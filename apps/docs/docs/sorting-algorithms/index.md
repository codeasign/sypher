# Sorting Algorithms

Estimated completion: 8-12 hours - Written for total beginners, no prior algorithms experience assumed

## What This Is

A beginner-friendly course covering the sorting algorithms every engineer eventually
runs into — from the simplest comparison sorts to the hybrid algorithm powering every
modern language's built-in sort. Each module is a single self-contained page written
in plain English, with an everyday analogy, an ASCII diagram walking through a real
pass step by step, and a plain-language explanation of how fast the algorithm is and
how much memory it needs. Every module also has an interactive in-browser code editor
supporting nine languages — Python, C, C++, Java, C#, JavaScript, TypeScript, Rust,
and Go — plus a full set of worked solutions.

New to algorithms? Start with Module 0 — it covers Big-O notation, stability, and
in-place sorting in plain language before the algorithm modules assume you know them.

## How to Use This Course

Module 0 is a beginner primer — no code, just the vocabulary (Big-O notation,
stability, in-place) that every later module assumes. Skip it if you already know
these terms. Every module after that (1-11) follows the same five-part format:
Summary, Pseudocode, ASCII Diagram, How It Works, and Practice test cases — followed
by an interactive code editor (9 languages) and a Solutions section with a worked
answer in each. Work through modules 1-3 first — the simple O(n^2) comparison sorts
that set up every idea that follows. Modules 4-6 cover the O(n log n) divide-and-conquer
and heap-based sorts. Modules 7-9 shift to non-comparison sorts that beat O(n log n)
under specific constraints. Module 10 covers Shell Sort as an Insertion Sort
generalization, and module 11 covers Tim Sort — the hybrid every mainstream language
actually uses. Module 12 is a decision guide for picking the right algorithm given a
real constraint.
