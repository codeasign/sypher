## Summary

Bubble Sort works by comparing two numbers that are sitting right next to each other and swapping them if they're in the wrong order. It walks down the whole list doing this over and over, and each pass pushes the biggest number left in the unsorted part all the way to the end — like a bubble floating up to the top. It's one of the easiest sorting methods to picture, but it's slow on big lists because it may need a lot of passes and comparisons to finish.

## Pseudocode

```text
1. Look at the first two numbers in the list.
2. If the left one is bigger than the right one, swap them.
3. Move one step to the right and repeat step 2 with the next pair.
4. Keep going until you reach the end of the list — that's one full pass.
5. If you made at least one swap during that pass, do another pass from the start.
6. If you made zero swaps during a pass, the list is already sorted — stop.
```

## Worked Example

We'll trace `[4, 2, 7, 2, 9, 1]` through every pass. Notice the two `2`s — this list will show what happens to equal values along the way.

Pass 1 moves through the list left to right: `4` and `2` swap, `4` and `7` stay put, `7` and `2` swap, `7` and `9` stay put, and `9` and `1` swap. Bubble Sort never builds a new list; it just keeps rearranging this same one, which is what people mean when they call a sort **in-place**.

The rest of the pass swaps `4` and `1`, adding `4` to the settled group.

One last comparison swaps `2` and `1`, and the list is fully sorted: `[1, 2, 2, 4, 7, 9]`. Notice the two `2`s are still in their original relative order from start to finish — that's Bubble Sort's stability paying off. If any pass had found zero swaps, it would have stopped early instead of running all the way to pass 5.

## Complexity

- **Best case:** if the list is already sorted, the very first pass finds zero swaps and stops right away, so it barely does any work (`O(n)`).
- **Average case:** on a typically shuffled list, it ends up comparing and swapping most pairs across most passes (`O(n²)`).
- **Worst case:** if the list is in reverse order, every pass finds a swap and it has to run all `n - 1` passes, comparing almost everything against everything else (`O(n²)`).
- **Space:** it only ever swaps numbers inside the same list — no second list is built — so it needs no extra memory beyond a couple of temporary variables (`O(1)`).

## Applications

Bubble Sort is mostly a **teaching algorithm** — it's the first sort many people learn because it's easy to picture, not because it's fast. In real software, you won't find it doing the heavy lifting:

- It shows up in intro programming courses and textbooks to teach the idea of comparing and swapping before moving on to faster algorithms.
- It's occasionally used to sort tiny, already-nearly-sorted lists (a handful of items) where the simplicity of the code matters more than raw speed.
- It can be handy as a rough "is this list sorted yet" check, since a single pass with no swaps tells you the answer.
- Real-world programming languages don't use it for their built-in sort functions — Python and Java use **Tim Sort**, and many C++ standard library implementations use variations of **Quick Sort** or **Intro Sort**, because those scale far better on large data.

    n = len(array)
    for i in range(n - 1):
        swapped = False
        for j in range(n - 1 - i):
            if array[j] > array[j + 1]:
                array[j], array[j + 1] = array[j + 1], array[j]
                swapped = True
        if not swapped:
            break
    return array`}</CodeBlock>
