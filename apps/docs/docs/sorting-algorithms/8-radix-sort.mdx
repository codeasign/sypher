## Summary

Radix Sort sorts numbers by looking at one digit at a time instead of comparing whole numbers to each other. It starts with the ones digit and sorts the whole list by just that digit, then moves to the tens digit and sorts again, then the hundreds digit, and so on, until it's worked through every digit of the biggest number. Each of those digit-by-digit sorting passes uses the counting trick from Counting Sort. It stays fast and predictable, but the more digits the numbers have, the more passes it needs to make.

## Pseudocode

```text
1. If the list is empty, stop — there's nothing to sort.
2. Find the biggest number in the list, so you know how many digits you'll need to work through.
3. Look at the ones digit of every number, and sort the whole list based on just that digit (using the counting trick from Counting Sort).
4. Look at the tens digit of every number, and sort the whole list based on just that digit.
5. Keep moving one digit to the left each time — hundreds, thousands, and so on — sorting the whole list by that digit each pass.
6. Stop once you've processed the last digit of the biggest number — the list is now fully sorted.
```

## Worked Example

We'll trace `[170, 45, 75, 90, 802, 24, 45]` through every digit pass. The biggest number, `802`, has three digits, so Radix Sort needs exactly three passes — ones, tens, then hundreds. Notice the two `45`s — this list will show what happens to equal values along the way.

Radix Sort never compares `170` to `45` directly — it only looks at the last digit of each number and drops it into the matching bucket (`0`-`9`). Numbers with the same ones digit, like `170` and `90` (both end in `0`), land in the same bucket and keep their original left-to-right order inside it. That's what makes this a **stable** sort: equal digits never get reordered relative to each other.

Most numbers have a hundreds digit of `0`, since only `170` and `802` are three digits long, so bucket `0` fills up with five numbers that still keep their pass-2 order among themselves. After this last pass the list is fully sorted: `[24, 45, 45, 75, 90, 170, 802]` — and the two `45`s are still in their original relative order from the very first list. Radix Sort also never sorts inside the same array slot by slot the way Bubble Sort does — each pass builds a fresh bucketed list — so it is **not in-place**; it needs extra room for the buckets and the rebuilt list every pass.

## Complexity

- **Best case:** it always runs every digit pass in full no matter what — it never checks whether the list is already sorted, so there's no shortcut for a lucky starting order (`O(d × n)`).
- **Average case:** the same amount of work happens on a typically shuffled list, because every pass buckets every number exactly once regardless of how mixed up they are (`O(d × n)`).
- **Worst case:** still `O(d × n)` — the only thing that can slow it down is `d`, the number of digits in the biggest number, not how out of order the list starts.
- **Space:** each pass needs a fresh output list plus 10 buckets to hold numbers by digit before rebuilding the list, so it needs extra memory proportional to the list size (`O(n + k)`, where `k` is the number of buckets, 10 for base-10 digits).

Radix Sort is **not a comparison-based sort** — it never asks "is this number bigger than that one?" It only ever asks "what's this digit?", which is why its speed depends on the number of digits `d` instead of the usual `n log n` comparison sorts are stuck with.

## Applications

- Sorting large batches of fixed-width numbers fast, such as employee IDs, invoice numbers, or product codes, where the digit count stays small no matter how many items you're sorting.
- Historically the technique behind mechanical **punch-card sorting machines**, which physically sorted stacks of cards one column (digit) at a time — the direct ancestor of the "one digit at a time" idea used here.
- Sorting IP addresses or MAC addresses, which are fixed-width numbers under the hood, by working through them octet by octet or byte by byte.
- Used as a building block inside suffix array construction algorithms (like the DC3/skew algorithm) in string processing and bioinformatics, where fixed-length keys need to be sorted over and over.
- Sorting large sets of bounded-length integer keys in database and external-sorting systems, where avoiding direct comparisons between huge keys saves real time.

    if not array:
        return array
    max_val = max(array)
    current = array[:]
    exp = 1
    while max_val // exp > 0:
        current = _counting_sort_by_digit(current, exp)

    return current


def _counting_sort_by_digit(array: list[int], exp: int) -> list[int]:
    n = len(array)
    output = [0] * n
    count = [0] * 10
    for value in array:
        digit = (value // exp) % 10
        count[digit] += 1
    for d in range(1, 10):
        count[d] += count[d - 1]
    for i in range(n - 1, -1, -1):
        digit = (array[i] // exp) % 10
        count[digit] -= 1
        output[count[digit]] = array[i]
    return output`}</CodeBlock>

    if (n == 0) {
        return NULL;
    }
    int max_val = array[0];
    for (int i = 1; i < n; i++) {
        if (array[i] > max_val) {
            max_val = array[i];
        }
    }
    int *current = malloc(n * sizeof(int));
    memcpy(current, array, n * sizeof(int));
    for (long exp = 1; max_val / exp > 0; exp *= 10) {
        int *output = malloc(n * sizeof(int));
        int count[10] = {0};
        for (int i = 0; i < n; i++) {
            int digit = (current[i] / exp) % 10;
            count[digit]++;
        }
        for (int d = 1; d < 10; d++) {
            count[d] += count[d - 1];
        }
        for (int i = n - 1; i >= 0; i--) {
            int digit = (current[i] / exp) % 10;
            output[--count[digit]] = current[i];
        }
        free(current);
        current = output;
    }
    return current;
}`}</CodeBlock>

<CodeBlock language="rust">{`pub fn radix_sort(array: &mut Vec<i32>) -> &mut Vec<i32> {
    if array.is_empty() {
        return array;
    }
    let max_val = *array.iter().max().unwrap();
    let mut current = array.clone();
    let mut exp: i64 = 1;
    while (max_val as i64) / exp > 0 {
        current = counting_sort_by_digit(&current, exp);
        exp *= 10;
    }
    *array = current;
    array
}

fn counting_sort_by_digit(array: &Vec<i32>, exp: i64) -> Vec<i32> {
    let n = array.len();
    let mut output = vec![0; n];
    let mut count = [0usize; 10];
    for &v in array.iter() {
        let digit = ((v as i64 / exp) % 10) as usize;
        count[digit] += 1;
    }
    for d in 1..10 {
        count[d] += count[d - 1];
    }
    for i in (0..n).rev() {
        let digit = ((array[i] as i64 / exp) % 10) as usize;
        count[digit] -= 1;
        output[count[digit]] = array[i];
    }
    output
}`}</CodeBlock>
