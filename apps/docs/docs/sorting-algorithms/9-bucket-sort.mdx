## Summary

Bucket Sort sorts a list by first tossing each number into one of a few rough groups, kind of like sorting mail by zip code region before a mail carrier deals with any of it individually. Once the numbers are split into these small groups ("buckets"), it tidies up each group on its own using a simple method, then lines the groups up in order to read out the final sorted list. This only works well when the numbers are spread out fairly evenly across the range — if too many numbers land in the same group, that group turns into one big pile again, and sorting it takes a while.

## Pseudocode

```text
1. If the list is empty, you're done — return an empty list.
2. Find the biggest number in the list.
3. Decide how many buckets to use — use one bucket for every number in the list.
4. For each number, work out which bucket it belongs in: multiply the number by
   the bucket count, then divide by (biggest number + 1). This sends small
   numbers to early buckets and large numbers to later buckets.
5. Drop each number into its bucket.
6. Go through the buckets one at a time and tidy up the few numbers inside each
   one (using a simple method called insertion sort — build up a sorted mini-list
   one number at a time).
7. Read the buckets out in order, front to back — that's your sorted list.
```

## How It Works

- Look at the diagram: the numbers `8, 3, 1, 9, 6` each get sent to one of 5 groups, based roughly on how big they are compared to the biggest number in the list (`9`). Small numbers land in the early groups, big numbers land in the later groups.
- Group 4 happens to catch two numbers, `9` and `8`, because they're both close in size. Every other group only gets one number (or none).
- Once every number has been dropped into a group, each group is tidied up on its own — since groups are small, this is quick even with a simple method.
- Finally, the groups are read out in order, left to right. Because the groups themselves are already in size order, and each group is sorted inside itself, the whole list comes out sorted: `[1, 3, 6, 8, 9]`.
- This method needs some extra room to hold the temporary groups — it doesn't sort the original list directly in place the way some other methods do.
- If two numbers are exactly equal, this method keeps them in the same left-to-right order they started in, because they're added to their group in that order and the per-group tidy-up preserves it too.
- How fast is it? If the numbers are spread out evenly across the range, this is fast — not much more work than just reading through the list once (`O(n)`). But if a lot of numbers pile into the same group, that group becomes one big pile again, and tidying it up takes as long as if you'd never split into groups at all (`O(n²)`) in the worst case.
