## Summary

Selection Sort works by repeatedly finding the smallest number left in the unsorted part of the list and moving it to the front. Think of it like picking the smallest card out of a messy pile and setting it down, then finding the next smallest and setting it right after, until the whole pile is sorted. It's easy to follow but still slow on big lists, since it has to look through the entire remaining pile every single time just to find the smallest one.

## Pseudocode

```text
1. Mark the first position in the list as the start of the "unsorted" part.
2. Scan through the unsorted part and remember the position of the smallest number you find.
3. Swap that smallest number into the first position of the unsorted part.
4. Move the boundary of the unsorted part one step to the right.
5. Repeat steps 2-4 until only one number is left вҖ” the list is now sorted.
```

## Worked Example

Starting list: `[29, 10, 14, 37, 13, 10, 8]`. Scan every number once to find the smallest вҖ” that's `8`, sitting at the very end. Swap it with whatever's in the front slot (`29`), so `8` locks into position `0`. Note that the two `10`s were compared during this scan and the earlier one won the tie, but that's not a guarantee вҖ” selection sort can swap a number past an equal one elsewhere in the list, so it doesn't reliably keep equal values in their original order. That's what people mean when they call it an "unstable" sort. It also doesn't need any extra list to work in вҖ” it just rearranges this same list, which is why it's called an "in-place" sort.

## Complexity

- **Best case**: Even if the list is already sorted, selection sort still scans the entire unsorted part on every pass to make sure it isn't missing a smaller value вҖ” so it does the same amount of work regardless (`O(nВІ)`).
- **Average case**: Same story for a randomly shuffled list вҖ” every pass scans everything that's left, no matter what order the numbers started in (`O(nВІ)`).
- **Worst case**: A list in reverse order costs exactly the same as any other case, since the scanning happens no matter how scrambled the list is (`O(nВІ)`).
- **Space**: It rearranges the list in place, using just a couple of extra variables to track the smallest value's position вҖ” it doesn't need to build a second list (`O(1)`).

## Applications

- Mostly used to teach how sorting works, not used in real production sorting вҖ” its `O(nВІ)` behavior is too slow for large, real-world data compared to `O(n log n)` options like merge sort or quick sort.
- Its one genuine niche: situations where swapping (writing) elements is much more expensive than comparing them, such as sorting data on flash memory or EEPROM where every write wears the hardware down a little. Selection sort makes at most `n - 1` swaps total, no matter how scrambled the list is вҖ” far fewer writes than bubble or insertion sort.
- Shows up occasionally in embedded systems and firmware for sorting small, fixed-size lists where minimizing writes matters more than raw speed.
- Used as a stepping stone when teaching heap sort, which is essentially the same "pick the smallest/largest remaining value" idea, sped up by using a heap instead of a linear scan to find that value.

    n = len(nums)

    for i in range(n - 1):
        min_index = i

        for j in range(i + 1, n):
            if nums[j] < nums[min_index]:
                min_index = j

        if min_index != i:
