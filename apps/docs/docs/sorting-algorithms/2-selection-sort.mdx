## Summary

Selection Sort works by repeatedly finding the smallest number left in the unsorted part of the list and moving it to the front. Think of it like picking the smallest card out of a messy pile and setting it down, then finding the next smallest and setting it right after, until the whole pile is sorted. It's easy to follow but still slow on big lists, since it has to look through the entire remaining pile every single time just to find the smallest one.

## Pseudocode

```text
1. Mark the first position in the list as the start of the "unsorted" part.
2. Scan through the unsorted part and remember the position of the smallest number you find.
3. Swap that smallest number into the first position of the unsorted part.
4. Move the boundary of the unsorted part one step to the right.
5. Repeat steps 2-4 until only one number is left — the list is now sorted.
```

## How It Works

- Start with `[5, 3, 8, 1]`. The whole list is unsorted so far. Scan through it looking for the smallest value: `5`, then `3` (smaller), then `8` (not smaller), then `1` (smaller than `3`) — so `1` is the smallest.
- Swap `1` into the front position, giving `[1, 3, 8, 5]`. That front slot is done — it holds the right value for good.
- Repeat the same idea on what's left, `[3, 8, 5]`: scan for the smallest (`3`), which is already at the front, so nothing moves there. Then scan `[8, 5]`, find `5` is smaller, and swap it to the front of that section.
- Each pass only does one swap, no matter how many numbers are out of place — it just takes the whole pass to figure out which one to move.
- It doesn't need extra space to work — it rearranges the same list in place instead of building a new one.
- Selection Sort can move a number past an equal one while hunting for the smallest, so if two items are equal, it doesn't reliably leave them in the order they started in.
- Whether the list starts sorted or completely backwards, Selection Sort always scans the whole unsorted part every single pass looking for the minimum, so it does roughly the same amount of work either way (this is called O(n²) time, in both the best and worst case).
