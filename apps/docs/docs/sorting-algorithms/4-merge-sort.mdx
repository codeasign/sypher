## Summary

Merge Sort works by splitting the list in half again and again until each piece has just one number left вҖ” a list of one is already sorted. Then it merges those tiny pieces back together two at a time, always picking the smaller of the two current front numbers, until the whole list is back together in order. This "split it up, then merge it back together" idea is called divide and conquer. It stays reliably fast even on big or badly-ordered lists.

## Pseudocode

```text
1. If the list has one number or none, it's already sorted.
2. Otherwise, treat each single number as a sorted mini-list of size 1.
3. Merge neighboring mini-lists two at a time into sorted lists of size 2.
4. Merge those neighboring lists into sorted lists of size 4, then size 8, and so on, doubling the size each round.
5. Keep merging neighboring sorted chunks until only one chunk вҖ” the whole list вҖ” is left.
```

### Recursive Version

```text
1. If the list has one number or none, it's already sorted вҖ” stop.
2. Split the list into two halves.
3. Sort the left half using these same steps (split it again if it's still more than one number).
4. Sort the right half the same way.
5. Merge the two sorted halves back into one list: keep comparing the front numbers of each half and take the smaller one first.
6. Once one half runs out of numbers, add all the remaining numbers from the other half.
```

Compare `3` and `3` вҖ” they're equal, and the merge always takes the number from the left piece first when there's a tie, so the `3` from `[3, 5, 8]` goes next. ## Complexity

- **Best case:** even if the list is already sorted, Merge Sort still splits it all the way down and merges every piece back up the same way every time, so it does the same amount of work regardless (`O(n log n)`).
- **Average case:** on a typically shuffled list, it splits and merges exactly the same way (`O(n log n)`).
- **Worst case:** even if the list is in the worst possible order, the split-then-merge pattern never changes, so it still finishes in (`O(n log n)`).
- **Space:** every merge step builds a brand-new list instead of rearranging numbers in place, so at any one time it needs roughly as much extra memory as the list itself (`O(n)`).

## Applications

- **External sorting** вҖ” sorting data too big to fit in memory at once, like huge log files or database tables, by sorting chunks on disk and merging the sorted chunks together.
- **Tim Sort**, the hybrid sort behind Python's `sorted()`/`list.sort()` and Java's `Arrays.sort()` for objects, is built on merge sort's merging idea combined with insertion sort for small runs.
- Anywhere **sort stability** matters вҖ” for example sorting a list of orders by date while keeping same-date orders in their original order вҖ” since Merge Sort never lets equal values swap past each other.
- **Merging results from multiple sources**, like combining several already-sorted result sets from different database shards or servers into one sorted stream.
- **Sorting linked lists** вҖ” Merge Sort only needs to split and re-link pieces, not jump to random positions, which makes it a natural fit for linked lists where index-based swapping is expensive.

    if len(array) <= 1:
        return array[:]

    mid = len(array) // 2
    left = merge_sort(array[:mid])
    right = merge_sort(array[mid:])

    return _merge(left, right)


def _merge(left: list[int], right: list[int]) -> list[int]:
    result = []
    i = j = 0

    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1

    result.extend(left[i:])
    result.extend(right[j:])
    return result`}</CodeBlock>

int *merge(const int *left, int left_size, const int *right, int right_size) {
    int *result = (int *)malloc((left_size + right_size) * sizeof(int));
    int i = 0, j = 0, k = 0;

    while (i < left_size && j < right_size) {
        if (left[i] <= right[j]) {
            result[k++] = left[i++];
        } else {
            result[k++] = right[j++];
        }
    }
    while (i < left_size) {
        result[k++] = left[i++];
    }
    while (j < right_size) {
        result[k++] = right[j++];
    }
    return result;
}

int *merge_sort(const int *array, int size) {
    if (size <= 1) {
        int *result = (int *)malloc(size * sizeof(int));
        if (size == 1) {
            result[0] = array[0];
        }
        return result;
    }

    int mid = size / 2;
    int *left = merge_sort(array, mid);
    int *right = merge_sort(array + mid, size - mid);

    int *result = merge(left, mid, right, size - mid);
    free(left);
    free(right);
    return result;
}`}</CodeBlock>

std::vector<int> merge(const std::vector<int>& left, const std::vector<int>& right) {
    std::vector<int> result;
    result.reserve(left.size() + right.size());
    size_t i = 0, j = 0;

    while (i < left.size() && j < right.size()) {
        if (left[i] <= right[j]) {
            result.push_back(left[i++]);
        } else {
            result.push_back(right[j++]);
        }
    }
    while (i < left.size()) {
        result.push_back(left[i++]);
    }
    while (j < right.size()) {
        result.push_back(right[j++]);
    }
    return result;
}

std::vector<int> mergeSort(const std::vector<int>& array) {
    if (array.size() <= 1) {
        return array;
    }

    size_t mid = array.size() / 2;
    std::vector<int> left(array.begin(), array.begin() + mid);
    std::vector<int> right(array.begin() + mid, array.end());

    return merge(mergeSort(left), mergeSort(right));
}`}</CodeBlock>

            result.push(right[j]);
            j += 1;
