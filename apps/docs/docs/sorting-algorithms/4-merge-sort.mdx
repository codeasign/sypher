## Summary

Merge Sort works by splitting the list in half again and again until each piece has just one number left — a list of one is already sorted. Then it merges those tiny pieces back together two at a time, always picking the smaller of the two current front numbers, until the whole list is back together in order. This "split it up, then merge it back together" idea is called divide and conquer. It stays reliably fast even on big or badly-ordered lists.

## Pseudocode

```text
1. If the list has one number or none, it's already sorted — stop.
2. Split the list into two halves.
3. Sort the left half using these same steps (split it again if it's still more than one number).
4. Sort the right half the same way.
5. Merge the two sorted halves back into one list: keep comparing the front numbers of each half and take the smaller one first.
6. Once one half runs out of numbers, add all the remaining numbers from the other half.
```

## How It Works

- Follow the diagram from the top. `[5, 3, 8, 1]` gets split into `[5, 3]` and `[8, 1]`, and each of those gets split again into single numbers: `[5]`, `[3]`, `[8]`, `[1]`. A list with just one number is already sorted, so splitting stops there.
- Now merge back up. `[5]` and `[3]` merge by comparing their only numbers and putting the smaller one first, giving `[3, 5]`. In the same way, `[8]` and `[1]` merge into `[1, 8]`.
- Merge those two results together: compare the front numbers, `3` and `1` — `1` is smaller, so it goes first. Then compare `3` and `8` — `3` is smaller, so it goes next. Then `5` and `8` — `5` goes next, and `8` is all that's left, so it goes last. The final merged list is `[1, 3, 5, 8]`.
- This split-then-merge idea is called divide and conquer: break a big problem into small, easy pieces, solve each tiny piece, then combine the solutions back together.
- Merge Sort needs extra space to work — the merge step builds a brand-new list instead of rearranging numbers inside the original one.
- When two numbers are equal, the merge step always takes the one from the left piece first, so equal numbers keep the order they started in.
- Merge Sort always splits the list exactly in half and always merges the same way, no matter how the numbers started out — so it takes roughly the same amount of time whether the list is already sorted or completely backwards (this is called O(n log n) time, both best and worst case).
