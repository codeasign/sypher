## Summary

Quick Sort picks one number from the list — called the pivot — and splits everything else into two groups: numbers smaller than the pivot, and numbers bigger than it. It puts the smaller group first, then the pivot, then the bigger group, so the pivot lands exactly where it belongs. Then it does the exact same trick again on each of those two groups, and keeps going until every group is down to one number. It's usually quick, but if it keeps picking unlucky pivots, it can slow down a lot.

## Pseudocode

```text
1. If the current part of the list has one item or none, it's already sorted — stop.
2. Pick the last item in that part of the list as the pivot.
3. Go through every other item in that part and compare it to the pivot.
4. Slide each item that's smaller than or equal to the pivot into a "smaller" group on the left.
5. Once every item has been checked, place the pivot right after the "smaller" group — now everything left of it is smaller, and everything right of it is bigger.
6. Repeat steps 1-5 separately on the "smaller" group and the "bigger" group.
7. Keep repeating until every group has just one item left — the whole list is now sorted.
```

## How It Works

- In the diagram, we start with `5, 3, 8, 1, 9` and pick `5` — the last number — as the pivot.
- We go through the rest of the numbers one by one and sort them into two piles: `3` and `1` are smaller than `5`, so they go in the "smaller" pile; `8` and `9` are bigger, so they go in the "bigger" pile.
- Once every number has been placed in a pile, `5` slides into the gap between the two piles. That's now its final position — it will never move again.
- Quick Sort then repeats this same process separately on the "smaller" pile (`3, 1`) and the "bigger" pile (`8, 9`), picking a new pivot each time, until every pile has just one number left.
- Quick Sort is "in-place," which just means it rearranges the numbers inside the same list you gave it, instead of building a whole new list somewhere else.
- Quick Sort is not "stable." That means if two equal numbers are in the list, this method doesn't promise they'll stay in the same order relative to each other after sorting.
- How fast is it? If the pivot roughly splits each group in half, Quick Sort flies through the list (this is the `O(n log n)` case). But if you keep getting unlucky pivots — for example, always picking the last number as pivot on a list that's already sorted — the groups barely shrink each time, and Quick Sort ends up comparing almost every number to every other number (the much slower `O(n²)` case).
