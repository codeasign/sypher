## Summary

Quick Sort picks one number from the list — called the pivot — and splits everything else into two groups: numbers smaller than the pivot, and numbers bigger than it. It puts the smaller group first, then the pivot, then the bigger group, so the pivot lands exactly where it belongs. Then it does the exact same trick again on each of those two groups, and keeps going until every group is down to one number. It's usually quick, but if it keeps picking unlucky pivots, it can slow down a lot.

## Pseudocode

```text
1. If the current part of the list has one item or none, it's already sorted — stop.
2. Pick the last item in that part of the list as the pivot.
3. Go through every other item in that part and compare it to the pivot.
4. Slide each item that's smaller than or equal to the pivot into a "smaller" group on the left.
5. Once every item has been checked, place the pivot right after the "smaller" group — now everything left of it is smaller, and everything right of it is bigger.
6. Repeat steps 1-5 separately on the "smaller" group and the "bigger" group.
7. Keep repeating until every group has just one item left — the whole list is now sorted.
```

### Recursive Version

```text
function quickSort(list, low, high):
    if low is greater than or equal to high:
        return  # zero or one item, already sorted

    pivotIndex = partition(list, low, high)
    quickSort(list, low, pivotIndex - 1)       # sort the smaller group
    quickSort(list, pivotIndex + 1, high)      # sort the bigger group

function partition(list, low, high):
    pivot = list[high]                          # last item is the pivot
    boundary = low - 1                           # edge of the "smaller" group so far

    for j from low to high - 1:
        if list[j] is smaller than or equal to pivot:
            boundary = boundary + 1
            swap list[boundary] and list[j]

    swap list[boundary + 1] and list[high]       # drop the pivot into its final spot
    return boundary + 1
```

The `partition` step is the heart of Quick Sort. It walks through every item in the current range once, sliding anything smaller-or-equal to the pivot toward the left edge, then finishes by swapping the pivot into the one gap left over — right between the smaller group and the bigger group. `quickSort` just calls `partition` to find that gap, then calls itself on the two groups that fall on either side of it.

Quick Sort picks one number to act as the pivot — here it's the simplest choice, the last number in the list, which is `3`. Every other number will be compared against `3` to decide which side of it they belong on.

After comparing every other number to `3`, only `2` and `1` are smaller, so they slide to the left; `4`, `7`, `6`, `8`, and `9` are all bigger, so they end up on the right. The pivot `3` drops into the gap between the two groups — that's its final position, it will never move again. Quick Sort is "in-place," which means it rearranges these numbers inside the same list instead of copying them into a brand new one.

Quick Sort now repeats the same trick on the smaller group by itself. Its pivot is `1`, and the only other number, `2`, is bigger, so it moves to the right of `1`. That leaves this group fully sorted as `1, 2` — with only one number left on each side of the pivot, there's nothing left to split.

This time the pivot, `9`, happens to already be the biggest number in the group, so every other number counts as "smaller" and none of them move. The pivot just confirms its own spot at the end. This is the pattern that makes Quick Sort slow when it repeats: the group barely shrinks, dropping by only one number (the pivot) each time instead of splitting roughly in half.

Same story here: `8` is the biggest number in this group, so `4`, `7`, and `6` all count as smaller and stay put. The group shrinks by just one number again.

This group finally splits into two real pieces: `4` is smaller than `6`, so it stays on the left, and `7` is bigger, so it swaps to the right. Now every piece on both sides of the original list has just one number left in it.

Once every group is down to one number, there's nothing left to compare or swap — stitching the pieces back together in the order they landed gives the fully sorted list. Quick Sort is not "stable," which means if two equal numbers appear in the list, this method doesn't promise they'll keep their original order relative to each other after sorting.

## Complexity

- **Best case**: If the pivot happens to split each group roughly in half every time, Quick Sort works through the whole list very efficiently (`O(n log n)`).
- **Average case**: For a typical shuffled list, pivots usually split groups reasonably well, so it still runs fast on average (`O(n log n)`).
- **Worst case**: If the pivot keeps landing on the smallest or biggest number in its group — like always picking the last item as pivot on a list that's already sorted, or one built by an adversary to defeat that choice — each group only shrinks by one number instead of splitting in half, so Quick Sort ends up comparing almost every number to every other number (`O(n²)`).
- **Space**: Quick Sort sorts the list in-place, but it still needs a small amount of extra memory to keep track of which group it's currently working on as it recurses (`O(log n)` on average, `O(n)` in the worst case).

## Applications

- General-purpose in-memory sorting — it's fast in practice and doesn't need extra memory for a second copy of the data.
- Used inside many standard library sort functions for primitive types, such as C's `qsort` and tuned variants inside Java's `Arrays.sort` for primitive arrays and the C++ Standard Library's `std::sort` (usually as part of a hybrid with other algorithms to avoid the worst case).
- Used in database engines and query planners for in-memory sorting of result sets before returning or merging them.
- Median-of-three and randomized pivot-selection variants are used in performance-sensitive systems code specifically to avoid the `O(n²)` worst case on already-sorted or adversarial input.

    def partition(low: int, high: int) -> int:
        pivot = array[high]
        i = low - 1

        for j in range(low, high):
            if array[j] <= pivot:
                i += 1
                array[i], array[j] = array[j], array[i]

        array[i + 1], array[high] = array[high], array[i + 1]
        return i + 1

    def sort(low: int, high: int) -> None:
        if low < high:
            pivot_index = partition(low, high)
            sort(low, pivot_index - 1)
            sort(pivot_index + 1, high)

    sort(0, len(array) - 1)
    return array`}</CodeBlock>

    let pivot = array[high];
    let mut i = low;

    for j in low..high {
        if array[j] <= pivot {
            array.swap(i, j);
            i += 1;
        }
    }

    array.swap(i, high);
