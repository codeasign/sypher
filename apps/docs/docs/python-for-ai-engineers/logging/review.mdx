# Logging — Review

⏱ 3 minutes

## Key Takeaways

- Use `logging.getLogger(__name__)` for module-level loggers and `logging.basicConfig()` for quick setup.
- Log levels in increasing severity: `DEBUG`, `INFO`, `WARNING`, `ERROR`, `CRITICAL` — set a level and only messages at or above that level are output.
- Configure handlers (console, file) with formatters for flexible output routing.
- Use `logger.exception()` in `except` blocks to capture the full traceback.
- Never log sensitive data like passwords, API keys, or personal information.

## Recap in 60 Seconds

If you only remember three things:

1. Replace `print()` with `logging.info()` in production code — it gives you timestamps, levels, and filterable output.
2. Set both the logger level and the handler level — forgetting the logger level is the most common configuration bug.
3. Use `logger.exception()` in exception handlers to preserve the traceback.
