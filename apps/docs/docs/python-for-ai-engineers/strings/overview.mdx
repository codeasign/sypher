## Why This Matters

Strings are the most manipulated data type in AI engineering. Every prompt you send to an LLM is a string. Every API response you parse is a string. Every dataset you preprocess — CSV, JSON, text files — is full of strings you need to clean, split, join, and reformat. Prompt engineering is essentially applied string manipulation: you construct a template, insert variables, and verify the result is well-formed.

Beyond prompts, strings show up in logging, configuration, error messages, and user interfaces. The difference between a cleanly formatted log and a mess of concatenated pieces is the difference between being able to debug a production issue in five minutes versus an hour. Mastering Python's string tools is one of the highest-leverage skills for day-to-day productivity.

## The Problem

You have a log line: `"ERROR 2026-06-30 14:30:22 model=gpt-4 latency=2342ms"`. You need to extract the model name and latency value. You could count characters and use hard-coded indices, but that breaks if the format changes. You could use regular expressions, but that is overkill for a structured line. What you need is a set of predictable, readable string operations — `split()`, `strip()`, `find()` — that handle the extraction cleanly without fragile indexing.

## The Concept

A string is an **immutable sequence of Unicode characters**. Immutable means once created, you cannot change a string — any operation that appears to modify a string actually creates a new one. Sequence means you can access individual characters by position, loop over them, and take slices.

Think of a string as a row of lockers. Each locker holds one character and has an index number. You can look into any locker (`s[0]`), peer into a range of lockers (`s[2:5]`), or walk down the row (`for ch in s`). But you cannot reach into a locker and change what is inside — you can only build a new row.
