# Strings — Common Mistakes

⏱ 6 minutes

## When NOT to Use Strings

Not everything should be a string. Use appropriate types for your data:

- **Numbers:** Use `int` or `float` for arithmetic — do not store "42" as a string and call `int()` every time you need to calculate.
- **Dates:** Use `datetime.datetime` objects, not strings — they support arithmetic, timezone conversion, and formatting.
- **Structured data:** Use `dict` or JSON, not hand-parsed strings. Strings are for text, not for encoding structured information that has a native representation.

## The Pattern to Remember

- **Pattern:** Create with quotes → manipulate with methods → format with f-strings.
- **When to use:** Any text — prompts, logs, user messages, configuration values, API responses.
- **When NOT to use:** Numbers you compute on, dates you manipulate, structured data that belongs in a dict or a dataclass.
- **Mental model:** A string is a read-only sequence of characters. Every method returns a new string — the original is never modified.

## Common Mistakes

### Mistake 1: Off-by-One Errors in Slicing

**Mistake:** Expecting `s[0:5]` to include index 5.

```python
s = "HelloWorld"
print(s[0:5])  # "Hello" — not "HelloW"
```

**Why it happens:** The slice `start:stop` includes `start` but excludes `stop`. This is consistent with Python's zero-based indexing and with `range(5)` producing 0, 1, 2, 3, 4 (five values).

**Symptoms:** You get one fewer character than expected, or an empty slice when you expected a single character.

**Fix:** Remember the "fencepost" model — slice indices are positions between characters. `s[0:5]` takes the five characters between position 0 and position 5. Use `len(s)` to find the last valid index for a slice.

### Mistake 2: Modifying a String In-Place

**Mistake:** Writing `s[0] = "J"` to change the first character.

**Why it happens:** Many languages (C, Java) allow this. Python strings are immutable.

**Symptoms:** `TypeError: 'str' object does not support item assignment`

**Fix:** Build a new string:

```python
s = "hello"
s = "J" + s[1:]  # "Jello"
```

### Mistake 3: Confusing `split()` and `split(' ')`

```python
line = "a  b   c"  # Multiple spaces between words
print(line.split())      # ['a', 'b', 'c'] — splits on any whitespace
print(line.split(' '))   # ['a', '', 'b', '', '', 'c'] — splits on single space
```

**Why it happens:** `split()` with no argument splits on any run of whitespace and discards empty strings. `split(' ')` splits on each individual space character, preserving empty strings between consecutive spaces.

**Symptoms:** Your CSV parsing produces empty fields where there are double spaces, or your whitespace handling misses spacing variations.

**Fix:** Use `split()` (no argument) for whitespace-delimited text. Use `split(',')` for CSV — but be aware of edge cases, and prefer the `csv` module for real CSV data.

### Mistake 4: Using `+` in a Loop to Build Large Strings

```python
# Inefficient
result = ""
for token in tokens:
    result += token + " "
```

**Why it happens:** The `+=` operator looks concise and readable. You do not notice the performance problem until `tokens` reaches thousands of elements.

**Symptoms:** The loop runs fine on small data but becomes surprisingly slow as input grows. CPU profiling shows time spent in string operations.

**Fix:** Collect parts in a list and `join()` once:

```python
parts = []
for token in tokens:
    parts.append(token)
result = " ".join(parts)
```

### Mistake 5: Not Stripping User Input

**Mistake:** Using `input()` directly without calling `.strip()`.

```python
name = input("Enter name: ")  # User types "  Alice  " with spaces
print(f"Hello, {name}!")      # "Hello,   Alice  !"
```

**Why it happens:** Users type spaces, hit enter with accidental whitespace, or copy-paste extra characters. The newline is stripped by `input()`, but leading/trailing spaces remain.

**Symptoms:** Conditional checks fail (`if name == "Alice"` is `False` even though the user typed "Alice"), logs have unexpected whitespace, file lookups fail.

**Fix:** Always call `.strip()` on user input:

```python
name = input("Enter name: ").strip()
```
