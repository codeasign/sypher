# CSV

⏱ 10 minutes · Difficulty: Beginner

## What You Will Learn

By the end of this lesson you will be able to:

- Read and write CSV files with Python's `csv` module
- Use `DictReader` and `DictWriter` for named column access
- Handle different delimiters, quoting styles, and encoding issues
- Filter, sort, and aggregate CSV data programmatically
- Process large CSV files in chunks

## Prerequisites

You should already understand:

- [File Handling](../file-handling/overview) — CSV is a file format; reading and writing follow the same `with open()` pattern
- [Lists](../lists/overview) — CSV rows map to lists of strings
- [Dictionaries](../dictionaries/overview) — `DictReader` maps column headers to dict keys

## Why This Matters

CSV is the default format for tabular data in AI engineering. Datasets from Kaggle, Hugging Face, and research repositories are distributed as CSV. Experiment results are logged to CSV. Evaluation metrics across model runs are stored in CSV. Ground truth labels, prediction outputs, and feature tables are all CSV.

CSV has no schema, no type system, and no standard encoding — every column is a string until you convert it. This simplicity makes it universal but also error-prone. Understanding how to read, write, and clean CSV data correctly is essential before you can use it with pandas or NumPy.

## The Problem

You have 10,000 evaluation results in a CSV file. You need to:

- Skip malformed rows (lines with wrong column count)
- Convert string columns to the correct numeric types
- Filter rows where accuracy is above 0.9
- Write the filtered results to a new CSV

Python's `csv` module handles the parsing; you handle the validation and conversion.

## The Concept

CSV is a row-based format where each line represents one row and values are separated by a delimiter (usually a comma). The first row often contains column headers. Values containing the delimiter, newlines, or quotes are wrapped in double quotes.

The `csv` module has two reader styles:

- `csv.reader` — returns each row as a list of strings
- `csv.DictReader` — returns each row as a dict keyed by the header row

Both iterate lazily over the file — they do not load the entire file into memory.
