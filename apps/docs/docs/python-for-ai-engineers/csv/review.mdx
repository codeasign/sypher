# CSV — Review

⏱ 3 minutes

## Key Takeaways

- Always open CSV files with `newline=""` and `encoding="utf-8"` (or `"utf-8-sig"` for Excel files).
- Every CSV value is a string — convert to the correct type explicitly when reading.
- Use `csv.DictReader` / `csv.DictWriter` for column-name-based access; use `csv.reader` / `csv.writer` for positional access.
- Validate and clean CSV data before using it — handle missing values, wrong column counts, and whitespace.
- Detect the delimiter and quoting style explicitly; do not assume comma.

## Recap in 60 Seconds

If you only remember three things:

1. `csv.reader` returns lists; `csv.DictReader` returns dicts. Always open files with `newline=""`.
2. Every value is a string — `float(row["accuracy"])` is required before doing math.
3. Validate rows before using them — skip missing values, wrong column counts, and unparseable numbers.
