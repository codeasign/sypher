# Your First Python Program — Hands On

⏱ 10 minutes

## Breaking It Down

### Step 1: Write a Simple Script

Create `greet.py` in your editor:

```python
# greet.py — Reads a name and prints a greeting
name = input("Enter your name: ")
message = "Hello, " + name + "! Welcome to Python for AI."
print(message)
```

Run it:

```bash
python3 greet.py
```

Type your name when prompted and press Enter. You should see:

```
Enter your name: Jenny
Hello, Jenny! Welcome to Python for AI.
```

### Step 2: Understand What Each Line Does

| Line | What it does |
|------|-------------|
| `name = input("Enter your name: ")` | Prints the prompt, waits for keyboard input, stores the result as a **string** in `name` |
| `message = "Hello, " + name + "! ..."` | **Concatenates** three strings into one and stores the result in `message` |
| `print(message)` | Sends the value of `message` to the terminal |

The `+` operator on strings joins them end-to-end. If `name` is `"Jenny"`, then `"Hello, " + "Jenny"` produces `"Hello, Jenny"`.

### Step 3: Add a Numeric Calculation

Extend the script to tell the user how many characters their name has:

```python
# greet.py — Extended with a calculation
name = input("Enter your name: ")
message = "Hello, " + name + "! Welcome to Python for AI."
print(message)

# Calculate and display the length of the name
length = len(name)
print("Your name has", length, "characters.")
```

Run it:

```bash
python3 greet.py
```

If you enter `Jenny`, output:

```
Enter your name: Jenny
Hello, Jenny! Welcome to Python for AI.
Your name has 5 characters.
```

The `len()` function returns the number of characters in a string. `Jenny` has 5 characters. When you pass multiple arguments to `print()` separated by commas, Python inserts a space between them automatically.

### Step 4: Read Command-Line Arguments

Instead of interactive input, you can pass arguments on the command line. This is how production scripts are typically invoked:

```python
# greet_cli.py — Reads a name from the command line
import sys

# sys.argv is a list: [script_name, arg1, arg2, ...]
if len(sys.argv) < 2:
    print("Usage: python3 greet_cli.py <name>")
    sys.exit(1)

name = sys.argv[1]
message = "Hello, " + name + "! Welcome to Python for AI."
print(message)
```

Run it:

```bash
python3 greet_cli.py Jenny
# Hello, Jenny! Welcome to Python for AI.
```

If you run it without a name, it shows a usage message and exits with a non-zero status code, signalling an error to the shell.

### Step 5: Build Something Real — A Simple Data Log

Combine everything into a script that simulates logging an AI model prediction:

```python
# predict_log.py — Simulates logging a model prediction
import sys
from datetime import datetime

# Get inputs from command line
if len(sys.argv) < 3:
    print("Usage: python3 predict_log.py <model_version> <confidence>")
    sys.exit(1)

model_version = sys.argv[1]
confidence = float(sys.argv[2])

# Create a summary string
timestamp = str(datetime.now())
summary = (
    "[" + timestamp + "] "
    "Model: " + model_version + " | "
    "Confidence: " + str(confidence) + " | "
    "Threshold: 0.5"
)

print(summary)
print("Prediction logged.")
```

```bash
python3 predict_log.py v2.1.0 0.87
```

Expected output (timestamp will differ):

```
[2026-06-30 14:30:22.123456] Model: v2.1.0 | Confidence: 0.87 | Threshold: 0.5
Prediction logged.
```

This structure — command-line arguments, string formatting, and output — is the skeleton of countless Python scripts used in AI pipelines.

## Security Considerations

`input()` in Python 3 returns a string — it does not evaluate it as code. In Python 2, `input()` was dangerous because it evaluated the input as a Python expression, but Python 2 reached end of life in 2020 and you should never use it. In Python 3, the equivalent of Python 2's dangerous behaviour is `eval(input())`, which you should never use with untrusted input. Keep `input()` as-is and you are safe.
