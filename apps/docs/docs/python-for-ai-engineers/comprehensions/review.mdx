# Comprehensions — Review

⏱ 3 minutes

## Key Takeaways

- Use `[expr for item in iterable if cond]` for list comprehensions, `{k: v for ...}` for dicts, `{expr for ...}` for sets.
- The `if` at the end filters elements; a ternary `if/else` in the expression transforms each element but keeps all.
- Nested comprehensions read left-to-right — same order as nested for loops — but keep them simple for readability.
- Comprehensions are faster than manual loops because the iteration runs at C speed.
- Use generator expressions `(expr for ...)` instead of list comprehensions for very large sequences to save memory.

## Recap in 60 Seconds

If you only remember three things:

1. A comprehension is `[transform(item) for item in iterable if condition]` — read it as a single sentence.
2. Use comprehensions when you want a new collection; use loops when you want side effects.
3. Dict comprehensions `{k: v for k, v in ...}` and set comprehensions `{expr for ...}` use the same pattern as list comprehensions but with different bracket types.
