## Why This Matters

Comprehensions are one of Python's most distinctive and beloved features. They let you express common data transformation patterns — map every element, filter by condition, build lookup tables — in a single readable line. In AI engineering, you will use them constantly: cleaning token lists, filtering predictions by confidence, building feature dictionaries, and transforming batch results.

Code that uses comprehensions is not only shorter but typically faster, because the iteration happens at C speed inside the interpreter rather than at Python bytecode speed in a manual loop.

## The Problem

You have a list of model prediction scores and need to extract only those above 0.5, doubled:

```python
scores = [0.12, 0.87, 0.45, 0.92, 0.33, 0.78]
# You want: [1.74, 1.84, 1.56]
```

With a manual loop:

```python
result = []
for s in scores:
    if s > 0.5:
        result.append(s * 2)
```

This is six lines for a simple operation. The logic — filter, then transform — is buried in the loop mechanics. A comprehension expresses the same intent in one line with no boilerplate.

## The Concept

A comprehension is an expression that builds a new collection by iterating over an existing one, optionally filtering elements. Every comprehension has the same structure:

```python
[expression for item in iterable if condition]
```

The `if condition` part is optional. Without it, you transform every element. With it, you filter and transform.
