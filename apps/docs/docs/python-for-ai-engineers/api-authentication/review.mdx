# API Authentication — Review

⏱ 3 minutes

## Key Takeaways

- Never hardcode API keys — use environment variables loaded at runtime.
- Use `Authorization: Bearer <token>` headers for most AI APIs; inject them via session headers or httpx auth transports.
- Handle token expiry by caching the expiry time and refreshing before it expires, with a buffer.
- Logging authentication headers leaks credentials — always sanitize before logging.
- Use separate, scoped keys for different environments and rotate them regularly.

## Recap in 60 Seconds

If you only remember three things:

1. Load credentials from environment variables — never from source code or config files committed to git.
2. Send auth in the `Authorization` header via session defaults — never in query parameters.
3. Track token expiry and refresh proactively; retry on 401 in case of a race condition.
