# Sets

⏱ 8 minutes · Difficulty: Beginner

## What You Will Learn

By the end of this lesson you will be able to:

- Create sets and understand their unordered, unique-element semantics
- Add and remove elements with `add()`, `remove()`, `discard()`, `pop()`
- Perform mathematical set operations — union, intersection, difference, symmetric difference
- Use sets for fast membership testing and deduplication
- Choose between frozen sets and mutable sets

## Prerequisites

You should already understand:

- [Lists](../lists/overview) — sets are another collection type with different semantics
- [Dictionaries](../dictionaries/overview) — sets share the {} notation and hash-based internals with dicts

## Why This Matters

Sets solve two problems that appear constantly in AI engineering:

**Deduplication:** You have 10,000 prompt-response pairs collected from an experiment. Many are duplicates. A set finds the unique items instantly.

**Membership testing:** You have a list of 50,000 known stopwords and need to check if each token in a document is a stopword. `token in stopword_list` is O(n) per check — 50,000 comparisons per token. `token in stopword_set` is O(1) — one hash lookup, regardless of the set size.

Beyond these, set operations let you answer questions like "which labels exist in both evaluation sets?" (intersection) and "which models are new this month?" (difference) with clean, readable code.

## The Problem

Checking `x in list` on a list of 100,000 elements takes 50,000 comparisons on average. That is fine for a single check, but if you do it in a loop — "check if each of 50,000 tokens is a stopword" — you perform up to 5 billion comparisons. This is the kind of performance problem that makes a script take ten minutes instead of two seconds.

The fix is sets. A set uses a hash table internally — same mechanism as a dictionary, but with only keys and no values — giving O(1) membership checks.

## The Concept

A set is an **unordered collection of unique, hashable elements**. Unordered means elements have no position — you cannot index a set with `s[0]`. Unique means every element appears at most once. Hashable means elements must be immutable — strings, numbers, tuples — but not lists or dictionaries.
