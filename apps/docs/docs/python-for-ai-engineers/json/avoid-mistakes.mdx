# JSON — Common Mistakes

⏱ 5 minutes

## When NOT to Use JSON

- **Binary data:** JSON cannot represent raw bytes. Use base64 encoding for binary payloads, or use a binary format like Protocol Buffers or MessagePack.
- **Very large numeric arrays:** JSON parsing of huge arrays is slow and memory-intensive. Use NumPy's `.npy` format or Parquet instead of storing arrays as JSON lists.
- **Performance-critical serialization:** For high-throughput inter-process communication, use msgpack, protobuf, or flatbuffers — JSON parsing is relatively slow.
- **Data that needs comments:** JSON does not support comments. Use YAML or TOML for configuration files that need inline documentation.

## The Pattern to Remember

- **Pattern:** `json.dumps(data, indent=2)` to serialize, `json.loads(string)` to deserialize. Use `dump`/`load` (no `s`) with file objects. Use `.get()` instead of `[]` for safe navigation of external data.
- **When to use:** API payloads and responses, configuration files, experiment logs, data interchange between systems.
- **When NOT to use:** Binary data, performance-critical serialization, configuration needing comments, large numeric arrays.
- **Mental model:** JSON is a text format with six types — object, array, string, number, boolean, null. The `json` module converts between JSON text and Python objects.

## Common Mistakes

### Mistake 1: Using Single Quotes in JSON Strings

```python
import json

data = "{'model': 'gpt-4'}"  # Not valid JSON!

try:
    parsed = json.loads(data)  # Raises json.JSONDecodeError
except json.JSONDecodeError as e:
    print(e)  # Expecting property name enclosed in double quotes
```

**Why it happens:** JSON requires double quotes for strings and keys. Python accepts both single and double quotes, so you naturally write JSON-like strings with single quotes. But `json.loads()` strictly follows the JSON spec.

**Symptoms:** `json.JSONDecodeError: Expecting property name enclosed in double quotes` when parsing.

**Fix:** Use double quotes for all JSON strings. Use `json.dumps()` to generate JSON from Python objects — it always uses double quotes correctly. For the reverse direction, use `json.loads()` on properly quoted strings.

### Mistake 2: Forgetting That `json.dumps` Fails on Non-Serializable Types

```python
import json
from datetime import datetime

data = {"timestamp": datetime.now()}

try:
    json.dumps(data)  # TypeError!
except TypeError as e:
    print(e)  # Object of type datetime is not JSON serializable
```

**Why it happens:** `json.dumps()` only handles the basic six types. Python objects like `datetime`, `Decimal`, `set`, `numpy.ndarray`, and custom classes are not directly serializable.

**Symptoms:** `TypeError: Object of type X is not JSON serializable`.

**Fix:** Provide a `default` function to convert non-serializable types, or convert them manually before serializing:

```python
json.dumps(data, default=str)  # Converts everything to string
# Better: write a custom serializer specific to your types
```

### Mistake 3: Loading an Entire Large JSON File Into Memory

```python
import json

# For a 2 GB JSON file — this will crash or swap heavily
with open("huge_dataset.json", "r", encoding="utf-8") as f:
    data = json.load(f)  # 2 GB in memory!
```

**Why it happens:** `json.load()` parses the entire file and constructs the full Python object tree in memory. For large files, this exhausts RAM.

**Symptoms:** `MemoryError`, excessive swapping, or the program being killed by the OS.

**Fix:** Use JSON Lines format (one JSON object per line) to process records incrementally, or use `ijson` for streaming JSON parsing of deeply nested structures.

### Mistake 4: Not Validating Untrusted JSON Structure

```python
import json

untrusted = '{"choices": null}'
parsed = json.loads(untrusted)

# This will crash
first = parsed["choices"][0]  # TypeError: 'NoneType' object is not subscriptable
```

**Why it happens:** `json.loads()` parses whatever valid JSON you give it — it does not validate the structure matches your expectations. An API might return an error response instead of the expected data shape.

**Symptoms:** `TypeError`, `KeyError`, or `IndexError` when accessing nested fields that do not exist in the parsed structure.

**Fix:** Always validate structure with `.get()` and explicit checks:

```python
choices = parsed.get("choices")
if choices and isinstance(choices, list) and len(choices) > 0:
    content = choices[0].get("message", {}).get("content")
```

### Mistake 5: Serializing a Dict with Non-String Keys

```python
import json

data = {1: "one", 2: "two"}

# Works — keys are converted to strings
json_str = json.dumps(data)
print(json_str)  # {"1": "one", "2": "two"}

# But the round-trip changes the type!
parsed = json.loads(json_str)
print(list(parsed.keys()))  # ['1', '2'] — strings, not ints!
```

**Why it happens:** JSON object keys must be strings. When serializing, `json.dumps()` converts non-string keys to strings. When deserializing, all keys become strings — information about the original key type is lost.

**Symptoms:** Round-trip serialization changes key types from int/tuple to string.

**Fix:** Convert keys explicitly if the type matters:

```python
data = {1: "one", 2: "two"}
# Convert keys explicitly to preserve type info
serializable = {str(k): {"key_type": type(k).__name__, "value": v}
                for k, v in data.items()}
```
