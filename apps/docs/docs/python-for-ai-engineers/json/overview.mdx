# JSON

⏱ 10 minutes · Difficulty: Beginner

## What You Will Learn

By the end of this lesson you will be able to:

- Serialize Python objects to JSON and deserialize JSON back to Python
- Read and write JSON files with `json.dump()` and `json.load()`
- Handle nested JSON structures from API responses
- Validate JSON data and handle parsing errors
- Work with JSON Lines format for streaming data

## Prerequisites

You should already understand:

- [File Handling](../file-handling/overview) — JSON is read from and written to files
- [Dictionaries](../dictionaries/overview) — JSON objects map directly to Python dicts
- [Lists](../lists/overview) — JSON arrays map directly to Python lists

## Why This Matters

JSON is the universal data interchange format for web APIs. Every model inference request you send to an AI provider — OpenAI, Anthropic, Groq — uses JSON. Every API response you parse is JSON. Every configuration file for experiment tracking, hyperparameter tuning, and pipeline orchestration is JSON.

In AI engineering, you will:
- Send JSON payloads to model APIs
- Parse JSON responses containing predictions, logprobs, and metadata
- Read JSON configuration files for experiment parameters
- Write JSON results for downstream analysis
- Handle streaming JSON from batch inference pipelines

## The Problem

You call an LLM API and get back a JSON response with nested fields: choices, message content, token usage, finish reason. You need to extract the generated text and the token counts. Without understanding JSON structure and Python's `json` module, you cannot reliably navigate the response.

## The Concept

JSON (JavaScript Object Notation) represents data as key-value pairs and ordered lists. It has exactly six value types:

| JSON type | Python type |
|-----------|-------------|
| object — `{...}` | dict |
| array — `[...]` | list |
| string — `"..."` | str |
| number — `123` / `1.5` | int / float |
| boolean — `true` / `false` | bool |
| null | None |

The `json` module converts between JSON strings and Python objects automatically. `json.loads()` reads a string; `json.dumps()` writes one. The file variants `json.load()` and `json.dump()` work the same way with file objects.
