# JSON — Review

⏱ 3 minutes

## Key Takeaways

- Use `json.dumps()` / `json.dump()` to serialize Python objects to JSON strings/files, `json.loads()` / `json.load()` to deserialize back.
- JSON has six types: object, array, string, number, boolean, null — mapped to Python dict, list, str, int/float, bool, None.
- Always use `.get()` with defaults when navigating parsed JSON from external sources — the structure may not match your expectations.
- Provide a `default` function to `json.dumps()` for non-serializable types like `datetime` and NumPy arrays.
- Use JSON Lines (one object per line) for streaming large datasets.

## Recap in 60 Seconds

If you only remember three things:

1. `json.dumps(data, indent=2)` writes; `json.loads(string)` reads. Use the file variants (`dump`/`load`) with open files.
2. JSON keys and strings must use double quotes — use `json.dumps()` to generate valid JSON, never hand-write it.
3. Validate structure with `.get()` and isinstance checks before accessing nested fields in untrusted data.
