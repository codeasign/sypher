# Virtual Environments — Hands On

⏱ 12 minutes

## Breaking It Down

### Step 1: Create a Virtual Environment

Navigate to your project directory and run:

```bash
mkdir my_ai_project
cd my_ai_project

# Create a virtual environment named "venv" inside the project
python3 -m venv venv
```

This creates a `venv/` directory containing:

```
venv/
  bin/              # Python interpreter, pip, activate script (Linux/macOS)
  include/
  lib/
    python3.12/
      site-packages/  # Empty — ready for your packages
```

On Windows, the structure uses `Scripts\` instead of `bin/`:

```
venv/
  Scripts\          # python.exe, pip.exe, Activate.ps1
  Lib\
    site-packages\
```

### Step 2: Activate the Environment

**macOS / Linux:**

```bash
source venv/bin/activate
```

**Windows (PowerShell):**

```powershell
.\venv\Scripts\Activate.ps1
```

**Windows (cmd.exe):**

```cmd
.\venv\Scripts\activate.bat
```

After activation, your prompt changes to show the environment name:

```
(venv) your-machine:my_ai_project you$
```

This visual cue is the easiest way to confirm you are inside an environment. The `which python3` command also confirms it:

```bash
which python3
# /Users/you/my_ai_project/venv/bin/python3   (inside env)
# /usr/local/bin/python3                       (outside env)
```

### Step 3: Install Packages

With the environment active, install packages normally:

```bash
pip install requests
pip install numpy
```

Or install multiple in one command:

```bash
pip install requests numpy pandas
```

Check what is installed:

```bash
pip list
```

You should see your installed packages plus `pip` and `setuptools`.

### Step 4: Freeze Dependencies

Generate a `requirements.txt` file that records every installed package and its exact version:

```bash
pip freeze > requirements.txt
```

This file is the standard way to share dependencies. Its contents look like:

```
numpy==1.26.0
pandas==2.1.0
python-dateutil==2.8.2
pytz==2023.3
requests==2.31.0
```

### Step 5: Recreate the Environment from requirements.txt

Another developer (or your future self) can recreate the exact same environment:

```bash
# Create a fresh environment
python3 -m venv venv
source venv/bin/activate

# Install everything from the file
pip install -r requirements.txt
```

This guarantees identical dependency versions across machines — critical for reproducibility in AI projects.

### Step 6: Deactivate and Delete

When you are done working:

```bash
deactivate
```

This restores your shell PATH to its original state. To delete the environment entirely (it is just a directory):

```bash
rm -rf venv/
```

Create a new one when you need it again. Never commit `venv/` to version control — it is machine-specific and reproducible from `requirements.txt`.

## Build Something Real — AI Project Setup Script

Create a shell script that sets up a complete AI project environment:

```bash
# setup_project.sh — Creates a new AI project with venv and core deps
PROJECT_NAME=${1:-ai_project}

echo "Creating project: $PROJECT_NAME"
mkdir "$PROJECT_NAME"
cd "$PROJECT_NAME"

echo "Creating virtual environment..."
python3 -m venv venv
source venv/bin/activate

echo "Installing core AI packages..."
pip install numpy pandas requests openai scikit-learn

echo "Freezing dependencies..."
pip freeze > requirements.txt

echo "Project ready! Activate with: cd $PROJECT_NAME && source venv/bin/activate"
```

Run it:

```bash
bash setup_project.sh my_first_ai_project
```
