# Virtual Environments — Review

⏱ 3 minutes

## Key Takeaways

- `python3 -m venv venv` creates an isolated Python environment in the `venv/` directory.
- Activate with `source venv/bin/activate` (Linux/macOS) or `.\venv\Scripts\Activate.ps1` (Windows PowerShell) — your prompt changes to `(venv)` as confirmation.
- `pip freeze > requirements.txt` records exact dependency versions; `pip install -r requirements.txt` recreates them.
- Never commit `venv/` to version control — it is machine-specific and reproducible from `requirements.txt`.
- If you relocate a project, delete and recreate the virtual environment rather than moving it.

## Recap in 60 Seconds

If you only remember three things:

1. Create a virtual environment per project with `python3 -m venv venv` and always activate it before installing or running anything.
2. Use `pip freeze > requirements.txt` to save dependencies and `pip install -r requirements.txt` to restore them.
3. The `venv/` directory is disposable — delete and recreate freely; your `requirements.txt` is the source of truth.
