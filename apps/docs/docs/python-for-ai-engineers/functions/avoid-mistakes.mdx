# Functions — Common Mistakes

⏱ 7 minutes

## When NOT to Use Functions

- **Trivial operations used once:** `result = max(values) * 2` does not need a function wrapper.
- **Configuration data:** Use constants or data structures, not functions, for static values.
- **Class methods:** If a function operates on an object's state and needs many related functions, consider a class instead.

## The Pattern to Remember

- **Pattern:** `def name(parameters) -> ReturnType:` → body → `return value`. Use type hints and docstrings.
- **When to use:** Any logic that is called more than once, or that has a clear single responsibility.
- **When NOT to use:** Trivial one-use operations, static configuration.
- **Mental model:** A function is a machine — input goes in, processing happens, output comes out. The machine is isolated from the outside world.

## Common Mistakes

### Mistake 1: Mutable Default Arguments

```python
def add_item(item, cache=[]):  # ❌
    cache.append(item)
    return cache

print(add_item("a"))  # ['a']
print(add_item("b"))  # ['a', 'b'] — the same list!
print(add_item("c"))  # ['a', 'b', 'c'] — unintended accumulation
```

**Why it happens:** The default value is evaluated once when the function is defined, not each time the function is called. All calls share the same mutable object.

**Symptoms:** State leaks between calls. The function behaves differently on the second call than the first.

**Fix:** Use `None` and create the mutable inside the function:

```python
def add_item(item, cache=None):
    if cache is None:
        cache = []
    cache.append(item)
    return cache
```

### Mistake 2: Forgetting the Return Statement

```python
def process(text):
    result = text.strip().lower()
    # Forgot: return result

cleaned = process("  Hello  ")
print(cleaned)  # None
```

**Why it happens:** Every function returns `None` by default. If you do not write `return`, that is what the caller gets.

**Symptoms:** The caller receives `None` and either crashes (`AttributeError: 'NoneType' object has no attribute...`) or silently proceeds with no data.

**Fix:** Always include an explicit `return` statement. If a function truly has no return value, document that.

### Mistake 3: Modifying a Mutable Argument In-Place (Side Effects)

```python
def add_defaults(config):
    config["temperature"] = 0.7  # Modifies the original dict!
    config["max_tokens"] = 1024
    return config

original = {"model": "gpt-4"}
result = add_defaults(original)
print(original)  # {'model': 'gpt-4', 'temperature': 0.7, 'max_tokens': 1024} — changed!
```

**Why it happens:** Python passes arguments by reference. Modifying a mutable argument inside the function changes the original object outside the function.

**Symptoms:** The caller's data is unexpectedly modified. This is especially dangerous when the same dict is used in multiple places.

**Fix:** Copy before modifying:

```python
def add_defaults(config):
    config = config.copy()  # Work on a copy
    config["temperature"] = 0.7
    config["max_tokens"] = 1024
    return config
```

### Mistake 4: Confusing Local and Global Scope

```python
total = 0

def add_to_total(x):
    total += x  # UnboundLocalError: local variable 'total' referenced before assignment
```

**Why it happens:** Python decides a variable is local if it is assigned anywhere in the function. Since `total` is assigned (`total += x`), it is treated as local — and it has no value yet.

**Symptoms:** `UnboundLocalError`.

**Fix:** Pass the value as a parameter and return the updated value:

```python
total = 0

def add_to_total(current_total, x):
    return current_total + x

total = add_to_total(total, 5)
```

Or use `global`, but prefer the parameter approach.

### Mistake 5: Too Many Parameters

```python
# Hard to read, hard to call correctly
def create_pipeline(source, destination, format, delimiter, encoding,
                     compression, max_size, overwrite, backup, log_level):
    pass
```

**Why it happens:** You keep adding parameters as requirements grow, without refactoring.

**Symptoms:** Callers have no idea what parameters to pass or in what order. Missed parameters produce subtle bugs.

**Fix:** Group related parameters into a dict or a dataclass:

```python
from dataclasses import dataclass

@dataclass
class PipelineConfig:
    source: str
    destination: str
    format: str = "json"
    delimiter: str = ","
    encoding: str = "utf-8"
    compression: str | None = None
    max_size: int = 10_000_000
    overwrite: bool = False
    backup: bool = True
    log_level: str = "INFO"

def create_pipeline(config: PipelineConfig):
    print(f"Processing {config.source} -> {config.destination}")
```
