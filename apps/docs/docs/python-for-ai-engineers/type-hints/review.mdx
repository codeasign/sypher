# Type Hints — Review

⏱ 3 minutes

## Key Takeaways

- Type hints document function signatures — they are ignored at runtime but checked by tools like `mypy`.
- Use `list[str]`, `dict[str, float]`, `tuple[int, str]` for collections (Python 3.9+), and `X | None` or `Optional[X]` for nullable values.
- `Any` disables type checking — use it sparingly at typed/untyped boundaries.
- Use `TypeVar` to write generic functions that work with multiple types while preserving type safety.
- Type hints do not enforce types at runtime — use `isinstance()` if you need runtime guards.

## Recap in 60 Seconds

If you only remember three things:

1. Annotate every function parameter and return value — `def f(x: str) -> int:` — it costs nothing and documents your code.
2. Use `list[str]` not `typing.List[str]`, and `str | None` not `Optional[str]` (Python 3.10+).
3. Run `mypy` on your code to catch type errors before runtime — add it to your CI pipeline.
