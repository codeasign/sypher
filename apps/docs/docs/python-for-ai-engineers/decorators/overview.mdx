## Why This Matters

Decorators let you add behaviour to functions without modifying their code. This is essential for cross-cutting concerns — things that apply to many functions: log every API call, time every pipeline step, retry on failure, cache results, validate inputs, enforce rate limits.

In AI engineering, you will use decorators for: timing inference calls, logging model inputs and outputs, retrying API requests with backoff, caching expensive computations, and registering event handlers. Frameworks like Flask, Click, and pytest use decorators extensively, and understanding how they work lets you use those frameworks effectively and build your own.

## The Problem

You have five functions that call external APIs. Each one needs logging, timing, and retry logic. Without decorators, you would add the same boilerplate to every function:

```python
def call_openai(prompt):
    start = time.time()
    log.info("Calling OpenAI...")
    try:
        result = ...
        log.info(f"OpenAI returned in {time.time() - start:.2f}s")
        return result
    except Exception:
        log.error("OpenAI call failed")
        raise
```

This is repetitive, error-prone, and makes the actual logic harder to read. A decorator lets you define this wrapper once and apply it to any function with a single `@` line.

## The Concept

A decorator is a function that takes another function as an argument, wraps it with additional behaviour, and returns the wrapped function. The `@` syntax is syntactic sugar:

```python
@timer
def train_model():
    ...

# Is equivalent to:
def train_model():
    ...
train_model = timer(train_model)
```

The decorator function typically defines an inner `wrapper` function that calls the original and adds behaviour before or after.
