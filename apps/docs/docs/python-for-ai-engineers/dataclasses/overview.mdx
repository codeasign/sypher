## Why This Matters

In AI engineering, you constantly define data containers — a model prediction with text, confidence, and metadata; an experiment result with metrics and parameters; a configuration with multiple fields. Before dataclasses, you had two choices: use a dict (no autocomplete, no type checking, keys can be misspelled) or write a full class with `__init__`, `__repr__`, `__eq__`, and other boilerplate.

Dataclasses give you the best of both: a minimal definition that auto-generates the boilerplate, with type annotations that enable IDE support and static analysis.

## The Problem

You need a structure to hold a prediction result: input text, predicted label, confidence score, and model name. With a dict:

```python
result = {"text": "Great!", "label": "positive", "confidence": 0.95, "model": "gpt-4"}
# No autocomplete, no validation — result["lable"] silently returns KeyError
```

With a plain class:

```python
class Prediction:
    def __init__(self, text, label, confidence, model):
        self.text = text
        self.label = label
        self.confidence = confidence
        self.model = model

    def __repr__(self):
        return f"Prediction(text={self.text!r}, label={self.label!r}, ...)"
```

There is so much boilerplate for what is essentially four fields and the logic to initialise them. Dataclasses eliminate this repetition.

## The Concept

A dataclass is a class decorated with `@dataclass`. Python automatically generates `__init__`, `__repr__`, `__eq__`, and optionally `__hash__`, `__lt__`, and other methods based solely on the type-annotated fields you declare.
