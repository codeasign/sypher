## Why This Matters

Embeddings are the foundation of semantic search, clustering, recommendation systems, and retrieval-augmented generation (RAG). An embedding converts text — a sentence, a paragraph, an entire document — into a list of numbers (a vector) that captures its meaning. Similar texts have vectors that are close together in vector space.

Companies use embeddings for everything: GitHub Copilot retrieves relevant code snippets via embeddings, Notion's AI search uses embeddings, Pinterest recommends pins by embedding similarity. Every RAG pipeline — the standard architecture for grounding LLMs in your own data — starts with embeddings.

## The Problem

Keyword search fails on synonyms and context. Searching "cheap flights" does not match "low-cost air travel" even though they mean the same thing. Adding synonyms manually is endless maintenance. You need a way to find text by meaning, not by exact word matches.

Embeddings solve this by placing both the query and the documents in the same semantic space. You embed everything — the query and all documents — then find the closest vectors. The matching happens in meaning-space, not word-space.

## The Concept

An embedding model is a neural network that maps text to a fixed-size vector (e.g., 1536 dimensions for OpenAI's `text-embedding-3-small`). The model is trained so that texts with similar meaning produce vectors that are close together, measured by cosine similarity.

Key properties:

- **Fixed size regardless of input length.** A single word and a 1000-word document both produce a 1536-dimensional vector.
- **Semantic, not lexical.** "Car" and "automobile" have similar vectors despite sharing no characters.
- **Dense.** Every dimension carries information (compared to sparse one-hot encodings).
- **Direction matters.** The cosine of the angle between two vectors measures their similarity, from -1 (opposite) to 1 (identical).
