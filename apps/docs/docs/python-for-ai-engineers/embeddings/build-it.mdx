# Embeddings — Hands On

⏱ 15 minutes

## Breaking It Down

### Step 1: Generate an Embedding

OpenAI provides the most widely-used embedding API. Install the SDK:

```bash
pip install openai numpy
```

Then send text to the embedding endpoint:

```python
# generate_embedding.py
import os
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

response = client.embeddings.create(
    model="text-embedding-3-small",
    input="Machine learning is transforming healthcare.",
)

embedding = response.data[0].embedding
print(f"Dimensions: {len(embedding)}")   # Output: 1536
print(f"First 5 values: {embedding[:5]}")
# Output: [-0.008, 0.021, -0.035, 0.012, 0.053] (example values)
```

The `text-embedding-3-small` model produces 1536-dimensional vectors. The larger `text-embedding-3-large` produces 3072 dimensions. More dimensions capture more nuance but use more storage and compute.

### Step 2: Cosine Similarity

Cosine similarity measures how close two vectors are. It ranges from -1 (opposite directions) to 1 (same direction). For embeddings, values are almost always between 0 and 1.

```python
# cosine_similarity.py
import numpy as np


def cosine_similarity(a: list[float], b: list[float]) -> float:
    """Compute cosine similarity between two vectors."""
    a = np.array(a)
    b = np.array(b)
    return float(np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b)))


# Example: semantic similarity
vec1 = [0.1, 0.3, 0.8, 0.2]  # "I love Python programming"
vec2 = [0.15, 0.28, 0.75, 0.22]  # "Python is my favourite language"
vec3 = [0.9, 0.1, 0.05, 0.7]  # "The stock market rose today"

print(f"Similar texts:  {cosine_similarity(vec1, vec2):.4f}")  # ~0.97
print(f"Different texts: {cosine_similarity(vec1, vec3):.4f}")  # ~0.40
```

Similar texts have cosine similarity above 0.8. Different topics score below 0.5. The exact threshold depends on your domain.

### Step 3: Batch Embedding for Efficiency

Embedding one text at a time is slow. The API accepts a batch of inputs:

```python
# batch_embed.py
import os
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

texts = [
    "Python is a versatile programming language.",
    "JavaScript runs in the browser.",
    "Rust provides memory safety without garbage collection.",
    "Machine learning models require large datasets.",
    "The Eiffel Tower is in Paris.",
]

response = client.embeddings.create(
    model="text-embedding-3-small",
    input=texts,
)

# Each input gets its own embedding
embeddings = {texts[i]: data.embedding for i, data in enumerate(response.data)}
print(f"Generated {len(embeddings)} embeddings")
# Output: Generated 5 embeddings
```

Batch requests are more efficient than individual calls — the API processes them as a single request, reducing overhead.

### Step 4: Semantic Search In Memory

Build a search engine over a corpus: embed all documents, embed the query, find the closest vectors.

```python
# semantic_search.py
import os
import numpy as np
from openai import OpenAI


class SemanticSearch:
    """In-memory semantic search over a document collection."""

    def __init__(self):
        self.client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])
        self.documents: list[str] = []
        self.embeddings: list[list[float]] = []

    def add_documents(self, documents: list[str]):
        """Embed and store documents."""
        if not documents:
            return
        response = self.client.embeddings.create(
            model="text-embedding-3-small",
            input=documents,
        )
        self.documents.extend(documents)
        self.embeddings.extend(d.data.embedding for d in response.data)

    def search(self, query: str, top_k: int = 3) -> list[dict]:
        """Return top-k documents most similar to the query.

        Args:
            query: Natural language search query.
            top_k: Number of results to return.

        Returns:
            List of {document, score} dicts, highest score first.
        """
        if not self.documents:
            return []

        # Embed the query
        response = self.client.embeddings.create(
            model="text-embedding-3-small",
            input=query,
        )
        query_emb = np.array(response.data[0].embedding)

        # Compute similarities
        scores = []
        for doc_emb in self.embeddings:
            doc_emb = np.array(doc_emb)
            sim = float(np.dot(query_emb, doc_emb) /
                        (np.linalg.norm(query_emb) * np.linalg.norm(doc_emb)))
            scores.append(sim)

        # Get top-k indices
        top_indices = np.argsort(scores)[-top_k:][::-1]

        return [
            {"document": self.documents[i], "score": round(scores[i], 4)}
            for i in top_indices
        ]


# Usage
searcher = SemanticSearch()
searcher.add_documents([
    "Python is great for data science and machine learning.",
    "JavaScript is the language of the web, running in every browser.",
    "Rust offers performance comparable to C++ with modern safety guarantees.",
    "Pandas and NumPy are essential Python data science libraries.",
    "The Eiffel Tower was built in 1889 for the World's Fair.",
])

results = searcher.search("data science libraries")
for r in results:
    print(f"  {r['score']:.4f}  {r['document']}")

# Expected output:
#   0.8923  Pandas and NumPy are essential Python data science libraries.
#   0.7641  Python is great for data science and machine learning.
#   0.4321  JavaScript is the language of the web...
```

## Build Something Real — Document Search with Persistent Storage

The in-memory approach works for small datasets but does not persist. For production, store embeddings in a vector database (Chroma, Pinecone, Weaviate). Here is a persistent version using Chroma:

```bash
pip install chromadb
```

```python
# persistent_search.py
import os
import chromadb
from openai import OpenAI


class PersistentVectorSearch:
    """Semantic search backed by Chroma for persistence."""

    def __init__(self, collection_name: str = "documents"):
        self.openai = OpenAI(api_key=os.environ["OPENAI_API_KEY"])
        self.chroma = chromadb.PersistentClient(path="./chroma_db")
        self.collection = self.chroma.get_or_create_collection(
            name=collection_name,
            metadata={"hnsw:space": "cosine"},  # Use cosine similarity
        )

    def add_documents(self, documents: list[str], ids: list[str] | None = None):
        """Embed documents with OpenAI and store in Chroma."""
        if ids is None:
            ids = [f"doc_{i}" for i in range(len(documents))]

        response = self.openai.embeddings.create(
            model="text-embedding-3-small",
            input=documents,
        )
        embeddings = [d.embedding for d in response.data]

        self.collection.add(
            embeddings=embeddings,
            documents=documents,
            ids=ids,
        )

    def search(self, query: str, top_k: int = 5) -> list[dict]:
        """Search by embedding similarity."""
        response = self.openai.embeddings.create(
            model="text-embedding-3-small",
            input=query,
        )
        query_emb = response.data[0].embedding

        results = self.collection.query(
            query_embeddings=[query_emb],
            n_results=top_k,
        )

        return [
            {"document": doc, "score": round(dist, 4)}
            for doc, dist in zip(
                results["documents"][0],
                results["distances"][0],  # Chroma returns distances, not similarities
            )
        ]


# Usage
vsearch = PersistentVectorSearch()
vsearch.add_documents([
    "Python 3.13 introduces a JIT compiler for performance improvements.",
    "The Anthropic API supports structured output via tool use.",
    "Retrieval-augmented generation combines search with LLMs.",
    "Docker containers provide reproducible environments for deployment.",
    "Pytest fixtures help manage test dependencies and setup.",
])

results = vsearch.search("API structured output")
for r in results:
    print(f"  {r['score']:.4f}  {r['document']}")
```

Chroma stores vectors on disk and rebuilds them on restart. For larger-scale production, use Pinecone or Weaviate.

## Performance Considerations

- **Embedding cost.** OpenAI charges per token for embeddings. `text-embedding-3-small` costs $0.02 per million tokens — very cheap for small-to-medium collections. A document collection of 10GB text costs roughly $0.20 to embed.
- **Storage.** 1536-dimensional float32 vectors: 6 KB per document. 1 million documents = 6 GB for vectors alone, plus index overhead.
- **Search speed.** Brute-force cosine similarity is O(N*d) per query. For collections over 100K documents, use an approximate nearest neighbour (ANN) index like HNSW (supported by Chroma and Pinecone).

## Security Considerations

- **Data sent to embedding APIs** is processed by the provider. Do not embed sensitive or proprietary text with third-party embedding APIs. Use a local embedding model (e.g., `sentence-transformers/all-MiniLM-L6-v2`) for sensitive data.
- **Access control.** If your search system returns documents based on embedding similarity, ensure the user is authorised to see every document returned. Embedding similarity does not respect access control.
