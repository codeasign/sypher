# AI Pipelines — Review

⏱ 3 minutes

## Key Takeaways

- Structure multi-step AI workflows as pipelines of independent stages, each with its own retry policy and error handling.
- Use Pydantic models for typed stage inputs and outputs — catch mismatches at stage boundaries, not in production.
- Wrap each item's pipeline run in try/except so one failure does not crash the entire batch.
- Limit concurrent LLM calls with a semaphore to avoid rate limits.
- Log every stage outcome (success, failure, latency, token usage) for observability and debugging.

## Recap in 60 Seconds

If you only remember three things:

1. Each pipeline stage is a function with retries and its own error boundary — one crash does not stop everything.
2. Type all stage inputs and outputs with Pydantic to catch interface mismatches immediately.
3. Throttle concurrent LLM calls with a semaphore — rate limits are guaranteed if you do not.
