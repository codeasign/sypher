# Async Python — Hands On

⏱ 18 minutes

## Breaking It Down

### Step 1: Your First Async Function

```python
# first_async.py
import asyncio

async def say_hello():
    print("Hello")
    await asyncio.sleep(1)  # Simulate an I/O wait
    print("World")

# Running an async function requires an event loop
asyncio.run(say_hello())
# Hello
# (1 second pause)
# World
```

`async def` defines a coroutine function. `await` pauses the coroutine until the awaited operation completes. `asyncio.run()` is the entry point — it creates an event loop, runs the coroutine, and shuts down the loop.

### Step 2: Running Multiple Coroutines Sequentially vs Concurrently

```python
# sequential_vs_concurrent.py
import asyncio
import time

async def fetch_data(item_id: int) -> str:
    """Simulate an API call that takes 1 second."""
    await asyncio.sleep(1)
    return f"Data for item {item_id}"

async def sequential():
    """Run tasks one at a time — total ~3 seconds."""
    start = time.time()
    result1 = await fetch_data(1)
    result2 = await fetch_data(2)
    result3 = await fetch_data(3)
    elapsed = time.time() - start
    print(f"Sequential: {result1}, {result2}, {result3} ({elapsed:.1f}s)")

async def concurrent():
    """Run tasks concurrently — total ~1 second."""
    start = time.time()
    results = await asyncio.gather(
        fetch_data(1),
        fetch_data(2),
        fetch_data(3),
    )
    elapsed = time.time() - start
    print(f"Concurrent: {results} ({elapsed:.1f}s)")

asyncio.run(sequential())  # ~3 seconds
asyncio.run(concurrent())  # ~1 second
```

`asyncio.gather()` runs all provided coroutines concurrently and returns a list of their results in the same order.

### Step 3: Gathering a Dynamic List of Tasks

When you have a dynamic number of tasks, create a list of coroutines and unpack it:

```python
# gather_list.py
import asyncio

async def process_prompt(prompt: str) -> str:
    """Simulate processing a prompt through an LLM."""
    delay = len(prompt) * 0.1  # Longer prompts take longer
    await asyncio.sleep(delay)
    return f"Response to: {prompt[:20]}..."

async def main():
    prompts = [
        "What is Python?",
        "Explain async/await.",
        "How does list comprehension work?",
        "What is the difference between a list and a tuple?",
        "Explain NumPy broadcasting.",
    ]

    # Create a coroutine for each prompt
    tasks = [process_prompt(p) for p in prompts]

    # Run all concurrently
    results = await asyncio.gather(*tasks)

    for prompt, result in zip(prompts, results):
        print(f"{prompt[:25]:25s} → {result}")

asyncio.run(main())
```

### Step 4: Limiting Concurrency with Semaphore

Without a limit, `asyncio.gather()` runs all tasks at once. For 1,000 tasks, this can overwhelm your network, the server, or both:

```python
# semaphore.py
import asyncio

async def call_api(session_id: int) -> str:
    """Simulate an API call."""
    await asyncio.sleep(1)
    return f"Session {session_id} complete"

async def main():
    sem = asyncio.Semaphore(5)  # Max 5 concurrent calls

    async def limited_call(session_id: int) -> str:
        async with sem:
            return await call_api(session_id)

    # Create 20 tasks
    tasks = [limited_call(i) for i in range(20)]

    start = asyncio.get_event_loop().time()
    results = await asyncio.gather(*tasks)
    elapsed = asyncio.get_event_loop().time() - start

    print(f"Completed {len(results)} tasks with concurrency 5 in {elapsed:.1f}s")
    # With 20 tasks at 5 concurrent, total ~4 seconds (20/5 = 4 batches × 1s)

asyncio.run(main())
```

The `Semaphore` ensures only 5 tasks run at any moment. The remaining tasks wait in the queue until a slot opens.

### Step 5: Background Tasks with `create_task`

Sometimes you want to start a task and continue working while it runs:

```python
# background_tasks.py
import asyncio

async def background_worker(name: str, interval: float):
    """Run indefinitely, doing periodic work."""
    for i in range(5):
        await asyncio.sleep(interval)
        print(f"  [{name}] Tick {i + 1}")
    return f"{name} done"

async def main():
    # Start background tasks — they run in the background
    task1 = asyncio.create_task(background_worker("worker-A", 0.5))
    task2 = asyncio.create_task(background_worker("worker-B", 0.8))

    # Do main work while background tasks run
    print("Main: Starting main work...")
    for i in range(3):
        await asyncio.sleep(1.0)
        print(f"Main: Working... step {i + 1}")
    print("Main: Main work complete.")

    # Wait for background tasks to finish
    result1 = await task1
    result2 = await task2
    print(f"Results: {result1}, {result2}")

asyncio.run(main())
```

`create_task()` schedules a coroutine to run in the background. The event loop interleaves it with other tasks. Use `await task` to retrieve its result when you need it.

### Step 6: Async Generator — Streaming Data

Async generators yield values as they become available, without blocking the event loop:

```python
# async_gen.py
import asyncio

async def stream_tokens(text: str):
    """Simulate streaming tokens one at a time."""
    for word in text.split():
        await asyncio.sleep(0.2)  # Simulate network delay per token
        yield word

async def main():
    prompt = "Async Python makes concurrent programming elegant"
    print("Streaming tokens:")
    async for token in stream_tokens(prompt):
        print(f"  Received: {token}")

asyncio.run(main())
```

Async generators use `async for` instead of `for`. They are the foundation for streaming LLM responses, processing large files line by line, and handling real-time data feeds.

## Build Something Real — Async Batch Processor

```python
# async_batch.py
import asyncio
import time

class AsyncBatchProcessor:
    """Process a batch of items concurrently with rate limiting."""

    def __init__(self, concurrency: int = 10):
        self.sem = asyncio.Semaphore(concurrency)
        self.results = []
        self.errors = []

    async def process_one(self, item: dict) -> dict:
        """Process a single item (simulate API call)."""
        async with self.sem:
            # Simulate variable processing time
            delay = 0.5 + (item.get("complexity", 1) * 0.3)
            await asyncio.sleep(delay)

            # Simulate occasional failure
            if item.get("should_fail"):
                raise ValueError(f"Item {item['id']} failed processing")

            return {
                "id": item["id"],
                "result": f"Processed in {delay:.1f}s",
                "latency": delay,
            }

    async def run_batch(self, items: list[dict]) -> dict:
        """Process all items concurrently."""
        start = time.time()

        async def safe_process(item: dict):
            try:
                result = await self.process_one(item)
                self.results.append(result)
            except Exception as e:
                self.errors.append({"id": item["id"], "error": str(e)})

        tasks = [safe_process(item) for item in items]
        await asyncio.gather(*tasks)

        elapsed = time.time() - start
        return {
            "total": len(items),
            "success": len(self.results),
            "failed": len(self.errors),
            "elapsed_seconds": round(elapsed, 2),
            "avg_latency": round(
                sum(r["latency"] for r in self.results) / max(len(self.results), 1),
                2,
            ),
        }

async def main():
    # Create a batch of 20 items with varying complexity
    items = [
        {"id": i, "complexity": (i % 3) + 1, "should_fail": (i % 7 == 0)}
        for i in range(20)
    ]

    processor = AsyncBatchProcessor(concurrency=5)
    summary = await processor.run_batch(items)

    print(f"Batch Summary:")
    print(f"  Total:    {summary['total']}")
    print(f"  Success:  {summary['success']}")
    print(f"  Failed:   {summary['failed']}")
    print(f"  Elapsed:  {summary['elapsed_seconds']}s")
    print(f"  Avg lat:  {summary['avg_latency']}s")

    # Expected: 20 items, concurrency 5
    # Each item takes 0.8–1.4s, so worst-case: ~4 batches × 1.4s = ~5.6s
    # Expected failures: ids 0, 7, 14 → 3 failures, 17 successes

asyncio.run(main())
```

Summary calculation: `sum(latency) / count = avg_latency`. Items 7 and 14 fail (i % 7 == 0), so 17 succeed. With 5 concurrent slots and items ranging 0.8–1.4s latency, total elapsed is roughly `ceil(20/5) × 1.4 = 5.6s`.

## Performance Considerations

- **Async is for I/O, not CPU:** Async does not make CPU-bound code faster. For numerical computation, use NumPy, multiprocessing, or PyTorch with GPU.
- **Event loop overhead:** A single `await` has microsecond-level overhead. For millions of operations, this adds up — batch your work where possible.
- **Connection pooling:** Async HTTP clients (like httpx.AsyncClient) maintain their own connection pools. Reuse the client across requests.
- **Rate limiting:** Server-imposed rate limits (429 Too Many Requests) may cap your effective concurrency regardless of your local settings.
- **Task cancellation:** Long-running tasks can be cancelled with `task.cancel()`. Use `asyncio.shield()` for operations that should not be interrupted.
