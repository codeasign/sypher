# Async Python

⏱ 12 minutes · Difficulty: Advanced

## What You Will Learn

By the end of this lesson you will be able to:

- Write and execute async functions using `async` and `await`
- Run multiple coroutines concurrently with `asyncio.gather()`
- Create and manage background tasks with `asyncio.create_task()`
- Use `asyncio.Semaphore` to limit concurrency
- Choose between sync and async for different workloads

## Why This Matters

Modern AI workloads are I/O-bound: waiting for API responses, reading files, streaming data, querying databases. During those waits, your program is idle — the CPU does nothing while the network or disk catches up.

Async Python lets your program do other work while waiting. Instead of sitting idle during a 2-second API call, it can dispatch 100 concurrent calls and process results as they arrive. This is how production AI pipelines achieve throughput on large batches of inference requests.

## The Problem

You have 100 prompts to send to an LLM API. Each takes about 2 seconds to respond. With synchronous code, you send one request, wait 2 seconds, get the response, send the next — total time: 200 seconds.

With async code, you send all 100 requests at once and wait for all of them to complete. Total time: roughly 2 seconds (plus some overhead). The CPU is not doing more work — it is just not sitting idle during network I/O.

## The Concept

Async Python is built on **coroutines** — functions that can pause their execution at `await` points and resume later. The event loop manages which coroutine runs at any given moment. When a coroutine hits an `await`, it yields control to the event loop, which can run another coroutine that is ready.

Key distinction: async is **concurrency** (managing multiple tasks with overlapping timelines) on a single thread, not **parallelism** (running multiple threads/processes simultaneously on different cores).
