# Async Python — Review

⏱ 3 minutes

## Key Takeaways

- Define coroutines with `async def`, call them with `await`, run the entry point with `asyncio.run()`.
- Use `asyncio.gather()` to run multiple coroutines concurrently; use `return_exceptions=True` to handle individual failures.
- Limit concurrency with `asyncio.Semaphore` to avoid overwhelming servers or local resources.
- Never call blocking sync functions (`time.sleep()`, `requests.get()`) inside async code — use async equivalents.
- Use `asyncio.create_task()` for background work, and always `await` the task before the event loop exits.

## Recap in 60 Seconds

If you only remember three things:

1. Async is concurrency on one thread — `await` yields to the event loop so other coroutines can run during I/O waits.
2. `asyncio.gather()` runs tasks concurrently; `asyncio.Semaphore` limits how many run at once.
3. Never block the event loop with sync I/O — use `asyncio.sleep()`, `httpx.AsyncClient`, and `aiofiles` instead.
