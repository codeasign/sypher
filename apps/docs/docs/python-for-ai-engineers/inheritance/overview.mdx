# Inheritance

⏱ 10 minutes · Difficulty: Intermediate

## What You Will Learn

By the end of this lesson you will be able to:

- Create a subclass that inherits attributes and methods from a parent class
- Override methods in a subclass to customise behaviour
- Use `super()` to call the parent class's methods
- Understand Python's Method Resolution Order (MRO) for multiple inheritance
- Use abstract base classes to define interfaces

## Prerequisites

You should already understand:

- [Classes](../classes/overview) — inheritance builds on class definitions
- [Functions](../functions/overview) — method overriding is redefining a function in a subclass

## Why This Matters

Inheritance lets you define common behaviour in one place and reuse it across multiple classes. In AI engineering, you might have a base `BaseModel` class with common methods (`load()`, `save()`, `evaluate()`) and subclasses for specific model types (`GPTModel`, `ClaudeModel`, `LlamaModel`) that override just the parts that differ.

Inheritance is also the foundation of frameworks: when you write a custom PyTorch module, you inherit from `torch.nn.Module` and override `forward()`. The framework calls your code through the inherited interface — this pattern (the Template Method pattern) is everywhere in Python.

## The Problem

You have three model integrations — OpenAI, Anthropic, and Llama — each with its own API, but they share common behaviour: loading API keys, handling rate limits, logging calls, retrying on failure. Without inheritance, you duplicate the common code in each class — violating DRY and making maintenance a nightmare. With inheritance, the common code lives in a parent class and each subclass implements only what is specific to that provider.

## The Concept

Inheritance creates an **is-a** relationship. If `GPTModel` inherits from `BaseModel`, then `GPTModel` is a `BaseModel` — it has all the same attributes and methods, plus any it adds or overrides.

`super()` lets a subclass call the parent class's version of a method — essential when the subclass wants to extend the parent's behaviour rather than replace it entirely.
