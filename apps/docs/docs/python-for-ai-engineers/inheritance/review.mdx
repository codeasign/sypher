# Inheritance — Review

⏱ 3 minutes

## Key Takeaways

- Inheritance models **is-a** relationships — a `GPTModel` is a `BaseModel`.
- Call `super().__init__()` and `super().method()` to extend the parent's behaviour rather than replacing it.
- Use abstract base classes (`ABC`, `@abstractmethod`) to define interfaces that subclasses must implement.
- Python's MRO (C3 linearisation) resolves method lookup order in multiple inheritance — left-to-right, depth-first.
- Prefer composition over inheritance for has-a relationships.

## Recap in 60 Seconds

If you only remember three things:

1. Use inheritance for is-a relationships and `super()` to extend parent behaviour — do not forget to call `super().__init__()`.
2. Define abstract base classes with `@abstractmethod` to enforce interface contracts on subclasses.
3. Prefer composition over inheritance for code reuse — "has a" is usually better than "is a."
