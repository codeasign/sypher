# LLM API Basics — Hands On

⏱ 15 minutes

## Breaking It Down

### Step 1: Install the SDKs and Set Up Your API Key

Both Anthropic and OpenAI publish official Python SDKs on PyPI.

```bash
pip install anthropic openai python-dotenv
```

Create a `.env` file in your project root — never hardcode keys:

```bash
# .env
ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-...
```

Tools like `python-dotenv` load these into `os.environ` at runtime. Alternatively, set them in your shell profile or CI/CD secrets manager.

### Step 2: Your First API Call

Here is the minimal pattern for calling both providers.

```python
# first_call.py
import os
from dotenv import load_dotenv

load_dotenv()

# --- Anthropic ---
from anthropic import Anthropic

anthropic_client = Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])

response = anthropic_client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    messages=[
        {"role": "user", "content": "What is the capital of Japan?"}
    ],
)

print(response.content[0].text)
# Output: Tokyo
```

```python
# --- OpenAI ---
from openai import OpenAI

openai_client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

response = openai_client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "user", "content": "What is the capital of Japan?"}
    ],
)

print(response.choices[0].message.content)
# Output: Tokyo
```

The response object contains the generated text, token usage, model metadata, and the stop reason.

### Step 3: Multi-turn Conversations

LLMs are stateless — each call is independent. To simulate conversation history, you send the full message list every time.

```python
import os
from anthropic import Anthropic

client = Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])

messages = [
    {"role": "user", "content": "My name is Alice."},
    {"role": "assistant", "content": "Nice to meet you, Alice!"},
    {"role": "user", "content": "What is my name?"},
]

response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=256,
    messages=messages,
)

print(response.content[0].text)
# Output: Your name is Alice.
```

Every turn appends to the list. The provider charges for all tokens in the messages array, so longer histories cost more.

### Step 4: Streaming

Streaming renders tokens as they arrive instead of waiting for the full response. This is essential for chat interfaces and real-time applications.

```python
# streaming.py
import os
from anthropic import Anthropic

client = Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])

with client.messages.stream(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    messages=[{"role": "user", "content": "Write a haiku about Python."}],
) as stream:
    for text in stream.text_stream:
        print(text, end="", flush=True)
    # Prints tokens one by one as they arrive
    # The context manager handles the final message assembly
```

The OpenAI equivalent uses a generator:

```python
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

stream = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "Write a haiku about Python."}],
    stream=True,
)

for chunk in stream:
    delta = chunk.choices[0].delta.content
    if delta:
        print(delta, end="", flush=True)
```

### Step 5: System Prompts

System prompts set the model's behaviour and persona before the conversation starts.

```python
import os
from anthropic import Anthropic

client = Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])

response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    system="You are a senior Python reviewer. Point out bugs and suggest improvements concisely.",
    messages=[
        {"role": "user", "content": "def add(a,b): return a+b"}
    ],
)

print(response.content[0].text)
```

With OpenAI, the system message is the first entry in the messages array:

```python
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "system", "content": "You are a senior Python reviewer. Point out bugs and suggest improvements concisely."},
        {"role": "user", "content": "def add(a,b): return a+b"},
    ],
)

print(response.choices[0].message.content)
```

## Build Something Real — LLM Client Wrapper

The code below wraps both providers behind a unified interface with error handling, token tracking, and retries. This is the pattern used in production AI applications.

```python
# llm_client.py
import os
import time
import logging
from dataclasses import dataclass
from typing import Optional

logger = logging.getLogger(__name__)


@dataclass
class LLMResponse:
    text: str
    model: str
    input_tokens: int
    output_tokens: int
    latency_ms: float


class LLMClient:
    """Unified client for Anthropic and OpenAI APIs."""

    def __init__(self, provider: str = "anthropic"):
        self.provider = provider
        if provider == "anthropic":
            from anthropic import Anthropic
            self._client = Anthropic(
                api_key=os.environ["ANTHROPIC_API_KEY"]
            )
        elif provider == "openai":
            from openai import OpenAI
            self._client = OpenAI(
                api_key=os.environ["OPENAI_API_KEY"]
            )
        else:
            raise ValueError(f"Unsupported provider: {provider}")

    def generate(
        self,
        messages: list[dict],
        system: Optional[str] = None,
        model: Optional[str] = None,
        max_tokens: int = 1024,
        temperature: float = 0.7,
        max_retries: int = 3,
    ) -> LLMResponse:
        """Send a completion request with automatic retry on transient errors.

        Args:
            messages: Conversation history as role/content dicts.
            system: Optional system prompt.
            model: Model identifier (provider default if omitted).
            max_tokens: Maximum output tokens.
            temperature: Sampling temperature (0.0 to 1.0).
            max_retries: Number of retries on 5xx or timeout.

        Returns:
            LLMResponse with text, token usage, and latency.
        """
        last_error = None
        for attempt in range(max_retries):
            try:
                start = time.perf_counter()
                response = self._call(
                    messages, system, model, max_tokens, temperature
                )
                elapsed = time.perf_counter() - start

                return LLMResponse(
                    text=self._extract_text(response),
                    model=self._extract_model(response),
                    input_tokens=self._extract_input_tokens(response),
                    output_tokens=self._extract_output_tokens(response),
                    latency_ms=round(elapsed * 1000),
                )
            except (ConnectionError, TimeoutError) as e:
                last_error = e
                wait = 2 ** attempt  # Exponential backoff: 1s, 2s, 4s
                logger.warning(
                    "Request failed (attempt %d/%d): %s. Retrying in %ds",
                    attempt + 1, max_retries, e, wait,
                )
                time.sleep(wait)

        raise RuntimeError(
            f"Request failed after {max_retries} retries"
        ) from last_error

    def _call(self, messages, system, model, max_tokens, temperature):
        if self.provider == "anthropic":
            kwargs = dict(
                model=model or "claude-sonnet-4-20250514",
                max_tokens=max_tokens,
                temperature=temperature,
                messages=messages,
            )
            if system:
                kwargs["system"] = system
            return self._client.messages.create(**kwargs)
        else:
            if system:
                messages = [
                    {"role": "system", "content": system}
                ] + messages
            return self._client.chat.completions.create(
                model=model or "gpt-4o",
                max_tokens=max_tokens,
                temperature=temperature,
                messages=messages,
            )

    def _extract_text(self, response) -> str:
        if self.provider == "anthropic":
            return response.content[0].text
        return response.choices[0].message.content

    def _extract_model(self, response) -> str:
        return response.model

    def _extract_input_tokens(self, response) -> int:
        if self.provider == "anthropic":
            return response.usage.input_tokens
        return response.usage.prompt_tokens

    def _extract_output_tokens(self, response) -> int:
        if self.provider == "anthropic":
            return response.usage.output_tokens
        return response.usage.completion_tokens


# Usage
if __name__ == "__main__":
    logging.basicConfig(level=logging.INFO)
    client = LLMClient(provider="anthropic")
    result = client.generate(
        messages=[{"role": "user", "content": "What is 2 + 2?"}],
        max_tokens=128,
    )
    print(f"Response: {result.text}")
    print(f"Model: {result.model}")
    print(f"Tokens: {result.input_tokens} in / {result.output_tokens} out")
    print(f"Latency: {result.latency_ms}ms")
```

## Performance Considerations

Token costs add up fast. At $3 per million input tokens for Claude Sonnet 4, sending a 10K-token conversation costs $0.03 per call — trivial for one request, but at 10,000 calls per day that is $300 daily. Track every token. Use the shortest model that meets your quality bar for each task (Haiku 3.5 for simple classification, Sonnet 4 for complex reasoning). Cache repeated system prompts where the provider supports it.

## Security Considerations

- **Never hardcode API keys.** Use environment variables or a secrets manager (AWS Secrets Manager, Vault).
- **Validate user input before sending it to the model.** Prompt injection is a real attack vector — users can craft input that overrides your system prompt.
- **Set explicit `max_tokens` limits.** Without one, a misconfigured or adversarial prompt could generate an unbounded response and run up your bill.
- **Log token usage but never log full prompts or responses** that contain user data or API keys.
