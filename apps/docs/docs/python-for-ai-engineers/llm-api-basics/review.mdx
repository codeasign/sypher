# LLM API Basics — Review

⏱ 3 minutes

## Key Takeaways

- Install the provider's SDK (`anthropic`, `openai`) and authenticate with an API key from an environment variable.
- Send messages as a list of role/content dicts; the full conversation history is sent on every call.
- Use streaming for real-time output and set explicit `max_tokens` to control costs and latency.
- Implement retry logic with exponential backoff for production reliability — API calls fail regularly.
- Never hardcode API keys. Use `.env` files locally and a secrets manager in production.

## Recap in 60 Seconds

If you only remember three things:

1. An LLM API call is a JSON HTTP request: messages go in, text comes out, tokens get counted.
2. Stream for user-facing latency, always set `max_tokens`, and always retry transient failures.
3. Keep your API keys out of your code — environment variables are the minimum, secrets managers are better.
