# File Handling — Review

⏱ 3 minutes

## Key Takeaways

- Always use `with open(...) as f:` — the context manager guarantees the file is closed, even on exceptions.
- Always specify `encoding="utf-8"` — the default encoding varies by OS, leading to cross-platform bugs.
- Use `"r"` for reading, `"w"` for writing (overwrites!), `"a"` for appending, `"x"` for exclusive create.
- Use `pathlib.Path` for cross-platform path operations instead of string concatenation.
- Do not read entire large files into memory — iterate line-by-line or chunk-by-chunk.

## Recap in 60 Seconds

If you only remember three things:

1. `with open(...) as f:` is the only pattern you should use — never `open()` without `with`.
2. Always pass `encoding="utf-8"` — it saves hours of cross-platform debugging.
3. Use `json.dump()` / `json.load()` for structured data, `csv.writer` / `csv.reader` for tabular data, and `pathlib.Path` for all path operations.
