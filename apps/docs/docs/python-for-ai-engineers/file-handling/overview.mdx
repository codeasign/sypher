# File Handling

⏱ 10 minutes · Difficulty: Beginner

## What You Will Learn

By the end of this lesson you will be able to:

- Read from and write to text files using `open()` and context managers
- Use the `with` statement to ensure files are closed automatically
- Read and write JSON files for structured data
- Read and write CSV files using the `csv` module
- Work with file paths using `pathlib`

## Prerequisites

You should already understand:

- [Modules](../modules/overview) — file handling uses stdlib modules like `json`, `csv`, and `pathlib`
- [Strings](../strings/overview) — file content is text that you will manipulate
- [Dictionaries](../dictionaries/overview) — JSON maps to Python dicts

## Why This Matters

AI engineering is fundamentally data-intensive. You load training data from files, save model outputs, log metrics to CSV, read configuration from JSON, and write results for downstream consumption. Every pipeline starts and ends with file I/O.

Beyond data, you will work with prompt templates stored in files, save checkpoints during training, log errors to files for debugging, and process large text corpora. File handling is not a peripheral skill — it is how data enters and leaves your program.

## The Problem

You train a model and want to save the evaluation results — accuracy, precision, recall, F1 — so you can compare them across experiments. You could print them to the terminal and copy-paste into a spreadsheet. But that does not scale to 100 experiments. You need to write structured data to files in a format that can be read back by analysis scripts and notebooks.

The same pattern applies to everything you do: read input from a file, process it, write output to a file. The file is the boundary between your program and the world.

## The Concept

Python's `open()` function returns a **file object** that provides access to the file's contents. The `with` statement creates a **context manager** that automatically closes the file when the block exits — even if an exception occurs. This is the standard way to work with files in Python.

File modes tell Python how to open the file:

| Mode | Meaning |
|------|---------|
| `"r"` | Read (default) — file must exist |
| `"w"` | Write — creates or overwrites |
| `"a"` | Append — writes at the end of the file |
| `"x"` | Exclusive create — fails if file exists |
| `"rb"` / `"wb"` | Binary mode — for non-text files |
