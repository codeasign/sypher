# Composition — Review

⏱ 3 minutes

## Key Takeaways

- Composition models **has-a** relationships — a Pipeline has Transformers, a Service has a Backend.
- Inject dependencies through the constructor instead of creating them inside — this enables swapping and testing.
- Prefer composition over inheritance for code reuse — it is more flexible and less tightly coupled.
- The Delegation pattern lets each composed object focus on one responsibility.
- Composition works with dependency injection to make every component independently testable.

## Recap in 60 Seconds

If you only remember three things:

1. Composition is "has-a" — build complex objects by combining simpler objects that each do one thing.
2. Inject dependencies via `__init__` — never create them inside the constructor if you need testability.
3. Prefer composition over inheritance for most code reuse scenarios — it is more flexible and less fragile.
