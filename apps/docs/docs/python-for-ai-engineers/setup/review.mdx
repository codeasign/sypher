# Setting Up Python — Review

⏱ 3 minutes

## Key Takeaways

- Install Python intentionally using the official installer or a trusted package manager, and verify the version with `python3 --version`.
- Always use `python3 -m pip install <package>` instead of raw `pip` to guarantee the interpreter and package manager are paired.
- The most common setup failure is PATH-related — either Python was not added to PATH or the wrong interpreter is first in the search order.
- For development, install a modern Python (3.12+) — the version your OS ships is for system tools, not for your projects.
- Virtual environments (covered next) are the standard way to isolate project dependencies and prevent version conflicts.

## Recap in 60 Seconds

If you only remember three things:

1. Install Python from python.org or Homebrew, always check the "Add to PATH" box or use `brew link` — then restart your terminal.
2. Verify with `python3 --version` and `python3 -m pip --version` before trusting the installation.
3. Use `python3 -m pip install` (not `pip`) to keep interpreter and packages paired.
