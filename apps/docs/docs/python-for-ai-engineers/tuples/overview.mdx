## Why This Matters

Tuples are everywhere in Python, often without you noticing. Function return values are tuples. The `enumerate()` function yields `(index, value)` tuples. Dictionary `.items()` returns `(key, value)` tuples. When you use `a, b = b, a` to swap variables, Python is creating and unpacking a tuple.

In AI engineering, tuples are useful for representing fixed records — a coordinate `(x, y)`, a model version `(major, minor, patch)`, a configuration pair `(name, value)`. Because tuples are immutable, they can be used as dictionary keys, unlike lists. They also signal to other developers: "this data structure has a fixed number of elements that should not change."

## The Problem

You write a function that returns multiple values — a prediction label, a confidence score, and the processing time:

```python
def predict(text):
    # ... inference ...
    return "positive", 0.92, 0.034
```

Without tuples, you would need to return a list (mutable, suggestible to change) or define a class (overkill for a simple return). Tuples give you a lightweight, immutable container that clearly communicates: "these three values belong together and are not meant to be modified."

## The Concept

A tuple is an **ordered, immutable sequence**. It differs from a list in exactly one way: you cannot add, remove, or change elements after creation. Everything else — indexing, slicing, iteration — works identically.

Because tuples are immutable, they are **hashable** (if all their elements are also hashable), meaning they can be used as dictionary keys and set members. Lists cannot.

Think of a tuple as a sealed envelope. The contents are fixed once sealed. You can look inside (index, slice, iterate), but you cannot change what is inside. If you need a different set of values, you create a new envelope.
