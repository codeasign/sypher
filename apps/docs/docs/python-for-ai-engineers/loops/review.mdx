# Loops — Review

⏱ 3 minutes

## Key Takeaways

- Use `for item in iterable` for clean iteration; use `enumerate()` when you need the index, `zip()` to iterate multiple sequences in parallel.
- `for`/`else` runs the `else` block only if the loop completed without hitting `break` — useful for search loops.
- `while` loops require explicit progress toward the termination condition — always include a counter or break path.
- List comprehensions (`[expr for x in data if condition]`) are faster and more readable than manual `for` loops with `append()`.
- Never modify a list while iterating over it — iterate over a copy instead.

## Recap in 60 Seconds

If you only remember three things:

1. Use `for item in list` directly — avoid `for i in range(len(list))` unless you specifically need the index.
2. Pair `enumerate()` with indices and `zip()` with parallel sequences — they are more readable than manual counters.
3. Prefer comprehensions over manual loops for simple transformations — they are faster and more Pythonic.
