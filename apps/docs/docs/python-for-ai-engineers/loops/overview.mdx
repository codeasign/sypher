## Why This Matters

Loops are how you process batches of data. In AI engineering, you loop over training samples, evaluation records, API responses, token sequences, file paths, and configuration entries. A single inference call on one input becomes a loop when you scale to 10,000 inputs. A data-cleaning script that works on one file becomes a loop when you point it at a directory.

Knowing how to write clean, efficient loops — and when to avoid loops by using vectorised operations — separates productive engineers from those who fight their tools.

## The Problem

You have 5,000 customer reviews and need to send each one to an LLM for classification. You could copy-paste the inference code 5,000 times. Or you could write a loop: `for review in reviews: result = classify(review)` — two lines that replace thousands.

The challenge is not just repeating an action, but doing it with variants: skip certain items, stop early if something goes wrong, pair inputs with expected outputs, track progress. Python's loop tools handle all of these cleanly.

## The Concept

A `for` loop in Python iterates over an **iterable** — any object that can produce elements one at a time. Lists, strings, dicts, files, and generators are all iterable. The loop variable takes each element in turn, and the indented body runs once per element.

A `while` loop repeats as long as a condition is `True`. Use it when the number of iterations is not known in advance — waiting for an API to respond, polling a file, processing until a condition is met.
