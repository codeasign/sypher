# Loops — Common Mistakes

⏱ 6 minutes

## When NOT to Use Loops

- **NumPy/Pandas operations:** If you are looping over arrays to do arithmetic, use vectorised operations instead. `a + b` on numpy arrays is 10-100x faster than a loop.
- **List comprehensions:** Use a comprehension for simple transformations: `[x*2 for x in data]` instead of a loop with `.append()`.
- **map/filter:** For applying a function to every element, `map()` can be cleaner — but comprehensions are generally preferred in Python.

## The Pattern to Remember

- **Pattern:** Use `for item in iterable` by default; use `enumerate()` for index access; use `zip()` for parallel iteration; use `while` for condition-based repetition.
- **When to use:** Any time you need to repeat an operation over a sequence or until a condition changes.
- **When NOT to use:** Vectorised numpy/pandas operations, simple transformations (use comprehensions), infinite polling loops without a timeout.
- **Mental model:** A for loop processes each item in a sequence automatically; a while loop keeps going as long as a condition is true.

## Common Mistakes

### Mistake 1: Modifying a List While Iterating Over It

```python
# Bug — skips elements
items = [1, 2, 3, 4, 5]
for item in items:
    if item % 2 == 0:
        items.remove(item)
print(items)  # [1, 3, 5] — works here but not reliable
```

**Why it happens:** Removing an element shifts all subsequent indices down by one. The loop advances to the next index, skipping the element that shifted into the current position.

**Symptoms:** Some elements are not processed. The result changes depending on how many elements you remove.

**Fix:** Iterate over a copy, or build a new list:

```python
items = [item for item in items if item % 2 != 0]
```

### Mistake 2: Off-by-One with range()

```python
# Wanted: loop 5 times (indices 0-4)
for i in range(1, 5):
    print(i)  # Prints 1, 2, 3, 4 — that is 4 times, not 5
```

**Why it happens:** `range(start, stop)` is exclusive on the stop value. `range(5)` gives `0, 1, 2, 3, 4` (5 values). `range(1, 5)` gives `1, 2, 3, 4` (4 values).

**Symptoms:** You iterate one fewer time than expected.

**Fix:** When you want N values starting from 0, use `range(N)`. When you need to iterate a specific number of times, double-check the stop value.

### Mistake 3: Forgetting That zip Stops at the Shortest Input

```python
models = ["gpt-4", "claude-3"]          # 2 items
scores = [0.95, 0.91, 0.87, 0.88]      # 4 items

for model, score in zip(models, scores):
    print(f"{model}: {score}")
# gpt-4: 0.95
# claude-3: 0.91
# The last 2 scores are silently ignored!
```

**Why it happens:** `zip()` stops at the shortest iterable by design.

**Symptoms:** Data loss — elements at the end of longer sequences are never processed with no error message.

**Fix:** If you need all elements, use `zip_longest` from `itertools`:

```python
from itertools import zip_longest
for model, score in zip_longest(models, scores, fillvalue="N/A"):
    print(f"{model}: {score}")
# gpt-4: 0.95
# claude-3: 0.91
# N/A: 0.87
# N/A: 0.88
```

### Mistake 4: Infinite While Loop

```python
# Bug — never decreases, so the condition is always True
x = 10
while x > 0:
    print(x)
    # forgot: x -= 1
```

**Why it happens:** The loop body does not modify the condition variable. If the condition starts as `True`, it stays `True` forever.

**Symptoms:** The program hangs, consuming CPU indefinitely. You must kill the process.

**Fix:** Always ensure the loop body makes progress toward making the condition `False`. Use a `break` as a safety net:

```python
max_iterations = 1000
count = 0
while some_condition and count < max_iterations:
    # ... loop body ...
    count += 1
```

### Mistake 5: Using `append` in a Loop Instead of a Comprehension

```python
# Inefficient and verbose
squares = []
for x in range(100):
    squares.append(x ** 2)

# Better
squares = [x ** 2 for x in range(100)]
```

**Why it happens:** You learned `for` loops first and have not yet internalised comprehensions.

**Symptoms:** The code works but is slower (`.append()` in a loop involves attribute lookup and method call overhead per iteration) and more verbose than necessary.

**Fix:** Use list comprehensions for simple transformations:

```python
# Filter + transform
results = [process(x) for x in data if x > 0]

# Dict comprehension
square_map = {x: x ** 2 for x in range(10)}

# Set comprehension
unique_lengths = {len(word) for word in words}
```
