# Loops — Hands On

⏱ 14 minutes

## Breaking It Down

### Step 1: Basic for Loop Over a List

```python
# basic_for.py
models = ["gpt-4", "claude-3", "llama-3", "gemini-pro"]

for model in models:
    print(f"Running benchmark on {model}...")

# Running benchmark on gpt-4...
# Running benchmark on claude-3...
# Running benchmark on llama-3...
# Running benchmark on gemini-pro...
```

### Step 2: Looping with range()

`range(start, stop, step)` produces a sequence of integers without creating a full list in memory:

```python
# range_demo.py

# range(stop) — 0 to stop-1
for i in range(5):
    print(i)           # 0, 1, 2, 3, 4

# range(start, stop) — inclusive start, exclusive stop
for i in range(2, 6):
    print(i)           # 2, 3, 4, 5

# range(start, stop, step) — step controls increment
for i in range(0, 10, 2):
    print(i)           # 0, 2, 4, 6, 8

# Backwards
for i in range(5, 0, -1):
    print(i)           # 5, 4, 3, 2, 1
```

`range(5)` produces `0, 1, 2, 3, 4` — that is 5 values, starting from 0. This matches Python's zero-based indexing.

### Step 3: Looping with enumerate()

When you need both the index and the value:

```python
# enumerate_demo.py
configs = [
    {"model": "gpt-4", "temperature": 0.7},
    {"model": "claude-3", "temperature": 0.5},
    {"model": "llama-3", "temperature": 0.3},
]

for index, config in enumerate(configs):
    print(f"{index}: {config['model']} at temp {config['temperature']}")

# 0: gpt-4 at temp 0.7
# 1: claude-3 at temp 0.5
# 2: llama-3 at temp 0.3

# Start from a non-zero index
for index, item in enumerate(["a", "b", "c"], start=1):
    print(f"{index}: {item}")
# 1: a
# 2: b
# 3: c
```

### Step 4: Looping with zip()

When you need to iterate over multiple sequences in parallel:

```python
# zip_demo.py
models = ["gpt-4", "claude-3", "llama-3"]
latencies_ms = [1200, 1800, 900]
costs_per_1k = [0.03, 0.015, 0.002]

for model, latency, cost in zip(models, latencies_ms, costs_per_1k):
    print(f"{model}: {latency}ms, ${cost}/1k tokens")

# gpt-4: 1200ms, $0.03/1k tokens
# claude-3: 1800ms, $0.015/1k tokens
# llama-3: 900ms, $0.002/1k tokens

# zip stops at the shortest sequence
short = [1, 2, 3]
longer = ["a", "b", "c", "d"]
for a, b in zip(short, longer):
    print(a, b)
# 1 a
# 2 b
# 3 c  — "d" is not included
```

### Step 5: break, continue, and else

```python
# flow_control.py

# break — exit the loop entirely
for model in ["gpt-4", "claude-3", "llama-3", "gemini-pro"]:
    if model == "llama-3":
        print(f"Found {model}, stopping search.")
        break
    print(f"Checking {model}...")
# Checking gpt-4...
# Checking claude-3...
# Found llama-3, stopping search.

# continue — skip to the next iteration
scores = [0.92, 0.45, 0.88, 0.32, 0.95]
for score in scores:
    if score < 0.5:
        continue  # Skip low-confidence predictions
    print(f"Processing score: {score}")
# Processing score: 0.92
# Processing score: 0.88
# Processing score: 0.95

# for/else — else runs only if break was NOT hit
models = ["gpt-4", "claude-3", "llama-3"]
target = "claude-4"  # Not in the list
for model in models:
    if model == target:
        print(f"Found {target}")
        break
else:
    print(f"{target} not found in the list")
```

### Step 6: while Loops

Use `while` for condition-based repetition:

```python
# while_demo.py

# Retry loop with exponential backoff
import time

def call_api():
    """Simulated API call that fails sometimes."""
    import random
    return random.choice([True, False])

max_retries = 5
attempt = 1
wait = 1  # seconds

while attempt <= max_retries:
    print(f"Attempt {attempt}/{max_retries}...")
    if call_api():
        print("Success!")
        break
    if attempt < max_retries:
        print(f"Failed, waiting {wait}s...")
        time.sleep(wait)
        wait *= 2  # Exponential backoff: 1, 2, 4, 8 seconds
    attempt += 1
else:
    print("All retries exhausted.")
```

### Step 7: Build Something Real — Batch Inference Processor

```python
# batch_processor.py — Process a batch of inputs with progress tracking

import time

def mock_inference(text):
    """Simulate LLM inference."""
    time.sleep(0.1)  # Simulate latency
    return {"input": text, "tokens": len(text.split()), "sentiment": "positive"}

def process_batch(inputs, batch_size=5):
    """Process inputs in batches with progress reporting."""
    results = []
    total = len(inputs)

    for i in range(0, total, batch_size):
        batch = inputs[i:i + batch_size]
        batch_num = (i // batch_size) + 1
        total_batches = (total + batch_size - 1) // batch_size
        print(f"Batch {batch_num}/{total_batches} ({i+1}-{min(i+batch_size, total)} of {total})")

        for item in batch:
            result = mock_inference(item)
            results.append(result)

    return results

inputs = [
    "This product is amazing",
    "Not what I expected",
    "Decent quality for the price",
    "Customer support was unhelpful",
    "Would recommend to a friend",
    "Average at best",
    "Exceeded my expectations",
    "Needs improvement",
    "Perfect for daily use",
    "Could be better",
]

results = process_batch(inputs, batch_size=3)
print(f"\nProcessed {len(results)} inputs")

# Count tokens efficiently
total_tokens = sum(r["tokens"] for r in results)
print(f"Total tokens processed: {total_tokens}")
```

## Performance Considerations

- **for loops are inherently serial** — each iteration waits for the previous one. For CPU-bound work, consider `multiprocessing` or vectorised numpy operations.
- **List comprehensions are faster** than `for` loops with `.append()` — the comprehension executes at C speed inside the interpreter. Use them for simple transformations.
- **Avoid `range(len(list))`** — use `enumerate()` when you need indices, direct iteration when you do not. `range(len(list))` requires an extra lookup on every iteration.
- **break early** when possible — if you are searching for one item, stop as soon as you find it.
