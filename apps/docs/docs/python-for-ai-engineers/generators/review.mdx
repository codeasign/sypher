# Generators — Review

⏱ 3 minutes

## Key Takeaways

- Generator functions use `yield` to produce values lazily — one at a time, on demand.
- Generator expressions `(expr for x in iterable)` are the lazy equivalent of list comprehensions and use O(1) memory.
- A generator can be iterated only once — after exhaustion, recreate it or materialise to a list.
- Chain generators to build memory-efficient data pipelines that process one item at a time.
- Use `yield from` to delegate to a sub-generator or iterable.

## Recap in 60 Seconds

If you only remember three things:

1. Generators yield values one at a time and pause between yields — they never hold the entire dataset in memory.
2. Use generator expressions `(x for x in data)` instead of list comprehensions for large datasets where you only need to iterate once.
3. Generators are single-use — you cannot iterate twice over the same generator; recreate it if you need another pass.
