# Data Visualization — Common Mistakes

⏱ 5 minutes

## When NOT to Use matplotlib

- **Interactive web dashboards:** Use Plotly, Bokeh, or a dedicated dashboard framework — matplotlib is designed for static figures.
- **Geospatial data:** Use geopandas or folium for maps.
- **Statistical visualizations:** Use seaborn (built on matplotlib) for heatmaps, violin plots, pair plots, and regression plots.
- **Extremely large datasets (millions of points):** Use datashader or downsample before plotting — matplotlib slows down significantly with too many elements.

## The Pattern to Remember

- **Pattern:** `fig, ax = plt.subplots()` → `ax.plot(data)` → customize with `ax.set_xlabel()`, `ax.set_title()`, etc. → `fig.savefig()` or `plt.show()`.
- **When to use:** Publication-quality static charts, training curves, performance comparisons, data exploration, reports.
- **When NOT to use:** Interactive dashboards, geospatial maps, web applications with live data, massive point clouds.
- **Mental model:** A figure contains one or more axes. You draw on the axes, not the figure. Every visual element is customizable through the axes object.

## Common Mistakes

### Mistake 1: Using pyplot Globally Instead of the Object Interface

```python
import matplotlib.pyplot as plt

# Fragile — modifies the global figure state
plt.plot(x, y)
plt.xlabel("x")
plt.title("My Plot")
plt.show()  # Which figure is this showing?
```

**Why it happens:** The pyplot interface (`plt.plot()`) modifies a hidden "current figure." In scripts, notebooks, and functions, this leads to accidental cross-talk between plots.

**Symptoms:** Labels from a previous plot appear on the current plot. `plt.show()` shows an unexpected figure.

**Fix:** Use the object-oriented interface exclusively:

```python
fig, ax = plt.subplots()
ax.plot(x, y)
ax.set_xlabel("x")
ax.set_title("My Plot")
fig.savefig("plot.png")
```

### Mistake 2: Forgetting `plt.tight_layout()` or `bbox_inches="tight"`

```python
fig, ax = plt.subplots()
ax.set_xlabel("A very long label that gets cut off at the edge of the saved image")
fig.savefig("plot.png")  # X-axis label is cropped
```

**Why it happens:** matplotlib does not automatically expand the figure boundaries to include labels, titles, and tick labels. They extend beyond the saved area.

**Symptoms:** Labels, legends, or titles are cut off in the saved image even though they look fine on screen.

**Fix:** Always call `plt.tight_layout()` before saving, or pass `bbox_inches="tight"` to `savefig()`:

```python
fig.savefig("plot.png", bbox_inches="tight")
```

### Mistake 3: Creating Multiple Figures Without Closing Them

```python
import matplotlib.pyplot as plt
import numpy as np

for i in range(100):
    fig, ax = plt.subplots()
    ax.plot(np.random.randn(100))
    # fig is never closed — 100 figures accumulate in memory!
```

**Why it happens:** Each `plt.subplots()` creates a new figure object. If you do not close them, matplotlib holds references to all of them, consuming memory.

**Symptoms:** Memory usage grows unboundedly in loops. Eventually the program slows or crashes.

**Fix:** Close figures when you are done, or use `plt.close("all")` at the end:

```python
for i in range(100):
    fig, ax = plt.subplots()
    ax.plot(np.random.randn(100))
    fig.savefig(f"plot_{i}.png")
    plt.close(fig)  # Free memory
```

### Mistake 4: Not Setting Appropriate Axis Limits

```python
import matplotlib.pyplot as plt

data = [0.92, 0.95, 0.88, 0.96, 0.91]
fig, ax = plt.subplots()
ax.plot(data)
# Default y-axis limits are [0.87, 0.97]
# Small changes look dramatic because the axis starts near the data
```

**Why it happens:** matplotlib auto-scales to fit the data tightly. If your data ranges from 0.88 to 0.96, the y-axis may span only 0.1 units, making small fluctuations look like large changes.

**Symptoms:** Charts that exaggerate differences. A drop from 0.94 to 0.92 looks catastrophic because the axis does not start at 0.

**Fix:** Set y-axis limits explicitly when the scale matters:

```python
ax.set_ylim(0, 1.0)  # Full range for percentages
# Or use a consistent range across subplots
```

### Mistake 5: Ignoring the Difference Between `plt.close(fig)` and `fig.clear()`

```python
import matplotlib.pyplot as plt

fig, ax = plt.subplots()
# ... draw ...
fig.clear()   # Removes the axes but keeps the figure object — hidden memory
plt.close(fig)  # Fully releases the figure resources
```

**Why it happens:** `clear()` removes drawn content but keeps the figure allocated. `close()` releases the figure back to the operating system. Using `clear()` in loops leaks memory.

**Symptoms:** Plots get slower over time. Memory grows. Figures accumulate in memory.

**Fix:** Use `plt.close(fig)` when you are done with a figure. Use `clear()` only when you intend to redraw the same figure (e.g., for animation frames).
