## Why This Matters

In AI engineering, most data does not arrive as clean NumPy arrays. It arrives as CSV exports from databases, JSON responses from APIs, logs from experiments, or spreadsheets from collaborators. Before you can train a model or analyze results, you need to clean, transform, and explore the data.

Pandas is the standard tool for this. A DataFrame — a labeled 2D table — is the closest data structure to a spreadsheet or SQL table in Python. You load raw data, inspect it, filter rows, create new columns, handle missing values, and compute grouped statistics, all in a few lines of code.

## The Problem

You ran 1,000 model evaluation runs and saved the results to a CSV. You want to:

- Find the average accuracy per model type
- Filter out experiments where the model crashed (accuracy = 0)
- Group by the dataset version to see if performance improved over time

With plain Python and CSV modules, this is a significant amount of code:

```python
import csv

with open("results.csv") as f:
    rows = list(csv.DictReader(f))

avg_by_model = {}
for r in rows:
    if float(r["accuracy"]) == 0:
        continue
    model = r["model"]
    if model not in avg_by_model:
        avg_by_model[model] = []
    avg_by_model[model].append(float(r["accuracy"]))

result = {m: sum(v)/len(v) for m, v in avg_by_model.items()}
```

With pandas, the same analysis is a short expression.

## The Concept

A **DataFrame** is a 2D table with labeled rows (index) and columns. A **Series** is a single labeled column. Every column has a single dtype (like a NumPy array), but different columns can have different dtypes — one string, one float, one int.

Think of a DataFrame as a dict of aligned Series, or a spreadsheet with superpowers.
