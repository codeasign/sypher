- [Error Handling](../error-handling/overview) вҖ” HTTP errors must be caught and handled in both sync and async code

## Why This Matters

httpx is the modern HTTP client for Python. It provides the same familiar API as `requests` but adds async support, HTTP/2, and a more consistent timeout model. In AI engineering, you increasingly need async HTTP for:

- Making many concurrent LLM calls (evaluation benchmarks, batch inference)
- Streaming token-by-token responses from model APIs
- Building async web services and agents that wait on external APIs without blocking

## The Problem

You have 100 prompts to send to an LLM API. With `requests`, you can send them sequentially вҖ” one at a time вҖ” which takes `100 Г— latency_per_call`. With httpx's async client, you can send them concurrently, completing the entire batch in roughly the time of the single slowest call. For a 2-second API, that is the difference between 200 seconds and 2 seconds.

## The Concept

httpx provides two interfaces:

- `httpx.Client` вҖ” synchronous API, identical in usage to `requests.Session`
- `httpx.AsyncClient` вҖ” async API, used with `async`/`await` and `asyncio`

Both support the same features: connection pooling, timeout control, streaming, HTTP/2, and custom transports.
