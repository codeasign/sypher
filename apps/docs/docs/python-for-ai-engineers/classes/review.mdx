# Classes — Review

⏱ 3 minutes

## Key Takeaways

- Define a class with `class Name:`, initialise with `__init__(self, ...)`, create instances with `Name(args)`.
- Every instance method receives `self` as the first argument — Python provides it automatically.
- Use `@classmethod` for alternative constructors, `@staticmethod` for utility functions, and `@property` for controlled attribute access.
- Use `__str__` for user-facing string representation and `__repr__` for developer-facing unambiguous representation.
- Never use mutable class attributes for per-instance state — initialise in `__init__` instead.

## Recap in 60 Seconds

If you only remember three things:

1. `self` is the instance — every method must have it as the first parameter. Calling `obj.method()` automatically passes `obj` as `self`.
2. Use `@property` to add validation or computed attributes without changing how the attribute is accessed.
3. `__repr__` is for developers (unambiguous), `__str__` is for users (readable) — implement both.
