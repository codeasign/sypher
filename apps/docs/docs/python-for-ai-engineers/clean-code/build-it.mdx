# Clean Code — Hands On

⏱ 18 minutes

## Breaking It Down

### Step 1: Name Things What They Are

The most impactful change you can make is using descriptive names:

```python
# ❌ Unclear names
d = Path("data")
f = d / "reviews.csv"
r = []

# ✅ Clear names
data_dir = Path("data")
reviews_path = data_dir / "reviews.csv"
reviews = []
```

Good names answer three questions: what is this, why does it exist, and how should I use it?

| Bad Name | Good Name | Reason |
|----------|-----------|--------|
| `d` | `data_dir` | Describes what the variable holds |
| `f` | `reviews_path` | Describes both type (path) and content (reviews) |
| `r` | `reviews` | Plural indicates a collection |
| `x` | `api_response` | Describes what the API returned |
| `res` | `results` | Standard, clear name for output collection |
| `good` | `correct_count` | Describes what is being counted |

### Step 2: Extract Each Concern into Its Own Function

The original script does three things: load data, classify, evaluate. Extract each into a function:

```python
# data_loading.py
from pathlib import Path
from typing import List, Dict
import csv


def load_reviews(path: str) -> List[Dict[str, str]]:
    """Load review data from a CSV file.

    Args:
        path: Path to a CSV with 'text' and 'label' columns.

    Returns:
        List of dictionaries, each with 'text' and 'label' keys.
    """
    reviews_path = Path(path)
    if not reviews_path.exists():
        raise FileNotFoundError(f"File not found: {path}")

    reviews = []
    with open(reviews_path, "r") as file:
        reader = csv.DictReader(file)
        for row in reader:
            reviews.append({"text": row["text"], "label": row["label"]})

    return reviews
```

```python
# classification.py
from typing import List, Dict
from openai import OpenAI


def classify_sentiment(
    texts: List[str],
    model: str = "gpt-4o-mini",
) -> List[str]:
    """Classify each text as 'positive' or 'negative' using an LLM.

    Args:
        texts: List of text strings to classify.
        model: OpenAI model identifier to use.

    Returns:
        List of predicted labels in the same order as input texts.
    """
    client = OpenAI()
    predictions = []

    for text in texts:
        response = client.chat.completions.create(
            model=model,
            temperature=0.0,
            messages=[
                {
                    "role": "system",
                    "content": "Classify the sentiment as positive or negative. "
                               "Respond with one word only.",
                },
                {"role": "user", "content": text},
            ],
        )
        prediction = response.choices[0].message.content.strip().lower()
        predictions.append(prediction)

    return predictions
```

```python
# evaluation.py
from typing import List


def compute_accuracy(predictions: List[str], labels: List[str]) -> float:
    """Calculate the fraction of predictions that match the true labels.

    Normalises both sides by stripping whitespace and lowercasing.

    Args:
        predictions: List of predicted labels.
        labels: List of ground-truth labels.

    Returns:
        Accuracy as a float between 0.0 and 1.0.

    Raises:
        ValueError: If the lists have different lengths.
    """
    if len(predictions) != len(labels):
        raise ValueError(
            f"Length mismatch: predictions={len(predictions)}, labels={len(labels)}"
        )

    if not predictions:
        return 0.0

    correct = sum(
        1 for p, l in zip(predictions, labels)
        if p.strip().lower() == l.strip().lower()
    )
    return correct / len(predictions)
```

### Step 3: Use Type Hints in Every Function Signature

Type hints serve as documentation that never goes out of date:

```python
from typing import List, Dict, Optional
from pathlib import Path


def load_config(path: str, env: Optional[str] = None) -> Dict[str, object]:
    """Load configuration with optional environment override."""
    # Implementation...
    pass
```

Type hints catch bugs before runtime when used with a type checker like `mypy`:

```bash
pip install mypy
mypy src/my_project/
```

### Step 4: Write Docstrings That Explain Why, Not What

A good docstring answers questions the code alone cannot:

```python
# ❌ Explains what the code already shows
def load_reviews(path):
    """Open file, create reader, loop over rows, append to list, return list."""
    ...

# ✅ Explains intent, constraints, and side effects
def load_reviews(path: str) -> List[Dict[str, str]]:
    """Load review data from a CSV file.

    The CSV must have 'text' and 'label' columns. The 'label' column
    should contain 'positive' or 'negative' strings. No preprocessing
    or cleaning is applied — whitespace is preserved.

    Args:
        path: Absolute or relative path to the CSV file.

    Returns:
        List of dictionaries, one per row, with 'text' and 'label' keys.

    Raises:
        FileNotFoundError: If the file does not exist at the given path.
        ValueError: If required columns are missing.
    """
```

Use the Google-style docstring format shown above — it renders well in IDE tooltips and works with documentation generators like Sphinx.

### Step 5: Build Something Real — Full Refactored Pipeline

Here is the original messy script refactored into clean, modular code:

```python
# scripts/run_sentiment.py
"""CLI entry point for the sentiment classification pipeline."""

import sys
import argparse
from pathlib import Path
from typing import List, Dict

import csv
from openai import OpenAI


def load_reviews(path: str) -> List[Dict[str, str]]:
    """Load review data from a CSV file with 'text' and 'label' columns."""
    reviews_path = Path(path)
    if not reviews_path.exists():
        raise FileNotFoundError(f"Reviews file not found: {path}")

    with open(reviews_path, "r") as file:
        reader = csv.DictReader(file)
        reviews = list(reader)

    required = {"text", "label"}
    first_row = reviews[0] if reviews else {}
    if not required.issubset(first_row.keys()):
        raise ValueError(
            f"CSV must contain columns: {required}. "
            f"Found: {set(first_row.keys()) if reviews else 'empty file'}"
        )
    return reviews


def classify_sentiment(
    texts: List[str],
    model: str = "gpt-4o-mini",
    temperature: float = 0.0,
) -> List[str]:
    """Classify each text as positive or negative using an LLM API.

    Args:
        texts: Text strings to classify.
        model: OpenAI model name.
        temperature: Sampling temperature (0.0 for deterministic).

    Returns:
        Predicted labels in the same order as input texts.
    """
    client = OpenAI()
    predictions: List[str] = []

    for text in texts:
        response = client.chat.completions.create(
            model=model,
            temperature=temperature,
            messages=[
                {
                    "role": "system",
                    "content": "Classify the sentiment as positive or negative. "
                               "Respond with one word only."
                },
                {"role": "user", "content": text},
            ],
        )
        prediction = response.choices[0].message.content.strip().lower()
        predictions.append(prediction)

    return predictions


def compute_accuracy(predictions: List[str], labels: List[str]) -> float:
    """Calculate accuracy by comparing predictions to ground-truth labels.

    Both lists are normalised by stripping whitespace and lowercasing
    before comparison.
    """
    if len(predictions) != len(labels):
        raise ValueError(
            f"Prediction/label length mismatch: "
            f"{len(predictions)} vs {len(labels)}"
        )
    if not predictions:
        return 0.0
    correct = sum(
        1 for p, l in zip(predictions, labels)
        if p.strip().lower() == l.strip().lower()
    )
    return correct / len(predictions)


def main() -> int:
    """Parse arguments, run pipeline, print results."""
    parser = argparse.ArgumentParser(
        description="Run sentiment classification on a CSV of reviews"
    )
    parser.add_argument("input_csv", help="Path to CSV with 'text' and 'label' columns")
    parser.add_argument("--model", default="gpt-4o-mini", help="OpenAI model to use")
    args = parser.parse_args()

    reviews = load_reviews(args.input_csv)
    texts = [r["text"] for r in reviews]
    labels = [r["label"] for r in reviews]

    predictions = classify_sentiment(texts, model=args.model)
    accuracy = compute_accuracy(predictions, labels)

    print(f"Processed {len(reviews)} reviews")
    print(f"Accuracy: {accuracy:.2%}")
    return 0


if __name__ == "__main__":
    sys.exit(main())
```

### Step 6: Run the Linter

Use `ruff` to enforce clean code automatically:

```bash
pip install ruff
ruff check src/
ruff format src/
```

Configure it in `pyproject.toml`:

```toml
[tool.ruff]
line-length = 88
target-version = "py310"

[tool.ruff.lint]
select = ["E", "F", "I", "N", "W", "D"]
```

Running `ruff check` before every commit catches issues like unused imports, missing docstrings, and naming violations before they reach code review.

## Security Considerations

When writing clean code for AI systems, be careful about logging — you might inadvertently log prompt inputs or API responses that contain sensitive user data. Use structured logging with clear redaction policies rather than `print()`:

```python
import logging

logger = logging.getLogger(__name__)

def classify_text(text: str) -> str:
    logger.info("Classifying text of length %d characters", len(text))
    # Never log the actual text content without explicit sanitisation
    ...
```
