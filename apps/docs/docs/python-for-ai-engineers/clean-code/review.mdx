# Clean Code — Review

⏱ 3 minutes

## Key Takeaways

- Use descriptive names that answer what, why, and how — comments are a code smell for poor naming.
- Each function should do exactly one thing. If you cannot name it with a precise verb, it does too much.
- Type hints and docstrings (Google-style) are documentation that never goes out of date.
- Never use bare `except:` — always catch specific exception types.
- Always apply `@functools.wraps` in your decorators to preserve function metadata.

## Recap in 60 Seconds

If you only remember three things:

1. Extract one task per function, name it precisely, and document *why* with a docstring.
2. Use type hints on every function signature — they catch bugs before runtime.
3. Run a linter (ruff, mypy) in CI so clean code is enforced, not optional.
