# Prompt Engineering — Review

⏱ 3 minutes

## Key Takeaways

- A system prompt sets persona, tone, and constraints; write it in specific, behavioural terms, not vague greetings.
- Few-shot examples anchor the output format — show 2-3 examples covering edge cases, not just the typical case.
- Chain-of-thought prompting improves multi-step reasoning accuracy by forcing the model to show its work.
- Use temperature 0.0 for deterministic tasks (classification, extraction) and 0.7-0.9 for creative tasks.
- Structure prompts with context first, then instruction last — the model pays most attention to recency.

## Recap in 60 Seconds

If you only remember three things:

1. System prompt sets the rules, few-shot examples set the format, instruction at the end sets the task.
2. Temperature 0.0 gives deterministic output; only raise it when you want variation.
3. Prompt injection is a real security risk — validate and sanitise user input before interpolating it into prompts.
