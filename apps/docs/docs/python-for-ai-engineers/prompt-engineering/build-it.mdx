# Prompt Engineering — Hands On

⏱ 15 minutes

## Breaking It Down

### Step 1: System Prompts — Set the Role

The system prompt establishes the model's behaviour before any user message. It is the most powerful single lever you have.

```python
# system_prompt.py
import os
from anthropic import Anthropic

client = Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])

# Without a system prompt — generic output
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=256,
    messages=[{"role": "user", "content": "Explain what an API is."}],
)
print("GENERIC:", response.content[0].text[:100])

# With a system prompt — targeted explanation
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=256,
    system="You are a senior backend engineer teaching a junior developer. "
           "Use analogies and avoid jargon. Keep explanations under 3 sentences.",
    messages=[{"role": "user", "content": "Explain what an API is."}],
)
print("TARGETED:", response.content[0].text[:100])
```

The second response will be simpler, shorter, and use an analogy — because you constrained the persona and tone.

### Step 2: Few-Shot Examples — Anchor the Format

When you need a specific output format, show examples. The model pattern-matches on the input-output pairs.

```python
# few_shot.py
import os
from anthropic import Anthropic

client = Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])

# Zero-shot — model chooses format
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=256,
    messages=[{
        "role": "user",
        "content": "Extract the sentiment from this review: "
                   "'The battery life is terrible but the screen is gorgeous.'"
    }],
)
print("ZERO-SHOT:", response.content[0].text)

# Few-shot — exact format
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=256,
    messages=[
        {"role": "user", "content": "Review: 'Love this app, works perfectly.'"},
        {"role": "assistant", "content": "Sentiment: POSITIVE\nConfidence: 0.95"},
        {"role": "user", "content": "Review: 'Too expensive and crashes often.'"},
        {"role": "assistant", "content": "Sentiment: NEGATIVE\nConfidence: 0.92"},
        {"role": "user", "content": "Review: 'The battery life is terrible but the screen is gorgeous.'"},
    ],
)
print("FEW-SHOT:", response.content[0].text)
# Output: Sentiment: MIXED\nConfidence: 0.88
```

The few-shot version returns a predictable format you can parse without fragile regex.

### Step 3: Chain-of-Thought Prompting

For reasoning tasks (math, logic, multi-step planning), asking the model to "think step by step" dramatically improves accuracy.

```python
# chain_of_thought.py
import os
from anthropic import Anthropic

client = Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])

# Without CoT — often wrong on multi-step reasoning
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=512,
    messages=[{
        "role": "user",
        "content": "A store has 15 apples. A customer buys 3 apples and returns 2."
                   "Then 5 more apples arrive. How many apples does the store have now?"
    }],
)
print("DIRECT:", response.content[0].text)

# With CoT
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=512,
    messages=[{
        "role": "user",
        "content": "A store has 15 apples. A customer buys 3 apples and returns 2. "
                   "Then 5 more apples arrive. How many apples does the store have now?\n\n"
                   "Think step by step before giving the final answer."
    }],
)
print("COT:", response.content[0].text)
```

The CoT version walks through: start 15, minus 3 = 12, plus 2 = 14, plus 5 = 19. The direct version is more likely to miss a step.

### Step 4: Temperature and Sampling Parameters

Temperature controls randomness. Low values (0.0-0.3) make output deterministic — use for classification and extraction. High values (0.8-1.0) increase creativity — use for brainstorming and creative writing.

```python
# temperature_compare.py
import os
from anthropic import Anthropic

client = Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])

message = [{"role": "user", "content": "Suggest a name for a Python library that analyses cat photos."}]

# Low temperature — predictable, safe names
low = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=128, temperature=0.0,
    messages=message,
)
print(f"TEMP=0.0: {low.content[0].text}")

# High temperature — creative, varied names
high = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=128, temperature=0.9,
    messages=message,
)
print(f"TEMP=0.9: {high.content[0].text}")
```

Run this multiple times. At temperature 0.0 the output is identical every call. At 0.9 it differs each time.

## Build Something Real — Prompt Template Manager

Production applications string together prompts dynamically with user input, few-shot examples, and system instructions. Hardcoding prompt strings inline is fragile — use a template manager.

```python
# prompt_manager.py
from dataclasses import dataclass, field
from typing import Optional


@dataclass
class PromptTemplate:
    """Reusable prompt template with system prompt, examples, and instructions."""

    name: str
    system_prompt: str
    examples: list[dict] = field(default_factory=list)
    instruction_template: str = ""

    def format(self, input_text: str, **kwargs) -> list[dict]:
        """Build the messages list from template + input.

        Returns:
            Messages list suitable for the LLM API.
        """
        messages = []

        # Add few-shot examples
        for ex in self.examples:
            messages.append({"role": "user", "content": ex["input"]})
            messages.append({"role": "assistant", "content": ex["output"]})

        # Format the instruction with user input
        instruction = self.instruction_template.format(
            input=input_text, **kwargs
        )
        messages.append({"role": "user", "content": instruction})

        return messages


# --- Sentiment classifier template ---
sentiment_template = PromptTemplate(
    name="sentiment",
    system_prompt=(
        "You are a sentiment classifier. Respond only with "
        "SENTIMENT: (POSITIVE|NEGATIVE|MIXED|NEUTRAL) and CONFIDENCE: (0.0-1.0). "
        "No other text."
    ),
    examples=[
        {"input": "Review: 'Absolutely love this product!'",
         "output": "SENTIMENT: POSITIVE\nCONFIDENCE: 0.98"},
        {"input": "Review: 'Waste of money, broke in a day.'",
         "output": "SENTIMENT: NEGATIVE\nCONFIDENCE: 0.95"},
        {"input": "Review: 'It is okay for the price.'",
         "output": "SENTIMENT: NEUTRAL\nCONFIDENCE: 0.70"},
    ],
    instruction_template="{input}",
)

# Usage
messages = sentiment_template.format(
    input_text="Review: 'Great battery, terrible camera.'"
)
# messages now contains: 3 examples (6 messages) + the final instruction

print("Generated messages:", len(messages), "turns")
```

With this pattern, you define prompts declaratively, test them independently, and reuse them across your application.

## Performance Considerations

- **Longer prompts cost more.** Every example and instruction token is billed. Keep few-shot examples minimal — 2-3 is often enough. Remove redundant system prompt clauses.
- **Caching system prompts.** Anthropic supports prompt caching for system prompts and large context blocks. A cached prompt is much cheaper on subsequent calls.
- **Chain-of-thought increases output tokens.** A CoT prompt that generates 500 reasoning tokens before a 20-token answer costs ~25x more than a direct prompt. Use CoT only when reasoning quality is critical.

## Security Considerations

- **Prompt injection.** A user can craft input that overrides your system prompt: "Ignore previous instructions and act as a different AI." Validate and sanitise any user-provided text that gets interpolated into prompts.
- **Output validation.** Never trust the LLM's output format. Parse with regex or a schema validator and reject malformed responses. A prompt-engineered format can still break.
- **Data leakage.** User input in few-shot examples or system prompts may be visible to the provider. Do not put PII or secrets in prompts.
