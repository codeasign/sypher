## Why This Matters

Prompt engineering is the primary way you control an LLM's output. The same model asked the same question with different wording can produce a correct answer, a hallucination, or a refusal. Companies that deploy LLMs in production invest heavily in prompt engineering because a well-crafted prompt is the difference between a feature that works reliably and one that fails unpredictably.

This is not an art — it is an engineering discipline with measurable techniques: system messages set the role and constraints, few-shot examples anchor output format, chain-of-thought improves reasoning accuracy, and temperature controls creativity versus determinism. Every AI engineer needs these techniques.

## The Problem

You ask an LLM to "extract the key points from this article." The response is a paragraph with bullet points, some commentary, and a markdown table. You ask again and get a completely different format. You try to parse the output programmatically and it breaks on the variation.

The model is not being difficult — it just does not know what format you want unless you show it. The naive approach of writing a one-sentence instruction gives the model too much freedom. You need structured prompting techniques that constrain the output space.

## The Concept

A prompt has three structural layers:

1. **System prompt** — sets the model's persona, constraints, and behaviour for the entire conversation. Applied once, it influences every response.
2. **Few-shot examples** — show the model exactly what you want by example. The model is very good at pattern-matching, so 2-3 well-chosen examples often outperform a detailed instruction.
3. **Instructions** — the specific task, placed at the end of the prompt where the model pays most attention (recency bias).

Each layer constrains the output more tightly. A system prompt alone gives broad direction; adding examples gives precise format control.
