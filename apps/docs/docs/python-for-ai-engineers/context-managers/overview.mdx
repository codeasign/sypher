# Context Managers

⏱ 10 minutes · Difficulty: Intermediate

## What You Will Learn

By the end of this lesson you will be able to:

- Write a custom context manager using a class with `__enter__` and `__exit__`
- Write a context manager using `@contextmanager` from `contextlib`
- Use context managers for timing, file handling, and temporary changes

## Prerequisites

You should already understand:

- [Decorators](../decorators/overview) — `@contextmanager` is a decorator-based pattern
- [File handling](../file-handling/overview) — `with open(...)` is the most common context manager
- [Exception handling basics](https://docs.python.org/3/tutorial/errors.html) — `__exit__` can handle exceptions

## Why This Matters

Context managers automate setup and teardown. You use them already with `with open(...) as f:` — the file is automatically closed when the block exits. The same pattern applies to: database connections that need committing or rolling back, model inference sessions that need GPU memory cleanup, temporary directory creation, environment variable changes, and timing blocks.

Writing your own context managers means you can encapsulate any "do this before, do that after" pattern into a reusable, safe block that guarantees cleanup even when exceptions occur.

## The Problem

You train a model and need to record how long it takes. Without a context manager:

```python
start = time.time()
train_model()
elapsed = time.time() - start
print(f"Training took {elapsed:.2f}s")
```

This works but scatters timing logic across your code. If `train_model()` raises an exception, you miss the timing. A context manager encapsulates the start/stop timing into a reusable block that always runs the "after" step.

## The Concept

A context manager is an object that defines `__enter__` and `__exit__` methods. The `with` statement calls `__enter__` when the block starts, and calls `__exit__` when the block ends — whether by normal completion or exception. This guarantees cleanup always runs.

Python also provides `contextlib.contextmanager` which lets you write a context manager as a generator function with a single `yield`.
