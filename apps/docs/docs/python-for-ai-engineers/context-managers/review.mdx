# Context Managers — Review

⏱ 3 minutes

## Key Takeaways

- Context managers guarantee cleanup via `__enter__` and `__exit__` — use `with` to invoke them.
- Write custom managers with `@contextmanager` and a generator containing one `yield` wrapped in `try/finally`.
- The `@contextmanager` yield is the boundary between setup code (before) and teardown code (after).
- Do not return `True` from `__exit__` unless you explicitly want to suppress exceptions.
- Use `contextlib.suppress` to ignore specific exceptions, or write your own with `@contextmanager`.

## Recap in 60 Seconds

If you only remember three things:

1. Always wrap `yield` in `try/finally` when writing a `@contextmanager` — otherwise teardown code may not run if the block raises.
2. `__exit__` returning `True` suppresses exceptions — return `False` or `None` to propagate them.
3. Built-in context managers like `with open(...)` exist for files; write custom ones for timing, DB sessions, model loading, and temporary state changes.
