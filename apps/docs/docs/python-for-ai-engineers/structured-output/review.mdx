# Structured Output — Review

⏱ 3 minutes

## Key Takeaways

- Define output schemas as Pydantic models and pass their JSON schema to the API via tool use or response_format.
- Structured output eliminates fragile regex parsing — the API constrains token generation to match your schema.
- Always validate the response with Pydantic; structured output reduces but does not eliminate validation errors.
- Make realistically-missing fields `Optional` with a default of `None` to avoid spurious extraction failures.
- Keep schemas simple — flat structures with few fields are faster to generate and less error-prone than deeply nested ones.

## Recap in 60 Seconds

If you only remember three things:

1. Pass the schema to the API so the model generates valid JSON matching your structure — no regex or markdown stripping needed.
2. Always validate the response through Pydantic — structured output can still produce type errors or missing fields.
3. Use `Optional` for fields that may realistically be absent from the source text.
