# Structured Output

⏱ 12 minutes · Difficulty: Advanced

- Handle schema violations and partial failures gracefully

## Prerequisites

You should already understand:

- [Prompt Engineering](../prompt-engineering/overview) — structured output is a prompt engineering technique enforced at the API level
- [JSON](../json/overview) — schemas and responses are JSON
- [Type Hints](../type-hints/overview) — Pydantic models use Python type annotations
- [Dataclasses](../dataclasses/overview) — Pydantic is a superset of dataclass-style definitions

## Why This Matters

The most common production problem with LLM integration is parsing the output. A model asked to return JSON may: wrap it in markdown, add a preamble ("Here is the data you requested..."), forget a closing brace, use single quotes, or include extra fields. Each variation breaks your parser.

Structured output solves this at the API level. Both Anthropic and OpenAI support forcing the model to return output that matches a JSON schema you provide. If the model produces invalid output, the API returns an error — it does not silently return malformed data. This shifts validation from fragile regex to compile-time schema enforcement.

## The Problem

You ask an LLM to extract a person's name, age, and occupation from text. The response is:

```json
{name: "Alice", "age": "thirty", job: "Engineer"}
```

This is invalid JSON (keys unquoted, age as a string, extra key). Your application crashes. You add more prompt instructions. The next response wraps the JSON in a markdown code block. You add regex extraction. The next response changes "Engineer" to "engineer". Your validation breaks again.

This cat-and-mouse game is unsustainable. You need a mechanism where the API guarantees the output matches your schema.

## The Concept

Structured output works by attaching a schema to the API request. The model generates tokens constrained to produce valid JSON that conforms to your schema. The API layer enforces this by rejecting invalid token sequences during generation.

The schema is defined as a JSON Schema or a Pydantic model. You send it in the request alongside the prompt. The response contains a `parsed` field that is already validated — no parsing step needed on your side.
