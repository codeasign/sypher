# Magic Methods — Review

⏱ 3 minutes

## Key Takeaways

- Magic methods let your objects integrate with Python's built-in syntax — `len()`, `str()`, `[]`, `==`, `+`, `in`, `for`, etc.
- Always implement `__repr__` for every class — it helps with debugging and logging.
- When you define `__eq__`, define `__hash__` too — always keep them consistent.
- Use `__getitem__` and `__len__` to make objects container-like; use `__iter__` to make them iterable.
- Implement `__bool__` explicitly if the default (based on `__len__`) does not match your object's semantics.

## Recap in 60 Seconds

If you only remember three things:

1. `__repr__` is the most important magic method — implement it on every class you write for clearer debugging.
2. Implement `__eq__` and `__hash__` together when you want value equality — never one without the other.
3. Magic methods are hooks into Python syntax — `obj[key]` calls `__getitem__`, `for x in obj` calls `__iter__`, `str(obj)` calls `__str__`.
