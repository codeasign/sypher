# Pytest — Review

⏱ 3 minutes

## Key Takeaways

- Write test functions named `test_*` with plain `assert` statements — no special API needed.
- Use fixtures for reusable test data and mock clients; control their scope (`function`, `session`) to balance isolation and speed.
- Parametrize tests with `@pytest.mark.parametrize` to cover multiple inputs without duplicating code.
- Mock all external API calls (LLM providers) so tests are fast, deterministic, and cost-free.
- Test error paths as thoroughly as happy paths — empty inputs, malformed responses, timeout exceptions.

## Recap in 60 Seconds

If you only remember three things:

1. One test per behaviour, not one test per function — test observable inputs and outputs.
2. Mock every external API call so tests run offline, instantly, and cost nothing.
3. Test error paths: empty inputs, malformed responses, and exception cases are where production bugs hide.
