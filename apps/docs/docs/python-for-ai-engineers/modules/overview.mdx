# Modules and Packages

⏱ 10 minutes · Difficulty: Beginner

## What You Will Learn

By the end of this lesson you will be able to:

- Import standard library modules and third-party packages
- Create your own module and import it from another script
- Understand the `if __name__ == "__main__"` guard
- Use `pip` to install third-party packages
- Structure a Python project with multiple modules

## Prerequisites

You should already understand:

- [Functions](../functions/overview) — modules are where you store your functions
- [Virtual environments](../virtual-environments/overview) — how to isolate project dependencies
- [File system basics](https://docs.python.org/3/tutorial/modules.html) — directories and files

## Why This Matters

No real AI project lives in a single file. You will have one file for data loading, another for model configuration, another for evaluation, and another for the main pipeline. Modules are how you split code across files while keeping it importable and organised.

Beyond your own code, you will import from the Python standard library (`os`, `sys`, `json`, `csv`, `re`, `collections`, `pathlib`) and from third-party libraries (`numpy`, `transformers`, `openai`, `torch`). Understanding how the import system works — where Python looks for modules, how it resolves names, and how to structure your own packages — is essential for any project that goes beyond a single script.

## The Problem

You write a data-loading function and a classification function in the same file. The file grows to 500 lines. You need to use the data-loading function in another project — but you cannot import a function from a script that also prints output and defines unrelated things. You need to separate reusable code into its own file that can be imported without executing side effects.

## The Concept

A **module** is any `.py` file. A **package** is a directory of `.py` files containing an `__init__.py` (Python 3.3+ supports namespace packages without `__init__.py`, but it is conventional to include one).

When you write `import x`, Python:

1. Looks for `x` in `sys.modules` (already imported modules).
2. Looks for `x` in the directories listed in `sys.path` (which includes the script's directory, the standard library directory, and installed packages).
3. If found, executes the module and makes it available as a namespace.
