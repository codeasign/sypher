# Control Flow

⏱ 10 minutes · Difficulty: Beginner

## What You Will Learn

- Evaluate truthy and falsy values in Python conditions

- Use the `match`/`case` statement (Python 3.10+) for pattern matching

- [Variables and data types](../variables/overview)

## Why This Matters

Every non-trivial program makes decisions. If the confidence score is above 0.9, use the prediction directly. If the response contains a code block, format it for display. If the API returns a 429 status, retry with backoff. If the model name contains "gpt-4", use one tokeniser; otherwise, use another.

Control flow is how you encode these decisions. Well-written conditionals make the logic of your program obvious. Poorly written conditionals — deeply nested, relying on obscure truthiness rules — are a common source of bugs and maintenance headaches.

## The Problem

You need to classify a sentiment score into a label:

```
score > 0.6 → "positive"
0.3 < score <= 0.6 → "neutral"
score <= 0.3 → "negative"
score is None → "unknown"
```

Without conditionals, you would need separate functions for each case and convoluted logic to handle the boundaries. With `if`/`elif`/`else`, the boundaries are explicit, readable, and easy to verify.

## The Concept

Control flow lets you execute different code paths based on conditions. The `if` statement evaluates a **condition** as a boolean. If the condition is `True`, the indented block runs. If `False`, Python checks `elif` conditions in order. If none match, the `else` block runs.

A condition can be any expression. Python considers some values **falsy** (evaluate to `False`): `False`, `0`, `0.0`, `""` (empty string), `[]` (empty list), `None`, and empty dicts/sets. Everything else is **truthy**.
