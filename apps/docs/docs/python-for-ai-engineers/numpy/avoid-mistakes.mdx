# NumPy — Common Mistakes

⏱ 6 minutes

## When NOT to Use NumPy

- **Small data (a few hundred elements):** The overhead of creating a NumPy array from a list often exceeds the speed gain. For small collections, Python lists are simpler and fast enough.
- **Heterogeneous data:** NumPy arrays require one dtype. If your data mixes strings, numbers, and dates, use pandas or plain Python objects.
- **Non-numerical data:** NumPy excels at numbers. For text, use Python strings. For categorical data, use pandas or scikit-learn encoders.
- **GPU workloads:** For deep learning, use PyTorch or TensorFlow tensors directly — they support GPU acceleration that NumPy cannot use.

## The Pattern to Remember

- **Pattern:** `import numpy as np` — create arrays with `np.array()`, operate element-wise, use boolean indexing for filtering, use ufuncs for math.
- **When to use:** Numerical computation, signal processing, feature engineering, linear algebra, random sampling, performance-critical math.
- **When NOT to use:** Small data, heterogeneous data, text processing, GPU workloads, simple data structures.
- **Mental model:** A NumPy array is a dense memory block where every operation is a tight C loop — write array-level expressions, not element-level loops.

## Common Mistakes

### Mistake 1: Using Python Loops Instead of Vectorized Operations

```python
import numpy as np

arr = np.array([1, 2, 3, 4, 5])

# Slow — Python loop
result = []
for x in arr:
    result.append(x ** 2)

# Fast — vectorized
result = arr ** 2
```

**Why it happens:** You think in Python loops because that is how lists work. But NumPy arrays have vectorized operations that apply to the entire array at once.

**Symptoms:** Code that works but is 10–100x slower than expected. Iterating over a NumPy array with a `for` loop is a sign you are missing the vectorized equivalent.

**Fix:** Replace element-level operations with array-level expressions. If you need a conditional, use `np.where()` or boolean indexing instead of a loop.

### Mistake 2: Confusing Copy and View

```python
import numpy as np

arr = np.array([1, 2, 3, 4, 5])
subset = arr[:3]    # View, not a copy
subset[0] = 999
print(arr)          # [999   2   3   4   5] — original changed!

# Correct — explicit copy
subset = arr[:3].copy()
subset[0] = 0
print(arr)          # [999   2   3   4   5] — original unchanged
```

**Why it happens:** Slicing returns a view for performance (no data copy). You modify the "subset" expecting it to be independent.

**Symptoms:** Values in the original array change unexpectedly. The bug is hard to trace because the modifying code is far from where the slice was taken.

**Fix:** Call `.copy()` when you need an independent array. Know that slicing, `reshape` (on contiguous arrays), and `T` on 2D arrays return views.

### Mistake 3: Forgetting That Boolean Indexing Returns a Copy

```python
import numpy as np

arr = np.array([1, 2, 3, 4, 5])
mask = arr > 2
subset = arr[mask]  # This IS a copy
subset[0] = 999
print(arr)          # [1 2 3 4 5] — original unchanged
```

Unlike slicing, boolean indexing always returns a copy. This is the opposite of the slicing rule and catches people who assume the same behaviour.

**Symptoms:** Trying to modify filtered elements in-place does not affect the original array. You fix the copy, not the source.

**Fix:** Use `np.where()` to find indices when you need to modify the original in-place, or assign back: `arr[arr > 2] = 999`.

### Mistake 4: Mismatched Shapes in Operations

```python
import numpy as np

a = np.array([1, 2, 3])       # shape (3,)
b = np.array([[1, 2], [3, 4], [5, 6]])  # shape (3, 2)

try:
    result = a + b  # ValueError: operands could not be broadcast together
except ValueError as e:
    print(e)  # shapes (3,) and (3,2) not aligned
```

**Why it happens:** Broadcasting requires the trailing dimensions to match or be 1. When they do not, NumPy raises an error.

**Symptoms:** `ValueError: operands could not be broadcast together with shapes ...`

**Fix:** Check shapes with `.shape` before operations. Reshape with `.reshape()` or add dimensions with `np.newaxis`:

```python
a_2d = a[:, np.newaxis]  # shape (3, 1) — now broadcastable with (3, 2)
print(a_2d + b)
# [[ 2  3]
#  [ 5  6]
#  [ 8  9]]
```

### Mistake 5: Using `np.array` on a List of Lists with Different Lengths

```python
import numpy as np

# Jagged list — inner lists have different lengths
jagged = [[1, 2, 3], [4, 5]]

try:
    arr = np.array(jagged)  # Creates array of Python lists, not 2D!
except ValueError as e:
    print(e)

# What you actually get:
arr = np.array(jagged, dtype=object)
print(arr)          # [list([1, 2, 3]) list([4, 5])]
print(arr.shape)    # (2,) — not (2, 3)!
```

**Why it happens:** NumPy cannot infer a consistent shape from a ragged nested sequence, so it falls back to an array of Python objects — losing all performance benefits.

**Symptoms:** The array has `dtype=object` and shape `(n,)` instead of a meaningful 2D shape. Operations like `.sum()` and `.mean()` do not work element-wise.

**Fix:** Ensure all inner sequences have the same length. Pad shorter lists with `np.nan` or a sentinel value, or use a list comprehension to normalize lengths before creating the array.
