## Why This Matters

Every numerical operation in AI engineering — from simple feature scaling to training loop internals — runs on arrays of numbers. Python lists are too slow for this: each element is a Python object, iteration happens at Python bytecode speed, and there is no support for element-wise math without explicit loops.

NumPy solves this with **ndarrays**: homogeneous, contiguous blocks of memory that the CPU can process in tight C loops. A single NumPy operation like `arr * 2` runs at compiled speed, transforming every element in one call. This is the foundation that pandas, scikit-learn, PyTorch, and TensorFlow all build on.

## The Problem

You have 10,000 model prediction scores and a confidence threshold. You want to filter scores above the threshold, square them (to penalize low-confidence predictions asymmetrically), and compute the mean of the result. In pure Python:

```python
scores = [0.12, 0.87, 0.45, 0.92, 0.33, 0.78]
threshold = 0.5

result = []
for s in scores:
    if s > threshold:
        result.append(s ** 2)

mean = sum(result) / len(result)  # (0.7569 + 0.8464 + 0.6084) / 3 = 0.7372...
```

Six lines for a simple pipeline. With NumPy it is one expression, faster and clearer.

## The Concept

A NumPy array is a grid of values of the same type. The key difference from a Python list: everything in the array is stored as raw C primitives in a single memory block, and operations apply to all elements at once — you never write a loop over individual elements.

```python
import numpy as np

scores = np.array([0.12, 0.87, 0.45, 0.92, 0.33, 0.78])
threshold = 0.5

mean = scores[scores > threshold] ** 2  # Filter, then square, all at C speed
print(f"{mean:.4f}")  # 0.7372
```

The expression `scores > threshold` produces a boolean array. Using it as an index selects only the `True` positions. Squaring applies element-wise. The entire pipeline is one line of NumPy, and it runs 10–50x faster than the loop equivalent on large data.
