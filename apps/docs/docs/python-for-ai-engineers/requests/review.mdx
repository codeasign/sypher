# Requests — Review

⏱ 3 minutes

## Key Takeaways

- Use `requests.get()` for GET requests, `requests.post(json=...)` for POST with JSON bodies.
- Always set a `timeout` — without it, requests can hang indefinitely.
- Always call `response.raise_for_status()` to catch HTTP errors.
- Use `requests.Session()` with default headers for multiple requests to the same server — connection reuse is faster.
- Never hardcode API keys — use environment variables.

## Recap in 60 Seconds

If you only remember three things:

1. `session.post(url, json=payload, timeout=10)` then `response.raise_for_status()` then `response.json()` — the three-step pattern for every API call.
2. Use a `Session` object for multiple requests to avoid repeated TLS handshakes.
3. Always set `timeout` and always check `response.ok` or call `raise_for_status()`.
