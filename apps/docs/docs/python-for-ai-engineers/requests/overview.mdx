# Requests

⏱ 10 minutes · Difficulty: Intermediate

## What You Will Learn

By the end of this lesson you will be able to:

- Send GET and POST requests with the `requests` library
- Set headers, query parameters, and JSON bodies
- Handle HTTP response status codes and errors
- Use sessions for connection reuse and persistent headers
- Build an API client for an AI model provider

## Prerequisites

You should already understand:

- [Dictionaries](../dictionaries/overview) — request parameters and response data are dicts
- [JSON](../json/overview) — API payloads and responses use JSON
- [Error Handling](../error-handling/overview) — HTTP errors must be caught and handled

## Why This Matters

AI models are primarily accessed through HTTP APIs. Every inference call to OpenAI, Anthropic, Groq, or any model provider is an HTTP request — you send a JSON payload with your prompt and parameters, and you receive a JSON response with the model's output.

The `requests` library is the standard tool for this in Python. It handles connection pooling, timeout management, authentication headers, error responses, and streaming — all the details you need for production API clients. Without it, you would be writing raw socket code.

## The Problem

You need to call an LLM API: send a prompt, get a completion. With Python's built-in `urllib.request`, a simple API call requires multiple lines of boilerplate for encoding the body, setting headers, handling the response, and decoding the result. `requests` reduces the entire interaction to a single function call.

## The Concept

An HTTP request has four parts:

1. **Method** — what you want to do: GET (retrieve), POST (create), PUT (update), DELETE (remove)
2. **URL** — where you are sending the request
3. **Headers** — metadata about the request (content type, authentication)
4. **Body** — data sent with POST, PUT, and PATCH requests

The response has three parts:

1. **Status code** — 200 OK, 201 Created, 400 Bad Request, 401 Unauthorized, 500 Server Error
2. **Headers** — metadata about the response
3. **Body** — the data the server returned
