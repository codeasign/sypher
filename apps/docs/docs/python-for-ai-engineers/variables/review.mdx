# Variables — Review

⏱ 3 minutes

## Key Takeaways

- Variables are labels on objects, not boxes — they reference values in memory and have no fixed type.
- Python's core data types are `int`, `float`, `str`, `bool`, and `NoneType`; use `type()` to discover a value's type at runtime.
- Convert between types explicitly with `int()`, `float()`, `str()`, and `bool()` — never rely on implicit coercion.
- Follow PEP 8 naming: `snake_case` for variables, `UPPER_CASE` for constants, avoid shadowing built-ins.
- Use `==` for value equality, `is` only for `None` checks.

## Recap in 60 Seconds

If you only remember three things:

1. Python uses dynamic typing — the variable has no type, only the object does. `type()` tells you what you are working with.
2. Name variables clearly with `snake_case`, never shadow built-ins like `list` or `str`, and keep each variable's type stable within a function.
3. Use `==` to compare values, `is` only to check `None`. Integer division with `/` returns a `float` — use `//` for floor division.
