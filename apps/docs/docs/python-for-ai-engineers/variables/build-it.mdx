# Variables and Data Types — Hands On

⏱ 12 minutes

## Breaking It Down

### Step 1: Core Data Types

Create `types_demo.py`:

```python
# types_demo.py — Demonstrates every core data type

# Integer: whole numbers
count = 42
temperature = -5

# Float: decimal numbers
pi = 3.14159
learning_rate = 0.001

# String: text — single or double quotes
name = "AI Engineer"
greeting = 'Hello'

# Boolean: True or False
is_ready = True
has_error = False

# NoneType: represents "no value"
result = None

# Check the type of any value with type()
print(type(count))         # <class 'int'>
print(type(pi))            # <class 'float'>
print(type(name))          # <class 'str'>
print(type(is_ready))      # <class 'bool'>
print(type(result))        # <class 'NoneType'>
```

### Step 2: Dynamic Typing in Action

The same variable can hold different types at different times:

```python
# dynamic_demo.py
value = 42
print(type(value))  # <class 'int'>

value = 3.14
print(type(value))  # <class 'float'>

value = "now a string"
print(type(value))  # <class 'str'>
```

This is legal but rarely a good idea in real code. In practice, a variable should hold one logical type throughout its lifetime — Python will not enforce it, but your teammates will.

### Step 3: Type Conversion

Explicit conversion between types — also called **casting**:

```python
# casting_demo.py

# String to integer
age_str = "25"
age = int(age_str)
print(age + 1)  # 26 — works because age is now an int

# Integer to string
score = 98
message = "Score: " + str(score)
print(message)  # Score: 98

# String to float
rate = float("0.85")
print(rate * 100)  # 85.0

# Anything to boolean
print(bool(1))      # True
print(bool(0))      # False
print(bool(""))     # False — empty string is falsy
print(bool("hi"))   # True — non-empty string is truthy
print(bool(None))   # False — None is always falsy
```

Conversion rules:

- `int("25")` — works if the string is a valid integer literal
- `int("25.5")` — raises `ValueError`; use `float()` then `int()` to truncate: `int(float("25.5"))` → `25`
- `int(True)` → `1`; `int(False)` → `0`

### Step 4: Multiple Assignment

Python lets you assign multiple variables in one line:

```python
# multiple_assignment.py

# Same value to multiple variables
x = y = z = 0
# x, y, z are all 0

# Different values in one line
name, age, active = "Alice", 30, True
# name = "Alice", age = 30, active = True

# Swapping two variables — no temporary variable needed
a, b = 5, 10
a, b = b, a
# a = 10, b = 5
```

The swap works because Python evaluates the right-hand side first (producing a tuple `(10, 5)`), then unpacks it into `a` and `b`.

### Step 5: Naming Conventions (PEP 8)

Python has strong conventions for variable naming. These are not enforced by the compiler but are universally followed:

```python
# ✅ Good — snake_case for variables and functions
model_name = "gpt-4"
batch_size = 32
learning_rate = 0.001

# ✅ Good — UPPER_CASE for constants
MAX_RETRIES = 3
API_ENDPOINT = "https://api.example.com"

# ✅ Good — leading underscore for internal / private use
_internal_cache = {}

# ❌ Bad — camelCase (used in JavaScript but not Python)
modelName = "gpt-4"      # Avoid

# ❌ Bad — single-letter names (except in loops)
x = "some value"          # Avoid — what is x?

# ❌ Bad — names that shadow built-ins
list = [1, 2, 3]          # Avoid — overrides the list() built-in
```

### Step 6: Checking Type in Practice

When processing AI data, you often need to verify types at runtime:

```python
# type_checking.py

def process_prediction(value):
    """Process a model prediction, handling different input types."""
    if value is None:
        return 0.0  # No prediction available
    if isinstance(value, bool):
        return 1.0 if value else 0.0
    if isinstance(value, (int, float)):
        return float(value)
    if isinstance(value, str):
        try:
            return float(value)
        except ValueError:
            return 0.0
    return 0.0

print(process_prediction(0.87))           # 0.87
print(process_prediction(None))           # 0.0
print(process_prediction(True))           # 1.0
print(process_prediction("0.95"))         # 0.95
print(process_prediction("invalid"))      # 0.0
```
