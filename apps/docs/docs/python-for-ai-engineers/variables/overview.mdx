# Variables and Data Types

⏱ 10 minutes · Difficulty: Beginner

By the end of this lesson you will be able to:

- Use type conversion functions like `int()`, `float()`, `str()`, `bool()`
- Follow Python naming conventions for variables
- Understand how variables reference objects in memory

## Why This Matters

Every program manipulates data — numbers, text, flags, lists. Variables are how you name and store that data so you can refer to it later. In AI engineering, you will assign tensors to variables, store configuration in dictionaries, pass model outputs between processing stages, and track state through long pipelines. Understanding how Python's variables actually work — that they are references to objects, not boxes that hold values — prevents a class of bugs that trip up developers coming from statically-typed languages.

The way Python handles types is fundamentally different from languages like Java or C++. Python uses **dynamic typing**: a variable can hold a string at one point and an integer later. This flexibility speeds up development but requires discipline to avoid confusing yourself and your teammates. Every AI library you use — PyTorch, TensorFlow, scikit-learn — relies on clear, well-named variables to keep complex pipelines readable.

## The Problem

You write `count = 5` and later `count = "five"`. The code runs without complaint. But downstream code that expected a number now receives a string, producing a `TypeError` at runtime. Unlike statically-typed languages that catch this at compile time, Python only discovers the error when the offending line executes. Without understanding that variables are just **names bound to objects**, you cannot predict when this will happen or how to guard against it.

## The Concept

Think of a variable as a **label** you stick on an object. The object lives in memory; the label is how you refer to it. You can stick multiple labels on the same object, or take a label off one object and put it on another. The object itself does not care what label you use.

When you write `x = 5`, Python:

1. Creates an integer object `5` in memory.
2. Binds the name `x` to that object.

When you then write `x = "hello"`, Python:

1. Creates a string object `"hello"` in memory.
2. Re-binds the name `x` to the new object.
3. The old integer `5` is now unreferenced and will be cleaned up by garbage collection.

This is called **dynamic typing** — the variable `x` has no type itself; only the object it currently references has a type.
