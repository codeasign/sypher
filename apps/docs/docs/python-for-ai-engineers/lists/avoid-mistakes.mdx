# Lists — Common Mistakes

⏱ 6 minutes

## When NOT to Use Lists

Lists are versatile, but not always the right choice:

- **Fast membership checks:** Use a `set` — `x in my_set` is O(1) versus O(n) for a list.
- **Key-value lookups:** Use a `dict` — lists store values by position, dicts by key.
- **Fixed-size, same-type sequences:** Use a `tuple` — immutable, slightly faster, and documents that the data should not change.
- **Large numerical data:** Use `numpy.ndarray` — lists store Python objects (8+ bytes each), arrays store raw C values (4-8 bytes each).

## The Pattern to Remember

- **Pattern:** `append()` to grow, index/slice to access, `for item in list` to iterate, `.copy()` to duplicate.
- **When to use:** Ordered collections that grow, shrink, or change over time.
- **When NOT to use:** Unordered collections (use set), lookups by key (use dict), large numerical arrays (use numpy).
- **Mental model:** A list is a mutable train of items. Each item has a position. You can add, remove, or change carriages.

## Common Mistakes

### Mistake 1: Modifying a List While Iterating Over It

```python
# Bug: removes only every other element
numbers = [1, 2, 3, 4, 5]
for n in numbers:
    if n % 2 == 0:
        numbers.remove(n)
print(numbers)  # [1, 3, 5] — works here by luck, fails in general
```

**Why it happens:** When you remove an element during iteration, the indices shift. The loop moves to the next index but the element that shifted into the current index is skipped.

**Symptoms:** Elements are missed — some items that should be removed stay in the list, or some items that should stay are incorrectly skipped.

**Fix:** Iterate over a copy:

```python
numbers = [1, 2, 3, 4, 5]
for n in numbers.copy():
    if n % 2 == 0:
        numbers.remove(n)
```

Or use a list comprehension to build a new list:

```python
numbers = [n for n in [1, 2, 3, 4, 5] if n % 2 != 0]
```

### Mistake 2: Confusing `append()` and `extend()`

```python
a = [1, 2]
a.append([3, 4])
print(a)  # [1, 2, [3, 4]] — nested list added as one element

b = [1, 2]
b.extend([3, 4])
print(b)  # [1, 2, 3, 4] — elements added individually
```

**Why it happens:** Both add items to the end, but `append()` adds its argument as a single element, while `extend()` iterates over its argument and adds each element separately.

**Symptoms:** You wind up with nested lists when you expected a flat list, or vice versa.

**Fix:** Use `append()` when adding one item. Use `extend()` or `+=` when merging another collection.

### Mistake 3: Using a List as a Default Argument

```python
def add_item(item, cache=[]):  # ❌ Dangerous default
    cache.append(item)
    return cache

print(add_item("a"))  # ['a']
print(add_item("b"))  # ['a', 'b'] — Not what you expect!
```

**Why it happens:** Default arguments are evaluated once at function definition time, not each time the function is called. All calls share the same list object.

**Symptoms:** The default value accumulates state across calls. Each call appends to the same list from previous calls.

**Fix:** Use `None` as the default and create a new list inside the function:

```python
def add_item(item, cache=None):
    if cache is None:
        cache = []
    cache.append(item)
    return cache
```

### Mistake 4: Index Out of Range

```python
items = [1, 2, 3]
print(items[3])  # IndexError: list index out of range
```

**Why it happens:** Indices start at 0, so a list of length 3 has valid indices 0, 1, 2.

**Symptoms:** `IndexError` at runtime.

**Fix:** Always check the length before accessing an index, or use `.append()` and process the last element with `items[-1]` instead of tracking indices:

```python
if len(items) > 3:
    print(items[3])
```

### Mistake 5: Using `=` Instead of `.copy()` When Duplicating

```python
original = [1, 2, 3]
copied = original        # ❌ This is a reference, not a copy
copied.append(4)
print(original)          # [1, 2, 3, 4] — original changed
```

**Why it happens:** You think `=` creates a new list. It creates a new reference to the same list object.

**Symptoms:** Modifying what you thought was a copy also changes the original.

**Fix:** Use `original.copy()`, `list(original)`, or `original[:]` to create a shallow copy. Use `copy.deepcopy()` for nested lists.
