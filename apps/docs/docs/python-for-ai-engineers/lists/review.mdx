# Lists — Review

⏱ 3 minutes

## Key Takeaways

- Lists are ordered, mutable sequences — use `append()` to grow, `pop()` to shrink, `[i]` to access.
- Slicing (`list[start:stop:step]`) uses exclusive stop — same indexing convention as strings.
- Assignment (`=`) creates a reference, not a copy — use `.copy()` or `list()` for shallow copies.
- Do not modify a list while iterating over it — iterate over a copy instead.
- Never use a mutable list as a function default argument — use `None` and create a new list inside the function.

## Recap in 60 Seconds

If you only remember three things:

1. `append()` to add one item, `extend()` to merge another list, `pop()` to remove and return the last item.
2. Assignment creates a reference — always copy intentionally with `list.copy()` or `list(original)`.
3. List membership (`x in list`) is O(n); membership checks in a set are O(1) — choose accordingly.
