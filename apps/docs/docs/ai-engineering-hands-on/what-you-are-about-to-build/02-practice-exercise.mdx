# Build the AI Research Engine

⏱ 20 minutes · Difficulty: Beginner

This is the exercise where you build your first AI-powered application. By the end, you will have a working command-line research assistant that calls a real LLM API, displays intelligent answers, and saves your conversation history.

## Prerequisites

Before starting, make sure you have:

- Python 3.10 or later installed (check with `python --version`)
- An Anthropic API key (sign up at console.anthropic.com)
- A text editor or IDE (VS Code recommended)
- An internet connection (the script calls a remote API)

## Setup

Create a dedicated folder for your AI Research Engine project. This keeps everything organized and makes it easy to find later.

```bash
mkdir ai-research-engine
cd ai-research-engine
```

Create and activate a virtual environment to isolate your project dependencies:

```bash
python -m venv venv
```

This creates a folder called `venv` inside your project directory that contains its own Python interpreter and package directory. Any packages you install while the virtual environment is active will go into this folder instead of your system-wide Python installation.

On macOS and Linux:

```bash
source venv/bin/activate
```

On Windows:

```bash
venv\Scripts\activate
```

After activation, your terminal prompt should show `(venv)` at the beginning, like this:

```
(venv) ai-research-engine $
```

This tells you the virtual environment is active. If you close your terminal and come back later, you need to reactivate it with the same command.

Install the Anthropic SDK — this is the Python library that handles communication with the API:

```bash
pip install anthropic
```

Set your API key as an environment variable. Replace `your-key-here` with the actual key from your Anthropic console.

On macOS and Linux:

```bash
export ANTHROPIC_API_KEY="sk-ant-your-actual-api-key-here"
```

On Windows PowerShell:

```powershell
$env:ANTHROPIC_API_KEY = "sk-ant-your-actual-api-key-here"
```

On Windows Command Prompt:

```cmd
set ANTHROPIC_API_KEY=sk-ant-your-actual-api-key-here
```

Verify the key is set correctly:

```bash
echo $ANTHROPIC_API_KEY
```

You should see the key string printed. If it shows nothing, the environment variable is not set.

## The Build

Follow these steps in order. Each step builds on the previous one. The final result is a complete, working AI Research Engine.

### Step 1: Create the Script File

Create a new file called `research.py` in your project folder. This will hold all the code for your AI Research Engine.

Open your editor and create `ai-research-engine/research.py`.

### Step 2: Write the Import and API Key Setup

Add these lines to the top of `research.py`:

```python
import os
import anthropic

api_key = os.environ["ANTHROPIC_API_KEY"]
client = anthropic.Anthropic(api_key=api_key)
```

**What this does:** The `os` module gives your script access to the operating system. The `anthropic` module is the SDK you installed. The script reads your API key from the environment variable and creates a client object that handles all communication with the API.

**Expected behavior:** Running just these lines will not produce any output yet, but if the API key is missing, Python will raise a `KeyError` with the message `'ANTHROPIC_API_KEY'`.

### Step 3: Write the Question-Prompting Function

Add this function after the client setup:

```python
def get_question():
    print("AI Research Engine")
    print("Ask any question (type 'quit' to exit)")
    print("-" * 40)
    return input("Your question: ")
```

**What this does:** This function displays a header and prompts for a question. The `input()` function reads text from the terminal and returns it as a string.

**Expected output when tested:**

```
AI Research Engine
Ask any question (type 'quit' to exit)
----------------------------------------
Your question:
```

The program will wait for you to type something and press Enter.

### Step 4: Write the API-Calling Function

Add this function to send the question to the LLM:

```python
def ask_ai(question):
    message = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=1024,
        system="You are a helpful research assistant. Answer clearly and accurately.",
        messages=[
            {"role": "user", "content": question}
        ]
    )
    return message.content[0].text
```

**What this does:** This function constructs a request with your question and sends it to the Anthropic API. The `model` parameter selects the Claude Sonnet model. The `max_tokens` parameter limits the response length. The `system` parameter sets the AI's behavior. The function returns the text of the response.

**Expected behavior:** This function will work only when called with a valid question string and a properly set API key. If the API key is invalid, it raises an `anthropic.AuthenticationError`.

### Step 5: Write the Conversation-Logging Function

Add this function to save exchanges to a file:

```python
def save_exchange(question, answer):
    with open("research_log.txt", "a", encoding="utf-8") as f:
        f.write(f"Q: {question}\n")
        f.write(f"A: {answer}\n")
        f.write("-" * 60 + "\n")
```

**What this does:** This function opens (or creates) a file called `research_log.txt` in append mode and writes the question-answer pair separated by a line of dashes. The `"a"` mode means each call adds new content without overwriting previous entries.

**Expected behavior:** The first call creates `research_log.txt` in the project folder. Each subsequent call appends to the same file. If the file cannot be written (permissions issue), Python raises a `PermissionError`.

### Step 6: Write the Main Loop

Add the main program loop that ties everything together:

```python
def main():
    while True:
        question = get_question()

        if question.lower() == "quit":
            print("Research log saved to research_log.txt")
            print("Goodbye!")
            break

        if question.strip() == "":
            print("Please type a question.")
            continue

        print("\nAsking the AI Research Engine...\n")
        try:
            answer = ask_ai(question)
            print(f"Answer:\n{answer}\n")
            save_exchange(question, answer)
        except Exception as e:
            print(f"An error occurred: {e}")
            print("Check your API key and internet connection.\n")

if __name__ == "__main__":
    main()
```

**What this does:** The `while True` loop keeps asking for questions until you type `quit`. Inside the loop:
1. It calls `get_question()` to prompt for input
2. It checks for the `quit` command and exits gracefully
3. It checks for empty input and prompts again
4. It calls `ask_ai()` to get a response from the LLM
5. It prints the answer
6. It saves the exchange to the log file
7. If anything goes wrong, it catches the error and shows a helpful message

**Expected behavior of the complete program:**

```
$ python research.py

AI Research Engine
Ask any question (type 'quit' to exit)
----------------------------------------
Your question: What is the tallest mountain on Earth?

Asking the AI Research Engine...

Answer:
The tallest mountain on Earth is Mount Everest, which reaches
29,031.7 feet (8,848.86 meters) above sea level. It is located
in the Himalayan mountain range on the border between Nepal and
the Tibet Autonomous Region of China.

However, if you measure from base to peak rather than sea level,
Mauna Kea in Hawaii is taller at over 33,500 feet (10,210 meters)
from its base on the ocean floor to its summit.

---

Ask another question (type 'quit' to exit)
----------------------------------------
Your question: quit

Research log saved to research_log.txt
Goodbye!
```

### Step 7: Check Your Research Log

Open the file `research_log.txt` to confirm your conversation was saved:

```bash
cat research_log.txt
```

You should see your question and answer written in the file with a line of dashes separating each exchange.

**Expected output:**

```
Q: What is the tallest mountain on Earth?
A: The tallest mountain on Earth is Mount Everest...
------------------------------------------------------------
```

### Step 8: Run It Again and Ask a Follow-Up

Run the program again and ask a related question. Notice that each session appends to the same log file, building a permanent research record.

```bash
python research.py
```

Type: "How was Mount Everest formed?"

Exit the program and check the log again. You now have two entries in your research log, showing a mini research thread on the topic of mountains.

## Common Errors

Here are the most common issues you will encounter and how to fix them.

### Error 1: ModuleNotFoundError

```
ModuleNotFoundError: No module named 'anthropic'
```

**Cause:** You did not install the Anthropic SDK, or you installed it outside your virtual environment.

**Fix:** Activate your virtual environment and run:

```bash
pip install anthropic
```

Verify the installation:

```bash
pip list | grep anthropic
```

### Error 2: KeyError

```
KeyError: 'ANTHROPIC_API_KEY'
```

**Cause:** The environment variable `ANTHROPIC_API_KEY` is not set in your current terminal session.

**Fix:** Set the environment variable (see the Setup section above). Remember that environment variables are session-specific — if you open a new terminal window, you need to set it again.

```bash
export ANTHROPIC_API_KEY="sk-ant-your-actual-key-here"
```

### Error 3: AuthenticationError

```
anthropic.AuthenticationError: ...
```

**Cause:** Your API key is invalid, expired, or mistyped.

**Fix:** Check that the key is correct by printing it:

```bash
echo $ANTHROPIC_API_KEY
```

Compare it against the key in your Anthropic console. If it is wrong, set it again. If it has been exposed, generate a new key in the console.

### Error 4: APIConnectionError

```
anthropic.APIConnectionError: ...
```

**Cause:** Your internet connection is down, or the API endpoint is unreachable.

**Fix:** Check your internet connection:

```bash
ping api.anthropic.com
```

If the ping fails, fix your connection and try again. If the ping works, the API may be temporarily down — wait a moment and retry.

### Error 5: PermissionError

```
PermissionError: [Errno 13] Permission denied: 'research_log.txt'
```

**Cause:** Your script does not have permission to write to the current directory.

**Fix:** Make sure you are running the script from a directory where you have write access. Your home directory or `Documents` folder always works. Avoid system directories like `/` or `C:\Program Files`.

### Error 6: IndentationError

```
IndentationError: expected an indented block
```

**Cause:** Python uses indentation (4 spaces by convention) to define blocks of code. If your indentation is inconsistent or missing, the script will not run.

**Fix:** Use the exact indentation shown in the code examples. Configure your editor to use 4 spaces for Python files, and never mix spaces and tabs.

### Error 7: AttributeError

```
AttributeError: 'NoneType' object has no attribute 'content'
```

**Cause:** The API returned a response that did not contain the expected content structure. This can happen if the model was unavailable, the request was malformed, or a newer SDK version changed the response format.

**Fix:** Add error handling around the API call. Check that `message.content` exists before accessing it. Verify you are using a compatible version of the SDK:

```bash
pip install --upgrade anthropic
```

## What You Just Built

You built a working AI Research Engine. Here is what your application does:

- Accepts any question in plain English
- Sends it securely to a state-of-the-art LLM via an API
- Returns an intelligent, contextual answer in seconds
- Saves every exchange to a searchable log file
- Handles errors gracefully without crashing

This is the same fundamental architecture used by ChatGPT, Claude.ai, Perplexity, and thousands of AI-powered products. The components are the same: a client application (your script), an API endpoint, a large language model, and a data store (your log file).

- Extract structured data like JSON and tables from responses

- Let the AI search documents and the web before answering
- Give the AI tools it can use — running code, looking up facts
- Build multi-agent systems where multiple AIs collaborate
