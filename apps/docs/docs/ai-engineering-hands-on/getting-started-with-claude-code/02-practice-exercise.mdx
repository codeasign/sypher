This section walks you through installing Claude Code, running your first interactive session, and mastering the essential commands and flags. Each step builds on the previous one, so follow them in order.

## Setup

Before you start this exercise, make sure you have:

- **Node.js 18 or higher** installed (`node --version`)
- **npm** available (`npm --version`)
- **An Anthropic account** with Claude Code access. Sign up at [console.anthropic.com](https://console.anthropic.com)
- **A small project to practice on.** If you do not have one, create a minimal project:

```bash
mkdir claude-code-practice
cd claude-code-practice
npm init -y
echo "console.log('hello world');" > index.js
```

### Install Claude Code

Open a terminal and run:

```bash
npm install -g @anthropic-ai/claude-code
```

### Verify Installation

```bash
claude --version
```

Expected output (version may vary):

```
@anthropic-ai/claude-code/2.1.200 (win32 x64)
```

### First Launch Authentication

The first time you run `claude`, it will prompt you to authenticate:

```bash
claude
```

A browser window opens asking you to log in to your Anthropic account and authorize the CLI. Complete the authorization, then return to the terminal. You should see the Claude Code prompt (`▶`).

If you prefer to use an API key instead of OAuth, set the environment variable:

```bash
export ANTHROPIC_API_KEY="sk-ant-your-key-here"
```

Then launch Claude Code again. It will use the API key for authentication.

### Create a Practice Project

Create a small project to use during the exercises:

```bash
mkdir -p /tmp/claude-code-practice
cd /tmp/claude-code-practice
git init
echo "node_modules/" > .gitignore
echo "node_modules/" > .claudeignore
```

Create a simple Python file to work with:

```python
# calculator.py
def add(a, b):
    return a + b

def subtract(a, b):
    return a - b

def multiply(a, b):
    return a * b

def divide(a, b):
    if b == 0:
        raise ValueError("Cannot divide by zero")
    return a / b
```

## The Build — Step by Step

### Step 1: Launch Claude Code Interactively

Start Claude Code in your practice project directory:

```bash
cd /tmp/claude-code-practice
claude
```

After authentication, you see the Claude Code prompt. The terminal may show a brief startup message with session information. At the `▶` prompt, type:

```
/status
```

Claude Code responds with information about the current git status, the directory you are in, and session details. This is useful for verifying what context Claude Code has loaded.

Expected output shows something like:

```
Git status: clean (main)
Working directory: /tmp/claude-code-practice
```

This confirms Claude Code is running and aware of your project. The `/status` command is one of the most useful slash commands — use it whenever you want to check what Claude Code knows about your current state.

### Step 2: Ask a Question About Your Code

At the `▶` prompt, type:

```
what does this project contain?
```

Claude Code reads the directory structure and responds with a summary of what it found. It reads the `calculator.py` file and `package.json` and describes the project.

Expected behavior: Claude Code lists the files in the directory (`calculator.py`, `index.js`, `package.json`) and describes what each file does. It may note that `calculator.py` has basic arithmetic functions and `index.js` is a minimal Node.js script.

This demonstrates the key capability: Claude Code reads your project into context and can answer questions about it without you needing to open files manually.

### Step 3: Ask Claude to Make a Code Change

Now ask Claude Code to add a new function. At the `▶` prompt, type:

```
Add a power function to calculator.py that raises a to the power of b. Add it after the divide function.
```

Claude Code reads `calculator.py`, plans the change, and presents a diff:

```
─── a/calculator.py
+++ b/calculator.py
@@ -9,3 +9,7 @@ def divide(a, b):
     if b == 0:
         raise ValueError("Cannot divide by zero")
     return a / b
+
+
+def power(a, b):
+    return a ** b
```

The approval prompt shows the file path, the diff, and waits for your input. Type **Y** to approve.

After approving, Claude Code applies the change and reports success. Verify the file was updated:

```bash
cat calculator.py
```

You should see the `power` function at the end of the file.

### Step 4: Run a One-Shot Prompt with `-p`

Exit the current session by typing:

```
/bye
```

Now use the `-p` flag to run a one-shot prompt without entering interactive mode:

```bash
claude -p "read calculator.py and tell me what functions it has"
```

Claude Code launches, reads the file, responds with the list of functions, and exits. This is useful for quick questions where you do not need an interactive session.

Expected output (approximate):

```
The calculator.py file contains the following functions:
- add(a, b): returns a + b
- subtract(a, b): returns a - b
- multiply(a, b): returns a * b
- divide(a, b): returns a / b, raises ValueError if b is zero
- power(a, b): returns a raised to the power of b
```

The `-p` flag costs one request and exits immediately. Use it for quick questions, code reviews, or integrating Claude Code into scripts.

### Step 5: Use the `--model` Flag

Run the same question with a different model:

```bash
claude --model opus -p "read calculator.py and suggest one improvement"
```

This runs the prompt using `claude-opus-4-20250514` instead of the default Sonnet. The response may be more detailed or thoughtful since Opus is optimized for deeper reasoning.

Compare the response with what you got from Sonnet. You may notice:

- Opus tends to give more thorough explanations
- Opus may suggest more architectural improvements
- Opus takes slightly longer to respond

To use the default Sonnet model explicitly:

```bash
claude --model sonnet -p "read calculator.py and suggest one improvement"
```

### Step 6: Resume a Session with `--resume`

Start an interactive session and build some context:

```bash
claude
```

At the `▶` prompt, type:

```
read calculator.py and explain the divide function to me
```

Claude Code reads the file and explains the divide function. Now exit the session with:

```
/bye
```

Now resume the session:

```bash
claude --resume
```

Claude Code reloads the conversation history from the previous session. You can ask a follow-up question that references the previous context:

```
what about the power function I added earlier?
```

Claude Code remembers the context from the previous session and can answer about the power function even though it was discussed before you exited. This is useful for long-running tasks where you need to pause and come back later.

### Step 7: Interrupt Claude Code Mid-Response

Start a new session:

```bash
claude
```

Ask a question that will generate a long response:

```
explain the entire history of version control systems in detail
```

As Claude Code starts generating its response, press **Ctrl+C** once. The response stops mid-stream. The `▶` prompt reappears.

Now type a different, shorter question:

```
what is git?
```

Claude Code responds to the new question, ignoring the interrupted request. This demonstrates how Ctrl+C lets you course-correct when Claude is heading in the wrong direction. Instead of waiting for the full response and then correcting, you stop mid-stream and redirect immediately. This saves time and reduces cost because you are not paying for a long response you will not use.

Press **Ctrl+C** twice to exit the session. The first Ctrl+C stops the current response. The second Ctrl+C exits. This double-press pattern is intentional — it prevents accidental exits.

### Step 8: Use the `--verbose` Flag

Run a one-shot prompt with verbose output:

```bash
claude --verbose -p "what is the current directory?"
```

With `--verbose`, Claude Code shows every tool call it makes. You see:

- The `Read` tool being called to check files
- The `Bash` tool being used to list directory contents
- The `Grep` tool being used for searches

This is useful for debugging why Claude Code is behaving in a certain way or for understanding the exact steps it takes to complete a task.

You can also try the `claude execute` command, which runs a shell command through Claude Code without entering an interactive session:

```bash
claude execute "node index.js"
```

This is a convenience shortcut that wraps `claude -p "run this command"` with simpler syntax. It is useful for running commands where you want Claude Code to handle any errors or issues that arise.

## Common Errors at This Stage

### Error 1: npm install fails with permission error

```
npm ERR! Error: EACCES: permission denied, mkdir '/usr/local/lib/node_modules/@anthropic-ai'
```

**Cause:** You do not have permission to write to the global npm directory.

**Fix:** Use a version manager like nvm, or install with sudo (on Linux/Mac):
```bash
sudo npm install -g @anthropic-ai/claude-code
```
On Windows, run the terminal as Administrator.

### Error 2: "claude: command not found"

```
bash: claude: command not found
```

**Cause:** The npm global install directory is not in your PATH.

**Fix:** Find the npm global directory and add it to your PATH:
```bash
npm root -g
# Example output: /usr/local/lib/node_modules
# The bin directory is one level up: /usr/local/bin
export PATH="/usr/local/bin:$PATH"
```
Add this to your `~/.bashrc` or `~/.zshrc` to make it permanent.

### Error 3: Authentication fails on first launch

```
Error: Authentication failed. Please try again.
```

**Cause:** The OAuth flow did not complete successfully, or the API key is invalid.

**Fix:** Try launching with an explicit API key:
```bash
export ANTHROPIC_API_KEY="sk-ant-your-key"
claude
```
If using OAuth, make sure you complete the browser authorization within the timeout window (typically 5 minutes). Clear any cached tokens and try again:
```bash
rm -rf ~/.claude/credentials
```

### Error 4: Rate limit exceeded during session

```
Rate limit exceeded. Please wait before sending more requests.
```

**Cause:** You hit the per-minute or per-day rate limit for your account tier.

**Fix:** Wait a few minutes and try again. Reduce the number of requests per session by combining multiple questions into one prompt. Check your rate limits at console.anthropic.com.

### Error 5: Session history too large

```
Warning: Session history is large. Consider clearing context with /clear.
```

**Cause:** Your conversation has grown very long, consuming context window space.

**Fix:** Use `/clear` to reset the conversation history while keeping the session alive. Claude Code will need to re-read files that were previously discussed, but the session token and cost tracking reset.

### Error 6: Model not available

```
Error: Model "claude-sonnet-3" is not available for Claude Code.
```

**Cause:** You specified a model that does not exist or is not available for Claude Code.

**Fix:** Use only the supported models:
- `claude-sonnet-4-20250514` (default)
- `claude-opus-4-20250514`

Check the latest model names in the Claude Code documentation. You can also run `claude --help` to see available model options.

- Knowledge of the `--verbose` flag for debugging tool execution
