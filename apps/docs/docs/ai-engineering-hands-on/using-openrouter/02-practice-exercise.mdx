Build a multi-provider AI client that can route requests across different LLM providers and gracefully handle failures. By the end of this exercise you will have a working Python application that uses OpenRouter to query models from different providers and fall back when one provider is unavailable.

## Setup

### Prerequisites

Before starting, make sure you have the following installed:

- Python 3.10 or later
- pip (Python package manager)
- An OpenRouter account with an API key

### Create a project directory

Open a terminal and create a new directory for this exercise:

```bash
mkdir openrouter-practice
cd openrouter-practice
```

### Create a virtual environment

Isolate your project dependencies:

```bash
python -m venv venv
source venv/bin/activate    # On Linux/macOS
```

On Windows:

```powershell
python -m venv venv
.\venv\Scripts\activate
```

### Install dependencies

Create a `requirements.txt` file:

```txt
requests>=2.31.0
python-dotenv>=1.0.0
```

Install the dependencies:

```bash
pip install -r requirements.txt
```

### Set up your API key

Create a `.env` file in your project directory:

```txt
OPENROUTER_API_KEY=sk-or-v1-your-key-here
```

Replace `sk-or-v1-your-key-here` with your actual OpenRouter API key. The `.env` file is for local development only -- never commit it to version control. Create a `.gitignore` file to keep it safe:

```txt
.env
venv/
__pycache__/
*.pyc
```

## The Build -- Step by Step

### Step 1: Test your API connection

Create a file called `test_connection.py` that verifies your API key and connection to OpenRouter:

```python
import os
import requests
from dotenv import load_dotenv

load_dotenv()

API_KEY = os.getenv("OPENROUTER_API_KEY")
if not API_KEY:
    raise ValueError("OPENROUTER_API_KEY not found in environment")

url = "https://openrouter.ai/api/v1/models"
headers = {"Authorization": f"Bearer {API_KEY}"}

response = requests.get(url, headers=headers, timeout=30)
response.raise_for_status()

data = response.json()
print(f"Connection successful. Found {len(data.get('data', []))} models.")
print(f"First model: {data['data'][0]['id']}")
```

Run the script:

```bash
python test_connection.py
```

**Expected output:**

```
Connection successful. Found 300+ models.
First model: openai/gpt-4o
```

The exact count will vary as OpenRouter adds new models. If you see an error, check your API key and internet connection.

**Checkpoint:** You have confirmed that your API key works and that you can reach the OpenRouter API. This is the foundation for every subsequent step.

### Step 2: Implement a simple chat completion function

Create a file called `chat.py` with a function that sends a message to any OpenRouter model:

```python
import os
import requests
from dotenv import load_dotenv

load_dotenv()

API_KEY = os.getenv("OPENROUTER_API_KEY")
BASE_URL = "https://openrouter.ai/api/v1"


def chat_completion(model: str, message: str, system_prompt: str = None) -> str:
    """Send a chat completion request to OpenRouter and return the response text."""
    messages = []
    if system_prompt:
        messages.append({"role": "system", "content": system_prompt})
    messages.append({"role": "user", "content": message})

    response = requests.post(
        url=f"{BASE_URL}/chat/completions",
        headers={
            "Authorization": f"Bearer {API_KEY}",
            "Content-Type": "application/json",
        },
        json={
            "model": model,
            "messages": messages,
            "max_tokens": 500,
        },
        timeout=30,
    )
    response.raise_for_status()
    data = response.json()
    return data["choices"][0]["message"]["content"]


if __name__ == "__main__":
    result = chat_completion(
        model="openai/gpt-4o",
        message="What is the capital of Japan?",
    )
    print(result)
```

Run it:

```bash
python chat.py
```

**Expected output:**

```
The capital of Japan is Tokyo.
```

**Explanation:** The function constructs a standard chat completions request, sends it to OpenRouter, and extracts the response text. The `model` parameter uses OpenRouter's `provider/model` slug format. OpenRouter routes this request to a provider that serves GPT-4o.

**Checkpoint:** You have a reusable function that can query any model through OpenRouter by changing the `model` argument.

### Step 4: Implement provider routing preferences

Create `routed_chat.py` that lets you control which provider handles your request. This is useful when you want to enforce a specific provider order for reliability or compliance:

```python
import os
import requests
from dotenv import load_dotenv

load_dotenv()

API_KEY = os.getenv("OPENROUTER_API_KEY")
BASE_URL = "https://openrouter.ai/api/v1"


def routed_completion(
    model: str,
    message: str,
    provider_order: list = None,
    allow_fallbacks: bool = True,
    max_price_prompt: float = None,
    max_price_completion: float = None,
) -> str:
    """Send a chat completion with explicit routing preferences."""
    messages = [{"role": "user", "content": message}]

    provider_config = {
        "allow_fallbacks": allow_fallbacks,
    }
    if provider_order:
        provider_config["order"] = provider_order
    if max_price_prompt is not None or max_price_completion is not None:
        provider_config["max_price"] = {}
        if max_price_prompt is not None:
            provider_config["max_price"]["prompt"] = max_price_prompt
        if max_price_completion is not None:
            provider_config["max_price"]["completion"] = max_price_completion

    response = requests.post(
        url=f"{BASE_URL}/chat/completions",
        headers={
            "Authorization": f"Bearer {API_KEY}",
            "Content-Type": "application/json",
        },
        json={
            "model": model,
            "messages": messages,
            "provider": provider_config,
            "max_tokens": 300,
        },
        timeout=30,
    )
    response.raise_for_status()
    data = response.json()
    return data["choices"][0]["message"]["content"]


if __name__ == "__main__":
    # Prefer Anthropic, fall back to OpenAI if Anthropic is unavailable
    result = routed_completion(
        model="anthropic/claude-sonnet-4-20250514",
        message="What are the benefits of API gateways?",
        provider_order=["Anthropic", "OpenAI"],
        max_price_prompt=3e-7,
        max_price_completion=2e-6,
    )
    print(result)
```

Run it:

```bash
python routed_chat.py
```

**Explanation:** The `provider` object in the request body tells OpenRouter exactly how to route. The `order` list specifies the exact priority of providers. The `max_price` fields cap the per-token cost. If no provider can serve the request within your constraints, OpenRouter returns an error rather than exceeding your budget.

**Checkpoint:** You can now control provider selection and enforce cost limits at the request level.

### Step 5: Build a robust fallback system

The most important feature for production reliability is automatic fallback across models and providers. Create `fallback_client.py`:

```python
import os
import time
import requests
from dotenv import load_dotenv

load_dotenv()

API_KEY = os.getenv("OPENROUTER_API_KEY")
BASE_URL = "https://openrouter.ai/api/v1"

FALLBACK_CHAIN = [
    {"model": "openai/gpt-4o", "provider_order": None},
    {"model": "anthropic/claude-sonnet-4-20250514", "provider_order": ["Anthropic", "Amazon Bedrock"]},
    {"model": "google/gemini-2.0-flash-001", "provider_order": ["Google", "AI21"]},
    {"model": "mistralai/mistral-small-2501", "provider_order": None},
]


def fallback_completion(message: str, system_prompt: str = None, max_retries: int = 2) -> dict:
    """Try each model in the fallback chain until one succeeds."""
    messages = []
    if system_prompt:
        messages.append({"role": "system", "content": system_prompt})
    messages.append({"role": "user", "content": message})

    for entry in FALLBACK_CHAIN:
        for attempt in range(max_retries):
            try:
                provider_config = {"allow_fallbacks": True}
                if entry["provider_order"]:
                    provider_config["order"] = entry["provider_order"]

                response = requests.post(
                    url=f"{BASE_URL}/chat/completions",
                    headers={
                        "Authorization": f"Bearer {API_KEY}",
                        "Content-Type": "application/json",
                    },
                    json={
                        "model": entry["model"],
                        "messages": messages,
                        "provider": provider_config,
                        "max_tokens": 500,
                    },
                    timeout=30,
                )
                response.raise_for_status()
                data = response.json()
                print(f"SUCCESS: {data['model']}")
                return data

            except requests.exceptions.RequestException as e:
                print(f"FAIL: {entry['model']} (attempt {attempt + 1}): {e}")
                time.sleep(1)

    raise RuntimeError("All models in fallback chain failed")


if __name__ == "__main__":
    result = fallback_completion(
        message="Write a haiku about API gateways.",
        system_prompt="You are a poet who writes about technology.",
    )
    print(f"\nResponse: {result['choices'][0]['message']['content']}")
    print(f"\nToken usage: {result['usage']}")
```

Run it:

```bash
python fallback_client.py
```

**Expected output:**

```
FAIL: openai/gpt-4o (attempt 1): 503 Server Error
FAIL: openai/gpt-4o (attempt 2): 503 Server Error
SUCCESS: anthropic/claude-sonnet-4-20250514

Response: Gateways bridge worlds,
Tokens flow through open doors,
Models find their path.

Token usage: {'prompt_tokens': 30, 'completion_tokens': 15, 'total_tokens': 45}
```

The exact output depends on which providers are available. If all providers are healthy, the first model succeeds immediately. The key point is that the client never fails as long as at least one model in the chain is available.

**Explanation:** The fallback chain iterates through a list of models. Each model gets up to `max_retries` attempts. If all attempts for one model fail (due to outage, rate limit, or timeout), it moves to the next model. This pattern provides resilience against both single-provider outages and transient network errors.

**Checkpoint:** You have a production-ready fallback client that can survive multi-provider outages.

Now combine everything into a complete multi-provider chat application. Create `multi_provider_chat.py`:

```python
import os
import sys
import time
import requests
from dotenv import load_dotenv

load_dotenv()

API_KEY = os.getenv("OPENROUTER_API_KEY")
BASE_URL = "https://openrouter.ai/api/v1"

FALLBACK_CHAIN = [
    {"model": "openai/gpt-4o", "label": "GPT-4o (OpenAI)"},
    {"model": "anthropic/claude-sonnet-4-20250514", "label": "Claude Sonnet 4 (Anthropic)"},
    {"model": "google/gemini-2.0-flash-001", "label": "Gemini 2.0 Flash (Google)"},
    {"model": "mistralai/mistral-small-2501", "label": "Mistral Small (Mistral)"},
]


def get_completion(messages: list) -> dict:
    for entry in FALLBACK_CHAIN:
        for attempt in range(2):
            try:
                response = requests.post(
                    url=f"{BASE_URL}/chat/completions",
                    headers={
                        "Authorization": f"Bearer {API_KEY}",
                        "Content-Type": "application/json",
                    },
                    json={
                        "model": entry["model"],
                        "messages": messages,
                        "provider": {"allow_fallbacks": True},
                        "max_tokens": 500,
                    },
                    timeout=30,
                )
                response.raise_for_status()
                data = response.json()
                print(f"Served by: {entry['label']}")
                return data
            except requests.exceptions.RequestException as e:
                print(f"  {entry['label']}: {e}")
                time.sleep(0.5)
    return None


def main():
    print("Multi-Provider Chat (type 'quit' to exit)")
    print("=" * 40)

    while True:
        user_input = input("\nYou: ").strip()
        if user_input.lower() in ("quit", "exit"):
            break

        messages = [{"role": "user", "content": user_input}]
        result = get_completion(messages)

        if result:
            print(f"\nAI: {result['choices'][0]['message']['content']}")
            usage = result.get("usage", {})
            print(f"\n[Tokens: {usage.get('total_tokens', 'N/A')}]")
        else:
            print("All providers unavailable. Please try again later.")


if __name__ == "__main__":
    main()
```

Run it:

```bash
python multi_provider_chat.py
```

Try typing a question and see which provider handles it. Type `quit` to exit.

```
Multi-Provider Chat (type 'quit' to exit)
========================================

Served by: GPT-4o (OpenAI)

AI: OpenRouter is a unified API gateway that provides access to multiple
large language model providers through a single interface, allowing you to
route requests to different models and providers based on your preferences.

[Tokens: 45]

You: quit
```

**Explanation:** This is a complete interactive application. It accepts user input, sends it through the fallback chain, and displays the response with metadata. The fallback mechanism ensures the application keeps working even if several providers are unavailable.

**Checkpoint:** You have a working multi-provider chat application. This is the same pattern used in production systems for high-availability AI applications.

## Common Errors

### Error 1: Authentication failure

**Error message:**

```
401 Client Error: Unauthorized for url: https://openrouter.ai/api/v1/chat/completions
```

**Cause:** The API key is missing, incorrect, or not being loaded from the environment.

**Fix:** Verify that your `.env` file contains the correct key and that you called `load_dotenv()` before accessing the environment variable. Check that the key starts with `sk-or-v1-`.

```bash
# Check that the key is loaded
python -c "from dotenv import load_dotenv; import os; load_dotenv(); print(os.getenv('OPENROUTER_API_KEY')[:15] + '...')"
```

### Error 2: Model not found

**Error message:**

```
404 Client Error: Not Found for url: https://openrouter.ai/api/v1/chat/completions
```

**Cause:** The model slug you specified does not exist on OpenRouter. Model names change frequently as providers update their offerings.

**Fix:** Query the models endpoint to find the correct slug:

```bash
curl -s "https://openrouter.ai/api/v1/models" \
  -H "Authorization: Bearer $OPENROUTER_API_KEY" | \
  python -c "import sys,json; data=json.load(sys.stdin); [print(m['id']) for m in data.get('data',[])]" | \
  grep -i "gpt-4"
```

### Error 3: Rate limit exceeded

**Error message:**

```
429 Client Error: Too Many Requests for url: https://openrouter.ai/api/v1/chat/completions
```

**Cause:** You have exceeded the rate limit for your account tier. Free tier accounts have 20 requests per minute and 50 requests per day.

**Fix:** Add a delay between requests or upgrade your account:

```python
import time
time.sleep(3)  # Wait 3 seconds between requests
```

### Error 4: Provider returned an error

**Error message:**

```
502 Server Error: Bad Gateway for url: https://openrouter.ai/api/v1/chat/completions
```

**Cause:** OpenRouter successfully received your request but the upstream provider returned an error. This could be a provider outage, a rate limit at the provider level, or a temporary issue.

**Fix:** This is exactly the scenario your fallback chain handles. The error is expected and normal -- your code should catch this exception and try the next model in the chain. Do not treat 502 and 503 errors as fatal.

### Error 5: Request timeout

**Error message:**

```
requests.exceptions.ConnectionError: HTTPSConnectionPool(host='openrouter.ai', port=443): Max retries exceeded
```

**Cause:** The request took longer than the configured timeout. This can happen with slow models or network congestion.

**Fix:** Increase the timeout or use a faster model suffix:

```python
# Use the :nitro suffix for the fastest provider
model="openai/gpt-4o:nitro"

# Increase timeout
response = requests.post(url, headers=headers, json=payload, timeout=60)
```
