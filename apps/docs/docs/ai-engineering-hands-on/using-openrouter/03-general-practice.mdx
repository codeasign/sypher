## Tier 1 -- Beginner

### Exercise 1: List available models

**Task:** Write a script that fetches the list of models from OpenRouter and prints only the model IDs that contain "gemini" in their name. Display the total count of matching models.

**Hint:** Use the `GET /models` endpoint. The response contains a `data` array. Each item has an `id` field. Filter with Python's `in` operator or a list comprehension.

**Solution:**

```python
import os
import requests
from dotenv import load_dotenv

load_dotenv()

API_KEY = os.getenv("OPENROUTER_API_KEY")
url = "https://openrouter.ai/api/v1/models"
headers = {"Authorization": f"Bearer {API_KEY}"}

response = requests.get(url, headers=headers, timeout=30)
data = response.json()

gemini_models = [m["id"] for m in data.get("data", []) if "gemini" in m["id"].lower()]

print(f"Found {len(gemini_models)} Gemini models:")
for model_id in gemini_models:
    print(f"  - {model_id}")
```

**Expected output (abbreviated):**

```
Found 5 Gemini models:
  - google/gemini-2.0-flash-001
  - google/gemini-2.0-flash-lite-001
  - google/gemini-2.0-pro-001
  - google/gemini-2.5-pro-001
  - google/gemini-2.5-flash-001
```

### Exercise 2: Extract response metadata

**Task:** Send a chat completion request and print the following metadata from the response: the model used, the total tokens consumed, and the finish reason. Enable the `X-OpenRouter-Metadata` header to get provider information.

**Hint:** Pass `extra_headers` to the request. The metadata header is `X-OpenRouter-Metadata` set to `"true"`. The provider information is returned in the response body.

**Solution:**

```python
import os
import requests
from dotenv import load_dotenv

load_dotenv()

API_KEY = os.getenv("OPENROUTER_API_KEY")
url = "https://openrouter.ai/api/v1/chat/completions"
headers = {
    "Authorization": f"Bearer {API_KEY}",
    "Content-Type": "application/json",
    "X-OpenRouter-Metadata": "true",
}

payload = {
    "model": "openai/gpt-4o",
    "messages": [{"role": "user", "content": "What is the speed of light?"}],
    "max_tokens": 100,
}

response = requests.post(url, headers=headers, json=payload, timeout=30)
data = response.json()

print(f"Model: {data['model']}")
print(f"Total tokens: {data['usage']['total_tokens']}")
print(f"Finish reason: {data['choices'][0]['finish_reason']}")
print(f"Provider: {data.get('provider', 'Not available')}")
```

### Exercise 3: Cost-aware request

**Task:** Send a chat completion request to `openai/gpt-4o:floor` (the cheapest provider) and print the estimated cost of the request. The pricing for each model is available from the `GET /models` endpoint. Calculate the cost as:

```
prompt_cost = prompt_tokens * float(pricing["prompt"])
completion_cost = completion_tokens * float(pricing["completion"])
total_cost = prompt_cost + completion_cost
```

**Hint:** First fetch the pricing for the model from the models endpoint, then use those prices to calculate the cost of your actual request.

**Solution:**

```python
import os
import requests
from dotenv import load_dotenv

load_dotenv()

API_KEY = os.getenv("OPENROUTER_API_KEY")
BASE_URL = "https://openrouter.ai/api/v1"
headers = {"Authorization": f"Bearer {API_KEY}"}

# Step 1: Get pricing for the model
model_id = "openai/gpt-4o"
models_resp = requests.get(f"{BASE_URL}/models", headers=headers, timeout=30)
models_data = models_resp.json()

pricing = None
for m in models_data.get("data", []):
    if m["id"] == model_id:
        pricing = m.get("pricing")
        break

if not pricing:
    print(f"Could not find pricing for {model_id}")
    exit(1)

print(f"Pricing: prompt=${pricing['prompt']}/token, completion=${pricing['completion']}/token")

# Step 2: Send a request
payload = {
    "model": f"{model_id}:floor",
    "messages": [{"role": "user", "content": "Explain quantum computing in one sentence."}],
    "max_tokens": 100,
}

response = requests.post(
    f"{BASE_URL}/chat/completions",
    headers={**headers, "Content-Type": "application/json"},
    json=payload,
    timeout=30,
)
data = response.json()

# Step 3: Calculate cost
usage = data["usage"]
prompt_cost = usage["prompt_tokens"] * float(pricing["prompt"])
completion_cost = usage["completion_tokens"] * float(pricing["completion"])
total_cost = prompt_cost + completion_cost

print(f"Prompt tokens: {usage['prompt_tokens']} (${prompt_cost:.8f})")
print(f"Completion tokens: {usage['completion_tokens']} (${completion_cost:.8f})")
print(f"Total cost: ${total_cost:.8f}")
print(f"Response: {data['choices'][0]['message']['content']}")
```

### Exercise 4: Simple streaming response

**Task:** Send a request with `stream=True` and print each token as it arrives. Use the OpenAI Python SDK with the OpenRouter base URL.

**Hint:** Set `base_url="https://openrouter.ai/api/v1"` when creating the OpenAI client. Use `stream=True` in the `create` call. Iterate over the stream and print each chunk's delta content.

**Solution:**

```python
import os
from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()

client = OpenAI(
    base_url="https://openrouter.ai/api/v1",
    api_key=os.getenv("OPENROUTER_API_KEY"),
)

stream = client.chat.completions.create(
    model="openai/gpt-4o",
    messages=[{"role": "user", "content": "Count from 1 to 10."}],
    stream=True,
    max_tokens=200,
)

print("Streaming response:")
for chunk in stream:
    if chunk.choices and chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="", flush=True)
print()
```

## Tier 2 -- Intermediate

### Exercise 5: Build a model comparison tool

**Task:** Create a script that sends the same prompt to three different models and saves the responses to a JSON file. Include the following metadata for each response: model name, provider, token usage, latency in seconds, and cost.

**Hint:** Use `time.time()` to measure latency before and after each request. Collect all responses into a list of dictionaries. Use `json.dump()` to write the results to a file.

**Solution:**

```python
import os
import json
import time
import requests
from dotenv import load_dotenv

load_dotenv()

API_KEY = os.getenv("OPENROUTER_API_KEY")
BASE_URL = "https://openrouter.ai/api/v1"
headers = {"Authorization": f"Bearer {API_KEY}"}

# Get pricing for cost calculation
models_resp = requests.get(f"{BASE_URL}/models", headers=headers, timeout=30)
models_data = models_resp.json()
pricing_map = {}
for m in models_data.get("data", []):
    pricing_map[m["id"]] = m.get("pricing", {})

# Fetch model pricing for cost calculation
models_to_test = [
    "openai/gpt-4o",
    "anthropic/claude-sonnet-4-20250514",
    "google/gemini-2.0-flash-001",
]

prompt = "Explain the concept of a distributed system in three sentences."

results = []

for model in models_to_test:
    print(f"Testing {model}...")
    start = time.time()

    response = requests.post(
        f"{BASE_URL}/chat/completions",
        headers={**headers, "Content-Type": "application/json"},
        json={
            "model": model,
            "messages": [{"role": "user", "content": prompt}],
            "max_tokens": 200,
            "provider": {"allow_fallbacks": False},
        },
        timeout=30,
    )

    elapsed = time.time() - start
    data = response.json()
    usage = data.get("usage", {})
    pricing = pricing_map.get(model, {})

    prompt_cost = usage.get("prompt_tokens", 0) * float(pricing.get("prompt", 0))
    completion_cost = usage.get("completion_tokens", 0) * float(pricing.get("completion", 0))

    results.append({
        "model": model,
        "response": data["choices"][0]["message"]["content"],
        "tokens": usage,
        "latency_seconds": round(elapsed, 2),
        "cost_usd": round(prompt_cost + completion_cost, 10),
    })

with open("model_comparison.json", "w") as f:
    json.dump(results, f, indent=2)

print(f"\nSaved results to model_comparison.json")
for r in results:
    print(f"  {r['model']}: {r['latency_seconds']}s, ${r['cost_usd']:.10f}")
```

### Exercise 6: Implement a cost-aware router

**Task:** Build a function that takes a user message and a budget in dollars, and automatically selects the cheapest model that can handle the request within budget. Use the `:floor` suffix and the `provider.max_price` field to enforce the budget.

**Hint:** Iterate through models from cheapest to most expensive. Fetch pricing from the models endpoint. Use `provider.max_price` to cap costs. Return the first model that fits within the budget.

**Solution:**

```python
import os
import requests
from dotenv import load_dotenv

load_dotenv()

API_KEY = os.getenv("OPENROUTER_API_KEY")
BASE_URL = "https://openrouter.ai/api/v1"
headers = {"Authorization": f"Bearer {API_KEY}"}


def get_models_sorted_by_price():
    """Fetch models and sort by prompt price (ascending)."""
    response = requests.get(f"{BASE_URL}/models", headers=headers, timeout=30)
    data = response.json()

    models = []
    for m in data.get("data", []):
        pricing = m.get("pricing", {})
        prompt_price = float(pricing.get("prompt", "inf"))
        models.append({
            "id": m["id"],
            "prompt_price": prompt_price,
            "completion_price": float(pricing.get("completion", "inf")),
        })

    # Sort by prompt price, ascending
    models.sort(key=lambda x: x["prompt_price"])
    return models


def cheapest_within_budget(message: str, max_budget_usd: float) -> str:
    """Find the cheapest model that can handle the request within budget."""
    models = get_models_sorted_by_price()

    for m in models:
        # Estimate 2x prompt tokens for safety margin
        estimated_prompt_tokens = len(message.split()) * 2
        estimated_cost = estimated_prompt_tokens * m["prompt_price"]

        if estimated_cost > max_budget_usd:
            continue

        try:
            payload = {
                "model": f"{m['id']}:floor",
                "messages": [{"role": "user", "content": message}],
                "max_tokens": 100,
                "provider": {
                    "max_price": {
                        "prompt": m["prompt_price"] * 1.1,
                        "completion": m["completion_price"] * 1.1,
                    }
                },
            }
            response = requests.post(
                f"{BASE_URL}/chat/completions",
                headers={**headers, "Content-Type": "application/json"},
                json=payload,
                timeout=30,
            )
            response.raise_for_status()
            data = response.json()
            return f"Model: {data['model']}\nResponse: {data['choices'][0]['message']['content']}"

        except requests.exceptions.RequestException:
            continue

    return "No model available within budget."


if __name__ == "__main__":
    result = cheapest_within_budget(
        message="What is Python?",
        max_budget_usd=0.0001,
    )
    print(result)
```

### Exercise 7: Zero-data-retention proxy

**Task:** Create a wrapper that routes all requests exclusively to providers that do not store request data. Use the `data_collection` and `zdr` provider settings. Verify that the request was routed correctly by checking the provider metadata.

**Hint:** Set `provider.data_collection` to `"deny"` and `provider.zdr` to `True`. Not all providers support zero data retention. Use the `X-OpenRouter-Metadata` header to see which provider handled the request.

**Solution:**

```python
import os
import requests
from dotenv import load_dotenv

load_dotenv()

API_KEY = os.getenv("OPENROUTER_API_KEY")
BASE_URL = "https://openrouter.ai/api/v1"


def zdr_completion(model: str, message: str) -> dict:
    """Send request with zero data retention requirements."""
    payload = {
        "model": model,
        "messages": [{"role": "user", "content": message}],
        "max_tokens": 200,
        "provider": {
            "data_collection": "deny",
            "zdr": True,
            "allow_fallbacks": True,
        },
    }

    response = requests.post(
        f"{BASE_URL}/chat/completions",
        headers={
            "Authorization": f"Bearer {API_KEY}",
            "Content-Type": "application/json",
            "X-OpenRouter-Metadata": "true",
        },
        json=payload,
        timeout=30,
    )
    response.raise_for_status()
    return response.json()


if __name__ == "__main__":
    result = zdr_completion(
        model="openai/gpt-4o",
        message="This is a test message with no data retention.",
    )
    print(f"Model: {result['model']}")
    print(f"Response: {result['choices'][0]['message']['content']}")
    print(f"Provider info: {result.get('provider', 'Not available')}")
    print(f"Token usage: {result['usage']}")
```

### Exercise 8: Provider-specific fallback with logging

**Task:** Build a client that logs every request attempt to a file named `openrouter.log`. Each log entry should include the timestamp, model attempted, provider, success/failure status, latency, and cost. Use the logging module.

**Hint:** Use Python's `logging` module to write to a file. Set up a `logging.FileHandler` with a formatter. Log before and after each request attempt.

**Solution:**

```python
import os
import time
import logging
import requests
from dotenv import load_dotenv

load_dotenv()

# Set up logging
logging.basicConfig(
    filename="openrouter.log",
    level=logging.INFO,
    format="%(asctime)s | %(levelname)s | %(message)s",
    datefmt="%Y-%m-%d %H:%M:%S",
)

API_KEY = os.getenv("OPENROUTER_API_KEY")
BASE_URL = "https://openrouter.ai/api/v1"

FALLBACK_MODELS = [
    "openai/gpt-4o",
    "anthropic/claude-sonnet-4-20250514",
    "google/gemini-2.0-flash-001",
]


def logged_completion(message: str) -> str:
    """Send a completion request with full logging."""
    for model in FALLBACK_MODELS:
        start = time.time()
        logging.info(f"Attempting model={model}")

        try:
            response = requests.post(
                f"{BASE_URL}/chat/completions",
                headers={
                    "Authorization": f"Bearer {API_KEY}",
                    "Content-Type": "application/json",
                },
                json={
                    "model": model,
                    "messages": [{"role": "user", "content": message}],
                    "max_tokens": 200,
                },
                timeout=30,
            )
            response.raise_for_status()
            data = response.json()
            elapsed = time.time() - start
            cost = response.headers.get("X-OpenRouter-Cost", "N/A")

            logging.info(
                f"SUCCESS model={model} "
                f"latency={elapsed:.2f}s "
                f"tokens={data['usage']['total_tokens']} "
                f"cost={cost}"
            )
            return data["choices"][0]["message"]["content"]

        except requests.exceptions.RequestException as e:
            elapsed = time.time() - start
            logging.error(f"FAILURE model={model} latency={elapsed:.2f}s error={e}")
            continue

    logging.critical("All models in fallback chain failed")
    return "Service unavailable."


if __name__ == "__main__":
    result = logged_completion("What is the tallest mountain in the world?")
    print(f"Result: {result}")
    print("\nCheck openrouter.log for details.")
```

### Exercise 9: Multi-provider model cascading

**Scenario:** You are building a two-stage AI pipeline. The first stage uses a cheap, fast model to extract keywords from user input. The second stage uses an expensive, high-quality model to generate a detailed response based on those keywords. The catch is that both stages must use different providers, and the second stage must fall back to a different model if the first choice is unavailable.

**Requirements:**
- Stage 1: Use `google/gemini-2.0-flash-lite` or similar cheap model to extract 3-5 keywords from the user's question
- Stage 2: Use `anthropic/claude-sonnet-4-20250514` to write a detailed paragraph using those keywords, with fallback to `openai/gpt-4o`
- Log the keywords extracted and the model used for each stage
- Handle errors gracefully at each stage

**Approach:** Build a pipeline function that calls the first model, parses the keyword output, then passes the keywords as context to the second model. Each stage has its own error handling and retry logic.

**Key considerations:**
- How do you parse the keywords from the first model's output reliably?
- What happens if the first stage succeeds but the second stage fails?
- Should you use different `max_tokens` values for each stage?
- How do you log intermediate results for debugging?

### Exercise 10: Cost-monitoring dashboard script

**Scenario:** You need to monitor your OpenRouter spending across multiple models and providers. Your manager wants a daily report showing total spend, spend per model, spend per provider, and average latency. You have access to the OpenRouter API and can log every request to a local database.

**Requirements:**
- Log every request to a SQLite database: timestamp, model, provider, tokens, cost, latency
- Write a summary function that queries the database and returns:
  - Total spend
  - Spend per model (sorted by most expensive)
  - Spend per provider
  - Average latency per model
- Output the summary as a formatted table

**Approach:** Use Python's `sqlite3` module (built-in) to create a local database. Create a `requests_log` table with columns for all metadata. Insert a row after every successful API call. Write a separate `report.py` script that queries the database and formats the output.

**Key considerations:**
- What database schema handles the metadata you need to query?
- How do you handle the case where cost information is not available in the response?
- What indexes would speed up the reporting queries?
- How do you prevent the database from growing unbounded?

### Exercise 11: Dynamic provider routing based on content type

**Scenario:** Your application handles three types of requests: simple factual queries, creative writing, and code generation. Each type has different quality and cost requirements. **Requirements:**
- Classify user input into one of three categories: `factual`, `creative`, `code`
- Route factual queries to the cheapest capable model
- Route creative writing to the best quality model (Anthropic preferred)
- Route code generation to a model with strong coding capabilities (OpenAI preferred)
- Each route must have at least one fallback model
- Use OpenRouter's provider routing features to enforce preferences

**Approach:** Build a classifier that detects the content type from the user's message. Use keywords or a simple heuristic for classification. Maintain a routing table that maps each content type to a primary and fallback model. Use OpenRouter's `provider.order` and `models` array to implement the routing.

**Key considerations:**
- How accurate does your classifier need to be? What are the consequences of misclassification?
- Should you allow the user to override the routing?
- How do you measure whether the routing is cost-effective?
- Could you use OpenRouter's `:floor` suffix for the factual route and `:nitro` for interactive routes?

### Exercise 12: Multi-region availability with latency optimization

**Scenario:** Your users are distributed across the US, Europe, and Asia. You want to route requests to the provider that offers the lowest latency for each region. Some providers have data centers in specific regions, and you need to respect data residency requirements.

**Requirements:**
- Detect the user's region from their IP address or a provided header
- Route US users to the lowest-latency US provider
- Route European users to a provider with GDPR-compliant data centers
- Route Asian users to a provider with Asian data centers
- Each region must have a fallback provider
- Log which region and provider handled each request

**Approach:** Use OpenRouter's provider routing to specify preferred providers per region. Since OpenRouter does not natively support geo-based routing, implement the geo-detection at the application level and select the appropriate model/provider configuration based on the detected region.

**Key considerations:**
- How do you determine which provider has the lowest latency for a given region without empirical testing?
- What happens if the preferred provider for a region is unavailable?
- How do you test latency differences between providers?
- Does OpenRouter's gateway add more latency for users far from their servers?
