## Setup

Before you start this exercise, make sure you have:

- **Python 3.10 or higher** installed (`python --version`)
- **An OpenAI API key** with credits. Sign up at [platform.openai.com](https://platform.openai.com) if you do not have one
- **pip** available (`pip --version`)

### Install the OpenAI SDK

Create a new directory for this project and install the SDK:

```bash
mkdir openai-caller
cd openai-caller
pip install openai
```

### Set Your API Key

Create a `.env` file in your project directory:

```
OPENAI_API_KEY=sk-proj-your-key-here
```

**Never commit this file.** Create `.gitignore` and add `.env` to it:

```
.env
```

Then load it in your Python scripts:

```python
import os
from dotenv import load_dotenv

load_dotenv()
```

Or set the environment variable directly in your terminal:

```bash
export OPENAI_API_KEY="sk-proj-your-key-here"
```

### Verify Installation

```bash
python -c "import openai; print(openai.__version__)"
```

Expected output (version may vary):

```
2.44.0
```

## The Build — Step by Step

### Step 1: Create the Client

Create a file called `chat.py` with your first imports and client setup:

```python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ["OPENAI_API_KEY"]
)

print("Client created successfully")
```

When you call `OpenAI()`, the SDK checks that `OPENAI_API_KEY` is set. If you leave out the `api_key` parameter, it reads the variable automatically.

Run it:

```bash
python chat.py
```

Expected output:

```
Client created successfully
```

### Step 2: Make Your First API Call

Replace the content of `chat.py` with a full request:

```python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ["OPENAI_API_KEY"]
)

completion = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "user", "content": "What is the capital of Japan?"}
    ]
)

print(completion.choices[0].message.content)
```

Run it:

```bash
python chat.py
```

Expected output (exact wording may vary):

```
The capital of Japan is Tokyo.
```

**What just happened:** Your code sent a single user message to GPT-4o-mini, the model generated a response, and you printed the text. This is the simplest possible API call.

### Step 3: Add a System Prompt

A system prompt (or `developer` role in the latest OpenAI convention) sets the behavior of the assistant. Replace `chat.py` with:

```python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ["OPENAI_API_KEY"]
)

completion = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "developer", "content": "You are a historian. Answer every question with a one-sentence historical fact, then stop."},
        {"role": "user", "content": "What is the capital of Japan?"}
    ]
)

print(completion.choices[0].message.content)
```

Run it:

```bash
python chat.py
```

Expected output:

```
Tokyo became the de facto capital of Japan in 1603 when Tokugawa Ieyasu established his shogunate there, though the imperial capital remained in Kyoto until 1868.
```

Notice the different tone — the model adapted to the "historian" persona.

### Step 4: Read the Token Usage

Add usage tracking to see exactly what you are being billed for:

```python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ["OPENAI_API_KEY"]
)

completion = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "developer", "content": "Answer in one sentence."},
        {"role": "user", "content": "Explain what a Python decorator is."}
    ]
)

print("Response:", completion.choices[0].message.content)
print()
print("--- Token Usage ---")
print(f"Prompt tokens:     {completion.usage.prompt_tokens}")
print(f"Completion tokens: {completion.usage.completion_tokens}")
print(f"Total tokens:      {completion.usage.total_tokens}")

# Calculate estimated cost (approximate)
input_cost_per_1m = 0.15   # gpt-4o-mini input
output_cost_per_1m = 0.60  # gpt-4o-mini output
cost = (completion.usage.prompt_tokens / 1_000_000 * input_cost_per_1m +
        completion.usage.completion_tokens / 1_000_000 * output_cost_per_1m)
print(f"Estimated cost:    ${cost:.6f}")
```

Run it:

```bash
python chat.py
```

Expected output (tokens will vary):

```
Response: A Python decorator is a function that allows you to modify the behavior of another function or class without changing its source code.

--- Token Usage ---
Prompt tokens:     29
Completion tokens: 25
Total tokens:      54
Estimated cost:    $0.000019
```

Fifty-four tokens for a fraction of a cent. This is why API-based AI is so accessible — individual calls are cheap, and costs only add up at scale.

### Step 5: Handle Errors Gracefully

Real applications need error handling. Add a try-except block:

```python
import os
from openai import OpenAI, APIError, RateLimitError, APIConnectionError

client = OpenAI(
    api_key=os.environ["OPENAI_API_KEY"]
)

try:
    completion = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": "Hello"}],
        max_tokens=50
    )
    print(completion.choices[0].message.content)
    print(f"Finish reason: {completion.choices[0].finish_reason}")

except RateLimitError:
    print("Rate limit hit. Wait a moment and try again.")
except APIConnectionError:
    print("Could not connect to OpenAI. Check your internet connection.")
except APIError as e:
    print(f"OpenAI API returned an error: {e}")
except Exception as e:
    print(f"Unexpected error: {e}")
```

This is a production-ready pattern. Never call the API without error handling.

### Step 6: Build a Simple Chat Loop

Create `chat_loop.py`:

```python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ["OPENAI_API_KEY"]
)

messages = [
    {"role": "developer", "content": "You are a helpful assistant. Keep answers brief."}
]

print("Chat with GPT-4o-mini. Type 'quit' to exit.")
print("-" * 50)

while True:
    user_input = input("\nYou: ")
    if user_input.lower() == "quit":
        break

    messages.append({"role": "user", "content": user_input})

    completion = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=messages,
        max_tokens=200
    )

    reply = completion.choices[0].message.content
    print(f"\nAssistant: {reply}")

    messages.append({"role": "assistant", "content": reply})
```

Run it:

```bash
python chat_loop.py
```

This is a real chat application. Each turn appends both the user input and the assistant response to the messages list, so the model remembers the conversation history.

## Common Errors at This Stage

### Error 1: API key not set

```
openai.AuthenticationError: The api_key client option must be set either by passing api_key to the client or by setting the OPENAI_API_KEY environment variable
```

**Cause:** You did not set `OPENAI_API_KEY` as an environment variable or pass it to the client.
**Fix:** Run `export OPENAI_API_KEY="sk-proj-..."` before running your script, or pass it directly: `OpenAI(api_key="sk-proj-...")`.

### Error 2: Insufficient quota

```
openai.RateLimitError: 429 Too Many Requests
```

**Cause:** You exceeded your per-minute rate limit or have exhausted your usage quota.
**Fix:** Wait 60 seconds and retry. Check your usage at platform.openai.com/account/usage.

### Error 3: Model not found

```
openai.NotFoundError: 404 The model `gpt-4o-old` does not exist
```

**Cause:** You used a model name that does not exist or has been deprecated.
**Fix:** Check the current model list at platform.openai.com/docs/models. Use the exact model string as listed.

### Error 4: Invalid messages format

```
openai.BadRequestError: 400 messages must be an array of objects with role and content
```

**Cause:** You passed a string instead of a list, or omitted the `role` key.
**Fix:** Always pass messages as `[{"role": "user", "content": "your text here"}]`.

### Error 5: Request too large

```
openai.BadRequestError: 400 This model's maximum context length is 128000 tokens
```

**Cause:** Your messages plus the generated response exceed the model's context window.
**Fix:** Reduce the number of messages, shorten each message, or lower `max_tokens`.

### Error 6: Connection timeout

```
openai.APIConnectionError: Connection error
```

**Cause:** Your machine cannot reach api.openai.com (firewall, VPN, no internet).
**Fix:** Check your internet connection and firewall settings. If behind a corporate proxy, configure it with `OpenAI(http_client=httpx.Client(proxies="http://proxy:8080"))`.
