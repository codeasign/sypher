## Tier 1 — Beginner

### Exercise 1: Make a Greeting Call

**Task:** Write a Python script that sends the message "Say hello to the world in pirate speak" to GPT-4o-mini and prints the response.

**Hint:** You only need one user message. The model will naturally adopt the requested tone.

### Solution

```python
import os
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

completion = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "user", "content": "Say hello to the world in pirate speak"}
    ]
)

print(completion.choices[0].message.content)
```

Expected output:

```
Ahoy there, matey! The whole wide world be greetin' ye with a hearty "Ahoy!"
```

### Exercise 4: Switch Models

**Task:** Ask the same question to three different models: `gpt-4o-mini`, `gpt-4o`, and `o4-mini`. Print the response, token usage, and model name for each.

**Hint:** Loop over a list of model names. Compare the response quality and length.

### Solution

```python
import os
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

models = ["gpt-4o-mini", "gpt-4o", "o4-mini"]
question = "Explain quantum entanglement in one paragraph."

for model in models:
    completion = client.chat.completions.create(
        model=model,
        messages=[{"role": "user", "content": question}],
        max_tokens=200
    )
    print(f"--- {model} ---")
    print(completion.choices[0].message.content)
    print(f"Tokens: {completion.usage.total_tokens}")
    print()
```

### Exercise 1: Build a Summarizer

**Task:** Create a script that takes a long block of text (hardcoded or pasted) and sends it to the API with a system prompt instructing the model to summarize it in three bullet points. Print the summary.

**Hint:** Use the `developer` role for the instruction and the `user` role for the text.

### Solution

```python
import os
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

long_text = """
The Python programming language was created by Guido van Rossum in the late 1980s
and was first released in 1991. It was designed with an emphasis on code readability
and simplicity. Python uses significant indentation to delimit blocks of code,
rather than curly braces or keywords. It supports multiple programming paradigms,
including structured, object-oriented, and functional programming. Python is
dynamically typed and garbage-collected. It has a large standard library that
includes modules for file I/O, system calls, sockets, and more. Python's popularity
has grown significantly, especially in data science, machine learning, and web
development. Major companies like Google, Netflix, and NASA use Python.
"""

completion = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "developer", "content": "Summarize the following text in exactly three bullet points."},
        {"role": "user", "content": long_text}
    ],
    max_tokens=300
)

print(completion.choices[0].message.content)
```

### Exercise 2: Temperature Comparison

**Task:** Ask the same question five times with `temperature=0.0` and five times with `temperature=1.5`. Print the responses. Observe how temperature 0.0 produces identical outputs while temperature 1.5 produces varied outputs.

**Hint:** Use a for loop. Compare results manually.

### Solution

```python
import os
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

prompt = "Name three programming languages that start with 'J'."

print("=== Temperature 0.0 (deterministic) ===")
for i in range(5):
    completion = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": prompt}],
        temperature=0.0,
        max_tokens=50
    )
    print(f"{i+1}. {completion.choices[0].message.content}")

print("\n=== Temperature 1.5 (creative) ===")
for i in range(5):
    completion = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": prompt}],
        temperature=1.5,
        max_tokens=50
    )
    print(f"{i+1}. {completion.choices[0].message.content}")
```

At temperature 0.0, all five responses will be identical. At 1.5, they will differ because the model samples from less likely tokens.

### Exercise 3: Multi-Turn Conversation

**Task:** Create a script that asks the model to suggest a dinner recipe, then asks for a shopping list for that recipe, then asks for step-by-step cooking instructions. Each turn must include the full conversation history.

**Hint:** Append each user input and assistant response to the messages list. The model needs the history to remember which recipe it suggested.

### Solution

```python
import os
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

messages = [
    {"role": "developer", "content": "You are a helpful cooking assistant."}
]

# Turn 1
messages.append({"role": "user", "content": "Suggest a simple pasta dish for dinner."})
completion = client.chat.completions.create(model="gpt-4o-mini", messages=messages)
reply1 = completion.choices[0].message.content
messages.append({"role": "assistant", "content": reply1})
print("Turn 1 — Recipe:", reply1[:100], "...\n")

# Turn 2
messages.append({"role": "user", "content": "What ingredients do I need for that?"})
completion = client.chat.completions.create(model="gpt-4o-mini", messages=messages)
reply2 = completion.choices[0].message.content
messages.append({"role": "assistant", "content": reply2})
print("Turn 2 — Ingredients:", reply2[:100], "...\n")

# Turn 3
messages.append({"role": "user", "content": "Give me step-by-step cooking instructions."})
completion = client.chat.completions.create(model="gpt-4o-mini", messages=messages)
reply3 = completion.choices[0].message.content
print("Turn 3 — Instructions:", reply3[:100], "...")
```

### Exercise 4: Error-Handling Wrapper

**Task:** Write a function `safe_chat()` that wraps `client.chat.completions.create()` with error handling for authentication errors, rate limits, and network errors. It should retry once on rate limit errors after waiting 10 seconds.

**Hint:** Use `time.sleep(10)` for the rate limit retry. Catch `RateLimitError`, `AuthenticationError`, and `APIConnectionError` from `openai`.

### Solution

```python
import os
import time
from openai import OpenAI, RateLimitError, AuthenticationError, APIConnectionError

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

def safe_chat(model, messages, max_retries=1):
    for attempt in range(max_retries + 1):
        try:
            return client.chat.completions.create(
                model=model,
                messages=messages
            )
        except RateLimitError:
            if attempt < max_retries:
                print(f"Rate limited. Retrying in 10s...")
                time.sleep(10)
            else:
                print("Rate limit exceeded. Try again later.")
                return None
        except AuthenticationError:
            print("Invalid API key. Check your OPENAI_API_KEY.")
            return None
        except APIConnectionError:
            print("Network error. Check your internet connection.")
            return None

result = safe_chat("gpt-4o-mini", [{"role": "user", "content": "Hello"}])
if result:
    print(result.choices[0].message.content)
```

### Exercise 1: Token Cost Estimator

**Task:** Write a script that reads a text file, sends it to GPT-4o-mini for summarization, and estimates the cost before making the call using a rough heuristic (4 characters ≈ 1 token for English text). After the call, compare your estimate with the actual token usage. Print both.

**Hint:** Use `len(text) / 4` as an estimator for prompt tokens. Multiply by the per-token cost for `gpt-4o-mini` ($0.15 per 1M input tokens, $0.60 per 1M output tokens).

### Solution

```python
import os
from openai import OpenAI

# Configuration
INPUT_COST_PER_1M = 0.15
OUTPUT_COST_PER_1M = 0.60

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

# Read the file
text = """
Your long text here. Replace this with content from a file:
with open("my_document.txt", "r") as f:
    text = f.read()
"""

# Estimate before calling
estimated_input_tokens = len(text) / 4
estimated_input_cost = estimated_input_tokens / 1000000 * INPUT_COST_PER_1M
print(f"Estimated input tokens: {int(estimated_input_tokens)}")
print(f"Estimated input cost: ${estimated_input_cost:.8f}")

# Make the call
completion = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "developer", "content": "Summarize this text."},
        {"role": "user", "content": text}
    ]
)

# Compare with actual
actual_input = completion.usage.prompt_tokens
actual_cost = (actual_input / 1000000 * INPUT_COST_PER_1M +
               completion.usage.completion_tokens / 1000000 * OUTPUT_COST_PER_1M)

print(f"Actual input tokens: {actual_input}")
print(f"Actual cost: ${actual_cost:.8f}")
print(f"Estimate accuracy: {abs(1 - estimated_input_tokens / actual_input) * 100:.1f}% off")

print("\nSummary:")
print(completion.choices[0].message.content)
```

### Exercise 2: Batch Question Answering with Delays

**Task:** You have 20 questions you want to ask the API one by one. Write a script that sends each question, prints the answer, and waits 1 second between requests to stay under rate limits. Track the total tokens and cost across all questions.

**Hint:** Use `time.sleep(1)` between calls. Accumulate token usage by summing `completion.usage.total_tokens` across all calls.

### Solution

```python
import os
import time
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

questions = [
    "What is the capital of France?",
    "What is 2 + 2?",
    "What is the boiling point of water?",
    "Who wrote Romeo and Juliet?",
    "What planet is known as the Red Planet?",
    "What is the speed of light?",
    "What year did World War II end?",
    "What is the largest ocean?",
    "What language is spoken in Brazil?",
    "What is the square root of 144?"
]

total_tokens = 0
total_cost = 0.0
INPUT_COST = 0.15 / 1000000     # per token
OUTPUT_COST = 0.60 / 1000000    # per token

for i, question in enumerate(questions, 1):
    try:
        completion = client.chat.completions.create(
            model="gpt-4o-mini",
            messages=[{"role": "user", "content": question}]
        )
        answer = completion.choices[0].message.content
        tokens = completion.usage.total_tokens
        cost = (completion.usage.prompt_tokens * INPUT_COST +
                completion.usage.completion_tokens * OUTPUT_COST)

        total_tokens += tokens
        total_cost += cost

        print(f"{i}. Q: {question}")
        print(f"   A: {answer[:80]}...")
        print(f"   Tokens: {tokens}, Cost: ${cost:.6f}\n")

        time.sleep(1)  # Rate limit buffer

    except Exception as e:
        print(f"Error on question {i}: {e}")

print(f"\nTotal tokens: {total_tokens}")
print(f"Total cost: ${total_cost:.4f}")
```

### Exercise 3: Streaming Chat with Token-by-Token Display

**Task:** Create a chat script that streams responses from the API and prints each token as it arrives, giving a typewriter effect. The script should maintain conversation history across turns.

**Hint:** Use `stream=True` in the API call and iterate over the response: `for chunk in completion:`. Each chunk has the next token in `chunk.choices[0].delta.content`.

### Solution

```python
import os
from openai import OpenAI
import sys

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

messages = [
    {"role": "developer", "content": "You are a helpful assistant. Keep answers brief."}
]

print("Streaming chat. Type 'quit' to exit.\n")

while True:
    user_input = input("You: ")
    if user_input.lower() == "quit":
        break

    messages.append({"role": "user", "content": user_input})
    print("Assistant: ", end="", flush=True)

    collected = []
    stream = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=messages,
        stream=True,
        max_tokens=300
    )

    for chunk in stream:
        if chunk.choices[0].delta.content is not None:
            content = chunk.choices[0].delta.content
            print(content, end="", flush=True)
            collected.append(content)

    print("\n")
    messages.append({"role": "assistant", "content": "".join(collected)})
```

This streaming approach gives a real-time feel and is the standard pattern for chat interfaces. The response starts appearing in under a second because tokens are displayed as they are generated, not all at once at the end.
