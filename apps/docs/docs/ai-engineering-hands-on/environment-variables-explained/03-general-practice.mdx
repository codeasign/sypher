# Environment Variables Explained from Zero -- Practice Exercises

⏱ 30 minutes · Difficulty: Beginner to Intermediate

These exercises reinforce the concepts from the lesson. Work through them at your own pace. Each exercise includes a hint and a solution, but try to solve it on your own first.

## Tier 1: Beginner

### Exercise 1: Print all environment variables

Your operating system has dozens of environment variables set by default -- things like `HOME`, `PATH`, `USER`, and `SHELL` on macOS/Linux, or `USERPROFILE`, `PATH`, and `APPDATA` on Windows.

Write a Python script that prints every environment variable and its value, one per line, in the format `KEY = VALUE`.

**Hint:** `os.environ` behaves like a dictionary. You can iterate over it with `for key in os.environ:`.

<details>
<summary>Click for solution</summary>

```python
import os

for key in os.environ:
    value = os.environ[key]
    print(f"{key} = {value}")
```

Or using `.items()`:

```python
import os

for key, value in os.environ.items():
    print(f"{key} = {value}")
```
</details>

### Exercise 2: Check if a specific variable exists

Write a script that asks you to type a variable name (like `PATH` or `HOME`), then tells you whether that variable exists and what its value is.

Use Python's `input()` function to read what they type.

**Expected behavior:**

- If the variable exists: print "Found: PATH = /usr/bin:/bin:/usr/sbin"
- If the variable does NOT exist: print "Variable NOT_FOUND is not set"

**Hint:** Use `os.environ.get()` and check if the result is `None`.

<details>
<summary>Click for solution</summary>

```python
import os

var_name = input("Enter an environment variable name: ")
value = os.environ.get(var_name)

    print(f"Variable {var_name} is not set")

    print(f"Found: {var_name} = {value}")
```

### Exercise 4: Create a config helper function

Write a reusable function called `require_env(name)` that:

1. Tries to read an environment variable by name
2. If the variable exists, returns its value
3. If the variable does not exist, prints a message and exits the program with `exit(1)`

Then test it by calling `require_env("PATH")` and `require_env("NONEXISTENT_VAR_99999")`.

**Hint:** Use `os.environ.get()` and check for `None`. Use `exit(1)` to stop the program.

<details>
<summary>Click for solution</summary>

```python
import os

def require_env(name):
    value = os.environ.get(name)
    if value is None:
        print(f"ERROR: Required environment variable '{name}' is not set.")
        exit(1)
    return value

# Test with a variable that exists
path = require_env("PATH")
print(f"PATH = {path[:50]}...")  # print first 50 chars

# Test with a variable that does not exist
api_key = require_env("NONEXISTENT_VAR_99999")
print(f"API key: {api_key}")  # this line never runs
```
</details>

## Tier 2: Intermediate

### Exercise 5: Parse a .env file manually

Before using `python-dotenv`, people wrote their own `.env` parsers. Write a function called `load_env_file(filepath)` that opens a `.env` file and returns a dictionary of key-value pairs.

Your function should:

1. Open the file
2. Skip blank lines and lines that start with `#`
3. Split each remaining line on the first `=` sign
4. Strip whitespace from the key and value
5. Return a dictionary

Then test it by creating a small `.env` file and calling your function.

**Hint:** Use `str.strip()`, `str.startswith()`, and `str.split("=", 1)`.

<details>
<summary>Click for solution</summary>

```python
import os

def load_env_file(filepath):
    """Read a .env file and return a dictionary of variables."""
    result = {}
    with open(filepath, "r") as f:
        for line in f:
            line = line.strip()
            # Skip blank lines and comments
            if not line or line.startswith("#"):
                continue
            # Split on the first equals sign
            parts = line.split("=", 1)
            key = parts[0].strip()
            value = parts[1].strip() if len(parts) > 1 else ""
            result[key] = value
    return result

# Create a test .env file
with open("test.env", "w") as f:
    f.write("# This is a test\n")
    f.write("NAME=Alice\n")
    f.write("ROLE=Engineer\n")
    f.write("SCORE=100\n")

# Load and print
env_vars = load_env_file("test.env")
for key, value in env_vars.items():
    print(f"{key} = {value}")
```

Expected output:

```
NAME = Alice
ROLE = Engineer
SCORE = 100
```
</details>

### Exercise 6: Sensible defaults with .env overrides

Write a script that defines a set of default configuration values in a dictionary, then overrides them with values from a `.env` file if they exist.

Your script should:

1. Define defaults: `{"DB_HOST": "localhost", "DB_PORT": "5432", "DB_NAME": "default_db", "DEBUG": "false"}`
2. Load a `.env` file
3. Override any default where a matching key exists in the environment
4. Print the final configuration

**Hint:** Loop over your defaults and replace with `os.environ.get(key, default)`.

<details>
<summary>Click for solution</summary>

```python
import os
from dotenv import load_dotenv

load_dotenv()

# Define defaults
config_defaults = {
    "DB_HOST": "localhost",
    "DB_PORT": "5432",
    "DB_NAME": "default_db",
    "DEBUG": "false",
}

# Build final config by overriding defaults with env vars
final_config = {}
for key, default_value in config_defaults.items():
    final_config[key] = os.environ.get(key, default_value)

# Print the configuration
print("Final configuration:")
for key, value in final_config.items():
    print(f"  {key} = {value}")
```

Create a `.env` file:

```bash
DB_NAME=production_db
DEBUG=true
```

Expected output:

```
Final configuration:
  DB_HOST = localhost
  DB_PORT = 5432
  DB_NAME = production_db
  DEBUG = true
```

`DB_HOST` and `DB_PORT` fall back to their defaults. `DB_NAME` and `DEBUG` are overridden by the `.env` file.
</details>

### Exercise 7: Environment-aware greeting

Write a script that behaves differently depending on which environment variable called `ENVIRONMENT` is set.

If `ENVIRONMENT` is set to:

- `"development"` or not set at all: print "Running in development mode with verbose logging"
- `"staging"`: print "Running in staging mode with moderate logging"
- `"production"`: print "Running in production mode with minimal logging"
- Any other value: print "Warning: Unknown environment setting"

**Hint:** Use `os.environ.get("ENVIRONMENT", "development")` and chain `if`/`elif` statements.

<details>
<summary>Click for solution</summary>

```python
import os
from dotenv import load_dotenv

load_dotenv()

env = os.environ.get("ENVIRONMENT", "development")

if env == "development":
    print("Running in development mode with verbose logging")
elif env == "staging":
    print("Running in staging mode with moderate logging")
elif env == "production":
    print("Running in production mode with minimal logging")

    print(f"Warning: Unknown environment setting: {env}")
```

Test it with different settings:

```bash
# Running in development mode with verbose logging

python script.py
# Running in production mode with minimal logging
```

## Tier 3: Advanced

### Exercise 8: Multi-file environment loader

Build a function that loads environment variables from multiple `.env` files in order. Later files should NOT override variables that earlier files already set.

Create three files:

`base.env`:
```bash
APP_NAME=MyApp
LOG_LEVEL=info
DB_HOST=localhost
```

`override.env`:
```bash
LOG_LEVEL=debug
DB_HOST=override-db.example.com
```

`secret.env`:
```bash
API_KEY=sk-secret-key-here
```

Write a script that:

1. Loads `base.env` first
2. Loads `override.env` second (should NOT override `LOG_LEVEL` or `DB_HOST` since they were already set)
3. Loads `secret.env` third

Print the final state of all variables.

**Hint:** Before setting a variable, check if it already exists in your result dictionary. Only set it if it does not.

<details>
<summary>Click for solution</summary>

```python
import os

def load_env_file(filepath, existing=None):
    """Read a .env file and return variables, not overriding existing ones."""
    if existing is None:
        existing = {}
    with open(filepath, "r") as f:
        for line in f:
            line = line.strip()
            if not line or line.startswith("#"):
                continue
            parts = line.split("=", 1)
            key = parts[0].strip()
            value = parts[1].strip() if len(parts) > 1 else ""
            # Only set if not already present
            if key not in existing:
                existing[key] = value
    return existing

# Load in order
all_vars = {}

all_vars = load_env_file("base.env", all_vars)
print("After base.env:", all_vars)

all_vars = load_env_file("override.env", all_vars)
print("After override.env:", all_vars)

all_vars = load_env_file("secret.env", all_vars)
print("After secret.env:", all_vars)
```

Expected output:

```
After base.env: {'APP_NAME': 'MyApp', 'LOG_LEVEL': 'info', 'DB_HOST': 'localhost'}
After override.env: {'APP_NAME': 'MyApp', 'LOG_LEVEL': 'info', 'DB_HOST': 'localhost'}
After secret.env: {'APP_NAME': 'MyApp', 'LOG_LEVEL': 'info', 'DB_HOST': 'localhost', 'API_KEY': 'sk-secret-key-here'}
```

The override file had `LOG_LEVEL=debug` and `DB_HOST=override-db.example.com`, but they were ignored because those keys already existed from `base.env`.
</details>

### Exercise 9: Environment variable validator

Build a validator function that checks whether a set of required environment variables are set and validates their values meet certain rules.

Write a function called `validate_env(rules)` that takes a dictionary of rules like this:

```python
rules = {
    "API_KEY": {"required": True, "min_length": 20},
    "DB_PORT": {"required": True, "pattern": r"^\d+$"},
    "LOG_LEVEL": {"required": False, "allowed": ["debug", "info", "warning", "error"]},
    "APP_NAME": {"required": True},
}
```

The function should:

1. Check that all required variables are set
2. For variables that are set, apply additional checks
3. Print a clear error for each violation
4. Return `True` if all checks pass, `False` otherwise

**Hint:** Use `os.environ.get()` to check existence. For the "pattern" check, you can use Python's `re` module and `re.fullmatch()`.

<details>
<summary>Click for solution</summary>

```python
import os
import re
from dotenv import load_dotenv

load_dotenv()

def validate_env(rules):
    """Validate environment variables against a set of rules."""
    all_valid = True

    for var_name, checks in rules.items():
        value = os.environ.get(var_name)

        # Check required
        if checks.get("required", False):
            if not value:
                print(f"FAIL: {var_name} is required but not set")
                all_valid = False
                continue

        # Skip remaining checks if value is not set
        if not value:
            continue

        # Check min_length
        min_length = checks.get("min_length")
        if min_length is not None and len(value) < min_length:
            print(f"FAIL: {var_name} must be at least {min_length} characters (got {len(value)})")
            all_valid = False

        # Check pattern
        pattern = checks.get("pattern")
        if pattern is not None:
            if not re.fullmatch(pattern, value):
                print(f"FAIL: {var_name} value '{value}' does not match pattern {pattern}")
                all_valid = False

        # Check allowed values
        allowed = checks.get("allowed")
        if allowed is not None and value not in allowed:
            print(f"FAIL: {var_name} must be one of {allowed}, got '{value}'")
            all_valid = False

    return all_valid


# Define validation rules
rules = {
    "API_KEY": {"required": True, "min_length": 20},
    "DB_PORT": {"required": True, "pattern": r"^\d+$"},
    "LOG_LEVEL": {"required": False, "allowed": ["debug", "info", "warning", "error"]},
    "APP_NAME": {"required": True},
}

# Run validation
if validate_env(rules):
    print("All environment variables are valid. Starting application...")
else:
    print("Environment validation failed. Fix the issues above and try again.")
    exit(1)
```

Create a `.env` file:

```bash
API_KEY=sk-short
DB_PORT=not-a-number
APP_NAME=
```

Expected output:

```
FAIL: API_KEY must be at least 20 characters (got 8)
FAIL: DB_PORT value 'not-a-number' does not match pattern ^\d+$
FAIL: APP_NAME is required but not set
Environment validation failed. Fix the issues above and try again.
```
</details>

### Exercise 10: Debug mode toggle

Many applications use an environment variable called `DEBUG` to enable or disable detailed logging. Build a small application that respects this pattern.

Write a script that:

1. Reads a `DEBUG` environment variable (default: `"false"`)
2. If `DEBUG` is `"true"` or `"1"` (case-insensitive), turn on debug logging
3. Define a function called `debug_log(message)` that only prints when debug mode is on
4. Define a function called `info_log(message)` that always prints
5. Simulate an application startup with a few steps, using both log levels

**Hint:** Use `os.environ.get("DEBUG", "false").lower() in ("true", "1")` for the check.

<details>
<summary>Click for solution</summary>

```python
import os
from dotenv import load_dotenv

load_dotenv()

# Determine debug mode
debug_mode = os.environ.get("DEBUG", "false").lower() in ("true", "1")

def debug_log(message):
    """Only print if debug mode is enabled."""
    if debug_mode:
        print(f"[DEBUG] {message}")

def info_log(message):
    """Always print."""
    print(f"[INFO]  {message}")

# Simulate application startup
info_log("Starting application...")

debug_log("Loading configuration from environment")
debug_log(f"  DEBUG mode: {debug_mode}")
debug_log(f"  USER: {os.environ.get('USER', 'unknown')}")

info_log("Connecting to database...")
debug_log("  Connection string assembled")
debug_log("  Opening connection pool")

info_log("Application ready.")
```

Run without DEBUG:

```bash
python app.py
```

Expected output:

```
[INFO]  Starting application...
[INFO]  Connecting to database...
[INFO]  Application ready.
```

Run with DEBUG:

```bash
DEBUG=true python app.py
```

Expected output:

```
[INFO]  Starting application...
[DEBUG] Loading configuration from environment
[DEBUG]   DEBUG mode: True
[DEBUG]   USER: your-username
[INFO]  Connecting to database...
[DEBUG]   Connection string assembled
[DEBUG]   Opening connection pool
[INFO]  Application ready.
```

This pattern lets you add verbose logging during development without cluttering output in production.
</details>
