You will now make a real API call from your own machine. By the end of this exercise, you will have sent a prompt to an LLM, received a response, and handled the result in Python. No guesswork — every step is laid out below.

## Setup

### Prerequisites

- **Python 3.10 or later**. Open a terminal and run `python --version` to check. If you do not have Python installed, download it from python.org.
- **A code editor**. Any text editor works. VS Code is a good free option.
- **An Anthropic API key**. Sign up at console.anthropic.com and create a key. Copy the key string — it starts with `sk-ant-`.

### Install the SDK

Create a new folder for this project, then create a file called `requirements.txt` with this content:

```text
anthropic>=0.70.0
python-dotenv>=1.0.0
```

Open a terminal in your project folder and run:

```bash
pip install -r requirements.txt
```

This installs the Anthropic Python SDK (the library that builds HTTP requests for you) and python-dotenv (a helper for loading environment variables from a file).

### Set Your API Key

Create a file named `.env` in your project folder with this content:

```text
ANTHROPIC_API_KEY=sk-ant-your-key-here
```

Replace `sk-ant-your-key-here` with your actual API key. Then create a file called `.gitignore` with this content:

```text
.env
```

This ensures your key never gets committed to git.

## The Build — Step by Step

### Step 1: Verify Your Environment

Create a file called `check_env.py`:

```python
import os
from dotenv import load_dotenv

load_dotenv()
key = os.environ.get("ANTHROPIC_API_KEY")

if key and key.startswith("sk-ant-"):
    print("Key found and looks valid.")
    print(f"Key starts with: {key[:7]}...")
    print(f"Key length: {len(key)} characters")
else:
    print("Key not found or looks incorrect.")
    print("Make sure your .env file has: ANTHROPIC_API_KEY=sk-ant-...")
```

Run it:

```bash
python check_env.py
```

Expected output:

```text
Key found and looks valid.
Key starts with: sk-ant-...
Key length: 98 characters
```

If the key is not found, check that your `.env` file is in the same folder as the script and contains no extra spaces around the equals sign.

### Step 2: Make Your First API Call

Create a file called `first_call.py`:

```python
import os
from dotenv import load_dotenv
from anthropic import Anthropic

load_dotenv()

client = Anthropic(
    api_key=os.environ["ANTHROPIC_API_KEY"]
)

response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=100,
    messages=[
        {"role": "user", "content": "Say hello in one word."}
    ]
)

print(response.content[0].text)
```

Run it:

```bash
python first_call.py
```

Expected output:

```text
Hello
```

The exact word may vary — the model might say "Greetings" or "Hi" — but it will be a single-word greeting. This proves your key works, the SDK connects to the server, and the model responds.

### Step 3: Inspect the Full Response Object

Modify your script to print the full response:

```python
import os
from dotenv import load_dotenv
from anthropic import Anthropic

load_dotenv()

client = Anthropic(
    api_key=os.environ["ANTHROPIC_API_KEY"]
)

response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=100,
    messages=[
        {"role": "user", "content": "Say hello in one word."}
    ]
)

# Print the entire response object
print(type(response))
print(response)
```

Run it:

```bash
python first_call.py
```

Expected output (formatted for readability):

```text
<class 'anthropic.types.message.Message'>
Message(
    id='msg_01ABCDEF1234567890',
    content=[TextBlock(text='Hello', type='text')],
    model='claude-sonnet-4-20250514',
    role='assistant',
    stop_reason='end_turn',
    stop_sequence=None,
    type='message',
    usage=Usage(input_tokens=10, output_tokens=1)
)
```

The response object contains everything: the generated text, the model used, why it stopped, and the exact token count. Notice `output_tokens=1` — that single word cost a fraction of a cent to generate.

### Step 4: Ask a Multi-Sentence Question

Now increase the complexity. Create `ask_france.py`:

```python
import os
from dotenv import load_dotenv
from anthropic import Anthropic

load_dotenv()

client = Anthropic(
    api_key=os.environ["ANTHROPIC_API_KEY"]
)

response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=300,
    messages=[
        {"role": "user", "content": "What is the capital of France? Tell me three interesting facts about it."}
    ]
)

print(response.content[0].text)
print()
print("--- Usage ---")
print(f"Input tokens: {response.usage.input_tokens}")
print(f"Output tokens: {response.usage.output_tokens}")
```

Run it:

```bash
python ask_france.py
```

Expected output (exact wording may differ):

```text
The capital of France is Paris.

Three interesting facts about Paris:

1. The Eiffel Tower was originally built as a temporary structure for the 1889 World's Fair and was almost torn down in 1909.

2. Paris has only one stop sign in the entire city — the vast majority of intersections use priority-to-the-right rules instead.

3. The Louvre Museum would take approximately 100 days to see every piece of art if you spent 30 seconds on each item.

--- Usage ---
Input tokens: 22
Output tokens: 120
```

You now know exactly how many tokens each side of the conversation consumed.

### Step 5: Add a System Prompt

A system prompt sets the model's behavior before the user message. Create `pirate_mode.py`:

```python
import os
from dotenv import load_dotenv
from anthropic import Anthropic

load_dotenv()

client = Anthropic(
    api_key=os.environ["ANTHROPIC_API_KEY"]
)

response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=150,
    system="You are a pirate. Speak like a pirate in every response.",
    messages=[
        {"role": "user", "content": "What is the capital of France?"}
    ]
)

print(response.content[0].text)
```

Run it:

```bash
python pirate_mode.py
```

Expected output:

```text
Arrr, the capital o' France be Paris, me hearty! A fine city on the Seine River, full o' treasure and... well, more baguettes than gold doubloons, but a worthy port o' call all the same!
```

The system prompt changes the model's tone completely without changing the question. You can use this to set roles, enforce formatting rules, or constrain the model to specific topics.

### Step 6: Stream the Response Token by Token

Create `stream_demo.py`:

```python
import os
from dotenv import load_dotenv
from anthropic import Anthropic

load_dotenv()

client = Anthropic(
    api_key=os.environ["ANTHROPIC_API_KEY"]
)

with client.messages.stream(
    model="claude-sonnet-4-20250514",
    max_tokens=100,
    messages=[
        {"role": "user", "content": "Count from 1 to 5, one number per line."}
    ]
) as stream:
    for text in stream.text_stream:
        print(text, end="", flush=True)

print()
```

Run it:

```bash
python stream_demo.py
```

Expected output (each character appears incrementally):

```text
1
2
3
4
5
```

With streaming, you see the output as the model generates it. This is the technique behind every chat interface that types out responses in real time.

### Step 7: Count Tokens Without Making a Full Call

Sometimes you want to know how many tokens your prompt uses before you send it. The SDK provides a token counting method:

```python
import os
from dotenv import load_dotenv
from anthropic import Anthropic

load_dotenv()

client = Anthropic(
    api_key=os.environ["ANTHROPIC_API_KEY"]
)

count = client.messages.count_tokens(
    model="claude-sonnet-4-20250514",
    messages=[
        {"role": "user", "content": "What is the capital of France? Tell me three interesting facts about it."}
    ]
)

print(f"Input token count: {count.input_tokens}")
```

Run it:

```bash
python count_tokens.py
```

Expected output:

```text
Input token count: 22
```

This call is free — it does not run the model. Use it to estimate costs before sending expensive prompts.

## Common Errors at This Stage

### Error 1: `anthropic.AuthenticationError`

```text
anthropic.AuthenticationError: Error code: 401 - {"error":{"type":"authentication_error","message":"invalid x-api-key"}}
```

**Cause:** Your API key is missing, incorrect, or being read from the wrong place. The server rejected your key.

**Fix:** Check that your `.env` file has the correct key and that `load_dotenv()` runs before you access the environment variable. Print the key value (first few characters) to verify: `print(os.environ["ANTHROPIC_API_KEY"][:7])`.

### Error 2: `anthropic.NotFoundError`

```text
anthropic.NotFoundError: Error code: 404 - {"error":{"type":"not_found_error","message":"model: claude-sonnet-4-20250514"}}
```

**Cause:** The model string does not exist or has been deprecated. Model names change when new versions are released.

**Fix:** Check the current model name in the Anthropic documentation or in your account dashboard. Replace the string with the correct identifier.

### Error 3: `anthropic.RateLimitError`

```text
anthropic.RateLimitError: Error code: 429 - {"error":{"type":"rate_limit_error","message":"Rate limit exceeded."}}
```

**Cause:** You sent too many requests in a short time window.

**Fix:** Wait a minute before retrying. If you need higher limits, check your plan level at console.anthropic.com. In production code, add retry logic with exponential backoff.

### Error 4: `ModuleNotFoundError: No module named 'anthropic'`

```text
ModuleNotFoundError: No module named 'anthropic'
```

**Cause:** The Anthropic SDK is not installed.

**Fix:** Run `pip install -r requirements.txt` again. If you are using a virtual environment, make sure it is activated before running the command. Run `pip list | grep anthropic` to confirm the package is installed.

### Error 5: `KeyError: 'ANTHROPIC_API_KEY'`

```text
KeyError: 'ANTHROPIC_API_KEY'
```

**Cause:** Python tried to read the environment variable, but it was not set.

**Fix:** Make sure your `.env` file exists and contains `ANTHROPIC_API_KEY=sk-ant-...` with no extra quotes or spaces. Verify that `load_dotenv()` is called before `os.environ["ANTHROPIC_API_KEY"]`. The `.env` file must be in the same directory as your script.
