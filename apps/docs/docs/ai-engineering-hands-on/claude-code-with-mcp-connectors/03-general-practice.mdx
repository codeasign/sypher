## Tier 1 — Beginner

### Exercise 1.1: Add a Delete Tool to Your SQLite Server

**Task:** Add a new tool called `delete_row` to the SQLite MCP server from the practice exercise. The tool should delete a row from a specified table by ID, but only allow deletion from the `users` table.

**Hint:** Follow the same pattern as `insert_row`. Use `db.prepare('DELETE FROM table WHERE id = ?').run(id)`. Remember to check the table name against an allowed list.

**Solution:**

```javascript
server.tool(
  'delete_row',
  'Delete a row from a table by ID',
  {
    table: { type: 'string', description: 'The table name to delete from' },
    id: { type: 'number', description: 'The ID of the row to delete' }
  },
  async (args) => {
    const allowedTables = ['users'];
    if (!allowedTables.includes(args.table)) {
      return { content: [{ type: 'text', text: `Error: Table "${args.table}" is not allowed` }], isError: true };
    }
    try {
      const result = db.prepare(`DELETE FROM ${args.table} WHERE id = ?`).run(args.id);
      if (result.changes === 0) {
        return { content: [{ type: 'text', text: `No row found with ID ${args.id}` }] };
      }
      return { content: [{ type: 'text', text: `Deleted row with ID ${args.id} from ${args.table}` }] };
    } catch (err) {
      return { content: [{ type: 'text', text: `Delete error: ${err.message}` }], isError: true };
    }
  }
);
```

Restart Claude Code to pick up the changes. Then try: "Delete user Bob from the database."

### Exercise 1.2: Create a Simple Filesystem MCP Server

**Task:** Build an MCP server that exposes two tools: `list_files` to list files in a directory, and `read_file` to read a file's contents. Both tools should only operate within a single allowed directory.

**Hint:** Use `fs.readdirSync` and `fs.readFileSync` from Node.js built-in `fs` module. Resolve paths with `path.resolve` and check that the resolved path starts with the allowed directory.

**Solution:**

```javascript
const { McpServer } = require('@modelcontextprotocol/sdk/server/mcp.js');
const { StdioServerTransport } = require('@modelcontextprotocol/sdk/server/stdio.js');
const fs = require('fs');
const path = require('path');

const ALLOWED_DIR = path.resolve(process.env.ALLOWED_DIR || '.');

function isPathSafe(targetPath) {
  return path.resolve(ALLOWED_DIR, targetPath).startsWith(ALLOWED_DIR);
}

const server = new McpServer({ name: 'safe-filesystem', version: '1.0.0' });

server.tool('list_files', 'List files in a directory', {
  dir: { type: 'string', description: 'Directory path relative to allowed root' }
}, async (args) => {
  if (!isPathSafe(args.dir)) {
    return { content: [{ type: 'text', text: 'Error: Path outside allowed directory' }], isError: true };
  }
  try {
    const entries = fs.readdirSync(path.resolve(ALLOWED_DIR, args.dir), { withFileTypes: true });
    const listing = entries.map(e => ({
      name: e.name,
      type: e.isDirectory() ? 'directory' : 'file',
      size: e.isFile() ? fs.statSync(path.join(ALLOWED_DIR, args.dir, e.name)).size : null
    }));
    return { content: [{ type: 'text', text: JSON.stringify(listing, null, 2) }] };
  } catch (err) {
    return { content: [{ type: 'text', text: `Error: ${err.message}` }], isError: true };
  }
});

server.tool('read_file', 'Read the contents of a file', {
  filepath: { type: 'string', description: 'File path relative to allowed root' }
}, async (args) => {
  if (!isPathSafe(args.filepath)) {
    return { content: [{ type: 'text', text: 'Error: Path outside allowed directory' }], isError: true };
  }
  try {
    const content = fs.readFileSync(path.resolve(ALLOWED_DIR, args.filepath), 'utf-8');
    return { content: [{ type: 'text', text: content }] };
  } catch (err) {
    return { content: [{ type: 'text', text: `Error: ${err.message}` }], isError: true };
  }
});

async function main() {
  const transport = new StdioServerTransport();
  await server.connect(transport);
  console.error('Safe filesystem MCP server running');
}
main().catch((err) => { console.error('Server failed:', err); process.exit(1); });
```

Configure it in `.claude/settings.json` with `ALLOWED_DIR` set to a safe directory like `/tmp/sandbox`.

### Exercise 1.3: Configure Two MCP Servers in Settings

**Task:** Create a `.claude/settings.json` that connects both the SQLite server and the filesystem server at the same time.

**Hint:** The `mcpServers` object can hold multiple entries. Each entry needs a unique key, a command, and optional args and env.

**Solution:**

```json
{
  "mcpServers": {
    "sqlite-connector": {
      "command": "node",
      "args": ["/home/you/mcp-sqlite-server/server.js"],
      "env": { "DB_PATH": "/home/you/mcp-sqlite-server/sample.db" }
    },
    "safe-filesystem": {
      "command": "node",
      "args": ["/home/you/mcp-filesystem-server/server.js"],
      "env": { "ALLOWED_DIR": "/home/you/sandbox" }
    }
  }
}
```

Start Claude Code and run `/mcp`. You should see both servers listed with their respective tools.

### Exercise 1.4: Manage MCP Servers with Slash Commands

**Task:** Practice using the four main `/mcp` slash commands to manage your connected servers.

**Hint:** Run `/mcp` inside Claude Code to see the current state. Use `/mcp disconnect sqlite-connector` to disconnect, then `/mcp connect sqlite-connector` to reconnect.

**Solution:**

Start Claude Code with the dual-server configuration above. Then try these commands in order:

```text
/mcp
```

This shows both servers. Now disconnect one:

```text
/mcp disconnect safe-filesystem
```

You should see a confirmation. Run `/mcp` again — only the SQLite server should appear. Now reconnect:

```text
/mcp connect safe-filesystem
```

Both servers should be back. Check the logs of a server:

```text
/mcp logs sqlite-connector
```

These commands give you full control over which servers are active without restarting Claude Code.

## Tier 2 — Intermediate

### Exercise 2.1: Build an MCP Server That Calls a REST API

**Task:** Create an MCP server that fetches data from a public REST API (like JSONPlaceholder) and exposes tools for getting posts and users.

**Hint:** Use the built-in `fetch` function available in Node.js 18+. The tool should accept an ID parameter and return formatted JSON.

**Solution:**

```javascript
const { McpServer } = require('@modelcontextprotocol/sdk/server/mcp.js');
const { StdioServerTransport } = require('@modelcontextprotocol/sdk/server/stdio.js');

const server = new McpServer({ name: 'json-api-connector', version: '1.0.0' });

server.tool('get_post', 'Fetch a blog post by ID from JSONPlaceholder', {
  id: { type: 'number', description: 'The post ID (1-100)', minimum: 1, maximum: 100 }
}, async (args) => {
  try {
    const response = await fetch(`https://jsonplaceholder.typicode.com/posts/${args.id}`);
    if (!response.ok) throw new Error(`HTTP ${response.status}`);
    const data = await response.json();
    return { content: [{ type: 'text', text: JSON.stringify(data, null, 2) }] };
  } catch (err) {
    return { content: [{ type: 'text', text: `API error: ${err.message}` }], isError: true };
  }
});

server.tool('get_user_posts', 'Fetch all posts by a user', {
  userId: { type: 'number', description: 'The user ID (1-10)', minimum: 1, maximum: 10 }
}, async (args) => {
  try {
    const response = await fetch(`https://jsonplaceholder.typicode.com/users/${args.userId}/posts`);
    if (!response.ok) throw new Error(`HTTP ${response.status}`);
    const data = await response.json();
    return { content: [{ type: 'text', text: JSON.stringify(data, null, 2) }] };
  } catch (err) {
    return { content: [{ type: 'text', text: `API error: ${err.message}` }], isError: true };
  }
});

async function main() {
  const transport = new StdioServerTransport();
  await server.connect(transport);
  console.error('JSON API MCP server running');
}
main().catch(console.error);
```

Test it by asking Claude Code: "Show me post number 5 from the blog API."

### Exercise 2.2: Chain Multiple Tool Calls in One Session

**Task:** Use the SQLite MCP server and the filesystem MCP server together. Ask Claude Code to export the users table to a JSON file on the filesystem, using only natural language.

**Hint:** Claude Code can discover tools across multiple MCP servers. It will figure out which server provides which tool and call them in sequence. The query tool comes from `sqlite-connector`, and the file writing would come from `safe-filesystem`.

**Solution:**

Start Claude Code with both servers configured. Then prompt:

```text
Query all users from the database and write the results to a file called users-export.json in the sandbox directory.
```

Claude Code will first call `query_database` to get the users, then use the filesystem server's tools to write the result. If your filesystem server does not have a write tool, Claude Code will tell you. Add this `write_file` tool to your filesystem server:

```javascript
server.tool('write_file', 'Write content to a file', {
  filepath: { type: 'string', description: 'File path relative to allowed root' },
  content: { type: 'string', description: 'Content to write' }
}, async (args) => {
  if (!isPathSafe(args.filepath)) {
    return { content: [{ type: 'text', text: 'Error: Path outside allowed directory' }], isError: true };
  }
  try {
    const fullPath = path.resolve(ALLOWED_DIR, args.filepath);
    fs.writeFileSync(fullPath, args.content, 'utf-8');
    return { content: [{ type: 'text', text: `File written to ${fullPath}` }] };
  } catch (err) {
    return { content: [{ type: 'text', text: `Write error: ${err.message}` }], isError: true };
  }
});
```

### Exercise 2.3: Add Input Validation to Your Server

**Task:** Enhance your SQLite MCP server so that all query results are capped at 100 rows and string inputs are sanitized for length.

**Hint:** Modify the `query_database` handler to append `LIMIT 100` to SELECT queries that do not already have a LIMIT clause. For the `insert_row` tool, validate that string values do not exceed 256 characters.

**Solution:**

Add this validation helper at the top of your server file:

```javascript
function enforceLimit(sql) {
  const trimmed = sql.trim();
  if (!/LIMIT\s+\d+/i.test(trimmed)) {
    const base = trimmed.endsWith(';') ? trimmed.slice(0, -1) : trimmed;
    return base + ' LIMIT 100';
  }
  return trimmed;
}

function validateStringLength(obj, maxLen) {
  for (const [key, value] of Object.entries(obj)) {
    if (typeof value === 'string' && value.length > maxLen) {
      throw new Error(`Field "${key}" exceeds maximum length of ${maxLen} characters`);
    }
  }
}
```

Then modify `query_database` to call `enforceLimit`:

```javascript
const safeSql = enforceLimit(args.sql);
const rows = db.prepare(safeSql).all();
```

And modify `insert_row` to call `validateStringLength`:

```javascript
validateStringLength(args.data, 256);
```

### Exercise 2.4: Implement Tool Result Caching

**Task:** Add in-memory caching to the `get_schema` tool so that repeated calls within the same session do not re-query the database schema.

**Hint:** Use a `Map` as a cache with a time-to-live (TTL) of 60 seconds. Check the cache before querying the database.

**Solution:**

Add at the top of your server file:

```javascript
const cache = new Map();
const CACHE_TTL_MS = 60000;

function getCached(key) {
  const entry = cache.get(key);
  if (entry && (Date.now() - entry.timestamp) < CACHE_TTL_MS) return entry.data;
  return null;
}

function setCache(key, data) {
  cache.set(key, { data, timestamp: Date.now() });
}
```

Then modify the `get_schema` handler:

```javascript
server.tool('get_schema', 'Get the schema of all tables', {
  table: { type: 'string', description: 'Optional table name' }
}, async (args) => {
  const cacheKey = args.table || '__all__';
  const cached = getCached(cacheKey);
  if (cached) return { content: [{ type: 'text', text: cached }] };
  try {
    const tables = db.prepare(args.table
      ? "SELECT name FROM sqlite_master WHERE type='table' AND name = ?"
      : "SELECT name FROM sqlite_master WHERE type='table'").all(args.table ? [args.table] : undefined);
    const schema = {};
    for (const t of tables) {
      schema[t.name] = db.prepare(`PRAGMA table_info(${t.name})`).all();
    }
    const result = JSON.stringify(schema, null, 2);
    setCache(cacheKey, result);
    return { content: [{ type: 'text', text: result }] };
  } catch (err) {
    return { content: [{ type: 'text', text: `Schema error: ${err.message}` }], isError: true };
  }
});
```

## Tier 3 — Advanced

### Exercise 3.1: Build a Docker Status MCP Server

**Task:** Create an MCP server that uses the Docker CLI to inspect running containers, view logs, and restart services. All command arguments must be strictly validated to prevent command injection.

**Hint:** Use `execSync` with `stdio: 'pipe'` to capture output. Validate container names against a regex pattern (alphanumeric and hyphens only). Never pass raw user input into a shell command.

**Solution:**

```javascript
const { McpServer } = require('@modelcontextprotocol/sdk/server/mcp.js');
const { StdioServerTransport } = require('@modelcontextprotocol/sdk/server/stdio.js');
const { execSync } = require('child_process');

const CONTAINER_REGEX = /^[a-zA-Z0-9][a-zA-Z0-9_-]*$/;

function runDocker(args) {
  try {
    return execSync(`docker ${args}`, { encoding: 'utf-8', maxBuffer: 10 * 1024 * 1024, timeout: 30000 });
  } catch (err) {
    throw new Error(`Docker command failed: ${err.stderr || err.message}`);
  }
}

const server = new McpServer({ name: 'docker-connector', version: '1.0.0' });

server.tool('list_containers', 'List all running Docker containers', {}, async () => {
  try {
    const output = runDocker('ps --format "{{.ID}}\t{{.Image}}\t{{.Status}}\t{{.Names}}"');
    const containers = output.trim().split('\n').filter(Boolean).map(line => {
      const [id, image, status, ...nameParts] = line.split('\t');
      return { id, image, status, name: nameParts.join('\t') };
    });
    return { content: [{ type: 'text', text: JSON.stringify(containers, null, 2) }] };
  } catch (err) {
    return { content: [{ type: 'text', text: `Error: ${err.message}` }], isError: true };
  }
});

server.tool('view_container_logs', 'View the last N log lines from a container', {
  container: { type: 'string', description: 'Container name or ID' },
  lines: { type: 'number', description: 'Number of lines to show', default: 50 }
}, async (args) => {
  if (!CONTAINER_REGEX.test(args.container)) {
    return { content: [{ type: 'text', text: 'Error: Invalid container name' }], isError: true };
  }
  try {
    const output = runDocker(`logs --tail ${args.lines} ${args.container}`);
    return { content: [{ type: 'text', text: output }] };
  } catch (err) {
    return { content: [{ type: 'text', text: `Error: ${err.message}` }], isError: true };
  }
});

server.tool('restart_container', 'Restart a Docker container', {
  container: { type: 'string', description: 'Container name or ID' }
}, async (args) => {
  if (!CONTAINER_REGEX.test(args.container)) {
    return { content: [{ type: 'text', text: 'Error: Invalid container name' }], isError: true };
  }
  try {
    runDocker(`restart ${args.container}`);
    return { content: [{ type: 'text', text: `Container ${args.container} restarted` }] };
  } catch (err) {
    return { content: [{ type: 'text', text: `Error: ${err.message}` }], isError: true };
  }
});

async function main() {
  const transport = new StdioServerTransport();
  await server.connect(transport);
  console.error('Docker connector MCP server running');
}
main().catch((err) => { console.error('Server failed:', err); process.exit(1); });
```

Test it by asking Claude Code: "Show me my running containers and the logs from the most recent one."

### Exercise 3.3: Build an MCP Server with SSE Transport

**Task:** Create an MCP server that uses SSE (Server-Sent Events) transport instead of stdio, allowing it to run as a network service that multiple Claude Code instances can connect to.

**Hint:** The MCP SDK provides `SSEServerTransport`. You need an HTTP server to handle the SSE connection and POST messages. The server must handle one GET endpoint for the SSE stream and one POST endpoint for receiving messages.

**Solution:**

```javascript
const { McpServer } = require('@modelcontextprotocol/sdk/server/mcp.js');
const { SSEServerTransport } = require('@modelcontextprotocol/sdk/server/sse.js');
const http = require('http');

const server = new McpServer({ name: 'remote-sqlite-server', version: '1.0.0' });

server.tool('ping', 'Verify connectivity', {}, async () => {
  return { content: [{ type: 'text', text: 'pong' }] };
});

const PORT = process.env.MCP_PORT || 3100;
let transport = null;

const httpServer = http.createServer(async (req, res) => {
  if (req.method === 'GET' && req.url === '/sse') {
    transport = new SSEServerTransport('/messages', res);
    await server.connect(transport);
  } else if (req.method === 'POST' && req.url === '/messages') {
    if (transport) {
      await transport.handlePostMessage(req, res);
    } else {
      res.writeHead(400);
      res.end('No active SSE session');
    }
  } else {
    res.writeHead(404);
    res.end('Not found');
  }
});

httpServer.listen(PORT, () => {
  console.error(`MCP SSE server listening on port ${PORT}`);
});
```

Configure Claude Code to connect to this server using the `url` field in settings.json:

```json
{
  "mcpServers": {
    "remote-sqlite": {
      "url": "http://localhost:3100/sse"
    }
  }
}
```

The SSE transport lets the MCP server run as a persistent network service, independent of any single Claude Code session. This is useful for shared team resources like a central database or a monitoring dashboard.
