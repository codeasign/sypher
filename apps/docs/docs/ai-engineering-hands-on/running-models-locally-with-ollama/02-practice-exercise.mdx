## Setup

Before you begin, make sure you have the following installed on your machine.

### Install Ollama

Download the installer for your operating system from ollama.ai and run it.

- **Windows**: Download the Windows installer and run it. Ollama runs as a system tray application.
- **macOS**: Download the macOS app and drag it to your Applications folder.
- **Linux**: Use the install script or your package manager.

Linux install via script:

```bash
curl -fsSL https://ollama.ai/install.sh | sh
```

Verify the installation:

```bash
ollama --version
```

### Pull a Model

The default installation does not include any models. You must pull at least one model.

```bash
ollama pull gemma3:2b
```

This downloads a 1.7 GB model that works well on most laptops. If you have limited disk space or an older machine, use an even smaller model:

```bash
ollama pull phi3:mini
```

### Install Python Packages

Create a new project directory and a virtual environment:

```bash
mkdir ollama-practice
cd ollama-practice
python -m venv venv
```

Activate the environment:

- Windows: `venv\Scripts\activate`
- macOS/Linux: `source venv/bin/activate`

Install the required packages:

```bash
pip install ollama openai
```

The ollama package provides the native Python SDK. The openai package lets you test the OpenAI-compatible endpoint.

## The Build

Follow these steps to build a complete local LLM workflow. Each step builds on the previous one.

### Step 1: Verify Ollama is Running

Before making any API calls, verify that the Ollama server is running.

Open a terminal and check the server status:

```bash
ollama list
```

If the server is not running, start it:

```bash
ollama serve
```

On Windows and macOS, the server starts automatically when the Ollama application is running. Look for the Ollama icon in the system tray or menu bar.

### Step 2: Test the Model via CLI

Start an interactive chat session with your model.

```bash
ollama run gemma3:2b
```

Once the prompt appears, type a few test messages:

```
>>> Write a haiku about programming.
>>> What is the capital of Mongolia?
>>> /bye
```

Observe the response speed. On a modern laptop CPU, you should see 5-15 tokens per second for a 2B-3B parameter model.

Now test with a single-shot prompt:

```bash
ollama run gemma3:2b "Explain the difference between TCP and UDP in one paragraph."
```

This is useful for scripting and automation, where you want a response without maintaining an interactive session.

### Step 3: Use the Python SDK for Chat

Create a file called chat_demo.py:

```python
from ollama import chat

response = chat(
    model='gemma3:2b',
    messages=[
        {'role': 'user', 'content': 'List three benefits of using version control.'}
    ]
)

print(response.message.content)
```

Run it:

```bash
python chat_demo.py
```

The script sends a single message, receives the response, and prints it. The SDK handles all the HTTP communication with the local Ollama server.

### Step 4: Build a Multi-Turn Conversation

Create a file called conversation.py:

```python
from ollama import chat

messages = [
    {'role': 'system', 'content': 'You are a helpful math tutor for elementary school students.'},
    {'role': 'user', 'content': 'What is a fraction?'},
]

# First turn
response = chat(model='gemma3:2b', messages=messages)
print('Tutor:', response.message.content)

# Add the response to history
messages.append({'role': 'assistant', 'content': response.message.content})
messages.append({'role': 'user', 'content': 'Can you give me an example of 1/4?'})

# Second turn
response = chat(model='gemma3:2b', messages=messages)
print()
print('Tutor:', response.message.content)
```

Run it:

```bash
python conversation.py
```

This demonstrates how to maintain conversation context by passing the full message history on each request. Each call includes the system prompt, all previous user messages, and all previous assistant responses.

### Step 5: Use the OpenAI-Compatible Endpoint

Create a file called openai_compat.py:

```python
from openai import OpenAI

client = OpenAI(
    base_url='http://localhost:11434/v1/',
    api_key='ollama'
)

response = client.chat.completions.create(
    model='gemma3:2b',
    messages=[
        {'role': 'system', 'content': 'You are a concise assistant.'},
        {'role': 'user', 'content': 'What is 7 * 8?'}
    ]
)

print(response.choices[0].message.content)
```

Run it:

```bash
python openai_compat.py
```

This script uses the OpenAI Python SDK, but points it at your local Ollama server instead of the OpenAI API. The response format is identical to what OpenAI returns, so any code written for OpenAI can switch to Ollama by changing the base_url and api_key.

### Step 6: Stream a Response

Create a file called streaming_demo.py:

```python
from ollama import chat

stream = chat(
    model='gemma3:2b',
    messages=[
        {'role': 'user', 'content': 'Write a short story about a robot learning to paint.'}
    ],
    stream=True
)

print('Story:')
for chunk in stream:
    print(chunk['message']['content'], end='', flush=True)
print()
```

Run it:

```bash
python streaming_demo.py
```

Streaming shows each token as it is generated, rather than waiting for the full response. This provides a better user experience for interactive applications.

### Step 7: Generate Embeddings

Create a file called embeddings_demo.py:

```python
from ollama import embed

response = embed(
    model='gemma3:2b',
    input='Machine learning is a subset of artificial intelligence.'
)

print(f'Embedding dimension: {len(response.embeddings[0])}')
print(f'First 5 values: {response.embeddings[0][:5]}')
```

Run it:

```bash
python embeddings_demo.py
```

Embeddings convert text into numerical vectors. These vectors are used for semantic search, clustering, and retrieval-augmented generation (RAG) pipelines. The embedding dimension varies by model.

## Common Errors

### Error 1: Ollama Not Running

**Error message:**

```
requests.exceptions.ConnectionError: HTTPConnectionPool(host='localhost', port=11434):
Max retries exceeded with url: /api/chat
```

**Cause:** The Ollama server is not started.

**Fix:** Start the Ollama application or run ollama serve in a terminal. Verify with ollama list.

```bash
ollama serve
```

In a new terminal:

```bash
ollama list
```

### Error 2: Model Not Found

**Error message:**

```
ollama.ResponseError: model "gemma3:2b" not found
```

**Cause:** The model has not been pulled yet, or the model name is misspelled.

**Fix:** Pull the model with the correct name.

```bash
ollama pull gemma3:2b
```

List available models to confirm:

```bash
ollama list
```

Check for typographical errors in the model name. Model names are case-sensitive.

### Error 3: Port Conflict

**Error message:**

```
listen tcp 127.0.0.1:11434: bind: address already in use
```

**Cause:** Another process is already using port 11434. This can happen if you have multiple instances of Ollama running, or another application is using the same port.

**Fix:** Check what is using the port and stop the conflicting process. On Windows:

```bash
netstat -ano | findstr :11434
```

On macOS/Linux:

```bash
lsof -i :11434
```

Alternatively, configure Ollama to use a different port:

```bash
set OLLAMA_HOST=127.0.0.1:11435
ollama serve
```

### Error 4: Out of Memory

**Error message:**

```
Error: LLAMA API error: CUDA OOM or system error: cannot allocate memory
```

Or the process is killed without an error message.

**Cause:** The model requires more RAM or VRAM than your system has available.

**Fix:** Use a smaller model or a lower quantization level. Close other memory-intensive applications before running the model.

```bash
ollama pull phi3:mini
ollama run phi3:mini
```

Check memory usage:

```bash
ollama ps
```

This shows all models currently loaded in memory and their resource usage.

### Error 5: OpenAI SDK API Key

**Error message (when using OpenAI SDK):**

```
openai.AuthenticationError: No API key provided.
```

**Cause:** The OpenAI SDK requires an api_key parameter, even when using Ollama. Some SDK versions enforce this check before making the request.

**Fix:** Pass any non-empty string as the api_key. Ollama does not validate it.

```python
client = OpenAI(
    base_url='http://localhost:11434/v1/',
    api_key='ollama'
)
```

### Error 6: Wrong Endpoint Path

**Error message:**

```
404 Not Found
```

**Cause:** The URL path is incorrect. The OpenAI-compatible endpoint requires the full path including /v1/.

**Fix:** Ensure the base_url ends with /v1/:

```python
client = OpenAI(
    base_url='http://localhost:11434/v1/',  # note the /v1/ suffix
    api_key='ollama'
)
```

## What You Just Built

You built a complete local LLM inference pipeline with the following capabilities:

1. **CLI interaction**: Running models interactively from the terminal
2. **Python SDK integration**: Sending prompts and receiving responses programmatically
3. **Multi-turn conversations**: Maintaining context across multiple message exchanges
4. **OpenAI SDK compatibility**: Using existing OpenAI code with a local model
5. **Streaming responses**: Displaying tokens as they are generated

The same pattern extends to any use case: chatbots, code assistants, summarization tools, classification pipelines, and RAG applications. The only difference between local and cloud inference is the endpoint your code connects to.

## Testing Your Setup

### Verify the OpenAI-Compatible Endpoint

Use curl to verify the OpenAI-compatible endpoint is working:

```bash
curl -X POST http://localhost:11434/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gemma3:2b",
    "messages": [{"role": "user", "content": "Say hello in French."}],
    "stream": false
  }'
```

Expected response contains a JSON object with the model response. The structure includes choices, message, and usage fields matching the OpenAI API format.

### Verify the Embeddings Endpoint

Test the embeddings endpoint directly:

```bash
curl -X POST http://localhost:11434/api/embed \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gemma3:2b",
    "input": "Test sentence for embedding"
  }'
```

The response contains an embeddings array of floating-point numbers. Each embedding is a vector of fixed dimension determined by the model.

## Complete Working Example

Here is a complete script that combines all the concepts from this exercise. It loads a document, generates embeddings, answers a question using RAG, and streams the response.

Create a file called complete_demo.py:

```python
import time
from ollama import chat, embed

# Step 1: Define documents
documents = [
    "Ollama is a tool for running LLMs locally on your own hardware.",
    "Models are downloaded from the Ollama library using ollama pull.",
    "The Ollama server exposes a REST API on port 11434.",
    "OpenAI-compatible endpoints are available at /v1/.",
    "Python SDK is available via pip install ollama.",
    "Modelfiles customize model behavior with system prompts and parameters.",
    "Streaming returns tokens one at a time as they are generated.",
    "Embeddings convert text into numerical vectors for similarity search.",
]

print("Step 1: Loading documents...")
print(f"Loaded {len(documents)} documents.")

# Step 2: Generate embeddings
print("\nStep 2: Generating document embeddings...")
doc_embeddings = []
for doc in documents:
    response = embed(model='gemma3:2b', input=doc)
    doc_embeddings.append(response.embeddings[0])
print(f"Generated {len(doc_embeddings)} embeddings.")

# Step 3: Query
question = "How do I download a model?"
print(f"\nStep 3: Processing query: {question}")

# Step 4: Find relevant documents
import math
q_embed = embed(model='gemma3:2b', input=question).embeddings[0]

def cosine_similarity(a, b):
    dot = sum(x*y for x, y in zip(a, b))
    na = math.sqrt(sum(x*x for x in a))
    nb = math.sqrt(sum(y*y for y in b))
    return dot / (na * nb)

similarities = [(cosine_similarity(q_embed, de), i)
                for i, de in enumerate(doc_embeddings)]
similarities.sort(reverse=True)
top_context = "\n".join([documents[i] for _, i in similarities[:3]])

print(f"Top relevant documents: {[documents[i] for _, i in similarities[:3]]}")

# Step 5: Generate answer
print("\nStep 5: Generating answer with context...")
prompt = f"""Answer based on the context below.

Context:
{top_context}

Question: {question}

Answer:"""

start = time.perf_counter()
stream = chat(
    model='gemma3:2b',
    messages=[{'role': 'user', 'content': prompt}],
    stream=True
)

print("Answer: ", end='', flush=True)
for chunk in stream:
    print(chunk['message']['content'], end='', flush=True)

elapsed = time.perf_counter() - start
print(f"\n\nCompleted in {elapsed:.2f} seconds.")
```

Run it:

```bash
python complete_demo.py
```

This script ties together everything: loading content, embedding it, searching semantically, and generating a contextual answer with streaming output.

## Troubleshooting Checklist

If you encounter problems during the exercise, go through this checklist in order.

### 1. Is Ollama running?

Run ollama list. If it returns an error, Ollama is not running. Start the Ollama application or run ollama serve.

### 2. Is the model downloaded?

Run ollama list. If your model is not in the list, run ollama pull gemma3:2b.

### 3. Is the correct model name in code?

Model names are case-sensitive and include the version tag. gemma3:2b is not the same as Gemma3:2b or gemma3:2B. Verify the exact name from ollama list.

### 4. Is the Python environment set up?

Run pip list to verify the ollama package is installed. If not, run pip install ollama. Make sure you are in the correct virtual environment.

### 5. Is the port available?

Check that port 11434 is not blocked by a firewall or used by another application. Run ollama serve in a terminal and watch for startup messages.

### 6. Is there enough memory?

If the model loads but crashes during inference, you may be running out of memory. Check available RAM and try a smaller model like phi3:mini.

### 7. Are you using the correct endpoint path?

For the OpenAI SDK, ensure the base URL ends with /v1/. The full path is http://localhost:11434/v1/.
