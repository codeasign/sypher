## The Build — Step by Step

### Step 1: Non-Streaming Baseline

Create `compare_streaming.py` to establish a baseline for comparison:

```python
import os
import time
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

start = time.time()

completion = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Write a paragraph about the history of the internet."}],
    max_tokens=300
)

elapsed = time.time() - start
response = completion.choices[0].message.content

print("Non-streaming response:")
print(response)
print(f"\nTime: {elapsed:.2f}s")
print(f"Characters: {len(response)}")
```

Run it:

```bash
python compare_streaming.py
```

Expected output: The complete response appears after several seconds, all at once alert.

```The internet began as a US military research project called ARPANET in the 1960s...
Time: 3.45s
Characters: 487
```

### Step 2: Your First Streaming Call

Create `first_stream.py`:

```python
import os
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

stream = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Write a paragraph about the history of the internet."}],
    max_tokens=300,
    stream=True
)

print("Streaming response:")
for chunk in stream:
    if chunk.choices[0].delta.content is not None:
        print(chunk.choices[0].delta.content, end="", flush=True)

print("\n\n--- Stream complete ---")
```

Run it:

```bash
python first_stream.py
```

**What to observe:** The first token appears almost immediately, and the rest of the text fills in progressively. Compare the experience with the non-streaming version. The total time is similar, but the perceived speed is dramatically different.

### Step 3: Add Timing to See the First-Token Latency

Replace `first_stream.py` with a timed version:

```python
import os
import time
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

stream = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Write a paragraph about the history of the internet."}],
    max_tokens=300,
    stream=True
)

start = time.time()
first_token = True
collected = []

for chunk in stream:
    if chunk.choices[0].delta.content is not None:
        content = chunk.choices[0].delta.content
        if first_token:
            first_token_latency = time.time() - start
            print(f"First token arrived in {first_token_latency:.3f}s\n")
            first_token = False
        print(content, end="", flush=True)
        collected.append(content)

total_time = time.time() - start
full_response = "".join(collected)

print(f"\n\nTotal time: {total_time:.2f}s")
print(f"First-token latency: {first_token_latency:.3f}s")
print(f"Response length: {len(full_response)} chars")
```

Run it:

```bash
python first_stream.py
```

Expected output (times will vary):

```
First token arrived in 0.412s

The internet began as a US military research project called ARPANET...

Total time: 3.87s
First-token latency: 0.412s
Response length: 502 chars
```

Notice the first token arrives in ~400ms but the total time is ~4s. Streaming gives the user something to read 3.5 seconds earlier than non-streaming.

### Step 4: Accumulate the Full Response

Update `first_stream.py` to accumulate and then use the complete response:

```python
import os
import time
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

stream = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "List three programming languages and their main uses."}],
    max_tokens=200,
    stream=True
)

collected = []
for chunk in stream:
    if chunk.choices[0].delta.content is not None:
        content = chunk.choices[0].delta.content
        print(content, end="", flush=True)
        collected.append(content)

full_response = "".join(collected)
print(f"\n\n=== Full Response ({len(full_response)} chars) ===")
print(full_response)
```

The key insight: you both display tokens in real-time AND store the complete response for later use — logging, saving to a database, or further processing.

### Step 5: Streaming with Token Usage Tracking

```python
import os
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

stream = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "What are the benefits of streaming?"}],
    max_tokens=200,
    stream=True,
    stream_options={"include_usage": True}
)

collected = []
for chunk in stream:
    if chunk.choices[0].delta.content is not None:
        content = chunk.choices[0].delta.content
        print(content, end="", flush=True)
        collected.append(content)
    if chunk.usage:
        print(f"\n\n--- Token Usage ---")
        print(f"Prompt tokens: {chunk.usage.prompt_tokens}")
        print(f"Completion tokens: {chunk.usage.completion_tokens}")
        print(f"Total tokens: {chunk.usage.total_tokens}")
```

Run it:

```bash
python first_stream.py
```

The usage data appears at the very end — it is only available once the stream completes.

### Step 6: Streaming Chat Loop with Typewriter Effect

Create `streaming_chat.py`:

```python
import os
import sys
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

messages = [
    {"role": "developer", "content": "You are a helpful assistant. Keep answers brief."}
]

print("Streaming chat. Type 'quit' to exit.")
print("-" * 40)

while True:
    user_input = input("\nYou: ")
    if user_input.lower() == "quit":
        break

    messages.append({"role": "user", "content": user_input})
    collected = []
    print("Assistant: ", end="", flush=True)

    stream = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=messages,
        max_tokens=300,
        stream=True
    )

    for chunk in stream:
        if chunk.choices[0].delta.content is not None:
            content = chunk.choices[0].delta.content
            print(content, end="", flush=True)
            collected.append(content)

    full_response = "".join(collected)
    print()
    messages.append({"role": "assistant", "content": full_response})
```

Run it:

```bash
python streaming_chat.py
```

Type a few messages and observe how the response appears character by character — this is the same pattern used by ChatGPT and Claude.ai.

## Common Errors at This Stage

### Error 1: Forgetting to check for None

```python
TypeError: sequence item 0: expected str instance, NoneType found
```

**Cause:** You tried to join collected chunks without filtering `None` values. Some streaming chunks have `delta.content = None`.
**Fix:** Always check `if chunk.choices[0].delta.content is not None` before using the value.

### Error 2: Using `end=""` without `flush=True`

```python
# Tokens buffer instead of appearing immediately
```

**Cause:** Python buffers stdout. Without `flush=True`, tokens are held in the buffer until a newline or the buffer fills.
**Fix:** Use `print(content, end="", flush=True)` or `sys.stdout.write(content); sys.stdout.flush()`.

### Error 3: Missing `stream_options` for usage data

```python
# chunk.usage is always None
```

**Cause:** OpenAI requires explicit `stream_options={"include_usage": True}` to include usage in streaming responses.
**Fix:** Add the parameter to your `create()` call. Without it, `chunk.usage` is never populated.

### Error 4: Reusing a consumed stream

```python
stream = client.chat.completions.create(..., stream=True)
for chunk in stream:
    print(chunk.choices[0].delta.content)
# Second iteration gets nothing — stream is exhausted
```

**Cause:** Streams are iterators that can only be consumed once.
**Fix:** Always accumulate the full response during iteration. Do not try to iterate the same stream twice.

### Error 5: Anthropic's context manager

```python
# Wrong — missing `with`
stream = client.messages.create(..., stream=True)
for event in stream:
    ...

# Correct
with client.messages.create(..., stream=True) as stream:
    for event in stream:
        ...
```

**Cause:** Anthropic's streaming API uses a context manager. Without `with`, the stream may not close properly.
**Fix:** Always use `with client.messages.create(..., stream=True) as stream:`.

### Error 6: Not accumulating — losing the response

```python
# Only prints, never saves
for chunk in stream:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="")
# The full response is now lost — cannot log or store it
```

**Cause:** You only displayed the output without accumulating it into a variable.
**Fix:** Always accumulate: `collected.append(content)` and then `full_response = "".join(collected)`.
