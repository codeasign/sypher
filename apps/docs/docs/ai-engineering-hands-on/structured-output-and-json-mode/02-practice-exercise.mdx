## The Build — Step by Step

### Step 1: The Problem — Unreliable JSON from Free Text

Create `unreliable.py` to see what happens when you ask for JSON without structured output:

```python
import os
import json
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

completion = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "developer", "content": "Return the following as JSON: name, age, city."},
        {"role": "user", "content": "John is 30 years old and lives in New York."}
    ]
)

print("Raw response:")
print(repr(completion.choices[0].message.content))
print()

try:
    data = json.loads(completion.choices[0].message.content)
    print("Parsed:", data)
except json.JSONDecodeError as e:
    print(f"Failed to parse: {e}")
```

Run it a few times:

```bash
python unreliable.py
python unreliable.py
python unreliable.py
```

Expected: The raw response may include extra text like `"Here is the JSON:"`, or markdown code fences `` ```json ``. Sometimes it parses cleanly, sometimes it does not. This unpredictability is the problem structured output solves.

### Step 2: Your First Structured Output Call

Create `structured.py`:

```python
import os
import json
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

schema = {
    "name": "extract_person",
    "schema": {
        "type": "object",
        "properties": {
            "name": {"type": "string", "description": "The person's full name"},
            "age": {"type": "integer", "description": "The person's age in years"},
            "city": {"type": "string", "description": "The city where the person lives"}
        },
        "required": ["name", "age", "city"],
        "additionalProperties": False
    },
    "strict": True
}

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "developer", "content": "Extract person information from the text."},
        {"role": "user", "content": "John is 30 years old and lives in New York."}
    ],
    response_format={
        "type": "json_schema",
        "json_schema": schema
    }
)

data = json.loads(response.choices[0].message.content)
print("Structured output:")
print(data)
print(f"\nName: {data['name']}")
print(f"Age: {data['age']}")
print(f"City: {data['city']}")
```

Run it:

```bash
python structured.py
```

Expected output:

```python
Structured output:
{'name': 'John', 'age': 30, 'city': 'New York'}

Name: John
Age: 30
City: New York
```

Every run produces the exact same shape. No extra text. No markdown. No parsing failures.

### Step 3: Extract Invoice Data with a Complex Schema

Create `invoice_extract.py`:

```python
import os
import json
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

invoice_schema = {
    "name": "extract_invoice",
    "schema": {
        "type": "object",
        "properties": {
            "vendor_name": {"type": "string", "description": "The vendor or company name"},
            "invoice_number": {"type": "string", "description": "Invoice reference number"},
            "invoice_date": {"type": "string", "description": "Date in YYYY-MM-DD format"},
            "total_amount": {"type": "number", "description": "Total amount due"},
            "currency": {"type": "string", "enum": ["USD", "EUR", "GBP", "JPY"]},
            "line_items": {
                "type": "array",
                "description": "Individual line items on the invoice",
                "items": {
                    "type": "object",
                    "properties": {
                        "description": {"type": "string"},
                        "quantity": {"type": "integer"},
                        "unit_price": {"type": "number"},
                        "total": {"type": "number"}
                    },
                    "required": ["description", "quantity", "unit_price", "total"],
                    "additionalProperties": False
                }
            }
        },
        "required": ["vendor_name", "invoice_number", "invoice_date", "total_amount", "currency", "line_items"],
        "additionalProperties": False
    },
    "strict": True
}

invoice_text = """
INVOICE #INV-2024-00789
Date: March 15, 2024

Vendor: TechCorp Solutions Inc.
123 Business Ave, San Francisco, CA

Items:
1. Server Maintenance - 2 hours @ $150/hr = $300.00
2. Cloud Storage - 500GB @ $0.10/GB = $50.00
3. SSL Certificate - 1 year @ $199.00 = $199.00

Total: $549.00
Payment Terms: Net 30
"""

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "developer", "content": "Extract the invoice data from the text below."},
        {"role": "user", "content": invoice_text}
    ],
    response_format={
        "type": "json_schema",
        "json_schema": invoice_schema
    }
)

invoice = json.loads(response.choices[0].message.content)

print(f"Vendor: {invoice['vendor_name']}")
print(f"Invoice: {invoice['invoice_number']}")
print(f"Date: {invoice['invoice_date']}")
print(f"Total: {invoice['currency']} {invoice['total_amount']:.2f}")
print(f"\nLine items ({len(invoice['line_items'])}):")
for item in invoice['line_items']:
    print(f"  - {item['description']}: {item['quantity']} × ${item['unit_price']:.2f} = ${item['total']:.2f}")
```

Run it:

```bash
python invoice_extract.py
```

Expected output:

```
Vendor: TechCorp Solutions Inc.
Invoice: INV-2024-00789
Date: 2024-03-15
Total: USD 549.00

Line items (3):
  - Server Maintenance: 2 × $150.00 = $300.00
  - Cloud Storage: 500 × $0.10 = $50.00
  - SSL Certificate: 1 × $199.00 = $199.00
```

### Step 4: Handle Refusals

Create `refusal_handling.py`:

```python
import os
import json
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

schema = {
    "name": "classify_sentiment",
    "schema": {
        "type": "object",
        "properties": {
            "sentiment": {"type": "string", "enum": ["positive", "negative", "neutral"]},
            "confidence": {"type": "number", "description": "Confidence score 0.0 to 1.0"}
        },
        "required": ["sentiment", "confidence"],
        "additionalProperties": False
    },
    "strict": True
}

test_inputs = [
    "I love this product! It is amazing!",
    "This is the worst experience I have ever had.",
    "The weather today is cloudy."
]

for text in test_inputs:
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {"role": "developer", "content": "Classify the sentiment of the text."},
            {"role": "user", "content": text}
        ],
        response_format={
            "type": "json_schema",
            "json_schema": schema
        }
    )

    if response.choices[0].message.refusal:
        print(f"Refused for: '{text}'")
        print(f"  Reason: {response.choices[0].message.refusal}")
    else:
        data = json.loads(response.choices[0].message.content)
        print(f"'{text}' → {data['sentiment']} (conf: {data['confidence']})")
    print()
```

### Step 5: Batch Processing with Structured Output

Create `batch_process.py`:

```python
import os
import json
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

extract_schema = {
    "name": "extract_entity",
    "schema": {
        "type": "object",
        "properties": {
            "entities": {
                "type": "array",
                "items": {
                    "type": "object",
                    "properties": {
                        "name": {"type": "string"},
                        "type": {"type": "string", "enum": ["person", "organization", "location", "date", "other"]},
                        "mention": {"type": "string", "description": "The exact text mentioning this entity"}
                    },
                    "required": ["name", "type", "mention"],
                    "additionalProperties": False
                }
            }
        },
        "required": ["entities"],
        "additionalProperties": False
    },
    "strict": True
}

documents = [
    "Apple Inc. was founded by Steve Jobs in Cupertino, California on April 1, 1976.",
    "Microsoft and OpenAI announced a partnership in San Francisco.",
    "Jane visited the British Museum in London last Tuesday."
]

for i, doc in enumerate(documents, 1):
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {"role": "developer", "content": "Extract all named entities from the text."},
            {"role": "user", "content": doc}
        ],
        response_format={
            "type": "json_schema",
            "json_schema": extract_schema
        }
    )

    result = json.loads(response.choices[0].message.content)
    print(f"Document {i}:")
    for entity in result["entities"]:
        print(f"  [{entity['type']:12s}] {entity['name']} (\"{entity['mention']}\")")
    print()
```

### Step 6: Anthropic Structured Output via Tool Use

Create `claude_structured.py`:

```python
import os
from anthropic import Anthropic

client = Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])

response = client.messages.create(
    model="claude-haiku-4-5-20251001",
    max_tokens=1024,
    messages=[
        {"role": "user", "content": "Extract: John (age 30) from New York, and Sarah (age 25) from Boston."}
    ],
    tools=[
        {
            "name": "extract_people",
            "description": "Extract people's information from text",
            "input_schema": {
                "type": "object",
                "properties": {
                    "people": {
                        "type": "array",
                        "items": {
                            "type": "object",
                            "properties": {
                                "name": {"type": "string"},
                                "age": {"type": "integer"},
                                "city": {"type": "string"}
                            },
                            "required": ["name", "age", "city"],
                            "additionalProperties": False
                        }
                    }
                },
                "required": ["people"],
                "additionalProperties": False
            }
        }
    ],
    tool_choice={"type": "tool", "name": "extract_people"}
)

for block in response.content:
    if block.type == "tool_use":
        data = block.input
        for person in data["people"]:
            print(f"{person['name']}, {person['age']}, from {person['city']}")
```

Run it:

```bash
python claude_structured.py
```

Expected output:

```
John, 30, from New York
Sarah, 25, from Boston
```

## Common Errors at This Stage

### Error 1: Schema name conflict

```
openai.BadRequestError: 400 Schema name 'extract' conflicts with existing tool/function name
```

**Cause:** The schema name matches an internal function name or another tool's name.
**Fix:** Use a unique, descriptive name like `"extract_invoice_data"` rather than generic names like `"extract"` or `"get_data"`.

### Error 2: Unsupported schema feature

```
openai.BadRequestError: 400 Unsupported schema feature: anyOf
```

**Cause:** The schema uses an unsupported feature like `anyOf`, `oneOf`, or `$ref`.
**Fix:** Use inline schema definitions only. Replace `anyOf` with optional fields (leave out of `required`).

### Error 3: Missing required parameter with legacy JSON mode

```
openai.BadRequestError: 400 When using response_format with type "json_object", you must include a system or developer message instructing the model to output JSON
```

**Cause:** Legacy `json_object` mode requires an explicit instruction in the prompt.
**Fix:** Use `json_schema` mode instead, which does not have this requirement, or add a developer message mentioning JSON output.

### Error 4: Anthropic tool_choice issues

```
anthropic.BadRequestError: 400 tool_choice: tool "extract_data" not found in tools
```

**Cause:** The tool name in `tool_choice` does not match any tool in the `tools` array.
**Fix:** Ensure the name matches exactly. It is case-sensitive.

### Error 5: Missing the `refusal` check

```python
# Crash when model refuses
data = json.loads(response.choices[0].message.content)
# TypeError: 'NoneType' object is not iterable if content is None due to refusal
```

**Cause:** You assumed content is always populated. When the model refuses, `content` may be set differently.
**Fix:** Always check `response.choices[0].message.refusal` before accessing content.
