You have been using an auto-complete feature in your code editor for months — GitHub Copilot powered by Claude. It works great. So you decide to build your own code assistant. You sign up for an Anthropic API key, install the SDK, and make your first call. It works. Then you deploy it. After a week, you get an email: your bill is $340. You check the logs and find that a bug in your retry logic was sending the same request 150 times per minute. Each request was using `claude-opus-4-8`, the most expensive model, because you never configured which model to use. The unlimited loop had no bound on `max_tokens`, so some responses ran to 8,000 tokens.

That experience is common. The Anthropic API is powerful — Claude can handle complex reasoning, long documents, and tool use. But calling it correctly means understanding the SDK, the model pricing, and how to control response length and cost.

## The Concept

Calling the Anthropic API means sending a structured request to Claude — Anthropic's family of language models — through the official Python SDK. Claude processes your input and generates a response. Think of it like sending a letter to an expert consultant: you write your question with clear instructions (the prompt), include any reference material (the context), specify how long the answer should be (`max_tokens`), and the consultant writes back. The consultant charges by the word, so you want to be precise about what you ask and how much you want back.

## API Reference

| Method / Parameter | What It Does | Example |
|---|---|---|
| `pip install anthropic` | Installs the official Anthropic Python SDK (v0.116.0) | `pip install anthropic` |
| `from anthropic import Anthropic` | Imports the client class | `from anthropic import Anthropic` |
| `client = Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])` | Creates a client using your API key from an environment variable | `client = Anthropic()` — reads `ANTHROPIC_API_KEY` by default |
| `client.messages.create(model="claude-sonnet-5", max_tokens=1024, messages=[...])` | Sends a message to Claude and returns the response | `client.messages.create(model="claude-sonnet-5", max_tokens=1024, messages=[{"role":"user","content":"Hello"}])` |
| `model` parameter | The Claude model to use. See table below | `model="claude-haiku-4-5-20251001"` |
| `max_tokens` (required) | Maximum tokens Claude can generate in the response | `max_tokens=500` |
| `system` parameter (optional) | System prompt that sets Claude's behavior | `system="You are a helpful coding assistant."` |
| `messages` parameter | Array of message objects with `role` and `content`. Roles: `user`, `assistant` | `messages=[{"role":"user","content":"What is Python?"}]` |
| `temperature` parameter (0.0–1.0) | Controls randomness. Lower is more deterministic | `temperature=0.0` |
| `stream` parameter (bool) | If true, receive tokens incrementally | `stream=True` |
| `message.content[0].text` | Accesses the response text from a message | `print(message.content[0].text)` |

## How It Actually Works — Step by Step

### Step 1: Client Initialization

When you call `Anthropic()`, the SDK checks your environment for `ANTHROPIC_API_KEY`. It then configures an HTTP client (based on `httpx`) with:

- Base URL: `https://api.anthropic.com`
- Default headers: `x-api-key`, `anthropic-version` (API version header)
- Timeout: default 600 seconds (10 minutes)

### Step 2: Building the Request

The `messages.create()` method serializes your parameters into a JSON payload:

```json
{
  "model": "claude-sonnet-5",
  "max_tokens": 1024,
  "system": "You are a helpful assistant.",
  "messages": [
    {"role": "user", "content": "What is the capital of France?"}
  ]
}
```

Unlike some other providers, `max_tokens` is **required** for Anthropic — you must always set it.

### Step 3: Authentication and Routing

The API validates your `x-api-key` against their authentication service. On success, the request is routed to an inference server that has the requested model loaded. If the model is cold (not loaded), the server loads it — this adds latency, typically 2-10 seconds for the first request.

### Step 4: Claude Generates the Response

Claude processes your message through its neural network. Key differences from other providers:

- Claude uses a **constitutional AI** approach — the model has built-in guardrails that influence token selection
- Claude supports **extended thinking** (visible chain-of-thought) for complex reasoning tasks
- The model can output multiple **content blocks** (text, tool_use) in a single response

Generation continues until one of these conditions is met:

1. Claude produces a stop sequence (`"\n\nHuman:"` or `"\n\nAssistant:"`)
2. The response reaches `max_tokens`
3. A content filter triggers

### Step 5: The Response Object

Anthropic's API returns a `Message` object with a different structure than OpenAI's:

```python
Message(
    id="msg_01ABCD2efgh3",
    type="message",
    role="assistant",
    content=[TextBlock(text="The capital of France is Paris.", type="text")],
    model="claude-sonnet-5",
    stop_reason="end_turn",
    stop_sequence=None,
    usage=Usage(input_tokens=14, output_tokens=8)
)
```

Key differences from OpenAI:

| Concept | OpenAI | Anthropic |
|---|---|---|
| Response access | `choices[0].message.content` | `content[0].text` |
| Stop reason | `choices[0].finish_reason` | `stop_reason` |
| Usage | `usage.prompt_tokens` etc. | `usage.input_tokens`, `.output_tokens` |
| Multiple content blocks | One message per choice | One message with multiple blocks in `content` array |

### Step 6: Content Blocks — Multi-Part Responses

Claude can return multiple content blocks in one response. For example, a response that uses a tool looks like this:

```python
content = [
    TextBlock(text="Let me look that up for you.", type="text"),
    ToolUseBlock(
        name="search_web",
        input={"query": "current population of France"},
        type="tool_use"
    )
]
```

You iterate over `content` and handle each block by `type`:

```python
for block in message.content:
    if block.type == "text":
        print(block.text)
    elif block.type == "tool_use":
        print(f"Tool call: {block.name}({block.input})")
```

## Cost, Limits, and Safety

### Token Costs (approximate, as of July 2026)

| Model | Input (per 1M tokens) | Output (per 1M tokens) |
|---|---|---|
| `claude-fable-5` | $15.00 | $75.00 |
| `claude-opus-4-8` | $15.00 | $75.00 |
| `claude-sonnet-5` | $3.00 | $15.00 |
| `claude-haiku-4-5` | $0.25 | $1.25 |

A single call to `claude-sonnet-5` with 500 input tokens and 200 output tokens costs roughly `(500/1M * $3.00) + (200/1M * $15.00)` = $0.0045. At 1,000 calls per day, that is about $4.50 per day for Sonnet. Using Opus would cost about $18 per day for the same volume.

### Rate Limits

Anthropic rate limits vary by tier. Typical starting limits are 5 requests per minute and 1,000 per day. Higher tiers unlock more capacity. Check your current limits at console.anthropic.com.

### Safety Rules

**Never hardcode your API key.** Always use an environment variable:

```python
import os
from anthropic import Anthropic

client = Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])
```

**Set `max_tokens` on every call.** Unlike some providers, this is required by Anthropic's API. A reasonable default is 1024 for chat, 4096 for analysis tasks.

**Always set `max_tokens` below your budget threshold.** If you set `max_tokens=100000`, a single long response could cost over $1.50 with Sonnet or $7.50 with Opus.

**Use lower-cost models for prototyping.** Start with `claude-haiku-4-5` during development. Upgrade to Sonnet or Opus only when you need the extra capability.

**Log token usage.** In production, log `message.usage.input_tokens` and `message.usage.output_tokens` for every call. This lets you detect unexpected spikes.

## When to Use This / When Not To

**Use the Anthropic API when:**
- You need a model with strong reasoning and instruction following
- You need a very long context window (up to 1M tokens with Sonnet/Opus/Fable)
- You want built-in safety features (constitutional AI)
- You need tool use / function calling capabilities
- You are building applications where extended thinking (visible chain-of-thought) is valuable

**Do not use the Anthropic API when:**
- You need to process data offline or behind a strict firewall
- You need the lowest possible per-call cost — consider Haiku or a local model
- Your use case requires heavy multimodal (image generation) — Claude supports image input but not image generation
- You are building a very high-frequency, low-latency application — each call has network latency (typically 1-5 seconds)

## Mental Model Check

**Question 1:** What parameter is required for every Anthropic API call that is optional with some other providers?

  `max_tokens` — it is a required parameter and must always be set explicitly.

---

**Question 2:** How do you access the response text from a Claude API response?

  Through `message.content[0].text`. The content is a list of content blocks; for text-only responses the first block is a `TextBlock` and its `.text` attribute contains the response.

---

**Question 3:** What does `stop_reason: "tool_use"` mean in a Claude response?

  It means Claude decided to call a tool function. The response will contain a `ToolUseBlock` in the content array with the tool name and input parameters. You must execute the tool and send the result back.

---

**Question 4:** Which Claude model is the best balance of speed, cost, and capability for most applications?

  `claude-sonnet-5`. It offers strong reasoning at roughly one-fifth the cost of Opus while still being fast enough for interactive use.

---

**Question 5:** How do the token usage fields in an Anthropic response differ from OpenAI's?

  Anthropic uses `usage.input_tokens` and `usage.output_tokens`. OpenAI uses `usage.prompt_tokens` and `usage.completion_tokens`. The concepts are the same but the field names differ.
