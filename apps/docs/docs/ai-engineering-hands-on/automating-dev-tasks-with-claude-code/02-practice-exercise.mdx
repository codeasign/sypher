In this exercise you will build a GitHub Actions workflow that uses Claude Code to automatically review pull requests, generate tests, and update documentation. By the end, you will have a working CI automation pipeline that you can adapt for your own repositories.

## Setup

Before you start, make sure you have the following prerequisites.

### Prerequisites

- A GitHub repository with at least one pull request you can use for testing
- An Anthropic API key with access to Claude models
- Node.js 18 or later installed on your local machine
- Python 3.9 or later installed on your local machine
- A GitHub account with repository access

### Local Setup

Clone the repository you want to run automation on:

```bash
git clone https://github.com/your-username/your-repo.git
cd your-repo
```

Create a virtual environment and install the required packages:

```bash
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
```

Create a `requirements.txt` file with the following content:

```txt
pytest>=8.0.0
black>=24.0.0
ruff>=0.3.0
```

Install the dependencies:

```bash
pip install -r requirements.txt
```

### Environment Variables

You need to configure the `ANTHROPIC_API_KEY` environment variable in your GitHub repository secrets. Do not hardcode the key in any file.

Navigate to your GitHub repository, then go to Settings, Secrets and variables, Actions. Add a new repository secret with the name `ANTHROPIC_API_KEY` and your API key as the value.

For local testing, set the variable in your terminal:

```bash
export ANTHROPIC_API_KEY="your-api-key-here"
```

On Windows PowerShell:

```powershell
$env:ANTHROPIC_API_KEY = "your-api-key-here"
```

## The Build -- Step by Step

You will create a GitHub Actions workflow file that runs Claude Code on every pull request. The workflow will review the code diff, generate tests for new functions, and update documentation.

### Step 1: Create the Workflow Directory and File

GitHub Actions workflows live in the `.github/workflows/` directory. Create this directory and a new workflow file.

```bash
mkdir -p .github/workflows
```

Create the file `.github/workflows/claude-code-review.yml` with the following initial content:

```yaml
name: Claude Code PR Review

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  claude-review:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4
        with:
          fetch-depth: 0
```

This workflow triggers on two pull request events: `opened` (when a new PR is created) and `synchronize` (when new commits are pushed to an existing PR). The `fetch-depth: 0` in the checkout step ensures the full git history is available, which Claude Code needs to understand the context of changes.

Expected output: The file is created with no errors. GitHub will automatically detect the workflow file on the next push.

### Step 2: Install Claude Code in the Workflow

Add a step to install Claude Code. Claude Code is distributed as an npm package.

```yaml
      - name: Install Claude Code
        run: npm install -g @anthropic-ai/claude-code
```

Add this step after the checkout step. The `-g` flag installs Claude Code globally so it is available as the `claude` command.

Expected output: The workflow installs Claude Code successfully. The installation takes approximately 30 seconds.

### Step 3: Set Up the API Key

Add an environment variable step that reads the API key from GitHub Secrets. This step also verifies that the key is present.

```yaml
      - name: Verify API key
        run: |
          if [ -z "${{ secrets.ANTHROPIC_API_KEY }}" ]; then
            echo "Error: ANTHROPIC_API_KEY is not set"
            exit 1
          fi
          echo "API key is set"
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

Expected output: If the secret is configured, you see "API key is set" in the workflow logs. If it is missing, the workflow fails with a clear error message.

### Step 4: Run Claude Code for PR Review

This is the core step. You will invoke Claude Code with a prompt that asks it to review the PR diff.

```yaml
      - name: Claude Code PR Review
        id: claude-review
        run: |
          claude -p "Review the changes in this pull request. Focus on:
          1. Missing tests for new functions
          2. Code style violations
          3. Potential bugs or edge cases
          4. Missing documentation
          5. Security issues

          Output a JSON report with:
          - summary: one-paragraph overview
          - issues: list of issues found with severity (low/medium/high)
          - files_reviewed: list of files examined
          - recommendations: actionable suggestions

          The output must be valid JSON." \
            --dangerously-skip-permissions \
            --allowedTools Read,Grep,Glob \
            --output-format json \
            --shutdown-timeout 180000 \
            --max-threads 1 > claude-review-output.json 2>&1
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

The prompt is specific and structured. It asks for five categories of review and specifies the exact output format. The flags are critical:

- `--dangerously-skip-permissions` allows unattended operation
- `--allowedTools Read,Grep,Glob` restricts Claude Code to read-only tools
- `--output-format json` produces structured output
- `--shutdown-timeout 180000` kills the process after 3 minutes
- `--max-threads 1` prevents parallel operations

Expected output: The command creates a file called `claude-review-output.json` in the working directory containing the JSON review report.

### Step 5: Parse the JSON Output and Post a Comment

Add a step that reads the JSON output and posts it as a comment on the pull request. This step uses the GitHub CLI (`gh`) which is pre-installed on GitHub Actions runners.

```yaml
      - name: Post review comment
        run: |
          if [ ! -f claude-review-output.json ]; then
            echo "No review output found"
            exit 0
          fi

          SUMMARY=$(jq -r '.summary // "No summary provided"' claude-review-output.json)
          ISSUES=$(jq -r '.issues // [] | .[] | "- [\(.severity)] \(.description)"' claude-review-output.json 2>/dev/null || echo "No issues found")
          RECOMMENDATIONS=$(jq -r '.recommendations // [] | .[] | "- \(.)"' claude-review-output.json 2>/dev/null || echo "No recommendations")

          COMMENT="## Claude Code Review\n\n### Summary\n$SUMMARY\n\n### Issues Found\n$ISSUES\n\n### Recommendations\n$RECOMMENDATIONS\n\n_This review was generated automatically._"

          gh pr comment ${{ github.event.pull_request.number }} --body "$COMMENT"
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

The `jq` commands extract the summary, issues, and recommendations from the JSON output. The `gh pr comment` command posts the formatted comment to the PR. The `GITHUB_TOKEN` secret is automatically provided by GitHub Actions.

Expected output: A comment appears on the pull request with the Claude Code review results. The comment includes a summary, a list of issues with severity levels, and actionable recommendations.

### Step 6: Add Test Generation for Changed Functions

Add a second Claude Code invocation that generates tests for any new or modified functions in the PR. This step runs after the review step and generates test files.

```yaml
      - name: Generate tests for changed files
        id: generate-tests
        run: |
          claude -p "For each Python file changed in this pull request, check if there are corresponding tests. If tests are missing, generate them following the project's test conventions. Output a JSON list of:
          - file: the source file examined
          - test_file: the test file created (or existing)
          - status: created/exists/skipped
          - coverage_estimate: approximate coverage percentage

          Save generated test files using Write tool." \
            --dangerously-skip-permissions \
            --allowedTools Read,Write,Grep,Glob \
            --output-format json \
            --shutdown-timeout 300000 > claude-test-output.json 2>&1
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

This step allows `Write` in addition to read-only tools because Claude Code needs to create new test files. The timeout is longer (5 minutes) because test generation is more complex than review.

Expected output: A file called `claude-test-output.json` containing the test generation report. New test files are created in the repository. Note that changes to the working directory are not automatically committed -- you need an explicit commit step.

### Step 7: Commit and Push Generated Changes

Add a step that commits any files Claude Code created and pushes them to a new branch. This creates a separate PR that the team can review.

```yaml
      - name: Create PR with generated changes
        if: success()
        run: |
          BRANCH_NAME="claude-auto-$(date +%s)"

          git config user.name "Claude Code Automation"
          git config user.email "claude-code@example.com"

          git checkout -b "$BRANCH_NAME"

          if git diff --quiet; then
            echo "No changes to commit"
            exit 0
          fi

          git add -A
          git commit -m "Automated changes from Claude Code review

          Generated tests and documentation updates
          based on PR #${{ github.event.pull_request.number }}"

          git push origin "$BRANCH_NAME"

          gh pr create \
            --title "Automated changes for PR #${{ github.event.pull_request.number }}" \
            --body "This PR contains changes generated automatically by Claude Code:

          - Generated tests for new or modified functions
          - Updated documentation to reflect code changes

          Please review before merging." \
            --base "${{ github.base_ref }}" \
            --head "$BRANCH_NAME"
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

This step creates a new branch, commits any changes Claude Code made, pushes the branch, and opens a new pull request. The branch name includes a timestamp to ensure uniqueness.

Expected output: A new pull request appears in the repository containing the auto-generated test files and documentation updates.

### Step 8: Full Workflow File

Here is the complete workflow file. All steps combined:

```yaml
name: Claude Code PR Review

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  claude-review:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Install dependencies
        run: |
          pip install pytest black ruff
          npm install -g @anthropic-ai/claude-code

      - name: Verify API key
        run: |
          if [ -z "${{ secrets.ANTHROPIC_API_KEY }}" ]; then
            echo "Error: ANTHROPIC_API_KEY is not set"
            exit 1
          fi

      - name: Claude Code PR Review
        id: claude-review
        run: |
          claude -p "Review the changes in this pull request. Focus on:
          1. Missing tests for new functions
          2. Code style violations
          3. Potential bugs or edge cases
          4. Missing documentation
          5. Security issues

          Output a JSON report with summary, issues, files_reviewed, and recommendations." \
            --dangerously-skip-permissions \
            --allowedTools Read,Grep,Glob \
            --output-format json \
            --shutdown-timeout 180000 > claude-review-output.json 2>&1
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}

      - name: Post review comment
        run: |
          if [ -f claude-review-output.json ]; then
            SUMMARY=$(jq -r '.summary' claude-review-output.json)
            gh pr comment ${{ github.event.pull_request.number }} --body "## Claude Code Review\n\n$SUMMARY"
          fi
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

      - name: Generate tests
        id: generate-tests
        run: |
          claude -p "For each Python file changed in this PR, generate tests if missing.
          Output JSON with file, test_file, status, and coverage_estimate." \
            --dangerously-skip-permissions \
            --allowedTools Read,Write,Grep,Glob \
            --output-format json \
            --shutdown-timeout 300000 > claude-test-output.json 2>&1
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}

      - name: Commit and create PR
        if: success()
        run: |
          BRANCH="claude-auto-$(date +%s)"
          git config user.name "Claude Code Automation"
          git config user.email "claude-code@example.com"
          git checkout -b "$BRANCH"
          if ! git diff --quiet; then
            git add -A
            git commit -m "Automated changes from Claude Code"
            git push origin "$BRANCH"
            gh pr create --title "Automated changes" --body "Auto-generated by Claude Code" \
              --base "${{ github.base_ref }}" --head "$BRANCH"
          fi
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

## Common Errors at This Stage

### Error 1: API Key Not Found

```
Error: ANTHROPIC_API_KEY environment variable is not set
Claude Code requires an API key to authenticate.
```

**Cause:** The `ANTHROPIC_API_KEY` secret is not configured in the GitHub repository secrets, or the environment variable is not being passed to the Claude Code step.

**Fix:** Go to your GitHub repository Settings, then Secrets and variables, then Actions. Add a new repository secret called `ANTHROPIC_API_KEY` with your API key as the value. Make sure the `env:` block in the workflow step references `${{ secrets.ANTHROPIC_API_KEY }}` correctly.

### Error 2: Permission Denied

```
Permission denied: Claude Code attempted to use Write tool but it is not in the allowed tools list.
```

**Cause:** The `--allowedTools` flag restricts Claude Code to specific tools. If a step needs to create files but `Write` is not listed, Claude Code cannot do it.

**Fix:** Add `Write` to the `--allowedTools` list for steps that need to create or modify files. For review-only steps, keep the list read-only (Read, Grep, Glob). Match the allowed tools to the task requirements.

### Error 3: JSON Parsing Failure

```
parse error: Unmatched delimiter at line 3, column 1
jq: error: Cannot iterate over null
```

**Cause:** The Claude Code output was truncated or malformed, or the process was killed by the timeout before finishing. The CI pipeline then tries to parse incomplete JSON and fails.

**Fix:** Increase `--shutdown-timeout` to give Claude Code more time to complete. Add a validation check in the workflow that verifies the JSON is valid before trying to parse it. For example: `jq empty claude-review-output.json || echo "Invalid JSON"`.

### Error 4: Workflow Trigger Not Firing

```
No workflow runs found for the latest push.
```

**Cause:** The workflow file has a syntax error or the `on:` trigger conditions do not match the actual event. For example, if the trigger is `pull_request` but you pushed to a branch without opening a PR, the workflow will not run.

**Fix:** Check the workflow file syntax using `actionlint` or the GitHub Actions tab in the repository. Make sure the trigger event matches what you are testing. Use `workflow_dispatch` as an additional trigger so you can run the workflow manually for testing.

### Error 5: Git Push Fails with Authentication Error

```
remote: Permission to user/repo.git denied to github-actions[bot]
fatal: unable to access 'https://github.com/user/repo.git/': The requested URL returned error: 403
```

**Cause:** The `GITHUB_TOKEN` provided by GitHub Actions does not have permission to push to the repository. This can happen if the repository has branch protection rules that restrict who can push.

**Fix:** Use a personal access token instead of the default `GITHUB_TOKEN`. Create a fine-grained personal access token with `contents: write` and `pull requests: write` permissions, add it as a repository secret, and use it in the workflow.

### Error 6: Rate Limit Exceeded

```
Error: 429 Too Many Requests
Rate limit exceeded for API key.
```

**Cause:** Multiple concurrent workflow runs are hitting the Anthropic API rate limit. This is common when several PRs are opened at the same time.

**Fix:** Implement a queue or delay mechanism. Use `--shutdown-timeout` to bound each run. Add a retry step in the workflow that waits and retries on 429 responses. Consider staggering CI triggers by adding a random delay before Claude Code runs.

## What You Just Built

You built a complete GitHub Actions workflow that automates development tasks with Claude Code. Here is what your pipeline does:

- **Automated PR review.** Every pull request is reviewed by Claude Code for missing tests, code style violations, potential bugs, missing documentation, and security issues. The review results are posted as a comment on the PR.

- **Automated test generation.** Claude Code examines the changed files in each PR and generates test files for any new or modified functions. The generated tests follow your project's existing test conventions.

- **Automated PR creation.** Changes generated by Claude Code -- test files, documentation updates -- are committed to a new branch and opened as a separate pull request for the team to review. This keeps the generated changes separate from the original PR.

- **Structured output for CI integration.** The JSON output format allows the CI pipeline to parse Claude Code results programmatically and take different actions based on the content.

- **Safety boundaries with allowed tools.** The read-only review step uses `--allowedTools Read,Grep,Glob` to prevent any modifications during the review phase. The test generation step adds `Write` only when file creation is required.

- **Cost control with timeouts.** The `--shutdown-timeout` flag ensures that no Claude Code run exceeds its budgeted time, preventing runaway costs.
