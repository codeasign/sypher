In this exercise, you connect the Weather MCP server from the previous lesson to Claude, verify it independently with the MCP Inspector, and then debug the connection errors you are most likely to hit along the way. By the end, you will have Claude actually calling `get_forecast` and `get_alerts` and returning real weather data.

## Setup

Before you start, make sure you have the Weather MCP server from the previous lesson on disk, in its own project directory. This exercise assumes it is a Python server built with the `mcp` SDK's `FastMCP` class, exposing two tools: `get_forecast(latitude, longitude)` and `get_alerts(state)`.

You will need:

- **Python 3.10 or higher** (`python --version`)
- **uv**, the Python package and project manager used by the MCP quickstart (`uv --version`; install with `curl -LsSf https://astral.sh/uv/install.sh | sh` on macOS/Linux, or `powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"` on Windows)
- **Node.js 18 or higher**, only needed to run the MCP Inspector via `npx` (`node --version`)
- **Claude Desktop** installed and updated to its latest version, or **Claude Code** installed via npm

If you have not kept your weather project from the previous lesson, recreate the minimum version here:

```bash
uv init weather
cd weather
uv venv
```

Activate the virtual environment. On macOS/Linux:

```bash
source .venv/bin/activate
```

On Windows:

```powershell
.venv\Scripts\activate
```

Install dependencies:

```bash
uv add "mcp[cli]" httpx
```

Create `weather.py` with this content:

```python
from typing import Any

import httpx
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("weather")

NWS_API_BASE = "https://api.weather.gov"
USER_AGENT = "weather-app/1.0"


async def make_nws_request(url: str) -> dict[str, Any] | None:
    """Make a request to the NWS API with proper error handling."""
    headers = {"User-Agent": USER_AGENT, "Accept": "application/geo+json"}
    async with httpx.AsyncClient() as client:
        try:
            response = await client.get(url, headers=headers, timeout=30.0)
            response.raise_for_status()
            return response.json()
        except Exception:
            return None


def format_alert(feature: dict) -> str:
    """Format an alert feature into a readable string."""
    props = feature["properties"]
    return f"""
Event: {props.get("event", "Unknown")}
Area: {props.get("areaDesc", "Unknown")}
Severity: {props.get("severity", "Unknown")}
Description: {props.get("description", "No description available")}
Instructions: {props.get("instruction", "No specific instructions provided")}
"""


@mcp.tool()
async def get_alerts(state: str) -> str:
    """Get weather alerts for a US state.

    Args:
        state: Two-letter US state code (e.g. CA, NY)
    """
    url = f"{NWS_API_BASE}/alerts/active/area/{state}"
    data = await make_nws_request(url)

    if not data or "features" not in data:
        return "Unable to fetch alerts or no alerts found."

    if not data["features"]:
        return "No active alerts for this state."

    alerts = [format_alert(feature) for feature in data["features"]]
    return "\n---\n".join(alerts)


@mcp.tool()
async def get_forecast(latitude: float, longitude: float) -> str:
    """Get weather forecast for a location.

    Args:
        latitude: Latitude of the location
        longitude: Longitude of the location
    """
    points_url = f"{NWS_API_BASE}/points/{latitude},{longitude}"
    points_data = await make_nws_request(points_url)

    if not points_data:
        return "Unable to fetch forecast data for this location."

    forecast_url = points_data["properties"]["forecast"]
    forecast_data = await make_nws_request(forecast_url)

    if not forecast_data:
        return "Unable to fetch detailed forecast."

    periods = forecast_data["properties"]["periods"]
    forecasts = []
    for period in periods[:5]:
        forecast = f"""
{period["name"]}:
Temperature: {period["temperature"]}°{period["temperatureUnit"]}
Wind: {period["windSpeed"]} {period["windDirection"]}
Forecast: {period["detailedForecast"]}
"""
        forecasts.append(forecast)

    return "\n---\n".join(forecasts)


def main():
    mcp.run(transport="stdio")


if __name__ == "__main__":
    main()
```

Confirm the server runs on its own before touching any Claude configuration:

```bash
uv run weather.py
```

Expected output — the process starts and then appears to hang, printing nothing:

```text
(no output — the server is waiting on stdin for JSON-RPC messages)
```

That silence is correct. A stdio MCP server does not print a banner; it waits for a client to talk to it. Press `Ctrl+C` to stop it once you have confirmed it does not immediately exit with an error.

Now find the absolute path to your project directory, because every configuration step from here on needs it:

```bash
pwd
```

```text
/home/you/projects/weather
```

On Windows, run the equivalent from Command Prompt:

```powershell
cd
```

```text
C:\Users\you\projects\weather
```

Write this path down. Copy it exactly — you will paste it into JSON in the next step.

## The Build — Step by Step

### Step 1: Verify the server with the MCP Inspector before touching Claude

Before connecting anything to Claude, confirm the server itself works using the MCP Inspector, a standalone tool that speaks MCP without needing any AI client at all.

```bash
npx @modelcontextprotocol/inspector uv run weather.py
```

Expected output:

```text
Starting MCP inspector...
⚙️ Proxy server listening on localhost:6277
🔍 MCP Inspector is up and running at http://localhost:6274
```

Open `http://localhost:6274` in your browser. Click "Connect," then open the "Tools" tab. You should see `get_alerts` and `get_forecast` listed, each with their input schema.

**Checkpoint:** click `get_forecast`, enter `latitude: 47.6062` and `longitude: -122.3321` (Seattle), and click "Run Tool." You should get back a multi-day forecast as plain text. If this step fails, the problem is in your server code, not in Claude's configuration — fix it here first, because nothing downstream will work until this does.

### Step 2: Locate Claude Desktop's configuration file

Find the config file for your platform. On macOS, use a text editor of your choice, or run:

```bash
code ~/Library/Application\ Support/Claude/claude_desktop_config.json
```

On Windows, in PowerShell:

```powershell
code $env:AppData\Claude\claude_desktop_config.json
```

If the file does not exist yet, create it. An empty starting file looks like this:

```json
{
  "mcpServers": {}
}
```

**Checkpoint:** confirm the file opens (or is created) at that exact path before continuing. If your editor created it somewhere else, Claude Desktop will never see it.

### Step 3: Add the weather server entry

Add your server under the `mcpServers` key, using the absolute path you copied earlier. On macOS/Linux:

```json
{
  "mcpServers": {
    "weather": {
      "command": "uv",
      "args": [
        "--directory",
        "/home/you/projects/weather",
        "run",
        "weather.py"
      ]
    }
  }
}
```

On Windows, use double backslashes or forward slashes — a single backslash inside a JSON string is an escape character and will corrupt the path:

```json
{
  "mcpServers": {
    "weather": {
      "command": "uv",
      "args": [
        "--directory",
        "C:\\Users\\you\\projects\\weather",
        "run",
        "weather.py"
      ]
    }
  }
}
```

If `uv` is not on Claude's PATH, you may need the full path to the `uv` executable instead of just `"uv"`. Find it with:

```bash
which uv
```

or on Windows:

```powershell
where.exe uv
```

**Checkpoint:** run the JSON through a validator before saving — a trailing comma or unmatched brace here is the single most common reason a server never appears in Claude at all.

```bash
python -m json.tool < claude_desktop_config.json
```

Expected output is the same JSON printed back to you, confirming it parses. Any `JSONDecodeError` here must be fixed before restarting Claude.

### Step 4: Add environment variables (if your server needs them)

The weather server above needs no API key, since the National Weather Service API is free and keyless. To see how you would pass a secret to a server that does need one, add a placeholder entry using `${VAR_NAME}` expansion rather than a literal value:

```json
{
  "mcpServers": {
    "weather": {
      "command": "uv",
      "args": [
        "--directory",
        "/home/you/projects/weather",
        "run",
        "weather.py"
      ],
      "env": {
        "WEATHER_LOG_LEVEL": "${WEATHER_LOG_LEVEL}"
      }
    }
  }
}
```

Set the variable in your shell before launching Claude Desktop (on most platforms, Claude Desktop inherits environment variables from the shell or session that launches it):

```bash
export WEATHER_LOG_LEVEL="debug"
```

**Checkpoint:** never write a real secret value directly in this file — always reference it with `${VAR_NAME}` and set the actual value in your shell environment or OS-level environment settings.

### Step 5: Restart Claude Desktop completely and verify discovery

Fully quit Claude Desktop — not just close the window. On Windows, right-click the Claude icon in the system tray and select "Quit." On macOS, use `Cmd+Q` or "Quit Claude" from the menu bar. Then relaunch the app.

Look for the connector icon in the message composer (a plus icon, or "Add files, connectors, and more"). Click it and hover over the connectors menu.

**Checkpoint:** you should see `weather` listed with its two tools. If it does not appear, proceed straight to the "Common Errors at This Stage" section below before continuing.

### Step 6: Connect the same server to Claude Code

If you also use Claude Code from the terminal, register the same server there instead of hand-editing a JSON file:

```bash
claude mcp add weather -- uv run --directory /home/you/projects/weather weather.py
```

Expected output:

```text
Added stdio MCP server "weather" to local config
```

Verify it registered correctly:

```bash
claude mcp list
```

Expected output:

```text
weather: uv run --directory /home/you/projects/weather weather.py - ✓ Connected
```

**Checkpoint:** if the status shows anything other than "Connected," run `claude mcp get weather` to see the specific failure reason before moving on.

To share this server definition with a team by committing it to version control instead of keeping it personal, use project scope:

```bash
claude mcp add --scope project weather -- uv run --directory /home/you/projects/weather weather.py
```

This writes the entry to `.mcp.json` in your project root instead of your personal `~/.claude.json`.

### Step 7: Ask Claude to call your tools and observe what happens

Start a conversation in Claude Desktop or Claude Code and ask a question that only your tool can answer:

```text
What's the weather forecast for Seattle? Use latitude 47.6062, longitude -122.3321.
```

Expected behavior: Claude recognizes that `get_forecast` matches the request, shows you a permission prompt (in Claude Code) or an inline "used weather" indicator (in Claude Desktop) naming the tool and its arguments, and after you approve it, responds with a natural-language summary built from the tool's returned forecast text.

```text
Claude: I'll check the forecast for that location.

[Uses weather - get_forecast]

Here's the forecast for Seattle:
- Tonight: Partly cloudy, low around 52°F, light SW wind
- Tomorrow: Mostly sunny, high near 68°F
...
```

**Checkpoint:** this is the end state of the exercise — a full round trip from your question, through Claude's reasoning, through the tool call, to your own server's code, and back. If you get here, the connection is fully working.

## Common Errors at This Stage

**Error: server does not appear in the connectors menu at all, with no error shown**

```text
(no error — "weather" is simply absent from the list)
```

Cause: almost always a JSON syntax error in the config file, or the file living at the wrong path. Claude Desktop fails silently rather than showing a parse error in the main window.

Fix: validate the JSON with `python -m json.tool < claude_desktop_config.json` and confirm the file path matches exactly what your platform expects (`%APPDATA%\Claude\claude_desktop_config.json` on Windows).

**Error: server shows a red or gray "disconnected" indicator**

```text
weather: uv run --directory /home/you/projects/weather weather.py - ✗ Failed to connect
```

Cause: the `command` field points to an executable Claude cannot find (often because `uv` is not on Claude's PATH, even though it is on yours), or the working directory Claude spawns from does not match the relative path you assumed.

Fix: replace `"command": "uv"` with the full absolute path from `which uv` / `where.exe uv`, and confirm every path in `args` is absolute, not relative.

**Error: `JSONDecodeError: Expecting ',' delimiter` when validating the config file**

```text
json.decoder.JSONDecodeError: Expecting ',' delimiter: line 8 column 5 (char 142)
```

Cause: a missing comma between two server entries, or a trailing comma after the last entry in an object — both are invalid JSON even though trailing commas are fine in JavaScript.

Fix: run the file through a JSON validator and fix the reported line. Re-run the validator until it prints the parsed structure back with no error.

**Error: tool calls fail with "server has no tools" or the tools list is empty**

```text
weather: ✓ Connected (0 tools)
```

Cause: the `@mcp.tool()` decorator was not applied to your function, the function was defined after `mcp.run()` was called, or an exception during tool registration was silently swallowed.

Fix: run the server standalone through the MCP Inspector (`npx @modelcontextprotocol/inspector uv run weather.py`) and check the "Tools" tab — if the Inspector also shows zero tools, the bug is in your server code, not your Claude configuration.

**Error: environment variable not resolved, server crashes on a missing key**

```text
KeyError: 'WEATHER_LOG_LEVEL'
```

Cause: the `${VAR_NAME}` in your config was not matched by any variable actually set in the environment Claude launched from — commonly because you set it in one terminal session and then launched Claude Desktop from a completely different login session that never inherited it.

Fix: for Claude Code, confirm with `claude mcp get weather` that the environment block resolved as expected. For Claude Desktop, set persistent environment variables at the OS level (System Properties on Windows, your shell profile plus a full logout/login on macOS) rather than a one-off `export` in a terminal Claude Desktop was not launched from.

## What You Just Built

- A verified, standalone check of your MCP server using the MCP Inspector, completely independent of any AI client
- A working `claude_desktop_config.json` entry that lets Claude Desktop discover, launch, and talk to your weather server over stdio
- The same server registered with Claude Code using `claude mcp add`, plus the project-scoped `--scope project` variant for sharing with a team
- A safe pattern for passing environment variables to an MCP server using `${VAR_NAME}` expansion instead of hardcoded secrets
- A debugging workflow that isolates server-code bugs (caught by the Inspector) from configuration and transport bugs (caught by `claude mcp list` / `claude mcp get` and JSON validation)
- A full end-to-end tool call, where a natural-language question to Claude triggered your own Python code and returned real weather data
