# Installing Python and Using the Terminal — Practice Exercises

These exercises test and extend what you learned in the overview and practice exercise. They are organised by difficulty. Complete them in order — each tier assumes you can do everything in the previous one.

## Tier 1 — Beginner

These exercises reinforce the basic skills you just learned. If you get stuck, refer back to the overview and practice exercise steps.

### Exercise 1 — Verify Your Environment

Open a terminal and run the following commands. For each one, write down the output and confirm you understand what it means.

```bash
python --version
pip --version
python -c "print('Environment verified')"
```

Now write a one-line Python script called `check.py` that prints the string `"Ready for AI engineering"`. Run it from the terminal.

**Check your work:** The output should be exactly:

```text
Ready for AI engineering
```

If it is not, check that you have quotation marks around the string and parentheses around the argument to `print`.

### Exercise 2 — Directory Structure

Create the following directory structure inside your `ai-learning` folder using only terminal commands:

```
ai-learning/
  section-01/
  section-02/
  section-03/
  notes/
```

Use `mkdir` to create each directory. Then create an empty file called `readme.txt` inside `notes/` using one of these commands:

- On macOS/Linux: `touch notes/readme.txt`
- On Windows: `type nul > notes\readme.txt` (in Command Prompt) or `New-Item notes/readme.txt` (in PowerShell)

List the contents of `notes/` to confirm the file exists. On macOS/Linux use `ls notes/`. On Windows use `dir notes`.

**Check your work:** The listing should show `readme.txt`.

### Exercise 3 — Navigate and Confirm Your Location

Open a terminal and practice moving around your file system using only commands. Start from your home directory, then navigate into the `ai-learning` directory you created earlier. Confirm you are in the right place by printing the current working directory path.

From inside `ai-learning`, create a subdirectory called `practice` and move into it. Then move back up to `ai-learning` using `..`.

**Check your work:**

On macOS/Linux:

```bash
pwd
```

Should show something like `/home/yourname/ai-learning`.

On Windows:

```bash
cd
```

Should show something like `C:\Users\YourName\ai-learning`.

If you cannot find your `ai-learning` folder, search for it:

On macOS/Linux:

```bash
find ~ -type d -name "ai-learning"
```

On Windows Command Prompt:

```bash
dir /s ai-learning
```

### Exercise 4 — The REPL as a Calculator

Start the Python REPL and use it as a calculator to compute:

1. 15 multiplied by 37
2. 1024 divided by 8
3. 2 raised to the power of 10 (hint: `**` is the exponent operator)
4. The remainder when 100 is divided by 7 (hint: `%` is the modulo operator)

Exit the REPL when you are finished.

<details>
<summary>Hints</summary>

```python
>>> 15 * 37
555
>>> 1024 / 8
128.0
>>> 2 ** 10
1024
>>> 100 % 7
2
```

The `**` operator raises a number to a power. The `%` operator gives the remainder after division.
</details>

### Exercise 5 — Run a Script With Multiple Lines

Create a new file called `multiline.py` in your `first-scripts` directory with the following content:

```python
# This is a comment — Python ignores lines starting with #
# Comments help you and others understand the code

print("Line 1: Starting the script")
print("Line 2: Still going")

print("Line 3: Almost done")
```

Run it from the terminal. Then create a copy called `multiline_copy.py` using the terminal:

- On macOS/Linux: `cp multiline.py multiline_copy.py`

Run `multiline_copy.py` to confirm the copy works.

**Check your work:** Both files should produce the same four lines of output.

## Tier 2 — Intermediate

These exercises require combining the skills you learned in new ways. Read error messages carefully — they often tell you exactly what to fix.

### Exercise 6 — Explore Python's Built-in Help

The Python REPL includes a built-in help system. Start the REPL and type:

```python
>>> help()
```

This opens the interactive help utility. Type `modules` and press Enter to see a list of every Python module available on your system. This list is long — press Enter to scroll line by line, or press Space to scroll page by page. Type `q` to quit the list.

Now ask for help on a specific function:

```python
>>> help(print)
```

Read the documentation for the `print` function. Pay attention to the `sep` and `end` parameters. After reading, press `q` to exit help. Then exit the REPL.

Outside the REPL, create a file called `custom_print.py` that uses `sep` and `end` to print `1-2-3-4` on a single line. The `sep` parameter sets the separator between multiple arguments. The `end` parameter sets what comes after the last argument (default is a newline).

<details>
<summary>Solution</summary>

```python
print(1, 2, 3, 4, sep="-", end="")
```

Running `python custom_print.py` produces `1-2-3-4` with no trailing newline.
</details>

Open your terminal and make sure you are in your home directory:

On macOS/Linux:

On Windows (PowerShell):

### Exercise 8 — Write a Greeting Script That Uses Input

Create a file called `greet.py` with this content:

```python
name = input("What is your name? ")
print(f"Hello, {name}! Welcome to AI engineering.")
```

Run the script. It will wait for you to type your name and press Enter. Type your name and press Enter.

**Expected output** (if you type "Alice"):

```text
What is your name? Alice
Hello, Alice! Welcome to AI engineering.
```

Now modify the script to ask for the user's name and favourite number, then print a message combining both. For example: "Alice, your favourite number is 42."

<details>
<summary>Solution</summary>

```python
name = input("What is your name? ")
number = input("What is your favourite number? ")
print(f"{name}, your favourite number is {number}.")
```

Note: `input()` always returns a string. If you need to do arithmetic with the number, you would convert it with `int()` or `float()`.
</details>

### Exercise 9 — Create a Simple Log File

Create a file called `logger.py` in your `first-scripts` directory with this content:

```python
from datetime import datetime

timestamp = datetime.now().strftime("%Y-%m-%d %H:%M:%S")
message = f"[{timestamp}] Script ran successfully"

print(message)

with open("run.log", "a") as log_file:
    log_file.write(message + "\n")

print("Logged to run.log")
```

Run this script twice, waiting a few seconds between runs. Then check the contents of `run.log`:

On macOS/Linux:

```bash
cat run.log
```

On Windows:

```bash
type run.log
```

**Expected output** (timestamps will differ):

```text
[2026-07-03 14:30:01] Script ran successfully
[2026-07-03 14:30:15] Script ran successfully
```

Each run appends a new line to `run.log`. The `with open(...) as log_file` pattern automatically closes the file when the block finishes.

## Tier 3 — Advanced

These exercises challenge you to combine multiple skills and solve problems without step-by-step instructions. Read the problem carefully and break it down into pieces you already know.

### Exercise 10 — Terminal-Based Project Generator

Write a Python script called `project_generator.py` that automates the creation of a standard project directory structure. When run, it should:

1. Ask the user for a project name (using `input()`)
2. Create a directory with that name
3. Inside that directory, create these subdirectories: `data`, `scripts`, `notebooks`, `outputs`
4. Inside the `scripts` directory, create an empty file called `__init__.py`
5. Inside the `notebooks` directory, create an empty file called `.gitkeep`
6. Print a summary of what was created

Run the script with the project name `my-first-project` and confirm the structure was created correctly using terminal commands to list the directories.

<details>
<summary>Solution</summary>

```python
import os

project_name = input("Enter project name: ")

os.makedirs(f"{project_name}/data", exist_ok=True)
os.makedirs(f"{project_name}/scripts", exist_ok=True)
os.makedirs(f"{project_name}/notebooks", exist_ok=True)
os.makedirs(f"{project_name}/outputs", exist_ok=True)

with open(f"{project_name}/scripts/__init__.py", "w") as f:
    pass

with open(f"{project_name}/notebooks/.gitkeep", "w") as f:
    pass

print(f"Project '{project_name}' created with:")
print("  - data/")
print("  - scripts/__init__.py")
print("  - notebooks/.gitkeep")
print("  - outputs/")
```

Run it with `python project_generator.py`, enter `my-first-project`, and verify with `ls my-first-project` (or `dir my-first-project` on Windows).
</details>

### Exercise 11 — Diagnose and Fix a Broken Script

Below is a script called `broken.py` that contains several errors. Write it to a file exactly as shown, then diagnose and fix each error.

```python
print("Starting diagnostics")
print("Step 1: Checking Python version"
print("Step 2: This line has a problem
      print("Step 3: This line is indented wrong")
prnit("Step 4: This has a typo")
  print("Step 5: Unexpected indent")
print("Step 6: Done")
```

Try to run it:

```bash
python broken.py
```

You will see a `SyntaxError` pointing to line 3. Fix that error, save the file, and run again. A new error will appear on a different line. Fix each error one at a time until the script runs without errors and prints all six lines.

<details>
<summary>Solution</summary>

The fixed script should look like this:

```python
print("Starting diagnostics")
print("Step 1: Checking Python version")
print("Step 2: This line has a problem")
print("Step 3: This line is indented wrong")
print("Step 4: This has a typo")
print("Step 5: Unexpected indent")
print("Step 6: Done")
```

The errors were:
- Line 3: Missing closing parenthesis `)`
- Line 4: Missing closing parenthesis and closing quotation mark
- Line 5: Unexpected indentation (no block context requires it)
- Line 6: Typo — `prnit` should be `print`
- Line 7: Unexpected indentation
</details>

### Exercise 12 — Command History and Terminal Tricks

Modern terminals have features that make your life easier. Practice each of these:

1. **Command history:** Press the Up arrow key several times. You should see previous commands reappear. Press Enter to re-run one. Press Down to move forward through history.

2. **Tab completion:** Type `pyt` and press Tab. If Python is the only command starting with `pyt`, the terminal should complete it to `python`. If there are multiple matches, press Tab twice to see them.

3. **Clear the screen:** If your terminal feels cluttered, type `clear` (macOS/Linux) or `cls` (Windows) to clear it.

4. **Cancel a command:** If you start typing a command and change your mind, press `Ctrl+C` to cancel it. Try it: type `python` (without pressing Enter), then press `Ctrl+C`. You should be returned to a fresh prompt.

Write a short summary in a file called `terminal-tricks.txt` listing the three shortcuts you found most useful. Use the terminal to create this file — not a graphical editor.

<details>
<summary>Hint</summary>

On macOS/Linux, create the file with:

```bash
cat > terminal-tricks.txt
```

Type your summary, then press `Ctrl+D` to save and exit.

On Windows Command Prompt:

```bash
copy con terminal-tricks.txt
```

Type your summary, then press `Ctrl+Z` then Enter to save.

On Windows PowerShell:

```bash
Set-Content terminal-tricks.txt "Line 1`nLine 2"
```
</details>

## What to Do If You Are Stuck

Every AI engineer — including professionals with years of experience — gets stuck on setup issues at some point. When something does not work:

1. **Read the error message.** It is not random noise. It tells you the file, the line number, and usually the kind of problem. The most useful skill you can develop is reading error messages carefully.

2. **Check your spelling.** `prnit` does not work. `Print` with a capital P does not work (Python is case-sensitive). Missing colons, parentheses, and quotation marks are responsible for the majority of beginner errors.

3. **Check your directory.** Are you in the same folder as your script? Run `pwd` (macOS/Linux) or `cd` (Windows) to confirm.

4. **Re-run the exercise from scratch.** If something is fundamentally broken, sometimes the fastest path forward is to delete a misconfigured installation and start over. Python can be installed and uninstalled without harming your computer.

5. **Use the REPL to test small pieces.** If a script is not working, copy the problematic line into the REPL and see what happens. The REPL gives you instant feedback without needing to re-run the whole script.
