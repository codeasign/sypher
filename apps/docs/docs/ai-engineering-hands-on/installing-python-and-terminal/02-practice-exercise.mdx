# Installing Python and Using the Terminal — Practice Exercise

Now you will apply everything from the overview. By the end of this exercise, you will have Python installed and running, a terminal workflow you can navigate comfortably, and a verified setup ready for the next section.

## Setup

Before you begin, make sure you have:

- A computer with internet access
- Administrator or install permission on your machine
- About 15 to 20 minutes of uninterrupted time

Open a terminal window now. If you are on Windows, open PowerShell or Command Prompt from the Start menu. On macOS, open Terminal from Applications > Utilities. On Linux, use your terminal emulator (Ctrl+Alt+T usually works).

Keep this window open — you will use it for every step.

## The Build

This exercise has 6 steps. Complete each one before moving to the next. Expected output is shown after each command so you know whether it worked.

### Step 1 — Check What Python You Have (If Any)

Before installing anything, check whether Python is already on your system. This avoids installing a second copy unnecessarily.

Run in your terminal:

```bash
python --version
```

**Expected outcome one** — Python is not installed:

```text
'python' is not recognized as an internal or external command,
operable program or batch file.
```

**Expected outcome two** — Python is installed and the version is 3.12 or higher:

```text
Python 3.12.4
```

**Expected outcome three** — Python is installed but uses the `python3` name (common on macOS and Linux):

```bash
python3 --version
```

```text
Python 3.12.4
```

If you received outcome one, proceed to Step 2 to install Python. If you received outcome two or three and the version is 3.12 or higher, you can skip to Step 3. If the version is older than 3.12, it is worth upgrading so everything in this course works without compatibility surprises.

### Step 2 — Install Python

**Windows instructions:**

Visit python.org and download the latest Python 3.12 or 3.13 installer for Windows. Run the installer. At the very first screen, find the checkbox that says "Add Python to PATH" and make sure it is checked. Then click "Install Now."

After installation completes, click "Close." Open a new terminal window (the PATH change only applies to new terminal sessions) and verify:

```bash
python --version
```

```text
Python 3.12.4
```

**macOS instructions:**

Visit python.org and download the latest Python 3.12 or 3.13 installer for macOS. Run the package installer and follow the prompts. After installation, open a new terminal and verify:

```bash
python3 --version
```

```text
Python 3.12.4
```

**Linux (Ubuntu/Debian) instructions:**

Open a terminal and run:

```bash
sudo apt update
sudo apt install python3 python3-pip python3-venv
```

Then verify:

```bash
python3 --version
```

```text
Python 3.12.x
```

If you are on Fedora, use `sudo dnf install python3` instead. On Arch Linux, use `sudo pacman -S python python-pip`.

### Step 3 — Check That pip Works

pip is the Python package installer. Verify it is functional:

On Windows:

```bash
pip --version
```

```text
pip 24.0 from C:\Users\YourName\AppData\Local\Programs\Python\Python312\site-packages\pip (python 3.12)
```

On macOS/Linux:

```bash
pip3 --version
```

```text
pip 24.0 from /usr/local/lib/python3.12/site-packages/pip (python 3.12)
```

The version number may differ — what matters is that pip prints a version and mentions "python 3.12" (or whatever version you installed). If you get a `'pip' is not recognized` error, re-run the Python installer on Windows and make sure pip is included in the installation. On macOS/Linux, install pip through your package manager or with:

```bash
python3 -m ensurepip --upgrade
```

### Step 4 — Create a Project Directory

A project directory keeps your code organised. Build the habit now.

```bash
# Create a top-level folder for your AI learning journey
mkdir ai-learning

# Move into that folder
cd ai-learning

# Create a subfolder for today's work
mkdir first-scripts

# Move into the subfolder
cd first-scripts
```

**Expected outcome:** the `cd` commands should produce no output — they silently change your directory. To confirm where you are:

On macOS/Linux:

```bash
pwd
```

```text
/home/yourname/ai-learning/first-scripts
```

On Windows (using `cd` with no arguments to show the current directory):

```bash
cd
```

```text
C:\Users\YourName\ai-learning\first-scripts
```

### Step 5 — Write and Run Your First Script

Open VS Code (or your preferred text editor). Create a new file called `hello_ai.py` inside the `first-scripts` folder.

Type the following exactly:

```python
print("Hello, AI World!")
print("Python is installed and working.")
```

Save the file. Back in the terminal, make sure you are in the `first-scripts` directory, then run:

On Windows:

```bash
python hello_ai.py
```

On macOS/Linux:

```bash
python3 hello_ai.py
```

**Expected output:**

```text
Hello, AI World!
Python is installed and working.
```

If you see a `SyntaxError`, check for missing quotation marks, mismatched parentheses, or extra spaces before `print`. If you see `No such file or directory` (macOS/Linux) or `The system cannot find the file specified` (Windows), you are in the wrong directory — use `ls` or `dir` to list files, then `cd` to the correct folder.

### Step 6 — Experiment With the REPL

The Python REPL is your sandbox for trying things out. Start it:

```bash
python
```

Or on macOS/Linux:

```bash
python3
```

**Expected output:**

```text
Python 3.12.4 ( ... ) on ...
Type "help", "copyright", "credits" or "license" for more information.
>>>
```

Now type these lines one at a time, pressing Enter after each:

```python
>>> 5 + 3
8
>>> "AI" + " Engineering"
'AI Engineering'
>>> for i in range(3):
...     print(f"Loop {i}")
...
Loop 0
Loop 1
Loop 2
```

The `...` prompt appears when Python expects more input — type the next indented line and press Enter, then press Enter again on a blank line to finish.

When you are done, exit the REPL:

```python
>>> exit()
```

Or press `Ctrl+Z` then Enter on Windows, or `Ctrl+D` on macOS/Linux.

You are back at your terminal prompt. Your environment is fully set up and verified.

## Common Errors

### Error 1 — "Python is not recognized" (Windows)

**What you see:**

```text
'python' is not recognized as an internal or external command,
operable program or batch file.
```

**Why it happens:** The Python executable's directory is not in your system's PATH environment variable. Windows does not know where to find the `python` command.

**Fix:** Re-run the Python installer. At the first screen, check the box that says "Add Python to PATH," then click "Install Now." After installation, close and re-open your terminal — PATH changes only apply to new terminal sessions. Then run `python --version` again.

### Error 2 — "pip is not recognized" (Windows)

**What you see:**

```text
'pip' is not recognized as an internal or external command
```

**Why it happens:** pip was not included in the Python installation, or Python's Scripts directory is not on PATH.

**Fix:** Re-run the installer and ensure pip is selected in the optional features. Alternatively, use the safer invocation that always pairs pip with the correct Python:

```bash
python -m pip --version
```

This runs pip as a module of the Python interpreter, bypassing PATH lookup entirely.

### Error 3 — ModuleNotFoundError When Importing

**What you see:**

```python
>>> import numpy
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
ModuleNotFoundError: No module named 'numpy'
```

**Why it happens:** You installed the package with one Python interpreter but are importing it in a different one. This is the most common environment problem in Python.

**Fix:** Always install packages with the paired command:

```bash
python -m pip install numpy
```

Then import from the same Python:

```bash
python -c "import numpy; print(numpy.__version__)"
```

### Error 4 — SyntaxError: invalid syntax

**What you see:**

```python
>>> print "Hello"
  File "<stdin>", line 1
    print "Hello"
    ^^^^^^^^^^^^
SyntaxError: Missing parentheses in call to 'print'. Did you mean print(...)?
```

**Why it happens:** In Python 3, `print` is a function that requires parentheses. In Python 2, `print` was a statement and did not need them. You are running Python 3, which requires parentheses.

**Fix:** Add parentheses:

```python
>>> print("Hello")
Hello
```

### Error 5 — Permission Denied (macOS/Linux)

**What you see:**

```text
$ pip install numpy
error: externally-managed-environment
This environment is externally managed
```

**Why it happens:** Modern Linux distributions prevent pip from installing system-wide packages to avoid conflicts with the system package manager.

**Fix:** Use a virtual environment (covered in a later lesson) or install with the `--user` flag:

```bash
pip install --user numpy
```

Or use the safer module-based approach:

```bash
python3 -m pip install --user numpy
```

## What You Just Built

You have built a complete, verified Python development environment. Here is what that means concretely:

- **Python is installed** and accessible from your terminal by name (`python` or `python3`)
- **pip is functional** and paired with your Python interpreter — you can install any of the 500,000+ packages on PyPI
- **A project directory exists** with a clean structure (`ai-learning/first-scripts/`) that you can extend for the rest of this course
- **A script file runs** — you wrote `hello_ai.py`, executed it, and saw output. This is the same pattern you will use for every Python program you ever write: create a `.py` file, run it with `python filename.py`
- **You have used the REPL** interactively, which means you have a tool for quick experiments throughout your learning journey

From here, every new skill you learn sits on top of this foundation. When the next lesson says "open your terminal and run Python," you will already know exactly what that means and how to do it.
