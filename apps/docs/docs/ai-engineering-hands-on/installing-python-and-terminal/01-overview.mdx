# Installing Python and Using the Terminal

## What You Will Learn

By the end of this lesson you will be able to:

- Install Python 3.12+ on Windows, macOS, or Linux
- Open and navigate the terminal with basic commands
- Verify your Python installation from the command line
- Write and execute your first Python script
- Understand what the terminal is and why AI engineers use it daily

## Prerequisites

You need a computer and permission to install software. No programming experience is required — this lesson starts from absolute zero.

## Problem Story

You sit down to follow an AI tutorial. The first step says: "Open your terminal and run `python --version`." You stare at your screen. What is a terminal? Where is it? You open the application launcher, search for "terminal" — nothing appears. You try "command prompt" instead. A black window opens with a blinking cursor. You type `python --version` and press Enter. The response is:

```text
'python' is not recognized as an internal or external command,
operable program or batch file.
```

This is the single most common obstacle for new developers. You have the will to learn, but the machine speaks a language you have not learned yet. The error message does not tell you what went wrong or how to fix it. You feel stuck before you have written a single line of code.

This exact scenario stops thousands of aspiring AI engineers every day. The fix is straightforward once you understand what is happening: your computer does not know where Python lives yet. You need to install it, and you need to learn a handful of terminal commands to talk to your computer the way developers do.

By the end of this lesson, not only will you have Python running — you will understand why the terminal is the control center for everything you do in AI engineering.

## The Concept

### What Is a Terminal?

A terminal (also called a command line or shell) is a text-based interface to your computer's operating system. Instead of clicking icons and dragging windows with a mouse, you type commands and see text responses. This may feel like a step backward — why type when you can click? — but the terminal gives you powers that graphical interfaces cannot.

When you click a button in a graphical app, that click triggers a pre-programmed command behind the scenes. The terminal lets you issue those commands directly, without any app in the middle. That means you can automate repetitive tasks, run programs with precise arguments, see exactly what went wrong when something fails, and chain multiple operations together into pipelines.

There are several different terminal programs, but they all work the same basic way. On Windows, you might use Command Prompt or PowerShell. On macOS, the default is simply called Terminal. On Linux, common options include GNOME Terminal, Konsole, or xterm. The differences between them matter very little for what you will do — any of them works fine for running Python.

When you open a terminal, you see a prompt. It might look like this:

```text
C:\Users\YourName>
```

Or on macOS/Linux:

```text
yourname@computer:~$
```

This prompt tells you the terminal is ready to accept a command. The `~` character (tilde) is a shortcut for your home directory. The `$` or `>` symbol is the prompt character — it is not part of the commands you type.

Every command you type is executed by a program called a shell. The shell interprets your typed words, finds the matching program, runs it, and displays the output. The most common shell is called Bash (Bourne Again SHell) on macOS and Linux, and PowerShell on Windows. When you type `python hello_ai.py`, the shell finds the `python` program, hands it the argument `hello_ai.py`, and Python runs your script.

For AI engineering specifically, almost every tool you will use — Python, pip, Jupyter, Git, Docker, cloud CLIs — is designed to be driven from the terminal. The terminal is the common language that connects all your tools.

### What Is the PATH Environment Variable?

The PATH is one of the most important concepts to understand. It is a list of directories that your operating system searches whenever you type a command. When you type `python`, the OS looks through every directory in PATH, in order, until it finds a program called `python`.

You can see your PATH yourself:

On macOS/Linux:

```bash
echo $PATH
```

On Windows:

```bash
echo %PATH%
```

You will see a long string of directories separated by colons (macOS/Linux) or semicolons (Windows). For example:

```text
/usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin
```

If Python's installation directory is not in this list, you get the `'python' is not recognized` error. The installer's "Add Python to PATH" option simply adds Python's directory to this list so your OS can find it.

### What Is Python?

Python is a programming language, which means it is a formal system for giving instructions to a computer. When you write Python code in a file and run it, the Python interpreter reads your instructions, translates them into machine operations, and executes them.

Python is particularly well suited for AI engineering because:

- Its syntax is readable and concise, so you spend less time wrestling with punctuation and more time thinking about logic
- It has an enormous ecosystem of libraries for mathematics (NumPy), data processing (pandas), machine learning (scikit-learn, PyTorch), and AI APIs (OpenAI SDK, Anthropic SDK)
- The community is large and beginner-friendly, so help is always available
- It runs on every major operating system without modification
- It has a REPL (interactive mode) that lets you test ideas instantly without creating a file

Think of Python as the engine and the terminal as the driver's seat. You need both to go anywhere.

### How Python and pip Work Together

Python ships with a tool called pip (Pip Installs Packages) that downloads and installs third-party libraries from the Python Package Index (PyPI). PyPI hosts over 500,000 packages contributed by the community.

When you run `pip install numpy`, pip downloads the package from PyPI, unpacks it, and copies the files into a directory called `site-packages` inside your Python installation. When your script runs `import numpy`, Python looks in `site-packages` for a folder named `numpy`.

The critical detail is that each Python installation has its own `site-packages`. If you have Python 3.11 and Python 3.12 installed, packages installed with one are invisible to the other. This is why `python -m pip install numpy` is safer than bare `pip install numpy` — it guarantees you are using the pip that belongs to the specific Python interpreter you will run your script with.

### Reference Table

| Command / Concept | What It Does | Example |
|---|---|---|
| `python --version` | Prints the installed Python version | `python --version` → `Python 3.12.4` |

| `exit()` or `Ctrl+Z` then Enter (Windows) | Exits the Python REPL back to the terminal | `exit()` |
| `pip install <package>` | Installs a Python package from PyPI | `pip install requests` |

| `mkdir <name>` | Creates a new directory | `mkdir my-project` |
| `pwd` (macOS/Linux) or `cd` alone (Windows) | Prints the current working directory path | `pwd` → `/Users/you/projects` |
| `echo <text>` | Prints text to the terminal | `echo "Hello"` → `Hello` |
| `clear` (macOS/Linux) or `cls` (Windows) | Clears the terminal screen | `clear` |
| `$PATH` (or `%PATH%` on Windows) | An environment variable listing directories where the OS looks for executables | `echo $PATH` shows all executable-search directories |
| Script `#!` (shebang) | A special first line in scripts on macOS/Linux that tells the OS which interpreter to use | `#!/usr/bin/env python3` |

## How It Works

The process of installing Python and running your first script follows a clear, repeatable sequence. Each step builds on the one before it.

### Step 1 — Download Python

Go to the official Python website at python.org and download the latest version of Python 3.12 or newer. On Windows, download the executable installer (the 64-bit version). On macOS, download the macOS 64-bit universal2 installer. On Linux, Python often comes pre-installed, but you may need to install the latest version through your package manager.

On Windows, the installer includes a critical checkbox labeled "Add Python to PATH" — check this box before clicking "Install Now." This is the step that prevents the `'python' is not recognized` error from the problem story. Without it, Windows does not know where to find the `python` command.

The installer also lets you customise the installation location. The default is usually best, but if you change it, make a note of where you installed Python — you may need it later for PATH troubleshooting.

On macOS, the installer adds Python to PATH automatically. After installation, you may need to grant permission for the installer in System Settings > Privacy & Security — look for a message saying the Python installer was blocked and click "Allow."

On Linux, you may need to install the `python3` and `python3-pip` packages:

```bash
# Ubuntu / Debian
sudo apt update
sudo apt install python3 python3-pip python3-venv
```

The `python3-venv` package is worth installing now even though you will not use it until a later lesson — it enables virtual environments, which are the standard way to keep each project's dependencies isolated from others.

### Step 2 — Verify the Installation

Open your terminal. On Windows, search for "Command Prompt" or "PowerShell" in the Start menu. On macOS, open "Terminal" from Applications > Utilities. On Linux, press `Ctrl+Alt+T` or open "Terminal" from the applications menu.

Type the following command and press Enter:

```bash
python --version
```

If you see output like `Python 3.12.4`, the installation succeeded.

On some systems, especially macOS and Linux, you may need to type `python3` instead of `python`:

```bash
python3 --version
```

If you still see an error, your PATH may not include the Python directory. On Windows, re-run the installer and ensure "Add Python to PATH" is checked. On macOS, the installer usually handles this. On Linux, confirm the package installed correctly with `which python3`.

If you are on Windows and checked "Add Python to PATH" but the error persists, you may have multiple terminal windows open. Close all terminal windows and open a fresh one — PATH changes only take effect in newly opened terminals. If it still fails, restart your computer and try again.

If you are on macOS and seeing a security warning instead of a version number, go to System Settings > Privacy & Security and look for a message about the Python installer being blocked. Click "Allow" next to it, then re-open Terminal and try `python3 --version` again.

### Step 3 — Install a Code Editor

You can write Python in any text editor, but a good code editor makes a tremendous difference. For AI engineering, two free options are most popular:

- **VS Code** (Visual Studio Code) — the industry standard for AI development, with extensions for Python, Jupyter notebooks, and GitHub Copilot
- **PyCharm Community Edition** — a dedicated Python IDE with excellent built-in tools

VS Code is the recommended choice for this course because it is lightweight, free, and has a rich ecosystem of extensions for AI development. It also has an integrated terminal, which means you can run commands without switching windows.

Install VS Code from code.visualstudio.com. After installation, open it, click the Extensions icon on the left sidebar (or press `Ctrl+Shift+X`), search for "Python" and install the Microsoft Python extension (publisher: Microsoft). This gives you syntax highlighting, code completion, and debugging for Python.

Two more extensions worth installing now:

- **Python Debugger** (Microsoft) — lets you step through your code line by line to find bugs
- **Jupyter** (Microsoft) — enables running Jupyter notebooks inside VS Code, which you will use extensively in later sections

### Step 4 — Create a Project Directory

A project directory is a folder that holds all the files for one project. Organising your code into named directories is one of the simplest habits that separates organised engineers from messy ones.

Back in your terminal, create a directory for your AI learning projects:

```bash
# Create the directory
mkdir ai-learning

# Move into it
cd ai-learning

# Confirm you are in the right place
pwd
```

On Windows, the `pwd` command does not exist — use `cd` with no arguments to print the current directory, or use `echo %cd%`.

### Step 5 — Write Your First Python Script

Open VS Code. Go to File > Open Folder and select the `ai-learning` folder you just created. In VS Code, create a new file called `hello_ai.py` by clicking the New File icon or pressing `Ctrl+N` and saving it with that name.

Type the following into the file:

```python
print("Hello, AI World!")
print("Python is installed and working.")
```

Save the file with `Ctrl+S`.

### Step 6 — Run Your Script From the Terminal

In VS Code, open the integrated terminal by going to Terminal > New Terminal (or pressing the backtick key). The terminal should already be in your `ai-learning` directory.

Run your script with:

```bash
python hello_ai.py
```

Or on macOS/Linux:

```bash
python3 hello_ai.py
```

You should see:

```text
Hello, AI World!
Python is installed and working.
```

If you see an error like `SyntaxError`, double-check that you have typed the quotation marks and parentheses correctly. Python requires exact syntax.

If you see `ModuleNotFoundError`, it means you are running a different Python interpreter than expected — verify with `python --version` that you are using the version you installed.

### Step 7 — The Python REPL

Beyond running scripts from files, Python also offers a REPL (Read-Eval-Print Loop). When you type `python` with no script name, it starts an interactive session where you type Python code and see results immediately. This is excellent for experimenting with small pieces of code.

Try it:

```bash
python
```

You should see something like:

```python
Python 3.12.4 (tags/v3.12.4:8e8a4ba, Jun  6 2024, 19:43:15) [MSC v.1940 64 bit (AMD64)] on win32
Type "help", "copyright", "credits" or "license" for more information.
>>>
```

The `>>>` prompt means Python is waiting for your commands. Type:

```python
>>> 2 + 2
4
>>> print("The REPL is interactive")
The REPL is interactive
>>> exit()
```

Press `Ctrl+Z` then Enter on Windows, or `Ctrl+D` on macOS/Linux, or type `exit()` to leave the REPL.

### Step 8 — Install a Python Package With pip

Python packages are reusable libraries written by the community. The package installer `pip` downloads and installs them from the Python Package Index (PyPI). AI engineers install packages constantly — almost nothing you build will use only the Python standard library.

Install a package used extensively in AI data processing:

```bash
pip install numpy
```

Or if your system uses `python3`:

```bash
python3 -m pip install numpy
```

The `-m pip` flag explicitly runs pip through your Python interpreter, which guarantees that the package ends up in the correct site-packages directory. This is the safest way to install packages and avoids the mismatch problem described earlier.

Once installed, test it from the REPL:

```python
>>> import numpy
>>> numpy.__version__
'1.26.4'
```

If you see `ModuleNotFoundError` when running `import numpy`, pip installed the package to a different Python than the one running your REPL. Always use `python -m pip install` to keep them paired.

### Understanding pip Output

When you install a package, pip prints several lines showing what it is doing:

```text
$ python -m pip install numpy
Collecting numpy
  Downloading numpy-1.26.4-cp312-cp312-win_amd64.whl (15.8 MB)
     ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ 15.8/15.8 MB 8.2 MB/s eta 0:00:00
Installing collected packages: numpy
Successfully installed numpy-1.26.4
```

Each line has meaning:

- **Collecting numpy** — pip searched PyPI and found the `numpy` package
- **Downloading** — pip is downloading the package file. The `cp312` part means it is compiled for Python 3.12. The progress bar shows download speed and estimated time
- **Installing collected packages** — pip is copying files into the correct `site-packages` directory
- **Successfully installed** — the package is ready to use

If you see warnings about a newer version of pip being available, you can upgrade pip itself:

```bash
python -m pip install --upgrade pip
```

## Cost and Safety

### Cost

Python itself is free and open source. Every package on PyPI is free to download and use. There is no cost to installing Python, writing scripts, or using pip. The code editor (VS Code) is also free. Your only cost is your time and your computer's electricity.

### Safety

Only download Python from the official website (python.org). Third-party sources may bundle malware or outdated versions. When using pip, be cautious with packages that have very few downloads or suspicious names — supply chain attacks on PyPI do occur, though they are rare. Stick to well-known packages in AI engineering (NumPy, pandas, requests, openai, torch, transformers, and so on) and verify the package name before installing.

pip shows a warning if the package name looks suspicious. If you see such a warning, stop and double-check the spelling.

### Limits

The terminal is extremely powerful but also unforgiving. Commands run immediately with no undo button. `rm -rf` on macOS/Linux or `del /s` on Windows will permanently delete files. Always double-check a destructive command before pressing Enter, especially when using `sudo` (macOS/Linux) or running as Administrator (Windows).

Python itself does not have guardrails against accidentally overwriting files, deleting data, or making network requests. As you write more complex scripts, build in safety checks and test on non-critical data first.

## When to Use and When Not to Use

### Use the terminal when:

- You need to install or update software
- You want to run a Python script or start a Jupyter notebook
- You need to see exact error messages (they are more detailed than GUI equivalents)
- You are automating a repetitive task
- You are working on a remote server (no graphical interface available)

### Use a graphical interface when:

- You are browsing files visually to find something
- You are editing code and want syntax highlighting and autocompletion (VS Code provides these in a graphical window)
- You are viewing images, plots, or data visualisations
- You are using a tool like Jupyter that offers a mixed code-and-output interface

The two approaches are complementary. Professional AI engineers use the terminal for installation, automation, and execution, and use graphical editors for writing and debugging code.

### What Python Version Should You Use?

Python releases new versions every year. At the time of this course, Python 3.12 and 3.13 are the current stable releases. Any version 3.10 or later will work for everything in this course.

The convention is straightforward: the first number (3) is the major version, the second number (12) is the minor version, and the third number (4) is the patch version. Patch versions indicate bug fixes and security updates — you should always use the latest patch within your chosen minor version.

If you work on a team that uses a specific Python version, match that version exactly. If you are on your own, install the latest stable release from python.org. This course assumes Python 3.12 or newer.

### Common Terminal Shortcuts

These keyboard shortcuts work in most terminals and will save you time:

| Shortcut | Action |
|---|---|
| Up Arrow / Down Arrow | Scroll through command history |
| Tab | Auto-complete file and directory names |
| Ctrl+C | Cancel the currently running command |
| Ctrl+L | Clear the screen (same as `clear` or `cls`) |
| Ctrl+A | Move cursor to the beginning of the line |
| Ctrl+E | Move cursor to the end of the line |
| Ctrl+U | Delete everything from cursor to the beginning |
| Ctrl+K | Delete everything from cursor to the end |

Learning these shortcuts gradually will make you faster. Start with Up Arrow (re-run previous commands) and Tab (auto-complete filenames) — these two save the most time.

### Question 1

You just installed Python and typed `python --version` but got `'python' is not recognized`. What is the most likely cause?

<details>
<summary>Answer</summary>

Python was not added to your system PATH during installation. On Windows, re-run the installer and check the "Add Python to PATH" box before clicking Install. On macOS/Linux, you may have installed only part of the package — run `which python3` to see if Python is present under a different name.
</details>

### Question 2

What is the difference between running `pip install numpy` and `python -m pip install numpy`?

<details>
<summary>Answer</summary>

`python -m pip` explicitly runs pip using the Python interpreter you type before it. This guarantees the installed package is placed in that interpreter's site-packages. The bare `pip install numpy` runs whichever pip is first in your PATH, which may belong to a different Python. Using `python -m pip` is always safer for this reason.
</details>

### Question 3

You are in a directory called `projects` and want to move into a subdirectory called `ai-learning`. What command do you use?

<details>
<summary>Answer</summary>

`cd ai-learning` changes the current working directory to `ai-learning`. You can verify you are in the right place with `pwd` on macOS/Linux or `cd` on Windows.
</details>

### Question 4

What is the Python REPL and when would you use it instead of writing a script?

<details>
<summary>Answer</summary>

The REPL (Read-Eval-Print Loop) is an interactive Python session started with the `python` command and no script name. It shows a `>>>` prompt and evaluates each line of code immediately. Use the REPL for quick experiments, testing small code snippets, or exploring a library's features. Use a script file when you want to write code longer than a few lines that you intend to run multiple times.
</details>

### Question 5

You wrote `hello_ai.py` with `print("Hello, AI World!")` and ran `python hello_ai.py` from the terminal. Nothing happened — no output, no error. What might be wrong?

<details>
<summary>Answer</summary>

The most likely cause is that you are not in the same directory as `hello_ai.py`. The `python` command looks for the file in the current working directory. Use `ls` (or `dir` on Windows) to see what files are in your current directory. If `hello_ai.py` is not listed, use `cd` to navigate to the folder where you saved it, or provide the full path to the file.
