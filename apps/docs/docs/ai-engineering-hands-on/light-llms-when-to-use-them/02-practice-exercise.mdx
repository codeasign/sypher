In this exercise, you will build a smart query classifier and router. The system takes a user question, classifies its complexity using a light model, and then routes it to either a light model for a quick answer or a heavy model for a detailed response. You will see firsthand how much time and money this saves compared to sending everything to the heavy model.

## Setup

### Prerequisites

- Python 3.10 or higher installed on your machine
- An OpenAI API key (we will use OpenAI for both the light and heavy models)
- Basic familiarity with the terminal and environment variables

### Install Dependencies

Create a new directory for this project and add a `requirements.txt` file:

```txt
openai>=1.55.0
python-dotenv>=1.0.0
```

Install the dependencies:

```bash
pip install -r requirements.txt
```

### Set Your API Key

Create a `.env` file in the project directory:

```bash
echo "OPENAI_API_KEY=your-key-here" > .env
```

Replace `your-key-here` with your actual OpenAI API key. Never commit this file to version control. Add `.env` to your `.gitignore` file if you have one.

## The Build -- Step by Step

### Step 1: Create the Project Structure

Create a file called `smart_router.py` in your project directory. This file will contain all the logic for this exercise.

Add the imports at the top:

```python
import os
from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()
client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])
```

**Checkpoint:** Run the file to verify no import errors:

```bash
python smart_router.py
```

You should see no output. If you see an import error, verify the packages are installed correctly. Common issues include having multiple Python versions installed where `pip` installs to a different one than `python` uses.

### Step 2: Build the Complexity Classifier

The classifier uses a light model (GPT-4o-mini) to score a question's complexity from 1 to 10. A score of 1 means the question is trivial (for example, "What is 2 plus 2?"). A score of 10 means it requires deep reasoning (for example, "Explain the implications of quantum decoherence on the measurement problem").

Add this function to `smart_router.py`:

```python
SYSTEM_PROMPT_CLASSIFIER = """
You are a query complexity classifier. Analyze the user's question and
return a single integer between 1 and 10 representing its complexity.

1-3: Simple factual questions, basic math, greetings, simple instructions
4-6: Moderate complexity, requires some explanation, multi-step but straightforward
7-10: Complex reasoning, analysis, creative work, multi-step problem solving

Return ONLY the integer. No explanation, no formatting.
"""

def classify_complexity(question: str) -> int:
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {"role": "system", "content": SYSTEM_PROMPT_CLASSIFIER},
            {"role": "user", "content": question}
        ],
        temperature=0.0,
        max_tokens=5
    )
    return int(response.choices[0].message.content.strip())
```

**Checkpoint:** Add a test call at the bottom of the file and run it:

```python
if __name__ == "__main__":
    score = classify_complexity("What is the capital of France?")
    print(f"Complexity score: {score}")
```

Run the file:

```bash
python smart_router.py
```

Expected output:

```
Complexity score: 1
```

The classifier correctly identifies that a simple factual question has low complexity. If you see a different score in the 1-3 range, that is fine -- the model may vary slightly between runs.

### Step 3: Build the Answer Functions

Now add two answer functions: one for the light model and one for the heavy model.

```python
LIGHT_MODEL = "gpt-4o-mini"
HEAVY_MODEL = "gpt-4o"

def answer_with_light_model(question: str) -> str:
    response = client.chat.completions.create(
        model=LIGHT_MODEL,
        messages=[
            {"role": "system", "content": "You are a helpful assistant. Answer concisely in one or two sentences."},
            {"role": "user", "content": question}
        ],
        max_tokens=200
    )
    return response.choices[0].message.content

def answer_with_heavy_model(question: str) -> str:
    response = client.chat.completions.create(
        model=HEAVY_MODEL,
        messages=[
            {"role": "system", "content": "You are a helpful assistant. Provide a thorough, detailed answer."},
            {"role": "user", "content": question}
        ],
        max_tokens=1000
    )
    return response.choices[0].message.content
```

**Checkpoint:** Call both functions to verify they work:

```python
if __name__ == "__main__":
    q = "What is the capital of France?"
    light = answer_with_light_model(q)
    heavy = answer_with_heavy_model(q)
    print(f"Light: {light}")
    print(f"Heavy: {heavy}")
```

Expected output:

```
Light: The capital of France is Paris.
Heavy: The capital of France is Paris, which has been the country's political and cultural center for centuries. It is located in the north-central part of the country along the Seine River...
```

The light model gives a concise answer. The heavy model gives a detailed one. Both are correct, but the light model used fewer tokens and was faster.

### Step 4: Build the Router

The router combines the classifier and the answer functions. It uses a complexity threshold to decide whether to use the light model or the heavy model.

```python
COMPLEXITY_THRESHOLD = 5

def route_and_answer(question: str) -> dict:
    score = classify_complexity(question)
    
    if score <= COMPLEXITY_THRESHOLD:
        answer = answer_with_light_model(question)
        model_used = LIGHT_MODEL
    else:
        answer = answer_with_heavy_model(question)
        model_used = HEAVY_MODEL
    
    return {
        "question": question,
        "complexity_score": score,
        "model_used": model_used,
        "answer": answer
    }
```

### Step 6: Add Cost Tracking

The router is working, but you cannot see the cost savings. Add a tracking function that estimates the token usage and cost for each question.

```python
def estimate_cost(model: str, input_text: str, output_text: str) -> float:
    """Estimate cost in USD for a single API call."""
    input_tokens = len(input_text.split()) * 1.3  # rough estimate
    output_tokens = len(output_text.split()) * 1.3
    
    if model == LIGHT_MODEL:
        input_cost = (input_tokens / 1_000_000) * 0.15
        output_cost = (output_tokens / 1_000_000) * 0.60
    else:
        input_cost = (input_tokens / 1_000_000) * 2.50
        output_cost = (output_tokens / 1_000_000) * 10.00
    
    return input_cost + output_cost

def run_tests_with_costs():
    test_questions = [
        "What is 2 plus 2?",
        "Who wrote Romeo and Juliet?",
        "Explain how a car engine works in simple terms.",
        "Write a Python function that implements a binary search tree with insert, delete, and search methods.",
        "Analyze the economic impact of inflation on middle-class households in developing countries.",
        "What color is the sky?",
        "Design a fault-tolerant microservices architecture for a global e-commerce platform."
    ]
    
    total_cost = 0.0
    total_cost_if_all_heavy = 0.0
    
    for q in test_questions:
        result = route_and_answer(q)
        cost = estimate_cost(result["model_used"], q, result["answer"])
        heavy_cost = estimate_cost(HEAVY_MODEL, q, result["answer"])
        total_cost += cost
        total_cost_if_all_heavy += heavy_cost
        
        print(f"Complexity: {result['complexity_score']:2d} | Model: {result['model_used']:15s} | Cost: ${cost:.6f}")
    
    print("-" * 60)
    print(f"Total cost with routing:      ${total_cost:.6f}")
    print(f"Total cost if all heavy:      ${total_cost_if_all_heavy:.6f}")
    print(f"Savings:                      ${total_cost_if_all_heavy - total_cost:.6f}")
    print(f"Savings percentage:           {((1 - total_cost / total_cost_if_all_heavy) * 100):.1f}%")

if __name__ == "__main__":
    run_tests_with_costs()
```

## Common Errors at This Stage

### Error 1: Invalid API Key

```
openai.AuthenticationError: 
The api_key client option must be set either by passing api_key to the client or by setting the OPENAI_API_KEY environment variable
```

**Cause:** The `.env` file is missing or the key is not set. **Fix:** Verify that `.env` exists in the project directory and contains `OPENAI_API_KEY=sk-...` with your actual key. Also verify the key is active in your OpenAI dashboard.

### Error 2: Rate Limit Exceeded

```
openai.RateLimitError: 
Error code: 429 - You exceeded your current quota, please check your plan and billing details.
```

**Cause:** You have exceeded your API rate limit or your free tier quota has expired. **Fix:** Add a small delay between requests using `time.sleep(0.5)`, or check your OpenAI usage dashboard to see if you need to add billing information.

### Error 3: Model Not Found

```
openai.BadRequestError: 
Error code: 404 - The model `gpt-4o-mini` does not exist or you do not have access to it.
```

**Cause:** The model string is incorrect or your account does not have access to that model. **Fix:** Verify the model name on the OpenAI models page. You can list available models with:

```python
models = client.models.list()
for model in models:
    print(model.id)
```

### Error 4: Classifier Returns Non-Integer

```
ValueError: invalid literal for int() with base 10: 'The complexity score is 3'
```

**Cause:** The classifier model returned a full sentence instead of just a number. **Fix:** The system prompt instructs the model to return only the integer, but sometimes it ignores this. Update the parsing to be more robust:

```python
def classify_complexity(question: str) -> int:
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {"role": "system", "content": SYSTEM_PROMPT_CLASSIFIER},
            {"role": "user", "content": question}
        ],
        temperature=0.0,
        max_tokens=5
    )
    text = response.choices[0].message.content.strip()
    # Extract the first integer found in the response
    import re
    match = re.search(r'\d+', text)
    if match:
        return int(match.group())
    return 5  # default to medium complexity if parsing fails
```

### Error 5: Token Limit Exceeded

```
openai.BadRequestError: 
Error code: 400 - This model's maximum context length is 16385 tokens.
```

**Cause:** Your question plus the system prompt exceeded the model's context window. **Fix:** For the light model, keep questions under 10,000 tokens. The classifier is designed for short queries, not long documents. If you need to classify long documents, truncate them first.

## What You Just Built

- A **complexity classifier** that uses a light model (GPT-4o-mini) to score questions from 1 to 10
- A **router** that sends simple questions to the light model and complex questions to the heavy model
- A **cost tracking system** that shows the real dollar savings from routing
- A **horizontally scalable architecture** -- you can add more model tiers or change the threshold without changing the core logic
- A **production-ready pattern** -- this same approach is used by companies processing millions of requests per day to cut API costs by 60 to 90 percent

The core insight is that most of your traffic is simple, and light models handle simple tasks perfectly well. The money you save on simple traffic pays for the heavy model calls on complex traffic, and the whole system runs faster as a result.
