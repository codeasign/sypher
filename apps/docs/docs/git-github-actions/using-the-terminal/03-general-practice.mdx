These exercises are organized by difficulty. Start with Tier 1 and work your way up. Each exercise specifies a starting directory and the expected outcome. Attempt each one before looking at the hint.

All exercises assume a Unix-like terminal (Linux, macOS, or Windows Git Bash). If you are on Windows Command Prompt or PowerShell, some commands differ — use Git Bash for full compatibility.

For every exercise, follow the same pattern: read the prompt, write down the command you think will work, run it, and check whether the result matches the expected outcome. If it does not match, read the error message and adjust your approach.

## Tier 1 — Beginner

These exercises use a single command each. The goal is to build muscle memory for the basic syntax.

### Exercise 1: Where Are You?

Open your terminal. Without moving anywhere, print the full path of your current location.

**Expected outcome:** The terminal displays a path like `/home/you` or `/Users/you`.

<details>
<summary>Hint</summary>

The command is `pwd`. It has no options or arguments — just type it and press Enter.
</details>

<details>
<summary>Solution</summary>

```bash
pwd
```

The output shows your current working directory. This is the directory the shell considers "here" until you `cd` somewhere else.

If your prompt shows something like `~/projects`, the `pwd` output will show the full absolute path: `/home/you/projects`.
</details>

### Exercise 2: List Everything in Your Home Directory

List all files and directories in your home directory, including hidden files (dotfiles), in detailed format.

**Expected outcome:** A listing of all files, their permissions, owner, size, and last modified date. Hidden files (names starting with `.`) are also shown.

<details>
<summary>Hint</summary>

Two flags are needed: `-a` for all files including hidden ones, and `-l` for detailed format. They can be combined as `-la` or `-al`.
</details>

<details>
<summary>Solution</summary>

```bash
ls -la ~
```

The tilde `~` is shorthand for your home directory. You could also run `ls -la /home/you` (substituting your username) or just `ls -la` if you are already in your home directory.

Look for files starting with a dot like `.bashrc`, `.profile`, `.gitconfig`, or `.ssh`. These are configuration files that control how your shell and tools behave. They are hidden by default because you rarely need to see them.
</details>

### Exercise 3: Create a Directory for a New Project

Create a directory called `my-first-project` in your home directory. Then create three subdirectories inside it: `src`, `tests`, and `docs`.

**Expected outcome:** The directory structure `/home/you/my-first-project/` exists with `src`, `tests`, and `docs` inside it.

<details>
<summary>Hint</summary>

You can create all three subdirectories in one command by passing multiple names to `mkdir`.
</details>

<details>
<summary>Solution</summary>

```bash
mkdir ~/my-first-project
mkdir ~/my-first-project/src ~/my-first-project/tests ~/my-first-project/docs
```

Or, if you navigate into the project directory first:

```bash
mkdir ~/my-first-project
cd ~/my-first-project
mkdir src tests docs
```

Verify with `ls -la ~/my-first-project`. The output should show the three subdirectories, each marked with a `d` at the start of the permissions column indicating they are directories.
</details>

### Exercise 4: Create and Populate a README File

In your `my-first-project` directory, create a file called `README.md` with the content `# My First Project`.

**Expected outcome:** The file `~/my-first-project/README.md` exists and contains the line `# My First Project`.

<details>
<summary>Hint</summary>

Use `echo` with the redirect operator `>` to write text into a new file.
</details>

<details>
<summary>Solution</summary>

```bash
echo "# My First Project" > ~/my-first-project/README.md
```

Verify with:

```bash
cat ~/my-first-project/README.md
```

The output should be `# My First Project`. The `>` operator created the file because it did not exist, then wrote the text into it. If the file had already existed, `>` would have overwritten it completely.
</details>

## Tier 2 — Intermediate

These exercises combine multiple commands to accomplish a task. Plan the sequence before you start typing.

### Exercise 1: Set Up a Web Project

Create a directory called `web-app` in your home directory. Inside it, create the following structure:

```
web-app/
├── index.html
├── css/
│   └── style.css
└── js/
    └── app.js
```

Then write a basic HTML skeleton into `index.html`. The file should contain at least the `<!DOCTYPE html>` declaration and the opening and closing `<html>`, `<head>`, and `<body>` tags.

**Expected outcome:** The full directory structure exists, and `index.html` contains valid HTML markup.

<details>
<summary>Hint</summary>

Create the directories first using `mkdir -p` to handle nested paths. Use `touch` to create files. Use `echo` or multiple `echo` commands with `>>` to write the HTML content.
</details>

<details>
<summary>Solution</summary>

```bash
mkdir -p ~/web-app/css ~/web-app/js
touch ~/web-app/index.html ~/web-app/css/style.css ~/web-app/js/app.js
echo "<!DOCTYPE html>" > ~/web-app/index.html
echo "<html>" >> ~/web-app/index.html
echo "<head><title>My App</title></head>" >> ~/web-app/index.html
echo "<body><h1>Hello</h1></body>" >> ~/web-app/index.html
echo "</html>" >> ~/web-app/index.html
```

Verify the structure:

```bash
ls -R ~/web-app
cat ~/web-app/index.html
```

The `-R` flag on `ls` lists directories recursively, showing the full tree. `ls -R` is essential for checking nested structures.
</details>

### Exercise 2: Organize Downloads

Pretend you have a directory called `~/downloads` containing these files:

```
image1.png
image2.png
document.pdf
notes.txt
script.py
data.csv
photo.jpg
```

Create this directory with these files using `touch` and `mkdir`. Then organize them into subdirectories: `images` for `.png` and `.jpg` files, `documents` for `.pdf` and `.txt` files, `code` for `.py` files, and `data` for `.csv` files.

**Expected outcome:** Each file is moved into the appropriate subdirectory. The final structure has four subdirectories instead of eight loose files.

<details>
<summary>Hint</summary>

Use `mkdir` to create the category directories, then `mv` with wildcard patterns like `*.png` to move multiple files at once.
</details>

<details>
<summary>Solution</summary>

First create the mock directory:

```bash
mkdir ~/downloads-test
cd ~/downloads-test
touch image1.png image2.png document.pdf notes.txt script.py data.csv photo.jpg
```

Then organize:

```bash
mkdir images documents code data
mv *.png *.jpg images/
mv *.pdf *.txt documents/
mv *.py code/
mv *.csv data/
```

Verify:

```bash
ls -R
```

You should see the four subdirectories, each containing the expected files. Notice how the wildcard `*.png` expanded to match `image1.png` and `image2.png` before the `mv` command ran. The shell handles the expansion, not `mv` itself.
</details>

### Exercise 3: Create a Diary With Daily Entries

Create a directory called `diary` with subdirectories for each month: `jan`, `feb`, `mar`. Inside each month directory, create entry files named `day-01.txt`, `day-02.txt`, and `day-03.txt`. Each file should contain the text `Entry for [month] [day]` (for example, `Entry for jan 01`).

**Expected outcome:** Nine files exist across three directories. Each file contains the correct text for its month and day.

<details>
<summary>Hint</summary>

Use `mkdir -p` to create all directories at once, then use a loop or a series of `echo` commands. A simple loop in bash: `for month in jan feb mar; do for day in 01 02 03; do echo "Entry for $month $day" > ~/diary/$month/day-$day.txt; done; done`
</details>

<details>
<summary>Solution</summary>

```bash
mkdir -p ~/diary/jan ~/diary/feb ~/diary/mar
for month in jan feb mar; do
  for day in 01 02 03; do
    echo "Entry for $month $day" > ~/diary/$month/day-$day.txt
  done
done
```

Verify with:

```bash
ls -R ~/diary
cat ~/diary/jan/day-01.txt
cat ~/diary/mar/day-03.txt
```

The nested `for` loop iterates through each month, and for each month it iterates through each day. This is a pattern you will use frequently: iterate over a set of values and perform a command for each one.
</details>

### Exercise 4: Backup and Restore a File

Create a file called `important-notes.txt` in `~/my-first-project` with at least three lines of content. Make a copy of it called `important-notes-backup.txt`. Then delete the original. Verify the backup exists and the original is gone.

**Expected outcome:** The backup file exists and contains the same content as the deleted original.

<details>
<summary>Hint</summary>

Use `echo` with `>>` to build a multi-line file, then `cp` to back it up, then `rm` to delete the original.
</details>

<details>
<summary>Solution</summary>

```bash
cd ~/my-first-project
echo "Note 1: Buy groceries" > important-notes.txt
echo "Note 2: Call dentist" >> important-notes.txt
echo "Note 3: Submit report" >> important-notes.txt
cp important-notes.txt important-notes-backup.txt
rm important-notes.txt
ls
cat important-notes-backup.txt
```

The `ls` command should show `important-notes-backup.txt` but not `important-notes.txt`. The `cat` command confirms the content was preserved in the backup. This is the exact workflow you will use when you need to modify a file but want to keep a copy of the original.
</details>

## Tier 3 — Advanced

These exercises simulate real-world scenarios you will encounter as a developer. They require planning and combining multiple concepts.

### Exercise 1: Set Up a Python Project Structure

Create a standard Python project structure in `~/python-project` with the following layout:

```
python-project/
├── README.md
├── setup.py
├── requirements.txt
├── .gitignore
├── src/
│   └── my_package/
│       ├── __init__.py
│       ├── main.py
│       └── utils.py
└── tests/
    ├── __init__.py
    └── test_main.py
```

Write appropriate initial content into each file:
- `README.md` should contain a project title and a one-line description
- `setup.py` should contain a Python `setuptools.setup()` call with a name, version, and author
- `.gitignore` should list `__pycache__/` and `*.pyc`
- `main.py` should contain a `def main():` function with a print statement
- `test_main.py` should contain an `import pytest` statement and an empty test function

**Expected outcome:** All directories and files exist. Each file contains at least some relevant content. The `ls -R` command shows the complete structure.

<details>
<summary>Hint</summary>

Create all directories with a single `mkdir -p` command, then use `touch` for empty files and `echo` for files with content. For multi-line content in files, use `echo` with `>>` to append each line.
</details>

<details>
<summary>Solution</summary>

```bash
mkdir -p ~/python-project/src/my_package ~/python-project/tests
touch ~/python-project/src/my_package/__init__.py
touch ~/python-project/tests/__init__.py
touch ~/python-project/src/my_package/utils.py

echo "# Python Project" > ~/python-project/README.md
echo "A sample Python project structure." >> ~/python-project/README.md

echo "from setuptools import setup, find_packages" > ~/python-project/setup.py
echo "setup(name='python-project', version='0.1.0', author='You', packages=find_packages('src'), package_dir={'': 'src'})" >> ~/python-project/setup.py

echo "pytest>=7.0" > ~/python-project/requirements.txt

echo "__pycache__/" > ~/python-project/.gitignore
echo "*.pyc" >> ~/python-project/.gitignore
echo ".env" >> ~/python-project/.gitignore
echo "*.egg-info/" >> ~/python-project/.gitignore

echo "def main():" > ~/python-project/src/my_package/main.py
echo "    print('Hello from Python project')" >> ~/python-project/src/my_package/main.py
echo "if __name__ == '__main__':" >> ~/python-project/src/my_package/main.py
echo "    main()" >> ~/python-project/src/my_package/main.py

echo "import pytest" > ~/python-project/tests/test_main.py
echo "def test_placeholder():" >> ~/python-project/tests/test_main.py
echo "    assert True" >> ~/python-project/tests/test_main.py

ls -R ~/python-project
```
</details>

### Exercise 2: Batch Rename All .txt Files to .md

Create a directory called `~/batch-rename-test` containing ten files: `file-01.txt` through `file-10.txt`. Each file should contain its own name as content (for example, `file-01.txt` contains the text `file-01`).

Then rename all ten `.txt` files to `.md` files without typing each name individually. After the rename, verify that no `.txt` files remain and that ten `.md` files exist with the correct content.

**Expected outcome:** All ten files are renamed from `file-01.txt` ... `file-10.txt` to `file-01.md` ... `file-10.md`. Their content is preserved.

<details>
<summary>Hint</summary>

Create the files using a `for` loop. For the rename, use a `for` loop that iterates over `*.txt` files and uses `mv` with parameter expansion to replace the extension: `mv "$file" "${file%.txt}.md"`. The `%` operator removes the shortest matching suffix from the variable.
</details>

<details>
<summary>Solution</summary>

```bash
mkdir ~/batch-rename-test
cd ~/batch-rename-test

for i in $(seq -w 1 10); do
  echo "file-$i" > "file-$i.txt"
done

ls
# Output: file-01.txt  file-02.txt  ...  file-10.txt

for file in *.txt; do
  mv "$file" "${file%.txt}.md"
done

ls
# Output: file-01.md  file-02.md  ...  file-10.md

# Verify content preserved
cat file-01.md
# Output: file-01
```

The `seq -w 1 10` generates numbers 01 through 10 with zero-padding. The `${file%.txt}` syntax removes the `.txt` suffix from the variable, and `.md` is appended to form the new name. This parameter expansion syntax works in any POSIX-compliant shell.
</details>

### Exercise 3: Find and Clean Up Large Temporary Files

Create a directory called `~/cleanup-test` with the following mock files:

- `report-2024-01.log` (3 KB)
- `report-2024-02.log` (5 KB)
- `debug-output.tmp` (1 KB)
- `cache-data.tmp` (50 bytes)
- `notes.txt` (200 bytes)
- `temp-session.tmp` (100 bytes)

Use `dd` or `mkfile` to create files with actual sizes (or use `fallocate` on Linux). After creating them, list the contents sorted by size. Then remove all `.tmp` files. Finally, confirm that only the `.log` and `.txt` files remain.

**Expected outcome:** Three `.tmp` files are deleted. Three `.log` files and one `.txt` file remain. The `ls -l` listing confirms the correct file sizes.

<details>
<summary>Hint</summary>

Use `dd if=/dev/zero of=filename bs=1024 count=N` to create a file of N kilobytes. Use `ls -lS` to sort by size (capital S flag). Use `rm *.tmp` to remove all temporary files at once.
</details>

<details>
<summary>Solution</summary>

```bash
mkdir ~/cleanup-test
cd ~/cleanup-test

# Create files with specific sizes using dd (works on Linux and macOS)
dd if=/dev/zero of=report-2024-01.log bs=1024 count=3 2>/dev/null
dd if=/dev/zero of=report-2024-02.log bs=1024 count=5 2>/dev/null
dd if=/dev/zero of=debug-output.tmp bs=1024 count=1 2>/dev/null
dd if=/dev/zero of=cache-data.tmp bs=50 count=1 2>/dev/null
dd if=/dev/zero of=notes.txt bs=200 count=1 2>/dev/null
dd if=/dev/zero of=temp-session.tmp bs=100 count=1 2>/dev/null

# List sorted by size (largest first)
ls -lS

# Remove all .tmp files
rm *.tmp

# Confirm cleanup
ls -l
```

The `2>/dev/null` suppresses the `dd` output messages (records in/out). After running `rm *.tmp`, only the `.log` and `.txt` files should remain. The `*` wildcard matched all three `.tmp` files and passed them to `rm` in a single command.
</details>

## Bonus Challenge: Create a Complete Project From Scratch

For this challenge, do not create any helper files. Build everything from the terminal commands you have learned.

Create a directory called `bonus-project` in your home directory with the following structure and content:

```
bonus-project/
├── README.md          (contains: "# Bonus Project" and a blank line and "Created with terminal commands.")
├── src/
│   ├── utils/
│   │   ├── helpers.js  (contains: "// Utility functions")
│   │   └── validators.js (contains: "// Validation functions")
│   └── main.js          (contains: "console.log('Starting bonus-project');")
├── tests/
│   └── test-main.js     (empty file)
├── .gitignore           (contains: "node_modules/" and ".env")
└── docs/
    └── architecture.md  (contains: "# Architecture" and "This project follows a modular structure.")
```

**Requirements:** Use at least one `mkdir -p` command, at least one `echo` with `>` and one with `>>`, and at least one `touch` command. Verify the complete structure using `ls -R`.

<details>
<summary>Solution (one approach)</summary>

```bash
cd ~
mkdir -p bonus-project/src/utils bonus-project/tests bonus-project/docs
touch bonus-project/tests/test-main.js

echo "# Bonus Project" > bonus-project/README.md
echo "" >> bonus-project/README.md
echo "Created with terminal commands." >> bonus-project/README.md

echo "// Utility functions" > bonus-project/src/utils/helpers.js
echo "// Validation functions" > bonus-project/src/utils/validators.js
echo "console.log('Starting bonus-project');" > bonus-project/src/main.js

echo "node_modules/" > bonus-project/.gitignore
echo ".env" >> bonus-project/.gitignore

echo "# Architecture" > bonus-project/docs/architecture.md
echo "This project follows a modular structure." >> bonus-project/docs/architecture.md

ls -R bonus-project
```
</details>

## Self-Check

After completing these exercises, you should be able to perform each of the following without looking at the solution:

1. **pwd** — Print your current directory.
2. **ls** — List files with detailed format, including hidden files, and sort by different criteria.
3. **cd** — Navigate to any directory using absolute or relative paths.
4. **mkdir** — Create single directories and nested directory trees.
5. **touch** — Create empty files.
6. **echo with redirection** — Write content to files (`>` for overwrite, `>>` for append).
7. **cp** — Copy files and directories.
8. **mv** — Rename files, move files between directories.
9. **rm** — Delete files and directories individually or in groups.
10. **Wildcards** — Use `*` to match multiple files by pattern.
11. **Loops** — Use `for` loops to repeat operations across multiple files.
12. **Parameter expansion** — Use `${variable%pattern}` to strip suffixes and transform filenames.

If any of these still feel unfamiliar, repeat the corresponding exercise until the command comes naturally. The terminal becomes intuitive through repetition, not theory. Try creating your own practice scenarios — invent a directory structure and challenge yourself to build it from memory.
