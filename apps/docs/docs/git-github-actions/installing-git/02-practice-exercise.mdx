This exercise walks you through the entire process of installing Git and configuring it for daily use. By the end, you will have a working Git installation with your identity, editor, and default branch settings in place.

---

## Setup

Before you begin, determine what state your machine is in.

### Check if Git is already installed

Open a terminal and run:

```bash
git --version
```

One of three things will happen:

- **You see a version number** (for example, `git version 2.45.1`). Git is already installed. Skip to the configuration section of this exercise.
- **You see an error** (for example, `command not found: git` or `'git' is not recognized`). Git is not installed. Proceed to the installation section.
- **You see a version number that is older than 2.28**. Git is installed but outdated. You should update to a newer version so that you can use `init.defaultBranch`.

### Check your operating system

Knowing your OS determines which installation method you use:

```bash
# Windows
systeminfo | findstr /B /C:"OS Name"

# macOS
sw_vers

# Linux
cat /etc/os-release
```

Write down your OS before proceeding. You will pick the installation method that matches it.

### Check if you have a package manager

- **Windows 10 1809 or later**: You have `winget` built in. Run `winget --version` to confirm.
- **Windows (any version)**: You may have Chocolatey or Scoop. Run `choco --version` or `scoop --version`.
- **macOS**: You may have Homebrew. Run `brew --version` to confirm.
- **Linux**: You have a package manager built in. Run `apt --version`, `dnf --version`, or `pacman --version` depending on your distribution.

If you are on macOS and do not have Homebrew, you can install it by running the command at `brew.sh` or use the Xcode Command Line Tools instead.

## The Walkthrough

Follow each step in order. Do not skip steps. After each step, verify the result before moving to the next one.

### Step 1: Check your current Git version

```bash
git --version
```

If this command succeeds, write down the version number. If it fails, proceed to Step 2.

**Expected output (example):** `git version 2.45.1`

### Step 2: Install Git (if needed)

Choose the method that matches your operating system.

**Windows users:**

```bash
# Option A (recommended) — using winget
winget install Git.Git --silent

# Option B — using Chocolatey (if installed)
choco install git -y

# Option C — manual download
# Open https://git-scm.com/download/win in your browser
# Download the 64-bit Git for Windows Standalone Installer
# Run the installer and choose the following options:
#   - Select Components: leave defaults
#   - Choosing the default editor: select "Use Visual Studio Code as Git's default editor"
#   - Adjusting your PATH environment: select "Git from the command line and also from 3rd-party software"
#   - Choosing HTTPS transport backend: select "Use the OpenSSL library"
#   - Configuring the line ending conversions: select "Checkout Windows-style, commit Unix-style line endings"
#   - Configuring the terminal emulator: select "Use Windows' default console window"
#   - Click Install
```

**macOS users:**

```bash
# Option A (recommended) — using Homebrew
brew install git

# Option B — using Xcode Command Line Tools
xcode-select --install

# Option C — manual download
# Open https://git-scm.com/download/mac in your browser
# Download the macOS installer (.dmg)
# Open the .dmg file and run the .pkg installer
```

**Linux users:**

```bash
# Debian / Ubuntu / Linux Mint / Raspberry Pi OS
sudo apt-get update
sudo apt-get install git -y

# Fedora / RHEL / CentOS 8+
sudo dnf install git

# openSUSE
sudo zypper install git

# Arch Linux / Manjaro / EndeavourOS
sudo pacman -S git
```

### Step 3: Verify the installation

Open a **new** terminal window (this is important — the old one still has the old PATH). Then run:

```bash
git --version
```

**Expected output:** A version string like `git version 2.45.1`. If you still see an error, close the terminal, open another new one, and try again. If the error persists after two attempts, see the Common Errors section below.

### Step 4: Set your global user name

```bash
git config --global user.name "Your Full Name"
```

Replace `Your Full Name` with your actual name. Use the name you want to appear on your commits — this is usually your first and last name as it appears on your professional profiles.

**Verify:**

```bash
git config --global user.name
```

**Expected output:** The name you just set.

### Step 5: Set your global user email

```bash
git config --global user.email "your.email@example.com"
```

Replace `your.email@example.com` with your actual email address. This should be the email associated with your GitHub or GitLab account if you have one.

**Verify:**

```bash
git config --global user.email
```

**Expected output:** The email address you just set.

### Step 6: Set your default editor

```bash
# VS Code
git config --global core.editor "code --wait"

# Or nano (terminal-based, no GUI needed)
# git config --global core.editor "nano"

# Or vim
# git config --global core.editor "vim"
```

Choose the editor you are most comfortable with. VS Code is a good default if you have it installed. The `--wait` flag tells your terminal to pause until you close the editor window.

**Verify:**

```bash
git config --global core.editor
```

**Expected output:** The editor command you set (for example, `code --wait`).

### Step 7: Set the default branch name

```bash
git config --global init.defaultBranch main
```

This ensures that every new repository you create starts with a branch named `main` instead of `master`. This is a widely adopted convention in the industry.

**Verify:**

```bash
git config --global init.defaultBranch
```

**Expected output:** `main`

### Step 8: View your full configuration

```bash
git config --list --show-origin
```

This command prints every setting Git is currently using, along with the file path where each setting was defined. Scan the output to confirm that your four settings (user.name, user.email, core.editor, init.defaultBranch) are present.

**Expected output (example):**

```
file:C:/Program Files/Git/etc/gitconfig    core.symlinks=false
file:C:/Users/ada/.gitconfig               user.name=Ada Lovelace
file:C:/Users/ada/.gitconfig               user.email=ada@example.com
file:C:/Users/ada/.gitconfig               core.editor=code --wait
file:C:/Users/ada/.gitconfig               init.defaultBranch=main
```

## Common Errors

### Error 1: "git is not recognized" (Windows) or "command not found: git" (macOS/Linux)

**Exact message (Windows):**

```
git : The term 'git' is not recognized as the name of a cmdlet, function, script file, or operable program.
```

**Exact message (macOS/Linux):**

```
zsh: command not found: git
```

**Cause:** The Git executable is not on your PATH. This means either Git is not installed, or it is installed but the installer did not add the Git directory to the PATH environment variable.

**Fix:**

1. If Git is not installed, follow the installation steps in the Walkthrough section above.
2. If Git is installed but PATH is missing, re-run the installer. On Windows, watch for the "Adjusting your PATH environment" screen and select "Git from the command line and also from 3rd-party software." On macOS, installing via Homebrew places Git in `/usr/local/bin`, which is usually on the PATH — if it is not, add this line to `~/.zshrc`:
   ```bash
   export PATH="/usr/local/bin:$PATH"
   ```
3. After fixing the PATH, close your terminal, open a new one, and run `git --version` again.

### Error 2: "Please tell me who you are" when committing

**Exact message:**

```
*** Please tell me who you are.

Run

  git config --global user.email "you@example.com"
  git config --global user.email

  git config --global user.name "Your Name"
  git config --global user.name

to set your account's default identity.
Omit --global to set the identity only in this repository.
```

**Cause:** You tried to run `git commit` without setting `user.name` and `user.email` at any configuration level. Git requires an identity for every commit.

**Fix:**

```bash
git config --global user.name "Your Full Name"
git config --global user.email "your.email@example.com"
```

After setting these, the commit command that failed will work again. You do not need to re-type the commit command — Git remembers the staged changes.

### Error 3: "fatal: not in a git repository"

**Exact message:**

```
fatal: not a git repository (or any of the parent directories): .git
```

**Cause:** You ran a Git command that requires a repository (for example, `git commit`, `git branch`, `git status`) in a directory that has never been initialized with Git.

**Fix:**

```bash
git init
```

This creates a new `.git` directory in the current folder, turning it into a Git repository. The command that failed will then work. Note that this is not an error in the installation — it means you are not inside a Git repository yet.

### Error 4: "unsafe repository" (Git 2.35.2 and later)

**Exact message:**

```
fatal: detected dubious ownership in repository at '/path/to/repo'
To add an exception for this directory, call:

        git config --global --add safe.directory /path/to/repo
```

**Cause:** Git 2.35.2 introduced a security check that flags repositories owned by a different user. This commonly happens on Linux when you clone a repository with `sudo` or when the repository directory is owned by a different system user.

**Fix:**

```bash
git config --global --add safe.directory /path/to/repo
```

Replace `/path/to/repo` with the actual path shown in the error message. You can also add multiple directories, or add `*` (not recommended for security reasons) to trust all directories:

```bash
git config --global --add safe.directory '*'
```

### Error 5: "detected dubious ownership" on Windows (Git for Windows)

**Exact message:**

```
fatal: detected dubious ownership in repository at 'C:/Users/ada/Projects/my-repo'
```

**Cause:** The same security check as Error 4, but triggered by a different user context — for example, if you cloned the repository as Administrator and are now accessing it as a regular user, or if the repository is on a mounted network drive.

**Fix:** Same as Error 4 — add the repository path to the safe.directory list:

```bash
git config --global --add safe.directory "C:/Users/ada/Projects/my-repo"
```

## What You Just Learned

- You can verify whether Git is installed with a single command: `git --version`.
- The installation method depends on your operating system: `winget` for Windows, `brew` for macOS, and `apt`/`dnf`/`pacman` for Linux.
- After installation, you must open a new terminal window before Git is available on the command line.
- Git requires `user.name` and `user.email` to be set before you can create commits. Without these, Git refuses to commit.
- The `--global` flag writes settings to `~/.gitconfig`, which applies to every repository on your machine.
- The `--show-origin` flag reveals the file path where each configuration value is defined, which helps you debug unexpected behavior.
- You can configure the default editor (`core.editor`) and the default branch name (`init.defaultBranch`) to match your preferences.
- Most Git errors that beginners encounter are not programming errors — they are configuration errors. The error messages tell you what is missing and what command to run to fix it.
