# K Closest Points to Origin

**Difficulty:** Medium

Given an array of `points` where `points[i] = [xi, yi]` represents a point on the X-Y plane, and an integer `k`, return the `k` closest points to the origin `(0, 0)`.

The distance between two points is the **Euclidean distance**: `sqrt(x^2 + y^2)`. You do not need to sort them — return any valid set of k points.

## Example 1

**Input:** `points = [[1,3],[-2,2]]`, `k = 1`  
**Output:** `[[-2,2]]`  
**Explanation:** The distance of `[1,3]` is `sqrt(10)`. The distance of `[-2,2]` is `sqrt(8)`. Since `sqrt(8) < sqrt(10)`, `[-2,2]` is closer.

## Example 2

**Input:** `points = [[3,3],[5,-1],[-2,4]]`, `k = 2`  
**Output:** `[[-2,4],[3,3]]`  
**Explanation:** Distances: `sqrt(18)`, `sqrt(26)`, `sqrt(20)`. The two closest are `[3,3]` and `[-2,4]`.

## Input Specification

The first line contains two integers `N` and `K` — the number of points and how many closest to return.  
The second line contains `2*N` space-separated integers representing the points as `x1 y1 x2 y2 ... xN yN`.

## Output Specification

Print all K closest points, each as `x y`, all on one line separated by spaces, sorted by x-coordinate (then y-coordinate for ties). This deterministic ordering allows automated testing.

## Constraints

- `1 <= k <= points.length <= 10^4`
- `-10^4 <= xi, yi <= 10^4`

## Hints

<details>
<summary>Hint 1</summary>

You do not need to compute the actual Euclidean distance. Since `sqrt` is monotonically increasing, comparing `x^2 + y^2` gives the same ordering as comparing the actual distances. This avoids floating-point arithmetic entirely.

</details>

<details>
<summary>Hint 2</summary>

A brute-force approach sorts all points by distance in O(n log n) time. Can you do better? Think about using a **max-heap of size k** that always keeps only the k closest points seen so far.

</details>

<details>
<summary>Hint 3</summary>

Push `(distance, x, y)` onto a max-heap. After each push, if the heap size exceeds `k`, pop the farthest point. At the end, the heap contains exactly the k closest points in O(n log k) time.

</details>
