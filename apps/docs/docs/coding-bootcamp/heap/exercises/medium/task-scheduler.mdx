# Task Scheduler

**Difficulty:** Medium

You are given a list of CPU tasks represented by uppercase letters (e.g., `["A","A","A","B","B","B"]`) and a non-negative integer `n` representing the **cooldown period** between two executions of the same task. At each time unit, the CPU can execute one task or sit idle.

Return the **minimum number of time units** required to finish all the tasks.

## Example 1

**Input:** `tasks = ["A","A","A","B","B","B"]`, `n = 2`
**Output:** `8`
**Explanation:** A valid schedule is `A → B → idle → A → B → idle → A → B`. That is 8 time units total.

## Example 2

**Input:** `tasks = ["A","A","A","A","A","A"]`, `n = 2`
**Output:** `16`
**Explanation:** A must wait 2 units after each execution. Schedule: `A → idle → idle → A → idle → idle → A → idle → idle → A → idle → idle → A → idle → idle → A`.

## Example 3

**Input:** `tasks = ["A","B","C","D","E","F"]`, `n = 2`
**Output:** `6`
**Explanation:** All tasks are unique, so no idle time is needed: `A → B → C → D → E → F`.

## Input Specification

- The first line contains space-separated single uppercase letters representing the task list.
- The second line contains a single non-negative integer `n` (the cooldown period).

## Output Specification

Print a single integer — the minimum number of time units to complete all tasks.

## Constraints

- `1 <= tasks.length <= 10^4`
- Each task is an uppercase English letter (`'A'` to `'Z'`).
- `0 <= n <= 100`

## Hints

<details>
<summary>Hint 1</summary>

Count the frequency of each task. The most frequent task is the bottleneck — it determines the minimum "frame" structure for the schedule. Try thinking about how many idle slots the most frequent task forces.

</details>

<details>
<summary>Hint 2</summary>

If the most frequent task has frequency `maxFreq`, it creates `(maxFreq - 1)` gaps between its executions, each of length `n`. The formula `(maxFreq - 1) * (n + 1)` gives the length of the skeleton — but you still need to add the number of tasks that also reach `maxFreq` (they fill the last slot).

</details>

<details>
<summary>Hint 3</summary>

The answer is `max(len(tasks), (maxFreq - 1) * (n + 1) + maxCount)`, where `maxCount` is the number of distinct tasks that share the maximum frequency. The `max` with `len(tasks)` handles the case where there are enough diverse tasks to eliminate all idle time.

</details>
