# Find Median from Data Stream

**Difficulty:** Hard

The **median** is the middle value in an ordered integer list. If the size of the list is even, the median is the average of the two middle values.

Design a class `MedianFinder` that:
- `addNum(num)` — adds an integer from the data stream to the data structure
- `findMedian()` — returns the median of all elements so far

## Example 1

**Input:**  
`addNum(1)`  
`addNum(2)`  
`findMedian() → 1.5`  
`addNum(3)`  
`findMedian() → 2.0`

**Explanation:** After adding 1, the sorted list is `[1]` → median = 1.0. After adding 2, `[1, 2]` → median = 1.5. After adding 3, `[1, 2, 3]` → median = 2.0.

## Example 2

**Input:**  
`addNum(5)`  
`addNum(15)`  
`findMedian() → 10.0`  
`addNum(1)`  
`findMedian() → 5.0`

**Explanation:** After adding 5 and 15, the sorted list is `[5, 15]` → median = 10.0. After adding 1, the sorted list is `[1, 5, 15]` → median = 5.0.

## Input Specification

The first line contains an integer `n`. Then `n` lines follow, each containing a single integer to be added to the stream.

## Output Specification

After each addition, output the median as a float with exactly one decimal place, one per line.

## Constraints

- `1 <= n <= 10^5`
- `-10^5 <= num <= 10^5`
- At most `5 * 10^4` calls to `addNum()` and `findMedian()` combined

## Hints

<details>
<summary>Hint 1</summary>

Think about using two heaps — a max-heap for the smaller half and a min-heap for the larger half of the numbers.
</details>

<details>
<summary>Hint 2</summary>

Keep the sizes of the two heaps balanced so that the median is always at the top of one of them.
</details>

<details>
<summary>Hint 3</summary>

The max-heap (or a min-heap with negated values) stores the smaller half, and the min-heap stores the larger half.
</details>
