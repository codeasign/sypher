# Sliding Window

The Sliding Window technique maintains a window of elements and slides it across the data structure to solve subarray/substring problems efficiently.

⏱ 20 minutes

## What You Will Learn

- What the Sliding Window pattern is and when to use it
- Fixed-size vs. variable-size window variants
- How to recognize sliding window problems
- Complexity analysis and common pitfalls

## The Problem

Many array/string problems ask you to find a contiguous subarray or substring satisfying a condition. The naive approach checks every possible subarray (O(n²)). Sliding Window reduces this to O(n) by reusing previous computations.

## The Concept

Maintain two pointers (left, right) that define a window. Expand the right pointer to grow the window, and shrink from the left when the condition is violated. Track the optimal answer as the window slides across the input.

## Common Patterns

### Fixed-size Window
Window size is constant. Slide across the array computing something (max, min, avg) in O(1) per step using the previous window's value minus the outgoing element plus the incoming.

### Variable-size Window  
Window expands and contracts based on a condition. Typical for "longest substring with at most K distinct characters" or "minimum window containing all targets".

## Complexity

| Metric | Complexity |
|--------|-----------|
| Time   | O(n) — each element visited at most twice (once by each pointer) |
| Space  | O(1) or O(k) — depends on auxiliary data structure (hash map for frequencies) |

## When to Use Sliding Window

- Subarray/substring problems with a **contiguous** constraint
- The problem asks for **minimum/maximum** length satisfying a condition
- You need to track **frequencies** or **counts** within a window
- O(n²) brute force can be improved
