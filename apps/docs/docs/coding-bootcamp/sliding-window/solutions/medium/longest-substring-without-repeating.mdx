# Longest Substring Without Repeating Characters — Solution

## Problem Overview

Given a string `s`, find the length of the longest substring (contiguous) without repeating characters.

## Brute Force Approach

**Idea:** Check every possible substring and test whether it contains all unique characters.

**Algorithm:**
1. Initialize `maxLen = 0`
2. For each `i` from 0 to n-1:
   - For each `j` from i to n-1:
     - Create substring `s[i:j+1]`
     - Check if all characters in the substring are unique (e.g., using a set)
     - If unique, update `maxLen = max(maxLen, j - i + 1)`
3. Return `maxLen`

**Complexity Analysis:**
- **Time:** O(n³) — O(n²) substrings, and checking each for uniqueness takes O(n) in the worst case
- **Space:** O(min(n, m)) — the set used for uniqueness checking, where m is the character set size

**Why this is suboptimal:** The brute force approach is impractical for n = 5 × 10⁴. It redundantly checks substrings that share overlapping characters.

## Optimal Approach: Sliding Window with Hash Map

**Idea:** Maintain a sliding window and a hash map that stores the last index where each character was seen. When we encounter a character already in the window, jump the left pointer past its previous occurrence instead of incrementing one by one.

**Algorithm:**
1. Initialize `left = 0`, `maxLen = 0`, and an empty hash map `lastSeen`
2. For `right` from 0 to n-1:
   - If `s[right]` is in `lastSeen` and its index is >= `left`:
     - Move `left` to `lastSeen[s[right]] + 1` (skip past the duplicate)
   - Update `lastSeen[s[right]] = right`
   - Update `maxLen = max(maxLen, right - left + 1)`
3. Return `maxLen`

**Why This Works:** The `left` pointer always stays at or before the first position of the current window. When a duplicate is found, we know the window can't extend past the previous occurrence of that character without including a duplicate, so we jump the left pointer directly past it. The hash map gives us O(1) lookup for the last occurrence of each character.

**Complexity Analysis:**
- **Time:** O(n) — each character is processed once by the `right` pointer; the `left` pointer only moves forward
- **Space:** O(min(m, n)) — the hash map stores at most the size of the character set (m = 128 for ASCII, 26 for lowercase letters, etc.)

### Edge Cases to Consider

- **Empty string**: The loop never executes, and `maxLen` remains 0 — correct.
- **Single character**: `maxLen` becomes 1.
- **All unique characters**: The window grows to the full string length. The `if` condition inside the loop is never triggered.
- **All same character**: The window always has size 1. Each step finds a duplicate at `left - 1` and moves `left` to `right`.
- **Spaces and special characters**: The hash map handles any character, including spaces, digits, and symbols.

### Why the Jumping Left Pointer Works

The key insight is that when we encounter a duplicate character at position `right`, the previous occurrence of that character is at `lastSeen[s[right]]`. Any window that includes both positions would contain a duplicate. Therefore, the new window must start at `lastSeen[s[right]] + 1` (immediately after the previous occurrence) to exclude the duplicate. This jump is correct because all characters between `left` and `lastSeen[s[right]]` are irrelevant — they would still include the duplicate if we kept them.

class Solution {
    public int lengthOfLongestSubstring(String s) {
        int n = s.length();
        int maxLen = 0;  // Stores the length of the longest substring found
        java.util.Map<Character, Integer> lastSeen = new java.util.HashMap<>();

        // left pointer defines the start of the current window
        for (int right = 0, left = 0; right < n; right++) {
            char c = s.charAt(right);
            // If c was seen before and is within the current window, move left past it
            if (lastSeen.containsKey(c) && lastSeen.get(c) >= left) {
                left = lastSeen.get(c) + 1;  // Jump left past the duplicate
            }
            // Update the last seen index of the current character
            lastSeen.put(c, right);
            // Update the maximum length found so far
            maxLen = Math.max(maxLen, right - left + 1);
        }
        return maxLen;
    }
}

        }
    }

## Key Takeaways

- **Sliding window with a hash map** is the standard pattern for substring problems with the "no repeating characters" constraint
- The key optimization is **jumping the left pointer** directly past the previous occurrence of a duplicate character, rather than incrementing one step at a time
- The approach is **O(n) time and O(min(m, n)) space** — optimal for this problem
- The hash map stores the **last seen index** of each character, which enables the O(1) left-pointer jump
- For ASCII-only inputs, a fixed-size array of 128 integers is more efficient than a hash map
