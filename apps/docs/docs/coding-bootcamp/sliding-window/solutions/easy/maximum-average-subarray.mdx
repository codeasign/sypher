# Maximum Average Subarray — Solution

## Problem Overview

Given an integer array `nums` of length `n` and an integer `k`, find the maximum average value of any contiguous subarray of length `k` and return that value (rounded to 5 decimal places).

## Brute Force Approach

**Idea:** For every possible starting position of a subarray of length `k`, compute the sum of its `k` elements, keep track of the maximum sum, and divide by `k` at the end.

**Algorithm:**
1. Initialize `maxSum = -infinity`
2. For each `i` from 0 to `n - k`:
   - Compute `sum = nums[i] + nums[i+1] + ... + nums[i+k-1]`
   - Update `maxSum = max(maxSum, sum)`
3. Return `maxSum / k`

**Complexity Analysis:**
- **Time:** O(n * k) — for each of the `n-k+1` starting positions, we sum `k` elements
- **Space:** O(1) — only a few variables

**Why this is suboptimal:** The inner loop recomputes the sum from scratch for each window, discarding the work done for the previous window. This leads to O(n*k) time, which is too slow for large inputs.

## Optimal Approach: Sliding Window

**Idea:** Instead of recomputing the sum for each window from scratch, reuse the previous window's sum. When the window slides by one position, subtract the element that leaves the window and add the new element that enters it.

**Algorithm:**
1. Compute the sum of the first `k` elements: `windowSum = sum(nums[0:k])`
2. Set `maxSum = windowSum`
3. For each `i` from `k` to `n - 1`:
   - `windowSum = windowSum - nums[i - k] + nums[i]`
   - Update `maxSum = max(maxSum, windowSum)`
4. Return `maxSum / k`

**Why This Works:** A contiguous subarray of length `k` is a sliding window of fixed size. By maintaining the running sum of the current window, each slide is O(1) instead of O(k). The window covers every possible contiguous subarray of length `k` exactly once.

**Complexity Analysis:**
- **Time:** O(n) — each element is visited at most twice (once in the initial sum, once as it enters or leaves)
- **Space:** O(1) — only a constant amount of extra memory

## Deep Dive: How the Sliding Window Works

### Visual Walkthrough

For `nums = [1, 12, -5, -6, 50, 3]` and `k = 4`:

```
Initial window sum = 1 + 12 + (-5) + (-6) = 2
maxSum = 2

Window [1, 12, -5, -6]
         ↑            ↑
        left         right

Step 1: Slide window right by 1
  Subtract nums[0] = 1 (leaving the window)
  Add nums[4] = 50 (entering the window)
  newSum = 2 - 1 + 50 = 51
  maxSum = max(2, 51) = 51

Window [12, -5, -6, 50]
         ↑             ↑
        left          right

Step 2: Slide window right by 1
  Subtract nums[1] = 12 (leaving the window)
  Add nums[5] = 3 (entering the window)
  newSum = 51 - 12 + 3 = 42
  maxSum = max(51, 42) = 51

Window [-5, -6, 50, 3]
         ↑            ↑
        left         right

Final result: maxSum / k = 51 / 4 = 12.75
```

### Edge Cases to Consider

- **k = n**: The entire array is the only window. The algorithm computes the initial sum and never enters the loop.
- **k = 1**: Each element is a window of size 1. The algorithm finds the maximum element.
- **All negative numbers**: The maximum sum will be negative (the least negative window). The algorithm handles this correctly since `maxSum` starts as the first window's sum.
- **Large values**: The `windowSum` must be stored in a 64-bit integer (or equivalent) to handle `n * max(nums[i])` up to 10^5 * 10^4 = 10^9, which fits in a 32-bit signed integer, but best practice is to use a 64-bit integer.

### Common Mistakes

- **Starting maxSum at 0**: If all numbers are negative, `maxSum = 0` would never be updated, leading to an incorrect positive average. Always initialize `maxSum` to the first window's sum.
- **Integer division**: Dividing an integer sum by `k` performs integer division in many languages. Cast to `double` or `float` before division.
- **Off-by-one errors**: The loop condition `i < n` (or `i <= n - 1`) and the indexing `nums[i - k]` for the element leaving the window must be carefully aligned.

class Solution {
    public double findMaxAverage(int[] nums, int k) {
        // Compute sum of the first window of size k
        int windowSum = 0;
        for (int i = 0; i < k; i++) {
            windowSum += nums[i];
        }
        int maxSum = windowSum;  // Initialize maxSum with the first window's sum

        // Slide the window: subtract the element leaving, add the element entering
        for (int i = k; i < nums.length; i++) {
            windowSum = windowSum - nums[i - k] + nums[i];
            maxSum = Math.max(maxSum, windowSum);  // Track the maximum window sum
        }

        // Return the maximum average as a double with 5 decimal precision
        return (double) maxSum / k;
    }
}

fn find_max_average(nums: Vec<i32>, k: i32) -> f64 {
    let k = k as usize;
    // Compute the sum of the first window
    let mut window_sum: i32 = nums[..k].iter().sum();

        if window_sum > max_sum {
            max_sum = window_sum;  // Update max if current window is larger
        }
    }

## Key Takeaways

- **Sliding window of fixed size** is the optimal pattern when the problem asks for a contiguous subarray of a specific length
- **Reusing the previous computation** is the core insight: instead of summing `k` elements each time, update the sum in O(1) by subtracting the element that leaves and adding the element that enters
- The approach is **O(n) time and O(1) space** — optimal for this problem
- Always initialize `maxSum` (or `maxAverage`) to the first window's value, not 0, to handle all-negative inputs correctly
- Use floating-point division when computing the average, not integer division
