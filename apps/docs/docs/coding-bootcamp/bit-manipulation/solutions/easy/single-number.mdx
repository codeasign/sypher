# Single Number — Solution

## Problem Overview

Given a non-empty array of integers where every element appears twice except for one, find that single element. The solution must run in linear time and use constant extra space.

## Brute Force Approach

**Idea:** Use a hash map (dictionary) to count occurrences of each number, then find the one with count 1.

**Algorithm:**
1. Create an empty dictionary/hash map
2. Iterate through the array, incrementing the count for each element
3. Iterate through the dictionary and return the key with value 1

**Complexity Analysis:**
- **Time:** O(n) — we iterate through the array once to count, then again through the map
- **Space:** O(n) — we store up to n/2 + 1 key-value pairs in the hash map

**Why this could be better:** While this works in O(n) time, it uses O(n) extra space. The optimal approach eliminates the extra space requirement entirely.

## Optimal Approach: XOR Cancellation

**Idea:** XOR all elements together. Since `a ^ a = 0` and XOR is commutative/associative, all duplicate pairs cancel out, leaving only the unique element.

**Algorithm:**
1. Initialize `result = 0`
2. XOR `result` with every number in the array
3. Return `result`

**Why This Works:**
- XOR of a number with itself is 0: `x ^ x = 0`
- XOR of a number with 0 is itself: `x ^ 0 = x`
- XOR is commutative and associative, so the order doesn't matter
- All pairs cancel out: `a ^ b ^ a ^ b ^ c = (a ^ a) ^ (b ^ b) ^ c = 0 ^ 0 ^ c = c`

**Trace for `[4, 1, 2, 1, 2]`:**
```
result = 0
result ^ 4 = 4
result ^ 1 = 5
result ^ 2 = 7
result ^ 1 = 6
result ^ 2 = 4  ← unique element
```

**Complexity Analysis:**
- **Time:** O(n) — single pass through the array
- **Space:** O(1) — only one integer variable

## Deep Dive: Why XOR Works for This Problem

### The XOR Property in Detail

The XOR operation has a unique property that makes it perfect for this problem: it is its own inverse. This means:

- `x ^ y ^ y = x` — applying XOR twice with the same value returns the original
- `x ^ y ^ x = y` — the order doesn't matter

For this problem, every pair of identical numbers XORs to 0, and the remaining single number XORs with 0 to give itself.

### Edge Cases

- **Single element array:** The loop runs once, `result ^= nums[0]` returns `nums[0]` because `0 ^ x = x`
- **Negative numbers:** XOR works identically on two's complement binary representation
- **Large numbers:** No overflow concerns — XOR operates on the bit level

### Alternative: Math Approach

A different approach is `2 * sum(set(nums)) - sum(nums)`. This also finds the unique element:
- The sum of the set gives each distinct number once
- Doubling it gives the sum if every number appeared twice
- Subtracting the actual sum leaves the unique element

This approach is O(n) time and O(n) space (for the set), so it's not as good as the XOR approach.

class Solution {
    public int singleNumber(int[] nums) {
        int result = 0;                 // Initialize result to 0
        for (int num : nums) {          // Iterate through each element
            result ^= num;              // XOR with current element
        }
        return result;                  // Return the unique element
    }
}

## Key Takeaways

- **XOR cancellation** is the key pattern for finding a unique element in a sea of pairs
- The solution is O(n) time and O(1) space — optimal
- XOR is both commutative and associative, so order doesn't matter
- This pattern generalizes: any element that appears an even number of times cancels out
