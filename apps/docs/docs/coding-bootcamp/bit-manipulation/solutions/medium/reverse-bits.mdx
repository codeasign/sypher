# Reverse Bits — Solution

## Problem Overview

Reverse the bits of a given 32-bit unsigned integer. For example, if the input is `00000010100101000001111010011100`, the output should be `00111001011110000010100101000000`.

## Brute Force Approach

**Idea:** Iterate through all 32 bits of the input. For each bit, extract it and place it in the reversed position.

**Algorithm:**
1. Initialize `result = 0`
2. For each bit position `i` from 0 to 31:
   - Extract bit `i` of `n`: `bit = (n >> i) & 1`
   - Place it in position `(31 - i)` of result: `result |= (bit << (31 - i))`
3. Return `result`

**Complexity Analysis:**
- **Time:** O(32) = O(1) — fixed number of iterations for 32-bit integer
- **Space:** O(1) — only a few integer variables

**Why this could be better:** While this works, it uses a loop that could be optimized with divide-and-conquer techniques or lookup tables.

## Optimal Approach: Bit-by-Bit Reversal

**Idea:** Process each bit of the input and build the result by shifting and masking.

**Algorithm:**
1. Initialize `result = 0`
2. Loop 32 times:
   - Shift `result` left by 1 to make room for the next bit
   - Extract the LSB of `n`: `n & 1`
   - Add it to result: `result |= (n & 1)`
   - Shift `n` right by 1 to process the next bit
3. Return `result`

**Why This Works:**
- Each iteration takes the LSB of `n` and places it as the next bit of `result`
- The first LSB of `n` becomes the MSB of `result` after 31 shifts
- This is equivalent to reversing all 32 bits

**Trace for `n = 43261596` (binary: `00000010100101000001111010011100`):**
```
Initial: n = 00000010100101000001111010011100, result = 0

Step 1: result << 1 = 0, LSB of n = 0, result = 0, n >> 1
Step 2: result << 1 = 0, LSB of n = 0, result = 0, n >> 1
Step 3: result << 1 = 0, LSB of n = 1, result = 1, n >> 1
...
After 32 steps: result = 00111001011110000010100101000000
```

**Complexity Analysis:**
- **Time:** O(32) = O(1) — fixed number of iterations
- **Space:** O(1) — only two integer variables

## Deep Dive: Divide and Conquer Optimization

### Reversing with Bit Masks

A more efficient approach uses a divide-and-conquer strategy with carefully crafted masks:

```
Step 1: Reverse bytes      (swap adjacent 16-bit halves)
Step 2: Reverse 8-bit halves within each 16-bit half
Step 3: Reverse 4-bit halves within each 8-bit half
Step 4: Reverse 2-bit halves within each 4-bit half
Step 5: Reverse adjacent bits
```

This reduces the number of operations but has the same O(1) complexity.

### Lookup Table Approach

For even more speed, precompute the reversed value for each byte (256 entries) and reverse 4 bytes independently:

```
result = (reverseByte[n & 0xFF] << 24) |
         (reverseByte[(n >> 8) & 0xFF] << 16) |
         (reverseByte[(n >> 16) & 0xFF] << 8) |
         reverseByte[(n >> 24) & 0xFF]
```

This trades O(1) extra memory for fewer operations, which can be faster in practice.

### Edge Cases

- **n = 0:** All bits are 0, result is 0
- **n = 2³² − 1 (all 1s):** Reversing all 1s gives all 1s, so result = 2³² − 1
- **n = 1 (only LSB set):** After reversal, the MSB is set, giving result = 2³¹

## Key Takeaways

- **Bit-by-bit reversal** is the standard approach: for each of 32 bits, extract the LSB and build the result from MSB to LSB
- The complexity is O(1) since the number of bits is fixed at 32
- For performance-critical code, divide-and-conquer or lookup table approaches can be faster
- This technique generalizes to reversing any fixed-width bit string
