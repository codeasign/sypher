# Reverse Integer

**Difficulty:** Medium

Given a signed 32-bit integer `x`, return `x` with its digits reversed. If reversing `x` causes the value to go outside the signed 32-bit integer range `[-2^31, 2^31 - 1]`, return `0`.

**Note:** This problem uses the overflow constraint as its core challenge — you cannot simply convert to a string and reverse without thinking about bounds.

## Example 1

**Input:** `x = 123`
**Output:** `321`

## Example 2

**Input:** `x = -123`
**Output:** `-321`
**Explanation:** The sign is preserved. `-123` reversed gives `-321`.

## Example 3

**Input:** `x = 120`
**Output:** `21`
**Explanation:** Trailing zeros are dropped — `021` is just `21`.

## Example 4

**Input:** `x = 1534236469`
**Output:** `0`
**Explanation:** `9646324351` overflows 32-bit signed range, so return `0`.

## Input Specification

A single line containing one signed integer `x`.

## Output Specification

Print the reversed integer, or `0` if the result would overflow the 32-bit signed integer range.

## Constraints

- `-2^31 <= x <= 2^31 - 1`

## Hints

<details>
<summary>Hint 1</summary>

Extract digits one at a time using modulo and integer division. Build the reversed number by multiplying the running result by 10 and adding the next digit.

</details>

<details>
<summary>Hint 2</summary>

Check for overflow **before** you add the next digit, not after. If `result > INT_MAX / 10`, the next multiply-by-10 will overflow. Handle the boundary case `result == INT_MAX / 10` separately.

</details>

<details>
<summary>Hint 3</summary>

In Python, integers have arbitrary precision, so you can build the full reversed value and clamp it at the end. In languages with fixed-width integers (C, Java, C++, Rust), you must guard each step. The signed 32-bit bounds are `-2147483648` to `2147483647`.

</details>
