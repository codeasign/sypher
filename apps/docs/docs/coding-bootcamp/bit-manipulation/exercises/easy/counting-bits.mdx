# Counting Bits

**Difficulty:** Easy

Given an integer `n`, return an array `ans` of length `n + 1` such that for each `i` (0 ≤ i ≤ n), `ans[i]` is the **number of 1 bits** in the binary representation of `i`.

## Example 1

**Input:** `n = 2`  
**Output:** `[0, 1, 1]`  
**Explanation:**
- `0` → `0` (0 bits)
- `1` → `1` (1 bit)
- `2` → `10` (1 bit)

## Example 2

**Input:** `n = 5`  
**Output:** `[0, 1, 1, 2, 1, 2]`  
**Explanation:**
- `0` → `0` (0 bits)
- `1` → `1` (1 bit)
- `2` → `10` (1 bit)
- `3` → `11` (2 bits)
- `4` → `100` (1 bit)
- `5` → `101` (2 bits)

## Input Specification

A single integer `n`.

## Output Specification

Print `n + 1` space-separated integers — the count of 1 bits for each number from 0 to n.

## Constraints

- `0 <= n <= 10⁵`

## Hints

<details>
<summary>Hint 1</summary>

Think about the relationship between `i` and `i >> 1` (integer division by 2).
</details>

<details>
<summary>Hint 2</summary>

For even numbers, `i` has the same number of 1 bits as `i / 2` because shifting right drops a 0.
</details>

<details>
<summary>Hint 3</summary>

For odd numbers, `i` has one more 1 bit than `i / 2` because the LSB is 1.
</details>

</ProblemEditor>
