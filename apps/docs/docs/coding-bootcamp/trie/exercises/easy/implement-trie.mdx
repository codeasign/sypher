# Implement Trie

**Difficulty:** Easy

A **Trie** (pronounced "try") or prefix tree is a tree data structure used to efficiently store and retrieve keys in a set of strings.

Implement the `Trie` class:

- `Trie()` — Initializes the trie object.
- `void insert(String word)` — Inserts the string `word` into the trie.
- `boolean search(String word)` — Returns `true` if the string `word` is in the trie (i.e., was inserted before), and `false` otherwise.
- `boolean startsWith(String prefix)` — Returns `true` if there is a previously inserted string `word` that has the prefix `prefix`, and `false` otherwise.

## Example

**Input:**
```
insert apple
search apple
search app
startsWith app
insert app
search app
```

**Output:**
```
true
false
true
true
```

**Explanation:**
- `insert("apple")` — inserts the word "apple"
- `search("apple")` — returns true (exists)
- `search("app")` — returns false (not inserted yet)
- `startsWith("app")` — returns true (prefix of "apple")
- `insert("app")` — inserts "app"
- `search("app")` — returns true (now exists)

## Input Specification

Each line of input contains a command followed by a word (or just a command for the empty string). Commands are:
- `insert <word>` — insert the word
- `search <word>` — check if the word exists (print `true` or `false`)
- `startsWith <prefix>` — check if any word starts with the given prefix (print `true` or `false`)

## Output Specification

For each `search` and `startsWith` command, print `true` or `false` on its own line.

## Constraints

- `1 <= word.length, prefix.length <= 2000`
- `word` and `prefix` consist only of lowercase English letters
- The empty string `""` is a valid word/prefix
- At most `3 * 10⁴` calls will be made to `insert`, `search`, and `startsWith`

## Hints

<details>
<summary>Hint 1</summary>

Each Trie node should store an array or map of children nodes (one per possible letter) and a boolean flag indicating whether the node represents the end of a word.
</details>

<details>
<summary>Hint 2</summary>

When inserting a word, traverse character by character. Create new nodes as needed. Mark the final node as an end-of-word node.
</details>

<details>
<summary>Hint 3</summary>

For `search`, traverse character by character. If at any point a child is missing, return `false`. At the end, check the end-of-word flag.
</details>

<details>
<summary>Hint 4</summary>

`startsWith` is identical to `search` except it does not check the end-of-word flag — it only needs to verify that the entire prefix path exists.
</details>
