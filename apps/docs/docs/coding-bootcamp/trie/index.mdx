# Trie

A Trie (pronounced "try"), also known as a prefix tree, is a tree-based data structure that stores strings by splitting them into individual characters. Each node represents a single character, and paths from the root to leaf nodes form complete words.

## The Problem

Many string problems require fast prefix lookups — checking if a word exists, or finding all words that share a given prefix. A Hash Map can answer "does this exact word exist?" in O(1) time, but it cannot efficiently answer "does any word start with this prefix?" without scanning every key.

A Trie solves this by organizing strings by their common prefixes. Instead of storing each word as a separate key, the Trie shares nodes for shared prefixes. This means:

- Inserting a word takes O(L) time (where L is the word length)
- Searching for a word or prefix takes O(L) time
- Space is shared across words, making it memory-efficient for large dictionaries

## The Concept

Each node in a Trie contains:
- A **map or array of children** (one per possible character, typically 26 for lowercase letters)
- A **boolean flag** indicating whether the node marks the end of a valid word

## Complexity

| Operation | Complexity |
|-----------|-----------|
| Insert    | O(L) — L is the length of the word |
| Search    | O(L) — traverse L characters down the tree |
| StartsWith (prefix) | O(L) — same as search, no end-of-word check needed |
| Space     | O(N × L) — N words of average length L, worst case when no prefixes are shared |

## Common Applications

### Autocomplete

When a user types a prefix, the Trie finds the node for that prefix, then traverses all descendant nodes to collect possible completions. This is much faster than filtering a list of all words.

### Spell Checker

A Trie can quickly verify whether a word exists in its dictionary. For suggestions, you can traverse the Trie and find words that are within a small edit distance (Levenshtein distance) from the misspelled word.

### Prefix Matching

Any problem that asks "find all strings that start with X" is a natural fit for a Trie. Examples include:
- Longest common prefix among a set of strings
- Finding words that match a pattern with wildcards

### Word Search (Board Games)

In problems like "Word Search II," a Trie stores the dictionary of target words. As you traverse the board, you check the Trie to see if the current path forms a valid prefix — pruning branches that cannot lead to a word.

## When to Use a Trie vs. Hash Map

| Scenario | Use Trie | Use Hash Map |
|----------|----------|-------------|
| Exact word lookup | O(L) — good | O(1) — better |
| Prefix lookup | O(L) — excellent | O(N × L) — must scan all keys |
| Autocomplete | O(L + results) — excellent | O(N × L) — must filter all keys |
| Memory for shared prefixes | Efficient — shares prefixes | Wasted — stores each key separately |
| Wildcard / pattern matching | Possible — traverse with backtracking | Difficult — would need to scan all keys |

## Worked Example: Insert and Search

Given a dictionary of words: `["cat", "car", "dog", "do"]`

Let's trace inserting **"cat"**:

```
Step 1: Start at root
         root.children['c'] = new Node('c')  →  move to 'c'

Step 2: At node 'c'
         'c'.children['a'] = new Node('a')   →  move to 'a'

Step 3: At node 'a'
         'a'.children['t'] = new Node('t')   →  move to 't'

Step 4: At node 't'
         't'.isEndOfWord = true              →  "cat" is stored
```

Now let's search for **"car"**:

```
Step 1: Start at root, look for 'c'  →  found
Step 2: At 'c', look for 'a'          →  found
Step 3: At 'a', look for 'r'          →  found
Step 4: At 'r', check isEndOfWord     →  false → "car" is NOT in the dictionary
```

But after inserting "car", step 4 would return true.
