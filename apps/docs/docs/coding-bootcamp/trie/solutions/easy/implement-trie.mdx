# Implement Trie — Solution

## Problem Overview

Implement a Trie (prefix tree) with three operations: `insert`, `search`, and `startsWith`. The challenge is to design the tree nodes and traversal logic to support O(L) operations where L is the word length.

## Approach: Array-Based Trie

**Idea:** Each node stores an array of 26 child references (one per lowercase letter) and a boolean flag indicating whether the node marks the end of a word.

**Algorithm:**
1. Create a `TrieNode` class with `children[26]` (or a dictionary) and `isEnd` boolean
2. **Insert:** Traverse the word character by character. If a child for the current character doesn't exist, create it. Move to the child. After processing all characters, mark the final node as an end-of-word.
3. **Search:** Traverse the word character by character. If a child is missing, return `false`. After traversing all characters, return the `isEnd` flag of the final node.
4. **startsWith:** Same as search, but return `true` if the entire prefix path exists (regardless of `isEnd`).

**Why This Works:** By organizing strings by their common prefixes, each operation only needs to follow L pointers — there is no scanning through unrelated words or hash collisions.

**Complexity Analysis:**
- **Time:** O(L) for each operation — L is the length of the word or prefix
- **Space:** O(N × L) in the worst case, where N is the number of words and L is the average word length

## Deep Dive

### Trie Node Design

There are two common approaches for storing children:

1. **Array-based (size 26):** Fast O(1) child lookup by index (`idx = char - 'a'`). Uses more memory per node (26 pointers, most are null). Best for lowercase-only input.
2. **Hash map-based:** Only stores existing children. More memory-efficient for sparse sets. Lookup is O(1) on average but with more overhead.

For problems with only lowercase letters, the array approach is preferred for its speed and simplicity.

### Edge Cases

- **Empty string `""`:** A valid word/prefix. The `insert ""` operation should set the root's `isEnd` flag.
- **Repeated insertions:** Inserting the same word multiple times should not cause errors — just ensure `isEnd` stays `true`.
- **Word is a prefix of another word:** e.g., "app" and "apple". Both should be searchable independently. The node at "app" must have `isEnd = true` even though it also has a child.

class TrieNode:
    def __init__(self):
        self.children = [None] * 26  # Array of 26 child nodes (a-z)
        self.is_end = False          # Marks the end of a valid word

class Trie:
    def __init__(self):
        self.root = TrieNode()       # Root node (empty character)

    def insert(self, word: str) -> None:
        node = self.root
        for ch in word:
            idx = ord(ch) - ord('a')
            if node.children[idx] is None:
                node.children[idx] = TrieNode()  # Create new node if missing
            node = node.children[idx]            # Move to child
        node.is_end = True                       # Mark word end

    def search(self, word: str) -> bool:
        node = self.root
        for ch in word:
            idx = ord(ch) - ord('a')
            if node.children[idx] is None:
                return False                     # Prefix not found
            node = node.children[idx]
        return node.is_end                       # Must be end of a word

    def startsWith(self, prefix: str) -> bool:
        node = self.root
        for ch in prefix:
            idx = ord(ch) - ord('a')
            if node.children[idx] is None:
                return False                     # Prefix not found
            node = node.children[idx]
        return True                              # Prefix exists

struct TrieNode {
    children: Vec<Option<Box<TrieNode>>>,  // Box for heap allocation
    is_end: bool,
}

impl TrieNode {
    fn new() -> Self {
        TrieNode {
            children: (0..26).map(|_| None).collect(),  // 26 empty children
            is_end: false,
        }
    }
}

struct Trie {
    root: TrieNode,
}

impl Trie {
    fn new() -> Self {
        Trie { root: TrieNode::new() }
    }

    fn insert(&mut self, word: &str) {
        let mut node = &mut self.root;
        for ch in word.chars() {
            let idx = (ch as usize) - ('a' as usize);
            if node.children[idx].is_none() {
                node.children[idx] = Some(Box::new(TrieNode::new()));  // Create if missing
            }
            node = node.children[idx].as_mut().unwrap();  // Move to child
        }
        node.is_end = true;  // Mark word end
    }

    fn search(&self, word: &str) -> bool {
        let mut node = &self.root;
        for ch in word.chars() {
            let idx = (ch as usize) - ('a' as usize);
            match &node.children[idx] {
                Some(child) => node = child.as_ref(),  // Move to child
                None => return false,                   // Prefix missing
            }
        }
        node.is_end  // Must be end of a word
    }

    fn starts_with(&self, prefix: &str) -> bool {
        let mut node = &self.root;
        for ch in prefix.chars() {
            let idx = (ch as usize) - ('a' as usize);
            match &node.children[idx] {
                Some(child) => node = child.as_ref(),  // Move to child
                None => return false,                   // Prefix missing
            }
        }
        true  // Entire prefix path exists
    }
}

#include <stdlib.h>
#include <string.h>

typedef struct TrieNode {
    struct TrieNode* children[26];  // Array of 26 child pointers
    bool isEnd;
} TrieNode;

typedef struct {
    TrieNode* root;
} Trie;

TrieNode* createNode() {
    TrieNode* node = (TrieNode*)calloc(1, sizeof(TrieNode));  // calloc zeros memory
    node->isEnd = false;
    return node;
}

Trie* trieCreate() {
    Trie* obj = (Trie*)malloc(sizeof(Trie));
    obj->root = createNode();
    return obj;
}

void trieInsert(Trie* obj, char* word) {
    TrieNode* node = obj->root;
    for (int i = 0; word[i] != '\0'; i++) {
        int idx = word[i] - 'a';
        if (node->children[idx] == NULL) {
            node->children[idx] = createNode();  // Create if missing
        }
        node = node->children[idx];                // Move to child
    }
    node->isEnd = true;                            // Mark word end
}

bool trieSearch(Trie* obj, char* word) {
    TrieNode* node = obj->root;
    for (int i = 0; word[i] != '\0'; i++) {
        int idx = word[i] - 'a';
        if (node->children[idx] == NULL) return false;  // Prefix missing
        node = node->children[idx];
    }
    return node->isEnd;  // Must be end of a word
}

bool trieStartsWith(Trie* obj, char* prefix) {
    TrieNode* node = obj->root;
    for (int i = 0; prefix[i] != '\0'; i++) {
        int idx = prefix[i] - 'a';
        if (node->children[idx] == NULL) return false;  // Prefix missing
        node = node->children[idx];
    }
    return true;  // Entire prefix path exists
}

void trieFreeNode(TrieNode* node) {
    if (!node) return;
    for (int i = 0; i < 26; i++) {
        trieFreeNode(node->children[i]);  // Recursively free children
    }
    free(node);
}

void trieFree(Trie* obj) {
    trieFreeNode(obj->root);
    free(obj);
}

## Key Takeaways

- Each Trie node uses O(1) lookups via array indexing (`char - 'a'`)
- Insert and search both traverse exactly L nodes — no backtracking needed
- The `isEnd` flag is what distinguishes a word from a prefix
- `startsWith` is simpler than `search` because it doesn't check `isEnd`
- Array-based children (size 26) is optimal for lowercase English letters
