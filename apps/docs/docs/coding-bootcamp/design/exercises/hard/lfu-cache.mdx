# LFU Cache

**Difficulty:** Hard

Design and implement a data structure for a **Least Frequently Used (LFU)** cache.

Implement the `LFUCache` class:

- `LFUCache(int capacity)` initializes the object with the `capacity` of the data structure.
- `int get(int key)` gets the value of the `key` if the key exists in the cache. Otherwise, returns `-1`.
- `void put(int key, int value)` updates the value of the `key` if present, or inserts the key if not already present. When the cache reaches its `capacity`, it should invalidate and remove the **least frequently used** key before inserting a new item. For this problem, when there is a **tie** (i.e., two or more keys with the same frequency), the **least recently used** key would be invalidated.

To determine the least frequently used key, a **use counter** is maintained for each key in the cache. The key with the smallest use counter is the least frequently used key. When a key is first inserted into the cache, its use counter is set to `1` (due to the `put` operation). The use counter for a key in the cache is incremented whenever a `get` or `put` operation is called on it.

The functions `get` and `put` must each run in O(1) average time complexity.

## Example 1

**Input:**
```
["LFUCache","put","put","get","put","get","get","put","get","get","get"]
[[2],[1,1],[2,2],[1],[3,3],[2],[3],[4,4],[1],[3],[4]]
```

**Output:**
```
[null,null,null,1,null,-1,3,null,-1,3,4]
```

**Explanation:**
```
LFUCache lfu = new LFUCache(2);
lfu.put(1, 1);   // cache=[1,_], cnt(1)=1
lfu.put(2, 2);   // cache=[2,1], cnt(2)=1, cnt(1)=1
lfu.get(1);      // return 1, cache=[1,2], cnt(2)=1, cnt(1)=2
lfu.put(3, 3);   // 2 is the LFU key because cnt(2)=1 is smallest, evicts 2, cache=[3,1], cnt(3)=1, cnt(1)=2
lfu.get(2);      // return -1 (not found)
lfu.get(3);      // return 3, cache=[3,1], cnt(3)=2, cnt(1)=2
lfu.put(4, 4);   // Both 1 and 3 have cnt=2, but 1 is LRU, evicts 1, cache=[4,3], cnt(4)=1, cnt(3)=2
lfu.get(1);      // return -1 (not found)
lfu.get(3);      // return 3, cache=[3,4], cnt(3)=3, cnt(4)=1
lfu.get(4);      // return 4, cache=[4,3], cnt(4)=2, cnt(3)=3
```

## Input Specification

The first line contains `LFUCache N` where N is the capacity. Subsequent lines contain commands:
- `put key value` — insert or update key with value
- `get key` — output the value for key or -1

## Output Specification

For each `get` command, output the result on its own line.

## Constraints

- `0 <= capacity <= 10^4`
- `0 <= key <= 10^5`
- `0 <= value <= 10^5`
- At most `2 * 10^5` calls will be made to `get` and `put`.

## Hints

<details>
<summary>Hint 1</summary>

Use a hash map for O(1) key lookup. For each key, store its value, frequency, and a reference to its position in the frequency list.
</details>

<details>
<summary>Hint 2</summary>

Maintain a frequency map where each frequency bucket contains a doubly linked list of keys at that frequency, ordered by recency. Also track the minimum frequency across all keys.
</details>

<details>
<summary>Hint 3</summary>

When a key is accessed, remove it from its current frequency bucket, increment its frequency, and add it to the head of the next frequency bucket. Update the minimum frequency if needed.
</details>

<details>
<summary>Hint 4</summary>

For eviction, remove the tail of the linked list at the minimum frequency bucket. This gives you the least recently used item among the least frequently used items.
</details>
