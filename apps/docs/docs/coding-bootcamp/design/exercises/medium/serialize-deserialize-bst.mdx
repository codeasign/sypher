# Serialize and Deserialize BST

**Difficulty:** Medium

Serialization is the process of converting a data structure or object into a sequence of bits so that it can be stored in a file or memory buffer, or transmitted across a network connection and reconstructed later.

Design an algorithm to **serialize** and **deserialize** a Binary Search Tree. There is no restriction on how your serialization/deserialization algorithm should work. You need to ensure that a BST can be serialized to a string and that this string can be deserialized to the original tree structure.

## Example 1

**Input:** BST with preorder traversal `4 2 1 3 6 5 7`

```
        4
       / \
      2   6
     / \ / \
    1  3 5  7
```

**Serialized:** `"4 2 1 3 6 5 7"`

**Deserialized preorder output:** `4 2 1 3 6 5 7`

## Example 2

**Input:** BST with preorder traversal `5 3 7`

```
    5
   / \
  3   7
```

**Serialized:** `"5 3 7"`

**Deserialized preorder output:** `5 3 7`

## Example 3

**Input:** Single node BST `1`

**Serialized:** `"1"`

**Deserialized preorder output:** `1`

## Input Specification

A single line of space-separated integers representing the **preorder traversal** of a valid BST. The harness builds the BST from these values and passes the root to your `serialize` function.

## Output Specification

Print the space-separated preorder traversal of the BST recovered after calling `serialize` then `deserialize`. A correct solution will match the input exactly (since the input is already in preorder form).

## Constraints

- `1 <= number of nodes <= 10^4`
- `1 <= Node.val <= 10^5`
- All node values are **unique**
- The input is guaranteed to be a valid BST preorder sequence

## Hints

<details>
<summary>Hint 1</summary>

The preorder traversal of a BST (root, left subtree, right subtree) uniquely identifies the tree. Serializing as a space-separated preorder sequence is both compact and easy to reconstruct.

</details>

<details>
<summary>Hint 2</summary>

To deserialize, use the BST property: given a root value, all values less than the root belong to the left subtree, and all values greater belong to the right subtree. Process the preorder sequence with a pointer (or a deque/queue) and reconstruct recursively using a `(min, max)` bound to determine where each subtree ends.

</details>

<details>
<summary>Hint 3</summary>

For efficient deserialization, maintain a running index into the token list. At each recursive call, pass the current valid value range `(min_val, max_val)`. If the next token falls outside this range, stop building the current subtree and return `None` — the token belongs to a higher level of the tree.

</details>

</ProblemEditor>
