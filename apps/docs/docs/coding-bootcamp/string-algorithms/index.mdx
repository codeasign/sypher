# String Algorithms

String algorithms are fundamental techniques for processing, searching, and manipulating text data вҖ” one of the most common data types in programming.

вҸұ 25 minutes

## What You Will Learn

- Core string processing techniques: parsing, pattern matching, and encoding/decoding
- When to apply two-pointer and sliding window patterns on strings
- How KMP and Robin-Karp string matching algorithms work
- Complexity analysis of common string operations

## The Problem

String operations appear in nearly every programming domain: validating input, searching documents, transforming data, and building output. Naive approaches (e.g., nested loops for pattern matching) often run in O(n*m) or worse. Efficient string algorithms reduce this to O(n+m) or O(n).

## Core Concepts

### String Parsing

Parsing extracts structured information from raw text. Common patterns include splitting on delimiters, scanning character-by-character, and using state machines to track context.

### Pattern Matching

Finding a substring (pattern) within a larger string (text). The naive approach checks every possible starting position вҖ” O(n*m). Advanced algorithms like KMP and Robin-Karp achieve O(n+m).

### Encoding / Decoding

Transforming data between representations. Examples include run-length encoding (compress repeated characters), URL encoding, and nested string expansion like `"3[a2[c]]"`.

## Complexity

| Algorithm | Time | Space | Use Case |
|-----------|------|-------|----------|
| Naive pattern matching | O(n*m) | O(1) | Short strings |
| KMP (Knuth-Morris-Pratt) | O(n+m) | O(m) | Repeated pattern matching |
| Robin-Karp (rolling hash) | O(n+m) avg, O(n*m) worst | O(1) | Multiple pattern search |
| Two-pointer string scan | O(n) | O(1) | Palindrome, prefix checks |
| Sliding window on string | O(n) | O(1) or O(k) | Substring constraints |
| StringBuilder / join | O(n) | O(n) | Efficient string building |
| Run-length encoding | O(n) | O(n) | String compression |

## Common Patterns

### Two Pointers on Strings

Two pointers moving in opposite directions (palindrome check) or same direction (prefix matching). Uses O(1) extra space.

### Sliding Window

A variable-size window moves across the string, maintaining a constraint (e.g., longest substring without repeating characters).

### String Building

Building strings efficiently using `StringBuilder` / `list` / `Vec<char>` to avoid O(n^2) cost from repeated immutable string concatenation.

### KMP (Knuth-Morris-Pratt)

Preprocesses the pattern to build a **prefix function** (LPS array) that tells KMP how far to shift on a mismatch, skipping redundant comparisons.

### Robin-Karp

Uses a rolling hash to compare the pattern with substrings of the text in O(1) average time per position.

## Worked Example: Longest Common Prefix

Given an array of strings `["flower", "flow", "flight"]`, find the longest common prefix.

```
Step 1: Take the first string as reference: "flower"
Step 2: Compare character by character with each subsequent string

         flower
         flow          вҶҗ 'f' matches, 'l' matches, 'o' matches, 'w' matches
         flight        вҶҗ 'f' matches, 'l' matches, 'o' vs 'i' вҖ” mismatch!

         Common prefix so far: "fl"
         No more characters to check вҶ’ return "fl"
```

**Approach:** Start with prefix = first string. For each other string, shorten the prefix until it matches the start of that string. Time O(n*m) where n = number of strings, m = length of shortest string.

## When to Use String Algorithms

- The problem asks about **substrings, palindromes, or anagrams**
- You need to **parse or validate** string input
- The problem involves **encoding or decoding** patterns
- You need **efficient string search** in a large text
- **String building** with concatenation in a loop (use StringBuilder!)
- Character frequency **counting** and comparison
