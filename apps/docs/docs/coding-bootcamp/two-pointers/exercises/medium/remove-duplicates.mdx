# Remove Duplicates from Sorted Array II

**Difficulty:** Medium

Given an integer array `nums` sorted in **non-decreasing order**,
remove some duplicates **in-place** such that each unique element
appears **at most twice**. The relative order of the elements should
be kept the same.

Return `k` after placing the final result in the first `k` slots of
`nums`.

Do **not** allocate extra space for another array. You must do this
by modifying the input array in-place with O(1) extra memory.

## Example 1

**Input:** `nums = [1,1,1,2,2,3]`  
**Output:** `5, nums = [1,1,2,2,3,_]`  
**Explanation:** Your function should return `k = 5`, with the first
five elements of `nums` being `1, 1, 2, 2, 3`. The remaining
elements are not important.

## Example 2

**Input:** `nums = [0,0,1,1,1,1,2,3,3]`  
**Output:** `7, nums = [0,0,1,1,2,3,3,_,_]`  
**Explanation:** Your function should return `k = 7`.

## Input Specification

The first line contains an integer `n` (the length of the array).
The second line contains `n` space-separated integers in
non-decreasing order representing the array `nums`.

## Output Specification

The first line contains an integer `k` (the new length after removing
duplicates). The second line contains the first `k` elements of the
modified array as space-separated integers.

## Constraints

- `1 <= nums.length <= 3 * 10⁴`
- `-10⁴ <= nums[i] <= 10⁴`
- `nums` is sorted in **non-decreasing** order

## Hints

<details>
<summary>Hint 1</summary>

Use the same-direction (slow/fast) pointer technique. The slow pointer
marks where the next allowed element goes.
</details>

<details>
<summary>Hint 2</summary>

Compare `nums[fast]` with `nums[slow - 2]`. If they differ, it's safe
to write — it means there are at most 2 copies of that value so far.
</details>

<details>
<summary>Hint 3</summary>

Start both pointers at index 2. Elements at indices 0 and 1 are always
kept.
</details>

</ProblemEditor>
