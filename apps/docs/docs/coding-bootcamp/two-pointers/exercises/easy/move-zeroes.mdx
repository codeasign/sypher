# Move Zeroes

**Difficulty:** Easy

Given an integer array `nums`, move all `0`s to the **end** of the array while **maintaining the relative order** of the non-zero elements.

## Example 1

**Input:** `nums = [0, 1, 0, 3, 12]`  
**Output:** `[1, 3, 12, 0, 0]`  
**Explanation:** The non-zero elements `[1, 3, 12]` retain their relative order and the two zeros are moved to the end.

## Example 2

**Input:** `nums = [0]`  
**Output:** `[0]`  
**Explanation:** Only one element — a zero — so it stays in place.

## Example 3

**Input:** `nums = [1, 2, 3]`  
**Output:** `[1, 2, 3]`  
**Explanation:** No zeros present, the array is unchanged.

## Input Specification

A single line containing space-separated integers representing the array `nums`.

## Output Specification

Print the modified array as space-separated integers on a single line, with all zeros moved to the end and non-zero elements in their original relative order.

## Constraints

- `1 <= nums.length <= 10^4`
- `-2^31 <= nums[i] <= 2^31 - 1`

## Hints

<details>
<summary>Hint 1</summary>

Think about maintaining a **write pointer** that tracks where the next non-zero element should be placed. Iterate through the array and whenever you find a non-zero, put it at the write pointer position and advance the write pointer.

</details>

<details>
<summary>Hint 2</summary>

After placing all non-zero elements at the front, the remaining positions (from write pointer to end) should all be filled with `0`. This gives you an O(n) time, O(1) space solution.

</details>

<details>
<summary>Hint 3</summary>

Alternatively, you can use a **swap approach**: keep a slow pointer `i` and a fast pointer `j`. Whenever `nums[j]` is non-zero, swap `nums[i]` with `nums[j]` and advance `i`. This also preserves relative order of non-zero elements.

</details>

</ProblemEditor>
