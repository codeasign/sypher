# Trapping Rain Water

**Difficulty:** Hard

Given `n` non-negative integers representing an elevation map where
the width of each bar is `1`, compute how much water it can trap
after raining.

## Example 1

**Input:** `height = [0,1,0,2,1,0,1,3,2,1,2,1]`  
**Output:** `6`  
**Explanation:** The elevation map (black bars) traps 6 units of blue
water as visualized above.

## Example 2

**Input:** `height = [4,2,0,3,2,5]`  
**Output:** `9`  

## Input Specification

The first line contains an integer `n` (the number of bars in the
elevation map). The second line contains `n` non-negative integers
representing the heights of the bars.

## Output Specification

Print a single integer representing the total units of water that can
be trapped.

## Constraints

- `n == height.length`
- `1 <= n <= 2 * 10⁴`
- `0 <= height[i] <= 10⁵`

## Hints

<details>
<summary>Hint 1</summary>

At any position, the water trapped = min(max height to the left,
max height to the right) - current height.
</details>

<details>
<summary>Hint 2</summary>

Use two pointers to track the running maximum from each side without
extra space.
</details>

<details>
<summary>Hint 3</summary>

Always process the side with the smaller height — that side determines
the water level.
</details>

</ProblemEditor>
