# Three Sum — Solution

## Problem Overview

Given an integer array `nums`, return all unique triplets `[nums[i], nums[j], nums[k]]` such that `i != j`, `i != k`, `j != k`, and `nums[i] + nums[j] + nums[k] == 0`. The solution set must not contain duplicate triplets.

## Brute Force Approach

**Idea:** Use three nested loops to check every possible triplet.

**Algorithm:**
1. For each `i` from 0 to n-3:
   - For each `j` from i+1 to n-2:
     - For each `k` from j+1 to n-1:
       - If `nums[i] + nums[j] + nums[k] == 0`, add `[nums[i], nums[j], nums[k]]` to results
2. Deduplicate the results (e.g., by sorting each triplet and using a set)

**Complexity Analysis:**
- **Time:** O(n³) — three nested loops plus deduplication overhead
- **Space:** O(n) for storing results and the deduplication set

**Why this is suboptimal:** Three nested loops are impractical for n = 3000 (the constraint). The optimal approach reduces this to O(n²) using sorting and two pointers.

## Optimal Approach: Sorting + Two Pointers

**Idea:** Sort the array first. Then, for each element `nums[i]`, use two pointers on the remaining subarray to find pairs that sum to `-nums[i]`. Sort once, skip duplicates at every level.

**Algorithm:**
1. Sort `nums` in ascending order
2. For each index `i` from 0 to n-3:
   - Skip if `nums[i] == nums[i-1]` (duplicate — would produce same triplets)
   - Set `left = i + 1`, `right = n - 1`
   - While `left < right`:
     - Compute `sum = nums[i] + nums[left] + nums[right]`
     - If `sum == 0`:
       - Record `[nums[i], nums[left], nums[right]]`
       - Skip duplicate values from the left and right
       - Move both pointers inward
     - If `sum < 0`: increment `left` (need larger sum)
     - If `sum > 0`: decrement `right` (need smaller sum)

**Why This Works:** Sorting transforms the problem into the Two Sum II pattern. By fixing one element, we reduce three-sum to two-sum on the remaining subarray. Skipping duplicates at all levels ensures unique triplets without needing a set.

**Complexity Analysis:**
- **Time:** O(n²) — sorting is O(n log n), and the two-pointer scan is O(n) for each of n elements
- **Space:** O(1) excluding the output — sorting is in-place

## Deep Dive: How Sorting + Two Pointers Works

### Visual Walkthrough

For `nums = [-1, 0, 1, 2, -1, -4]`:

```
Step 1: Sort → [-4, -1, -1, 0, 1, 2]

Step 2: Fix i = 0 (nums[i] = -4)
        Target pair sum = 4
        left = 1, right = 5
        [-4, -1, -1, 0, 1, 2]
           ↑   ↑              ↑
           i  left          right
        sum = -1 + 2 = 1 < 4 → left++
        ... no pair sums to 4 with -4, move i forward

Step 3: Fix i = 1 (nums[i] = -1)
        Target pair sum = 1
        left = 2, right = 5
        [-4, -1, -1, 0, 1, 2]
               ↑   ↑         ↑
               i  left      right
        sum = -1 + 2 = 1 == target → record [-1, -1, 2]
        Skip duplicates, move pointers
        ...
        Found: [-1, -1, 2] and [-1, 0, 1]
```

### Why Sorting First?

Sorting transforms the problem into a series of Two Sum II subproblems. By fixing one element, we reduce the 3-sum to a 2-sum on the remaining right portion. Without sorting, we'd need O(n³) time or a hash set approach.

### Duplicate Handling Strategy

Duplicates are skipped at THREE levels:
1. **Outer loop**: Skip `nums[i]` if it equals `nums[i-1]`
2. **Left pointer**: After finding a triplet, skip all duplicate `nums[left]` values
3. **Right pointer**: After finding a triplet, skip all duplicate `nums[right]` values

This ensures no duplicate triplets in the output.

```java
import java.util.*;

class Solution {
    public List<List<Integer>> threeSum(int[] nums) {
        // Step 1: Sort the array — enables the two-pointer technique
        Arrays.sort(nums);
        List<List<Integer>> result = new ArrayList<>();
        int n = nums.length;

        // Step 2: Fix one element at a time with the outer loop
        for (int i = 0; i < n - 2; i++) {
            // Skip duplicate values for the fixed element
            if (i > 0 && nums[i] == nums[i - 1]) continue;

            // Step 3: Two pointers on the remaining subarray
            int left = i + 1, right = n - 1;
            while (left < right) {
                int sum = nums[i] + nums[left] + nums[right];

                if (sum == 0) {
                    // Found a valid triplet — record it
                    result.add(Arrays.asList(nums[i], nums[left], nums[right]));

                    // Skip duplicate values from both sides
                    while (left < right && nums[left] == nums[left + 1]) left++;
                    while (left < right && nums[right] == nums[right - 1]) right--;

                    // Move both pointers inward for the next search
                    left++; right--;
                } else if (sum < 0) {
                    // Sum too small — move left pointer to increase it
                    left++;
                } else {
                    // Sum too large — move right pointer to decrease it
                    right--;
                }
            }
        }
        return result;
    }
}
```

```rust
fn three_sum(nums: &mut Vec<i32>) -> Vec<Vec<i32>> {
    // Step 1: Sort the array — enables the two-pointer technique
    nums.sort();
```

## Key Takeaways

- **Sort + Two Pointers** is the canonical pattern for k-sum problems (k ≥ 2)
- **Duplicate handling** is critical: skip at both the outer loop and inner while loop to avoid duplicate triplets
- Each fixed element reduces the problem to Two Sum II on the subarray to its right
- O(n²) is optimal for three-sum since the problem is inherently quadratic
