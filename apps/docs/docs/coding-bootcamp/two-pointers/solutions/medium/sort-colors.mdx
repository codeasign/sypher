# Sort Colors (Dutch National Flag) — Solution

## Problem Overview

Given an array `nums` containing only the integers `0`, `1`, and `2`, sort it **in-place** so that all `0`s come first, followed by all `1`s, and then all `2`s. Solve this without using a built-in sort function and in a **single pass** (O(n) time, O(1) space).

## Brute Force Approach: Counting Sort

**Idea:** Count the occurrences of each value (0, 1, 2), then overwrite the array with the correct number of each value in order.

**Algorithm:**
1. Count the number of 0s, 1s, and 2s in the array
2. Overwrite the first `count0` positions with `0`
3. Overwrite the next `count1` positions with `1`
4. Overwrite the remaining positions with `2`

**Complexity Analysis:**
- **Time:** O(n) — two passes over the array
- **Space:** O(1) — only three counter variables

**Why this is not optimal:** While O(n) time and O(1) space, this requires two passes over the array. The Dutch National Flag algorithm achieves the same result in a single pass.

## Optimal Approach: Dutch National Flag (Three Pointers)

**Idea:** Use three pointers — `low`, `mid`, and `high` — to partition the array into three regions: all 0s (before `low`), all 1s (between `low` and `mid`), and all 2s (after `high`). The `mid` pointer scans through the array, swapping elements into their correct regions.

**Algorithm:**
1. Initialize `low = 0`, `mid = 0`, `high = n - 1`
2. While `mid <= high`:
   - If `nums[mid] == 0`: swap `nums[low]` with `nums[mid]`, increment `low` and `mid`
   - If `nums[mid] == 1`: increment `mid` (already in the correct region)
   - If `nums[mid] == 2`: swap `nums[mid]` with `nums[high]`, decrement `high` (do NOT increment `mid` — the swapped-in value needs to be examined)

**Why This Works:** The algorithm maintains the invariant:
- Elements before `low` are all 0s
- Elements between `low` and `mid` are all 1s
- Elements after `high` are all 2s
- Elements between `mid` and `high` are unprocessed

When `nums[mid] == 2`, we swap it with `nums[high]` and decrement `high`. Importantly, we do NOT advance `mid` because the value swapped from `high` could be 0, 1, or 2 and needs to be processed. When `nums[mid] == 0`, we swap with `nums[low]` and advance both `low` and `mid` — the value at `low` is guaranteed to be 1 (since all 1s are between `low` and `mid`), so it's safe to advance `mid`.

**Visual Walkthrough:**
```
Initial: nums = [2, 0, 2, 1, 1, 0]
                 ↑
               low=0, mid=0
                 ↑
                high=5

Step 1: nums[mid]=2 → swap with nums[high]=0
        nums = [0, 0, 2, 1, 1, 2]
                 ↑
               low=0, mid=0
                             ↑
                           high=4

Step 2: nums[mid]=0 → swap with nums[low]=0, low=1, mid=1
        nums = [0, 0, 2, 1, 1, 2]
                    ↑
                  low=1, mid=1
                             ↑
                           high=4

Step 3: nums[mid]=0 → swap with nums[low]=0, low=2, mid=2
        nums = [0, 0, 2, 1, 1, 2]
                       ↑
                     low=2, mid=2
                             ↑
                           high=4

Step 4: nums[mid]=2 → swap with nums[high]=1, high=3
        nums = [0, 0, 1, 1, 2, 2]
                       ↑
                     low=2, mid=2
                       ↑
                     high=3

Step 5: nums[mid]=1 → mid=3
        nums[mid]=1 → mid=4
        mid=4 > high=3 → exit

Result: [0, 0, 1, 1, 2, 2]
```

**Complexity Analysis:**
- **Time:** O(n) — single pass, each element is examined at most twice
- **Space:** O(1) — in-place modification with only three pointer variables

def sort_colors(nums: list) -> list:
    # Three pointers: low, mid, and high
    low = 0
    mid = 0
    high = len(nums) - 1

    # Process all elements between mid and high
    while mid <= high:
        if nums[mid] == 0:
            # 0 belongs at the low position — swap and advance both
            nums[low], nums[mid] = nums[mid], nums[low]
            low += 1
            mid += 1
        elif nums[mid] == 1:
            # 1 belongs in the middle — just advance mid
            mid += 1
        else:  # nums[mid] == 2
            # 2 belongs at the high position — swap and decrement high
            nums[mid], nums[high] = nums[high], nums[mid]
            high -= 1
            # Do NOT advance mid — the swapped-in value needs processing

    return nums

fn sort_colors(nums: &mut Vec<i32>) {
    // Three pointers: low, mid, and high
    let mut low = 0;
    let mut mid = 0;
    let mut high = nums.len() - 1;

    // Process all elements between mid and high
    while mid <= high {
        if nums[mid] == 0 {
            // 0 belongs at the low position — swap and advance both
            nums.swap(low, mid);
            low += 1;
            mid += 1;
        } else if nums[mid] == 1 {
            // 1 belongs in the middle — just advance mid
            mid += 1;
        } else {  // nums[mid] == 2
            // 2 belongs at the high position — swap and decrement high
            nums.swap(mid, high);
            if high == 0 { break; }  // Prevent underflow
            high -= 1;
            // Do NOT advance mid — the swapped-in value needs processing
        }
    }
}

## Key Takeaways

- **Dutch National Flag** is the canonical three-pointer algorithm for partitioning an array with three distinct values
- The invariant (`low` marks end of 0s, `mid` marks end of 1s, `high` marks start of 2s) guarantees correctness
- **Not advancing `mid` after a 2-swap** is the critical detail — the swapped-in value from `high` is unprocessed
- The algorithm is **O(n) time, O(1) space** in a single pass — optimal for this problem
- This pattern generalizes to any 3-way partitioning problem
