# Valid Palindrome — Solution

## Problem Overview

A phrase is a **palindrome** if, after converting all uppercase letters into lowercase and removing all non-alphanumeric characters, it reads the same forward and backward. Given a string `s`, return `true` if it is a palindrome, `false` otherwise.

## Brute Force Approach

**Idea:** Clean the string by filtering out non-alphanumeric characters and converting to lowercase, then compare the cleaned string with its reverse.

**Algorithm:**
1. Create a new string `cleaned` by iterating through `s`, keeping only alphanumeric characters (lowercased)
2. Compare `cleaned` with `cleaned[::-1]` (its reverse)
3. Return `true` if they match, `false` otherwise

**Complexity Analysis:**
- **Time:** O(n) — we iterate through the string once to clean it, then again to reverse and compare
- **Space:** O(n) — we create a new cleaned string of up to length n

**Why this could be better:** While this approach works, it uses O(n) extra space and essentially requires two passes. The optimal approach solves the problem in-place.

## Optimal Approach: Two Pointers

**Idea:** Use two pointers starting at opposite ends of the string, moving toward each other. Skip non-alphanumeric characters and compare the characters case-insensitively.

**Algorithm:**
1. Initialize `left = 0` and `right = s.length - 1`
2. While `left < right`:
   - Skip non-alphanumeric characters from the left by incrementing `left`
   - Skip non-alphanumeric characters from the right by decrementing `right`
   - Compare `s[left]` and `s[right]` case-insensitively
   - If they differ, return `false`
   - Move both pointers inward (`left++`, `right--`)
3. Return `true` if all pairs match

**Why This Works:** By checking characters from both ends simultaneously and skipping non-alphanumeric characters, we effectively compare the cleaned version of the string without ever constructing it. The two pointers converge in the middle, ensuring we check each pair exactly once.

**Complexity Analysis:**
- **Time:** O(n) — each character is visited at most once
- **Space:** O(1) — only two pointer variables

## Deep Dive: How the Two-Pointer Technique Works

### Visual Walkthrough

For the input `s = "A man, a plan, a canal: Panama"`, let's trace the algorithm:

```
Initial: left = 0, right = 29
s = "A man, a plan, a canal: Panama"
      ↑                              ↑
    left                           right

Step 1: s[left]='A' is alphanumeric, s[right]='a' is alphanumeric
        Compare lowercased: 'a' == 'a' ✓ → left=1, right=28

Step 2: s[left]=' ' is NOT alphanumeric → left=2
Step 2: s[left]='m' is alphanumeric, s[right]='m' is alphanumeric
        Compare: 'm' == 'm' ✓ → left=3, right=27

... continues until pointers cross ...
```

### Edge Cases to Consider

- **Empty string or single character**: Always a palindrome. The loop condition `left < right` immediately fails, returning `true`.
- **Only non-alphanumeric characters**: All characters get skipped; the pointers never find a valid pair to compare, so the function returns `true`.
- **Mixed case**: `Character.toLowerCase()` handles case-insensitive comparison.
- **Very long strings**: The algorithm is O(n) and uses no extra space, so it handles the maximum constraint of 2×10⁵ characters efficiently.

### Why Not Use Regex to Clean the String First?

A common alternative is to strip non-alphanumeric characters with regex and then compare forwards/backwards. While cleaner, this:
1. Creates a new string (O(n) extra space)
2. Requires a regex pass and a reverse pass (two iterations)
3. The two-pointer approach achieves the same result in one pass with O(1) space

class Solution {
    public boolean isPalindrome(String s) {
        int left = 0, right = s.length() - 1;  // Initialize two pointers at opposite ends
        while (left < right) {                  // Continue until pointers meet or cross
            // Skip non-alphanumeric characters from the left
            while (left < right && !Character.isLetterOrDigit(s.charAt(left)))
                left++;
            // Skip non-alphanumeric characters from the right
            while (left < right && !Character.isLetterOrDigit(s.charAt(right)))
                right--;
            // Compare characters case-insensitively
            if (Character.toLowerCase(s.charAt(left)) !=
                Character.toLowerCase(s.charAt(right)))
                return false;  // Mismatch found — not a palindrome
            left++;   // Move left pointer inward
            right--;  // Move right pointer inward
        }
        return true;  // All pairs matched — it's a palindrome
    }
}

fn is_palindrome(s: &str) -> bool {
    let chars: Vec<char> = s.chars().collect();  // Collect chars for indexing
    let mut left = 0;                              // Left pointer start
    let mut right = chars.len().saturating_sub(1); // Right pointer at end (handles empty)
    while left < right {                           // Loop until pointers cross
        // Skip non-alphanumeric characters from the left
        while left < right && !chars[left].is_alphanumeric() { left += 1; }
        // Skip non-alphanumeric characters from the right
        while left < right && !chars[right].is_alphanumeric() { right -= 1; }
        // Compare characters case-insensitively
        if chars[left].to_ascii_lowercase() != chars[right].to_ascii_lowercase() {
            return false;  // Mismatch found
        }
        left += 1;                                   // Advance left
        right = right.saturating_sub(1);             // Advance right (safe decrement)
    }
    true  // All pairs matched
}

## Key Takeaways

- **Two pointers from opposite ends** is the standard pattern for palindrome checking
- **Skipping unwanted characters** is done inside the while loop, not as a preprocessing step
- The approach is **O(n) time and O(1) space** — optimal for this problem
- Case-insensitive comparison and alphanumeric filtering are the two core operations
