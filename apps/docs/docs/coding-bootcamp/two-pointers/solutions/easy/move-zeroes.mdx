# Move Zeroes — Solution

## Problem Overview

Given an integer array `nums`, move all `0`s to the **end** of the array while **maintaining the relative order** of the non-zero elements. This must be done **in-place** with O(1) extra space.

## Brute Force Approach

**Idea:** Create a new array containing only the non-zero elements (preserving order), then fill the rest with zeros.

**Algorithm:**
1. Create a new array `result` of the same length, initialized with zeros
2. Iterate through `nums`, copying each non-zero element to `result` in order
3. Copy `result` back to `nums`

**Complexity Analysis:**
- **Time:** O(n) — one pass to filter, one pass to fill
- **Space:** O(n) — extra array of size n

**Why this is suboptimal:** The problem explicitly requires O(1) extra space. The brute force approach uses O(n) extra space.

## Optimal Approach: Two Pointers (Write Pointer)

**Idea:** Use a write pointer that tracks where the next non-zero element should go. First pass: write all non-zero elements to the front. Second pass: fill the remaining positions with zeros.

**Algorithm:**
1. Initialize `writePos = 0` (the position to write the next non-zero element)
2. For each element `num` in `nums`:
   - If `num != 0`, set `nums[writePos] = num` and increment `writePos`
3. For positions from `writePos` to `n-1`, set `nums[pos] = 0`

**Why This Works:** The write pointer ensures all non-zero elements are compacted at the front in their original relative order. Then we fill the tail with zeros — since we know exactly how many non-zero elements exist (the final value of `writePos`), we know exactly how many zeros to add.

**Visual Walkthrough:**
```
Initial: nums = [0, 1, 0, 3, 12], writePos = 0

Step 1: nums[0] = 0 → skip, writePos = 0
Step 2: nums[1] = 1 ≠ 0 → nums[0] = 1, writePos = 1
Step 3: nums[2] = 0 → skip, writePos = 1
Step 4: nums[3] = 3 ≠ 0 → nums[1] = 3, writePos = 2
Step 5: nums[4] = 12 ≠ 0 → nums[2] = 12, writePos = 3

After first pass: nums = [1, 3, 12, 3, 12], writePos = 3
Fill zeros from writePos to end: nums[3] = 0, nums[4] = 0
Result: [1, 3, 12, 0, 0]
```

**Complexity Analysis:**
- **Time:** O(n) — two passes over the array
- **Space:** O(1) — in-place modification

## Alternative: Swap Approach (Single Pass)

**Idea:** Use a slow pointer `i` and a fast pointer `j`. When `nums[j]` is non-zero, swap `nums[i]` with `nums[j]` and advance `i`. This preserves order and moves zeros to the end in a single pass.

**Algorithm:**
1. Initialize `i = 0` (slow pointer)
2. For `j = 0` to `n-1` (fast pointer):
   - If `nums[j] != 0`, swap `nums[i]` and `nums[j]`, then increment `i`

**Complexity Analysis:**
- **Time:** O(n) — single pass
- **Space:** O(1) — in-place modification

def move_zeroes(nums: list) -> list:
    # Write pointer: tracks where the next non-zero element goes
    write_pos = 0

    # First pass: move all non-zero elements to the front
    for num in nums:
        if num != 0:
            nums[write_pos] = num  # Place non-zero at write position
            write_pos += 1          # Advance write pointer

    for i in range(write_pos, len(nums)):
        nums[i] = 0

## Key Takeaways

- **Two-pointer (write pointer)** is the standard pattern for in-place array compaction
- The write pointer approach is **O(n) time, O(1) space** — optimal for this problem
- The swap-based single-pass variant is another common solution that achieves the same complexity
- The key insight: separate the concerns of "which elements to keep" and "where to put them"
