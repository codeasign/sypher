# 4Sum II

**Difficulty:** Medium

Given four integer arrays `nums1`, `nums2`, `nums3`, and `nums4`, all of length `n`, return the number of tuples `(i, j, k, l)` such that:

- `0 <= i, j, k, l < n`
- `nums1[i] + nums2[j] + nums3[k] + nums4[l] == 0`

## Example 1

**Input:**
```
nums1 = [1, 2]
nums2 = [-2, -1]
nums3 = [-1, 2]
nums4 = [0, 2]
```
**Output:** `2`

**Explanation:** The two tuples are:
- `(0, 0, 0, 1)` → `1 + (-2) + (-1) + 2 = 0`
- `(1, 1, 0, 0)` → `2 + (-1) + (-1) + 0 = 0`

## Example 2

**Input:**
```
nums1 = [0]
nums2 = [0]
nums3 = [0]
nums4 = [0]
```
**Output:** `1`

## Example 3

**Input:**
```
nums1 = [1, -1]
nums2 = [-1, 1]
nums3 = [1, -1]
nums4 = [-1, 1]
```
**Output:** `8`

**Explanation:** Every combination of one element from each array sums to 0.

## Input Specification

Four lines, each containing space-separated integers representing `nums1`, `nums2`, `nums3`, and `nums4` respectively.

## Output Specification

Print a single integer — the count of 4-tuples `(i, j, k, l)` such that `nums1[i] + nums2[j] + nums3[k] + nums4[l] == 0`.

## Constraints

- `n == nums1.length == nums2.length == nums3.length == nums4.length`
- `1 <= n <= 200`
- `-2^28 <= nums1[i], nums2[i], nums3[i], nums4[i] <= 2^28`

## Hints

<details>
<summary>Hint 1</summary>

A brute-force approach that checks all four-element combinations is O(n^4). Can you split the problem into two halves to do better?

</details>

<details>
<summary>Hint 2</summary>

Compute all pairwise sums from `nums1` and `nums2` and store them in a hash map. Then for each pair from `nums3` and `nums4`, look up the negated sum.

</details>

<details>
<summary>Hint 3</summary>

For each pair `(k, l)`, you need `nums1[i] + nums2[j] == -(nums3[k] + nums4[l])`. If the hash map stores the count of each sum from the first two arrays, just add the count at `-(nums3[k] + nums4[l])` to the result.

</details>
