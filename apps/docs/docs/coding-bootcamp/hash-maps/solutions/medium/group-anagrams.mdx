# Group Anagrams — Solution

## Problem Overview

Given an array of strings `strs`, group the **anagrams** together. You can return the answer in any order. An anagram is a word or phrase formed by rearranging the letters of a different word or phrase, using all the original letters exactly once.

## Brute Force Approach: Compare Every Pair

**Idea:** For each string, compare it with every other string to check if they are anagrams, and group matching ones together.

**Algorithm:**
1. Initialize a list of groups (each group is a list of strings)
2. For each string `s` in `strs`:
   - Check if `s` is an anagram of any existing group's representative
   - If yes, add `s` to that group
   - If no, create a new group with `s`
3. Return all groups

**Complexity Analysis:**
- **Time:** O(n^2 * k) — for each of n strings, we compare with up to n groups, each comparison O(k) to check anagram status
- **Space:** O(n * k) — storing all strings in groups

**Why this could be better:** Comparing every pair is prohibitively slow. A hash map with a canonical key groups all anagrams in a single pass.

## Optimal Approach: Sorted String Key

**Idea:** Two strings are anagrams if they have the same characters in the same frequency. Sorting each string produces a canonical form that is identical for all anagrams. Use this sorted string as a key in a hash map, and append each original string to the corresponding group.

**Algorithm:**
1. Create a hash map `groups` mapping a sorted string key to a list of original strings
2. For each string `s` in `strs`:
   - Sort the characters of `s` to produce `key`
   - Append `s` to `groups[key]`
3. Return all values from `groups`

**Why This Works:** Anagrams produce the same sorted string (e.g., "eat" and "tea" both sort to "aet"). By grouping strings with the same sorted key, we collect all anagrams together in one pass.

**Complexity Analysis:**
- **Time:** O(n * k log k) where n is the number of strings and k is the maximum length of a string — sorting each string dominates
- **Space:** O(n * k) — storing all strings in the hash map groups

**Optimization:** For lowercase letters, use a 26-element character frequency array as the key (e.g., "#1#0#2#...") to avoid sorting, achieving O(n * k) time.

## Deep Dive: How the Sorted-String Key Works

### Visual Walkthrough

For the input `strs = ["eat", "tea", "tan", "ate", "nat", "bat"]`, let's trace the algorithm:

```
Step 1: Process "eat"
  sorted = "aet" → groups["aet"] = ["eat"]

Step 2: Process "tea"
  sorted = "aet" → groups["aet"] = ["eat", "tea"]

Step 3: Process "tan"
  sorted = "ant" → groups["aet"] = ["eat", "tea"]
                        groups["ant"] = ["tan"]

Step 4: Process "ate"
  sorted = "aet" → groups["aet"] = ["eat", "tea", "ate"]
                        groups["ant"] = ["tan"]

Step 5: Process "nat"
  sorted = "ant" → groups["aet"] = ["eat", "tea", "ate"]
                        groups["ant"] = ["tan", "nat"]

Step 6: Process "bat"
  sorted = "abt" → groups["aet"] = ["eat", "tea", "ate"]
                        groups["ant"] = ["tan", "nat"]
                        groups["abt"] = ["bat"]

Result: [["eat", "tea", "ate"], ["tan", "nat"], ["bat"]]
```

### Edge Cases to Consider

- **Empty input array**: Returns an empty list — no groups to form.
- **Single string**: Returns a single group containing that string.
- **All strings are anagrams**: All strings map to the same key, producing one large group.
- **No strings are anagrams**: Each string produces a unique key, producing n groups of size 1.
- **Empty strings**: An empty string has a sorted key of "" (empty), so all empty strings group together.
- **Duplicate strings**: Duplicates naturally fall into the same group.

### Why Not Use a Frequency Array as the Key?

The sorted-string approach is simple and works for any character set. A frequency-array key (e.g., a 26-element tuple converted to a string like "#1#2#0#...") avoids the O(k log k) sorting cost, reducing total time to O(n * k). However, it requires:
- Knowing the character set size (e.g., 26 for lowercase letters)
- Converting the array to a hashable key (string concatenation)

The sorted-string approach is preferred for its simplicity unless the strings are very long.

def group_anagrams(strs: list[str]) -> list[list[str]]:
    from collections import defaultdict

    groups = defaultdict(list)  # Map sorted key -> list of strings

    for s in strs:
        # Sort the string to create a canonical key
        key = ''.join(sorted(s))
        groups[key].append(s)

    return list(groups.values())

import java.util.*;

class Solution {
    public List<List<String>> groupAnagrams(String[] strs) {
        Map<String, List<String>> groups = new HashMap<>();  // Map sorted key -> list of strings

        for (String s : strs) {
            // Sort the string to create a canonical key
            char[] chars = s.toCharArray();
            Arrays.sort(chars);
            String key = new String(chars);

            // Add to the appropriate group
            groups.computeIfAbsent(key, k -> new ArrayList<>()).add(s);
        }

        return new ArrayList<>(groups.values());
    }
}

fn group_anagrams(strs: Vec<String>) -> Vec<Vec<String>> {

    for s in strs {

        let mut chars: Vec<char> = s.chars().collect();
        chars.sort_unstable();
        let key: String = chars.into_iter().collect();

import "sort"

func groupAnagrams(strs []string) [][]string {
    groups := make(map[string][]string)  // Map sorted key -> list of strings

    for _, s := range strs {
        // Sort the string to create a canonical key
        chars := []byte(s)
        sort.Slice(chars, func(i, j int) bool { return chars[i] < chars[j] })
        key := string(chars)

        groups[key] = append(groups[key], s)
    }

    result := make([][]string, 0, len(groups))
    for _, group := range groups {
        result = append(result, group)
    }
    return result
}

## Key Takeaways

- **Canonical form via sorting** — anagrams produce the same sorted string, making it a perfect hash map key.
- **Single pass grouping** — each string is processed exactly once; the hash map handles the grouping.
- **Frequency array alternative** — use a 26-element count array as the key to avoid O(k log k) sorting, achieving O(n * k) time.
- **Edge cases handled naturally** — empty strings, duplicates, and single characters all work without special logic.
