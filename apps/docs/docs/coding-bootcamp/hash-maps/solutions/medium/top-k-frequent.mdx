# Top K Frequent Elements — Solution

## Problem Overview

Given an integer array `nums` and an integer `k`, return the `k` most frequent elements. You may return the answer in any order.

## Brute Force Approach: Sort by Frequency

**Idea:** Count the frequency of each element, then sort the unique elements by their frequency in descending order and take the first `k`.

**Algorithm:**
1. Count frequencies using a hash map
2. Collect all unique elements into a list
3. Sort the list by frequency in descending order
4. Return the first `k` elements

**Complexity Analysis:**
- **Time:** O(n + m log m) where m is the number of unique elements — counting is O(n), sorting is O(m log m)
- **Space:** O(n) — storing the hash map and sorted list

**Why this could be better:** Sorting all unique elements is unnecessary when we only need the top `k`. We can avoid the O(m log m) sort by using bucket sort or a min-heap.

## Optimal Approach: Bucket Sort

**Idea:** Since the maximum frequency of any element is at most the length of the array, we can use bucket sort. Create an array of buckets where `buckets[freq]` stores all numbers with that frequency, then collect from the highest frequency downward.

**Algorithm:**
1. Count frequencies using a hash map
2. Create an array `buckets` of size `n + 1` where `buckets[i]` is a list of numbers with frequency `i`
3. For each `(num, count)` in the frequency map, append `num` to `buckets[count]`
4. Iterate from the highest frequency (`n`) down to `1`, collecting numbers until we have `k` elements

**Why This Works:** The maximum possible frequency of any element is `n` (the array length). By creating a bucket for each possible frequency, we can group elements by frequency in O(n) time without sorting. Collecting from the highest bucket downward naturally gives us the most frequent elements first.

**Complexity Analysis:**
- **Time:** O(n) — one pass for counting, one pass for bucket fill, one pass for collection
- **Space:** O(n) — hash map and bucket array

## Deep Dive: How Bucket Sort Works

### Visual Walkthrough

For the input `nums = [1, 1, 1, 2, 2, 3], k = 2`, let's trace the algorithm:

```
Step 1: Count frequencies
  freq = {1: 3, 2: 2, 3: 1}

Step 2: Create buckets (size = n + 1 = 7)
  buckets[0] = []
  buckets[1] = [3]      ← numbers with frequency 1
  buckets[2] = [2]      ← numbers with frequency 2
  buckets[3] = [1]      ← numbers with frequency 3
  buckets[4] = []
  buckets[5] = []
  buckets[6] = []

Step 3: Collect from highest frequency downward
  Start at i = 6, go down to 1
  i = 6: buckets[6] is empty → skip
  i = 5: buckets[5] is empty → skip
  i = 4: buckets[4] is empty → skip
  i = 3: buckets[3] = [1] → result = [1], need 1 more
  i = 2: buckets[2] = [2] → result = [1, 2], have k = 2 → return [1, 2]
```

### Edge Cases to Consider

- **k equals number of unique elements**: All elements are returned.
- **All elements are the same**: Only one unique element, frequency equals `n`. The bucket at index `n` contains it.
- **Multiple elements with the same frequency**: They are collected together in the same bucket, and any ordering among them is valid.
- **Single element array**: Only one element with frequency 1; returns that element.
- **Large k**: k is guaranteed to be within the range of unique elements, so the collection always finds enough elements.

### Why Not Use a Min-Heap?

A min-heap of size k is another valid approach:
- Maintain a heap of size k, always keeping the k most frequent elements seen so far
- O(n log k) time, O(n + k) space
- Advantage: works well when k is small relative to n
- Tradeoff: O(n log k) vs O(n) for bucket sort

Bucket sort is optimal when the frequency range is bounded by n (which it always is here).

def top_k_frequent(nums: list[int], k: int) -> list[int]:
    from collections import defaultdict

    # Step 1: Count frequencies
    freq = defaultdict(int)
    for num in nums:
        freq[num] += 1

    # Step 2: Create buckets where index = frequency
    # Maximum frequency is at most len(nums)
    buckets = [[] for _ in range(len(nums) + 1)]
    for num, count in freq.items():
        buckets[count].append(num)

    # Step 3: Collect top k from highest frequency buckets
    result = []
    for i in range(len(buckets) - 1, 0, -1):
        for num in buckets[i]:
            result.append(num)
            if len(result) == k:
                return result

    return result

import java.util.*;

class Solution {
    public int[] topKFrequent(int[] nums, int k) {
        // Step 1: Count frequencies
        Map<Integer, Integer> freq = new HashMap<>();
        for (int num : nums) {
            freq.put(num, freq.getOrDefault(num, 0) + 1);
        }

        // Step 2: Create buckets where index = frequency
        List<Integer>[] buckets = new List[nums.length + 1];
        for (int i = 0; i < buckets.length; i++) {
            buckets[i] = new ArrayList<>();
        }
        for (Map.Entry<Integer, Integer> entry : freq.entrySet()) {
            buckets[entry.getValue()].add(entry.getKey());
        }

        // Step 3: Collect top k from highest frequency buckets
        int[] result = new int[k];
        int idx = 0;
        for (int i = buckets.length - 1; i >= 0 && idx < k; i--) {
            for (int num : buckets[i]) {
                result[idx++] = num;
                if (idx == k) break;
            }
        }

        return result;
    }
}

use std::collections::HashMap;

fn top_k_frequent(nums: Vec<i32>, k: i32) -> Vec<i32> {
    let k = k as usize;

    // Step 1: Count frequencies
    let mut freq = HashMap::new();
    for num in nums.iter() {
        *freq.entry(*num).or_insert(0) += 1;
    }

    // Step 2: Create buckets where index = frequency
    let mut buckets: Vec<Vec<i32>> = vec![vec![]; nums.len() + 1];
    for (num, count) in freq {
        buckets[count as usize].push(num);
    }

    // Step 3: Collect top k from highest frequency buckets
    let mut result = Vec::new();
    for i in (0..buckets.len()).rev() {
        for num in buckets[i].iter() {
            result.push(*num);
            if result.len() == k {
                return result;
            }
        }
    }

    result
}

#include <stdlib.h>

// Note: Full C solution requires a hash map implementation.
// This shows the conceptual approach using a simplified counting method.

typedef struct {
    int key;
    int value;
} MapEntry;

int cmp_freq(const void* a, const void* b) {
    MapEntry* ea = (MapEntry*)a;
    MapEntry* eb = (MapEntry*)b;
    return eb->value - ea->value;  // Descending order by frequency
}

int* topKFrequent(int* nums, int numsSize, int k, int* returnSize) {
    *returnSize = k;

    // Count frequencies using an offset array (simplified for bounded range)
    // For production, use a proper hash table
    MapEntry entries[20001];  // Offset by 10000 for negative values
    int entryCount = 0;

    int freq[20001] = {0};
    for (int i = 0; i < numsSize; i++) {
        freq[nums[i] + 10000]++;
    }

    // Collect non-zero entries
    for (int i = 0; i < 20001; i++) {
        if (freq[i] > 0) {
            entries[entryCount].key = i - 10000;
            entries[entryCount].value = freq[i];
            entryCount++;
        }
    }

    // Sort by frequency descending
    qsort(entries, entryCount, sizeof(MapEntry), cmp_freq);

    // Take top k
    int* result = (int*)malloc(k * sizeof(int));
    for (int i = 0; i < k; i++) {
        result[i] = entries[i].key;
    }

    return result;
}

func topKFrequent(nums []int, k int) []int {
    // Step 1: Count frequencies
    freq := make(map[int]int)
    for _, num := range nums {
        freq[num]++
    }

    // Step 2: Create buckets where index = frequency
    buckets := make([][]int, len(nums)+1)
    for num, count := range freq {
        buckets[count] = append(buckets[count], num)
    }

    // Step 3: Collect top k from highest frequency buckets
    result := make([]int, 0, k)
    for i := len(buckets) - 1; i >= 0 && len(result) < k; i-- {
        for _, num := range buckets[i] {
            result = append(result, num)
            if len(result) == k {
                return result
            }
        }
    }

    return result
}

## Key Takeaways

- **Bucket sort exploits the frequency bound** — since max frequency equals array length, we can sort in O(n) time.
- **Alternative: Min-heap of size k** — O(n log k) time, useful when k is small.
- **Alternative: Quickselect** — O(n) average time on the frequency array.
- **The hash map is essential** — it provides the frequency counts that drive the bucket sort.
- **Any order in the result is acceptable** — elements with the same frequency can appear in any order.
