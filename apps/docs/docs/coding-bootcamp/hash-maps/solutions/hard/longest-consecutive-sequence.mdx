# Longest Consecutive Sequence — Solution

## Problem Overview

Given an unsorted array of integers `nums`, return the length of the longest consecutive elements sequence. You must write an algorithm that runs in **O(n)** time.

## Brute Force Approach: Sorting

**Idea:** Sort the array, then scan for consecutive sequences.

**Algorithm:**
1. Sort the array in ascending order
2. Iterate through the sorted array, counting consecutive elements
3. When a gap is found, reset the counter and track the maximum length found
4. Return the maximum length

**Complexity Analysis:**
- **Time:** O(n log n) — sorting dominates the runtime
- **Space:** O(1) or O(n) depending on the sorting algorithm

**Why this could be better:** Sorting gives O(n log n), but the problem explicitly requires O(n) time. We need to use a hash set to achieve linear time.

## Optimal Approach: Hash Set Sequence Detection

**Idea:** Insert all numbers into a hash set for O(1) lookups. Then iterate through the set and only count a sequence when the current number is the **start** of a sequence (i.e., `num - 1` is not in the set). This ensures each number is visited at most twice, achieving O(n) time.

**Algorithm:**
1. Insert all numbers into a hash set `numSet`
2. Initialize `longest = 0`
3. For each `num` in `numSet`:
   - If `num - 1` is NOT in `numSet` (this is the start of a sequence):
     - Set `current = num` and `length = 1`
     - While `current + 1` is in `numSet`, increment `current` and `length`
     - Update `longest = max(longest, length)`
4. Return `longest`

**Why This Works:** By only counting from the start of each sequence, we avoid the O(n^2) behavior of checking every number's full sequence repeatedly. The sequence start check (`num - 1` not in set) ensures each element is processed at most once as part of a sequence count.

**Complexity Analysis:**
- **Time:** O(n) — each number is inserted into the set once and visited at most once during sequence counting
- **Space:** O(n) — the hash set stores all unique numbers

## Deep Dive: How Sequence Detection Works

### Visual Walkthrough

For the input `nums = [100, 4, 200, 1, 3, 2]`, let's trace the algorithm:

```
Step 1: Build hash set
  numSet = {100, 4, 200, 1, 3, 2}

Step 2: Iterate through the set
  num = 100: 99 NOT in set → start of sequence
    current = 100, length = 1
    101 NOT in set → stop
    longest = max(0, 1) = 1

  num = 4: 3 IS in set → NOT a sequence start → skip

  num = 200: 199 NOT in set → start of sequence
    current = 200, length = 1
    201 NOT in set → stop
    longest = max(1, 1) = 1

  num = 1: 0 NOT in set → start of sequence
    current = 1, length = 1
    2 IS in set → current = 2, length = 2
    3 IS in set → current = 3, length = 3
    4 IS in set → current = 4, length = 4
    5 NOT in set → stop
    longest = max(1, 4) = 4

  result = 4
```

### Edge Cases to Consider

- **Empty array**: Return 0 — no sequences exist.
- **Single element**: Return 1 — a single element is a sequence of length 1.
- **All identical elements**: The set deduplicates to one element, so the longest sequence is 1.
- **All elements already consecutive**: The entire array forms one sequence. The algorithm counts it in one pass.
- **Negative numbers**: The hash set handles negative numbers naturally; the consecutive check works with negative values.
- **Duplicates**: The hash set automatically deduplicates, so duplicate values don't inflate the sequence length.

### Why Not Use a Union-Find Data Structure?

Union-Find (Disjoint Set Union) can also solve this problem by unioning consecutive numbers. While it also achieves O(n * alpha(n)) time, the hash set approach is simpler and avoids the overhead of implementing a DSU.

def longest_consecutive(nums: list[int]) -> int:
    # Insert all numbers into a hash set for O(1) lookups

class Solution {
    public int longestConsecutive(int[] nums) {
        // Insert all numbers into a hash set
        Set<Integer> numSet = new HashSet<>();
        for (int num : nums) {
            numSet.add(num);

                int length = 1;

                // Count consecutive numbers
                while (numSet.contains(current + 1)) {
                    current++;
                    length++;
                }

                // Update the longest sequence found
                longest = Math.max(longest, length);
            }
        }

        return longest;
    }
}

#include <stdlib.h>
#include <stdbool.h>

// Simple hash set implementation using a fixed-size table with chaining
#define HASH_SET_SIZE 10007

typedef struct Node {
    int val;
    struct Node* next;
} Node;

void insert(Node** buckets, int key, int val) {
    Node* newNode = (Node*)malloc(sizeof(Node));
    newNode->val = val;
    newNode->next = buckets[key];
    buckets[key] = newNode;
}

bool contains(Node** buckets, int key, int val) {
    Node* curr = buckets[key];
    while (curr) {
        if (curr->val == val) return true;
        curr = curr->next;
    }
    return false;
}

int longestConsecutive(int* nums, int numsSize) {
    // Build hash set
    Node* buckets[HASH_SET_SIZE] = {NULL};

    for (int i = 0; i < numsSize; i++) {
        int key = nums[i] % HASH_SET_SIZE;
        if (key < 0) key = -key;  // Handle negative numbers
        if (!contains(buckets, key, nums[i])) {
            insert(buckets, key, nums[i]);
        }
    }

    int longest = 0;

    for (int i = 0; i < numsSize; i++) {
        int key = nums[i] % HASH_SET_SIZE;
        if (key < 0) key = -key;

        // Only start if this is the start of a sequence
        int prevKey = (nums[i] - 1) % HASH_SET_SIZE;
        if (prevKey < 0) prevKey = -prevKey;

        if (!contains(buckets, prevKey, nums[i] - 1)) {
            int current = nums[i];
            int length = 1;

            while (1) {
                int nextKey = (current + 1) % HASH_SET_SIZE;
                if (nextKey < 0) nextKey = -nextKey;
                if (!contains(buckets, nextKey, current + 1)) break;
                current++;
                length++;
            }

            if (length > longest) longest = length;
        }
    }

    return longest;
}

func longestConsecutive(nums []int) int {

    numSet := make(map[int]struct{})
    for _, num := range nums {

    }

                    break
                }
                current++
                length++
            }

            // Update the longest sequence found
            if length > longest {
                longest = length
            }
        }
    }

    return longest
}

## Key Takeaways

- **O(n) time is the constraint** — sorting would give O(n log n), but the hash set approach achieves the required O(n).
- **Only count from sequence starts** — by checking `num - 1` is not in the set, each number is processed at most twice.
- **Hash set deduplication** — duplicates are automatically handled, preventing inflated sequence counts.
- **The approach is optimal** — you must at least look at each element once, so O(n) is the lower bound.
- **Alternative: Union-Find** — DSU can also solve this in near-linear time, but the hash set approach is simpler.
