# Valid Anagram — Solution

## Problem Overview

Given two strings `s` and `t`, return `true` if `t` is an **anagram** of `s`, and `false` otherwise. An anagram is a word or phrase formed by rearranging the letters of a different word or phrase, using all the original letters exactly once.

## Brute Force Approach: Sorting

**Idea:** Sort both strings alphabetically. If they are anagrams, the sorted versions will be identical.

**Algorithm:**
1. If the lengths of `s` and `t` differ, return `false`
2. Sort the characters in `s` and `t`
3. Compare the sorted strings — return `true` if they match, `false` otherwise

**Complexity Analysis:**
- **Time:** O(n log n + m log m) — sorting dominates the runtime
- **Space:** O(n + m) — we create sorted copies of both strings

**Why this could be better:** Sorting is unnecessary when we only need to compare character frequencies. A hash map can achieve O(n) time by counting characters directly.

## Optimal Approach: Character Frequency Count

**Idea:** Use a hash map to count character occurrences in `s`, then decrement counts for characters in `t`. If any count goes negative or any character in `t` is missing from the map, the strings are not anagrams.

**Algorithm:**
1. If `len(s) != len(t)`, return `false` immediately
2. Create a hash map `counts`
3. Iterate through `s`: increment `counts[ch]` for each character
4. Iterate through `t`: if `ch` is not in `counts`, return `false`; decrement `counts[ch]`; if `counts[ch] < 0`, return `false`
5. Return `true` if all checks pass

**Why This Works:** Anagrams have identical character frequencies. By incrementing counts for `s` and decrementing for `t`, the net count for each character must be zero. A negative count or missing key means `t` has a character `s` does not, or has more of it than `s` does.

**Complexity Analysis:**
- **Time:** O(n + m) — each string is traversed exactly once
- **Space:** O(k) where k is the size of the character set (at most 26 for lowercase letters, or O(n) for general Unicode)

## Deep Dive: How the Frequency Count Works

### Visual Walkthrough

For the input `s = "anagram"`, `t = "nagaram"`, let's trace the algorithm:

```
Step 1: Count characters in s = "anagram"
counts = {a: 3, n: 1, g: 1, r: 1, m: 1}

Step 2: Decrement using characters in t = "nagaram"
  'n' → counts = {a: 3, n: 0, g: 1, r: 1, m: 1}
  'a' → counts = {a: 2, n: 0, g: 1, r: 1, m: 1}
  'g' → counts = {a: 2, n: 0, g: 0, r: 1, m: 1}
  'a' → counts = {a: 1, n: 0, g: 0, r: 1, m: 1}
  'r' → counts = {a: 1, n: 0, g: 0, r: 0, m: 1}
  'a' → counts = {a: 0, n: 0, g: 0, r: 0, m: 1}
  'm' → counts = {a: 0, n: 0, g: 0, r: 0, m: 0}

All counts are zero → return true
```

### Edge Cases to Consider

- **Different lengths**: Return `false` immediately — anagrams must have the same length.
- **Empty strings**: Both empty strings are anagrams (lengths equal, loop falls through, returns `true`).
- **Single character**: `s = "a"`, `t = "a"` returns `true`; `s = "a"`, `t = "b"` returns `false`.
- **All characters the same**: `s = "aaa"`, `t = "aaa"` works correctly.
- **Case sensitivity**: For the problem constraints (lowercase only), case is not an issue. For Unicode, normalize before comparing.

### Why Not Use Sorting?

Sorting is the simplest correct approach, but it costs O(n log n). Counting with a hash map achieves O(n) time by processing each character exactly once. For the maximum constraint of 5 x 10^4 characters, O(n) is significantly faster than O(n log n).

def is_anagram(s: str, t: str) -> bool:
    # If lengths differ, they cannot be anagrams
    if len(s) != len(t):
        return False

    # Count character frequencies in s
    counts = {}
    for ch in s:
        counts[ch] = counts.get(ch, 0) + 1

    # Decrement counts using characters from t
    for ch in t:
        if ch not in counts:     # Character not in s at all
            return False
        counts[ch] -= 1
        if counts[ch] < 0:       # t has more of this char than s
            return False

    return True  # All counts balanced

import java.util.HashMap;
import java.util.Map;

class Solution {
    public boolean isAnagram(String s, String t) {
        // If lengths differ, they cannot be anagrams
        if (s.length() != t.length()) {
            return false;
        }

        // Count character frequencies using a hash map
        Map<Character, Integer> counts = new HashMap<>();

        // Increment counts for each character in s
        for (char ch : s.toCharArray()) {
            counts.put(ch, counts.getOrDefault(ch, 0) + 1);
        }

        // Decrement counts for each character in t
        for (char ch : t.toCharArray()) {
            if (!counts.containsKey(ch)) {
                return false;          // Character not in s
            }
            counts.put(ch, counts.get(ch) - 1);
            if (counts.get(ch) < 0) {
                return false;          // t has more of this char
            }
        }

        return true;  // All counts balanced
    }
}

use std::collections::HashMap;

fn is_anagram(s: String, t: String) -> bool {
    // If lengths differ, they cannot be anagrams
    if s.len() != t.len() {
        return false;
    }

    // Count character frequencies
    let mut counts: HashMap<char, i32> = HashMap::new();

    // Increment for characters in s
    for ch in s.chars() {
        *counts.entry(ch).or_insert(0) += 1;
    }

    // Decrement for characters in t
    for ch in t.chars() {
        if let Some(count) = counts.get_mut(&ch) {
            *count -= 1;
            if *count < 0 {
                return false;    // t has more of this char
            }
        } else {
            return false;        // Character not in s
        }
    }

    true  // All counts balanced
}

bool isAnagram(char* s, char* t) {
    int lenS = strlen(s);
    int lenT = strlen(t);

    // If lengths differ, they cannot be anagrams
    if (lenS != lenT) {
        return false;
    }

    // For lowercase letters, a fixed array is more efficient
    // than a hash map (26 possible characters)
    int counts[26] = {0};

    // Increment counts for characters in s
    for (int i = 0; i < lenS; i++) {
        counts[s[i] - 'a']++;
    }

    // Decrement counts for characters in t
    for (int i = 0; i < lenS; i++) {
        counts[t[i] - 'a']--;
    }

    // Check all counts are zero
    for (int i = 0; i < 26; i++) {
        if (counts[i] != 0) {
            return false;  // Mismatched frequency
        }
    }

    return true;  // All counts balanced
}

## Key Takeaways

- **Length check first** — a quick early exit when lengths differ makes the rest of the algorithm simpler.
- **Hash map for frequency counting** — O(1) average lookups give O(n) overall time.
- **Array optimization** — when the character set is constrained (e.g., only lowercase letters), a fixed-size array outperforms a hash map.
- **The sorting alternative** — O(n log n) time is correct but slower; the counting approach is optimal.
