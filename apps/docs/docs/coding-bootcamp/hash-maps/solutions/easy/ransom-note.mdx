# Ransom Note — Solution

## Problem Overview

Given two strings `ransomNote` and `magazine`, return `true` if `ransomNote` can be constructed by using the letters from `magazine` and `false` otherwise. Each letter in `magazine` can only be used once.

## Approach: Frequency Counting

**Idea:** Count how many times each letter appears in the magazine. Then iterate through the ransom note, decrementing each letter's count. If any count drops below zero, the ransom note cannot be constructed.

**Algorithm:**
1. Create a frequency map (or array of size 26 for lowercase letters) to count letters in `magazine`
2. For each character in `magazine`, increment its count
3. For each character in `ransomNote`:
   - If the count for that character is 0, return `false`
   - Decrement the count
4. Return `true` if all characters are available

**Complexity Analysis:**
- **Time:** O(n + m) where n is the length of `ransomNote` and m is the length of `magazine`
- **Space:** O(1) — the alphabet is fixed at 26 characters (or O(k) for a general hash map)

def ransom_note(note: str, magazine: str) -> bool:
    freq = {}
    for ch in magazine:
        freq[ch] = freq.get(ch, 0) + 1
    for ch in note:
        if freq.get(ch, 0) == 0:
            return False
        freq[ch] -= 1
    return True

class Solution {
    public boolean canConstruct(String ransomNote, String magazine) {
        int[] freq = new int[26];
        for (char c : magazine.toCharArray()) {
            freq[c - 'a']++;
        }
        for (char c : ransomNote.toCharArray()) {
            if (freq[c - 'a'] == 0) return false;
            freq[c - 'a']--;
        }
        return true;
    }
}

fn can_construct(ransom_note: &str, magazine: &str) -> bool {

## Key Takeaways

- **Fixed-size frequency array** (size 26 for lowercase letters) is more efficient than a hash map when the character set is known and limited
- **Greedy consumption** of character counts works because each letter can be used at most once
- Both O(n + m) time and O(1) space are optimal
