# Contains Duplicate — Solution

## Problem Overview

Given an integer array `nums`, return `true` if any value appears **at least twice** in the array, and return `false` if every element is distinct.

## Brute Force Approach: Nested Loops

**Idea:** Compare every pair of elements to check for duplicates.

**Algorithm:**
1. Loop through each index `i` from `0` to `n-1`
2. For each `i`, loop through `j` from `i+1` to `n-1`
3. If `nums[i] == nums[j]`, return `true`
4. If no pair matches, return `false`

**Complexity Analysis:**
- **Time:** O(n^2) — in the worst case we check every pair
- **Space:** O(1) — no extra space

**Why this could be better:** O(n^2) is too slow for arrays of size up to 10^5. We can solve this in O(n) time using a hash set to track seen elements.

## Optimal Approach: Hash Set

**Idea:** Use a hash set to store numbers we have already seen. For each number, check if it is already in the set. If so, a duplicate exists.

**Algorithm:**
1. Create an empty hash set `seen`
2. Iterate through each `num` in `nums`:
   - If `num` is already in `seen`, return `true`
   - Add `num` to `seen`
3. If the loop finishes, return `false`

**Why This Works:** A hash set provides O(1) average-time lookups. By tracking each number as we traverse the array, we detect the first duplicate the moment it appears. If no number appears twice, the set will contain all elements by the end.

**Complexity Analysis:**
- **Time:** O(n) — single pass through the array with O(1) hash set operations
- **Space:** O(n) — in the worst case, we store all n elements in the hash set

## Deep Dive: How the Hash Set Approach Works

### Visual Walkthrough

For the input `nums = [1, 2, 3, 1]`, let's trace the algorithm:

```
Initial: seen = {}

Step 1: num = 1
  1 is NOT in seen → add 1
  seen = {1}

Step 2: num = 2
  2 is NOT in seen → add 2
  seen = {1, 2}

Step 3: num = 3
  3 is NOT in seen → add 3
  seen = {1, 2, 3}

Step 4: num = 1
  1 IS in seen → return true (duplicate found)
```

### Edge Cases to Consider

- **Empty array or single element**: No duplicates possible. The loop never finds a match, returns `false`.
- **All distinct elements**: The set grows to contain all elements, and the loop finishes normally.
- **All identical elements**: The first element is added to the set; the second element immediately triggers a duplicate.
- **Negative numbers**: Hash sets handle negative integers naturally.
- **Very large arrays**: The algorithm is O(n) and handles the maximum constraint of 10^5 elements efficiently.

### Why Not Sort?

Sorting the array and checking adjacent elements is an alternative approach:
- **Sorting:** O(n log n) time, O(1) space (in-place sort)
- **Hash set:** O(n) time, O(n) space

The hash set approach is the most natural choice for this problem — it directly addresses the question "have I seen this before?" with minimal overhead.

#include <stdbool.h>
#include <stdlib.h>

// Simple hash set using a fixed-size table with chaining
#define HASH_SET_SIZE 10007

typedef struct Node {
    int val;
    struct Node* next;
} Node;

bool containsDuplicate(int* nums, int numsSize) {
    // Allocate hash table buckets (zero-initialized)
    Node* buckets[HASH_SET_SIZE] = {NULL};

    for (int i = 0; i < numsSize; i++) {
        int key = nums[i] % HASH_SET_SIZE;
        if (key < 0) key = -key;  // Handle negative numbers

        // Search for the value in the bucket
        Node* curr = buckets[key];
        while (curr != NULL) {
            if (curr->val == nums[i]) {
                return true;      // Duplicate found
            }
            curr = curr->next;
        }

        // Insert new value at the head of the bucket
        Node* newNode = (Node*)malloc(sizeof(Node));
        newNode->val = nums[i];
        newNode->next = buckets[key];
        buckets[key] = newNode;
    }

    return false;  // All elements are distinct
}

## Key Takeaways

- **Hash set is the natural choice** — it directly models the "have I seen this before?" question with O(1) lookups.
- **Early exit** — return as soon as the first duplicate is found, avoiding unnecessary work.
- **Space-time tradeoff** — O(n) space buys O(n) time instead of O(n log n) with sorting.
- **Alternative (sorting)** — sort the array, then check adjacent elements: O(n log n) time, O(1) space.
