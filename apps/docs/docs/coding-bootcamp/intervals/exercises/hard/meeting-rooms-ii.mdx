# Meeting Rooms II

**Difficulty:** Hard

Given an array of meeting time intervals consisting of start and end times `[start, end]`, find the **minimum number of conference rooms** required to host all meetings without conflicts.

## Example 1

**Input:** `intervals = [[0, 30], [5, 10], [15, 20]]`  
**Output:** `2`  
**Explanation:** Meeting [0, 30] overlaps with both [5, 10] and [15, 20]. At peak, two meetings run concurrently, so 2 rooms are needed.

## Example 2

**Input:** `intervals = [[7, 10], [2, 4]]`  
**Output:** `1`  
**Explanation:** The meetings do not overlap, so 1 room suffices.

## Input Specification

The first line contains an integer `n`, the number of intervals.  
Each of the next `n` lines contains two integers `start` and `end` representing a meeting interval.

## Output Specification

Print a single integer representing the minimum number of conference rooms required.

## Constraints

- `0 <= n <= 10⁴`
- `-10⁶ <= start < end <= 10⁶`

## Hints

<details>
<summary>Hint 1</summary>

Think of this as finding the maximum number of overlapping intervals at any point in time. The minimum rooms needed equals the maximum concurrency.
</details>

<details>
<summary>Hint 2</summary>

Separate start and end times into two arrays. Sort both. Use two pointers to simulate rooms being booked and freed as time progresses.
</details>

<details>
<summary>Hint 3</summary>

Alternatively, use a min-heap (priority queue) to track the end times of meetings currently using a room. When a new meeting starts, free any rooms whose end time has passed.
</details>

</ProblemEditor>
