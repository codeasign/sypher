# Merge Intervals

**Difficulty:** Medium

Given an array of intervals where `intervals[i] = [start, end]`, merge all overlapping intervals and return an array of the non-overlapping intervals that cover all the intervals in the input.

## Example 1

**Input:** `intervals = [[1, 3], [2, 6], [8, 10], [15, 18]]`  
**Output:** `[[1, 6], [8, 10], [15, 18]]`  
**Explanation:** Since intervals `[1, 3]` and `[2, 6]` overlap, they are merged into `[1, 6]`.

## Example 2

**Input:** `intervals = [[1, 4], [4, 5]]`  
**Output:** `[[1, 5]]`  
**Explanation:** Intervals `[1, 4]` and `[4, 5]` are considered overlapping (the end of one equals the start of another).

## Input Specification

The first line contains an integer `n`, the number of intervals.  
Each of the next `n` lines contains two integers `start` and `end` representing an interval.

## Output Specification

Print each merged interval on its own line as `start end`. If there are no intervals, output nothing.

## Constraints

- `0 <= n <= 10⁴`
- `-10⁶ <= start <= end <= 10⁶`

## Hints

<details>
<summary>Hint 1</summary>

Sort the intervals by start time. This is the first step in almost every interval problem.
</details>

<details>
<summary>Hint 2</summary>

After sorting, iterate through the intervals. Keep track of the current merged interval. If the next interval overlaps with it, extend the current interval's end. Otherwise, add the current interval to the result and start a new one.
</details>

<details>
<summary>Hint 3</summary>

Two intervals overlap if `current_end >= next_start`. When merging, take the maximum of the two end values: `new_end = max(current_end, next_end)`.
</details>
