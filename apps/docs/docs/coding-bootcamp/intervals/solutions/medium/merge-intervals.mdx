# Merge Intervals — Solution

## Problem Overview

Given an array of intervals, merge all overlapping intervals. This is the natural extension of the Meeting Rooms problem: instead of just detecting overlap, we need to combine overlapping intervals into a single continuous interval.

## Brute Force Approach

**Idea:** For each interval, check every other interval. If they overlap, merge them and restart the process until no more merges are possible.

**Algorithm:**
1. For each interval `i`, compare it with every other interval `j`
2. If `i` and `j` overlap, replace `i` with the merged interval `[min(start_i, start_j), max(end_i, end_j)]`, remove `j`, and restart from the beginning
3. Repeat until no more merges are possible

**Complexity Analysis:**
- **Time:** O(n³) in the worst case — each merge restarts the scan
- **Space:** O(1) — modification is done in-place (excluding the removal cost)

**Why this could be better:** This approach is extremely inefficient for large inputs. Sorting first lets us merge in a single pass.

## Optimal Approach: Sort and Merge

**Idea:** Sort intervals by start time, then iterate in a single pass, merging overlapping intervals as we go.

**Algorithm:**
1. Sort intervals by start time
2. Initialize `result` with the first interval
3. For each remaining interval `[start, end]`:
   - If `start <= last_end` (overlap or adjacent): update `last_end = max(last_end, end)`
   - Else: add the new interval to `result`
4. Return `result`

**Why This Works:** After sorting by start time, overlapping intervals become adjacent in the sorted order. We can process them linearly, merging as we go, because any interval that will merge with the current one must come immediately after it in the sorted order (or be reachable through a chain of adjacent merges).

**Complexity Analysis:**
- **Time:** O(n log n) — dominated by sorting
- **Space:** O(n) — to store the result (or O(1) if merging in-place)

## Deep Dive: How the Sort and Merge Works

### Visual Walkthrough

For the input `intervals = [[1, 3], [2, 6], [8, 10], [15, 18]]`:

```
Step 1: Sort by start time (already sorted)
        [(1,3), (2,6), (8,10), (15,18)]

Step 2: Initialize result with first interval
        result = [(1,3)]
        last_end = 3

Step 3: Process remaining intervals
        Interval (2,6): 2 <= 3 → overlap
            last_end = max(3, 6) = 6
            result = [(1,6)]

        Interval (8,10): 8 <= 6? No → no overlap
            Add (8,10) to result
            result = [(1,6), (8,10)]
            last_end = 10

        Interval (15,18): 15 <= 10? No → no overlap
            Add (15,18) to result
            result = [(1,6), (8,10), (15,18)]

Result: [[1,6], [8,10], [15,18]]
```

### Handling Adjacent Intervals

In this problem, intervals `[1, 4]` and `[4, 5]` are considered overlapping because `4 >= 4`. This means back-to-back meetings should be merged into a single interval `[1, 5]`. The condition `current_end >= next_start` (rather than `>`) handles this.

### Edge Cases

- **Empty input (n = 0):** Return an empty list
- **Single interval:** Return it as-is
- **One interval completely inside another:** `[1, 10]` and `[2, 5]` should merge to `[1, 10]` (take max of ends)
- **Multiple merges chaining:** `[1, 3], [2, 4], [3, 5]` all merge into `[1, 5]`
- **All identical intervals:** `[2, 2], [2, 2], [2, 2]` all merge into `[2, 2]`
- **Negative values:** Handled naturally since sorting and comparison work on all integers

### Why Start-Time Sorting Matters

Sorting by start time creates a linear ordering where:
- All intervals that could merge with interval `i` are guaranteed to appear after `i` in the sorted list
- You never need to look backward — once an interval is finalized in the result, no future interval can merge with it (because any future interval has a greater or equal start time)
- This enables the single-pass O(n) merge after the O(n log n) sort

import java.util.Arrays;
import java.util.ArrayList;
import java.util.List;

class Solution {
    public int[][] merge(int[][] intervals) {
        if (intervals.length == 0) return new int[0][];

        // Sort by start time
        Arrays.sort(intervals, (a, b) -> Integer.compare(a[0], b[0]));

        List<int[]> result = new ArrayList<>();
        int[] current = intervals[0];
        result.add(current);

        for (int i = 1; i < intervals.length; i++) {
            int[] next = intervals[i];
            // If current interval overlaps with next, merge
            if (next[0] <= current[1]) {
                current[1] = Math.max(current[1], next[1]);
            } else {
                current = next;
                result.add(current);
            }
        }
        return result.toArray(new int[result.size()][]);
    }
}

fn merge_intervals(intervals: Vec<Vec<i32>>) -> Vec<Vec<i32>> {
    if intervals.is_empty() {
        return vec![];
    }

    let mut intervals = intervals;
    // Sort by start time
    intervals.sort_by(|a, b| a[0].cmp(&b[0]));

    let mut result: Vec<Vec<i32>> = vec![intervals[0].clone()];

    for i in 1..intervals.len() {
        let current_end = result.last().unwrap()[1];
        let next_start = intervals[i][0];
        let next_end = intervals[i][1];
        // If current interval overlaps with next, merge
        if next_start <= current_end {
            result.last_mut().unwrap()[1] = current_end.max(next_end);
        } else {
            result.push(intervals[i].clone());
        }
    }
    result
}

int cmp(const void* a, const void* b) {
    int* ia = *(int**)a;
    int* ib = *(int**)b;
    return ia[0] - ib[0];
}

int** merge(int** intervals, int intervalsSize, int* intervalsColSize, int* returnSize, int** returnColumnSizes) {
    if (intervalsSize == 0) {
        *returnSize = 0;
        *returnColumnSizes = NULL;
        return NULL;
    }

    // Sort by start time
    qsort(intervals, intervalsSize, sizeof(int*), cmp);

    int** result = malloc(intervalsSize * sizeof(int*));
    *returnColumnSizes = malloc(intervalsSize * sizeof(int));

    result[0] = malloc(2 * sizeof(int));
    result[0][0] = intervals[0][0];
    result[0][1] = intervals[0][1];
    (*returnColumnSizes)[0] = 2;
    int count = 1;

    for (int i = 1; i < intervalsSize; i++) {
        int lastEnd = result[count - 1][1];
        int nextStart = intervals[i][0];
        int nextEnd = intervals[i][1];
        // If current interval overlaps with next, merge
        if (nextStart <= lastEnd) {
            if (nextEnd > lastEnd) {
                result[count - 1][1] = nextEnd;
            }
        } else {
            result[count] = malloc(2 * sizeof(int));
            result[count][0] = nextStart;
            result[count][1] = nextEnd;
            (*returnColumnSizes)[count] = 2;
            count++;
        }
    }

    *returnSize = count;
    return result;
}

using System;

class Solution {
    public int[][] Merge(int[][] intervals) {
        if (intervals.Length == 0) return new int[0][];

        // Sort by start time
        var sorted = intervals.OrderBy(x => x[0]).ToArray();

        var result = new List<int[]> { sorted[0] };

        for (int i = 1; i < sorted.Length; i++) {
            int currentEnd = result[result.Count - 1][1];
            int nextStart = sorted[i][0];
            int nextEnd = sorted[i][1];
            // If current interval overlaps with next, merge
            if (nextStart <= currentEnd) {
                result[result.Count - 1][1] = Math.Max(currentEnd, nextEnd);
            } else {
                result.Add(sorted[i]);
            }
        }
        return result.ToArray();
    }
}

## Key Takeaways

- **Sort then merge** is the canonical approach for interval merging
- Use `<=` (not `<`) for the overlap condition when adjacent intervals should merge
- When merging, take the maximum of the two end values — not just the second interval's end
- The single-pass merge works because sorting guarantees correct ordering
- This pattern is the building block for many harder interval problems
