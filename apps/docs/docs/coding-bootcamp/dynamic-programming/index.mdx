# Dynamic Programming

Dynamic Programming breaks problems into overlapping subproblems and stores results to avoid recomputation.

⏱ 30 minutes

## What You Will Learn

- Identifying DP problems: optimal substructure and overlapping subproblems
- Top-down (memoization) vs bottom-up (tabulation)
- Common DP patterns: 0/1 Knapsack, LCS, LIS

## The Concept

Solve a problem by combining solutions to subproblems. Store (memoize) subproblem results to avoid recomputation. This transforms exponential brute force into polynomial time.

## Common Patterns

| Pattern | Example | Approach |
|---------|---------|----------|
| Fibonacci-style | Climbing Stairs | dp[i] = dp[i-1] + dp[i-2] |
| 0/1 Knapsack | Subset Sum | dp[i][w] = max(dp[i-1][w], dp[i-1][w-wi] + vi) |
| Longest Common | LCS, LIS | dp[i][j] based on match/mismatch |
| Grid | Unique Paths | dp[r][c] = dp[r-1][c] + dp[r][c-1] |
| Interval | Burst Balloons | dp[i][j] over subarray range |
| State Machine | Best Time to Buy/Sell Stock | dp[i][k][hold] state transitions |

## Complexity

- **Time**: O(n × m) typical for 2D DP, varies by problem
- **Space**: O(n × m) for table, can often be optimized to O(m) or O(n)
