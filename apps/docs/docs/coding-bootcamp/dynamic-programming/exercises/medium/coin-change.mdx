# Coin Change

**Difficulty:** Medium

You are given an integer array `coins` representing coins of different denominations and an integer `amount` representing a total amount of money.

Return the **fewest number of coins** needed to make up that amount. If that amount of money cannot be made up by any combination of the coins, return `-1`.

You may assume that you have an **infinite number** of each kind of coin.

## Example 1

**Input:** `coins = [1, 2, 5]`, `amount = 11`  
**Output:** `3`  
**Explanation:** `11 = 5 + 5 + 1`

## Example 2

**Input:** `coins = [2]`, `amount = 3`  
**Output:** `-1`

## Example 3

**Input:** `coins = [1]`, `amount = 0`  
**Output:** `0`

## Input Specification

The first line contains two integers: `n` (the number of coin denominations) and `amount` (the target amount).

The second line contains `n` space-separated integers representing the coin denominations.

## Output Specification

Print a single integer: the minimum number of coins needed to make the amount, or `-1` if impossible.

## Constraints

- `1 <= n <= 100`
- `1 <= coins[i] <= 10^4`
- `0 <= amount <= 10^4`

## Hints

<details>
<summary>Hint 1</summary>

Think of this as a dynamic programming problem where you build up from smaller amounts to larger ones. If you know the minimum coins needed for every amount less than `i`, how can you compute the minimum for `i`?
</details>

<details>
<summary>Hint 2</summary>

For each amount `i`, try subtracting each coin denomination. If `i - coin >= 0`, then `dp[i] = min(dp[i], dp[i - coin] + 1)`.
</details>

<details>
<summary>Hint 3</summary>

Initialize `dp[0] = 0` and all other entries to a large number (like `amount + 1` or `infinity`). If `dp[amount]` remains large, return `-1`.
</details>
