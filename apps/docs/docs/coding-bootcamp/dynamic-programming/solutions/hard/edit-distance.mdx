# Edit Distance — Solution

## Problem Overview

Given two strings `word1` and `word2`, find the minimum number of operations (insert, delete, replace) to convert one to the other. This is also known as the **Levenshtein distance**, a classic 2D dynamic programming problem that measures how dissimilar two strings are.

## Brute Force Approach: Recursion

**Idea:** Recursively explore all possible edit sequences at each character position.

**Algorithm:**
1. If `word1` is empty, return `len(word2)` (insert all word2 characters)
2. If `word2` is empty, return `len(word1)` (delete all word1 characters)
3. If first characters match, recursively process the rest
4. Otherwise, consider three operations and take the minimum:
   - Insert: `1 + f(word1, word2[1:])`
   - Delete: `1 + f(word1[1:], word2)`
   - Replace: `1 + f(word1[1:], word2[1:])`

**Complexity Analysis:**
- **Time:** O(3^(m+n)) — exponential, three branches at each position
- **Space:** O(m + n) — recursion stack depth

**Why this is inefficient:** The recursion explores every possible edit sequence. For strings of length 500, this is astronomically infeasible.

## Optimal Approach: Dynamic Programming (Bottom-Up 2D Table)

**Idea:** Build a 2D DP table where `dp[i][j]` is the edit distance between `word1[0..i-1]` and `word2[0..j-1]`. The answer is `dp[m][n]`.

**Algorithm:**
1. Let `m = len(word1)`, `n = len(word2)`
2. Create a 2D array `dp` of size `(m+1) x (n+1)`
3. Initialize:
   - `dp[i][0] = i` for all i (i deletions to turn word1 prefix into empty string)
   - `dp[0][j] = j` for all j (j insertions to turn empty string into word2 prefix)
4. For each `i` from 1 to `m` and `j` from 1 to `n`:
   - If `word1[i-1] == word2[j-1]`: `dp[i][j] = dp[i-1][j-1]` (no operation needed)
   - Else: `dp[i][j] = 1 + min(dp[i-1][j]`, `dp[i][j-1]`, `dp[i-1][j-1])`
     - `dp[i-1][j]` = delete word1[i-1]
     - `dp[i][j-1]` = insert word2[j-1] into word1
     - `dp[i-1][j-1]` = replace word1[i-1] with word2[j-1]
5. Return `dp[m][n]`

**Why This Works:** The DP table captures all possible prefix combinations. At each cell, we consider the cost of the three operations relative to already-computed subproblems. The minimum of the three choices gives the optimal edit distance for that prefix.

**Complexity Analysis:**
- **Time:** O(m * n) — fill a table of size (m+1) x (n+1)
- **Space:** O(m * n) — the full 2D table

## Space-Optimized Approach

**Idea:** Since each cell only depends on the current row and the previous row, we can reduce space to O(min(m, n)) using two 1D arrays.

**Algorithm:**
1. Use two rows: `prev` (for i-1) and `curr` (for i)
2. Initialize `prev[j] = j` (first row = insertions)
3. For each row i, compute `curr` from `prev` and the current character comparison
4. Swap `prev` and `curr` after each row

**Complexity Analysis:**
- **Time:** O(m * n) — same as 2D version
- **Space:** O(min(m, n)) — only two 1D arrays

Tracing the solution path (highlighted cells that lead to the final answer):

1. Start at dp[5][3] = 3 (word1="horse", word2="ros")
2. word1[4]='e' ≠ word2[2]='s' → min of three neighbors + 1
   - dp[4][3] = 2 (delete 'e'), dp[5][2] = 4 (insert 's'), dp[4][2] = 2 (replace 'e'→'s')
   - Minimum is 2 (via delete or replace), so dp[5][3] = 2 + 1 = 3
3. Following back through dp[4][3] = 2:
   - word1[3]='s' vs word2[2]='s' → match → dp[3][2] = 2
4. dp[3][2] = 2: word1[2]='r' vs word2[1]='o' → mismatch
   - dp[2][2] = 1 (delete 'r'), dp[3][1] = 2 (insert 'o'), dp[2][1] = 1 (replace 'r'→'o')
   - Takes dp[2][2] = 1 → delete 'r'
5. dp[2][2] = 1: word1[1]='o' vs word2[1]='o' → match → dp[1][1] = 1
6. dp[1][1] = 1: word1[0]='h' vs word2[0]='r' → mismatch
   - dp[0][1] = 1 (delete 'h'), dp[1][0] = 1 (insert 'r'), dp[0][0] = 0 (replace 'h'→'r')
   - Takes dp[0][0] + 1 = 1 → replace 'h' with 'r'

Result: replace 'h'→'r', delete 'r', delete 'e' = 3 operations. Matches the expected output.

### Edge Cases to Consider

- **Empty strings**: If one string is empty, the distance is the length of the other (all inserts or all deletes). The initialization handles this.
- **Identical strings**: All characters match, so the diagonal never increases. Answer is 0.
- **Single character strings**: Minimum of insert/delete/replace operations (0 if same, 1 if different).
- **Completely different strings**: Every character position requires a replacement. Answer is the length of the longer string (or min(m,n) replaces + the difference in lengths as inserts/deletes).

```java
class Solution {
    public int minDistance(String word1, String word2) {
        int m = word1.length();
        int n = word2.length();

        // 2D DP table: dp[i][j] = edit distance for prefixes of length i and j
        int[][] dp = new int[m + 1][n + 1];

        // Base case: converting to/from empty string requires all characters
        for (int i = 0; i <= m; i++) dp[i][0] = i;  // i deletions
        for (int j = 0; j <= n; j++) dp[0][j] = j;  // j insertions

        // Fill the DP table row by row
        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                if (word1.charAt(i - 1) == word2.charAt(j - 1)) {
                    // Characters match — no operation needed, take diagonal value
                    dp[i][j] = dp[i - 1][j - 1];
                } else {
                    // Characters differ — take minimum of three operations
                    dp[i][j] = 1 + Math.min(
                        dp[i - 1][j],      // Delete word1[i-1]
                        Math.min(
                            dp[i][j - 1],      // Insert word2[j-1] into word1
                            dp[i - 1][j - 1]   // Replace word1[i-1] with word2[j-1]
                        )
                    );
                }
            }
        }

        return dp[m][n];  // Edit distance for the full strings
    }
}
```

```rust
fn min_distance(word1: &str, word2: &str) -> i32 {
    let m = word1.len();
    let n = word2.len();

    // Use two rows for space optimization: only need previous and current
    let mut prev: Vec<i32> = (0..=n as i32).collect();  // First row: dp[0][j] = j
    let mut curr: Vec<i32> = vec![0; n + 1];

    // Process each character of word1
    for (i, c1) in word1.chars().enumerate() {
        curr[0] = (i + 1) as i32;  // dp[i+1][0] = i+1 (deletion count)
        for (j, c2) in word2.chars().enumerate() {
            if c1 == c2 {
                // Characters match — no operation, carry forward diagonal value
                curr[j + 1] = prev[j];
            } else {
                // Mismatch — take minimum of delete, insert, replace
                curr[j + 1] = 1 + std::cmp::min(
                    prev[j + 1],  // Delete current word1 character
                    std::cmp::min(
                        curr[j],      // Insert current word2 character
                        prev[j]       // Replace current character
                    )
                );
            }
        }
        // Swap rows for next iteration
        std::mem::swap(&mut prev, &mut curr);
    }

    prev[n]  // Answer is in prev after the last swap
}
```

```c
int minDistance(char* word1, char* word2) {
    int m = strlen(word1);
    int n = strlen(word2);

    // Allocate 2D DP table
    int dp[505][505];  // Max constraint is 500, +1 for safety

    // Base cases: empty string conversions

    for (int j = 0; j <= n; j++) dp[0][j] = j;  // j insertions

                // Characters match — no additional cost
                dp[i][j] = dp[i - 1][j - 1];

                // Find the cheapest operation among the three choices
                int del = dp[i - 1][j];        // Delete from word1
                int ins = dp[i][j - 1];        // Insert into word1
                int rep = dp[i - 1][j - 1];    // Replace in word1
                int min = del < ins ? del : ins;
                min = min < rep ? min : rep;
                dp[i][j] = 1 + min;  // Add 1 for the operation cost
            }
        }
    }

    return dp[m][n];  // Edit distance for full strings
}
```

```csharp
class Solution {
    public int MinDistance(string word1, string word2) {
        int m = word1.Length;
        int n = word2.Length;

        // 2D DP array
        int[,] dp = new int[m + 1, n + 1];

        // Base cases: converting to/from empty string
        for (int i = 0; i <= m; i++) dp[i, 0] = i;  // i deletions
        for (int j = 0; j <= n; j++) dp[0, j] = j;  // j insertions

        // Fill DP table
        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {

                    // Characters match, no cost — take diagonal
                    dp[i, j] = dp[i - 1, j - 1];
                } else {
                    // Minimum of three operations
                    int delete = dp[i - 1, j];      // Delete from word1
                    int insert = dp[i, j - 1];      // Insert into word1
                    int replace = dp[i - 1, j - 1]; // Replace in word1
                    dp[i, j] = 1 + Math.Min(delete, Math.Min(insert, replace));
                }
            }
        }

        return dp[m, n];  // Edit distance for full strings
    }
}
```

## Key Takeaways

- **2D DP for string comparison**: The classic pattern for edit distance, longest common subsequence, and similar problems uses a 2D table comparing prefixes.
- **Three operations, three transitions**: Delete `(i-1, j)`, Insert `(i, j-1)`, Replace `(i-1, j-1)`. The minimum of these three gives the optimal cost.
- **Space optimization**: Only two rows are needed at any time, reducing O(m*n) space to O(min(m, n)).
- **Base case is critical**: The first row and column represent conversions to/from an empty string. Getting this right is essential for the recurrence to work.
- **Match = no cost**: When characters match, the cost carries over diagonally from `dp[i-1][j-1]` without incrementing.
- Applications extend beyond strings: DNA sequence alignment, spell checking, plagiarism detection, and natural language processing all use variants of edit distance.
