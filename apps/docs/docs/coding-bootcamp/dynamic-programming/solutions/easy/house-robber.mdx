# House Robber — Solution

## Problem Overview

Given an array `nums` where `nums[i]` is the money at house `i`, find the maximum sum you can collect without robbing two adjacent houses. This is a classic **1D dynamic programming** problem — a linear recurrence with two states per position.

## Brute Force Approach: Generate All Subsets

**Idea:** Generate every subset of houses that has no adjacent elements, compute the sum for each, and track the maximum.

**Algorithm:**
1. Use bit masking or recursion to enumerate all subsets of indices.
2. For each subset, check that no two selected indices are adjacent.
3. If valid, compute the sum and update the maximum.

**Complexity Analysis:**
- **Time:** O(2^n) — exponential, completely infeasible for n = 100.
- **Space:** O(n) — recursion depth or bit mask storage.

**Why this could be better:** The brute force is only viable for n  20. The DP solution solves it in O(n) time, which is required for the full constraint range.

## Optimal Approach: Dynamic Programming

**Idea:** Define `dp[i]` as the maximum money that can be robbed from the first `i` houses. At each house, two choices exist: skip it (keep `dp[i-1]`) or rob it (take `nums[i-1]` plus `dp[i-2]`). The optimal choice is the maximum of these two.

**Recurrence:**
```
dp[0] = 0                         // no houses
dp[1] = nums[0]                   // only one house
dp[i] = max(dp[i-1], dp[i-2] + nums[i-1])   // skip or rob
```

**Algorithm:**
1. Handle edge cases: if `nums` is empty, return 0. If it has one element, return that element.
2. Initialize two variables: `prev2 = 0` (dp[i-2]) and `prev1 = nums[0]` (dp[i-1]).
3. Iterate from i = 2 to n:
   - `curr = max(prev1, prev2 + nums[i-1])`
   - `prev2 = prev1`, `prev1 = curr`
4. Return `prev1`.

**Why This Works:** The recurrence captures all valid subsets because the decision at house `i` only depends on the best solution up to `i-1` (skip) and the best solution up to `i-2` plus house `i` (rob). This optimal substructure is the hallmark of DP.

**Complexity Analysis:**
- **Time:** O(n) — single pass through the array.
- **Space:** O(1) — only two variables needed.

### Edge Cases

- **Empty array**: No houses — return 0.
- **Single house**: Return `nums[0]`.
- **Two houses**: Return `max(nums[0], nums[1])` since you can only rob one.
- **All zeros**: No money anywhere — return 0.
- **All same values**: Alternating pick yields `ceil(n/2) * value`.
- **Increasing values**: The optimal pattern may involve skipping two in a row.

### Formal Proof of Optimal Substructure

Let `S_i` be an optimal solution for the first `i` houses. Consider the last house `i-1`:
- Case 1: `S_i` does not include house `i-1`. Then `S_i` is also an optimal solution for the first `i-1` houses, so its value is `dp[i-1]`.
- Case 2: `S_i` includes house `i-1`. Then house `i-2` cannot be included, so `S_i` consists of house `i-1` plus an optimal solution for the first `i-2` houses. Its value is `nums[i-1] + dp[i-2]`.

Since these two cases cover all possibilities, `dp[i] = max(dp[i-1], dp[i-2] + nums[i-1])` is correct.

```python
def rob(nums: list[int]) -> int:
    # Edge case: no houses to rob
    if not nums:
        return 0
    # Edge case: only one house
    if len(nums) == 1:
        return nums[0]

    # prev2 = dp[i-2], prev1 = dp[i-1]
    prev2 = 0          # dp[0]: no houses, no money
    prev1 = nums[0]    # dp[1]: best with just the first house

    for i in range(2, len(nums) + 1):
        # At house i-1: max of skipping it or robbing it
        curr = max(prev1, prev2 + nums[i - 1])
        # Slide the window forward
        prev2, prev1 = prev1, curr

    return prev1  # dp[n]: best with all n houses
```

class Solution {
    public int rob(int[] nums) {
        // Edge case: no houses
        if (nums.length == 0) return 0;
        // Edge case: single house
        if (nums.length == 1) return nums[0];

        // prev2 = dp[i-2], prev1 = dp[i-1]
        int prev2 = 0;          // dp[0]
        int prev1 = nums[0];    // dp[1]

        for (int i = 2; i <= nums.length; i++) {
            // Max of skipping house i-1 or robbing it
            int curr = Math.max(prev1, prev2 + nums[i - 1]);
            // Shift the DP window
            prev2 = prev1;
            prev1 = curr;
        }

        return prev1;  // dp[n]
    }
}

func rob(nums []int) int {
    // Edge case: no houses
    if len(nums) == 0 {
        return 0
    }
    // Edge case: single house
    if len(nums) == 1 {
        return nums[0]
    }

    // prev2 = dp[i-2], prev1 = dp[i-1]
    prev2 := 0          // dp[0]
    prev1 := nums[0]    // dp[1]

    for i := 2; i <= len(nums); i++ {
        // Max of skipping house i-1 or robbing it
        curr := prev1
        if prev2 + nums[i-1] > curr {
            curr = prev2 + nums[i-1]
        }
        // Slide the window forward
        prev2 = prev1
        prev1 = curr
    }

    return prev1  // dp[n]
}

## Key Takeaways

- **Optimal substructure**: The best decision at each step depends only on the previous two subproblem solutions — no need to know the full history.
- **Space optimization**: The recurrence only needs `dp[i-1]` and `dp[i-2]`, reducing space from O(n) to O(1).
- **Pattern recognition**: "Cannot take adjacent items" is a canonical DP pattern. Variations include House Robber II (circular) and House Robber III (tree).
- **Think bottom-up**: Start with the smallest subproblems (0 houses, 1 house) and build up iteratively.
