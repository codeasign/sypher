# Longest Increasing Subsequence — Solution

## Problem Overview

Find the length of the longest strictly increasing subsequence in an array of integers. A subsequence preserves order but can skip elements. This is a classic **sequence DP** problem with an O(n log n) optimization using patience sorting.

## Approach 1: Dynamic Programming (O(n^2))

**Idea:** Let `dp[i]` = length of the LIS ending at index `i`. For each `i`, check all previous indices `j < i`. If `nums[j] < nums[i]`, then `dp[i] = max(dp[i], dp[j] + 1)`.

**Algorithm:**
1. Initialize `dp` array of length n with all 1s (each element alone is a subsequence of length 1).
2. For `i` from 1 to n-1:
   - For `j` from 0 to i-1:
     - If `nums[j] < nums[i]`, update `dp[i] = max(dp[i], dp[j] + 1)`.
3. Return `max(dp)`.

**Complexity Analysis:**
- **Time:** O(n^2) — nested loops.
- **Space:** O(n) — dp array.

## Approach 2: Patience Sorting with Binary Search (O(n log n))

**Idea:** Maintain an array `tails` where `tails[k]` is the smallest possible tail value of an increasing subsequence of length `k+1`. For each number, find the first `tails` element that is >= it using binary search, and replace it. If the number is greater than all tails, append it.

This works because `tails` is always sorted. The length of `tails` at the end is the LIS length.

**Algorithm:**
1. Initialize an empty list `tails`.
2. For each `num` in `nums`:
   - Use binary search to find the insertion point: the first index where `tails[i] >= num`.
   - If found, replace `tails[i] = num`.
   - If not found (num > all tails), append `num` to `tails`.
3. Return `len(tails)`.

**Why This Works:** The algorithm processes elements one at a time, maintaining the invariant that `tails[k]` represents the best possible ending value for a subsequence of length `k+1`. By always keeping the smallest possible tail, we maximize opportunities to extend subsequences.

**Complexity Analysis:**
- **Time:** O(n log n) — binary search for each element.
- **Space:** O(n) — tails array.

## Deep Dive: Patience Sorting Walkthrough

### Visual Walkthrough

For `nums = [10, 9, 2, 5, 3, 7, 101, 18]`:

```
Step  num    tails (after)         LIS length
─────────────────────────────────────────────
 1    10     [10]                  1
 2     9     [9]                   1   ← replaced 10 (smaller tail is better)
 3     2     [2]                   1   ← replaced 9
 4     5     [2, 5]                2   ← appended (5 > all tails)
 5     3     [2, 3]                2   ← replaced 5
 6     7     [2, 3, 7]             3   ← appended (7 > all tails)
 7   101     [2, 3, 7, 101]        4   ← appended
 8    18     [2, 3, 7, 18]         4   ← replaced 101

Result: LIS length = 4 (tails array: [2, 3, 7, 18])
```

The actual LIS is [2, 3, 7, 101] (or [2, 3, 5, 7, ...]) — the `tails` array doesn't store the actual subsequence, just the best tails.

### Edge Cases

- **Empty array**: Return 0.
- **Single element**: Return 1.
- **All decreasing**: LIS is 1 (pick any single element).
- **All equal**: LIS is 1 (strictly increasing means no equal values allowed).
- **All increasing**: LIS is n (the entire array).
- **Duplicates**: Strictly increasing means `nums[j] < nums[i]`, not `<=`. Duplicates cannot be part of the same LIS.

def length_of_lis(nums: list[int]) -> int:
    # tails[k] = smallest possible tail for subsequence of length k+1
    tails = []

    for num in nums:
        # Binary search: find first index where tails[i] >= num
        left, right = 0, len(tails)
        while left < right:
            mid = (left + right) // 2
            if tails[mid] < num:
                left = mid + 1  # num is larger, search right half
            else:
                right = mid     # nums[mid] >= num, search left half

        if left == len(tails):
            tails.append(num)       # num extends the longest subsequence
        else:
            tails[left] = num       # replace to keep tails minimal

    return len(tails)

class Solution {
    public int lengthOfLIS(int[] nums) {
        // tails[k] = smallest possible tail for subsequence of length k+1
        int[] tails = new int[nums.length];
        int size = 0;  // current length of tails

        for (int num : nums) {
            // Binary search: find first index where tails[i] >= num
            int left = 0, right = size;
            while (left < right) {
                int mid = left + (right - left) / 2;
                if (tails[mid] < num) {
                    left = mid + 1;  // num is larger, search right
                } else {
                    right = mid;     // tails[mid] >= num, search left
                }
            }

            tails[left] = num;       // place num in the tails array
            if (left == size) size++; // num extends the longest subsequence
        }

        return size;
    }
}

pub fn length_of_lis(nums: Vec<i32>) -> i32 {
    // tails[k] = smallest possible tail for subsequence of length k+1
    let mut tails: Vec<i32> = Vec::new();

    for &num in &nums {
        // Binary search: find first index where tails[i] >= num
        let pos = match tails.binary_search(&num) {
            Ok(p) => p,      // exact match found, replace at p
            Err(p) => p,     // insertion point for num
        };

        if pos == tails.len() {
            tails.push(num);       // num extends the longest subsequence
        } else {
            tails[pos] = num;      // replace with smaller tail
        }
    }

    tails.len() as i32
}

## Key Takeaways

- **Two approaches**: O(n^2) DP for intuition, O(n log n) patience sorting for performance.
- **Patience sorting**: The algorithm is named after the card game patience — each pile keeps the smallest possible top card, and a new card is placed on the leftmost pile where it fits.
- **`tails` is always sorted**: This is the invariant that enables binary search. Each replacement keeps it sorted.
- **Not the actual subsequence**: The `tails` array tracks lengths, not the actual LIS values. Recovering the actual subsequence requires additional tracking.
