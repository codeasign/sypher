# Accounts Merge — Solution

## Problem Overview

Given a list of `accounts` where each element `accounts[i]` is a list of strings: `[name, email1, email2, ...]`. If two accounts share a common email address, they belong to the same person. Merge the accounts so that each person has a single account with their name and all their unique emails sorted in lexicographic order. Accounts that do not share any emails are considered separate people.

**Example:**
```
Input:
accounts = [
  ["John","john@mail.com","john00@mail.com"],
  ["John","johnny@mail.com","john@mail.com"],
  ["John","john_newyork@mail.com"],
  ["Mary","mary@mail.com"]
]

Output:
[
  ["John","john00@mail.com","john@mail.com","johnny@mail.com"],
  ["John","john_newyork@mail.com"],
  ["Mary","mary@mail.com"]
]
```
Accounts 0 and 1 share `john@mail.com` and are merged.

## Brute Force Approach

Build an adjacency graph where each email is a node and edges connect emails that belong to the same account. Then run DFS/BFS from each unvisited email to find all connected components (each component belongs to one person).

**Algorithm:**
1. Build a hash map from email to its name.
2. Build an adjacency list: for each account, connect every pair of emails in that account with an undirected edge.
3. Perform DFS from each unvisited email, collecting all reachable emails into a component.
4. For each component, sort the emails and prepend the name.

**Complexity:** O(N * K * log K) time, where N is the number of accounts and K is the average number of emails per account. O(N * K) space for the graph.

**Why Union-Find is better:** The graph approach requires building explicit adjacency lists and managing visited flags. Union-Find handles the grouping implicitly through parent pointers, reducing memory overhead and simplifying the merge logic.

## Optimal Approach: Union-Find

Map each email to a unique integer ID. Use Union-Find to union emails that belong to the same account. After processing all accounts, group emails by their root.

**Algorithm:**
1. Create a hash map `emailToId` mapping each email to a unique integer ID.
2. Create a hash map `emailToName` mapping each email to the account name.
3. Initialize Union-Find with a capacity equal to the total number of unique emails.
4. For each account:
   - For each email after the name, union the first email's ID with the current email's ID.
5. Build a map from root ID to list of emails belonging to that root.
6. For each group, sort the emails and prepend the name.

**Complexity:** O(N * K * alpha(N * K)) time for unions, plus O(M log M) for sorting each group (where M is the total number of unique emails). O(N * K) space.

## Deep Dive

Walk through the example:

```
accounts = [
  ["John","a@mail.com","b@mail.com"],     // Account 0
  ["John","c@mail.com","a@mail.com"],     // Account 1
  ["John","d@mail.com"],                  // Account 2
  ["Mary","e@mail.com"]                   // Account 3
]

Step 1: Assign IDs to unique emails:
  a@mail.com → 0, b@mail.com → 1, c@mail.com → 2, d@mail.com → 3, e@mail.com → 4

Step 2: Union emails within each account:
  Account 0: union(0, 1)  → parent = [0, 0, 2, 3, 4]
  Account 1: union(2, 0)  → find(2)=2, find(0)=0, union → parent = [2, 0, 2, 3, 4]
             (Now a@mail.com, b@mail.com, c@mail.com are all connected)
  Account 2: d@mail.com alone → no union needed
  Account 3: e@mail.com alone → no union needed

Step 3: Group by root:
  Root 2: [a@mail.com, b@mail.com, c@mail.com]  → ["John", "a@mail.com", "b@mail.com", "c@mail.com"]
  Root 3: [d@mail.com]                          → ["John", "d@mail.com"]
  Root 4: [e@mail.com]                          → ["Mary", "e@mail.com"]
```

class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n

    def find(self, x: int) -> int:
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]

    def union(self, x: int, y: int) -> None:
        px, py = self.find(x), self.find(y)
        if px == py:
            return
        if self.rank[px] < self.rank[py]:
            self.parent[px] = py
        elif self.rank[px] > self.rank[py]:
            self.parent[py] = px
        else:
            self.parent[py] = px
            self.rank[px] += 1


class Solution:
    def accountsMerge(self, accounts: list[list[str]]) -> list[list[str]]:
        email_to_id = {}
        email_to_name = {}

        # Assign unique IDs to each email
        for account in accounts:
            name = account[0]
            for email in account[1:]:
                if email not in email_to_id:
                    email_to_id[email] = len(email_to_id)
                    email_to_name[email] = name

        uf = UnionFind(len(email_to_id))

        # Union emails within each account
        for account in accounts:
            first_email = account[1]
            first_id = email_to_id[first_email]
            for email in account[2:]:
                uf.union(first_id, email_to_id[email])

        # Group emails by root
        root_to_emails: dict[int, list[str]] = {}
        for email, eid in email_to_id.items():
            root = uf.find(eid)
            root_to_emails.setdefault(root, []).append(email)

        # Build result: sorted emails with name
        result = []
        for emails in root_to_emails.values():
            name = email_to_name[emails[0]]
            result.append([name] + sorted(emails))

        return result

class Solution {
    class UnionFind {
        int[] parent, rank;
        UnionFind(int n) {
            parent = new int[n];
            rank = new int[n];
            for (int i = 0; i < n; i++) parent[i] = i;
        }
        int find(int x) {
            if (parent[x] != x) parent[x] = find(parent[x]);
            return parent[x];
        }
        void union(int x, int y) {
            int px = find(x), py = find(y);
            if (px == py) return;
            if (rank[px] < rank[py]) parent[px] = py;
            else if (rank[px] > rank[py]) parent[py] = px;
            else { parent[py] = px; rank[px]++; }
        }
    }

    public List<List<String>> accountsMerge(List<List<String>> accounts) {
        Map<String, Integer> emailToId = new HashMap<>();
        Map<String, String> emailToName = new HashMap<>();

        // Assign unique IDs to each email
        int id = 0;
        for (List<String> account : accounts) {
            String name = account.get(0);
            for (int i = 1; i < account.size(); i++) {
                String email = account.get(i);
                if (!emailToId.containsKey(email)) {
                    emailToId.put(email, id++);
                    emailToName.put(email, name);
                }
            }
        }

        UnionFind uf = new UnionFind(id);

        // Union emails within each account
        for (List<String> account : accounts) {
            int firstId = emailToId.get(account.get(1));
            for (int i = 2; i < account.size(); i++) {
                uf.union(firstId, emailToId.get(account.get(i)));
            }
        }

        // Group emails by root
        Map<Integer, List<String>> rootToEmails = new HashMap<>();
        for (Map.Entry<String, Integer> entry : emailToId.entrySet()) {
            int root = uf.find(entry.getValue());
            rootToEmails.computeIfAbsent(root, k -> new ArrayList<>()).add(entry.getKey());
        }

        // Build result: sorted emails with name
        List<List<String>> result = new ArrayList<>();
        for (List<String> emails : rootToEmails.values()) {
            Collections.sort(emails);
            String name = emailToName.get(emails.get(0));
            List<String> merged = new ArrayList<>();
            merged.add(name);
            merged.addAll(emails);
            result.add(merged);
        }
        return result;
    }
}

class UnionFind {
public:
    vector<int> parent, rank;
    UnionFind(int n) : parent(n), rank(n, 0) {
        for (int i = 0; i < n; i++) parent[i] = i;
    }
    int find(int x) {
        if (parent[x] != x) parent[x] = find(parent[x]);
        return parent[x];
    }
    void unite(int x, int y) {
        int px = find(x), py = find(y);
        if (px == py) return;
        if (rank[px] < rank[py]) parent[px] = py;
        else if (rank[px] > rank[py]) parent[py] = px;
        else { parent[py] = px; rank[px]++; }
    }
};

class Solution {
public:
    vector<vector<string>> accountsMerge(vector<vector<string>>& accounts) {
        unordered_map<string, int> emailToId;
        unordered_map<string, string> emailToName;

        // Assign unique IDs to each email
        int id = 0;
        for (auto& account : accounts) {
            string& name = account[0];
            for (int i = 1; i < account.size(); i++) {
                string& email = account[i];
                if (!emailToId.count(email)) {
                    emailToId[email] = id++;
                    emailToName[email] = name;
                }
            }
        }

        UnionFind uf(id);

        // Union emails within each account
        for (auto& account : accounts) {
            int firstId = emailToId[account[1]];
            for (int i = 2; i < account.size(); i++) {
                uf.unite(firstId, emailToId[account[i]]);
            }
        }

        // Group emails by root
        unordered_map<int, vector<string>> rootToEmails;
        for (auto& [email, eid] : emailToId) {
            int root = uf.find(eid);
            rootToEmails[root].push_back(email);
        }

        // Build result: sorted emails with name
        vector<vector<string>> result;
        for (auto& [root, emails] : rootToEmails) {
            sort(emails.begin(), emails.end());
            string name = emailToName[emails[0]];
            vector<string> merged = {name};
            merged.insert(merged.end(), emails.begin(), emails.end());
            result.push_back(merged);
        }
        return result;
    }
};

        emails.sort();
        const name = emailToName.get(emails[0]);
        result.push([name, ...emails]);
    }
    return result;

use std::collections::HashMap;

struct UnionFind {
    parent: Vec<usize>,
    rank: Vec<i32>,
}

impl UnionFind {
    fn new(n: usize) -> Self {
        let parent: Vec<usize> = (0..n).collect();
        let rank = vec![0; n];
        UnionFind { parent, rank }
    }

    fn find(&mut self, x: usize) -> usize {
        if self.parent[x] != x {
            self.parent[x] = self.find(self.parent[x]);
        }
        self.parent[x]
    }

    fn union(&mut self, x: usize, y: usize) {
        let px = self.find(x);
        let py = self.find(y);
        if px == py { return; }
        if self.rank[px] < self.rank[py] { self.parent[px] = py; }
        else if self.rank[px] > self.rank[py] { self.parent[py] = px; }
        else { self.parent[py] = px; self.rank[px] += 1; }
    }
}

impl Solution {
    pub fn accounts_merge(accounts: Vec<Vec<String>>) -> Vec<Vec<String>> {
        let mut email_to_id: HashMap<String, usize> = HashMap::new();
        let mut email_to_name: HashMap<String, String> = HashMap::new();

        // Assign unique IDs to each email
        let mut id = 0;
        for account in &accounts {
            let name = &account[0];
            for email in account.iter().skip(1) {
                if !email_to_id.contains_key(email) {
                    email_to_id.insert(email.clone(), id);
                    email_to_name.insert(email.clone(), name.clone());
                    id += 1;
                }
            }
        }

        let mut uf = UnionFind::new(id);

        // Union emails within each account
        for account in &accounts {
            let first_id = email_to_id[&account[1]];
            for email in account.iter().skip(2) {
                uf.union(first_id, email_to_id[email]);
            }
        }

        // Group emails by root
        let mut root_to_emails: HashMap<usize, Vec<String>> = HashMap::new();
        for (email, &eid) in &email_to_id {
            let root = uf.find(eid);
            root_to_emails.entry(root).or_default().push(email.clone());
        }

        // Build result: sorted emails with name
        let mut result: Vec<Vec<String>> = Vec::new();
        for emails in root_to_emails.values_mut() {
            emails.sort();
            let name = email_to_name[&emails[0]].clone();
            let mut merged = vec![name];
            merged.append(emails);
            result.push(merged);
        }
        result
    }
}

typedef struct {
    int* parent;
    int* rank;
    int capacity;
} UnionFind;

UnionFind* uf_create(int n) {
    UnionFind* uf = malloc(sizeof(UnionFind));
    uf->parent = malloc(n * sizeof(int));
    uf->rank = calloc(n, sizeof(int));
    uf->capacity = n;
    for (int i = 0; i < n; i++) uf->parent[i] = i;
    return uf;
}

int uf_find(UnionFind* uf, int x) {
    if (uf->parent[x] != x) uf->parent[x] = uf_find(uf, uf->parent[x]);
    return uf->parent[x];
}

public class Solution {
    class UnionFind {
        private int[] parent;
        private int[] rank;

        public UnionFind(int n) {
            parent = new int[n];
            rank = new int[n];
            for (int i = 0; i < n; i++) parent[i] = i;
        }

        public int Find(int x) {
            if (parent[x] != x) parent[x] = Find(parent[x]);
            return parent[x];
        }

        public void Union(int x, int y) {
            int px = Find(x), py = Find(y);
            if (px == py) return;
            if (rank[px] < rank[py]) parent[px] = py;
            else if (rank[px] > rank[py]) parent[py] = px;
            else { parent[py] = px; rank[px]++; }
        }
    }

    public IList<IList<string>> AccountsMerge(IList<IList<string>> accounts) {
        var emailToId = new Dictionary<string, int>();
        var emailToName = new Dictionary<string, string>();

        // Assign unique IDs to each email
        int id = 0;
        foreach (var account in accounts) {
            string name = account[0];
            for (int i = 1; i < account.Count; i++) {
                string email = account[i];
                if (!emailToId.ContainsKey(email)) {
                    emailToId[email] = id++;
                    emailToName[email] = name;
                }
            }
        }

        var uf = new UnionFind(id);

        // Union emails within each account
        foreach (var account in accounts) {
            int firstId = emailToId[account[1]];
            for (int i = 2; i < account.Count; i++) {
                uf.Union(firstId, emailToId[account[i]]);
            }
        }

        // Group emails by root
        var rootToEmails = new Dictionary<int, List<string>>();
        foreach (var kvp in emailToId) {
            int root = uf.Find(kvp.Value);
            if (!rootToEmails.ContainsKey(root))
                rootToEmails[root] = new List<string>();
            rootToEmails[root].Add(kvp.Key);
        }

        // Build result: sorted emails with name
        var result = new List<IList<string>>();
        foreach (var emails in rootToEmails.Values) {
            emails.Sort();
            string name = emailToName[emails[0]];
            var merged = new List<string> { name };
            merged.AddRange(emails);
            result.Add(merged);
        }
        return result;
    }
}

## Key Takeaways

- Union-Find with a hash map is a powerful combination for grouping problems where elements share a common attribute.
- Each email gets a unique ID; the Union-Find structure tracks which group each email belongs to.
- The key insight is that all emails within a single account must share the same root — we union the first email with every other email in that account.
- After all unions, grouping by root gives us the merged accounts. Sorting each group ensures the output is in lexicographic order.
- This approach gracefully handles duplicate emails across accounts without creating duplicate entries.
