# Longest Consecutive Sequence — Solution

## Problem Overview

Given an unsorted array of integers `nums`, return the length of the longest consecutive elements sequence. A **consecutive sequence** is a sequence of elements where each element is exactly one greater than the previous element. The algorithm must run in **O(n)** time.

**Example:**
```
Input: nums = [100, 4, 200, 1, 3, 2]
Output: 4
Explanation: The longest consecutive sequence is [1, 2, 3, 4] with length 4.
```

## Brute Force Approach

For each element, check if `num + 1`, `num + 2`, etc. exist in the array using linear search or a sorted array.

**Algorithm:**
1. Sort the array (O(n log n)).
2. Scan the sorted array and count consecutive runs.
3. Track the maximum run length.

**Complexity:** O(n log n) time due to sorting, O(1) or O(n) space depending on sorting implementation.

**Why Union-Find is interesting:** While the hash set approach is the optimal O(n) solution, Union-Find provides an alternative O(n) approach that demonstrates how disjoint sets can solve sequence-grouping problems. The hash set approach is simpler, but Union-Find is valuable when the problem evolves to include dynamic additions or queries.

## Optimal Approach: Union-Find

Use Union-Find to group consecutive numbers together. Each number is unioned with `num + 1` (if `num + 1` exists in the set). The size of the largest connected component is the answer.

**Algorithm:**
1. Convert `nums` to a set for O(1) lookups.
2. Create a map from each unique number to its index in the Union-Find structure.
3. For each unique number `num`:
   - If `num + 1` exists in the set, union the indices of `num` and `num + 1`.
4. Find the size of each component by counting how many nodes share each root.
5. Return the maximum component size.

**Complexity:** O(n * alpha(n)) time, O(n) space.

## Hash Set Approach (Alternative O(n) Solution)

For completeness, the simpler O(n) approach uses a hash set:
1. Add all numbers to a set.
2. For each number, check if it is the *start* of a sequence (i.e., `num - 1` is not in the set).
3. If it is a start, count consecutive numbers (`num + 1`, `num + 2`, ...) in the set.
4. Return the maximum count.

## Deep Dive

Walk through the example `nums = [100, 4, 200, 1, 3, 2]`:

```
Step 1: Add all unique numbers to a set and assign indices:
  num_set = {100, 4, 200, 1, 3, 2}
  index_map: 100→0, 4→1, 200→2, 1→3, 3→4, 2→5

Step 2: Union each num with num+1 (if num+1 exists):
  num=100: 101 not in set → skip
  num=4:   5 not in set → skip
  num=200: 201 not in set → skip
  num=1:   2 in set → union(1, 2) → parent = [0,1,2,1,4,1]
  num=3:   4 in set → union(3, 4) → parent = [0,1,2,3,3,1]
  num=2:   3 in set → union(2, 3) → parent = [0,1,3,3,3,1]
            find(2)=1, find(3)=3 → union → parent = [0,1,3,1,3,1]

Step 3: Count component sizes:
  Root 0: node 100 → size 1
  Root 1: nodes 1, 2, 3, 4 → size 4
  Root 2: node 200 → size 1

Step 4: Maximum size = 4 ✅
```

class Solution:
    def longestConsecutive(self, nums: list[int]) -> int:
        if not nums:
            return 0

        num_set = set(nums)
        # Map each number to a unique index
        num_to_idx = {num: i for i, num in enumerate(num_set)}
        n = len(num_set)
        parent = list(range(n))
        rank = [0] * n

        def find(x: int) -> int:
            if parent[x] != x:
                parent[x] = find(parent[x])
            return parent[x]

        def union(x: int, y: int) -> None:
            px, py = find(x), find(y)
            if px == py:
                return
            if rank[px] < rank[py]:
                parent[px] = py
            elif rank[px] > rank[py]:
                parent[py] = px
            else:
                parent[py] = px
                rank[px] += 1

        # Union each number with its successor if it exists
        for num in num_set:
            if num + 1 in num_set:
                union(num_to_idx[num], num_to_idx[num + 1])

        # Count component sizes
        size = {}
        for i in range(n):
            root = find(i)
            size[root] = size.get(root, 0) + 1

        return max(size.values())

class Solution {
    public int longestConsecutive(int[] nums) {
        if (nums.length == 0) return 0;

        Set<Integer> numSet = new HashSet<>();
        for (int num : nums) numSet.add(num);

        Map<Integer, Integer> numToIdx = new HashMap<>();
        int idx = 0;
        for (int num : numSet) numToIdx.put(num, idx++);

        int n = numSet.size();
        int[] parent = new int[n];
        int[] rank = new int[n];
        for (int i = 0; i < n; i++) parent[i] = i;

        // Union each number with its successor if it exists
        for (int num : numSet) {
            if (numSet.contains(num + 1)) {
                union(parent, rank, numToIdx.get(num), numToIdx.get(num + 1));
            }
        }

        // Count component sizes
        int[] size = new int[n];
        int maxSize = 0;
        for (int i = 0; i < n; i++) {
            int root = find(parent, i);
            size[root]++;
            maxSize = Math.max(maxSize, size[root]);
        }
        return maxSize;
    }

    private int find(int[] parent, int x) {
        if (parent[x] != x) parent[x] = find(parent, parent[x]);
        return parent[x];
    }

    private void union(int[] parent, int[] rank, int x, int y) {
        int px = find(parent, x), py = find(parent, y);
        if (px == py) return;
        if (rank[px] < rank[py]) parent[px] = py;
        else if (rank[px] > rank[py]) parent[py] = px;
        else { parent[py] = px; rank[px]++; }
    }
}

use std::collections::{HashMap, HashSet};

impl Solution {
    pub fn longest_consecutive(nums: Vec<i32>) -> i32 {
        if nums.is_empty() { return 0; }

        let num_set: HashSet<i32> = nums.into_iter().collect();
        let num_to_idx: HashMap<i32, usize> = num_set.iter().cloned().enumerate()
            .map(|(i, num)| (num, i)).collect();

        let n = num_set.len();
        let mut parent: Vec<usize> = (0..n).collect();
        let mut rank = vec![0; n];

        fn find(parent: &mut Vec<usize>, x: usize) -> usize {
            if parent[x] != x {
                parent[x] = find(parent, parent[x]);
            }
            parent[x]
        }

        fn union(parent: &mut Vec<usize>, rank: &mut Vec<i32>, x: usize, y: usize) {
            let px = find(parent, x);
            let py = find(parent, y);
            if px == py { return; }
            if rank[px] < rank[py] { parent[px] = py; }
            else if rank[px] > rank[py] { parent[py] = px; }
            else { parent[py] = px; rank[px] += 1; }
        }

        // Union each number with its successor if it exists
        for &num in &num_set {
            if num_set.contains(&(num + 1)) {
                union(&mut parent, &mut rank, num_to_idx[&num], num_to_idx[&(num + 1)]);
            }
        }

        // Count component sizes
        let mut size = vec![0; n];
        let mut max_size = 0;
        for i in 0..n {
            let root = find(&mut parent, i);
            size[root] += 1;
            max_size = max_size.max(size[root]);
        }
        max_size
    }
}

// Note: C lacks a built-in hash set, so we pre-process sorted unique values.
// This is a simplified approach that sorts first, then uses Union-Find on
// consecutive elements. The O(n log n) sort is the bottleneck here.

int cmp(const void* a, const void* b) {
    return *(int*)a - *(int*)b;
}

int find(int* parent, int x) {
    if (parent[x] != x) parent[x] = find(parent, parent[x]);
    return parent[x];
}

void unionSet(int* parent, int* rank, int x, int y) {
    int px = find(parent, x), py = find(parent, y);
    if (px == py) return;
    if (rank[px] < rank[py]) parent[px] = py;
    else if (rank[px] > rank[py]) parent[py] = px;
    else { parent[py] = px; rank[px]++; }
}

int longestConsecutive(int* nums, int numsSize) {
    if (numsSize == 0) return 0;

    // Sort to group consecutive numbers
    qsort(nums, numsSize, sizeof(int), cmp);

    // Remove duplicates and build unique array
    int* uniq = malloc(numsSize * sizeof(int));
    int n = 0;
    for (int i = 0; i < numsSize; i++) {
        if (i == 0 || nums[i] != nums[i - 1]) uniq[n++] = nums[i];
    }

    int parent[n], rank[n];
    for (int i = 0; i < n; i++) { parent[i] = i; rank[i] = 0; }

    // Union consecutive numbers (difference of 1)
    for (int i = 1; i < n; i++) {
        if ((long long)uniq[i] - uniq[i - 1] == 1) {
            unionSet(parent, rank, i, i - 1);
        }
    }

    // Count component sizes
    int* size = calloc(n, sizeof(int));
    int maxSize = 0;
    for (int i = 0; i < n; i++) {
        int root = find(parent, i);
        size[root]++;
        if (size[root] > maxSize) maxSize = size[root];
    }

    free(uniq);
    free(size);
    return maxSize;
}

## Key Takeaways

- Union-Find can solve sequence grouping by unioning each element with its successor. The size of the largest component gives the longest consecutive sequence.
- The hash set approach (checking only sequence starts) is simpler and achieves the same O(n) time, but Union-Find offers advantages for dynamic scenarios where elements are added incrementally.
- Always deduplicate numbers before building the Union-Find structure — duplicates don't affect the longest consecutive sequence length.
- The challenge of this problem is achieving O(n) time. Both the hash set approach and the Union-Find approach meet this requirement.
