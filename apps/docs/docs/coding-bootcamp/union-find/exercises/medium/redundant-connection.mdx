# Redundant Connection

**Difficulty:** Medium

In a graph of `n` nodes labeled from `1` to `n`, find an edge whose removal would make the graph a tree (acyclic and connected). If multiple solutions exist, return the edge that occurs **last** in the input.

A **tree** is an undirected graph that is connected and has no cycles.

You are given a 2D array `edges` where `edges[i] = [u_i, v_i]` (1-indexed). Find the redundant connection.

## Example 1

**Input:**
```
3
1 2
1 3
2 3
```

**Output:**
```
2 3
```

**Explanation:** The graph has 3 nodes and 3 edges. Removing `[2, 3]` makes it a tree. Removing `[1, 3]` would also work, but `[2, 3]` occurs later in the input.

## Example 2

**Input:**
```
4
1 2
2 3
3 4
1 4
```

**Output:**
```
1 4
```

**Explanation:** The edge `[1, 4]` creates a cycle `1-2-3-4-1`. Removing it makes the graph a tree.

## Input Specification

The first line contains an integer `n` (the number of nodes and also the number of edges).

The next `n` lines each contain two integers `u` and `v` representing an undirected edge between nodes `u` and `v`.

## Output Specification

Print the two integers `u` and `v` of the redundant edge, separated by a space. If no redundant edge exists, print a blank line.

## Constraints

- `2 <= n <= 1000`
- `1 <= u_i, v_i <= n`
- The input graph has exactly `n` edges

## Hints

<details>
<summary>Hint 1</summary>

Use union-find (DSU) to detect cycles. As you process each edge, check if its two endpoints are already in the same connected component. If they are, this edge creates a cycle.

</details>

<details>
<summary>Hint 2</summary>

Process edges in order. Keep track of the last edge that creates a cycle. Return that edge at the end.

</details>

<details>
<summary>Hint 3</summary>

Initialize a parent array of size `n + 1` (1-indexed). Each node starts as its own parent. Implement `find` with path compression and `union` with rank/size optimization.

</details>
