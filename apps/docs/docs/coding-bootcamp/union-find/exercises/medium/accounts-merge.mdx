# Accounts Merge

**Difficulty:** Medium

Given a list of `accounts` where each element `accounts[i]` is a list of strings, the first element `accounts[i][0]` is a name, and the remaining elements are emails representing the account.

Merge accounts that share at least one common email. Two accounts that share a common email belong to the same person. Note that even if two accounts have the same name, they may belong to different people — names are not unique. However, it is guaranteed that if two accounts have a common email, they belong to the same person.

Return the merged accounts. Each merged account must have the name first, followed by its emails in **sorted order**.

## Example 1

**Input:**
```
1
John 3 johnsmith@mail.com john_newyork@mail.com john00@mail.com
2
John 2 johnsmith@mail.com johnnybravo@mail.com
3
Mary 1 mary@mail.com
```

**Output:**
```
John john00@mail.com,john_newyork@mail.com,johnnybravo@mail.com,johnsmith@mail.com
Mary mary@mail.com
```

**Explanation:** The first and second accounts share `johnsmith@mail.com`, so they merge. Mary's account has no common email, so it remains separate.

## Example 2

**Input:**
```
2
John 2 johnsmith@mail.com john00@mail.com
John 2 johnnybravo@mail.com johnsmith@mail.com
```

**Output:**
```
John john00@mail.com,johnnybravo@mail.com,johnsmith@mail.com
```

**Explanation:** Both accounts share `johnsmith@mail.com` and merge.

## Input Specification

The first line contains an integer `n`, the number of accounts.

For each account, the line contains:
- A name (string)
- An integer `m` (number of emails)
- Followed by `m` distinct email addresses (strings)

Each line has the format: `name m email1 email2 ... emailM`

## Output Specification

Print each merged account on its own line. Each line starts with the name followed by a space, then the sorted unique email addresses separated by commas.

Print accounts in any order (the judge will check sets of emails per name).

## Constraints

- `1 <= accounts.length <= 1000`
- `1 <= emails[i].length <= 40`
- `1 <= name.length <= 20`
- The total number of emails across all accounts is at most `10^4`

## Hints

<details>
<summary>Hint 1</summary>

Model this as a graph problem. Each email is a node. Two emails from the same account are connected by an edge. Use union-find (DSU) to group emails into connected components.

</details>

<details>
<summary>Hint 2</summary>

Map each email to a unique integer ID. When processing an account, union all its email IDs together. Also track which name is associated with each email.

</details>

<details>
<summary>Hint 3</summary>

After union-find, collect all emails belonging to each root component. Sort the emails within each component and attach the name. Use a `TreeSet` or sorted list to maintain order.

</details>
