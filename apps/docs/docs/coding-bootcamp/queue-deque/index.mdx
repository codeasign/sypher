# Queue & Deque

The Queue (FIFO) and Deque (double-ended) patterns are essential for problems involving order preservation, sliding windows, BFS traversal, and task scheduling.

⏱ 20 minutes

## What You Will Learn

- Basic queue operations and FIFO behavior
- Deque (double-ended queue) for sliding window problems
- Applications: BFS, task scheduling, level-order traversal

## The Concept

A queue stores elements in First-In-First-Out (FIFO) order. Enqueue adds to the rear, dequeue removes from the front. A deque (double-ended queue) allows insertion and removal at both ends, making it ideal for sliding window maximum and minimum problems.

## Common Applications

| Problem | Approach |
|---------|----------|
| Implement Queue using Stacks | Two stacks reverse order |
| Moving Average from Data Stream | Queue maintains window |
| Sliding Window Maximum | Monotonic deque |
| Design Circular Queue | Array with head/tail pointers |
| Constrained Subsequence Sum | Deque DP optimization |
| BFS Traversal | Queue for level-order |
| Task Scheduling | Queue for FIFO execution |

## Complexity

| Operation | Time | Space |
|-----------|------|-------|
| Enqueue / Dequeue (Queue) | O(1) | O(1) |
| push_front / push_back (Deque) | O(1) | O(1) |
| pop_front / pop_back (Deque) | O(1) | O(1) |
| Sliding window (monotonic deque) | O(n) | O(k) |
