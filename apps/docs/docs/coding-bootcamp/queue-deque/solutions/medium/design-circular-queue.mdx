# Design Circular Queue — Solution

## Problem Overview

Design a circular queue using a fixed-size array. The last position connects back to the first position to form a circle. Operations: `enQueue`, `deQueue`, `Front`, `Rear`, `isEmpty`, `isFull`.

## Approach: Array with Modulo Arithmetic

**Idea:** Use a fixed-size array with head and tail pointers that wrap around using modulo arithmetic. Track the count of elements to distinguish between empty and full states.

**Algorithm:**
1. Store elements in an array of size `capacity`.
2. `head` points to the front element. `count` tracks the number of elements.
3. `enQueue(value)`: If full, return false. Otherwise, place `value` at position `(head + count) % capacity` and increment count.
4. `deQueue()`: If empty, return false. Otherwise, move `head` forward by 1 (mod capacity) and decrement count.
5. `Front()`: If empty, return -1. Otherwise, return `data[head]`.
6. `Rear()`: If empty, return -1. Otherwise, return `data[(head + count - 1) % capacity]`.
7. `isEmpty()`: Return `count == 0`.
8. `isFull()`: Return `count == capacity`.

**Why This Works:** The modulo arithmetic creates the circular behavior. The count-based approach cleanly separates empty and full states without needing to waste a slot.

**Complexity Analysis:**
- **Time:** O(1) for all operations.
- **Space:** O(k) — array of size k.

### Visual Walkthrough

For `MyCircularQueue(3)` with operations `enQueue(1), enQueue(2), enQueue(3), enQueue(4), deQueue(), enQueue(4)`:

```
Initial: head=0, count=0, data=[_, _, _]

enQueue(1):  idx=(0+0)%3=0, data[0]=1, count=1
             head=0, data=[1, _, _]

enQueue(2):  idx=(0+1)%3=1, data[1]=2, count=2
             head=0, data=[1, 2, _]

enQueue(3):  idx=(0+2)%3=2, data[2]=3, count=3
             head=0, data=[1, 2, 3]

enQueue(4):  count=3 == capacity=3 → return false

deQueue():   head=1, count=2
             head=1, data=[1, 2, 3]

enQueue(4):  idx=(1+2)%3=0, data[0]=4, count=3
             head=1, data=[4, 2, 3]

Rear():      idx=(1+3-1)%3=0, data[0]=4 → return 4
```

### Edge Cases

- **Empty queue**: `Front()` and `Rear()` return -1.
- **Full queue**: `enQueue()` returns false.
- **Single element**: Both `Front()` and `Rear()` return the same element.
- **Wrap-around**: After dequeue and enqueue, new elements fill positions that were previously dequeued.

class MyCircularQueue {
    private int[] data;     // fixed-size array for storage
    private int head;       // index of the front element
    private int count;      // number of elements currently in the queue
    private int capacity;   // maximum capacity of the queue

    public MyCircularQueue(int k) {
        data = new int[k];
        capacity = k;
        head = 0;
        count = 0;
    }

    // Insert an element into the circular queue. Return true if successful.
    public boolean enQueue(int value) {
        if (count == capacity) return false;
        // Calculate the position for the new element using modulo arithmetic
        data[(head + count) % capacity] = value;
        count++;
        return true;
    }

    // Delete an element from the circular queue. Return true if successful.
    public boolean deQueue() {
        if (count == 0) return false;
        // Move the head forward by one position, wrapping around if needed
        head = (head + 1) % capacity;
        count--;
        return true;
    }

    // Get the front element from the queue. Return -1 if empty.
    public int Front() {
        return count == 0 ? -1 : data[head];
    }

    // Get the last element from the queue. Return -1 if empty.
    public int Rear() {
        if (count == 0) return -1;
        // Calculate the rear index using modulo arithmetic
        return data[(head + count - 1) % capacity];
    }

    // Check whether the circular queue is empty.
    public boolean isEmpty() {
        return count == 0;
    }

    // Check whether the circular queue is full.
    public boolean isFull() {
        return count == capacity;
    }
}

struct MyCircularQueue {
    data: Vec<i32>,   // fixed-size array for storage
    capacity: i32,    // maximum capacity of the queue
    head: i32,        // index of the front element
    count: i32,       // number of elements currently in the queue
}

impl MyCircularQueue {
    fn new(k: i32) -> Self {
        MyCircularQueue {
            data: vec![0; k as usize],
            capacity: k,
            head: 0,
            count: 0,
        }
    }

    // Insert an element into the circular queue. Return true if successful.
    fn en_queue(&mut self, value: i32) -> bool {
        if self.count == self.capacity { return false; }
        let idx = ((self.head + self.count) % self.capacity) as usize;
        self.data[idx] = value;
        self.count += 1;
        true
    }

    // Delete an element from the circular queue. Return true if successful.
    fn de_queue(&mut self) -> bool {
        if self.count == 0 { return false; }
        self.head = (self.head + 1) % self.capacity;
        self.count -= 1;
        true
    }

    // Get the front element from the queue. Return -1 if empty.
    fn front(&self) -> i32 {
        if self.count == 0 { -1 } else { self.data[self.head as usize] }
    }

    // Get the last element from the queue. Return -1 if empty.
    fn rear(&self) -> i32 {
        if self.count == 0 { return -1; }
        let idx = ((self.head + self.count - 1) % self.capacity) as usize;
        self.data[idx]
    }

    // Check whether the circular queue is empty.
    fn is_empty(&self) -> bool {
        self.count == 0
    }

    // Check whether the circular queue is full.
    fn is_full(&self) -> bool {
        self.count == self.capacity
    }
}

## Key Takeaways

- **Modulo arithmetic** creates the circular wrapping behavior with O(1) operations.
- **Count-based tracking** cleanly distinguishes empty vs. full without wasting a slot.
- **Head pointer** moves forward on dequeue — no shifting needed.
- Array indices are calculated on the fly: front = `head`, rear = `(head + count - 1) % capacity`.
