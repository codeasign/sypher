# Implement Queue using Stacks — Solution

## Problem Overview

Implement a FIFO queue using only two stacks. The queue must support `push` (add to back), `pop` (remove from front), `peek` (get front element), and `empty` (check if empty).

## Approach: Two Stacks

**Idea:** Use two stacks: `input` for push operations and `output` for pop/peek operations. When the output stack is empty, transfer all elements from input to output, reversing their order. This reversal converts LIFO to FIFO.

**Algorithm:**
1. `push(x)`: Push x onto the input stack (O(1))
2. `pop()`: If output is empty, transfer all elements from input to output. Pop from output.
3. `peek()`: Same transfer as pop, but just return the top of output without removing it.
4. `empty()`: Return true if both stacks are empty.

**Why This Works:** The transfer from input to output reverses the order of elements. The first element pushed becomes the bottom of the input stack, but when transferred, it becomes the top of the output stack, ready to be popped first (FIFO).

**Complexity Analysis:**
- **Time:** O(1) amortized per operation. Each element is pushed to input once, transferred to output once, and popped from output once — three operations total per element over its lifetime.
- **Space:** O(n) — we store all elements across two stacks.

## Deep Dive

### Visual Walkthrough

For the sequence: `push(1), push(2), push(3), pop(), peek(), pop()`:

```
push(1):   input=[1],          output=[]
push(2):   input=[1,2],        output=[]
push(3):   input=[1,2,3],      output=[]

pop():     input=[1,2,3],      output=[]     (output empty, transfer)
           input=[],           output=[3,2,1]
           return pop output → 1
           input=[],           output=[3,2]

peek():    input=[],           output=[3,2]  (output not empty)
           return peak output → 2

pop():     input=[],           output=[3,2]  (output not empty)
           return pop output → 2
           input=[],           output=[3]
```

### Edge Cases

- **Empty queue**: Both stacks are empty — `empty()` returns true.
- **Pop on empty queue**: Not expected per constraints, but would throw an error if attempted.
- **Peek after pop**: Works seamlessly since the output stack still has elements.
- **Mixed operations**: Elements are only transferred when output is empty, ensuring each element is transferred at most once.

import java.util.Stack;

class MyQueue {
    // input stack for push operations
    private Stack<Integer> input;
    // output stack for pop/peek operations
    private Stack<Integer> output;

    public MyQueue() {
        input = new Stack<>();
        output = new Stack<>();
    }

    // Push element x to the back of the queue
    public void push(int x) {
        input.push(x);
    }

    // Remove and return the front element
    public int pop() {
        peek(); // ensure output has elements
        return output.pop();
    }

    // Return the front element without removing it
    public int peek() {
        if (output.isEmpty()) {
            // Transfer all elements from input to output
            // This reverses the order, achieving FIFO
            while (!input.isEmpty()) {
                output.push(input.pop());
            }
        }
        return output.peek();
    }

    // Return whether the queue is empty
    public boolean empty() {
        return input.isEmpty() && output.isEmpty();
    }
}

struct MyQueue {
    // input stack for push operations
    input: Vec<i32>,
    // output stack for pop/peek operations
    output: Vec<i32>,
}

impl MyQueue {
    fn new() -> Self {
        MyQueue {
            input: Vec::new(),
            output: Vec::new(),
        }
    }

    // Push element x to the back of the queue
    fn push(&mut self, x: i32) {
        self.input.push(x);
    }

    // Remove and return the front element
    fn pop(&mut self) -> i32 {
        if self.output.is_empty() {
            // Transfer all elements from input to output
            while let Some(x) = self.input.pop() {
                self.output.push(x);
            }
        }
        self.output.pop().unwrap()
    }

    // Return the front element without removing it
    fn peek(&mut self) -> i32 {
        if self.output.is_empty() {
            while let Some(x) = self.input.pop() {
                self.output.push(x);
            }
        }
        *self.output.last().unwrap()
    }

    // Return whether the queue is empty
    fn empty(&self) -> bool {
        self.input.is_empty() && self.output.is_empty()
    }
}

#include <stdbool.h>
#include <stdlib.h>

#define MAX_SIZE 10000

typedef struct {
    int data[MAX_SIZE];
    int top;
} Stack;

typedef struct {
    Stack input;  // input stack for push operations
    Stack output; // output stack for pop/peek operations
} MyQueue;

// Initialize stack
void initStack(Stack* s) {
    s->top = -1;
}

// Push element onto stack
void pushStack(Stack* s, int x) {
    s->data[++s->top] = x;
}

// Pop element from stack
int popStack(Stack* s) {
    return s->data[s->top--];
}

// Peek at top of stack
int peekStack(Stack* s) {
    return s->data[s->top];
}

// Check if stack is empty
bool emptyStack(Stack* s) {
    return s->top == -1;
}

MyQueue* myQueueCreate() {
    MyQueue* q = (MyQueue*)malloc(sizeof(MyQueue));
    initStack(&q->input);
    initStack(&q->output);
    return q;
}

void myQueuePush(MyQueue* obj, int x) {
    pushStack(&obj->input, x);
}

int myQueuePop(MyQueue* obj) {
    int val = myQueuePeek(obj);
    popStack(&obj->output);
    return val;
}

int myQueuePeek(MyQueue* obj) {
    if (emptyStack(&obj->output)) {
        // Transfer all elements from input to output
        // This reverses the order, achieving FIFO
        while (!emptyStack(&obj->input)) {
            pushStack(&obj->output, popStack(&obj->input));
        }
    }
    return peekStack(&obj->output);
}

bool myQueueEmpty(MyQueue* obj) {
    return emptyStack(&obj->input) && emptyStack(&obj->output);
}

void myQueueFree(MyQueue* obj) {
    free(obj);
}

type MyQueue struct {
    // input stack for push operations
    input  []int
    // output stack for pop/peek operations
    output []int
}

func Constructor() MyQueue {
    return MyQueue{
        input:  make([]int, 0),
        output: make([]int, 0),
    }
}

// Push element x to the back of the queue
func (this *MyQueue) Push(x int) {
    this.input = append(this.input, x)
}

// Remove and return the front element
func (this *MyQueue) Pop() int {
    if len(this.output) == 0 {
        // Transfer all elements from input to output
        // This reverses the order, achieving FIFO
        for len(this.input) > 0 {
            n := len(this.input) - 1
            this.output = append(this.output, this.input[n])
            this.input = this.input[:n]
        }
    }
    n := len(this.output) - 1
    val := this.output[n]
    this.output = this.output[:n]
    return val
}

// Return the front element without removing it
func (this *MyQueue) Peek() int {
    if len(this.output) == 0 {
        for len(this.input) > 0 {
            n := len(this.input) - 1
            this.output = append(this.output, this.input[n])
            this.input = this.input[:n]
        }
    }
    return this.output[len(this.output)-1]
}

// Return whether the queue is empty
func (this *MyQueue) Empty() bool {
    return len(this.input) == 0 && len(this.output) == 0
}

## Key Takeaways

- **Two stacks reverse the order** of elements, converting LIFO to FIFO.
- **Amortized O(1)** per operation — each element is moved at most twice (pushed to input, transferred to output, popped from output).
- The **lazy transfer** approach (only transfer when output is empty) is the key optimization.
- Always check **both stacks** for `empty()` — elements might be in either stack.
