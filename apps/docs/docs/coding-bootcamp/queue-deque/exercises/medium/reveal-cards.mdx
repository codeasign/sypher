# Reveal Cards in Increasing Order

**Difficulty:** Medium

You have a deck of `n` integers (the card values). You need to **rearrange the deck** so that when you perform the following reveal process, the cards are revealed in increasing order:

1. Take the top card of the deck and reveal it.
2. If there is still a card in the deck, take the new top card and move it to the bottom of the deck.
3. Repeat until all cards are revealed.

Return the ordering of the deck that produces an increasing reveal sequence.

## Example 1

**Input:** `deck = [17, 13, 11, 2, 3, 5, 7]`  
**Output:** `[2, 13, 3, 11, 5, 17, 7]`  
**Explanation:** The deck is revealed in this order: 2, 3, 5, 7, 11, 13, 17 (increasing).

## Example 2

**Input:** `deck = [1, 2, 3, 4]`  
**Output:** `[1, 3, 2, 4]`  
**Explanation:** Revealing gives: 1, 2, 3, 4.

## Example 3

**Input:** `deck = [1, 1000]`  
**Output:** `[1, 1000]`  
**Explanation:** Two cards: reveal 1 first (smallest), then 1000.

## Input Specification

A single line containing space-separated integers — the card values in the input deck (order does not matter; you must rearrange them).

## Output Specification

Print space-separated integers representing the rearranged deck such that revealing cards one-by-one (top card revealed, next card moved to bottom) produces values in increasing order.

## Constraints

- `1 <= deck.length <= 1000`
- `1 <= deck[i] <= 10^6`
- All values in `deck` are **unique**.

## Hints

<details>
<summary>Hint 1</summary>

Sort the deck first. The smallest card should be revealed first — it belongs at the position that is picked first during the reveal simulation. Think about which position index gets picked at each step.

</details>

<details>
<summary>Hint 2</summary>

Simulate the reveal process on indices using a queue. Initialize the queue with `[0, 1, 2, ..., n-1]`. For each sorted card, assign it to `queue.popleft()`, then rotate by moving `queue.popleft()` to `queue.append(...)`. This tells you which result index receives each sorted card.

</details>

<details>
<summary>Hint 3</summary>

After computing the index-to-card mapping with the queue simulation, fill a result array: `result[index] = sorted_card`. The queue simulation runs in O(n) time after the O(n log n) sort, giving an overall O(n log n) solution.

</details>

</ProblemEditor>
