# Constrained Subsequence Sum

**Difficulty:** Hard

Given an integer array `nums` and an integer `k`, return the maximum sum of a **non-empty subsequence** of `nums` such that for every two **consecutive** integers in the subsequence, `nums[i]` and `nums[j]` (where `i < j`), the condition `j - i <= k` is satisfied.

A subsequence is obtained by deleting some elements from an array without changing the order of the remaining elements.

## Example

**Input:** `nums = [10, 2, -10, 5, 20]`, `k = 3`

**Output:** `37`

**Explanation:**
The subsequence is `[10, 2, 5, 20]` with sum 37.
- `10` and `2` have index difference 1 (`<=` 3)
- `2` and `5` have index difference 2 (`<=` 3)
- `5` and `20` have index difference 1 (`<=` 3)

## Input Specification

First line: integer `k` (maximum gap), followed by space-separated integers of `nums`.

## Output Specification

A single integer representing the maximum sum.

## Constraints

- `1 <= k <= nums.length <= 10^5`
- `-10^4 <= nums[i] <= 10^4`

## Hints

<details>
<summary>Hint 1</summary>

This is a DP problem where dp[i] represents the maximum sum of a subsequence ending at index i (including nums[i]).
</details>

<details>
<summary>Hint 2</summary>

The recurrence is: dp[i] = nums[i] + max(0, max(dp[j] for j in [i-k, i-1])). Use a deque to maintain the maximum dp value in the sliding window.
</details>

<details>
<summary>Hint 3</summary>

The deque stores indices in decreasing order of dp values. When the front index falls out of the window (i - k), remove it.
</details>
