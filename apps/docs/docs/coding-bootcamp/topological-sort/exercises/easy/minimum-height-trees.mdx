# Minimum Height Trees

**Difficulty:** Easy

A tree is an undirected graph with `n` nodes labeled `0` to `n - 1` and exactly `n - 1` edges. Given this tree, you can pick any node as a root and the **height** of the tree rooted at that node is the number of edges on the longest downward path from that root to a leaf.

Return **a list of all nodes** which, when used as the root, produce a tree of **minimum height**. The answer can be returned in any order.

## Example 1

**Input:** `n = 4`, edges = `[[1,0],[1,2],[1,3]]`  
**Output:** `[1]`  
**Explanation:** Rooting at node 1 gives height 1. Rooting at any leaf gives height 2.

## Example 2

**Input:** `n = 6`, edges = `[[3,0],[3,1],[3,2],[3,4],[4,5]]`  
**Output:** `[3,4]`  
**Explanation:** Rooting at node 3 or node 4 both yield height 2, which is optimal.

## Example 3

**Input:** `n = 1`, edges = `[]`  
**Output:** `[0]`  
**Explanation:** A single node is always the root of a height-0 tree.

## Input Specification

The first line is a single integer `n` (the number of nodes).  
The second line contains the edges as consecutive pairs `u v u v ...` (space-separated). If there are no edges, this line is the word `none`.

## Output Specification

Print the root node labels (0-indexed) that produce minimum height trees, **sorted in ascending order**, space-separated.

## Constraints

- `1 <= n <= 2 * 10^4`
- `0 <= edges.length == n - 1`
- `0 <= u, v < n`
- `u != v`
- All pairs `(u, v)` are distinct.
- The input represents a valid tree.

## Hints

<details>
<summary>Hint 1</summary>

The answer always contains at most **2 nodes** (proof: the centroid of a tree). Think about trimming the outermost leaves layer by layer — this is essentially Kahn's algorithm (topological sort) applied to an undirected tree.

</details>

<details>
<summary>Hint 2</summary>

Build an adjacency list and track each node's degree. Collect all leaf nodes (degree == 1) into a queue. Repeatedly remove them, decrementing their neighbors' degrees. When a neighbor reaches degree 1, it becomes the new leaf. Stop when 1 or 2 nodes remain.

</details>

<details>
<summary>Hint 3</summary>

The loop condition is `remaining_nodes > 2`. At each iteration, peel off the current set of leaves and reduce `remaining_nodes` by the number of leaves removed. The nodes left at the end are the answer.

</details>

</ProblemEditor>
