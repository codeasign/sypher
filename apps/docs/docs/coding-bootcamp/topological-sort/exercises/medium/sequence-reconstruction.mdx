# Sequence Reconstruction

**Difficulty:** Medium

Given a sequence `org` which is a permutation of integers from `1` to `n`, and a list of sequences `seqs`, determine if `org` is the **only** shortest supersequence that can be reconstructed from `seqs`.

A sequence `org` can be reconstructed from `seqs` if `org` is a supersequence of every sequence in `seqs` (each `seqs[i]` is a subsequence of `org`). Return `true` if and only if `org` is the **unique** shortest common supersequence — meaning no other permutation of `1..n` satisfies all the ordering constraints from `seqs`.

## Example 1

**Input:** `org = [1, 2, 3]`, `seqs = [[1, 2], [1, 3], [2, 3]]`

**Output:** `true`

**Explanation:** The sequences `[1,2]`, `[1,3]`, and `[2,3]` together fix the order `1 → 2 → 3`. No other permutation satisfies all three constraints simultaneously.

## Example 2

**Input:** `org = [1, 2, 3]`, `seqs = [[1, 2], [1, 3]]`

**Output:** `false`

**Explanation:** `[1,2]` requires `1` before `2` and `[1,3]` requires `1` before `3`, but neither constrains whether `2` or `3` comes first. Both `[1,2,3]` and `[1,3,2]` are valid reconstructions, so `org` is not unique.

## Example 3

**Input:** `org = [4, 1, 5, 2, 6, 3]`, `seqs = [[5, 2, 6, 3], [4, 1, 5, 2]]`

**Output:** `false`

**Explanation:** Several permutations satisfy both sequences. The ordering of `3` relative to items not in its seq is unconstrained.

## Input Specification

The first line contains space-separated integers representing `org`.
The second line contains `k`, the number of sequences in `seqs`.
The next `k` lines each contain one sequence as space-separated integers.

## Output Specification

Print `true` if `org` is the unique shortest supersequence of `seqs`, otherwise print `false`.

## Constraints

- `1 <= n <= 10^4`
- `1 <= len(org) == n`
- `0 <= k <= 10^4`
- `1 <= len(seqs[i]) <= 10^4`
- All values in `org` and `seqs` are in the range `[1, n]`
- `org` is a permutation of `[1, 2, ..., n]`

## Hints

<details>
<summary>Hint 1</summary>

Build a directed graph from the ordering constraints. For each sequence in `seqs`, every consecutive pair `(seqs[i][j], seqs[i][j+1])` becomes a directed edge. Then apply topological sort (Kahn's algorithm) on the graph.

</details>

<details>
<summary>Hint 2</summary>

For `org` to be the **unique** reconstruction, at every step of Kahn's algorithm the queue must contain **exactly one** node (no ties). If at any point the queue has more than one node with in-degree 0, multiple orderings are possible and `org` is not unique.

</details>

<details>
<summary>Hint 3</summary>

There are several edge cases to handle: if `k == 0` (no sequences) and `n > 1`, return `false` (no constraints means many valid orderings). Also check that the resulting topological order equals `org` and covers all `n` nodes. Any value in `seqs` that is outside `[1, n]` should cause an immediate `false` return.

</details>
