# Parallel Courses

**Difficulty:** Hard

You are given 
 courses labeled from  to 
 - 1 and a list of prerequisites where prerequisites[i] = [a, b] indicates that you **must** take course  before course .

In one semester, you can take **any number** of courses as long as you have already taken all of their prerequisites in previous semesters.

Return the **minimum number of semesters** needed to take all courses. If it is impossible to finish all courses (there is a cycle in the prerequisites), return -1.

## Example 1

**Input:** 
 = 4, prerequisites = [[1, 0], [2, 0], [3, 1], [3, 2]]

**Output:** 3

**Explanation:**
- Semester 1: Take course 0 (no prerequisites)
- Semester 2: Take courses 1 and 2 (prerequisite: course 0)
- Semester 3: Take course 3 (prerequisites: courses 1 and 2)

## Example 2

**Input:** 
 = 3, prerequisites = [[1, 0], [2, 1]]

**Output:** 3

**Explanation:**
- Semester 1: Take course 0
- Semester 2: Take course 1
- Semester 3: Take course 2

## Example 3

**Input:** 
 = 2, prerequisites = [[1, 0], [0, 1]]

**Output:** -1

**Explanation:** A cycle exists (0 requires 1, 1 requires 0), so it is impossible to complete all courses.

## Input Specification

The first line contains an integer 
, the number of courses. The second line contains an integer p, the number of prerequisites. The next p lines each contain two integers  and  representing a prerequisite [a, b].

## Output Specification

Print the minimum number of semesters needed, or -1 if impossible.

## Constraints

- `1 <= n <= 2000`
- `0 <= p <= 5000`
- `0 <= a, b < n`
- All prerequisite pairs are unique

## Hints

<details>
<summary>Hint 1</summary>

This is a topological sort problem with a twist. You need to track the minimum number of semesters (levels) in the BFS-based Kahn's algorithm. Process all courses that become available in the same semester together in a batch.

</details>

<details>
<summary>Hint 2</summary>

Use Kahn's algorithm but process the queue in levels. For each semester, take all courses that currently have in-degree 0. This naturally gives the minimum number of semesters because you take every available course each semester.

</details>

<details>
<summary>Hint 3</summary>

If the number of processed courses is less than 
 after the algorithm finishes, the graph has a cycle. Return -1 in that case.

</details>
