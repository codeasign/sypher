# Topological Sort

A topological sort (topological ordering) of a Directed Acyclic Graph (DAG) is a linear ordering of vertices such that for every directed edge `u → v`, vertex `u` comes before `v` in the ordering. Topological sort is the foundation of dependency resolution — if A depends on B, B must come before A in the order.

## The Concept

A topological ordering exists if and only if the graph is a **Directed Acyclic Graph (DAG)**. If the graph contains a cycle, no linear ordering can satisfy all edge constraints — this is how topological sort doubles as a cycle detection algorithm.

### Kahn's Algorithm (BFS)

Kahn's algorithm uses in-degree counting and a queue:

1. Compute the in-degree (number of incoming edges) for each vertex
2. Enqueue all vertices with in-degree 0 (no prerequisites)
3. While the queue is not empty:
   - Dequeue a vertex and add it to the result
   - For each neighbor, decrement its in-degree
   - If a neighbor's in-degree becomes 0, enqueue it
4. If the result contains all vertices, a valid topological order exists. Otherwise, a cycle exists.

### DFS-Based Approach

The DFS approach uses a stack and a visited set:

1. Run DFS from each unvisited vertex
2. Mark vertices as "visiting" (in the current recursion stack)
3. After visiting all neighbors, push the vertex onto a stack (post-order)
4. If a "visiting" vertex is encountered again, a cycle is detected
5. The final stack (popped) gives the topological order

## Complexity

| Aspect | Complexity |
|--------|-----------|
| Time | O(V + E) — each vertex and edge is processed once |
| Space | O(V) — adjacency list, queue/stack, and in-degree array |

## Kahn's Algorithm vs. DFS Approach

| Feature | Kahn's Algorithm (BFS) | DFS-Based Approach |
|---------|----------------------|-------------------|
| Data structure | Queue (in-degree tracking) | Stack (recursion + post-order) |
| Cycle detection | Result size < V | Detects "visiting" vertex in recursion |
| Output order | Naturally from "first" to "last" | Naturally from "last" to "first" (reverse stack) |
| Iterative | Yes | Usually recursive (may overflow for large graphs) |
| Intuition | "Remove nodes with no dependencies" | "Finish dependencies before their dependents" |

## Common Applications

### Course Scheduling

Given a list of courses with prerequisites, determine whether all courses can be completed (cycle detection) and in what order. This is the classic LeetCode problem "Course Schedule" and its variants.

### Build Systems (Make, Bazel, Gradle)

Build systems use topological sort to determine the order in which to compile source files. If file A depends on file B, B must be compiled before A. Topological sort ensures dependencies are built before their dependents.

### Dependency Resolution (Package Managers)

When installing packages, package managers (npm, pip, Maven) resolve dependencies by topologically sorting the dependency graph. This ensures each package is installed before any package that depends on it.

### Task Scheduling

Any system where tasks have prerequisites can use topological sort to determine a valid execution order. This includes data pipelines, ETL jobs, and workflow engines.

## Worked Example: Kahn's Algorithm

Given a graph with 6 vertices and edges: `[(5,2), (5,0), (4,0), (4,1), (2,3), (3,1)]` where `(u,v)` means `u → v` (u must come before v).

```
Vertices: 0, 1, 2, 3, 4, 5

Edges:
  5 → 2, 5 → 0
  4 → 0, 4 → 1
  2 → 3
  3 → 1

In-degree:  [2, 2, 1, 1, 0, 0]
Vertex:     0  1  2  3  4  5

Step 1: Queue = [4, 5]  (in-degree 0)
Step 2: Dequeue 4 → result = [4]
        Decrement neighbors: 0 in-degree 2→1, 1 in-degree 2→1
        Queue = [5]
Step 3: Dequeue 5 → result = [4, 5]
        Decrement neighbors: 2 in-degree 1→0, 0 in-degree 1→0
        Queue = [2, 0]
Step 4: Dequeue 2 → result = [4, 5, 2]
        Decrement neighbors: 3 in-degree 1→0
        Queue = [0, 3]
Step 5: Dequeue 0 → result = [4, 5, 2, 0]
        No outgoing edges from 0
        Queue = [3]
Step 6: Dequeue 3 → result = [4, 5, 2, 0, 3]
        Decrement neighbors: 1 in-degree 1→0
        Queue = [1]
Step 7: Dequeue 1 → result = [4, 5, 2, 0, 3, 1]
        Queue = []

Result: [4, 5, 2, 0, 3, 1] — a valid topological order
```

## Worked Example: DFS-Based Approach

Same graph: edges `[(5,2), (5,0), (4,0), (4,1), (2,3), (3,1)]`

```
DFS(5) → DFS(2) → DFS(3) → DFS(1) → push 1 → push 3 → push 2
         DFS(0) → push 0
         → push 5
DFS(4) → push 4

Stack (bottom to top): [1, 3, 2, 0, 5, 4]
Reverse: [4, 5, 0, 2, 3, 1]

But wait — this is also a valid topological order: [4, 5, 0, 2, 3, 1]
```

Note: There can be multiple valid topological orderings for the same graph. Both `[4, 5, 2, 0, 3, 1]` and `[4, 5, 0, 2, 3, 1]` satisfy all edge constraints.
