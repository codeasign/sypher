# Minimum Increment to Make Array Unique

**Difficulty:** Hard

You are given an integer array `nums`. In one move, you can pick an index `i` where `0 <= i < nums.length` and increment `nums[i]` by `1`.

Return the **minimum number of moves** required to make every value in `nums` **unique**.

## Example 1

**Input:** `nums = [1, 2, 2]`

**Output:** `1`

**Explanation:** Increment the second `2` to `3`. Now `nums = [1, 2, 3]`, all unique. One move total.

## Example 2

**Input:** `nums = [3, 2, 1, 2, 1, 7]`

**Output:** `6`

**Explanation:** After sorting, `nums = [1, 1, 2, 2, 3, 7]`. Increment the second `1` to `4` (3 moves), the second `2` to `5` (3 moves). Total = 6 moves. Result: `[1, 4, 2, 5, 3, 7]`.

## Input Specification

A single line containing space-separated integers representing the array `nums`.

## Output Specification

Print a single integer — the minimum number of increment operations needed.

## Constraints

- `1 <= nums.length <= 10^5`
- `0 <= nums[i] <= 10^5`

## Hints

<details>
<summary>Hint 1</summary>

Sort the array first. Then iterate, ensuring each element is at least one greater than the previous element.

</details>

<details>
<summary>Hint 2</summary>

Keep track of the "minimum required value" for the current element. If the current element is less than this, you must increment it.

</details>

<details>
<summary>Hint 3</summary>

The prefix sum concept applies here: as you process sorted values, the running "ceiling" acts like a prefix max that tracks what the next available value should be.

</details>
