# Prefix Sum

The **prefix sum** (also called cumulative sum) is a preprocessing technique that allows you to answer range sum queries in constant time after an initial O(n) computation. It is one of the most fundamental algorithmic tools in competitive programming and technical interviews.

⏱ 20 minutes

## What You Will Learn

- What a prefix sum array is and how to build it
- How to answer range sum queries in O(1)
- The difference array pattern for efficient range updates
- How to recognize problems solvable with prefix sums
- Complexity analysis and trade-offs

## The Problem

Given an array `A` of `n` elements, you need to answer many queries asking for the sum of elements in a subarray `A[l..r]`. The naive approach sums from `l` to `r` each time — O(n) per query. With `q` queries, that is O(n × q), which is too slow for large inputs.

Prefix sum reduces this to O(1) per query after an O(n) precomputation.

## The Concept

Build a cumulative sum array `P` where `P[i]` stores the sum of all elements from index 0 up to index `i`. Then any range sum is computed with a single subtraction.

## Building the Prefix Sum Array

The algorithm is straightforward:

```python
def build_prefix_sum(arr):
    n = len(arr)
    prefix = [0] * n
    prefix[0] = arr[0]
    for i in range(1, n):
        prefix[i] = prefix[i - 1] + arr[i]
    return prefix
```

**Range query:**
```python
def range_sum(prefix, l, r):
    if l == 0:
        return prefix[r]
    return prefix[r] - prefix[l - 1]
```

## Complexity

| Operation | Time | Space |
|-----------|------|-------|
| Precomputation (build P) | O(n) | O(n) |
| Range sum query (l, r) | O(1) | O(1) |
| Update (add value at index) | O(1) to update A, O(n) to rebuild P | — |

## Worked Example

**Input array:** `A = [3, 1, 4, 1, 5, 9, 2]`

**Step 1 — Build the prefix sum array:**
```
P[0] = A[0]             = 3
P[1] = P[0] + A[1]      = 3 + 1  = 4
P[2] = P[1] + A[2]      = 4 + 4  = 8
P[3] = P[2] + A[3]      = 8 + 1  = 9
P[4] = P[3] + A[4]      = 9 + 5  = 14
P[5] = P[4] + A[5]      = 14 + 9 = 23
P[6] = P[5] + A[6]      = 23 + 2 = 25
```

Result: `P = [3, 4, 8, 9, 14, 23, 25]`

**Step 2 — Answer range queries:**

- `sum(1, 3)` = A[1] + A[2] + A[3] = 1 + 4 + 1 = 6
  Using prefix sum: `P[3] - P[0]` = 9 - 3 = 6

- `sum(0, 4)` = A[0] + A[1] + A[2] + A[3] + A[4] = 3 + 1 + 4 + 1 + 5 = 14
  Using prefix sum: `P[4]` = 14 (P[4] already sums from index 0)

- `sum(2, 6)` = A[2] + A[3] + A[4] + A[5] + A[6] = 4 + 1 + 5 + 9 + 2 = 21
  Using prefix sum: `P[6] - P[1]` = 25 - 4 = 21

## Common Patterns

### Subarray Sum Equals Target

When combined with a hash map, prefix sums can count subarrays with a given sum in O(n). If `prefix[j] - prefix[i] == target`, then the subarray `(i+1..j)` has sum = target.

```python
def subarray_sum(nums, k):
    count = 0
    prefix_sum = 0
    prefix_map = {0: 1}  # sum 0 occurs once (empty prefix)

    for num in nums:
        prefix_sum += num
        # If prefix_sum - k exists in map, those subarrays end here
        count += prefix_map.get(prefix_sum - k, 0)
        prefix_map[prefix_sum] = prefix_map.get(prefix_sum, 0) + 1

    return count
```

### Pivot / Equilibrium Index

An index `i` is a pivot if the sum of elements left of `i` equals the sum of elements to its right. This is efficiently checked with prefix sums.

### 2D Prefix Sum

Extends to matrices: `P[i][j]` = sum of all elements in the rectangle from `(0,0)` to `(i,j)`. Any sub-rectangle sum is computed in O(1) using inclusion-exclusion:

```
Sum(r1, c1, r2, c2) = P[r2][c2] - P[r1-1][c2] - P[r2][c1-1] + P[r1-1][c1-1]
```

## Difference Array (Range Updates)

The **difference array** is the inverse of prefix sum. It is used when you need to apply **many range updates** (adding a value to every element in a range) and then query the final array once.

### The Concept

Given an array `A`, the difference array `D` is defined as:

```
D[0] = A[0]
D[i] = A[i] - A[i-1]  (for i > 0)
```

To add `val` to every element in range `[l, r]`:
- `D[l] += val`
- `D[r + 1] -= val` (if `r + 1 < n`)

After all updates, reconstruct the final array by computing the prefix sum of `D`.

### Difference Array Algorithm

```python
def apply_range_updates(n, updates):
    # updates: list of (l, r, val) tuples
    diff = [0] * (n + 1)  # extra element for sentinel

    for l, r, val in updates:
        diff[l] += val
        diff[r + 1] -= val

    # Reconstruct final array
    result = [0] * n
    curr = 0
    for i in range(n):
        curr += diff[i]
        result[i] = curr

    return result
```

### When to Use Difference Array

- You have an array and many range increment/decrement operations
- The array is only queried **after all updates** are applied
- You need O(1) per update and O(n) final reconstruction
- Example problems: corporate flight bookings, range addition, car pooling

## When to Use Prefix Sum

- You need to answer **many range sum queries** on a **static array** (no updates between queries)
- You need to check **subarray sums against a target value**
- You need to find **pivot/equilibrium indices**
- Problems involve **running totals or cumulative distributions**

Do **not** use prefix sum when the array is frequently updated between queries — consider a Fenwick tree (Binary Indexed Tree) or segment tree instead.
