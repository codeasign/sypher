# Minimum Increment to Make Array Unique — Solution

## Problem Overview

Given an array of integers, find the minimum number of moves (incrementing any element by 1) to make every value unique.

## Brute Force Approach

**Idea:** Sort the array, then for each element, if it is less than or equal to the previous element, increment it repeatedly until it is strictly greater than the previous element.

**Algorithm:**
1. Sort the array
2. For each index `i` from `1` to `n-1`:
   - While `nums[i] <= nums[i-1]`:
     - Increment `nums[i]` by 1
     - Increment `moves` counter
3. Return total `moves`

**Complexity Analysis:**
- **Time:** O(n^2) in the worst case — each element may need many increments
- **Space:** O(1) — sorts and modifies in place (or O(n) if copying)

**Why this could be better:** Incrementing one step at a time is wasteful. We can compute the required increment mathematically in O(1) per element instead of looping.

## Optimal Approach: Greedy with Sorting

**Idea:** Sort the array and maintain a running `minRequired` value that represents the smallest available value for the current element. If the current element is already greater than `minRequired`, it is unique and becomes the new baseline. Otherwise, add the difference to the move count and bump `minRequired` by 1.

**Algorithm:**
1. Sort `nums` in ascending order
2. Initialize `moves = 0` and `minRequired = 0`
3. For each `num` in sorted `nums`:
   - If `num >= minRequired`: set `minRequired = num + 1`
   - Else: `moves += minRequired - num` and `minRequired += 1`
4. Return `moves`

**Why This Works:** After sorting, each element should be at least 1 greater than the previous distinct value to maintain uniqueness. The variable `minRequired` tracks the smallest value the current element can take. When `num` is already above `minRequired`, no increment is needed and `minRequired` jumps to `num`. When `num` is at or below `minRequired`, the element must be pushed up to `minRequired`, costing `(minRequired - num + 1)` moves for that single increment — no step-by-step looping needed.

**Complexity Analysis:**
- **Time:** O(n log n) — dominated by sorting. Can be O(n) with counting sort when values are bounded (e.g., 0 to 10^5).
- **Space:** O(1) — sorts in place (or O(n) if a copy is needed)

def min_increment_for_unique(nums: list) -> int:
    nums.sort()
    moves = 0
    min_required = 0

## Key Takeaways

- **Sorting + greedy tracking** is the optimal pattern for making array elements unique
- The **minimum required** variable elegantly avoids O(n^2) nested loops
- Each element's increment is computed mathematically, not iteratively
- The same idea can be extended with counting sort for O(n) performance when the input value range is small
