# Find Pivot Index — Solution

## Problem Overview

Given an array of integers `nums`, find the left-most pivot index where the sum of elements to the left equals the sum of elements to the right. Return `-1` if no such index exists.

## Brute Force Approach

**Idea:** For each index, compute the left sum and right sum by iterating through the array.

**Algorithm:**
1. For each index `i` from `0` to `n-1`:
   - Compute `left_sum` = sum of `nums[0..i-1]`
   - Compute `right_sum` = sum of `nums[i+1..n-1]`
   - If equal, return `i`
2. Return `-1` if no pivot found

**Complexity Analysis:**
- **Time:** O(n^2) — for each index, we sum the left and right sides
- **Space:** O(1) — no extra space

**Why this could be better:** The repeated summing is wasteful. We can compute the total sum once and use running sums.

## Optimal Approach: Prefix Sum in One Pass

**Idea:** Compute the total sum of the array. Then iterate, maintaining a running `left_sum`. At each index `i`, the right sum is `total_sum - left_sum - nums[i]`. Check if `left_sum == right_sum`.

**Algorithm:**
1. Compute `total_sum = sum(nums)`
2. Initialize `left_sum = 0`
3. For each index `i`:
   - `right_sum = total_sum - left_sum - nums[i]`
   - If `left_sum == right_sum`, return `i`
   - Add `nums[i]` to `left_sum`
4. Return `-1`

**Why This Works:** The pivot index is the only position where the left cumulative sum equals the right cumulative sum. By maintaining a running left sum and computing the right sum from the total, we check each index in constant time.

**Complexity Analysis:**
- **Time:** O(n) — one pass to compute total, one pass to find pivot
- **Space:** O(1) — only a few variables

class Solution {
    public int pivotIndex(int[] nums) {
        int totalSum = 0;
        for (int num : nums) totalSum += num;

        int leftSum = 0;
        for (int i = 0; i < nums.length; i++) {
            int rightSum = totalSum - leftSum - nums[i];
            if (leftSum == rightSum) return i;
            leftSum += nums[i];
        }
        return -1;
    }
}

## Key Takeaways

- The **running sum** pattern is a variant of prefix sums
- Computing the total sum first avoids repeated calculations
- Return the first match to satisfy the "left-most pivot" requirement
