# Permutations — Solution

## Problem Overview

Given an array of distinct integers, return all possible permutations. For `n = 3`, there are `3! = 6` permutations. Unlike subsets, every permutation includes all elements in a different order.

## Brute Force Approach

**Idea:** Generate all permutations by repeatedly picking an unused element. Iterate through all elements, maintain a set of used indices.

**Algorithm:**
1. For each position, try every unused element
2. Include it, mark it used, recurse for the next position
3. Unmark it and try the next element

**Complexity Analysis:**
- **Time:** O(n * n!) — there are n! permutations, and each takes O(n) to build
- **Space:** O(n) — recursion depth of n, plus the boolean array for tracking used elements

**Why this could be better:** This is actually the optimal approach already. The brute force IS the backtracking approach — there is no simpler or more naive solution that is conceptually different since permutations inherently require O(n!) enumeration.

## Optimal Approach: Backtracking with Swap

**Idea:** Instead of using a separate `used` array, we can swap elements in-place. This is more memory efficient and is the standard technique for generating permutations.

**Algorithm:**
1. Define a recursive function `backtrack(first)`:
   - If `first == n`, add a copy of the current array to result
   - Loop `i` from `first` to `n - 1`:
     - Swap `nums[first]` with `nums[i]`
     - Recurse: `backtrack(first + 1)`
     - Swap back (backtrack)
2. Call `backtrack(0)` and return result

**Why This Works:** By swapping elements, we rearrange the array in-place. The element at position `first` represents the current choice for that position. After placing each element, we move to the next position. The swaps ensure we explore all possible arrangements without needing extra space for a `used` array.

**Complexity Analysis:**
- **Time:** O(n * n!) — n! permutations, each costing O(n) to copy at the leaf
- **Space:** O(n) — recursion stack depth of n (no extra `used` array)

### Using a `used` Array (Alternative Approach)

If the swap approach feels confusing, a more intuitive alternative uses a `used` boolean array:

```python
def permute(nums):
    result = []
    current = []
    used = [False] * len(nums)

    def backtrack():
        if len(current) == len(nums):
            result.append(current[:])
            return
        for i in range(len(nums)):
            if not used[i]:
                used[i] = True
                current.append(nums[i])
                backtrack()
                current.pop()
                used[i] = False

    backtrack()
    return result
```

This approach is easier to understand but uses O(n) extra space for the `used` array.

### Edge Cases

- **Single element (n=1):** Only one permutation. The loop adds `[nums[0]]` directly.
- **Two elements (n=2):** Two permutations — the element in the first position and its swap.
- **Negative numbers:** Work identically; swaps treat negatives the same as positives.
- **Large n (n=8):** 8! = 40,320 permutations. This is the upper bound within the time limit.

import java.util.*;

class Solution {
    public List<List<Integer>> permute(int[] nums) {
        List<List<Integer>> result = new ArrayList<>();
        List<Integer> current = new ArrayList<>();
        for (int num : nums) current.add(num);
        backtrack(0, current, result);
        return result;
    }

    private void backtrack(int first, List<Integer> current, List<List<Integer>> result) {
        if (first == current.size()) {
            result.add(new ArrayList<>(current));  // Complete permutation found
            return;
        }
        for (int i = first; i < current.size(); i++) {
            Collections.swap(current, first, i);   // Place element at position 'first'
            backtrack(first + 1, current, result);  // Recurse for next position
            Collections.swap(current, first, i);   // Swap back (backtrack)
        }
    }
}

fn permute(nums: Vec<i32>) -> Vec<Vec<i32>> {
    let mut result: Vec<Vec<i32>> = Vec::new();
    let mut nums = nums;  // Make mutable

    fn backtrack(first: usize, nums: &mut Vec<i32>, result: &mut Vec<Vec<i32>>) {
        if first == nums.len() {
            result.push(nums.clone());  // Complete permutation found
            return;
        }
        for i in first..nums.len() {
            nums.swap(first, i);      // Swap element into position
            backtrack(first + 1, nums, result);  // Recurse
            nums.swap(first, i);      // Swap back (backtrack)
        }
    }

    backtrack(0, &mut nums, &mut result);
    result
}

#include <stdlib.h>
#include <string.h>

void swap(int* a, int* b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

void backtrack(int* nums, int numsSize, int first, int** result, int* returnSize, int** returnColumnSizes) {
    if (first == numsSize) {
        result[*returnSize] = (int*)malloc(numsSize * sizeof(int));
        memcpy(result[*returnSize], nums, numsSize * sizeof(int));
        (*returnColumnSizes)[*returnSize] = numsSize;
        (*returnSize)++;
        return;
    }
    for (int i = first; i < numsSize; i++) {
        swap(&nums[first], &nums[i]);     // Swap element into position
        backtrack(nums, numsSize, first + 1, result, returnSize, returnColumnSizes);
        swap(&nums[first], &nums[i]);     // Swap back (backtrack)
    }
}

int** permute(int* nums, int numsSize, int* returnSize, int** returnColumnSizes) {
    int total = 1;
    for (int i = 2; i <= numsSize; i++) total *= i;  // n!
    int** result = (int**)malloc(total * sizeof(int*));
    *returnColumnSizes = (int*)malloc(total * sizeof(int));
    *returnSize = 0;
    backtrack(nums, numsSize, 0, result, returnSize, returnColumnSizes);
    return result;
}

using System.Collections.Generic;

class Solution {
    public IList<IList<int>> Permute(int[] nums) {
        IList<IList<int>> result = new List<IList<int>>();
        IList<int> current = new List<int>(nums);
        Backtrack(0, current, result);
        return result;
    }

    private void Backtrack(int first, IList<int> current, IList<IList<int>> result) {
        if (first == current.Count) {
            result.Add(new List<int>(current));  // Complete permutation found
            return;
        }
        for (int i = first; i < current.Count; i++) {
            Swap(current, first, i);            // Place element at position 'first'
            Backtrack(first + 1, current, result);  // Recurse for next position
            Swap(current, first, i);            // Swap back (backtrack)
        }
    }

    private void Swap(IList<int> list, int i, int j) {
        int temp = list[i];
        list[i] = list[j];
        list[j] = temp;
    }
}

## Key Takeaways

- **Permutations use the same backtracking template** as subsets, but with a key difference: the base case adds only when the permutation is complete (all elements placed)
- **Swap-based approach** avoids extra space for a `used` array — it rearranges the input array in-place
- The **choose-explore-unchoose** pattern still applies: swap, recurse, swap back
- Time complexity O(n * n!) is optimal since we must generate all n! permutations
- Permutations differ from subsets in that **every element must be used exactly once**
