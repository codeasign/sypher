# Generate Parentheses — Solution

## Problem Overview

Given `n` pairs of parentheses, generate all **valid** (well-formed) combinations of parentheses. A valid combination means every opening parenthesis `(` has a matching closing parenthesis `)` and they are properly nested.

## Approach: Backtracking

**Idea:** Build the parentheses string step by step using recursion. At each step, decide whether to add `(` or `)`, while maintaining validity constraints.

**Algorithm:**
1. Use a recursive backtracking helper that tracks:
   - `open_n`: number of `(` used so far
   - `close_n`: number of `)` used so far
   - `path`: the current string being built
2. **Base case:** When `open_n == n` and `close_n == n`, we have a valid combination — add it to the result list
3. **Recursive case — try `(` first:** If `open_n < n`, add `(` and recurse
4. **Recursive case — try `)`:** If `close_n < open_n`, add `)` and recurse (ensures we never close more than we've opened)
5. By trying `(` before `)` at each step, the results are generated in lexicographic order

**Why This Works:** The two validity checks ensure we never generate an invalid sequence:
- `open_n < n` prevents too many opening parentheses
- `close_n < open_n` prevents a closing parenthesis without a matching open one

**Complexity Analysis:**
- **Time:** O(2^(2n)) in the worst case (Catalan number C_n = (1/(n+1)) * binom(2n, n) ≈ 4^n / (n^(3/2) * sqrt(π)))
- **Space:** O(n) for the recursion stack, plus O(C_n * n) for the output

def generate_parentheses(n: int) -> list[str]:
    result = []

    def backtrack(open_n: int, close_n: int, path: str) -> None:
        # Base case: used all n pairs
        if open_n == n and close_n == n:
            result.append(path)
            return
        # Try adding an opening parenthesis
        if open_n < n:
            backtrack(open_n + 1, close_n, path + '(')
        # Try adding a closing parenthesis (only if it's valid)
        if close_n < open_n:
            backtrack(open_n, close_n + 1, path + ')')

    backtrack(0, 0, '')
    return result

fn generate_parentheses(n: i32) -> Vec<String> {
    let mut result = Vec::new();
    let mut path = String::new();
    backtrack(n, 0, 0, &mut path, &mut result);
    result

fn backtrack(n: i32, open_n: i32, close_n: i32, path: &mut String, result: &mut Vec<String>) {
    // Base case: used all n pairs
    if open_n == n && close_n == n {
        result.push(path.clone());
        return;
    }
    // Try adding an opening parenthesis
    if open_n < n {
        path.push('(');
        backtrack(n, open_n + 1, close_n, path, result);
        path.pop();
    }
    // Try adding a closing parenthesis (only if it's valid)
    if close_n < open_n {
        path.push(')');
        backtrack(n, open_n, close_n + 1, path, result);
        path.pop();
    }
}

#include <stdio.h>
#include <stdlib.h>
#include <string.h>

void backtrack(int openN, int closeN, int n, char* path, int depth, char** result, int* count) {
    // Base case: used all n pairs
    if (openN == n && closeN == n) {
        path[depth] = '\0';
        result[*count] = strdup(path);
        (*count)++;
        return;
    }
    // Try adding an opening parenthesis
    if (openN < n) {
        path[depth] = '(';
        backtrack(openN + 1, closeN, n, path, depth + 1, result, count);
    }
    // Try adding a closing parenthesis (only if it's valid)
    if (closeN < openN) {
        path[depth] = ')';
        backtrack(openN, closeN + 1, n, path, depth + 1, result, count);
    }
}

char** generateParentheses(int n, int* returnSize) {
    int capacity = 1 << (2 * n);
    char** result = (char**)malloc(capacity * sizeof(char*));
    char* path = (char*)malloc((2 * n + 1) * sizeof(char));
    *returnSize = 0;
    backtrack(0, 0, n, path, 0, result, returnSize);
    free(path);
    return result;
}

## Key Takeaways

- **Backtracking** naturally generates all valid combinations by exploring choices incrementally
- The **validity conditions** (`open < n` and `close < open`) prevent invalid sequences
- Always trying `(` before `)` produces results in a predictable order
- This is a classic example of **constrained combinatorial generation**
