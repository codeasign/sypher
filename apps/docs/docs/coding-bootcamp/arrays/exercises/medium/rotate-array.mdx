# Rotate Array

**Difficulty:** Medium

Given an integer array `nums`, rotate the array to the right by `k` steps, where `k` is non-negative.

Return the rotated array.

## Example 1

**Input:** `nums = [1, 2, 3, 4, 5, 6, 7]`, `k = 3`  
**Output:** `[5, 6, 7, 1, 2, 3, 4]`  
**Explanation:**  
- Rotate 1 step to the right: `[7, 1, 2, 3, 4, 5, 6]`  
- Rotate 2 steps to the right: `[6, 7, 1, 2, 3, 4, 5]`  
- Rotate 3 steps to the right: `[5, 6, 7, 1, 2, 3, 4]`

## Example 2

**Input:** `nums = [-1, -100, 3, 99]`, `k = 2`  
**Output:** `[3, 99, -1, -100]`  
**Explanation:**  
- Rotate 1 step to the right: `[99, -1, -100, 3]`  
- Rotate 2 steps to the right: `[3, 99, -1, -100]`

## Input Specification

The first line contains space-separated integers representing the array `nums`.
The second line contains a single integer `k` — the number of rotation steps.

## Output Specification

Print space-separated integers representing the rotated array.

## Constraints

- `1 <= nums.length <= 10^5`
- `-2^31 <= nums[i] <= 2^31 - 1`
- `0 <= k <= 10^5`

## Hints

<details>
<summary>Hint 1</summary>

The naive approach moves elements one by one — O(n * k) time. Can you do it in O(n)?

</details>

<details>
<summary>Hint 2</summary>

Notice that rotating by `k` is the same as rotating by `k % n`. Use the reversal algorithm: reverse the whole array, then reverse the first `k` elements, then reverse the last `n - k` elements.

</details>

<details>
<summary>Hint 3</summary>

The reversal algorithm uses O(1) extra space. Think about how reversing different parts of the array achieves the rotation.

</details>

</ProblemEditor>
