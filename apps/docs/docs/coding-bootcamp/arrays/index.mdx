# Arrays

The Arrays pattern covers traversal, in-place modification, prefix/suffix
arrays, and simulation over array data structures.

вҸұ 15 minutes

## What You Will Learn

- Array traversal techniques вҖ” forward, backward, and two-pointer
- In-place modification strategies вҖ” shifting, swapping, overwriting
- Prefix and suffix precomputation for range queries
- Simulation problems that follow rules step by step

## The Concept

Arrays store elements in contiguous memory, giving O(1) random access by
index. This fundamental property вҖ” knowing the exact position of every
element вҖ” enables a family of efficient patterns.

**Forward traversal** visits each element from left to right, the most
natural loop. **Backward traversal** goes right to left, useful when later
elements influence earlier ones. **Two-pointer traversal** uses two indices
that move toward each other or in the same direction, often reducing O(nВІ)
nested loops to O(n).

**In-place modification** avoids allocating extra arrays by reusing the
existing storage вҖ” shifting elements to fill gaps, swapping to reorder,
or overwriting to filter. **Prefix and suffix arrays** precompute running
totals (sums, products, maxima) so any range query becomes O(1).

**Simulation** problems describe a process with precise rules вҖ” walk
through the array step by step, updating state according to those rules.

## Key Patterns

### Forward / Backward Traversal

The simplest and most common pattern. A single loop walks the array in
one direction.

```python
# Forward traversal
for i in range(len(arr)):
    process(arr[i])

# Backward traversal
for i in range(len(arr) - 1, -1, -1):
    process(arr[i])
```

Use forward when each element depends only on earlier elements; use
backward when each element depends on later elements.

### In-place Modification

Modify the array without allocating extra O(n) space. Common techniques:

- **Shifting** вҖ” move elements right or left to insert or delete
- **Swapping** вҖ” exchange two elements to reorder
- **Overwriting** вҖ” write a new value into an index that has already been
  consumed (e.g., removing duplicates from a sorted array)

```python
# Remove duplicates in-place (sorted array)
write = 1
for read in range(1, len(arr)):
    if arr[read] != arr[read - 1]:
        arr[write] = arr[read]
        write += 1
# arr[:write] contains the unique elements
```

### Prefix / Suffix Precomputation

Build auxiliary arrays that store cumulative results, then answer range
queries in O(1).

| Problem | Prefix Array | Query |
|---------|-------------|-------|
| Range sum | `prefix[i] = sum of arr[0..i]` | `sum(l..r) = prefix[r] - prefix[l-1]` |
| Range product (no zero) | `prefix[i] = product of arr[0..i]` | `product(l..r) = prefix[r] // prefix[l-1]` |
| Running max/min | `prefix[i] = max(arr[0..i])` | `max(l..r) = max(prefix[r], ...)` |

```python
# Build prefix sum
prefix = [0] * len(arr)
running = 0
for i, val in enumerate(arr):
    running += val
    prefix[i] = running

# Query sum of arr[2..5]
result = prefix[5] - (prefix[1] if 1 >= 0 else 0)
```

### Simulation

The problem describes rules вҖ” implement them faithfully. Common
simulation problems over arrays:

- **Game of Life** вҖ” apply birth/death rules to every cell
- **Spiral Matrix** вҖ” walk the perimeter, then recurse inward
- **Robot in a Grid** вҖ” follow movement instructions on a grid

```python
# Spiral Matrix simulation
top, bottom = 0, len(matrix) - 1
left, right = 0, len(matrix[0]) - 1
result = []

while top <= bottom and left <= right:
    # Traverse right along top row
    for col in range(left, right + 1):
        result.append(matrix[top][col])
    top += 1

    # Traverse down along right column
    for row in range(top, bottom + 1):
        result.append(matrix[row][right])
    right -= 1

    # Traverse left along bottom row
    if top <= bottom:
        for col in range(right, left - 1, -1):
            result.append(matrix[bottom][col])
        bottom -= 1

    # Traverse up along left column
    if left <= right:
        for row in range(bottom, top - 1, -1):
            result.append(matrix[row][left])
        left += 1
```

## Complexity

| Metric | Complexity |
|--------|-----------|
| Forward / backward traversal | O(n) time, O(1) space |
| In-place modification | O(n) time, O(1) space |
| Prefix / suffix precomputation | O(n) time, O(n) space |
| Simulation | O(n Г— m) time, O(1) extra space |

## Common Applications

| Application | Pattern | Example |
|-------------|---------|---------|
| Running totals | Prefix sum | Cumulative sum of daily sales |
| Rotate array | In-place swap | Rotate array by k positions |
| Remove duplicates | In-place overwrite | Deduplicate sorted list |
| Game simulation | Walk-through | Conway's Game of Life |
| Matrix traversal | Bounded walk | Spiral order of a matrix |
| Stock prices | Forward + backward scan | Best time to buy and sell stock |
| Product except self | Prefix Г— suffix | Product of all elements except current |

## When to Use Arrays Pattern

- You need to process **sequential** data in one or more passes
- **Index-based access** is sufficient (no need for key-value lookups)
- You need **O(1) memory** for small, fixed-size data
- The problem benefits from **precomputing range aggregates**
