# Rotate Array — Solution

## Problem Overview

Given an integer array `nums`, rotate the array to the right by `k` steps, where `k` is non-negative. Rotate in-place with O(1) extra space.

## Brute Force Approach

**Idea:** Rotate the array one step at a time, k times. Each rotation shifts every element one position to the right, with the last element wrapping around to the front.

**Algorithm:**
1. Normalize `k = k % n` (if k >= n, rotating by n brings us back to the original)
2. Repeat k times:
   - Store the last element in a temporary variable
   - Shift all elements one position to the right
   - Place the saved last element at index 0
3. The array is now rotated by k positions

**Complexity Analysis:**
- **Time:** O(n × k) — each of k rotations shifts all n elements
- **Space:** O(1) — only a single temporary variable

**Why this is suboptimal:** For large k (up to 10⁵), this becomes O(n²) in the worst case. The optimal approach achieves O(n) using a clever reversal trick.

## Optimal Approach: Reverse Three Times

**Idea:** Reverse the entire array, then reverse the first k elements, then reverse the remaining n-k elements. This achieves the desired rotation in-place.

**Algorithm:**
1. Normalize `k = k % n` to handle cases where k >= n
2. **Reverse the entire array:** `reverse(nums, 0, n - 1)`
3. **Reverse the first k elements:** `reverse(nums, 0, k - 1)`
4. **Reverse the remaining n-k elements:** `reverse(nums, k, n - 1)`

**Why This Works:** The three reversals rearrange the array into the rotated order. Let's trace with `nums = [1,2,3,4,5,6,7]`, `k = 3`:
- Original: `[1, 2, 3, 4, 5, 6, 7]`
- Reverse all: `[7, 6, 5, 4, 3, 2, 1]`
- Reverse first k (3): `[5, 6, 7, 4, 3, 2, 1]`
- Reverse last n-k (4): `[5, 6, 7, 1, 2, 3, 4]` ✅

**Complexity Analysis:**
- **Time:** O(n) — each element is visited twice (once in the full reverse, once in its partial reverse)
- **Space:** O(1) — all operations are in-place using swaps

def rotate(nums: list[int], k: int) -> list[int]:
    # Helper function to reverse a portion of the array in-place
    def reverse(arr: list[int], left: int, right: int) -> None:
        while left < right:
            # Swap elements at the two ends
            arr[left], arr[right] = arr[right], arr[left]
            left += 1
            right -= 1

    n = len(nums)
    # Normalize k to handle cases where k >= n
    k = k % n
    if k == 0:
        return nums  # No rotation needed

    # Step 1: Reverse the entire array
    reverse(nums, 0, n - 1)
    # Step 2: Reverse the first k elements
    reverse(nums, 0, k - 1)
    # Step 3: Reverse the remaining n-k elements
    reverse(nums, k, n - 1)
    return nums

            int temp = nums[left];
            nums[left] = nums[right];
            nums[right] = temp;
            left++;
            right--;
        }
    }
}

## Key Takeaways

- **Three-reversal trick** is the canonical O(1) space solution for array rotation
- Always **normalize k** with `k = k % n` first — rotating by n returns to the original
- The reversal pattern works for both left and right rotation with slight modifications
- Each reversal is O(n) with O(1) space, making the overall approach optimal
