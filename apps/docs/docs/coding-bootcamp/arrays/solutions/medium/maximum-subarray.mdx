# Maximum Subarray — Solution

## Problem Overview

Find the contiguous subarray with the largest sum. This is solved optimally with **Kadane's Algorithm** — one of the most important greedy/DP patterns.

## Approach: Kadane's Algorithm

**Idea:** Maintain a running `curr_sum` and a `best_sum`. At each element, decide whether to extend the current subarray or start fresh. If `curr_sum` becomes negative, reset it to 0 (the current element is better than keeping a negative sum).

**Algorithm:**
1. Initialize `best_sum = nums[0]` (or very small number), `curr_sum = 0`.
2. For each `num` in `nums`:
   - `curr_sum = max(num, curr_sum + num)` — extend or start new.
   - `best_sum = max(best_sum, curr_sum)`.
3. Return `best_sum`.

**Complexity Analysis:**
- **Time:** O(n) — single pass.
- **Space:** O(1) — constant.

def max_subarray(nums: list[int]) -> int:
    curr_sum = 0
    best_sum = float('-inf')

    for num in nums:
        # Extend existing subarray or start new one at num
        curr_sum = max(num, curr_sum + num)
        # Update global maximum
        best_sum = max(best_sum, curr_sum)

    return best_sum

func maxSubArray(nums []int) int {
    if len(nums) == 0 {
        return 0
    }
    currSum := nums[0]
    bestSum := nums[0]

    for _, num := range nums[1:] {
        if currSum+num > num {
            currSum = currSum + num
        } else {
            currSum = num
        }
        if currSum > bestSum {
            bestSum = currSum
        }
    }

    return bestSum
}

#include <limits.h>

int maxSubArray(int* nums, int numsSize) {
    int currSum = 0;
    int bestSum = INT_MIN;

    for (int i = 0; i < numsSize; i++) {
        int extended = currSum + nums[i];

        if (currSum > bestSum) bestSum = currSum;

## Key Takeaways

- **Kadane's Algorithm**: If the running sum drops below 0, reset it. The max at each step is either the current element or the running sum plus the current element.
- **Divide and conquer**: An O(n log n) approach exists (split array, compute cross sum) but Kadane's is simpler and faster.
