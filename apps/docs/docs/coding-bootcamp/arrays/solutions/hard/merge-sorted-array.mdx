# Merge Sorted Array — Solution

## Problem Overview

You are given two sorted integer arrays `nums1` and `nums2`. Merge `nums2` into `nums1` as one sorted array. `nums1` has a length of `m + n`, where the first `m` elements are the actual elements and the last `n` elements are zeros (placeholders). `nums2` has length `n`.

## Brute Force Approach

**Idea:** Concatenate `nums2` onto the end of `nums1` (overwriting the placeholder zeros), then sort the entire array.

**Algorithm:**
1. Copy all elements of `nums2` into the last `n` positions of `nums1` (indices m through m+n-1)
2. Sort the entire `nums1` array using a standard sorting algorithm

**Complexity Analysis:**
- **Time:** O((m+n) log(m+n)) — dominated by the sort operation
- **Space:** O(1) — sorting is typically in-place

**Why this is suboptimal:** We are not taking advantage of the fact that both input arrays are already sorted. The optimal approach merges in O(m+n) time by using a two-pointer technique.

## Optimal Approach: Two Pointers from the End

**Idea:** Use two pointers to read from the end of each array and write the largest remaining element to the end of `nums1`, working backwards to avoid overwriting elements that haven't been processed yet.

**Algorithm:**
1. Set `p1 = m - 1` (last actual element in nums1), `p2 = n - 1` (last element in nums2), and `p = m + n - 1` (last position in nums1)
2. While both `p1 >= 0` and `p2 >= 0`:
   - If `nums1[p1] > nums2[p2]`, place `nums1[p1]` at `nums1[p]`, decrement `p1`
   - Otherwise, place `nums2[p2]` at `nums1[p]`, decrement `p2`
   - Decrement `p`
3. If `p2` elements remain (nums2 has leftover smaller elements), copy them into `nums1`
4. No need to copy leftover `nums1` elements — they are already in place

**Why This Works:** By filling from the end, we avoid the need for a temporary array. The largest elements are placed at the end first, ensuring we never overwrite an unprocessed element in `nums1`. When one array is exhausted, the remaining elements from the other array are already in their correct positions (for `nums1`) or need to be copied over (for `nums2`).

**Complexity Analysis:**
- **Time:** O(m + n) — each element is processed exactly once
- **Space:** O(1) — all operations are in-place

def merge(nums1: list[int], m: int, nums2: list[int], n: int) -> None:
    """
    Merge nums2 into nums1 in-place, assuming nums1 has size m + n.
    """
    # Set pointers at the end of each array
    p1 = m - 1          # Last actual element in nums1
    p2 = n - 1          # Last element in nums2
    p = m + n - 1       # Last position in nums1 (write pointer)

    # Compare from the end and place the larger element at position p
    while p1 >= 0 and p2 >= 0:
        if nums1[p1] > nums2[p2]:
            # nums1 has the larger element — place it at the end
            nums1[p] = nums1[p1]
            p1 -= 1
        else:
            # nums2 has the larger (or equal) element
            nums1[p] = nums2[p2]
            p2 -= 1
        p -= 1  # Move the write pointer left

    # If nums2 has leftover elements, copy them into nums1
    # (nums1 leftovers are already in their correct positions)
    while p2 >= 0:
        nums1[p] = nums2[p2]
        p2 -= 1
        p -= 1

## Key Takeaways

- **Merge from the end** avoids overwriting elements and eliminates the need for extra space
- Always process from the **largest** elements down to the smallest when working in-place at the end of an array
- The `nums1` array already has space allocated — the zeros are placeholders, not data
- After the main loop, only `nums2` leftovers need to be copied; `nums1` leftovers are already in position
