## What This Is

A crash course in algorithmic coding patterns. Each module covers one
pattern — you'll learn the theory, solve curated problems of increasing
difficulty, and study idiomatic solutions in 6 programming languages.

## Why Coding Patterns Matter

Interview problems and real-world code challenges follow recurring
patterns. Master the pattern, and you can solve any problem in its
family — instead of memorizing hundreds of individual solutions.

## Skills Gained

- Recognize which pattern fits a given problem
- Implement two-pointer, sliding window, and other fundamental algorithms
- Write clean, correct code in Python, Java, C++, TypeScript,
  JavaScript, and Rust
- Test solutions against sample and hidden test cases via an embedded
  Judge0 code runner

## Prerequisites

Basic programming knowledge in any language. Familiarity with arrays,
strings, and hash maps is helpful but not required.

## How the Bootcamp Works

Each pattern module has three parts:

1. **Theory** — Learn the pattern: what it is, when to use it,
   complexity analysis, and a worked example.
2. **Exercises** — Solve problems grouped by difficulty (Easy → Medium
   → Hard). Write code in the embedded editor and run it against
   test cases.
3. **Solutions** — Study idiomatic solutions in all 6 languages
   side-by-side.

## Patterns

- **Arrays** — Traversal, in-place modification, prefix/suffix arrays, and simulation over arrays
- **Hash Maps / Hash Sets** — Frequency counting, fast lookup, and deduplication with O(1) average access
- **Prefix Sum** — Running sum precomputation for O(1) range queries and subarray sum problems
- **Two Pointers** — Using left/right converging or same-direction pointers to solve array and string problems efficiently (O(n) time, O(1) space)
- **Sliding Window** — Maintaining a window of elements to solve subarray/substring problems in O(n) time
- **Binary Search** — Finding elements in sorted arrays and searching the answer space in O(log n) time
- **Fast & Slow Pointers** — Floyd's cycle detection, finding middle nodes, and cycle-related problems
- **Linked List** — Pointer manipulation, reversal, cycle detection, and merging linked lists
- **Stack & Monotonic Stack** — LIFO structures for nested patterns, next greater/smaller element, and expression evaluation
- **Queue & Deque** — FIFO and double-ended structures for BFS, sliding windows, and scheduling
- **Tree DFS & BFS** — Recursive and level-order tree traversals for depth, path, and tree property problems
- **Binary Search Tree (BST)** — Ordered traversal, validation, search, and manipulation of BSTs
- **Heap / Priority Queue** — Using priority queues for Top K, median finding, and merging sorted data
- **Intervals** — Merging, inserting, and finding overlaps among ranges
- **Greedy** — Local optimum choices leading to global optimum for scheduling, jumps, and optimization
- **Dynamic Programming** — Breaking problems into overlapping subproblems with memoization for polynomial-time solutions
- **Backtracking** — Systematically searching for solutions by building candidates and pruning invalid branches
- **Trie** — Prefix tree for efficient word search, autocomplete, and pattern matching
- **Topological Sort** — DAG ordering for dependency resolution and course scheduling
- **Union Find (DSU)** — Disjoint set union for dynamic connectivity and component tracking
- **Graph Algorithms** — Adjacency lists, DFS/BFS traversal, cloning, connected components, and shortest paths
- **Matrix Traversal** — Direction vectors, spiral traversal, grid rotations, and in-place matrix operations
- **Bit Manipulation** — XOR tricks, bit masking, and shift operations for space/time optimization
- **Math & Number Theory** — GCD, prime sieving, modular arithmetic, and exponentiation
- **String Algorithms** — Parsing, pattern matching (KMP), encoding/decoding, and string manipulation
- **Simulation** — Following exact rules, managing state, and stepping through processes
- **Design** — Designing data structures (LRU/LFU caches, Min Stack, Browser History)
