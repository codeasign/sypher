# Binary Search

Binary Search finds elements in a sorted array by repeatedly dividing the search range in half.

⏱ 20 minutes

## What You Will Learn

- Classic binary search and its variants
- How to handle rotated arrays and unknown boundaries
- When to use binary search on the answer space

## The Problem

Finding an element in a sorted array can be done in O(log n) by eliminating half the search space at each step.

## The Concept

Start with left and right pointers at the ends of the search space. At each step, examine the middle element. If it matches, return. If the target is smaller, search the left half; otherwise, search the right half.

## Variants

- **Standard**: Find exact element in sorted array
- **First/Last position**: Find boundaries of duplicate elements
- **Rotated array**: Search in array that's been rotated
- **Answer space**: Binary search on the result space (predicate-based)
- **2D matrix**: Extend binary search to two dimensions

## When to Use Binary Search

- The input is **sorted** (or can be sorted)
- You're asked for O(log n) time complexity
- The problem has a **monotonic predicate** — if condition(x) is true, then condition(x+1) is also true
