# Binary Search — Solution

## Problem Overview

Given a **sorted** array of integers and a `target` value, return the index of the target if found, or `-1` if not found. The array is sorted in ascending order.

## Brute Force Approach: Linear Search

**Idea:** Scan every element from left to right until you find the target.

**Algorithm:**
1. Loop through each index `i` from `0` to `n-1`
2. If `nums[i] == target`, return `i`
3. If the loop finishes, return `-1`

**Complexity Analysis:**
- **Time:** O(n) — in the worst case, we examine every element
- **Space:** O(1) — no extra space

**Why this could be better:** This ignores the fact that the array is sorted. We can do much better by exploiting the sorted order.

## Optimal Approach: Binary Search

**Idea:** Repeatedly divide the search range in half. Compare the middle element with the target and eliminate the half that cannot contain the target.

**Algorithm:**
1. Initialize `left = 0` and `right = n - 1`
2. While `left <= right`:
   - Compute `mid = left + (right - left) / 2` (avoids integer overflow)
   - If `nums[mid] == target`, return `mid`
   - If `nums[mid] < target`, discard the left half: `left = mid + 1`
   - If `nums[mid] > target`, discard the right half: `right = mid - 1`
3. If the loop exits, return `-1`

**Why This Works:** Since the array is sorted, every element to the left of `mid` is less than or equal to `nums[mid]`, and every element to the right is greater than or equal to `nums[mid]`. By comparing the target to the middle element, we can always eliminate one half of the remaining search space.

**Complexity Analysis:**
- **Time:** O(log n) — each iteration halves the search space
- **Space:** O(1) — only a few integer variables

## Deep Dive: How Binary Search Works

### Visual Walkthrough

For the input `n = 7, target = 6, nums = [1, 2, 3, 4, 5, 6, 7]`, let's trace the algorithm:

```
Initial: left = 0, right = 6
nums = [1, 2, 3, 4, 5, 6, 7]
         ↑                 ↑
       left              right

Step 1: mid = 0 + (6 - 0) / 2 = 3
nums[3] = 4
4 < 6 → target is in the right half
left = mid + 1 = 4

nums = [1, 2, 3, 4, 5, 6, 7]
                     ↑     ↑
                   left  right

Step 2: mid = 4 + (6 - 4) / 2 = 5
nums[5] = 6
6 == 6 → target found at index 5
```

### Edge Cases to Consider

- **Target at the first element (index 0):** The first comparison at `mid` may or may not hit index 0 depending on the array size. The algorithm converges correctly.
- **Target at the last element (index n-1):** Similarly, the algorithm narrows down to the rightmost element.
- **Target not present:** The loop exits when `left > right` and returns `-1`.
- **Single element array:** `left = right = 0`. If `nums[0] == target`, return 0; otherwise return -1.
- **Duplicates:** Any occurrence of the target is valid. Standard binary search returns the first middle where it finds the target.
- **Integer overflow:** Traditional `mid = (left + right) / 2` can overflow for very large arrays. Using `mid = left + (right - left) / 2` avoids this.

### Why Binary Search is Optimal

A sorted array of `n` elements can only have one correct index for the target. Each comparison yields a binary outcome (go left or go right), which gives us at most `log₂(n)` pieces of information. No comparison-based algorithm can do better than O(log n) for searching in a sorted array.

class Solution {
    public int binarySearch(int[] nums, int target) {
        int left = 0;                              // Start of search range
        int right = nums.length - 1;               // End of search range

        while (left <= right) {                     // Continue while range is valid
            int mid = left + (right - left) / 2;   // Compute mid to avoid overflow

            if (nums[mid] == target) {             // Found the target
                return mid;                        // Return its index
            } else if (nums[mid] < target) {       // Target is in the right half
                left = mid + 1;                    // Narrow search to right half
            } else {                                // Target is in the left half
                right = mid - 1;                   // Narrow search to left half
            }
        }

        return -1;                                 // Target not found
    }
}

## Key Takeaways

- **Binary search exploits the sorted property** to achieve O(log n) search time.
- **The while loop condition is `left <= right`**, not `left < right`, to correctly handle the case when the target is at the final `left == right` position.
- **Use `mid = left + (right - left) / 2`** to prevent integer overflow.
- **The algorithm is deterministic** — for a given sorted array, it always returns the same result.
- Binary search is a foundational technique used in countless algorithms and data structures (BSTs, heaps, segment trees, etc.).
