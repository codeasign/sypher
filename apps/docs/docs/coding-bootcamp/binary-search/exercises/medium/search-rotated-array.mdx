# Search in Rotated Sorted Array

**Difficulty:** Medium

There is an integer array `nums` sorted in ascending order (with **distinct**
values). The array is then **rotated** at an unknown pivot index `k`
(`1 <= k < n`), resulting in the array
`nums[k], nums[k+1], ..., nums[n-1], nums[0], nums[1], ..., nums[k-1]`.

Given the rotated array `nums` and a `target` value, return the **index**
of the target if it is in the array, or `-1` if it is not.

You must write an algorithm with **O(log n)** runtime complexity.

## Example 1

**Input:** `n = 5, target = 0, nums = [4, 5, 1, 2, 3]`  
**Output:** `2`  
**Explanation:** The target `0` is at index 2. The array was originally
`[1, 2, 3, 4, 5]` and was rotated at pivot index 2.

## Example 2

**Input:** `n = 5, target = 3, nums = [4, 5, 1, 2, 3]`  
**Output:** `4`  
**Explanation:** The target `3` is at the end of the array.

## Example 3

**Input:** `n = 5, target = 6, nums = [4, 5, 1, 2, 3]`  
**Output:** `-1`  
**Explanation:** The target `6` is not present in the array.

## Input Specification

The first line contains two space-separated integers: `n` (the length of
the array) and `target` (the value to search for).

The second line contains `n` space-separated integers representing the
rotated sorted array `nums`.

## Output Specification

Print a single integer: the 0-based index of `target` in the array, or
`-1` if the target is not found.

## Constraints

- `1 <= n <= 10^5`
- `-10^4 <= nums[i] <= 10^4`
- All values in `nums` are **distinct** (except test cases denoted explicitly).
- `nums` is guaranteed to be rotated at some pivot index unknown to you.

## Hints

<details>
<summary>Hint 1</summary>

Even though the array is rotated, one half of the array (either from
`left` to `mid` or from `mid` to `right`) will always be fully sorted
in ascending order.
</details>

<details>
<summary>Hint 2</summary>

Compare `nums[left]` with `nums[mid]` to determine which half is sorted.
Then check if the target lies within that sorted half's range.
</details>

<details>
<summary>Hint 3</summary>

If the target lies within the sorted half, continue binary search in that
half. Otherwise, search in the other half.
</details>
