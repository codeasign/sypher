# Median of Two Sorted Arrays

**Difficulty:** Hard

Given two sorted arrays `nums1` and `nums2` of size `n` and `m`
respectively, return the **median** of the two sorted arrays.

The overall run time complexity should be **O(log(min(n, m)))**.

## Example 1

**Input:** `nums1 = [1, 3], nums2 = [2]`  
**Output:** `2.0`  
**Explanation:** The merged array `[1, 2, 3]` has median 2.

## Example 2

**Input:** `nums1 = [1, 2], nums2 = [3, 4]`  
**Output:** `2.5`  
**Explanation:** The merged array `[1, 2, 3, 4]` has median (2 + 3) / 2 = 2.5.

## Example 3

**Input:** `nums1 = [1, 5, 9], nums2 = [3]`  
**Output:** `4.0`  
**Explanation:** The merged array `[1, 3, 5, 9]` has median 4.

## Input Specification

The first line contains two space-separated integers: `n` and `m`
(the lengths of the two arrays).

The second line contains `n` space-separated integers representing
`nums1` (if `n > 0`; otherwise this line is empty).

The third line contains `m` space-separated integers representing
`nums2` (if `m > 0`; otherwise this line is empty).

## Output Specification

Print a single floating-point number: the median of the two sorted
arrays, formatted with one decimal place.

## Constraints

- `0 <= n, m <= 1000`
- `-10^6 <= nums1[i], nums2[i] <= 10^6`
- Both `nums1` and `nums2` are sorted in ascending order.
- `n + m >= 1` (one array may be empty, but both arrays are never simultaneously empty)

## Hints

<details>
<summary>Hint 1</summary>

The median divides the combined set into two halves of equal size (or
differing by at most one). Think about how to partition both arrays
simultaneously.
</details>

<details>
<summary>Hint 2</summary>

Binary search on the **smaller** array to find the correct partition
point. The partition in the larger array is determined automatically
by the total number of elements needed.
</details>

<details>
<summary>Hint 3</summary>

The key condition for a valid partition is:
`max(left_part) <= min(right_part)`. Check the boundary elements
between the two partitions to verify this condition.
</details>
