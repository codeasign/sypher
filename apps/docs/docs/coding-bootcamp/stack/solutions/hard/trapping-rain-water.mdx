# Trapping Rain Water — Solution

## Problem Overview

Given an array `heights` of non-negative integers representing an elevation map (each bar has width 1), compute the total amount of rainwater that can be trapped between the bars after raining.

## Brute Force Approach

**Idea:** For each bar, compute the maximum height to its left and the maximum height to its right. The water trapped above the bar is `min(maxLeft, maxRight) - height[i]`.

**Algorithm:**
1. Initialize `totalWater = 0`
2. For each index `i` from 0 to n-1:
   - Find `maxLeft = max(heights[0..i])`
   - Find `maxRight = max(heights[i..n-1])`
   - Add `min(maxLeft, maxRight) - heights[i]` to `totalWater`
3. Return `totalWater`

**Complexity Analysis:**
- **Time:** O(n²) — for each of n bars, we scan n elements left and right
- **Space:** O(1) — only a few variables

**Why this is suboptimal:** For n = 10^5, O(n²) is too slow. The monotonic stack and two-pointer approaches both achieve O(n).

## Optimal Approach: Monotonic Decreasing Stack

**Idea:** Use a monotonic decreasing stack to find "valleys" (low points) between taller bars. When we encounter a bar taller than the top of the stack, we know we have found a right boundary. The popped bar forms the bottom of a valley, the new top of the stack is the left boundary, and the current bar is the right boundary.

**Algorithm:**
1. Initialize `totalWater = 0` and an empty stack
2. For each index `i` from 0 to n-1:
   - While the stack is not empty and `heights[i] > heights[stack.top()]`:
     - Pop the top index `bottom = stack.pop()`
     - If the stack is now empty, break (no left boundary)
     - Let `left = stack.top()` (left boundary index)
     - Let `width = i - left - 1` (distance between boundaries)
     - Let `height = min(heights[left], heights[i]) - heights[bottom]` (water depth above bottom)
     - Add `width * height` to `totalWater`
   - Push `i` onto the stack
3. Return `totalWater`

**Why This Works:** The stack maintains indices in decreasing order of height. When a taller bar appears, it forms a right boundary for all lower bars on the stack. For each lower bar (the bottom), the bar just below it in the stack is the left boundary. The water volume is the width (distance between boundaries) times the height difference (min of boundaries minus bottom).

**Complexity Analysis:**
- **Time:** O(n) — each index is pushed and popped at most once
- **Space:** O(n) — the stack holds up to n indices

### Key Insight

The stack approach treats each valley as a layered container: for each bottom bar, the water held above it is bounded by the nearest taller bars on both sides. By processing in a single pass from left to right, we handle nested valleys automatically.

### Comparing with Other Approaches

- **Two Pointer (O(n), O(1))**: Uses left and right pointers with max tracking. More space-efficient but less intuitive.
- **Dynamic Programming (O(n), O(n))**: Precomputes left and right max arrays. Simple but uses extra space.
- **Monotonic Stack (O(n), O(n))**: Processes in one pass, naturally handles complex valley shapes. Good for understanding the structure.

### Edge Cases

- **Empty or single element**: No water can be trapped, result is 0.
- **Non-decreasing heights**: Water flows off to the right, none trapped.
- **Non-increasing heights**: Water flows off to the left, none trapped.
- **All equal heights**: No valleys, no water trapped.
- **Two tall walls with a deep valley**: Maximum water is trapped.

import java.util.Stack;

class Solution {
    public int trap(int[] heights) {
        int n = heights.length;
        int totalWater = 0;
        Stack<Integer> stack = new Stack<>();  // Monotonic decreasing stack of indices

        // Process each bar from left to right
        for (int i = 0; i < n; i++) {
            // While the current bar is taller than the top of the stack
            while (!stack.isEmpty() && heights[i] > heights[stack.peek()]) {
                int bottom = stack.pop();      // The valley bottom
                if (stack.isEmpty()) break;    // No left boundary

                int left = stack.peek();        // Left boundary index
                int width = i - left - 1;       // Distance between boundaries
                int height = Math.min(heights[left], heights[i]) - heights[bottom];
                totalWater += width * height;
            }
            stack.push(i);  // Push current index onto stack
        }

        return totalWater;
    }
}

fn trap(heights: Vec<i32>) -> i32 {
    let n = heights.len();
    let mut total_water = 0;

            let height = heights[left].min(heights[i]) - heights[bottom];

    }

#include <stdlib.h>

int trap(int* heights, int n) {
    int totalWater = 0;
    int* stack = (int*)malloc(n * sizeof(int));  // Monotonic decreasing stack of indices
    int top = -1;  // Stack pointer, -1 means empty

    // Process each bar from left to right
    for (int i = 0; i < n; i++) {
        // While the current bar is taller than the top of the stack
        while (top >= 0 && heights[i] > heights[stack[top]]) {
            int bottom = stack[top--];  // The valley bottom
            if (top < 0) break;  // No left boundary

            int left = stack[top];  // Left boundary index
            int width = i - left - 1;  // Distance between boundaries
            int height = (heights[left] < heights[i] ? heights[left] : heights[i]) - heights[bottom];
            totalWater += width * height;
        }
        stack[++top] = i;  // Push current index onto stack
    }

    free(stack);
    return totalWater;
}

## Key Takeaways

- **Monotonic decreasing stack** naturally finds "valleys" between taller boundary bars
- The stack processes **layered water volumes** in a single pass — each popped bar represents the bottom of a valley
- The water volume formula is: `width * min(leftBound, rightBound) - bottomHeight`
- Each element is **pushed and popped at most once**, giving O(n) time complexity
- This approach elegantly handles nested valleys and complex elevation profiles
