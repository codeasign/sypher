# Sliding Window Maximum — Solution

## Problem Overview

Given an array `nums` and a sliding window of size `k`, return the maximum value in each window as the window moves from left to right by one position each step.

## Brute Force Approach

**Idea:** For each window position, scan all k elements to find the maximum.

**Algorithm:**
1. Initialize `result = []`
2. For each starting index `i` from 0 to n-k:
   - Scan `nums[i..i+k-1]` to find the maximum
   - Append the maximum to `result`
3. Return `result`

**Complexity Analysis:**
- **Time:** O(n * k) — for each of n-k+1 windows, we scan k elements
- **Space:** O(n-k+1) — the output array

**Why this is suboptimal:** For n = 10^5 and k = 5 * 10^4, O(n * k) = 5 * 10^9 operations, which is far too slow.

## Optimal Approach: Monotonic Deque (Double-Ended Queue)

**Idea:** Use a deque to maintain indices of potential maximum elements in the current window. The deque is kept in decreasing order of values (monotonic decreasing). The front of the deque always holds the index of the maximum element in the current window.

**Algorithm:**
1. Initialize an empty deque and an empty result array
2. For each index `i` from 0 to n-1:
   - **Remove out-of-window indices**: If the front of the deque is `<= i - k`, pop it from the front
   - **Maintain monotonic decreasing order**: While the deque is not empty and `nums[i] > nums[deque.back()]`, pop from the back
   - **Add current index**: Push `i` to the back of the deque
   - **Record maximum**: If `i >= k - 1` (the first window is complete), the front of the deque is the maximum for the current window
3. Return `result`

**Why This Works:** The deque maintains a list of indices in decreasing order of their values. When a new element arrives, all smaller elements in the deque are removed from the back — they can never be the maximum for any future window because the new element is larger and to the right (so it will outlast them). The front of the deque is always the maximum for the current window, and we remove expired indices from the front.

**Complexity Analysis:**
- **Time:** O(n) — each index is added and removed from the deque at most once
- **Space:** O(k) — the deque holds at most k elements at any time

### Why the Deque Works

- **Front of deque** = index of the maximum element in the current window
- **Back of deque** = index of the smallest element that could become the maximum in a future window
- When a new element is larger than the back, we remove the back because it will never be the maximum for any window that includes the new element (the new element is larger and to the right)
- When a window slides, we remove the front if it is out of bounds

### Monotonic Queue vs. Monotonic Stack

A monotonic deque (or "monotonic queue") is similar to a monotonic stack but allows removal from both ends:
- **Stack**: LIFO, used when the pattern is "next greater element" — you only need to compare with the most recent element
- **Deque**: Double-ended, used when the pattern is "sliding window" — you need to remove elements from both ends as the window slides

### Edge Cases

- **k = 1**: Each element is its own window maximum. The deque holds one element at a time.
- **k = n**: Only one window. The deque processes all elements and the front is the maximum.
- **Decreasing array**: The deque holds only the current element (all previous are larger, so they stay in front until out of window).
- **Increasing array**: The deque holds only the current element (all previous are smaller, so they are popped from the back).
- **All equal values**: The deque grows to size k, then each element is replaced as the window slides.

```java
import java.util.ArrayDeque;
import java.util.Deque;

class Solution {
    public int[] maxSlidingWindow(int[] nums, int k) {
        int n = nums.length;
        if (n == 0 || k == 0) return new int[0];

        int[] result = new int[n - k + 1];
        Deque<Integer> deque = new ArrayDeque<>();  // Monotonic decreasing deque of indices

        // Process each element
        for (int i = 0; i < n; i++) {
            // Step 1: Remove indices that are out of the current window
            if (!deque.isEmpty() && deque.peekFirst() <= i - k) {
                deque.pollFirst();
            }

            // Step 2: Maintain monotonic decreasing order — remove smaller elements
            while (!deque.isEmpty() && nums[i] > nums[deque.peekLast()]) {
                deque.pollLast();
            }

            // Step 3: Add current index
            deque.offerLast(i);

            // Step 4: Record the maximum when the first window is complete
            if (i >= k - 1) {
                result[i - k + 1] = nums[deque.peekFirst()];
            }
        }

        return result;
    }
}
```

```rust
use std::collections::VecDeque;

fn max_sliding_window(nums: Vec<i32>, k: usize) -> Vec<i32> {
    let n = nums.len();
    if n == 0 || k == 0 {
        return vec![];
    }

    let mut result = Vec::new();
    let mut deque: VecDeque<usize> = VecDeque::new();  // Monotonic decreasing deque of indices

    // Process each element
    for i in 0..n {
        // Step 1: Remove indices that are out of the current window
        if let Some(&front) = deque.front() {
            if front <= i.saturating_sub(k) {
                deque.pop_front();
            }
        }

        // Step 2: Maintain monotonic decreasing order
        while let Some(&back) = deque.back() {
            if nums[i] <= nums[back] {
                break;
            }
            deque.pop_back();
        }

        // Step 3: Add current index
        deque.push_back(i);

        // Step 4: Record the maximum when the first window is complete
        if i >= k - 1 {
            result.push(nums[deque[0]]);
        }
    }

    result
}
```

```c
#include <stdlib.h>

int* maxSlidingWindow(int* nums, int n, int k, int* returnSize) {
    if (n == 0 || k == 0) {
        *returnSize = 0;
        return NULL;
    }

    *returnSize = n - k + 1;
    int* result = (int*)malloc((*returnSize) * sizeof(int));
    int* deque = (int*)malloc(n * sizeof(int));  // Monotonic decreasing deque of indices
    int front = 0, back = 0;  // Deque pointers (front < back means non-empty)

    // Process each element
    int ri = 0;
    for (int i = 0; i < n; i++) {
        // Step 1: Remove indices that are out of the current window
        if (front < back && deque[front] <= i - k) {
            front++;
        }

        // Step 2: Maintain monotonic decreasing order
        while (front < back && nums[i] > nums[deque[back - 1]]) {
            back--;
        }

        // Step 3: Add current index
        deque[back++] = i;

        // Step 4: Record the maximum when the first window is complete
        if (i >= k - 1) {
            result[ri++] = nums[deque[front]];
        }
    }

    free(deque);
    return result;
}
```

```csharp
using System.Collections.Generic;

class Solution {
    public int[] MaxSlidingWindow(int[] nums, int k) {
        int n = nums.Length;
        if (n == 0 || k == 0) return new int[0];

        int[] result = new int[n - k + 1];
        LinkedList<int> deque = new LinkedList<int>();  // Monotonic decreasing deque of indices

        // Process each element
        for (int i = 0; i < n; i++) {
            // Step 1: Remove indices that are out of the current window
            if (deque.Count > 0 && deque.First.Value <= i - k) {
                deque.RemoveFirst();
            }

            // Step 2: Maintain monotonic decreasing order
            while (deque.Count > 0 && nums[i] > nums[deque.Last.Value]) {
                deque.RemoveLast();
            }

            // Step 3: Add current index
            deque.AddLast(i);

            // Step 4: Record the maximum when the first window is complete
            if (i >= k - 1) {
                result[i - k + 1] = nums[deque.First.Value];
            }
        }

        return result;
    }
}
```

## Key Takeaways

- **Monotonic deque (double-ended queue)** is the optimal data structure for sliding window maximum problems
- The deque maintains a **monotonic decreasing order** of values — the front is always the maximum
- Each element is **added and removed at most once**, giving O(n) time complexity
- **Remove from front** when indices are out of the window; **remove from back** when smaller values are superseded by a larger new value
- This pattern is distinct from the monotonic stack — use a deque when you need to remove from both ends (sliding window), and a stack when you only need to remove from one end (next greater element)
