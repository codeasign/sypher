# Largest Rectangle in Histogram — Solution

## Problem Overview

Given an array `heights` representing bar heights in a histogram (each bar has width 1), find the largest rectangle area that can be formed entirely within the histogram. The rectangle spans contiguous bars and its height is limited by the shortest bar in that segment.

## Brute Force Approach

**Idea:** For each bar, consider it as the shortest bar in a rectangle and expand left and right as far as possible.

**Algorithm:**
1. Initialize `maxArea = 0`
2. For each index `i` from 0 to n-1:
   - Expand left from `i` while `heights[left] >= heights[i]`, recording the leftmost index
   - Expand right from `i` while `heights[right] >= heights[i]`, recording the rightmost index
   - Compute `width = right - left + 1` and `area = heights[i] * width`
   - Update `maxArea = max(maxArea, area)`
3. Return `maxArea`

**Complexity Analysis:**
- **Time:** O(n²) — for each of n bars, we scan up to n bars left and right
- **Space:** O(1) — only a few variables

**Why this is suboptimal:** For n = 10^5, O(n²) is impractical. The monotonic stack approach reduces this to O(n) by efficiently computing the left and right boundaries for all bars in a single pass.

## Optimal Approach: Monotonic Increasing Stack

**Idea:** Use a monotonic increasing stack to find the previous smaller element (left boundary) and next smaller element (right boundary) for each bar in a single pass. When a bar is popped from the stack, we know its height and its exact boundaries.

**Algorithm:**
1. Append a sentinel height of 0 to the end of `heights` to force all bars to be processed
2. Initialize `maxArea = 0` and an empty stack
3. For each index `i` from 0 to n (including the sentinel):
   - While the stack is not empty and `heights[i] < heights[stack.top()]`:
     - Pop `height = heights[stack.pop()]`
     - The right boundary is `i` (the current index is the first bar shorter than height)
     - The left boundary is `stack.top()` if the stack is not empty, or `-1` (the previous bar shorter than height)
     - Compute `width = i - leftBoundary - 1`
     - Compute `area = height * width`
     - Update `maxArea = max(maxArea, area)`
   - Push `i` onto the stack
4. Return `maxArea`

**Why This Works:** The stack maintains indices of bars in increasing order of height. When we encounter a bar shorter than the top of the stack, we know the top bar's right boundary (the current index). The left boundary is the new top of the stack (the previous shorter bar). This gives us the exact width for a rectangle at the popped bar's height.

**Complexity Analysis:**
- **Time:** O(n) — each index is pushed and popped at most once
- **Space:** O(n) — the stack holds up to n indices

## Deep Dive: How the Monotonic Stack Works

### Visual Walkthrough

For `heights = [2, 1, 5, 6, 2, 3]`, let's trace the algorithm (with sentinel 0 at index 6):

```

i=3, h=6: h >= heights[stack.top()]=5, push 3. stack = [1, 2, 3]

i=4, h=2: h < heights[stack.top()]=6, pop index 3 (height=6)

           h < heights[stack.top()]=5, pop index 2 (height=5)

i=6, h=0: h < heights[stack.top()]=3, pop index 5 (height=3)
           left = 4 (stack.top() = 4), right = 6

           h < heights[stack.top()]=2, pop index 4 (height=2)
           left = 1 (stack.top() = 1), right = 6
           width = 6 - 1 - 1 = 4, area = 2*4 = 8, maxArea = 10
           h < heights[stack.top()]=1, pop index 1 (height=1)
           left = -1 (stack empty), right = 6
           width = 6 - (-1) - 1 = 6, area = 1*6 = 6, maxArea = 10
```

### Understanding the Boundary Calculation

```
For bar at index i with height h:
  left_boundary = index of the nearest bar to the left with height < h
                  (or -1 if none exists)
  right_boundary = index of the nearest bar to the right with height < h
                   (or n if none exists)
  width = right_boundary - left_boundary - 1
  area = h * width
```

The stack naturally maintains the left boundaries because elements below the top are always shorter and to the left. When we pop a bar, the new top (or -1) is the left boundary, and the current index is the right boundary.

### Why the Sentinel Value?

Adding a 0-height bar at the end forces all remaining bars in the stack to be processed. Without it, bars that extend to the end of the array would never be popped and their areas would be missed. The sentinel guarantees every bar is evaluated.

### Edge Cases to Consider

- **Single bar**: Area is the bar's height times width 1.
- **All bars same height**: The largest rectangle spans the entire histogram: `height * n`.
- **Strictly increasing heights**: Each bar's rectangle extends left to the beginning. The largest rectangle uses all bars at the shortest height.
- **Strictly decreasing heights**: Same as increasing but mirrored.
- **Zero-height bars**: Any rectangle including a zero-height bar has area 0, so they effectively serve as boundaries.
- **Large values**: Heights up to 10^4 and n up to 10^5 mean the area can be up to 10^9, which fits in a 32-bit signed integer.

```java
import java.util.Stack;

class Solution {
    public int largestRectangleArea(int[] heights) {
        // Step 1: Create an extended array with a sentinel 0 at the end
        int n = heights.length;
        int[] h = new int[n + 1];
        System.arraycopy(heights, 0, h, 0, n);
        h[n] = 0;  // Sentinel — forces all elements out of the stack

        int maxArea = 0;
        Stack<Integer> stack = new Stack<>();  // Monotonic increasing stack of indices

        // Step 2: Process each bar (including the sentinel)
        for (int i = 0; i <= n; i++) {
            // While the current bar is shorter than the top of the stack
            while (!stack.isEmpty() && h[i] < h[stack.peek()]) {
                int height = h[stack.pop()];   // Height of the bar being evaluated
                // Left boundary: the bar just below the popped one in the stack (or -1)
                int left = stack.isEmpty() ? -1 : stack.peek();
                // Right boundary: the current index (first bar shorter than height)
                int width = i - left - 1;
                maxArea = Math.max(maxArea, height * width);
            }
            stack.push(i);  // Push current index onto stack
        }

        return maxArea;
    }
}
```

```rust
fn largest_rectangle_area(heights: Vec<i32>) -> i32 {
    // Step 1: Add a sentinel 0 at the end to force processing of all bars
    let mut h = heights.clone();
    h.push(0);  // Sentinel

    let n = h.len();
    let mut max_area = 0;
    let mut stack: Vec<usize> = Vec::new();  // Monotonic increasing stack of indices

    // Step 2: Process each bar (including the sentinel)
    for i in 0..n {
        // While the current bar is shorter than the top of the stack
        while let Some(&top) = stack.last() {
            if h[i] >= h[top] {
                break;
            }
            let height = h[stack.pop().unwrap()];  // Height of the bar being evaluated
            // Left boundary: the bar just below in the stack (or -1 as usize)
            let left = if stack.is_empty() {
                usize::MAX  // sentinel for -1
            } else {
                stack[stack.len() - 1]
            };
            // Right boundary: the current index
            let width = if left == usize::MAX {
                i
            } else {
                i - left - 1
            };
            max_area = max_area.max(height * width as i32);
        }
        stack.push(i);  // Push current index onto stack
    }

    max_area
}
```

```c
#include <stdlib.h>

int largestRectangleArea(int* heights, int n) {
    // Step 1: Allocate extended array with sentinel 0 at the end
    int* h = (int*)malloc((n + 1) * sizeof(int));
    for (int i = 0; i < n; i++) h[i] = heights[i];
    h[n] = 0;  // Sentinel

    int* stack = (int*)malloc((n + 1) * sizeof(int));
    int top = -1;  // Stack pointer, -1 means empty
    int maxArea = 0;

    // Step 2: Process each bar (including the sentinel)
    for (int i = 0; i <= n; i++) {
        // While the current bar is shorter than the top of the stack
        while (top >= 0 && h[i] < h[stack[top]]) {
            int height = h[stack[top--]];  // Height of the bar being evaluated
            // Left boundary: the bar just below in the stack (or -1)
            int left = (top >= 0) ? stack[top] : -1;
            // Right boundary: the current index
            int width = i - left - 1;
            if (height * width > maxArea) maxArea = height * width;
        }
        stack[++top] = i;  // Push current index onto stack
    }

    free(stack);
    free(h);
    return maxArea;
}
```

## Key Takeaways

- **Monotonic increasing stack** is the key pattern for finding the "next smaller element" on both sides of each bar
- The **sentinel value** (height 0 at the end) is a clever trick to avoid post-processing remaining elements
- The **boundary calculation** `width = rightBoundary - leftBoundary - 1` correctly captures the span of bars with height >= current bar
- Each element is **pushed and popped only once**, giving O(n) time — optimal for this problem
- Understanding the stack's role in maintaining **increasing heights** is essential: when the order breaks, we process the taller bars with their known boundaries
