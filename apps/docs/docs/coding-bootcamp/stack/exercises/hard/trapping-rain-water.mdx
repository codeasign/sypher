# Trapping Rain Water

**Difficulty:** Hard

Given `n` non-negative integers representing an elevation map where the width of each bar is 1, compute how much water it can trap after raining.

## Example 1

**Input:** `heights = [0, 1, 0, 2, 1, 0, 1, 3, 2, 1, 2, 1]`  
**Output:** `6`  
**Explanation:** The elevation map is shown in the diagram above. Six units of rainwater (▓) are trapped between the bars.

## Example 2

**Input:** `heights = [4, 2, 0, 3, 2, 5]`  
**Output:** `9`  

## Example 3

**Input:** `heights = [1, 0, 1]`  
**Output:** `1`  

## Input Specification

The first line contains an integer `n` (the length of the array). The second line contains `n` space-separated integers representing the bar heights.

## Output Specification

Print a single integer — the total amount of rainwater that can be trapped.

## Constraints

- `1 <= heights.length <= 10^5`
- `0 <= heights[i] <= 10^4`

## Hints

<details>
<summary>Hint 1</summary>

Water trapped above a bar is determined by the minimum of the tallest bar to its left and the tallest bar to its right, minus the bar's own height.
</details>

<details>
<summary>Hint 2</summary>

A monotonic decreasing stack can help you find the "valleys" between taller bars. When you encounter a bar taller than the top of the stack, you can compute trapped water using the popped bar as the bottom of a valley.
</details>

<details>
<summary>Hint 3</summary>

The stack approach processes the array in one pass: when a higher bar appears, pop the lower bar as the bottom, use the new top as the left boundary, and the current bar as the right boundary.
</details>

</ProblemEditor>
