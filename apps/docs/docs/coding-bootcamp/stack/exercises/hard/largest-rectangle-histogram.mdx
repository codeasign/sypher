# Largest Rectangle in Histogram

**Difficulty:** Hard

Given an array of integers `heights` representing the heights of bars in a histogram (where the width of each bar is 1), return the area of the largest rectangle that can be formed entirely within the histogram.

The rectangle must be composed of contiguous bars and span the full height of the shortest bar in that segment.

## Example 1

**Input:** `heights = [2, 1, 5, 6, 2, 3]`  
**Output:** `10`  
**Explanation:** The largest rectangle has an area of 10, formed by bars at indices 2 and 3 (heights 5 and 6), spanning a width of 2 at a height of 5.

## Example 2

**Input:** `heights = [2, 4]`  
**Output:** `4`  
**Explanation:** The entire histogram forms a rectangle of width 2 and height 2, or the bar of height 4 alone gives area 4.

## Example 3

**Input:** `heights = [2, 1, 2]`  
**Output:** `3`  
**Explanation:** The largest rectangle spans all three bars at a height of 1 (area = 3). The smaller rectangle of height 2 at index 0 or 2 gives area 2.

## Input Specification

The first line contains an integer `n` (the length of the array). The second line contains `n` space-separated integers representing the histogram bar heights.

## Output Specification

Print a single integer — the maximum rectangle area.

## Constraints

- `1 <= heights.length <= 10^5`
- `0 <= heights[i] <= 10^4`

## Hints

<details>
<summary>Hint 1</summary>

For each bar, think about the widest rectangle that uses that bar's height. The rectangle extends left and right until it hits a bar shorter than the current bar.
</details>

<details>
<summary>Hint 2</summary>

A monotonic increasing stack can help find the "previous smaller element" and "next smaller element" for each bar in O(n) time.
</details>

<details>
<summary>Hint 3</summary>

Use a sentinel value (height 0) at the end of the array to force all remaining bars in the stack to be processed.
</details>
