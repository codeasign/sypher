# Permutation Sequence

The set `[1, 2, 3, ..., n]` contains a total of `n!` unique permutations.

By listing and labeling all permutations in order (lexicographically), you get a sequence. Given `n` and `k`, return the **k-th** permutation (1-indexed).

## Examples

**Example 1:**
```
Input: n = 3, k = 3
Output: "213"
```
The permutations of [1, 2, 3] in order are:
1. "123"
2. "132"
3. "213"
4. "231"
5. "312"
6. "321"

**Example 2:**
```
Input: n = 4, k = 9
Output: "2314"
```

**Example 3:**
```
Input: n = 3, k = 1
Output: "123"
```

## Constraints

- `1 <= n <= 9`
- `1 <= k <= n!`

## Hints

<details>
<summary>Hint 1</summary>

The key insight is the factorial number system. For a given position, there are (n-1)! permutations that start with each digit. Use division to determine which digit comes next.

</details>

<details>
<summary>Hint 2</summary>

Keep a list of available digits. For each position, compute the index as (k-1) / (remaining factorial) and select that digit. Update k to k % (remaining factorial) for the next position.

</details>

<details>
<summary>Hint 3</summary>

Precompute factorials to avoid recomputing them repeatedly. Since n `<=` 9, factorials fit in 32-bit integers.

</details>

</ProblemEditor>
