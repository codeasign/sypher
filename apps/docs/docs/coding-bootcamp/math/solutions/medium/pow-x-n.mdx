# Pow(x, n) — Solution

## Approach

Use **exponentiation by squaring**. The idea is to repeatedly square `x` and halve `n`. When `n` is odd, multiply the result by the current `x`. This reduces the time from O(n) to O(log n).

For negative exponents, compute `x^(-n) = 1 / x^n`. We handle this by negating `n` and dividing 1 by the result.

**Time Complexity:** O(log n)
**Space Complexity:** O(1)

## Steps

1. Handle the edge case `n = 0` → return 1
2. Convert to long to handle `n = -2³¹` safely
3. If `n` is negative, convert `x = 1/x` and `n = -n`
4. While `n > 0`:
   - If `n` is odd, multiply result by `x`
   - Square `x` (x = x * x)
   - Halve `n` (n = n / 2)
5. Return result

## Visual

```
Compute 3^13:

n = 13 (binary: 1101)

Iteration 1: n=13 (odd)  → result = 1 * 3 = 3,  x = 3² = 9,   n = 6
Iteration 2: n=6  (even) → result = 3,          x = 9² = 81,  n = 3
Iteration 3: n=3  (odd)  → result = 3 * 81 = 243, x = 81² = 6561, n = 1
Iteration 4: n=1  (odd)  → result = 243 * 6561 = 1594323, x = ... n = 0

Result: 1594323 ✅
```

def my_pow(x: float, n: int) -> float:
    # Handle zero exponent
    if n == 0:
        return 1.0

    # Use long to handle -2^31 safely
    exp = abs(n)

    result = 1.0
    while exp > 0:
        # If exponent is odd, multiply result by current x
        if exp & 1:
            result *= x
        # Square x and halve exponent
        x *= x
        exp >>= 1

    # Handle negative exponent
    return result if n > 0 else 1.0 / result

## Key Takeaways

- Exponentiation by squaring reduces O(n) to O(log n) — a massive improvement for large exponents
- The bit check `exp & 1` tests if the exponent is odd
- Right shift `exp >>= 1` divides the exponent by 2
- Always use a 64-bit integer to handle the negation of `INT_MIN` (-2³¹)
- For negative exponents, compute the positive power first and take the reciprocal
