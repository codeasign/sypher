# Happy Number — Solution

## Approach

We simulate the process of replacing a number with the sum of the squares of its digits. The key insight is that unhappy numbers enter a cycle. We can detect this cycle using Floyd's cycle detection algorithm (tortoise and hare) to avoid extra space, or use a HashSet to track seen numbers.

**Time Complexity:** O(log n) — each step reduces the number of digits significantly
**Space Complexity:** O(1) with Floyd's algorithm, O(log n) with HashSet

## Steps

1. Define a helper function to compute the sum of squares of digits
2. Use two pointers: slow (moves one step) and fast (moves two steps)
3. If slow and fast meet and are not 1, a cycle is detected → return false
4. If either reaches 1, return true

## Visual

```
n = 19

Step 1: 1² + 9² = 82
Step 2: 8² + 2² = 68
Step 3: 6² + 8² = 100
Step 4: 1² + 0² + 0² = 1  → happy!

n = 2

Step 1: 2² = 4
Step 2: 4² = 16
Step 3: 1² + 6² = 37
Step 4: 3² + 7² = 58
Step 5: 5² + 8² = 89
Step 6: 8² + 9² = 145
Step 7: 1² + 4² + 5² = 42
Step 8: 4² + 2² = 20
Step 9: 2² + 0² = 4  → back to 4, cycle detected → not happy
```

```python
def is_happy(n: int) -> bool:
    def sum_squares(num: int) -> int:
        total = 0
        while num > 0:
            digit = num % 10
            total += digit * digit
            num //= 10
        return total

    # Floyd's cycle detection
    slow = n
    fast = sum_squares(n)

    while fast != 1 and slow != fast:
        slow = sum_squares(slow)           # move one step
        fast = sum_squares(sum_squares(fast))  # move two steps

    return fast == 1
```

## Key Takeaways

- The sum of squares of digits of any number is bounded (max 243 for 3-digit numbers)
- Floyd's cycle detection gives O(1) space complexity
- Alternative: using a HashSet to detect cycles is simpler but uses O(log n) space
- The number of steps is bounded by the cycle length, which is small
