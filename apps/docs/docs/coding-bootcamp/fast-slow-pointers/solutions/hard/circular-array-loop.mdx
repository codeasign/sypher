# Circular Array Loop — Solution

## Problem Overview

You are given a circular array `nums` of non-zero integers. If `nums[i]` is positive, move forward `nums[i]` steps; if negative, move backward `nums[i]` steps. The array is circular, so moving past the end wraps to the beginning, and moving before the start wraps to the end. Determine if there is a cycle of length at least 2 where all elements in the cycle have the same direction (all positive or all negative).

## Brute Force Approach

**Idea:** For each starting index, simulate the traversal and track visited indices using a hash set. Check if a cycle forms and validate the cycle length and direction.

**Algorithm:**
1. For each index `i` from 0 to n-1:
   - Use a hash set `visited` to track indices in the current path
   - Start at index `i` and follow the movement rules
   - Track the direction (sign) of the first element
   - If we revisit an index, check if the cycle length > 1 and all elements have the same direction
   - If the direction changes or we hit a self-loop (length 1), break and try the next start

**Complexity Analysis:**
- **Time:** O(n²) — for each starting index, we may traverse up to n elements
- **Space:** O(n) — the hash set for each starting index

**Why this could be better:** The brute force approach is too slow for large inputs. The optimal approach uses Floyd's cycle detection to check each element in O(1) space and uses a visited marking scheme to avoid redundant work.

## Optimal Approach: Fast and Slow Pointers with Direction Check

**Idea:** Use Floyd's cycle detection on each unvisited element. For each starting index, run slow and fast pointers while enforcing that all elements in the path have the same direction. Mark visited elements to avoid reprocessing. When slow and fast meet, verify the cycle has length > 1 (not a self-loop).

**Algorithm:**
1. Define a helper `next(i)` that returns the next index considering circular wrapping: `((i + nums[i]) % n + n) % n`
2. Initialize a `visited` array of booleans (all false)
3. For each index `i` from 0 to n-1:
   - If already visited, skip
   - Set `slow = i`, `fast = i`
   - Record the direction `dir = nums[i] > 0`
   - While true:
     - Mark `slow` as visited
     - If `nums[slow]` has a different direction than `dir`, break (invalid path)
     - Advance `slow` by one step
     - If `nums[fast]` has a different direction, break
     - Advance `fast` by one step
     - If `nums[fast]` has a different direction, break
     - Advance `fast` by one step (second step)
     - If `slow == fast`:
       - If `slow != next(slow)`, we found a valid cycle of length > 1 — return `true`
       - Otherwise, it's a self-loop (length 1) — break
4. Return `false` (no valid cycle found)

**Why This Works:** The visited array ensures each element is processed at most once. The direction check ensures all elements in the cycle move the same way. The self-loop check ensures the cycle length is at least 2. Together, these guarantee we only return `true` for valid cycles.

**Complexity Analysis:**
- **Time:** O(n) — each element is visited at most once
- **Space:** O(n) — the visited array

def circularArrayLoop(nums: list[int]) -> bool:
    n = len(nums)
    visited = [0] * n

    def next_idx(i):
        # Compute next index with proper circular wrapping
        return (i + nums[i]) % n

    for i in range(n):
        if visited[i]:
            continue

        slow = fast = i
        direction = nums[i] > 0  # True = forward, False = backward

        while True:
            visited[slow] = 1

            # Check direction for slow pointer
            if (nums[slow] > 0) != direction:
                break
            slow = next_idx(slow)

            # Check direction for fast pointer (first step)
            if (nums[fast] > 0) != direction:
                break
            fast = next_idx(fast)

            # Check direction for fast pointer (second step)
            if (nums[fast] > 0) != direction:
                break
            fast = next_idx(fast)

            if slow == fast:
                # Verify cycle length > 1 (not a self-loop)
                if slow != next_idx(slow):
                    return True
                break

    return False

class Solution {
    private int next(int[] nums, int i) {
        int n = nums.length;
        return ((i + nums[i]) % n + n) % n;
    }

    public boolean circularArrayLoop(int[] nums) {
        int n = nums.length;
        boolean[] visited = new boolean[n];

        for (int i = 0; i < n; i++) {
            if (visited[i]) continue;

            int slow = i, fast = i;
            boolean dir = nums[i] > 0;

            while (true) {
                visited[slow] = true;

                if ((nums[slow] > 0) != dir) break;
                slow = next(nums, slow);

                if ((nums[fast] > 0) != dir) break;
                fast = next(nums, fast);

                if ((nums[fast] > 0) != dir) break;
                fast = next(nums, fast);

                if (slow == fast) {
                    if (slow != next(nums, slow)) return true;
                    break;
                }
            }
        }
        return false;
    }
}

fn circular_array_loop(nums: &[i32]) -> bool {
    let n = nums.len();
    let mut visited = vec![false; n];
    let next = |i: usize| -> usize {
        ((i as i32 + nums[i]) % n as i32 + n as i32) as usize % n

        if visited[i] { continue; }

        let mut slow = i;
        let mut fast = i;
        let dir = nums[i] > 0;

            visited[slow] = true;

            slow = next(slow);

            if (nums[fast] > 0) != dir { break; }
            fast = next(fast);

            if (nums[fast] > 0) != dir { break; }
            fast = next(fast);

            if slow == fast {
                if slow != next(slow) { return true; }
                break;
            }
        }
    }
    false
}

## Key Takeaways

- **Circular array loops** extend Floyd's algorithm with direction checking
- Both pointers must verify direction consistency at every step
- Self-loops (cycles of length 1) must be explicitly rejected
- A visited array avoids redundant processing of previously explored elements
- The time complexity is O(n) because each element is marked visited at most once
