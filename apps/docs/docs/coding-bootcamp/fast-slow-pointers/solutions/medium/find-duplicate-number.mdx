# Find the Duplicate Number — Solution

## Problem Overview

Given an array of `n + 1` integers where each integer is in the range `[1, n]`, there is exactly one repeated number. Find the duplicate without modifying the array and using only O(1) extra space.

## Brute Force Approach

**Idea:** Use a hash set to track seen numbers. When we encounter a number already in the set, it is the duplicate.

**Algorithm:**
1. Initialize an empty hash set `seen`
2. For each number `num` in `nums`:
   - If `num` is in `seen`, return `num`
   - Add `num` to `seen`
3. This point is never reached (the problem guarantees a duplicate)

**Complexity Analysis:**
- **Time:** O(n) — one pass through the array
- **Space:** O(n) — the hash set may store up to n elements

**Why this could be better:** The hash set approach is simple and O(n) time, but uses O(n) space. The problem explicitly requires O(1) extra space.

## Alternative: Sorting

**Idea:** Sort the array, then scan for adjacent duplicates.

**Algorithm:**
1. Sort `nums` in ascending order
2. For each adjacent pair `nums[i]` and `nums[i+1]`:
   - If they are equal, return `nums[i]`

**Complexity Analysis:**
- **Time:** O(n log n) — dominated by sorting
- **Space:** O(1) or O(n) depending on sorting algorithm, but it modifies the array

**Why this violates constraints:** The problem states "without modifying the array," so sorting is not allowed.

## Optimal Approach: Floyd's Cycle Detection

**Idea:** Treat the array as a functional graph (a linked list) where each index `i` points to `nums[i]`. Since `nums[i]` is in `[1, n]` and there are `n + 1` elements, the values are valid indices, forming a linked-list-like structure. The duplicate number creates a cycle in this graph. Floyd's algorithm finds the cycle entry point, which is the duplicate.

**Algorithm:**
1. Phase 1 — Detect cycle:
   - Initialize `slow = nums[0]`, `fast = nums[0]`
   - Move `slow` one step: `slow = nums[slow]`
   - Move `fast` two steps: `fast = nums[nums[fast]]`
   - Repeat until `slow == fast` (they meet inside the cycle)
2. Phase 2 — Find cycle entry (the duplicate):
   - Reset `slow = nums[0]`
   - Move both pointers one step at a time: `slow = nums[slow]`, `fast = nums[fast]`
   - When they meet again, the meeting point is the cycle entry, which is the duplicate number

**Why This Works:** The array of length `n + 1` with values in `[1, n]` defines a function `f(i) = nums[i]` that maps each index to a value. Since there are `n + 1` indices but only `n` possible values, by the pigeonhole principle at least one value appears twice. This creates a cycle in the functional graph. The entry point of the cycle is the value that is pointed to by two different indices — the duplicate number.

**Complexity Analysis:**
- **Time:** O(n) — each pointer moves at most n steps
- **Space:** O(1) — only two pointer variables

def findDuplicate(nums: list[int]) -> int:
    # Phase 1: Detect cycle using Floyd's algorithm
    slow = nums[0]
    fast = nums[0]
    while True:
        slow = nums[slow]              # Move one step
        fast = nums[nums[fast]]        # Move two steps
        if slow == fast:               # They met — cycle detected
            break

    # Phase 2: Find the cycle entry point (the duplicate)
    slow = nums[0]
    while slow != fast:
        slow = nums[slow]              # Move one step from start
        fast = nums[fast]              # Move one step from meeting point

    return slow

## Key Takeaways

- Treating an array as a **functional graph** (linked list) is a powerful insight
- The duplicate number problem is isomorphic to finding the cycle start in a linked list
- Floyd's algorithm works in O(n) time and O(1) space — no array modification needed
- Phase 1 detects the cycle; Phase 2 finds the entry point (the duplicate)
- This approach satisfies all constraints: constant space, no modification, O(n) time
