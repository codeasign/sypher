# Palindrome Linked List — Solution

## Problem Overview

Given the `head` of a singly linked list, determine whether it is a palindrome. A linked list is a palindrome if the sequence of values reads the same forwards and backwards.

Return `true` if the linked list is a palindrome, `false` otherwise.

## Brute Force Approach

**Idea:** Copy the linked list values into an array, then check whether the array is a palindrome.

**Algorithm:**
1. Traverse the list and copy each node's value into an array
2. Use two pointers (left and right) to check if the array reads the same forwards and backwards
3. If all pairs match, return `true`; otherwise, return `false`

**Complexity Analysis:**
- **Time:** O(n) — one pass to copy values, one pass to check
- **Space:** O(n) — the array stores all node values

**Why this could be better:** The array approach is simple and passes all test cases, but uses O(n) extra space. The optimal approach uses O(1) space by modifying the list in-place.

## Optimal Approach: Fast and Slow Pointers with List Reversal

**Idea:** Use the fast and slow pointer technique to find the middle of the list, reverse the second half in-place, then compare the first half with the reversed second half.

**Algorithm:**
1. **Find the middle** — Use slow and fast pointers. When fast reaches the end (or `null`), slow is at the middle (for even-length lists, it's at the start of the second half).
2. **Reverse the second half** — Starting from the slow pointer, reverse the linked list in-place.
3. **Compare both halves** — Walk two pointers simultaneously — one from the head, one from the reversed second half — comparing values at each step.
4. If all values match, return `true`; otherwise, return `false`.

For odd-length lists, the middle node does not need to be compared (it is its own mirror).

**Complexity Analysis:**
- **Time:** O(n) — each step is a single pass: find middle (O(n)), reverse half (O(n)), compare (O(n))
- **Space:** O(1) — only pointer variables, regardless of list size

## Deep Dive

The algorithm modifies the list in-place by reversing the second half. After the comparison, the list is left in a modified state. This is acceptable since the problem only requires a boolean answer and the modification is O(1) space.

Key insight: Finding the middle with the fast-slow pointer naturally places the slow pointer at the start of the second half for even-length lists (e.g., for 4 nodes, slow lands at index 2 — the third node — which is the second half). For odd-length lists (e.g., 5 nodes), slow lands at index 2 (the third node, the exact middle), so the second half starts from `slow.next`, skipping the middle node in the comparison.

class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next


def is_palindrome(head):
    if not head or not head.next:
        return True

    # Phase 1: Find the middle using fast and slow pointers
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next

    # Phase 2: Reverse the second half
    prev = None
    while slow:
        next_node = slow.next
        slow.next = prev
        prev = slow
        slow = next_node

    # Phase 3: Compare first half and reversed second half
    left, right = head, prev
    while right:
        if left.val != right.val:
            return False
        left = left.next
        right = right.next

    return True

fn is_palindrome(head: Option<Box<ListNode>>) -> bool {
    // Phase 1: Find length of the list
    let mut len = 0;
    let mut curr = &head;
    while let Some(node) = curr {
        len += 1;
        curr = &node.next;
    }

    if len <= 1 { return true; }

    // Phase 2: Collect first half values into a vector
    let mut vals = Vec::new();
    let mut curr = &head;
    for _ in 0..len / 2 {
        if let Some(node) = curr {
            vals.push(node.val);
            curr = &node.next;
        }
    }

    // Phase 3: Skip middle element for odd length, then compare
    let mut curr = &head;
    if len % 2 == 1 {
        for _ in 0..(len / 2 + 1) {
            if let Some(node) = curr {
                curr = &node.next;
            }
        }
    } else {
        for _ in 0..(len / 2) {
            if let Some(node) = curr {
                curr = &node.next;
            }
        }
    }

    for &v in vals.iter().rev() {
        if let Some(node) = curr {
            if node.val != v { return false; }
            curr = &node.next;
        }
    }
    true
}

## Key Takeaways

- **The fast-slow pointer technique** finds the middle of the linked list efficiently
- **Reversing the second half** in-place enables O(1) space comparison
- The algorithm uses three linear passes, each O(n), giving O(n) total time
- This is a classic combination of two patterns: fast-slow pointers and linked list reversal
- The order of operations matters: find middle first, then reverse from the middle
