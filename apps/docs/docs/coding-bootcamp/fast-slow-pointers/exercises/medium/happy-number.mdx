# Happy Number

Write an algorithm to determine if a number `n` is **happy**.

A happy number is defined by the following process:

1. Starting with any positive integer, replace the number by the sum of the squares of its digits.
2. Repeat the process until the number equals 1 (where it will stay), or it **loops endlessly in a cycle** that does not include 1.
3. Those numbers for which this process ends in 1 are happy.

Return `true` if `n` is a happy number, otherwise return `false`.

## Examples

**Example 1:**
```
Input: n = 19
Output: true
Explanation:
1² + 9² = 82
8² + 2² = 68
6² + 8² = 100
1² + 0² + 0² = 1
```

**Example 2:**
```
Input: n = 2
Output: false
```

## Constraints

- `1 <= n <= 2^31 - 1`

## Hints

<details>
<summary>Hint 1</summary>

The sequence of digit square sums will either reach 1 or enter a cycle. Use Floyd's cycle detection (fast and slow pointers) to detect the cycle efficiently.
</details>

<details>
<summary>Hint 2</summary>

The slow pointer computes sumSquares once per iteration; the fast pointer computes it twice. If slow == fast and fast != 1, there is a cycle.
</details>

</ProblemEditor>
