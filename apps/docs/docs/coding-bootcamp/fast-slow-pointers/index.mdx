# Fast & Slow Pointers

The **Fast & Slow Pointers** pattern (also known as the **Tortoise and Hare** algorithm) uses two pointers that move through a sequence at different speeds. The fast pointer moves two steps at a time while the slow pointer moves one step. This simple difference lets us detect cycles, find middle nodes, and solve a variety of problems in O(1) extra space.

## The Tortoise and Hare Algorithm

The core idea comes from Floyd's Cycle Detection algorithm:

1. **Slow pointer** moves one step per iteration
2. **Fast pointer** moves two steps per iteration
3. If there is a cycle, the fast pointer will eventually lap the slow pointer and they will meet
4. If there is no cycle, the fast pointer reaches the end

## Key Properties

- **Cycle detection**: If fast and slow meet, a cycle exists
- **Cycle entry point**: After detecting a cycle, reset one pointer to head and move both one step вҖ” they meet at the cycle start
- **Middle of list**: When fast reaches the end, slow is at the middle
- **N-th from end**: Move fast n steps ahead, then advance both вҖ” slow lands at the target

## Complexity

- **Time**: O(n)
- **Space**: O(1)

## Common Applications

- **Cycle detection** in linked lists and sequences
- **Finding the middle** of a linked list or array in one pass
- **Happy Number**: Detecting cycles in digit square sums
- **Find the duplicate number** in an array (Floyd's on the index-value graph)
- **Circular array loops**: Detecting cycles with direction constraints

## Detecting Cycle Entry Point

Once fast and slow meet, a cycle is confirmed. To find the cycle's start:

1. Reset one pointer to the head
2. Move both pointers one step at a time
3. The point where they meet is the cycle's entry

```
Head в”Җв”Җ> 1 в”Җв”Җ> 2 в”Җв”Җ> 3 в”Җв”Җ> 4 в”Җв”Җ> 5 в”Җв”Җ> 6
                          в–І           в”Ӯ
                          в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

## Finding the Middle Node

When the fast pointer reaches the end (or null), the slow pointer is at the middle:

```
Head в”Җв”Җ> 1 в”Җв”Җ> 2 в”Җв”Җ> 3 в”Җв”Җ> 4 в”Җв”Җ> 5 в”Җв”Җ> null
                          S           F
                         (mid)
```

## Worked Example: Happy Number

**Problem**: Starting from any positive integer, replace it with the sum of squares of its digits. Continue until the number equals 1 (happy) or loops endlessly (unhappy). Determine if a number is happy.

**Floyd's approach**:

- slow = sumSquares(n)
- fast = sumSquares(sumSquares(n))
- If slow == fast (and not 1), a cycle exists вҶ’ unhappy
- If slow == 1 вҶ’ happy

```
n = 19
  slow = sumSquares(19) = 1ВІ + 9ВІ = 82
  fast = sumSquares(82) = 64 + 4 = 68
  slow = sumSquares(82) = 68
  fast = sumSquares(68) = 100
  slow = sumSquares(68) = 100
  fast = sumSquares(1) = 1
  slow = sumSquares(100) = 1
  вҶ’ 19 is happy!
```
