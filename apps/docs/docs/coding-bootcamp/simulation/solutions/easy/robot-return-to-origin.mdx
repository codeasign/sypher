# Robot Return to Origin — Solution

## Problem Overview

Simulate a robot moving on a 2D plane. Track x and y coordinates and check if both are zero after all moves.

## Brute Force Approach

Count the occurrences of each move direction and check if U == D and L == R.

```
Count 'U' and 'D' — if counts differ, return false
Count 'L' and 'R' — if counts differ, return false
Return true
```

**Complexity**: O(n) time, O(1) space

## Optimal Approach

Track position as we go. Start at (0, 0) and update x and y for each move character. At the end, return whether both are zero.

This is already the optimal approach — the problem is inherently O(n) since every move must be processed.

## Deep Dive

The state machine approach is straightforward: the robot's position is the state, and each move is a transition. This is the simplest form of simulation — direct state tracking with no data structures needed.

def judgeCircle(moves: str) -> bool:
    x = y = 0
    for c in moves:
        if c == 'U': y += 1
        elif c == 'D': y -= 1
        elif c == 'L': x -= 1
        elif c == 'R': x += 1
    return x == 0 and y == 0

class Solution {
    public boolean judgeCircle(String moves) {
        int x = 0, y = 0;
        for (char c : moves.toCharArray()) {
            if (c == 'U') y++;
            else if (c == 'D') y--;
            else if (c == 'L') x--;
            else if (c == 'R') x++;
        }
        return x == 0 && y == 0;
    }
}
