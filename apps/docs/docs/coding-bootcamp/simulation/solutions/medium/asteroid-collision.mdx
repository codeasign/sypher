# Asteroid Collision — Solution

## Problem Overview

Simulate asteroid collisions where asteroids moving right (+) and left (-) can collide. The larger asteroid destroys the smaller one; same size destroys both.

## Brute Force Approach

Simulate step by step, removing collided asteroids and rechecking from the beginning after each collision. This is inefficient as it may require O(n²) iterations.

**Complexity**: O(n²) time, O(n) space

## Optimal Approach

Use a stack to simulate collisions in a single pass:

1. Iterate through each asteroid
2. For a moving-right asteroid (positive), push it onto the stack
3. For a moving-left asteroid (negative), check for collisions with the top of the stack
4. While the stack top is moving right and smaller than the current asteroid, pop it
5. If equal sizes, pop both
6. If the stack top is moving left (or empty), push the current asteroid

The key insight: only a moving-right asteroid followed by a moving-left asteroid can collide. Asteroids moving in the same direction never meet.

## Deep Dive

The stack maintains the state of surviving asteroids. When a moving-left asteroid encounters a moving-right asteroid, we simulate the collision by comparing sizes. This cascading check handles chain reactions (e.g., [10, 2, -5] where 2 explodes against -5, then 10 explodes against -5).

class Solution {
    public int[] asteroidCollision(int[] asteroids) {
        Stack<Integer> stack = new Stack<>();
        for (int a : asteroids) {
            boolean exploded = false;
            while (!stack.isEmpty() && a < 0 && stack.peek() > 0) {
                if (stack.peek() < -a) {
                    stack.pop();
                    continue;
                } else if (stack.peek() == -a) {
                    stack.pop();
                }
                exploded = true;
                break;
            }
            if (!exploded) stack.push(a);
        }
        int[] result = new int[stack.size()];
        for (int i = 0; i < result.length; i++) result[i] = stack.get(i);
        return result;
    }
}

        stack<int> st;
        for (int a : asteroids) {
            bool exploded = false;
            while (!st.empty() && a < 0 && st.top() > 0) {
                if (st.top() < -a) {
                    st.pop();
                    continue;

                    st.pop();

                break;

        }

        reverse(result.begin(), result.end());
        return result;
    }

fn asteroid_collision(asteroids: Vec<i32>) -> Vec<i32> {

int* asteroidCollision(int* asteroids, int asteroidsSize, int* returnSize) {
    int* stack = malloc(asteroidsSize * sizeof(int));
    int top = -1;
    for (int i = 0; i < asteroidsSize; i++) {
        int a = asteroids[i];
        int exploded = 0;
        while (top >= 0 && a < 0 && stack[top] > 0) {
            if (stack[top] < -a) { top--; continue; }
            else if (stack[top] == -a) { top--; }
            exploded = 1; break;
        }
        if (!exploded) stack[++top] = a;
    }
    *returnSize = top + 1;
    return stack;
}

public class Solution {
    public int[] AsteroidCollision(int[] asteroids) {
        var stack = new List<int>();
        foreach (int a in asteroids) {
            bool exploded = false;
            while (stack.Count > 0 && a < 0 && stack[stack.Count - 1] > 0) {
                if (stack[stack.Count - 1] < -a) {
                    stack.RemoveAt(stack.Count - 1);
                    continue;
                } else if (stack[stack.Count - 1] == -a) {
                    stack.RemoveAt(stack.Count - 1);
                }
                exploded = true; break;
            }
            if (!exploded) stack.Add(a);
        }
        return stack.ToArray();
    }
}

func asteroidCollision(asteroids []int) []int {

        exploded := false

            } else if stack[len(stack)-1] == -a {

## Key Takeaways

- Stack elegantly handles cascading collision logic
- The key condition for collision: top moving right AND new asteroid moving left
- The `while` loop handles multiple collisions from one asteroid
