# Spiral Matrix II

**Difficulty:** Medium

Given a positive integer `n`, generate an `n x n` matrix filled with elements from `1` to `n²` in **spiral order**.

## Examples

**Example 1:**
```
Input: n = 3
Output:
[[1, 2, 3],
 [8, 9, 4],
 [7, 6, 5]]
Flattened output: 1 2 3 8 9 4 7 6 5
```

**Example 2:**
```
Input: n = 1
Output: [[1]]
Flattened output: 1
```

**Example 3:**
```
Input: n = 4
Output:
[[ 1,  2,  3,  4],
 [12, 13, 14,  5],
 [11, 16, 15,  6],
 [10,  9,  8,  7]]
Flattened output: 1 2 3 4 12 13 14 5 11 16 15 6 10 9 8 7
```

## Input Specification

A single line containing one integer `n`.

## Output Specification

Print all `n²` integers in row-major order (row 0 left to right, then row 1 left to right, and so on), all on one line separated by spaces. This is the flattened representation of the spiral matrix.

## Constraints

- `1 <= n <= 20`

## Hints

<details>
<summary>Hint 1</summary>

Maintain four boundary pointers: `top`, `bottom`, `left`, `right`. Fill values going right along the top row, then down along the right column, then left along the bottom row, then up along the left column. After each direction, shrink the corresponding boundary inward.

</details>

<details>
<summary>Hint 2</summary>

Use a counter starting at 1. Keep looping while the counter is at most `n * n`. Each of the four directional fills advances the counter and updates the matrix. Shrink the boundary after each side is filled.

</details>

<details>
<summary>Hint 3</summary>

Alternatively, simulate movement with a direction array `[(0,1),(1,0),(0,-1),(-1,0)]` for right, down, left, up. When you hit a wall or a cell that is already filled, rotate to the next direction. This avoids managing four boundary variables explicitly.

</details>

</ProblemEditor>
