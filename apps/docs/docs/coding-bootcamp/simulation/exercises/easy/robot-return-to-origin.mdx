# Robot Return to Origin

There is a robot starting at the position (0, 0), the origin, on a 2D plane. Given a sequence of its moves, judge if this robot **ends up at (0, 0)** after it completes its moves.

You are given a string `moves` that represents the move sequence of the robot where `moves[i]` is one of the following:

- `'U'`: moves the robot up one unit (y + 1)
- `'D'`: moves the robot down one unit (y - 1)
- `'L'`: moves the robot left one unit (x - 1)
- `'R'`: moves the robot right one unit (x + 1)

Return `true` if the robot returns to the origin after all of its moves, otherwise return `false`.

## Examples

**Example 1:**
```
Input: moves = "UD"
Output: true
Explanation: The robot moves up once, then down once. Net change is zero.
```

**Example 2:**
```
Input: moves = "LL"
Output: false
Explanation: The robot moves left twice. Net horizontal position is -2.
```

**Example 3:**
```
Input: moves = "UDLR"
Output: true
Explanation: Up, down, left, right — the robot ends at the origin.
```

## Constraints

- `1 <= moves.length <= 10^4`
- `moves[i]` is one of `'U'`, `'D'`, `'L'`, or `'R'`

## Hints

<details>
<summary>Hint 1</summary>

Track the x and y coordinates separately. Start at (0, 0) and update for each move.
</details>

<details>
<summary>Hint 2</summary>

The robot returns to the origin only if the number of U moves equals D moves AND the number of L moves equals R moves.
</details>

</ProblemEditor>
