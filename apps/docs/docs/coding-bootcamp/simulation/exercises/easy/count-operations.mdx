# Count Operations to Obtain Zero

**Difficulty:** Easy

You are given two non-negative integers `num1` and `num2`. In one operation, if `num1 >= num2`, you subtract `num2` from `num1`; otherwise you subtract `num1` from `num2`. Return the **number of operations** required to make either number equal to zero.

## Example 1

**Input:** `num1 = 2`, `num2 = 3`
**Output:** `3`
**Explanation:**
- Operation 1: num1 &lt; num2, so num2 = 3 - 2 = 1. Now: (2, 1).
- Operation 2: num1 &gt;= num2, so num1 = 2 - 1 = 1. Now: (1, 1).
- Operation 3: num1 &gt;= num2, so num1 = 1 - 1 = 0. Now: (0, 1).
- num1 is zero — stop. Total = **3**.

## Example 2

**Input:** `num1 = 10`, `num2 = 10`
**Output:** `1`
**Explanation:**
- Operation 1: num1 &gt;= num2, so num1 = 10 - 10 = 0. Now: (0, 10).
- num1 is zero — stop. Total = **1**.

## Example 3

**Input:** `num1 = 1`, `num2 = 5`
**Output:** `5`
**Explanation:** Each step subtracts 1 from 5, requiring 5 operations.

## Input Specification

A single line containing two space-separated non-negative integers `num1` and `num2`.

## Output Specification

Print a single integer — the total number of operations needed to make either `num1` or `num2` equal to zero.

## Constraints

- `0 <= num1, num2 <= 10^5`

## Hints

<details>
<summary>Hint 1</summary>

Simulate the process directly: loop while both numbers are non-zero, and in each iteration subtract the smaller from the larger (incrementing an operation counter).

</details>

<details>
<summary>Hint 2</summary>

When one number is a multiple of the other (e.g., 20 and 4), you don't need to subtract one at a time — you can use integer division to figure out how many subtractions happen at once before the roles switch: `num1 // num2` steps. This is the key to an efficient solution.

</details>

<details>
<summary>Hint 3</summary>

The optimized approach is similar to the Euclidean GCD algorithm. Add `num1 // num2` to the count, then replace `num1` with `num1 % num2`, and swap roles. This reduces the problem size quickly and runs in O(log(min(num1, num2))) time.

</details>
