# Find Center of Star Graph — Solution

## Problem Overview

A star graph has one center node connected to all other `n - 1` nodes, with no other edges. Given the edge list, find the center.

## Key Insight: The Center Appears in Every Edge

In a star graph, the center node is an endpoint of **every** edge. By looking at just the first two edges, we can identify the center: it is the node that appears in **both** edges.

**Proof:** Every edge touches the center. So edge 1 is `(center, a)` and edge 2 is `(center, b)`. The common node between these two edges must be the center.

## Algorithm

1. Look at the first edge: `edges[0] = [a, b]`
2. Look at the second edge: `edges[1] = [c, d]`
3. If `a == c` or `a == d`, return `a`; otherwise return `b`

**Complexity Analysis:**
- **Time:** O(1) — only two edges examined
- **Space:** O(1) — no extra data structures

class Solution {
    public int findCenter(int[][] edges) {
        // The center node appears in both of the first two edges
        int a = edges[0][0], b = edges[0][1];  // Nodes in first edge
        int c = edges[1][0], d = edges[1][1];  // Nodes in second edge
        // If a appears in the second edge, it is the center
        if (a == c || a == d) return a;
        // Otherwise b must be the center
        return b;
    }
}

## Key Takeaways

- **O(1) solution** — you don't need to build the entire graph or count degrees
- The insight is that the center node must be present in **every** edge, so check just the first two
- This is a great example of reading the problem constraints carefully to find a trivial solution
- Always ask: is there a mathematical property that bypasses the brute force approach?
