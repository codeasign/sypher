# Max Area of Island — Solution

## Problem Overview

Find the largest connected component of 1s in a binary grid, where connections are 4-directional (up, down, left, right).

## Approach: DFS (Recursive)

**Idea:** Iterate over every cell in the grid. When we encounter an unvisited land cell, run DFS to explore the full island and compute its area. Track the maximum area seen.

**Algorithm:**
1. Initialize `maxArea = 0` and `visited` matrix (or modify grid in-place)
2. For each cell `(i, j)` in the grid:
   - If the cell is land and not visited, run DFS starting from it
   - DFS returns the area of the island
   - Update `maxArea = max(maxArea, area)`
3. Return `maxArea`

**DFS helper:** Check bounds and if the cell is land; mark visited; return 1 + sum of DFS results from 4 neighbors

**Complexity Analysis:**
- **Time:** O(m * n) — each cell is visited at most once
- **Space:** O(m * n) — recursion stack in worst case (entire grid is one island)

class Solution {
    private int m, n;

    public int maxAreaOfIsland(int[][] grid) {
        m = grid.length;
        n = grid[0].length;
        int maxArea = 0;
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                if (grid[i][j] == 1) {
                    maxArea = Math.max(maxArea, dfs(grid, i, j));  // Explore island
                }
            }
        }
        return maxArea;
    }

    private int dfs(int[][] grid, int i, int j) {
        // Check bounds and if cell is land
        if (i < 0 || i >= m || j < 0 || j >= n || grid[i][j] == 0) return 0;
        grid[i][j] = 0;  // Sink the island to mark visited
        // Explore 4 directions and sum area
        return 1 + dfs(grid, i + 1, j) + dfs(grid, i - 1, j)
               + dfs(grid, i, j + 1) + dfs(grid, i, j - 1);
    }
}

fn max_area_of_island(grid: Vec<Vec<i32>>) -> i32 {
    let mut grid = grid;  // Make mutable for in-place modification
    let m = grid.len();
    let n = grid[0].len();
    let mut max_area = 0;

    // Recursive DFS helper
    fn dfs(grid: &mut Vec<Vec<i32>>, i: i32, j: i32, m: i32, n: i32) -> i32 {
        if i < 0 || i >= m || j < 0 || j >= n || grid[i as usize][j as usize] == 0 {
            return 0;
        }
        grid[i as usize][j as usize] = 0;  // Sink the island
        // Sum area of current cell plus 4-directional neighbors
        1 + dfs(grid, i + 1, j, m, n) + dfs(grid, i - 1, j, m, n)
          + dfs(grid, i, j + 1, m, n) + dfs(grid, i, j - 1, m, n)
    }

    for i in 0..m {
        for j in 0..n {
            if grid[i][j] == 1 {
                let area = dfs(&mut grid, i as i32, j as i32, m as i32, n as i32);
                max_area = max_area.max(area);  // Track maximum
            }
        }
    }
    max_area
}

## Key Takeaways

- **Grid traversal with DFS** is the standard approach for island-counting problems
- **Sink the island** (set grid[i][j] = 0) avoids needing a separate visited array
- **Direction arrays** (or 4 separate calls) handle the 4-directional connectivity
- The same pattern works for counting islands, perimeter, and other island variants
