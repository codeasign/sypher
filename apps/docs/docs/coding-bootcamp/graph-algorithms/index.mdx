# Graph Algorithms

Graph algorithms are a fundamental category of coding interview problems. A **graph** is a collection of nodes (vertices) connected by edges. Graphs model real-world relationships: social networks, road maps, web pages, and dependency systems all behave as graphs.

## Graph Representations

There are three common ways to represent a graph in code:

### Adjacency List

An array (or map) where each node points to a list of its neighbors. This is the most popular representation for interviews because it is space-efficient for sparse graphs.

```python
# Adjacency list as a dictionary
graph = {
    0: [1, 3],
    1: [0, 2],
    2: [1, 3],
    3: [0, 2]
}
```

### Adjacency Matrix

A `V x V` boolean matrix where `matrix[i][j] = 1` means there is an edge from `i` to `j`. Space: O(V²). Good for dense graphs but wasteful for sparse ones.

```python
# Adjacency matrix for the same graph
#  0  1  2  3
# ────────────
# 0  1  1  0  1
# 1  1  0  1  0
# 2  0  1  0  1
# 3  1  0  1  0
```

### Edge List

A simple list of `(u, v)` pairs. Easy to iterate but slow for neighbor lookups.

```python
edges = [(0, 1), (0, 3), (1, 2), (2, 3)]
```

## Depth-First Search (DFS)

**DFS** explores a graph by going as deep as possible along each branch before backtracking. It uses a stack (explicitly or via recursion).

```python
def dfs(graph, start, visited=None):
    if visited is None:
        visited = set()
    visited.add(start)
    for neighbor in graph[start]:
        if neighbor not in visited:
            dfs(graph, neighbor, visited)
    return visited
```

**Time:** O(V + E) — each node and edge visited once  
**Space:** O(V) — recursion stack / visited set

### Key Use Cases

- Finding connected components
- Cycle detection in directed / undirected graphs
- Topological sorting (post-order DFS)
- Solving puzzles and mazes (path existence)

## Breadth-First Search (BFS)

**BFS** explores a graph level by level — visiting all neighbors before moving deeper. It uses a queue.

```python
from collections import deque

def bfs(graph, start):
    visited = {start}
    queue = deque([start])
    while queue:
        node = queue.popleft()
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
    return visited
```

**Time:** O(V + E) — each node and edge visited once  
**Space:** O(V) — queue and visited set

### Key Use Cases

- Shortest path in unweighted graphs
- Level-order traversal
- Finding connected components
- Bipartite checking

## Connected Components

In an undirected graph, a **connected component** is a maximal set of nodes where every pair is connected by a path. To find all components, run DFS or BFS on every unvisited node.

```python
def count_components(graph, n):
    visited = set()
    count = 0
    for node in range(n):
        if node not in visited:
            dfs(graph, node, visited)
            count += 1
    return count
```

## Cycle Detection

### Undirected Graph

A graph has a cycle if, during DFS, we find an edge to an already-visited node that is **not** the parent of the current node.

```python
def has_cycle_undirected(graph, n):
    visited = [False] * n

    def dfs(node, parent):
        visited[node] = True
        for neighbor in graph[node]:
            if not visited[neighbor]:
                if dfs(neighbor, node):
                    return True
            elif neighbor != parent:
                return True
        return False

    for i in range(n):
        if not visited[i]:
            if dfs(i, -1):
                return True
    return False
```

### Directed Graph

For directed graphs, track nodes **on the current recursion path** (a stack) in addition to visited. A back edge to a node on the path means a cycle.

```python
def has_cycle_directed(graph, n):
    WHITE, GRAY, BLACK = 0, 1, 2
    color = [WHITE] * n

    def dfs(node):
        color[node] = GRAY
        for neighbor in graph[node]:
            if color[neighbor] == GRAY:
                return True
            if color[neighbor] == WHITE:
                if dfs(neighbor):
                    return True
        color[node] = BLACK
        return False

    for i in range(n):
        if color[i] == WHITE:
            if dfs(i):
                return True
    return False
```

## Bipartite Checking

A graph is **bipartite** if its nodes can be colored with two colors such that no two adjacent nodes share the same color. Use BFS to assign alternating colors: if a neighbor already has the same color, the graph is not bipartite.

```python
def is_bipartite(graph, n):
    color = [-1] * n  # -1 = uncolored, 0 and 1 are the two colors
    for start in range(n):
        if color[start] == -1:
            queue = deque([start])
            color[start] = 0
            while queue:
                node = queue.popleft()
                for neighbor in graph[node]:
                    if color[neighbor] == -1:
                        color[neighbor] = 1 - color[node]
                        queue.append(neighbor)
                    elif color[neighbor] == color[node]:
                        return False
    return True
```

## Graph Coloring

Beyond bipartite checking, graph coloring problems ask whether a graph can be colored with `k` colors so that adjacent nodes have different colors. General k-coloring is NP-hard, but **2-coloring** (bipartite) is O(V+E).

## Worked Example: Find if a Path Exists

**Problem:** Given `n` nodes, an edge list, and a start and end node, determine if a path exists between start and end.

**Approach:** Build an adjacency list, then run DFS or BFS from the start. If the end is reachable, return true.

```python
from collections import defaultdict, deque

def valid_path(n, edges, start, end):
    # Build adjacency list
    graph = defaultdict(list)
    for u, v in edges:
        graph[u].append(v)
        graph[v].append(u)

    # BFS from start
    visited = {start}
    queue = deque([start])
    while queue:
        node = queue.popleft()
        if node == end:
            return True
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
    return False
```

**Walkthrough for n=6, edges=[(0,1),(1,2),(2,3),(4,5)], start=0, end=3:**

```
Graph:
  0 ─ 1 ─ 2 ─ 3        4 ─ 5

BFS from 0:
  Queue: [0]
  Pop 0 → visit [1]
  Queue: [1]
  Pop 1 → visit [2]
  Queue: [2]
  Pop 2 → visit [3]
  Queue: [3]
  Pop 3 → node == end → return True
```

## Key Techniques Cheat Sheet

| Technique | When to Use | Complexity |
|---|---|---|
| DFS (recursive) | Path existence, connected components, topological sort | O(V+E) |
| BFS (queue) | Shortest path (unweighted), level-order, bipartite | O(V+E) |
| Union-Find | Dynamic connectivity, Kruskal's MST | O(α(V)) |
| Dijkstra | Shortest path (weighted, non-negative) | O(E log V) |
| Topological Sort | Dependency ordering (DAG) | O(V+E) |
| Tarjan / Kosaraju | Strongly connected components | O(V+E) |
