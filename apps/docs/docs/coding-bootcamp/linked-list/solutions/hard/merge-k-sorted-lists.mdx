# Merge K Sorted Lists — Solution

## Problem Overview

Given an array of `k` sorted linked lists, merge them into a single sorted linked list and return the head. Each individual list is sorted in ascending order, and the merged result must also be sorted in ascending order.

## Brute Force Approach

**Idea:** Collect all node values from all k lists into an array, sort the array, then build a new linked list from the sorted values.

**Algorithm:**
1. Traverse all k lists, appending each node's value to a single array
2. Sort the array in ascending order
3. Build a new linked list from the sorted array values
4. Return the head of the new list

**Complexity Analysis:**
- **Time:** O(N log N) — we collect N values (O(N)) and sort them (O(N log N))
- **Space:** O(N) — we store all N values in an array

**Why this could be better:** This approach ignores the fact that the input lists are already sorted. By using that property, we can achieve O(N log k) time without needing to sort all values from scratch.

## Approach 2: Merge One by One (Pairwise Merge)

**Idea:** Merge two lists at a time using the standard "merge two sorted lists" algorithm. Start with an empty result, then merge each list into the result one at a time.

**Algorithm:**
1. Initialize `result = null`
2. For each list in the array:
   - Merge `result` with the current list using the two-pointer merge technique
   - Set `result` to the merged head
3. Return `result`

**Complexity Analysis:**
- **Time:** O(k * N) — the first merge processes 2 lists, the second processes 3 lists, etc. This leads to O(k * N) total operations.
- **Space:** O(1) — only a few pointer variables

**Why this could be better:** The first merge is small, but later merges involve increasingly large lists. This approach is slow for large k.

## Optimal Approach: Divide and Conquer (Pairwise Merging)

**Idea:** Use a divide-and-conquer strategy. Repeatedly merge pairs of lists until only one list remains. This is similar to merge sort — we pair up the lists, merge each pair, and halve the number of lists each round.

**Algorithm:**
1. While there is more than one list in the array:
   - Pair up adjacent lists (0 with 1, 2 with 3, etc.)
   - Merge each pair into a single sorted list using the standard two-pointer merge
   - Replace the pair with the merged result
2. Return the remaining list (or null if the array was empty)

**Why This Works:** Each round halves the number of lists. Merging two lists of average size N/k takes O(N/k) time. The total work per round is O(N) (since we merge all pairs), and there are O(log k) rounds, giving O(N log k) total.

**Complexity Analysis:**
- **Time:** O(N log k) — each node is processed once per round, and there are log k rounds
- **Space:** O(1) — no extra space beyond the recursive stack (if implemented iteratively, or O(log k) for recursion)

## Alternative Optimal Approach: Min-Heap (Priority Queue)

**Idea:** Push the first node of each of the k lists into a min-heap. Repeatedly pop the smallest node, append it to the result, and push its next node (if any) into the heap.

**Algorithm:**
1. Create a min-heap and push the head of each non-empty list
2. Create a dummy head node to simplify building the result list
3. While the heap is not empty:
   - Pop the smallest node from the heap
   - Append it to the result list (advance the tail pointer)
   - If the popped node has a next node, push that next node into the heap
4. Return the dummy head's next pointer

**Complexity Analysis:**
- **Time:** O(N log k) — each push/pop on the heap is O(log k), performed N times
- **Space:** O(k) — the heap holds at most one node from each of the k lists

## Deep Dive: Divide and Conquer vs. Min-Heap

### Visual Walkthrough (Divide and Conquer)

For input `lists = [[1, 4, 5], [1, 3, 4], [2, 6]]`:

```
Round 1: Merge pairs
  Merge list 0 [1, 4, 5] with list 1 [1, 3, 4] -> [1, 1, 3, 4, 4, 5]
  Merge list 2 [2, 6] with null -> [2, 6]
  Remaining lists: [[1, 1, 3, 4, 4, 5], [2, 6]]

Round 2: Merge the final pair
  Merge [1, 1, 3, 4, 4, 5] with [2, 6] -> [1, 1, 2, 3, 4, 4, 5, 6]
  Result: [1, 1, 2, 3, 4, 4, 5, 6]
```

### Visual Walkthrough (Min-Heap)

Initial heap: [1 (from list 0), 1 (from list 1), 2 (from list 2)]
Pop 1 (list 0), push next 4. Heap: [1 (list 1), 2 (list 2), 4 (list 0)]
Pop 1 (list 1), push next 3. Heap: [2 (list 2), 3 (list 1), 4 (list 0)]
Pop 2 (list 2), push next 6. Heap: [3 (list 1), 4 (list 0), 6 (list 2)]
Pop 3 (list 1), push next 4. Heap: [4 (list 0), 4 (list 1), 6 (list 2)]
... continues until all nodes are processed.

### Edge Cases to Consider

- **k = 0 (empty array):** Return null immediately.
- **k = 1 (single list):** Return that list directly — no merging needed.
- **All lists are empty:** The heap never receives any nodes, so the result is null. The divide-and-conquer approach also returns null after all pairs merge to nothing.
- **Lists of different lengths:** Both approaches handle varying lengths naturally — the heap only pushes nodes that exist, and divide-and-conquer merges whatever is present.
- **All values are the same:** The comparator will treat them as equal, and the order of equal values doesn't matter as long as they are all included.

class Solution {
    public ListNode mergeKLists(ListNode[] lists) {
        if (lists == null || lists.length == 0) return null;  // Empty input — no result

        return merge(lists, 0, lists.length - 1);  // Divide and conquer from 0 to last index
    }

    // Recursively merge lists from index left to right
    private ListNode merge(ListNode[] lists, int left, int right) {
        if (left == right) return lists[left];     // Single list — return it as-is
        if (left > right) return null;             // Invalid range — no list

        int mid = left + (right - left) / 2;       // Find the middle index (avoids overflow)
        ListNode l1 = merge(lists, left, mid);     // Recursively merge the left half
        ListNode l2 = merge(lists, mid + 1, right); // Recursively merge the right half
        return mergeTwoLists(l1, l2);               // Merge the two halves together
    }

    // Standard two-pointer merge of two sorted linked lists
    private ListNode mergeTwoLists(ListNode l1, ListNode l2) {
        ListNode dummy = new ListNode(0);          // Dummy node to simplify building the result
        ListNode tail = dummy;                     // Tail pointer for appending nodes

        while (l1 != null && l2 != null) {         // While both lists have nodes remaining
            if (l1.val < l2.val) {                 // Take the smaller value
                tail.next = l1;                    // Append l1 node
                l1 = l1.next;                      // Advance l1 pointer
            } else {
                tail.next = l2;                    // Append l2 node
                l2 = l2.next;                      // Advance l2 pointer
            }
            tail = tail.next;                      // Advance the tail pointer
        }

        // Attach any remaining nodes from the non-empty list
        tail.next = (l1 != null) ? l1 : l2;
        return dummy.next;                          // Return the merged head (skip dummy)
    }
}

fn merge_k_lists(lists: Vec<Option<Box<ListNode>>>) -> Option<Box<ListNode>> {
    if lists.is_empty() { return None; }                              // No lists — no result
    merge(&lists, 0, lists.len() - 1)                                 // Divide and conquer
}

fn merge(lists: &Vec<Option<Box<ListNode>>>, left: usize, right: usize) -> Option<Box<ListNode>> {
    if left == right { return lists[left].clone(); }                  // Single list — return clone
    if left > right { return None; }                                  // Invalid range — no list

    let mid = left + (right - left) / 2;                              // Find the middle index
    let l1 = merge(lists, left, mid);                                 // Recursively merge left half
    let l2 = merge(lists, mid + 1, right);                            // Recursively merge right half
    merge_two_lists(l1, l2)                                            // Merge the two halves
}

fn merge_two_lists(l1: Option<Box<ListNode>>, l2: Option<Box<ListNode>>) -> Option<Box<ListNode>> {
    match (l1, l2) {
        (None, None) => None,                                          // Both empty — no result
        (Some(n), None) | (None, Some(n)) => Some(n),                  // One empty — return the other
        (Some(n1), Some(n2)) => {                                       // Both non-empty — pick smaller
            if n1.val < n2.val {
                Some(Box::new(ListNode {
                    val: n1.val,
                    next: merge_two_lists(n1.next, Some(n2)),           // Recursively merge the rest
                }))
            } else {
                Some(Box::new(ListNode {
                    val: n2.val,
                    next: merge_two_lists(Some(n1), n2.next),           // Recursively merge the rest
                }))
            }
        }
    }
}

## Key Takeaways

- **Divide and conquer** (pairwise merging) achieves O(N log k) time with O(1) extra space, making it the most memory-efficient optimal solution
- **Min-heap** is an alternative O(N log k) solution that uses O(k) space, which can be more intuitive for some developers
- The **merge-two-lists** helper is the building block for both approaches and is a classic problem on its own
- The dummy head pattern simplifies linked list construction by eliminating the need to handle the first node as a special case
- Always consider the edge cases: k = 0, k = 1, empty lists, and lists of varying lengths
