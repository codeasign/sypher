# Reverse Linked List — Solution

## Problem Overview

Given the `head` of a singly linked list, reverse the list so that the tail becomes the new head and all `next` pointers face the opposite direction. Return the new head.

## Brute Force Approach

**Idea:** Traverse the list once to collect all values into an array, then build a new reversed list from the array.

**Algorithm:**
1. Traverse the linked list, storing each node's value in an array
2. Create a new linked list by iterating through the array in reverse order
3. Return the head of the new list

**Complexity Analysis:**
- **Time:** O(n) — we traverse the list twice (once to collect values, once to build the reversed list)
- **Space:** O(n) — we store all n values in an array

**Why this could be better:** This approach destroys the original list structure and uses O(n) extra space. The optimal approach reverses the list in-place without allocating additional memory.

## Optimal Approach: Iterative Reversal (Three Pointers)

**Idea:** Walk through the list, reversing each node's `next` pointer to point backward. Use three pointers to track the previous node, current node, and next node to avoid losing the rest of the list.

**Algorithm:**
1. Initialize `prev = null` (or `None`) and `current = head`
2. While `current` is not null:
   - Save `next_temp = current.next` (so we don't lose the rest of the list)
   - Set `current.next = prev` (reverse the pointer)
   - Move `prev` to `current` (advance prev)
   - Move `current` to `next_temp` (advance current)
3. Return `prev` as the new head (prev is now the original tail)

**Why This Works:** Each iteration detaches the current node from the forward direction and reattaches it pointing backward. The `prev` pointer tracks the reversed portion that grows one node at a time, while `current` advances through the original list. By the time `current` reaches `null`, `prev` is at the original tail, which is now the head of the reversed list.

**Complexity Analysis:**
- **Time:** O(n) — each node is visited exactly once
- **Space:** O(1) — only three pointer variables

## Deep Dive: Pointer Mechanics

### Visual Walkthrough

For the input list `1 -> 2 -> 3 -> 4 -> 5 -> null`, let's trace the algorithm:

```
Initial state:
prev = null
current = 1

Step 1: Save next_temp = 2 (the rest of the list)
        Set 1.next = null (point backward to prev)
        prev = 1
        current = 2
        Result: null <- 1    2 -> 3 -> 4 -> 5 -> null

Step 2: Save next_temp = 3
        Set 2.next = 1 (point backward)
        prev = 2
        current = 3
        Result: null <- 1 <- 2    3 -> 4 -> 5 -> null

Step 3: Save next_temp = 4
        Set 3.next = 2
        prev = 3
        current = 4
        Result: null <- 1 <- 2 <- 3    4 -> 5 -> null

Step 4: Save next_temp = 5
        Set 4.next = 3
        prev = 4
        current = 5
        Result: null <- 1 <- 2 <- 3 <- 4    5 -> null

Step 5: Save next_temp = null
        Set 5.next = 4
        prev = 5
        current = null
        Result: null <- 1 <- 2 <- 3 <- 4 <- 5

Loop ends (current = null). Return prev = 5.
Reversed list: 5 -> 4 -> 3 -> 2 -> 1 -> null
```

### Edge Cases to Consider

- **Empty list (head = null):** The while loop doesn't execute, and we return `prev` which is `null`. Correct.
- **Single node:** The loop runs once, reversing the single node's `next` to `null`, and returns that node as the new head. Correct.
- **Two nodes:** The loop runs twice, reversing both pointers. After the second iteration, `prev` is the original tail (now the new head). Correct.

### Recursive Approach

Reversing a linked list can also be done recursively. The idea is to reverse the rest of the list starting from `head.next`, then attach `head` to the end of the reversed portion:

```
Base case: if head is null or head.next is null, return head
Recursive step:
  1. new_head = reverse_list(head.next)
  2. head.next.next = head    (make the next node point back to head)
  3. head.next = null          (head becomes the new tail)
  4. return new_head
```

**Space complexity of recursion:** O(n) due to the call stack, making the iterative approach preferable for large lists where stack overflow is a concern.

class Solution {
    public ListNode reverseLinkedList(ListNode head) {
        ListNode prev = null;      // Track the reversed portion (starts empty)
        ListNode current = head;   // Start at the original head
        ListNode nextTemp;         // Temporary storage for the next node

        while (current != null) {            // Walk through the entire list
            nextTemp = current.next;         // Save the next node before breaking the link
            current.next = prev;             // Reverse: point current node backward
            prev = current;                  // Move prev forward to include current in reversed portion
            current = nextTemp;              // Move current forward to the next original node
        }
        return prev;                         // prev is now the head of the reversed list
    }
}

## Key Takeaways

- **Three-pointer iterative reversal** is the standard O(n) time, O(1) space solution
- The key insight is to save `current.next` before overwriting it, then advance both `prev` and `current`
- After the loop, `prev` is always the new head (the original tail of the list)
- This pattern is foundational for many linked list problems (palindrome checking, reversal in groups, etc.)
- The recursive approach is more elegant but uses O(n) stack space, so the iterative version is preferred for production code
