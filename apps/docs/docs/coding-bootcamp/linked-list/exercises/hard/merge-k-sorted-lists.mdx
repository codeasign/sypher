# Merge K Sorted Lists

**Difficulty:** Hard

You are given an array of `k` linked-lists, each linked-list is sorted in ascending order. Merge all the linked lists into one sorted linked list and return it.

## Example 1

**Input:** `lists = [[1, 4, 5], [1, 3, 4], [2, 6]]`  
**Output:** `[1, 1, 2, 3, 4, 4, 5, 6]`  
**Explanation:** The three sorted lists are merged into a single sorted list.

## Example 2

**Input:** `lists = [[]]`  
**Output:** `[]`  
**Explanation:** A single empty list produces an empty result.

## Input Specification

The first line contains an integer `k` (the number of linked lists). For each of the `k` lists, the first line is `n` (number of nodes in this list), followed by a line with `n` space-separated integers (the node values in order). If `n = 0`, the list is empty.

## Output Specification

Print the merged sorted values, space-separated, on a single line. If the result is empty, print an empty line.

## Constraints

- `k` ≤ 10⁴
- Total number of nodes across all lists ≤ 10⁵
- -10⁹ ≤ `Node.val` ≤ 10⁹

## Hints

<details>
<summary>Hint 1</summary>

Start by merging two sorted lists at a time. Can you merge all k lists by repeatedly merging pairs?
</details>

<details>
<summary>Hint 2</summary>

Use a min-heap (priority queue) to always extract the smallest node among all k heads.
</details>

<details>
<summary>Hint 3</summary>

The divide-and-conquer approach (merge pairs recursively) has O(N log k) time complexity and O(1) extra space.
</details>

</ProblemEditor>
