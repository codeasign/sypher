# Non-overlapping Intervals

**Difficulty:** Medium

Given an array of intervals `intervals` where `intervals[i] = [start_i, end_i]`, return the minimum number of intervals you need to remove to make the rest of the intervals non-overlapping.

## Example 1

**Input:** `intervals = [[1,2],[2,3],[3,4],[1,3]]`  
**Output:** `1`  
**Explanation:** Remove [1,3] to make the rest non-overlapping.

## Example 2

**Input:** `intervals = [[1,2],[1,2],[1,2]]`  
**Output:** `2`  
**Explanation:** You need to remove two [1,2] intervals.

## Example 3

**Input:** `intervals = [[1,2],[2,3]]`  
**Output:** `0`  
**Explanation:** No overlapping intervals.

## Input Specification

- A single line with 2n space-separated integers: `start_1 end_1 start_2 end_2 ... start_n end_n`.
- `1 <= intervals.length <= 10^5`

## Output Specification

Print a single integer: the minimum number of intervals to remove.

## Constraints

- `1 <= intervals.length <= 10^5`
- `-5 * 10^4 <= start_i < end_i <= 5 * 10^4`

## Hints

<details>
<summary>Hint 1</summary>

Sort intervals by their end time. Greedily pick the interval that ends earliest.
</details>

<details>
<summary>Hint 2</summary>

Keep track of the current end time. If the next interval starts before the current end, it overlaps and must be removed.
</details>

<details>
<summary>Hint 3</summary>

This is the classic "Maximum Non-overlapping Intervals" problem — the minimum to remove = total - maximum non-overlapping.
</details>
