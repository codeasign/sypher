# Maximum Depth of Binary Tree

**Difficulty:** Easy

Given the `root` of a binary tree, return its **maximum depth**.

A binary tree's **maximum depth** is the number of nodes along the longest path from the root node down to the farthest leaf node.

## Example 1

```
Tree:
    3
   / \
  9  20
    /  \
   15   7
```

**Input:**
```
7
3 9 20 -1 -1 15 7
```

**Output:** `3`

**Explanation:** The longest path is `3 -> 20 -> 15` (or `3 -> 20 -> 7`), which has 3 nodes.

## Example 2

```
Tree:
    1
     \
      2
```

**Input:**
```
3
1 -1 2
```

**Output:** `2`

**Explanation:** The longest path is `1 -> 2`, which has 2 nodes.

## Example 3

```
Tree:  (empty)
```

**Input:**
```
0
```

**Output:** `0`

**Explanation:** An empty tree has a maximum depth of 0.

## Input Specification

- First line: an integer `n` — number of nodes in the tree (`0 <= n <= 10^4`)
- Second line: `n` space-separated integers representing the level-order traversal of the tree (use `-1` for null nodes). Omitted if `n == 0`.

## Output Specification

Print a single integer — the maximum depth of the tree.

## Constraints

- The number of nodes in the tree is in the range `[0, 10^4]`
- `-100 <= Node.val <= 100`

## Hints

<details>
<summary>Hint 1</summary>

Think recursively: the maximum depth of a tree is 1 (for the current node) plus the maximum of the depths of its left and right subtrees.
</details>

<details>
<summary>Hint 2</summary>

The base case is straightforward: if the node is `null`, the depth is 0.
</details>

<details>
<summary>Hint 3</summary>

You can also solve this iteratively using BFS (level-order traversal). The number of levels you traverse equals the maximum depth.
</details>
