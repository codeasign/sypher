# Word Ladder

**Difficulty:** Hard

A **transformation sequence** from word `beginWord` to word `endWord` using a dictionary `wordList` is a sequence of words such that:

- The first word in the sequence is `beginWord`.
- The last word in the sequence is `endWord`.
- Only one letter can be changed at a time.
- Each transformed word must exist in `wordList`.

Given two words, `beginWord` and `endWord`, and a dictionary `wordList`, return the **length of the shortest transformation sequence** from `beginWord` to `endWord`. If no such sequence exists, return `0`.

Note that `beginWord` is **not** required to be in `wordList`, but `endWord` must be in `wordList` for a valid transformation.

## Example 1

**Input:**
```
beginWord = "hit"
endWord = "cog"
wordList = ["hot","dot","dog","lot","log","cog"]
```
**Output:** `5`
**Explanation:** One of the shortest transformation sequences is `"hit" → "hot" → "dot" → "dog" → "cog"`, which has length 5.

## Example 2

**Input:**
```
beginWord = "hit"
endWord = "cog"
wordList = ["hot","dot","dog","lot","log"]
```
**Output:** `0`
**Explanation:** `endWord` `"cog"` is not in `wordList`, so no valid transformation exists.

## Example 3

**Input:**
```
beginWord = "hit"
endWord = "hot"
wordList = ["hot"]
```
**Output:** `2`
**Explanation:** `"hit" → "hot"` is a single-step transformation with length 2.

## Input Specification

The input consists of exactly four lines:
1. A single word `beginWord`
2. A single word `endWord`
3. An integer `n`, the size of the word list
4. `n` space-separated words representing the word list (empty line if `n = 0`)

## Output Specification

Print a single integer representing the length of the shortest transformation sequence from `beginWord` to `endWord`. If no such sequence exists, print `0`.

## Constraints

- `1 <= beginWord.length <= 10`
- `beginWord.length == endWord.length`
- `0 <= wordList.length <= 5000`
- `1 <= wordList[i].length <= 10`
- All words consist of lowercase English letters
- All words in `wordList` are unique

## Hints

<details>
<summary>Hint 1</summary>

This is a shortest path problem on an unweighted graph — use BFS.
</details>

<details>
<summary>Hint 2</summary>

Each word is a node, and an edge exists if two words differ by exactly one character.
</details>

<details>
<summary>Hint 3</summary>

Build a set for O(1) word lookup, then generate all possible next words by changing one character at a time.
</details>

<details>
<summary>Hint 4</summary>

BFS guarantees the shortest path when all edges have equal weight (unweighted graph).
</details>
