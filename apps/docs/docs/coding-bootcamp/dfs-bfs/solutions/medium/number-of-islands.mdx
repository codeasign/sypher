# Number of Islands — Solution

## Problem Overview

Given an `m x n` 2D binary grid where `'1'` represents land and `'0'` represents water, count the number of **islands**. An island is formed by connecting adjacent land cells horizontally or vertically (4-directional). All four edges of the grid are surrounded by water.

## Brute Force Approach

**Idea:** Use a separate visited array to track which cells have been explored. Iterate through every cell; when land is found and not yet visited, perform DFS/BFS to mark all connected land.

**Algorithm:**
1. Create a `visited` boolean matrix of the same dimensions, initialized to `false`
2. For each cell `(r, c)` in the grid:
   - If `grid[r][c] == '1'` and `visited[r][c] == false`:
     - Increment island count
     - Perform DFS (or BFS) to mark all reachable land cells as visited
3. Return the island count

**Complexity Analysis:**
- **Time:** O(m × n) — each cell is visited at most once
- **Space:** O(m × n) — the visited array uses O(m × n) extra memory

**Why this could be better:** The visited array doubles the memory footprint. The optimal approach reuses the input grid to mark visited cells, reducing space to O(1) auxiliary (plus the recursion stack).

## Optimal Approach: DFS with In-Place Modification

**Idea:** Instead of a separate visited array, modify the grid in-place by setting visited land cells to `'0'`. This eliminates the need for extra memory while keeping the same time complexity.

**Algorithm:**
1. Initialize `count = 0`
2. Get `rows = grid.length` and `cols = grid[0].length`
3. For each cell `(r, c)` in the grid:
   - If `grid[r][c] == '1'`:
     - Increment `count` (found a new island)
     - Call `dfs(grid, r, c)` to mark all connected land as visited
4. Return `count`

**DFS helper function:**
1. Check bounds: if `r` or `c` is out of bounds, or `grid[r][c] == '0'`, return
2. Mark the current cell as visited by setting `grid[r][c] = '0'`
3. Recursively call DFS on all 4 neighbors: up, down, left, right

**Why This Works:** When we find a `'1'`, we know it belongs to an undiscovered island. By performing DFS from that cell, we visit every land cell connected to it and mark them all as `'0'`. This ensures that subsequent cells belonging to the same island are not counted again. The key insight is that we can safely destroy the input grid because we only need the count, not the original grid afterward.

**Complexity Analysis:**
- **Time:** O(m × n) — each cell is visited at most once
- **Space:** O(m × n) worst case — the recursion stack can go up to m × n in the worst case (e.g., a grid that is entirely land, where DFS recurses through all cells)

  After DFS completes, all cells of this island are now '0':
    [0] [0]  0

### Edge Cases to Consider

- **Empty grid**: If `grid` is `null` or empty, return `0` immediately.
- **Single cell**: A single `'1'` is one island; a single `'0'` is zero islands.
- **All land**: The entire grid is one big island — DFS will traverse every cell.
- **All water**: Zero islands — the loop finds no `'1'` cells.
- **Diagonal land only**: `'1'` cells at diagonal positions (e.g., `(0,0)` and `(1,1)`) are NOT connected since only horizontal and vertical adjacency counts.
- **Single row or column**: Islands are still connected left-to-right or top-to-bottom respectively.
- **Maximum constraints (300 × 300)**: The recursion depth may reach 90,000 in the worst case. Some languages or environments may stack overflow — an iterative BFS or explicit stack DFS can be used as an alternative.

### Why DFS Instead of BFS?

Both DFS and BFS work for this problem. DFS is slightly simpler to implement recursively, and the recursion naturally mirrors the depth-first exploration. However, for very large grids, the recursion depth could be a concern (stack overflow). In that case:
- Use an explicit stack (iterative DFS) to avoid recursion limits
- Or use BFS with a queue, which has the same O(m × n) time and space complexity

class Solution {
    public int numIslands(char[][] grid) {
        // Edge case: empty grid
        if (grid == null || grid.length == 0) return 0;

        int rows = grid.length;          // Number of rows in the grid
        int cols = grid[0].length;       // Number of columns in the grid
        int count = 0;                   // Island counter

        // Iterate through every cell in the grid
        for (int r = 0; r < rows; r++) {
            for (int c = 0; c < cols; c++) {
                // Found an unvisited land cell — start of a new island
                if (grid[r][c] == '1') {
                    count++;                    // Increment island count
                    dfs(grid, r, c, rows, cols); // Explore all connected land
                }
            }
        }
        return count;
    }

    // Depth-first search to mark all land cells in the current island
    private void dfs(char[][] grid, int r, int c, int rows, int cols) {
        // Base case: out of bounds or water cell — stop exploring
        if (r < 0 || r >= rows || c < 0 || c >= cols || grid[r][c] == '0') {
            return;
        }

        // Mark the current land cell as visited by setting it to '0'
        grid[r][c] = '0';

        // Recursively explore all 4-directional neighbors
        dfs(grid, r + 1, c, rows, cols);  // Down
        dfs(grid, r - 1, c, rows, cols);  // Up
        dfs(grid, r, c + 1, rows, cols);  // Right
        dfs(grid, r, c - 1, rows, cols);  // Left
    }
}

## Key Takeaways

- **DFS with in-place modification** is the standard pattern for counting connected components in a grid
- **Sinking the island** (setting `'1'` to `'0'`) eliminates the need for a separate visited array
- The approach is **O(m × n) time** — each cell is visited at most once
- **Space is O(m × n) in the worst case** due to the recursion stack (a fully filled grid)
- For very large grids, consider an **iterative DFS** (explicit stack) or **BFS** (queue) to avoid stack overflow
- The same "sink and explore" pattern applies to other grid-based connected component problems (e.g., Max Area of Island, Number of Closed Islands)
