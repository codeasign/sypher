# Maximum Depth of Binary Tree — Solution

## Problem Overview

Given the `root` of a binary tree, find its **maximum depth** — the number of nodes along the longest path from the root node down to the farthest leaf node.

## Approach: Recursive DFS (Depth-First Search)

**Idea:** The maximum depth of a tree is defined recursively: it is `1` (for the current node) plus the maximum of the depths of its left and right subtrees.

**Algorithm:**
1. If `root` is `null`, return `0` (an empty tree has depth 0)
2. Recursively compute `leftDepth = maxDepth(root.left)`
3. Recursively compute `rightDepth = maxDepth(root.right)`
4. Return `1 + max(leftDepth, rightDepth)`

**Why This Works:** The definition of tree depth is inherently recursive — the depth rooted at any node depends only on the depths of its children. By computing depths bottom-up, each node's depth is determined by the deeper of its two subtrees.

**Complexity Analysis:**
- **Time:** O(n) — each node is visited exactly once
- **Space:** O(h) — the recursion stack depth equals the height of the tree `h`. In the worst case (skewed tree), `h = n`. In the best case (balanced tree), `h = log n`

## BFS Alternative: Level-Order Traversal

**Idea:** Perform a level-order traversal (BFS) using a queue. The number of levels visited equals the maximum depth.

**Algorithm:**
1. If `root` is `null`, return `0`
2. Initialize a queue with `root` and a `depth` counter at `0`
3. While the queue is not empty:
   - Increment `depth`
   - Process all nodes at the current level (the current queue size)
   - Enqueue their children for the next level
4. Return `depth`

**Complexity Analysis:**
- **Time:** O(n) — each node is visited once
- **Space:** O(w) — the queue holds at most the width of the tree `w`, which can be up to `n/2` for a full tree

## Deep Dive: Recursive Depth Calculation

### Visual Walkthrough

For the tree `[3, 9, 20, -1, -1, 15, 7]`:

```
    3
   / \
  9  20
    /  \
   15   7
```

Tracing the recursion:

```
maxDepth(3):
  left = maxDepth(9):
    left = maxDepth(null) = 0
    right = maxDepth(null) = 0
    return 1 + max(0, 0) = 1

  right = maxDepth(20):
    left = maxDepth(15):
      left = maxDepth(null) = 0
      right = maxDepth(null) = 0
      return 1 + max(0, 0) = 1
    right = maxDepth(7):
      left = maxDepth(null) = 0
      right = maxDepth(null) = 0
      return 1 + max(0, 0) = 1
    return 1 + max(1, 1) = 2

  return 1 + max(1, 2) = 3
```

Result: `3`

### Edge Cases to Consider

- **Empty tree**: The base case returns `0` immediately.
- **Single node**: Both left and right are `null`, so `1 + max(0, 0) = 1`.
- **Skewed tree (all left children)**: The recursion follows the chain to the deepest leaf, returning a depth equal to the number of nodes.
- **Negative values**: Node values are irrelevant to depth calculation — only the structure matters.
- **Very deep trees**: In languages with recursion limits, consider the iterative BFS approach to avoid stack overflow.

### Why Recursion Is Natural

Tree depth is defined recursively: the depth at any node is `1 + max(depth(child_left), depth(child_right))`. The code mirrors this definition exactly. Recursion also handles the branching naturally — each recursive call explores a different subtree, and the `max()` function combines results.

## Key Takeaways

- **Recursive DFS** is the most natural solution — it mirrors the recursive definition of tree depth
- **BFS (level-order)** is a good alternative when recursion depth might be a concern
- Both approaches are **O(n) time** — each node is visited exactly once
- The problem introduces the fundamental pattern of **recursive tree traversal**: compute values for children, then combine at the parent
- This pattern appears in many other tree problems (diameter, balanced tree, max path sum)
