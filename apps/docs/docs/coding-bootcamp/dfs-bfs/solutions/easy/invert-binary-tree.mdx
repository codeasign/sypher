# Invert Binary Tree — Solution

## Problem Overview

Given the `root` of a binary tree, **invert** the tree (create a mirror image) by swapping the left and right children of every node, and return the root.

## Approach: Recursive DFS

**Idea:** Invert the tree recursively. For each node, swap its left and right children, then recursively invert the left and right subtrees.

**Algorithm:**
1. If `root` is `null`, return `null` (base case)
2. Swap the left and right children of the current node
3. Recursively call `invertTree(root.left)` to invert the left subtree
4. Recursively call `invertTree(root.right)` to invert the right subtree
5. Return the root

**Why This Works:** Inverting a tree is defined recursively. To invert a tree at node `n`, we need to swap `n.left` and `n.right`, and then invert both subtrees. The base case — a null node — is already inverted (there's nothing to swap).

**Complexity Analysis:**
- **Time:** O(n) — each node is visited exactly once
- **Space:** O(h) — the recursion stack depth equals the height of the tree `h`

## BFS Alternative: Iterative Level-Order

**Idea:** Use a queue to perform BFS. For each node processed, swap its children and enqueue them for further processing.

**Algorithm:**
1. If `root` is `null`, return `null`
2. Initialize a queue with `root`
3. While the queue is not empty:
   - Dequeue a node
   - Swap its left and right children
   - Enqueue the left child (if not null)
   - Enqueue the right child (if not null)
4. Return the root

**Complexity Analysis:**
- **Time:** O(n) — each node is visited once
- **Space:** O(w) — the queue holds at most the width of the tree

## Deep Dive: Recursive Inversion

### Visual Walkthrough

For the tree `[4, 2, 7, 1, 3, 6, 9]`:

```
Original:
    4
   / \
  2   7
 / \ / \
1  3 6  9

Step 1: Start at root (4)
        Swap children: left=2, right=7 → swap → left=7, right=2
        Recurse on left child (7):
          Swap children: left=6, right=9 → swap → left=9, right=6
          Recurse on left child (9): both children null → return
          Recurse on right child (6): both children null → return
          Return 7
        Recurse on right child (2):
          Swap children: left=1, right=3 → swap → left=3, right=1
          Recurse on left child (3): both children null → return
          Recurse on right child (1): both children null → return
          Return 2
        Return 4

Inverted:
    4
   / \
  7   2
 / \ / \
9  6 3  1
```

### Edge Cases to Consider

- **Empty tree**: Return `null` immediately.
- **Single node**: No children to swap — the node is returned as-is.
- **Unbalanced tree**: The operation works regardless of shape. A skewed tree requires O(n) recursion depth.
- **Tree with only left children**: After inversion, all nodes become right children.
- **Tree with only right children**: After inversion, all nodes become left children.

### Why Recursion Works

Inversion is a natural fit for recursion because the structure of the operation mirrors the tree structure. At each node, we perform the same operation (swap children), then recurse on both subtrees. The process is independent at each level, making it a clean divide-and-conquer approach.

class Solution {
    public TreeNode invertTree(TreeNode root) {
        // Base case: null node is already inverted
        if (root == null) return null;

        // Swap the left and right children
        TreeNode temp = root.left;
        root.left = root.right;
        root.right = temp;

        // Recursively invert the left and right subtrees
        invertTree(root.left);
        invertTree(root.right);

        // Return the root of the inverted tree
        return root;
    }
}

fn invert_tree(root: Option<Box<TreeNode>>) -> Option<Box<TreeNode>> {
    match root {
        // Base case: null node is already inverted
        None => None,
        Some(mut node) => {
            // Swap the left and right children using std::mem::swap
            std::mem::swap(&mut node.left, &mut node.right);

            node.left = invert_tree(node.left);
            node.right = invert_tree(node.right);

            // Return the root of the inverted tree
            Some(node)
        }
    }
}

## Key Takeaways

- **Recursion is the cleanest approach** — the operation mirrors the tree structure perfectly
- **Swap then recurse**: The order of operations (swap first, then recurse) doesn't matter for correctness, but swapping first is intuitive
- **O(n) time and O(h) space** — optimal for this problem
- The iterative BFS approach is equally valid and avoids recursion depth concerns
- Inverting a tree is a classic introductory problem that teaches the fundamental **recursive tree traversal** pattern
