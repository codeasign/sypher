# Validate Binary Search Tree — Solution

## Problem Overview

Given the root of a binary tree, determine if it is a valid BST. A valid BST requires that for every node, all values in its left subtree are less than the node's value, and all values in its right subtree are greater than the node's value.

## Approach 1: Recursive Range Validation (Optimal)

**Idea:** Traverse the tree recursively, passing down a valid range `(min, max)` for each node. For the root, the range is unbounded (-infinity, +infinity). For a left child, update the max to the parent's value. For a right child, update the min to the parent's value. If any node's value falls outside its allowed range, the tree is invalid.

**Algorithm:**
1. Define a helper function `validate(node, low, high)` that checks if `node` is valid
2. If node is null, return true (empty tree is valid)
3. If `node.val <= low` or `node.val >= high`, return false (out of range)
4. Recursively validate left subtree with range `(low, node.val)`
5. Recursively validate right subtree with range `(node.val, high)`
6. Return true only if both subtrees are valid

**Why This Works:** The BST property is transitive. If a node must be less than its grandparent (because it's in the left subtree), then all descendants in that left subtree must also be less than the grandparent. Passing down the range enforces this transitivity.

**Complexity Analysis:**
- **Time:** O(n) — each node is visited exactly once
- **Space:** O(h) — recursion stack depth equals the height of the tree

## Approach 2: In-Order Traversal (Alternative)

**Idea:** Perform an in-order traversal. Since in-order traversal of a valid BST produces a strictly increasing sequence, we can track the previously visited value and verify that each subsequent value is greater.

**Algorithm:**
1. Initialize `prev` to null
2. Perform in-order traversal (left, visit, right)
3. When visiting each node, if `prev != null` and `node.val <= prev`, return false
4. Set `prev = node.val`
5. Return true if all nodes pass the check

    fn validate(node: Option<Rc<RefCell<TreeNode>>>, low: Option<i64>, high: Option<i64>) -> bool {
        if let Some(n) = node {                  // If node exists, check its value

        }

## Key Takeaways

- **A common mistake is only checking immediate children** — the range validation approach correctly enforces the BST property across all levels
- **Using null for unbounded ranges** works well in languages with nullable types; for primitives, use sentinel values like Long.MIN_VALUE/Long.MAX_VALUE
- **Watch for integer overflow** — if the problem allows values at the boundary of the integer range, use a wider type (long/i64) for the range parameters
- **The in-order traversal approach** is simpler to implement but requires O(n) space in the worst case for the recursion stack
- Both approaches run in O(n) time and are optimal
