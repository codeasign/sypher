# Recover Binary Search Tree — Solution

## Problem Overview

You are given the root of a BST where exactly two nodes have been swapped by mistake. You must recover the tree by swapping the values of these two nodes back.

The key insight: performing an in-order traversal on a valid BST yields a strictly increasing sequence. After swapping two nodes, the in-order sequence will have either one or two inversions (pairs where the order is violated).

## Approach: In-Order Traversal with Pointer Tracking

**Idea:** Perform an in-order traversal and track two nodes that are misplaced. When we find a pair where `prev.val > curr.val`, the first occurrence identifies the first swapped node (prev), and the last occurrence identifies the second swapped node (curr). After the traversal, swap the values of the two identified nodes.

**Algorithm:**
1. Initialize three pointers: `first = null`, `second = null`, `prev = null`
2. Perform an in-order traversal. For each visited node:
   - If `prev != null` and `prev.val > node.val`:
     - If this is the first inversion, set `first = prev` and `second = node`
     - If this is a subsequent inversion, set `second = node` (update the second node)
   - Set `prev = node`
3. After traversal, swap the values of `first` and `second`

**Why This Works:**

In a valid BST, in-order traversal gives: `[a1, a2, a3, ..., an]` where `a1 < a2 < a3 < ... < an`.

After swapping two nodes `x` and `y` (where `x > y`), the sequence might look like:
- If the swapped nodes are not adjacent: `[... y, ..., x ...]` — two inversions
  - First inversion: `prev = x`, `curr = some_value_greater_than_y` → `x > curr`
  - Second inversion: `prev = some_value`, `curr = y` → `some_value > y`
- If the swapped nodes are adjacent: `[... y, x ...]` — one inversion
  - Only inversion: `prev = y`, `curr = x` → `y > x`

Our algorithm correctly handles both cases because:
- On the first inversion, we set `first = prev` (the larger element) and `second = curr` (the smaller element)
- On a second inversion (if any), we update `second = curr` to capture the truly smaller element

**Complexity Analysis:**
- **Time:** O(n) — we visit each node exactly once
- **Space:** O(h) — recursion stack depth, where h is the tree height

## Morris Traversal (O(1) Space Follow-up)

To achieve truly O(1) extra space (excluding recursion stack), use Morris traversal. Morris traversal uses threaded binary tree pointers to avoid using a stack or recursion. The algorithm is the same as above but uses a Morris in-order traversal instead of recursive or stack-based traversal.

class Solution {
    private TreeNode first = null;               // First misplaced node
    private TreeNode second = null;              // Second misplaced node
    private TreeNode prev = null;                // Previous node in in-order

    public void recoverTree(TreeNode root) {
        inorder(root);                           // Find the two swapped nodes

        // Swap the values of the two misplaced nodes
        int temp = first.val;
        first.val = second.val;
        second.val = temp;
    }

    private void inorder(TreeNode node) {
        if (node == null) return;

        inorder(node.left);                      // Visit left subtree

        // Detect inversion: prev.val > current.val
        if (prev != null && prev.val > node.val) {
            if (first == null) {
                first = prev;                    // First inversion — mark prev
            }
            second = node;                       // Always update second to current
        }
        prev = node;                             // Update previous node

        inorder(node.right);                     // Visit right subtree
    }
}

use std::cell::RefCell;
use std::rc::Rc;

fn recover_tree(root: Option<Rc<RefCell<TreeNode>>>) {
    // We use Rc/RefCell for shared mutable state in the recursive closure
    let first: Rc<RefCell<Option<Rc<RefCell<TreeNode>>>>> = Rc::new(RefCell::new(None));
    let second: Rc<RefCell<Option<Rc<RefCell<TreeNode>>>>> = Rc::new(RefCell::new(None));
    let prev: Rc<RefCell<Option<Rc<RefCell<TreeNode>>>>> = Rc::new(RefCell::new(None));

    fn inorder(
        node: Option<Rc<RefCell<TreeNode>>>,
        first: Rc<RefCell<Option<Rc<RefCell<TreeNode>>>>>,
        second: Rc<RefCell<Option<Rc<RefCell<TreeNode>>>>>,
        prev: Rc<RefCell<Option<Rc<RefCell<TreeNode>>>>>,
    ) {
        if let Some(n) = node {
            inorder(n.borrow().left.clone(), first.clone(), second.clone(), prev.clone());

            // Detect inversion
            let p = prev.borrow().clone();
            if let Some(prev_node) = p {
                if prev_node.borrow().val > n.borrow().val {
                    if first.borrow().is_none() {
                        *first.borrow_mut() = Some(prev_node.clone());
                    }
                    *second.borrow_mut() = Some(n.clone());
                }
            }
            *prev.borrow_mut() = Some(n.clone());

            inorder(n.borrow().right.clone(), first, second, prev);
        }
    }

    inorder(root, first.clone(), second.clone(), prev.clone());

    // Swap the values of the two misplaced nodes
    let f = first.borrow().clone().unwrap();
    let s = second.borrow().clone().unwrap();
    let temp = f.borrow().val;
    f.borrow_mut().val = s.borrow().val;
    s.borrow_mut().val = temp;
}

## Edge Cases

- **Adjacent swapped nodes:** Only one inversion is detected. `first` is set on that inversion, `second` is also set. The swap still works correctly.
- **Non-adjacent swapped nodes:** Two inversions are detected. `first` is set on the first inversion, `second` is updated on the second inversion (overwriting the first detection of `second`). This correctly captures the two truly misplaced nodes.
- **Root is one of the swapped nodes:** Works correctly — the algorithm doesn't depend on where the nodes are in the tree.
- **Single node or empty tree:** The check `prev != null` prevents any action. The tree is already valid (swapping two nodes in a tree of less than 2 nodes is not possible).

## Key Takeaways

- **In-order traversal of a BST must be strictly increasing** — two violations signal swapped adjacent nodes; one violation with a later second violation signals non-adjacent swapped nodes
- **The first inversion identifies the larger element (first), and the last inversion identifies the smaller element (second)** — this is the core insight
- **Always update `second` to the current node** on every inversion, not just the first one
- **The solution is O(n) time and O(h) space** — optimal for this problem
- **Morris traversal** can reduce space to O(1) for the follow-up requirement
