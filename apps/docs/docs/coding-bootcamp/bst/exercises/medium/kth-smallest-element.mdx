# Kth Smallest Element in a BST

**Difficulty:** Medium

Given the `root` of a Binary Search Tree and an integer `k`, return the `kth` smallest value (1-indexed) among all the nodes in the tree.

## Example 1

**Input:**
```
3 1
3 1 4
```

The BST is:
```
    3
   / \
  1   4
```

**Output:** `1`
**Explanation:** The sorted values are `[1, 3, 4]`. The 1st smallest is `1`.

## Example 2

**Input:**
```
3 3
3 1 4
```

**Output:** `4`
**Explanation:** The sorted values are `[1, 3, 4]`. The 3rd smallest is `4`.

## Example 3

**Input:**
```
5 3
5 3 8 2 4
```

The BST is:
```
    5
   / \
  3   8
 / \
2   4
```

**Output:** `4`
**Explanation:** The sorted values are `[2, 3, 4, 5, 8]`. The 3rd smallest is `4`.

## Input Specification

- First line: two space-separated integers `n` (number of nodes) and `k` (1-indexed position)
- Second line: `n` space-separated integers representing the level-order traversal of the BST (use `-1` for null nodes)

## Output Specification

Print a single integer: the kth smallest value in the BST. If the tree is empty, print `-1`.

## Constraints

- The number of nodes is in the range `[0, 10^4]`
- `0 < k <= n` (k is always valid for non-empty trees)
- `-10^4 <= Node.val <= 10^4`

## Hints

<details>
<summary>Hint 1</summary>

An in-order traversal of a BST visits nodes in ascending order. You can simply perform an in-order traversal and stop at the kth visited node.
</details>

<details>
<summary>Hint 2</summary>

For an iterative approach, use an explicit stack to simulate in-order traversal. This avoids recursion overhead and can stop early once the kth element is found.
</details>

<details>
<summary>Hint 3</summary>

You can also use a recursive in-order traversal with a counter. Start from the leftmost node, increment the counter as you visit nodes, and return the value when the counter reaches k.
</details>
