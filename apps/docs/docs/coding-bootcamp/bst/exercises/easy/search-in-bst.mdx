# Search in a Binary Search Tree

**Difficulty:** Easy

Given the `root` of a binary search tree and a `target` value, determine whether the target value exists in the BST.

Return `true` if the target exists, and `false` otherwise.

## Example 1

**Input:**
```
5
4 2 7 1 3
2
```

The BST is:
```
    4
   / \
  2   7
 / \
1   3
```

**Output:** `true`
**Explanation:** The target `2` exists in the tree.

## Example 2

**Input:**
```
5
4 2 7 1 3
5
```

**Output:** `false`
**Explanation:** The target `5` does not exist in the tree.

## Example 3

**Input:**
```
1
10
10
```

**Output:** `true`
**Explanation:** A single-node tree contains the target.

## Input Specification

- First line: an integer `n` — number of nodes in the BST (`0 <= n <= 10^4`)
- Second line: `n` space-separated integers representing the level-order traversal of the BST (use `-1` for null nodes)
- Third line: an integer `target` — the value to search for

## Output Specification

Print `true` if the target exists in the BST, otherwise `false`.

## Constraints

- The number of nodes is in the range `[0, 10^4]`
- `-10^5 <= Node.val <= 10^5`
- `-10^5 <= target <= 10^5`
- The input tree is a valid BST (every left node < root < every right node)

## Hints

<details>
<summary>Hint 1</summary>

The BST property tells you exactly which direction to go at each node. If the target is less than the current node's value, search the left subtree. If greater, search the right subtree.
</details>

<details>
<summary>Hint 2</summary>

You can implement this either recursively or iteratively. Both approaches are O(log n) average time.
</details>

<details>
<summary>Hint 3</summary>

Start at the root. While the current node is not null, compare its value with the target. If they match, return true. Otherwise, move to the left or right child based on the comparison. If you reach null, the target is not in the tree.
</details>
