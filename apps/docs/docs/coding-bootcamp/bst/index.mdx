# Binary Search Tree (BST)

A Binary Search Tree is a node-based binary tree data structure where every node follows a specific ordering property: all values in the left subtree are less than the node's value, and all values in the right subtree are greater than the node's value.

⏱ 20 minutes

## What You Will Learn

- BST properties and structural rules (left < root < right)
- How to search, insert, and traverse a BST efficiently
- The significance of in-order traversal
- How to validate, find kth smallest, and recover a BST

## The Problem

BST problems appear frequently in coding interviews. They test your understanding of tree traversal, recursive thinking, and the ordering property. Common LeetCode problems include validating a BST, finding the kth smallest element, searching in a BST, and recovering a BST with swapped nodes.

## The Concept

A BST is a binary tree where each node has at most two children, and the following property holds for every node:

- Every node in the **left subtree** has a value **less than** the node's value
- Every node in the **right subtree** has a value **greater than** the node's value

This ordering property enables binary search on the tree: at each node, you can decide which subtree to explore, eliminating half the remaining nodes.

## BST Properties

### Ordering Property

For every node `n` with value `v`:
- All values in `n.left` subtree are **strictly less than** `v`
- All values in `n.right` subtree are **strictly greater than** `v`

### No Duplicates (Typically)

Standard BST definitions do not allow duplicate values. Some variations allow duplicates by storing them in the left or right subtree consistently.

### Recursive Nature

Every subtree of a BST is also a valid BST. This makes recursion a natural fit for BST operations.

## Key Operations

### Search

Start at the root. At each node, compare the target with the current node's value. If equal, return the node. If the target is smaller, go left. If larger, go right. If you reach a null node, the target is not in the tree.

### Insertion

Search for the value. If you find it, it already exists (no duplicates). If you reach a null position, insert the new node there while maintaining the BST property.

### Deletion

Three cases: deleting a leaf (remove directly), deleting a node with one child (replace with the child), deleting a node with two children (replace with in-order successor or predecessor).

## In-Order Traversal

In-order traversal visits nodes in the order: **left → root → right**. For a BST, this produces values in **sorted ascending order**. This is the most important traversal pattern for BST problems.

## Complexity

| Operation | Average (Balanced) | Worst (Skewed) |
|-----------|-------------------|----------------|
| Search    | O(log n)          | O(n)           |
| Insert    | O(log n)          | O(n)           |
| Delete    | O(log n)          | O(n)           |
| In-order  | O(n)              | O(n)           |
| Space     | O(n)              | O(n)           |

**Memory:** O(n) for storing n nodes (each node stores value + two child pointers).

## Common Applications

- **Ordered traversal**: Get all values in sorted order via in-order traversal
- **Validation**: Check if a binary tree maintains the BST property
- **Search**: Efficiently find if a value exists in the tree
- **LCA (Lowest Common Ancestor)**: Find the common ancestor of two nodes in a BST by using the ordering property
- **Range queries**: Find all values within a given range
- **Kth smallest/largest**: Use in-order traversal to find the kth element
- **Self-balancing trees**: AVL, Red-Black trees build on BST with balancing guarantees

## Worked Example: Search in BST

Given the BST below, search for the value `6`:

```
      8
     / \
    3   12
   / \
  1   6
```

**Step-by-step:**
1. Start at root `8`. Is `6 == 8`? No. Is `6 < 8`? Yes, go left.
2. Move to node `3`. Is `6 == 3`? No. Is `6 < 3`? No, go right.
3. Move to node `6`. Is `6 == 6`? Yes. Found the node.

**Return:** The node containing `6`.

## When to Use BST Techniques

- The problem involves a **binary tree with an ordering property**
- You need **sorted order** of elements — consider **in-order traversal**
- You need to **validate the ordering** of a tree
- The problem mentions **kth smallest** or **kth largest** in a tree
- The problem involves **two swapped nodes** in a tree
- You need efficient **search, insert, or delete** operations
