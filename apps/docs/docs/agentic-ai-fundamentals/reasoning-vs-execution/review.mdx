# Reasoning vs. Execution — Review

*Reading time: 3 minutes*

## Key Takeaways

- Reasoning is deciding what to do; execution is doing it — they have different cost and capability profiles
- Separating reasoning and execution lets you use a large model for planning and a smaller model for tool calls, reducing costs
- Fused reasoning+execution is simpler and appropriate for short agent runs or simple tool calls
- The separation requires structured output from the reasoning phase that the execution phase can follow reliably
- Execution failures must be fed back to the reasoning phase for proper recovery

## Recap in 60 Seconds

1. Reasoning needs capability (big model); execution needs speed and precision (small model) — match the model to the job
2. Measure the reasoning-to-execution token ratio before deciding to separate them
3. Make the reasoning output highly structured so the execution phase cannot misinterpret the plan
