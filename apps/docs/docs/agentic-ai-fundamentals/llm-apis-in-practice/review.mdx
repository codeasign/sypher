# LLM APIs in Practice — Review

*Reading time: 3 minutes*

## Key Takeaways

- Streaming reduces perceived latency by delivering tokens as they are generated
- A production API client should handle retries with exponential backoff for transient errors
- Classify API errors into retriable (rate limits, server errors) and non-retriable (auth, bad request)
- Log token usage on every request for cost tracking and optimization
- Configure timeouts explicitly based on expected input size and output length
- Model names should be configuration values, not hardcoded strings

## Recap in 60 Seconds

1. Wrap API calls in a client that handles retries, streaming, cost tracking, and structured responses — never call the SDK directly from application code
2. Classify errors before retrying: rate limits and server errors get backoff; auth errors fail fast
3. Track every token — cost visibility is the foundation of optimization
