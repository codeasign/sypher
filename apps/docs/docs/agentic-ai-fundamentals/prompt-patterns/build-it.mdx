# Prompt Patterns — Build It

*Reading time: 12 minutes*

## Breaking It Down

### 1. Label-Constraint Pattern (Classification)

The label-constraint pattern gives the model a fixed set of labels and forces a single selection:

```python
import anthropic

client = anthropic.Anthropic()


def classify_with_labels(text: str, labels: list[str]) -> str:
    """Classify text into one of the provided labels."""
    labels_str = ", ".join(labels)
    response = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=10,
        temperature=0,
        system=(
            f"Classify the input into exactly one category: {labels_str}."
            "\nRespond with only the category name."
        ),
        messages=[{"role": "user", "content": text}]
    )
    return response.content[0].text.strip()


categories = ["COMPLAINT", "PRAISE", "QUESTION", "FEATURE_REQUEST"]

texts = [
    "This app is terrible, it crashes every time I open it.",
    "I love the new design, great work team!",
    "How do I reset my password?",
    "It would be great if you added dark mode support.",
]

for text in texts:
    label = classify_with_labels(text, categories)
    print(f"[{label:16s}] {text[:40]}")
```

The label-constraint pattern works because the output space is tightly bounded. With `max_tokens=10` and a strict format instruction, the model has no room to deviate.

### 2. Persona Pattern

The persona pattern adapts content for a specific audience:

```python
def explain_with_persona(topic: str, persona: str) -> str:
    """Explain a topic as a specific persona."""
    response = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=200,
        temperature=0.3,
        system=f"You are {persona}.",
        messages=[{"role": "user", "content": f"Explain {topic}"}]
    )
    return response.content[0].text


personas = [
    "a kindergarten teacher explaining to 5-year-olds",
    "a CEO explaining to the board of directors",
]

for persona in personas:
    print(f"\n=== {persona} ===")
    print(explain_with_persona("blockchain", persona))
```

The same underlying content (blockchain) produces completely different explanations based on the persona. The kindergarten teacher version uses simple analogies; the CEO version focuses on strategic implications.

### 3. Template Pattern

The template pattern provides a structured format with placeholders to fill:

```python
def extract_with_template(text: str) -> dict:
    """Extract structured data using a template pattern."""
    response = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=200,
        temperature=0,
        messages=[{
            "role": "user",
            "content": (
                "Extract the following information from the text below.\n\n"
                "Template:\n"
                "- name: <full name>\n"
                "- email: <email address>\n"
                "- department: <department name>\n"
                "- urgency: <HIGH, MEDIUM, or LOW>\n"
                "- summary: <one sentence description>\n\n"
                "Text: " + text
            )
        }]
    )
    return response.content[0].text


email = """
Hi, my name is Jane Smith and I'm writing from the Engineering department.
My email is jane.smith@company.com. Our production server has been down for
two hours and we need immediate assistance. This is critical.
"""

print(extract_with_template(email))
```

The template gives the model exact field names and formats to use, making the output easy to parse programmatically.

### 4. Comparison Pattern

The comparison pattern evaluates options against explicit criteria:

```python
def compare_options(
    options: list[str],
    criteria: list[str]
) -> str:
    """Compare options using specified criteria."""
    criteria_str = "\n".join(f"- {c}" for c in criteria)
    options_str = "\n".join(f"- {o}" for o in options)

    response = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=400,
        temperature=0,
        messages=[{
            "role": "user",
            "content": (
                f"Compare the following cloud providers using these criteria:\n\n"
                f"Criteria:\n{criteria_str}\n\n"
                f"Options:\n{options_str}\n\n"
                "Format: For each criterion, list which option is best and why. "
                "End with an overall recommendation."
            )
        }]
    )
    return response.content[0].text


providers = [
    "AWS: mature, broadest service catalog, pay-as-you-go pricing",
    "GCP: strong in data/ML, competitive pricing, good network",
    "Azure: best Microsoft integration, strong enterprise features",
]
criteria = [
    "Global availability (number of regions)",
    "Pricing for small teams",
    "Machine learning services",
    "Ease of getting started",
]

print(compare_options(providers, criteria))
```

## How It Works Internally

Each pattern exploits a different property of how LLMs process and generate text.

The label-constraint pattern works by narrowing the output distribution to a few high-probability tokens. When you set `max_tokens=10` and specify "respond with only the category name," the model assigns nearly all probability mass to the tokens representing your labels.

The persona pattern works by shifting the entire output distribution toward the statistical patterns of the specified role. The model has seen vast amounts of text written from different perspectives and activates the relevant patterns when the persona is named.

The template pattern exploits the model's pattern-matching ability. When you provide a structured template, the model treats it as a completion task — it "fills in the blanks" rather than generating free text.

## Build Something Real

Build a prompt pattern selector that automatically picks the right pattern for a task:

```python
import anthropic
from enum import Enum

client = anthropic.Anthropic()


class TaskType(Enum):
    CLASSIFICATION = "classification"
    EXTRACTION = "extraction"
    EXPLANATION = "explanation"
    COMPARISON = "comparison"


PATTERN_TEMPLATES = {
    TaskType.CLASSIFICATION: {
        "system": "Classify the input into exactly one category. Respond with only the category name.",
        "user": "Categories: {categories}\n\nText: {input}"
    },
    TaskType.EXTRACTION: {
        "system": "Extract structured information from the text. Follow the template exactly.",
        "user": "Extract from this text:\n{input}\n\nFields to extract:\n{fields}"
    },
    TaskType.EXPLANATION: {
        "system": "You are {persona}.",
        "user": "Explain {topic} in a way that matches your persona."
    },
    TaskType.COMPARISON: {
        "system": "Compare items objectively using the provided criteria.",
        "user": "Items:\n{items}\n\nCriteria:\n{criteria}\n\nFormat each criterion with the best option and why."
    },
}


def apply_pattern(task_type: TaskType, **kwargs) -> str:
    """Apply the appropriate prompt pattern for a task."""
    template = PATTERN_TEMPLATES[task_type]

    if task_type == TaskType.CLASSIFICATION:
        system = template["system"]
        user = template["user"].format(
            categories=", ".join(kwargs["categories"]),
            input=kwargs["input"]
        )
    elif task_type == TaskType.EXTRACTION:
        system = template["system"]
        fields_text = "\n".join(f"- {f}" for f in kwargs["fields"])
        user = template["user"].format(
            input=kwargs["input"],
            fields=fields_text
        )
    elif task_type == TaskType.EXPLANATION:
        system = template["system"].format(persona=kwargs["persona"])
        user = template["user"].format(topic=kwargs["topic"])
    elif task_type == TaskType.COMPARISON:
        items_text = "\n".join(f"- {i}" for i in kwargs["items"])
        criteria_text = "\n".join(f"- {c}" for c in kwargs["criteria"])
        user = template["user"].format(items=items_text, criteria=criteria_text)
        system = template["system"]

    response = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=300,
        temperature=0,
        system=system,
        messages=[{"role": "user", "content": user}]
    )
    return response.content[0].text


# Example usage
result = apply_pattern(
    TaskType.CLASSIFICATION,
    categories=["URGENT", "NORMAL", "LOW"],
    input="The server is down and customers cannot access their accounts."
)
print(f"Classification result: {result}")
```

This pattern selector abstracts away the prompt engineering details, letting application code specify what to do without worrying about how to prompt for it.
