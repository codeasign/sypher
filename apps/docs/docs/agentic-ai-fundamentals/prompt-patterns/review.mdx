# Prompt Patterns — Review

*Reading time: 3 minutes*

## Key Takeaways

- Prompt patterns are reusable structures for common task types — classification, extraction, explanation, comparison
- The label-constraint pattern with a fixed label set prevents the model from inventing new categories
- The persona pattern shifts the model's output distribution by assigning a specific role
- The template pattern treats generation as filling in structured blanks
- Match the pattern to the task — do not apply a single pattern to everything

## Recap in 60 Seconds

1. Recognize the task type and select the matching pattern rather than designing from scratch every time
2. Use one primary pattern per API call — combining too many patterns confuses the model
3. Always validate pattern outputs and test across diverse inputs before going to production
