# Prompt Patterns

*Reading time: 5 minutes*

**Difficulty:** Beginner

## What You Will Learn

By the end of this lesson you will be able to:

- Identify and apply the most effective prompt patterns for different tasks
- Select the right pattern based on your goal (classification, extraction, generation, reasoning)
- Combine multiple patterns when a single pattern is insufficient
- Recognize when a pattern is being misapplied

## Why This Matters

Prompt patterns are reusable solutions to common LLM interaction problems. Instead of designing every prompt from scratch, you learn a library of patterns and compose them like functions. This makes your prompts more predictable, easier to debug, and faster to write.

Experienced AI engineers recognize patterns in user requirements: "This is a classification task — use the label-constraint pattern." "The user wants to compare options — use the comparison matrix pattern." "The model needs to reason through a complex problem — use the step-back pattern." Pattern recognition turns prompt engineering from an art into an engineering discipline.

## The Problem

You are asked to build five different LLM features: a content classifier, a data extractor, a summarizer, a comparison tool, and a creative writer. Each seems different, so you write five completely different prompts from scratch. The classifier prompt uses a different structure than the extractor. The summarizer handles formatting differently than the comparator.

Maintaining these prompts is a nightmare. Each has its own conventions, its own quirks, and its own failure modes. When you need to update the output format across all five, you have to edit each one individually.

What if, instead, all five used a shared pattern with the same structure, just different content? This is the value of prompt patterns.

## The Concept

A prompt pattern is a reusable template structure for a class of tasks. Each pattern has:

- A **structure** — the order and grouping of instructions
- A **rationale** — why this structure works for this task type
- **Variants** — modifications for different scenarios
- **Anti-patterns** — common ways to misuse it

The core patterns every AI engineer should know:

**Label-constraint pattern** — used for classification. Give the model a fixed set of labels and require it to pick exactly one. Prevents the model from inventing new categories.

**Persona pattern** — assign the model a specific role. Changes vocabulary, depth, and perspective. Useful when the same content must be presented differently to different audiences.

**Template pattern** — provide a structured format with slots. Fill in the slots with specific content. Useful for data extraction, form filling, and structured generation.

**Comparison pattern** — ask the model to evaluate options against explicit criteria. Prevents the model from making vague, unsupported comparisons.

**Chain-of-thought pattern** — ask the model to reason step by step before answering. Improves accuracy on multi-step reasoning tasks. (Covered in depth in its own lesson.)
