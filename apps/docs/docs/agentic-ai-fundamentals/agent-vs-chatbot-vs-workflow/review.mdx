# Agent vs. Chatbot vs. Workflow — Review

*Reading time: 3 minutes*

## Key Takeaways

- Chatbots generate text but cannot act; workflows follow fixed paths; agents use LLMs to decide the next action
- The same task can be implemented as a chatbot, workflow, or agent — each with different trade-offs in cost, latency, and reliability
- Start with the simplest architecture and escalate to more complex ones only when needed
- A triage system that classifies requests and routes to the appropriate handler combines all three architectures effectively
- Every architecture needs an escape hatch for cases it cannot handle

## Recap in 60 Seconds

1. Let the task's need for autonomy determine the architecture — not your excitement about the technology
2. Chatbots are cheap and simple but cannot act; workflows are reliable but rigid; agents are flexible but expensive and unpredictable
3. Build a triage layer to route simple requests to chatbots, routine lookups to workflows, and complex cases to agents
