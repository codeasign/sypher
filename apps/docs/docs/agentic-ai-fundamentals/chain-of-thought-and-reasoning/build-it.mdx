# Chain of Thought and Reasoning — Build It

*Reading time: 12 minutes*

## Breaking It Down

### 1. Direct Answer vs. Chain of Thought

Compare the direct approach with zero-shot CoT on a reasoning problem:

```python
import anthropic

client = anthropic.Anthropic()


def direct_answer(question: str) -> str:
    """Ask the model to answer directly, no reasoning."""
    response = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=500,
        temperature=0,
        messages=[{"role": "user", "content": question}]
    )
    return response.content[0].text


def chain_of_thought(question: str) -> str:
    """Ask the model to think step by step before answering."""
    response = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=500,
        temperature=0,
        system="First, reason step by step inside <reasoning> tags. Then provide the final answer inside <answer> tags.",
        messages=[{"role": "user", "content": question}]
    )
    return response.content[0].text


question = """
A store sells apples for $0.50 each and oranges for $0.75 each.
If Sarah buys 4 apples and 3 oranges, and pays with a $10 bill,
how much change does she receive?
"""

print("=== DIRECT ANSWER ===")
print(direct_answer(question))

print("\n=== CHAIN OF THOUGHT ===")
print(chain_of_thought(question))
```

The direct answer might be correct (it often is for simple problems). The CoT answer shows the full reasoning — cost of apples, cost of oranges, total, subtraction from $10. If the answer is wrong, the CoT trace tells you exactly which step failed.

### 2. Zero-Shot Chain of Thought

The simplest CoT technique: append "Let's think step by step" to the prompt:

```python
def zero_shot_cot(problem: str) -> str:
    """Zero-shot chain of thought — just ask for step-by-step reasoning."""
    response = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=500,
        temperature=0,
        messages=[{
            "role": "user",
            "content": f"{problem}\n\nLet's think step by step."
        }]
    )
    return response.content[0].text


problems = [
    "A bat and a ball cost $1.10 in total. The bat costs $1.00 more than the ball. How much does the ball cost?",
    "If it takes 5 machines 5 minutes to make 5 widgets, how long would it take 100 machines to make 100 widgets?",
    "In a race, if you pass the person in second place, what position are you in?",
]

for problem in problems:
    print(f"\nProblem: {problem[:60]}...")
    print(f"Answer: {zero_shot_cot(problem)}")
```

Zero-shot CoT works remarkably well. The simple phrase "Let's think step by step" triggers the model's latent reasoning capability. For the bat-and-ball problem (which famously trips up humans who answer intuitively), the model walks through the algebra correctly.

### 3. Few-Shot Chain of Thought

For more complex reasoning, provide examples showing the reasoning format:

```python
def few_shot_cot(problem: str) -> str:
    """Few-shot chain of thought with reasoning examples."""
    messages = [
        {"role": "user", "content": "A bakery bakes 120 loaves of bread each day. They sell 3/4 of them by noon. How many loaves do they sell by noon?"},
        {"role": "assistant", "content": "<reasoning>Total loaves: 120\nSold by noon: 3/4 of 120 = 120 * 3/4 = 90\nCheck: 90 is less than 120, which makes sense.</reasoning>\n<answer>90</answer>"},
        {"role": "user", "content": "A train travels at 60 miles per hour. How far does it travel in 45 minutes?"},
        {"role": "assistant", "content": "<reasoning>Speed: 60 mph\nTime: 45 minutes = 45/60 = 0.75 hours\nDistance = speed × time = 60 × 0.75 = 45 miles</reasoning>\n<answer>45 miles</answer>"},
        {"role": "user", "content": problem}
    ]

    response = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=500,
        temperature=0,
        messages=messages
    )
    return response.content[0].text


complex_problem = """
A library has 2,500 books. 40% are fiction. Of the fiction books, 30% are science fiction.
How many science fiction books are in the library?
"""
print(few_shot_cot(complex_problem))
```

The few-shot examples define the reasoning format (inside tags) and show the expected level of detail. The model follows the pattern for new problems.

## How It Works Internally

Chain-of-thought prompting exploits the autoregressive nature of language models. When the model generates text, each token's prediction depends on all previous tokens. By generating intermediate reasoning tokens, the model creates additional context that informs the final answer.

The reasoning tokens serve as scratch space. The model uses them to offload intermediate computations that would otherwise have to be compressed into a single forward pass. This is analogous to a human working through a math problem on paper rather than doing it all in their head.

The structured format with `<reasoning>` and `<answer>` tags is important because it separates the reasoning process from the final answer. Your application code can extract the answer from the tags for programmatic use while retaining the reasoning trace for debugging and auditing.

## Build Something Real

Build a reasoning evaluator that compares direct and CoT answers on multiple problems:

```python
import anthropic
import re

client = anthropic.Anthropic()


def evaluate_with_cot(problem: str) -> dict:
    """Solve a problem with chain of thought and return structured results."""
    response = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=500,
        temperature=0,
        system=(
            "Solve the problem step by step. Put your reasoning in <reasoning> tags "
            "and your final answer in <answer> tags. "
            "The answer should be a single number or short phrase."
        ),
        messages=[{"role": "user", "content": problem}]
    )

    text = response.content[0].text

    # Extract reasoning and answer from tags
    reasoning_match = re.search(r"<reasoning>(.*?)</reasoning>", text, re.DOTALL)
    answer_match = re.search(r"<answer>(.*?)</answer>", text, re.DOTALL)

    return {
        "full_output": text,
        "reasoning": reasoning_match.group(1).strip() if reasoning_match else "",
        "answer": answer_match.group(1).strip() if answer_match else "",
        "input_tokens": response.usage.input_tokens,
        "output_tokens": response.usage.output_tokens,
    }


# Test problems covering different reasoning domains
problems = [
    {
        "id": "math_1",
        "question": "A company has 48 employees. 2/3 of them work remotely. How many remote employees are there?",
    },
    {
        "id": "logic_1",
        "question": "All cats are mammals. Some mammals are dogs. Therefore, some cats are dogs. Is this conclusion valid? Explain why or why not.",
    },
    {
        "id": "planning_1",
        "question": "You need to bake a cake that takes 45 minutes in the oven and 15 minutes to cool. You also need to make frosting that takes 10 minutes. The party starts at 3:00 PM. What is the latest time you can start baking?",
    },
]

for problem in problems:
    print(f"\n{'='*60}")
    print(f"Problem [{problem['id']}]: {problem['question'][:80]}...")
    result = evaluate_with_cot(problem["question"])
    print(f"\nReasoning:\n{result['reasoning']}")
    print(f"\nAnswer: {result['answer']}")
    print(f"Tokens: {result['input_tokens']}in / {result['output_tokens']}out")
```

The reasoning evaluator gives you structured access to the model's thinking process. You can compare answers across problems, audit reasoning quality, and measure the token cost of the CoT approach.
