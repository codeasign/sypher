# Chain of Thought and Reasoning — Review

*Reading time: 3 minutes*

## Key Takeaways

- Chain-of-thought prompting forces the model to externalize its reasoning, dramatically improving accuracy on multi-step tasks
- Zero-shot CoT ("Let's think step by step") works surprisingly well without any examples
- Few-shot CoT with reasoning examples keeps the model on track for complex problems
- Use structured tags like `<reasoning>` and `<answer>` to separate reasoning from the final answer
- CoT is not free — it adds latency and token costs. Use it only when it measurably improves results

## Recap in 60 Seconds

1. If a problem requires multiple steps or implicit assumptions, use CoT — the intermediate tokens serve as scratch space
2. Structure your CoT output with tags so you can extract the answer programmatically from the reasoning trace
3. Always benchmark CoT against direct answering — if accuracy does not improve, drop it
