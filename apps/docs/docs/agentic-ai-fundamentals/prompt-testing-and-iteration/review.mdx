# Prompt Testing and Iteration — Review

*Reading time: 3 minutes*

## Key Takeaways

- Treat prompts like code — test them against a curated dataset before deploying
- A good test dataset has 20-100 cases covering all categories, edge cases, and boundary conditions
- Use an A/B testing framework to compare prompt versions on the same dataset
- Evaluate per-category accuracy, not just overall accuracy — minority classes matter
- Reserve a held-out validation set to detect overfitting

## Recap in 60 Seconds

1. Build a test dataset with at least 20-30 diverse examples including edge cases, then run every prompt change against it
2. Track per-category accuracy — overall accuracy hides failures on rare but critical categories
3. Version your prompts and their evaluation reports so you can detect regressions and roll back when needed
