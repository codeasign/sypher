# System Prompts — Review

*Reading time: 3 minutes*

## Key Takeaways

- System prompts set persistent behavior for the entire conversation — they are not part of the messages array
- Use system prompts for invariants (role, tone, rules) and user messages for the current task
- Different roles in the system prompt produce fundamentally different outputs from the same user message
- System prompts with explicit guardrails constrain the model's scope effectively
- Version your system prompts and update them when the application changes

## Recap in 60 Seconds

1. The system prompt is the constitution — it sets rules that last the entire conversation, so invest time in getting it right
2. Split concerns: behavioral rules go in the system prompt, active tasks go in the user message
3. Combine system prompts with output validation — prompting reduces bad behavior but does not eliminate it entirely
