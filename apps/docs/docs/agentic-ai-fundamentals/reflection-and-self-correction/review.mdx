# Reflection and Self-Correction — Review

*Reading time: 3 minutes*

## Key Takeaways

- Reflection asks the model to critique its own output before delivering it to the user
- The self-correction loop: produce, reflect, revise, repeat until passing or max revisions
- Use a scoring rubric with clear pass/fail thresholds to prevent over-correction
- Set a hard limit on revisions (2-3) to prevent infinite correction loops
- The reflection model must be at least as capable as the generating model

## Recap in 60 Seconds

1. Reflection turns the model from a generator into an evaluator — it checks its own work
2. Set max_revisions to 2-3 to prevent runaway loops and control costs
3. If the reflection cannot quote specific evidence for an issue, the issue may not be real
