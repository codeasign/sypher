# Agent Tools and Tool Calling — Review

*Reading time: 3 minutes*

## Key Takeaways

- Every tool has three parts: a definition (JSON schema), an implementation (the function), and a result (returned to the model)
- Tool descriptions are the model's primary signal for choosing between tools — make them specific and distinct from each other
- Always validate tool arguments before passing them to the implementation function
- Handle parallel tool calls — the model may request multiple tools in a single response
- Return descriptive error messages so the model can recover from tool failures

## Recap in 60 Seconds

1. A tool's name and description determine whether the model calls the right one — invest in clear, contrasting descriptions
2. Always validate arguments, always wrap execution in try/except, always return the result to the model
3. Keep each tool focused on one operation; avoid tools with more than 5 required parameters
