# Prompt Engineering Basics — Build It

*Reading time: 10 minutes*

## Breaking It Down

### 1. Vague Prompt vs. Structured Prompt

Start by seeing the difference between an underspecified prompt and a well-structured one:

```python
import anthropic

client = anthropic.Anthropic()


def ask_vague():
    """A prompt with no structure or constraints."""
    return client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=300,
        temperature=0,
        messages=[
            {"role": "user", "content": "Tell me about Python decorators."}
        ]
    ).content[0].text


def ask_structured():
    """A prompt with role, task, format, and constraints."""
    return client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=300,
        temperature=0,
        system=(
            "You are a senior Python developer teaching junior engineers."
        ),
        messages=[
            {
                "role": "user",
                "content": (
                    "Explain Python decorators.\n\n"
                    "Task: Provide a definition, a simple code example, and "
                    "one real-world use case.\n\n"
                    "Format: Use three sections with headers. "
                    "Each code example must be complete and runnable.\n\n"
                    "Constraints:\n"
                    "- Definition: max 3 sentences\n"
                    "- Code example: max 15 lines\n"
                    "- Use case: max 4 sentences\n"
                    "- Target audience: a developer who knows functions and classes"
                )
            }
        ]
    ).content[0].text


print("=== VAGUE PROMPT ===")
print(ask_vague()[:200])

print("\n=== STRUCTURED PROMPT ===")
print(ask_structured()[:200])
```

The vague prompt returns a wandering introduction. The structured prompt returns precisely what was asked for. The difference is not in the model — it is in the prompt design.

### 2. Role Prompting

The role you assign fundamentally shapes the output:

```python
def ask_with_role(role: str) -> str:
    """Test how different roles affect the response."""
    response = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=150,
        temperature=0,
        system=f"You are {role}.",
        messages=[
            {"role": "user", "content": "Explain what an API is."}
        ]
    )
    return response.content[0].text


roles = ["a 10-year-old child", "a senior software engineer", "a product manager"]

for role in roles:
    print(f"\n=== Role: {role} ===")
    print(ask_with_role(role))
```

Each role produces a distinctly different explanation — appropriate vocabulary, examples, and level of detail for the assigned persona.

### 3. The Format Constraint

Explicit format instructions are critical for programmatic consumption:

```python
def ask_with_format(format_instruction: str) -> str:
    """Test how format constraints affect output structure."""
    response = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=200,
        temperature=0,
        messages=[
            {
                "role": "user",
                "content": (
                    f"List 3 machine learning framework libraries.\n\n"
                    f"Format instruction: {format_instruction}"
                )
            }
        ]
    )
    return response.content[0].text


# Compare with no format instruction
print("No format instruction:")
print(ask_with_format("No specific format needed."))

# Compare with JSON format
print("\nJSON format:")
print(ask_with_format("Return as a JSON array of objects with keys: name, company, primary_use"))

# Compare with table format
print("\nMarkdown table:")
print(ask_with_format("Return as a markdown table with columns: name | company | type"))
```

The JSON format instruction returns programmatically usable data. The table format returns a nicely formatted table. Without a format instruction, the model might return any structure it chooses.

## How It Works Internally

When the model processes a prompt, every token in the system message and user message influences the attention patterns in the transformer layers. The role instruction ("You are a senior engineer") modifies the distribution of output tokens toward the statistical patterns associated with that role in the training data. Format constraints work the same way — they bias the model toward token sequences that look like the requested structure.

This is why prompt ordering matters. Instructions at the end of a long prompt sometimes have less influence than instructions at the beginning (a phenomenon called recency bias in some models, or primacy bias in others). The system prompt, which appears before the conversation, sets the overall context that influences how every subsequent message is interpreted.

## Build Something Real

Build a prompt testing harness that compares different prompt structures side by side:

```python
import anthropic
from dataclasses import dataclass

client = anthropic.Anthropic()


@dataclass
class PromptTemplate:
    """A reusable prompt structure for testing."""
    role: str
    task: str
    format: str
    constraints: list[str]


def evaluate_prompt(template: PromptTemplate) -> dict:
    """Run a prompt and return the output along with metadata."""
    constraints_text = "\n".join(f"- {c}" for c in template.constraints)
    prompt = (
        f"{template.task}\n\n"
        f"Format: {template.format}\n\n"
        f"Constraints:\n{constraints_text}"
    )

    response = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=300,
        temperature=0,
        system=f"You are {template.role}.",
        messages=[{"role": "user", "content": prompt}]
    )

    return {
        "output": response.content[0].text,
        "input_tokens": response.usage.input_tokens,
        "output_tokens": response.usage.output_tokens
    }


# Test two prompt designs for the same task
prompt_a = PromptTemplate(
    role="a technical writer",
    task="Explain the differences between SQL and NoSQL databases.",
    format="Use a comparison table.",
    constraints=["Max 100 words total", "Focus on practical use cases"]
)

prompt_b = PromptTemplate(
    role="a senior database architect",
    task="Explain the differences between SQL and NoSQL databases.",
    format="Write 3 paragraphs covering: data model, scalability, and use cases.",
    constraints=["Write for an audience of junior developers", "Include a concrete example of when to use each"]
)

for name, prompt in [("Prompt A: Technical writer", prompt_a), ("Prompt B: DB architect", prompt_b)]:
    result = evaluate_prompt(prompt)
    print(f"\n=== {name} ===")
    print(f"Input tokens: {result['input_tokens']}, Output tokens: {result['output_tokens']}")
    print(f"Output:\n{result['output'][:150]}...")
```

This harness lets you iterate on prompt design systematically — you change one element at a time and compare results objectively.
