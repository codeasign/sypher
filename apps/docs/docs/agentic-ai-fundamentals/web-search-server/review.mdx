# Review: Web Search MCP Server

⏱️ 5 minutes

## Key Takeaways

1. A web search MCP server bridges the agent's static training data with live internet content through tools like `brave_web_search` and `fetch_page`
2. Always specify result freshness for time-sensitive queries — the default ranking favors authority over recency
3. Use the minimum `count` value needed to conserve API quota and reduce latency
4. The `fetch_page` tool must strip HTML, trim content, and reject non-HTTPS URLs for safe operation
5. Cache similar queries client-side to avoid burning through API quota on repeated requests

## Recap in 60 Seconds

If you only remember three things:

1. **Freshness matters** — for current topics, use the news search tool or pass a `freshness` parameter
2. **Do not trust snippets** — they are truncated and often incomplete; fetch the full page for authoritative answers
3. **Protect your API key** — strip it from error messages before returning them to the agent
