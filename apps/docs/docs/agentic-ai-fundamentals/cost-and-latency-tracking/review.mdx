# Review: Cost and Latency Tracking

⏱️ 5 minutes

## Key Takeaways

1. Track input and output token costs separately — output tokens cost 3-5x more per token and contribute roughly half of total LLM cost
2. Model the cumulative context window growth — each tool call appends its result to the conversation, making later calls more expensive
3. Tool call count is the highest-leverage optimization target — each call adds one full LLM round-trip (2-10 seconds plus $0.02-$0.10)
4. Report p95 latency, not average — the tail matters more for user experience than the mean
5. Always track cost alongside task success rate — optimizing latency or cost at the expense of correctness is a net loss

## Recap in 60 Seconds

If you only remember three things:

1. **Track all three** — cost, latency, and tool call count; they move together
2. **Context window grows** — the 10th LLM call costs more than the 1st because it includes all prior history
3. **Tool calls drive everything** — reducing tool calls from 8 to 4 roughly halves both cost and latency
