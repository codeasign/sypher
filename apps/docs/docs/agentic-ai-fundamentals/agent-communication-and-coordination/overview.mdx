# Agent Communication and Coordination

*Reading time: 6 minutes*

**Difficulty:** Advanced

- Choose between synchronous and asynchronous agent communication

## Why This Matters

Giving each agent a narrow tool set solves accuracy and safety, but it introduces a new problem: how do agents share information? If the research agent discovers a crucial fact that changes the writer's direction, how does that fact reach the writer?

Without a communication protocol, agents become silos. Each agent produces output that is useful only to the orchestrator, not to other agents. This works for sequential pipelines, but many tasks require two-way collaboration — a reviewer should be able to ask the writer for revisions, and a research agent should be able to ask the data analyst for supporting numbers.

In production multi-agent systems, communication is the hardest problem to get right. An agent that misunderstands another agent's output can cascade errors through the system. A missing message can leave the system waiting forever. A poorly designed message format can force agents to parse free-text messages, defeating the purpose of structured data exchange.

## The Problem

You have a research agent and a writing agent working on the same document. The research agent finds five data points. The writing agent only uses three because the other two were buried in a long paragraph of free text. The reviewer rejects the document for missing data. The research agent never knew the writer missed its findings.

Without structured communication, agents lose information in translation. Each agent packages its output in its own format, and the receiving agent must parse and interpret it. This is fragile, error-prone, and impossible to debug when things go wrong. You need a shared communication protocol that all agents agree on.

## The Concept

Agent communication works best when it is structured, typed, and asynchronous. Instead of agents sending free-text messages to each other, they send structured payloads — dictionaries or typed objects — with known schemas. A message bus or registry handles delivery and ensures messages are not lost.

There are two fundamental coordination patterns:

**Direct messaging** — one agent sends a message directly to another agent by name. The sender knows who it is talking to. This works well when the communication pattern is known in advance (research always feeds to writing).

**Message bus** — agents publish messages to named channels, and any agent subscribed to that channel receives them. The sender does not know who is listening. This works well for broadcast events (e.g., "database schema changed") and for systems where agents are added and removed dynamically.
