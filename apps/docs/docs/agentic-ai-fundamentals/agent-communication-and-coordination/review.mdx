# Agent Communication and Coordination — Review

*Reading time: 3 minutes*

## Key Takeaways

- Inter-agent communication must use structured, typed messages with known schemas — free text causes parsing errors and context loss
- Direct messaging works for stable, known topologies; message buses work for dynamic, multi-agent systems
- Always define a conflict resolution strategy before agents produce contradictory results
- Use asynchronous messaging to avoid blocking the entire pipeline on a slow agent
- Every message needs a TTL — without it, a crashed agent stalls the system indefinitely

## Recap in 60 Seconds

1. Define a structured message schema (sender, recipient, type, payload) before building your agents — treat it like an API contract
2. Use a message bus for systems with three or more agents and dynamic topologies; use direct messaging for fixed two-agent pipelines
3. Always handle conflicts, timeouts, and out-of-order delivery — the happy path is not enough for production
