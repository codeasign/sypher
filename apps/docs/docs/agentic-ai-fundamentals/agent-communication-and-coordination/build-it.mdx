# Agent Communication and Coordination — Build It

*Reading time: 15 minutes*

## Breaking It Down

### 1. Structured Message Format

Every message between agents follows a known schema. This prevents the parsing errors and context loss that happen with free-text communication.

```python
from dataclasses import dataclass, field
from datetime import datetime, timezone


@dataclass
class AgentMessage:
    """A structured message that agents exchange."""
    sender: str
    recipient: str | None  # None = broadcast
    message_type: str       # "data", "request", "response", "error"
    payload: dict
    timestamp: str = field(default_factory=lambda: datetime.now(timezone.utc).isoformat())
    message_id: str = ""

    def to_dict(self) -> dict:
        return {
            "sender": self.sender,
            "recipient": self.recipient,
            "message_type": self.message_type,
            "payload": self.payload,
            "timestamp": self.timestamp,
            "message_id": self.message_id,
        }
```

### 2. Direct Messaging

In direct messaging, the sender specifies the recipient by name. This pattern works when the communication topology is stable and known ahead of time.

```python
@dataclass
class DirectMessenger:
    """Direct messaging system where agents address each other by name."""
    mailboxes: dict[str, list[AgentMessage]] = field(default_factory=dict)

    def register_agent(self, name: str):
        """Register an agent's mailbox."""
        if name not in self.mailboxes:
            self.mailboxes[name] = []

    def send(self, message: AgentMessage):
        """Send a message directly to the named recipient."""
        if message.recipient not in self.mailboxes:
            raise ValueError(f"Unknown recipient: {message.recipient}")
        self.mailboxes[message.recipient].append(message)

    def receive(self, agent_name: str) -> list[AgentMessage]:
        """Retrieve all pending messages for an agent."""
        messages = list(self.mailboxes.get(agent_name, []))
        self.mailboxes[agent_name] = []
        return messages


# Example: Research agent sends structured data to the writer
messenger = DirectMessenger()
messenger.register_agent("researcher")
messenger.register_agent("writer")

research_msg = AgentMessage(
    sender="researcher",
    recipient="writer",
    message_type="data",
    payload={
        "findings": [
            {"key": "market_size", "value": "4.2B", "confidence": 0.9},
            {"key": "growth_rate", "value": "15.6%", "confidence": 0.85},
        ],
        "sources": ["Industry Report 2025"],
    },
    message_id="msg-001",
)

messenger.send(research_msg)

# Writer checks its inbox
inbox = messenger.receive("writer")
for msg in inbox:
    print(f"From: {msg.sender}, Type: {msg.message_type}")
    print(f"Data points received: {len(msg.payload['findings'])}")
```

### 3. Message Bus (Pub/Sub)

The message bus decouples senders from receivers. Agents publish to channels, and any agent subscribed to a channel receives the message. This pattern scales to many agents and dynamic topologies.

```python
@dataclass
class MessageBus:
    """Pub/sub message bus for agent communication."""
    subscriptions: dict[str, list[str]] = field(default_factory=dict)
    queues: dict[str, list[AgentMessage]] = field(default_factory=dict)

    def subscribe(self, agent_name: str, channel: str):
        """Subscribe an agent to a channel."""
        if channel not in self.subscriptions:
            self.subscriptions[channel] = []
            self.queues[channel] = []
        self.subscriptions[channel].append(agent_name)

    def publish(self, message: AgentMessage, channel: str):
        """Publish a message to all subscribers of a channel."""
        self.queues.setdefault(channel, []).append(message)

    def poll(self, agent_name: str) -> list[AgentMessage]:
        """Poll all channels for messages intended for this agent."""
        messages = []
        for channel, subscribers in self.subscriptions.items():
            if agent_name in subscribers:
                queue = self.queues.get(channel, [])
                # Collect and clear this agent's messages
                for msg in queue:
                    if msg.recipient is None or msg.recipient == agent_name:
                        messages.append(msg)
                # Remove consumed messages from queue
                self.queues[channel] = [
                    m for m in queue
                    if m not in messages
                ]
        return messages


# Example: agents collaborate on a document via the bus
bus = MessageBus()
bus.subscribe("researcher", "document-workflow")
bus.subscribe("writer", "document-workflow")
bus.subscribe("reviewer", "document-workflow")

# Research agent publishes findings
bus.publish(
    AgentMessage(
        sender="researcher",
        recipient=None,  # broadcast to all subscribers
        message_type="data",
        payload={"findings": "Market data collected"},
        message_id="msg-002",
    ),
    channel="document-workflow",
)

# Reviewer publishes quality feedback
bus.publish(
    AgentMessage(
        sender="reviewer",
        recipient="writer",  # direct within the bus
        message_type="request",
        payload={"revisions": ["Add source citations to section 2"]},
        message_id="msg-003",
    ),
    channel="document-workflow",
)

# Writer polls for updates
updates = bus.poll("writer")
print(f"Writer received {len(updates)} messages:")
for u in updates:
    print(f"  - From: {u.sender}, Type: {u.message_type}")
```

### 4. Conflict Resolution

When two agents produce contradictory results, the system needs a conflict resolution strategy. The simplest approach is a voting mechanism where a third agent or the orchestrator breaks the tie.

```python
from enum import Enum


class ConflictStrategy(Enum):
    MAJORITY_VOTE = "majority_vote"
    TRUST_SCORE = "trust_score"
    SUPERVISOR_DECIDES = "supervisor_decides"
    LATEST_WINS = "latest_wins"


@dataclass
class ConflictResolver:
    """Resolves conflicts between agents producing contradictory results."""
    strategy: ConflictStrategy = ConflictStrategy.SUPERVISOR_DECIDES
    trust_scores: dict[str, float] = field(default_factory=dict)

    def resolve(self, claims: list[dict]) -> dict:
        """Resolve a set of conflicting claims from different agents."""
        if not claims:
            return {}

        if len(claims) == 1:
            return claims[0]

        if self.strategy == ConflictStrategy.MAJORITY_VOTE:
            return self._majority_vote(claims)

        elif self.strategy == ConflictStrategy.TRUST_SCORE:
            return self._trust_based(claims)

        elif self.strategy == ConflictStrategy.LATEST_WINS:
            return max(claims, key=lambda c: c.get("timestamp", ""))

        else:
            # SUPERVISOR_DECIDES — use the highest-confidence agent
            return self._trust_based(claims)

    def _majority_vote(self, claims: list[dict]) -> dict:
        """Return the most common value across conflicting agents."""
        values = {}
        for claim in claims:
            key = str(claim.get("value"))
            values[key] = values.get(key, 0) + 1
        winner = max(values, key=values.get)
        for claim in claims:
            if str(claim.get("value")) == winner:
                return claim
        return claims[0]

    def _trust_based(self, claims: list[dict]) -> dict:
        """Return the claim from the highest-trust agent."""
        scored = [
            (claim, self.trust_scores.get(claim.get("source", ""), 0.5))
            for claim in claims
        ]
        return max(scored, key=lambda x: x[1])[0]


# Example: three agents report different sales numbers
resolver = ConflictResolver(strategy=ConflictStrategy.MAJORITY_VOTE)

claims = [
    {"source": "database_agent", "value": 2045000, "timestamp": "2025-06-01"},
    {"source": "analytics_agent", "value": 2100000, "timestamp": "2025-06-01"},
    {"source": "api_agent", "value": 2045000, "timestamp": "2025-05-30"},
]

resolved = resolver.resolve(claims)
print(f"Resolved value: {resolved['value']} (from {resolved['source']})")
```

## How It Works Internally

When an agent receives a message, it does not merely append it to conversation history. The agent's message-handling layer should parse the structured payload, validate it against the expected schema, and integrate it into the agent's context in a way that distinguishes it from user input and tool results. This prevents the agent from treating structured data as free-text instructions.

## Performance Considerations

The message bus introduces a polling overhead. Each agent that polls the bus reads through all unprocessed messages to find its own. For systems with many agents, use a channel-per-agent approach so agents poll only their own queue rather than scanning all channels. The performance impact becomes noticeable when you have more than 20 agents polling the same bus.

## Security Considerations

Never let agents parse raw messages from untrusted sources within the system. Validate that every message conforms to the expected schema before passing it to an agent's reasoning loop. An agent that receives a malformed message could misinterpret it as a task instruction, which is a prompt injection vector within your own system.
