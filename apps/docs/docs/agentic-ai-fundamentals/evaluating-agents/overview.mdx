## Why This Matters

Agent evaluation is the hardest open problem in production AI engineering today. Unlike a traditional ML model where you measure accuracy on a held-out test set, an agent is a stochastic system that makes multiple decisions, calls tools, and interacts with an environment. Its outputs are not just text — they are sequences of actions, some of which change real-world state.

Companies deploying agents in production report that evaluation accounts for 40-60% of their engineering effort. The reason is fundamental: agents are non-deterministic. The same prompt with the same model can produce different tool call sequences across runs. A fix that improves one task often breaks another. Without rigorous evaluation, you ship agents that work 80% of the time and fail in unpredictable ways.

The stakes are high. A customer support agent that occasionally hallucinates a refund gives away real money. A code generation agent that sometimes introduces a security vulnerability creates a liability. A deployment agent that picks the wrong version takes down production. Evaluation is how you catch these failures before they reach users.

## The Problem

Your agent answers a user's question by searching the web, reading a file, running code, and synthesizing an answer. Is that the right answer? Did the agent pick the right search query? Did it read the correct file? Did it waste three unnecessary tool calls? Did it handle the edge case where the file does not exist? Did it fabricate data when the search returned no results?

Traditional evaluation metrics like accuracy or BLEU score do not capture these dimensions. You need a framework that measures:

- **Did the agent achieve the user's goal?** (task success)
- **How efficiently did it do so?** (cost, latency, tool call count)
- **Did it make any unsafe decisions?** (safety violations)
- **How does it handle failures?** (robustness)
- **Does it degrade gracefully under pressure?** (stress testing)

Each of these requires different evaluation methods, different datasets, and different metrics.

## The Concept

Agent evaluation is a multi-dimensional measurement framework. You define the tasks your agent should perform, create a test dataset with expected outcomes, run the agent against those tasks, and measure its performance across several axes.

The key insight is that you evaluate at three levels:

1. **Step level** — did the agent pick the right tool for each step?
2. **Trajectory level** — was the overall sequence of steps efficient and correct?
3. **Outcome level** — did the end result satisfy the user's goal?

Most teams start with outcome-level evaluation (pass/fail on the final result) and add trajectory and step-level evaluation as they discover specific failure modes. The goal is not perfection on every metric — it is understanding where your agent falls short so you can improve it.
