# Review: Evaluating Agents

⏱️ 5 minutes

## Key Takeaways

1. Agent evaluation requires three levels of measurement: step-level (single tool calls), trajectory-level (the full sequence), and outcome-level (the final result)
2. Always include at least 30% edge-case tasks in your evaluation dataset — the happy path is not the whole story
3. Use a different model for LLM-as-Judge evaluation than the model powering the agent to avoid shared blind spots
4. Track multiple metrics (success rate, latency, tool call count, safety violations) as a composite score rather than optimizing one in isolation
5. Rotate eval tasks regularly to prevent overfitting to a fixed test set

## Recap in 60 Seconds

If you only remember three things:

1. **Three levels matter** — outcome, trajectory, and step-level evaluation each catch different failure modes
2. **Judge with a different model** — do not use the same model to grade itself
3. **Balance your metrics** — success rate alone hides latency, cost, and safety problems
