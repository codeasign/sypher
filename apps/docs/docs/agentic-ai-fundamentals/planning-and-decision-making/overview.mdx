# Planning and Decision Making

*Reading time: 6 minutes*

**Difficulty:** Advanced

- Distinguish between flat planning, hierarchical planning, and dynamic replanning
- Implement a planner that decomposes complex tasks into subtasks
- Handle ambiguous or incomplete user requests with clarification steps
- Design decision-making strategies that balance exploration and exploitation

## Why This Matters

A planning agent without a planning strategy does not plan — it reacts. It takes whatever action the next token suggests, which may not be the action that makes progress toward the goal. Planning is what separates a systematic agent from a reactive one.

Production agent systems at companies like Google (Vertex AI Agent Builder) and Microsoft (AutoGen) use structured planning approaches. Their agents do not just execute the next plausible action — they maintain a plan, track progress, and adjust when steps fail. Without planning, agents chase dead ends, repeat completed steps, and miss the overall goal.

## The Problem

You give your agent a complex task: "Plan a team offsite for 20 people. Find a venue within a 2-hour drive, compare three catering options, check availability for the second Friday of next month, and draft an invitation email."

The agent calls `search_venues`, gets results, calls `compare_catering`, gets prices, calls `check_availability` — and discovers the venue is booked on that date. It does not go back to search for other venues. It reports "task failed" and stops.

The agent had no plan. It took the first action available at each step. When a step failed, it had no fallback strategy. A planning agent would have generated multiple venue options before checking availability, so a booking failure would trigger a fallback, not a task failure.

## The Concept

Planning in agents takes several forms:

**Flat planning**. The agent generates a complete plan upfront, then executes each step in order. This works for well-understood tasks where all steps are known in advance. The plan is fixed — execution failures require a full replan.

**Hierarchical planning**. The agent decomposes the task into subtasks, each of which may be further decomposed. A venue-search subtask might have sub-steps: search, check ratings, verify availability. Each level of the hierarchy is independently planned and executed.

**Dynamic replanning**. The agent maintains a current plan but revises it based on execution results. When a step fails, the agent replans from the current state rather than restarting. This is essential for real-world tasks where unexpected obstacles occur.

Decision-making strategies determine how the agent chooses between competing actions:

- **Greedy**: Always pick the action with the highest immediate reward. Fast but may miss better long-term outcomes.
- **Lookahead**: Simulate the consequences of each action before choosing. More expensive but more robust.
- **Approval-gated**: Propose the next action for human approval before executing. Slowest but safest.
