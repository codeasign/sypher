# ReAct Pattern — Avoid Mistakes

*Reading time: 5 minutes*

## When NOT to Use This

ReAct is powerful but not universal:

- **Tasks requiring no tool calls**: If every answer comes from the model's training data, ReAct adds unnecessary reasoning tokens.
- **Very simple tasks**: "What is 2+2?" does not need a thought-action loop.
- **Streaming or real-time systems**: ReAct's turn-based structure adds latency that streaming systems cannot tolerate.
- **When reasoning traces must be hidden**: The thought process is visible in the output. If you cannot expose the model's reasoning, do not use ReAct.

## The Pattern to Remember

**Pattern**: Interleave reasoning (Thought) with action (Action/Tool call) at every turn. Each turn produces a traceable decision.

When to use:
- Multi-step reasoning + tool calling tasks
- Applications where auditability matters
- Debugging complex agent behavior

When NOT to use:
- Single-turn lookups
- Real-time streaming
- Tasks requiring hidden reasoning

**Mental model**: ReAct is like showing your work on a math test. The grader can see every step. If the final answer is wrong, the steps show where the error occurred.

## Common Mistakes

### Mistake 1: The Model Does Not Externalize Reasoning

**Why it happens**: The system prompt does not explicitly ask for reasoning, or the model skips it to save tokens.

**Symptoms**: The agent calls tools without any visible reasoning. The trace is empty. Debugging is impossible.

**Fix**: Make reasoning mandatory. Use a system prompt that requires "Thought:" before every "Action:". Use structured tags like `<thought>` if the model consistently skips natural language reasoning.

### Mistake 2: Circular Reasoning

**Why it happens**: The model keeps thinking without taking action. It goes around in circles analyzing the same problem.

**Symptoms**: The trace shows multiple thought steps with no tool calls. The agent is overthinking.

**Fix**: Limit the number of consecutive thought-only turns. If the model produces 3 thoughts without an action, inject a prompt to choose a direction and act.

### Mistake 3: Reasoning That Contradicts Actions

**Why it happens**: The model's stated reasoning does not match the actual tool call it makes.

**Symptoms**: The thought says "I need to search the database" but the action is `send_email`. The trace shows conflicting information.

**Fix**: Validate that the action matches the stated intention. If they diverge, log a warning. In critical systems, abort and ask the model to re-evaluate.

### Mistake 4: Too Much Context from Past Thoughts

**Why it happens**: The reasoning trace accumulates every thought, filling the context window with internal monologue.

**Symptoms**: The context window fills with old thoughts rather than tool results. The agent loses factual information because thoughts crowd out observations.

**Fix**: Prune old thoughts. Keep the last 2-3 reasoning steps and discard earlier ones. Tool results and observations should take priority in the context window.

### Mistake 5: The Model Treats Its Own Reasoning as Fact

**Why it happens**: The model generates a plausible-sounding thought, then treats it as confirmed truth without evidence.

**Symptoms**: The trace shows "Thought: I know the capital of France is London" — the model reasons itself into a wrong conclusion and acts on it.

**Fix**: Prompt the model to distinguish between "what I know" and "what I need to verify." Facts should come from tool results, not from the model's training data.
