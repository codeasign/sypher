# Review: Tool Evaluation

⏱️ 5 minutes

## Key Takeaways

1. Tool evaluation measures invocation rate, parameter accuracy, error rate, and description relevance for every tool your agent uses
2. Distinguish between validation errors (agent passed wrong params) and execution errors (backend server failure) — they have different root causes and fixes
3. Include survivorship bias in your analysis — the tools the agent never calls may be the ones with the worst descriptions
4. Report p50 and p95 latency including error calls, not just successful ones, to capture the true tail
5. Optimize invocation rates against expected usage from eval tasks, not against balanced distributions across all tools

## Recap in 60 Seconds

If you only remember three things:

1. **Separate error categories** — validation errors are agent-side; execution errors are server-side
2. **Watch for hidden tools** — zero invocation rate means a poor description or conflicting tool design
3. **Measure what you do not see** — missed opportunities (expected tool not called) matter as much as errors
