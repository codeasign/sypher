# Agent Memory

*Reading time: 6 minutes*

**Difficulty:** Intermediate

## What You Will Learn

By the end of this lesson you will be able to:

- Distinguish between short-term, long-term, and episodic memory in agents
- Implement conversation history management within context window limits
- Design retrieval-augmented memory systems for long-running agents
- Choose the right memory strategy based on task duration and complexity

## Prerequisites

You should already understand:

- How context windows work (covered in [Tokens and Context Windows](../tokens-and-context-windows/overview))
- How agents maintain state across turns (covered in [Anatomy of an Agent](../anatomy-of-an-agent/overview))
- Basic vector database concepts for retrieval

## Why This Matters

An agent without memory answers every question as if it is the first question. It forgets what it just did, what the user told it five steps ago, and what results it already obtained. This is not just inefficient — it is dangerous. An agent that forgets it already booked a flight might book it again.

Production agents handle memory deliberately. They decide what to keep in the context window (immediate history), what to store externally (facts, summaries, embeddings), and what to discard (completed intermediate steps). The difference between a prototype agent that loses track after three turns and a production agent that runs for hours is entirely in the memory architecture.

## The Problem

Your research agent runs a series of web searches, reads articles, and compiles a report. After the fifth search, the context window is nearly full. The agent starts forgetting earlier search results. When it writes the final report, it omits the first two sources entirely because the token window pushed them out.

You cannot just increase the context window — that increases cost and latency. You need a memory system that selectively preserves important information and discards what is no longer needed. The agent must remember the key facts from all ten searches without keeping all ten raw search results in context.

## The Concept

Agent memory is divided into three types, inspired by cognitive science:

**Short-term memory**. The conversation history in the model's context window. This is the fastest memory to access and the most detailed. Its capacity is limited by the model's context window size. Short-term memory holds the current task state, recent observations, and pending actions.

**Long-term memory**. Information stored outside the context window and retrieved when needed. This can be a vector database of past conversations, a key-value store of facts the agent has learned, or a file system of completed work products. Long-term memory persists across sessions.

**Episodic memory**. Records of past episodes — what the agent did, what happened, what went wrong. Episodic memory helps the agent learn from experience within a single session. When a tool call fails, the agent records the failure and avoids the same mistake.

Memory management strategies:

- **Sliding window**: Keep the most recent N messages. Old messages are dropped regardless of importance. Simple but lossy.
- **Summarization**: Periodically summarize older conversation turns and replace them with the summary. Preserves information at reduced fidelity.
- **Retrieval-augmented**: Store all messages externally in a vector database. At each step, retrieve the most relevant messages back into context.
