# What Is MCP? — Review

*Reading time: 3 minutes*

## Key Takeaways

- MCP (Model Context Protocol) is an open standard protocol for connecting AI agents to tools, data, and prompts — think of it as USB-C for AI
- It uses JSON-RPC 2.0 for all messages, making it language-agnostic and well-structured
- Three core primitives: tools (callable functions), resources (readable data), and prompts (reusable templates)
- Every connection starts with an `initialize` handshake to negotiate protocol version and capabilities
- The `notifications/initialized` message is required after initialization — forgetting it is the most common integration mistake

## Recap in 60 Seconds

1. MCP standardizes how agents discover and use tools — one protocol for filesystem, database, Slack, and any other service
2. JSON-RPC 2.0 messages flow over stdio (local child processes) or SSE (remote HTTP servers) between MCP client and MCP server
3. Always complete the full initialization handshake (initialize request, initialize response, initialized notification) before making tool calls
