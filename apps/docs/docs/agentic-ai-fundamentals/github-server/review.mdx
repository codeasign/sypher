# Review: GitHub MCP Server

⏱️ 5 minutes

## Key Takeaways

1. The GitHub MCP server wraps the GitHub REST API into typed tools like `create_issue`, `search_code`, and `create_pull_request_review`
2. Authentication uses a GitHub personal access token set via the `GITHUB_TOKEN` environment variable
3. Always scope the token to the minimum set of repos and permissions — use fine-grained tokens over classic tokens
4. Rate limiting and pagination are handled by the official server, but custom implementations must address both
5. Combine the GitHub server with the filesystem server for powerful workflows — read code, then open issues or PRs based on findings

## Recap in 60 Seconds

If you only remember three things:

1. **Scope your token tightly** — a full `repo` token lets the agent delete repos; use fine-grained tokens with minimal permissions
2. **Combine servers** — filesystem server to read and understand code + GitHub server to act on it
3. **Handle pagination and rate limits** — the API returns at most 30 items per page and 5000 requests per hour
