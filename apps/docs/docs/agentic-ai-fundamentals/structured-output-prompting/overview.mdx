## Why This Matters

Natural language output is great for human consumption. But most production AI systems need to process model output programmatically — extract data, populate databases, trigger API calls, drive UI components. For the model output to be useful in code, it must be structured, predictable, and parseable.

The model can produce structured output, but it requires careful prompt design. A missing comma, an extra field, or an inconsistent key name breaks your parser. Teaching the model to produce reliable structured output is one of the highest-impact skills in AI engineering — it is the difference between "the output looks right" and "the output works in our system."

## The Problem

You ask the model to return customer information from an email. It returns:

```
Name: John Smith
Email: john@email.com
Order: #12345
Issue: late delivery
```

This looks parseable, until the model returns:

```
Customer: John Smith (john@email.com) — order #12345
Problem with delivery: package is late
```

The format changed. Your parser breaks. You add strict instructions, and the model follows them for a while, but occasionally a different format slips through on edge cases.

You need a reliable way to get the model to output data in a format your code can parse without failure — every time.

## The Concept

Structured output prompting is the set of techniques for getting LLMs to produce output in a machine-parseable format. The key approaches, from simplest to most powerful:

**Format specification in the prompt**: You describe the output format explicitly in the prompt. "Return a JSON object with keys: name, email, order_id, and issue." This works well for simple structures but can fail on complex schemas.

**Schema as a constraint**: You provide a JSON Schema or similar schema definition and instruct the model to follow it. This is more precise than natural language format descriptions.

**Few-shot with structured examples**: You show examples of the exact output format you want. The model learns the format from the examples more reliably than from instructions alone.

**JSON mode / Structured Outputs**: Some APIs offer a mode that constrains the model's output to valid JSON. The model still needs the schema in the prompt, but JSON mode guarantees parseable JSON syntax.

**Tool use / Function calling**: The most reliable approach. You define a tool or function with a typed schema, and the model's output is constrained to match that schema. The API handles the formatting.

The general rule: the more the API enforces the structure, the more reliable the output. Prompting for JSON works 80-90% of the time. JSON mode gets 95-99%. Tool use with a schema gets 99.9%.
