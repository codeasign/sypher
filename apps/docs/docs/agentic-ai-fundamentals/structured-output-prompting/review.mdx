# Structured Output Prompting — Review

*Reading time: 3 minutes*

## Key Takeaways

- Prompt-only structured output works 80-90% of the time; JSON mode gets 95-99%; tool use with schemas gets 99.9%
- Always validate and clean raw model output before parsing — strip markdown fences, search for JSON boundaries
- Validate not just JSON syntax but also the values and types against your schema
- Few-shot JSON examples teach the model the exact format more reliably than schema descriptions alone
- The output is an interface contract — version your schemas and handle violations gracefully

## Recap in 60 Seconds

1. The most reliable structured output comes from API-level constraints (tool use / function calling), not prompt instructions — prefer the API when you can
2. Always validate output after generation — valid JSON does not mean valid data for your schema
3. Handle real-world messiness: strip prefixes, fix trailing commas, retry on failure
