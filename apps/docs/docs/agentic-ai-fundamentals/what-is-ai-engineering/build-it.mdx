# What is AI Engineering — Build It

*Reading time: 8 minutes*

## Breaking It Down

### 1. Setting Up Your Environment

You need Python 3.10 or later and the Anthropic SDK. Create a virtual environment and install the dependency:

```bash
python -m venv .venv
source .venv/bin/activate   # On Windows: .venv\Scripts\activate
pip install anthropic
```

Store your API key as an environment variable rather than hardcoding it. Create a `.env` file or export it directly:

```bash
export ANTHROPIC_API_KEY="sk-ant-your-key-here"
```

Never commit API keys to version control. Add `.env` to your `.gitignore`.

### 2. Your First API Call

The simplest possible LLM interaction sends a message and prints the response. Here is a complete script:

```python
import anthropic
import os

# The SDK reads ANTHROPIC_API_KEY from the environment automatically
client = anthropic.Anthropic()

# Send a message and get a response
response = client.messages.create(
    model="claude-3-5-sonnet-20241022",
    max_tokens=256,
    temperature=0,
    system="You are a helpful assistant that gives concise answers.",
    messages=[
        {"role": "user", "content": "What is AI engineering in two sentences?"}
    ]
)

# Extract and print the response text
print(response.content[0].text)
```

Running this prints something like:

> AI engineering is the discipline of building production software systems that integrate large language models as components. It combines traditional software engineering practices with prompt engineering, evaluation, and orchestration to create reliable AI-powered applications.

The `client.messages.create()` method takes several important parameters. `model` selects which LLM to use. `max_tokens` limits the length of the response. `temperature` controls randomness — 0 makes output deterministic, higher values add variability. `system` sets the assistant's behavior. `messages` is the conversation history.

### 3. Understanding the Response Object

The response is a structured object with metadata you will use in production:

```python
response = client.messages.create(
    model="claude-3-5-sonnet-20241022",
    max_tokens=256,
    messages=[{"role": "user", "content": "Say hello"}]
)

# The response contains rich metadata
print(f"Model used: {response.model}")
print(f"Input tokens: {response.usage.input_tokens}")
print(f"Output tokens: {response.usage.output_tokens}")
print(f"Stop reason: {response.stop_reason}")
print(f"Response text: {response.content[0].text}")
```

The `usage` field shows token counts you need for cost tracking. The `stop_reason` tells you why the model stopped — `"end_turn"` means it finished naturally, `"max_tokens"` means you hit the length limit.

### 4. Making a Multi-Turn Conversation

Real applications send multiple messages to maintain context. Each call builds on the previous exchange:

```python
import anthropic

client = anthropic.Anthropic()

conversation = [
    {"role": "user", "content": "What is the capital of France?"}
]

response = client.messages.create(
    model="claude-3-5-sonnet-20241022",
    max_tokens=256,
    messages=conversation
)

# Add the assistant's response to the conversation
conversation.append(
    {"role": "assistant", "content": response.content[0].text}
)

# Ask a follow-up that depends on the previous answer
conversation.append(
    {"role": "user", "content": "What is the population of that city?"}
)

response = client.messages.create(
    model="claude-3-5-sonnet-20241022",
    max_tokens=256,
    messages=conversation
)

print(response.content[0].text)
```

Notice that the messages array grows with each turn. The LLM has no memory of previous calls — you must send the entire conversation history each time. This is called stateless interaction, and it is fundamental to how LLM APIs work.

## How It Works Internally

When you call `client.messages.create()`, several things happen behind the scenes:

1. **Serialization**: The SDK converts your Python objects into an HTTP request with a JSON body conforming to the Messages API schema.
2. **Authentication**: Your API key is sent as an HTTP header. The server validates it against your account.
3. **Inference**: The model processes your messages through its transformer architecture. Every token in your prompt attends to every other token. The model predicts the next token, then feeds that token back in as input, repeating until it reaches `max_tokens` or predicts a stop token.
4. **Response assembly**: The output tokens are decoded back to text, wrapped in metadata, and returned as the API response.
5. **Billing**: The provider counts input and output tokens from your request and deducts from your account balance.

The entire round trip takes anywhere from a few hundred milliseconds to tens of seconds depending on model size, response length, and current load.

## Build Something Real

Let's build a simple customer intent classifier — the kind of component that powers routing in AI support systems:

```python
import anthropic

client = anthropic.Anthropic()

def classify_intent(message: str) -> str:
    """Classify a customer message into one of four intent categories."""
    response = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=50,
        temperature=0,
        system=(
            "You classify customer messages into exactly one category: "
            "BILLING, TECHNICAL, ACCOUNT, or GENERAL. "
            "Respond with only the category name, nothing else."
        ),
        messages=[
            {"role": "user", "content": message}
        ]
    )
    return response.content[0].text.strip()

# Test with different customer messages
test_messages = [
    "I was charged twice for my subscription last month",
    "The login page returns a 500 error when I click submit",
    "I forgot my password and need to reset it",
    "What are your business hours during holidays?"
]

for msg in test_messages:
    intent = classify_intent(msg)
    print(f"Message: {msg}")
    print(f"Intent: {intent}\n")
```

The `system` prompt defines the assistant's behavior — in this case, a strict classification task. The `temperature=0` ensures consistent results. The output constraint ("Respond with only the category name") makes the output parseable without additional processing.
