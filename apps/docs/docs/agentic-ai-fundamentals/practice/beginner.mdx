## Exercise 1: Structured Prompt for Information Extraction

**Problem:** Write a system prompt and a function `extract_meeting_info(text: str) -> dict` that uses an LLM to extract the date, attendees list, and action items from a meeting transcript. Use structured output (JSON mode or tool use) so the result is always parseable.

**Expected input:**
```python
transcript = (
    "On March 15th, we had a sprint review with Alice, Bob, and Charlie. "
    "Alice will deploy the new auth module by Friday. Bob needs to update the API docs. "
    "Charlie is responsible for running the regression tests before the release."
)
result = extract_meeting_info(transcript)
```

**Expected output:** A dict with keys `date`, `attendees` (list), and `action_items` (list of strings), for example:
```python
{
    "date": "March 15th",
    "attendees": ["Alice", "Bob", "Charlie"],
    "action_items": [
        "Alice: deploy the new auth module by Friday",
        "Bob: update the API docs",
        "Charlie: run regression tests before release"
    ]
}
```

**Hints:**
- Use the Anthropic or OpenAI SDK with a `response_format` or tool-use schema.
- Define the expected structure in the system prompt or schema.
- The model will naturally extract dates and names — your job is to constrain the output format.
- Handle the case where no date or no attendees are found gracefully.

**Constraints:** Maximum 30 lines of code. Must use structured output (not free-text + regex). Must return a valid `dict`.

## Exercise 2: Tool Definition for a Weather Agent

**Problem:** Define a tool `get_weather` that an agent can call to fetch weather data, and write a simple agent loop that lets a user ask about weather for any city. The tool should accept a city name and optional country code, and return a mock weather report.

**Expected input:** User asks `"What is the weather in Tokyo?"` or `"Weather in London, GB"`

**Expected output:** The agent calls the tool and returns a formatted weather string like:
```text
Weather in Tokyo:
  Temperature: 22°C
  Conditions: Partly cloudy
  Humidity: 65%
```

**Hints:**
- Define the tool with a JSON schema (parameters: `city` required string, `country_code` optional string).
- Write a mock tool function that returns plausible data for at least 5 cities.
- Use a simple loop: get user input, append to messages, call the LLM with the tool definition, check for tool calls, execute and return the result.
- No need for a multi-turn conversation — one question, one answer.

**Constraints:** Must use proper tool/function calling (not free-text). Mock data for at least 5 cities. Maximum 50 lines of code.

## Exercise 3: Agent Observability — Logging a Session

**Problem:** Instrument a minimal agent session with structured logging. Write a function `run_logged_agent(user_query: str, logger: AgentLogger)` that:
1. Logs the user input
2. Simulates an LLM call (print a placeholder)
3. Logs the LLM call with token counts
4. Simulates one tool call (`search_knowledge_base`)
5. Logs the tool call with duration
6. Logs the final response

Use this `AgentLogger` class:

```python
import json
import uuid
from datetime import datetime

class AgentLogger:
    def __init__(self):
        self.session_id = str(uuid.uuid4())[:8]
        self.events = []

    def log(self, event_type: str, payload: dict):
        self.events.append({
            "timestamp": datetime.now().isoformat(),
            "session_id": self.session_id,
            "event_type": event_type,
            "payload": payload,
        })

    def export(self) -> str:
        return json.dumps(self.events, indent=2)
```

**Expected output:** `logger.export()` returns a JSON string with at least 4 events: `user_input`, `llm_call`, `tool_call`, and `response`.

**Hints:**
- The `llm_call` payload should include `model`, `prompt_tokens`, and `max_tokens`.
- The `tool_call` payload should include `tool_name`, `arguments`, and `duration_ms`.
- Simulate a realistic duration using `time.sleep(0.1)` or a random value.
- The `response` payload should include the response text and `completion_tokens`.

**Constraints:** Must produce valid JSON output. Must include at least 4 event types. Must use the provided `AgentLogger` class.
