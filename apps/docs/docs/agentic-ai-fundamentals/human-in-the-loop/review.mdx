# Human in the Loop — Review

*Reading time: 3 minutes*

## Key Takeaways

- Human-in-the-loop adds approval gates at critical decision points — the agent pauses and waits for human input
- Human-on-the-loop streams actions to a monitor who can intervene but does not need to approve every action
- Approval fatigue sets in when too many gates require human attention — use risk tiers to gate only high-risk actions
- Always set a timeout on human approval requests so the agent does not hang indefinitely
- Every human decision must be logged for audit trails

## Recap in 60 Seconds

1. Only gate irreversible or high-risk actions — too many gates cause approval fatigue and defeat the purpose
2. Set a timeout on every approval request; use fallback behavior (reject, escalate, or auto-approve with warning)
3. Present full context to the human: what the agent has done, why this action is needed, and what will happen next
