# Review: Slack MCP Server

⏱️ 5 minutes

## Key Takeaways

1. Always scope the bot token to the minimum set of channels and permissions — use `/invite @BotName` to join only the channels it needs
2. Format messages using Slack's `mrkdwn` syntax — `*bold*`, `` `code` ``, and `>quote` — for proper rendering
3. The `search:read` scope provides full workspace search; `channels:history` only covers messages after the bot joins
4. Combine related content into single messages to avoid Slack API rate limits (max 1 message per ~second per channel)
5. Never check the bot token into version control — use environment variables and rotate regularly

## Recap in 60 Seconds

If you only remember three things:

1. **Scope the bot tightly** — invite it only to channels it needs, and use the minimum OAuth scopes
2. **Format messages properly** — use Slack `mrkdwn` for bold, code blocks, and lists
3. **Combine messages** — one well-formatted message beats multiple rapid-fire calls that hit rate limits
