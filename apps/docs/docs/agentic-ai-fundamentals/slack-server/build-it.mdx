# Build It: Configuring a Slack MCP Server

⏱️ 20 minutes

## Breaking It Down

### 1. Creating a Slack App and Getting a Bot Token

Go to [api.slack.com/apps](https://api.slack.com/apps) and click **Create New App > From scratch**. Name it (e.g., "Claude Bot") and select your workspace.

Navigate to **OAuth & Permissions** and add these Bot Token Scopes:

| Scope | Purpose |
|---|---|
| `channels:history` | Read messages in public channels |
| `channels:read` | View public channel names and membership |
| `chat:write` | Send messages as the bot |
| `groups:history` | Read messages in private channels |
| `groups:read` | View private channel names |
| `search:read` | Search all messages across the workspace |
| `users:read` | Look up user display names and emails |

Click **Install to Workspace**, then copy the **Bot User OAuth Token** (starts with `xoxb-`).

### 2. Configuring the Slack MCP Server

Install the official MCP Slack server:

```bash
npm install -g @anthropic/server-slack
```

Configure it in your `.claude/settings.json`:

```json
{
  "mcpServers": {
    "slack": {
      "command": "npx",
      "args": [
        "-y",
        "@anthropic/server-slack"
      ],
      "env": {
        "SLACK_BOT_TOKEN": "xoxb-your-token-here",
        "SLACK_TEAM_ID": "T01234567"
      }
    }
  }
}
```

The `SLACK_TEAM_ID` is your workspace's team ID. You can find it by visiting `https://api.slack.com/methods/team.info/test` while logged in, or from the Slack app under **About > More > Team ID**.

### 3. Available Tools

Once connected, the server exposes these tools:

| Tool | Description |
|---|---|
| `slack_list_channels` | List all channels the bot has access to |
| `slack_get_channel_history` | Get recent messages from a channel |
| `slack_send_message` | Send a message to a channel or thread |
| `slack_search_messages` | Search across workspace messages |
| `slack_get_thread_replies` | Get all replies in a thread |
| `slack_get_user_profile` | Get a user's display name and status |
| `slack_add_reaction` | Add an emoji reaction to a message |

### 4. Real-World Usage Scenarios

**Search past conversations:**

Ask the agent: "Find the discussion about migrating from webpack to Vite that happened in #engineering last month."

The agent calls `slack_search_messages(query: "migrate from webpack to vite" channel:"engineering")` and returns the relevant thread with timestamps and authors:

```
#engineering — 3 matches found:

1. Alice (May 12): "Has anyone tried migrating from webpack to Vite yet?"
2. Bob (May 12): "We did it on the dashboard app. Reduced build times by 80%. Happy to share notes."
3. Alice (May 13): "Bob, would love to see those notes. Can you share them in this thread?"
```

**Send notifications:**

Ask the agent: "Let #deployments know that the staging deploy finished successfully."

The agent calls `slack_send_message(channel: "deployments", text: "✅ Staging deploy completed successfully at 14:32 UTC. Version: v2.4.1")`.

**Summarize threads:**

Ask: "What was decided in the #design sprint planning thread?"

The agent calls `slack_get_channel_history(channel: "design")` to find the most recent thread, then `slack_get_thread_replies(channel: "design", ts: "1712345678.000000")` to read the full discussion, and synthesizes a summary.

### 5. Building a Custom Slack Server with the MCP SDK

For custom filtering logic, additional tools, or multi-workspace support:

```python
# slack_server.py
import os
from mcp import Server
from mcp.types import TextContent
from slack_sdk import WebClient
from slack_sdk.errors import SlackApiError

SLACK_BOT_TOKEN = os.environ["SLACK_BOT_TOKEN"]
client = WebClient(token=SLACK_BOT_TOKEN)
server = Server("custom-slack")

@server.tool("slack_send_message")
async def slack_send_message(channel: str, text: str) -> TextContent:
    """Send a message to a Slack channel or thread."""
    try:
        result = client.chat_postMessage(channel=channel, text=text)
        return TextContent(
            type="text",
            text=f"Message sent to {channel} (ts: {result['ts']})"
        )
    except SlackApiError as e:
        return TextContent(type="text", text=f"Error: {e.response['error']}")

@server.tool("slack_search_messages")
async def slack_search_messages(query: str) -> TextContent:
    """Search across workspace messages."""
    try:
        result = client.search_messages(query=query, count=5)
        messages = result["messages"]["matches"]
        output_lines = []
        for msg in messages:
            channel = msg["channel"]["name"]
            user = msg.get("username", "unknown")
            text = msg["text"][:200]
            ts = msg["ts"]
            output_lines.append(f"[#{channel}] {user} ({ts}): {text}")
        return TextContent(
            type="text",
            text="\n\n".join(output_lines) if output_lines else "No matches found"
        )
    except SlackApiError as e:
        return TextContent(type="text", text=f"Error: {e.response['error']}")

@server.tool("slack_list_channels")
async def slack_list_channels() -> TextContent:
    """List public channels the bot belongs to."""
    try:
        result = client.conversations_list(types="public_channel", limit=50)
        channels = [f"#{c['name']} ({c['id']}) — {c.get('purpose', {}).get('value', '')[:50]}"
                    for c in result["channels"]]
        return TextContent(
            type="text",
            text="\n".join(channels) if channels else "No channels found"
        )
    except SlackApiError as e:
        return TextContent(type="text", text=f"Error: {e.response['error']}")
```

Run it:

```bash
SLACK_BOT_TOKEN=xoxb-xxx python slack_server.py --transport stdio
```

## Security Considerations

The bot token grants real write access to your Slack workspace. Protect it carefully:

- Never check the token into version control — use environment variables
- Restrict the bot to specific channels using Slack's **Event Subscriptions** channel filter
- Use the minimum scopes needed — do not add `admin` scopes
- Regularly rotate the token through the Slack API dashboard
- Consider a dedicated bot user (not your personal account) so actions are clearly attributed
