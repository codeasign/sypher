# Building a Slack MCP Server

⏱️ 15 minutes · Difficulty: Intermediate

## What You Will Learn

By the end of this lesson you will be able to:

- Create a Slack app and configure it with the correct bot token scopes
- Install and configure a Slack MCP server in Claude Code
- Give your agent tools to read messages, send messages, and search Slack history
- Scope the bot to specific channels for security

## Prerequisites

You should already understand:

- How MCP servers are configured with environment variables (see [GitHub Server](../github-server/overview))
- What a Slack workspace is and how apps are installed
- OAuth token concepts — bot tokens vs user tokens, token scopes
- The difference between public channels, private channels, and DMs

## Why This Matters

An agent that can read and write in Slack becomes a member of your team. It can answer questions by searching past conversations, notify channels when builds complete, triage support requests by reading threads, and summarize long discussions for teammates who were out of office.

Engineering teams use Slack MCP servers to automate incident response — an agent watches for alert messages, looks up the relevant runbook from a database, and posts resolution steps in the thread. Customer success teams use them to answer common questions by searching past resolutions. Product teams use them to gather feature requests from multiple channels and compile them into a report.

The Slack MCP server is particularly powerful when combined with the database and filesystem servers: an agent can read a Slack thread about a bug, search the database for related error records, read the relevant source code, and post a fix in the thread — all from a single request.

## The Problem

Your team lives in Slack. Questions, decisions, bug reports, feature requests, deployment notifications — all flow through channels. When a developer asks "did anyone discuss the API rate limiting issue last week?", the answer is buried in days of Slack history that the agent cannot access.

Without a Slack MCP server, the agent is blind to your team's communication. It cannot search channel history, cannot send notifications, cannot participate in threads. You would need to manually copy-paste relevant Slack conversations into the agent's context — which defeats the purpose of having an agent.

## The Concept

A Slack MCP server acts as a bridge between the MCP protocol and the Slack Web API. It authenticates as a Slack bot installed in your workspace, with permissions scoped to specific channels. The server exposes tools like `slack_list_channels`, `slack_get_channel_history`, `slack_send_message`, `slack_search_messages`, and `slack_get_thread_replies`.

Each tool wraps one or more Slack API calls. `slack_send_message`, for example, calls `chat.postMessage` with the channel ID, text, and optional thread timestamp. The server handles Slack API pagination, rate limiting, and token refresh so the agent only sees clean, typed tool interfaces.
