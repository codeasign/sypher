# Tokens and Context Windows — Review

*Reading time: 3 minutes*

## Key Takeaways

- Tokens are subword units produced by a tokenizer — one English word averages about 1.3 tokens
- The context window includes both prompt and response tokens; total must stay within the model's limit
- Always budget for the response when checking context window fit
- Token costs scale linearly with usage — calculate cost per request before deploying at scale
- Truncation strategies (sliding window, summarization) are essential for long-running conversations
- Different models have different window sizes and costs — parameterize your strategy by model

## Recap in 60 Seconds

1. Every API call costs tokens on both input and output — know your token counts the same way you know your memory usage
2. The context window is shared between your prompt and the model's response — plan for both
3. Token optimization = cost optimization. A 20% reduction in prompt tokens across thousands of requests saves real money
