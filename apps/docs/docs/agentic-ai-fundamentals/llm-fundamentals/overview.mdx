# LLM Fundamentals

*Reading time: 5 minutes*

**Difficulty:** Beginner

## What You Will Learn

By the end of this lesson you will be able to:

- Explain what a large language model is at the architecture level
- Describe the three stages of LLM training: pretraining, fine-tuning, and alignment
- List the key inference parameters and how they affect output
- Distinguish between model capabilities and limitations accurately

## Prerequisites

You should already understand:

- What an API call to an LLM looks like (covered in [What is AI Engineering](../what-is-ai-engineering/overview))
- Basic concepts of probability and prediction
- Familiarity with the idea of neural networks (no equations required)

## Why This Matters

Every prompt you write, every parameter you tune, and every production decision you make depends on how the underlying model works. Engineers who treat the LLM as a black box inevitably hit walls: they do not understand why increasing temperature sometimes breaks structured output, why the model repeats itself in long responses, or why a well-crafted prompt works on one model but fails on another.

The difference between a prompt engineer who can reliably ship production features and one who guesses at random is an accurate mental model of what is happening inside the transformer. This lesson gives you that mental model without requiring a machine learning PhD.

## The Problem

You write what you think is the perfect prompt — clear instructions, good examples, proper formatting. Sometimes it works brilliantly. Other times the model ignores a key instruction, invents a fact, or produces garbled output for no apparent reason.

You try to fix it by adding more instructions, but that makes the prompt long and fragile. You adjust parameters, but the results are inconsistent. Without understanding why the model behaves this way, you are debugging in the dark. The root cause is almost never "the model is broken" — it is a mismatch between what you asked for and how the model's architecture processes your input.

## The Concept

A large language model is a neural network trained to predict the next token in a sequence. That is it at the core. Every capability you see — answering questions, writing code, summarizing documents — emerges from this simple objective applied at enormous scale.

The model is built on the transformer architecture, which uses a mechanism called attention to weigh the importance of every token relative to every other token in the input. When you send a prompt, the model:

1. Tokenizes your text into subword units
2. Processes each token through layers of attention and feed-forward computation
3. Generates a probability distribution over all possible next tokens
4. Samples from that distribution to pick the next token
5. Feeds the new token back into the input and repeats steps 2-5

The training happens in three stages:

**Pretraining** exposes the model to a vast corpus of text from the internet, books, and code. The model learns grammar, facts, reasoning patterns, and world knowledge — compressed into its weights. This is the expensive stage, costing millions of dollars and taking months.

**Fine-tuning** trains the model on curated instruction-response pairs. This teaches it to follow instructions instead of just predicting the next token. A conversational model does not naturally know how to answer a question — fine-tuning teaches that format.

**Alignment** (RLHF or constitutional AI) further trains the model to prefer helpful, honest, and harmless responses. This reduces harmful outputs and refusals while teaching the model to say "I do not know" rather than making things up.
