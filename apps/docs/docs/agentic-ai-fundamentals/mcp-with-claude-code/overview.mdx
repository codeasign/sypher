# MCP with Claude Code

*Reading time: 6 minutes*

**Difficulty:** Intermediate

## What You Will Learn

By the end of this lesson you will be able to:

- Configure MCP servers in Claude Code's settings.json
- Understand the MCP server configuration format
- Extend Claude Code's capabilities with custom MCP servers
- Troubleshoot MCP connection issues in Claude Code

## Prerequisites

You should already understand:

- What MCP is and how it works (covered in [What Is MCP?](../what-is-mcp/overview))
- How to build an MCP server (covered in [Building an MCP Server](../building-an-mcp-server/overview))
- Basic familiarity with Claude Code CLI
- Understanding of JSON configuration files

## Why This Matters

Claude Code has a built-in MCP client. When you configure an MCP server in Claude Code's settings, every Claude Code session automatically connects to that server, discovers its tools, and makes them available for use. This means you do not need to write a custom MCP client — you configure once and all your tools are available in every session.

This is the most practical MCP workflow for AI engineers. Instead of building a custom agent with an MCP client, you add MCP servers to Claude Code's configuration and get access to file system tools, database tools, API gateways, and any other MCP server you build — all from the Claude Code CLI.

The configuration format is straightforward but must be exact. A typo in the settings.json path or JSON structure means the server does not load, and Claude Code silently skips it. Understanding the exact format and how to debug connection issues is essential for production use.

## The Problem

You have built an MCP server that provides database query tools for your team's PostgreSQL instance. The server works when you test it with the MCP Inspector. Now you want every Claude Code session on your machine to have access to these database tools. You do not want to build a custom client, write integration code, or modify Claude Code's source.

Claude Code already has the client built in. You just need to tell it where your server is and how to start it. The configuration lives in Claude Code's settings.json file, under the `mcp_servers` key.

## The Concept

Claude Code reads MCP server configurations from its settings file. Each server entry specifies a command to run (the server process) and arguments. When Claude Code starts, it spawns each configured server as a child process (stdio transport), runs the MCP initialization handshake, discovers the server's tools, and makes them available to the model.

The configuration is transport-level: you provide the command, and Claude Code handles the MCP protocol. You do not need to write any client code — Claude Code is the client.

Servers are started lazily in some configurations or eagerly at startup. Claude Code manages the lifecycle — it spawns the process, initializes the connection, discovers capabilities, and keeps the connection alive for the duration of the session.

When you ask Claude Code a question that requires a tool, the model sees all tools from all connected MCP servers alongside its built-in tools. It chooses the right tool based on the tool's name and description, just as it would with any other tool.
