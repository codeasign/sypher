# MCP with Claude Code — Build It

*Reading time: 14 minutes*

## Breaking It Down

### 1. Understanding the Configuration Format

Claude Code reads MCP server configurations from its settings files. The format uses an `mcp_servers` top-level key where each entry defines the command to run for that server.

Claude Code looks for settings in two locations (loaded in order, later files override earlier ones):

- **Global**: `~/.claude/settings.json` — applies to all projects
- **Project**: `.claude/settings.local.json` — applies to the current project only
- **Project shared**: `.claude/settings.json` — checked into version control, shared with the team

For MCP servers, the configuration format is:

```json
{
  "mcp_servers": {
    "server-name": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "."],
      "env": {
        "API_KEY": "optional-environment-variable"
      }
    }
  }
}
```

Each server entry has:

| Field     | Required | Description |
|-----------|----------|-------------|
| `command` | Yes      | The executable to run (e.g., `npx`, `python`, `node`) |
| `args`    | No       | Array of command-line arguments |
| `env`     | No       | Object of environment variables for the server process |

### 2. Adding a Filesystem Server

The official `@modelcontextprotocol/server-filesystem` server provides file read, write, and search tools. It is the simplest way to verify your MCP configuration works.

Add this to your project's `.claude/settings.local.json`:

```json
{
  "mcp_servers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "."
      ]
    }
  }
}
```

To add it globally so it is available in every project, add the same block to `~/.claude/settings.json`.

After saving the file, restart Claude Code. You should see a message indicating MCP servers are loading. If the server starts successfully, Claude Code will have tools like `read_file`, `write_file`, `search_files`, and `get_file_info` available.

### 3. Adding a Custom Python MCP Server

For a server you built yourself, use the `python` command with your script's path:

```json
{
  "mcp_servers": {
    "custom-server": {
      "command": "python",
      "args": [
        "path/to/your_server.py"
      ]
    }
  }
}
```

You can use absolute paths or paths relative to the project root. If your server requires environment variables:

```json
{
  "mcp_servers": {
    "database-gateway": {
      "command": "python",
      "args": ["db_server.py"],
      "env": {
        "DB_HOST": "localhost",
        "DB_PORT": "5432",
        "DB_NAME": "analytics"
      }
    }
  }
}
```

### 4. Multiple Servers

You can configure multiple servers in the same `mcp_servers` object. Claude Code starts all of them and merges their tools.

```json
{
  "mcp_servers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "."]
    },
    "web-search": {
      "command": "python",
      "args": ["-m", "web_search_server"]
    },
    "database": {
      "command": "python",
      "args": ["db_mcp_server.py"],
      "env": {
        "DB_URL": "postgresql://localhost:5432/mydb"
      }
    }
  }
}
```

When multiple servers are configured, Claude Code:
1. Spawns each server as a separate child process
2. Initializes each connection independently
3. Merges all tool definitions into a single tool list
4. Routes each tool call to the correct server based on the tool name

### 5. Verifying Your Configuration

To verify that your MCP servers are working:

1. Start Claude Code in the project directory
2. Ask Claude to list the available tools with a prompt like "What tools do you have available?"
3. If your servers loaded correctly, Claude should mention tools from your custom servers

If a server fails to load, Claude Code continues without that server's tools. Check for issues:

```bash
# Test the server command independently
npx -y @modelcontextprotocol/server-filesystem .

# Or for a Python server
python your_server.py
```

If the server works independently but not in Claude Code, check:

- Is the settings file valid JSON?
- Is the path to the script correct (absolute or project-relative)?
- Are all required environment variables set?
- Does the server follow the MCP protocol correctly (proper initialization handshake)?

### 6. Troubleshooting Common Issues

```bash
# Check if Claude Code's settings are valid JSON
python -c "import json; json.load(open('.claude/settings.local.json'))"

# Run the MCP server directly to test it
echo '{"jsonrpc":"2.0","id":1,"method":"tools/list","params":{}}' \
  | python your_server.py

# Expected output should be a valid JSON-RPC response with tools
```

If the server does not output anything when you pipe a `tools/list` request, the server may not be reading from stdin correctly or may have a startup error. Check stderr for error messages.

## How It Works Internally

When Claude Code starts, it reads `mcp_servers` from the merged settings files (global + project). For each server entry, it spawns a child process with the given command and arguments. It sends the MCP initialization sequence over stdin/stdout, waits for the initialize response, sends the `initialized` notification, and calls `tools/list` to discover available tools.

The discovered tools are merged with Claude Code's built-in tools. The model sees all tools from all servers as if they were native. When the model decides to call a server's tool, Claude Code translates the tool call into an MCP `tools/call` JSON-RPC message, sends it over the server's stdin, and returns the result to the model.

If a server crashes or fails to initialize, Claude Code logs the error and continues without that server's tools. It does not crash the session — the failure is isolated to the individual server.

## Build Something Real

Build a complete project setup with two MCP servers: a filesystem server for your project and a custom Python server that provides project-specific tools.

First, create a simple custom MCP server for project analysis:

```python
# project_mcp_server.py — A simple MCP server for project analysis
import asyncio
import json
import os
from pathlib import Path

from mcp.server import Server, NotificationOptions
from mcp.server.models import InitializationOptions
from mcp.types import TextContent, Tool


server = Server("project-analytics")


@server.list_tools()
async def list_tools() -> list[Tool]:
    return [
        Tool(
            name="analyze_project_size",
            description="Count files and directories in the project",
            inputSchema={
                "type": "object",
                "properties": {
                    "path": {
                        "type": "string",
                        "description": "Directory to analyze (relative to project root)",
                    },
                },
            },
        ),
        Tool(
            name="find_largest_files",
            description="Find the largest files in the project",
            inputSchema={
                "type": "object",
                "properties": {
                    "count": {
                        "type": "number",
                        "description": "Number of files to return",
                    },
                    "min_size_mb": {
                        "type": "number",
                        "description": "Minimum file size in MB",
                    },
                },
            },
        ),
    ]


@server.call_tool()
async def call_tool(name: str, arguments: dict) -> list[TextContent]:
    if name == "analyze_project_size":
        target = Path(arguments.get("path", "."))
        files = sum(1 for _ in target.rglob("*") if _.is_file())
        dirs = sum(1 for _ in target.rglob("*") if _.is_dir())
        return [TextContent(
            type="text",
            text=f"Project stats for {target}:\n"
                 f"  Files: {files}\n"
                 f"  Directories: {dirs}\n"
                 f"  Total items: {files + dirs}",
        )]

    if name == "find_largest_files":
        count = arguments.get("count", 10)
        min_bytes = arguments.get("min_size_mb", 1) * 1024 * 1024
        files = []
        for f in Path(".").rglob("*"):
            if f.is_file() and f.stat().st_size >= min_bytes:
                files.append((f, f.stat().st_size))
        files.sort(key=lambda x: -x[1])
        top = files[:count]
        lines = [f"Top {len(top)} largest files:"]
        for path, size in top:
            mb = size / (1024 * 1024)
            lines.append(f"  {path} — {mb:.1f} MB")
        return [TextContent(type="text", text="\n".join(lines))]

    raise ValueError(f"Unknown tool: {name}")


async def main():
    async with mcp.server.stdio() as (read_stream, write_stream):
        await server.run(
            read_stream,
            write_stream,
            InitializationOptions(
                server_name="project-analytics",
                server_version="1.0.0",
                capabilities=server.get_capabilities(
                    notification_options=NotificationOptions(),
                    experimental_capabilities={},
                ),
            ),
        )


if __name__ == "__main__":
    asyncio.run(main())
```

Then configure it in `.claude/settings.local.json`:

```json
{
  "mcp_servers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "."]
    },
    "project-analytics": {
      "command": "python",
      "args": ["project_mcp_server.py"]
    }
  }
}
```

Start Claude Code and try: "Analyze this project and find the largest files over 500KB."

## Security Considerations

MCP servers in Claude Code run with the same permissions as the Claude Code process. A filesystem server configured with access to your project directory can read and write files in that directory. Only configure servers you trust and only grant them access to the directories and data they need. The `env` field in the server configuration should never contain secrets that are checked into version-controlled settings files — use `.claude/settings.local.json` (gitignored) for environment-specific secrets.
