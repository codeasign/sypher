# Observe-Think-Act Loop

*Reading time: 5 minutes*

**Difficulty:** Intermediate

## What You Will Learn

By the end of this lesson you will be able to:

- Describe the observe-think-act loop as the fundamental execution pattern for agents
- Implement each phase of the loop with correct message passing
- Recognize when the loop is stuck and how to break out
- Measure the loop's efficiency in terms of turns per task

- How tools are called and results returned (covered in [Agent Tools and Tool Calling](../agent-tools-and-tool-calling/overview))

## Why This Matters

Every agent runs some version of an observe-think-act loop. Whether you use ReAct, Plan-and-Execute, or a custom pattern, the underlying cycle is the same: the agent observes its environment, thinks about what to do, and acts. Understanding this loop at its most basic level is the foundation for every advanced agent pattern.

Production agents at companies like Replit (AI coding assistant), GitHub (Copilot Chat), and Notion (AI Q&A) all use variants of this loop. When your agent behaves unexpectedly — calling the wrong tool, repeating itself, or getting stuck — the root cause is almost always in the loop implementation: a missing observation, a skipped thinking step, or an action that did not update the state correctly.

## The Problem

Your agent receives a request: "Find the latest sales report, summarize it, and email the summary to the team."

The agent calls `search_files("sales report")`, gets a list, calls `read_file("Q3_report.pdf")`, and then... nothing. It stops. It found the report, read it, but never summarized or emailed it.

The loop exited early because the model's response after reading the file was a text summary — which the orchestrator treated as the final answer. The agent forgot to check whether the task was actually complete. The loop terminated before the task was done.

You need a loop that knows when it is done — not just when the model stops requesting tools.

## The Concept

The observe-think-act loop has three phases:

**Observe**. The agent takes in new information from its environment. This includes user messages, tool results, system state changes, and memory retrievals. Observation updates the agent's understanding of the current situation.

**Think**. The agent processes what it observed. This may involve reasoning about the next step, evaluating options, checking progress against the goal, or retrieving relevant memories. Thinking determines what action to take.

**Act**. The agent executes an action — calling a tool, producing output, or requesting clarification. The action changes the state of the world or advances the task.

After the act phase, the loop repeats: the result of the action becomes a new observation.

The loop must have a clear termination condition. Common conditions:

- The task is complete (all required actions have been taken)
- The agent is stuck (same state reached multiple times)
- A safety limit is hit (max turns, max time, max cost)
