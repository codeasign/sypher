# Observe-Think-Act Loop — Review

*Reading time: 3 minutes*

## Key Takeaways

- The observe-think-act loop is the fundamental execution pattern for all agents
- Each turn has three phases: observe (receive information), think (decide), act (execute)
- Termination conditions must be explicit: max turns, max time, stall detection, task completion
- Always set a hard limit on the loop — unlimited loops are unlimited costs
- Structured state tracking (turns, timing, tokens) is essential for debugging

## Recap in 60 Seconds

1. Observe = receive new information; Think = decide what to do; Act = execute the decision — repeat until done
2. Every loop needs both a success condition (task complete) and failure conditions (limits, stalls, errors)
3. Log every turn with timing and token data — you cannot fix a loop you cannot observe
