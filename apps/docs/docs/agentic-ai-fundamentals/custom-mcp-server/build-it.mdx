# Build It: Scaffolding a Custom MCP Server

⏱️ 25 minutes

## Breaking It Down

### 1. Setting Up the Project

Create a new project directory and install the MCP Python SDK:

```bash
mkdir my-mcp-server
cd my-mcp-server
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
pip install mcp httpx
```

The `mcp` package provides the `Server` class, tool decorators, transport handlers, and protocol serialization. `httpx` is used here for HTTP calls to the backend API, but you can substitute any Python client library.

Create your main server file:

```bash
touch server.py
```

### 2. Defining Your First Tool

Open `server.py` and scaffold the server:

```python
# server.py
import os
from mcp import Server
from mcp.types import TextContent

# Create the server instance with a unique name
server = Server("my-custom-server")

@server.tool("hello")
async def hello(name: str = "World") -> TextContent:
    """A simple greeting tool — demonstrates the basic pattern."""
    return TextContent(type="text", text=f"Hello, {name}!")
```

Run it in stdio mode:

```bash
python server.py --transport stdio
```

Test it by piping a JSON-RPC request, or configure it in Claude Code:

```json
{
  "mcpServers": {
    "my-custom-server": {
      "command": "python",
      "args": ["/path/to/my-mcp-server/server.py", "--transport", "stdio"]
    }
  }
}
```

### 3. Adding Typed Parameters and Descriptions

The SDK reads type annotations to generate the tool schema. Add more complex types:

```python
from typing import Optional
from pydantic import BaseModel

class DeploymentConfig(BaseModel):
    service: str
    version: str
    environment: str = "staging"
    timeout_seconds: int = 300
    rollback_on_failure: bool = True

@server.tool("deploy_service")
async def deploy_service(config: DeploymentConfig) -> TextContent:
    """Deploy a service version to an environment."""
    # In production, this would call your internal deploy API
    result = (
        f"Deploying {config.service} v{config.version} "
        f"to {config.environment} "
        f"(timeout: {config.timeout_seconds}s, "
        f"rollback: {config.rollback_on_failure})"
    )
    return TextContent(type="text", text=result)
```

The agent now sees a tool named `deploy_service` with parameters `service` (string, required), `version` (string, required), `environment` (string, default "staging"), `timeout_seconds` (integer, default 300), and `rollback_on_failure` (boolean, default true).

### 4. Adding Resources

Resources let the agent read data without calling a tool. They are similar to static endpoints:

```python
@server.resource("config://environments")
async def get_environments() -> TextContent:
    """List all available deployment environments."""
    environments = ["development", "staging", "production"]
    return TextContent(type="text", text="\n".join(environments))

@server.resource("config://services")
async def get_services() -> TextContent:
    """List all deployable services."""
    services = ["api-gateway", "user-service", "order-service", "notification-service"]
    return TextContent(type="text", text="\n".join(services))
```

Resources are accessed via their URI scheme (`config://` in this example). When the agent encounters a need for deployment context, it can read `config://environments` and `config://services` to understand the available targets.

### 5. Adding Prompts

Prompts are reusable templates that help the agent handle common tasks:

```python
@server.prompt("deployment_checklist")
async def deployment_checklist(service: str, environment: str) -> str:
    """Get a deployment checklist for a service and environment."""
    return f"""
## Deployment Checklist for {service} → {environment}

Before deploying, confirm:
1. All tests pass on the main branch
2. The version tag exists in the container registry
3. Database migrations have been reviewed
4. The runbook has been updated
5. The on-call engineer has been notified

Use `deploy_service` to proceed with the deployment.
"""
```

### 6. Building the Complete Server

Here is a complete custom server that wraps a hypothetical deployment API:

```python
# deploy_mcp_server.py
import os
import httpx
from typing import Optional
from mcp import Server
from mcp.types import TextContent
from pydantic import BaseModel

DEPLOY_API_BASE = os.environ.get("DEPLOY_API_BASE", "http://localhost:8080")
DEPLOY_API_KEY = os.environ.get("DEPLOY_API_KEY", "")

server = Server("deployment-server")

class DeployRequest(BaseModel):
    service: str
    version: str
    environment: str = "staging"
    timeout_seconds: int = 300

@server.resource("deploy://services")
async def list_services() -> TextContent:
    """List all available services that can be deployed."""
    async with httpx.AsyncClient() as client:
        resp = await client.get(
            f"{DEPLOY_API_BASE}/services",
            headers={"Authorization": f"Bearer {DEPLOY_API_KEY}"},
        )
    resp.raise_for_status()
    services = resp.json()
    return TextContent(
        type="text",
        text="\n".join(f"{s['name']} (latest: {s['latest_version']})" for s in services)
    )

@server.tool("deploy")
async def deploy(config: DeployRequest) -> TextContent:
    """Deploy a service to a target environment."""
    async with httpx.AsyncClient() as client:
        resp = await client.post(
            f"{DEPLOY_API_BASE}/deploy",
            json=config.model_dump(),
            headers={"Authorization": f"Bearer {DEPLOY_API_KEY}"},
            timeout=config.timeout_seconds,
        )
    if resp.status_code == 202:
        data = resp.json()
        return TextContent(
            type="text",
            text=f"Deployment started: {data['deployment_id']}\n"
                 f"Track progress: {data['status_url']}"
        )
    else:
        return TextContent(type="text", text=f"Deploy failed: {resp.text}")

@server.tool("rollback")
async def rollback(service: str, environment: str) -> TextContent:
    """Rollback a service in an environment to the previous version."""
    async with httpx.AsyncClient() as client:
        resp = await client.post(
            f"{DEPLOY_API_BASE}/rollback",
            json={"service": service, "environment": environment},
            headers={"Authorization": f"Bearer {DEPLOY_API_KEY}"},
        )
    resp.raise_for_status()
    data = resp.json()
    return TextContent(
        type="text",
        text=f"Rollback initiated: {service} in {environment} → v{data['previous_version']}"
    )

@server.tool("deployment_status")
async def deployment_status(deployment_id: str) -> TextContent:
    """Check the status of a deployment by its ID."""
    async with httpx.AsyncClient() as client:
        resp = await client.get(
            f"{DEPLOY_API_BASE}/deployments/{deployment_id}",
            headers={"Authorization": f"Bearer {DEPLOY_API_KEY}"},
        )
    resp.raise_for_status()
    data = resp.json()
    status = data["status"]  # pending, running, succeeded, failed, rolled_back
    return TextContent(type="text", text=f"Deployment {deployment_id}: {status}")

@server.prompt("rollback_guide")
async def rollback_guide(service: str, environment: str) -> str:
    """Guide the user through rolling back a deployment."""
    return f"""
## Rollback Guide: {service} in {environment}

1. Check the current deployment status with `deployment_status`
2. Verify that a previous version exists in the deployment history
3. Use `rollback` to revert to the previous version
4. Monitor the rollback progress
5. Notify the team in Slack

Current environment: {environment}
Service: {service}
"""
```

### 7. Transport Configuration

Configure the server for either transport:

```python
if __name__ == "__main__":
    import sys

    if "--transport" in sys.argv:
        transport = sys.argv[sys.argv.index("--transport") + 1]
    else:
        transport = "stdio"

    if transport == "sse":
        from mcp.server.sse import SseServerTransport
        import uvicorn
        from starlette.applications import Starlette
        from starlette.routing import Route

        sse = SseServerTransport("http://localhost:8000")

        async def handle_sse(request):
            async with sse.connect_sse(request.scope, request.receive, request._send) as streams:
                await server.run(streams[0], streams[1], server.create_initialization_options())

        app = Starlette(routes=[Route("/sse", endpoint=handle_sse)])
        uvicorn.run(app, host="0.0.0.0", port=8000)
    else:
        from mcp.server.stdio import stdio_server
        import anyio

        async def main():
            async with stdio_server() as (read_stream, write_stream):
                await server.run(read_stream, write_stream, server.create_initialization_options())

        anyio.run(main)
```

## Build Something Real

Combine what you have learned across Section 7 into a single, integrated agent setup. Configure all six custom servers in your `.claude/settings.json`:

```json
{
  "mcpServers": {
    "filesystem": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-filesystem", "/workspace"] },
    "github": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-github"], "env": { "GITHUB_TOKEN": "${GITHUB_TOKEN}" } },
    "sqlite": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-sqlite", "/workspace/data.db"] },
    "web-search": { "command": "npx", "args": ["-y", "@anthropic/server-brave-search"], "env": { "BRAVE_API_KEY": "${BRAVE_API_KEY}" } },
    "slack": { "command": "npx", "args": ["-y", "@anthropic/server-slack"], "env": { "SLACK_BOT_TOKEN": "${SLACK_BOT_TOKEN}", "SLACK_TEAM_ID": "T01234567" } },
    "custom-deploy": { "command": "python", "args": ["/path/to/deploy_mcp_server.py", "--transport", "stdio"], "env": { "DEPLOY_API_KEY": "${DEPLOY_API_KEY}" } }
  }
}
```

Now one agent session can: read your codebase (filesystem), search for bugs (web search), query the database for error rates (SQLite), open GitHub issues for what it finds (GitHub), notify Slack about the findings (Slack), and deploy a fix (custom deploy server). Six servers, one agent, zero context switching.
