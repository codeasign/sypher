# Review: Custom MCP Server

⏱️ 5 minutes

## Key Takeaways

1. A custom MCP server wraps any internal API or service behind typed, documented MCP tools using the Python SDK
2. Always define explicit typed parameters for every tool — `**kwargs` hides the schema from the agent
3. Handle backend errors gracefully by returning user-friendly messages instead of crashing the server
4. Manage HTTP client and database connection lifecycles — create once, clean up on shutdown
5. Check the transport mode at startup and use the correct handler for stdio or SSE

## Recap in 60 Seconds

If you only remember three things:

1. **Typed parameters** — every tool needs explicit, typed parameters so the agent knows how to call it
2. **Graceful error handling** — catch backend exceptions and return descriptive messages, never crash
3. **Resource lifecycle** — create HTTP clients once, close them on shutdown
