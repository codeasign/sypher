# Building a Custom MCP Server

⏱️ 20 minutes · Difficulty: Advanced

## What You Will Learn

By the end of this lesson you will be able to:

- Scaffold an MCP server from scratch using the Python SDK
- Define tools, resources, and prompts for your custom server
- Implement proper error handling, logging, and transport configuration
- Package and deploy your server for use with any MCP client

## Prerequisites

You should already understand:

- How MCP primitives fit together (see [MCP Primitives](../mcp-primitives/overview))
- The difference between stdio and SSE transports (see [Transports](../transports/overview))
- Python async programming with `async` / `await`
- Basic familiarity with the previous MCP server lessons (filesystem, GitHub, database, web search, Slack)

## Why This Matters

The pre-built MCP servers cover common use cases, but every team has unique services, internal APIs, and custom data sources that an agent needs to access. Your team might have an internal API for deployment orchestration, a proprietary data warehouse, a ticketing system like Jira or Linear, or a custom machine learning model that needs to be queried.

Building a custom MCP server is how you connect your agent to anything that does not have an off-the-shelf MCP server. It is the difference between an agent that can only use generic tools and an agent that is deeply integrated with your specific infrastructure.

Companies that deploy AI agents in production inevitably build custom MCP servers. A deployment agent needs a server that wraps your CI/CD pipeline. A customer support agent needs a server that queries your order management system. A data analysis agent needs a server that connects to your OLAP data warehouse. Each of these requires a custom MCP server tailored to the specific API and data model.

## The Problem

You have an internal API — maybe a REST API for your deployment system, a gRPC endpoint for your recommendation engine, or a GraphQL endpoint for your product catalog. You want your agent to interact with this API. But the agent does not know the API schema, the authentication mechanism, or the request/response format.

You could give the agent a generic HTTP tool and describe the API in the system prompt. But this is brittle — the agent may construct malformed requests, misunderstand the response format, or fail to handle authentication errors. A custom MCP server solves this by wrapping each API operation in a typed, documented tool that the agent can call correctly every time.

## The Concept

A custom MCP server is a program that implements the MCP protocol, registers tools and resources, and runs on either stdio transport (for local use) or SSE transport (for remote use). The MCP Python SDK provides a `Server` class that handles protocol negotiation, capability advertisement, and message serialization — you just define the tools and resources.

Each tool is an async function decorated with `@server.tool(...)`. The decorator takes a name, description, and optional JSON Schema for parameters. The SDK generates the tool definition from the function signature and type annotations, so the agent sees a complete schema for every tool. If your API changes, you update the function, and the schema updates automatically.
