# Specialist Agents — Build It

*Reading time: 16 minutes*

## Breaking It Down

### 1. Specialist Agent Template

Every specialist agent follows the same structure: a system prompt that defines its role, a tool set scoped to its domain, and a method that processes input and returns output. The template below captures this pattern.

```python
from dataclasses import dataclass, field
from typing import Callable


@dataclass
class SpecialistAgent:
    """An agent scoped to a single domain with its own tools and prompt."""
    name: str
    system_prompt: str
    tools: dict[str, Callable] = field(default_factory=dict)
    max_turns: int = 5

    def register_tool(self, name: str, handler: Callable, description: str = ""):
        """Register a tool available to this specialist."""
        self.tools[name] = handler

    def process(self, task: str) -> str:
        """
        Execute a task using only this specialist's tools.
        In production, this would call an LLM with the specialist's prompt
        and tool definitions.
        """
        print(f"[{self.name}] Processing: {task}")

        # Simulated reasoning — in production the LLM decides tool calls
        result_parts = [f"=== {self.name} OUTPUT ==="]

        for tool_name, handler in self.tools.items():
            try:
                output = handler(task)
                if output:
                    result_parts.append(f"[{tool_name}]: {output}")
            except Exception as e:
                result_parts.append(f"[{tool_name}]: Error — {e}")

        return "\n".join(result_parts)
```

### 2. Data Analyst Specialist

The data analyst agent knows only about data operations — querying, statistical analysis, and formatting. It has no tools for file modification, email, or web access.

```python
class DataAnalystAgent(SpecialistAgent):
    """Specialist agent for data analysis. No write or network access."""

    def __init__(self):
        super().__init__(
            name="DataAnalyst",
            system_prompt=(
                "You are a data analyst agent. Your job is to query, "
                "aggregate, and summarize data. You never modify data. "
                "You never access the filesystem or network outside "
                "your approved data sources."
            ),
        )
        self.register_tool(
            "query_sales_data",
            lambda task: (
                "Q1: $420K, Q2: $510K, Q3: $485K, Q4: $630K"
            ),
            "Query quarterly sales figures from the database"
        )
        self.register_tool(
            "compute_statistics",
            lambda task: (
                "Mean: $511.25K, Median: $497.5K, "
                "StdDev: $79.3K, Growth QoQ: +10.7%"
            ),
            "Compute summary statistics on provided data"
        )
        self.register_tool(
            "identify_trends",
            lambda task: (
                "Trend: steady growth through Q4 with seasonal dip in Q3. "
                "Outlier: Q4 spike correlates with product launch."
            ),
            "Identify patterns and trends in the data"
        )


analyst = DataAnalystAgent()
result = analyst.process("Analyze 2025 quarterly sales")
print(result)
```

### 3. Report Writer Specialist

The report writer agent transforms structured data into formatted documents. It has tools for formatting, templating, and rendering — but it cannot query databases or search the web. It must receive its data as input.

```python
class ReportWriterAgent(SpecialistAgent):
    """Specialist agent for document generation. No data access."""

    def __init__(self):
        super().__init__(
            name="ReportWriter",
            system_prompt=(
                "You are a report writer agent. You receive structured "
                "data and transform it into well-formatted documents. "
                "You do not query data yourself."
            ),
        )
        self.register_tool(
            "format_executive_summary",
            lambda data: (
                "## Executive Summary\n\n"
                f"Based on the analysis, {data.get('trend', 'the data')} "
                "indicates positive momentum. "
                f"Revenue grew to ${data.get('total_revenue', 0):.1f}K "
                "for the fiscal year."
            ),
            "Generate an executive summary from analysis data"
        )
        self.register_tool(
            "format_table",
            lambda data: (
                "| Quarter | Revenue | Growth |\n"
                "|---------|---------|--------|\n"
                "| Q1      | $420K   | —      |\n"
                "| Q2      | $510K   | +21.4% |\n"
                "| Q3      | $485K   | -4.9%  |\n"
                "| Q4      | $630K   | +29.9% |\n"
            ),
            "Format data as a Markdown table"
        )
        self.register_tool(
            "add_recommendations",
            lambda data: (
                "## Recommendations\n\n"
                "1. Invest in the product category that drove Q4 growth\n"
                "2. Investigate Q3 dip — determine if seasonal or structural\n"
                "3. Set Q1 target at $550K based on momentum"
            ),
            "Generate data-driven recommendations"
        )


writer = ReportWriterAgent()
data = {"trend": "steady growth through Q4", "total_revenue": 2045.0}
result = writer.process(str(data))
print(result)
```

### 4. Routing via Orchestrator

The orchestrator examines each incoming task, determines which specialist should handle it, and routes accordingly. This keeps the specialists isolated from each other.

```python
@dataclass
class TaskRouter:
    """Routes tasks to the appropriate specialist agent."""
    specialists: dict[str, SpecialistAgent] = field(default_factory=dict)

    def register(self, domain: str, agent: SpecialistAgent):
        self.specialists[domain] = agent

    def route(self, task: str) -> str:
        """Route a task to the matching specialist based on domain keywords."""
        task_lower = task.lower()

        if any(kw in task_lower for kw in ["analyze", "query", "data", "statistics"]):
            return "data"
        elif any(kw in task_lower for kw in ["write", "report", "document", "summarize"]):
            return "report"
        elif any(kw in task_lower for kw in ["code", "implement", "debug", "test"]):
            return "code"
        else:
            return "general"

    def run(self, task: str) -> dict[str, str]:
        """Run a task through the appropriate specialist pipeline."""
        domain = self.route(task)
        agent = self.specialists.get(domain)

        if not agent:
            return {"domain": domain, "result": f"No specialist registered for {domain}"}

        result = agent.process(task)
        return {"domain": domain, "result": result}


# Set up the routing system
router = TaskRouter()
router.register("data", DataAnalystAgent())
router.register("report", ReportWriterAgent())

# Test with different task types
tasks = [
    "Analyze the quarterly sales data and identify trends",
    "Write a report summarizing the key findings",
    "Implement a binary search function",
]

for task in tasks:
    output = router.run(task)
    print(f"\nTask: {task}")
    print(f"Routed to: {output['domain']}")
    print(f"Result:\n{output['result'][:150]}...")
    print("---")
```

## How It Works Internally

Each specialist agent loads only its own tools into the model's context. This means the model's attention is focused on a smaller set of possible actions. When a data agent has 5 tools instead of 30, the model's per-tool accuracy increases because it does not need to distinguish between similar tool descriptions across domains.

The orchestrator acts as a firewall between specialists. If the data analyst agent is compromised or malfunctions, it still cannot access the file system because its tool set does not include file operations. This defense-in-depth property is the primary security benefit of specialist architectures.

## Build Something Real

Build a customer support triage system with three specialist agents: a billing agent, a technical support agent, and an account management agent.

```python
@dataclass
class CustomerSupportSystem:
    """Production support triage with specialist agents."""

    def __init__(self):
        self.billing_agent = self._create_billing_agent()
        self.tech_agent = self._create_tech_agent()
        self.account_agent = self._create_account_agent()

    def _create_billing_agent(self) -> SpecialistAgent:
        agent = SpecialistAgent(
            name="BillingSupport",
            system_prompt=(
                "You are a billing support specialist. You handle "
                "invoices, payments, refunds, and subscription changes. "
                "You never give technical advice."
            ),
        )
        agent.register_tool(
            "lookup_invoice",
            lambda task: (
                "Invoice INV-4242: $299.99, due 2025-06-15, "
                "status: overdue by 14 days"
            ),
            "Look up customer invoice by ID"
        )
        agent.register_tool(
            "process_refund",
            lambda task: "Refund of $299.99 initiated. Reference: RFD-8821.",
            "Process a refund for a given invoice"
        )
        return agent

    def _create_tech_agent(self) -> SpecialistAgent:
        agent = SpecialistAgent(
            name="TechnicalSupport",
            system_prompt=(
                "You are a technical support specialist. You help with "
                "product issues, troubleshooting, and bug reports. "
                "You never handle billing or account changes."
            ),
        )
        agent.register_tool(
            "search_knowledge_base",
            lambda task: "KB article #404: Reset API key in 3 steps.",
            "Search the knowledge base for solutions"
        )
        agent.register_tool(
            "check_service_status",
            lambda task: "All services operational. Last incident: 72h ago.",
            "Check current service health status"
        )
        return agent

    def _create_account_agent(self) -> SpecialistAgent:
        agent = SpecialistAgent(
            name="AccountManagement",
            system_prompt=(
                "You are an account management specialist. You handle "
                "profile updates, team management, permissions, and "
                "account security. You never modify billing data."
            ),
        )
        agent.register_tool(
            "update_profile",
            lambda task: "Profile updated successfully.",
            "Update customer profile information"
        )
        agent.register_tool(
            "manage_team_members",
            lambda task: "Team member added: user@example.com (Admin role).",
            "Add or remove team members"
        )
        return agent

    def triage(self, query: str) -> dict:
        """Route a customer query to the right specialist and return the result."""
        domain_map = {
            "billing": ["invoice", "payment", "refund", "charge", "subscription"],
            "tech": ["error", "bug", "crash", "not working", "install", "login"],
            "account": ["profile", "password", "team", "permission", "security"],
        }
        query_lower = query.lower()
        for domain, keywords in domain_map.items():
            if any(kw in query_lower for kw in keywords):
                agent = getattr(self, f"{domain}_agent")
                result = agent.process(query)
                return {"domain": domain, "result": result, "agent": agent.name}

        return {"domain": "unknown", "result": "Unable to route query.",
                "agent": "none"}


support = CustomerSupportSystem()

queries = [
    "I need a refund for invoice INV-4242",
    "The app crashes when I try to log in",
    "Can you add user@example.com to my team?",
]

for q in queries:
    response = support.triage(q)
    print(f"Query: {q}")
    print(f"Assigned to: {response['agent']}")
    print(f"Result excerpt: {response['result'][:80]}...")
    print()
```

This triage system demonstrates the core benefit of specialist agents: each agent's tool set and prompt is focused, making the system more accurate, easier to debug, and inherently safer.

## Performance Considerations

Each specialist agent processes tasks independently, so they can run in parallel. If a task requires both billing and account changes, run both specialists concurrently and merge their results. This halves wall-clock time compared to a single generalist that handles both domains sequentially.
