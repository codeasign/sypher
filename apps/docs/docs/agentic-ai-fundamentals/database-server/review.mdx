# Review: Database MCP Server

⏱️ 5 minutes

## Key Takeaways

1. Always configure production database MCP servers with `PG_READ_ONLY=true` to prevent accidental writes
2. The agent discovers schemas automatically through `list_tables` and `describe_table` — it does not need you to describe the schema
3. Row limits (default 1000) and query timeouts (default 30 seconds) prevent runaway queries from consuming resources
4. Use separate server configurations for read-only (production) and read-write (staging) access
5. Parameterize all queries in custom database MCP servers to prevent SQL injection through tool parameters

## Recap in 60 Seconds

If you only remember three things:

1. **Default to read-only** — production databases should never be writable through MCP
2. **Let the agent introspect** — it will call `list_tables` and `describe_table` automatically to understand your schema
3. **Set limits** — 1000 rows per query and a 30-second timeout prevent resource exhaustion
