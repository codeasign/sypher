# Review: Configuration Management

⏱️ 5 minutes

## Key Takeaways

1. Configuration must be layered: defaults provide safe fallbacks, environment files override per-environment, secrets are injected from outside, and runtime overrides apply last
2. Validate all configuration at startup with a schema — Pydantic models catch invalid values before they cause silent runtime failures
3. Never store secrets in configuration files or version control — use environment variables or a secrets vault with access control
4. Feature flags let you toggle agent behaviors without redeploying, enabling gradual rollout and A/B testing of agent capabilities
5. Config drift between environments is a leading cause of production incidents — treat config files as code and deploy them through the same pipeline

## Recap in 60 Seconds

If you only remember three things:

1. **Layer your config** — defaults, environment file, secrets, runtime overrides. Each layer only specifies what differs from the layer below.
2. **Validate at startup, not runtime** — a Pydantic model catches a typo in `model.name` before it costs you API calls and debugging time.
3. **Secrets are not config** — API keys, passwords, and tokens live in environment variables or a vault, never in YAML files committed to git.
