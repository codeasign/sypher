# Prompt Templates — Review

*Reading time: 3 minutes*

## Key Takeaways

- Prompt templates separate prompt design from application code, improving maintainability
- Use safe substitution methods that treat variables as literal text to prevent template injection
- Always estimate rendered prompt length and truncate variable content to fit within context windows
- Store templates in external files with versioning for independent updates and rollback capability
- Conditional template sections let you compose prompts from reusable building blocks

## Recap in 60 Seconds

1. Centralize prompts in a template library — application code calls templates by name, not by building prompts inline
2. Handle user input safely with substitution that escapes template syntax
3. Version your templates and store them outside application code for independent deployment
