ChatGPT popularized the conversational AI interface, but it's a hosted service with limitations: no custom instructions, no model switching between providers, no local data control, and subscription costs that add up for teams. Building your own chat interface gives you full control over models, data, features, and cost.

## What You'll Build

You'll build **ChatClone**, a full-stack ChatGPT alternative with:

- **Streaming chat UI** вҖ” real-time token-by-token response display
- **Multi-model support** вҖ” Claude Haiku, Sonnet, GPT-4o-mini, and local Ollama
- **Conversation management** вҖ” create, rename, delete, search conversations
- **Markdown rendering** вҖ” code blocks, tables, lists, images in responses
- **System prompt customization** вҖ” per-conversation system instructions
- **Conversation export** вҖ” download as Markdown, JSON, or plain text
- **API key management** вҖ” bring your own keys for each provider
- **Token usage tracking** вҖ” per-conversation token and cost breakdown

## Technology Decisions

| Component | Choice | Why this, not X |
|---|---|---|
| Frontend | React + Tailwind CSS | Most popular, great ecosystem. Vue is simpler but fewer chat UI examples |
| Streaming | Server-Sent Events (SSE) | Native browser support, simpler than WebSocket for one-way streaming |
| Backend | FastAPI + `sse-starlette` | Native async, SSE support. Flask can't do true streaming without hacks |
| Database | SQLite | Zero config, sufficient for single-user. PostgreSQL needed for multi-tenant |
| Markdown | `react-markdown` + `react-syntax-highlighter` | Renders code blocks beautifully. DangerouslySetInnerHTML is a security risk |
| Local LLM | Ollama | Free, private, runs on consumer hardware. llama.cpp is more powerful but harder to set up |
