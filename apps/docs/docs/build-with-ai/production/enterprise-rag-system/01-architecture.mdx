In 2024,摩根大通 reported that their analysts spent roughly 40% of their time searching for and consolidating information across internal documents, research reports, and regulatory filings before they could begin actual analysis. This isn't unique to finance — every knowledge-intensive organization faces the same bottleneck. Internal knowledge lives in wikis, Slack archives, code comments, Notion docs, and PDFs that search engines can't index and keyword search can't understand.

A standard vector-search RAG pipeline solves the retrieval problem but fails in production: no access control, no audit trail, no handling of updates or deletions, and no cost controls. Building a RAG system that actually works inside an organization means solving for security, freshness, and reliability — not just embedding and similarity search.

## What You'll Build

You'll build **RAGFlow**, a complete enterprise RAG system with these capabilities:

- **Document ingestion pipeline** that processes PDFs, Markdown, plain text, and code files
- **Chunking with metadata** — files split intelligently with source tracking
- **Hybrid search** — vector similarity combined with keyword (BM25) fallback
- **Access control** — per-document access groups enforced at query time
- **Audit logging** — every query, document view, and system change recorded
- **CLI and REST API** — query via terminal or integrate into existing tools
- **Token cost tracking** — per-query breakdown with configurable budgets
- **Streaming responses** — results arrive as the LLM generates them

### Data Flow

1. **Ingestion:** Documents enter via API or CLI, get chunked (recursive character split with overlap), embedded via `text-embedding-3-small`, and stored in ChromaDB with full metadata
2. **Query:** A natural language query arrives → hybrid search runs (vector + BM25) → results filtered by user's access groups → top-K passed to LLM with context → response streamed back
3. **Audit:** Every query, document access, and ingestion event is logged to SQLite with timestamps and user identity

## Technology Decisions

| Component | Choice | Why this, not X |
|---|---|---|
| Vector Store | ChromaDB | Persistent, no external service, supports metadata filtering. Not Pinecone (needs cloud account) or Weaviate (overkill for MVP) |
| Embeddings | `text-embedding-3-small` (OpenAI) | Best quality-per-dollar at $0.02/1M tokens. Trade-off: requires API key — Ollama is free but accuracy drops 5-10% in benchmarks |
| LLM | Claude Haiku 3.5 (default), Sonnet 4 (optional) | Haiku is 5x cheaper than Sonnet for retrieval tasks. Sonnet reserved for complex multi-document synthesis |
| Document Store | SQLite with FTS5 | Zero-config, supports full-text search (BM25 via FTS5). PostgreSQL would scale better but adds setup cost |
| Chunking | RecursiveCharacterTextSplitter (LangChain) | Fast, deterministic, language-aware. Semantic chunking is more accurate but 10x slower for initial builds |
| API Framework | FastAPI | Native async, auto-docs, Pydantic validation. Flask Sync is simpler but blocks on LLM calls |
| Auth | API keys + scoped tokens | Simple to implement, works with existing enterprise API gateways. OAuth2 would be more standard but adds complexity for MVP |

## Data Model

```python
from pydantic import BaseModel
from typing import List, Optional
from datetime import datetime
import uuid

class Document(BaseModel):
    id: str = uuid.uuid4().hex[:12]
    filename: str
    source_path: str
    content_type: str  # "pdf", "md", "txt", "py", "js", etc.
    access_group: str  # "engineering", "legal", "all", etc.
    created_at: datetime = datetime.utcnow()
    updated_at: datetime = datetime.utcnow()
    chunk_count: int = 0
    checksum: str       # SHA-256 of original file

class Chunk(BaseModel):
    id: str = uuid.uuid4().hex[:12]
    document_id: str
    content: str
    chunk_index: int
    embedding: List[float]
    metadata: dict = {}

class QueryLog(BaseModel):
    id: str = uuid.uuid4().hex[:12]
    user: str
    query: str
    access_group: str
    documents_retrieved: List[str]
    tokens_used: int
    latency_ms: int
    timestamp: datetime = datetime.utcnow()

class APIKey(BaseModel):
    key_hash: str        # SHA-256 of the raw key
    user: str
    access_group: str
    is_active: bool = True
    created_at: datetime = datetime.utcnow()
```

All data stored in SQLite except embeddings, which live in ChromaDB with the chunk's `id` as the overlap key.

## API Surface

### Ingestion

| Method | Path | Input | Output | Errors |
|---|---|---|---|---|
| POST | `/api/v1/ingest` | `multipart/form-data` (file + access_group) | `{ "document_id", "chunks", "status" }` | 400 (unsupported type), 401 (bad key) |
| GET | `/api/v1/documents` | query: `?group=` | `[{ id, filename, access_group, chunk_count }]` | 401 |
| DELETE | `/api/v1/documents/{id}` | path param | `{ "deleted": true }` | 404, 401 |

### Query

| Method | Path | Input | Output | Errors |
|---|---|---|---|---|
| POST | `/api/v1/query` | `{ "query", "stream": bool }` | `{ "answer", "sources": [...], "tokens_used", "latency_ms" }` | 401, 422 (validation) |
| GET | `/api/v1/query/history` | query: `?limit=20` | `[{ query, timestamp, tokens_used }]` | 401 |

### Admin

| Method | Path | Input | Output | Errors |
|---|---|---|---|---|
| POST | `/api/v1/admin/keys` | `{ "user", "access_group" }` | `{ "api_key": "rag-skr..." }` | 401 (admin only) |
| DELETE | `/api/v1/admin/keys/{key_hash}` | path param | `{ "deactivated": true }` | 404, 401 |
