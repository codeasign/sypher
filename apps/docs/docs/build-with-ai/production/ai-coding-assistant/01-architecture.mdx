Developers spend roughly 35% of their time reading and understanding existing code, according to a 2023 study by the IEEE. For teams working across multiple languages and large monorepos, that number can reach 50%. The problem isn't writing code — it's understanding what existing code does, why it was written a certain way, and where to make changes safely.

An AI coding assistant that runs locally, understands your full project context, and can explain, review, and modify code reduces context-switching and speeds up onboarding to new codebases.

## What You'll Build

You'll build **CodeMate**, a terminal-based AI coding assistant with:

- **Project indexing** — reads your codebase, builds a searchable index of files, symbols, and imports
- **Code explanation** — explain any function, class, or file in natural language
- **Code review** — finds bugs, style issues, and security problems in a diff or file
- **Code generation** — generate new functions, tests, or entire files from descriptions
- **Context-aware** — understands imports, dependencies, and type relationships across your project
- **Diff preview** — shows proposed changes before applying them
- **Language server integration** — uses tree-sitter for accurate syntax parsing

### Data Flow

1. User runs `codemate explain path/to/file.py --function parse_data`
2. Project indexer finds the file, extracts the function body and its dependencies
3. Context collector gathers relevant imports, type definitions, and callers
4. LLM prompt constructed with the code and context
5. Explanation streamed back to the terminal

## Technology Decisions

| Component | Choice | Why this, not X |
|---|---|---|
| Language Parsing | tree-sitter | Accurate AST parsing for 40+ languages. Regex-based parsing breaks on complex syntax |
| Code Index | ChromaDB | Fast semantic search across the codebase. grep is faster for exact matches but misses semantic relationships |
| LLM | Claude Sonnet 4 | Code understanding requires deep reasoning. Haiku is cheaper but misses subtle bugs in review |
| CLI Framework | Click | Simple, well-documented, great for composable commands. Typer is newer but less battle-tested |
| Diff Engine | difflib (built-in) | Zero dependencies, handles most cases. `git diff` integration for projects already in git |

## Data Model

```python
from pydantic import BaseModel
from typing import Optional
from datetime import datetime

class CodeSymbol(BaseModel):
    name: str
    kind: str  # function, class, method, variable, import
    file_path: str
    start_line: int
    end_line: int
    docstring: Optional[str] = None
    signature: Optional[str] = None

class IndexEntry(BaseModel):
    file_path: str
    language: str
    symbols: list[CodeSymbol]
    last_indexed: datetime
    checksum: str

class ReviewFinding(BaseModel):
    severity: str  # error, warning, info
    line: int
    column: Optional[int] = None
    message: str
    suggestion: str
    category: str  # bug, style, security, perf, design
```
