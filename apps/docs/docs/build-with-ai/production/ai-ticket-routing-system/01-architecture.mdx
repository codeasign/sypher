Large support teams often have specialized agents — billing experts, technical support engineers, account managers, and product specialists. Routing tickets to the right person (or AI agent) is a coordination problem. A supervisor agent that delegates to worker agents, each with its own expertise and tools, can handle this at scale.

## What You'll Build

You'll build **RouteBot**, a multi-agent ticket routing system with:

- **Supervisor agent** — receives the ticket, determines which specialist to route to
- **Billing specialist agent** — handles invoices, payments, refunds, subscription changes
- **Technical support specialist** — handles bugs, errors, configuration, API issues
- **Account specialist** — handles account access, permissions, team management
- **Product specialist** — handles feature requests, roadmap questions, feedback
- **Escalation coordinator** — handles multi-specialty tickets or handoffs

## Technology Decisions

| Component | Choice | Why this, not X |
|---|---|---|
| Orchestration | LangGraph | Native multi-agent support with supervisor pattern. Manual routing doesn't scale |
| Specialist Agents | Claude Haiku 3.5 | Each specialist is a focused prompt — no need for Sonnet's broad reasoning |
| Supervisor | Claude Sonnet 4 | The router needs to understand nuance across all domains |
| Communication | Structured state (shared dict) | Simpler than message queues for this scale. Redis streams would be needed for distributed agents |
