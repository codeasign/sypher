It is 11 PM on a Saturday. A customer just received a shipping confirmation for the wrong item. Your support team is offline until Monday. The customer opens a chat, types "I got the wrong product — what do I do?" and gets... an auto-reply with a link to the returns policy. Not helpful. By Monday, the customer has left a 1-star review and initiated a chargeback.

Good customer support is fast, personalised, and available 24/7. Most companies can't afford a 24-hour support team, but they can build an AI support bot that handles the 80% of routine queries — order status, password resets, refund policies, shipping questions — and seamlessly escalates the remaining 20% to human agents with full context.

In this project you'll build exactly that: an AI-powered customer support bot that answers product and account questions from a knowledge base, handles multi-turn conversations, recognises when it cannot help, and prepares a clean handoff to a human agent.

## What You'll Build

A Python CLI tool that simulates a customer support chat session. The system:

- Loads a knowledge base of FAQs, policies, and product documentation
- Answers customer questions by retrieving relevant knowledge base entries
- Maintains conversation context across multiple turns
- Detects when the query requires human intervention (refunds, account issues, complaints)
- Prepares a structured summary for human agent handoff
- Tracks unresolved issues for follow-up
- Supports customisation with your own product information

## Concepts You Need

**Intent classification.** Before answering, you need to understand what the customer wants: a shipping inquiry, a product question, a complaint, a return request. You classify the intent using keyword matching or the LLM itself. This determines which knowledge base section to query and how to route the conversation.

**Knowledge base retrieval with semantic search.** The support bot needs a source of truth — product descriptions, shipping policies, return windows, troubleshooting guides. Instead of hardcoding answers, you store this information in a knowledge base and retrieve the most relevant entries for each query using embedding similarity. This keeps answers accurate and maintainable.

**Escalation detection and handoff.** Not every query should be handled by AI. The bot must recognise its limits: refund amounts over a threshold, account security issues, complex complaints. When it detects a query that needs a human, it should collect the relevant context and prepare a structured handoff summary so the human agent picks up without repeating questions.

**Conversation state management.** A support conversation is multi-turn — the customer might ask a follow-up ("What about international shipping?") or provide additional details ("My order number is 12345"). The bot must track the conversation state: what the issue is, what information has been collected, and what resolution steps have been attempted.

### Step 2: Build the knowledge base loader

Create `knowledge_base.py`:

```python
import os
from pathlib import Path
from typing import List, Dict
from sentence_transformers import SentenceTransformer
import numpy as np
import faiss

class KnowledgeBase:
    def __init__(self, kb_dir: str = "knowledge_base"):
        self.kb_dir = Path(kb_dir)
        self.entries: List[Dict[str, str]] = []
        self.embeddings = None
        self.index = None
        self.model = SentenceTransformer("all-MiniLM-L6-v2")

    def load(self):
        """Load all markdown files from the knowledge base directory."""
        self.entries = []
        for md_file in sorted(self.kb_dir.glob("*.md")):
            content = md_file.read_text(encoding="utf-8")
            # Split into sections by headings
            sections = content.split("\n## ")
            for i, section in enumerate(sections):
                if section.strip():
                    heading = section.split("\n")[0].strip().lstrip("# ")
                    body = "\n".join(section.split("\n")[1:]).strip()
                    entry = {
                        "source": md_file.stem,
                        "heading": heading,
                        "content": f"## {heading}\n{body}" if heading else content,
                    }
                    self.entries.append(entry)

        # Build index
        texts = [e["content"] for e in self.entries]
        if texts:
            self.embeddings = self.model.encode(texts, show_progress_bar=False)
            self.index = faiss.IndexFlatL2(self.embeddings.shape[1])
            self.index.add(np.array(self.embeddings).astype("float32"))
        print(f"Loaded {len(self.entries)} entries from {len(texts)} sources")

    def search(self, query: str, top_k: int = 3) -> List[Dict[str, str]]:
        """Search the knowledge base for relevant entries."""
        if not self.index or not self.entries:
            return []
        query_vec = self.model.encode([query])
        distances, indices = self.index.search(
            np.array(query_vec).astype("float32"), top_k
        )
        results = []
        for i in indices[0]:
            if i < len(self.entries):
                results.append(self.entries[i])
        return results

    def get_all_text(self) -> str:
        """Return all knowledge base entries as formatted text."""
        return "\n\n".join(
            f"[{e['source']}] {e['content']}" for e in self.entries
        )
```

### Step 3: Build the support bot agent

Create `support_bot.py`:

```python
import os
import sys
import json
from pathlib import Path
from dotenv import load_dotenv

load_dotenv()

ANTHROPIC_API_KEY = os.environ.get("ANTHROPIC_API_KEY")
if not ANTHROPIC_API_KEY:
    print("Error: ANTHROPIC_API_KEY not set in .env")
    sys.exit(1)

import anthropic
client = anthropic.Anthropic(api_key=ANTHROPIC_API_KEY)

from knowledge_base import KnowledgeBase

class SupportBot:
    def __init__(self):
        self.kb = KnowledgeBase()
        self.kb.load()
        self.conversation_history = []
        self.escalated = False
        self.customer_info = {}

    def _classify_intent(self, query: str) -> str:
        """Classify the customer's intent."""
        response = client.messages.create(
            model="claude-3-5-haiku-20241022",
            max_tokens=50,
            system=(
                "Classify this customer support query into exactly one category: "
                "shipping, returns, product_info, account, complaint, other. "
                "Respond with ONLY the category name, nothing else."
            ),
            messages=[{"role": "user", "content": query}]
        )
        return response.content[0].text.strip().lower()

    def _should_escalate(self, query: str) -> tuple:
        """Check if query should be escalated to a human."""
        response = client.messages.create(
            model="claude-3-5-haiku-20241022",
            max_tokens=100,
            system=(
                "Determine if this customer support query requires a human agent. "
                "Return a JSON object: {\"escalate\": bool, \"reason\": \"...\"}\n\n"
                "Escalate for: refund amounts over $100, account security issues, "
                "legal threats, complex complaints, requests for manager, "
                "sensitive account changes, or anything the bot clearly cannot handle."
            ),
            messages=[{"role": "user", "content": query}]
        )
        try:
            return json.loads(response.content[0].text)
        except (json.JSONDecodeError, AttributeError):
            return {"escalate": False, "reason": ""}

    def _get_kb_context(self, query: str) -> str:
        """Get relevant knowledge base context for a query."""
        results = self.kb.search(query)
        if not results:
            return ""
        sections = []
        for r in results:
            sections.append(
                f"[Source: {r['source']} - {r['heading']}]\n{r['content']}"
            )
        return "\n\n".join(sections)

    def respond(self, query: str) -> str:
        """Process a customer query and return a response."""
        # Check for escalation
        escalation = self._should_escalate(query)
        if escalation.get("escalate"):
            self.escalated = True
            handoff = self._prepare_handoff(query)
            self.conversation_history.append({"role": "user", "content": query})
            return f"I need to transfer you to a human agent.\n\n{handoff}"

        # Classify intent and get KB context
        intent = self._classify_intent(query)
        kb_context = self._get_kb_context(query)

        # Build response
        system_prompt = (
            "You are a helpful customer support agent for an online store. "
            "Use the provided knowledge base context to answer the customer's question. "
            "Be friendly, concise, and professional. If the context doesn't contain "
            "the answer, say you need to check with a specialist and offer to escalate. "
            "Ask for order numbers or account details when relevant."
        )
        messages = self.conversation_history + [
            {
                "role": "user",
                "content": (
                    f"Customer intent: {intent}\n\n"
                    f"Knowledge base context:\n{kb_context}\n\n"
                    f"Customer message: {query}"
                )
            }
        ]
        response = client.messages.create(
            model="claude-3-5-haiku-20241022",
            max_tokens=500,
            system=system_prompt,
            messages=messages
        )

        answer = response.content[0].text
        self.conversation_history.append({"role": "user", "content": query})
        self.conversation_history.append({"role": "assistant", "content": answer})
        return answer

    def _prepare_handoff(self, query: str) -> str:
        """Prepare a structured summary for human agent handoff."""
        history_str = "\n".join(
            f"{m['role'].upper()}: {m['content'][:200]}"
            for m in self.conversation_history[-6:]
        )
        return (
            f"\n--- Human Agent Handoff ---\n"
            f"Customer message: {query}\n"
            f"Conversation so far:\n{history_str}\n"
            f"--- End Handoff ---\n"
            f"A human agent will be with you shortly."
        )
```

**What happened:** The `SupportBot` class ties everything together. For each customer query, it checks if escalation is needed, classifies the intent, retrieves relevant knowledge base entries, and generates a response grounded in the KB. Conversation history is maintained so follow-up questions work naturally.

### Step 4: Build the interactive CLI

```python
def main():
    print("=" * 55)
    print("  AI Customer Support Bot")
    print("  (type 'quit' to exit)")
    print("=" * 55)

    bot = SupportBot()

    print("\nBot: Hello! I'm your support assistant. How can I help you today?\n")

    while True:
        query = input("You: ").strip()
        if query.lower() in ("quit", "exit"):
            print("\nBot: Thank you for chatting! Have a great day.")
            break
        if not query:
            continue

        response = bot.respond(query)
        print(f"\nBot: {response}\n")

if __name__ == "__main__":
    main()
```

## Understand What Happened

**What each API call actually did.** Each customer query triggered up to three LLM calls: one for intent classification (50 tokens, cheap), one for escalation check (100 tokens), and one for the main response (500 tokens). The intent classification and escalation check are small, fast prompts that route the conversation. The main response includes the KB context and generates the answer.

**Token usage and cost for a typical run.** A typical 5-turn conversation uses roughly 3,500–5,000 input tokens and 800–1,500 output tokens, costing approximately $0.01–$0.02. Most of the cost is in the main response generation.

**Performance characteristics.** Intent classification and escalation checks each take under 1 second. The main response takes 1–3 seconds. Knowledge base search is sub-millisecond. Total per-turn latency is 2–4 seconds.

**3 common mistakes teams make with this pattern:**

- **Not including conversation history in escalation handoff.** A handoff that only says "customer is unhappy" forces the human agent to ask "What have you already tried?" — defeating the purpose of the bot. Always include the full conversation context and attempted resolutions.

- **Storing the entire conversation in the prompt.** Long conversations exceed context windows and increase cost. Keep only the last 3–5 exchanges in the history sent to the LLM. Older context should be summarised rather than included verbatim.

- **Failing to handle off-topic queries.** Without a guardrail, the support bot can answer questions about cooking recipes or philosophy. Always include a system prompt instruction: "If the query is not about our products or services, politely redirect to support topics."
