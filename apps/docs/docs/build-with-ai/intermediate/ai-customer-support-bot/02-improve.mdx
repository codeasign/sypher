## Improve It

### 1. Add sentiment tracking for emotional awareness

Detect customer emotion and adjust the bot's tone accordingly:

```python
import json

def analyze_sentiment(query: str) -> dict:
    """Analyze customer sentiment from their message."""
    response = client.messages.create(
        model="claude-3-5-haiku-20241022",
        max_tokens=100,
        system=(
            "Analyze the customer's sentiment. Return JSON with: "
            "sentiment (positive/neutral/negative/frustrated/angry), "
            "urgency (low/medium/high), "
            "and a one-sentence summary of their emotional state."
        ),
        messages=[{"role": "user", "content": query}]
    )
    try:
        return json.loads(response.content[0].text)
    except (json.JSONDecodeError, AttributeError):
        return {"sentiment": "neutral", "urgency": "low"}

class SupportBot:
    def respond(self, query: str) -> str:
        sentiment = analyze_sentiment(query)
        # Adjust response based on sentiment
        tone_guide = {
            "positive": "Match their positive tone, be warm.",
            "neutral": "Be professional and helpful.",
            "negative": "Acknowledge their frustration, apologise genuinely.",
            "frustrated": "Empathize first, then offer concrete solutions. "
                         "Keep responses shorter than usual.",
            "angry": "Apologize sincerely. Focus on solving the problem. "
                    "Avoid explanations or policy justifications early in the response.",
        }
        tone = tone_guide.get(sentiment.get("sentiment", "neutral"),
                              "Be professional and helpful.")
        # ... rest of respond() with sentiment added to system prompt
```

**Why it matters in production:** A frustrated customer who gets a cheerful "Have a great day!" message feels unheard. Adjusting tone based on sentiment — apologetic for angry customers, warm for happy ones — improves satisfaction scores by up to 30%.

### 2. Add conversation analytics and reporting

Track metrics to understand bot performance:

```python
import json
from datetime import datetime, date
from collections import Counter

class AnalyticsTracker:
    def __init__(self, log_path: str = "support_analytics.json"):
        self.log_path = Path(log_path)
        self.sessions = self._load()

    def _load(self) -> list:
        if self.log_path.exists():
            data = self.log_path.read_text(encoding="utf-8")
            return json.loads(data) if data.strip() else []
        return []

    def log_session(self, session: dict):
        self.sessions.append(session)
        self._save()

    def _save(self):
        self.log_path.write_text(json.dumps(self.sessions, indent=2), encoding="utf-8")

    def generate_report(self):
        today = date.today()
        today_sessions = [s for s in self.sessions
                          if s.get("date", "").startswith(str(today))]
        total = len(self.sessions)
        today_count = len(today_sessions)
        escalated = sum(1 for s in self.sessions if s.get("escalated"))
        resolved = sum(1 for s in self.sessions if s.get("resolved"))

        intent_counts = Counter(
            s.get("intent") for s in self.sessions if s.get("intent")
        )

        print(f"\n{'=' * 45}")
        print(f"Support Bot Analytics Report")
        print(f"{'=' * 45}")
        print(f"Total conversations: {total}")
        print(f"Today: {today_count}")
        print(f"Resolved without escalation: {resolved} "
              f"({resolved / total * 100:.0f}%" if total else "N/A")
        print(f"Escalated to human: {escalated} "
              f"({escalated / total * 100:.0f}%" if total else "N/A")
        print(f"\nIntent breakdown:")
        for intent, count in intent_counts.most_common():
            print(f"  {intent}: {count} ({count / total * 100:.0f}%)")
```

**Why it matters in production:** Analytics answer the critical questions: What are customers asking about most? How often does the bot resolve without escalation? Which intents have the highest escalation rate? This data drives knowledge base improvements and identifies gaps.

### 3. Add proactive greeting based on detected intent

Instead of a generic greeting, personalise based on page context:

```python
def generate_greeting(page_context: str = None) -> str:
    """Generate a contextual greeting based on where the customer is."""
    if not page_context:
        return "Hello! I'm your support assistant. How can I help you today?"

    response = client.messages.create(
        model="claude-3-5-haiku-20241022",
        max_tokens=100,
        system=(
            "Generate a short, friendly support greeting based on which "
            "page the customer is on. Be specific, not generic."
        ),
        messages=[{
            "role": "user",
            "content": f"The customer is on the {page_context} page. Generate a greeting."
        }]
    )
    return response.content[0].text
```

**Why it matters in production:** A customer on the returns page who gets "Welcome! How can I help?" misses the opportunity for "I see you're looking at returns — I can help you start a return in under a minute." Contextual greetings increase engagement rates significantly.

### 4. Add rate limiting and token budget

```python
import time
from collections import deque

class RateLimiter:
    def __init__(self, max_calls: int = 30, window_seconds: int = 60):
        self.max_calls = max_calls
        self.window = window_seconds
        self.call_timestamps = deque()

    def can_proceed(self) -> bool:
        now = time.time()
        # Remove old timestamps
        while self.call_timestamps and self.call_timestamps[0] < now - self.window:
            self.call_timestamps.popleft()
        if len(self.call_timestamps) >= self.max_calls:
            return False
        self.call_timestamps.append(now)
        return True

class SupportBot:
    def __init__(self):
        self.rate_limiter = RateLimiter()
        # ... existing init

    def respond(self, query: str) -> str:
        if not self.rate_limiter.can_proceed():
            return ("I'm experiencing high demand right now. "
                    "Please wait a moment and try again, or email us at support@example.com.")
        # ... rest of respond
```

**Why it matters in production:** Without rate limiting, a single abusive user or a traffic spike can exhaust your API budget in minutes. Rate limiting with a token budget protects both your costs and your API rate limits.

### 5. Add logging and error handling

```python
import logging

logger = logging.getLogger(__name__)

            escalation = self._should_escalate(query)
            if escalation.get("escalate"):
                logger.warning(f"Escalation triggered: {escalation.get('reason')}")
                # ... escalation handling
            intent = self._classify_intent(query)
            logger.info(f"Classified as: {intent}")
            # ... rest
        except Exception as e:
            logger.error(f"Error processing query: {e}", exc_info=True)
```

**Why it matters in production:** Logging every query, intent classification, escalation, and error gives you the data to debug issues, identify patterns, and improve the bot. Without logs, a bot that starts giving bad answers is invisible until customers complain.

## Deploy It

Deploy as a FastAPI web chat interface on Railway.

### Step 1: Create the web application

Create `app.py`:

```python
import os
import sys
from pathlib import Path
from dotenv import load_dotenv
from fastapi import FastAPI, Form
from fastapi.responses import HTMLResponse
from pydantic import BaseModel

load_dotenv()
ANTHROPIC_API_KEY = os.environ.get("ANTHROPIC_API_KEY")
if not ANTHROPIC_API_KEY:
    print("Error: ANTHROPIC_API_KEY not set")
    sys.exit(1)

import anthropic
client = anthropic.Anthropic(api_key=ANTHROPIC_API_KEY)

from support_bot import SupportBot, analyze_sentiment

app = FastAPI(title="Customer Support Bot")

# Global bot instance
bot = SupportBot()
sessions = {}

class ChatRequest(BaseModel):
    session_id: str
    message: str

@app.get("/", response_class=HTMLResponse)
async def index():
    return """
    <html>
    <head><title>Customer Support</title>
    <style>
      body { font-family: system-ui, sans-serif; max-width: 700px; margin: 0 auto; padding: 2rem; }
      #chat { height: 400px; overflow-y: auto; border: 1px solid #ddd; padding: 1rem; border-radius: 8px; margin-bottom: 1rem; }
      .msg { margin: 0.5rem 0; padding: 0.5rem 1rem; border-radius: 8px; }
      .user { background: #007bff; color: #fff; margin-left: 20%; }
      .bot { background: #f1f1f1; margin-right: 20%; }
      .input-row { display: flex; gap: 0.5rem; }
      input { flex: 1; padding: 0.75rem; border: 1px solid #ddd; border-radius: 4px; }
      button { padding: 0.75rem 1.5rem; background: #007bff; color: #fff; border: none; border-radius: 4px; cursor: pointer; }
    </style></head>
    <body>
      <h1>Customer Support</h1>
      <div id="chat"><div class="msg bot">Hello! How can I help you today?</div></div>
      <div class="input-row">
        <input type="text" id="input" placeholder="Type your message..." />
        <button onclick="send()">Send</button>
      </div>
      <script>
        let sessionId = 's' + Date.now();
        async function send() {
          const input = document.getElementById('input');
          const msg = input.value.trim();
          if (!msg) return;
          input.value = '';
          document.getElementById('chat').innerHTML += '<div class="msg user">' + msg + '</div>';
          const res = await fetch('/chat', {
            method: 'POST',
            headers: {'Content-Type': 'application/json'},
            body: JSON.stringify({ session_id: sessionId, message: msg })
          });
          const data = await res.json();
          document.getElementById('chat').innerHTML += '<div class="msg bot">' + data.response + '</div>';
          document.getElementById('chat').scrollTop = document.getElementById('chat').scrollHeight;
        }
        document.getElementById('input').addEventListener('keypress', function(e) {
          if (e.key === 'Enter') send();
        });
      </script>
    </body>
    </html>
    """

@app.post("/chat")
async def chat(req: ChatRequest):
    session = sessions.get(req.session_id)
    if not session:
        session = {"history": []}
        sessions[req.session_id] = session
    response = bot.respond(req.message)
    return {"response": response}
```

## Challenge Yourself

**Beginner — Add a knowledge base admin command.** Create a `--add-faq` CLI command that lets you add a new FAQ entry to the knowledge base without editing a file. The entry should be embedded and indexed immediately without restarting the bot.

**Intermediate — Multi-language support.** Add automatic language detection to the bot. If a customer messages in Spanish, the bot should detect the language, respond in Spanish, and search translated versions of the knowledge base. Use the LLM to translate KB entries on the fly.

**Advanced — Ticket system integration.** Build a simple ticketing system that creates a ticket for each escalation. The ticket should include the customer's messages, the bot's responses, the escalation reason, and a priority score. Store tickets in a JSON file and provide a `--list-tickets` command to view open tickets.
