You had an idea on your morning walk. By the time you got to your desk, it was gone. You recorded a voice memo during a meeting but never transcribed it. You have 47 audio notes on your phone, each labelled "Voice Memo 001" through "Voice Memo 047," and you have no idea which one contains the brilliant thought you had about the product roadmap last Tuesday.

Voice notes are the most natural way to capture ideas — you speak faster than you type, and talking feels like thinking. But raw audio is nearly impossible to search, skim, or organise. A 5-minute voice memo might contain three distinct ideas, two action items, and a question — but all you see is a length in minutes and a default filename.

In this project you'll build a tool that changes that: an AI voice notes summariser that transcribes audio files, extracts key insights, categorises the content, and produces structured, searchable summaries. By the end, you'll be able to record a voice memo, run it through the tool, and get a clean document you can search, share, and act on.

## What You'll Build

A Python CLI tool that processes audio recordings and produces structured summaries. The system:

- Accepts audio files in common formats (MP3, WAV, M4A)
- Transcribes speech to text using OpenAI Whisper (local, free, and private)
- Analyses the transcript with an LLM to extract key points, action items, and questions
- Categorises the note by type (idea, meeting, todo, journal, lecture)
- Outputs a structured Markdown summary with title, category, key points, and action items
- Archives summaries in a searchable local database

## Concepts You Need

**Automatic Speech Recognition (ASR) with Whisper.** OpenAI Whisper is a local speech-to-text model that runs on your machine. It transcribes audio into text with high accuracy, even with background noise, multiple speakers, and different accents. You use the `whisper` Python package to load the model and transcribe files. The "small" model runs on a laptop CPU in roughly real-time (1 minute of audio takes ~1 minute to transcribe).

**Audio format handling.** Audio files come in many formats and sample rates. You use `ffmpeg` (via Whisper or pydub) to normalise the audio to the format the model expects: 16kHz mono WAV. This is a standard preprocessing step for any ASR pipeline.

**LLM-based text structuring.** Raw transcripts are messy — filled with false starts, filler words, and run-on sentences. The LLM cleans this up: it extracts the core ideas, organises them into a logical structure, identifies action items, and generates a concise title. This transforms rambling speech into a clean, readable document.

## Build It

### Step 1: Set up the transcription module

Create `voice_note.py`:

```python
import os
import sys
import json
from pathlib import Path
from datetime import datetime
from dotenv import load_dotenv

load_dotenv()

ANTHROPIC_API_KEY = os.environ.get("ANTHROPIC_API_KEY")
if not ANTHROPIC_API_KEY:
    print("Error: ANTHROPIC_API_KEY not set in .env")
    sys.exit(1)

import anthropic
client = anthropic.Anthropic(api_key=ANTHROPIC_API_KEY)

import whisper

def transcribe_audio(audio_path: str, model_size: str = "small") -> dict:
    """Transcribe an audio file using Whisper."""
    print(f"Loading Whisper model ({model_size})...")
    model = whisper.load_model(model_size)
    print(f"Transcribing: {audio_path}")
    result = model.transcribe(audio_path)
    print(f"Transcription complete: {len(result['text'])} characters")
    return result
```

```bash
"
```

**What happened:** `whisper.load_model("small")` downloads the ~460MB model on first run and caches it. The `model.transcribe()` call loads the audio, normalises it to 16kHz mono, runs it through the neural network, and returns the transcribed text along with per-segment timestamps and confidence scores.

### Step 2: Build the transcript analysis function

Add the function that analyses the transcript with an LLM:

```python
SUMMARY_PROMPT = """You are analysing a voice note transcript. Extract the following:

1. TITLE: A concise, descriptive title (5-10 words)
2. CATEGORY: One of: idea, meeting, todo, journal, lecture, other
3. KEY POINTS: 3-7 bullet points capturing the main ideas
4. ACTION ITEMS: Any tasks or follow-ups mentioned, with owner if specified
5. QUESTIONS: Any open questions raised in the recording
6. SUMMARY: 2-3 paragraph summary that reads like a polished document

Format the output as Markdown with clear section headings.
Clean up the transcript: remove filler words, fix run-on sentences, organise logically.
Maintain the original meaning and any specific details, numbers, or names."""

def analyse_transcript(transcript: str, duration_seconds: float = 0) -> str:
    """Analyse a transcript and produce a structured summary."""
    duration_info = f"\nAudio duration: {duration_seconds:.1f} seconds"
    if duration_seconds > 0:
        duration_info += f" ({duration_seconds / 60:.1f} minutes)"

    response = client.messages.create(
        model="claude-3-5-haiku-20241022",
        max_tokens=1500,
        system=SUMMARY_PROMPT,
        messages=[{
            "role": "user",
            "content": f"Transcript:{duration_info}\n\n{transcript}"
        }]
    )

    summary = response.content[0].text
    input_tokens = response.usage.input_tokens
    output_tokens = response.usage.output_tokens
    print(f"  Analysis: {input_tokens} in / {output_tokens} out "
          f"(~${input_tokens * 3e-6 + output_tokens * 15e-6:.4f})")
    return summary
```

**What happened:** The LLM prompt is designed to transform raw, messy speech into a structured document. It extracts the essence of the recording while discarding the verbal noise. The output is clean Markdown that reads like a well-written note.

### Step 3: Build the main processing function

```python
def process_audio(audio_path: str, whisper_model_size: str = "small") -> dict:
    """Process an audio file: transcribe + analyse. Returns structured result."""
    audio_path = str(audio_path)
    transcription = transcribe_audio(audio_path, whisper_model_size)
    transcript_text = transcription["text"]
    duration = transcription.get("duration", 0)

    print("Analysing transcript...")
    summary = analyse_transcript(transcript_text, duration)

    # Extract category from the summary
    category = "other"
    for line in summary.split("\n"):
        if line.lower().startswith("category"):
            category = line.split(":", 1)[-1].strip().lower()
            break

    return {
        "audio_file": Path(audio_path).name,
        "transcript": transcript_text,
        "summary": summary,
        "category": category,
        "duration": duration,
        "timestamp": datetime.now().isoformat(),
    }
```

### Step 4: Build the archive module

Create `archive.py`:

```python
import json
from pathlib import Path
from datetime import datetime
from typing import List, Dict, Optional

class Archive:
    def __init__(self, db_path: str = "voice_notes_archive.json"):
        self.db_path = Path(db_path)
        self.notes = self._load()

    def _load(self) -> list:
        if self.db_path.exists():
            data = self.db_path.read_text(encoding="utf-8")
            return json.loads(data) if data.strip() else []
        return []

    def _save(self):
        self.db_path.write_text(json.dumps(self.notes, indent=2), encoding="utf-8")

    def add(self, note: dict) -> str:
        """Add a note to the archive. Returns the note ID."""
        note_id = f"note_{datetime.now().strftime('%Y%m%d_%H%M%S')}"
        note["id"] = note_id
        self.notes.insert(0, note)
        self._save()
        return note_id

    def list_notes(self, category: Optional[str] = None) -> List[Dict]:
        """List all notes, optionally filtered by category."""
        if category:
            return [n for n in self.notes if n.get("category") == category]
        return self.notes

    def search(self, query: str) -> List[Dict]:
        """Simple keyword search across notes."""
        query = query.lower()
        results = []
        for note in self.notes:
            if (query in note.get("summary", "").lower()
                    or query in note.get("transcript", "").lower()):
                results.append(note)
        return results

    def get(self, note_id: str) -> Optional[Dict]:
        """Get a specific note by ID."""
        for note in self.notes:
            if note.get("id") == note_id:
                return note
        return None

    def delete(self, note_id: str) -> bool:
        """Delete a note by ID."""
        for i, note in enumerate(self.notes):
            if note.get("id") == note_id:
                self.notes.pop(i)
                self._save()
                return True
        return False

    def stats(self) -> dict:
        """Return archive statistics."""
        categories = {}
        for note in self.notes:
            cat = note.get("category", "other")
            categories[cat] = categories.get(cat, 0) + 1
        total_duration = sum(n.get("duration", 0) for n in self.notes)
        return {
            "total_notes": len(self.notes),
            "categories": categories,
            "total_duration_minutes": round(total_duration / 60, 1),
        }
```

**What happened:** The archive module stores all processed voice notes in a JSON file. It supports listing, searching, filtering by category, and deleting. The JSON format is simple, portable, and easily backed up. No database server needed.

### Step 5: Build the main CLI

Add the main function to `voice_note.py`:

```python
from archive import Archive

def main():
    archive = Archive()

    print("=" * 50)
    print("  AI Voice Notes Summarizer")
    print("=" * 50)

    while True:
        print("\nCommands:")
        print("  process <audio_file>   - Transcribe and summarise a voice note")
        print("  list [category]        - List archived notes")
        print("  search <query>         - Search notes")
        print("  get <note_id>          - View a specific note")
        print("  stats                  - Archive statistics")
        print("  quit                   - Exit")
        print("-" * 50)

        cmd = input("> ").strip().split(maxsplit=1)
        if not cmd:
            continue

        action = cmd[0].lower()

        if action == "quit":
            break

        elif action == "process":
            if len(cmd) < 2:
                print("Usage: process <audio_file>")
                continue
            audio_path = cmd[1]
            if not os.path.exists(audio_path):
                print(f"File not found: {audio_path}")
                continue
            result = process_audio(audio_path)
            note_id = archive.add(result)
            print(f"\n{result['summary']}\n")
            print(f"Saved to archive: {note_id}")

        elif action == "list":
            category = cmd[1].lower() if len(cmd) > 1 else None
            notes = archive.list_notes(category)
            if not notes:
                print("No notes found")
                continue
            print(f"\n{'ID':<30} {'Category':<12} {'Duration':<10} {'Title'}")
            print("-" * 80)
            for note in notes:
                # Extract title from summary
                title_line = ""
                for line in note.get("summary", "").split("\n"):
                    if line.strip().startswith("## Title"):
                        title_line = line.split(":", 1)[-1].strip()
                        break
                dur = f"{note.get('duration', 0):.0f}s"
                print(f"{note['id']:<30} {note.get('category', '?'):<12} {dur:<10} {title_line[:40]}")

        elif action == "search":
            if len(cmd) < 2:
                print("Usage: search <query>")
                continue
            results = archive.search(cmd[1])
            print(f"Found {len(results)} matching notes:")
            for r in results:
                print(f"  {r['id']}: {r.get('audio_file', 'unknown')}")

        elif action == "get":
            if len(cmd) < 2:
                print("Usage: get <note_id>")
                continue
            note = archive.get(cmd[1])
            if note:
                print(f"\n{note['summary']}")
                print(f"\n--- Transcript ---")
                print(note["transcript"][:500] + "...")
            else:
                print(f"Note not found: {cmd[1]}")

        elif action == "stats":
            stats = archive.stats()
            print(f"Total notes: {stats['total_notes']}")
            print(f"Total duration: {stats['total_duration_minutes']} minutes")
            print(f"\nCategories:")
            for cat, count in stats['categories'].items():
                print(f"  {cat}: {count}")

        else:
            print(f"Unknown command: {action}")

if __name__ == "__main__":
    main()
```

## Understand What Happened

**What each API call actually did.** The Whisper model ran entirely on your local machine — no internet, no API cost. It loaded the audio, resampled it to 16kHz mono, ran it through the encoder-decoder transformer, and output text with per-segment timestamps. The Anthropic API call then processed the raw transcript and returned a structured summary.

**Token usage and cost for a typical run.** A 3-minute voice note (~500 words) uses roughly 800–1,000 input tokens for the LLM analysis and generates ~400–500 output tokens. The cost is approximately $0.002 per note. The Whisper transcription is free (local). A batch of 50 voice notes costs under $0.10.

**Performance characteristics.** Whisper "small" model processes audio at roughly real-time speed on a modern laptop. A 3-minute recording takes ~3 minutes to transcribe on CPU, ~30 seconds on GPU. The LLM analysis adds 2–3 seconds. The system is CPU-bound during transcription and network-bound during analysis.

**3 common mistakes teams make with this pattern:**

- **Using a Whisper model that's too large for the hardware.** The "large" model is more accurate but 10x slower. For voice notes with clear speech, the "small" or "base" model is sufficient and runs 5x faster. Always match the model size to the audio quality and hardware.

- **Not handling long recordings.** Whisper has a context window of ~30 seconds of audio per segment. For recordings longer than a few minutes, Whisper handles the segmentation automatically, but very long recordings (1 hour+) can produce inconsistent transcript quality. Chunk long recordings into 10-minute segments for best results.

- **Skipping audio preprocessing.** Low-quality recordings with background noise produce poor transcripts. Using a noise gate or high-pass filter before transcription can dramatically improve accuracy. In production, add a preprocessing step with `pydub` or `sox` to normalise volume and reduce noise.
