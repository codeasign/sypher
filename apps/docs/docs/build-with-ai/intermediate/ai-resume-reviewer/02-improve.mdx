## Improve It

### 1. Add PDF resume parsing with PyMuPDF

Many resumes come as PDFs. Add automatic text extraction:

```python
import fitz  # PyMuPDF

def extract_resume_from_pdf(pdf_path: str) -> str:
    """Extract text from a PDF resume."""
    try:
        doc = fitz.open(pdf_path)
        text_parts = []
        for page in doc:
            page_text = page.get_text()
            if page_text.strip():
                text_parts.append(page_text)
        doc.close()
        full_text = "\n\n".join(text_parts)
        print(f"Extracted {len(full_text)} characters from PDF ({len(doc)} pages)")
        return full_text
    except Exception as e:
        print(f"Error reading PDF: {e}")
        return ""

def load_text(source: str) -> str:
    """Load text from a file or return the input as-is."""
    path = Path(source)
    if not path.exists():
        return source
    if path.suffix.lower() == ".pdf":
        return extract_resume_from_pdf(source)
    return path.read_text(encoding="utf-8")
```

**Why it matters in production:** Most resumes are PDFs. Requiring users to paste text from a PDF adds friction and loses formatting information. Direct PDF parsing makes the tool seamless — upload and go.

### 2. Add before-and-after score comparison

Track how resume changes affect the score:

```python
import json
from datetime import datetime

class ScoreTracker:
    def __init__(self, db_path: str = "score_history.json"):
        self.db_path = Path(db_path)
        self.history = self._load()

    def _load(self) -> list:
        if self.db_path.exists():
            data = self.db_path.read_text(encoding="utf-8")
            return json.loads(data) if data.strip() else []
        return []

    def _save(self):
        self.db_path.write_text(json.dumps(self.history, indent=2), encoding="utf-8")

    def record_review(self, resume_id: str, analysis: dict):
        entry = {
            "resume_id": resume_id,
            "timestamp": datetime.now().isoformat(),
            "overall_score": analysis.get("overall_score"),
            "ats_score": analysis.get("ats_fit_score"),
            "dimensions": analysis.get("dimensions", {})
        }
        self.history.append(entry)
        self._save()
        print(f"Score recorded: {entry['overall_score']}/100")

    def show_progress(self, resume_id: str):
        entries = [e for e in self.history if e["resume_id"] == resume_id]
        if not entries:
            print(f"No history for resume: {resume_id}")
            return
        print(f"\nScore Progress for {resume_id}:")
        for i, entry in enumerate(entries):
            print(f"  #{i + 1}: {entry['overall_score']}/100 "
                  f"(ATS: {entry['ats_score']}/100) - {entry['timestamp'][:10]}")
        if len(entries) > 1:
            first = entries[0]["overall_score"]
            last = entries[-1]["overall_score"]
            change = last - first
            sign = "+" if change >= 0 else ""
            print(f"\n  Change: {sign}{change} points")
```

**Why it matters in production:** A single score is a snapshot. Tracking scores over iterations shows whether resume changes are actually moving the needle. Job seekers who iterate based on feedback see 20–40 point improvements after 3–4 revision cycles.

### 3. Add batch comparison for multiple candidates

Compare multiple resumes against the same job description:

```python
from concurrent.futures import ThreadPoolExecutor, as_completed

def batch_compare(resume_files: list, jd_text: str) -> list:
    """Compare multiple resumes against one job description."""
    results = []

    def process_resume(resume_file):
        resume_text = load_text(resume_file)
        analysis = analyze_resume(resume_text, jd_text)
        return {
            "file": Path(resume_file).name,
            "overall_score": analysis.get("overall_score"),
            "ats_score": analysis.get("ats_fit_score"),
            "strengths": analysis.get("strengths", []),
            "critical_gaps": analysis.get("critical_gaps", []),
        }

    with ThreadPoolExecutor(max_workers=5) as executor:
        futures = {executor.submit(process_resume, f): f for f in resume_files}
        for future in as_completed(futures):
            results.append(future.result())

    # Sort by score descending
    results.sort(key=lambda r: r.get("overall_score", 0), reverse=True)

    print(f"\n{'=' * 60}")
    print(f"Batch Comparison Results ({len(results)} candidates)")
    print(f"{'=' * 60}")
    print(f"{'Rank':<6} {'Candidate':<20} {'Score':<8} {'ATS':<8} {'Gaps'}")
    print(f"{'-' * 60}")
    for rank, r in enumerate(results, 1):
        gaps = "; ".join(r.get("critical_gaps", [])[:2])
        print(f"{rank:<6} {r['file']:<20} {r['overall_score']:<8} "
              f"{r['ats_score']:<8} {gaps[:35]}")

    return results
```

**Why it matters in production:** Recruiters screening multiple candidates need a ranked comparison, not individual reports. Batch comparison with parallel API calls processes 10 candidates in the same wall-clock time as one, making it practical for real hiring workflows.

### 4. Add retry with exponential backoff

```python
import time
import random

def analyze_resume_with_retry(
    resume_text: str,
    job_description: str,
    max_retries: int = 3
) -> dict:
    """Analyse with automatic retry on transient errors."""
    last_error = None
    for attempt in range(max_retries):
        try:
            return analyze_resume(resume_text, job_description)
        except (anthropic.APIError, anthropic.APITimeoutError,
                anthropic.RateLimitError, anthropic.InternalServerError) as e:
            last_error = e
            if attempt < max_retries - 1:
                wait = (2 ** attempt) + random.uniform(0, 1)
                print(f"API error (attempt {attempt + 1}), retrying in {wait:.1f}s...")
                time.sleep(wait)
    print(f"Failed after {max_retries} attempts")
    return {"overall_score": 0, "error": str(last_error)}
```

**Why it matters in production:** Transient API errors are inevitable in production. Retry with backoff turns a 1% error rate into a 0.001% effective error rate after 3 retries, making the service reliable enough for user-facing use.

## Deploy It

Deploy as a FastAPI web application on Railway with a simple form interface.

### Step 1: Create the web application

Create `app.py`:

```python
import os
import sys
import json
from pathlib import Path
from dotenv import load_dotenv
from fastapi import FastAPI, UploadFile, File, Form, HTTPException
from fastapi.responses import HTMLResponse
from typing import Optional

load_dotenv()

ANTHROPIC_API_KEY = os.environ.get("ANTHROPIC_API_KEY")
if not ANTHROPIC_API_KEY:
    print("Error: ANTHROPIC_API_KEY not set")
    sys.exit(1)

import anthropic
client = anthropic.Anthropic(api_key=ANTHROPIC_API_KEY)

# Import your reviewer functions
from reviewer import analyze_resume, format_report, load_text

app = FastAPI(title="AI Resume Reviewer")

def extract_text_from_upload(file: UploadFile) -> str:
    content = file.file.read()
    if file.filename.endswith(".pdf"):
        import fitz
        doc = fitz.open(stream=content, filetype="pdf")
        text = "\n\n".join(page.get_text() for page in doc)
        doc.close()
        return text
    return content.decode("utf-8")

@app.get("/", response_class=HTMLResponse)
async def index():
    return """
    <html>
    <head><title>AI Resume Reviewer</title>
    <style>
      body { font-family: system-ui, sans-serif; max-width: 900px; margin: 0 auto; padding: 2rem; }

    <body>
      <h1>AI Resume Reviewer</h1>
      <form id="reviewForm">
        <h3>Resume</h3>

        <h3>Job Description</h3>

        document.getElementById('resumeFile').onchange = async (e) => {

          if (!file) return;

          document.querySelector('textarea[name="resume"]').value = text;
        };
        document.getElementById('reviewForm').onsubmit = async (e) => {
          e.preventDefault();
          document.getElementById('loading').style.display = 'block';
          const form = new FormData(e.target);
          const res = await fetch('/review', { method: 'POST', body: form });
          const data = await res.json();
          document.getElementById('loading').style.display = 'none';
          document.getElementById('report').innerText = data.report;
        };
      </script>
    </body>
    </html>
    """

@app.post("/review")
async def review(
    resume: str = Form(...),
    jd: str = Form(...)
):
    if not resume.strip() or not jd.strip():
        raise HTTPException(400, "Both resume and job description are required")
    analysis = analyze_resume(resume, jd)
    report = format_report(analysis)
    return {"report": report, "score": analysis.get("overall_score")}
```

## Challenge Yourself

**Beginner — Skill gap heatmap.** Create a visual heatmap (using HTML/CSS) that highlights which required skills from the job description are present, partially present, or missing in the resume. Colour-code it: green for strong matches, yellow for partial, red for gaps. Display it above the text report.

**Intermediate — Coach mode with personalised suggestions.** Add a "Coach Mode" that analyses the resume without a job description and provides general improvement suggestions: career narrative, formatting gaps, missing sections (summary, achievements, certifications). Use a separate prompt template for this mode.

**Advanced — Resume rewriter.** Build a feature that generates rewritten versions of weak resume sections. The user highlights a section (e.g. "Fixed bugs and improved test coverage") and the tool suggests three rewritten versions that use stronger action verbs, quantify impact, and align with the job description. Use few-shot prompting with before/after examples.
