## Deploy It

### Dockerfile

```dockerfile
FROM python:3.11-slim AS builder
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

FROM python:3.11-slim
WORKDIR /app
COPY --from=builder /usr/local/lib/python3.11/site-packages /usr/local/lib/python3.11/site-packages
COPY . .
RUN mkdir -p data
EXPOSE 8000
CMD ["uvicorn", "app.api:app", "--host", "0.0.0.0", "--port", "8000"]
```

### GitHub Actions CI/CD

```yaml
# .github/workflows/deploy.yml
name: Deploy ResearchNet
on:
  push:
    branches: [main]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"
      - run: pip install -r requirements.txt
      - run: pytest tests/

  deploy:
    needs: test
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: superfly/flyctl-actions/setup-flyctl@master
      - run: flyctl deploy --remote-only
        env:
          FLY_API_TOKEN: ${{ secrets.FLY_API_TOKEN }}
```

## Portfolio Presentation

### 3-Sentence Summary

"I built ResearchNet, a multi-agent research platform where an orchestrator agent decomposes questions, specialized agents research in parallel, a critic reviews findings, and the system iterates until the report meets quality standards. It uses LangGraph for agent orchestration, Claude Sonnet 4 for complex reasoning, and Haiku 3.5 for high-volume parallel research tasks. The system produces cited, structured reports in 2-5 minutes at a cost of $0.05-0.20 per session."

### 5 Technical Decisions to Highlight

1. **Orchestrator uses Sonnet 4, researchers use Haiku 3.5** — Right model for the right job
2. **Iterative refinement with a critic agent** — Not a single-pass system
3. **DuckDuckGo API over Google Search API** — Zero cost, same quality for general research
4. **SSE streaming** — Real-time agent activity without WebSocket complexity
5. **LangGraph state graph** — Testable, debuggable agent workflows

### 3 Interview Questions to Expect

1. "How do you prevent infinite loops in the iterative refinement?" — Answer: max_iterations check and confidence threshold
2. "How do you handle conflicting information from different sources?" — Answer: Fact-checker cross-references, flagged contradictions in the report
3. "What's the bottleneck in production?" — Answer: Sequential research of sub-questions (can parallelize), and LLM latency
