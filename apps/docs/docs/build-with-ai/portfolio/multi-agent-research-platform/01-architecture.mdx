Enterprise research teams spend 40-60% of their time gathering and synthesizing information before producing analysis. McKinsey estimates that knowledge workers spend 1.8 hours daily — 9.3 hours per week — searching for information. A multi-agent research system that parallelizes search, analysis, fact-checking, and synthesis reduces this to minutes while producing more thorough, cited results than any single pass.

## What You'll Build

You'll build **ResearchNet**, a multi-agent research platform with:

- **Orchestrator agent** — decomposes a research question into sub-questions and assigns them to specialists
- **Web researcher agent** — searches the web, fetches pages, extracts relevant content
- **Code analyzer agent** — reads and analyzes code repositories, documentation
- **Fact-checker agent** — cross-references claims across multiple sources
- **Synthesis agent** — combines findings into a structured report
- **Critic agent** — reviews the report for gaps, contradictions, and unsupported claims
- **Iterative refinement** — the critic's feedback triggers another research cycle
- **Web UI** — real-time view of agent activity, progress, and final report
- **Persistence** — full research session history with searchable results

## Component Breakdown

| Component | Responsibility | Technology | Interface |
|---|---|---|---|
| Orchestrator | Decompose query, assign tasks, manage state, trigger iteration | Claude Sonnet 4 | Receives query, emits structured task list, receives results |
| Web Researcher | Search web, fetch pages, extract content | Claude Haiku 3.5 + httpx | Receives search query, returns structured findings with URLs |
| Code Analyst | Read repos, analyze code, extract patterns | Claude Sonnet 4 | Receives repo URL/file path, returns analysis |
| Fact-Checker | Cross-reference claims, flag contradictions | Claude Haiku 3.5 | Receives claims, returns verification status per claim |
| Synthesis | Compile findings into structured report | Claude Sonnet 4 | Receives all findings, returns report |
| Critic | Review for gaps, bias, unsupported claims | Claude Sonnet 4 | Receives report, returns feedback with revision requests |

## Technology Decisions

| Component | Choice | Why this, not X |
|---|---|---|
| Orchestration | LangGraph | Native multi-agent supervision. Manual chain-of-thought doesn't handle branching |
| Web Search | DuckDuckGo (free) + httpx fetching | No API key needed. Google Search API costs $5/1K queries |
| LLM — Orchestrator | Claude Sonnet 4 | Decomposition and iteration need deep reasoning. Haiku fragments research questions |
| LLM — Researchers | Claude Haiku 3.5 | High volume of parallel calls. Sonnet would 5x cost for marginal accuracy gain |
| Persistence | SQLite + JSON | Simple, zero-config. PostgreSQL when scaling to multi-user |
| Frontend | React + Tailwind | Real-time agent progress display. Static reports would miss the live aspect |
| Streaming | Server-Sent Events | Agent progress pushes to UI. WebSocket is overkill for one-directional updates |

## Data Architecture

```python
from pydantic import BaseModel
from typing import List, Optional, Literal
from datetime import datetime

class ResearchSession(BaseModel):
    id: str
    query: str
    status: Literal["in_progress", "completed", "failed"]
    sub_questions: List[str] = []
    created_at: datetime = datetime.utcnow()
    completed_at: Optional[datetime] = None

class Finding(BaseModel):
    id: str
    session_id: str
    agent: str  # "web_researcher", "code_analyst", "fact_checker"
    sub_question: str
    content: str
    sources: List[str] = []
    confidence: float = 0.0
    created_at: datetime = datetime.utcnow()

class Report(BaseModel):
    id: str
    session_id: str
    title: str
    executive_summary: str
    sections: List[dict] = []
    findings: List[str] = []
    contradictions: List[str] = []
    confidence: float = 0.0
    iteration: int = 0
    created_at: datetime = datetime.utcnow()

class CriticFeedback(BaseModel):
    report_id: str
    gaps: List[str] = []
    unsupported_claims: List[str] = []
    contradictions: List[str] = []
    improvement_suggestions: List[str] = []
    score: float = 0.0
    requires_revision: bool = False
```

## Security Architecture

- **API keys** stored in environment variables, never in code
- **User sessions** isolated — one research session per user
- **Rate limiting** — 10 research queries per hour per user (cost control)
- **Output sanitization** — HTML-escaped in reports, no raw markdown execution
- **Web search** — results filtered through content extraction, no direct JS execution
