A 2024 survey by Arize AI found that 60% of organizations deploying LLMs have no observability for their AI systems. When a model degrades, cost spikes, or safety violations occur, teams have no way to trace the issue back to the specific prompt, model version, or input that caused it. In regulated industries, this lack of observability is a compliance risk.

## What You'll Build

You'll build **ObserveAI**, an observability platform with:

- **LLM call instrumentation** — drop-in middleware that captures every API call
- **Token and cost tracking** — per-call, per-user, per-model breakdown
- **Latency monitoring** — histograms, p50/p95/p99, and anomaly detection
- **Error tracking** — rate limits, timeouts, invalid responses, with stack traces
- **Safety monitoring** — flag content policy violations, prompt injections, PII leaks
- **Dashboards** — real-time web dashboard with configurable time ranges
- **Alerts** — webhook/email alerts for cost spikes, error rate thresholds
- **Traces** — full request/response logging with searchable history

## Technology Decisions

| Component | Choice | Why this, not X |
|---|---|---|
| Instrumentation | Python decorator + FastAPI middleware | Zero-code changes for existing apps. Proxy-based instrumentation is more powerful but complex |
| Storage | SQLite (time-partitioned) | Simple, queryable. TimescaleDB needed at 10K+ calls/day |
| Cost Tracking | Token counting + provider rate cards | Real-time, no external billing API needed |
| Anomaly Detection | Statistical (z-score) | No training data needed. ML-based detection is more accurate but requires labeled history |

## Data Model

```python
from pydantic import BaseModel
from typing import Optional, Literal
from datetime import datetime

class LLMCall(BaseModel):
    id: str
    timestamp: datetime
    model: str
    provider: str
    prompt_tokens: int
    completion_tokens: int
    total_tokens: int
    latency_ms: int
    cost: float
    error: Optional[str] = None
    user_id: Optional[str] = None
    tags: dict = {}
    prompt_preview: str
    response_preview: str

class SafetyEvent(BaseModel):
    id: str
    call_id: str
    event_type: Literal["prompt_injection", "pii_leak", "content_violation", "unsafe_output"]
    severity: Literal["low", "medium", "high", "critical"]
    detail: str
    timestamp: datetime

class Alert(BaseModel):
    id: str
    metric: str
    threshold: float
    current_value: float
    severity: str
    message: str
    created_at: datetime
```
