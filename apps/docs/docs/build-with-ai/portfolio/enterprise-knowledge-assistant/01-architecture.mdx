Enterprise knowledge is scattered across wikis, Notion docs, GitHub repositories, Slack messages, Google Docs, and internal tools. Employees spend an estimated 2.5 hours per day searching for information. A centralized knowledge assistant that indexes all sources, enforces access control, and provides a single chat interface can save 5+ hours per employee per week.

## What You'll Build

You'll build **KnowBot**, an enterprise knowledge assistant with:

- **Multi-source indexing** — Notion, GitHub repos, local Markdown, Confluence (API stubs)
- **Multi-tenant access control** — teams see only their authorized documents
- **Hybrid search** — vector + keyword with tenant-scoped filtering
- **Slack integration** — ask questions directly from Slack
- **Audit logging** — every query logged with user, tenant, timestamp
- **Caching** — frequent queries cached to reduce cost and latency
- **Admin dashboard** — usage stats, popular queries, top users

## Technology Decisions

| Component | Choice | Why this, not X |
|---|---|---|
| Vector Store | ChromaDB (tenant-collection pattern) | Each tenant gets their own collection for natural isolation |
| Multi-tenancy | Collection-per-tenant | Tenant A's data never mixes with Tenant B's. Row-level filtering is complex at scale |
| Slack Integration | Slack Bolt SDK | Official SDK, handles events. Raw WebSocket is more work |
| Caching | SQLite + TTL | Simple, fast. Redis would be better but adds infrastructure |

## Data Model

```python
from pydantic import BaseModel
from typing import Optional, Literal
from datetime import datetime

class Tenant(BaseModel):
    id: str
    name: str
    created_at: datetime

class KnowledgeSource(BaseModel):
    id: str
    tenant_id: str
    source_type: Literal["notion", "github", "markdown", "confluence"]
    config: dict
    last_indexed: Optional[datetime] = None
    document_count: int = 0

class QueryLog(BaseModel):
    id: str
    tenant_id: str
    user_id: str
    query: str
    response_preview: str
    tokens_used: int
    latency_ms: int
    timestamp: datetime
```
