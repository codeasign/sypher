You have a big interview coming up and you need to practice. You ask a friend to run a mock interview, but they go easy on you — nodding along, saying "that sounds good" — and you finish with no idea whether your answer was actually strong. You try practicing alone, staring at a list of questions from the internet, but without feedback you are just reciting into the void. The problem is not a lack of preparation material. It is a lack of a realistic, honest practice partner who will tell you when your answer falls flat.

Interviewing is a skill that needs deliberate practice. You need to hear real questions, formulate answers under pressure, and get specific feedback on what worked and what did not. You need to know if you are rambling, if your examples lack structure, and if your answers match what hiring managers actually look for. You need a coach that never gets tired, never holds back, and is always available at 2 AM when the nerves hit. That coach is Claude Haiku.

### What You Will Build

A Python CLI tool that simulates a job interview for a specific role using Claude Haiku. It asks behavioral and technical questions, evaluates your answers, gives constructive feedback, and tracks your improvement across multiple practice sessions.

- Selects a target role: Software Engineer, Product Manager, Data Scientist, or Engineering Manager
- Generates role-specific interview questions dynamically via Claude Haiku
- Asks one question at a time and waits for your typed answer
- Evaluates each answer on clarity, relevance, structure, and impact (0-100 scale)
- Provides specific strengths, areas for improvement, and a sample strong answer
- Saves scores per session to a local JSON file
- Displays a progress table showing your average score over time
- Costs under a nickel per full session

### Concept in 5 Minutes

Think of an LLM as a patient interview coach who has conducted thousands of mock interviews across every role at every company. This coach never gets tired, never judges you, and always gives honest feedback tailored to the role you are targeting. You tell it which role to practice, and it generates realistic questions, listens to your answers, and evaluates them against the same criteria a real hiring manager would use.

Create `interview_coach.py` in `~/interview-coach/`. This version loads environment variables, defines available roles, and presents a selection menu.

#### Step 4: Build the Interviewer — Question Generation

Add the Anthropic SDK import, initialize the client, and add a function that generates role-specific questions via Claude Haiku:

```python
from anthropic import Anthropic

client = Anthropic(api_key=api_key)


def generate_questions(role: dict, count: int = 6) -> list[str]:
    """Generate interview questions using Claude Haiku."""
    response = client.messages.create(
        model="claude-3-5-haiku-latest",
        max_tokens=MAX_TOKENS_PER_RUN,
        system=(
            f"You are a hiring manager. Generate {count} questions for a "
            f"{role['name']} position — 3 behavioral and 3 {role['style']}. "
            f"Output as a JSON array of strings."
        ),
        messages=[
            {"role": "user", "content": f"Generate {count} questions for {role['name']}."}
        ],
    )
    text = response.content[0].text.strip()
    if "[" in text:
        text = text[text.index("["):text.rindex("]") + 1]
    return json.loads(text)[:count]
```

The system prompt tells Claude to act as a hiring manager and produce a JSON array of questions. The response parsing extracts the array from whatever surrounding text Claude adds. To test, call `generate_questions` from `main()` and print the result.

#### Step 5: Build the Answer Collector and Evaluator

Add two functions — one to collect multi-line answers terminated by a double Enter, and one to send each answer to Claude for structured evaluation:

```python
def ask_question(question: str, num: int, total: int) -> str:
    """Display a question and collect the user's typed answer."""
    print(f"\n{'=' * 60}")
    print(f"Question {num} of {total}")
    print(f"{'=' * 60}\n{question}\n")
    print("Type your answer. Press Enter twice to finish:\n")
    lines = []
    while True:
        line = input()
        if line == "" and (not lines or lines[-1] == ""):
            if lines:
                lines.pop()
            break
        lines.append(line)
    return "\n".join(lines)


def evaluate_answer(question: str, answer: str, role: dict) -> dict:
    """Evaluate the user's answer and return a score with feedback."""
    response = client.messages.create(
        model="claude-3-5-haiku-latest",
        max_tokens=MAX_TOKENS_PER_RUN,
        system=(
            "You are an expert interview coach. Return a JSON object with: "
            "score (0-100), star_used (bool), clarity (0-25), "
            "relevance (0-25), structure (0-25), impact (0-25), "
            "strengths (list), improvements (list), sample_answer (string)."
        ),
        messages=[{
            "role": "user",
            "content": f"Role: {role['name']}\nQuestion: {question}\n\nAnswer: {answer}\n\nEvaluate."
        }],
    )
    text = response.content[0].text.strip()
    if "{" in text:
        text = text[text.index("{"):text.rindex("}") + 1]
    return json.loads(text)


def display_feedback(ev: dict, num: int) -> None:
    """Print evaluation feedback to the terminal."""
    print(f"\n{'=' * 60}")
    print(f"Feedback for Question {num}")
    print(f"{'=' * 60}")
    score = ev["score"]
    bar = "█" * int(score / 100 * 50) + "░" * (50 - int(score / 100 * 50))
    print(f"\nOverall Score: {score}/100  [{bar}]")
    print(f"  Clarity: {ev['clarity']}/25 | Relevance: {ev['relevance']}/25")
    print(f"  Structure: {ev['structure']}/25 | Impact: {ev['impact']}/25")
    if ev.get("star_used"):
        print("  STAR Method: Used")
    print(f"\nStrengths:")
    for s in ev["strengths"]:
        print(f"  + {s}")
    print(f"\nAreas to Improve:")
    for imp in ev["improvements"]:
        print(f"  - {imp}")
    print(f"\nSample Strong Answer:\n  {ev['sample_answer']}")
    input("\nPress Enter to continue...")
```

The evaluation prompt asks Claude to score the answer on four dimensions (each 25 points), check for STAR method usage, list strengths and improvements, and provide a sample strong answer. The visual score bar gives you an instant sense of how you performed.

#### Step 6: Add Session Logging and Progress Tracking

Add persistence — `save_session` writes scores to `~/interview-coach/session_log.json`, and `show_progress` reads it back to display trends:

```python
from datetime import datetime
from pathlib import Path

LOG_PATH = Path.home() / "interview-coach" / "session_log.json"


def save_session(role_name: str, scores: list[int]) -> list[dict]:
    """Save session results to the JSON log file."""
    LOG_PATH.parent.mkdir(parents=True, exist_ok=True)
    session = {
        "date": datetime.now().isoformat(),
        "role": role_name,
        "scores": scores,
        "average": round(sum(scores) / len(scores), 1) if scores else 0,
    }
    history = []
    if LOG_PATH.exists():
        with open(LOG_PATH, "r") as f:
            history = json.load(f)
    history.append(session)
    with open(LOG_PATH, "w") as f:
        json.dump(history, f, indent=2)
    return history


def show_progress(history: list[dict]) -> None:
    """Display a progress table of recent sessions."""
    print(f"\n{'=' * 60}")
    print("Your Progress Over Time\n")
    print(f"{'Session':<10} {'Role':<20} {'Avg Score':<12} {'Date':<12}")
    print("-" * 60)
    for i, s in enumerate(history[-10:], max(1, len(history) - 9)):
        print(f"{i:<10} {s['role']:<20} {s['average']:<12.1f} {s['date'][:10]:<12}")
    if len(history) >= 2:
        lf = [s["average"] for s in history[-5:]]
        print(f"\nTrend (last 5): {'improving' if lf[-1] > lf[0] else 'needs work'}")
    print()
```

#### Step 7: Wire It All Together — Complete Application

Assemble all the pieces. The final `interview_coach.py` combines the role selector, question generator, answer collector, evaluator, and session logger. The `main()` function orchestrates the full flow:

```python
def main() -> None:
    role = select_role()
    print(f"\nPreparing questions for {role['name']}...")
    questions = generate_questions(role)
    total = len(questions)
    print(f"Starting interview with {total} questions.\n")

    scores = []
    total_tokens = 0
    for i, q in enumerate(questions, 1):
        if total_tokens > MAX_TOKENS_PER_RUN:
            print(f"Token budget reached. Increase MAX_TOKENS_PER_RUN in .env.")
            break
        answer = ask_question(q, i, total)
        print("\nEvaluating your answer...")
        ev = evaluate_answer(q, answer, role)
        total_tokens += 500
        display_feedback(ev, i)
        scores.append(ev["score"])

    print(f"\n{'=' * 60}")
    print("Session Complete!")
    print(f"{'=' * 60}")
    if scores:
        print(f"\nAverage: {sum(scores)/len(scores):.1f}/100")
        print(f"Best:    {max(scores):.1f}/100")
        print(f"Focus:   {min(scores):.1f}/100")

    history = save_session(role["name"], scores)
    show_progress(history)
    print("Good luck with your interviews!")


if __name__ == "__main__":
    main()
```

### Make It Yours

Three customizations to tailor the coach to your needs:

**1. Add custom questions.** Prepend a `CUSTOM_QUESTIONS` list to the generated questions so you always practice your weakest areas:

```python
CUSTOM_QUESTIONS = ["Describe balancing technical debt with feature delivery."]
# Prepend: return CUSTOM_QUESTIONS + generated[:count - len(CUSTOM_QUESTIONS)]
```

**2. Add difficulty levels.** Prompt for a level (junior, mid, senior, staff) and pass it to the system prompt. Junior questions focus on fundamentals; staff questions probe architecture and strategic thinking.

**3. Add company-specific styles.** Different companies have different interview cultures. Add a company selector:

```python
STYLES = {
    "amazon": "Use Amazon Leadership Principles. Expect STAR-format answers.",
    "google": "Focus on analytical problem-solving and data-driven decisions.",
    "startup": "Emphasize versatility, speed, and comfort with ambiguity.",
}
```

The evaluator then checks for company-specific signals.
