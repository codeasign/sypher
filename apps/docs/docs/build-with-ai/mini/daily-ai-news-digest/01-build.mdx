You check your Twitter feed. Then Hacker News. Then the Anthropic blog. Then Hugging Face daily papers. Then five newsletters you subscribed to and never read. Forty-five minutes later you have a vague sense that some company released something, a new paper claims to beat GPT on some benchmark you do not fully understand, and someone on Reddit is very upset about it. You still do not know what actually matters.

This is the AI news firehose problem. The field moves so fast that keeping up feels like a full-time job — except it is not your job. You are an engineer, a student, or a product builder who needs to stay informed but cannot afford to spend an hour each morning curating feeds and filtering noise. The FOMO is real, but the time cost of feeding it is worse. What you actually need is a single, concise summary of the day's most relevant developments, delivered to your inbox or saved to a file, written by someone (or something) that has read everything so you do not have to.

### What You'll Build

A Python script that automates your morning AI news reading. It fetches articles from three AI-focused RSS feeds, deduplicates stories that appear across multiple sources, summarizes each article using Claude Haiku, ranks them by relevance, and outputs a clean Markdown digest file. Specifically, it will:

- Fetch articles from Hugging Face Daily Papers, Google AI Blog, and the Anthropic Blog RSS feeds
- Deduplicate articles that share similar URLs or titles (no reading the same announcement twice)
- Summarize each article in one paragraph using `claude-haiku-3-5`
- Rank all articles by estimated relevance to an AI engineer
- Write a formatted `digest.md` file you can open in any Markdown viewer

### Concept in 5 Minutes

The mental model is simple: think of the LLM as your personal news curator. You do not have time to read one hundred headlines and decide which ten matter. The LLM has all the time in the world. Your job is to connect the LLM to the raw feed, tell it what you care about, and let it do the filtering and summarization. The script acts as a pipeline with four stages: collect, deduplicate, summarize, rank. Each stage is a Python function. The RSS fetcher is the only stage that touches the network. The deduplication stage compares titles and URLs. The summarization stage calls Claude once per unique article. The ranking stage calls Claude once more with all summaries and asks it to order them by relevance. The result is a digest file you can open in two seconds.

#### Step 3: Create the RSS feed fetcher

Create `digest.py` with the feed-fetching logic:

```python
import os
import json
import hashlib
from datetime import datetime
from pathlib import Path
from typing import List, Dict
import feedparser
from anthropic import Anthropic

# ---------------------------------------------------------------
# Configuration
# ---------------------------------------------------------------
RSS_FEEDS = [
    "https://huggingface.co/api/daily_papers?format=atom",
    "https://blog.google/technology/ai/rss/",
    "https://www.anthropic.com/news/rss.xml",
]

OUTPUT_DIR = Path.home() / "ai-digest"
MAX_ARTICLES = 20
MAX_SUMMARY_TOKENS = 300
MODEL_NAME = "claude-haiku-3-5-20241022"

# Spend guard rails
MAX_ITERATIONS = 50
MAX_TOKENS_PER_RUN = 50000


def fetch_feeds(feeds: List[str]) -> List[Dict]:
    """Fetch all articles from a list of RSS feeds."""
    articles = []
    for url in feeds:
        try:
            feed = feedparser.parse(url)
            for entry in feed.entries[:MAX_ARTICLES]:
                articles.append({
                    "title": entry.get("title", "Untitled"),
                    "url": entry.get("link", ""),
                    "summary": entry.get("summary", ""),
                    "published": entry.get("published", ""),
                    "source": url,
                })
            print(f"  Fetched {len(feed.entries[:MAX_ARTICLES])} articles from {url}")
        except Exception as e:
            print(f"  Error fetching {url}: {e}")
    return articles


if __name__ == "__main__":
    print("Fetching RSS feeds...")
    all_articles = fetch_feeds(RSS_FEEDS)
    print(f"Total articles fetched: {len(all_articles)}")
```

#### Step 4: Add deduplication

Add the deduplication function after `fetch_feeds`:

```python
def normalize_url(url: str) -> str:
    """Strip trailing slashes and query params for comparison."""
    return url.rstrip("/").split("?")[0].split("#")[0]


def title_similarity(t1: str, t2: str) -> float:
    """Simple word-overlap similarity for deduplication."""
    words1 = set(t1.lower().split())
    words2 = set(t2.lower().split())
    if not words1 or not words2:
        return 0.0
    intersection = words1 & words2
    union = words1 | words2
    return len(intersection) / len(union)


def deduplicate(articles: List[Dict], threshold: float = 0.6) -> List[Dict]:
    """Remove duplicates by URL and by title similarity."""
    seen_urls = set()
    seen_titles = []
    unique = []
    for article in articles:
        norm_url = normalize_url(article["url"])
        if norm_url in seen_urls:
            continue
        is_duplicate_title = any(
            title_similarity(article["title"], t) > threshold
            for t in seen_titles
        )
        if is_duplicate_title:
            continue
        seen_urls.add(norm_url)
        seen_titles.append(article["title"])
        unique.append(article)
    return unique
```

Then update the `if __name__` block:

A value of 0.6 catches duplicate coverage without being so aggressive that it merges different stories about the same company.

#### Step 5: Add the summarizer and ranker

Add the summarization function. It calls Claude Haiku for each article, then calls Claude once more to rank everything:

```python
anthropic_client = Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])


def summarize_article(article: Dict) -> str:
    """Summarize a single article using Claude Haiku."""
    prompt = f"""You are an AI news curator. Summarize the following article in one paragraph (under {MAX_SUMMARY_TOKENS} tokens).
Include what happened, why it matters for AI engineers, and who should care.

Title: {article["title"]}
Content: {article["summary"]}
"""
    response = anthropic_client.messages.create(
        model=MODEL_NAME,
        max_tokens=MAX_SUMMARY_TOKENS,
        temperature=0.3,
        messages=[{"role": "user", "content": prompt}],
    )
    return response.content[0].text


def rank_articles(
    summaries: List[Dict], top_n: int = 10
) -> List[Dict]:
    """Rank summarized articles by relevance to an AI engineer."""
    summary_texts = "\n\n".join(
        f"Article {i+1}: {s['title']}\nSummary: {s['summary']}"
        for i, s in enumerate(summaries)
    )
    prompt = f"""You are an AI news ranking system. Below are summaries of today's AI news.
Rank them by relevance to a working AI engineer (not a researcher). Consider:
- Practical applicability (can they use it today?)
- Industry impact (does it change how AI is built?)
- Novelty (is this genuinely new or just incremental?)
- Actionability (does it require them to change anything?)

Return your ranking as a JSON list of article numbers in order, most relevant first.
Example: {{"ranked": [3, 1, 5, 2, 4]}}

Summaries:
{summary_texts}
"""
    response = anthropic_client.messages.create(
        model=MODEL_NAME,
        max_tokens=1000,
        temperature=0.0,
        messages=[{"role": "user", "content": prompt}],
    )
    try:
        ranked = json.loads(response.content[0].text)
        ranked_indices = ranked["ranked"]
    except (json.JSONDecodeError, KeyError):
        print("Warning: Could not parse ranking, using original order")
        ranked_indices = list(range(1, len(summaries) + 1))

    ranked_articles = []
    for idx in ranked_indices:
        if 1 <= idx <= len(summaries):
            ranked_articles.append(summaries[idx - 1])
    return ranked_articles[:top_n]
```

**Explanation:** `summarize_article` sends a structured prompt that includes the article title and summary content. The temperature of 0.3 gives consistent but not rigid output. The `rank_articles` function sends all summaries and asks for a single JSON response. The model returns a ranked list of article numbers, which the function uses to reorder the articles. Adding a spend guard — if too many articles arrive, the function stops early:

```python
def summarize_all_safe(articles: List[Dict]) -> List[Dict]:
    """Summarize with iteration and token spend guards."""
    summaries = []
    total_tokens_used = 0
    for i, article in enumerate(articles):
        if i >= MAX_ITERATIONS:
            print(f"  Reached max iterations ({MAX_ITERATIONS}), stopping summarization")
            break
        summary = summarize_article(article)
        token_estimate = len(summary.split()) * 1.5  # rough token estimate
        total_tokens_used += token_estimate
        if total_tokens_used > MAX_TOKENS_PER_RUN:
            print(f"  Reached max tokens per run ({MAX_TOKENS_PER_RUN}), stopping")
            break
        summaries.append({
            "title": article["title"],
            "url": article["url"],
            "summary": summary,
            "source": article["source"],
        })
        print(f"  Summarized {i+1}/{len(articles)}: {article['title'][:50]}...")
    return summaries
```

#### Step 6: Assemble the full pipeline

Replace the entire `if __name__` block with the complete pipeline:

```python
def write_digest(summaries: List[Dict], output_path: Path):
    """Write ranked summaries to a markdown digest file."""
    date_str = datetime.now().strftime("%A, %B %d, %Y")
    lines = [
        f"# AI News Digest — {date_str}",
        "",
        f"_{len(summaries)} curated articles from your AI feeds._",
        "",
        "---",
        "",
    ]
    for i, article in enumerate(summaries, 1):
        lines.extend([
            f"## {i}. {article['title']}",
            "",
            f"[{article['url']}]({article['url']})",
            "",
            article["summary"],
            "",
            "---",
            "",
        ])
    output_path.write_text("\n".join(lines), encoding="utf-8")
    print(f"Digest written to {output_path}")


if __name__ == "__main__":
    print("=" * 50)
    print("  AI News Digest Generator")
    print("=" * 50)

    # Step 1: Fetch
    print("\n[1/4] Fetching RSS feeds...")
    all_articles = fetch_feeds(RSS_FEEDS)
    print(f"  Total articles fetched: {len(all_articles)}")

    # Step 2: Deduplicate
    print("\n[2/4] Deduplicating...")
    unique_articles = deduplicate(all_articles)
    print(f"  Unique articles: {len(unique_articles)} (removed {len(all_articles) - len(unique_articles)})")

    # Step 3: Summarize
    print("\n[3/4] Summarizing articles with Claude Haiku...")
    api_key = os.environ.get("ANTHROPIC_API_KEY")
    if not api_key:
        print("  ERROR: ANTHROPIC_API_KEY not set. Create a .env file with your key.")
        exit(1)
    summarized = summarize_all_safe(unique_articles)
    print(f"  Summarized {len(summarized)} articles")

    # Step 4: Rank and write
    print("\n[4/4] Ranking and writing digest...")
    OUTPUT_DIR.mkdir(parents=True, exist_ok=True)
    ranked = rank_articles(summarized, top_n=10)
    digest_path = OUTPUT_DIR / "digest.md"
    write_digest(ranked, digest_path)

    print("\n" + "=" * 50)
    print("  Done! Open your digest:")
    print(f"  {digest_path}")
    print("=" * 50)
```

Three ways to customize the digest for your own workflow.

**2. Change to weekly digest mode**

Modify the script to aggregate across seven days instead of one. Add a `DAYS_BACK` configuration variable and filter articles by comparing the `published` field:

```python
from datetime import datetime, timedelta
import dateutil.parser

DAYS_BACK = 7

def filter_by_date(articles: List[Dict]) -> List[Dict]:
    cutoff = datetime.now() - timedelta(days=DAYS_BACK)
    recent = []
    for article in articles:
        try:
            pub_date = dateutil.parser.parse(article["published"])
            if pub_date >= cutoff:
                recent.append(article)
        except (ValueError, TypeError):
            recent.append(article)
    return recent
```

To use `dateutil.parser`, install `python-dateutil`:

```bash
pip install python-dateutil
```

**3. Add Slack or Discord webhook output**

Replace or supplement `write_digest` with a webhook POST. For Slack:

```python
import requests

SLACK_WEBHOOK_URL = os.environ.get("SLACK_WEBHOOK_URL", "")

def post_to_slack(summaries: List[Dict]):
    if not SLACK_WEBHOOK_URL:
        print("  No SLACK_WEBHOOK_URL set, skipping Slack post")
        return
    blocks = [{"type": "header", "text": {"type": "plain_text", "text": "AI News Digest"}}]
    for article in summaries:
        blocks.append({
            "type": "section",
            "text": {
                "type": "mrkdwn",
                "text": f"*{article['title']}*\n{article['summary'][:200]}...",
            },
        })
    payload = {"blocks": blocks}
    response = requests.post(SLACK_WEBHOOK_URL, json=payload)
    response.raise_for_status()
    print("  Posted to Slack")
```

Add the Slack webhook URL to your `.env` file and call `post_to_slack(ranked)` at the end of the main pipeline.
