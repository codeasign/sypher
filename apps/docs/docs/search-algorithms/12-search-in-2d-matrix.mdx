## Summary

Search in 2D Matrix finds a target number inside a grid where every row goes up left-to-right and every column goes up top-to-bottom. The trick: start at the top-right corner. If the number there is too big, step left. If it's too small, step down. Each step throws away a whole row or a whole column at once, so you never have to check most of the grid. It's not lightning-fast like plain binary search, but it's much quicker than checking every single cell.

## Pseudocode

```text
1. If the grid is empty, report "not found."
2. Start at the top-right corner of the grid.
3. Look at the number in the current cell.
4. If it matches the target, stop — you found it. Report its row and column.
5. If it's bigger than the target, move one step to the left.
6. If it's smaller than the target, move one step down.
7. If you move past the left edge or the bottom edge, report "not found."
```

## Worked Example

This only works if every row increases left-to-right and every column increases top-to-bottom — the whole grid is sorted in both directions at once, like a dictionary sorted by two different rules. We'll trace the search for `5` in the same 5x5 grid at every step, starting from the top-right corner.

## Complexity

- **Best case:** the top-right starting cell already holds the target, so it's found instantly (`O(1)`).
- **Average case:** each step throws away either a whole row or a whole column, so the pointer typically travels a distance proportional to the number of rows plus the number of columns (`O(rows + columns)`).
- **Worst case:** the target is missing, or it sits at the far end of the staircase path, so the pointer walks almost the full height plus the full width before giving up (`O(rows + columns)`).
- **Space:** it only tracks two numbers, the current row and the current column, no extra memory that grows with the grid (`O(1)`).

## Applications

- Spreadsheet and pivot-table lookups where values are sorted both across columns and down rows, such as a price table sorted by size across the top and by date down the side.
- Sorted database result grids, where a query planner can walk this same staircase pattern to avoid scanning every row when both dimensions are already ordered.
- Finding row minima in totally monotone ("Monge") matrices, the same top-right staircase idea used by the SMAWK algorithm to speed up certain dynamic-programming problems.
- Leaderboard and ranking grids sorted by two keys at once, like scores sorted by round across columns and by player down rows.
- Version and release-lookup tables sorted by two keys, such as major version down the rows and minor version across the columns.
