## Summary

Depth-First Search is a way to explore a graph or tree by picking one path and following it as far as it goes before backtracking to try another. Think of exploring a maze by always taking the next unexplored turn and only backing up once you hit a dead end — that's DFS. It works on connected networks of any shape, and it keeps a checklist of places already visited so it never loops forever, and it gets slower the more places and connections there are to explore.

## Pseudocode

```text
1. If the starting point isn't in the graph, report an empty result.
2. Keep a checklist of visited places, and a "to-do" pile (a stack) starting with just the starting point.
3. While the to-do pile isn't empty, take the most recently added place off the pile.
4. If it's already on the checklist, skip it and go back to step 3.
5. Otherwise, mark it visited and add it to the result list.
6. Add each of its neighbors that aren't already checked off onto the to-do pile.
7. Repeat until the to-do pile is empty, then report the result list.
```

## How It Works

- In the diagram, we start at `A`. We mark it visited and look at its neighbors, `B` and `C`.
- We dive into `B` first (going deep, not wide), mark it visited, and check its neighbor `D`.
- We dive into `D`, mark it visited. `D`'s other neighbor is `C`, but we haven't gotten there yet.
- With `B`'s branch fully explored, we backtrack and visit `C`. `C` also points to `D`, but `D` is already on our checklist, so we skip it instead of visiting it a second time.
- Final visit order: `A, B, D, C`.
- This works on a graph that's shaped like a tree, a chain, or even one full of loops — as long as we keep the visited checklist, we can never get stuck going around in circles.
- How fast is it? Since every place and every connection between places needs to be looked at exactly once, DFS is about equally fast whether you're lucky or unlucky — there's no real "best case" shortcut. It just scales with how big the graph is (this is called `O(V + E)` time, where `V` is the number of places/nodes and `E` is the number of connections/edges).
