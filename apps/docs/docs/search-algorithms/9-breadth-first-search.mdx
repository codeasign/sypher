def breadth_first_search(graph: dict[str, list[str]], start: str) -> list[str]:

    if start not in graph:

## Summary

Breadth-First Search explores a graph or tree one whole "ring" at a time, visiting everything close by before moving further out. Imagine dropping a pebble in a pond and watching the ripples spread outward evenly in all directions вҖ” that's BFS: it visits all of a starting point's direct neighbors first, then all of their neighbors, and so on. Because it spreads out evenly, it's the algorithm to use when you want the shortest route measured in number of steps. It gets slower the more places and connections there are, but it never wastes time backtracking.

## Pseudocode

```text
1. If the starting point isn't in the graph, report an empty result.
2. Keep a checklist of visited places, and a waiting line (a queue) starting with just the starting point. Mark the starting point visited right away.
3. While the waiting line isn't empty, take the place that's been waiting longest off the front of the line.
4. Add it to the result list.
5. Look at each of its neighbors. For any neighbor not already checked off, mark it visited and add it to the back of the waiting line.
6. Repeat until the waiting line is empty, then report the result list.
```

## Complexity

- **Best case** вҖ” Even if the target you're looking for happens to be right next to the start, BFS still has to enqueue and check every node and connection to know it's found everything at that distance, so there's no faster shortcut (`O(V + E)`).
- **Average case** вҖ” Same story: every place (`V`) and every connection (`E`) gets looked at once, so the work scales with the size of the whole graph (`O(V + E)`).
- **Worst case** вҖ” Even when the target is the very last node discovered, or doesn't exist at all, BFS still only ever visits each place and each connection once, so it stays at (`O(V + E)`).
- **Space** вҖ” BFS can end up holding an entire "ring" of places in its waiting line at once. For a wide graph, that ring can be almost as big as the whole graph, so memory use scales with the number of places (`O(V)`).

## Applications

- **Shortest path in unweighted graphs** вҖ” because BFS finishes an entire ring before starting the next one, the first time it reaches any node is guaranteed to be by the fewest possible hops.
- **"Degrees of separation" on social networks** вҖ” finding how many mutual-friend hops separate two people (the idea behind LinkedIn's "2nd/3rd degree connection" labels) is BFS from one person outward.
- **Web crawlers** вҖ” search engines like Google's crawler visit a starting page, then all pages it links to, then all of those, spreading outward ring by ring to index the web.
- **Network broadcast and routing protocols** вҖ” protocols that flood a message outward to every reachable device one hop at a time (like OSPF's link-state flooding) follow the same ring-by-ring spread as BFS.
- **Puzzle solvers that need the fewest moves** вҖ” solving a sliding puzzle or a Rubik's Cube in the minimum number of moves is BFS over every possible board state, since it guarantees the first solution found used the fewest steps.
