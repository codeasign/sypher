## Summary

A trie (say "try") is a tree built for spelling things out one letter at a time — every step down the tree spells out one more letter of a word, and words that start the same way share the same starting path. Trie Search checks whether a word (or just the beginning of a word) exists by walking down the tree one letter at a time, following the matching letter at each step. It stays fast no matter how many words are stored, because checking a word only takes as many steps as the word has letters — it never has to look at any of the other stored words.

## Pseudocode

```text
Adding a word to the trie:
1. Start at the top (the root).
2. For each letter in the word, in order: if there's no path for that letter yet, create one. Then move down that path.
3. After the last letter, mark that spot as "a complete word ends here."

Checking if a word was fully stored (search):
1. Start at the top.
2. For each letter in the word, in order: if there's no path for that letter, the word was never stored — stop and report "no."
3. If you successfully follow a path for every letter, check whether that final spot is marked "a complete word ends here." Report yes or no accordingly.

Checking if anything starts with a given prefix (startsWith):
1. Start at the top.
2. For each letter in the prefix, in order: if there's no path for that letter, report "no."
3. If you successfully follow a path for every letter, report "yes" — it doesn't matter whether that spot is marked as a complete word or not.
```

**Recursive Version**

```text
Recursive search(node, word, index):
1. If index has reached the end of the word, report whether the current node is marked "a complete word ends here."
2. Otherwise, look at the letter at this index. If there's no path from the current node for that letter, report "no."
3. Otherwise, move to the child reached by that letter and call search(child, word, index + 1). Whatever that call reports, report the same thing.
4. Kick the whole thing off with search(root, word, 0).

startsWith works the same way, except step 1 always reports "yes" once index reaches the end of the prefix — it never checks the "complete word" mark.
```

All three share the letters `c` and `a`, so those two steps are a single shared path — the tree only splits once it reaches the third letter, where `t` and `r` disagree. We'll search for `card`, reading one letter at a time and following the matching path.

That final node is marked `*`, meaning a complete word really does end there, so `search("card")` reports "yes." If that last node had no `*` mark, the search would still have to report "no," even though every letter matched — reaching a node only proves a word *starts* that way, it takes the `*` mark to prove a word *ends* there.

## Complexity

- **Best case:** if the very first letter of the word isn't in the trie at all, the search stops on the spot — instant (`O(1)`).
- **Average case:** the search walks down through however many letters happen to match before either hitting a missing path or running out of letters — still bounded by how long the word is (`O(L)`, where L is the length of the word being searched).
- **Worst case:** every letter of the word matches a path in the trie, so the search walks all the way down to the last letter — still only as many steps as the word has letters (`O(L)`).
- **Space:** every distinct starting sequence of letters shared by the stored words needs its own path in the tree, so in the worst case (no shared prefixes at all) storage grows with the number of words times how long they are (`O(N × L)`, where N is the number of stored words and L is their average length) — though shared prefixes usually make the real trie much smaller than that.

## Applications

- Autocomplete and search-suggestion boxes — search engines and IDEs walk a trie of previously seen terms to list every word that starts with what's been typed so far.
- Spell checkers and dictionary apps — checking whether a typed word exists in a large stored word list, letter by letter, without scanning every word.
- IP routing tables — routers store network address prefixes in a trie (often called a "prefix tree") to find the longest matching prefix for an incoming IP address.
- T9-style predictive text on phone keypads — matching a partial sequence of key presses against a dictionary trie to suggest whole words.
- Word games like Scrabble and Boggle solvers — checking whether a sequence of letters on the board is a valid word, or even a valid prefix worth continuing to explore, without rescanning the whole dictionary each time.

<CodeBlock language="c">{`#include <stdio.h>
#include <string.h>

#define MAX_TRIE_NODES 500
#define ALPHABET_SIZE 26

/*
 * Fixed-size array-based trie: trieNodes[i].children[c] stores the index of
 * the child reached via letter c ('a'-'z' mapped to 0-25), or -1 if absent.
 * trieNodeCount is a simple bump allocator; the root is always index 0.
 */
typedef struct {
    int children[ALPHABET_SIZE];
    int isEndOfWord;
} TrieNode;

TrieNode trieNodes[MAX_TRIE_NODES];
int trieNodeCount;

void trieInit(void) {
    trieNodeCount = 1;
    for (int i = 0; i < ALPHABET_SIZE; i++) {
        trieNodes[0].children[i] = -1;
    }
    trieNodes[0].isEndOfWord = 0;
}

void trieInsert(const char *word) {
    int current = 0;
    for (int i = 0; word[i] != '\\0'; i++) {
        int index = word[i] - 'a';
        if (trieNodes[current].children[index] == -1) {
            int newNode = trieNodeCount++;
            for (int j = 0; j < ALPHABET_SIZE; j++) {
                trieNodes[newNode].children[j] = -1;
            }
            trieNodes[newNode].isEndOfWord = 0;
            trieNodes[current].children[index] = newNode;
        }
        current = trieNodes[current].children[index];
    }
    trieNodes[current].isEndOfWord = 1;
}

static int trieFindNode(const char *s) {
    int current = 0;
    for (int i = 0; s[i] != '\\0'; i++) {
        int index = s[i] - 'a';
        if (trieNodes[current].children[index] == -1) {
            return -1;
        }
        current = trieNodes[current].children[index];
    }
    return current;
}

int trieSearch(const char *word) {
    int node = trieFindNode(word);
    return node != -1 && trieNodes[node].isEndOfWord;
}

int trieStartsWith(const char *prefix) {
    return trieFindNode(prefix) != -1;
}`}</CodeBlock>

    children: [Option<Box<TrieNode>>; 26],
    is_end_of_word: bool,
}

impl TrieNode {
    fn new() -> Self {
        TrieNode {
            children: std::array::from_fn(|_| None),
            is_end_of_word: false,
        }
    }
}

struct Trie {
    root: Box<TrieNode>,
}

impl Trie {
    fn new() -> Self {
        Trie { root: Box::new(TrieNode::new()) }
    }

    fn insert(&mut self, word: &str) {
        let mut node = &mut self.root;
        for ch in word.chars() {
            let index = (ch as u8 - b'a') as usize;
            if node.children[index].is_none() {
                node.children[index] = Some(Box::new(TrieNode::new()));
            }
            node = node.children[index].as_mut().unwrap();
        }
        node.is_end_of_word = true;
    }

    fn search(&self, word: &str) -> bool {
        match self.find_node(word) {
            Some(node) => node.is_end_of_word,
            None => false,
        }
    }

    fn starts_with(&self, prefix: &str) -> bool {
        self.find_node(prefix).is_some()
    }

    fn find_node(&self, s: &str) -> Option<&TrieNode> {
        let mut node = self.root.as_ref();
        for ch in s.chars() {
            let index = (ch as u8 - b'a') as usize;
            match &node.children[index] {
                Some(child) => node = child.as_ref(),
                None => return None,
            }
        }
        Some(node)
    }
