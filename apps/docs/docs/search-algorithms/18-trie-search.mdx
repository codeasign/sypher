#include <string.h>

#define MAX_TRIE_NODES 500
#define ALPHABET_SIZE 26

/*
 * Fixed-size array-based trie: trieNodes[i].children[c] stores the index of
 * the child reached via letter c ('a'-'z' mapped to 0-25), or -1 if absent.
 * trieNodeCount is a simple bump allocator; the root is always index 0.
 */
typedef struct {
    int children[ALPHABET_SIZE];
    int isEndOfWord;
} TrieNode;

TrieNode trieNodes[MAX_TRIE_NODES];
int trieNodeCount;

void trieInit(void) {
    trieNodeCount = 1;
    for (int i = 0; i < ALPHABET_SIZE; i++) {
        trieNodes[0].children[i] = -1;
    }
    trieNodes[0].isEndOfWord = 0;
}

void trieInsert(const char *word) {
    int current = 0;
    for (int i = 0; word[i] != '\\0'; i++) {
        int index = word[i] - 'a';
        if (trieNodes[current].children[index] == -1) {
            int newNode = trieNodeCount++;
            for (int j = 0; j < ALPHABET_SIZE; j++) {
                trieNodes[newNode].children[j] = -1;
            }
            trieNodes[newNode].isEndOfWord = 0;
            trieNodes[current].children[index] = newNode;
        }
        current = trieNodes[current].children[index];
    }
    trieNodes[current].isEndOfWord = 1;
}

static int trieFindNode(const char *s) {
    int current = 0;
    for (int i = 0; s[i] != '\\0'; i++) {
        int index = s[i] - 'a';
        if (trieNodes[current].children[index] == -1) {
            return -1;
        }
        current = trieNodes[current].children[index];
    }
    return current;
}

int trieSearch(const char *word) {
    int node = trieFindNode(word);
    return node != -1 && trieNodes[node].isEndOfWord;
}

int trieStartsWith(const char *prefix) {
    return trieFindNode(prefix) != -1;
}`}</CodeBlock>

## Summary

A trie (say "try") is a tree built for spelling things out one letter at a time — every step down the tree spells out one more letter of a word, and words that start the same way share the same starting path. Trie Search checks whether a word (or just the beginning of a word) exists by walking down the tree one letter at a time, following the matching letter at each step. It stays fast no matter how many words are stored, because checking a word only takes as many steps as the word has letters — it never has to look at any of the other stored words.

## Pseudocode

```text
Adding a word to the trie:
1. Start at the top (the root).
2. For each letter in the word, in order: if there's no path for that letter yet, create one. Then move down that path.
3. After the last letter, mark that spot as "a complete word ends here."

Checking if a word was fully stored (search):
1. Start at the top.
2. For each letter in the word, in order: if there's no path for that letter, the word was never stored — stop and report "no."
3. If you successfully follow a path for every letter, check whether that final spot is marked "a complete word ends here." Report yes or no accordingly.

Checking if anything starts with a given prefix (startsWith):
1. Start at the top.
2. For each letter in the prefix, in order: if there's no path for that letter, report "no."
3. If you successfully follow a path for every letter, report "yes" — it doesn't matter whether that spot is marked as a complete word or not.
```

## How It Works

- The diagram is built from four words: `apple`, `app`, `apricot`, `banana`. Follow the path `a → p → p`: three words all start this way, so they all share this same stretch of the tree instead of each having their own copy.
- At that second `p`, there's a `*` mark — that spot is a complete word (`app`) on its own, even though the path keeps going further for `apple` and `apricot`. From there, the tree branches: one way spells out `l → e` (finishing `apple`), the other spells out `r → i → c → o → t` (finishing `apricot`).
- To check `search("app")`: walk `a → p → p`, land on the marked spot, see the `*`, and report "yes, it's a complete word."
- To check `search("appl")`: walk `a → p → p → l`, which succeeds (there is a path for it, since it's partway through `apple`) — but that spot has no `*` mark, since no word ends exactly there. So `search` reports "no," even though `startsWith("appl")` would report "yes."
- That's the key precondition to understand: reaching a spot in the tree only proves a word *starts* that way — it takes the `*` mark to prove a word *ends* there. Mixing those two checks up is the most common way to get this wrong.
- How fast is it? Checking a word only takes as many steps as the word has letters, no matter how many other words are stored in the trie — checking a 5-letter word always takes 5 steps, whether the trie holds 10 words or 10 million (this is `O(L)` time, where L is the length of the word being checked). If a letter's path doesn't exist, the check can stop immediately without finishing the rest of the word.
