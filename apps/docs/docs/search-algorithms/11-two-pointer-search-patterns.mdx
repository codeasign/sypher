## Summary

Two-Pointer Search finds a pair of numbers in a sorted list that add up to a target number. The trick: put one pointer at the start of the list and one at the end, then move them toward each other based on whether the current pair's sum is too big or too small. Because the list is already sorted, this lets you skip huge chunks of possibilities instead of checking every pair one by one — it only looks at each number roughly once, so it stays fast even on long lists.

## Pseudocode

```text
1. Put a "left" pointer on the first number and a "right" pointer on the last number.
2. While left is still before right:
3.     Add the numbers at left and right together.
4.     If the sum equals the target, stop — you found the pair.
5.     If the sum is too small, move left one step forward (toward bigger numbers).
6.     If the sum is too big, move right one step backward (toward smaller numbers).
7. If left and right meet without a match, report "no pair found."
```

## How It Works

- This only works if the list is already sorted from smallest to biggest — that's what makes "too big" or "too small" a useful signal for which pointer to move.
- In the diagram, `left` starts on `1` and `right` starts on `15`. Their sum, `16`, is bigger than the target `9`, so the biggest number (`15`) is clearly too big to be part of the answer — move `right` back one step.
- Now `right` points at `11`. The sum `1 + 11 = 12` is still too big, so `right` moves back again, landing on `7`.
- Now the sum is `1 + 7 = 8`, which is too small this time — so instead move `left` forward, from `1` to `2`.
- The new sum is `2 + 7 = 9`, exactly the target. The pointers stop here and report the pair at positions `1` and `3`.
- If you're lucky and the matching pair sits right at the two ends of the list, you find it almost instantly. If the pair is buried in the middle, or there's no matching pair at all, the pointers have to keep marching toward each other, checking close to every number once (this is "O(n)" time — the "n" means however many numbers are in the list — with only "O(1)" extra memory used for the two pointers).
