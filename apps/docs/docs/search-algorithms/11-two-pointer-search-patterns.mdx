## Summary

Two-Pointer Search finds a pair of numbers in a sorted list that add up to a target number. The trick: put one pointer at the start of the list and one at the end, then move them toward each other based on whether the current pair's sum is too big or too small. Because the list is already sorted, this lets you skip huge chunks of possibilities instead of checking every pair one by one — it only looks at each number roughly once, so it stays fast even on long lists.

## Pseudocode

```text
1. Put a "left" pointer on the first number and a "right" pointer on the last number.
2. While left is still before right:
3.     Add the numbers at left and right together.
4.     If the sum equals the target, stop — you found the pair.
5.     If the sum is too small, move left one step forward (toward bigger numbers).
6.     If the sum is too big, move right one step backward (toward smaller numbers).
7. If left and right meet without a match, report "no pair found."
```

## Worked Example

This trick only works because the list is already sorted from smallest to biggest — like flipping toward the front or back of a sorted phone book instead of checking every page. That order is what makes "too big" or "too small" a useful signal for which pointer to move.

## Complexity

- **Best:** if the matching pair sits right at the two ends of the list, it's found on the very first check (`O(1)`).
- **Average:** the pointers typically step toward each other for a while before meeting, checking a good chunk of the list once (`O(n)`).
- **Worst:** if the pair is buried in the middle, or there's no matching pair at all, the pointers march all the way toward each other, checking close to every number once (`O(n)`).
- **Space:** only two pointers are ever needed, no matter how long the list is, so extra memory stays constant (`O(1)`).

## Applications

- **Two-sum style checks over sorted data** — fraud-detection and finance tools scanning a sorted list of transaction amounts for two entries that add up to a flagged total.
- **Merging two sorted lists** — the merge step of merge sort (and Timsort's merge phase, used inside Python's `list.sort` and Java's `Arrays.sort` for objects) walks two sorted runs with a pointer in each.
- **Palindrome checking** — string-validation libraries compare characters from both ends inward until the pointers meet.
- **Partitioning during quicksort** — the Lomuto and Hoare partition schemes scan from opposite ends of a range with two pointers to split it around a pivot.
- **Container-with-most-water style problems** — geometric and resource-scheduling algorithms that look for the best pair of boundaries by narrowing in from both ends.
