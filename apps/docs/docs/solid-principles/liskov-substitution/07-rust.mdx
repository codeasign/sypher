## Implementation

The violation shows a `Serializer` trait with both `serialize()` and `compress()` methods. A `PlainTextSerializer` cannot support compression but must implement the method, causing a panic. The correct version separates `Serialize` and `Compress` into distinct traits.

```rust
// LSP -- Rust 2021 edition implementation
// Domain: data serializer

// ============================================
// VIOLATION: Serializer trait couples
// serialize and compress, but plain text
// formats don't support compression
// ============================================

trait Serializer {
    fn serialize(&self, data: &str) -> String;
    fn compress(&self, data: &str) -> String;
}

struct JsonSerializer;

impl Serializer for JsonSerializer {
    fn serialize(&self, data: &str) -> String {
        format!(r#"{{"data": "{}"}}"#, data)
    }

    fn compress(&self, data: &str) -> String {
        format!("compressed::{}", data)
    }
}

struct PlainTextSerializer;

impl Serializer for PlainTextSerializer {
    fn serialize(&self, data: &str) -> String {
        data.to_string()
    }

    // VIOLATION: Plain text has no compression support,
    // but the trait contract promises every serializer does
    fn compress(&self, _data: &str) -> String {
        panic!("Plain text serializer does not support compression");
    }
}

fn save_with_compression(serializer: &impl Serializer, data: &str) -> String {
    let serialized = serializer.serialize(data);
    // VIOLATION: Assumes all serializers support compression
    serializer.compress(&serialized)
}

// ============================================
// CORRECT: Separate Serialize and Compress
// ============================================

trait Serialize {
    fn serialize(&self, data: &str) -> String;
}

trait Compress {
    fn compress(&self, data: &str) -> String;
}

struct JsonSerializer;

impl Serialize for JsonSerializer {
    fn serialize(&self, data: &str) -> String {
        format!(r#"{{"data": "{}"}}"#, data)
    }
}

impl Compress for JsonSerializer {
    fn compress(&self, data: &str) -> String {
        format!("gz::{}", data)
    }
}

struct PlainTextSerializer;

impl Serialize for PlainTextSerializer {
    fn serialize(&self, data: &str) -> String {
        data.to_string()
    }
}

// PlainTextSerializer does NOT implement Compress
// -- it's simply not a Compress, no panic needed

// A specialized compressed format
struct GzipSerializer;

impl Serialize for GzipSerializer {
    fn serialize(&self, data: &str) -> String {
        format!("gzipped:{}", data)
    }
}

impl Compress for GzipSerializer {
    fn compress(&self, data: &str) -> String {
        format!("gzip:{}", data)
    }
}

fn serialize_data(serializer: &impl Serialize, data: &str) -> String {
    serializer.serialize(data)
}

fn compress_data(compressor: &impl Compress, data: &str) -> String {
    compressor.compress(data)
}

fn serialize_and_compress(serializer: &(impl Serialize + Compress), data: &str) -> String {
    let serialized = serializer.serialize(data);
    serializer.compress(&serialized)
}

fn main() {
    println!("=== CORRECT IMPLEMENTATION ===");

    let json = JsonSerializer;
    let serialized = serialize_data(&json, "hello");
    println!("JSON serialized: {}", serialized);

    let compressed = compress_data(&json, "hello");
    println!("JSON compressed: {}", compressed);

    let combined = serialize_and_compress(&json, "hello");
    println!("JSON serialized + compressed: {}", combined);

    let plain = PlainTextSerializer;
    let text = serialize_data(&plain, "plain text");
    println!("Plain text: {}", text);

    // This would not compile -- PlainTextSerializer does not implement Compress
    // let result = serialize_and_compress(&plain, "test");

    let gzip = GzipSerializer;
    let gzipped = serialize_and_compress(&gzip, "data");
    println!("Gzip result: {}", gzipped);
}
```

### Key Language-Specific Notes

- Rust enforces substitutability at **compile time** through the trait system. A type can only implement a trait if it provides all required methods. There is no way to "partially implement" a trait -- you either implement it fully or not at all. This eliminates the "throw in override" LSP violation pattern entirely.
- **Blanket implementations** (e.g., `impl<T: Display> ToString for T`) demonstrate LSP at the standard library level: every type that implements `Display` automatically gets a `.to_string()` method, and all such types are substitutable wherever `ToString` is expected.
- **Trait objects** (`dyn Trait`) enable runtime polymorphism while preserving compile-time trait bounds. A `Vec<Box<dyn Serialize>>` can hold any serializer, and all are guaranteed to implement the full `Serialize` contract.
- Use **associated types** and **generic bounds** to express fine-grained substitutability constraints. A function bound with `T: Serialize + Compress` requires both traits, while `T: Serialize` only requires serialization. This is Rust's idiomatic approach to interface segregation and LSP.
