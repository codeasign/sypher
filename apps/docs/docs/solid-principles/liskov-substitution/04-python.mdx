## Implementation

The classic Rectangle-Square problem: `Rectangle` has independent `set_width`/`set_height`, but `Square` overrides them to keep both equal — breaking code that expects independent dimensions. The correct version uses a `Shape` ABC with `area()` that both implement as siblings.

```python
# LSP -- Python 3.11+ implementation
# Domain: shape area calculator
from abc import ABC, abstractmethod

# ============================================
# VIOLATION: Rectangle allows independent
# width/height; Square overrides to keep them
# equal, breaking the contract
# ============================================

class Rectangle:
    def __init__(self, width: float, height: float) -> None:
        self._width = width
        self._height = height

    def set_width(self, width: float) -> None:
        self._width = width

    def set_height(self, height: float) -> None:
        self._height = height

    def area(self) -> float:
        return self._width * self._height

class Square(Rectangle):
    # VIOLATION: Square constrains width == height,
    # but Rectangle code expects independent dimensions
    def set_width(self, width: float) -> None:
        self._width = width
        self._height = width  # Breaks caller's expectation

    def set_height(self, height: float) -> None:
        self._height = height
        self._width = height  # Breaks caller's expectation

def resize_rectangle(rect: Rectangle, new_width: float, new_height: float) -> float:
    rect.set_width(new_width)
    rect.set_height(new_height)
    return rect.area()

rect = Rectangle(4, 5)
print(f"Rectangle(4,5) resized to (6,7): area = {resize_rectangle(rect, 6, 7)}")
# Expected: 42

square = Square(4, 4)
print(f"Square(4,4) resized to (6,7): area = {resize_rectangle(square, 6, 7)}")
# VIOLATION: prints 49, not 42! The caller set width=6 and height=7,
# but Square coerced both to 7, silently producing wrong results.

# ============================================
# CORRECT: Separate Shape ABC, don't inherit
# Rectangle from Square or vice versa
# ============================================

class Shape(ABC):
    @abstractmethod
    def area(self) -> float:
        pass

class Rectangle(Shape):
    def __init__(self, width: float, height: float) -> None:
        self.width = width
        self.height = height

    def area(self) -> float:
        return self.width * self.height

class Square(Shape):
    def __init__(self, side: float) -> None:
        self.side = side

    def area(self) -> float:
        return self.side * self.side

# Functions that need to resize rectangles accept only Rectangle
def resize_rectangle(rect: Rectangle, new_width: float, new_height: float) -> float:
    rect.width = new_width
    rect.height = new_height
    return rect.area()

# Functions that work with any shape accept Shape
def print_area(shape: Shape) -> None:
    print(f"Area: {shape.area()}")

# Main driver
print("=== CORRECT IMPLEMENTATION ===")

rect = Rectangle(4, 5)
print_area(rect)
area = resize_rectangle(rect, 6, 7)
print(f"Rectangle resized: area = {area}")  # 42

sq = Square(4)
print_area(sq)
# resize_rectangle(sq, 6, 7)  # Type error -- Square is not Rectangle

# Both are Shapes though
shapes: list[Shape] = [Rectangle(3, 4), Square(5)]
for shape in shapes:
    print_area(shape)
```

### Key Language-Specific Notes

- Python's `ABC` (Abstract Base Class) from the `abc` module provides formal interface contracts. Subclasses must implement all abstract methods or they cannot be instantiated, enforcing part of the LSP contract.
- Python has **nominal subtyping** by default (a class is a subtype only if it explicitly inherits), but also supports **structural subtyping** through `Protocol` classes from `typing`. Protocols enable duck typing at the type-checker level without requiring inheritance.
- The `@abstractmethod` decorator prevents instantiation of incomplete subclasses at runtime, catching some LSP violations early. However, it does not enforce behavioral contracts (preconditions, postconditions, invariants) -- those must be documented and tested.
- Python's dynamic nature means LSP violations may only surface at runtime. Use type hints with `mypy` or `pyright` in strict mode to catch substitutability issues during development.
