## DIP in JavaScript

JavaScript's prototype-based inheritance and first-class functions offer multiple ways to implement the Dependency Inversion Principle. With ES2022 class syntax, private fields, and static initialization blocks, modern JavaScript can express the DIP as clearly as any statically-typed language.

This example builds a **notification system**. A high-level `NotificationService` needs to send messages through various channels. Without the DIP, it would be coupled to a specific email provider. With the DIP, it depends on an abstraction that any sender can implement.

## The Violation

In the violating version, `NotificationService` directly instantiates `SendGridEmailClient` and calls its methods. Switching to a different provider (Amazon SES, Mailgun, SMTP) requires editing `NotificationService`. Unit testing is impossible without a real SendGrid API key.

```javascript
// VIOLATION: NotificationService depends on a concrete SendGridEmailClient.
// Changing the email provider means modifying NotificationService.
// Testing requires a real SendGrid API key.

class SendGridEmailClient {
  #apiKey;

  constructor(apiKey) {
    this.#apiKey = apiKey;
  }

  async sendEmail(to, subject, body) {
    const response = await fetch('https://api.sendgrid.com/v3/mail/send', {
      method: 'POST',
      headers: {
        Authorization: `Bearer ${this.#apiKey}`,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        personalizations: [{ to: [{ email: to }] }],
        from: { email: 'noreply@example.com' },
        subject,
        content: [{ type: 'text/plain', value: body }],
      }),
    });

    if (!response.ok) {
      throw new Error(`SendGrid error: ${response.status}`);
    }
  }
}

class NotificationService {
  #sender;

  constructor() {
    // VIOLATION: Direct instantiation of concrete implementation.
    this.#sender = new SendGridEmailClient('SG.exampleApiKey12345');
  }

  async sendWelcomeEmail(userEmail, userName) {
    const subject = `Welcome, ${userName}!`;
    const body = `Hi ${userName},\n\nWelcome to our platform! We are excited to have you on board.`;
    await this.#sender.sendEmail(userEmail, subject, body);
  }

  async sendPasswordReset(userEmail, resetToken) {
    const subject = 'Password Reset Request';
    const body = `Click here to reset your password: https://example.com/reset?token=${resetToken}`;
    await this.#sender.sendEmail(userEmail, subject, body);
  }
}

// Usage
async function main() {
  const notificationService = new NotificationService();
  await notificationService.sendWelcomeEmail('alice@example.com', 'Alice');

  // PROBLEM: Cannot switch to SMTP without changing NotificationService.
  // PROBLEM: Cannot test welcome email logic without contacting SendGrid.
}

main().catch(console.error);

/* Expected output:
   (Sends email via SendGrid — requires real API key)
*/
```

## The Correct Implementation

We extract a `NotificationSender` abstraction (a simple class that defines the interface contract), then have `SendGridNotificationSender` and `SmtpNotificationSender` implement it. `NotificationService` depends only on the abstraction and receives its dependency through the constructor.

```javascript
// CORRECT: Both high-level and low-level modules depend on abstraction.
// NotificationSender is the abstraction (interface contract).
// NotificationService depends on the abstraction, not on concretions.

// ── Abstraction ──
// Defines the contract that all senders must follow.
class NotificationSender {
  /**
   * Sends a notification.
   * @param {string} to - Recipient address.
   * @param {string} subject - Message subject.
   * @param {string} body - Message body.
   * @returns {Promise<void>}
   */
  async send(to, subject, body) {
    throw new Error('Subclasses must implement send()');
  }
}

// ── Low-level implementation 1: SendGrid ──
class SendGridNotificationSender extends NotificationSender {
  #apiKey;

  constructor(apiKey) {
    super();
    this.#apiKey = apiKey;
  }

  async send(to, subject, body) {
    console.log(`[SendGrid] Sending to ${to}: ${subject}`);
    // Real implementation would call SendGrid API:
    // const response = await fetch('https://api.sendgrid.com/v3/mail/send', {
    //   method: 'POST',
    //   headers: {
    //     Authorization: `Bearer ${this.#apiKey}`,
    //     'Content-Type': 'application/json',
    //   },
    //   body: JSON.stringify({
    //     personalizations: [{ to: [{ email: to }] }],
    //     from: { email: 'noreply@example.com' },
    //     subject,
    //     content: [{ type: 'text/plain', value: body }],
    //   }),
    // });
  }
}

// ── Low-level implementation 2: SMTP ──
class SmtpNotificationSender extends NotificationSender {
  #host;
  #port;
  #username;
  #password;

  constructor(host, port, username, password) {
    super();
    this.#host = host;
    this.#port = port;
    this.#username = username;
    this.#password = password;
  }

  async send(to, subject, body) {
    console.log(`[SMTP] Sending to ${to}: ${subject}`);
    // Real implementation would use an SMTP library:
    // const transporter = nodemailer.createTransport({
    //   host: this.#host,
    //   port: this.#port,
    //   auth: { user: this.#username, pass: this.#password },
    // });
    // await transporter.sendMail({ from: '...', to, subject, text: body });
  }
}

// ── Low-level implementation 3: Console (for testing) ──
class ConsoleNotificationSender extends NotificationSender {
  async send(to, subject, body) {
    console.log(`[Console] TO: ${to}`);
    console.log(`[Console] SUBJECT: ${subject}`);
    console.log(`[Console] BODY: ${body}`);
    console.log('[Console] -- end of message --');
  }
}

// ── High-level module ──
// CORRECT: Depends only on NotificationSender abstraction.
// Does not know about SendGrid, SMTP, or console.
class NotificationService {
  #sender;

  constructor(sender) {
    // CORRECT: Dependency is injected, not created.
    // sender must conform to the NotificationSender interface.
    this.#sender = sender;
  }

  async sendWelcomeEmail(userEmail, userName) {
    const subject = `Welcome, ${userName}!`;
    const body = [
      `Hi ${userName},`,
      '',
      'Welcome to our platform! We are excited to have you on board.',
      '',
      'Get started by completing your profile:',
      'https://example.com/profile',
    ].join('\n');

    await this.#sender.send(userEmail, subject, body);
  }

  async sendPasswordReset(userEmail, resetToken) {
    const subject = 'Password Reset Request';
    const body = [
      'Hello,',
      '',
      'We received a request to reset your password.',
      `Click here to reset: https://example.com/reset?token=${resetToken}`,
      '',
      'If you did not request this, please ignore this email.',
    ].join('\n');

    await this.#sender.send(userEmail, subject, body);
  }

  async sendOrderConfirmation(userEmail, orderId, items) {
    const subject = `Order #${orderId} Confirmed`;
    const itemsList = items.map((item) => `  - ${item.name} (x${item.quantity})`).join('\n');
    const body = [
      'Thank you for your order!',
      '',
      `Order ID: ${orderId}`,
      'Items:',
      itemsList,
      '',
      'You will receive a shipping confirmation shortly.',
    ].join('\n');

    await this.#sender.send(userEmail, subject, body);
  }
}

// ── Composition root ──
// Wiring dependencies at the entry point.
async function main() {
  // Choose implementation at startup:
  const isProduction = process.env.NODE_ENV === 'production';
  const sender = isProduction
    ? new SendGridNotificationSender(process.env.SENDGRID_API_KEY)
    : new ConsoleNotificationSender();

  const notificationService = new NotificationService(sender);

  await notificationService.sendWelcomeEmail('alice@example.com', 'Alice');
  await notificationService.sendPasswordReset('bob@example.com', 'tok_abc123');
  await notificationService.sendOrderConfirmation(
    'carol@example.com',
    'ORD-0042',
    [
      { name: 'Wireless Mouse', quantity: 1 },
      { name: 'USB-C Hub', quantity: 2 },
    ]
  );
}

main().catch(console.error);

/* Expected output (development mode):

   https://example.com/profile
   [Console] -- end of message --
   [Console] TO: bob@example.com
   [Console] SUBJECT: Password Reset Request
   [Console] BODY: Hello,

   We received a request to reset your password.
   Click here to reset: https://example.com/reset?token=tok_abc123
   ...
   [Console] -- end of message --
   [Console] TO: carol@example.com
   [Console] SUBJECT: Order #ORD-0042 Confirmed
   [Console] BODY: Thank you for your order!
   ...
   [Console] -- end of message --
*/
```

## Key Points

- The base class `NotificationSender` acts as the abstraction. Any class that extends it and implements `send()` can be injected into `NotificationService`.
- The `NotificationService` constructor does not call `new` on any sender — it receives one. This is the hallmark of DIP compliance.
- Switching from SendGrid to SMTP requires zero changes to `NotificationService`. Add a new implementation and wire it in at the composition root.
- Testing is straightforward because you can inject `ConsoleNotificationSender` or a `MockNotificationSender` without real API credentials.
- JavaScript's duck typing means the abstraction can be even lighter: any object with a `send(to, subject, body)` method is acceptable. The base class is optional but documents the contract clearly.

## Testing with DIP

The DIP makes unit testing trivial. No real API keys, no network calls, no SMTP servers:

```javascript
import { describe, it, expect, vi } from 'vitest';

class MockNotificationSender {
  constructor() {
    this.sentMessages = [];
  }

  async send(to, subject, body) {
    this.sentMessages.push({ to, subject, body });
  }
}

describe('NotificationService', () => {
  it('sends a welcome email with the correct content', async () => {
    const mockSender = new MockNotificationSender();
    const service = new NotificationService(mockSender);

    await service.sendWelcomeEmail('test@example.com', 'Tester');

    expect(mockSender.sentMessages).toHaveLength(1);
    expect(mockSender.sentMessages[0].to).toBe('test@example.com');
    expect(mockSender.sentMessages[0].subject).toBe('Welcome, Tester!');
    expect(mockSender.sentMessages[0].body).toContain('complete your profile');
  });
});
```
