## DIP in Java

Java's interface system and constructor-based dependency injection make the Dependency Inversion Principle a standard practice. Modern Java with records, sealed classes, and the module system further reinforces clean abstraction boundaries.

This example builds an **authentication system**. A high-level `AuthService` handles user login, registration, and token management. Without the DIP, it would be coupled to specific JWT libraries and database drivers. With the DIP, it depends on abstractions that any JWT provider or user store can implement.

## The Violation

In the violating version, `AuthService` directly creates `JwtTokenProvider` and `MySqlUserDao` instances. Switching from JWT to Firebase tokens or from MySQL to PostgreSQL requires editing `AuthService`.

```java
// VIOLATION: AuthService depends on concrete JwtTokenProvider and MySqlUserDao.
// Changing token provider or database requires modifying AuthService.
// Testing requires real JWT secret and MySQL connection.

import java.util.HashMap;
import java.util.Map;
import java.util.Optional;
import java.util.UUID;

// ── Concrete classes ──

class User {
    private final String id;
    private final String email;
    private final String passwordHash;

    public User(String id, String email, String passwordHash) {
        this.id = id;
        this.email = email;
        this.passwordHash = passwordHash;
    }

    public String getId() { return id; }
    public String getEmail() { return email; }
    public String getPasswordHash() { return passwordHash; }
}

class MySqlUserDao {
    private final String connectionString;

    public MySqlUserDao(String connectionString) {
        this.connectionString = connectionString;
    }

    public Optional<User> findByEmail(String email) {
        // SELECT * FROM users WHERE email = ?
        System.out.println("[MySQL] Querying user by email: " + email);
        return Optional.of(new User(UUID.randomUUID().toString(), email, "hashed_password"));
    }

    public void save(User user) {
        // INSERT INTO users ...
        System.out.println("[MySQL] Saving user: " + user.getEmail());
    }
}

class JwtTokenProvider {
    private final String secretKey;
    private final long expirationMs;

    public JwtTokenProvider(String secretKey, long expirationMs) {
        this.secretKey = secretKey;
        this.expirationMs = expirationMs;
    }

    public String generateToken(String userId, String email) {
        // In production, use io.jsonwebtoken or similar library
        System.out.println("[JWT] Generating token for user: " + email);
        return "eyJhbGciOiJIUzI1NiJ9." + java.util.Base64.getEncoder()
            .encodeToString((userId + ":" + email).getBytes()) + ".signature";
    }

    public Optional<String> validateToken(String token) {
        System.out.println("[JWT] Validating token");
        return Optional.of("user-123");
    }
}

// ── High-level module ──

class AuthService {
    private final JwtTokenProvider tokenProvider;
    private final MySqlUserDao userDao;

    public AuthService() {
        // VIOLATION: Direct instantiation of concrete implementations.
        this.tokenProvider = new JwtTokenProvider("my-secret-key-12345", 3600000L);
        this.userDao = new MySqlUserDao("jdbc:mysql://localhost:3306/auth");
    }

    public String login(String email, String password) {
        Optional<User> userOpt = userDao.findByEmail(email);
        if (userOpt.isEmpty()) {
            throw new RuntimeException("User not found");
        }
        User user = userOpt.get();
        // Verify password hash (simplified)
        return tokenProvider.generateToken(user.getId(), user.getEmail());
    }

    public void register(String email, String password) {
        String id = UUID.randomUUID().toString();
        String hash = Integer.toHexString(password.hashCode()); // simplified
        User user = new User(id, email, hash);
        userDao.save(user);
    }

    public Optional<String> verifyToken(String token) {
        return tokenProvider.validateToken(token);
    }
}
```

## The Correct Implementation

We extract `TokenProvider` and `UserRepository` interfaces. Multiple implementations of each can be swapped at the composition root. `AuthService` receives its dependencies through the constructor and works with any implementation that satisfies the interfaces.

```java
// CORRECT: AuthService depends on TokenProvider and UserRepository interfaces.
// Implementations are swappable without changing the authentication logic.

import java.util.HashMap;
import java.util.Map;
import java.util.Optional;
import java.util.UUID;

// ── Domain classes ──

record User(String id, String email, String passwordHash) {}

// ── Abstractions ──

interface TokenProvider {
    String generateToken(String userId, String email);
    Optional<String> validateToken(String token);
}

interface UserRepository {
    Optional<User> findByEmail(String email);
    void save(User user);
}

// ── TokenProvider implementations ──

class JwtTokenProvider implements TokenProvider {
    private final String secretKey;
    private final long expirationMs;

    public JwtTokenProvider(String secretKey, long expirationMs) {
        this.secretKey = secretKey;
        this.expirationMs = expirationMs;
    }

    @Override
    public String generateToken(String userId, String email) {
        System.out.println("[JWT] Generating token for user: " + email);
        return "eyJhbGciOiJIUzI1NiJ9."
            + java.util.Base64.getEncoder().encodeToString((userId + ":" + email).getBytes())
            + ".signature";
    }

    @Override
    public Optional<String> validateToken(String token) {
        System.out.println("[JWT] Validating token");
        if (token != null && token.startsWith("eyJ")) {
            return Optional.of("user-123");
        }
        return Optional.empty();
    }
}

class FirebaseTokenProvider implements TokenProvider {
    private final String projectId;

    public FirebaseTokenProvider(String projectId) {
        this.projectId = projectId;
    }

    @Override
    public String generateToken(String userId, String email) {
        System.out.println("[Firebase] Creating custom token for user: " + email);
        // In production, use Firebase Admin SDK to create custom tokens
        return "firebase-custom-token-" + userId;
    }

    @Override
    public Optional<String> validateToken(String token) {
        System.out.println("[Firebase] Verifying ID token");
        // In production, use Firebase Admin SDK to verify
        if (token != null && token.startsWith("firebase-custom-token-")) {
            return Optional.of(token.substring("firebase-custom-token-".length()));
        }
        return Optional.empty();
    }
}

class TestTokenProvider implements TokenProvider {
    @Override
    public String generateToken(String userId, String email) {
        return "test-token-" + userId;
    }

    @Override
    public Optional<String> validateToken(String token) {
        if (token != null && token.startsWith("test-token-")) {
            return Optional.of(token.substring("test-token-".length()));
        }
        return Optional.empty();
    }
}

// ── UserRepository implementations ──

class MySqlUserRepository implements UserRepository {
    private final String connectionString;

    public MySqlUserRepository(String connectionString) {
        this.connectionString = connectionString;
    }

    @Override
    public Optional<User> findByEmail(String email) {
        System.out.println("[MySQL] Querying user by email: " + email);
        return Optional.of(new User(UUID.randomUUID().toString(), email, "hashed_password"));
    }

    @Override
    public void save(User user) {
        System.out.println("[MySQL] Saving user: " + user.email());
    }
}

class PostgresUserRepository implements UserRepository {
    private final String connectionString;

    public PostgresUserRepository(String connectionString) {
        this.connectionString = connectionString;
    }

    @Override
    public Optional<User> findByEmail(String email) {
        System.out.println("[PostgreSQL] Querying user by email: " + email);
        return Optional.of(new User(UUID.randomUUID().toString(), email, "hashed_password"));
    }

    @Override
    public void save(User user) {
        System.out.println("[PostgreSQL] Saving user: " + user.email());
    }
}

class InMemoryUserRepository implements UserRepository {
    private final Map<String, User> usersByEmail = new HashMap<>();

    @Override
    public Optional<User> findByEmail(String email) {
        return Optional.ofNullable(usersByEmail.get(email));
    }

    @Override
    public void save(User user) {
        usersByEmail.put(user.email(), user);
    }
}

// ── High-level module ──
// CORRECT: Depends only on TokenProvider and UserRepository interfaces.

class AuthService {
    private final TokenProvider tokenProvider;
    private final UserRepository userRepository;

    public AuthService(TokenProvider tokenProvider, UserRepository userRepository) {
        // CORRECT: Dependencies are injected, not created.
        this.tokenProvider = tokenProvider;
        this.userRepository = userRepository;
    }

    public String login(String email, String password) {
        Optional<User> userOpt = userRepository.findByEmail(email);
        if (userOpt.isEmpty()) {
            throw new RuntimeException("User not found: " + email);
        }
        User user = userOpt.get();
        // In production, verify password hash with BCrypt or similar
        System.out.println("[AuthService] Password verified for: " + email);
        return tokenProvider.generateToken(user.id(), user.email());
    }

    public void register(String email, String password) {
        String id = UUID.randomUUID().toString();
        String hash = Integer.toHexString(password.hashCode());
        User user = new User(id, email, hash);
        userRepository.save(user);
        System.out.println("[AuthService] Registered user: " + email);
    }

    public Optional<String> verifyToken(String token) {
        return tokenProvider.validateToken(token);
    }
}

// ── Composition root ──

public class Application {
    public static void main(String[] args) {
        System.out.println("=== Scenario 1: JWT + MySQL ===");
        AuthService production = new AuthService(
            new JwtTokenProvider("prod-secret-key", 3600000L),
            new MySqlUserRepository("jdbc:mysql://prod-host:3306/auth")
        );
        String token = production.login("alice@example.com", "password123");
        System.out.println("Token: " + token);
        production.verifyToken(token).ifPresent(userId ->
            System.out.println("Verified user ID: " + userId));
        System.out.println();

        System.out.println("=== Scenario 2: Firebase + PostgreSQL ===");
        AuthService cloudService = new AuthService(
            new FirebaseTokenProvider("my-project-123"),
            new PostgresUserRepository("jdbc:postgresql://cloud-db:5432/auth")
        );
        cloudService.register("bob@example.com", "securePass456");
        String firebaseToken = cloudService.login("bob@example.com", "securePass456");
        System.out.println("Firebase token: " + firebaseToken);
        System.out.println();

        System.out.println("=== Scenario 3: Test with in-memory storage ===");
        InMemoryUserRepository testRepo = new InMemoryUserRepository();
        AuthService testService = new AuthService(new TestTokenProvider(), testRepo);
        testService.register("test@example.com", "testPass");
        String testToken = testService.login("test@example.com", "testPass");
        testService.verifyToken(testToken).ifPresent(userId ->
            System.out.println("Test verified user ID: " + userId));
    }
}
```

## Key Points

- `TokenProvider` and `UserRepository` are Java interfaces. Any class implementing them can be wired into `AuthService`.
- `AuthService` never imports `JwtTokenProvider`, `FirebaseTokenProvider`, `MySqlUserRepository`, or `PostgresUserRepository`. It works purely through abstractions.
- Switching from JWT to Firebase tokens requires a new implementation of `TokenProvider` and a change only in the composition root (`main`). `AuthService` stays unchanged.
- `InMemoryUserRepository` makes unit testing trivial — no database needed.
- The composition root (the `main` method or a DI container configuration) is the only place where concrete classes are instantiated and wired together. This separation of concerns is a key benefit of DIP adherence.

## Testing with DIP

```java
class AuthServiceTest {
    public static void main(String[] args) {
        // Arrange
        InMemoryUserRepository userRepo = new InMemoryUserRepository();
        TestTokenProvider tokenProvider = new TestTokenProvider();
        AuthService service = new AuthService(tokenProvider, userRepo);

        // Act
        service.register("test@example.com", "password123");
        String token = service.login("test@example.com", "password123");
        Optional<String> userId = service.verifyToken(token);

        // Assert
        assert userId.isPresent() : "Token should be valid";
        assert userRepo.findByEmail("test@example.com").isPresent() : "User should exist";
        System.out.println("All tests passed!");
    }
}
```
