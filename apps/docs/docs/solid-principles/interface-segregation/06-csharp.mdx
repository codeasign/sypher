## Implementation

A C# 12 / .NET 8 implementation of ISP using a user management system. The violation shows an `IUserService` interface with `CreateUser()`, `UpdateUser()`, `DeleteUser()`, `SendWelcomeEmail()`, and `GenerateReport()` — a `BasicUserService` must implement report generation it doesn't need.

```csharp
// ISP — C# 12 / .NET 8 implementation
// Domain: User management

// ============================================================
// VIOLATION: Fat IUserService forces every user service to
// implement email sending and report generation
// ============================================================
//
// interface IUserService {
//     User CreateUser(string name, string email);
//     User UpdateUser(int id, string name);
//     void DeleteUser(int id);
//     void SendWelcomeEmail(int userId);
//     string GenerateReport(string format);
// }
//
// class BasicUserService : IUserService {
//     public User CreateUser(string name, string email) => new User(1, name, email);
//     public User UpdateUser(int id, string name) => new User(id, name, "test@test.com");
//     public void DeleteUser(int id) { }
//     public void SendWelcomeEmail(int userId) =>
//         throw new NotSupportedException("No email capability");
//     public string GenerateReport(string format) =>
//         throw new NotSupportedException("No reporting capability");
// }

// ============================================================
// CORRECT IMPLEMENTATION — Segregated interfaces
// ============================================================

// --- Data types ---

record User(int Id, string Name, string Email);

// --- Segregated interfaces ---

interface IUserRepository
{
    User CreateUser(string name, string email);
    User UpdateUser(int id, string name);
    void DeleteUser(int id);
    User? GetUser(int id);
}

interface IEmailService
{
    void SendWelcomeEmail(User user);
}

interface IReportGenerator
{
    string GenerateReport(string format);
}

// --- Implementations ---

sealed class InMemoryUserRepository : IUserRepository
{
    private readonly Dictionary<int, User> _users = new();
    private int _nextId = 1;

    public User CreateUser(string name, string email)
    {
        var user = new User(_nextId++, name, email);
        _users[user.Id] = user;
        return user;
    }

    public User UpdateUser(int id, string name)
    {
        if (!_users.TryGetValue(id, out var existing))
            throw new KeyNotFoundException($"User {id} not found");
        var updated = existing with { Name = name };
        _users[id] = updated;
        return updated;
    }

    public void DeleteUser(int id) => _users.Remove(id);

    public User? GetUser(int id) =>
        _users.TryGetValue(id, out var user) ? user : null;
}

sealed class SmtpEmailService : IEmailService
{
    public void SendWelcomeEmail(User user)
    {
        Console.WriteLine($"Sending welcome email to {user.Name} at {user.Email}");
    }
}

sealed class CsvReportGenerator : IReportGenerator
{
    public string GenerateReport(string format)
    {
        return $"Report generated in {format} format";
    }
}

// --- Orchestration ---

sealed class UserManagementApp
{
    private readonly IUserRepository _repo;
    private readonly IEmailService? _email;
    private readonly IReportGenerator? _reporter;

    public UserManagementApp(
        IUserRepository repo,
        IEmailService? email = null,
        IReportGenerator? reporter = null)
    {
        _repo = repo;
        _email = email;
        _reporter = reporter;
    }

    public User CreateUser(string name, string email)
    {
        var user = _repo.CreateUser(name, email);
        _email?.SendWelcomeEmail(user);  // Optional — depends on registration
        return user;
    }

    public string? GenerateReport(string format) =>
        _reporter?.GenerateReport(format);
}

// --- Main driver ---

var basicApp = new UserManagementApp(
    new InMemoryUserRepository()
);

var fullApp = new UserManagementApp(
    new InMemoryUserRepository(),
    new SmtpEmailService(),
    new CsvReportGenerator()
);

Console.WriteLine("=== Basic App (no email, no reports) ===");
var user1 = basicApp.CreateUser("Alice", "alice@example.com");
Console.WriteLine($"Created: {user1.Name} ({user1.Email})");
Console.WriteLine($"Report: {basicApp.GenerateReport("CSV") ?? "Not available"}");

Console.WriteLine("\n=== Full App ===");
var user2 = fullApp.CreateUser("Bob", "bob@example.com");
Console.WriteLine($"Created: {user2.Name} ({user2.Email})");
Console.WriteLine($"Report: {fullApp.GenerateReport("CSV")}");
```

Expected output:

```bash

Report: Not available

=== Full App ===

Created: Bob (bob@example.com)
Report: Report generated in CSV format
```

## Key Language-Specific Notes

- **Smallest interfaces possible** — Each interface (`IUserRepository`, `IEmailService`, `IReportGenerator`) covers exactly one capability domain. A class implements only what it needs.
- **Nullable dependency injection** — `IEmailService?` and `IReportGenerator?` are optional in the constructor, letting the app scale its capabilities without changing the orchestration class.
- **`sealed` classes** — Each implementation is sealed, meaning no further extension by inheritance. ISP-compliant interfaces can be implemented independently without a class hierarchy.
- **`record` types** for DTOs (`User`) provide immutability across interface boundaries, preventing coupling through shared mutable state.
