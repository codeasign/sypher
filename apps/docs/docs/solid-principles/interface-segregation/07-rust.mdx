## Implementation

A Rust 2021 edition implementation of ISP using a database operations system. The violation shows a `Database` trait with `connect()`, `query()`, `insert()`, `update()`, `delete()`, and `backup()` — a `ReadOnlyReplica` must implement write operations it cannot support.

```rust
// ISP — Rust 2021 edition implementation
// Domain: Database operations

use std::collections::HashMap;

// ============================================================
// VIOLATION: Fat Database trait forces ReadOnlyReplica to
// implement write operations it cannot meaningfully support
// ============================================================
//
// trait Database {
//     fn connect(&self) -> String;
//     fn query(&self, sql: &str) -> Vec<String>;
//     fn insert(&self, table: &str, data: &str) -> String;
//     fn update(&self, table: &str, id: &str, data: &str) -> String;
//     fn delete(&self, table: &str, id: &str) -> String;
//     fn backup(&self) -> String;
// }
//
// struct ReadOnlyReplica;
//
// impl Database for ReadOnlyReplica {
//     fn connect(&self) -> String { "Connected to replica".into() }
//     fn query(&self, sql: &str) -> Vec<String> { vec!["result".into()] }
//     fn insert(&self, _: &str, _: &str) -> String {
//         panic!("Read-only replica cannot insert")
//     }
//     fn update(&self, _: &str, _: &str, _: &str) -> String {
//         panic!("Read-only replica cannot update")
//     }
//     fn delete(&self, _: &str, _: &str) -> String {
//         panic!("Read-only replica cannot delete")
//     }
//     fn backup(&self) -> String { "Read-only replica backed up".into() }
// }

// ============================================================
// CORRECT IMPLEMENTATION — Segregated traits
// ============================================================

/// Trait for connectable database resources
trait Connectable {
    fn connect(&self) -> String;
}

/// Trait for read-only query operations
trait Queryable {
    fn query(&self, sql: &str) -> Vec<String>;
}

/// Trait for write operations (insert, update, delete)
trait Writeable {
    fn insert(&self, table: &str, data: &str) -> String;
    fn update(&self, table: &str, id: &str, data: &str) -> String;
    fn delete(&self, table: &str, id: &str) -> String;
}

/// Trait for backup operations
trait Backupable {
    fn backup(&self) -> String;
}

// --- Read-only replica: only Connectable + Queryable ---

struct ReadOnlyReplica {
    name: String,
}

impl ReadOnlyReplica {
    fn new(name: &str) -> Self {
        Self { name: name.to_string() }
    }
}

impl Connectable for ReadOnlyReplica {
    fn connect(&self) -> String {
        format!("Connected to replica: {}", self.name)
    }
}

impl Queryable for ReadOnlyReplica {
    fn query(&self, sql: &str) -> Vec<String> {
        vec![format!("[{}] Result for: {}", self.name, sql)]
    }
}

// --- Primary database: all traits ---

struct PrimaryDatabase {
    name: String,
    data: HashMap<String, Vec<String>>,
}

impl PrimaryDatabase {
    fn new(name: &str) -> Self {
        Self {
            name: name.to_string(),
            data: HashMap::new(),
        }
    }
}

impl Connectable for PrimaryDatabase {
    fn connect(&self) -> String {
        format!("Connected to primary: {}", self.name)
    }
}

impl Queryable for PrimaryDatabase {
    fn query(&self, sql: &str) -> Vec<String> {
        self.data.get(sql)
            .cloned()
            .unwrap_or_else(|| vec![format!("[{}] Query executed: {}", self.name, sql)])
    }
}

impl Writeable for PrimaryDatabase {
    fn insert(&self, table: &str, data: &str) -> String {
        format!("[{}] Inserted into {}: {}", self.name, table, data)
    }

    fn update(&self, table: &str, id: &str, data: &str) -> String {
        format!("[{}] Updated {} {}: {}", self.name, table, id, data)
    }

    fn delete(&self, table: &str, id: &str) -> String {
        format!("[{}] Deleted from {} where id={}", self.name, table, id)
    }
}

impl Backupable for PrimaryDatabase {
    fn backup(&self) -> String {
        format!("[{}] Full backup completed", self.name)
    }
}

// --- Functions constrained to specific capabilities ---

fn run_queries(db: &impl Queryable, queries: &[&str]) {
    for query in queries {
        for result in db.query(query) {
            println!("  {}", result);
        }
    }
}

fn write_data(db: &(impl Queryable + Writeable), table: &str, data: &str) {
    let id = db.insert(table, data);
    println!("  Write result: {}", id);
    let results = db.query(&format!("SELECT * FROM {table}"));
    for r in results {
        println!("  Verified: {}", r);
    }
}

// --- Main driver ---
fn main() {
    let replica = ReadOnlyReplica::new("replica-01");
    let primary = PrimaryDatabase::new("primary-db");

    println!("=== Read-Only Replica ===");
    println!("{}", replica.connect());
    run_queries(&replica, &["SELECT * FROM users", "SELECT * FROM orders"]);

    // replica.insert(...);  // Compile error! ReadOnlyReplica doesn't implement Writeable

    println!("\n=== Primary Database ===");
    println!("{}", primary.connect());
    write_data(&primary, "users", "Alice, Bob");
    println!("{}", primary.backup());
}
```

## Key Language-Specific Notes

- **Small, focused traits** — Each trait (`Connectable`, `Queryable`, `Writeable`, `Backupable`) covers exactly one capability. This is the idiomatic Rust application of ISP: one trait, one concern.
- **Trait bounds as capability contracts** — `fn run_queries(db: &impl Queryable)` constrains the function to types that can query. The compiler guarantees that only query-capable types are passed in.
- **Blanket implementations** — You could implement `Queryable` for any `Writeable` type with a blanket impl if every writable database should also be queryable, avoiding repetitive code.
- **No `panic!` stubs** — The correct version has zero panicking or no-op method implementations. Every trait method has a real implementation for the types that implement it.
- **Compile-time safety** — Calling `insert()` on a `ReadOnlyReplica` is a compile error, not a runtime panic. The Rust type system enforces ISP at compilation.
