Java's interface system and sealed types (introduced in Java 17) provide powerful tools for implementing the Open/Closed Principle. The `ServiceLoader` pattern allows runtime discovery of implementations, and sealed interfaces give compile-time exhaustiveness when needed.

## The Domain: Tax Calculator for Different Jurisdictions

A global e-commerce platform must calculate sales tax, VAT, or GST for every jurisdiction it ships to. Tax rules vary wildly between countries and change frequently. New jurisdictions are added as the company expands.

### The Violation

The following example shows a `TaxCalculator` class that branches on a country code. Every new jurisdiction requires editing this class.

```java
// VIOLATION: calculateTax uses if-else on country code.
// Adding a new jurisdiction means editing this class, risking
// incorrect tax calculations for all existing countries.

import java.math.BigDecimal;
import java.math.RoundingMode;

// VIOLATION: Branching on country code -- each new country requires
// adding an else-if block and retesting tax for every country.
class TaxCalculator {
    /**
     * Calculate tax for a given amount in a given jurisdiction.
     *
     * @param amount  The pre-tax amount (e.g., product price)
     * @param country Two-letter ISO country code
     * @return Tax amount
     */
    // VIOLATION: monolithic if-else chain -- every new country adds a branch
    public BigDecimal calculateTax(BigDecimal amount, String country) {
        if (country.equals("US")) {
            // US sales tax: varies by state, simplified to a flat 8%
            BigDecimal taxRate = new BigDecimal("0.08");
            BigDecimal tax = amount.multiply(taxRate);
            System.out.println("[US] Applying 8% sales tax");
            return tax.setScale(2, RoundingMode.HALF_UP);

        } else if (country.equals("UK")) {
            // UK VAT: 20% standard rate
            BigDecimal taxRate = new BigDecimal("0.20");
            BigDecimal tax = amount.multiply(taxRate);
            System.out.println("[UK] Applying 20% VAT");
            return tax.setScale(2, RoundingMode.HALF_UP);

        } else if (country.equals("DE")) {
            // Germany VAT: 19% standard rate
            BigDecimal taxRate = new BigDecimal("0.19");
            BigDecimal tax = amount.multiply(taxRate);
            System.out.println("[DE] Applying 19% VAT");
            return tax.setScale(2, RoundingMode.HALF_UP);

        } else if (country.equals("IN")) {
            // India GST: 18% standard rate
            BigDecimal taxRate = new BigDecimal("0.18");
            BigDecimal tax = amount.multiply(taxRate);
            System.out.println("[IN] Applying 18% GST");
            return tax.setScale(2, RoundingMode.HALF_UP);

        } else {
            throw new IllegalArgumentException("Unsupported country: " + country);
        }
    }

    public static void main(String[] args) {
        TaxCalculator calculator = new TaxCalculator();

        BigDecimal amount = new BigDecimal("100.00");

        System.out.println("Product price: $" + amount);
        System.out.println("US tax:  $" + calculator.calculateTax(amount, "US"));
        System.out.println("UK tax:  $" + calculator.calculateTax(amount, "UK"));
        System.out.println("DE tax:  $" + calculator.calculateTax(amount, "DE"));
        System.out.println("IN tax:  $" + calculator.calculateTax(amount, "IN"));
    }
}
```

Expected output:

```bash
Product price: $100.00
US tax:  $8.00
UK tax:  $20.00
DE tax:  $19.00
IN tax:  $18.00
```

### The Correct Implementation: Interface with Strategy

The solution defines a `TaxCalculator` interface. Each jurisdiction implements the interface. A registry maps country codes to calculator instances.

```java
// CORRECT: TaxCalculator interface defines the contract. Each jurisdiction
// is a separate implementation. The TaxService is closed for modification.

import java.math.BigDecimal;
import java.math.RoundingMode;
import java.util.HashMap;
import java.util.Map;
import java.util.ServiceLoader;

// --- Interface contract ---
interface TaxCalculator {
    /** Two-letter ISO country code this calculator supports. */
    String getCountryCode();

    /** Calculate tax for the given amount. */
    BigDecimal calculate(BigDecimal amount);

    /** Human-readable tax description (e.g., "8% sales tax"). */
    String getTaxDescription();
}

// --- Concrete implementations ---

class USTaxCalculator implements TaxCalculator {
    @Override
    public String getCountryCode() { return "US"; }

    @Override
    public BigDecimal calculate(BigDecimal amount) {
        // US: simplified flat 8% sales tax
        BigDecimal tax = amount.multiply(new BigDecimal("0.08"));
        System.out.println("[US] Applying 8% sales tax");
        return tax.setScale(2, RoundingMode.HALF_UP);
    }

    @Override
    public String getTaxDescription() { return "8% sales tax"; }
}

class UKTaxCalculator implements TaxCalculator {
    @Override
    public String getCountryCode() { return "UK"; }

    @Override
    public BigDecimal calculate(BigDecimal amount) {
        // UK: 20% VAT
        BigDecimal tax = amount.multiply(new BigDecimal("0.20"));
        System.out.println("[UK] Applying 20% VAT");
        return tax.setScale(2, RoundingMode.HALF_UP);
    }

    @Override
    public String getTaxDescription() { return "20% VAT"; }
}

class GermanyTaxCalculator implements TaxCalculator {
    @Override
    public String getCountryCode() { return "DE"; }

    @Override
    public BigDecimal calculate(BigDecimal amount) {
        // Germany: 19% VAT
        BigDecimal tax = amount.multiply(new BigDecimal("0.19"));
        System.out.println("[DE] Applying 19% VAT");
        return tax.setScale(2, RoundingMode.HALF_UP);
    }

    @Override
    public String getTaxDescription() { return "19% VAT"; }
}

class IndiaTaxCalculator implements TaxCalculator {
    @Override
    public String getCountryCode() { return "IN"; }

    @Override
    public BigDecimal calculate(BigDecimal amount) {
        // India: 18% GST
        BigDecimal tax = amount.multiply(new BigDecimal("0.18"));
        System.out.println("[IN] Applying 18% GST");
        return tax.setScale(2, RoundingMode.HALF_UP);
    }

    @Override
    public String getTaxDescription() { return "18% GST"; }
}

// --- Registry ---
class TaxRegistry {
    private final Map<String, TaxCalculator> calculators = new HashMap<>();

    public void register(TaxCalculator calculator) {
        calculators.put(calculator.getCountryCode(), calculator);
    }

    public TaxCalculator get(String countryCode) {
        TaxCalculator calculator = calculators.get(countryCode);
        if (calculator == null) {
            throw new IllegalArgumentException(
                "No tax calculator for country: " + countryCode
            );
        }
        return calculator;
    }

    public boolean supports(String countryCode) {
        return calculators.containsKey(countryCode);
    }

    public java.util.Set<String> getSupportedCountries() {
        return calculators.keySet();
    }
}

// --- Core service -- closed for modification ---
class TaxService {
    private final TaxRegistry registry;

    public TaxService(TaxRegistry registry) {
        this.registry = registry;
    }

    public BigDecimal calculateTax(BigDecimal amount, String countryCode) {
        TaxCalculator calculator = registry.get(countryCode);
        System.out.println("Calculating tax for " + countryCode
            + " (" + calculator.getTaxDescription() + ")");
        return calculator.calculate(amount);
    }

    public Map<String, BigDecimal> calculateForAll(
            BigDecimal amount,
            String... countries) {
        Map<String, BigDecimal> results = new HashMap<>();
        for (String country : countries) {
            if (registry.supports(country)) {
                results.put(country, calculateTax(amount, country));
            }
        }
        return results;
    }
}

// --- Main ---
public class TaxApplication {
    public static void main(String[] args) {
        // Setup registry
        TaxRegistry registry = new TaxRegistry();
        registry.register(new USTaxCalculator());
        registry.register(new UKTaxCalculator());
        registry.register(new GermanyTaxCalculator());
        registry.register(new IndiaTaxCalculator());

        TaxService service = new TaxService(registry);

        BigDecimal amount = new BigDecimal("100.00");
        System.out.println("Product price: $" + amount + "\n");

        // Calculate for individual countries
        System.out.println("US tax:  $" + service.calculateTax(amount, "US"));
        System.out.println("UK tax:  $" + service.calculateTax(amount, "UK"));
        System.out.println("DE tax:  $" + service.calculateTax(amount, "DE"));
        System.out.println("IN tax:  $" + service.calculateTax(amount, "IN"));

        // Bulk calculation
        System.out.println("\n=== Bulk calculation ===");
        Map<String, BigDecimal> allTaxes = service.calculateForAll(
            amount, "US", "UK", "DE", "IN"
        );
        for (var entry : allTaxes.entrySet()) {
            System.out.println(entry.getKey() + ": $" + entry.getValue());
        }

        // --- Adding a new jurisdiction without modifying existing code ---
        //
        // New requirement: Canada (HST). We create a new class and register it.
        // No existing files change.

        System.out.println("\n=== Adding Canada (no edits to existing code) ===");
        registry.register(new CanadaTaxCalculator());
        BigDecimal canadaTax = service.calculateTax(amount, "CA");
        System.out.println("CA tax: $" + canadaTax);
    }
}

// New jurisdiction -- separate file, no modifications to existing code
class CanadaTaxCalculator implements TaxCalculator {
    @Override
    public String getCountryCode() { return "CA"; }

    @Override
    public BigDecimal calculate(BigDecimal amount) {
        // Canada HST: 13% harmonized sales tax (simplified)
        BigDecimal tax = amount.multiply(new BigDecimal("0.13"));
        System.out.println("[CA] Applying 13% HST");
        return tax.setScale(2, RoundingMode.HALF_UP);
    }

    @Override
    public String getTaxDescription() { return "13% HST"; }
}
```

## Key Takeaways

- **Interface contracts** define the shape every tax calculator must satisfy. Java's interface default methods can add shared behavior (like validation) without breaking existing implementations.
- **`BigDecimal` for monetary values** demonstrates domain-appropriate types. Avoid `double` or `float` for financial calculations.
- **The `TaxRegistry`** uses a `HashMap<String, TaxCalculator>`. Each calculator self-registers its country code, keeping the registry simple and the calculators cohesive.
- **Java 17 sealed interfaces** could restrict which classes implement `TaxCalculator`:

  ```java
  sealed interface TaxCalculator permits USTaxCalculator, UKTaxCalculator, ... {}
  ```

  Sealed interfaces give compile-time exhaustiveness when switching over implementations, useful when the set of implementations is known and finite.

- **`ServiceLoader` pattern** (Java SPI) enables automatic discovery of implementations at runtime. Place implementations in `META-INF/services/` and the JVM loads them without explicit registration.
- **Adding a jurisdiction** is purely additive: write a new class implementing `TaxCalculator`, register it. `TaxService` never changes.
- The **bulk calculation** method `calculateForAll` demonstrates that derived operations built on the closed core also remain closed.
