# Software Engineering

*Estimated completion time: 40–60 hours*

**Difficulty:** Intermediate to Advanced

## What This Course Covers

This course covers the foundational principles and design patterns that every software engineer needs to write maintainable, scalable, and robust code. You will start with the SOLID principles — the five object-oriented design principles that form the bedrock of clean architecture — and then explore the Gang of Four design patterns organized by creational, structural, and behavioral categories.

## Why This Matters

Knowing a programming language is only the beginning. The difference between code that works and code that lasts comes down to design. SOLID principles help you write code that adapts to change without breaking, and design patterns give you a shared vocabulary for solving common problems. Together, they form the essential toolkit for professional software engineering.

## Skills You Will Gain

- Apply the five SOLID principles to write maintainable, loosely-coupled code
- Recognize and implement creational patterns: Factory Method, Abstract Factory, Singleton, Builder, Prototype
- Recognize and implement structural patterns: Adapter, Bridge, Composite, Decorator, Facade, Flyweight, Proxy
- Recognize and implement behavioral patterns: Chain of Responsibility, Command, Interpreter, Iterator, Mediator, Memento, Observer, State, Strategy, Template Method, Visitor
- Choose the right design pattern for a given problem
- Write clean, testable, and extensible code across multiple languages

## Course Sections

### SOLID Principles
Master the five principles of object-oriented design:
- **Single Responsibility** — one class, one reason to change
- **Open/Closed** — open for extension, closed for modification
- **Liskov Substitution** — subtypes must be substitutable for their base types
- **Interface Segregation** — many specific interfaces are better than one general interface
- **Dependency Inversion** — depend on abstractions, not concretions

### Design Patterns
Explore the three categories of Gang of Four design patterns:

**Creational** — object creation mechanisms:
Factory Method, Abstract Factory, Singleton, Builder, Prototype

**Structural** — object composition and relationships:
Adapter, Bridge, Composite, Decorator, Facade, Flyweight, Proxy

**Behavioral** — object communication and responsibility:
Chain of Responsibility, Command, Interpreter, Iterator, Mediator, Memento, Observer, State, Strategy, Template Method, Visitor

## Prerequisites

- Working knowledge of at least one object-oriented programming language (JavaScript, TypeScript, Python, Java, C#, or Rust)
- Understanding of classes, inheritance, interfaces, and polymorphism

## How to Use This Course

Each principle and pattern has multiple language-specific implementations so you can follow along in your preferred language or compare across languages. Start with the overview for each topic, then dive into the language-specific pages that interest you.
