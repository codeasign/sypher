## Implementation

Below is a complete Rust implementation of the Facade pattern using a cars domain. The `CarFacade` struct provides a unified interface for starting, driving, and stopping a modern car. Six subsystem structs — `Engine`, `Transmission`, `CoolingSystem`, `ElectricalSystem`, `FuelSystem`, and `BrakingSystem` — each encapsulate a specific vehicle domain. The facade owns all six subsystems and orchestrates the correct startup sequence, with a battery voltage check as a precondition before the starter relay engages.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌───────────────────────────────────────────┐
//   │   CarFacade (Facade)                      │  ← OOP: Composition, Encapsulation
//   ├───────────────────────────────────────────┤
//   │   - engine: Engine                        │
//   │   - transmission: Transmission            │
//   │   - cooling: CoolingSystem                │
//   │   - electrical: ElectricalSystem          │
//   │   - fuel_system: FuelSystem               │
//   │   - brakes: BrakingSystem                 │
//   ├───────────────────────────────────────────┤
//   │   + fn new() -> CarFacade                 │
//   │   + fn start_engine(&mut self)            │
//   │   + fn drive(&mut self, speed: u32)       │
//   │   + fn stop_engine(&mut self)             │
//   └───────────────────────────────────────────┘
```

#### OOP Concepts Applied

1. **Encapsulation** — Rust's module system and private fields enforce strict encapsulation. Each struct's fields are private by default. Only the public methods form the API surface.

2. **Composition** — `CarFacade` owns six subsystem structs. Rust's ownership model guarantees that the subsystems are dropped when the facade is dropped, preventing resource leaks.

3. **Abstraction** — The caller invokes `start_engine()`, `drive()`, and `stop_engine()`. The complex sequence of battery checks, fuel priming, and spark timing is hidden behind these three method names.

#### SOLID Principles Applied

4. **SRP** — `Engine` handles cranking and RPM. `FuelSystem` manages fuel pressure. `CarFacade` orchestrates the sequence but does not implement subsystem behavior.

5. **OCP** — New subsystems (e.g., `Turbocharger`, `HybridBattery`) can be added without changing existing subsystem structs. The facade is extended by adding a new field and integrating it into the orchestration methods.

6. **ISP** — Each subsystem struct exposes a minimal set of methods. `BrakingSystem` has only `release()`, `apply()`, and `is_engaged()`. Each subsystem provides only what it needs.

### Code

#### Subsystem: Engine

```rust
struct Engine {
    rpm: u32,
}

impl Engine {
    fn new() -> Self {
        Engine { rpm: 0 }
    }

    fn crank(&mut self) {
        self.rpm = 250;
        println!("Engine: cranking at 250 RPM");
    }

    fn inject_fuel(&self) {
        println!("Engine: injecting fuel into cylinders");
    }

    fn spark(&self) {
        println!("Engine: firing spark plugs");
    }

    fn get_idle_rpm(&self) -> u32 {
        850
    }

    fn get_rpm(&self) -> u32 {
        self.rpm
    }

    fn rev(&mut self, amount: u32) {
        self.rpm += amount;
        println!("Engine: revving to {} RPM", self.rpm);
    }

    fn stop(&mut self) {
        self.rpm = 0;
        println!("Engine: stopped");
    }
}
```

#### Facade

```rust
struct CarFacade {
    engine: Engine,
    transmission: Transmission,
    cooling: CoolingSystem,
    electrical: ElectricalSystem,
    fuel_system: FuelSystem,
    brakes: BrakingSystem,
}

impl CarFacade {
    fn new() -> Self {
        CarFacade {
            engine: Engine::new(),
            transmission: Transmission::new(),
            cooling: CoolingSystem::new(),
            electrical: ElectricalSystem::new(),
            fuel_system: FuelSystem::new(),
            brakes: BrakingSystem::new(),
        }
    }

    fn start_engine(&mut self) {
        println!("--- Starting Engine ---");
        let voltage = self.electrical.get_battery_voltage();
        if voltage < 12.0 {
            println!("ERROR: Battery voltage too low.");
            return;
        }
        self.electrical.power_ecu();
        self.electrical.power_accessories();
        self.electrical.close_starter_relay();
        self.cooling.run_water_pump();
        self.fuel_system.prime(55.0);
        self.engine.crank();
        self.engine.inject_fuel();
        self.engine.spark();
        println!("Engine started. Idle RPM: {}", self.engine.get_idle_rpm());
        self.cooling.run_radiator_fan();
        self.brakes.release();
    }

    fn drive(&mut self, speed: u32) {
        println!("--- Driving at {} RPM ---", speed);
        self.transmission.shift_to("Drive");
        self.engine.rev(speed);
        let target_temp = 90.0;
        self.cooling.set_thermostat(target_temp);
        println!("Driving: cruising at {} RPM", speed);
    }

    fn stop_engine(&mut self) {
        println!("--- Stopping Engine ---");
        self.transmission.shift_to("Park");
        self.brakes.apply();
        self.engine.stop();
        self.fuel_system.shutoff();
        println!("Engine stopped. All subsystems off.");
    }
}
```

#### Main / Driver

```rust
fn main() {
    let mut car = CarFacade::new();
    car.start_engine();
    println!();
    car.drive(2500);
    println!();
    car.stop_engine();
}
```

### Language-Specific Notes

1. **Ownership model** — The facade owns all six subsystem structs directly. When the `CarFacade` is dropped at the end of `main()`, Rust automatically drops each subsystem, preventing resource leaks without a garbage collector or manual `free` calls.

2. **`&mut self` on all facade methods** — Because the facade mutates its subsystems (cranking the engine, shifting gears, priming the fuel pump), all three public methods take `&mut self`. This ensures that only one reference to the facade exists at any time, preventing data races.

3. **Associated function `new()`** — Rust does not have constructors. The `CarFacade::new()` associated function is the idiomatic factory method that creates and returns a fully initialized instance.

4. **`println!` macro** — Rust uses `println!("...", args)` with format arguments rather than string interpolation. The compiler checks format strings at compile time, preventing format string mismatches.

5. **No null** — Rust has no null pointer. Every field is initialized in the `new()` associated function. This eliminates an entire class of null reference bugs that plague other languages.

6. **`String` vs `&str`** — The `Transmission` struct stores the gear as a `String` (owned, heap-allocated). The `shift_to` method accepts a `&str` (borrowed string slice), which is more flexible and allows passing both string literals and `String` references.
