## Implementation

Below is a complete TypeScript implementation of the Interpreter pattern using a Stock Market domain. The pattern defines a composable expression grammar for stock filtering. The `Expression` interface declares the `interpret()` method. Terminal expressions (`PriceGreaterThan`, `VolumeGreaterThan`, `TickerEquals`, `SectorEquals`) implement atomic comparisons. Nonterminal expressions (`AndExpr`, `OrExpr`, `NotExpr`) combine expressions logically. The `StockScreener` client evaluates an expression tree against a stock portfolio, collecting matches.

#### OOP Concepts Applied

1. **Abstraction** вҖ” The `Expression` interface defines a pure contract. All code depends on `Expression`, not on concrete classes.
2. **Encapsulation** вҖ” Each expression class stores its own data (thresholds, child expressions) and exposes only `interpret()`.
3. **Inheritance** вҖ” Classes implement the `Expression` interface, providing concrete `interpret()` bodies.
4. **Polymorphism** вҖ” The `screen()` method calls `interpret()` on any `Expression` reference. The runtime type determines the behavior.

#### SOLID Principles Applied

5. **SRP** вҖ” Each expression class evaluates exactly one grammar rule. `StockScreener` only iterates and collects results.
6. **OCP** вҖ” New expression types implement `Expression` without modifying existing expressions or the screener.
7. **LSP** вҖ” Every concrete expression is substitutable for `Expression`. The screener never needs to check concrete types.
8. **ISP** вҖ” The `Expression` interface is a single method вҖ” minimal and focused.
9. **DIP** вҖ” `StockScreener` depends on `Expression`, not on concrete implementations.

### StockData (Context)

```typescript
// в”Җв”Җ Context в”Җв”Җ
interface StockData {
  ticker: string;
  price: number;
  volume: number;
  sector: string;
}
```

### Expression (AbstractExpression)

```typescript
// в”Җв”Җ Abstract Expression в”Җв”Җ
interface Expression {
  interpret(stock: StockData): boolean;
}
```

### PriceGreaterThan (TerminalExpression)

```typescript
class PriceGreaterThan implements Expression {
  private threshold: number;

  constructor(threshold: number) {
    this.threshold = threshold;
  }

  interpret(stock: StockData): boolean {
    return stock.price > this.threshold;
  }
}
```

### VolumeGreaterThan (TerminalExpression)

```typescript
class VolumeGreaterThan implements Expression {
  private threshold: number;

  constructor(threshold: number) {
    this.threshold = threshold;
  }

  interpret(stock: StockData): boolean {
    return stock.volume > this.threshold;
  }
}
```

### TickerEquals (TerminalExpression)

```typescript
class TickerEquals implements Expression {
  private ticker: string;

  constructor(ticker: string) {
    this.ticker = ticker.toUpperCase();
  }

  interpret(stock: StockData): boolean {
    return stock.ticker === this.ticker;
  }
}
```

### SectorEquals (TerminalExpression)

```typescript
class SectorEquals implements Expression {
  private sector: string;

  constructor(sector: string) {
    this.sector = sector;
  }

  interpret(stock: StockData): boolean {
    return stock.sector === this.sector;
  }
}
```

### AndExpr (NonterminalExpression)

```typescript
class AndExpr implements Expression {
  private left: Expression;
  private right: Expression;

  constructor(left: Expression, right: Expression) {
    this.left = left;
    this.right = right;
  }

  interpret(stock: StockData): boolean {
    return this.left.interpret(stock) && this.right.interpret(stock);
  }
}
```

### OrExpr (NonterminalExpression)

```typescript
class OrExpr implements Expression {
  private left: Expression;
  private right: Expression;

  constructor(left: Expression, right: Expression) {
    this.left = left;
    this.right = right;
  }

  interpret(stock: StockData): boolean {
    return this.left.interpret(stock) || this.right.interpret(stock);
  }
}
```

### NotExpr (NonterminalExpression)

```typescript
class NotExpr implements Expression {
  private expr: Expression;

  constructor(expr: Expression) {
    this.expr = expr;
  }

  interpret(stock: StockData): boolean {
    return !this.expr.interpret(stock);
  }
}
```

### StockScreener (Client)

```typescript
// в”Җв”Җ Client в”Җв”Җ
class StockScreener {
  screen(stocks: StockData[], expression: Expression): StockData[] {
    const results: StockData[] = [];
    for (const stock of stocks) {
      if (expression.interpret(stock)) {
        results.push(stock);
      }
    }
    return results;
  }
}
```

### Language-Specific Notes

- **OOP concepts** вҖ” TypeScript's `interface` keyword provides first-class support for the abstraction participant. The `Expression` interface is the contract, and `implements Expression` on each concrete class enforces that all methods are present at compile time.
- **SOLID in practice** вҖ” TypeScript's structural type system means any object with a compatible `interpret()` method automatically satisfies the `Expression` interface (duck typing at the type level). This makes DIP and LSP compile-time enforced.
- **Idiomatic patterns** вҖ” The `private` keyword enforces encapsulation. Readonly `StockData` is an `interface` rather than a class, idiomatic for pure data objects in TypeScript. The `forEach` method with arrow functions provides clean iteration.
- **Type safety** вҖ” The `StockData[]` return type from `screen()` ensures consumers know the shape of results. The `number` type for price and volume prevents accidental string concatenation bugs.
