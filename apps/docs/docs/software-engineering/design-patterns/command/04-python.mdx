## Implementation

Below is a complete Python implementation of the Command pattern using a stock market trading domain. A `Broker` accepts buy and sell orders for stocks, executes them, supports cancellation (undo), and maintains a complete order history. Python's ABC module provides the abstract command interface, and dataclasses make the concrete command implementations concise and readable.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌──────────────────────────────────────────┐
//   │   class Order(ABC)                       │
//   │   ────────────────────────────────────── │
//   │   @abstractmethod                        │
//   │   execute()                              │ ← OOP: Abstraction
//   │   @abstractmethod                        │    ISP: minimal contract
//   │   undo()                                 │
//   └──────────────────┬───────────────────────┘
//                      ▲ extends
//   ┌──────────────────┴───────────────────────┐
//   │                                          │
//   ┌──────────────────┐  ┌──────────────────┐  ┌──────────────────┐
//   │   BuyOrder       │  │   SellOrder      │  │  CancelOrder     │
//   ├──────────────────┤  ├──────────────────┤  ├──────────────────┤
//   │ @dataclass       │  │ @dataclass       │  │ @dataclass       │
//   │ + execute()      │  │ + execute()      │  │ + execute()      │ ← OOP: Polymorphism
//   │   calls buy()    │  │   calls sell()   │  │   calls undo()   │    LSP: substitutable
//   │ + undo()         │  │ + undo()         │  │ + undo()         │    OCP: new order types
//   │   calls sell()   │  │   calls buy()    │  │   calls exec()   │
//   └────────┬─────────┘  └────────┬─────────┘  └────────┬─────────┘
//            │                     │                     │
//            ▼                     ▼                     ▼
//   ┌────────────────────────────────────────────────────────┐
//   │   class Stock (Receiver)                               │
//   ├────────────────────────────────────────────────────────┤
//   │   buy(quantity, price)                                 │ ← SRP: one responsibility
//   │   sell(quantity, price)                                │
//   │   get_info()                                           │
//   └────────────────────────────────────────────────────────┘
//
//   ┌────────────────────────────────────────────────────────┐
//   │   class Broker (Invoker)                               │
//   ├────────────────────────────────────────────────────────┤
//   │   order_history: list[Order]                           │ ← SRP: manages order lifecycle
//   │   place_order(order: Order)                            │    DIP: depends on Order
//   │   cancel_last_order()                                  │     abstraction, not
//   │   show_history()                                       │     concrete command types
//   └────────────────────────────────────────────────────────┘
```

#### OOP Concepts Applied

1. **Abstraction** — The `Order` ABC defines the abstract contract. Clients depend on `Order`, not on concrete command types.
2. **Encapsulation** — Each command hides its internal state. The `Stock` class encapsulates position data behind `buy()`, `sell()`, and `get_info()` methods.
3. **Inheritance** — `BuyOrder`, `SellOrder`, and `CancelOrder` all extend `Order` and implement the abstract methods.
4. **Polymorphism** — The `Broker` calls `order.execute()` on any `Order` reference. The correct implementation dispatches per type without isinstance checks.

#### SOLID Principles Applied

5. **SRP** — `Broker` only manages the order lifecycle. `Stock` only tracks position. Each command class encapsulates one trade type.
6. **OCP** — Adding a new `LimitOrder` command requires zero changes to `Broker` or `Stock`. Just extend `Order` and implement the methods.
7. **LSP** — Every concrete command type can substitute for `Order` without breaking the `Broker`.
8. **ISP** — The `Order` ABC has only two abstract methods. No class implements methods it does not need.
9. **DIP** — `Broker.place_order` accepts `Order` — an abstraction, not a concrete type.

```python
# Command Pattern — Python — Stock Market example

#               (ConcreteCommand), Stock (Receiver), Broker (Invoker)

# SOLID: SRP, OCP, LSP, ISP, DIP
# ================================================

from abc import ABC, abstractmethod
from dataclasses import dataclass, field
from typing import List
```

### Order (Command Interface)

```python
class Order(ABC):
    @abstractmethod
    def execute(self) -> None:
        ...

    @abstractmethod
    def undo(self) -> None:
        ...
```

### Stock (Receiver)

```python
@dataclass
class Stock:
    ticker: str
    price: float
    shares_outstanding: int

    def buy(self, quantity: int, price: float) -> None:
        self.shares_outstanding += quantity
        self.price = price
        print(f"Bought {quantity} shares of {self.ticker} at ${price:.2f}")

    def sell(self, quantity: int, price: float) -> None:
        if self.shares_outstanding >= quantity:
            self.shares_outstanding -= quantity
            self.price = price
            print(f"Sold {quantity} shares of {self.ticker} at ${price:.2f}")
        else:
            print(f"Insufficient shares of {self.ticker} to sell {quantity}")

    def get_info(self) -> str:
        return (
            f"{self.ticker}: ${self.price:.2f}/share, "
            f"{self.shares_outstanding} shares outstanding"
        )
```

### BuyOrder (ConcreteCommand)

```python
@dataclass
class BuyOrder(Order):
    stock: Stock
    quantity: int
    price: float

    def execute(self) -> None:
        self.stock.buy(self.quantity, self.price)

    def undo(self) -> None:
        self.stock.sell(self.quantity, self.price)
```

### SellOrder (ConcreteCommand)

```python
@dataclass
class SellOrder(Order):
    stock: Stock
    quantity: int
    price: float

    def execute(self) -> None:
        self.stock.sell(self.quantity, self.price)

    def undo(self) -> None:
        self.stock.buy(self.quantity, self.price)
```

### CancelOrder (ConcreteCommand)

```python
@dataclass
class CancelOrder(Order):
    target_order: Order
    was_executed: bool = False

    def execute(self) -> None:
        self.was_executed = True
        self.target_order.undo()
        print("CancelOrder executed — original order undone")

    def undo(self) -> None:
        if self.was_executed:
            self.target_order.execute()
            print("CancelOrder undone — original order re-applied")
```

### Broker (Invoker)

```python
class Broker:
    def __init__(self) -> None:
        self.order_history: List[Order] = []

    def place_order(self, order: Order) -> None:
        order.execute()
        self.order_history.append(order)
        print("Order recorded in history\n")

    def cancel_last_order(self) -> None:
        if not self.order_history:
            print("No orders to cancel\n")
            return
        last_order = self.order_history.pop()
        cancel_cmd = CancelOrder(last_order)
        cancel_cmd.execute()
        self.order_history.append(cancel_cmd)
        print("Cancellation recorded in history\n")

    def show_history(self) -> None:
        print("=== Order History ===")
        if not self.order_history:
            print("No orders in history")
            return
        for idx, order in enumerate(self.order_history, 1):
            type_name = type(order).__name__
            if isinstance(order, CancelOrder):
                details = (
                    f"CancelOrder targeting "
                    f"{type(order.target_order).__name__}"
                )
            else:
                details = (
                    f"{order.stock.ticker} x{order.quantity} "
                    f"@ ${order.price:.2f}"
                )
            print(f"  {idx}. {type_name}: {details}")
        print("")
```

### Main / Driver

```python
if __name__ == "__main__":
    print("=== Trading Session Started ===\n")

    aapl = Stock("AAPL", 175.50, 1000)
    googl = Stock("GOOGL", 141.25, 800)
    msft = Stock("MSFT", 378.90, 1200)

    broker = Broker()

    buy1 = BuyOrder(aapl, 100, 175.50)
    broker.place_order(buy1)

    sell1 = SellOrder(googl, 50, 141.25)
    broker.place_order(sell1)

    buy2 = BuyOrder(msft, 200, 378.90)
    broker.place_order(buy2)

    broker.show_history()

    print("=== Cancelling Last Order ===\n")
    broker.cancel_last_order()

    broker.show_history()

    print("=== Current Stock Positions ===")
    print(aapl.get_info())
    print(googl.get_info())
    print(msft.get_info())
```

### Language-Specific Notes

- **ABC module** — Python's `abc.ABC` and `@abstractmethod` provide the abstract base class mechanism. This enforces that subclasses implement `execute()` and `undo()`. Attempting to instantiate `Order` directly raises `TypeError`.
- **`@dataclass`** — The dataclass decorator auto-generates `__init__`, `__repr__`, `__eq__`, and `__hash__` methods. This makes the command classes concise and eliminates boilerplate.
- **Type hints** — All method signatures use type hints (`-> None`, `-> str`, `List[Order]`). These are optional at runtime but provide IDE support and static type checking with `mypy`.
- **`isinstance` for type checking** — The `show_history` method uses `isinstance(order, CancelOrder)` to distinguish command types. In more complex systems, you might add a `description` property to the `Order` ABC to avoid type checks entirely.
- **`__name__` guard** — The `if __name__ == "__main__":` guard ensures the driver code only runs when the file is executed directly, not when imported as a module.
