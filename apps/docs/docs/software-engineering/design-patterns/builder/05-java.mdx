The Builder pattern separates construction of a complex `LoanApplication` from its representation so the same process can produce different configurations. In this Java 17+ Banking example, the `LoanApplicationBuilder` interface defines steps for setting personal info, loan terms, interest rate, and documents. Three concrete builders — `PersonalLoanBuilder`, `MortgageBuilder`, and `AutoLoanBuilder` — implement those steps. The `LoanDirector` orchestrates the full construction sequence, calling builder methods in order and returning a ready-to-use `LoanApplication` record.

## How the Implementation Works

The demo program constructs three `LoanApplication` objects — a personal loan, a mortgage, and an auto loan — using the same `LoanDirector` with different builders. Each concrete builder tracks only the data relevant to its loan type. The director controls the construction sequence, ensuring every required field is populated before calling `getResult()`. Product immutability is enforced by making `LoanApplication` a Java `record`.

### Class Diagram (Pseudocode)

The `LoanApplication` product is a Java `record` with components for personal info, loan terms, interest rate, optional appraisal/vehicle info, a document list, and an application date. The `LoanApplicationBuilder` interface declares fluent setter methods that return the builder itself for chaining. Three concrete builders implement the interface, each adding type-specific fields (e.g., `MortgageBuilder` adds `setPropertyInfo()`). The `LoanDirector` holds a reference to a `LoanApplicationBuilder` and provides three construction methods, one per loan type, each calling the appropriate sequence of builder steps.

### OOP Concepts Applied

- **Abstraction** — The `LoanApplicationBuilder` interface hides how each loan type is assembled. Clients interact only with the abstract step methods.
- **Encapsulation** — Each concrete builder encapsulates its own loan parameters and internal state. No other class can modify the builder's fields directly.
- **Inheritance** — The three concrete builders implement the `LoanApplicationBuilder` interface, inheriting the method contract and the default `withDefaultDocuments()` method.
- **Polymorphism** — The `LoanDirector` operates on any `LoanApplicationBuilder` through the interface reference, enabling the same director to build different loan types.

### SOLID Principles Applied

- **Single Responsibility Principle** — Each builder class handles exactly one loan type. The director is responsible only for orchestrating the build sequence.
- **Open/Closed Principle** — New loan types can be added by implementing the `LoanApplicationBuilder` interface without modifying existing builders or the director.
- **Liskov Substitution Principle** — Any `LoanApplicationBuilder` can be passed to the `LoanDirector` without altering the correctness of the construction process.
- **Interface Segregation Principle** — The builder interface is minimal and focused on loan construction steps. No client depends on methods it does not use.
- **Dependency Inversion Principle** — The `LoanDirector` depends on the abstract `LoanApplicationBuilder` interface, not on concrete builder classes.

### LoanApplication (Product)
```java
// ── Product (immutable record) ─────────────────────────────────────────────

record LoanApplication(
    PersonalInfo personalInfo,
    String loanType,
    LoanTerms loanTerms,
    double interestRate,
    Appraisal appraisal,
    VehicleInfo vehicleInfo,
    String dealership,
    double downPayment,
    List<String> documents,
    LocalDate applicationDate
) {
    String formatAmount() {
        return String.format("$%.2f", loanTerms.amount());
    }

    String formatRate() {
        return String.format("%.2f%%", interestRate * 100.0);
    }

    String applicantName() {
        return personalInfo.firstName() + " " + personalInfo.lastName();
    }

    double calculateLTV() {
        if (appraisal == null || appraisal.appraisedValue() == 0.0) {
            return 0.0;
        }
        return (loanTerms.amount() - downPayment) / appraisal.appraisedValue() * 100.0;
    }

    String display() {
        var sb = new StringBuilder();
        if ("Personal".equals(loanType)) {
            sb.append("=== Personal Loan Application ===\n");
            sb.append(String.format(
                "Applicant: %s | Loan: %s over %dmo @ %s APR\n",
                applicantName(), formatAmount(), loanTerms.termMonths(), formatRate()));
            sb.append("Documents: ").append(String.join(", ", documents));
        } else if ("Mortgage".equals(loanType)) {
            sb.append("=== Mortgage Application ===\n");
            sb.append(String.format(
                "Applicant: %s | Loan: %s over %dmo @ %s APR\n",
                applicantName(), formatAmount(), loanTerms.termMonths(), formatRate()));
            sb.append(String.format(
                "Property: %s (Appraised: $%.2f)\n",
                appraisal.propertyAddress(), appraisal.appraisedValue()));
            sb.append(String.format(
                "Down Payment: $%.2f | LTV: %.2f%%\n", downPayment, calculateLTV()));
            sb.append("Documents: ").append(String.join(", ", documents));
        } else if ("Auto".equals(loanType)) {
            sb.append("=== Auto Loan Application ===\n");
            sb.append(String.format(
                "Applicant: %s | Loan: %s over %dmo @ %s APR\n",
                applicantName(), formatAmount(), loanTerms.termMonths(), formatRate()));
            sb.append(String.format(
                "Vehicle: %d %s %s (VIN: %s, Value: $%.2f)\n",
                vehicleInfo.year(), vehicleInfo.make(), vehicleInfo.model(),
                vehicleInfo.vin(), vehicleInfo.value()));
            sb.append("Dealership: ").append(dealership).append("\n");
            sb.append("Documents: ").append(String.join(", ", documents));
        }
        return sb.toString();
    }
}
```

### Builder Interface
```java
// ── Builder Interface ─────────────────────────────────────────────────────

interface LoanApplicationBuilder {
    LoanApplicationBuilder setPersonalInfo(PersonalInfo info);
    LoanApplicationBuilder setLoanTerms(LoanTerms terms);
    LoanApplicationBuilder setInterestRate(double rate);
    LoanApplicationBuilder addDocument(String type, String description);
    LoanApplication getResult();

    default LoanApplicationBuilder withDefaultDocuments() {
        addDocument("ID proof", "ID");
        return this;
    }
}
```

### PersonalLoanBuilder (ConcreteBuilder)
```java
// ── Concrete Builder: PersonalLoan ────────────────────────────────────────

class PersonalLoanBuilder implements LoanApplicationBuilder {
    private PersonalInfo personalInfo;
    private LoanTerms loanTerms;
    private double interestRate;
    private final List<String> documents = new ArrayList<>();

    @Override
    public LoanApplicationBuilder setPersonalInfo(PersonalInfo info) {
        this.personalInfo = info;
        return this;
    }

    @Override
    public LoanApplicationBuilder setLoanTerms(LoanTerms terms) {
        this.loanTerms = terms;
        return this;
    }

    @Override
    public LoanApplicationBuilder setInterestRate(double rate) {
        this.interestRate = rate;
        return this;
    }

    @Override
    public LoanApplicationBuilder addDocument(String type, String description) {
        documents.add(description);
        return this;
    }

    @Override
    public LoanApplication getResult() {
        return new LoanApplication(
            personalInfo, "Personal", loanTerms, interestRate,
            null, null, null, 0.0,
            List.copyOf(documents), LocalDate.now()
        );
    }
}
```

### LoanDirector (Director)
```java
// ── Director ──────────────────────────────────────────────────────────────

class LoanDirector {
    private LoanApplicationBuilder builder;

    public LoanDirector(LoanApplicationBuilder builder) {
        this.builder = builder;
    }

    public void changeBuilder(LoanApplicationBuilder builder) {
        this.builder = builder;
    }

    public LoanApplication constructPersonalLoan(
            PersonalInfo info, double amount, int term, double rate) {
        return builder
            .setPersonalInfo(info)
            .setLoanTerms(new LoanTerms(amount, term))
            .setInterestRate(rate)
            .withDefaultDocuments()
            .addDocument("Income", "Income Verification")
            .getResult();
    }

    public LoanApplication constructMortgage(
            PersonalInfo info, double amount, int term, double rate,
            Appraisal appraisal, double downPayment) {
        if (!(builder instanceof MortgageBuilder mb)) {
            throw new IllegalStateException("MortgageBuilder required");
        }
        mb.setPropertyInfo(appraisal, downPayment);
        return builder
            .setPersonalInfo(info)
            .setLoanTerms(new LoanTerms(amount, term))
            .setInterestRate(rate)
            .withDefaultDocuments()
            .addDocument("Tax", "Tax Returns")
            .addDocument("Appraisal", "Appraisal Report")
            .getResult();
    }

    public LoanApplication constructAutoLoan(
            PersonalInfo info, double amount, int term, double rate,
            VehicleInfo vehicleInfo, String dealership) {
        if (!(builder instanceof AutoLoanBuilder alb)) {
            throw new IllegalStateException("AutoLoanBuilder required");
        }
        alb.setVehicleInfo(vehicleInfo);
        alb.setDealership(dealership);
        return builder
            .setPersonalInfo(info)
            .setLoanTerms(new LoanTerms(amount, term))
            .setInterestRate(rate)
            .withDefaultDocuments()
            .addDocument("Vehicle", "Vehicle Registration")
            .getResult();
    }
}
```

### Main / Driver
```java
// ── Driver ─────────────────────────────────────────────────────────────────

public class BuilderPatternDemo {
    public static void main(String[] args) {
        var director = new LoanDirector(new PersonalLoanBuilder());

        // Alice Johnson — Personal Loan
        var alice = new PersonalInfo("Alice", "Johnson", "alice@example.com", "555-0101");
        var personalLoan = director.constructPersonalLoan(alice, 15000.0, 60, 0.1199);
        System.out.println(personalLoan.display());
        System.out.println();

        // Bob Martinez — Mortgage
        director.changeBuilder(new MortgageBuilder());
        var bob = new PersonalInfo("Bob", "Martinez", "bob@example.com", "555-0102");
        var appraisal = new Appraisal("123 Main St, Springfield", 450000.0);
        var mortgage = director.constructMortgage(bob, 350000.0, 360, 0.0625, appraisal, 70000.0);
        System.out.println(mortgage.display());
        System.out.println();

        // Carol Chen — Auto Loan
        director.changeBuilder(new AutoLoanBuilder());
        var carol = new PersonalInfo("Carol", "Chen", "carol@example.com", "555-0103");
        var vehicle = new VehicleInfo("Honda", "Accord", 2024,
            "1HGCV1F34PA012345", 35000.0);
        var autoLoan = director.constructAutoLoan(carol, 32000.0, 60, 0.0749,
            vehicle, "City Motors");
        System.out.println(autoLoan.display());
    }
}
```

### Language-Specific Notes

- **Records for immutable products** — `LoanApplication` is declared as a Java `record`, giving it a canonical constructor, accessor methods, `equals()`/`hashCode()`, and `toString()` automatically. The `List.copyOf()` call in each builder's `getResult()` returns an unmodifiable view, preserving immutability after construction.
- **Interface with default methods** — `withDefaultDocuments()` is a `default` method on the `LoanApplicationBuilder` interface. It calls the abstract `addDocument()` method, so all builders inherit the same default behavior without code duplication.
- **`String.format()` for consistent output** — Monetary values, interest rates, and LTV percentages are formatted with `String.format()` using `$%.2f` and `%.2f%%` patterns, guaranteeing uniform precision across all three loan types.
- **`var` for local type inference** — Local variables in the `main` method use `var` (e.g., `var director = new LoanDirector(...)`) to reduce syntactic noise. The compiler infers the static type from the initializer.
- **Instanceof pattern matching** — The director methods `constructMortgage()` and `constructAutoLoan()` use Java 17's pattern matching for `instanceof` (`builder instanceof MortgageBuilder mb`) to safely cast the builder to its concrete type for type-specific method calls.
- **Optional fields via null** — Because `LoanApplication` is a single record type, mortgage- and auto-specific fields (`appraisal`, `vehicleInfo`, `dealership`, `downPayment`) are set to `null` or `0.0` when they do not apply. An alternative design could use a sealed interface hierarchy or `Optional` wrappers.
