## Implementation

Below is a complete C# implementation of the Decorator pattern using a car dealership domain. C# uses an `interface` for the `Car` contract and an `abstract class` for the `CarFeature` decorator. The implementation leverages C# features such as expression-bodied members, nullable reference types, and string interpolation for clean, idiomatic code. The `Decimal` type is used for cost to avoid floating-point precision issues in financial calculations.

### How the Implementation Works

#### Class Diagram (Pseudocode)

#### OOP Concepts Applied

1. **Abstraction** — The `ICar` interface defines the abstract contract. Both `BaseCar` and all decorator classes implement `ICar`. Client code (the driver) depends on `ICar` and never references concrete types for behavioral operations.

2. **Polymorphism** — The driver stores a collection of `ICar` references. Each reference points to a different decorated variant, and the runtime type determines the actual behavior of `GetDescription()` and `GetCost()`.

3. **Composition over Inheritance** — `CarFeature` holds a reference to an `ICar` and delegates to it. Features are combined by composing objects at runtime, not by creating a hierarchy of subclasses.

#### SOLID Principles Applied

4. **SRP** — `BaseCar` handles only the base vehicle model and price. Each decorator handles exactly one optional feature. The driver handles configuration and display.

5. **OCP** — A new feature is added by creating a new concrete decorator class. No changes to `BaseCar`, `CarFeature`, or existing decorators are required.

6. **LSP** — Decorators maintain the `ICar` contract. Any decorated car can substitute for a `BaseCar` in code that expects `ICar`.

7. **ISP** — The `ICar` interface is minimal with two methods. No client is forced to depend on methods it does not use.

8. **DIP** — High-level modules depend on the `ICar` interface. `BaseCar` and decorators implement `ICar`. The driver constructs concrete instances but interacts with them through `ICar` references.

### Component interface

```csharp
public interface ICar
{
    string GetDescription();
    decimal GetCost();
}
```

### ConcreteComponent

```csharp
public class BaseCar : ICar
{
    private readonly string _model;
    private readonly decimal _baseCost;

    public BaseCar(string model, decimal baseCost)
    {
        _model = model;
        _baseCost = baseCost;
    }

    public string GetDescription() => _model;

    public decimal GetCost() => _baseCost;
}
```

### Decorator (Abstract)

```csharp
public abstract class CarFeature : ICar
{
    protected ICar Car { get; }

    protected CarFeature(ICar car)
    {
        Car = car ?? throw new ArgumentNullException(nameof(car));
    }

    public virtual string GetDescription() => Car.GetDescription();

    public virtual decimal GetCost() => Car.GetCost();
}
```

### ConcreteDecorator 1: LeatherSeats

```csharp
public class LeatherSeats : CarFeature
{
    private const decimal FeatureCost = 1500.00m;
    private const string FeatureName = "Leather Seats";

    public LeatherSeats(ICar car) : base(car) { }

    public override string GetDescription() =>
        $"{base.GetDescription()} + {FeatureName}";

    public override decimal GetCost() =>
        base.GetCost() + FeatureCost;
}
```

### Main / Driver

```csharp
public class Program
{
    public static void Main()
    {
        Console.WriteLine("=== C# Car Dealership Configuration System ===\n");

        // Build configurations
        ICar baseCamry = new BaseCar("Toyota Camry", 28000.00m);
        ICar withLeather = new LeatherSeats(baseCamry);
        ICar withSunroof = new Sunroof(baseCamry);
        ICar withLeatherSunroof = new Sunroof(new LeatherSeats(baseCamry));
        ICar ecoLuxury = new NavigationSystem(new LeatherSeats(baseCamry));
        ICar sportOnly = new SportPackage(baseCamry);
        ICar fullyLoaded = new SportPackage(
            new NavigationSystem(
                new Sunroof(
                    new LeatherSeats(baseCamry)
                )
            )
        );

        var configs = new Dictionary<string, ICar>
        {
            ["Base Model"] = baseCamry,
            ["With Leather"] = withLeather,
            ["With Sunroof"] = withSunroof,
            ["Leather + Sunroof"] = withLeatherSunroof,
            ["Eco Luxury"] = ecoLuxury,
            ["Sport Only"] = sportOnly,
            ["Fully Loaded"] = fullyLoaded,
        };

        // Print header
        Console.WriteLine($"{"Config",-18} {"Description",-60} {"Price",10}");
        Console.WriteLine(new string('-', 90));

        // Print each configuration
        foreach (var kvp in configs)
        {
            Console.WriteLine($"  {kvp.Key,-16} {kvp.Value.GetDescription(),-60} {kvp.Value.GetCost(),10:C2}");
        }

        // Price breakdown
        Console.WriteLine("\n--- Price Breakdown: Fully Loaded Camry ---");
        Console.WriteLine($"  Base car:                       {baseCamry.GetCost(),10:C2}");
        Console.WriteLine($"  + Leather Seats:                {1500.00m,10:C2}");
        Console.WriteLine($"  + Sunroof:                      {1200.00m,10:C2}");
        Console.WriteLine($"  + Navigation System:            {2000.00m,10:C2}");
        Console.WriteLine($"  + Sport Package:                {3500.00m,10:C2}");
        Console.WriteLine("  -----------------------------------------");
        Console.WriteLine($"  Total:                          {fullyLoaded.GetCost(),10:C2}");

        // Interface compliance
        Console.WriteLine("\n--- Interface Compliance ---");
        int i = 1;
        foreach (var kvp in configs)
        {
            Console.WriteLine($"  Config {i}: ICar interface = {kvp.Value is ICar}");
            i++;
        }

        // Reuse with Honda Accord
        Console.WriteLine("\n--- Honda Accord ---");
        ICar baseAccord = new BaseCar("Honda Accord", 27000.00m);
        ICar accordLuxury = new NavigationSystem(new LeatherSeats(baseAccord));
        Console.WriteLine($"  {accordLuxury.GetDescription()}");
        Console.WriteLine($"  Total: {accordLuxury.GetCost(),10:C2}");

        // Demonstrate order independence
        Console.WriteLine("\n--- Order Independence ---");
        ICar order1 = new Sunroof(new LeatherSeats(baseCamry));
        ICar order2 = new LeatherSeats(new Sunroof(baseCamry));
        Console.WriteLine($"  Sunroof(Leather): {order1.GetDescription()} — {order1.GetCost():C2}");
        Console.WriteLine($"  Leather(Sunroof): {order2.GetDescription()} — {order2.GetCost():C2}");
        Console.WriteLine($"  Same price: {order1.GetCost() == order2.GetCost()}");

        // Demonstrate null guard
        Console.WriteLine("\n--- Null Guard Verification ---");
        try
        {
            var invalid = new LeatherSeats(null!);
        }
        catch (ArgumentNullException)
        {
            Console.WriteLine("  CarFeature constructor rejects null: OK");
        }
    }
}
```

### Language-Specific Notes

- **`decimal` type for financial calculations** — C# provides the `decimal` keyword (128-bit precision) which avoids the floating-point rounding errors inherent in `float` and `double`. For a car dealership system where cost calculations must be exact, `decimal` is the appropriate choice. The `m` suffix (e.g., `28000.00m`) creates a decimal literal.

- **Expression-bodied members** — Simple methods such as `GetDescription() => _model` use C#'s expression-bodied member syntax (`=>`). This reduces boilerplate for single-expression methods while remaining fully compatible with the decorator pattern's virtual method dispatch.

- **Null guard in constructor** — The `CarFeature` abstract class constructor checks `car ?? throw new ArgumentNullException(nameof(car))`. This uses C#'s null-coalescing operator (`??`) to throw immediately if null is passed, preventing a `NullReferenceException` later in the decorator chain. This is a C# idiom for defensive programming.

- **`protected` property with getter** — Instead of a `protected` field, `CarFeature` exposes the wrapped car as a `protected ICar Car { get; }` auto-property with a getter only. This is more idiomatic C# than a bare field, as it encapsulates the field and can be overridden in subclasses if needed.

- **`const` vs `static readonly`** — Feature costs are declared as `private const decimal FeatureCost = 1500.00m`. Constants in C# are compile-time literals, which is appropriate for fixed feature prices. If prices were loaded from configuration at runtime, `private static readonly` would be used instead.

- **String interpolation with `$""`** — C#'s string interpolation (`$"{base.GetDescription()} + {FeatureName}"`) provides a readable alternative to concatenation. The interpolated string is compiled to a `string.Format` call, with no runtime overhead beyond the formatting itself.
