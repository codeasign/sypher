Rust implements the Adapter pattern through traits and struct composition. Unlike class-based languages, Rust uses traits to define shared behavior without inheritance. In the car domain, a Rust adapter implements the `CCSCharger` trait for a `TeslaToCCSAdapter` struct that wraps a `TeslaSupercharger`, enabling CCS-compatible charging station code to charge Tesla vehicles through zero-cost trait dispatch.

## How the Implementation Works

| Aspect | Implementation |
|---|---|
| **Target** | `trait CCSCharger` with `fn charge_dc(&self, power_kw: u32) -> String` |
| **Adaptee** | `struct TeslaSupercharger` with `fn send_charge(&self, voltage: u32, amperage: u32) -> String` |
| **Adapter** | `struct TeslaToCCSAdapter` that `impl CCSCharger for TeslaToCCSAdapter`, wraps `TeslaSupercharger` as a field |
| **Client** | `ChargingStation` method accepts `&impl CCSCharger` (static dispatch) or `&dyn CCSCharger` (dynamic dispatch) |
| **Ownership** | Adapter takes ownership of the adaptee; no borrowed references or lifetime annotations needed |
| **Dispatch** | Static dispatch via generics (zero-cost) or dynamic dispatch via trait objects |

### Target Trait

```rust
use std::f64::consts;

// ── Target Trait ──
/// The CCS (Combined Charging System) standard interface.
/// Any charger that implements this trait can communicate
/// with a CCS-compatible charging station.
trait CCSCharger {
    /// Deliver DC power at the requested kilowatt level.
    ///
    /// # Arguments
    /// * `power_kw` - Charging power in kilowatts (1 to 1000).
    ///
    /// # Returns
    /// A human-readable charging confirmation string.
    ///
    /// # Panics
    /// Panics if `power_kw` is 0 or exceeds 1000.
    fn charge_dc(&self, power_kw: u32) -> String;
}
```

### Adaptee

```rust
// ── Adaptee ──
/// Tesla's proprietary Supercharger hardware.
/// Uses a non-standard interface that expects voltage and amperage
/// separately, rather than the CCS convention of specifying power in kilowatts.
struct TeslaSupercharger;

impl TeslaSupercharger {
    /// Initiate a charge using Tesla's native protocol.
    ///
    /// # Arguments
    /// * `voltage` - Charging voltage in volts (typically 400 or 800).
    /// * `amperage` - Charging current in amperes.
    ///
    /// # Returns
    /// A confirmation from the Tesla charging hardware.
    fn send_charge(&self, voltage: u32, amperage: u32) -> String {
        let power_kw = (voltage * amperage) / 1000;
        format!(
            "Tesla Supercharger delivering {} kW at {}V / {}A via Tesla connector",
            power_kw, voltage, amperage
        )
    }
}
```

### Adapter

```rust
// ── Adapter ──
/// Adapts a TeslaSupercharger to the CCSCharger trait.
/// Tesla vehicles can now be charged at any CCS-compatible station.
///
/// Translation logic:
/// - Power < 350 kW uses 400V nominal (standard CCS).
/// - Power >= 350 kW uses 800V nominal (ultra-fast CCS).
/// - Amperage is derived from I = P / V, rounded up for safety.
struct TeslaToCCSAdapter {
    tesla_charger: TeslaSupercharger,
}

impl TeslaToCCSAdapter {
    /// Create a new adapter wrapping the given Tesla hardware.
    fn new(tesla_charger: TeslaSupercharger) -> Self {
        TeslaToCCSAdapter { tesla_charger }
    }
}

impl CCSCharger for TeslaToCCSAdapter {
    fn charge_dc(&self, power_kw: u32) -> String {
        assert!(
            power_kw > 0 && power_kw <= 1000,
            "Charging power must be between 1 and 1000 kW, got {}",
            power_kw
        );

        // Select voltage tier based on power requirement.
        let voltage: u32 = if power_kw > 350 { 800 } else { 400 };
        // Calculate required amperage: I = P / V (rounded up).
        let amperage: u32 = ((power_kw as f64 * 1000.0) / voltage as f64).ceil() as u32;

        println!(
            "  [Adapter] CCS {} kW -> Tesla {}V / {}A",
            power_kw, voltage, amperage
        );

        self.tesla_charger.send_charge(voltage, amperage)
    }
}
```

### Client

```rust
// ── Client ──
/// A public CCS charging station.
/// Accepts any CCSCharger implementation via dynamic dispatch.
struct ChargingStation {
    name: String,
    location: String,
}

impl ChargingStation {
    /// Create a new charging station.
    fn new(name: &str, location: &str) -> Self {
        ChargingStation {
            name: name.to_string(),
            location: location.to_string(),
        }
    }

    /// Service an electric vehicle at this station.
    ///
    /// # Arguments
    /// * `charger` - Any object that implements the CCSCharger trait.
    /// * `car_model` - The vehicle make and model for the log.
    /// * `required_kw` - The power the vehicle can accept in kilowatts.
    ///
    /// # Returns
    /// A summary string of the charging session.
    fn service_vehicle(
        &self,
        charger: &dyn CCSCharger,
        car_model: &str,
        required_kw: u32,
    ) -> String {
        println!(
            "[{}] {} at {}",
            chrono_now(),
            self.name,
            self.location
        );
        println!("  Vehicle: {}", car_model);
        println!("  Request: {} kW", required_kw);

        // The adapter's charge_dc is called through dynamic dispatch.
        let result = charger.charge_dc(required_kw);
        println!("  Result:  {}", result);
        println!("  Status:  Charging session completed successfully.");
        result
    }
}

/// Get a simple UTC timestamp string without pulling in the chrono crate.
fn chrono_now() -> String {
    use std::time::{SystemTime, UNIX_EPOCH};
    let duration = SystemTime::now()
        .duration_since(UNIX_EPOCH)
        .unwrap_or_default();
    let secs = duration.as_secs();
    let millis = duration.subsec_millis();

    // Convert to ISO-like format (approximate UTC).
    let days = secs / 86400;
    let time_secs = secs % 86400;
    let hours = time_secs / 3600;
    let minutes = (time_secs % 3600) / 60;
    let seconds = time_secs % 60;

    // A simple day counter relative to the Unix epoch.
    let year = 1970;
    let month = 1;
    let day = 1 + days as u16;

    format!(
        "{:04}-{:02}-{:02}T{:02}:{:02}:{:02}.{:03}Z",
        year, month, day, hours, minutes, seconds, millis
    )
}
```

### Main / Driver

```rust
// ── Driver ──
fn main() {
    println!("{}", "=".repeat(60));
    println!("ADAPTER PATTERN — Rust Tesla to CCS Demo");
    println!("{}", "=".repeat(60));
    println!();

    // Create the adaptee (Tesla's proprietary charging hardware).
    let tesla_hardware = TeslaSupercharger;

    // Wrap it in the adapter so it speaks the CCS language.
    let adapter = TeslaToCCSAdapter::new(tesla_hardware);

    // Create a CCS charging station.
    let station = ChargingStation::new(
        "Shell Recharge",
        "Love's Travel Stop, Las Vegas, NV",
    );

    // Define a fleet of Tesla vehicles needing a charge.
    let vehicles = vec![
        ("Tesla Model 3 Standard Range", 170u32),
        ("Tesla Model 3 Long Range", 250u32),
        ("Tesla Model Y Performance", 250u32),
        ("Tesla Model S Plaid", 200u32),
        ("Tesla Cybertruck", 350u32),
        ("Tesla Semi", 750u32),
    ];

    // Service each vehicle through the adapter.
    let mut session_logs: Vec<String> = Vec::new();
    for (model, power_kw) in &vehicles {
        println!("--- {} ({} kW) ---", model, power_kw);
        let log = station.service_vehicle(&adapter, model, *power_kw);
        session_logs.push(log);
        println!();
    }

    println!("{}", "=".repeat(60));
    println!("SESSION SUMMARY");
    println!("{}", "=".repeat(60));
    let success_count = session_logs.len();
    println!("Vehicles serviced: {}", vehicles.len());
    println!("Successful charges: {}", success_count);
    println!("All Tesla vehicles charged through the TeslaToCCSAdapter.");
}
```

## Language-Specific Notes

- Rust uses traits instead of interfaces or abstract classes. A trait defines a set of method signatures that types can implement. The `CCSCharger` trait is implemented for `TeslaToCCSAdapter` via `impl CCSCharger for TeslaToCCSAdapter`. There is no inheritance — the adapter is a plain struct that happens to implement the trait.

- The adapter struct owns the adaptee (`tesla_charger: TeslaSupercharger`) rather than holding a reference or a boxed trait object. Ownership means the adapter is responsible for the adaptee's lifetime. When the adapter goes out of scope, the adaptee is dropped automatically. This is the simplest and most performant arrangement.

- Dynamic dispatch via `&dyn CCSCharger` in the client method enables runtime polymorphism without generics. The `dyn` keyword indicates a trait object, and the `&` reference ensures the client does not take ownership. An alternative design uses generics: `fn service_vehicle<T: CCSCharger>(...)`, which compiles to static dispatch with zero runtime overhead but creates a separate monomorphized function for each concrete type.

- The `assert!` macro in `charge_dc` provides runtime validation. In a production Rust system, a `Result` return type might be preferred to handle errors without panicking. The assert is appropriate here because the values come from the driver code and invalid values indicate a programming error.

- The `chrono_now()` function is a manual timestamp implementation that avoids pulling in the `chrono` or `time` external crates. Real Rust projects typically use the `chrono` crate for production date-time handling, but this example keeps dependencies to zero by computing an approximate UTC timestamp from `SystemTime`.

- The `format!` macro in `send_charge` performs string interpolation at compile time, producing a heap-allocated `String`. Rust's `format!` is type-safe: if the arguments do not match the format specifiers, the program does not compile. This catches string formatting bugs during development rather than at runtime.
