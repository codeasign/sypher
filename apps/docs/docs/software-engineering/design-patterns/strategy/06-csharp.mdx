## Implementation

Below is a complete C# implementation of the Strategy pattern using the Stock Market domain. The `TradingBot` (Context) delegates trade signal generation to a pluggable `ITradingStrategy` interface. Three concrete strategies implement different trading algorithms: `MovingAverageCrossover`, `MomentumStrategy`, and `MeanReversionStrategy`. The implementation uses C# 12 / .NET 8 features including primary constructors, record types, and collection expressions. A fourth strategy (`RsiStrategy`) demonstrates OCP compliance by adding a new algorithm without modifying existing code.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌────────────────────────────────────────────────────┐
//   │ TradingBot (Context)                               │ ← OOP: Encapsulation
//   ├────────────────────────────────────────────────────┤    SRP: orchestrates
//   │ - Strategy: ITradingStrategy                       │     DIP: depends on interface
//   │ + SetStrategy(ITradingStrategy): void              │
//   │ + RunAnalysis(string, List<double>): Signal        │
//   └────────────────┬───────────────────────────────────┘
//                    │ delegates Analyze()
//                    ▼
//   ┌────────────────────────────────────────────────────┐
//   │ <<interface>> ITradingStrategy                     │ ← OOP: Abstraction
//   ├────────────────────────────────────────────────────┤    ISP: single method
//   │ + Analyze(List<double>): Signal                    │
//   └────────────────┬───────────────────────────────────┘
//                    ▲ implements
//   ┌────────────────┴───────────────────────────────────┐
//   │                                                    │
//   ┌────────────────┐ ┌────────────────┐               ┌──────────────────┐
//   │MovingAvg       │ │Momentum        │               │MeanReversion     │
//   │Crossover       │ │Strategy        │               │Strategy          │
//   ├────────────────┤ ├────────────────┤               ├──────────────────┤
//   │+ Analyze()     │ │+ Analyze()     │               │+ Analyze()       │
//   └────────────────┘ └────────────────┘               └──────────────────┘
//       OCP: new = new class, no existing code changed
```

#### OOP Concepts Applied

1. **Abstraction** — `ITradingStrategy` is a C# interface. All client code depends on this interface.
2. **Encapsulation** — Each strategy hides its parameters via `private readonly` fields. `TradingBot` encapsulates the strategy reference.
3. **Polymorphism** — `RunAnalysis()` calls `Strategy.Analyze(prices)`. The runtime dispatches to the correct concrete implementation.

#### SOLID Principles Applied

4. **SRP** — Each class has one responsibility. `TradingBot` orchestrates. Each strategy implements one algorithm.
5. **OCP** — New strategies implement `ITradingStrategy` without changing existing code.
6. **LSP** — Any concrete strategy is substitutable for `ITradingStrategy`.
7. **ISP** — The interface has a single method. No class implements unused methods.
8. **DIP** — `TradingBot` depends on `ITradingStrategy`. Concrete strategies are injected.

### MovingAverageCrossover (Concrete Strategy)

```csharp
// ── Concrete Strategy 1: Moving Average Crossover ──
// OOP role: Polymorphism — implements the strategy contract
// SOLID: SRP (only crossover), LSP (substitutable), OCP (extensible)
public sealed class MovingAverageCrossover : ITradingStrategy
{
    private readonly int _shortWindow;
    private readonly int _longWindow;

    public MovingAverageCrossover(int shortWindow = 20, int longWindow = 50)
    {
        if (shortWindow >= longWindow)
            throw new ArgumentException("shortWindow must be less than longWindow");
        _shortWindow = shortWindow;
        _longWindow = longWindow;
    }

    public Signal Analyze(List<double> prices)
    {
        if (prices.Count < _longWindow)
            return Signal.Hold;

        double shortMA = ComputeAverage(prices, _shortWindow);
        double longMA = ComputeAverage(prices, _longWindow);

        if (shortMA > longMA)
            return Signal.Buy;
        else if (shortMA < longMA)
            return Signal.Sell;
        return Signal.Hold;
    }

    private static double ComputeAverage(List<double> prices, int window)
    {
        double sum = 0;
        for (int i = prices.Count - window; i < prices.Count; i++)
            sum += prices[i];
        return sum / window;
    }
}

```

### MomentumStrategy (Concrete Strategy)

```csharp
// ── Concrete Strategy 2: Momentum Strategy ──
// OOP role: Polymorphism — implements the strategy contract
// SOLID: SRP (only momentum), LSP (substitutable), OCP (extensible)
public sealed class MomentumStrategy : ITradingStrategy
{
    private readonly int _lookbackPeriod;
    private readonly double _threshold;

    public MomentumStrategy(int lookbackPeriod = 10, double threshold = 0.05)
    {
        _lookbackPeriod = lookbackPeriod;
        _threshold = threshold;
    }

    public Signal Analyze(List<double> prices)
    {
        if (prices.Count < _lookbackPeriod + 1)
            return Signal.Hold;

        double currentPrice = prices[^1];
        double pastPrice = prices[^(1 + _lookbackPeriod)];
        double momentum = (currentPrice - pastPrice) / pastPrice;

        if (momentum > _threshold)
            return Signal.Buy;
        else if (momentum < -_threshold)
            return Signal.Sell;
        return Signal.Hold;
    }
}

```

### MeanReversionStrategy (Concrete Strategy)

```csharp
// ── Concrete Strategy 3: Mean Reversion Strategy ──
// OOP role: Polymorphism — implements the strategy contract
// SOLID: SRP (only reversion), LSP (substitutable), OCP (extensible)
public sealed class MeanReversionStrategy : ITradingStrategy
{
    private readonly int _window;
    private readonly double _deviationThreshold;

    public MeanReversionStrategy(int window = 20, double deviationThreshold = 0.03)
    {
        _window = window;
        _deviationThreshold = deviationThreshold;
    }

    public Signal Analyze(List<double> prices)
    {
        if (prices.Count < _window)
            return Signal.Hold;

        double sum = 0;
        for (int i = prices.Count - _window; i < prices.Count; i++)
            sum += prices[i];
        double mean = sum / _window;
        double current = prices[^1];
        double deviation = (current - mean) / mean;

        if (deviation < -_deviationThreshold)
            return Signal.Buy;
        else if (deviation > _deviationThreshold)
            return Signal.Sell;
        return Signal.Hold;
    }
}

```

### RsiStrategy (Concrete Strategy)

```csharp
// ── Concrete Strategy 4: RSI Strategy (added without modifying existing code) ──
// OOP role: OCP extension — new strategy, no existing code changed
// SOLID: OCP (open for extension, closed for modification)
public sealed class RsiStrategy : ITradingStrategy
{
    private readonly int _period;
    private readonly int _overbought;
    private readonly int _oversold;

    public RsiStrategy(int period = 14, int overbought = 70, int oversold = 30)
    {
        _period = period;
        _overbought = overbought;
        _oversold = oversold;
    }

    public Signal Analyze(List<double> prices)
    {
        if (prices.Count < _period + 1)
            return Signal.Hold;

        double totalGain = 0;
        double totalLoss = 0;

        int startIdx = prices.Count - _period;
        for (int i = startIdx; i < prices.Count; i++)
        {
            double change = prices[i] - prices[i - 1];
            if (change >= 0)
                totalGain += change;
            else
                totalLoss += Math.Abs(change);
        }

        double avgGain = totalGain / _period;
        double avgLoss = totalLoss / _period;

        if (avgLoss == 0)
            return Signal.Buy;

        double rs = avgGain / avgLoss;
        double rsi = 100.0 - (100.0 / (1.0 + rs));

        if (rsi >= _overbought)
            return Signal.Sell;
        else if (rsi <= _oversold)
            return Signal.Buy;
        return Signal.Hold;
    }
}

```

### TradingBot (Context)

```csharp
// ── Context: TradingBot ──
// OOP role: Encapsulation — holds strategy reference and delegates
// SOLID: SRP (only orchestrates), DIP (depends on ITradingStrategy interface)
public sealed class TradingBot
{
    private ITradingStrategy? _strategy;

    public void SetStrategy(ITradingStrategy strategy)
    {
        _strategy = strategy;
    }

    public string StrategyName => _strategy?.GetType().Name ?? "None";

    public Signal RunAnalysis(string ticker, List<double> prices)
    {
        if (_strategy is null)
            throw new InvalidOperationException("No strategy set. Call SetStrategy() first.");

        Signal signal = _strategy.Analyze(prices);
        LogSignal(ticker, signal);
        return signal;
    }

    private void LogSignal(string ticker, Signal signal)
    {
        string timestamp = DateTime.Now.ToString("yyyy-MM-dd HH:mm:ss");
        Console.WriteLine($"[{timestamp}] {ticker} -> {signal} (strategy: {StrategyName})");
    }
}

```

### Language-Specific Notes

- **Primary constructors (C# 12)** — The strategy classes use traditional constructor syntax, but C# 12 also supports primary constructors where parameters are declared directly on the class declaration: `public class MovingAverageCrossover(int shortWindow, int longWindow) : ITradingStrategy`. This reduces boilerplate for simple cases.
- **`sealed` classes** — Each concrete strategy is `sealed` to prevent inheritance. This ensures the OCP is applied correctly: new behavior comes from new classes, not from overriding existing ones.
- **Collection expressions (C# 12)** — The `[ ... ]` syntax for `List<double>` initialization is a C# 12 feature that replaces the older `new List<double> { ... }` syntax. It is more concise and consistent with other collection types.
- **Index-from-end operator `^`** — The `^1` operator accesses elements from the end of a list. `prices[^1]` retrieves the last element, equivalent to `prices[prices.Count - 1]`. This makes the momentum and reversion calculations more readable.
- **Tuple deconstruction** — The `foreach (var (name, instance) in strategies)` syntax deconstructs value tuples directly in the loop variable, providing a clean way to iterate over named strategy pairs.
