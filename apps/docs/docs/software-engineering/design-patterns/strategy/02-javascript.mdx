## Implementation

Below is a complete JavaScript implementation of the Strategy pattern using the Stock Market domain. The `TradingBot` (Context) delegates trade signal generation to a pluggable `TradingStrategy`. Three concrete strategies implement different trading algorithms: `MovingAverageCrossover`, `MomentumStrategy`, and `MeanReversionStrategy`. The bot can switch strategies at runtime without modifying its core logic. The implementation uses ES2022+ class syntax and demonstrates OOP concepts (abstraction via duck typing, encapsulation, polymorphism) and SOLID principles (SRP, OCP, LSP, DIP).

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌──────────────────────────────────────┐
//   │ TradingBot (Context)                 │ ← OOP: Encapsulation
//   ├──────────────────────────────────────┤    SRP: coordinates strategy
//   │ - strategy: { analyze(prices) }      │     DIP: depends on abstraction
//   │ + setStrategy(strategy): void         │
//   │ + runAnalysis(ticker, prices): Signal │
//   └────────────────┬─────────────────────┘
//                    │ delegates
//                    ▼
//   ┌──────────────────────────────────────┐
//   │ TradingStrategy (duck-typed contract)│ ← OOP: Abstraction
//   ├──────────────────────────────────────┤    ISP: single method
//   │ + analyze(prices): Signal             │
//   └────────────────┬─────────────────────┘
//                    ▲ implements
//   ┌────────────────┴─────────────────────┐
//   │                                      │
//   ┌────────────────┐ ┌────────────────┐ ┌──────────────────────┐
//   │MovingAvg       │ │Momentum        │ │MeanReversion         │
//   │Crossover       │ │Strategy        │ │Strategy              │
//   ├────────────────┤ ├────────────────┤ ├──────────────────────┤
//   │+ analyze()     │ │+ analyze()     │ │+ analyze()           │
//   └────────────────┘ └────────────────┘ └──────────────────────┘
//        ▲ OCP: new strategies added by
//        │   creating new classes — no
//        │   modifications to existing code
```

#### OOP Concepts Applied

1. **Abstraction** — The `TradingStrategy` contract is defined by convention (duck typing). Any object with an `analyze(prices)` method is a valid strategy. Clients depend on this contract, not concrete implementations.
2. **Encapsulation** — Each strategy class hides its internal parameters (window sizes, thresholds, lookback periods) behind its `analyze()` method. The `TradingBot` encapsulates the strategy reference and the orchestration logic.
3. **Polymorphism** — The `TradingBot.runAnalysis()` method calls `this.strategy.analyze(prices)` without knowing which concrete strategy is installed. The correct implementation runs based on the actual object type.

#### SOLID Principles Applied

4. **SRP** — `TradingBot` only manages strategy execution. Each strategy class only implements one algorithm. No class has more than one reason to change.
5. **OCP** — Adding a new strategy (e.g., `PairsTradingStrategy`) requires creating a new class and injecting it into the bot. No existing code is modified.
6. **LSP** — Every concrete strategy can substitute any other. The bot calls `analyze()` on any strategy and gets a valid `Signal` string.
7. **ISP** — The strategy contract is a single method `analyze(prices)`. No strategy is forced to implement methods it does not need.
8. **DIP** — `TradingBot` depends on the abstract strategy contract, not on concrete strategy classes. Strategies are injected from outside.

### MovingAverageCrossover (Concrete Strategy)

```javascript
// ── Concrete Strategy 1: Moving Average Crossover ──
// OOP role: Polymorphism — implements the algorithm contract
// SOLID: SRP (only crossover logic), LSP (substitutable), OCP (extensible)
class MovingAverageCrossover {
  #shortWindow;
  #longWindow;

  constructor(shortWindow = 20, longWindow = 50) {
    if (shortWindow >= longWindow) {
      throw new Error("shortWindow must be less than longWindow");
    }
    this.#shortWindow = shortWindow;
    this.#longWindow = longWindow;
  }

  analyze(prices) {
    if (prices.length < this.#longWindow) {
      return Signal.HOLD;
    }

    const shortMA = this.#computeAverage(prices, this.#shortWindow);
    const longMA = this.#computeAverage(prices, this.#longWindow);

    if (shortMA > longMA) {
      return Signal.BUY;
    } else if (shortMA < longMA) {
      return Signal.SELL;
    }
    return Signal.HOLD;
  }

  #computeAverage(prices, window) {
    const slice = prices.slice(-window);
    const sum = slice.reduce((acc, price) => acc + price, 0);
    return sum / slice.length;
  }
}

```

### MomentumStrategy (Concrete Strategy)

```javascript
// ── Concrete Strategy 2: Momentum Strategy ──
// OOP role: Polymorphism — implements the algorithm contract
// SOLID: SRP (only momentum logic), LSP (substitutable), OCP (extensible)
class MomentumStrategy {
  #lookbackPeriod;
  #threshold;

  constructor(lookbackPeriod = 10, threshold = 0.05) {
    this.#lookbackPeriod = lookbackPeriod;
    this.#threshold = threshold;
  }

  analyze(prices) {
    if (prices.length < this.#lookbackPeriod) {
      return Signal.HOLD;
    }

    const currentPrice = prices[prices.length - 1];
    const pastPrice = prices[prices.length - 1 - this.#lookbackPeriod];
    const momentum = (currentPrice - pastPrice) / pastPrice;

    if (momentum > this.#threshold) {
      return Signal.BUY;
    } else if (momentum < -this.#threshold) {
      return Signal.SELL;
    }
    return Signal.HOLD;
  }
}

```

### MeanReversionStrategy (Concrete Strategy)

```javascript
// ── Concrete Strategy 3: Mean Reversion Strategy ──
// OOP role: Polymorphism — implements the algorithm contract
// SOLID: SRP (only reversion logic), LSP (substitutable), OCP (extensible)
class MeanReversionStrategy {
  #window;
  #deviationThreshold;

  constructor(window = 20, deviationThreshold = 0.03) {
    this.#window = window;
    this.#deviationThreshold = deviationThreshold;
  }

  analyze(prices) {
    if (prices.length < this.#window) {
      return Signal.HOLD;
    }

    const slice = prices.slice(-this.#window);
    const mean = slice.reduce((acc, p) => acc + p, 0) / slice.length;
    const current = prices[prices.length - 1];
    const deviation = (current - mean) / mean;

    if (deviation < -this.#deviationThreshold) {
      // Price is below mean — expect reversion upward
      return Signal.BUY;
    } else if (deviation > this.#deviationThreshold) {
      // Price is above mean — expect reversion downward
      return Signal.SELL;
    }
    return Signal.HOLD;
  }
}

```

### RsiStrategy (Concrete Strategy)

```javascript
// ── Concrete Strategy 4: RSI Strategy ──
// OOP role: Polymorphism — additional strategy demonstrating OCP
// SOLID: OCP (new strategy, no existing code changed)
class RsiStrategy {
  #period;
  #overbought;
  #oversold;

  constructor(period = 14, overbought = 70, oversold = 30) {
    this.#period = period;
    this.#overbought = overbought;
    this.#oversold = oversold;
  }

  analyze(prices) {
    if (prices.length < this.#period + 1) {
      return Signal.HOLD;
    }

    const gains = [];
    const losses = [];
    for (let i = prices.length - this.#period; i < prices.length; i++) {
      const change = prices[i] - prices[i - 1];
      if (change >= 0) {
        gains.push(change);
        losses.push(0);
      } else {
        gains.push(0);
        losses.push(Math.abs(change));
      }
    }

    const avgGain = gains.reduce((a, b) => a + b, 0) / this.#period;
    const avgLoss = losses.reduce((a, b) => a + b, 0) / this.#period;

    if (avgLoss === 0) {
      return Signal.BUY;
    }

    const rs = avgGain / avgLoss;
    const rsi = 100 - 100 / (1 + rs);

    if (rsi >= this.#overbought) {
      return Signal.SELL;
    } else if (rsi <= this.#oversold) {
      return Signal.BUY;
    }
    return Signal.HOLD;
  }
}

```

### TradingBot (Context)

```javascript
// ── Context: TradingBot ──
// OOP role: Encapsulation — holds strategy reference and delegates
// SOLID: SRP (only orchestrates), DIP (depends on abstraction)
class TradingBot {
  #strategy;
  #portfolio;

  constructor() {
    this.#strategy = null;
    this.#portfolio = {};
  }

  setStrategy(strategy) {
    this.#strategy = strategy;
  }

  getStrategyName() {
    if (this.#strategy === null) {
      return "None";
    }
    return this.#strategy.constructor.name;
  }

  runAnalysis(ticker, prices) {
    if (this.#strategy === null) {
      throw new Error("No strategy set. Call setStrategy() first.");
    }
    const signal = this.#strategy.analyze(prices);
    this.#logSignal(ticker, signal);
    return signal;
  }

  #logSignal(ticker, signal) {
    const timestamp = new Date().toISOString().slice(0, 19).replace("T", " ");
    console.log(`[${timestamp}] ${ticker} -> ${signal} (strategy: ${this.getStrategyName()})`);
  }
}

```

### Language-Specific Notes

- **Duck typing for interfaces** — JavaScript has no interface keyword. The `TradingStrategy` contract is enforced by convention: any object with an `analyze(prices)` method is accepted. This is idiomatic JavaScript and follows the "duck typing" philosophy.
- **Private class fields** — The `#` prefix (e.g., `#shortWindow`, `#strategy`) creates true private fields, enforced by the JavaScript engine. This is an ES2022+ feature that improves encapsulation compared to older `_` conventions.
- **`Object.freeze` for enums** — Since JavaScript lacks a native enum, `Object.freeze` on a plain object creates an immutable constant map. This is the standard pattern for enumerations in modern JavaScript.
- **Method overriding via prototype chain** — Each concrete strategy class defines its own `analyze()` method, which overrides the inherited `Object.prototype` contract. The `TradingBot` calls `analyze()` polymorphically without knowing the concrete type.
