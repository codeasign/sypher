## Implementation

Below is a complete TypeScript implementation of the Flyweight pattern using a car configurator domain. The `CarSpec` interface defines the flyweight contract with a `display()` method. Three concrete flyweight classes (`EngineSpec`, `WheelSpec`, `PaintSpec`) implement this interface and store immutable intrinsic state. The `CarSpecFactory` manages a cache of flyweight objects, returning existing instances when available. The `CarConfiguration` class stores extrinsic state (VIN, owner, license plate, production date) and holds references to shared flyweight objects.

### How the Implementation Works

#### Class Diagram (Pseudocode)

#### OOP Concepts Applied

1. **Abstraction** — The `CarSpec` interface defines the `display()` method that all concrete flyweights must implement. The factory and client depend on this abstraction.

2. **Encapsulation** — Each flyweight hides its internal properties behind `readonly` modifiers. The factory encapsulates the cache data structure.

3. **Polymorphism** — The `CarConfiguration.display()` method calls `display()` on each flyweight reference without knowing the concrete type.

#### SOLID Principles Applied

4. **SRP** — Each flyweight class has a single responsibility. The factory manages caching. The configuration represents a car.

5. **OCP** — New flyweight types can be added by implementing `CarSpec` and adding a cache map in the factory. No existing classes need modification.

6. **LSP** — All concrete flyweights implement `CarSpec` and can be substituted wherever `CarSpec` is expected.

7. **ISP** — The `CarSpec` interface contains only the `display()` method. Flyweights are not forced to implement unused methods.

8. **DIP** — `CarConfiguration` depends on the `CarSpec` interface, not on concrete flyweight implementations.

### CarSpec (Flyweight Interface)

```typescript
// Flyweight Pattern — TypeScript — Car Configurator Example
// ==========================================================
// Participants: CarSpec (Flyweight), EngineSpec/WheelSpec/PaintSpec
//   (Concrete Flyweights), CarSpecFactory (FlyweightFactory),
//   CarConfiguration (Client)
// OOP: abstraction, encapsulation, polymorphism
// SOLID: SRP, OCP, LSP, ISP, DIP
// ==========================================================

// ── Flyweight interface ──
// OOP role: Abstraction + ISP
// SOLID: ISP, DIP
interface CarSpec {
  display(): string;
}

```

### EngineSpec

```typescript
// ── Concrete Flyweight: EngineSpec ──
// Intrinsic state: model, horsepower, displacement, cylinders, fuel type
// OOP role: Inheritance/Polymorphism
// SOLID: LSP, SRP
class EngineSpec implements CarSpec {
  public readonly model: string;
  public readonly horsepower: number;
  public readonly displacement: number;
  public readonly cylinders: number;
  public readonly fuelType: string;

  constructor(
    model: string,
    horsepower: number,
    displacement: number,
    cylinders: number,
    fuelType: string
  ) {
    this.model = model;
    this.horsepower = horsepower;
    this.displacement = displacement;
    this.cylinders = cylinders;
    this.fuelType = fuelType;
  }

  display(): string {
    return `${this.model} | ${this.horsepower} hp | ${this.displacement}L | ${this.cylinders} cyl | ${this.fuelType}`;
  }
}

```

### CarSpecFactory

```typescript
// ── Flyweight Factory ──
// OOP role: Composition
// SOLID: SRP (manages flyweight cache), DIP (returns CarSpec abstractions)
class CarSpecFactory {
  private engines: Map<string, EngineSpec> = new Map();
  private wheels: Map<string, WheelSpec> = new Map();
  private paints: Map<string, PaintSpec> = new Map();

  getEngine(
    model: string,
    horsepower: number,
    displacement: number,
    cylinders: number,
    fuelType: string
  ): EngineSpec {
    if (!this.engines.has(model)) {
      this.engines.set(
        model,
        new EngineSpec(model, horsepower, displacement, cylinders, fuelType)
      );
    }
    return this.engines.get(model)!;
  }

  getWheel(
    key: string,
    diameter: number,
    width: number,
    material: string,
    brand: string
  ): WheelSpec {
    if (!this.wheels.has(key)) {
      this.wheels.set(key, new WheelSpec(diameter, width, material, brand));
    }
    return this.wheels.get(key)!;
  }

  getPaint(code: string, name: string, finish: string, hexColor: string): PaintSpec {
    if (!this.paints.has(code)) {
      this.paints.set(code, new PaintSpec(code, name, finish, hexColor));
    }
    return this.paints.get(code)!;
  }

  getStats(): CacheStats {
    return {
      uniqueEngines: this.engines.size,
      uniqueWheels: this.wheels.size,
      uniquePaints: this.paints.size,
      totalUnique: this.engines.size + this.wheels.size + this.paints.size
    };
  }
}

```

### CarConfiguration

```typescript
// ── Client: CarConfiguration ──
// Extrinsic state: VIN, owner, license plate, production date
// References to shared flyweights: engine, wheels, paint
// OOP role: Encapsulation, Composition
// SOLID: SRP (represents one car), DIP (depends on CarSpec abstractions)
class CarConfiguration {
  constructor(
    private readonly vin: string,
    private readonly owner: string,
    private readonly licensePlate: string,
    private readonly productionDate: string,
    private readonly engine: EngineSpec,
    private readonly wheels: WheelSpec,
    private readonly paint: PaintSpec
  ) {}

  display(): string {
    const lines: string[] = [
      '--- Car Configuration ---',
      `VIN:             ${this.vin}`,
      `Owner:           ${this.owner}`,
      `License Plate:   ${this.licensePlate}`,
      `Production Date: ${this.productionDate}`,
      `Engine:          ${this.engine.display()}`,
      `Wheels:          ${this.wheels.display()}`,
      `Paint:           ${this.paint.display()}`,
      '---'
    ];
    return lines.join('\n');
  }
}

```

### Language-Specific Notes

- **`readonly` modifier for immutability** — TypeScript's `readonly` keyword on class properties ensures that the intrinsic state cannot be reassigned after construction. Since the flyweight properties are only set in the constructor and never mutated, `readonly` enforces the flyweight immutability invariant at compile time. Any attempt to reassign a `readonly` property produces a TypeScript compilation error.

- **TypeScript interface for the flyweight contract** — The `CarSpec` interface defines the structural contract for all flyweight types. TypeScript's structural type system means that any object with a `display(): string` method satisfies the interface, even without an explicit `implements` clause. This enables duck typing while providing compile-time type safety.

- **`CacheStats` interface for structured factory output** — The factory returns a typed `CacheStats` object instead of a raw string. This gives the caller type-safe access to individual statistics values. The `totalUnique` computed property is calculated in the method and returned as part of the interface.

- **Generic factory pattern** — Each flyweight type has its own `Map` in the factory. The factory methods are strongly typed: `getEngine()` returns `EngineSpec`, `getWheel()` returns `WheelSpec`, and `getPaint()` returns `PaintSpec`. The non-null assertion (`!`) on the `Map.get()` result is safe because the `has()` check guarantees the value exists.

- **`private readonly` constructor parameters** — The `CarConfiguration` class uses TypeScript's parameter property shorthand (`private readonly vin: string`) to declare and initialize properties in the constructor signature. This reduces boilerplate while maintaining encapsulation and immutability of the extrinsic state.
