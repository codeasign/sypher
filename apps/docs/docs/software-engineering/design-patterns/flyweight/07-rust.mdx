## Implementation

Below is a complete Rust implementation of the Flyweight pattern using a car configurator domain. The `CarSpec` trait defines the flyweight contract with a `display()` method. Three concrete flyweight structs (`EngineSpec`, `WheelSpec`, `PaintSpec`) implement this trait with immutable intrinsic state. The `CarSpecFactory` uses `HashMap` to cache flyweight objects and returns `Arc` references for shared ownership. The `CarConfiguration` struct stores extrinsic state (VIN, owner, license plate, production date) and holds `Arc` references to shared flyweight objects.

### How the Implementation Works

#### OOP Concepts Applied (Rust Idioms)

1. **Abstraction** — The `CarSpec` trait defines the `display()` contract. All client code depends on this trait via trait bounds.

2. **Encapsulation** — Flyweight struct fields are `pub` because Rust's ownership model makes immutability the default. The factory encapsulates the `HashMap` caches.

3. **Polymorphism** — While Rust does not have classical inheritance, trait objects (`&dyn CarSpec`) enable runtime polymorphism. In this implementation, each flyweight type is concrete and known at compile time.

#### SOLID Principles Applied

4. **SRP** — Each flyweight struct has a single responsibility. The factory manages caching. The configuration represents a car.

5. **OCP** — New flyweight types can be added by implementing `CarSpec` and adding a new cache `HashMap` in the factory.

6. **LSP** — All concrete flyweights implement `CarSpec` and can be substituted wherever the trait is expected.

7. **ISP** — The `CarSpec` trait contains only `display()`. Flyweights are not forced to implement unused methods.

8. **DIP** — `CarConfiguration` depends on `CarSpec` via trait bounds, not on concrete flyweight structs.

### CarSpec (Flyweight Interface)

```rust
// Flyweight Pattern — Rust — Car Configurator Example
// ==========================================================
// Participants: CarSpec (Flyweight), EngineSpec/WheelSpec/PaintSpec
//   (Concrete Flyweights), CarSpecFactory (FlyweightFactory),
//   CarConfiguration (Client)
// OOP: abstraction, encapsulation, polymorphism
// SOLID: SRP, OCP, LSP, ISP, DIP
// ==========================================================

use std::collections::HashMap;
use std::sync::Arc;


// ── Flyweight trait ──
// OOP role: Abstraction + ISP
// SOLID: ISP, DIP
trait CarSpec: Send + Sync {
    fn display(&self) -> String;
}

```

### EngineSpec

```rust
// ── Concrete Flyweight: EngineSpec ──
// Intrinsic state: model, horsepower, displacement, cylinders, fuel type
// OOP role: Implementation/Polymorphism
// SOLID: LSP, SRP
struct EngineSpec {
    model: String,
    horsepower: u32,
    displacement: f64,
    cylinders: u32,
    fuel_type: String,
}

impl CarSpec for EngineSpec {
    fn display(&self) -> String {
        format!(
            "{} | {} hp | {:.1}L | {} cyl | {}",
            self.model, self.horsepower, self.displacement, self.cylinders, self.fuel_type
        )
    }
}

```

### CarSpecFactory

```rust
// ── Flyweight Factory ──
// OOP role: Composition
// SOLID: SRP (manages flyweight cache), DIP (returns Arc<dyn CarSpec>)
struct CarSpecFactory {
    engines: HashMap<String, Arc<EngineSpec>>,
    wheels: HashMap<String, Arc<WheelSpec>>,
    paints: HashMap<String, Arc<PaintSpec>>,
}

impl CarSpecFactory {
    fn new() -> Self {
        CarSpecFactory {
            engines: HashMap::new(),
            wheels: HashMap::new(),
            paints: HashMap::new(),
        }
    }

    fn get_engine(
        &mut self,
        model: &str,
        horsepower: u32,
        displacement: f64,
        cylinders: u32,
        fuel_type: &str,
    ) -> Arc<EngineSpec> {
        self.engines
            .entry(model.to_string())
            .or_insert_with(|| {
                Arc::new(EngineSpec {
                    model: model.to_string(),
                    horsepower,
                    displacement,
                    cylinders,
                    fuel_type: fuel_type.to_string(),
                })
            })
            .clone()
    }

    fn get_wheel(
        &mut self,
        key: &str,
        diameter: f64,
        width: u32,
        material: &str,
        brand: &str,
    ) -> Arc<WheelSpec> {
        self.wheels
            .entry(key.to_string())
            .or_insert_with(|| {
                Arc::new(WheelSpec {
                    diameter,
                    width,
                    material: material.to_string(),
                    brand: brand.to_string(),
                })
            })
            .clone()
    }

    fn get_paint(
        &mut self,
        code: &str,
        name: &str,
        finish: &str,
        hex_color: &str,
    ) -> Arc<PaintSpec> {
        self.paints
            .entry(code.to_string())
            .or_insert_with(|| {
                Arc::new(PaintSpec {
                    code: code.to_string(),
                    name: name.to_string(),
                    finish: finish.to_string(),
                    hex_color: hex_color.to_string(),
                })
            })
            .clone()
    }

    fn get_stats(&self) -> CacheStats {
        CacheStats {
            unique_engines: self.engines.len(),
            unique_wheels: self.wheels.len(),
            unique_paints: self.paints.len(),
            total_unique: self.engines.len() + self.wheels.len() + self.paints.len(),
        }
    }
}

```

### CarConfiguration

```rust
// ── Client: CarConfiguration ──
// Extrinsic state: VIN, owner, license plate, production date
// References to shared flyweights via Arc
// OOP role: Encapsulation, Composition
// SOLID: SRP (represents one car), DIP (depends on CarSpec trait)
struct CarConfiguration {
    vin: String,
    owner: String,
    license_plate: String,
    production_date: String,
    engine: Arc<EngineSpec>,
    wheels: Arc<WheelSpec>,
    paint: Arc<PaintSpec>,
}

impl CarConfiguration {
    fn new(
        vin: &str,
        owner: &str,
        license_plate: &str,
        production_date: &str,
        engine: Arc<EngineSpec>,
        wheels: Arc<WheelSpec>,
        paint: Arc<PaintSpec>,
    ) -> Self {
        CarConfiguration {
            vin: vin.to_string(),
            owner: owner.to_string(),
            license_plate: license_plate.to_string(),
            production_date: production_date.to_string(),
            engine,
            wheels,
            paint,
        }
    }

    fn display(&self) -> String {
        format!(
            "--- Car Configuration ---\n\
             VIN:             {}\n\
             Owner:           {}\n\
             License Plate:   {}\n\
             Production Date: {}\n\
             Engine:          {}\n\
             Wheels:          {}\n\
             Paint:           {}\n\
             ---",
            self.vin,
            self.owner,
            self.license_plate,
            self.production_date,
            self.engine.display(),
            self.wheels.display(),
            self.paint.display(),
        )
    }
}

```

fn main() {
    let mut factory = CarSpecFactory::new();

    // Create flyweights (each unique spec is created once)
    let engine_2l = factory.get_engine("2.0L Turbo", 250, 2.0, 4, "Gasoline");
    let engine_3l = factory.get_engine("3.0L V6", 340, 3.0, 6, "Gasoline");
    let engine_electric = factory.get_engine("Electric Drive", 280, 0.0, 0, "Electric");

    let wheel_18 = factory.get_wheel("18x8 Sport", 18.0, 8, "Alloy", "Sport");
    let wheel_20 = factory.get_wheel("20x9 Performance", 20.0, 9, "Forged", "Performance");

    let paint_red = factory.get_paint("R3", "Racing Red", "Gloss", "#FF2020");
    let paint_blue = factory.get_paint("B7", "Midnight Blue", "Metallic", "#002060");

    // Create car configurations that share flyweights
    let cars = vec![
        CarConfiguration::new(

            "2026-01-15", engine_2l.clone(), wheel_18.clone(), paint_red.clone()),
        CarConfiguration::new(
            "WBA2345678901BCDE", "Bob Smith", "XYZ-5678",
            "2026-02-20", engine_2l.clone(), wheel_18.clone(), paint_blue.clone()),
        CarConfiguration::new(
            "WBA3456789012CDEF", "Carol Williams", "DEF-9012",
            "2026-03-10", engine_3l.clone(), wheel_20.clone(), paint_red.clone()),
        CarConfiguration::new(
            "WBA4567890123DEFG", "Dave Brown", "GHI-3456",
            "2026-04-05", engine_electric.clone(), wheel_20.clone(), paint_white.clone()),
        CarConfiguration::new(
            "WBA5678901234EFGH", "Eve Davis", "JKL-7890",
            "2026-05-18", engine_2l.clone(), wheel_18.clone(), paint_red.clone()),
        CarConfiguration::new(
            "WBA6789012345FGHI", "Frank Miller", "MNO-1234",
            "2026-06-01", engine_3l.clone(), wheel_20.clone(), paint_blue.clone()),
    ];

    // Display all car configurations
    println!("=== Car Configurator — Flyweight Pattern ===\n");

    for car in &cars {
        println!("{}\n", car.display());
    }

    // Demonstrate flyweight sharing
    println!("=== Flyweight Sharing Verification ===");
    println!("Car 1 and Car 5 share engine: {} (expected: true)",
        Arc::ptr_eq(&cars[0].engine, &cars[4].engine));

        Arc::ptr_eq(&cars[0].engine, &cars[3].engine));

        Arc::ptr_eq(&cars[0].wheels, &cars[1].wheels));
    println!("Car 1 and Car 3 share wheels: {} (expected: false)",
        Arc::ptr_eq(&cars[0].wheels, &cars[2].wheels));
    println!("Car 1 and Car 5 share paint: {} (expected: true)",
        Arc::ptr_eq(&cars[0].paint, &cars[4].paint));
    println!("Car 1 and Car 4 share paint: {} (expected: false)",
        Arc::ptr_eq(&cars[0].paint, &cars[3].paint));

    println!();
    println!("=== Factory Cache Statistics ===");
    let stats = factory.get_stats();
    println!("  Engines: {} unique", stats.unique_engines);
    println!("  Wheels:  {} unique", stats.unique_wheels);
    println!("  Paints:  {} unique", stats.unique_paints);
    println!("  Total:   {} unique flyweight objects", stats.total_unique);
    println!("  Shared across: {} car configurations", cars.len());
}

### Language-Specific Notes

- **`Arc` for shared ownership across flyweight consumers** — Rust's `Arc<T>` (Atomic Reference Counting) enables multiple `CarConfiguration` instances to share ownership of the same flyweight object. When `Arc::clone()` is called, the reference count is incremented atomically, and the underlying `T` is not duplicated. This is the idiomatic Rust approach to the Flyweight pattern: the factory stores `Arc<T>` values, and each client holds an `Arc<T>` reference to the shared flyweight.

- **`HashMap::entry().or_insert_with()` for cache-or-create logic** — The factory uses the `entry()` API with `or_insert_with()` to atomically check the cache and create the flyweight if absent. This is more idiomatic and efficient than the manual `contains_key()` + `insert()` pattern because it performs a single lookup instead of two. The closure passed to `or_insert_with()` is only executed on cache miss.

- **`Arc::ptr_eq()` for identity comparison** — The driver uses `Arc::ptr_eq()` to check whether two `Arc` references point to the same heap allocation. This is the Rust equivalent of reference equality and confirms that the flyweight objects are shared rather than duplicated. Unlike `Arc::eq()`, which compares the inner values, `ptr_eq()` compares the memory addresses of the reference-counted objects.

- **`Send + Sync` bounds on the trait** — The `CarSpec` trait requires `Send + Sync` because the flyweight objects are stored in `Arc` and may be accessed from multiple threads. `Send` allows the `Arc` to be transferred between threads, and `Sync` allows shared references to the flyweight from multiple threads. These bounds are automatically satisfied because all flyweight fields are immutable and implement `Send + Sync`.

- **`format!()` macro for string building** — The `display()` methods use Rust's `format!()` macro for string formatting. The `format!()` macro is type-safe and supports format specifiers like `{:.0}` (no decimal places), `{:.1}` (one decimal place), and `{}` (default display). This is the typical Rust approach to string formatting and avoids the runtime overhead of dynamic format strings.

- **No `dyn CarSpec` trait objects** — This implementation uses concrete generic types (`Arc<EngineSpec>`, `Arc<WheelSpec>`, `Arc<PaintSpec>`) rather than `Arc<dyn CarSpec>` trait objects. This is a deliberate design choice: using concrete types avoids the indirection of dynamic dispatch and allows the compiler to monomorphize the code for better performance. The trade-off is that the `CarConfiguration` struct must know the concrete flyweight types at compile time, but this is acceptable for the Cars domain where the set of flyweight types is fixed.
