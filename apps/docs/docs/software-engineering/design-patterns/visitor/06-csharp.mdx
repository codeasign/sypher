## Implementation

Below is a complete C# 12 / .NET 8 implementation of the Visitor pattern using a stock market portfolio domain. The `Portfolio` holds a heterogeneous collection of `Stock` and `Bond` objects (both implementing the `IElement` interface), and four visitor implementations perform distinct analyses: computing total market value, calculating annual dividend income, assessing risk levels, and generating a formatted portfolio report. C#'s interface system and pattern-matching capabilities provide type-safe double dispatch.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌────────────────────────────────────────────────┐
//   │   <<interface>> IVisitor                       │  <- OOP: Abstraction
//   ├────────────────────────────────────────────────┤     ISP: segregated per type
//   │   void VisitStock(Stock stock)                 │
//   │   void VisitBond(Bond bond)                    │
//   └────────────────┬───────────────────────────────┘
//                    ▲ implements
//   ┌────────────────┴───────────────────────────────┐
//   │                                                │
//   ┌────────────────────────┐   ┌──────────────────────────┐
//   │ ValueVisitor           │   │ DividendVisitor          │  <- OOP: Polymorphism
//   ├────────────────────────┤   ├──────────────────────────┤     SRP: one concern
//   │ - decimal TotalValue   │   │ - decimal TotalDividend  │     OCP: new = new impl
//   │ VisitStock(s) { ... }  │   │ VisitStock(s) { ... }    │
//   │ VisitBond(b) { ... }   │   │ VisitBond(b) { ... }     │
//   └────────────────────────┘   └──────────────────────────┘
//
//   ┌────────────────────────────────────────────────┐
//   │   <<interface>> IElement                       │  <- OOP: Abstraction
//   ├────────────────────────────────────────────────┤     ISP: single Accept method
//   │   void Accept(IVisitor visitor)                │
//   └────────────────┬───────────────────────────────┘
//                    ▲ implements
//   ┌────────────────┴───────────────────────────────┐
//   │                                                │
//   ┌────────────────────────┐   ┌──────────────────────────┐
//   │     Stock              │   │      Bond                │  <- OOP: Polymorphism
//   ├────────────────────────┤   ├──────────────────────────┤     LSP: both
//   │ Accept(v) {            │   │ Accept(v) {              │     substitutable
//   │   v.VisitStock(this);  │   │   v.VisitBond(this);     │     for IElement
//   │ }                      │   │ }                        │
//   └────────────────────────┘   └──────────────────────────┘
```

#### OOP Concepts Applied

1. **Abstraction** -- The `IVisitor` and `IElement` interfaces define the contracts (C# naming convention prefixes interfaces with `I`). All client code depends on these abstractions.
2. **Encapsulation** -- Each class hides its internal state. Visitors use `private` fields that are only accessible within the class.
3. **Inheritance (via `: IElement`)** -- `Stock` and `Bond` implement the `IElement` interface. Each provides its own `Accept` method that dispatches to the correct visitor method.
4. **Polymorphism** -- `Portfolio.AcceptAll()` calls `Accept()` on each `IElement` without knowing the concrete type. Interface method dispatch resolves the correct operation.

#### SOLID Principles Applied

5. **SRP** -- Each visitor class has a single responsibility: value calculation, dividend computation, risk assessment, or report generation. The `Portfolio` only manages the element collection.
6. **OCP** -- Adding a new analysis requires only a new class implementing `IVisitor`. No existing code needs modification.
7. **LSP** -- Any `IVisitor` implementation can be passed to `Portfolio.AcceptAll()` without breaking the iteration. `Stock` and `Bond` are fully substitutable for `IElement`.
8. **ISP** -- The `IVisitor` interface has exactly two methods -- one per visited element type. No implementor is forced to provide unused methods.
9. **DIP** -- `Portfolio` depends on the `IElement` abstraction. Visitors depend on the `IVisitor` interface contract. High-level code depends on abstractions.

### IElement (Interface)

```csharp
// OOP role: Abstraction
// SOLID: ISP (minimal contract), DIP (depended upon abstraction)
public interface IElement
{
    void Accept(IVisitor visitor);
}
```

### IVisitor (Interface)

```csharp
// OOP role: Abstraction
// SOLID: ISP (one method per element type), OCP (extensible)
public interface IVisitor
{
    void VisitStock(Stock stock);
    void VisitBond(Bond bond);
}
```

### Stock (Concrete Element)

```csharp
// OOP role: Polymorphism (implements IElement)
// SOLID: SRP (represents only stock data), LSP (substitutable for IElement)
public class Stock : IElement
{
    public string Ticker { get; }
    public string CompanyName { get; }
    public int Shares { get; }
    public decimal PricePerShare { get; }
    public decimal DividendYield { get; }

    public Stock(string ticker, string companyName, int shares,
                 decimal pricePerShare, decimal dividendYield)
    {
        Ticker = ticker;
        CompanyName = companyName;
        Shares = shares;
        PricePerShare = pricePerShare;
        DividendYield = dividendYield;
    }

    public void Accept(IVisitor visitor)
    {
        visitor.VisitStock(this);
    }
}
```

### Bond (Concrete Element)

```csharp
// OOP role: Polymorphism (implements IElement)
// SOLID: SRP (represents only bond data), LSP (substitutable for IElement)
public class Bond : IElement
{
    public string Name { get; }
    public decimal FaceValue { get; }
    public decimal CouponRate { get; }

    public Bond(string name, decimal faceValue, decimal couponRate,

    {
        Name = name;
        FaceValue = faceValue;
        CouponRate = couponRate;
        YearsToMaturity = yearsToMaturity;
    }

    public void Accept(IVisitor visitor)

        visitor.VisitBond(this);

```

### ValueVisitor (Concrete Visitor)

```csharp
// OOP role: Polymorphism (implements IVisitor)
// SOLID: SRP (only calculates total value), OCP (new analysis = new class)
public class ValueVisitor : IVisitor
{
    private decimal _totalValue;

    public void VisitStock(Stock stock)
    {
        _totalValue += stock.Shares * stock.PricePerShare;
    }

    public void VisitBond(Bond bond)
    {
        _totalValue += bond.FaceValue;
    }

    public decimal TotalValue => _totalValue;
}
```

### DividendVisitor (Concrete Visitor)

```csharp
// OOP role: Polymorphism (implements IVisitor)
// SOLID: SRP (only calculates annual income)
public class DividendVisitor : IVisitor
{
    private decimal _totalDividend;

    public void VisitStock(Stock stock)
    {
        decimal value = stock.Shares * stock.PricePerShare;
        _totalDividend += value * stock.DividendYield;
    }

    public void VisitBond(Bond bond)
    {
        _totalDividend += bond.FaceValue * bond.CouponRate;
    }

    public decimal TotalDividend => _totalDividend;
}
```

### RiskVisitor (Concrete Visitor)

```csharp
// OOP role: Polymorphism (implements IVisitor)
// SOLID: SRP (only assesses risk)
public class RiskVisitor : IVisitor
{
    private readonly List<string> _results = new();

    public void VisitStock(Stock stock)
    {
        if (stock.DividendYield > 0.04m)
        {
            _results.Add($"{stock.Ticker}: LOW RISK -- established dividend payer");
        }
        else if (stock.DividendYield > 0.01m)
        {
            _results.Add($"{stock.Ticker}: MEDIUM RISK -- moderate dividend");
        }
        else
        {
            _results.Add($"{stock.Ticker}: HIGH RISK -- low or no dividend");
        }
    }

    public void VisitBond(Bond bond)
    {
        if (bond.YearsToMaturity > 20)
        {
            _results.Add($"{bond.Name}: MEDIUM RISK -- long maturity");
        }
        else if (bond.YearsToMaturity > 10)
        {
            _results.Add($"{bond.Name}: LOW RISK -- medium maturity");
        }
        else
        {
            _results.Add($"{bond.Name}: LOW RISK -- short maturity");
        }
    }

    public IReadOnlyList<string> GetResults() => _results.AsReadOnly();
}
```

### ReportVisitor (Concrete Visitor)

```csharp
// OOP role: Polymorphism (implements IVisitor)
// SOLID: SRP (only generates formatted report)
public class ReportVisitor : IVisitor
{
    private readonly List<string> _lines = new();

    public void VisitStock(Stock stock)
    {
        decimal value = stock.Shares * stock.PricePerShare;
        decimal income = value * stock.DividendYield;
        string line = string.Format(
            "| {0,-6} | {1,-20} | {2,5}  | ${3,8:F2} | ${4,8:F2} | ${5,8:F2} |",
            stock.Ticker, stock.CompanyName,
            stock.Shares, stock.PricePerShare, value, income
        );
        _lines.Add(line);
    }

    public void VisitBond(Bond bond)
    {
        decimal income = bond.FaceValue * bond.CouponRate;
        string line = string.Format(
            "| {0,-6} | {1,-20} | {2,2}yr  | ${3,8:F2} | ${4,8:F2} | ${5,8:F2} |",
            bond.Name, "Bond",
            bond.YearsToMaturity, bond.FaceValue, bond.FaceValue, income
        );
        _lines.Add(line);
    }

    public string GetReport()
    {
        var sb = new StringBuilder();
        sb.AppendLine();
        sb.AppendLine("Portfolio Report:");
        sb.Append("| Ticker  | Name                 | Qty/Mat ");
        sb.AppendLine("| Price/Val  | Mkt Value  | Income    |");
        sb.Append("|---------+----------------------+---------");
        sb.AppendLine("+------------+------------+-----------|");
        foreach (string line in _lines)
        {
            sb.AppendLine(line);
        }
        return sb.ToString();
    }
}
```

### Portfolio (ObjectStructure)

```csharp
// OOP role: Composition (aggregates elements)
// SOLID: SRP (manages collection only), DIP (depends on IElement abstraction)
public class Portfolio
{
    private readonly List<IElement> _elements = new();

    public void Add(IElement element)
    {
        _elements.Add(element);
    }

    public void AcceptAll(IVisitor visitor)
    {
        foreach (IElement element in _elements)
        {
            element.Accept(visitor);
        }
    }
}
```

### Main / Driver

```csharp
public class VisitorDemo
{
    public static void Main()
    {
        var portfolio = new Portfolio();

        // Add stocks with realistic data
        portfolio.Add(new Stock("AAPL", "Apple Inc.", 100, 175.50m, 0.005m));
        portfolio.Add(new Stock("GOOGL", "Alphabet Inc.", 50, 140.25m, 0.004m));
        portfolio.Add(new Stock("MSFT", "Microsoft Corp.", 80, 380.75m, 0.007m));
        portfolio.Add(new Stock("JPM", "JPMorgan Chase", 60, 155.30m, 0.048m));

        // Add bonds
        portfolio.Add(new Bond("US10Y", 10000.0m, 0.043m, 10));
        portfolio.Add(new Bond("US30Y", 5000.0m, 0.048m, 30));

        // 1. ValueVisitor -- calculate total portfolio value
        var valueVisitor = new ValueVisitor();
        portfolio.AcceptAll(valueVisitor);
        Console.WriteLine($"Total Portfolio Value: ${valueVisitor.TotalValue:F2}");

        // 2. DividendVisitor -- calculate annual income
        var divVisitor = new DividendVisitor();
        portfolio.AcceptAll(divVisitor);
        Console.WriteLine($"Annual Dividend/Coupon Income: ${divVisitor.TotalDividend:F2}");

        // 3. RiskVisitor -- assess risk for each holding
        var riskVisitor = new RiskVisitor();
        portfolio.AcceptAll(riskVisitor);
        Console.WriteLine("\nRisk Assessment:");
        foreach (string result in riskVisitor.GetResults())
        {
            Console.WriteLine("  " + result);
        }

        // 4. ReportVisitor -- generate formatted portfolio report
        var reportVisitor = new ReportVisitor();
        portfolio.AcceptAll(reportVisitor);
        Console.WriteLine(reportVisitor.GetReport());
    }
}
```

### Language-Specific Notes

- **Interface naming convention `I` prefix** -- C# conventionally prefixes interface names with `I` (`IVisitor`, `IElement`). This distinguishes interfaces from classes at a glance and is idiomatic in the .NET ecosystem.
- **`decimal` type for financial calculations** -- C# uses `decimal` instead of `double` or `float` for monetary values. The `decimal` type has 28-29 significant digits and avoids the floating-point rounding errors that would accumulate in financial calculations. This is a best practice in .NET financial applications.
- **`IReadOnlyList` for defensive returns** -- `RiskVisitor.GetResults()` returns `_results.AsReadOnly()`, which wraps the list in a read-only wrapper. Callers can iterate but cannot modify the visitor's internal state. This is a stronger contract than Python's convention or Java's defensive copy.
- **`string.Format` with format specifiers** -- C#'s `string.Format` uses positional format items (`{0,-6}`, `{1,5}`, `{2:F2}`) with alignment and precision specifiers. This is the standard approach for formatted output in .NET, analogous to Java's `String.format`.
- **`var` keyword for concise declarations** -- C# 3.0+ `var` keyword infers the type from the right-hand side. `var portfolio = new Portfolio()` is equivalent to `Portfolio portfolio = new Portfolio()`. This reduces verbosity without sacrificing type safety.
- **Expression-bodied members** -- C# 6+ expression-bodied members (`public decimal TotalValue => _totalValue;`) provide a concise syntax for read-only properties. This is more idiomatic than writing a full getter method in modern C#.
