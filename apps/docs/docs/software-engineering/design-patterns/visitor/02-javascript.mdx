## Implementation

Below is a complete JavaScript (ES2022+) implementation of the Visitor pattern using a stock market portfolio domain. The `Portfolio` holds a collection of financial instruments -- `Stock` and `Bond` objects -- and four different visitors perform distinct analyses: computing total value, calculating annual dividend income, assessing risk, and generating a formatted report. Each visitor encapsulates a single algorithm, and the double-dispatch mechanism ensures the correct visit method runs for each instrument type.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌──────────────────────────────────┐
//   │   Visitor (base class)           │  <- OOP: Abstraction
//   ├──────────────────────────────────┤     ISP: segregated per element type
//   │   visitStock(stock)              │
//   │   visitBond(bond)               │
//   └────────────────┬─────────────────┘
//                    ▲ extends
//   ┌────────────────┴─────────────────┐
//   │                                  │
//   ┌──────────────────┐   ┌──────────────────────┐
//   │ ValueVisitor     │   │ DividendVisitor      │  <- OOP: Polymorphism
//   ├──────────────────┤   ├──────────────────────┤     SRP: one concern each
//   │ #totalValue: num │   │ #totalDividend: num  │     OCP: new = new subclass
//   │ visitStock()     │   │ visitStock()         │
//   │ visitBond()      │   │ visitBond()          │
//   └──────────────────┘   └──────────────────────┘
//
//   ┌──────────────────────────────────┐
//   │   Element (protocol)            │  <- OOP: Abstraction
//   ├──────────────────────────────────┤     ISP: minimal contract
//   │   accept(visitor)               │
//   └────────────────┬─────────────────┘
//                    ▲ implements
//   ┌────────────────┴─────────────────┐
//   │                                  │
//   ┌──────────────────┐   ┌──────────────────────┐
//   │     Stock        │   │      Bond            │  <- OOP: Polymorphism
//   ├──────────────────┤   ├──────────────────────┤     LSP: both usable
//   │ accept(v)        │   │ accept(v)            │     as Element
//   │   v.visit(this)  │   │   v.visit(this)      │
//   └──────────────────┘   └──────────────────────┘
```

#### OOP Concepts Applied

1. **Abstraction** -- The `Visitor` base class defines the contract with `visitStock()` and `visitBond()` methods. Subclasses implement the actual algorithm. The `Element` protocol defines the `accept` contract.
2. **Encapsulation** -- Each element class hides its internal data behind public getters. Visitors accumulate state privately (e.g., `#totalValue` in `ValueVisitor`).
3. **Inheritance** -- Concrete visitors extend the `Visitor` base class. Concrete elements implement the `accept` method from the `Element` interface.
4. **Polymorphism** -- The `Portfolio.acceptAll()` method calls `accept()` on each element without knowing whether it is a `Stock` or `Bond`. Through double dispatch, the correct visitor method is called per element type.

#### SOLID Principles Applied

5. **SRP** -- Each visitor has one reason to change: `ValueVisitor` only calculates value, `DividendVisitor` only computes dividends, `RiskVisitor` only assesses risk, `ReportVisitor` only generates reports. The `Portfolio` only manages the collection.
6. **OCP** -- New analyses are added by creating new visitor subclasses. No existing visitor or element code needs modification. The system is open for extension, closed for modification.
7. **LSP** -- Any concrete visitor can substitute for the `Visitor` base class without breaking the client. Similarly, `Stock` and `Bond` are substitutable for `Element`.
8. **ISP** -- The `Visitor` interface has exactly as many methods as there are element types. No class is forced to implement methods it does not use.
9. **DIP** -- `Portfolio` depends on the `Element` abstraction, not on concrete `Stock` or `Bond` types. Client code depends on the `Visitor` abstraction, not on concrete visitor types.

### Visitor (Base Class)

```javascript
// OOP role: Abstraction
// SOLID: ISP (segregated per element type), DIP (depended upon abstraction)
class Visitor {
  visitStock(stock) {
    throw new Error("visitStock must be overridden");
  }
  visitBond(bond) {
    throw new Error("visitBond must be overridden");
  }
}
```

### Stock (Concrete Element)

```javascript
// OOP role: Polymorphism (implements Element via accept)
// SOLID: SRP (represents only stock data), LSP (substitutable for Element)
class Stock {
  constructor(ticker, companyName, shares, pricePerShare, dividendYield) {
    this.ticker = ticker;
    this.companyName = companyName;
    this.shares = shares;
    this.pricePerShare = pricePerShare;
    this.dividendYield = dividendYield;
  }

  accept(visitor) {
    visitor.visitStock(this);
  }
}
```

### Bond (Concrete Element)

```javascript
// OOP role: Polymorphism (implements Element via accept)
// SOLID: SRP (represents only bond data), LSP (substitutable for Element)
class Bond {
  constructor(name, faceValue, couponRate, yearsToMaturity) {
    this.name = name;
    this.faceValue = faceValue;
    this.couponRate = couponRate;
    this.yearsToMaturity = yearsToMaturity;
  }

  accept(visitor) {
    visitor.visitBond(this);
  }
}
```

### ValueVisitor (Concrete Visitor)

```javascript
// OOP role: Polymorphism (extends Visitor)
// SOLID: SRP (only calculates total value), OCP (new analysis = new subclass)
class ValueVisitor extends Visitor {
  #totalValue = 0;

  visitStock(stock) {
    this.#totalValue += stock.shares * stock.pricePerShare;
  }

  visitBond(bond) {
    this.#totalValue += bond.faceValue;
  }

  getTotalValue() {
    return this.#totalValue;
  }
}
```

### DividendVisitor (Concrete Visitor)

```javascript
// OOP role: Polymorphism (extends Visitor)
// SOLID: SRP (only calculates annual income)
class DividendVisitor extends Visitor {
  #totalDividend = 0;

  visitStock(stock) {
    const value = stock.shares * stock.pricePerShare;
    this.#totalDividend += value * stock.dividendYield;
  }

  visitBond(bond) {
    this.#totalDividend += bond.faceValue * bond.couponRate;
  }

  getTotalDividend() {
    return this.#totalDividend;
  }
}
```

### RiskVisitor (Concrete Visitor)

```javascript
// OOP role: Polymorphism (extends Visitor)
// SOLID: SRP (only assesses risk)
class RiskVisitor extends Visitor {
  #results = [];

  visitStock(stock) {
    if (stock.dividendYield > 0.04) {
      this.#results.push(`${stock.ticker}: LOW RISK -- established dividend payer`);
    } else if (stock.dividendYield > 0.01) {
      this.#results.push(`${stock.ticker}: MEDIUM RISK -- moderate dividend`);
    } else {
      this.#results.push(`${stock.ticker}: HIGH RISK -- low or no dividend`);
    }
  }

  visitBond(bond) {
    if (bond.yearsToMaturity > 20) {
      this.#results.push(`${bond.name}: MEDIUM RISK -- long maturity`);
    } else if (bond.yearsToMaturity > 10) {
      this.#results.push(`${bond.name}: LOW RISK -- medium maturity`);
    } else {
      this.#results.push(`${bond.name}: LOW RISK -- short maturity`);
    }
  }

  getResults() {
    return [...this.#results];
  }
}
```

### ReportVisitor (Concrete Visitor)

```javascript
// OOP role: Polymorphism (extends Visitor)
// SOLID: SRP (only generates formatted report)
class ReportVisitor extends Visitor {
  #lines = [];

  visitStock(stock) {
    const value = stock.shares * stock.pricePerShare;
    const income = value * stock.dividendYield;
    const line =
      `| ${stock.ticker.padEnd(6)} | ${stock.companyName.padEnd(20)} ` +
      `| ${String(stock.shares).padStart(5)}  ` +
      `| $${stock.pricePerShare.toFixed(2).padStart(8)} ` +
      `| $${value.toFixed(2).padStart(8)} ` +
      `| $${income.toFixed(2).padStart(8)} |`;
    this.#lines.push(line);
  }

  visitBond(bond) {
    const income = bond.faceValue * bond.couponRate;
    const line =
      `| ${bond.name.padEnd(6)} | ${"Bond".padEnd(20)} ` +
      `| ${String(bond.yearsToMaturity).padStart(2)}yr  ` +
      `| $${bond.faceValue.toFixed(2).padStart(8)} ` +
      `| $${bond.faceValue.toFixed(2).padStart(8)} ` +
      `| $${income.toFixed(2).padStart(8)} |`;
    this.#lines.push(line);
  }

  getReport() {
    const header = "\nPortfolio Report:";
    const columns =
      "| Ticker  | Name                 | Qty/Mat | Price/Val  | Mkt Value  | Income    |";
    const separator =
      "|---------+----------------------+---------+------------+------------+-----------|";
    return [header, columns, separator, ...this.#lines].join("\n");
  }
}
```

### Portfolio (ObjectStructure)

```javascript
// OOP role: Composition (aggregates elements)
// SOLID: SRP (manages collection only), DIP (depends on Element abstraction)
class Portfolio {
  #instruments = [];

  add(instrument) {
    this.#instruments.push(instrument);
  }

  acceptAll(visitor) {
    for (const instrument of this.#instruments) {
      instrument.accept(visitor);
    }
  }
}
```

### Main / Driver

```javascript
const portfolio = new Portfolio();

// Add stocks with realistic data
portfolio.add(new Stock("AAPL", "Apple Inc.", 100, 175.50, 0.005));
portfolio.add(new Stock("GOOGL", "Alphabet Inc.", 50, 140.25, 0.004));
portfolio.add(new Stock("MSFT", "Microsoft Corp.", 80, 380.75, 0.007));
portfolio.add(new Stock("JPM", "JPMorgan Chase", 60, 155.30, 0.048));

// Add bonds
portfolio.add(new Bond("US10Y", 10000.0, 0.043, 10));
portfolio.add(new Bond("US30Y", 5000.0, 0.048, 30));

// 1. ValueVisitor -- calculate total portfolio value
const valueVisitor = new ValueVisitor();
portfolio.acceptAll(valueVisitor);
console.log(`Total Portfolio Value: $${valueVisitor.getTotalValue().toFixed(2)}`);

// 2. DividendVisitor -- calculate annual dividend / coupon income
const divVisitor = new DividendVisitor();
portfolio.acceptAll(divVisitor);
console.log(`Annual Dividend/Coupon Income: $${divVisitor.getTotalDividend().toFixed(2)}`);

// 3. RiskVisitor -- assess risk for each holding
const riskVisitor = new RiskVisitor();
portfolio.acceptAll(riskVisitor);
console.log("\nRisk Assessment:");
for (const result of riskVisitor.getResults()) {
  console.log("  " + result);
}

// 4. ReportVisitor -- generate formatted portfolio report
const reportVisitor = new ReportVisitor();
portfolio.acceptAll(reportVisitor);
console.log(reportVisitor.getReport());
```

### Language-Specific Notes

- **Dynamic dispatch via duck typing** -- JavaScript does not have formal interfaces. The `Element` protocol is enforced by convention: any object with an `accept(visitor)` method can be used as an element. This makes the Visitor pattern lightweight to implement but requires discipline to maintain contract compliance.
- **Private fields with `#`** -- ES2022+ private class fields (`#totalValue`, `#results`) provide true encapsulation. Visitors accumulate state privately, preventing external code from modifying their internal counters or result arrays.
- **`extends` for visitor hierarchy** -- JavaScript's `extends` keyword creates the Visitor class hierarchy. Each concrete visitor overrides `visitStock()` and `visitBond()` with its specific algorithm. The base class throws errors to catch unimplemented methods at development time.
- **Array spread for immutable access** -- `RiskVisitor.getResults()` returns `[...this.#results]` (a copy) rather than the internal array reference, protecting visitor state from external mutation.
- **`padStart`/`padEnd` for column alignment** -- JavaScript's string padding methods replace format specifiers found in other languages. The `ReportVisitor` uses these to align the table columns without external libraries.
- **No type safety at compile time** -- Unlike TypeScript, Java, or Rust, JavaScript provides no compile-time checking that visitors implement all required methods. Runtime type checking or thorough testing is essential to catch missing method implementations.
