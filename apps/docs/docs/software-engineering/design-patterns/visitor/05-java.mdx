## Implementation

Below is a complete Java 17+ implementation of the Visitor pattern using a stock market portfolio domain. The `Portfolio` holds a heterogeneous collection of `Stock` and `Bond` objects (both implementing the `Element` interface), and four visitor implementations perform distinct analyses: computing total market value, calculating annual dividend income, assessing risk levels, and generating a formatted portfolio report. Java's interface system and method overloading provide compile-time type safety for the double-dispatch mechanism.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌──────────────────────────────────────────────┐
//   │   <<interface>> Visitor                      │  <- OOP: Abstraction
//   ├──────────────────────────────────────────────┤     ISP: segregated per type
//   │   + void visitStock(Stock stock)             │
//   │   + void visitBond(Bond bond)                │
//   └────────────────┬─────────────────────────────┘
//                    ▲ implements
//   ┌────────────────┴─────────────────────────────┐
//   │                                              │
//   ┌────────────────────────┐   ┌──────────────────────────┐
//   │ ValueVisitor           │   │ DividendVisitor          │  <- OOP: Polymorphism
//   ├────────────────────────┤   ├──────────────────────────┤     SRP: one concern
//   │ - double totalValue    │   │ - double totalDividend   │     OCP: new = new impl
//   │ visitStock(s) { ... }  │   │ visitStock(s) { ... }    │
//   │ visitBond(b) { ... }   │   │ visitBond(b) { ... }     │
//   └────────────────────────┘   └──────────────────────────┘
//
//   ┌──────────────────────────────────────────────┐
//   │   <<interface>> Element                      │  <- OOP: Abstraction
//   ├──────────────────────────────────────────────┤     ISP: single accept method
//   │   + void accept(Visitor visitor)             │
//   └────────────────┬─────────────────────────────┘
//                    ▲ implements
//   ┌────────────────┴─────────────────────────────┐
//   │                                              │
//   ┌────────────────────────┐   ┌──────────────────────────┐
//   │     Stock              │   │      Bond                │  <- OOP: Polymorphism
//   ├────────────────────────┤   ├──────────────────────────┤     LSP: both
//   │ accept(v) {            │   │ accept(v) {              │     substitutable
//   │   v.visitStock(this);  │   │   v.visitBond(this);     │     for Element
//   │ }                      │   │ }                        │
//   └────────────────────────┘   └──────────────────────────┘
```

#### OOP Concepts Applied

1. **Abstraction** -- The `Visitor` and `Element` interfaces define the contracts. All client code depends on these abstractions rather than concrete types.
2. **Encapsulation** -- Each class hides its internal state. Visitors use `private` fields (`totalValue`, `results`, `lines`) that are only accessible within the class.
3. **Inheritance (via `implements`)** -- `Stock` and `Bond` implement the `Element` interface. Each provides its own `accept` method that dispatches to the correct visitor method.
4. **Polymorphism** -- `Portfolio.acceptAll()` calls `accept()` on each `Element` without knowing the concrete type. Method overloading on `visitStock`/`visitBond` resolves the correct operation.

#### SOLID Principles Applied

5. **SRP** -- Each visitor class has a single responsibility: value calculation, dividend computation, risk assessment, or report generation. The `Portfolio` only manages the element collection.
6. **OCP** -- Adding a new analysis requires only a new class implementing `Visitor`. No existing code needs modification.
7. **LSP** -- Any `Visitor` implementation can be passed to `Portfolio.acceptAll()` without breaking the iteration. `Stock` and `Bond` are fully substitutable for `Element`.
8. **ISP** -- The `Visitor` interface has exactly two methods -- one per visited element type. No implementor is forced to provide unused methods.
9. **DIP** -- `Portfolio` depends on the `Element` abstraction. Visitors depend on the `Visitor` interface contract. High-level code depends on abstractions, not concretions.

### Element (Interface)

```java
// OOP role: Abstraction
// SOLID: ISP (minimal contract), DIP (depended upon abstraction)
interface Element {
    void accept(Visitor visitor);
}
```

### Visitor (Interface)

```java
// OOP role: Abstraction
// SOLID: ISP (one method per element type), OCP (extensible)
interface Visitor {
    void visitStock(Stock stock);
    void visitBond(Bond bond);
}
```

### Stock (Concrete Element)

```java
// OOP role: Polymorphism (implements Element)
// SOLID: SRP (represents only stock data), LSP (substitutable for Element)
class Stock implements Element {
    private final String ticker;
    private final String companyName;
    private final int shares;
    private final double pricePerShare;
    private final double dividendYield;

    public Stock(String ticker, String companyName, int shares,
                 double pricePerShare, double dividendYield) {
        this.ticker = ticker;
        this.companyName = companyName;
        this.shares = shares;
        this.pricePerShare = pricePerShare;
        this.dividendYield = dividendYield;
    }

    public String getTicker() { return ticker; }
    public String getCompanyName() { return companyName; }
    public int getShares() { return shares; }
    public double getPricePerShare() { return pricePerShare; }
    public double getDividendYield() { return dividendYield; }

    @Override
    public void accept(Visitor visitor) {
        visitor.visitStock(this);
    }
}
```

### Bond (Concrete Element)

```java

class Bond implements Element {
```

### ValueVisitor (Concrete Visitor)

```java
// OOP role: Polymorphism (implements Visitor)
// SOLID: SRP (only calculates total value), OCP (new analysis = new class)
class ValueVisitor implements Visitor {
    private double totalValue = 0.0;

    @Override
    public void visitStock(Stock stock) {
        totalValue += stock.getShares() * stock.getPricePerShare();
    }

    @Override
    public void visitBond(Bond bond) {
        totalValue += bond.getFaceValue();
    }

    public double getTotalValue() {
        return totalValue;
    }
}
```

### DividendVisitor (Concrete Visitor)

```java
// OOP role: Polymorphism (implements Visitor)
// SOLID: SRP (only calculates annual income)
class DividendVisitor implements Visitor {
    private double totalDividend = 0.0;

    @Override
    public void visitStock(Stock stock) {
        double value = stock.getShares() * stock.getPricePerShare();
        totalDividend += value * stock.getDividendYield();
    }

    @Override
    public void visitBond(Bond bond) {
        totalDividend += bond.getFaceValue() * bond.getCouponRate();
    }

    public double getTotalDividend() {
        return totalDividend;
    }
}
```

### RiskVisitor (Concrete Visitor)

```java
// OOP role: Polymorphism (implements Visitor)
// SOLID: SRP (only assesses risk)
class RiskVisitor implements Visitor {
    private final List<String> results = new ArrayList<>();

    @Override
    public void visitStock(Stock stock) {
        if (stock.getDividendYield() > 0.04) {
            results.add(stock.getTicker()
                + ": LOW RISK -- established dividend payer");
        } else if (stock.getDividendYield() > 0.01) {
            results.add(stock.getTicker()
                + ": MEDIUM RISK -- moderate dividend");
        } else {
            results.add(stock.getTicker()
                + ": HIGH RISK -- low or no dividend");
        }
    }

    @Override
    public void visitBond(Bond bond) {
        if (bond.getYearsToMaturity() > 20) {
            results.add(bond.getName()
                + ": MEDIUM RISK -- long maturity");
        } else if (bond.getYearsToMaturity() > 10) {
            results.add(bond.getName()
                + ": LOW RISK -- medium maturity");
        } else {
            results.add(bond.getName()
                + ": LOW RISK -- short maturity");
        }
    }

    public List<String> getResults() {
        return new ArrayList<>(results);
    }
}
```

### ReportVisitor (Concrete Visitor)

```java
// OOP role: Polymorphism (implements Visitor)
// SOLID: SRP (only generates formatted report)
class ReportVisitor implements Visitor {
    private final List<String> lines = new ArrayList<>();

    @Override
    public void visitStock(Stock stock) {
        double value = stock.getShares() * stock.getPricePerShare();
        double income = value * stock.getDividendYield();
        String line = String.format(
            "| %-6s | %-20s | %5d  | $%8.2f | $%8.2f | $%8.2f |",
            stock.getTicker(), stock.getCompanyName(),
            stock.getShares(), stock.getPricePerShare(), value, income
        );
        lines.add(line);
    }

    @Override
    public void visitBond(Bond bond) {
        double income = bond.getFaceValue() * bond.getCouponRate();
        String line = String.format(
            "| %-6s | %-20s | %2dyr  | $%8.2f | $%8.2f | $%8.2f |",
            bond.getName(), "Bond",
            bond.getYearsToMaturity(), bond.getFaceValue(),
            bond.getFaceValue(), income
        );
        lines.add(line);
    }

    public String getReport() {
        StringBuilder sb = new StringBuilder();
        sb.append("\nPortfolio Report:\n");
        sb.append("| Ticker  | Name                 | Qty/Mat ");
        sb.append("| Price/Val  | Mkt Value  | Income    |\n");
        sb.append("|---------+----------------------+---------");
        sb.append("+------------+------------+-----------|\n");
        for (String line : lines) {
            sb.append(line).append("\n");
        }
        return sb.toString();
    }
}
```

### Portfolio (ObjectStructure)

```java
// OOP role: Composition (aggregates elements)
// SOLID: SRP (manages collection only), DIP (depends on Element abstraction)
class Portfolio {
    private final List<Element> elements = new ArrayList<>();

    public void add(Element element) {
        elements.add(element);
    }

    public void acceptAll(Visitor visitor) {
        for (Element element : elements) {
            element.accept(visitor);
        }
    }
}
```

### Main / Driver

```java
public class VisitorDemo {
    public static void main(String[] args) {
        Portfolio portfolio = new Portfolio();

        // Add stocks with realistic data
        portfolio.add(new Stock("AAPL", "Apple Inc.", 100, 175.50, 0.005));
        portfolio.add(new Stock("GOOGL", "Alphabet Inc.", 50, 140.25, 0.004));
        portfolio.add(new Stock("MSFT", "Microsoft Corp.", 80, 380.75, 0.007));
        portfolio.add(new Stock("JPM", "JPMorgan Chase", 60, 155.30, 0.048));

        // Add bonds
        portfolio.add(new Bond("US10Y", 10000.0, 0.043, 10));
        portfolio.add(new Bond("US30Y", 5000.0, 0.048, 30));

        // 1. ValueVisitor -- calculate total portfolio value
        ValueVisitor valueVisitor = new ValueVisitor();
        portfolio.acceptAll(valueVisitor);
        System.out.printf("Total Portfolio Value: $%.2f%n",
            valueVisitor.getTotalValue());

        // 2. DividendVisitor -- calculate annual income
        DividendVisitor divVisitor = new DividendVisitor();
        portfolio.acceptAll(divVisitor);
        System.out.printf("Annual Dividend/Coupon Income: $%.2f%n",
            divVisitor.getTotalDividend());

        // 3. RiskVisitor -- assess risk for each holding
        RiskVisitor riskVisitor = new RiskVisitor();
        portfolio.acceptAll(riskVisitor);
        System.out.println("\nRisk Assessment:");
        for (String result : riskVisitor.getResults()) {
            System.out.println("  " + result);
        }

        // 4. ReportVisitor -- generate formatted portfolio report
        ReportVisitor reportVisitor = new ReportVisitor();
        portfolio.acceptAll(reportVisitor);
        System.out.println(reportVisitor.getReport());
    }
}
```

### Language-Specific Notes

- **Interfaces for contract enforcement** -- Java's `interface` keyword provides compile-time enforcement of the Visitor contract. The `@Override` annotation ensures that methods actually override an interface method, catching typos and signature mismatches at compile time.
- **`final` fields for immutability** -- The `Stock` and `Bond` classes use `private final` fields, ensuring element data is immutable after construction. Visitors read data through getter methods but cannot modify the elements. This is a stronger guarantee than Python's convention-based approach.
- **`String.format()` for column alignment** -- Java's `String.format()` with format specifiers (`%-6s`, `%5d`, `$%8.2f`) handles column alignment for the report output. This is the standard Java approach for formatted console output, analogous to `printf` in C.
- **`ArrayList` with defensive copy** -- `RiskVisitor.getResults()` returns `new ArrayList<>(results)` instead of the internal list reference, preventing callers from modifying the visitor's accumulated state. This preserves encapsulation.
- **Method overloading for dispatch** -- Java resolves `visitStock` vs `visitBond` through method overloading. The `accept` method on each concrete element calls the correct overload based on `this`'s compile-time type, achieving double dispatch.
- **Single public class with inner classes** -- All classes are package-private, with only `VisitorDemo` being public (matching the filename). This keeps the example self-contained in a single file while demonstrating proper Java access control.
