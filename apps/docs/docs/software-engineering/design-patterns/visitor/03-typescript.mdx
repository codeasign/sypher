## Implementation

Below is a complete TypeScript implementation of the Visitor pattern using a stock market portfolio domain. The `Portfolio` holds a heterogeneous collection of `Stock` and `Bond` objects (both implementing the `Element` interface), and four visitor implementations perform distinct analyses: computing total market value, calculating annual dividend income, assessing risk levels, and generating a formatted portfolio report. TypeScript's interface system provides compile-time type safety for the double-dispatch mechanism.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌────────────────────────────────────────┐
//   │   interface Visitor                    │  <- OOP: Abstraction
//   ├────────────────────────────────────────┤     ISP: segregated per type
//   │   visitStock(stock: Stock): void       │
//   │   visitBond(bond: Bond): void          │
//   └────────────────┬───────────────────────┘
//                    ▲ implements
//   ┌────────────────┴───────────────────────┐
//   │                                        │
//   ┌────────────────────┐   ┌──────────────────────────┐
//   │ ValueVisitor       │   │ DividendVisitor          │  <- OOP: Polymorphism
//   ├────────────────────┤   ├──────────────────────────┤     SRP: one concern
//   │ -totalValue: number│   │ -totalDividend: number   │     OCP: new = new impl
//   │ visitStock(s) {...}│   │ visitStock(s) {...}      │
//   │ visitBond(b) {...} │   │ visitBond(b) {...}       │
//   └────────────────────┘   └──────────────────────────┘
//
//   ┌────────────────────────────────────────┐
//   │   interface Element                    │  <- OOP: Abstraction
//   ├────────────────────────────────────────┤     ISP: single accept method
//   │   accept(visitor: Visitor): void       │
//   └────────────────┬───────────────────────┘
//                    ▲ implements
//   ┌────────────────┴───────────────────────┐
//   │                                        │
//   ┌────────────────────┐   ┌──────────────────────────┐
//   │     Stock          │   │      Bond                │  <- OOP: Polymorphism
//   ├────────────────────┤   ├──────────────────────────┤     LSP: both
//   │ accept(v) {        │   │ accept(v) {              │     substitutable
//   │   v.visitStock(this)│   │   v.visitBond(this)     │     for Element
//   │ }                  │   │ }                        │
//   └────────────────────┘   └──────────────────────────┘
```

#### OOP Concepts Applied

1. **Abstraction** -- The `Visitor` and `Element` interfaces define the contracts. All client code depends on these abstractions rather than concrete types.
2. **Encapsulation** -- Each class hides its internal state. Visitors use private fields (`#totalValue`, `private results: string[]`) that are only accessible within the class.
3. **Inheritance (via `implements`)** -- `Stock` and `Bond` implement the `Element` interface. Each provides its own `accept` method that dispatches to the correct visitor method.
4. **Polymorphism** -- The `Portfolio.acceptAll()` method calls `accept()` on each `Element` without knowing the concrete type. Through double dispatch, the specific `visitStock` or `visitBond` method runs.

#### SOLID Principles Applied

5. **SRP** -- Each visitor class has a single responsibility: value calculation, dividend computation, risk assessment, or report generation. The `Portfolio` only manages the element collection.
6. **OCP** -- Adding a new analysis requires only a new class implementing the `Visitor` interface. No existing code needs modification.
7. **LSP** -- Any `Visitor` implementation can be passed to `Portfolio.acceptAll()` without breaking the iteration. `Stock` and `Bond` are fully substitutable for `Element`.
8. **ISP** -- The `Visitor` interface has exactly two methods -- one per element type. No implementor is forced to provide unused methods.
9. **DIP** -- `Portfolio` depends on the `Element` abstraction. Visitors depend on the `Visitor` interface contract. High-level analysis code depends on abstractions, not concretions.

### Element (Interface)

```typescript
// OOP role: Abstraction
// SOLID: ISP (minimal contract), DIP (depended upon abstraction)
interface Element {
  accept(visitor: Visitor): void;
}
```

### Visitor (Interface)

```typescript
// OOP role: Abstraction
// SOLID: ISP (one method per element type), OCP (extensible)
interface Visitor {
  visitStock(stock: Stock): void;
  visitBond(bond: Bond): void;
}
```

### Stock (Concrete Element)

```typescript
// OOP role: Polymorphism (implements Element)
// SOLID: SRP (represents only stock data), LSP (substitutable for Element)
class Stock implements Element {
  constructor(
    public readonly ticker: string,
    public readonly companyName: string,
    public readonly shares: number,
    public readonly pricePerShare: number,
    public readonly dividendYield: number
  ) {}

  accept(visitor: Visitor): void {
    visitor.visitStock(this);
  }
}
```

### Bond (Concrete Element)

```typescript
// OOP role: Polymorphism (implements Element)
// SOLID: SRP (represents only bond data), LSP (substitutable for Element)
class Bond implements Element {
  constructor(
    public readonly name: string,
    public readonly faceValue: number,
    public readonly couponRate: number,
    public readonly yearsToMaturity: number
  ) {}

  accept(visitor: Visitor): void {
    visitor.visitBond(this);
  }
}
```

### ValueVisitor (Concrete Visitor)

```typescript
// OOP role: Polymorphism (implements Visitor)
// SOLID: SRP (only calculates total value), OCP (new analysis = new class)
class ValueVisitor implements Visitor {
  private totalValue = 0;

  visitStock(stock: Stock): void {
    this.totalValue += stock.shares * stock.pricePerShare;
  }

  visitBond(bond: Bond): void {
    this.totalValue += bond.faceValue;
  }

  getTotalValue(): number {
    return this.totalValue;
  }
}
```

### DividendVisitor (Concrete Visitor)

```typescript
// OOP role: Polymorphism (implements Visitor)
// SOLID: SRP (only calculates annual income)
class DividendVisitor implements Visitor {
  private totalDividend = 0;

  visitStock(stock: Stock): void {
    const value = stock.shares * stock.pricePerShare;
    this.totalDividend += value * stock.dividendYield;
  }

  visitBond(bond: Bond): void {
    this.totalDividend += bond.faceValue * bond.couponRate;
  }

  getTotalDividend(): number {
    return this.totalDividend;
  }
}
```

### RiskVisitor (Concrete Visitor)

```typescript
// OOP role: Polymorphism (implements Visitor)
// SOLID: SRP (only assesses risk)
class RiskVisitor implements Visitor {
  private results: string[] = [];

  visitStock(stock: Stock): void {
    if (stock.dividendYield > 0.04) {
      this.results.push(`${stock.ticker}: LOW RISK -- established dividend payer`);
    } else if (stock.dividendYield > 0.01) {
      this.results.push(`${stock.ticker}: MEDIUM RISK -- moderate dividend`);
    } else {
      this.results.push(`${stock.ticker}: HIGH RISK -- low or no dividend`);
    }
  }

  visitBond(bond: Bond): void {
    if (bond.yearsToMaturity > 20) {
      this.results.push(`${bond.name}: MEDIUM RISK -- long maturity`);
    } else if (bond.yearsToMaturity > 10) {
      this.results.push(`${bond.name}: LOW RISK -- medium maturity`);
    } else {
      this.results.push(`${bond.name}: LOW RISK -- short maturity`);
    }
  }

  getResults(): readonly string[] {
    return this.results;
  }
}
```

### ReportVisitor (Concrete Visitor)

```typescript
// OOP role: Polymorphism (implements Visitor)
// SOLID: SRP (only generates formatted report)
class ReportVisitor implements Visitor {
  private lines: string[] = [];

  visitStock(stock: Stock): void {
    const value = stock.shares * stock.pricePerShare;
    const income = value * stock.dividendYield;
    const line =
      `| ${stock.ticker.padEnd(6)} | ${stock.companyName.padEnd(20)} ` +
      `| ${String(stock.shares).padStart(5)}  ` +
      `| $${stock.pricePerShare.toFixed(2).padStart(8)} ` +
      `| $${value.toFixed(2).padStart(8)} ` +
      `| $${income.toFixed(2).padStart(8)} |`;
    this.lines.push(line);
  }

  visitBond(bond: Bond): void {
    const income = bond.faceValue * bond.couponRate;
    const line =
      `| ${bond.name.padEnd(6)} | ${"Bond".padEnd(20)} ` +
      `| ${String(bond.yearsToMaturity).padStart(2)}yr  ` +
      `| $${bond.faceValue.toFixed(2).padStart(8)} ` +
      `| $${bond.faceValue.toFixed(2).padStart(8)} ` +
      `| $${income.toFixed(2).padStart(8)} |`;
    this.lines.push(line);
  }

  getReport(): string {
    const header = "\nPortfolio Report:";
    const columns =
      "| Ticker  | Name                 | Qty/Mat | Price/Val  | Mkt Value  | Income    |";
    const separator =
      "|---------+----------------------+---------+------------+------------+-----------|";
    return [header, columns, separator, ...this.lines].join("\n");
  }
}
```

### Portfolio (ObjectStructure)

```typescript
// OOP role: Composition (aggregates elements)
// SOLID: SRP (manages collection only), DIP (depends on Element abstraction)
class Portfolio {
  private elements: Element[] = [];

  add(element: Element): void {
    this.elements.push(element);
  }

  acceptAll(visitor: Visitor): void {
    for (const element of this.elements) {
      element.accept(visitor);
    }
  }
}
```

### Main / Driver

```typescript
const portfolio = new Portfolio();

// Add stocks with realistic data
portfolio.add(new Stock("AAPL", "Apple Inc.", 100, 175.50, 0.005));
portfolio.add(new Stock("GOOGL", "Alphabet Inc.", 50, 140.25, 0.004));
portfolio.add(new Stock("MSFT", "Microsoft Corp.", 80, 380.75, 0.007));
portfolio.add(new Stock("JPM", "JPMorgan Chase", 60, 155.30, 0.048));

// Add bonds
portfolio.add(new Bond("US10Y", 10000.0, 0.043, 10));
portfolio.add(new Bond("US30Y", 5000.0, 0.048, 30));

// 1. ValueVisitor -- calculate total portfolio value
const valueVisitor = new ValueVisitor();
portfolio.acceptAll(valueVisitor);
console.log(`Total Portfolio Value: $${valueVisitor.getTotalValue().toFixed(2)}`);

// 2. DividendVisitor -- calculate annual income
const divVisitor = new DividendVisitor();
portfolio.acceptAll(divVisitor);
console.log(`Annual Dividend/Coupon Income: $${divVisitor.getTotalDividend().toFixed(2)}`);

// 3. RiskVisitor -- assess risk for each holding
const riskVisitor = new RiskVisitor();
portfolio.acceptAll(riskVisitor);
console.log("\nRisk Assessment:");
for (const result of riskVisitor.getResults()) {
  console.log("  " + result);
}

// 4. ReportVisitor -- generate formatted portfolio report
const reportVisitor = new ReportVisitor();
portfolio.acceptAll(reportVisitor);
console.log(reportVisitor.getReport());
```

### Language-Specific Notes

- **Interfaces for compile-time safety** -- TypeScript's `interface` keyword provides compile-time enforcement of the Visitor contract. If a class claims to `implements Visitor` but omits `visitStock` or `visitBond`, the TypeScript compiler reports an error. This prevents the "missing method" bugs common in plain JavaScript implementations.
- **`readonly` modifier for immutability** -- The `Stock` and `Bond` classes use `public readonly` constructor parameters. This pattern (shorthand for declaring and assigning a field) ensures element data is immutable after construction. Visitors read data but cannot modify it, preserving encapsulation.
- **`private` access modifier** -- TypeScript's `private` keyword (plus the newer `#` private field syntax) enforces encapsulation at compile time. Visitor internal state (`totalValue`, `results`, `lines`) is not accessible from outside the class.
- **`readonly` return type** -- `RiskVisitor.getResults()` returns `readonly string[]` to signal that callers should not mutate the returned array. This is a compile-time contract that prevents accidental modification of visitor state.
- **Template literal types and `padStart`** -- TypeScript inherits JavaScript's string formatting capabilities. Template literals with `${}` provide string interpolation, while `padStart`/`padEnd` handle column alignment for the report output.
- **Interfaces are erased at runtime** -- TypeScript's type system is erased during compilation. The Visitor pattern's double-dispatch works through JavaScript's runtime dispatch, not TypeScript types. The interfaces provide compile-time safety, but runtime behavior is identical to the JavaScript implementation.
