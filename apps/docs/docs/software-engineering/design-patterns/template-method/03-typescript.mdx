## Implementation

Below is a complete TypeScript implementation of the Template Method pattern using a stock market trading domain. The abstract `TradingStrategy` class defines the `executeTrade()` template method with generic type safety. Concrete strategies `MovingAverageCrossover` and `MeanReversion` implement the primitive operations with full type annotations. The driver exercises both strategies with realistic AAPL, GOOGL, and MSFT price data and demonstrates type-safe signal handling via discriminated unions.

### How the Implementation Works

#### TypeScript-Specific Design

TypeScript brings several advantages to the Template Method pattern that plain JavaScript cannot provide:

- **Compile-time safety for primitive operations** — The `abstract` keyword enforces that subclasses implement `analyzeMarket()` and `calculatePositionSize()` at compile time, not just at runtime.
- **Discriminated union for `TradingDecision`** — The return type of `analyzeMarket()` uses a discriminated union to distinguish BUY, SELL, and HOLD signals, giving exhaustive type checking in client code.
- **Generic type parameter** — The strategy can accept typed price data (`number[]`) while remaining flexible enough to accommodate different data shapes across strategies.

#### OOP Concepts Applied

1. **Abstraction** — `TradingStrategy` defines the algorithm contract via abstract method signatures. Subclasses provide concrete implementations without exposing internal computation details.
2. **Encapsulation** — Each strategy hides its window sizes, lookback periods, and threshold constants as `private readonly` fields, inaccessible to external code.
3. **Inheritance** — Both concrete strategies inherit the `executeTrade()` skeleton, `placeOrder()`, and `logTrade()`.
4. **Polymorphism** — The driver iterates over `TradingStrategy<number[]>[]` and calls `executeTrade()` polymorphically.

#### SOLID Principles Applied

5. **SRP** — The algorithm structure lives in one place (the template method). Analysis logic lives in each concrete strategy. No class does more than one thing.
6. **OCP** — New strategies (e.g., `BollingerBandsStrategy`) are added as new subclasses without modifying the base class or existing strategies.
7. **LSP** — Every concrete strategy can substitute `TradingStrategy<number[]>` in client code. The contract guarantees that `executeTrade()` processes prices, computes a decision, and logs the result.
8. **ISP** — The primitive operation interface has exactly two required methods. Hooks have default no-op implementations. No class implements methods it does not need.
9. **DIP** — The driver depends on the abstract `TradingStrategy` type. Low-level analysis logic is injected via subclass polymorphism, not hardcoded in the client.

### Type Definitions

```typescript
// Template Method Pattern — TypeScript — Stock Market example
// ==========================================================
// Participants: TradingStrategy (AbstractClass),
// MovingAverageCrossover, MeanReversion (ConcreteClasses)
// OOP: abstraction, inheritance, polymorphism, encapsulation
// SOLID: SRP, OCP, LSP, ISP, DIP
// ==========================================================

// ── Type Definitions ──

type Signal = -1 | 0 | 1;

interface TradingDecision {
  signal: Signal;
  confidence: number;
}

interface Order {
  ticker: string;
  shares: number;
  signal: Signal;
  orderId: string;
  timestamp: string;
  status: 'FILLED' | 'REJECTED' | 'PENDING';
}

type PriceData = number[];
```

### TradingStrategy (AbstractClass)

```typescript
// ── AbstractClass: TradingStrategy ──
// OOP role: Abstraction + ISP (minimal required contract)
// SOLID: ISP, DIP
abstract class TradingStrategy {
  protected readonly MIN_CONFIDENCE = 0.15;
  protected readonly MAX_POSITION = 1000;

  // ── Template Method ──
  // 'final' equivalent: TypeScript has no native final keyword,
  // but making it non-abstract signals it should not be overridden.
  executeTrade(ticker: string, priceData: PriceData): void {
    const { signal, confidence } = this.analyzeMarket(priceData);

    // Hook 1: Pre-validate the signal
    if (!this.preValidate(signal, confidence)) {
      console.log(
        `[${ticker}] Trade rejected by pre-validation ` +
        `(confidence: ${confidence.toFixed(4)})`
      );
      return;
    }

    const shares = this.calculatePositionSize(confidence, this.MAX_POSITION);
    if (shares <= 0) {
      console.log(`[${ticker}] No trade: position size is zero`);
      return;
    }

    // Hook 2: Apply risk controls
    if (!this.applyRiskControls(ticker, shares)) {
      console.log(
        `[${ticker}] Trade rejected by risk controls (${shares} shares)`
      );
      return;
    }

    const order = this.placeOrder(ticker, shares, signal);
    this.logTrade(ticker, shares, signal, confidence, order);
  }

  // ── Primitive Operations ──
  protected abstract analyzeMarket(priceData: PriceData): TradingDecision;
  protected abstract calculatePositionSize(
    confidence: number,
    maxPosition: number
  ): number;

  // ── Hooks (optional overrides with defaults) ──
  protected preValidate(_signal: Signal, confidence: number): boolean {
    return confidence >= this.MIN_CONFIDENCE;
  }

  protected applyRiskControls(_ticker: string, _shares: number): boolean {
    return true;
  }

  // ── Concrete Methods ──
  protected placeOrder(
    ticker: string,
    shares: number,
    signal: Signal
  ): Order {
    return {
      ticker,
      shares,
      signal,
      orderId: `ORD-${Date.now()}`,
      timestamp: new Date().toISOString(),
      status: 'FILLED'
    };
  }

  protected logTrade(
    ticker: string,
    shares: number,
    signal: Signal,
    confidence: number,
    order: Order
  ): void {
    const ts = new Date().toLocaleString('en-US', { hour12: false });
    const direction = signal === 1 ? 'BUY' : signal === -1 ? 'SELL' : 'HOLD';
    console.log(
      `[${ts}] ${ticker}: ${direction} ${shares} shares ` +
      `(confidence: ${confidence.toFixed(4)}) — ` +
      `Order#${order.orderId} [${order.status}]`
    );
  }
}
```

### MovingAverageCrossover (ConcreteClass)

```typescript
// ── ConcreteClass: MovingAverageCrossover ──
// OOP role: Inheritance/Polymorphism
// SOLID: LSP, OCP, SRP
class MovingAverageCrossover extends TradingStrategy {
  private readonly SHORT_WINDOW = 3;
  private readonly LONG_WINDOW = 8;

  protected analyzeMarket(priceData: PriceData): TradingDecision {
    if (priceData.length < this.LONG_WINDOW) {
      return { signal: 0, confidence: 0 };
    }

    const shortMA = this.movingAverage(priceData, this.SHORT_WINDOW);
    const longMA = this.movingAverage(priceData, this.LONG_WINDOW);

    // Check for crossover
    const prevSlice = priceData.slice(-this.LONG_WINDOW - 1, -1);
    const prevShort = this.movingAverage(prevSlice, this.SHORT_WINDOW);
    const prevLong = this.movingAverage(prevSlice, this.LONG_WINDOW);

    if (shortMA > longMA && prevShort <= prevLong) {
      const confidence = Math.min((shortMA - longMA) / longMA, 1.0);
      return { signal: 1, confidence };
    } else if (shortMA < longMA && prevShort >= prevLong) {
      const confidence = Math.min((longMA - shortMA) / longMA, 1.0);
      return { signal: -1, confidence };
    }

    return { signal: 0, confidence: 0 };
  }

  private movingAverage(data: PriceData, window: number): number {
    const slice = data.slice(-window);
    return slice.reduce((a, b) => a + b, 0) / slice.length;
  }

  protected calculatePositionSize(
    confidence: number,
    maxPosition: number
  ): number {
    return Math.floor(maxPosition * Math.min(confidence * 1.2, 1.0));
  }
}
```

### MeanReversion (ConcreteClass)

```typescript
// ── ConcreteClass: MeanReversion ──
// OOP role: Inheritance/Polymorphism
// SOLID: LSP, OCP, SRP
class MeanReversion extends TradingStrategy {
  private readonly LOOKBACK = 10;
  private readonly DEVIATION_THRESHOLD = 0.02;
  private readonly MAX_TRADE_SIZE = 500;
  private readonly MIN_CONFIDENCE = 0.25;

  protected analyzeMarket(priceData: PriceData): TradingDecision {
    if (priceData.length < this.LOOKBACK + 1) {
      return { signal: 0, confidence: 0 };
    }

    const currentPrice = priceData[priceData.length - 1];
    const recentPrices = priceData.slice(-this.LOOKBACK);
    const meanPrice =
      recentPrices.reduce((a, b) => a + b, 0) / recentPrices.length;
    const deviation = (meanPrice - currentPrice) / meanPrice;

    if (deviation > this.DEVIATION_THRESHOLD) {
      return { signal: 1, confidence: Math.min(deviation, 0.5) };
    } else if (deviation < -this.DEVIATION_THRESHOLD) {
      return { signal: -1, confidence: Math.min(Math.abs(deviation), 0.5) };
    }

    return { signal: 0, confidence: 0 };
  }

  protected calculatePositionSize(
    confidence: number,
    maxPosition: number
  ): number {
    return Math.floor(maxPosition * confidence * 0.6);
  }

  // Override hook: tighter risk controls
  protected applyRiskControls(_ticker: string, shares: number): boolean {
    if (shares > this.MAX_TRADE_SIZE) {
      return false;
    }
    return super.applyRiskControls(_ticker, shares);
  }

  // Override hook: stricter confidence threshold
  protected preValidate(_signal: Signal, confidence: number): boolean {
    return confidence >= this.MIN_CONFIDENCE;
  }
}
```

      185.00, 184.50
    ],

      141.20, 140.80, 140.10, 139.50, 140.00,
      139.20, 138.80, 138.10, 137.50, 138.00,
      137.20, 136.80
    ],
    MSFT: [
      378.00, 379.50, 380.20, 378.80, 381.00,
      382.30, 383.00, 382.10, 384.50, 385.00,
      383.80, 386.20
    ]
  };

  const strategies: TradingStrategy[] = [
    new MovingAverageCrossover(),
    new MeanReversion()
  ];

  for (const ticker of Object.keys(marketData)) {
    console.log(`\n========== ${ticker} ==========`);
    const prices = marketData[ticker];

    for (const strategy of strategies) {
      const name = strategy.constructor.name;
      console.log(`\n--- ${name} ---`);
      strategy.executeTrade(ticker, prices);
    }
  }

  console.log('\n========== Portfolio Summary ==========');
  console.log('All trades executed via the Template Method pattern.');
  console.log('TypeScript enforced compile-time safety for all primitive operations.');
}

main();

### Language-Specific Notes

- **`abstract` keyword** — TypeScript's `abstract class` and `abstract method` provide compile-time enforcement that cannot be bypassed at runtime. Any subclass that fails to implement `analyzeMarket()` or `calculatePositionSize()` produces a compiler error.
- **`protected` access modifier** — The template method is public (called by external clients), while primitive operations and hooks are protected (only accessible within the class hierarchy). This enforces the pattern's intent: subclasses override steps but do not expose them to external callers.
- **Discriminated union signals** — The `Signal` type (`-1 | 0 | 1`) provides exhaustive type checking. Switch statements on `signal` must handle all three cases, preventing unhandled states at compile time.
- **`readonly` for constants** — Window sizes, thresholds, and limits are marked `private readonly`, making it clear that subclass parameters are immutable after construction. This communicates design intent to other developers reading the code.
