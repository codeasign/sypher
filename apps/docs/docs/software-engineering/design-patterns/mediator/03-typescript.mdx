## Implementation

Below is a complete TypeScript implementation of the Mediator pattern in the stock market domain. The `NasdaqExchange` mediator coordinates communication between `BuyAgent` and `SellAgent` instances, matching buy and sell orders without agents knowing about each other directly. The code is written in TypeScript strict mode with full type safety, using interfaces for contract definitions and abstract classes for shared behavior.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌──────────────────────────────────────┐
//   │   interface StockExchange             │ ← OOP: Abstraction via interface
//   ├──────────────────────────────────────┤    ISP: focused mediator methods
//   │   registerAgent(agent)               │    DIP: depends on TradingAgent
//   │   placeBid(agent, sym, qty, price)   │    not concrete classes
//   │   placeAsk(agent, sym, qty, price)   │
//   │   notify(agent, message)             │
//   │   broadcast(message)                 │
//   └────────────────┬─────────────────────┘
//                    ▲ implements
//   ┌────────────────┴─────────────────────┐
//   │   class NasdaqExchange                │ ← SRP: order matching only
//   │   implements StockExchange            │    OCP: new agent types
//   │   private agents: Map                 │    work without modification
//   │   private orderBook: Order[]          │
//   │   private matchOrders(): void         │
//   └────────────────┬─────────────────────┘
//                    │ coordinates
//   ┌────────────────┴─────────────────────┐
//   │   interface TradingAgent              │ ← LSP: all agents implement
//   ├──────────────────────────────────────┤    the same interface
//   │   getName(): string                  │    Polymorphism: exchange
//   │   getSymbol(): string                │    calls onMessage on any
//   │   onMessage(msg: string): void       │    TradingAgent uniformly
//   └────────────────┬─────────────────────┘
//                    ▲ implements
//   ┌────────────────┴─────────────────────┐
//   │   abstract class BaseTradingAgent     │ ← Encapsulation: private
//   ├──────────────────────────────────────┤    readonly fields
//   │   readonly name: string              │
//   │   readonly symbol: string            │
//   └────────────────┬─────────────────────┘
//                    ▲ extends
//   ┌────────────────┴─────────────────────┐
//   │   ┌────────────────┐  ┌────────────┐ │
//   │   │  BuyAgent      │  │ SellAgent  │ │
//   │   │  send(exch)    │  │ send(exch) │ │
//   │   └────────────────┘  └────────────┘ │
//   └───────────────────────────────────────┘
```

#### OOP Concepts Applied

1. **Abstraction** — The `StockExchange` interface defines the mediator contract without specifying implementation details. The `TradingAgent` interface defines the colleague contract. Concrete classes implement these interfaces, hiding their internal state behind private access modifiers.
2. **Encapsulation** — Private `readonly` fields in `BaseTradingAgent` (`name`, `symbol`) and private fields in `BuyAgent`/`SellAgent` (`quantity`, `maxPrice`/`minPrice`) enforce strict information hiding. The exchange's `#agents` and `#orderBook` (using the `private` keyword) cannot be accessed externally.
3. **Inheritance** — `BaseTradingAgent` provides shared implementations of `getName()`, `getSymbol()`, and `onMessage()`. `BuyAgent` and `SellAgent` extend it, inheriting the common behavior while adding domain-specific logic.
4. **Polymorphism** — The `NasdaqExchange` calls `agent.onMessage(message)` on any registered `TradingAgent` without knowing the concrete type. The `send()` method is called polymorphically through the `StockExchange` interface.

#### SOLID Principles Applied

5. **SRP** — `NasdaqExchange` is responsible only for order matching and agent notification. `BuyAgent` handles buy order creation. `SellAgent` handles sell order creation. Each class has a single responsibility.
6. **OCP** — New agent types (e.g., `MarketMakerAgent`) can be added by extending `BaseTradingAgent`. The `NasdaqExchange` mediator never needs to change — it works with any `TradingAgent` implementation.
7. **LSP** — Any class implementing `TradingAgent` or extending `BaseTradingAgent` can substitute for any other. The exchange treats all agents uniformly through the `TradingAgent` interface.
8. **ISP** — The `TradingAgent` interface defines only the minimal methods every agent needs (`getName`, `getSymbol`, `onMessage`). No agent is forced to depend on methods it does not use.
9. **DIP** — `NasdaqExchange` depends on the abstract `TradingAgent` interface, not on concrete agent classes. Agents depend on the abstract `StockExchange` interface, not on a concrete exchange.

### TradingAgent (Colleague Interface)
```typescript
// ── Colleague interface ──
interface TradingAgent {

}
```

### BaseTradingAgent (Abstract Base Class)
```typescript
// ── Abstract base class for trading agents ──
abstract class BaseTradingAgent implements TradingAgent {
  private readonly name: string;
  private readonly symbol: string;

  constructor(name: string, symbol: string) {
    this.name = name;
    this.symbol = symbol;
  }

  getName(): string {
    return this.name;
  }

  getSymbol(): string {
    return this.symbol;
  }

  onMessage(message: string): void {
    console.log(`[${this.name}] ${message}`);
  }

  /** Place an order — subclass responsibility */
  abstract send(exchange: StockExchange): void;
}
```

### NasdaqExchange (Concrete Mediator)
```typescript
// ── Concrete Mediator ──
class NasdaqExchange implements StockExchange {
  private readonly agents: Map<string, TradingAgent> = new Map();
  private readonly orderBook: Order[] = [];

  registerAgent(agent: TradingAgent): void {
    this.agents.set(agent.getName(), agent);
    this.notify(agent, `Welcome to NasdaqExchange, ${agent.getName()}!`);
  }

  placeBid(agent: TradingAgent, symbol: string, quantity: number, price: number): void {
    this.orderBook.push({ agent, symbol, quantity, price, type: 'bid' });
    this.matchOrders();
  }

  placeAsk(agent: TradingAgent, symbol: string, quantity: number, price: number): void {
    this.orderBook.push({ agent, symbol, quantity, price, type: 'ask' });
    this.matchOrders();
  }

  /** Match buy and sell orders by symbol and price */
  private matchOrders(): void {
    for (const bid of this.orderBook) {
      if (bid.type !== 'bid' || bid.quantity === 0) continue;
      for (const ask of this.orderBook) {
        if (ask.type !== 'ask' || ask.quantity === 0) continue;
        if (bid.symbol === ask.symbol && bid.price >= ask.price) {
          const fillQty = Math.min(bid.quantity, ask.quantity);
          bid.quantity -= fillQty;
          ask.quantity -= fillQty;

          this.notify(
            bid.agent,
            `FILLED: Bought ${fillQty} shares of ${bid.symbol} at $${ask.price.toFixed(2)}`
          );
          this.notify(
            ask.agent,
            `FILLED: Sold ${fillQty} shares of ${ask.symbol} at $${ask.price.toFixed(2)}`
          );
          this.broadcast(
            `TRADE: ${bid.agent.getName()} x ${ask.agent.getName()} — ${fillQty} ${bid.symbol} @ $${ask.price.toFixed(2)}`
          );
        }
      }
    }
    // Remove fully filled orders from the book
    this.orderBook.splice(0, this.orderBook.length, ...this.orderBook.filter((o) => o.quantity > 0));
  }

  notify(agent: TradingAgent, message: string): void {
    agent.onMessage(message);
  }

  broadcast(message: string): void {
    for (const agent of this.agents.values()) {
      agent.onMessage(message);
    }
  }
}
```

### BuyAgent (Concrete Colleague 1)
```typescript
// ── Concrete Colleague 1: BuyAgent ──
class BuyAgent extends BaseTradingAgent {
  private readonly quantity: number;
  private readonly maxPrice: number;

  constructor(name: string, symbol: string, quantity: number, maxPrice: number) {
    super(name, symbol);
    this.quantity = quantity;
    this.maxPrice = maxPrice;
  }

  /** Submit a buy order to the exchange */
  send(exchange: StockExchange): void {
    console.log(
      `${this.getName()}: Placing bid for ${this.quantity} ${this.getSymbol()} @ $${this.maxPrice.toFixed(2)}`
    );
    exchange.placeBid(this, this.getSymbol(), this.quantity, this.maxPrice);
  }
}
```

### Main / Driver
```typescript
// ── Main / Driver ──
const exchange: StockExchange = new NasdaqExchange();

// Create trading agents
const buyer1: BuyAgent = new BuyAgent('BuyAgent_AAPL', 'AAPL', 100, 150.50);
const seller1: SellAgent = new SellAgent('SellAgent_AAPL', 'AAPL', 100, 150.50);
const buyer2: BuyAgent = new BuyAgent('BuyAgent_GOOGL', 'GOOGL', 50, 180.30);
const seller2: SellAgent = new SellAgent('SellAgent_GOOGL', 'GOOGL', 50, 175.25);

// Register all agents with the exchange
console.log('=== Stock Market Mediator Demo ===\n');

exchange.registerAgent(buyer1);
exchange.registerAgent(seller1);
exchange.registerAgent(buyer2);
exchange.registerAgent(seller2);

console.log('');

// Submit orders — exchanges handle matching internally
buyer1.send(exchange);   // Bid AAPL 100 @ $150.50 — no matching ask yet
seller1.send(exchange);  // Ask AAPL 100 @ $150.50 — matches with buyer1

console.log('');

buyer2.send(exchange);   // Bid GOOGL 50 @ $180.30 — no matching ask yet
seller2.send(exchange);  // Ask GOOGL 50 @ $175.25 — matches immediately (bid >= ask)
```

### Language-Specific Notes

- **Interfaces for contract definition**: `StockExchange` and `TradingAgent` are defined as TypeScript interfaces, establishing clear contracts that all participants must follow. The compiler enforces that `NasdaqExchange` implements every method declared in `StockExchange`.
- **`implements` keyword**: `NasdaqExchange implements StockExchange` and `BaseTradingAgent implements TradingAgent` provide compile-time type checking against the interface contracts, catching missing method errors before runtime.
- **Readonly and access modifiers**: `private readonly name: string` provides compile-time guarantees that fields cannot be reassigned after construction and cannot be accessed outside the class. This enforces the mediator's encapsulation at the type level.
- **Strict null checks**: TypeScript's `strict` mode catches potential `null`/`undefined` access at compile time. The `Map` type system ensures that `this.agents.get(key)` returns `TradingAgent | undefined`, requiring explicit handling.
- **Union types for order types**: `type: 'bid' | 'ask'` uses a discriminated union to clearly distinguish buy and sell orders in the order book. The compiler can narrow the type based on comparisons, eliminating invalid states.
- **Generic collection types**: `Map<string, TradingAgent>` provides type-safe access to the agent registry. The compiler ensures that only `TradingAgent` instances can be stored and that keys are always strings, preventing runtime type errors.
