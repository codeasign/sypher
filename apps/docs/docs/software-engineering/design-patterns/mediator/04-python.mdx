The Mediator pattern centralises communication between objects so they interact indirectly through a mediator rather than directly with one another. In a stock market domain, a `StockExchange` mediator routes bids and asks between trading agents, matching buy and sell orders without the agents ever knowing about each other.

## How the Implementation Works

The Python Mediator implementation uses Abstract Base Classes (ABCs) from the `abc` module to define two contract hierarchies:

1. **`StockExchange(ABC)`** — the mediator contract declaring `register_agent`, `place_bid`, `place_ask`, and `broadcast`. Each method carries an `@abstractmethod` decorator so any concrete exchange must implement all four.

2. **`TradingAgent(ABC)`** — the colleague contract declaring `get_name`, `get_symbol`, and `on_message`. Colleagues communicate exclusively through the mediator, never directly.

3. **`NasdaqExchange(StockExchange)`** — the concrete mediator that holds a registry of agents (`_agents` dict) and an order book (`_order_book` list). When a bid or ask arrives it appends the order and immediately calls `_match_orders()`, which scans for compatible buy/sell pairs (same symbol, bid price >= ask price), executes the trade at the bid price, and notifies both parties.

4. **`BuyAgent` and `SellAgent(ABC)`** — concrete colleagues that encapsulate a trader's name, stock symbol, desired quantity, and price limit. Their `send` method delegates to the exchange, and `on_message` prints the notification to stdout.

5. **OOP concepts demonstrated**: abstraction (ABCs define pure contracts), encapsulation (private `__name`, `__symbol` attributes with `@property` accessors), composition (`NasdaqExchange` composes `TradingAgent` references), and message passing (agents talk only through the mediator).

6. **SOLID principles**: Single Responsibility (agents trade, exchanges match), Open/Closed (new agent types extend `TradingAgent` without modifying `StockExchange`), Liskov Substitution (any `TradingAgent` subclass works with any `StockExchange` subclass), Interface Segregation (small focused interfaces), Dependency Inversion (both layers depend on ABCs, not concrete classes).

### StockExchange (Mediator Interface)
```python
# ---------------------------------------------------------------------------
# Mediator interface
# ---------------------------------------------------------------------------

class StockExchange(ABC):

    @abstractmethod
    def register_agent(self, agent: "TradingAgent") -> None:
        """Add a trading agent to the exchange registry."""

    @abstractmethod
    def place_bid(self, agent: "TradingAgent", symbol: str,
                  quantity: int, price: float) -> None:
        """Submit a buy order on behalf of an agent."""

    @abstractmethod
    def place_ask(self, agent: "TradingAgent", symbol: str,
                  quantity: int, price: float) -> None:
        """Submit a sell order on behalf of an agent."""

    @abstractmethod
    def broadcast(self, message: str) -> None:
        """Send a message to every registered agent."""
```

### TradingAgent (Colleague Interface)
```python
# ---------------------------------------------------------------------------
# Colleague interface
# ---------------------------------------------------------------------------

class TradingAgent(ABC):

    @abstractmethod
    def get_name(self) -> str:
        """Return the agent's display name."""

    @abstractmethod
    def get_symbol(self) -> str:
        """Return the stock symbol this agent trades."""

    @abstractmethod
    def on_message(self, message: str) -> None:
        """Receive a notification from the exchange."""
```

### NasdaqExchange (Concrete Mediator)
```python
# ---------------------------------------------------------------------------
# Concrete mediator
# ---------------------------------------------------------------------------

class NasdaqExchange(StockExchange):

    def __init__(self) -> None:
        self._agents: dict[str, TradingAgent] = {}
        self._order_book: list[Order] = []

    def register_agent(self, agent: TradingAgent) -> None:
        self._agents[agent.get_name()] = agent
        self.broadcast(f"{agent.get_name()} has joined the exchange.")

    def place_bid(self, agent: TradingAgent, symbol: str,
                  quantity: int, price: float) -> None:
        order = Order(agent.get_name(), symbol, quantity, price, "buy")
        self._order_book.append(order)
        self.broadcast(
            f"NEW BID: {agent.get_name()} wants to buy {quantity} "
            f"shares of {symbol} @ ${price:.2f}"
        )
        trades = self._match_orders()
        for t in trades:
            self._agents[t.buyer].on_message(
                f"BOUGHT {t.quantity} shares of {t.symbol} "
                f"from {t.seller} @ ${t.price:.2f}"
            )
            self._agents[t.seller].on_message(
                f"SOLD {t.quantity} shares of {t.symbol} "
                f"to {t.buyer} @ ${t.price:.2f}"
            )

    def place_ask(self, agent: TradingAgent, symbol: str,
                  quantity: int, price: float) -> None:
        order = Order(agent.get_name(), symbol, quantity, price, "sell")
        self._order_book.append(order)
        self.broadcast(
            f"NEW ASK: {agent.get_name()} wants to sell {quantity} "
            f"shares of {symbol} @ ${price:.2f}"
        )
        trades = self._match_orders()
        for t in trades:
            self._agents[t.buyer].on_message(
                f"BOUGHT {t.quantity} shares of {t.symbol} "
                f"from {t.seller} @ ${t.price:.2f}"
            )
            self._agents[t.seller].on_message(
                f"SOLD {t.quantity} shares of {t.symbol} "
                f"to {t.buyer} @ ${t.price:.2f}"
            )

    def broadcast(self, message: str) -> None:
        print(f"[EXCHANGE] {message}")

    def _match_orders(self) -> list[Trade]:
        matched: list[Trade] = []
        remaining: list[Order] = []

        for order in self._order_book:
            if order.order_type == "buy":
                remaining.append(order)
            else:
                matched_order: Optional[Order] = None
                for i, buy in enumerate(remaining):
                    if buy.symbol == order.symbol and buy.price >= order.price:
                        matched_order = buy
                        break
                if matched_order is not None:
                    fill_qty = min(matched_order.quantity, order.quantity)
                    trade_price = matched_order.price
                    matched.append(Trade(
                        buyer=matched_order.agent_name,
                        seller=order.agent_name,
                        symbol=order.symbol,
                        quantity=fill_qty,
                        price=trade_price,
                    ))
                    remaining.remove(matched_order)
                    remaining.append(Order(
                        agent_name=matched_order.agent_name,
                        symbol=matched_order.symbol,
                        quantity=matched_order.quantity - fill_qty,
                        price=matched_order.price,
                        order_type="buy",
                    ))
                    remaining.append(Order(
                        agent_name=order.agent_name,
                        symbol=order.symbol,
                        quantity=order.quantity - fill_qty,
                        price=order.price,
                        order_type="sell",
                    ))

        self._order_book = [o for o in remaining if o.quantity > 0]
        return matched
```

### BuyAgent (Concrete Colleague 1)
```python
# ---------------------------------------------------------------------------
# Concrete colleagues
# ---------------------------------------------------------------------------

class BuyAgent(TradingAgent):

    def __init__(self, name: str, symbol: str,
                 quantity: int, price_limit: float) -> None:
        self.__name = name
        self.__symbol = symbol
        self.__quantity = quantity
        self.__price_limit = price_limit

    def get_name(self) -> str:
        return self.__name

    def get_symbol(self) -> str:
        return self.__symbol

    def on_message(self, message: str) -> None:
        print(f"[{self.__name}] {message}")

    def send(self, exchange: StockExchange) -> None:
        exchange.place_bid(self, self.__symbol,
                           self.__quantity, self.__price_limit)
```

### Main / Driver
```python
# ---------------------------------------------------------------------------
# Driver
# ---------------------------------------------------------------------------

if __name__ == "__main__":
    nasdaq = NasdaqExchange()

    buyer_aapl = BuyAgent("Alice", "AAPL", 100, 152.50)
    seller_aapl = SellAgent("Bob", "AAPL", 100, 151.00)
    buyer_googl = BuyAgent("Carol", "GOOGL", 50, 190.35)
    seller_googl = SellAgent("Dave", "GOOGL", 50, 189.30)

    for agent in (buyer_aapl, seller_aapl, buyer_googl, seller_googl):
        nasdaq.register_agent(agent)

    print("\n--- Placing AAPL orders ---")
    buyer_aapl.send(nasdaq)
    seller_aapl.send(nasdaq)

    print("\n--- Placing GOOGL orders ---")
    buyer_googl.send(nasdaq)
    seller_googl.send(nasdaq)
```

## Language-Specific Notes — Python

- **Abstract Base Classes (`abc.ABC`)** — provide a formal mechanism for defining interfaces with `@abstractmethod` decorators. Unlike duck typing alone, ABCs enforce that subclasses implement every abstract method at instantiation time, making contract violations fail fast.
- **`@dataclass`** — auto-generates `__init__`, `__repr__`, `__eq__`, and `__hash__` for data-centric classes like `Order` and `Trade`, eliminating boilerplate while keeping objects immutable-friendly via `frozen=True` when needed.
- **Type hints** — the code uses `dict[str, TradingAgent]`, `list[Order]`, `Optional[Order]`, and `-> None` return annotations. These are not enforced at runtime but enable static analysis with `mypy` or `pyright`.
- **Name mangling for encapsulation** — double-underscore attributes (`self.__name`) trigger Python's name mangling, producing `_BuyAgent__name` at runtime. This provides stronger encapsulation than a single underscore, which is merely a convention.
- **`__name__ == "__main__"` guard** — ensures the driver block only runs when the file is executed directly, not when imported as a module. This is the standard Python pattern for separate-able test/demo code.
- **Duck typing compatibility** — while this example uses ABCs for explicitness, a Pythonic alternative would drop the ABCs entirely and rely on duck typing: any object with the right methods can act as a `StockExchange`. The ABC approach adds self-documentation and early error checking.
