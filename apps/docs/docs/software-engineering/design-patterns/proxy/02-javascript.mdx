This page implements the Proxy design pattern in JavaScript ES2022+ using a Cars domain. The Proxy pattern provides a surrogate object that controls access to another object. Here, a `Car` interface is implemented by `SportsCar` (the real subject), while `LazyCarModelProxy` defers the expensive 3D model loading and `ValetCarProxy` restricts speed and access for valet parking scenarios.

## How the Implementation Works

### Class Diagram (Pseudocode)

The following table maps the participants to their OOP/SOLID roles:

```

  │ Participant            ──  OOP Role         ──  SOLID Principle         │
  ├──────────────────────────────────────────────────────────────────────────┤
  │ Car                    ──  Abstraction      ──  DIP (depend on          │
  │ (interface/base class) ──  (contract)       ──  abstractions)           │
  ├──────────────────────────────────────────────────────────────────────────┤
  │ SportsCar              ──  Inheritance      ──  LSP (substitutable      │
  │ (real subject)         ──  Polymorphism     ──  for Car)                │
  │                        ──                  ──  OCP (open for           │
  │                        ──                  ──  extension)               │
  ├──────────────────────────────────────────────────────────────────────────┤
  │ ValetCarProxy          ──  Encapsulation    ──  SRP (single reason      │
  │ LazyCarModelProxy      ──  (access logic    ──  to change: access       │
  │ (proxies)              ──   hidden)         ──  control / lazy load)    │
  ├──────────────────────────────────────────────────────────────────────────┤
  │ Driver / client        ──  Composition      ──  ISP (depends only       │
  │ (caller)               ──  (uses Car API)   ──  on Car interface)       │
  └──────────────────────────────────────────────────────────────────────────┘
```

### OOP Concepts Applied

- **Abstraction** — The `Car` base class defines a contract of methods (`displayDetails`, `drive`, `getMaxSpeed`, `accessTrunk`, `getVin`) that every car representation must implement. Callers interact only with the `Car` interface, never with concrete car internals or proxy logic.
- **Encapsulation** — The `SportsCar` hides its `#full3DModel`, `#trunkContents`, and `#gloveboxContents` behind private fields. The proxies encapsulate their access control rules and lazy initialization logic.
- **Inheritance** — `SportsCar`, `ValetCarProxy`, and `LazyCarModelProxy` extend `Car`, reusing shared fields (`#vin`, `#make`, `#model`, `#year`) and adding domain-specific state and behavior.
- **Polymorphism** — The driver calls methods on `Car` references without knowing whether they point to a `SportsCar`, `ValetCarProxy`, or `LazyCarModelProxy`. Each variant provides its own implementation of `drive()`, `displayDetails()`, and `accessTrunk()`.

### SOLID Principles Applied

- **Single Responsibility Principle** — `SportsCar` is responsible only for real car behavior. `ValetCarProxy` handles access control. `LazyCarModelProxy` handles lazy initialization. No class mixes concerns.
- **Open/Closed Principle** — Adding a new proxy type (e.g., `LoggingCarProxy` or `CachingCarProxy`) requires creating a new class that extends `Car`. Existing `SportsCar` and proxy classes remain unchanged.
- **Liskov Substitution Principle** — Every proxy can replace `Car` in client code without breaking correctness. `ValetCarProxy` and `LazyCarModelProxy` both adhere to the `Car` contract.
- **Interface Segregation Principle** — The `Car` base class exposes only the methods every car representation needs. Specialized methods like `applyInterest` or `getMonthlyPayment` from the banking domain are absent because they are irrelevant to cars.
- **Dependency Inversion Principle** — High-level driver code depends on the `Car` abstraction, not on concrete `SportsCar` or proxy classes. The driver receives `Car` references and operates through the abstract interface.

## Implementation

The complete JavaScript program below demonstrates the Proxy pattern in a Cars context. It is runnable in Node.js 20+ and uses ES2022+ class syntax with private fields, static helpers, and strict validation.

### Utility

```javascript
'use strict';

/** Generate a random vehicle identification number. */
function generateVin() {
  const chars = 'ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789';
  let vin = '';
  for (let i = 0; i < 17; i++) {
    vin += chars.charAt(Math.floor(Math.random() * chars.length));
  }
  return vin;
}
```

### Subject: Car (Abstract Base Class)

```javascript
class Car {
  /** @type {string} */ #vin;
  /** @type {string} */ #make;
  /** @type {string} */ #model;
  /** @type {number} */ #year;

  /**
   * @param {string} vin - Vehicle identification number.
   * @param {string} make - Manufacturer name.
   * @param {string} model - Model name.
   * @param {number} year - Model year.
   */
  constructor(vin, make, model, year) {
    if (new.target === Car) {
      throw new Error('Cannot instantiate abstract class Car directly.');
    }
    this.#vin = vin;
    this.#make = make;
    this.#model = model;
    this.#year = year;
  }

  /** @returns {string} */
  getVin() { return this.#vin; }

  /** @returns {string} */
  getMake() { return this.#make; }

  /** @returns {string} */
  getModel() { return this.#model; }

  /** @returns {number} */
  getYear() { return this.#year; }

  /** @returns {string} */
  getShortDescription() {
    return `${this.#year} ${this.#make} ${this.#model} (${this.#vin})`;
  }

  /** @returns {string} */
  displayDetails() { throw new Error('Must override displayDetails()'); }

  /**
   * @param {number} speed
   * @returns {string}
   */
  drive(speed) { throw new Error('Must override drive()'); }

  /** @returns {number} */
  getMaxSpeed() { throw new Error('Must override getMaxSpeed()'); }

  /** @returns {string} */
  accessTrunk() { throw new Error('Must override accessTrunk()'); }
}
```

### RealSubject: SportsCar

```javascript
class SportsCar extends Car {
  /** @type {number} */ #horsepower;
  /** @type {string} */ #engineType;
  /** @type {number} */ #acceleration0to60;
  /** @type {number} */ #maxSpeed;
  /** @type {object|null} */ #full3DModel;
  /** @type {string[]} */ #trunkContents;
  /** @type {string[]} */ #gloveboxContents;
  /** @type {boolean} */ #isEngineRunning;
  /** @type {number} */ #currentSpeed;

  /**
   * @param {string} vin
   * @param {string} make
   * @param {string} model
   * @param {number} year
   */
  constructor(vin, make, model, year) {
    super(vin, make, model, year);
    this.#horsepower = 750;
    this.#engineType = '5.2L V10 Twin-Turbo';
    this.#acceleration0to60 = 2.8;
    this.#maxSpeed = 200.0;
    this.#full3DModel = null;
    this.#trunkContents = ['First aid kit', 'Tool set', 'Owners manual'];
    this.#gloveboxContents = ['Registration', 'Insurance card', 'Service record'];
    this.#isEngineRunning = false;
    this.#currentSpeed = 0.0;
    this.#load3DModel();
  }

  /** Simulate expensive 3D model loading. */
  #load3DModel() {
    console.log(
      `  [SportsCar] Loading 3D model for ${this.getVin()}... ` +
      `200 MB of polygon data loaded.`
    );
    this.#full3DModel = {
      polygons: 450000,
      textures: ['carbon_fiber', 'leather_black', 'aluminum_brushed'],
      materials: ['forged_carbon', 'alcantara', 'titanium'],
    };
  }

  /**
   * Display full car specifications.
   * @returns {string}
   */
  displayDetails() {
    const modelData = this.#full3DModel;
    const trunk = this.#trunkContents.join(', ');
    const glovebox = this.#gloveboxContents.join(', ');
    return (
      `--- ${this.getShortDescription()} ---\n` +
      `  Engine: ${this.#engineType}\n` +
      `  Horsepower: ${this.#horsepower} hp\n` +
      `  0-60 mph: ${this.#acceleration0to60}s\n` +
      `  Top Speed: ${this.#maxSpeed} mph\n` +
      `  3D Model: ${modelData.polygons.toLocaleString()} polygons, ` +
      `${modelData.textures.length} textures\n` +
      `  Trunk: ${trunk}\n` +
      `  Glovebox: ${glovebox}`
    );
  }

  /**
   * Drive the car at the given speed.
   * @param {number} speed
   * @returns {string}
   */
  drive(speed) {
    const actualSpeed = Math.min(speed, this.#maxSpeed);
    this.#isEngineRunning = true;
    this.#currentSpeed = actualSpeed;
    const engineNote = 'ROAR! V10 screams through the rev range.';
    return (
      `  [SportsCar] ${engineNote}\n` +
      `  Speed: ${actualSpeed.toFixed(1)} mph | Gear: 7th | RPM: 8200`
    );
  }

  /** @returns {number} */
  getMaxSpeed() { return this.#maxSpeed; }

  /**
   * Open and access the trunk.
   * @returns {string}
   */
  accessTrunk() {
    const contents = this.#trunkContents.join(', ');
    return `  [SportsCar] Trunk opened. Contents: ${contents}`;
  }
}
```

### Virtual Proxy: LazyCarModelProxy

```javascript
class LazyCarModelProxy extends Car {
  /** @type {SportsCar|null} */ #realCar;

  /**
   * @param {string} vin
   * @param {string} make
   * @param {string} model
   * @param {number} year
   */
  constructor(vin, make, model, year) {
    super(vin, make, model, year);
    this.#realCar = null;
  }

  /**
   * Initialize the real car on first access.
   * @returns {SportsCar}
   */
  #getRealCar() {
    if (this.#realCar === null) {
      console.log(
        `  [LazyCarModelProxy] Initializing real SportsCar ` +
        `for ${this.getShortDescription()}...`
      );
      this.#realCar = new SportsCar(this.getVin(), this.getMake(), this.getModel(), this.getYear());
    } else {
      console.log(`  [LazyCarModelProxy] Reusing existing SportsCar instance.`);
    }
    return this.#realCar;
  }

  /**
   * Load details on demand — triggers real car creation.
   * @returns {string}
   */
  displayDetails() {
    return this.#getRealCar().displayDetails();
  }

  /**
   * Delegate to real car, initializing if needed.
   * @param {number} speed
   * @returns {string}
   */
  drive(speed) {
    return this.#getRealCar().drive(speed);
  }

  /** @returns {number} */
  getMaxSpeed() {
    return this.#getRealCar().getMaxSpeed();
  }

  /** @returns {string} */
  accessTrunk() {
    return this.#getRealCar().accessTrunk();
  }
}
```

### Protection Proxy: ValetCarProxy

```javascript
class ValetCarProxy extends Car {
  /** @type {Car} */ #realCar;
  /** @type {number} */ #maxValetSpeed;
  /** @type {string} */ #trunkDeniedMessage;

  /**
   * @param {Car} realCar - The real car to restrict access to.
   * @param {number} [maxValetSpeed=30] - Maximum speed allowed in mph.
   */
  constructor(realCar, maxValetSpeed = 30) {
    super(realCar.getVin(), realCar.getMake(), realCar.getModel(), realCar.getYear());
    this.#realCar = realCar;
    this.#maxValetSpeed = maxValetSpeed;
    this.#trunkDeniedMessage =
      '  [ValetCarProxy] ACCESS DENIED: Valet key does not permit trunk access.';
  }

  /**
   * Show limited information card for valet.
   * @returns {string}
   */
  displayDetails() {
    return (
      `--- Valet Card ---\n` +
      `  VIN: ${this.getVin()}\n` +
      `  Max Permitted Speed: ${this.#maxValetSpeed} mph\n` +
      `  Trunk Access: DENIED\n` +
      `  Glovebox Access: DENIED`
    );
  }

  /**
   * Enforce the valet speed limit.
   * @param {number} speed
   * @returns {string}
   */
  drive(speed) {
    const limitedSpeed = Math.min(speed, this.#maxValetSpeed);
    if (limitedSpeed < speed) {
      console.log(`  [ValetCarProxy] Speed limited to ${this.#maxValetSpeed} mph.`);
    }
    return this.#realCar.drive(limitedSpeed);
  }

  /** @returns {number} */
  getMaxSpeed() { return this.#maxValetSpeed; }

  /** @returns {string} */
  accessTrunk() { return this.#trunkDeniedMessage; }
}
```

```javascript
  console.log();

  // ── Scenario 2: Protection Proxy (Valet Key) ─────────────────────────────
  console.log('--- SCENARIO 2: Protection Proxy (Valet Key) ---');
  const realCar = new SportsCar(generateVin(), 'Sypher', 'GT-S', 2026);
  const valetKey = new ValetCarProxy(realCar);
  console.log();
  console.log('  Owner hands the valet key to the parking attendant.');
  console.log();

  console.log('  Valet: Displays info card:');
  console.log(valetKey.displayDetails());
  console.log();

  console.log('  Valet: Attempts to drive at 80 mph:');
  console.log(valetKey.drive(80.0));
  console.log();

  console.log('  Valet: Attempts to access the trunk:');
  console.log(valetKey.accessTrunk());
  console.log();

  // ── Scenario 3: Owner Returns — Full Access ─────────────────────────────
  console.log('--- SCENARIO 3: Owner Returns — Full Access ---');
  console.log('  Owner uses the real car key directly:');
  console.log(realCar.drive(120.0));
  console.log(realCar.accessTrunk());
  console.log();

  // ── Scenario 4: Polymorphic Iteration ────────────────────────────────────
  console.log('--- SCENARIO 4: Polymorphic Car Display ---');
  const cars = [lazyCar, valetKey];
  for (const car of cars) {
    console.log(`  [${car.getVin()}] Max Speed: ${car.getMaxSpeed()} mph`);
  }
  console.log();

  // ── Scenario 5: Demonstrating Lazy Loading Benefit ──────────────────────
  console.log('--- SCENARIO 5: Lazy Loading Benefit ---');
  console.log('  Creating 3 LazyCarModelProxies for showroom display...');
  const showroomCars = [
    new LazyCarModelProxy(generateVin(), 'Sypher', 'GT', 2026),
    new LazyCarModelProxy(generateVin(), 'Sypher', 'GT-S', 2026),
    new LazyCarModelProxy(generateVin(), 'Falcon', 'X1', 2026),
  ];
  console.log('  All 3 proxies created. Memory usage: minimal.');
  console.log('  No 3D models loaded. Showing thumbnails...');
  for (const car of showroomCars) {
    console.log(`    ${car.getShortDescription()}`);
  }
  console.log();
  console.log('  Customer examines one car in detail (triggers 3D model load):');
  console.log(showroomCars[0].displayDetails());
  console.log();

  console.log('='.repeat(72));
  console.log('  Proxy pattern demonstration complete.');
  console.log('='.repeat(72));
}

main();
```

## Language-Specific Notes

- **`new.target` for abstract class enforcement** — The `Car` base class uses `new.target === Car` in its constructor to prevent direct instantiation. JavaScript lacks the `abstract` keyword for classes, so this runtime check enforces that `Car` can only be extended, never instantiated directly. Any attempt to call `new Car(...)` throws at runtime, enforcing the DIP principle that clients must depend on concrete subtypes.

- **Private fields (`#`) for true encapsulation** — JavaScript ES2022+ `#` private fields provide hard runtime privacy. `#full3DModel`, `#trunkContents`, and `#gloveboxContents` in `SportsCar` are truly inaccessible from outside the class. This enforces the Encapsulation pillar of OOP more strictly than TypeScript's `private` keyword, which only enforces at compile time.

- **Lazy initialization via null-check pattern** — `LazyCarModelProxy.#getRealCar()` uses a simple `if (this.#realCar === null)` check to implement lazy loading. This is the most straightforward lazy initialization pattern in JavaScript. For production systems with concurrent access, consider using a lock or promise-based approach to prevent double initialization in async contexts.

- **Polymorphism through the prototype chain** — The `for...of` loop in Scenario 4 iterates over heterogeneous `Car` references (a `LazyCarModelProxy` and a `ValetCarProxy`) and calls `getMaxSpeed()` on each. JavaScript dispatches to the correct implementation via the prototype chain at runtime. The client code never knows which concrete type it holds.

- **No built-in Proxy for our pattern** — JavaScript has a built-in `Proxy` object (for metaprogramming traps like `get`, `set`, `apply`), but this implementation uses custom proxy classes to demonstrate the classical GoF Proxy pattern with explicit `Car` interface conformance. The built-in `Proxy` could be used for dynamic interception, but the class-based approach is clearer for educational purposes and provides compile-time type safety when used with TypeScript.

- **Default parameter for configurable speed limit** — The `ValetCarProxy` constructor accepts an optional `maxValetSpeed` parameter with a default of 30 mph. This allows different valet scenarios (parking lot vs. valet track) to configure the speed limit independently without subclassing. The `Math.min()` call in `drive()` ensures the limit is always enforced, even if the caller passes a negative or unsafe value.
