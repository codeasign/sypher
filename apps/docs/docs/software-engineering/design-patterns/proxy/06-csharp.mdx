This page implements the Proxy design pattern in C# 12 (.NET 8) using a Cars domain. The Proxy pattern provides a surrogate object that controls access to another object. Here, a `ICar` interface is implemented by `SportsCar` (the real subject), while `LazyCarModelProxy` defers expensive 3D model loading and `ValetCarProxy` restricts speed and access for valet parking. C#'s interface system, properties, and access modifiers enforce the pattern.

## How the Implementation Works

### Class Diagram (Pseudocode)

The following table maps the participants to their OOP/SOLID roles:

```
  ┌──────────────────────────────────────────────────────────────────────────┐
  │ Participant            ──  OOP Role         ──  SOLID Principle         │
  ├──────────────────────────────────────────────────────────────────────────┤
  │ ICar (interface)       ──  Abstraction      ──  DIP (depend on          │
  │                        ──  (contract)       ──  abstractions)           │
  ├──────────────────────────────────────────────────────────────────────────┤
  │ SportsCar              ──  Inheritance      ──  LSP (substitutable      │
  │ (real subject)         ──  Polymorphism     ──  for ICar)               │
  │                        ──  : ICar           ──  OCP (open for           │
  │                        ──                  ──  extension)               │
  ├──────────────────────────────────────────────────────────────────────────┤
  │ ValetCarProxy          ──  Encapsulation    ──  SRP (single reason      │
  │ LazyCarModelProxy      ──  (access logic    ──  to change: access       │
  │ (proxies)              ──   hidden)         ──  control / lazy load)    │
  ├──────────────────────────────────────────────────────────────────────────┤
  │ Driver / client        ──  Composition      ──  ISP (depends only       │
  │ (caller)               ──  (uses ICar API)  ──  on ICar interface)      │
  └──────────────────────────────────────────────────────────────────────────┘
```

### OOP Concepts Applied

- **Abstraction** — The `ICar` interface defines the contract with method signatures and auto-property getters. C# enforces this contract at compile time.
- **Encapsulation** — Internal state in `SportsCar` uses `private` fields and `readonly` modifiers. The proxies encapsulate their access control logic behind the `ICar` interface.
- **Inheritance and Interface Implementation** — `SportsCar`, `LazyCarModelProxy`, and `ValetCarProxy` implement `ICar` using the `: ICar` syntax.
- **Polymorphism** — The driver calls methods on `ICar` references without knowing the concrete type. C# resolves method calls at runtime via the virtual method table.

### SOLID Principles Applied

- **Single Responsibility Principle** — `SportsCar` handles real car behavior. `ValetCarProxy` handles access control. `LazyCarModelProxy` handles lazy initialization.
- **Open/Closed Principle** — Adding a new proxy type requires creating a new class that implements `ICar`. Existing classes remain unchanged.
- **Liskov Substitution Principle** — Every proxy can replace `ICar` in client code without breaking correctness.
- **Interface Segregation Principle** — The `ICar` interface exposes only the methods every car representation needs.
- **Dependency Inversion Principle** — High-level driver code depends on the `ICar` abstraction, not on concrete classes.

## Implementation

The complete C# program below demonstrates the Proxy pattern in a Cars context. It is runnable with .NET 8 and uses modern C# 12 features.

### Utility

```csharp
using System;
using System.Collections.Generic;
using System.Linq;

/// <summary>Generate a random 17-character vehicle identification number.</summary>
private static string GenerateVin()
{
    const string chars = "ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789";
    var random = new Random();
    return new string(Enumerable.Range(0, 17)
        .Select(_ => chars[random.Next(chars.Length)])
        .ToArray());
}
```

### Subject: ICar (Interface)

```csharp
public interface ICar
{
    string Vin { get; }
    string Make { get; }
    string Model { get; }
    int Year { get; }
    string GetShortDescription();
    string DisplayDetails();
    string Drive(double speed);
    double GetMaxSpeed();
    string AccessTrunk();
}
```

### RealSubject: SportsCar

```csharp
public class SportsCar : ICar
{
    public string Vin { get; }
    public string Make { get; }
    public string Model { get; }
    public int Year { get; }

    private readonly int horsepower = 750;
    private readonly string engineType = "5.2L V10 Twin-Turbo";
    private readonly double acceleration0to60 = 2.8;
    private readonly double maxSpeed = 200.0;
    private Dictionary<string, object>? full3DModel;
    private readonly List<string> trunkContents = new()
        { "First aid kit", "Tool set", "Owners manual" };
    private readonly List<string> gloveboxContents = new()
        { "Registration", "Insurance card", "Service record" };
    private bool isEngineRunning = false;
    private double currentSpeed = 0.0;

    public SportsCar(string vin, string make, string model, int year)
    {
        Vin = vin;
        Make = make;
        Model = model;
        Year = year;
        Load3DModel();
    }

    /// <summary>Simulate expensive 3D model loading.</summary>
    private void Load3DModel()
    {
        Console.WriteLine(
            $"  [SportsCar] Loading 3D model for {Vin}... " +
            "200 MB of polygon data loaded."
        );
        full3DModel = new()
        {
            ["polygons"] = 450_000,
            ["textures"] = new List<string>
                { "carbon_fiber", "leather_black", "aluminum_brushed" },
            ["materials"] = new List<string>
                { "forged_carbon", "alcantara", "titanium" },
        };
    }

    public string GetShortDescription() =>
        $"{Year} {Make} {Model} ({Vin})";

    public string DisplayDetails()
    {
        int polygons = (int)(full3DModel?["polygons"] ?? 0);
        var textures = (List<string>)(full3DModel?["textures"] ?? new());
        string trunk = string.Join(", ", trunkContents);
        string glovebox = string.Join(", ", gloveboxContents);
        return
            $"--- {GetShortDescription()} ---\n" +
            $"  Engine: {engineType}\n" +
            $"  Horsepower: {horsepower} hp\n" +
            $"  0-60 mph: {acceleration0to60}s\n" +
            $"  Top Speed: {maxSpeed} mph\n" +
            $"  3D Model: {polygons:N0} polygons, {textures.Count} textures\n" +
            $"  Trunk: {trunk}\n" +
            $"  Glovebox: {glovebox}";
    }

    public string Drive(double speed)
    {
        double actualSpeed = Math.Min(speed, maxSpeed);
        isEngineRunning = true;
        currentSpeed = actualSpeed;
        return
            $"  [SportsCar] ROAR! V10 screams through the rev range.\n" +
            $"  Speed: {actualSpeed:F1} mph | Gear: 7th | RPM: 8200";
    }

    public double GetMaxSpeed() => maxSpeed;

    public string AccessTrunk()
    {
        string contents = string.Join(", ", trunkContents);
        return $"  [SportsCar] Trunk opened. Contents: {contents}";
    }
}
```

### Virtual Proxy: LazyCarModelProxy

```csharp
public class LazyCarModelProxy : ICar
{
    private SportsCar? realCar = null;
    public string Vin { get; }
    public string Make { get; }
    public string Model { get; }
    public int Year { get; }

    public LazyCarModelProxy(string vin, string make, string model, int year)
    {
        Vin = vin;
        Make = make;
        Model = model;
        Year = year;
    }

    /// <summary>Initialize the real car on first access.</summary>
    private SportsCar GetRealCar()
    {
        if (realCar == null)
        {
            Console.WriteLine(
                $"  [LazyCarModelProxy] Initializing real SportsCar for " +
                $"{GetShortDescription()}..."
            );
            realCar = new SportsCar(Vin, Make, Model, Year);
        }
        else
        {
            Console.WriteLine("  [LazyCarModelProxy] Reusing existing SportsCar instance.");
        }
        return realCar;
    }

    public string GetShortDescription() =>
        $"{Year} {Make} {Model} ({Vin})";

    public string DisplayDetails() => GetRealCar().DisplayDetails();
    public string Drive(double speed) => GetRealCar().Drive(speed);
    public double GetMaxSpeed() => GetRealCar().GetMaxSpeed();
    public string AccessTrunk() => GetRealCar().AccessTrunk();
}
```

### Protection Proxy: ValetCarProxy

```csharp
public class ValetCarProxy : ICar
{
    private readonly ICar realCar;
    private readonly double maxValetSpeed;
    private readonly string trunkDeniedMessage;

    public ValetCarProxy(ICar realCar, double maxValetSpeed = 30.0)
    {
        this.realCar = realCar;
        this.maxValetSpeed = maxValetSpeed;
        this.trunkDeniedMessage =
            "  [ValetCarProxy] ACCESS DENIED: Valet key does not permit trunk access.";
    }

    public string Vin => realCar.Vin;
    public string Make => realCar.Make;
    public string Model => realCar.Model;
    public int Year => realCar.Year;

    public string GetShortDescription() => realCar.GetShortDescription();

    public string DisplayDetails() =>
        $"--- Valet Card ---\n" +
        $"  VIN: {Vin}\n" +
        $"  Max Permitted Speed: {maxValetSpeed} mph\n" +
        $"  Trunk Access: DENIED\n" +
        $"  Glovebox Access: DENIED";

    public string Drive(double speed)
    {
        double limitedSpeed = Math.Min(speed, maxValetSpeed);
        if (limitedSpeed < speed)
        {
            Console.WriteLine(
                $"  [ValetCarProxy] Speed limited to {maxValetSpeed} mph."
            );
        }
        return realCar.Drive(limitedSpeed);
    }

    public double GetMaxSpeed() => maxValetSpeed;

    public string AccessTrunk() => trunkDeniedMessage;
}
```

### Main / Driver

```csharp
public class Program
{
    public static void Main()
    {
        Console.WriteLine(new string('=', 72));
        Console.WriteLine("  PROXY PATTERN — Cars Domain (C# 12 / .NET 8)");
        Console.WriteLine(new string('=', 72));
        Console.WriteLine();

        // ── Scenario 1: Virtual Proxy (Lazy Loading) ──────────────────────
        Console.WriteLine("--- SCENARIO 1: Virtual Proxy (Lazy Loading) ---");
        ICar lazyCar = new LazyCarModelProxy(
            GenerateVin(), "Sypher", "GT", 2026
        );
        Console.WriteLine(
            "  Showroom: Created LazyCarModelProxy. No 3D model loaded yet."
        );
        Console.WriteLine();

        Console.WriteLine("  Customer browses thumbnail:");
        Console.WriteLine($"  {lazyCar.GetShortDescription()}");
        Console.WriteLine();

        Console.WriteLine("  Customer clicks \"View Full Details\":");
        string details = lazyCar.DisplayDetails();
        Console.WriteLine(details);
        Console.WriteLine();

        // ── Scenario 2: Protection Proxy (Valet Key) ───────────────────────
        Console.WriteLine("--- SCENARIO 2: Protection Proxy (Valet Key) ---");
        ICar realCar = new SportsCar(GenerateVin(), "Sypher", "GT-S", 2026);
        ICar valetKey = new ValetCarProxy(realCar);
        Console.WriteLine();

        Console.WriteLine("  Owner hands the valet key to the parking attendant.");
        Console.WriteLine();

        Console.WriteLine("  Valet: Displays info card:");
        Console.WriteLine(valetKey.DisplayDetails());
        Console.WriteLine();

        Console.WriteLine("  Valet: Attempts to drive at 80 mph:");
        Console.WriteLine(valetKey.Drive(80.0));
        Console.WriteLine();

        Console.WriteLine("  Valet: Attempts to access the trunk:");
        Console.WriteLine(valetKey.AccessTrunk());
        Console.WriteLine();

        // ── Scenario 3: Owner Returns — Full Access ───────────────────────
        Console.WriteLine("--- SCENARIO 3: Owner Returns --- Full Access ---");
        Console.WriteLine("  Owner uses the real car key directly:");
        Console.WriteLine(realCar.Drive(120.0));
        Console.WriteLine(realCar.AccessTrunk());
        Console.WriteLine();

        // ── Scenario 4: Polymorphic Iteration ──────────────────────────────
        Console.WriteLine("--- SCENARIO 4: Polymorphic Car Display ---");
        var cars = new List<ICar> { lazyCar, valetKey };
        foreach (var car in cars)
        {
            Console.WriteLine(
                $"  [{car.Vin}] Max Speed: {car.GetMaxSpeed()} mph"
            );
        }
        Console.WriteLine();

        // ── Scenario 5: Demonstrating Lazy Loading Benefit ────────────────
        Console.WriteLine("--- SCENARIO 5: Lazy Loading Benefit ---");
        var showroomCars = new LazyCarModelProxy[]
        {
            new(GenerateVin(), "Sypher", "GT", 2026),
            new(GenerateVin(), "Sypher", "GT-S", 2026),
            new(GenerateVin(), "Falcon", "X1", 2026),
        };
        Console.WriteLine("  All 3 proxies created. Memory usage: minimal.");
        Console.WriteLine("  Showing thumbnails...");
        foreach (var car in showroomCars)
        {
            Console.WriteLine($"    {car.GetShortDescription()}");
        }
        Console.WriteLine();

        Console.WriteLine("  Customer examines one car in detail:");
        Console.WriteLine(showroomCars[0].DisplayDetails());
        Console.WriteLine();

        Console.WriteLine(new string('=', 72));
        Console.WriteLine("  Proxy pattern demonstration complete.");
        Console.WriteLine(new string('=', 72));
    }
}
```

## Language-Specific Notes

- **Auto-implemented properties for concise accessors** — C#'s `{ get; }` auto-properties on the `ICar` interface provide a clean, concise way to declare read-only properties. The `SportsCar` class implements these with expression-bodied members (`=>`), making the code compact while maintaining full encapsulation. The properties are `init`-only in the constructor.

- **Constructor primary parameters (C# 12)** — The `LazyCarModelProxy` and `SportsCar` classes use traditional constructor parameter syntax. C# 12 primary constructors could also be used for even more concise declarations, but traditional constructors are clearer for educational purposes and preserve the ability to add validation logic.

- **`Dictionary<string, object>` for flexible 3D model data** — The `full3DModel` field uses `Dictionary<string, object>` with collection-expression syntax (`new() { ["key"] = value }`), which is a C# 12 feature. This provides a flexible, type-safe way to store heterogeneous 3D model data without creating a dedicated class.

- **`string.Join()` for list formatting** — C#'s `string.Join(", ", trunkContents)` provides a clean way to join collection elements into a comma-separated string. This is used in `DisplayDetails()` and `AccessTrunk()` to format the trunk and glovebox contents for display.

- **`Math.Min()` for speed clamping** — Both `SportsCar.Drive()` and `ValetCarProxy.Drive()` use `Math.Min()` to ensure the requested speed never exceeds the permitted maximum. This is a simple, reliable pattern for enforcing upper bounds on numeric parameters.

- **`List<ICar>` for polymorphic iteration** — The driver stores heterogeneous car references in a `List<ICar>` and iterates with `foreach`. The `var` keyword is used for local variable type inference, reducing syntactic noise while maintaining full type safety. The `ICar` interface ensures that only valid car operations are available on each element.
