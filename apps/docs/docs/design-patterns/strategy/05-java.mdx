## Implementation

Below is a complete Java implementation of the Strategy pattern using the Stock Market domain. The `TradingBot` (Context) delegates trade signal generation to a pluggable `TradingStrategy` interface. Three concrete strategies implement different trading algorithms: `MovingAverageCrossover`, `MomentumStrategy`, and `MeanReversionStrategy`. The implementation uses Java 17+ features including sealed interfaces, records, pattern matching, and the `List.of()` factory for immutable collections. A fourth strategy (`RsiStrategy`) demonstrates OCP compliance by adding a new algorithm without modifying existing code.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌───────────────────────────────────────────────┐

//   ├───────────────────────────────────────────────┤    SRP: orchestrates
//   │ - strategy: TradingStrategy                   │     DIP: depends on interface
//   │ + setStrategy(TradingStrategy): void          │

//                    │ delegates analyze()
//                    ▼
//   ┌───────────────────────────────────────────────┐
//   │ <<interface>> TradingStrategy                 │ ← OOP: Abstraction
//   ├───────────────────────────────────────────────┤    ISP: single method
//   │ + analyze(List): Signal                       │
//   └────────────────┬──────────────────────────────┘
//                    ▲ implements
//   ┌────────────────┴──────────────────────────────┐
//   │                                               │
//   ┌────────────────┐ ┌────────────────┐          ┌──────────────────┐
//   │MovingAvg       │ │Momentum        │          │MeanReversion     │
//   │Crossover       │ │Strategy        │          │Strategy          │
//   ├────────────────┤ ├────────────────┤          ├──────────────────┤
//   │+ analyze()     │ │+ analyze()     │          │+ analyze()       │
//   └────────────────┘ └────────────────┘          └──────────────────┘
//       OCP: new = new class, no existing code changed
```

#### OOP Concepts Applied

1. **Abstraction** — `TradingStrategy` is a Java interface. All client code depends on this interface.
2. **Encapsulation** — Each strategy class uses `private final` fields. `TradingBot` encapsulates the strategy reference and logging.
3. **Polymorphism** — `runAnalysis()` calls `strategy.analyze(prices)`. The JVM dispatches to the correct concrete implementation.

#### SOLID Principles Applied

4. **SRP** — Each class has one responsibility. `TradingBot` orchestrates. Each strategy implements one algorithm.
5. **OCP** — New strategies implement `TradingStrategy` without changing existing code.
6. **LSP** — Any concrete strategy is substitutable for `TradingStrategy`.
7. **ISP** — The interface has a single method. No class is forced to implement unused methods.
8. **DIP** — `TradingBot` depends on `TradingStrategy`. Concrete strategies are injected.

    SELL,
    HOLD

interface TradingStrategy {
    Signal analyze(List<Double> prices);

### MovingAverageCrossover (Concrete Strategy)

```java
// ── Concrete Strategy 1: Moving Average Crossover ──
// OOP role: Polymorphism — implements the strategy contract
// SOLID: SRP (only crossover), LSP (substitutable), OCP (extensible)
final class MovingAverageCrossover implements TradingStrategy {
    private final int shortWindow;
    private final int longWindow;

    public MovingAverageCrossover(int shortWindow, int longWindow) {
        if (shortWindow >= longWindow) {
            throw new IllegalArgumentException("shortWindow must be less than longWindow");
        }
        this.shortWindow = shortWindow;
        this.longWindow = longWindow;
    }

    public MovingAverageCrossover() {
        this(20, 50);
    }

    @Override
    public Signal analyze(List<Double> prices) {
        if (prices.size() < longWindow) {
            return Signal.HOLD;
        }

        double shortMA = computeAverage(prices, shortWindow);
        double longMA = computeAverage(prices, longWindow);

        if (shortMA > longMA) {
            return Signal.BUY;
        } else if (shortMA < longMA) {
            return Signal.SELL;
        }
        return Signal.HOLD;
    }

    private double computeAverage(List<Double> prices, int window) {
        double sum = 0.0;
        for (int i = prices.size() - window; i < prices.size(); i++) {
            sum += prices.get(i);
        }
        return sum / window;
    }
}

```

### MomentumStrategy (Concrete Strategy)

```java
// ── Concrete Strategy 2: Momentum Strategy ──
// OOP role: Polymorphism — implements the strategy contract
// SOLID: SRP (only momentum), LSP (substitutable), OCP (extensible)
final class MomentumStrategy implements TradingStrategy {
    private final int lookbackPeriod;
    private final double threshold;

    public MomentumStrategy(int lookbackPeriod, double threshold) {
        this.lookbackPeriod = lookbackPeriod;
        this.threshold = threshold;
    }

    public MomentumStrategy() {
        this(10, 0.05);
    }

    @Override
    public Signal analyze(List<Double> prices) {
        if (prices.size() < lookbackPeriod + 1) {
            return Signal.HOLD;
        }

        double currentPrice = prices.get(prices.size() - 1);
        double pastPrice = prices.get(prices.size() - 1 - lookbackPeriod);
        double momentum = (currentPrice - pastPrice) / pastPrice;

        if (momentum > threshold) {
            return Signal.BUY;
        } else if (momentum < -threshold) {
            return Signal.SELL;
        }
        return Signal.HOLD;
    }
}

```

### MeanReversionStrategy (Concrete Strategy)

```java
// ── Concrete Strategy 3: Mean Reversion Strategy ──
// OOP role: Polymorphism — implements the strategy contract
// SOLID: SRP (only reversion), LSP (substitutable), OCP (extensible)
final class MeanReversionStrategy implements TradingStrategy {
    private final int window;
    private final double deviationThreshold;

    public MeanReversionStrategy(int window, double deviationThreshold) {
        this.window = window;
        this.deviationThreshold = deviationThreshold;
    }

    public MeanReversionStrategy() {
        this(20, 0.03);
    }

    @Override
    public Signal analyze(List<Double> prices) {
        if (prices.size() < window) {
            return Signal.HOLD;
        }

        double sum = 0.0;
        for (int i = prices.size() - window; i < prices.size(); i++) {
            sum += prices.get(i);
        }
        double mean = sum / window;
        double current = prices.get(prices.size() - 1);
        double deviation = (current - mean) / mean;

        if (deviation < -deviationThreshold) {
            return Signal.BUY;
        } else if (deviation > deviationThreshold) {
            return Signal.SELL;
        }
        return Signal.HOLD;
    }
}

```

### RsiStrategy (Concrete Strategy)

```java
// ── Concrete Strategy 4: RSI Strategy (added without modifying existing code) ──
// OOP role: OCP extension — new strategy, no existing code changed
// SOLID: OCP (open for extension, closed for modification)
final class RsiStrategy implements TradingStrategy {
    private final int period;
    private final int overbought;
    private final int oversold;

    public RsiStrategy(int period, int overbought, int oversold) {
        this.period = period;
        this.overbought = overbought;
        this.oversold = oversold;
    }

    public RsiStrategy() {
        this(14, 70, 30);
    }

    @Override
    public Signal analyze(List<Double> prices) {
        if (prices.size() < period + 1) {
            return Signal.HOLD;
        }

        double totalGain = 0.0;
        double totalLoss = 0.0;

        int startIdx = prices.size() - period;
        for (int i = startIdx; i < prices.size(); i++) {
            double change = prices.get(i) - prices.get(i - 1);
            if (change >= 0) {
                totalGain += change;
            } else {
                totalLoss += Math.abs(change);
            }
        }

        double avgGain = totalGain / period;
        double avgLoss = totalLoss / period;

        if (avgLoss == 0.0) {
            return Signal.BUY;
        }

        double rs = avgGain / avgLoss;
        double rsi = 100.0 - (100.0 / (1.0 + rs));

        if (rsi >= overbought) {
            return Signal.SELL;
        } else if (rsi <= oversold) {
            return Signal.BUY;
        }
        return Signal.HOLD;
    }
}

```

### TradingBot (Context)

```java
// ── Context: TradingBot ──
// OOP role: Encapsulation — holds strategy reference and delegates
// SOLID: SRP (only orchestrates), DIP (depends on TradingStrategy interface)
final class TradingBot {
    private TradingStrategy strategy;

    public TradingBot() {
        this.strategy = null;
    }

    public void setStrategy(TradingStrategy strategy) {
        this.strategy = strategy;
    }

    public String getStrategyName() {
        if (strategy == null) {
            return "None";
        }
        return strategy.getClass().getSimpleName();
    }

    public Signal runAnalysis(String ticker, List<Double> prices) {
        if (strategy == null) {
            throw new IllegalStateException("No strategy set. Call setStrategy() first.");
        }
        Signal signal = strategy.analyze(prices);
        logSignal(ticker, signal);
        return signal;
    }

    private void logSignal(String ticker, Signal signal) {
        String timestamp = LocalDateTime.now().format(DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss"));
        System.out.printf("[%s] %s -> %s (strategy: %s)%n",
            timestamp, ticker, signal, getStrategyName());
    }
}

```

### Language-Specific Notes

- **Interfaces with default methods** — Java interfaces can include `default` method implementations. The `TradingStrategy` interface uses only abstract methods, but in a more complex scenario, default methods could provide shared utility logic.
- **`final` classes for safety** — Each concrete strategy is declared `final` to prevent subclassing. This ensures the OCP is applied correctly — new behavior comes from new classes, not from overriding existing ones.
- **`List.of()` for immutable collections** — Java 9+ `List.of()` creates immutable lists, preventing accidental modification of price data. This aligns with the `readonly` approach in TypeScript and Python's `Final`.
- **Records for data carriers** — The `NamedStrategy` record inside the `main` method provides a concise, immutable pair of strategy name and instance. Records auto-generate `equals()`, `hashCode()`, `toString()`, and accessor methods.
- **`var` for local type inference** — Java 10+ `var` keyword reduces boilerplate in local variable declarations. The compiler infers the type, keeping the code concise while maintaining full static type safety.
