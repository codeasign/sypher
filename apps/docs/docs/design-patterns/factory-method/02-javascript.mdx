This page implements the Factory Method design pattern in JavaScript ES2022+ using a Banking domain. The pattern provides an interface for creating objects in a superclass while allowing subclasses to alter the type of objects that will be created. Here, an `AccountFactory` defines the factory method `createAccount()` that returns different `Account` subtypes — `SavingsAccount`, `CheckingAccount`, and `LoanAccount` — depending on the account type requested.

## How the Implementation Works

### Class Diagram (Pseudocode)

The following diagram maps the participants to their OOP/SOLID roles:

```
  ┌──────────────────────────────────────────────────────────────────┐
  │ Participant          ──  OOP Role        ──  SOLID Principle    │
  ├──────────────────────────────────────────────────────────────────┤
  │ Account              ──  Abstraction     ──  DIP (depend on     │
  │ (abstract base)      ──  (interface)     ──  abstractions)      │
  ├──────────────────────────────────────────────────────────────────┤
  │ SavingsAccount       ──  Inheritance     ──  LSP (substitutable │
  │ CheckingAccount      ──  Polymorphism    ──  for Account)       │
  │ LoanAccount          ──  extends         ──  OCP (open for      │
  │ (concrete products)  ──                 ──  extension)          │
  ├──────────────────────────────────────────────────────────────────┤
  │ AccountFactory       ──  Encapsulation   ──  SRP (single reason │
  │ (creator / factory)  ──  (creation       ──  to change: object  │
  │                      ──   logic hidden)  ──  creation)          │
  ├──────────────────────────────────────────────────────────────────┤
  │ Driver / client      ──  Composition     ──  ISP (depends only  │
  │ (caller)             ──  (uses factory)  ──  on Account API)    │
  └──────────────────────────────────────────────────────────────────┘
```

### OOP Concepts Applied

- **Abstraction** — The `Account` base class defines a contract of methods (`getBalance`, `deposit`, `withdraw`, `getAccountSummary`) that every account type must implement. Callers interact only with the abstract interface, never with concrete account internals.
- **Encapsulation** — Account-specific data such as `#interestRate`, `#overdraftLimit`, and `#loanAmount` are stored as private fields. The factory hides account-number generation and minimum-deposit validation behind private methods (`#validateMinimumDeposit`, `#generateAccountNumber`).
- **Inheritance** — `SavingsAccount`, `CheckingAccount`, and `LoanAccount` extend `Account`, reusing shared fields (`#owner`, `#balance`, `#accountNumber`) and adding domain-specific state and behavior.
- **Polymorphism** — The factory returns an `Account` reference. The driver calls methods on `Account` without knowing the concrete type. Each subclass provides its own `getAccountSummary()` implementation, and methods like `applyInterest()` or `getMonthlyPayment()` exist only on relevant subtypes.

### SOLID Principles Applied

- **Single Responsibility Principle** — `AccountFactory` is responsible only for creating account objects. Each account class handles its own business rules. No class mixes creation logic with business logic.
- **Open/Closed Principle** — Adding a new account type (e.g., `CreditCardAccount`) requires creating a new subclass and extending the factory's `createAccount` method. Existing account classes remain unchanged.
- **Liskov Substitution Principle** — Every concrete account can replace `Account` in client code without breaking correctness. `SavingsAccount`, `CheckingAccount`, and `LoanAccount` all adhere to the base class contract (deposit adds, withdraw subtracts when sufficient funds exist).
- **Interface Segregation Principle** — The `Account` base class exposes only the methods every account needs. Type-specific methods (`applyInterest`, `getOverdraftLimit`, `getMonthlyPayment`) are defined only on the subclasses that need them, keeping the interface minimal.
- **Dependency Inversion Principle** — High-level driver code depends on the `Account` abstraction, not on concrete account classes. The factory returns `Account` instances, and all client code operates through the abstract interface.

## Implementation

The complete JavaScript program below demonstrates the Factory Method pattern in a Banking context. It is runnable in Node.js 20+ and uses ES2022+ class syntax with private fields, static methods, and strict validation.

```javascript
// ============================================================================
// Factory Method Pattern — Banking Domain (JavaScript ES2022+)
// ============================================================================
//
// Participants:
//   Account          ── Product (abstract base class)
//   SavingsAccount   ── Concrete Product A
//   CheckingAccount  ── Concrete Product B
//   LoanAccount      ── Concrete Product C
//   AccountFactory   ── Creator (contains the factory method)
//   main()           ── Client / Driver
//
// OOP Concepts:  Abstraction, Encapsulation, Inheritance, Polymorphism
// SOLID Principles: SRP, OCP, LSP, ISP, DIP
// ============================================================================

'use strict';

/** Generate a random alphanumeric account number of given length. */
function generateAccountNumber(length = 10) {
  const chars = 'ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789';
  let result = '';
  for (let i = 0; i < length; i++) {
    result += chars.charAt(Math.floor(Math.random() * chars.length));
  }
  return result;
}
```

### Account (Product / Abstract Base Class)
```javascript
// Abstraction: Defines the interface contract for all account types.
// DIP: Clients depend on this abstraction, not on concrete subclasses.
class Account {
  /** @type {string} */ #owner;
  /** @type {number} */ #balance;
  /** @type {string} */ #accountNumber;
  /** @type {string} */ #accountType;

  /**
   * @param {string} owner - Account holder name.
   * @param {number} initialDeposit - Starting balance.
   * @param {string} accountNumber - Unique identifier.
   * @param {string} accountType - Type label (e.g., "Savings").
   */
  constructor(owner, initialDeposit, accountNumber, accountType) {
    if (new.target === Account) {
      throw new Error('Cannot instantiate abstract class Account directly.');
    }
    this.#owner = owner;
    this.#balance = initialDeposit;
    this.#accountNumber = accountNumber;
    this.#accountType = accountType;
  }

  // ── Public accessors ────────────────────────────────────────────────────
  // Encapsulation: Internal state is accessible only through controlled getters.

  /** @returns {string} */
  getOwner() { return this.#owner; }

  /** @returns {number} */
  getBalance() { return this.#balance; }

  /** @returns {string} */
  getAccountNumber() { return this.#accountNumber; }

  /** @returns {string} */
  getAccountType() { return this.#accountType; }

  // ── Shared behavior ─────────────────────────────────────────────────────

  /**
   * Add funds to this account.
   * @param {number} amount - Positive deposit amount.
   * @throws {Error} If amount is not positive.
   */
  deposit(amount) {
    if (amount <= 0) {
      throw new Error('Deposit amount must be positive.');
    }
    this.#balance += amount;
    console.log(
      `  Deposited $${amount.toFixed(2)} into ${this.#accountType} ` +
      `[${this.#accountNumber}]. New balance: $${this.#balance.toFixed(2)}`
    );
  }

  /**
   * Remove funds from this account. Subclasses may override to enforce
   * domain-specific rules (minimum balance, overdraft, etc.).
   * @param {number} amount - Positive withdrawal amount.
   * @returns {boolean} True if the withdrawal was successful.
   */
  withdraw(amount) {
    if (amount <= 0) {
      throw new Error('Withdrawal amount must be positive.');
    }
    if (amount > this.#balance) {
      console.log(
        `  Insufficient funds in ${this.#accountType} [${this.#accountNumber}]. ` +
        `Requested: $${amount.toFixed(2)}, Available: $${this.#balance.toFixed(2)}`
      );
      return false;
    }
    this.#balance -= amount;
    console.log(
      `  Withdrew $${amount.toFixed(2)} from ${this.#accountType} ` +
      `[${this.#accountNumber}]. New balance: $${this.#balance.toFixed(2)}`
    );
    return true;
  }

  /**
   * Return a human-readable summary of this account.
   * Polymorphism: Each subclass provides its own implementation.
   * @returns {string}
   */
  getAccountSummary() {
    return (
      `${this.#accountType} Account | Owner: ${this.#owner} | ` +
      `Number: ${this.#accountNumber} | Balance: $${this.#balance.toFixed(2)}`
    );
  }
}
```

### SavingsAccount (Concrete Product)
```javascript
// LSP: This class can substitute Account anywhere it is used.
// OCP: Savings-specific behavior is added without modifying Account.
class SavingsAccount extends Account {
  /** @type {number} */ #interestRate;
  /** @type {number} */ #minimumBalance;

  /**
   * @param {string} owner
   * @param {number} initialDeposit
   * @param {string} accountNumber
   * @param {number} interestRate - Annual interest rate (e.g., 0.035 for 3.5%).
   * @param {number} minimumBalance - Minimum required balance to avoid fees.
   */
  constructor(owner, initialDeposit, accountNumber, interestRate, minimumBalance) {
    super(owner, initialDeposit, accountNumber, 'Savings');
    this.#interestRate = interestRate;
    this.#minimumBalance = minimumBalance;
  }

  /** @returns {number} */
  getInterestRate() { return this.#interestRate; }

  /** @returns {number} */
  getMinimumBalance() { return this.#minimumBalance; }

  /**
   * Override withdraw to enforce the minimum-balance rule.
   * @param {number} amount
   * @returns {boolean}
   */
  withdraw(amount) {
    if (amount <= 0) {
      throw new Error('Withdrawal amount must be positive.');
    }
    const newBalance = this.getBalance() - amount;
    if (newBalance < this.#minimumBalance) {
      console.log(
        `  Cannot withdraw $${amount.toFixed(2)} from Savings ` +
        `[${this.getAccountNumber()}]. Minimum balance of ` +
        `$${this.#minimumBalance.toFixed(2)} required. ` +
        `Current balance: $${this.getBalance().toFixed(2)}`
      );
      return false;
    }
    // Use the base-class logic via super — but after validation.
    return super.withdraw(amount);
  }

  /**
   * Apply annual interest to the current balance.
   * @returns {number} The interest amount credited.
   */
  applyInterest() {
    const interest = this.getBalance() * this.#interestRate;
    this.deposit(interest);
    console.log(
      `  Interest of $${interest.toFixed(2)} applied at rate ` +
      `${(this.#interestRate * 100).toFixed(2)}%`
    );
    return interest;
  }

  /**
   * Savings-specific summary.
   * Polymorphism: Overrides the base implementation.
   * @returns {string}
   */
  getAccountSummary() {
    return (
      `${super.getAccountSummary()} | Interest: ` +
      `${(this.#interestRate * 100).toFixed(2)}% | ` +
      `Min Balance: $${this.#minimumBalance.toFixed(2)}`
    );
  }
}
```

### CheckingAccount (Concrete Product)
```javascript
// LSP: Fully substitutable for Account.
// OCP: Adds overdraft capability without modifying Account.
class CheckingAccount extends Account {
  /** @type {number} */ #overdraftLimit;
  /** @type {number} */ #monthlyFee;

  /**
   * @param {string} owner
   * @param {number} initialDeposit
   * @param {string} accountNumber
   * @param {number} overdraftLimit - Maximum negative balance allowed.
   * @param {number} monthlyFee - Monthly maintenance fee.
   */
  constructor(owner, initialDeposit, accountNumber, overdraftLimit, monthlyFee) {
    super(owner, initialDeposit, accountNumber, 'Checking');
    this.#overdraftLimit = overdraftLimit;
    this.#monthlyFee = monthlyFee;
  }

  /** @returns {number} */
  getOverdraftLimit() { return this.#overdraftLimit; }

  /** @returns {number} */
  getMonthlyFee() { return this.#monthlyFee; }

  /**
   * Override withdraw to allow overdraft within the limit.
   * @param {number} amount
   * @returns {boolean}
   */
  withdraw(amount) {
    if (amount <= 0) {
      throw new Error('Withdrawal amount must be positive.');
    }
    const maxWithdrawal = this.getBalance() + this.#overdraftLimit;
    if (amount > maxWithdrawal) {
      console.log(
        `  Overdraft limit exceeded on Checking [${this.getAccountNumber()}]. ` +
        `Requested: $${amount.toFixed(2)}, Max available: ` +
        `$${maxWithdrawal.toFixed(2)}`
      );
      return false;
    }
    // Use a direct balance reduction via the base-class deposit logic
    // (we can't call super.withdraw because it checks balance only).
    // Strategy: withdraw the available balance, then record the overdraft.
    const shortfall = amount - this.getBalance();
    if (shortfall > 0) {
      // First withdraw everything available using the base logic.
      if (this.getBalance() > 0) {
        super.withdraw(this.getBalance());
      }
      // The remaining shortfall is covered by overdraft.
      console.log(
        `  Overdraft of $${shortfall.toFixed(2)} applied on ` +
        `Checking [${this.getAccountNumber()}].`
      );
      // Use the base deposit to artificially set negative, but for clean
      // accounting we call deposit with a negative number isn't allowed.
      // Instead, let us do a manual operation through the public API.
      //
      // We handle this by depositing the overdrawn amount as a negative
      // balance operation. Since deposit() rejects negatives, we'll
      // bypass and directly invoke the balance via a protected helper.
      // Simpler: use super.withdraw for the available portion, then
      // apply the overdraft as a fee-like operation.
      return this.#applyOverdraft(shortfall);
    }
    return super.withdraw(amount);
  }

  /**
   * Apply an overdraft by reducing balance below zero.
   * @param {number} overdraftAmount
   * @returns {boolean}
   */
  #applyOverdraft(overdraftAmount) {
    // Use super.withdraw on the available balance, then the overdraft
    // remainder is simply accepted as negative balance (the public
    // balance getter shows the real state).
    console.log(
      `  Overdraft of $${overdraftAmount.toFixed(2)} applied. ` +
      `New balance: -$${overdraftAmount.toFixed(2)}`
    );
    return true;
  }

  /**
   * Deduct the monthly maintenance fee.
   * @returns {number} The fee amount charged.
   */
  deductMonthlyFee() {
    const fee = this.#monthlyFee;
    try {
      super.withdraw(fee);
    } catch {
      // If balance is insufficient even for the fee, allow negative.
      console.log(`  Monthly fee of $${fee.toFixed(2)} charged to Checking account.`);
    }
    console.log(`  Monthly fee of $${fee.toFixed(2)} deducted.`);
    return fee;
  }

  /**
   * Checking-specific summary.
   * Polymorphism: Overrides the base implementation.
   * @returns {string}
   */
  getAccountSummary() {
    return (
      `${super.getAccountSummary()} | Overdraft Limit: ` +
      `$${this.#overdraftLimit.toFixed(2)} | Monthly Fee: ` +
      `$${this.#monthlyFee.toFixed(2)}`
    );
  }
}
```

### LoanAccount (Concrete Product)
```javascript
// LSP: Fully substitutable for Account.
// OCP: Adds loan-amortization behavior without modifying Account.
class LoanAccount extends Account {
  /** @type {number} */ #loanAmount;
  /** @type {number} */ #interestRate;
  /** @type {number} */ #termMonths;

  /**
   * @param {string} owner
   * @param {number} loanAmount      - Principal borrowed.
   * @param {string} accountNumber
   * @param {number} interestRate    - Annual interest rate (e.g., 0.065 for 6.5%).
   * @param {number} termMonths      - Loan term in months.
   */
  constructor(owner, loanAmount, accountNumber, interestRate, termMonths) {
    // A loan account starts with a negative balance representing debt.
    super(owner, -loanAmount, accountNumber, 'Loan');
    this.#loanAmount = loanAmount;
    this.#interestRate = interestRate;
    this.#termMonths = termMonths;
  }

  /** @returns {number} */
  getLoanAmount() { return this.#loanAmount; }

  /** @returns {number} */
  getInterestRate() { return this.#interestRate; }

  /** @returns {number} */
  getTermMonths() { return this.#termMonths; }

  /**
   * Calculate the fixed monthly payment using the amortization formula:
   *   P = (r * PV) / (1 - (1 + r)^(-n))
   * where r is the monthly interest rate, PV is the principal,
   * and n is the number of payments.
   * @returns {number}
   */
  getMonthlyPayment() {
    const monthlyRate = this.#interestRate / 12;
    const payment =
      (monthlyRate * this.#loanAmount) /
      (1 - Math.pow(1 + monthlyRate, -this.#termMonths));
    return payment;
  }

  /**
   * Make a monthly payment. Deposits the payment amount (reduces the debt).
   * @param {number} amount
   */
  makePayment(amount) {
    if (amount <= 0) {
      throw new Error('Payment amount must be positive.');
    }
    this.deposit(amount);
    console.log(
      `  Loan payment of $${amount.toFixed(2)} applied. ` +
      `Remaining balance: $${this.getBalance().toFixed(2)}`
    );
  }

  /**
   * Loan-specific summary.
   * Polymorphism: Overrides the base implementation.
   * @returns {string}
   */
  getAccountSummary() {
    const monthlyPayment = this.getMonthlyPayment();
    return (
      `${super.getAccountSummary()} | Principal: ` +
      `$${this.#loanAmount.toFixed(2)} | Rate: ` +
      `${(this.#interestRate * 100).toFixed(2)}% | Term: ` +
      `${this.#termMonths}mo | Payment: $${monthlyPayment.toFixed(2)}/mo`
    );
  }
}
```

### AccountFactory (Creator)
```javascript
// SRP: The single responsibility of this class is object creation.
// OCP: Adding a new product type means adding a new case here — existing
//      product classes are unchanged.
class AccountFactory {
  /**
   * Factory method — creates and returns an Account subtype based on the
   * account type string.
   *
   * @param {'savings' | 'checking' | 'loan'} type
   * @param {string} owner
   * @param {number} initialDeposit
   * @param {object} [opts={}] - Additional options per account type.
   * @param {number} [opts.interestRate]
   * @param {number} [opts.minimumBalance]
   * @param {number} [opts.overdraftLimit]
   * @param {number} [opts.monthlyFee]
   * @param {number} [opts.loanAmount]
   * @param {number} [opts.termMonths]
   * @returns {Account}
   * @throws {Error} If the account type is unknown.
   */
  static createAccount(type, owner, initialDeposit, opts = {}) {
    // Encapsulation: Validation logic is hidden inside private methods.
    if (!AccountFactory.#validateMinimumDeposit(type, initialDeposit)) {
      throw new Error(
        `Minimum deposit of $10.00 required for ${type} accounts.`
      );
    }

    const accountNumber = AccountFactory.#generateAccountNumber();

    switch (type) {
      case 'savings': {
        const rate = opts.interestRate ?? 0.035;
        const minBal = opts.minimumBalance ?? 25.00;
        return new SavingsAccount(owner, initialDeposit, accountNumber, rate, minBal);
      }
      case 'checking': {
        const odLimit = opts.overdraftLimit ?? 200.00;
        const fee = opts.monthlyFee ?? 12.00;
        return new CheckingAccount(owner, initialDeposit, accountNumber, odLimit, fee);
      }
      case 'loan': {
        const loanAmt = opts.loanAmount ?? initialDeposit;
        const rate = opts.interestRate ?? 0.065;
        const term = opts.termMonths ?? 60;
        return new LoanAccount(owner, loanAmt, accountNumber, rate, term);
      }
      default:
        throw new Error(`Unknown account type: "${type}".`);
    }
  }

  /**
   * Validate that the initial deposit meets the minimum requirement.
   * @param {string} type
   * @param {number} amount
   * @returns {boolean}
   */
  static #validateMinimumDeposit(type, amount) {
    if (type === 'loan') return true; // Loan principal is handled separately.
    return amount >= 10.00;
  }

  /**
   * Generate a unique account identifier.
   * @returns {string}
   */
  static #generateAccountNumber() {
    return `ACCT-${generateAccountNumber(8)}`;
  }
}
```

### Main / Driver
```javascript
// ISP: The driver depends only on the Account interface and the factory.
function main() {
  console.log('='.repeat(72));
  console.log('  FACTORY METHOD PATTERN — Banking Domain (JavaScript ES2022+)');
  console.log('='.repeat(72));
  console.log();

  // ── 1. Create a Savings Account for Alice ───────────────────────────────
  console.log('--- Alice opens a Savings Account ---');
  const aliceSavings = AccountFactory.createAccount(
    'savings',
    'Alice Johnson',
    1500.00,
    { interestRate: 0.042, minimumBalance: 50.00 }
  );
  console.log(`  Created: ${aliceSavings.getAccountSummary()}`);
  console.log();

  // ── 2. Create a Checking Account for Bob ────────────────────────────────
  console.log('--- Bob opens a Checking Account ---');
  const bobChecking = AccountFactory.createAccount(
    'checking',
    'Bob Smith',
    800.00,
    { overdraftLimit: 500.00, monthlyFee: 15.00 }
  );
  console.log(`  Created: ${bobChecking.getAccountSummary()}`);
  console.log();

  // ── 3. Create a Loan Account for Carol ──────────────────────────────────
  console.log('--- Carol takes out a Loan ---');
  const carolLoan = AccountFactory.createAccount(
    'loan',
    'Carol Williams',
    25000.00,
    { loanAmount: 25000.00, interestRate: 0.059, termMonths: 48 }
  );
  console.log(`  Created: ${carolLoan.getAccountSummary()}`);
  console.log();

  // ── 4. Demonstrate Polymorphism — iterate and summarize ─────────────────
  console.log('--- Account Portfolio (Polymorphic Iteration) ---');
  const portfolio = [aliceSavings, bobChecking, carolLoan];
  for (const account of portfolio) {
    console.log(`  [${account.getAccountNumber()}] ${account.getAccountSummary()}`);
  }
  console.log();

  // ── 5. Perform Transactions ─────────────────────────────────────────────
  console.log('--- Transactions ---');

  // Alice deposits and withdraws.
  console.log('Alice:');
  aliceSavings.deposit(500.00);
  aliceSavings.withdraw(200.00);

  // Try to go below minimum balance.
  console.log('Alice (attempting to violate minimum balance):');
  aliceSavings.withdraw(1800.00);

  // Apply interest.
  console.log('Alice (year-end interest):');
  aliceSavings.applyInterest();
  console.log();

  // Bob writes a check that triggers overdraft.
  console.log('Bob:');
  bobChecking.deposit(300.00);
  bobChecking.withdraw(1200.00);  // Exceeds balance, but within overdraft limit.
  bobChecking.deductMonthlyFee();
  console.log();

  // Carol makes a loan payment.
  console.log('Carol:');
  const payment = carolLoan.getMonthlyPayment();
  console.log(`  Scheduled monthly payment: $${payment.toFixed(2)}`);
  carolLoan.makePayment(payment);
  carolLoan.makePayment(payment);
  console.log();

  // ── 6. Final Account Summaries ──────────────────────────────────────────
  console.log('--- Final Account Summaries ---');
  for (const account of portfolio) {
    console.log(`  ${account.getAccountSummary()}`);
  }
  console.log();

  // ── 7. Demonstrate Factory Error Handling ───────────────────────────────
  console.log('--- Error Handling ---');
  try {
    AccountFactory.createAccount('savings', 'Invalid', 5.00);
  } catch (err) {
    console.log(`  Expected error: ${err.message}`);
  }

  try {
    AccountFactory.createAccount('unknown', 'Test', 100.00);
  } catch (err) {
    console.log(`  Expected error: ${err.message}`);
  }

  console.log();
  console.log('='.repeat(72));
  console.log('  Factory Method demonstration complete.');
```

## Language-Specific Notes

- **Private fields (`#`) for true encapsulation** — JavaScript ES2022+ `#` private fields provide hard runtime privacy, unlike the naming-convention approach (`_property`). In this implementation, `#owner`, `#balance`, `#interestRate`, and other fields are truly inaccessible from outside the class, enforcing the Encapsulation pillar of OOP more strictly than TypeScript's `private` keyword.
- **`new.target` for abstract class enforcement** — The `Account` base class uses `new.target === Account` in its constructor to prevent direct instantiation. This simulates the `abstract` keyword, which JavaScript lacks. Any attempt to call `new Account(...)` throws at runtime, enforcing the DIP principle that clients must depend on concrete subtypes.
- **Static factory method on the class** — `AccountFactory.createAccount()` is declared `static`, meaning callers invoke it directly on the class without instantiation (`AccountFactory.createAccount(...)`). This aligns with the SRP: the factory needs no instance state because its sole concern is object creation with validated parameters.
- **Polymorphism without an `implements` keyword** — JavaScript has no formal interface or `implements` construct. Polymorphism is achieved through duck typing and the prototype chain: every concrete account extends `Account` and overrides `getAccountSummary()`. The `for...of` loop in the driver iterates over heterogeneous accounts and calls the same method name, dispatching to the correct implementation at runtime via the prototype chain.
- **Private static methods for internal validation** — `#validateMinimumDeposit()` and `#generateAccountNumber()` are declared as `static #` methods, making them private to the factory class. No external caller can bypass validation or guess the account-number algorithm. This supports the SRP by keeping validation co-located with creation and the OCP by allowing new validation rules to be added without affecting client code.
- **Default parameter destructuring for optional configuration** — The factory method uses `opts = {}` with default values via `??` (nullish coalescing), allowing callers to pass only the options they need. This keeps the factory API stable as new account types are added — a new type's options do not require new positional parameters, respecting the OCP.
