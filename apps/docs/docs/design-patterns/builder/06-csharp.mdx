The Builder pattern in C# 12 / .NET 8 leverages language features like `record` types for immutable data transfer objects, `interface` definitions with default implementations, primary constructors, and nullable reference types. In this Banking example, an `ILoanApplicationBuilder` interface defines the steps for constructing a `LoanApplication` domain object. Three concrete builders — `PersonalLoanBuilder`, `MortgageBuilder`, and `AutoLoanBuilder` — each implement the same interface but produce loan applications with different structures and validation rules. A `LoanDirector` orchestrates the construction sequence for each loan type, keeping the client code decoupled from the assembly logic.

## How the Implementation Works

The `ILoanApplicationBuilder` interface declares the step-by-step construction methods. Each concrete builder maintains its own internal state — a `LoanApplication` being assembled — and returns the final product via `GetResult()`. The `LoanDirector` accepts any builder that conforms to the interface and calls the appropriate sequence of steps for each loan type. The product (`LoanApplication`) is a C# 12 `record` with immutable properties, making it safe to pass around after construction.

### Class Diagram (Pseudocode)

```
interface ILoanApplicationBuilder {
    void SetPersonalInfo(string firstName, string lastName, string email)
    void SetLoanTerms(string loanType, decimal amount, int termMonths)
    void SetInterestRate(double apr)
    void AddDocument(string name, string type)
    LoanApplication GetResult()
}

class PersonalLoanBuilder : ILoanApplicationBuilder { ... }
class MortgageBuilder : ILoanApplicationBuilder { ... }
class AutoLoanBuilder : ILoanApplicationBuilder { ... }

class LoanDirector {
    void ConstructPersonalLoan(ILoanApplicationBuilder builder)
    void ConstructMortgage(ILoanApplicationBuilder builder)
    void ConstructAutoLoan(ILoanApplicationBuilder builder)
}

record LoanApplication { ... }
```

### OOP Concepts Applied

- **Abstraction** — The `ILoanApplicationBuilder` interface hides the internal construction details behind a clean set of methods. Clients and the director interact only with the abstraction.
- **Encapsulation** — Each concrete builder encapsulates its own state (amount, term, documents, property info, vehicle info) and exposes only the builder interface methods to mutate it.
- **Inheritance** — All three concrete builders implement the same `ILoanApplicationBuilder` interface, allowing polymorphic substitution through the director.
- **Polymorphism** — The `LoanDirector` operates on `ILoanApplicationBuilder` references. The same director method produces different loan products depending on which concrete builder is passed in.

### SOLID Principles Applied

- **Single Responsibility Principle** — Each builder is responsible for constructing one kind of loan product. The director is responsible for orchestrating the build sequence. The product itself is a simple data carrier.
- **Open/Closed Principle** — New loan types (e.g., `StudentLoanBuilder`) can be added by implementing `ILoanApplicationBuilder` without modifying existing builders or the director.
- **Liskov Substitution Principle** — Any concrete builder can replace `ILoanApplicationBuilder` in the director without altering the correctness of the construction process.
- **Interface Segregation Principle** — The `ILoanApplicationBuilder` interface is focused on loan construction steps. Mortgage-specific methods like `SetPropertyInfo` are exposed optionally on `MortgageBuilder` rather than forced onto the interface.
- **Dependency Inversion Principle** — The `LoanDirector` depends on the `ILoanApplicationBuilder` abstraction, not on concrete builder classes. The client injects the desired builder at runtime.

```csharp

public sealed record PersonalInfo(
    string FirstName,

    int TermMonths,
```

### Builder Interface
```csharp

public interface ILoanApplicationBuilder
{

    void SetLoanTerms(string loanType, decimal amount, int termMonths);
    void SetInterestRate(double apr);
```

### PersonalLoanBuilder (ConcreteBuilder)
```csharp
// ── Concrete Builder: PersonalLoan ──────────────────────────────────
public sealed class PersonalLoanBuilder : ILoanApplicationBuilder
{
    private PersonalInfo? _personalInfo;
    private decimal _amount;
    private int _termMonths;
    private double _interestRate;
    private readonly List<LoanDocument> _documents = [];

    public void SetPersonalInfo(string firstName, string lastName, string email)
    {
        _personalInfo = new PersonalInfo(firstName, lastName, email);
    }

    public void SetLoanTerms(string loanType, decimal amount, int termMonths)
    {
        _amount = amount;
        _termMonths = termMonths;
    }

    public void SetInterestRate(double apr)
    {
        _interestRate = apr;
    }

    public void AddDocument(string name, string type)
    {
        _documents.Add(new LoanDocument(name, type));
    }

    public LoanApplication GetResult()
    {
        return new LoanApplication(
            PersonalInfo: _personalInfo,
            LoanType: "Personal",
            Amount: _amount,
            TermMonths: _termMonths,
            InterestRate: _interestRate,
            Documents: _documents.AsReadOnly()
        );
    }
}
```

### LoanDirector (Director)
```csharp
// ── Director ────────────────────────────────────────────────────────
public sealed class LoanDirector
{
    public void ConstructPersonalLoan(ILoanApplicationBuilder builder)
    {
        builder.SetPersonalInfo("Alice", "Johnson", "alice.johnson@example.com");
        builder.SetLoanTerms("Personal", 15000m, 60);
        builder.SetInterestRate(11.99);
        builder.AddDocument("Government ID", "Identification");
        builder.AddDocument("Pay Stubs", "Income Verification");
    }

    public void ConstructMortgage(ILoanApplicationBuilder builder)
    {
        builder.SetPersonalInfo("Bob", "Martinez", "bob.martinez@example.com");
        builder.SetLoanTerms("Mortgage", 350000m, 360);
        builder.SetInterestRate(6.25);

        if (builder is MortgageBuilder mb)
        {
            mb.SetPropertyInfo("123 Main St", "Springfield", 450000m);
            mb.SetDownPayment(70000m);
        }

        builder.AddDocument("Driver License", "ID");
        builder.AddDocument("Tax Returns", "Income Verification");
        builder.AddDocument("Appraisal Report", "Property Valuation");
    }

    public void ConstructAutoLoan(ILoanApplicationBuilder builder)
    {
        builder.SetPersonalInfo("Carol", "Chen", "carol.chen@example.com");
        builder.SetLoanTerms("Auto", 32000m, 60);
        builder.SetInterestRate(7.49);

        if (builder is AutoLoanBuilder alb)
        {
            alb.SetVehicleInfo(2024, "Honda", "Accord", "1HGCV1F34PA012345", 35000m);
            alb.SetDealership("City Motors");
        }

        builder.AddDocument("Driver License", "ID");
        builder.AddDocument("Vehicle Registration", "Registration");
    }
}
```

### Main / Driver
```csharp
// ── Driver ──────────────────────────────────────────────────────────
var director = new LoanDirector();

// Personal loan
var personalBuilder = new PersonalLoanBuilder();
director.ConstructPersonalLoan(personalBuilder);
var personalLoan = personalBuilder.GetResult();

Console.WriteLine("=== Personal Loan Application ===");
Console.WriteLine(
    $"Applicant: {personalLoan.PersonalInfo?.FirstName} {personalLoan.PersonalInfo?.LastName}" +
    $" | Loan: ${personalLoan.Amount:F2} over {personalLoan.TermMonths}mo" +
    $" @ {personalLoan.InterestRate:F2}% APR"
);
Console.WriteLine(
    "Documents: " + string.Join(", ",
        personalLoan.Documents.Select(d => d.Name))
);

Console.WriteLine();

// Mortgage
var mortgageBuilder = new MortgageBuilder();
director.ConstructMortgage(mortgageBuilder);
var mortgage = mortgageBuilder.GetResult();

Console.WriteLine("=== Mortgage Application ===");
Console.WriteLine(
    $"Applicant: {mortgage.PersonalInfo?.FirstName} {mortgage.PersonalInfo?.LastName}" +
    $" | Loan: ${mortgage.Amount:F2} over {mortgage.TermMonths}mo" +
    $" @ {mortgage.InterestRate:F2}% APR"
);

if (mortgage.PropertyInfo is { } prop)
{
    Console.WriteLine(
        $"Property: {prop.Address}, {prop.City}" +
        $" (Appraised: ${prop.AppraisedValue:F2})"
    );
}

if (mortgage.DownPayment.HasValue)
{
    var ltv = (mortgage.Amount - mortgage.DownPayment.Value) / (mortgage.PropertyInfo?.AppraisedValue ?? 1m) * 100m;
    Console.WriteLine(
        $"Down Payment: ${mortgage.DownPayment.Value:F2}" +
        $" | LTV: {ltv:F2}%"
    );
}

Console.WriteLine(
    "Documents: " + string.Join(", ",
        mortgage.Documents.Select(d => d.Name))
);

Console.WriteLine();

// Auto loan
var autoBuilder = new AutoLoanBuilder();
director.ConstructAutoLoan(autoBuilder);
var autoLoan = autoBuilder.GetResult();

Console.WriteLine("=== Auto Loan Application ===");
Console.WriteLine(
    $"Applicant: {autoLoan.PersonalInfo?.FirstName} {autoLoan.PersonalInfo?.LastName}" +
    $" | Loan: ${autoLoan.Amount:F2} over {autoLoan.TermMonths}mo" +
    $" @ {autoLoan.InterestRate:F2}% APR"
);

if (autoLoan.VehicleInfo is { } vehicle)
{
    Console.WriteLine(
        $"Vehicle: {vehicle.Year} {vehicle.Make} {vehicle.Model}" +
        $" (VIN: {vehicle.Vin}, Value: ${vehicle.Value:F2})"
    );
}

if (autoLoan.Dealership is { } dealer)
{
    Console.WriteLine($"Dealership: {dealer}");
}

Console.WriteLine(
    "Documents: " + string.Join(", ",
        autoLoan.Documents.Select(d => d.Name))
);

Console.WriteLine();
```

### Language-Specific Notes

- **Records for immutable DTOs** — `LoanApplication`, `PersonalInfo`, `PropertyInfo`, `VehicleInfo`, and `LoanDocument` are all declared as `record` types, giving value-based equality, immutability, and concise syntax with primary constructor parameters.
- **I-prefix interface naming** — The .NET convention prefixes interfaces with `I` (`ILoanApplicationBuilder`), which is idiomatic C# style and widely recognized across the ecosystem.
- **LINQ for document iteration** — `documents.Select(d => d.Name)` and `string.Join` provide a declarative, functional-style approach to formatting the document list without manual loops.
- **Primary constructors** — C# 12 primary constructors on `record` types eliminate boilerplate by directly declaring the constructor parameters as positional properties.
- **`decimal` for financial precision** — All monetary values (`amount`, `downPayment`, `appraisedValue`, `vehicleValue`) use the `decimal` type, which avoids the floating-point rounding errors inherent to `double` or `float`.
- **Nullable reference types** — Properties like `PersonalInfo?`, `PropertyInfo?`, `VehicleInfo?`, and `Dealership?` are explicitly nullable, requiring the caller to check for `null` before access (enforced by the compiler with nullable reference types enabled).
