The Builder pattern separates the construction of a complex object from its representation so the same construction process can produce different representations. In this Python implementation, we build `LoanApplication` objects—a rich domain object with personal info, loan terms, interest rate, collateral details, and supporting documents. An abstract builder interface (`LoanApplicationBuilder`) declares the construction steps, three concrete builders (`PersonalLoanBuilder`, `MortgageBuilder`, `AutoLoanBuilder`) implement each step for their loan product, and a `LoanDirector` orchestrates the sequence in a consistent order. The product is a `@dataclass` with a `summary()` method that renders the completed application.

## How the Implementation Works

### Class Diagram (Pseudocode)

The diagram above shows the full Builder pattern hierarchy. The abstract builder (`LoanApplicationBuilder`) defines the interface with five step methods. Each concrete builder stores intermediate state in private instance variables and exposes loan-specific setters (e.g., `set_property_info()` on `MortgageBuilder`). The `LoanDirector` accepts any builder via its constructor and calls the steps in the correct sequence for each loan product. The client code—in this case the script's driver—chooses a builder, passes it to the director, and retrieves the finished `LoanApplication` via `get_result()`.

### OOP Concepts Applied

- **Abstraction**: `LoanApplicationBuilder` exposes only the step methods needed to build a loan; internal state is hidden behind `@abstractmethod` declarations.
- **Encapsulation**: Each concrete builder stores its own working state (`_personal_info`, `_amount`, `_rate`, `_documents`, etc.) as private attributes, exposing only targeted setters for loan-specific data.
- **Inheritance**: `PersonalLoanBuilder`, `MortgageBuilder`, and `AutoLoanBuilder` inherit from `LoanApplicationBuilder` and share the same interface contract.
- **Polymorphism**: `LoanDirector` operates on the `LoanApplicationBuilder` abstract type—any concrete builder can be passed in without the director knowing which loan product is being built.

### SOLID Principles Applied

- **Single Responsibility**: Each concrete builder is responsible for constructing exactly one loan product; the director manages only the construction sequence.
- **Open/Closed**: New loan types (e.g., `StudentLoanBuilder`) can be added by implementing the builder interface without modifying existing builders or the director.
- **Liskov Substitution**: Any concrete builder can substitute for `LoanApplicationBuilder` in the director's constructor without altering the director's behavior.
- **Interface Segregation**: The builder interface is narrow—five focused methods—and loan-specific setters live only on the concrete builders that need them.
- **Dependency Inversion**: The director depends on the abstract `LoanApplicationBuilder` interface, not on concrete builder classes.

```python
# Builder Pattern — Python — Banking example
# ============================================
# Participants: LoanApplicationBuilder (ABC), PersonalLoanBuilder,
# MortgageBuilder, AutoLoanBuilder (ConcreteBuilders),
# LoanDirector (Director), LoanApplication (Product)
# OOP: abstraction, encapsulation, inheritance, polymorphism
# SOLID: SRP, OCP, LSP, ISP, DIP
# ============================================

from abc import ABC, abstractmethod
from dataclasses import dataclass, field
from typing import Optional


# ── Product ──────────────────────────────────────────────────────────

@dataclass
class LoanApplication:
    """Final product assembled by the builder."""
    applicant_name: str = ""
    loan_type: str = ""
    amount: float = 0.0
    term_months: int = 0
    interest_rate: float = 0.0
    documents: list[str] = field(default_factory=list)
    property_info: Optional[dict] = None
    down_payment: float = 0.0
    vehicle_info: Optional[dict] = None
    dealership: Optional[str] = None

    def summary(self) -> str:
        """Render a formatted summary of the completed application."""
        lines: list[str] = [
            f"=== {self.loan_type} Application ===",
            (
                f"Applicant: {self.applicant_name} | "
                f"Loan: ${self.amount:,.2f} over {self.term_months}mo "
                f"@ {self.interest_rate:.2f}% APR"
            ),
        ]
        if self.property_info is not None:
            appraisal = self.property_info["appraised_value"]
            ltv = (self.amount - self.down_payment) / appraisal * 100
            lines.append(
                f"Property: {self.property_info['address']} "
                f"(Appraised: ${appraisal:,.2f})"
            )
            lines.append(
                f"Down Payment: ${self.down_payment:,.2f} | "
                f"LTV: {ltv:.2f}%"
            )
        if self.vehicle_info is not None:
            v = self.vehicle_info
            lines.append(
                f"Vehicle: {v['year']} {v['make']} {v['model']} "
                f"(VIN: {v['vin']}, Value: ${v['value']:,.2f})"
            )
            lines.append(f"Dealership: {self.dealership}")
        lines.append(f"Documents: {', '.join(self.documents)}")
        return "\n".join(lines)
```

### Builder Interface (ABC)
```python
# ── Builder interface (ABC) ──────────────────────────────────────────

class LoanApplicationBuilder(ABC):
    """Abstract builder that defines the steps for constructing a loan application."""

    @abstractmethod
    def set_personal_info(self, name: str) -> None:
        """Record the applicant's full name."""

    @abstractmethod
    def set_loan_terms(self, amount: float, term_months: int) -> None:
        """Set the loan amount and repayment term in months."""

    @abstractmethod
    def set_interest_rate(self, rate: float) -> None:
        """Set the annual percentage rate (e.g., 6.25 for 6.25%)."""

    @abstractmethod
    def add_document(self, document: str) -> None:
        """Append a document name to the application package."""

    @abstractmethod
    def get_result(self) -> LoanApplication:
        """Return the fully constructed LoanApplication and reset the builder."""
```

### PersonalLoanBuilder (ConcreteBuilder)
```python
# ── Concrete builders ────────────────────────────────────────────────

class PersonalLoanBuilder(LoanApplicationBuilder):
    """Constructs an unsecured personal loan application."""

    def __init__(self) -> None:
        self._app: LoanApplication = LoanApplication(loan_type="Personal Loan")

    def set_personal_info(self, name: str) -> None:
        self._app.applicant_name = name

    def set_loan_terms(self, amount: float, term_months: int) -> None:
        self._app.amount = amount
        self._app.term_months = term_months

    def set_interest_rate(self, rate: float) -> None:
        self._app.interest_rate = rate

    def add_document(self, document: str) -> None:
        self._app.documents.append(document)

    def get_result(self) -> LoanApplication:
        result = self._app
        self._app = LoanApplication(loan_type="Personal Loan")
        return result
```

```python
class MortgageBuilder(LoanApplicationBuilder):
    """Constructs a mortgage application with property and down payment details."""

    def __init__(self) -> None:
        self._app: LoanApplication = LoanApplication(loan_type="Mortgage")

    def set_personal_info(self, name: str) -> None:
        self._app.applicant_name = name

    def set_loan_terms(self, amount: float, term_months: int) -> None:
        self._app.amount = amount
        self._app.term_months = term_months

    def set_interest_rate(self, rate: float) -> None:
        self._app.interest_rate = rate

    def add_document(self, document: str) -> None:
        self._app.documents.append(document)

    def set_property_info(self, address: str, appraised_value: float) -> None:
        """Set property address and appraised value."""
        self._app.property_info = {
            "address": address,
            "appraised_value": appraised_value,
        }

    def set_down_payment(self, amount: float) -> None:
        """Set the down payment amount."""
        self._app.down_payment = amount

    def get_result(self) -> LoanApplication:
        result = self._app
        self._app = LoanApplication(loan_type="Mortgage")
        return result
```

### LoanDirector (Director)
```python
# ── Director ──────────────────────────────────────────────────────────

class LoanDirector:
    """Orchestrates the builder steps to construct standard loan products."""

    def __init__(self, builder: LoanApplicationBuilder) -> None:
        self._builder = builder

    def construct_personal_loan(
        self, name: str, amount: float, term_months: int, rate: float
    ) -> None:
        """Build a personal loan with standard documentation."""
        self._builder.set_personal_info(name)
        self._builder.set_loan_terms(amount, term_months)
        self._builder.set_interest_rate(rate)
        self._builder.add_document("ID")
        self._builder.add_document("Income Verification")

    def construct_mortgage(
        self,
        name: str,
        amount: float,
        term_months: int,
        rate: float,
        property_address: str,
        appraised_value: float,
        down_payment: float,
    ) -> None:
        """Build a mortgage with property details and required disclosures."""
        self._builder.set_personal_info(name)
        self._builder.set_loan_terms(amount, term_months)
        self._builder.set_interest_rate(rate)
        self._builder.add_document("ID")
        self._builder.add_document("Tax Returns")
        self._builder.add_document("Appraisal Report")
        if isinstance(self._builder, MortgageBuilder):
            self._builder.set_property_info(property_address, appraised_value)
            self._builder.set_down_payment(down_payment)

    def construct_auto_loan(
        self,
        name: str,
        amount: float,
        term_months: int,
        rate: float,
        year: int,
        make: str,
        model: str,
        vin: str,
        value: float,
        dealership: str,
    ) -> None:
        """Build an auto loan with vehicle and dealership information."""
        self._builder.set_personal_info(name)
        self._builder.set_loan_terms(amount, term_months)
        self._builder.set_interest_rate(rate)
        self._builder.add_document("ID")
        self._builder.add_document("Vehicle Registration")
        if isinstance(self._builder, AutoLoanBuilder):
            self._builder.set_vehicle_info(year, make, model, vin, value)
            self._builder.set_dealership(dealership)
```

### Main / Driver
```python
# ── Driver ────────────────────────────────────────────────────────────

def main() -> None:
    """Exercise all three loan builder implementations."""

    # ── Personal Loan: Alice Johnson ────────────────────────────────
    personal_builder = PersonalLoanBuilder()
    director = LoanDirector(personal_builder)
    director.construct_personal_loan(
        name="Alice Johnson",
        amount=15000.00,
        term_months=60,
        rate=11.99,
    )
    alice_loan = personal_builder.get_result()
    print(alice_loan.summary())
    print()

    # ── Mortgage: Bob Martinez ──────────────────────────────────────
    mortgage_builder = MortgageBuilder()
    director = LoanDirector(mortgage_builder)
    director.construct_mortgage(
        name="Bob Martinez",
        amount=350000.00,
        term_months=360,
        rate=6.25,
        property_address="123 Main St, Springfield",
        appraised_value=450000.00,
        down_payment=70000.00,
    )
    bob_loan = mortgage_builder.get_result()
    print(bob_loan.summary())
    print()

    # ── Auto Loan: Carol Chen ───────────────────────────────────────
    auto_builder = AutoLoanBuilder()
    director = LoanDirector(auto_builder)
    director.construct_auto_loan(
        name="Carol Chen",

        model="Accord",

    )
    carol_loan = auto_builder.get_result()
    print(carol_loan.summary())


if __name__ == "__main__":
    main()
```

### Language-Specific Notes

- **ABCs and `@abstractmethod`**: Python's `abc` module provides the formal interface mechanism. Any class with an `@abstractmethod` cannot be instantiated directly—the runtime enforces the contract at construction time.
- **`@dataclass`**: The `LoanApplication` product uses `dataclasses.dataclass` with `field(default_factory=list)` to avoid the mutable-default-argument gotcha. This reduces boilerplate compared to a hand-written `__init__`.
- **Type hints**: Python 3.11+ supports `list[str]` (built-in generic syntax) and `Optional[dict]` for nullable fields. Type hints are optional at runtime but enable static analysis with mypy or pyright.
- **Snake_case**: All methods and variables follow PEP 8 conventions. The builder step methods (`set_personal_info`, `set_loan_terms`) are deliberately verbose to make the construction sequence self-documenting.
- **Duck typing vs. ABC**: The director could accept any object with the right methods (duck typing), but explicit ABC inheritance makes the contract visible in code, improves IDE autocompletion, and provides runtime enforcement.
- **f-strings for formatting**: The `summary()` method uses f-strings with format specifiers (`:,.2f`) for locale-aware currency formatting. This is the idiomatic Python approach for string interpolation.
