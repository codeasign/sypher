## Implementation

Below is a complete Python 3.11+ implementation of the Interpreter pattern using a Stock Market domain. The pattern defines a composable expression grammar for stock filtering. The `Expression` ABC defines the `interpret()` contract. Terminal expressions (`PriceGreaterThan`, `VolumeGreaterThan`, `TickerEquals`, `SectorEquals`) implement atomic comparisons against a `StockData` dataclass. Nonterminal expressions (`AndExpr`, `OrExpr`, `NotExpr`) compose expressions logically. The `StockScreener` evaluates an expression tree against a portfolio, collecting matching stocks.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌──────────────────────────────────────┐
//   │   <<ABC / abstract>>                 │
//   │   Expression (AbstractExpression)    │ ← OOP: Abstraction
//   ├──────────────────────────────────────┤    ISP: single @abstractmethod
//   │   @abstractmethod                    │
//   │   interpret(stock) -> bool           │
//   └────────────────┬─────────────────────┘
//                    ▲ inherits
//   ┌────────────────┴─────────────────────┐
//   │                │                     │
//   ┌────────────────┐  ┌──────────────────┐
//   │ TerminalExprs  │  │ NonterminalExprs │ ← OOP: Polymorphism
//   │ (PriceCheck,   │  │ (AndExpr,        │    LSP: all are
//   │  VolumeCheck,  │  │  OrExpr,         │    substitutable for
//   │  TickerMatch,  │  │  NotExpr)        │    Expression
//   │  SectorMatch)  │  │                  │
//   └────────────────┘  └──────────────────┘
//
//   ┌──────────────────────────────────────┐
//   │ StockScreener (Client)               │ ← SRP: only screens
//   │ + screen(stocks, expr): list         │    DIP: depends on
//   └──────────────────────────────────────┘    Expression ABC
```

#### OOP Concepts Applied

1. **Abstraction** — `Expression` is an abstract base class. Clients depend on `Expression`, never on concrete types.
2. **Encapsulation** — Each expression hides its internal data (thresholds, child expressions) and exposes only `interpret()`.
3. **Inheritance** — All concrete expressions inherit from `Expression` and implement `interpret()`.
4. **Polymorphism** — `StockScreener.screen()` calls `interpret()` on any `Expression` without type checking.

#### SOLID Principles Applied

5. **SRP** — Each expression class evaluates exactly one grammar rule. `StockScreener` only iterates and collects.
6. **OCP** — New expression types subclass `Expression` without modifying existing code.
7. **LSP** — Every concrete expression is substitutable for `Expression`. No `isinstance` checks needed.
8. **ISP** — The `Expression` ABC has a single `@abstractmethod` — `interpret()`.
9. **DIP** — `StockScreener` depends on `Expression` (the ABC), not on concrete implementations.

```python
# Interpreter Pattern — Python — Stock Market example
# ====================================================
# Participants: Expression (AbstractExpression), PriceGreaterThan,
# VolumeGreaterThan, TickerEquals, SectorEquals (TerminalExpressions),
# AndExpr, OrExpr, NotExpr (NonterminalExpressions),
# StockData (Context), StockScreener (Client)
# OOP: abstraction, inheritance, polymorphism, encapsulation
# SOLID: SRP, OCP, LSP, ISP, DIP
# ====================================================

from abc import ABC, abstractmethod
from dataclasses import dataclass
```

### StockData (Context)

```python
# ── Context ──
@dataclass
class StockData:
    ticker: str
    price: float
    volume: int
    sector: str
```

### Expression (AbstractExpression)

```python
# ── Abstract Expression ──
class Expression(ABC):
    @abstractmethod
    def interpret(self, stock: StockData) -> bool:
        """Evaluate this expression against the given stock data."""
        pass
```

### PriceGreaterThan (TerminalExpression)

```python
class PriceGreaterThan(Expression):
    def __init__(self, threshold: float):
        self._threshold = threshold

    def interpret(self, stock: StockData) -> bool:
        return stock.price > self._threshold
```

### VolumeGreaterThan (TerminalExpression)

```python
class VolumeGreaterThan(Expression):
    def __init__(self, threshold: int):
        self._threshold = threshold

    def interpret(self, stock: StockData) -> bool:
        return stock.volume > self._threshold
```

### TickerEquals (TerminalExpression)

```python
class TickerEquals(Expression):
    def __init__(self, ticker: str):
        self._ticker = ticker.upper()

    def interpret(self, stock: StockData) -> bool:
        return stock.ticker == self._ticker
```

### SectorEquals (TerminalExpression)

```python
class SectorEquals(Expression):
    def __init__(self, sector: str):
        self._sector = sector

    def interpret(self, stock: StockData) -> bool:
        return stock.sector == self._sector
```

### AndExpr (NonterminalExpression)

```python
class AndExpr(Expression):
    def __init__(self, left: Expression, right: Expression):
        self._left = left
        self._right = right

    def interpret(self, stock: StockData) -> bool:
        return self._left.interpret(stock) and self._right.interpret(stock)
```

### OrExpr (NonterminalExpression)

```python
class OrExpr(Expression):
    def __init__(self, left: Expression, right: Expression):
        self._left = left
        self._right = right

    def interpret(self, stock: StockData) -> bool:
        return self._left.interpret(stock) or self._right.interpret(stock)
```

### NotExpr (NonterminalExpression)

```python
class NotExpr(Expression):
    def __init__(self, expr: Expression):
        self._expr = expr

    def interpret(self, stock: StockData) -> bool:
        return not self._expr.interpret(stock)
```

### StockScreener (Client)

```python
# ── Client ──
class StockScreener:
    @staticmethod
    def screen(stocks: list[StockData], expression: Expression) -> list[StockData]:
        return [stock for stock in stocks if expression.interpret(stock)]
```

### Main / Driver

```python
# ── Main / Driver ──
if __name__ == "__main__":
    stocks = [
        StockData("AAPL", 175.50, 85000000, "Technology"),
        StockData("GOOGL", 142.30, 32000000, "Technology"),
        StockData("MSFT", 378.90, 25000000, "Technology"),
        StockData("JPM", 198.40, 12000000, "Financial"),
        StockData("BAC", 39.20, 45000000, "Financial"),
        StockData("AMZN", 178.25, 61000000, "Technology"),
        StockData("TSLA", 245.60, 98000000, "Automotive"),
        StockData("WMT", 72.80, 18000000, "Retail"),
        StockData("V", 275.30, 15000000, "Financial"),
        StockData("NVDA", 820.10, 52000000, "Technology"),
    ]

    screener = StockScreener()

    print("=== Stock Screener — Interpreter Pattern ===\n")

    # Query 1: price > 150 AND volume > 30000000
    query1 = AndExpr(PriceGreaterThan(150.0), VolumeGreaterThan(30000000))
    result1 = screener.screen(stocks, query1)
    print("Query 1: price > 150 AND volume > 30000000")
    for s in result1:
        print(f"  {s.ticker} — ${s.price:.2f} — {s.volume / 1_000_000:.1f}M vol")
    print(f"  Matched: {len(result1)} stocks\n")

    # Query 2: sector = "Financial" OR ticker = "MSFT"
    query2 = OrExpr(SectorEquals("Financial"), TickerEquals("MSFT"))
    result2 = screener.screen(stocks, query2)
    print('Query 2: sector = "Financial" OR ticker = "MSFT"')
    for s in result2:
        print(f"  {s.ticker} — ${s.price:.2f} — {s.sector}")
    print(f"  Matched: {len(result2)} stocks\n")

    # Query 3: NOT (price > 100) — stocks with price <= 100
    query3 = NotExpr(PriceGreaterThan(100.0))
    result3 = screener.screen(stocks, query3)
    print("Query 3: NOT (price > 100) — stocks with price <= 100")
    for s in result3:
        print(f"  {s.ticker} — ${s.price:.2f}")
    print(f"  Matched: {len(result3)} stocks\n")

    # Query 4: (ticker = "AAPL" OR ticker = "GOOGL") AND volume > 40000000
    query4 = AndExpr(
        OrExpr(TickerEquals("AAPL"), TickerEquals("GOOGL")),
        VolumeGreaterThan(40000000),
    )
    result4 = screener.screen(stocks, query4)
    print('Query 4: (ticker = "AAPL" OR ticker = "GOOGL") AND volume > 40000000')
    for s in result4:
        print(f"  {s.ticker} — ${s.price:.2f} — {s.volume / 1_000_000:.1f}M vol")
    print(f"  Matched: {len(result4)} stocks\n")

    # Query 5: sector = "Technology" OR sector = "Financial"
    query5 = OrExpr(SectorEquals("Technology"), SectorEquals("Financial"))
    result5 = screener.screen(stocks, query5)
    print('Query 5: sector = "Technology" OR sector = "Financial"')
    for s in result5:
        print(f"  {s.ticker} — ${s.price:.2f} — {s.sector}")
    print(f"  Matched: {len(result5)} stocks")
```

### Language-Specific Notes

- **OOP concepts** — Python uses `ABC` from the `abc` module for abstract base classes and `@abstractmethod` to enforce the contract. The `@dataclass` decorator provides concise, immutable context objects with auto-generated `__init__` and `__repr__`.
- **SOLID in practice** — Python's duck typing makes LSP and DIP natural: any object with an `interpret()` method that accepts `StockData` and returns `bool` can substitute for `Expression`. No explicit interface inheritance is strictly required.
- **Idiomatic patterns** — The `_name` convention for private attributes (name mangling not needed). List comprehensions (`[x for x in y if cond]`) provide concise, readable filtering in the `screen()` method. F-string formatting for monetary values (`${s.price:.2f}`).
- **No `isinstance` checking** — The `screen()` method calls `interpret()` polymorphically. The pattern is implemented purely through inheritance and method dispatch.
