## Implementation

Below is a complete C# 12 / .NET 8 implementation of the Interpreter pattern using a Stock Market domain. The pattern defines a composable expression grammar for stock filtering. The `IExpression` interface declares the `Interpret()` method. Terminal expressions (`PriceGreaterThan`, `VolumeGreaterThan`, `TickerEquals`, `SectorEquals`) implement atomic comparisons against a `StockData` record. Nonterminal expressions (`AndExpr`, `OrExpr`, `NotExpr`) compose expressions logically. The `StockScreener` evaluates an expression tree against a portfolio, collecting matching stocks.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌──────────────────────────────────────┐
//   │   <<interface>>                      │
//   │   IExpression (AbstractExpression)   │ ← OOP: Abstraction
//   ├──────────────────────────────────────┤    ISP: single method
//   │   + Interpret(StockData): bool       │
//   └────────────────┬─────────────────────┘
//                    ▲ implements
//   ┌────────────────┴─────────────────────┐
//   │                │                     │
//   ┌────────────────┐  ┌──────────────────┐
//   │ TerminalExprs  │  │ NonterminalExprs │ ← OOP: Polymorphism
//   │ (PriceCheck,   │  │ (AndExpr,        │    LSP: substitutable
//   │  VolumeCheck,  │  │  OrExpr,         │    for IExpression
//   │  TickerMatch,  │  │  NotExpr)        │
//   │  SectorMatch)  │  │                  │
//   └────────────────┘  └──────────────────┘
//
//   ┌──────────────────────────────────────┐
//   │ StockScreener (Client)               │ ← SRP: only screens
//   │ + Screen(List, IExpression): List    │    DIP: depends on
//   └──────────────────────────────────────┘    IExpression interface
```

#### OOP Concepts Applied

1. **Abstraction** — `IExpression` is the interface contract. All client code depends on `IExpression`, never on concrete types.
2. **Encapsulation** — Each expression class stores its private fields and exposes only `Interpret()`.
3. **Inheritance** — Classes implement the `IExpression` interface, providing concrete `Interpret()` bodies.
4. **Polymorphism** — `Screen()` calls `Interpret()` on any `IExpression` reference. The runtime dispatches to the correct implementation.

#### SOLID Principles Applied

5. **SRP** — Each expression class evaluates exactly one grammar rule. `StockScreener` only iterates and collects results.
6. **OCP** — New expression types implement `IExpression` without modifying existing expressions or the screener.
7. **LSP** — Every concrete expression is substitutable for `IExpression`. No type checking needed.
8. **ISP** — The `IExpression` interface is a single method — minimal and focused.
9. **DIP** — `StockScreener` depends on `IExpression`, not on concrete implementations.

### StockData (Context)

```csharp
// ── Context ──
record StockData(string Ticker, double Price, int Volume, string Sector);
```

### IExpression (AbstractExpression)

```csharp
// ── Abstract Expression ──
interface IExpression
{
    bool Interpret(StockData stock);
}
```

### PriceGreaterThan (TerminalExpression)

```csharp
class PriceGreaterThan : IExpression
{
    private readonly double _threshold;

    public PriceGreaterThan(double threshold)
    {
        _threshold = threshold;
    }

    public bool Interpret(StockData stock)
    {
        return stock.Price > _threshold;
    }
}
```

### VolumeGreaterThan (TerminalExpression)

```csharp
class VolumeGreaterThan : IExpression
{
    private readonly int _threshold;

    public VolumeGreaterThan(int threshold)
    {
        _threshold = threshold;
    }

    public bool Interpret(StockData stock)
    {
        return stock.Volume > _threshold;
    }
}
```

### TickerEquals (TerminalExpression)

```csharp
class TickerEquals : IExpression
{
    private readonly string _ticker;

    public TickerEquals(string ticker)
    {
        _ticker = ticker.ToUpperInvariant();
    }

    public bool Interpret(StockData stock)
    {
        return stock.Ticker == _ticker;
    }
}
```

### SectorEquals (TerminalExpression)

```csharp
class SectorEquals : IExpression
{
    private readonly string _sector;

    public SectorEquals(string sector)
    {
        _sector = sector;
    }

    public bool Interpret(StockData stock)
    {
        return stock.Sector == _sector;
    }
}
```

### AndExpr (NonterminalExpression)

```csharp
class AndExpr : IExpression
{
    private readonly IExpression _left;
    private readonly IExpression _right;

    public AndExpr(IExpression left, IExpression right)
    {
        _left = left;
        _right = right;
    }

    public bool Interpret(StockData stock)
    {
        return _left.Interpret(stock) && _right.Interpret(stock);
    }
}
```

### OrExpr (NonterminalExpression)

```csharp
class OrExpr : IExpression
{
    private readonly IExpression _left;
    private readonly IExpression _right;

    public OrExpr(IExpression left, IExpression right)
    {
        _left = left;
        _right = right;
    }

    public bool Interpret(StockData stock)
    {
        return _left.Interpret(stock) || _right.Interpret(stock);
    }
}
```

### NotExpr (NonterminalExpression)

```csharp
class NotExpr : IExpression
{
    private readonly IExpression _expr;

    public NotExpr(IExpression expr)
    {
        _expr = expr;
    }

    public bool Interpret(StockData stock)
    {
        return !_expr.Interpret(stock);
    }
}
```

### StockScreener (Client)

```csharp
// ── Client ──
class StockScreener
{
    public List<StockData> Screen(List<StockData> stocks, IExpression expression)
    {
        return stocks.Where(s => expression.Interpret(s)).ToList();
    }
}
```

### Main / Driver

```csharp
// ── Main / Driver ──
class InterpreterDemo
{
    static void Main()
    {
        var stocks = new List<StockData>
        {
            new("AAPL", 175.50, 85000000, "Technology"),
            new("GOOGL", 142.30, 32000000, "Technology"),
            new("MSFT", 378.90, 25000000, "Technology"),
            new("JPM", 198.40, 12000000, "Financial"),
            new("BAC", 39.20, 45000000, "Financial"),
            new("AMZN", 178.25, 61000000, "Technology"),
            new("TSLA", 245.60, 98000000, "Automotive"),
            new("WMT", 72.80, 18000000, "Retail"),
            new("V", 275.30, 15000000, "Financial"),
            new("NVDA", 820.10, 52000000, "Technology"),
        };

        var screener = new StockScreener();

        Console.WriteLine("=== Stock Screener — Interpreter Pattern ===");
        Console.WriteLine();

        // Query 1: price > 150 AND volume > 30000000
        IExpression query1 = new AndExpr(
            new PriceGreaterThan(150.0),
            new VolumeGreaterThan(30000000)
        );
        var result1 = screener.Screen(stocks, query1);
        Console.WriteLine("Query 1: price > 150 AND volume > 30000000");
        foreach (var s in result1)
        {
            Console.WriteLine($"  {s.Ticker} — ${s.Price:F2} — {s.Volume / 1_000_000.0:F1}M vol");
        }
        Console.WriteLine($"  Matched: {result1.Count} stocks");
        Console.WriteLine();

        // Query 2: sector = "Financial" OR ticker = "MSFT"
        IExpression query2 = new OrExpr(
            new SectorEquals("Financial"),
            new TickerEquals("MSFT")
        );
        var result2 = screener.Screen(stocks, query2);
        Console.WriteLine("Query 2: sector = \"Financial\" OR ticker = \"MSFT\"");
        foreach (var s in result2)
        {
            Console.WriteLine($"  {s.Ticker} — ${s.Price:F2} — {s.Sector}");
        }
        Console.WriteLine($"  Matched: {result2.Count} stocks");
        Console.WriteLine();

        // Query 3: NOT (price > 100) — stocks with price <= 100
        IExpression query3 = new NotExpr(new PriceGreaterThan(100.0));
        var result3 = screener.Screen(stocks, query3);
        Console.WriteLine("Query 3: NOT (price > 100) — stocks with price <= 100");
        foreach (var s in result3)
        {
            Console.WriteLine($"  {s.Ticker} — ${s.Price:F2}");
        }
        Console.WriteLine($"  Matched: {result3.Count} stocks");
        Console.WriteLine();

        // Query 4: (ticker = "AAPL" OR ticker = "GOOGL") AND volume > 40000000
        IExpression query4 = new AndExpr(
            new OrExpr(
                new TickerEquals("AAPL"),
                new TickerEquals("GOOGL")
            ),
            new VolumeGreaterThan(40000000)
        );
        var result4 = screener.Screen(stocks, query4);
        Console.WriteLine("Query 4: (ticker = \"AAPL\" OR ticker = \"GOOGL\") AND volume > 40000000");
        foreach (var s in result4)
        {
            Console.WriteLine($"  {s.Ticker} — ${s.Price:F2} — {s.Volume / 1_000_000.0:F1}M vol");
        }
        Console.WriteLine($"  Matched: {result4.Count} stocks");
        Console.WriteLine();

        // Query 5: sector = "Technology" OR sector = "Financial"
        IExpression query5 = new OrExpr(
            new SectorEquals("Technology"),
            new SectorEquals("Financial")
        );
        var result5 = screener.Screen(stocks, query5);
        Console.WriteLine("Query 5: sector = \"Technology\" OR sector = \"Financial\"");
        foreach (var s in result5)
        {
            Console.WriteLine($"  {s.Ticker} — ${s.Price:F2} — {s.Sector}");
        }
        Console.WriteLine($"  Matched: {result5.Count} stocks");
    }
}
```

### Language-Specific Notes

- **OOP concepts** — C# uses `interface` prefixed with `I` (idiomatic naming convention). The `record` keyword (C# 9+) provides immutable data carriers with value-based equality, `ToString()`, and positional construction.
- **SOLID in practice** — C#'s explicit interface implementation enforces ISP at compile time. The `readonly` keyword on fields ensures immutability. LINQ's `Where()` and `ToList()` provide declarative collection filtering in the `Screen()` method.
- **Idiomatic patterns** — `var` for local variable type inference. String interpolation (`$"{s.Ticker}"`) instead of composite formatting. `ToUpperInvariant()` for culture-safe string normalization. `_underscorePrefix` for private field naming convention.
- **Expression-bodied members** — While not used here for clarity, C# supports expression-bodied members (`bool Interpret(StockData s) => ...`) as a syntactic alternative for simple single-expression methods.
