## Implementation

Below is a complete C# implementation of the Command pattern using a stock market trading domain. A `Broker` accepts buy and sell orders for stocks, executes them, supports cancellation (undo), and maintains a complete order history. C#'s interface system provides the command contract, and the `List<Order>` collection provides type-safe order history management.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌──────────────────────────────────────────┐
//   │   interface IOrder                       │
//   │   ────────────────────────────────────── │
//   │   void Execute()                         │ ← OOP: Abstraction
//   │   void Undo()                            │    ISP: minimal contract
//   └──────────────────┬───────────────────────┘
//                      ▲ implements
//   ┌──────────────────┴───────────────────────┐
//   │                                          │
//   ┌──────────────────┐  ┌──────────────────┐  ┌──────────────────┐
//   │   BuyOrder       │  │   SellOrder      │  │  CancelOrder     │
//   ├──────────────────┤  ├──────────────────┤  ├──────────────────┤
//   │ + Execute()      │  │ + Execute()      │  │ + Execute()      │ ← OOP: Polymorphism
//   │   calls Buy()    │  │   calls Sell()   │  │   calls Undo()   │    LSP: substitutable
//   │ + Undo()         │  │ + Undo()         │  │ + Undo()         │    OCP: new order types
//   │   calls Sell()   │  │   calls Buy()    │  │   calls Exec()   │
//   └────────┬─────────┘  └────────┬─────────┘  └────────┬─────────┘
//            │                     │                     │
//            ▼                     ▼                     ▼
//   ┌────────────────────────────────────────────────────────┐
//   │   class Stock (Receiver)                               │
//   ├────────────────────────────────────────────────────────┤
//   │   void Buy(int quantity, decimal price)                │ ← SRP: one responsibility
//   │   void Sell(int quantity, decimal price)               │
//   │   string GetInfo()                                     │
//   └────────────────────────────────────────────────────────┘
//
//   ┌────────────────────────────────────────────────────────┐
//   │   class Broker (Invoker)                               │
//   ├────────────────────────────────────────────────────────┤
//   │   List<IOrder> orderHistory                            │ ← SRP: manages order lifecycle
//   │   void PlaceOrder(IOrder order)                        │    DIP: depends on IOrder
//   │   void CancelLastOrder()                               │     interface, not concrete
//   │   void ShowHistory()                                   │     command types
//   └────────────────────────────────────────────────────────┘
```

#### OOP Concepts Applied

1. **Abstraction** — The `IOrder` interface defines the contract. The `Broker` depends only on `IOrder`, never on concrete command classes.
2. **Encapsulation** — Each command hides its internal state. The `Stock` class encapsulates position data behind `Buy()`, `Sell()`, and `GetInfo()` methods.
3. **Inheritance** — `BuyOrder`, `SellOrder`, and `CancelOrder` all implement the `IOrder` interface.
4. **Polymorphism** — The `Broker` calls `order.Execute()` on any `IOrder` reference. The correct implementation dispatches per type at runtime.

#### SOLID Principles Applied

5. **SRP** — `Broker` only manages the order lifecycle. `Stock` only tracks position. Each command class encapsulates one trade type.
6. **OCP** — Adding a new `LimitOrder` command requires zero changes to `Broker` or `Stock`. Just implement the `IOrder` interface.
7. **LSP** — Every concrete command type is substitutable for `IOrder` without breaking the `Broker`.
8. **ISP** — The `IOrder` interface has only two methods. No class implements methods it does not need.
9. **DIP** — `Broker.PlaceOrder` accepts `IOrder` — an abstraction, not a concrete type.

### IOrder (Command Interface)

```csharp
public interface IOrder
{
    void Execute();
    void Undo();
}
```

### Stock (Receiver)

```csharp
public class Stock
{
    public string Ticker { get; }
    public decimal Price { get; private set; }
    public int SharesOutstanding { get; private set; }

    public Stock(string ticker, decimal price, int sharesOutstanding)
    {
        Ticker = ticker;
        Price = price;
        SharesOutstanding = sharesOutstanding;
    }

    public void Buy(int quantity, decimal price)
    {
        SharesOutstanding += quantity;
        Price = price;
        Console.WriteLine(
            $"Bought {quantity} shares of {Ticker} at ${price:F2}");
    }

    public void Sell(int quantity, decimal price)
    {
        if (SharesOutstanding >= quantity)
        {
            SharesOutstanding -= quantity;
            Price = price;
            Console.WriteLine(
                $"Sold {quantity} shares of {Ticker} at ${price:F2}");
        }
        else
        {
            Console.WriteLine(
                $"Insufficient shares of {Ticker} to sell {quantity}");
        }
    }

    public string GetInfo()
    {
        return
            $"{Ticker}: ${Price:F2}/share, " +
            $"{SharesOutstanding} shares outstanding";
    }
}
```

### BuyOrder (ConcreteCommand)

```csharp
public class BuyOrder : IOrder
{
    private readonly Stock _stock;
    private readonly int _quantity;
    private readonly decimal _price;

    public BuyOrder(Stock stock, int quantity, decimal price)
    {
        _stock = stock;
        _quantity = quantity;
        _price = price;
    }

    public void Execute() => _stock.Buy(_quantity, _price);
    public void Undo() => _stock.Sell(_quantity, _price);

    public Stock Stock => _stock;
    public int Quantity => _quantity;
    public decimal Price => _price;
}
```

### SellOrder (ConcreteCommand)

### CancelOrder (ConcreteCommand)

```csharp
public class CancelOrder : IOrder
{
    private readonly IOrder _targetOrder;
    private bool _wasExecuted;

    public CancelOrder(IOrder targetOrder)
    {
        _targetOrder = targetOrder;
        _wasExecuted = false;
    }

    public void Execute()
    {
        _wasExecuted = true;
        _targetOrder.Undo();
        Console.WriteLine("CancelOrder executed - original order undone");
    }

    public void Undo()
    {
        if (_wasExecuted)
        {
            _targetOrder.Execute();
            Console.WriteLine("CancelOrder undone - original order re-applied");
        }
    }

    public IOrder TargetOrder => _targetOrder;
}
```

### Broker (Invoker)

```csharp
public class Broker
{
    private readonly List<IOrder> _orderHistory = new();

    public void PlaceOrder(IOrder order)
    {
        order.Execute();
        _orderHistory.Add(order);
        Console.WriteLine("Order recorded in history\n");
    }

    public void CancelLastOrder()
    {
        if (_orderHistory.Count == 0)
        {
            Console.WriteLine("No orders to cancel\n");
            return;
        }
        IOrder lastOrder = _orderHistory[^1];
        _orderHistory.RemoveAt(_orderHistory.Count - 1);
        var cancelCmd = new CancelOrder(lastOrder);
        cancelCmd.Execute();
        _orderHistory.Add(cancelCmd);
        Console.WriteLine("Cancellation recorded in history\n");
    }

    public void ShowHistory()
    {
        Console.WriteLine("=== Order History ===");
        if (_orderHistory.Count == 0)
        {
            Console.WriteLine("No orders in history");
            return;
        }
        for (int i = 0; i < _orderHistory.Count; i++)
        {
            IOrder order = _orderHistory[i];
            string typeName = order.GetType().Name;
            string details;
            if (order is CancelOrder cmd)
            {
                details = $"CancelOrder targeting " +
                    $"{cmd.TargetOrder.GetType().Name}";
            }
            else if (order is BuyOrder buy)
            {
                details = $"{buy.Stock.Ticker} x{buy.Quantity} " +
                    $"@ ${buy.Price:F2}";
            }
            else if (order is SellOrder sell)
            {
                details = $"{sell.Stock.Ticker} x{sell.Quantity} " +
                    $"@ ${sell.Price:F2}";
            }
            else
            {
                details = "Unknown order type";
            }
            Console.WriteLine($"  {i + 1}. {typeName}: {details}");
        }
        Console.WriteLine();
    }
}
```

### Main / Driver

```csharp
class CommandDemo
{
    static void Main()
    {
        Console.WriteLine("=== Trading Session Started ===\n");

        var aapl = new Stock("AAPL", 175.50m, 1000);
        var googl = new Stock("GOOGL", 141.25m, 800);
        var msft = new Stock("MSFT", 378.90m, 1200);

        var broker = new Broker();

        var buy1 = new BuyOrder(aapl, 100, 175.50m);
        broker.PlaceOrder(buy1);

        var sell1 = new SellOrder(googl, 50, 141.25m);
        broker.PlaceOrder(sell1);

        var buy2 = new BuyOrder(msft, 200, 378.90m);
        broker.PlaceOrder(buy2);

        broker.ShowHistory();

        Console.WriteLine("=== Cancelling Last Order ===\n");
        broker.CancelLastOrder();

        broker.ShowHistory();

        Console.WriteLine("=== Current Stock Positions ===");
        Console.WriteLine(aapl.GetInfo());
        Console.WriteLine(googl.GetInfo());
        Console.WriteLine(msft.GetInfo());
    }
}
```

### Language-Specific Notes

- **Interface naming convention** — C# interfaces are conventionally prefixed with `I` (`IOrder`). This is a widely used naming convention that makes interfaces immediately identifiable.
- **`decimal` for financial values** — C#'s `decimal` type (128-bit) provides exact decimal arithmetic, unlike `float` or `double`. This is critical for financial applications where rounding errors are unacceptable. The `m` suffix (`175.50m`) creates a decimal literal.
- **Expression-bodied members** — `public void Execute() => _stock.Buy(_quantity, _price);` uses C# 6+ expression-bodied method syntax for one-line methods. This makes the command delegation concise.
- **Index from end operator** — `_orderHistory[^1]` uses C# 8.0's index-from-end operator (`^1` means "last element"). This is cleaner than `_orderHistory[_orderHistory.Count - 1]`.
- **Pattern matching `is`** — C# 7.0+ pattern matching with `order is CancelOrder cmd` combines type checking and variable assignment in one expression, eliminating explicit casts.
- **Primary constructor** — C# 12+ primary constructors could further simplify: `public class BuyOrder(Stock stock, int quantity, decimal price) : IOrder`. The compiler auto-generates the backing fields and constructor parameters.
