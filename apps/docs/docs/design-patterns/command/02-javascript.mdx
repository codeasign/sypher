## Implementation

Below is a complete JavaScript implementation of the Command pattern using a stock market trading domain. A `Broker` accepts buy and sell orders for stocks, executes them, supports cancellation (undo), and maintains a complete order history. Each order is a command object that encapsulates the trade details and delegates execution to the `Stock` receiver.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌──────────────────────────────────────────┐
//   │   class Order (base)                     │
//   │   ────────────────────────────────────── │
//   │   execute() / undo()                     │ ← OOP: Abstraction
//   └──────────────────┬───────────────────────┘    ISP: minimal interface
//                      ▲ extends
//   ┌──────────────────┴───────────────────────┐
//   │                                          │
//   ┌──────────────────┐  ┌──────────────────┐  ┌──────────────────┐
//   │   BuyOrder       │  │   SellOrder      │  │  CancelOrder     │
//   ├──────────────────┤  ├──────────────────┤  ├──────────────────┤
//   │ + execute()      │  │ + execute()      │  │ + execute()      │ ← OOP: Polymorphism
//   │   calls buy()    │  │   calls sell()   │  │   calls undo()   │    LSP: substitutable
//   │ + undo()         │  │ + undo()         │  │ + undo()         │    OCP: new order types
//   │   calls sell()   │  │   calls buy()    │  │   calls exec()   │
//   └────────┬─────────┘  └────────┬─────────┘  └────────┬─────────┘
//            │                     │                     │
//            ▼                     ▼                     ▼
//   ┌────────────────────────────────────────────────────────┐
//   │   class Stock (Receiver)                               │
//   ├────────────────────────────────────────────────────────┤
//   │   buy(quantity, price)                                 │ ← SRP: one responsibility
//   │   sell(quantity, price)                                │    (managing stock position)
//   │   getInfo()                                            │
//   └────────────────────────────────────────────────────────┘
//
//   ┌────────────────────────────────────────────────────────┐
//   │   class Broker (Invoker)                               │
//   ├────────────────────────────────────────────────────────┤
//   │   orderHistory: Order[]                                │ ← SRP: manages order lifecycle
//   │   placeOrder(order)                                    │    DIP: depends on Order, not
//   │   cancelLastOrder()                                    │     concrete types
//   │   showHistory()                                        │
//   └────────────────────────────────────────────────────────┘
```

#### OOP Concepts Applied

1. **Abstraction** — The `Order` base class defines the `execute()` and `undo()` contract. Concrete commands inherit from it and the `Broker` depends only on the abstract interface.
2. **Encapsulation** — Each command hides its internal state (stock reference, quantity, price). The `Stock` class encapsulates its position data behind `buy()`, `sell()`, and `getInfo()` methods.
3. **Inheritance** — `BuyOrder`, `SellOrder`, and `CancelOrder` all extend `Order` and share the common interface while providing their own implementations.
4. **Polymorphism** — The `Broker` calls `order.execute()` on any `Order` reference. The correct implementation runs per type without instanceof checks.

#### SOLID Principles Applied

5. **SRP** — `Broker` only manages order lifecycle. `Stock` only tracks position. Each command class only encapsulates one type of trade.
6. **OCP** — Adding a new `LimitOrder` command requires zero changes to `Broker` or `Stock`. The system is open for extension, closed for modification.
7. **LSP** — Every concrete command type can substitute for `Order` without breaking the `Broker`. The invoker calls `execute()` and `undo()` uniformly.
8. **ISP** — The `Order` interface is minimal: only `execute()` and `undo()`. No class is forced to implement methods it does not need.
9. **DIP** — The `Broker` depends on the abstract `Order` interface, not on concrete command types. It can accept any command without knowing its implementation details.

### Stock (Receiver)

```javascript
class Stock {
  constructor(ticker, price, sharesOutstanding) {
    this.ticker = ticker;
    this.price = price;
    this.sharesOutstanding = sharesOutstanding;
  }

  buy(quantity, price) {
    this.sharesOutstanding += quantity;
    this.price = price;
    console.log(
      `Bought ${quantity} shares of ${this.ticker} at $${price.toFixed(2)}`
    );
  }

  sell(quantity, price) {
    if (this.sharesOutstanding >= quantity) {
      this.sharesOutstanding -= quantity;
      this.price = price;
      console.log(
        `Sold ${quantity} shares of ${this.ticker} at $${price.toFixed(2)}`
      );
    } else {
      console.log(
        `Insufficient shares of ${this.ticker} to sell ${quantity}`
      );
    }
  }

  getInfo() {
    return (
      `${this.ticker}: $${this.price.toFixed(2)}/share, ` +
      `${this.sharesOutstanding} shares outstanding`
    );
  }
}
```

### BuyOrder (ConcreteCommand)

```javascript
class BuyOrder {
  constructor(stock, quantity, price) {
    this.stock = stock;
    this.quantity = quantity;
    this.price = price;
  }

  execute() {
    this.stock.buy(this.quantity, this.price);
  }

  undo() {
    this.stock.sell(this.quantity, this.price);
  }
}
```

### SellOrder (ConcreteCommand)

```javascript
class SellOrder {
  constructor(stock, quantity, price) {
    this.stock = stock;
    this.quantity = quantity;
    this.price = price;
  }

  execute() {
    this.stock.sell(this.quantity, this.price);
  }

  undo() {
    this.stock.buy(this.quantity, this.price);
  }
}
```

### CancelOrder (ConcreteCommand)

```javascript
class CancelOrder {
  constructor(targetOrder) {
    this.targetOrder = targetOrder;
    this.wasExecuted = false;
  }

  execute() {
    this.wasExecuted = true;
    this.targetOrder.undo();
    console.log('CancelOrder executed — original order undone');
  }

  undo() {
    if (this.wasExecuted) {
      this.targetOrder.execute();
      console.log('CancelOrder undone — original order re-applied');
    }
  }
}
```

### Broker (Invoker)

```javascript
class Broker {
  constructor() {
    this.orderHistory = [];
  }

  placeOrder(order) {
    order.execute();
    this.orderHistory.push(order);
    console.log('Order recorded in history\n');
  }

  cancelLastOrder() {
    if (this.orderHistory.length === 0) {
      console.log('No orders to cancel\n');
      return;
    }
    const lastOrder = this.orderHistory.pop();
    const cancelCmd = new CancelOrder(lastOrder);
    cancelCmd.execute();
    this.orderHistory.push(cancelCmd);
    console.log('Cancellation recorded in history\n');
  }

  showHistory() {
    console.log('=== Order History ===');
    if (this.orderHistory.length === 0) {
      console.log('No orders in history');
      return;
    }
    this.orderHistory.forEach((order, index) => {
      const type = order.constructor.name;
      let details;
      if (order instanceof CancelOrder) {
        details = `CancelOrder targeting ${order.targetOrder.constructor.name}`;
      } else {
        details = `${order.stock.ticker} x${order.quantity} @ $${order.price.toFixed(2)}`;
      }
      console.log(`  ${index + 1}. ${type}: ${details}`);
    });
    console.log('');
  }
}
```

### Main / Driver

```javascript
console.log('=== Trading Session Started ===\n');

const aapl = new Stock('AAPL', 175.50, 1000);
const googl = new Stock('GOOGL', 141.25, 800);
const msft = new Stock('MSFT', 378.90, 1200);

const broker = new Broker();

const buy1 = new BuyOrder(aapl, 100, 175.50);
broker.placeOrder(buy1);

const sell1 = new SellOrder(googl, 50, 141.25);
broker.placeOrder(sell1);

const buy2 = new BuyOrder(msft, 200, 378.90);
broker.placeOrder(buy2);

broker.showHistory();

console.log('=== Cancelling Last Order ===\n');
broker.cancelLastOrder();

broker.showHistory();

console.log('=== Current Stock Positions ===');
console.log(aapl.getInfo());
console.log(googl.getInfo());
console.log(msft.getInfo());
```

### Language-Specific Notes

- **No native interfaces** — JavaScript uses duck typing. Any object with `execute()` and `undo()` methods is a valid command. The `Order` base class is optional and used here for documentation purposes only.
- **`instanceof` for type checking** — The `Broker.showHistory()` method uses `instanceof CancelOrder` to distinguish cancel commands from regular orders for display purposes. This is a runtime check that TypeScript avoids with discriminated unions.
- **ES2022+ class fields** — Constructor initialization is the standard pattern. Private fields (`#field`) could be used for true encapsulation but are omitted here for readability.
- **Dynamic dispatch** — JavaScript's prototypal inheritance ensures that `order.execute()` always calls the correct method regardless of the concrete type, which is the core polymorphic behavior the Command pattern requires.
