Java implements the Composite pattern with a formal `CarPart` interface that declares all methods. Leaf classes (`Wheel`, `BrakePad`, `Piston`, `SparkPlug`) implement the interface directly, returning their own weight, price, and part count. The `CompositePart` abstract class implements `CarPart` and holds a `List<CarPart>` for children, delegating every data method to them. Concrete composites (`Engine`, `Suspension`, `Car`) extend `CompositePart` and add domain-specific fields. Java's checked exceptions and strong typing provide compile-time guarantees that only proper `CarPart` instances are added to the tree.

## How the Implementation Works

Java's `interface` keyword defines the `CarPart` contract. The `CompositePart` abstract class implements the child-management methods (`addChild`, `removeChild`, `getChild`) and provides the aggregation logic for `getWeight()`, `getPrice()`, and `getPartCount()` using a stream-based `sum()` over the children list. Concrete composites extend `CompositePart` and add their own fields. The driver constructs a tree of parts and calls `getWeight()`, `getPrice()`, and `getPartCount()` on the root `Car` object, which delegates recursively.

### OOP Principles Applied

| Principle | How It Is Applied |
|---|---|
| **Encapsulation** | Private fields (`weight`, `price`, `children`) are exposed only through public getter methods |
| **Inheritance** | `CompositePart` extends `Object` and implements `CarPart`; `Engine` extends `CompositePart` |
| **Polymorphism** | The `printPartsTree(CarPart part)` method works on any `CarPart` — JVM dynamic dispatch routes to the correct `getWeight()` implementation |
| **Open/Closed** | New leaf classes or composite classes can be added without modifying any existing code |
| **Single Responsibility** | Leaf classes own a single part's data; `CompositePart` owns aggregation; driver owns the construction |
| **Liskov Substitution** | Every `CompositePart` is a `CarPart`; every `Wheel` is a `CarPart`. Client code typed to `CarPart` accepts any implementation |

// Composite Pattern — Car Parts Hierarchy (Java)
import java.util.ArrayList;
import java.util.List;

interface CarPart {
    String getName();
    double getWeight();
    double getPrice();
    int getPartCount();

    default void addChild(CarPart part) {
        throw new UnsupportedOperationException(
            "addChild() not supported on " + getClass().getSimpleName()
        );
    }

    default void removeChild(CarPart part) {
        throw new UnsupportedOperationException(
            "removeChild() not supported on " + getClass().getSimpleName()
        );
    }

    default CarPart getChild(int index) {
        throw new UnsupportedOperationException(
            "getChild() not supported on " + getClass().getSimpleName()
        );
    }
}

### Composite

```java
abstract class CompositePart implements CarPart {
    private final String name;
    protected final List<CarPart> children;

    public CompositePart(String name) {
        this.name = name;
        this.children = new ArrayList<>();
    }

    @Override
    public String getName() { return name; }

    @Override
    public double getWeight() {
        return children.stream()
            .mapToDouble(CarPart::getWeight)
            .sum();
    }

    @Override
    public double getPrice() {
        return children.stream()
            .mapToDouble(CarPart::getPrice)
            .sum();
    }

    @Override
    public int getPartCount() {
        return children.stream()
            .mapToInt(CarPart::getPartCount)
            .sum();
    }

    @Override
    public void addChild(CarPart part) {
        if (part == null) {
            throw new IllegalArgumentException("Child cannot be null");
        }
        children.add(part);
    }

    @Override
    public void removeChild(CarPart part) {
        children.remove(part);
    }

    @Override
    public CarPart getChild(int index) {
        if (index < 0 || index >= children.size()) {
            throw new IndexOutOfBoundsException(
                "Child index " + index + " out of bounds"
            );
        }
        return children.get(index);
    }

    public List<CarPart> getChildren() {
        return new ArrayList<>(children);
    }
}
```

### Concrete Composites

### Print Helper

```java
class PartPrinter {
    public static void printPartsTree(CarPart part, String indent) {
        String weight = String.format("%.2f", part.getWeight());
        String price = String.format("%.2f", part.getPrice());
        int count = part.getPartCount();
        String kind = (part instanceof CompositePart) ? "Assembly" : "Part";
        System.out.printf(
            "%s%s: %s | Weight: %s kg | Price: $%s | Parts: %d%n",
            indent, kind, part.getName(), weight, price, count
        );
        if (part instanceof CompositePart) {
            for (CarPart child : ((CompositePart) part).getChildren()) {
                printPartsTree(child, indent + "  ");
            }
        }
    }
}
```

        Engine engine = new Engine(2.0, "gasoline");

        engine.addChild(new Piston("Piston-2", 0.45, 89.99, 83.0));

        engine.addChild(new Piston("Piston-4", 0.45, 89.99, 83.0));

        engine.addChild(new SparkPlug("SparkPlug-2", 0.05, 12.50, 7));
        engine.addChild(new SparkPlug("SparkPlug-3", 0.05, 12.50, 7));
        engine.addChild(new SparkPlug("SparkPlug-4", 0.05, 12.50, 7));

        // Build suspension: MacPherson strut
        Suspension suspension = new Suspension("MacPherson strut");
        suspension.addChild(new BrakePad("Front-Left-Pad", 0.65, 45.00, "ceramic"));
        suspension.addChild(new BrakePad("Front-Right-Pad", 0.65, 45.00, "ceramic"));
        suspension.addChild(new BrakePad("Rear-Left-Pad", 0.60, 42.00, "ceramic"));
        suspension.addChild(new BrakePad("Rear-Right-Pad", 0.60, 42.00, "ceramic"));

        // Build the car
        Car car = new Car("Honda", "Civic", 2025);
        car.addChild(engine);

        // Wheels
        car.addChild(new Wheel("Wheel-Front-Left", 15.0, 299.99, 18));
        car.addChild(new Wheel("Wheel-Front-Right", 15.0, 299.99, 18));
        car.addChild(new Wheel("Wheel-Rear-Left", 15.0, 299.99, 18));
        car.addChild(new Wheel("Wheel-Rear-Right", 15.0, 299.99, 18));

        car.addChild(suspension);

        // Print results
        System.out.println("=== Car Parts Hierarchy ===");
        PartPrinter.printPartsTree(car, "");

        System.out.println("\n=== Summary ===");
        System.out.printf("Vehicle: %s%n", car.getName());
        System.out.printf("Engine: %.1fL (%s)%n", engine.getDisplacementL(), engine.getFuelType());
        System.out.printf("Suspension: %s%n", suspension.getType());
        System.out.printf("Total weight: %.2f kg%n", car.getWeight());
        System.out.printf("Total price: $%.2f%n", car.getPrice());
        System.out.printf("Total part count: %d%n", car.getPartCount());
    }
}

## Language-Specific Notes

- Java's `interface` uses `default` methods for `addChild`, `removeChild`, and `getChild` on the `CarPart` interface, providing a default implementation that throws `UnsupportedOperationException` for leaf classes that do not override them.
- The `CompositePart` abstract class uses `List<CarPart>` and the Stream API (`stream().mapToDouble().sum()`) for idiomatic Java aggregation; the `getChildren()` method returns a defensive copy via `new ArrayList<>(children)`.
- The `final` keyword on fields (`private final String name`) ensures immutability of part data after construction, preventing accidental modifications.
- Java's `@Override` annotation provides compile-time verification that leaf methods correctly override the interface methods, catching typos in method signatures.
- The `instanceof` check in `printPartsTree` is the one acceptable place for type-checking the composite vs. leaf distinction, used only for display formatting rather than business logic.
- Compile and run: `javac CompositeDemo.java && java CompositeDemo` — no external dependencies required.
