The Composite pattern in JavaScript uses prototypal inheritance or ES6 classes to build a tree of car parts. A base `CarPart` class defines the common interface. Leaf classes (`Wheel`, `BrakePad`, `Piston`) implement the interface directly, returning their own weight, price, and part count. The `CompositePart` class (used by `Engine`, `Suspension`, and `Car`) aggregates child parts and delegates every operation to them recursively. JavaScript's dynamic typing makes the uniform interface natural — no explicit interface keyword is needed, only matching method names.

## How the Implementation Works

The JavaScript implementation uses a base `CarPart` class with default method stubs, then extends it into leaf subclasses and a `CompositePart` subclass. Leaves override data methods (`getWeight`, `getPrice`, `getPartCount`) and throw from `addChild`/`removeChild`/`getChild`. `CompositePart` maintains a `children` array and overrides every method to delegate to each child, aggregating results with `reduce`.

### OOP Principles Applied

| Principle | How It Is Applied |
|---|---|
| **Encapsulation** | Each part hides its internal weight, price, and children list behind accessor methods |
| **Inheritance** | `Wheel`, `BrakePad`, `Piston`, `CompositePart` all extend `CarPart` and inherit the interface shape |
| **Polymorphism** | The client calls `getWeight()` on any `CarPart` without knowing whether it is a leaf or composite; the correct behavior executes at runtime |
| **Open/Closed** | Adding a new leaf type (e.g., `Turbocharger`) or composite (e.g., `ExhaustSystem`) does not change any client or existing composite code |
| **Single Responsibility** | Leaf classes own a single part's data; `CompositePart` manages aggregation; `Car` builds the tree — each class has one reason to change |
| **Liskov Substitution** | Any `CarPart` can be passed to a function expecting a `CarPart` — leaf and composite are interchangeable through the interface |

### Component

```javascript
// Composite Pattern — Car Parts Hierarchy (JavaScript)
class CarPart {
  constructor(name) {
    if (new.target === CarPart) {
      throw new TypeError('CarPart is abstract — extend it');
    }
    this.name = name;
  }

  getName() {
    return this.name;
  }

  getWeight() {
    throw new Error('getWeight() must be overridden');
  }

  getPrice() {
    throw new Error('getPrice() must be overridden');
  }

  getPartCount() {
    throw new Error('getPartCount() must be overridden');
  }

  addChild(part) {
    throw new Error('addChild() is not supported on this part');
  }

  removeChild(part) {
    throw new Error('removeChild() is not supported on this part');
  }

  getChild(index) {
    throw new Error('getChild() is not supported on this part');
  }
}
```

### Composite

```javascript
class CompositePart extends CarPart {
  constructor(name) {
    super(name);
    this.children = [];
  }

  getWeight() {
    return this.children.reduce((sum, child) => sum + child.getWeight(), 0);
  }

  getPrice() {
    return this.children.reduce((sum, child) => sum + child.getPrice(), 0);
  }

  getPartCount() {
    return this.children.reduce(
      (count, child) => count + child.getPartCount(),
      0
    );
  }

  addChild(part) {
    if (!(part instanceof CarPart)) {
      throw new Error('Child must be a CarPart');
    }
    this.children.push(part);
  }

  removeChild(part) {
    const index = this.children.indexOf(part);
    if (index !== -1) {
      this.children.splice(index, 1);
    }
  }

  getChild(index) {
    if (index < 0 || index >= this.children.length) {
      throw new Error(`Child index ${index} out of bounds`);
    }
    return this.children[index];
  }

  getChildren() {
    return [...this.children];
  }
}
```

### Print Helper

```javascript
function printPartsTree(part, indent = '') {
  const weight = part.getWeight().toFixed(2);
  const price = part.getPrice().toFixed(2);
  const count = part.getPartCount();
  const type = part instanceof CompositePart ? 'Assembly' : 'Part';
  console.log(
    `${indent}${type}: ${part.getName()} | Weight: ${weight} kg | Price: $${price} | Parts: ${count}`
  );
  if (part instanceof CompositePart) {
    for (const child of part.getChildren()) {
      printPartsTree(child, indent + '  ');
    }
  }
}
```

## Language-Specific Notes

- JavaScript has no formal interface keyword or abstract class enforcement at runtime. The base `CarPart` class checks `new.target` in the constructor to simulate abstract instantiation prevention.
- The `reduce` method on arrays is the idiomatic JavaScript way to aggregate results from the children collection, replacing recursive loops in other languages.
- The `children` array access is protected through a `getChildren()` method that returns a shallow copy, preventing external mutation of the internal list.
- The pattern works equally well with factory functions and prototypal inheritance (pre-ES6 style), though ES6 class syntax is more readable and widely adopted.
- `removeChild` uses referential equality via `indexOf`; for production systems where the same part instance appears once, this suffices, but a BOM system might need ID-based removal.
- Node.js or Deno can run this file directly — no build step or dependencies required for the pattern itself.
