C# implements the Composite pattern using interfaces and abstract classes with the .NET framework's strong typing. The `ICarPart` interface declares the common contract. Leaf classes (`Wheel`, `BrakePad`, `Piston`, `SparkPlug`) implement `ICarPart` directly, returning their own weight, price, and part count. The `CompositePart` abstract class holds a `List<ICarPart>` and delegates `GetWeight()`, `GetPrice()`, and `GetPartCount()` to each child. Concrete composites (`Engine`, `Suspension`, `Car`) extend `CompositePart`, adding domain-specific properties. C# properties (`get; init;`) and LINQ provide modern, concise syntax for the pattern.

## How the Implementation Works

C#'s `interface` defines `ICarPart` with property `Name` and method signatures. The `CompositePart` abstract class implements `ICarPart` and provides the aggregation logic using LINQ's `Sum()` method. Leaf classes throw `NotSupportedException` from child-management methods. The driver uses collection initializer syntax and object initializers to build the tree. C# properties with `get; init;` accessors provide immutable data for leaf parts.

### OOP Principles Applied

| Principle | How It Is Applied |
|---|---|
| **Encapsulation** | Private fields (`_weight`, `_price`, `_children`) are accessed through properties and methods, never directly |
| **Inheritance** | `CompositePart` implements `ICarPart`; `Engine` extends `CompositePart` — a multi-level inheritance chain |
| **Polymorphism** | The `PrintPartsTree(ICarPart part)` method dispatches to the correct `GetWeight()` implementation at runtime via virtual dispatch |
| **Open/Closed** | Adding a `Turbocharger` leaf or `ExhaustSystem` composite requires zero changes to any existing code |
| **Single Responsibility** | Leaves own a single part's data; `CompositePart` owns aggregation; `Program` owns construction |
| **Liskov Substitution** | Every `Wheel` is an `ICarPart`; every `Engine` is an `ICarPart` — they are interchangeable in any method typed to `ICarPart` |

### Composite

```csharp
public abstract class CompositePart : ICarPart
{
    public string Name { get; }
    protected readonly List<ICarPart> Children;

    public CompositePart(string name)
    {
        Name = name;
        Children = new List<ICarPart>();
    }

    public virtual double GetWeight() =>
        Children.Sum(child => child.GetWeight());

    public virtual double GetPrice() =>
        Children.Sum(child => child.GetPrice());

    public virtual int GetPartCount() =>
        Children.Sum(child => child.GetPartCount());

    public void AddChild(ICarPart part)
    {
        ArgumentNullException.ThrowIfNull(part);
        Children.Add(part);
    }

    public void RemoveChild(ICarPart part) => Children.Remove(part);

    public ICarPart GetChild(int index)
    {
        if (index < 0 || index >= Children.Count)
        {
            throw new ArgumentOutOfRangeException(
                nameof(index),
                $"Child index {index} out of bounds"
            );
        }
        return Children[index];
    }

    public IReadOnlyList<ICarPart> GetChildren() => Children.AsReadOnly();
}
```

### Print Helper

```csharp
public static class PartPrinter
{
    public static void PrintPartsTree(ICarPart part, string indent = "")
    {
        string weight = part.GetWeight().ToString("F2");
        string price = part.GetPrice().ToString("F2");
        int count = part.GetPartCount();
        string kind = part is CompositePart ? "Assembly" : "Part";
        Console.WriteLine(
            $"{indent}{kind}: {part.Name} | " +
            $"Weight: {weight} kg | Price: ${price} | Parts: {count}"
        );
        if (part is CompositePart composite)
        {
            foreach (ICarPart child in composite.GetChildren())
            {
                PrintPartsTree(child, indent + "  ");
            }
        }
    }
}
```

        engine.AddChild(new SparkPlug("SparkPlug-3", 0.05, 12.50, 7));
        engine.AddChild(new SparkPlug("SparkPlug-4", 0.05, 12.50, 7));

        // Build suspension: MacPherson strut
        Suspension suspension = new("MacPherson strut");
        suspension.AddChild(new BrakePad("Front-Left-Pad", 0.65, 45.00, "ceramic"));
        suspension.AddChild(new BrakePad("Front-Right-Pad", 0.65, 45.00, "ceramic"));
        suspension.AddChild(new BrakePad("Rear-Left-Pad", 0.60, 42.00, "ceramic"));
        suspension.AddChild(new BrakePad("Rear-Right-Pad", 0.60, 42.00, "ceramic"));

        // Build the car
        Car car = new("Honda", "Civic", 2025);
        car.AddChild(engine);

        // Wheels
        car.AddChild(new Wheel("Wheel-Front-Left", 15.0, 299.99, 18));
        car.AddChild(new Wheel("Wheel-Front-Right", 15.0, 299.99, 18));
        car.AddChild(new Wheel("Wheel-Rear-Left", 15.0, 299.99, 18));
        car.AddChild(new Wheel("Wheel-Rear-Right", 15.0, 299.99, 18));

        car.AddChild(suspension);

        // Print results
        Console.WriteLine("=== Car Parts Hierarchy ===");
        PartPrinter.PrintPartsTree(car);

        Console.WriteLine("\n=== Summary ===");
        Console.WriteLine($"Vehicle: {car.Name}");
        Console.WriteLine($"Engine: {engine.DisplacementL}L ({engine.FuelType})");
        Console.WriteLine($"Suspension: {suspension.Type}");
        Console.WriteLine($"Total weight: {car.GetWeight():F2} kg");
        Console.WriteLine($"Total price: ${car.GetPrice():F2}");
        Console.WriteLine($"Total part count: {car.GetPartCount()}");
    }
}

## Language-Specific Notes

- C# uses the `default interface methods` feature (C# 8.0+) to provide default implementations for `AddChild`, `RemoveChild`, and `GetChild` on the `ICarPart` interface itself, making leaf classes cleaner.
- LINQ's `Sum()` method on `IEnumerable<T>` provides concise aggregation: `Children.Sum(child => child.GetWeight())` replaces manual loops.
- The `IReadOnlyList<ICarPart>` return type from `GetChildren()` (via `Children.AsReadOnly()`) prevents external callers from modifying the children list, enforcing encapsulation.
- C# 9 `init` accessors on properties could be used for immutable leaf construction, but the current code uses constructor assignment for broader compatibility.
- Pattern matching with `is` (`part is CompositePart composite`) provides a clean way to check composite vs. leaf in the print helper, with automatic variable capture.
- Compile and run: `dotnet new console -o CompositeDemo && copy this file into Program.cs && dotnet run` — or use any C# IDE with .NET 6.0 or later.
