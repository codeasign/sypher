## Implementation

Below is a complete Python implementation of the Template Method pattern using a stock market trading domain. The abstract `TradingStrategy` class uses Python's `ABC` module to define the template method `execute_trade()`. Concrete strategies `MovingAverageCrossover` and `MeanReversion` implement the primitive operations and optionally override hooks. The driver exercises both strategies with realistic AAPL, GOOGL, and MSFT price data using dataclasses for clean data structures.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌─────────────────────────────────────────────┐
//   │ TradingStrategy (ABC)                       │ ← OOP: Abstraction
//   ├─────────────────────────────────────────────┤    ISP: minimal contract
//   │ execute_trade(ticker, prices)               │     (analyze + size)
//   │ analyze_market(prices) -> (signal, conf)    │
//   │ calculate_position_size(conf, max) -> int    │
//   │ pre_validate(signal, conf) -> bool           │ ← hook
//   │ apply_risk_controls(ticker, shares) -> bool  │ ← hook
//   │ place_order(ticker, shares, sig) -> Order    │ ← concrete
//   │ log_trade(...)                               │ ← concrete
//   └──────────────────────┬──────────────────────┘
//                          ▲ extends
//   ┌──────────────────────┴──────────────────────┐
//   │                                             │
//   ┌─────────────────────────┐ ┌─────────────────────┐
//   │ MovingAverageCrossover  │ │ MeanReversion      │
//   ├─────────────────────────┤ ├─────────────────────┤
//   │ SHORT_WINDOW=3          │ │ LOOKBACK=10         │ ← OOP: Polymorphism
//   │ LONG_WINDOW=8           │ │ THRESHOLD=0.02      │    LSP: substitutable
//   │ analyze_market()        │ │ analyze_market()    │    OCP: extend with
//   │ calculate_position()    │ │ calculate_position()│    new subclasses
//   └─────────────────────────┘ └─────────────────────┘
```

#### OOP Concepts Applied

1. **Abstraction** — `TradingStrategy` inherits from `ABC` and declares abstract methods via the `@abstractmethod` decorator. Clients interact with strategies through the `execute_trade()` interface without knowing internal analysis details.
2. **Encapsulation** — Each strategy hides its window sizes, thresholds, and computational logic behind the `analyze_market()` and `calculate_position_size()` methods. The `Order` dataclass encapsulates order data.
3. **Inheritance** — `MovingAverageCrossover` and `MeanReversion` inherit `execute_trade()`, `place_order()`, and `log_trade()` from `TradingStrategy`.
4. **Polymorphism** — The driver iterates over a list of `TradingStrategy` instances and calls `execute_trade()` on each. Python's duck typing ensures the correct method runs per type.

#### SOLID Principles Applied

5. **SRP** — `TradingStrategy` has one responsibility: defining the trading workflow. Each concrete strategy has one responsibility: computing its specific analysis and sizing.
6. **OCP** — Adding a new strategy (e.g., `BollingerBandsStrategy`) requires only creating a new subclass. The template method in the base class and the driver code remain untouched.
7. **LSP** — Any concrete strategy can substitute `TradingStrategy` without breaking the driver. All strategies follow the same `execute_trade()` contract.
8. **ISP** — The abstract interface requires exactly two methods. Hooks are optional with default implementations. No class implements unused methods.
9. **DIP** — The driver depends on the abstract `TradingStrategy` type, not on concrete classes. High-level workflow is decoupled from low-level analysis.

### Data Structures

```python
# Template Method Pattern — Python — Stock Market example
# ======================================================
# Participants: TradingStrategy (AbstractClass),
# MovingAverageCrossover, MeanReversion (ConcreteClasses)
# OOP: abstraction, inheritance, polymorphism, encapsulation
# SOLID: SRP, OCP, LSP, ISP, DIP
# ======================================================

from abc import ABC, abstractmethod
from dataclasses import dataclass, field
from datetime import datetime
from typing import List, Tuple


# ── Data Structures ──

@dataclass
class TradingDecision:
    """Result of market analysis — signal and confidence level."""
    signal: int  # 1 = BUY, -1 = SELL, 0 = HOLD
    confidence: float  # 0.0 to 1.0


@dataclass
class Order:
    """Represents a filled order on the exchange."""
    ticker: str
    shares: int
    signal: int
    order_id: str = field(default_factory=lambda: f"ORD-{int(datetime.now().timestamp() * 1000)}")
    timestamp: str = field(default_factory=lambda: datetime.now().isoformat())
    status: str = "FILLED"

    def __str__(self) -> str:
        direction = "BUY" if self.signal > 0 else "SELL" if self.signal < 0 else "HOLD"
        return f"Order#{self.order_id}: {self.ticker} {direction} {self.shares} [{self.status}]"
```

### TradingStrategy (AbstractClass)

```python
# ── AbstractClass: TradingStrategy ──
# OOP role: Abstraction + ISP
# SOLID: ISP, DIP
class TradingStrategy(ABC):
    """Abstract base class defining the trading algorithm skeleton."""

    MIN_CONFIDENCE: float = 0.15
    MAX_POSITION: int = 1000

    # ── Template Method ──
    def execute_trade(self, ticker: str, price_data: List[float]) -> None:
        """Execute the trading algorithm following a fixed skeleton."""
        decision = self.analyze_market(price_data)

        # Hook 1: Pre-validate the signal
        if not self.pre_validate(decision.signal, decision.confidence):
            print(
                f"[{ticker}] Trade rejected by pre-validation "
                f"(confidence: {decision.confidence:.4f})"
            )
            return

        shares = self.calculate_position_size(decision.confidence, self.MAX_POSITION)
        if shares <= 0:
            print(f"[{ticker}] No trade: position size is zero")
            return

        # Hook 2: Apply risk controls
        if not self.apply_risk_controls(ticker, shares):
            print(
                f"[{ticker}] Trade rejected by risk controls "
                f"({shares} shares)"
            )
            return

        order = self.place_order(ticker, shares, decision.signal)
        self.log_trade(ticker, shares, decision.signal, decision.confidence, order)

    # ── Primitive Operations ──
    @abstractmethod
    def analyze_market(self, price_data: List[float]) -> TradingDecision:
        """Analyze price data and return a trading decision."""
        ...

    @abstractmethod
    def calculate_position_size(self, confidence: float, max_position: int) -> int:
        """Determine how many shares to trade based on confidence."""
        ...

    # ── Hooks (optional overrides) ──
    def pre_validate(self, signal: int, confidence: float) -> bool:
        """Pre-validation hook — default checks minimum confidence."""
        return confidence >= self.MIN_CONFIDENCE

    def apply_risk_controls(self, ticker: str, shares: int) -> bool:
        """Risk management hook — default allows all trades."""
        return True

    # ── Concrete Methods ──
    def place_order(self, ticker: str, shares: int, signal: int) -> Order:
        """Place an order on the exchange (concrete, shared by all strategies)."""
        return Order(ticker=ticker, shares=shares, signal=signal)

    def log_trade(
        self,
        ticker: str,
        shares: int,
        signal: int,
        confidence: float,
        order: Order
    ) -> None:
        """Log the completed trade in a consistent format."""
        ts = datetime.now().strftime("%m/%d/%Y, %I:%M:%S %p")
        direction = "BUY" if signal > 0 else "SELL" if signal < 0 else "HOLD"
        print(
            f"[{ts}] {ticker}: {direction} {shares} shares "
            f"(confidence: {confidence:.4f}) — {order}"
        )
```

### MovingAverageCrossover (ConcreteClass)

```python
# ── ConcreteClass: MovingAverageCrossover ──
# OOP role: Inheritance/Polymorphism
# SOLID: LSP, OCP, SRP
class MovingAverageCrossover(TradingStrategy):
    """Trades when a short-term moving average crosses a long-term one."""

    SHORT_WINDOW: int = 3
    LONG_WINDOW: int = 8

    def _moving_average(self, data: List[float], window: int) -> float:
        """Compute the simple moving average over the last `window` periods."""
        return sum(data[-window:]) / window

    def analyze_market(self, price_data: List[float]) -> TradingDecision:
        if len(price_data) < self.LONG_WINDOW:
            return TradingDecision(signal=0, confidence=0.0)

        short_ma = self._moving_average(price_data, self.SHORT_WINDOW)
        long_ma = self._moving_average(price_data, self.LONG_WINDOW)

        # Check for crossover by examining the previous period
        prev_data = price_data[:-1]
        prev_short = self._moving_average(prev_data, self.SHORT_WINDOW)
        prev_long = self._moving_average(prev_data, self.LONG_WINDOW)

        if short_ma > long_ma and prev_short <= prev_long:
            confidence = min((short_ma - long_ma) / long_ma, 1.0)
            return TradingDecision(signal=1, confidence=confidence)
        elif short_ma < long_ma and prev_short >= prev_long:
            confidence = min((long_ma - short_ma) / long_ma, 1.0)
            return TradingDecision(signal=-1, confidence=confidence)

        return TradingDecision(signal=0, confidence=0.0)

    def calculate_position_size(self, confidence: float, max_position: int) -> int:
        """Scale position by confidence with a multiplier."""
        return int(max_position * min(confidence * 1.2, 1.0))
```

### MeanReversion (ConcreteClass)

```python
# ── ConcreteClass: MeanReversion ──
# OOP role: Inheritance/Polymorphism
# SOLID: LSP, OCP, SRP
class MeanReversion(TradingStrategy):
    """Trades based on the assumption that prices revert to their mean."""

    LOOKBACK: int = 10
    DEVIATION_THRESHOLD: float = 0.02
    MAX_TRADE_SIZE: int = 500
    MIN_CONFIDENCE: float = 0.25

    def analyze_market(self, price_data: List[float]) -> TradingDecision:
        if len(price_data) < self.LOOKBACK + 1:
            return TradingDecision(signal=0, confidence=0.0)

        current_price = price_data[-1]
        recent_prices = price_data[-self.LOOKBACK:]
        mean_price = sum(recent_prices) / len(recent_prices)
        deviation = (mean_price - current_price) / mean_price

        if deviation > self.DEVIATION_THRESHOLD:
            return TradingDecision(signal=1, confidence=min(deviation, 0.5))
        elif deviation < -self.DEVIATION_THRESHOLD:
            return TradingDecision(signal=-1, confidence=min(abs(deviation), 0.5))

        return TradingDecision(signal=0, confidence=0.0)

    def calculate_position_size(self, confidence: float, max_position: int) -> int:
        """Mean reversion is more conservative — 60% of standard sizing."""
        return int(max_position * confidence * 0.6)

    # Override hook: tighter risk controls
    def apply_risk_controls(self, ticker: str, shares: int) -> bool:
        if shares > self.MAX_TRADE_SIZE:
            return False
        return super().apply_risk_controls(ticker, shares)

    # Override hook: stricter confidence threshold
    def pre_validate(self, signal: int, confidence: float) -> bool:
        return confidence >= self.MIN_CONFIDENCE
```

### Language-Specific Notes

- **`ABC` and `@abstractmethod`** — Python's `abc` module provides compile-time-like enforcement: any class that subclasses `TradingStrategy(ABC)` without implementing `analyze_market()` and `calculate_position_size()` raises a `TypeError` at instantiation time. This is as close to compile-time safety as Python gets.
- **`@dataclass` for data structures** — `TradingDecision` and `Order` use Python's `@dataclass` decorator with `field(default_factory=...)` for auto-generated order IDs and timestamps. This reduces boilerplate compared to handwritten `__init__` methods.
- **Type hints for clarity** — All method parameters and return types are annotated with type hints (`List[float]`, `Tuple[int, float]`, etc.). These are not enforced at runtime but provide excellent documentation and IDE support. Running `mypy` on this code will catch type errors without modifying behavior.
- **Super() delegation in hooks** — `MeanReversion.apply_risk_controls()` calls `super().apply_risk_controls()` before its own check, demonstrating how hooks can compose with parent behavior. This pattern is useful when you want to extend rather than replace the default hook behavior.
