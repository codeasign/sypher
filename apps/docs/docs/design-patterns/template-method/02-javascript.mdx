## Implementation

Below is a complete JavaScript implementation of the Template Method pattern using a stock market trading domain. The `TradingStrategy` abstract class defines the `executeTrade()` template method that outlines the trading algorithm. Concrete strategies `MovingAverageCrossover` and `MeanReversion` implement only the variant steps (`analyzeMarket`, `calculatePositionSize`) and optionally override hooks (`preValidate`, `applyRiskControls`). The driver exercises both strategies with realistic AAPL, GOOGL, and MSFT price data.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌─────────────────────────────────────────────────┐
//   │         TradingStrategy (AbstractClass)          │← OOP: Abstraction
//   ├─────────────────────────────────────────────────┤   ISP: focused on
//   │ + executeTrade(ticker, prices): void             │    trading lifecycle
//   │ # analyzeMarket(prices): {signal, confidence}    │
//   │ # calculatePositionSize(confidence, max): number │
//   │ # preValidate(signal, confidence): boolean       │← hook
//   │ # applyRiskControls(ticker, shares): boolean     │← hook
//   │ # placeOrder(ticker, shares, signal): Order      │← concrete
//   │ # logTrade(ticker, shares, signal, conf, order)  │← concrete
//   └────────────────────────┬─────────────────────────┘
//                            ▲ extends
//   ┌────────────────────────┴─────────────────────────┐
//   │                                                   │
//   ┌──────────────────────────┐  ┌──────────────────────────┐
//   │ MovingAverageCrossover   │  │ MeanReversion            │
//   ├──────────────────────────┤  ├──────────────────────────┤
//   │ analyzeMarket(prices)    │  │ analyzeMarket(prices)    │← OOP: Polymorphism
//   │ calculatePositionSize()  │  │ calculatePositionSize()  │   LSP: both behave
//   │ preValidate()            │  │ applyRiskControls()      │   as TradingStrategy
//   └──────────────────────────┘  └──────────────────────────┘   OCP: new strategies
//                                                                 without changing
//                                                                 the skeleton
```

#### OOP Concepts Applied

1. **Abstraction** — `TradingStrategy` defines the contract (template method + primitive operations) without revealing how concrete strategies compute signals or sizes.
2. **Encapsulation** — Each strategy hides its internal calculations (moving averages, lookback windows, thresholds) behind the `analyzeMarket()` method.
3. **Inheritance** — `MovingAverageCrossover` and `MeanReversion` inherit the `executeTrade()` template method, `placeOrder()`, and `logTrade()` from `TradingStrategy`.
4. **Polymorphism** — The driver iterates over an array of `TradingStrategy` references and calls `executeTrade()` on each. The correct implementation runs per type without branching.

#### SOLID Principles Applied

5. **SRP** — `TradingStrategy` has one responsibility: defining the trading algorithm skeleton. Each concrete strategy has one responsibility: implementing its specific analysis logic.
6. **OCP** — Adding a new strategy (e.g., `BollingerBands`) requires creating a new subclass that implements the two primitive operations. No existing code changes — the system is open for extension, closed for modification.
7. **LSP** — Every concrete strategy can substitute `TradingStrategy` without breaking the driver. All strategies respond to `executeTrade()` with the same contract guarantee.
8. **ISP** — The primitive operation interface is minimal: only `analyzeMarket()` and `calculatePositionSize()` must be implemented. Hooks are optional and have sensible defaults.
9. **DIP** — The driver depends on the abstract `TradingStrategy` type, not on concrete strategy classes. The high-level trading workflow does not depend on low-level analysis details.

### TradingStrategy (AbstractClass)

```javascript
// Template Method Pattern — JavaScript — Stock Market example
// ==========================================================
// Participants: TradingStrategy (AbstractClass),
// MovingAverageCrossover, MeanReversion (ConcreteClasses)
// OOP: abstraction, inheritance, polymorphism, encapsulation
// SOLID: SRP, OCP, LSP, ISP, DIP
// ==========================================================

// ── AbstractClass: TradingStrategy ──
// OOP role: Abstraction + ISP (minimal contract)
// SOLID: ISP, DIP
class TradingStrategy {
  constructor() {
    if (new.target === TradingStrategy) {
      throw new TypeError('Cannot instantiate abstract class TradingStrategy');
    }
    this.MIN_CONFIDENCE = 0.15;
    this.MAX_POSITION = 1000;
  }

  // ── Template Method ──
  // Defines the skeleton of the trading algorithm. Subclasses must not
  // override this method — they customize via primitive operations + hooks.
  executeTrade(ticker, priceData) {
    const { signal, confidence } = this.analyzeMarket(priceData);

    // Hook 1: Pre-validate the signal
    if (!this.preValidate(signal, confidence)) {
      console.log(`[${ticker}] Trade rejected by pre-validation (confidence: ${confidence.toFixed(2)})`);
      return;
    }

    const shares = this.calculatePositionSize(confidence, this.MAX_POSITION);
    if (shares <= 0) {
      console.log(`[${ticker}] No trade: position size is zero`);
      return;
    }

    // Hook 2: Apply risk controls
    if (!this.applyRiskControls(ticker, shares)) {
      console.log(`[${ticker}] Trade rejected by risk controls (${shares} shares)`);
      return;
    }

    const order = this.placeOrder(ticker, shares, signal);
    this.logTrade(ticker, shares, signal, confidence, order);
  }

  // ── Primitive Operations ──
  // Must be implemented by concrete strategies

  analyzeMarket(priceData) {
    throw new Error('analyzeMarket() must be overridden');
  }

  calculatePositionSize(confidence, maxPosition) {
    throw new Error('calculatePositionSize() must be overridden');
  }

  // ── Hooks (optional overrides) ──

  preValidate(signal, confidence) {
    return confidence >= this.MIN_CONFIDENCE;
  }

  applyRiskControls(ticker, shares) {
    // Default: always pass risk controls
    return true;
  }

  // ── Concrete Methods (shared by all strategies) ──

  placeOrder(ticker, shares, signal) {
    return {
      ticker,
      shares,
      signal,
      orderId: `ORD-${Date.now()}`,
      timestamp: new Date().toISOString(),
      status: 'FILLED'
    };
  }

  logTrade(ticker, shares, signal, confidence, order) {
    const ts = new Date().toLocaleString('en-US', { hour12: false });
    const direction = signal > 0 ? 'BUY' : (signal < 0 ? 'SELL' : 'HOLD');
    console.log(
      `[${ts}] ${ticker}: ${direction} ${shares} shares ` +
      `(confidence: ${confidence.toFixed(4)}) — Order#${order.orderId} [${order.status}]`
    );
  }
}
```

### MovingAverageCrossover (ConcreteClass)

```javascript
// ── ConcreteClass: MovingAverageCrossover ──
// OOP role: Inheritance/Polymorphism
// SOLID: LSP (substitutable for TradingStrategy), OCP (extensible), SRP
class MovingAverageCrossover extends TradingStrategy {
  constructor() {
    super();
    this.SHORT_WINDOW = 3;
    this.LONG_WINDOW = 8;
  }

  analyzeMarket(priceData) {
    if (priceData.length < this.LONG_WINDOW) {
      return { signal: 0, confidence: 0 };
    }

    const shortMA = this._movingAverage(priceData, this.SHORT_WINDOW);
    const longMA = this._movingAverage(priceData, this.LONG_WINDOW);

    // Crossover: when short MA crosses above long MA -> BUY
    // When short MA crosses below long MA -> SELL
    const prevShort = priceData.slice(-this.SHORT_WINDOW - 1, -1)
      .reduce((a, b) => a + b, 0) / this.SHORT_WINDOW;
    const prevLong = priceData.slice(-this.LONG_WINDOW - 1, -1)
      .reduce((a, b) => a + b, 0) / this.LONG_WINDOW;

    if (shortMA > longMA && prevShort <= prevLong) {
      // Bullish crossover
      const confidence = Math.min((shortMA - longMA) / longMA, 1.0);
      return { signal: 1, confidence };
    } else if (shortMA < longMA && prevShort >= prevLong) {
      // Bearish crossover
      const confidence = Math.min((longMA - shortMA) / longMA, 1.0);
      return { signal: -1, confidence };
    }

    return { signal: 0, confidence: 0 };
  }

  _movingAverage(data, window) {
    const slice = data.slice(-window);
    return slice.reduce((a, b) => a + b, 0) / slice.length;
  }

  calculatePositionSize(confidence, maxPosition) {
    // Scale position by confidence level
    return Math.floor(maxPosition * Math.min(confidence * 1.2, 1.0));
  }
}
```

### MeanReversion (ConcreteClass)

```javascript
// ── ConcreteClass: MeanReversion ──
// OOP role: Inheritance/Polymorphism
// SOLID: LSP (substitutable for TradingStrategy), OCP, SRP
class MeanReversion extends TradingStrategy {
  constructor() {
    super();
    this.LOOKBACK = 10;
    this.DEVIATION_THRESHOLD = 0.02; // 2% deviation triggers trade
    this.MAX_TRADE_SIZE = 500; // more conservative
    this.MIN_CONFIDENCE = 0.25; // stricter than default
  }

  analyzeMarket(priceData) {
    if (priceData.length < this.LOOKBACK + 1) {
      return { signal: 0, confidence: 0 };
    }

    const currentPrice = priceData[priceData.length - 1];
    const recentPrices = priceData.slice(-this.LOOKBACK);
    const meanPrice = recentPrices.reduce((a, b) => a + b, 0) / recentPrices.length;
    const deviation = (meanPrice - currentPrice) / meanPrice;

    if (deviation > this.DEVIATION_THRESHOLD) {
      // Price is below mean — expect reversion upward -> BUY
      return { signal: 1, Math.min(deviation, 0.5) };
    } else if (deviation < -this.DEVIATION_THRESHOLD) {
      // Price is above mean — expect reversion downward -> SELL
      return { signal: -1, confidence: Math.min(Math.abs(deviation), 0.5) };
    }

    return { signal: 0, confidence: 0 };
  }

  calculatePositionSize(confidence, maxPosition) {
    // Mean reversion is more conservative: 60% of standard sizing
    return Math.floor(maxPosition * confidence * 0.6);
  }

  // Override hook: mean reversion has tighter risk controls
  applyRiskControls(ticker, shares) {
    if (shares > this.MAX_TRADE_SIZE) {
      return false;
    }
    // Call parent hook as well (composition of hooks)
    return super.applyRiskControls(ticker, shares);
  }

  // Override hook: stricter confidence threshold
  preValidate(signal, confidence) {
    return confidence >= this.MIN_CONFIDENCE;
  }
}
```

      185.00, 184.50

      141.20, 140.80, 140.10, 139.50, 140.00,
      139.20, 138.80, 138.10, 137.50, 138.00,
      137.20, 136.80
    ],
    MSFT: [
      378.00, 379.50, 380.20, 378.80, 381.00,
      382.30, 383.00, 382.10, 384.50, 385.00,
      383.80, 386.20
    ]
  };

  const strategies = [
    new MovingAverageCrossover(),
    new MeanReversion()
  ];

  for (const ticker of Object.keys(marketData)) {
    console.log(`\n========== ${ticker} ==========`);
    const prices = marketData[ticker];

    for (const strategy of strategies) {
      const name = strategy.constructor.name;
      console.log(`\n--- ${name} ---`);
      strategy.executeTrade(ticker, prices);
    }
  }

  console.log('\n========== Portfolio Summary ==========');
  console.log('All trades executed successfully using the Template Method pattern.');
  console.log('The executeTrade() skeleton enforced consistent order across all strategies.');
}

main();

### Language-Specific Notes

- **ES2022+ class fields** — JavaScript uses `static` class fields for constants (`SHORT_WINDOW = 3`) and `constructor()` binding for instance-specific configuration. Private fields (`#field`) could further encapsulate internal strategy parameters, though the example uses protected-by-convention underscore prefixes.
- **Abstract class enforcement** — JavaScript does not have native abstract classes. The pattern uses `new.target` in the constructor to prevent direct instantiation of `TradingStrategy`, and the primitive operations throw `Error` if not overridden — a runtime check instead of compile-time enforcement.
- **Polymorphism via duck typing** — JavaScript's prototype-based inheritance means any object with `executeTrade()` conforms to the strategy interface. The driver iterates over strategies without needing a shared type annotation, relying on structural typing at runtime.
- **Template method immutability** — The `executeTrade()` method is intentionally not a getter or symbol-keyed property so that subclasses cannot accidentally override it via a common method name collision. If true sealing were needed, `Object.freeze()` could be applied to the prototype method.
