The JavaScript implementation of the Adapter pattern uses prototypal inheritance or ES6 class syntax to bridge incompatible interfaces. In the car domain, a JavaScript adapter can wrap a Tesla proprietary charging API behind a standard CCS charger interface, allowing any charging station to communicate with Tesla vehicles through a common protocol.

## How the Implementation Works

| Aspect | Implementation |
|---|---|
| **Target** | `CCSCharger` base class defines the expected interface (duck typing in JavaScript) |
| **Adaptee** | `TeslaSupercharger` has `sendCharge(voltage, amperage)` using proprietary Tesla protocol |
| **Adapter** | `TeslaToCCSAdapter` extends `CCSCharger`, wraps a `TeslaSupercharger`, translates CCS kw into voltage/amperage |
| **Client** | `ChargingStation` calls `chargeDC(powerKw)` on any `CCSCharger` without knowing the implementation |
| **Composition** | Adapter holds a reference to Adaptee — uses composition over inheritance |
| **Open/Closed** | New car brands can be supported by writing new adapters without modifying the charging station |

### Target Interface

```javascript
// ── Target Interface (duck-typed) ──
class CCSCharger {
  /**
   * Charge a vehicle at the specified power level using the CCS standard.
   * @param {number} powerKw - Requested charging power in kilowatts.
   * @returns {string} Charging status message.
   */
  chargeDC(powerKw) {
    throw new Error("CCSCharger subclasses must implement chargeDC()");
  }
}
```

### Adaptee

```javascript
// ── Adaptee ──
/**
 * Represents Tesla's proprietary Supercharger hardware.
 * Communicates using Tesla's own protocol with voltage and amperage parameters.
 */
class TeslaSupercharger {
  /**
   * Send a charge request using Tesla's proprietary interface.
   * @param {number} voltage - Charging voltage in volts.
   * @param {number} amperage - Charging current in amperes.
   * @returns {string} Confirmation from the Tesla charging hardware.
   */
  sendCharge(voltage, amperage) {
    const powerKw = (voltage * amperage) / 1000;
    return `Tesla Supercharger delivering ${powerKw} kW at ${voltage}V / ${amperage}A via Tesla connector`;
  }
}
```

### Adapter

```javascript
// ── Adapter ──
/**
 * Adapts a TeslaSupercharger to the CCSCharger interface.
 * Tesla vehicles can now charge at any CCS-compatible station.
 */
class TeslaToCCSAdapter extends CCSCharger {
  /**
   * @param {TeslaSupercharger} teslaCharger - The Tesla hardware to adapt.
   */
  constructor(teslaCharger) {
    super();
    /** @private */
    this._teslaCharger = teslaCharger;
  }

  /**
   * Convert a CCS standard charging request to Tesla's proprietary protocol.
   * CCS uses a nominal voltage of 400V (or 800V for newer stations).
   * The required amperage is derived from the power and voltage.
   *
   * @param {number} powerKw - Kilowatts requested via CCS standard.
   * @returns {string} Result from the Tesla charging hardware.
   */
  chargeDC(powerKw) {
    if (powerKw <= 0) {
      throw new Error("Power must be a positive value in kilowatts");
    }
    if (powerKw > 1000) {
      throw new Error("Requested power exceeds maximum of 1000 kW");
    }

    // CCS standard nominal voltage (800V for ultra-fast chargers)
    const voltage = powerKw > 350 ? 800 : 400;
    // Calculate amperage: Power (W) = Voltage (V) * Amperage (A)
    const amperage = Math.ceil((powerKw * 1000) / voltage);

    console.log(`  Adapter: Translating ${powerKw} kW CCS request to ${voltage}V / ${amperage}A for Tesla hardware`);
    return this._teslaCharger.sendCharge(voltage, amperage);
  }
}
```

### Client

```javascript
// ── Client ──
/**
 * A CCS-compatible charging station that can service any vehicle
 * presenting a CCSCharger interface.
 */
class ChargingStation {
  /**
   * @param {string} name - The station name and location.
   */
  constructor(name) {
    this.name = name;
  }

  /**
   * Service a vehicle at this charging station.
   *
   * @param {CCSCharger} charger - The CCS-compatible charger interface.
   * @param {string} carModel - The make and model of the vehicle.
   * @param {number} requiredKw - The power the vehicle needs in kilowatts.
   * @returns {string} The complete charging session log.
   */
  serviceVehicle(charger, carModel, requiredKw) {
    const timestamp = new Date().toISOString();
    console.log(`[${timestamp}] ${this.name}: Starting charge for ${carModel}`);
    console.log(`  Requested power: ${requiredKw} kW`);

    try {
      const result = charger.chargeDC(requiredKw);
      const message = `  Result: ${result}`;
      console.log(message);
      console.log(`  Status: Charge complete for ${carModel}`);
      return `${carModel} charged successfully: ${result}`;
    } catch (error) {
      const errorMsg = `  ERROR: ${error.message}`;
      console.error(errorMsg);
      return `${carModel} charge failed: ${error.message}`;
    }
  }
}
```

### Main / Driver

```javascript
// ── Driver ──
function main() {
  console.log("=".repeat(60));
  console.log("ADAPTER PATTERN — Tesla to CCS Charging Demo");
  console.log("=".repeat(60));
  console.log("");

  // Create the adaptee (Tesla's proprietary charging hardware).
  const teslaHardware = new TeslaSupercharger();

  // Wrap it in the adapter so it speaks the CCS language.
  const adapter = new TeslaToCCSAdapter(teslaHardware);

  // Create a CCS charging station that only knows the CCS interface.
  const station = new ChargingStation("Electrify America — Newark, DE");

  // Define a fleet of Tesla vehicles needing a charge.
  const vehicles = [
    { model: "Tesla Model 3 Long Range", powerKw: 250 },
    { model: "Tesla Model Y Performance", powerKw: 250 },
    { model: "Tesla Model S Plaid", powerKw: 200 },
    { model: "Tesla Cybertruck", powerKw: 350 },
    { model: "Tesla Roadster (upcoming)", powerKw: 400 }, // Triggers 800V mode
  ];

  // Service each vehicle through the adapter.
  // The station never knows it is talking to Tesla hardware.
  const results = [];
  for (const vehicle of vehicles) {
    console.log(`--- Charging: ${vehicle.model} ---`);
    const result = station.serviceVehicle(adapter, vehicle.model, vehicle.powerKw);
    results.push(result);
    console.log("");
  }

  console.log("=".repeat(60));
  console.log("CHARGING SESSION SUMMARY");
  console.log("=".repeat(60));
  for (const r of results) {
    console.log(`  ${r}`);
  }
  console.log("");
  console.log(`Total vehicles charged: ${vehicles.length}`);
  console.log("All charges completed through the TeslaToCCSAdapter.");
}

main();
```

## Language-Specific Notes

- JavaScript uses duck typing rather than formal interfaces. The `CCSCharger` base class with a throwing method in the parent serves as the interface contract, relying on runtime checks rather than compile-time enforcement. This is idiomatic for JavaScript but means interface violations are only caught at runtime.

- The `extends` keyword establishes the inheritance chain, but composition (holding a `_teslaCharger` reference) is preferred over inheriting from the Adaptee. This follows the composition-over-inheritance principle that the Adapter pattern endorses.

- ESLint or TypeScript in strict mode can flag missing `chargeDC` overrides at analysis time, providing a layer of safety without changing the runtime semantics.

- The adapter uses `Math.ceil` for amperage calculation to ensure the electrical system never exceeds the specified power delivery — a safety consideration that mirrors real charging hardware where rounding down could under-deliver promised power.

- Error handling with try/catch in the client demonstrates how adapter-layer exceptions (invalid power values) propagate clearly without leaking Adaptee-specific error types into the client interface.

- The JSDoc comments (`@param`, `@returns`, `@private`) serve as documentation contracts that IDEs and editors use for autocomplete and inline type checking, partially compensating for JavaScript's lack of compile-time type safety.
