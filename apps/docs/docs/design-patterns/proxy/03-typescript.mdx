This page implements the Proxy design pattern in TypeScript using a Cars domain. The Proxy pattern provides a surrogate object that controls access to another object. Here, a `Car` interface is implemented by `SportsCar` (the real subject), while `LazyCarModelProxy` defers expensive 3D model loading and `ValetCarProxy` restricts speed and access for valet parking. TypeScript's type system enforces the interface contract at compile time.

## How the Implementation Works

### Class Diagram (Pseudocode)

The following table maps the participants to their OOP/SOLID roles:

```
  ┌──────────────────────────────────────────────────────────────────────────┐
  │ Participant            ──  OOP Role         ──  SOLID Principle         │
  ├──────────────────────────────────────────────────────────────────────────┤
  │ Car (interface)        ──  Abstraction      ──  DIP (depend on          │
  │                        ──  (contract)       ──  abstractions)           │
  ├──────────────────────────────────────────────────────────────────────────┤
  │ SportsCar              ──  Inheritance      ──  LSP (substitutable      │
  │ (real subject)         ──  Polymorphism     ──  for Car)                │
  │                        ──  implements       ──  OCP (open for           │
  │                        ──                  ──  extension)               │
  ├──────────────────────────────────────────────────────────────────────────┤
  │ ValetCarProxy          ──  Encapsulation    ──  SRP (single reason      │
  │ LazyCarModelProxy      ──  (access logic    ──  to change: access       │
  │ (proxies)              ──   hidden)         ──  control / lazy load)    │
  ├──────────────────────────────────────────────────────────────────────────┤
  │ Driver / client        ──  Composition      ──  ISP (depends only       │
  │ (caller)               ──  (uses Car API)   ──  on Car interface)       │
  └──────────────────────────────────────────────────────────────────────────┘
```

### OOP Concepts Applied

- **Abstraction** — The `Car` interface defines a contract of methods (`displayDetails`, `drive`, `getMaxSpeed`, `accessTrunk`, `getVin`) that every car representation must implement. TypeScript enforces this contract at compile time.
- **Encapsulation** — The `SportsCar` hides its internal state through TypeScript's `private` keyword. The proxies encapsulate their access control rules and lazy initialization logic.
- **Inheritance and Interface Implementation** — `SportsCar`, `ValetCarProxy`, and `LazyCarModelProxy` implement the `Car` interface. TypeScript's structural typing ensures all required methods are present.
- **Polymorphism** — The driver calls methods on `Car` references without knowing whether they point to a `SportsCar`, `ValetCarProxy`, or `LazyCarModelProxy`. Each provides its own implementation of `drive()` and `displayDetails()`.

### SOLID Principles Applied

- **Single Responsibility Principle** — `SportsCar` handles real car behavior. `ValetCarProxy` handles access control. `LazyCarModelProxy` handles lazy initialization.
- **Open/Closed Principle** — Adding a new proxy type (e.g., `LoggingCarProxy`) requires creating a new class that implements `Car`. Existing classes remain unchanged.
- **Liskov Substitution Principle** — Every proxy can replace `Car` in client code without breaking correctness.
- **Interface Segregation Principle** — The `Car` interface exposes only the methods every car representation needs.
- **Dependency Inversion Principle** — High-level driver code depends on the `Car` abstraction, not on concrete classes.

## Implementation

The complete TypeScript program below demonstrates the Proxy pattern in a Cars context. It is fully typed and can be run with `ts-node`.

### Utility

```typescript
/** Generate a random 17-character vehicle identification number. */
function generateVin(): string {
  const chars = 'ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789';
  let vin = '';
  for (let i = 0; i < 17; i++) {
    vin += chars.charAt(Math.floor(Math.random() * chars.length));
  }
  return vin;
}
```

interface Car {

### RealSubject: SportsCar

```typescript
class SportsCar implements Car {
  private readonly vin: string;
  private readonly make: string;
  private readonly model: string;
  private readonly year: number;
  private readonly horsepower: number = 750;
  private readonly engineType: string = '5.2L V10 Twin-Turbo';
  private readonly acceleration0to60: number = 2.8;
  private readonly maxSpeed: number = 200.0;
  private full3DModel: object | null = null;
  private readonly trunkContents: string[] = ['First aid kit', 'Tool set', 'Owners manual'];
  private readonly gloveboxContents: string[] = ['Registration', 'Insurance card', 'Service record'];
  private isEngineRunning: boolean = false;
  private currentSpeed: number = 0.0;

  constructor(vin: string, make: string, model: string, year: number) {
    this.vin = vin;
    this.make = make;
    this.model = model;
    this.year = year;
    this.load3DModel();
  }

  /** Simulate expensive 3D model loading. */
  private load3DModel(): void {
    console.log(
      `  [SportsCar] Loading 3D model for ${this.vin}... ` +
      `200 MB of polygon data loaded.`
    );
    this.full3DModel = {
      polygons: 450000,
      textures: ['carbon_fiber', 'leather_black', 'aluminum_brushed'],
      materials: ['forged_carbon', 'alcantara', 'titanium'],
    };
  }

  getVin(): string { return this.vin; }
  getMake(): string { return this.make; }
  getModel(): string { return this.model; }
  getYear(): number { return this.year; }

  getShortDescription(): string {
    return `${this.year} ${this.make} ${this.model} (${this.vin})`;
  }

  displayDetails(): string {
    const modelData = this.full3DModel as { polygons: number; textures: string[] };
    const trunk = this.trunkContents.join(', ');
    const glovebox = this.gloveboxContents.join(', ');
    return (
      `--- ${this.getShortDescription()} ---\n` +
      `  Engine: ${this.engineType}\n` +
      `  Horsepower: ${this.horsepower} hp\n` +
      `  0-60 mph: ${this.acceleration0to60}s\n` +
      `  Top Speed: ${this.maxSpeed} mph\n` +
      `  3D Model: ${modelData.polygons.toLocaleString()} polygons, ` +
      `${modelData.textures.length} textures\n` +
      `  Trunk: ${trunk}\n` +
      `  Glovebox: ${glovebox}`
    );
  }

  drive(speed: number): string {
    const actualSpeed = Math.min(speed, this.maxSpeed);
    this.isEngineRunning = true;
    this.currentSpeed = actualSpeed;
    const engineNote = 'ROAR! V10 screams through the rev range.';
    return (
      `  [SportsCar] ${engineNote}\n` +
      `  Speed: ${actualSpeed.toFixed(1)} mph | Gear: 7th | RPM: 8200`
    );
  }

  getMaxSpeed(): number { return this.maxSpeed; }

  accessTrunk(): string {
    const contents = this.trunkContents.join(', ');
    return `  [SportsCar] Trunk opened. Contents: ${contents}`;
  }
}
```

### Virtual Proxy: LazyCarModelProxy

```typescript
class LazyCarModelProxy implements Car {
  private realCar: SportsCar | null = null;
  private readonly vin: string;
  private readonly make: string;
  private readonly model: string;
  private readonly year: number;

  constructor(vin: string, make: string, model: string, year: number) {
    this.vin = vin;
    this.make = make;
    this.model = model;
    this.year = year;
  }

  /**
   * Initialize the real car on first access.
   */
  private getRealCar(): SportsCar {
    if (this.realCar === null) {
      console.log(
        `  [LazyCarModelProxy] Initializing real SportsCar for ` +
        `${this.getShortDescription()}...`
      );
      this.realCar = new SportsCar(this.vin, this.make, this.model, this.year);
    } else {
      console.log(`  [LazyCarModelProxy] Reusing existing SportsCar instance.`);
    }
    return this.realCar;
  }

  getVin(): string { return this.vin; }
  getMake(): string { return this.make; }
  getModel(): string { return this.model; }
  getYear(): number { return this.year; }

  getShortDescription(): string {
    return `${this.year} ${this.make} ${this.model} (${this.vin})`;
  }

  displayDetails(): string {
    return this.getRealCar().displayDetails();
  }

  drive(speed: number): string {
    return this.getRealCar().drive(speed);
  }

  getMaxSpeed(): number {
    return this.getRealCar().getMaxSpeed();
  }

  accessTrunk(): string {
    return this.getRealCar().accessTrunk();
  }
}
```

### Protection Proxy: ValetCarProxy

```typescript
class ValetCarProxy implements Car {
  private readonly realCar: Car;
  private readonly maxValetSpeed: number;
  private readonly trunkDeniedMessage: string;

  constructor(realCar: Car, maxValetSpeed: number = 30) {
    this.realCar = realCar;
    this.maxValetSpeed = maxValetSpeed;
    this.trunkDeniedMessage =
      '  [ValetCarProxy] ACCESS DENIED: Valet key does not permit trunk access.';
  }

  getVin(): string { return this.realCar.getVin(); }
  getMake(): string { return this.realCar.getMake(); }
  getModel(): string { return this.realCar.getModel(); }
  getYear(): number { return this.realCar.getYear(); }

  getShortDescription(): string {
    return this.realCar.getShortDescription();
  }

  displayDetails(): string {
    return (
      `--- Valet Card ---\n` +
      `  VIN: ${this.getVin()}\n` +
      `  Max Permitted Speed: ${this.maxValetSpeed} mph\n` +
      `  Trunk Access: DENIED\n` +
      `  Glovebox Access: DENIED`
    );
  }

  drive(speed: number): string {
    const limitedSpeed = Math.min(speed, this.maxValetSpeed);
    if (limitedSpeed < speed) {
      console.log(`  [ValetCarProxy] Speed limited to ${this.maxValetSpeed} mph.`);
    }
    return this.realCar.drive(limitedSpeed);
  }

  getMaxSpeed(): number { return this.maxValetSpeed; }

  accessTrunk(): string {
    return this.trunkDeniedMessage;
  }
}
```

### Main / Driver

```typescript
function main(): void {
  console.log('='.repeat(72));
  console.log('  PROXY PATTERN — Cars Domain (TypeScript 5.x)');
  console.log('='.repeat(72));
  console.log();

  // ── Scenario 1: Virtual Proxy (Lazy Loading) ────────────────────────────
  console.log('--- SCENARIO 1: Virtual Proxy (Lazy Loading) ---');
  const lazyCar: Car = new LazyCarModelProxy(generateVin(), 'Sypher', 'GT', 2026);
  console.log('  Showroom: Created LazyCarModelProxy. No 3D model loaded yet.');
  console.log();

  console.log('  Customer browses thumbnail:');
  console.log(`  ${lazyCar.getShortDescription()}`);
  console.log();

  console.log('  Customer clicks "View Full Details":');
  const details: string = lazyCar.displayDetails();
  console.log(details);
  console.log();

  // ── Scenario 2: Protection Proxy (Valet Key) ─────────────────────────────
  console.log('--- SCENARIO 2: Protection Proxy (Valet Key) ---');
  const realCar: Car = new SportsCar(generateVin(), 'Sypher', 'GT-S', 2026);
  const valetKey: Car = new ValetCarProxy(realCar);
  console.log();

  console.log('  Owner hands the valet key to the parking attendant.');
  console.log();

  console.log('  Valet: Displays info card:');
  console.log(valetKey.displayDetails());
  console.log();

  console.log('  Valet: Attempts to drive at 80 mph:');
  console.log(valetKey.drive(80.0));
  console.log();

  console.log('  Valet: Attempts to access the trunk:');
  console.log(valetKey.accessTrunk());
  console.log();

  // ── Scenario 3: Owner Returns — Full Access ─────────────────────────────
  console.log('--- SCENARIO 3: Owner Returns — Full Access ---');
  console.log('  Owner uses the real car key directly:');
  console.log(realCar.drive(120.0));
  console.log(realCar.accessTrunk());
  console.log();

  // ── Scenario 4: Polymorphic Iteration ────────────────────────────────────
  console.log('--- SCENARIO 4: Polymorphic Car Display ---');
  const cars: Car[] = [lazyCar, valetKey];
  for (const car of cars) {
    console.log(`  [${car.getVin()}] Max Speed: ${car.getMaxSpeed()} mph`);
  }
  console.log();

  // ── Scenario 5: Demonstrating Lazy Loading Benefit ──────────────────────
  console.log('--- SCENARIO 5: Lazy Loading Benefit ---');
  const showroomCars: LazyCarModelProxy[] = [
    new LazyCarModelProxy(generateVin(), 'Sypher', 'GT', 2026),
    new LazyCarModelProxy(generateVin(), 'Sypher', 'GT-S', 2026),
    new LazyCarModelProxy(generateVin(), 'Falcon', 'X1', 2026),
  ];
  console.log('  All 3 proxies created. Memory usage: minimal.');
  console.log('  Showing thumbnails...');
  for (const car of showroomCars) {
    console.log(`    ${car.getShortDescription()}`);
  }
  console.log();

  console.log('  Customer examines one car in detail:');
  console.log(showroomCars[0].displayDetails());
  console.log();

  console.log('='.repeat(72));
  console.log('  Proxy pattern demonstration complete.');
  console.log('='.repeat(72));
}

main();
```

## Language-Specific Notes

- **Compile-time interface enforcement** — TypeScript's `Car` interface ensures at compile time that `SportsCar`, `LazyCarModelProxy`, and `ValetCarProxy` all provide the required methods with correct signatures. A missing method or a type mismatch produces a compilation error before the code runs. This is the key advantage over JavaScript's runtime duck typing.

- **`readonly` for immutable properties** — Fields like `vin`, `make`, `model`, and `year` are declared `readonly`, preventing accidental reassignment after construction. The `maxSpeed` and `engineType` are also `readonly` because they are intrinsic properties of the real car that should never change.

- **Union type for nullable proxy reference** — `LazyCarModelProxy.realCar` is typed as `SportsCar | null`, explicitly representing the uninitialized state. TypeScript forces null checks before accessing the real car — the `getRealCar()` method performs this check and throws if the proxy was somehow used without initialization, though in practice the lazy initialization ensures availability.

- **Interface as a contract for both real subject and proxies** — Unlike the JavaScript version which uses class-based inheritance with abstract-like stubs, the TypeScript version uses a pure `interface` for `Car`. Both `SportsCar` and the proxy classes implement the same interface. This is cleaner and more idiomatic in TypeScript, where `implements` is a first-class concept.

- **Type annotations on all public API surfaces** — Every method parameter and return type is explicitly annotated. This provides documentation-as-code and enables IDE autocompletion, refactoring tools, and compile-time verification that the proxy correctly implements the `Car` contract.

- **`as` type assertion for private model data** — In `displayDetails()`, the `full3DModel` is accessed with an `as { polygons: number; textures: string[] }` assertion because the field is private and was initialized by a private method. The assertion provides the shape information needed without exposing the type definition publicly.
