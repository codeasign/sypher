This page implements the Proxy design pattern in Rust using a Cars domain. The Proxy pattern provides a surrogate object that controls access to another object. Here, a `Car` trait is implemented by `SportsCar` (the real subject), while `LazyCarModelProxy` defers expensive 3D model loading via `OnceLock` and `ValetCarProxy` restricts speed and access for valet parking. Rust's trait system and ownership model enforce the pattern contract at compile time.

## How the Implementation Works

### Class Diagram (Pseudocode)

The following table maps the participants to their OOP/SOLID roles:

```
  ┌──────────────────────────────────────────────────────────────────────────┐
  │ Participant            ──  OOP Role         ──  SOLID Principle         │
  ├──────────────────────────────────────────────────────────────────────────┤
  │ Car (trait)            ──  Abstraction      ──  DIP (depend on          │
  │                        ──  (contract)       ──  abstractions)           │
  ├──────────────────────────────────────────────────────────────────────────┤
  │ SportsCar              ──  impl Trait       ──  LSP (substitutable      │
  │ (real subject)         ──  Polymorphism     ──  for Car)                │
  │                        ──                  ──  OCP (open for           │
  │                        ──                  ──  extension)               │
  ├──────────────────────────────────────────────────────────────────────────┤
  │ ValetCarProxy          ──  Encapsulation    ──  SRP (single reason      │
  │ LazyCarModelProxy      ──  (access logic    ──  to change: access       │
  │ (proxies)              ──   hidden)         ──  control / lazy load)    │
  ├──────────────────────────────────────────────────────────────────────────┤
  │ Driver / client        ──  Composition      ──  ISP (depends only       │
  │ (caller)               ──  (uses Car trait) ──  on Car trait)           │
  └──────────────────────────────────────────────────────────────────────────┘
```

### OOP Concepts Applied

- **Abstraction** — The `Car` trait defines the contract with method signatures. Rust enforces this contract at compile time — any type that claims to implement `Car` must provide all required methods.
- **Encapsulation** — Internal state in `SportsCar` uses `private` fields (module-level visibility by default). The proxies encapsulate their access control logic and lazy initialization behind the `Car` trait.
- **Trait Implementation** — `SportsCar`, `LazyCarModelProxy`, and `ValetCarProxy` implement the `Car` trait using the `impl Car for SportsCar` syntax.
- **Polymorphism** via Trait Objects — The driver uses `&dyn Car` references to hold heterogeneous car types. Rust resolves method calls at runtime via the vtable embedded in the trait object.

### SOLID Principles Applied

- **Single Responsibility Principle** — `SportsCar` handles real car behavior. `ValetCarProxy` handles access control. `LazyCarModelProxy` handles lazy initialization.
- **Open/Closed Principle** — Adding a new proxy type requires implementing `Car` for a new struct. Existing types remain unchanged.
- **Liskov Substitution Principle** — Every proxy can replace `&dyn Car` in client code without breaking correctness.
- **Interface Segregation Principle** — The `Car` trait exposes only the methods every car representation needs.
- **Dependency Inversion Principle** — High-level driver code depends on the `Car` trait, not on concrete types.

## Implementation

The complete Rust program below demonstrates the Proxy pattern in a Cars context. It is runnable with Rust 1.78+ and uses `OnceLock` for lazy initialization and trait objects for polymorphism.

### Utility and Supporting Types

```rust
use std::sync::OnceLock;
use rand::Rng;

/// Generate a random 17-character vehicle identification number.
fn generate_vin() -> String {
    let chars: Vec<char> = "ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789".chars().collect();
    let mut rng = rand::thread_rng();
    (0..17)
        .map(|_| chars[rng.gen_range(0..chars.len())])
        .collect()
}

/// Represents a loaded 3D model for a car.
struct ThreeDModel {
    polygons: u32,
    textures: Vec<String>,
    materials: Vec<String>,
}

impl ThreeDModel {
    fn new() -> Self {
        println!("  [SportsCar] Loading 3D model... 200 MB of polygon data loaded.");
        Self {
            polygons: 450_000,
            textures: vec![
                "carbon_fiber".to_string(),
                "leather_black".to_string(),
                "aluminum_brushed".to_string(),
            ],
            materials: vec![
                "forged_carbon".to_string(),
                "alcantara".to_string(),
                "titanium".to_string(),
            ],
        }
    }
}
```

### RealSubject: SportsCar

```rust
struct SportsCar {
    vin: String,
    make: String,
    model: String,
    year: u16,
    horsepower: u16,
    engine_type: String,
    acceleration_0_60: f64,
    max_speed: f64,
    full_3d_model: OnceLock<ThreeDModel>,
    trunk_contents: Vec<String>,
    is_engine_running: bool,
    current_speed: f64,
}

impl SportsCar {
    fn new(vin: String, make: String, model: String, year: u16) -> Self {
        Self {
            vin,
            make,
            model,
            year,
            horsepower: 750,
            engine_type: "5.2L V10 Twin-Turbo".to_string(),
            acceleration_0_60: 2.8,
            max_speed: 200.0,
            full_3d_model: OnceLock::new(),
            trunk_contents: vec![
                "First aid kit".to_string(),
                "Tool set".to_string(),
                "Owners manual".to_string(),
            ],
            is_engine_running: false,
            current_speed: 0.0,
        }
    }
}

impl Car for SportsCar {
    fn vin(&self) -> &str { &self.vin }
    fn make(&self) -> &str { &self.make }
    fn model(&self) -> &str { &self.model }
    fn year(&self) -> u16 { self.year }

    fn short_description(&self) -> String {
        format!("{} {} {} ({})", self.year, self.make, self.model, self.vin)
    }

    fn display_details(&self) -> String {
        let model = self.full_3d_model.get_or_init(|| {
            println!("  [SportsCar] Loading 3D model for {}...", self.vin);
            ThreeDModel::new()
        });
        let trunk = self.trunk_contents.join(", ");
        format!(
            "--- {} ---\n\
             Engine: {}\n\
             Horsepower: {} hp\n\
             0-60 mph: {:.1}s\n\
             Top Speed: {:.1} mph\n\
             3D Model: {} polygons, {} textures\n\
             Trunk: {}",
            self.short_description(),
            self.engine_type,
            self.horsepower,
            self.acceleration_0_60,
            self.max_speed,
            model.polygons,
            model.textures.len(),
            trunk,
        )
    }

    fn drive(&self, speed: f64) -> String {
        let actual_speed = speed.min(self.max_speed);
        format!(
            "  [SportsCar] ROAR! V10 screams through the rev range.\n\
             Speed: {:.1} mph | Gear: 7th | RPM: 8200",
            actual_speed
        )
    }

    fn get_max_speed(&self) -> f64 { self.max_speed }

    fn access_trunk(&self) -> String {
        let contents = self.trunk_contents.join(", ");
        format!("  [SportsCar] Trunk opened. Contents: {}", contents)
    }
}
```

### Virtual Proxy: LazyCarModelProxy

```rust
struct LazyCarModelProxy {
    vin: String,
    make: String,
    model: String,
    year: u16,
    real_car: OnceLock<SportsCar>,
}

impl LazyCarModelProxy {
    fn new(vin: String, make: String, model: String, year: u16) -> Self {
        Self {
            vin,
            make,
            model,
            year,
            real_car: OnceLock::new(),
        }
    }

    fn get_real_car(&self) -> &SportsCar {
        self.real_car.get_or_init(|| {
            println!(
                "  [LazyCarModelProxy] Initializing real SportsCar for {}...",
                self.short_description_impl()
            );
            SportsCar::new(
                self.vin.clone(),
                self.make.clone(),
                self.model.clone(),
                self.year,
            )
        })
    }

    fn short_description_impl(&self) -> String {
        format!("{} {} {} ({})", self.year, self.make, self.model, self.vin)
    }
}

impl Car for LazyCarModelProxy {
    fn vin(&self) -> &str { &self.vin }
    fn make(&self) -> &str { &self.make }
    fn model(&self) -> &str { &self.model }
    fn year(&self) -> u16 { self.year }

    fn short_description(&self) -> String {
        self.short_description_impl()
    }

    fn display_details(&self) -> String {
        self.get_real_car().display_details()
    }

    fn drive(&self, speed: f64) -> String {
        self.get_real_car().drive(speed)
    }

    fn get_max_speed(&self) -> f64 {
        self.get_real_car().get_max_speed()
    }

    fn access_trunk(&self) -> String {
        self.get_real_car().access_trunk()
    }
}
```

### Protection Proxy: ValetCarProxy

```rust
struct ValetCarProxy<'a> {
    real_car: &'a dyn Car,
    max_valet_speed: f64,
}

impl<'a> ValetCarProxy<'a> {
    fn new(real_car: &'a dyn Car, max_valet_speed: f64) -> Self {
        Self {
            real_car,
            max_valet_speed,
        }
    }
}

impl<'a> Car for ValetCarProxy<'a> {
    fn vin(&self) -> &str { self.real_car.vin() }
    fn make(&self) -> &str { self.real_car.make() }
    fn model(&self) -> &str { self.real_car.model() }
    fn year(&self) -> u16 { self.real_car.year() }

    fn short_description(&self) -> String {
        self.real_car.short_description()
    }

    fn display_details(&self) -> String {
        format!(
            "--- Valet Card ---\n\
             VIN: {}\n\
             Max Permitted Speed: {:.1} mph\n\
             Trunk Access: DENIED\n\
             Glovebox Access: DENIED",
            self.vin(),
            self.max_valet_speed,
        )
    }

    fn drive(&self, speed: f64) -> String {
        let limited_speed = speed.min(self.max_valet_speed);
        if limited_speed < speed {
            println!(
                "  [ValetCarProxy] Speed limited to {:.1} mph.",
                self.max_valet_speed
            );
        }
        self.real_car.drive(limited_speed)
    }

    fn get_max_speed(&self) -> f64 { self.max_valet_speed }

    fn access_trunk(&self) -> String {
        "  [ValetCarProxy] ACCESS DENIED: Valet key does not permit trunk access."
            .to_string()
    }
}
```

### Main / Driver

```rust
fn main() {
    println!("{}", "=".repeat(72));
    println!("  PROXY PATTERN — Cars Domain (Rust 1.78+)");
    println!("{}", "=".repeat(72));
    println!();

    // ── Scenario 1: Virtual Proxy (Lazy Loading) ────────────────────────────
    println!("--- SCENARIO 1: Virtual Proxy (Lazy Loading) ---");
    let lazy_car = LazyCarModelProxy::new(
        generate_vin(), "Sypher".to_string(), "GT".to_string(), 2026,
    );
    println!(
        "  Showroom: Created LazyCarModelProxy. No 3D model loaded yet."
    );
    println!();

    println!("  Customer browses thumbnail:");
    println!("  {}", lazy_car.short_description());
    println!();

    println!("  Customer clicks \"View Full Details\":");
    let details = lazy_car.display_details();
    println!("{}", details);
    println!();

    // ── Scenario 2: Protection Proxy (Valet Key) ─────────────────────────────
    println!("--- SCENARIO 2: Protection Proxy (Valet Key) ---");
    let real_car = SportsCar::new(
        generate_vin(), "Sypher".to_string(), "GT-S".to_string(), 2026,
    );
    let valet_key = ValetCarProxy::new(&real_car, 30.0);
    println!();

    println!("  Owner hands the valet key to the parking attendant.");
    println!();

    println!("  Valet: Displays info card:");
    println!("{}", valet_key.display_details());
    println!();

    println!("  Valet: Attempts to drive at 80 mph:");
    println!("{}", valet_key.drive(80.0));
    println!();

    println!("  Valet: Attempts to access the trunk:");
    println!("{}", valet_key.access_trunk());
    println!();

    // ── Scenario 3: Owner Returns — Full Access ─────────────────────────────
    println!("--- SCENARIO 3: Owner Returns --- Full Access ---");
    println!("  Owner uses the real car key directly:");
    println!("{}", real_car.drive(120.0));
    println!("{}", real_car.access_trunk());
    println!();

    // ── Scenario 4: Polymorphic Iteration ────────────────────────────────────
    println!("--- SCENARIO 4: Polymorphic Car Display ---");
    let cars: [&dyn Car; 2] = [&lazy_car, &valet_key];
    for car in &cars {
        println!(
            "  [{}] Max Speed: {:.0} mph",
            car.vin(),
            car.get_max_speed()
        );
    }
    println!();

    // ── Scenario 5: Demonstrating Lazy Loading Benefit ──────────────────────
    println!("--- SCENARIO 5: Lazy Loading Benefit ---");
    let showroom_cars = vec![
        LazyCarModelProxy::new(
            generate_vin(), "Sypher".to_string(), "GT".to_string(), 2026,
        ),
        LazyCarModelProxy::new(
            generate_vin(), "Sypher".to_string(), "GT-S".to_string(), 2026,
        ),
        LazyCarModelProxy::new(
            generate_vin(), "Falcon".to_string(), "X1".to_string(), 2026,
        ),
    ];
    println!("  All 3 proxies created. Memory usage: minimal.");
    println!("  Showing thumbnails...");
    for car in &showroom_cars {
        println!("    {}", car.short_description());
    }
    println!();

    println!("  Customer examines one car in detail:");
    println!("{}", showroom_cars[0].display_details());
    println!();

    println!("{}", "=".repeat(72));
    println!("  Proxy pattern demonstration complete.");
    println!("{}", "=".repeat(72));
}
```

## Language-Specific Notes

- **`OnceLock` for thread-safe lazy initialization** — Rust's `std::sync::OnceLock` provides a thread-safe, one-time initialization primitive. The `LazyCarModelProxy` and the `SportsCar. full_3d_model` field both use `OnceLock` to ensure the expensive initialization runs at most once, even if multiple threads access the proxy concurrently. The `get_or_init()` method atomically checks and initializes, making this pattern safe in concurrent contexts without external locking.

- **Trait objects (`&dyn Car`) for runtime polymorphism** — Rust achieves runtime polymorphism through trait objects. The `Car` trait is used via `&dyn Car` references, which carry a vtable pointer alongside the data pointer. The `main()` function stores different proxy types in `[&dyn Car; 2]` arrays and iterates over them, calling methods through dynamic dispatch. This is Rust's equivalent of Java's interface references or C#'s interface variables.

- **`impl Trait for Struct` rather than class inheritance** — Rust does not have classes or inheritance in the OOP sense. Instead, any struct can implement any trait from any crate. The `SportsCar` struct implements the `Car` trait, and the proxy structs also implement `Car`. There is no shared base class — each type provides its own implementation of the trait methods. This satisfies the LSP because any `&dyn Car` can be substituted for any other.

- **`OnceLock::get_or_init()` for the proxy pattern** — The `LazyCarModelProxy` uses `get_or_init()` with a closure that creates the `SportsCar`. This is a more idiomatic Rust approach than the manual null-check pattern used in other languages. The closure captures the proxy's `vin`, `make`, `model`, and `year` by cloning them, which is required because the closure may outlive the proxy's borrow.

- **Lifetime annotations on `ValetCarProxy`** — The `ValetCarProxy` struct has a lifetime parameter `'a` because it holds a reference `&'a dyn Car` to the real subject. This ensures that the real car outlives the proxy. Rust's borrow checker enforces this at compile time, preventing dangling references that could lead to use-after-free bugs. In other languages, this would be a runtime concern (or a null pointer exception).

- **`use std::fmt::Write` for future extensibility** — The import of `std::fmt::Write` is included for potential use with `write!()` macro for building strings. While the current implementation uses `format!()` and direct string concatenation, the import signals that the pattern could be extended to build longer strings with the `write!()` macro for better performance in production code.
