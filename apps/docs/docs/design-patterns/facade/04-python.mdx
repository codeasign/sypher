## Implementation

Below is a complete Python implementation of the Facade pattern using a cars domain. The `CarFacade` class provides a clean, high-level API for starting, driving, and stopping a modern car. Six subsystem classes — `Engine`, `Transmission`, `CoolingSystem`, `ElectricalSystem`, `FuelSystem`, and `BrakingSystem` — each handle a specific mechanical domain. The facade composes these subsystems and orchestrates the correct startup sequence, freeing the caller from any knowledge of the car's internal wiring.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌───────────────────────────────────────────┐
//   │   CarFacade (Facade)                      │  ← OOP: Composition, Encapsulation
//   ├───────────────────────────────────────────┤
//   │   - _engine: Engine                       │
//   │   - _transmission: Transmission           │
//   │   - _cooling: CoolingSystem               │
//   │   - _electrical: ElectricalSystem         │
//   │   - _fuel_system: FuelSystem              │
//   │   - _brakes: BrakingSystem                │
//   ├───────────────────────────────────────────┤
//   │   + start_engine()                        │
//   │   + drive(speed: int)                     │
//   │   + stop_engine()                         │
//   └───────────────────────────────────────────┘
```

#### OOP Concepts Applied

1. **Encapsulation** — Subsystem attributes use the `_` prefix convention to indicate privacy. The facade hides the orchestration sequence behind three public methods.

2. **Composition** — `CarFacade` composes six subsystem objects in `__init__`. This is a "has-a" structural relationship: a car facade has an engine, a transmission, etc.

3. **Duck Typing** — Python does not require formal interfaces. Any object with `crank()`, `injectFuel()`, and `spark()` methods can serve as the engine. This is a form of implicit polymorphism.

#### SOLID Principles Applied

4. **SRP** — Each subsystem class manages exactly one domain. `Engine` handles combustion and RPM. `FuelSystem` manages fuel pressure. `CarFacade` handles orchestration only.

5. **OCP** — Adding a `Turbocharger` subsystem requires adding a new class and wiring it into the facade. The existing subsystems and client code remain unchanged.

6. **ISP** — Each subsystem's public methods are cohesive. `BrakingSystem` exposes only `release()`, `apply()`, and `is_engaged()`. No class depends on methods it does not use.

### Code

#### Subsystem: Engine

```python
class Engine:
    """Subsystem: manages engine cranking, fuel injection, ignition, and RPM."""

    def __init__(self) -> None:
        self._rpm: int = 0

    def crank(self) -> None:
        self._rpm = 250
        print("Engine: cranking at 250 RPM")

    def inject_fuel(self) -> None:
        print("Engine: injecting fuel into cylinders")

    def spark(self) -> None:
        print("Engine: firing spark plugs")

    def get_idle_rpm(self) -> int:
        return 850

    def get_rpm(self) -> int:
        return self._rpm

    def rev(self, amount: int) -> None:
        self._rpm += amount
        print(f"Engine: revving to {self._rpm} RPM")

    def stop(self) -> None:
        self._rpm = 0
        print("Engine: stopped")
```

#### Facade

```python
class CarFacade:
    """Facade: provides a simplified interface to the car's subsystems."""

    def __init__(self) -> None:
        self._engine = Engine()
        self._transmission = Transmission()
        self._cooling = CoolingSystem()
        self._electrical = ElectricalSystem()
        self._fuel_system = FuelSystem()
        self._brakes = BrakingSystem()

    def start_engine(self) -> None:
        """Orchestrate the full engine start sequence."""
        print("--- Starting Engine ---")
        voltage = self._electrical.get_battery_voltage()
        if voltage < 12.0:
            print("ERROR: Battery voltage too low.")
            return
        self._electrical.power_ecu()
        self._electrical.power_accessories()
        self._electrical.close_starter_relay()
        self._cooling.run_water_pump()
        self._fuel_system.prime(55)
        self._engine.crank()
        self._engine.inject_fuel()
        self._engine.spark()
        print(f"Engine started. Idle RPM: {self._engine.get_idle_rpm()}")
        self._cooling.run_radiator_fan()
        self._brakes.release()

    def drive(self, speed: int) -> None:
        """Engage the transmission and rev the engine to the target speed."""
        print(f"--- Driving at {speed} RPM ---")
        self._transmission.shift_to("Drive")
        self._engine.rev(speed)
        target_temp = 90.0
        self._cooling.set_thermostat(target_temp)
        print(f"Driving: cruising at {speed} RPM")

    def stop_engine(self) -> None:
        """Safely shut down all subsystems."""
        print("--- Stopping Engine ---")
        self._transmission.shift_to("Park")
        self._brakes.apply()
        self._engine.stop()
        self._fuel_system.shutoff()
        print("Engine stopped. All subsystems off.")
```

#### Main / Driver

```python
def main() -> None:
    """Driver: demonstrate the Facade pattern with a full start-drive-stop cycle."""
    car = CarFacade()
    car.start_engine()
    print()
    car.drive(2500)
    print()
    car.stop_engine()


if __name__ == "__main__":
    main()
```

### Language-Specific Notes

1. **Underscore naming convention** — Python uses `_` prefix for private attributes by convention rather than enforcement. The language does not prevent external access to `_engine`, but the convention signals that it is an implementation detail.

2. **Duck typing** — The facade does not require formal interfaces. Any object with `crank()`, `inject_fuel()`, `spark()`, `rev()`, `stop()`, and `get_idle_rpm()` methods can be injected as the engine. This provides flexibility at the cost of compile-time safety.

3. **Type hints** — The code includes type hints (`int`, `float`, `str`, `None`) for self-documentation and IDE support. These are optional at runtime but improve code quality significantly.

4. **`if __name__ == "__main__"` guard** — The driver code is protected by the standard Python idiom, allowing the module to be imported without executing the demo.

5. **No properties for simple accessors** — Python's `@property` decorator could replace `get_rpm()` and `get_idle_rpm()` with attribute-style access (`engine.rpm`). The explicit method style is used here for consistency with the other language implementations.

6. **`time.sleep` for realism** — Uncommenting `time.sleep(0.5)` between subsystem calls would simulate real mechanical delays. The code runs instantly as-is, but the sequential order is preserved.
