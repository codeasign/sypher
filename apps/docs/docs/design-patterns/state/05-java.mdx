The Java 17+ implementation of the State pattern models a stock market order lifecycle. Each order transitions through well-defined states — New, Submitted, PartiallyFilled, Filled, Cancelled, and Rejected — with the `Order` context delegating state-specific behavior to the current `OrderState` implementation. The sealed interface hierarchy enforces a closed set of valid states at compile time, while polymorphic dispatch handles each state's unique transition rules at runtime.

## How the Implementation Works

### Class Diagram (Pseudocode)

```text
sealed interface OrderState permits NewState, SubmittedState, PartiallyFilledState,
                                     FilledState, CancelledState, RejectedState {
    + handleSubmit(Order)
    + handleFill(Order, int, double)
    + handleCancel(Order)
    + handleReject(Order, String)
    + getStatusLabel(): String
    + getAllowedOperations(): List<String>
}

final class NewState            implements OrderState { /* submit -> SubmittedState, cancel -> CancelledState */ }
final class SubmittedState      implements OrderState { /* fill -> FilledState or PartiallyFilledState, cancel -> CancelledState, reject -> RejectedState */ }
final class PartiallyFilledState implements OrderState { /* fill -> FilledState, cancel -> CancelledState */ }
final class FilledState         implements OrderState { /* terminal -- all operations fail */ }
final class CancelledState      implements OrderState { /* terminal -- all operations fail */ }
final class RejectedState       implements OrderState { /* terminal -- all operations fail */ }

final class Order {               // Context
    - symbol: String
    - quantity: int
    - filledQuantity: int
    - price: double
    - state: OrderState
    + submit(): void              // delegates to state.handleSubmit(this)
    + fill(qty, price): void      // delegates to state.handleFill(this, qty, price)
    + cancel(): void              // delegates to state.handleCancel(this)
    + reject(reason): void        // delegates to state.handleReject(this, reason)
    + getStatus(): String         // delegates to state.getStatusLabel()
    + setState(OrderState): void  // transitions to new state
    + addFill(qty, price): void   // accumulates filled quantity
    + isFullyFilled(): boolean    // checks if filledQuantity >= quantity
}
```

### OOP Concepts Applied

1. **Abstraction** — `OrderState` is a sealed interface that defines the contract for all lifecycle states. The `Order` never references concrete state classes directly — it calls methods on the `OrderState` abstraction.

2. **Encapsulation** — Each state class encapsulates its own transition rules, validation logic, and error messages. The `Order` encapsulates its mutable state reference (`state`). No external code can change the state directly — only through `setState()`.

3. **Inheritance** — All six concrete state classes implement the `OrderState` interface, providing their own version of each state method. The `sealed` keyword restricts the inheritance hierarchy to exactly these six classes.

4. **Polymorphism** — The `Order` calls `state.handleSubmit(this)` and the correct implementation executes depending on which concrete state is currently active — without any `if`/`switch` checks on status strings.

### SOLID Principles Applied

5. **SRP** — `Order` manages only order data and state delegation. Each state class handles only the behavior for one specific lifecycle phase. No class mixes responsibilities.

6. **OCP** — Adding a new state (e.g., `PendingReviewState`) requires creating one new `final` class and adding it to the `permits` clause. No existing state classes need modification unless they transition to the new state.

7. **LSP** — Every concrete state is a valid substitute for `OrderState`. The `Order` calls the same interface methods regardless of which state is active, and every state provides correct behavior for its phase.

8. **ISP** — The `OrderState` interface defines exactly the methods a state needs: four lifecycle handlers, a label getter, and an operations query. No class implements irrelevant methods.

9. **DIP** — `Order` depends on the `OrderState` abstraction, not on concrete state classes. States are injected via `setState()` and are fully interchangeable without changing the `Order` class.

## Implementation

```java
import java.util.List;

sealed interface OrderState permits NewState, SubmittedState, PartiallyFilledState,
    FilledState, CancelledState, RejectedState {

    void handleSubmit(Order order);
    void handleFill(Order order, int quantity, double price);
    void handleCancel(Order order);
    void handleReject(Order order, String reason);
    String getStatusLabel();
    List<String> getAllowedOperations();
}
```

### NewState

```java
final class NewState implements OrderState {
    @Override
    public void handleSubmit(Order order) {
        System.out.println(order.getSymbol() + ": New -> Submitted");
        order.setState(new SubmittedState());
    }

    @Override
    public void handleFill(Order order, int quantity, double price) {
        System.out.println("ERROR: Cannot fill — order must be submitted first");
    }

    @Override
    public void handleCancel(Order order) {
        System.out.println(order.getSymbol() + ": New -> Cancelled");
        order.setState(new CancelledState());
    }

    @Override
    public void handleReject(Order order, String reason) {
        System.out.println("ERROR: Cannot reject — order is not yet submitted");
    }

    @Override
    public String getStatusLabel() { return "New"; }

    @Override
    public List<String> getAllowedOperations() { return List.of("submit", "cancel"); }
}
```

### SubmittedState

```java
final class SubmittedState implements OrderState {
    @Override
    public void handleSubmit(Order order) {
        System.out.println("ERROR: Cannot submit — order is already submitted");
    }

    @Override
    public void handleFill(Order order, int quantity, double price) {
        order.addFill(quantity, price);
        if (order.isFullyFilled()) {
            System.out.println(order.getSymbol() + ": Filled " + order.getFilledQuantity()
                + " shares at $" + String.format("%.2f", price));
            order.setState(new FilledState());
        } else {
            System.out.println(order.getSymbol() + ": Partially filled "
                + order.getFilledQuantity() + "/" + order.getQuantity()
                + " shares at $" + String.format("%.2f", price));
            order.setState(new PartiallyFilledState());
        }
    }

    @Override
    public void handleCancel(Order order) {
        System.out.println(order.getSymbol() + ": Submitted -> Cancelled");
        order.setState(new CancelledState());
    }

    @Override
    public void handleReject(Order order, String reason) {
        System.out.println(order.getSymbol() + ": Rejected — " + reason);
        order.setState(new RejectedState());
    }

    @Override
    public String getStatusLabel() { return "Submitted"; }

    @Override
    public List<String> getAllowedOperations() { return List.of("fill", "cancel", "reject"); }
}
```

### PartiallyFilledState

```java
final class PartiallyFilledState implements OrderState {
    @Override
    public void handleSubmit(Order order) {
        System.out.println("ERROR: Cannot submit — order is partially filled");
    }

    @Override
    public void handleFill(Order order, int quantity, double price) {
        order.addFill(quantity, price);

        } else {
            System.out.println(order.getSymbol() + ": Partially filled "
                + order.getFilledQuantity() + "/" + order.getQuantity()
                + " shares at $" + String.format("%.2f", price));
        }
    }

    @Override
    public void handleCancel(Order order) {
        int remaining = order.getQuantity() - order.getFilledQuantity();
        System.out.println(order.getSymbol() + ": Cancelled " + remaining + " remaining shares");
        order.setState(new CancelledState());
    }

    @Override
    public void handleReject(Order order, String reason) {
        System.out.println("ERROR: Cannot reject — order is partially filled");
    }

    @Override
    public String getStatusLabel() { return "Partially Filled"; }

    @Override
    public List<String> getAllowedOperations() { return List.of("fill", "cancel"); }
}
```

### Order (Context)

```java
final class Order {
    private final String symbol;
    private final int quantity;
    private int filledQuantity;
    private double price;
    private OrderState state;

    public Order(String symbol, int quantity, double price) {
        this.symbol = symbol;
        this.quantity = quantity;
        this.price = price;
        this.filledQuantity = 0;
        this.state = new NewState();
    }

    public void submit() { state.handleSubmit(this); }
    public void fill(int quantity, double price) { state.handleFill(this, quantity, price); }
    public void cancel() { state.handleCancel(this); }
    public void reject(String reason) { state.handleReject(this, reason); }
    public String getStatus() { return state.getStatusLabel(); }
    public List<String> getAllowedOperations() { return state.getAllowedOperations(); }
    public void setState(OrderState newState) { this.state = newState; }
    public void addFill(int qty, double price) { this.filledQuantity += qty; this.price = price; }
    public boolean isFullyFilled() { return filledQuantity >= quantity; }

    public String getSymbol() { return symbol; }
    public int getQuantity() { return quantity; }
    public int getFilledQuantity() { return filledQuantity; }
    public double getPrice() { return price; }
}
```

### Main / Driver

```java
public class StatePatternDemo {
    public static void main(String[] args) {
        System.out.println("--- Order 1: AAPL 100 @ $175.50 ---");
        Order order1 = new Order("AAPL", 100, 175.50);
        order1.submit();
        order1.fill(100, 175.50);
        order1.cancel();

        System.out.println("\n--- Order 2: GOOGL 50 @ $140.25 ---");
        Order order2 = new Order("GOOGL", 50, 140.25);
        order2.submit();
        order2.fill(20, 140.25);
        order2.cancel();

        System.out.println("\n--- Order 3: MSFT 75 @ $378.80 ---");
        Order order3 = new Order("MSFT", 75, 378.80);
        order3.submit();
        order3.reject("Insufficient funds");

        System.out.println("\n=== Final States ===");
        System.out.println("AAPL: " + order1.getStatus());
        System.out.println("GOOGL: " + order2.getStatus());
        System.out.println("MSFT: " + order3.getStatus());
    }
}
```

### Language-Specific Notes

- **`sealed interface`** — Java 17's sealed types restrict the `OrderState` hierarchy to exactly six permitted subclasses. Any attempt to create an unauthorized subclass produces a compile-time error, making the state machine's completeness verifiable at build time.

- **`final` concrete states** — Each state class is marked `final`, preventing accidental subclassing that could break the sealed hierarchy or introduce unexpected state transitions.

- **`List.of()` factory methods** — `getAllowedOperations()` returns immutable lists created with `List.of()`. Terminal states return `List.of()` (empty list), while active states return their valid operations. The immutability prevents accidental modification by callers.

- **Type-safe delegation** — Each `Order` method delegates to the current state via `state.handleXxx(this)`. Passing `this` (the `Order` reference) allows the state to read and mutate the context freely through its public accessor methods.

- **`String.format()` for price precision** — Financial prices are formatted to two decimal places using `String.format("%.2f", price)`, ensuring consistent output like `$175.50` regardless of floating-point representation.

- **Single-file demo** — All classes are defined in one compilation unit. Java 17 permits non-public top-level classes in a single file, keeping the pattern self-contained for learning while remaining fully compilable and runnable.
