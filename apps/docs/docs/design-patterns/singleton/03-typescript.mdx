## Implementation

Below is a complete TypeScript implementation of the Singleton pattern using a banking domain. The `BankConfiguration` class uses strict TypeScript features including private static fields, readonly properties, and typed getter methods. Three client classes — `LoanDepartment`, `TransactionService`, and `ComplianceSystem` — all access the same `BankConfiguration` instance, demonstrating how the singleton guarantees consistent configuration across the entire banking application.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌──────────────────────────────────────┐
//   │   BankConfiguration (Singleton)      │  ← OOP: Encapsulation
//   ├──────────────────────────────────────┤     SRP: holds config data
//   │   - #instance: BankConfiguration     │
//   │   - #config: BankConfigData          │
//   ├──────────────────────────────────────┤
//   │   + static getInstance(): BankConf   │  ← Global access point
//   │   + getBaseInterestRate(): number    │
//   │   + getSavingsInterestRate(): number │
//   │   + getWireTransferFee(): number     │
//   │   + getMaxTransactionLimit(): number │
//   │   + getMinBalanceRequirement(): number│
//   │   + getKycApiEndpoint(): string      │
//   │   + getBankName(): string            │
//   └──────────────────────────────────────┘
```

#### OOP Concepts Applied

1. **Encapsulation** — The `BankConfiguration` class hides all configuration data behind private fields and exposes only typed getter methods. The `BankConfigData` interface defines the shape of the configuration data, but the internal storage is private.

2. **Abstraction** — Clients depend on the public getter interface of `BankConfiguration`. They do not need to know how the configuration is loaded, stored, or whether the values come from a file, environment variables, or hardcoded defaults.

3. **Polymorphism** — Not directly applicable to the Singleton pattern, but the three client classes all interact with the singleton through the same typed interface, and TypeScript's type system ensures type safety.

#### SOLID Principles Applied

4. **SRP** — `BankConfiguration` is responsible only for holding and providing access to banking configuration. The `getInstance()` method handles the uniqueness constraint. Each client class has a single responsibility — loan processing, transaction handling, or compliance.

5. **OCP** — New configuration properties can be added to `BankConfiguration` without changing any client code. The interface can be extended with new getter methods without breaking existing callers.

6. **LSP** — While there is no inheritance hierarchy here, the principle applies to the typed interface: any code that depends on the `BankConfiguration` type can rely on consistent behavior.

7. **ISP** — The getter methods are individually small and focused. A client that only needs the interest rate does not depend on methods related to fees or endpoints.

8. **DIP** — High-level client classes depend on the `BankConfiguration` abstraction (through its public getter interface), not on the details of how configuration is loaded or stored.

### BankConfiguration (Singleton)

```typescript
// Singleton Pattern — TypeScript — Banking example
// ==================================================
// Participants: BankConfiguration (Singleton),
// LoanDepartment, TransactionService, ComplianceSystem (Clients)
// OOP: encapsulation, abstraction
// SOLID: SRP, OCP, ISP, DIP
// ==================================================

// ── Configuration data interface ──
interface BankConfigData {
  readonly baseInterestRate: number;
  readonly savingsInterestRate: number;
  readonly wireTransferFee: number;
  readonly maxTransactionLimit: number;
  readonly minBalanceRequirement: number;
  readonly kycApiEndpoint: string;
  readonly bankName: string;
  readonly swiftCode: string;
  readonly reserveRequirement: number;
}

// ── Singleton ──
class BankConfiguration {
  private static #instance: BankConfiguration | null = null;
  private readonly #config: BankConfigData;

  // Private constructor — prevents external instantiation
  private constructor() {
    if (BankConfiguration.#instance !== null) {
      throw new Error("Use BankConfiguration.getInstance() to access the singleton");
    }
    this.#config = {
      baseInterestRate: 4.5,
      savingsInterestRate: 3.75,
      wireTransferFee: 25.00,
      maxTransactionLimit: 50000.00,
      minBalanceRequirement: 500.00,
      kycApiEndpoint: "https://api.globaltrustbank.com/kyc/v2",
      bankName: "Global Trust Bank",
      swiftCode: "GTBKUS44",
      reserveRequirement: 0.10,
    };
  }

  // Public static method — global access point
  static getInstance(): BankConfiguration {
    if (BankConfiguration.#instance === null) {
      BankConfiguration.#instance = new BankConfiguration();
    }
    return BankConfiguration.#instance;
  }

  // Public getter methods — typed read-only access
  getBaseInterestRate(): number {
    return this.#config.baseInterestRate;
  }

  getSavingsInterestRate(): number {
    return this.#config.savingsInterestRate;
  }

  getWireTransferFee(): number {
    return this.#config.wireTransferFee;
  }

  getMaxTransactionLimit(): number {
    return this.#config.maxTransactionLimit;
  }

  getMinBalanceRequirement(): number {
    return this.#config.minBalanceRequirement;
  }

  getKycApiEndpoint(): string {
    return this.#config.kycApiEndpoint;
  }

  getBankName(): string {
    return this.#config.bankName;
  }

  getSwiftCode(): string {
    return this.#config.swiftCode;
  }

  getReserveRequirement(): number {
    return this.#config.reserveRequirement;
  }
}
```

### LoanDepartment (Client 1)

```typescript
// ── Client 1: LoanDepartment ──
class LoanDepartment {
  processLoanApplication(amount: number, creditScore: number): void {
    const config = BankConfiguration.getInstance();
    let rate: number = config.getBaseInterestRate();
    if (creditScore > 700) {
      rate = rate - 1.5;
    }
    const annualInterest: number = amount * (rate / 100);
    console.log(`LoanDepartment: Processing $${amount.toFixed(2)} loan application`);
    console.log(`  Interest rate: ${rate}%`);
    console.log(`  Annual interest: $${annualInterest.toFixed(2)}`);
    console.log(`  Bank: ${config.getBankName()}`);
    console.log(`  SWIFT: ${config.getSwiftCode()}`);
  }
}
```

### TransactionService (Client 2)

```typescript
// ── Client 2: TransactionService ──
class TransactionService {
  processWireTransfer(amount: number): void {
    const config = BankConfiguration.getInstance();
    if (amount > config.getMaxTransactionLimit()) {
      console.log(`TransactionService: REJECTED — $${amount.toFixed(2)} exceeds limit of $${config.getMaxTransactionLimit().toFixed(2)}`);
      return;
    }
    const fee: number = config.getWireTransferFee();
    const total: number = amount + fee;
    console.log(`TransactionService: Processing wire transfer of $${amount.toFixed(2)}`);
    console.log(`  Wire fee: $${fee.toFixed(2)}`);
    console.log(`  Total debit: $${total.toFixed(2)}`);
    console.log(`  Bank: ${config.getBankName()}`);
  }

  checkSavingsRate(): void {
    const config = BankConfiguration.getInstance();
    const rate: number = config.getSavingsInterestRate();
    console.log(`TransactionService: Current savings interest rate is ${rate}%`);
  }
}
```

### ComplianceSystem (Client 3)

```typescript
// ── Client 3: ComplianceSystem ──
class ComplianceSystem {
  verifyCustomer(customerId: string): void {
    const config = BankConfiguration.getInstance();
    const endpoint: string = config.getKycApiEndpoint();
    const minBalance: number = config.getMinBalanceRequirement();
    console.log(`ComplianceSystem: Verifying customer ${customerId}`);
    console.log(`  KYC endpoint: ${endpoint}`);
    console.log(`  Minimum balance requirement: $${minBalance.toFixed(2)}`);
    console.log(`  Reserve requirement: ${(config.getReserveRequirement() * 100)}%`);
  }
}
```

### Main / Driver

```typescript
// ── Main / Driver ──
function main(): void {
  console.log("=== Banking System Initialization ===\n");

  const loanDept = new LoanDepartment();
  const transactionService = new TransactionService();
  const compliance = new ComplianceSystem();

  loanDept.processLoanApplication(250000.00, 720);
  console.log();
  transactionService.processWireTransfer(15000.00);
  console.log();
  transactionService.checkSavingsRate();
  console.log();
  compliance.verifyCustomer("CUST-4201");
  console.log();

  // Singleton identity verification
  console.log("=== Singleton Identity Verification ===");
  const ref1 = BankConfiguration.getInstance();
  const ref2 = BankConfiguration.getInstance();
  const ref3 = BankConfiguration.getInstance();
  console.log(`ref1 === ref2: ${ref1 === ref2}`);
  console.log(`ref2 === ref3: ${ref2 === ref3}`);
  console.log(`All references point to the same instance: ${(ref1 === ref2) && (ref2 === ref3)}`);
}

main();
```

### Language-Specific Notes

- **Private static fields** — TypeScript 3.8+ supports ECMAScript private fields with `#` prefix. The `#instance` field is a private static field accessible only within the class. Unlike TypeScript's `private` keyword, `#` provides true runtime privacy that cannot be bypassed.

- **Readonly interface** — The `BankConfigData` interface uses `readonly` modifiers on all properties, ensuring the configuration data cannot be mutated after construction. This aligns with the immutable-configuration pattern recommended for singletons.

- **Strict null checks** — The `#instance` field is typed as `BankConfiguration | null`, requiring explicit null checks in `getInstance()`. This is enforced by TypeScript's `strict` compiler mode.

- **Type safety** — All getter methods have explicit return types, providing compile-time guarantees that clients receive the correct types. The TypeScript compiler catches type mismatches before runtime.

- **No thread-safety** — Like JavaScript, standard TypeScript runs in a single-threaded event loop, so the simple null-check pattern is safe. For Node.js Worker Threads, a synchronization mechanism would be needed.
