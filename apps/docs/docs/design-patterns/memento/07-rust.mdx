The Memento pattern captures and externalizes an object's internal state so that the object can be restored to that state later without violating encapsulation. In Rust, the pattern leverages module-level privacy to enforce the memento's opacity: the `PortfolioSnapshot` struct is defined in the same module as the `Portfolio` originator, so `Portfolio` can read its fields directly, but those fields are not `pub`, preventing the caretaker or any external code from inspecting snapshot internals. This aligns with Rust's principle of zero-cost abstractions while maintaining strict encapsulation through the module system rather than runtime checks.

## How the Implementation Works

### Class Diagram (Pseudocode)

The diagram above shows the Memento pattern structure in Rust. The `Portfolio` originator holds a `HashMap<String, Holding>` for ticker positions and an `f64 cash_balance`. Before each state-changing operation, the caller invokes `portfolio.save()` which performs a deep clone of the internal data into a `PortfolioSnapshot` memento. The `PortfolioHistory` caretaker stores these snapshots in a `Vec` used as a stack, providing `push`, `pop`, and `can_undo` operations. When an undo is requested, the caretaker pops the most recent snapshot and passes it to `portfolio.restore()`, which overwrites the portfolio's internal fields from the snapshot.

### OOP Concepts Applied

- **Encapsulation**: `PortfolioSnapshot` fields are not `pub`, so only `Portfolio` (defined in the same module) can read them. The caretaker never inspects memento contents.
- **Polymorphism through trait bounds**: While not present in this minimal example, the pattern could be extended with a `Snapshot` trait that exposes narrow metadata (timestamp, label) while keeping state fields hidden.
- **Composition**: `Portfolio` composes `HashMap<String, Holding>` and `f64` for its state, and `PortfolioHistory` composes `Vec<PortfolioSnapshot>` for its stack, reflecting Rust's composition-oriented approach.
- **Move semantics for safe state transfer**: The `pop` method returns `Option<PortfolioSnapshot>` by value, transferring ownership of the snapshot out of the history stack.

### SOLID Principles Applied

- **Single Responsibility Principle**: `Portfolio` handles business logic (buy/sell), `PortfolioSnapshot` holds captured state, and `PortfolioHistory` manages the undo stack. Each struct has exactly one reason to change.
- **Open/Closed Principle**: New state fields can be added to `Portfolio` without modifying the caretaker or the snapshot storage mechanism, as long as `Clone` is derived.
- **Liskov Substitution Principle**: Any code that works with `PortfolioSnapshot` works uniformly with all snapshots regardless of when or from which `Portfolio` they were captured.
- **Interface Segregation Principle**: The caretaker interacts with snapshots only through `push`/`pop`/`can_undo`, never through state inspection. The memento exposes no interface to the caretaker beyond opaque storage.
- **Dependency Inversion Principle**: The caretaker depends on the abstract concept of a memento (storing and returning opaque `PortfolioSnapshot` values), not on the portfolio's internal state representation.

### Portfolio (Originator)

```rust
/// Originator — the portfolio whose state is saved and restored.
pub struct Portfolio {
    holdings: HashMap<String, Holding>,
    cash_balance: f64,
}

impl Portfolio {
    /// Create a new portfolio with the given initial cash balance.
    pub fn new(initial_cash: f64) -> Self {
        Portfolio {
            holdings: HashMap::new(),
            cash_balance: initial_cash,
        }
    }

    /// Buy shares of a stock at a given price per share. Deducts the cost
    /// from the cash balance and updates the average cost basis for the
    /// ticker's holding.
    pub fn buy_ticker(&mut self, symbol: &str, shares: i32, price: f64) {
        let cost = shares as f64 * price;
        self.cash_balance -= cost;

        let holding = self.holdings
            .entry(symbol.to_string())
            .or_insert(Holding {
                symbol: symbol.to_string(),
                shares: 0,
                average_cost: 0.0,
            });

        let total_cost = holding.average_cost * holding.shares as f64 + cost;
        holding.shares += shares;
        holding.average_cost = total_cost / holding.shares as f64;
    }

    /// Sell shares of a stock. Calculates proceeds using the average cost
    /// basis, adds proceeds to cash, and reduces or removes the holding.
    /// Returns an error if the symbol is not held or if insufficient shares
    /// are available.
    pub fn sell_ticker(&mut self, symbol: &str, shares: i32) -> Result<(), String> {
        match self.holdings.get_mut(symbol) {
            Some(holding) => {
                if holding.shares < shares {
                    return Err(format!(
                        "Not enough shares of {} to sell. Held: {}, requested: {}",
                        symbol, holding.shares, shares
                    ));
                }
                let proceeds = shares as f64 * holding.average_cost;
                self.cash_balance += proceeds;
                holding.shares -= shares;
                if holding.shares == 0 {
                    self.holdings.remove(symbol);
                }
                Ok(())
            }
            None => Err(format!("No holdings found for ticker symbol: {}", symbol)),
        }
    }

    /// Capture a snapshot of the portfolio's current internal state by
    /// performing a deep copy of the holdings map and cash balance.
    pub fn save(&self) -> PortfolioSnapshot {
        PortfolioSnapshot {
            holdings: self.holdings.clone(),
            cash_balance: self.cash_balance,
        }
    }

    /// Restore the portfolio's state from a previously saved snapshot.
    /// This overwrites the current holdings and cash balance with the
    /// values stored in the snapshot.
    pub fn restore(&mut self, snapshot: &PortfolioSnapshot) {
        self.holdings = snapshot.holdings.clone();
        self.cash_balance = snapshot.cash_balance;
    }

    /// Print the portfolio's current cash balance and all holdings to stdout.
    pub fn print_state(&self) {
        println!("Cash Balance: ${:.2}", self.cash_balance);
        if self.holdings.is_empty() {
            println!("  (no holdings)");
        } else {
            let mut symbols: Vec<&String> = self.holdings.keys().collect();
            symbols.sort();
            for symbol in symbols {
                let h = &self.holdings[symbol];
                let market_value = h.shares as f64 * h.average_cost;
                println!(
                    "  {}: {} shares @ ${:.2}/share (value: ${:.2})",
                    h.symbol, h.shares, h.average_cost, market_value
                );
            }
        }
        println!();
    }
}
```

### PortfolioHistory (Caretaker)

```rust
/// Caretaker — manages the undo history of portfolio snapshots. Snapshots
/// are stored in a Vec used as a stack, with the most recent snapshot at
/// the end.
pub struct PortfolioHistory {
    snapshots: Vec<PortfolioSnapshot>,
}

impl PortfolioHistory {
    /// Create an empty history with no snapshots.
    pub fn new() -> Self {
        PortfolioHistory {
            snapshots: Vec::new(),
        }
    }

    /// Push a snapshot onto the top of the history stack.
    pub fn push(&mut self, snapshot: PortfolioSnapshot) {
        self.snapshots.push(snapshot);
    }

    /// Pop and return the most recent snapshot, if any. Returns None if
    /// the history stack is empty.
    pub fn pop(&mut self) -> Option<PortfolioSnapshot> {
        self.snapshots.pop()
    }

    /// Returns true if there is at least one snapshot available for undo.
    pub fn can_undo(&self) -> bool {
        !self.snapshots.is_empty()
    }
}
```

### Main / Driver

```rust
fn main() {
    let mut portfolio = Portfolio::new(1_000_000.0);
    let mut history = PortfolioHistory::new();

    println!("=== Initial Portfolio ===");
    portfolio.print_state();

    // --- Trade 1: Buy 100 AAPL at $150 ---
    history.push(portfolio.save());
    portfolio.buy_ticker("AAPL", 100, 150.0);
    println!("=== After Buying 100 AAPL @ $150 ===");
    portfolio.print_state();

    // --- Trade 2: Buy 50 GOOGL at $2,800 ---
    history.push(portfolio.save());
    portfolio.buy_ticker("GOOGL", 50, 2800.0);
    println!("=== After Buying 50 GOOGL @ $2,800 ===");
    portfolio.print_state();

    // --- Trade 3: Buy 200 MSFT at $330 ---
    history.push(portfolio.save());
    portfolio.buy_ticker("MSFT", 200, 330.0);
    println!("=== After Buying 200 MSFT @ $330 ===");
    portfolio.print_state();

    // --- Undo: revert the MSFT buy, restoring the state from before trade 3 ---
    if let Some(snapshot) = history.pop() {
        portfolio.restore(&snapshot);
    }
    println!("=== After Undoing the MSFT Buy ===");
    portfolio.print_state();

    // --- Undo: revert the GOOGL buy, restoring the state from before trade 2 ---
    if let Some(snapshot) = history.pop() {
        portfolio.restore(&snapshot);
    }
    println!("=== After Undoing the GOOGL Buy ===");
    portfolio.print_state();

    // Show whether additional undo operations are available
    println!("Can undo further: {}", history.can_undo());
}
```

### Language-Specific Notes

- **Module-level privacy for encapsulation**: Rust has no `private` or `protected` access modifiers for cross-struct visibility. Instead, the `PortfolioSnapshot` struct is defined in the same module as `Portfolio`, and its fields lack the `pub` keyword. This means only code within the same module (including `Portfolio`) can read or write those fields, while `PortfolioHistory` and all external callers cannot access them. This achieves the same encapsulation as nested-class approaches in Java or C\# without runtime overhead.

- **`Clone` derive for snapshotting**: The `save()` method relies on the `Clone` derive macro to perform a deep copy of the `HashMap<String, Holding>`. This is a zero-cost abstraction: the compiler generates an efficient field-by-field copy, and any new fields added to `Holding` or `Portfolio` automatically participate in cloning as long as they implement `Clone`.

- **`Result` for fallible operations without exceptions**: The `sell_ticker` method returns `Result<(), String>` instead of throwing an exception. Callers handle errors with `match` or `if let`, and the `?` operator can propagate errors upward. This is idiomatic Rust and eliminates the need for try/catch blocks or unchecked runtime panics.

- **`Option` for safe pop operations**: The `pop()` method returns `Option<PortfolioSnapshot>` rather than throwing on an empty stack. The caller must handle both `Some(snapshot)` and `None` cases, typically with `if let Some(snapshot) = history.pop()`. This makes empty-stack errors a compile-time consideration rather than a runtime surprise.

- **`Vec` as an efficient stack**: The `PortfolioHistory` caretaker uses `Vec<PortfolioSnapshot>` as its underlying stack. `Vec::push` and `Vec::pop` are amortized O(1) operations, and `Vec` provides cache-friendly contiguous storage. The `can_undo()` method simply checks `!self.snapshots.is_empty()`, which is O(1).

- **No inheritance -- all composition**: Unlike class-based languages where the originator might extend a base `Stateful` class, Rust achieves reuse entirely through struct composition. `Portfolio` owns its `HashMap` and `f64` fields directly, and `PortfolioSnapshot` mirrors them. There is no shared base class, no virtual method dispatch, and no runtime overhead for state capture and restoration.
