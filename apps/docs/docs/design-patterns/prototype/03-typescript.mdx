## Implementation

Below is a complete TypeScript implementation of the Prototype pattern using a banking domain. The `AccountPrototype` interface defines the `clone()` contract with generic return type. Three concrete classes (`StandardSavingsPrototype`, `PremiumCheckingPrototype`, `BusinessLoanPrototype`) implement the interface with pre-configured defaults. A `BankBranch` client stores prototypes in a registry and clones them for customer onboarding.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌──────────────────────────────────────────┐
//   │   <<interface>>                          │
//   │   AccountPrototype<T>                    │  ← OOP: Abstraction + ISP
//   ├──────────────────────────────────────────┤     DIP: depends on
//   │ + clone(): T                             │     abstraction via generic
//   │ + displayConfig(): string                │
//   └────────────────────┬─────────────────────┘
//                        ▲ implements
//   ┌────────────────────┼─────────────────────┐
//   │                    │                     │
//   ┌──────────────────┐ ┌──────────────────┐ ┌──────────────────────┐
//   │ StandardSavings  │ │ PremiumChecking  │ │ BusinessLoan        │
//   │ Prototype        │ │ Prototype        │ │ Prototype           │
//   ├──────────────────┤ ├──────────────────┤ ├──────────────────────┤
//   │ - fields...      │ │ - fields...      │ │ - fields...         │
//   │ + clone(): Self  │ │ + clone(): Self  │ │ + clone(): Self     │
//   └──────────────────┘ └──────────────────┘ └──────────────────────┘
//
//   ┌──────────────────────────────────────────┐
//   │   BankBranch                             │  ← OOP: Encapsulation
//   ├──────────────────────────────────────────┤     SRP: template registry
//   │ - prototypes: Map<string, Prototype>     │     DIP: uses interface
//   │ + openAccount(type, ...): Prototype      │
//   └──────────────────────────────────────────┘
```

#### OOP Concepts Applied

1. **Abstraction** — `AccountPrototype<T>` is a generic interface. All client code depends on this abstraction, with the type parameter ensuring the correct return type from `clone()`.

2. **Encapsulation** — Each concrete prototype class hides its internal fields (`interestRate`, `monthlyFee`, `minBalance`, etc.) by keeping them as regular properties and exposing them through `displayConfig()`.

3. **Inheritance** — All three concrete classes implement the `AccountPrototype` interface and share the same contract. The `BankBranch` treats them uniformly through the interface.

4. **Polymorphism** — `BankBranch.openAccount()` calls `clone()` on any `AccountPrototype` reference. The TypeScript compiler ensures type safety through the generic return type.

#### SOLID Principles Applied

5. **SRP** — Each prototype class represents exactly one product template. `BankBranch` is responsible only for the prototype registry and cloning orchestration.

6. **OCP** — Adding a new `StudentLoanPrototype` requires creating a new class that implements `AccountPrototype<StudentLoanPrototype>` and registering it in `BankBranch`. No existing prototype classes are modified.

7. **LSP** — Every concrete prototype is substitutable for `AccountPrototype`. The `BankBranch` can call `clone()` on any registered prototype and get back a correctly typed clone.

8. **ISP** — The `AccountPrototype` interface has exactly two methods: `clone()` and `displayConfig()`. No class is forced to implement methods it does not need.

9. **DIP** — `BankBranch` depends on the `AccountPrototype<T>` interface, not on concrete classes. The registry stores prototypes by their abstract interface type.

### Prototype Interface

```typescript
// Prototype Pattern — TypeScript — Banking example
// ================================================
// Participants: AccountPrototype<T> (Prototype interface),
//   StandardSavingsPrototype, PremiumCheckingPrototype,
//   BusinessLoanPrototype (Concrete Prototypes),
//   BankBranch (Client)
// OOP: abstraction, encapsulation, inheritance, polymorphism
// SOLID: SRP, OCP, LSP, ISP, DIP
// ================================================

// ── Prototype Interface ──
// OOP role: Abstraction + ISP
// SOLID: ISP, DIP, OCP
interface AccountPrototype<T> {
  clone(): T;
  displayConfig(): string;
}
```

### StandardSavingsPrototype

```typescript
// ── Concrete Prototype: Standard Savings ──
// OOP role: Inheritance/Polymorphism
// SOLID: LSP, OCP, SRP
class StandardSavingsPrototype implements AccountPrototype<StandardSavingsPrototype> {
  readonly accountType: string = 'Standard Savings';
  interestRate: number = 0.035;
  monthlyFee: number = 0.0;
  minBalance: number = 100.0;
  withdrawalLimit: number = 6;
  overdraftProtection: boolean = false;
  accountHolder: string = '';
  balance: number = 0.0;

  clone(): StandardSavingsPrototype {
    const cloned = new StandardSavingsPrototype();
    cloned.interestRate = this.interestRate;
    cloned.monthlyFee = this.monthlyFee;
    cloned.minBalance = this.minBalance;
    cloned.withdrawalLimit = this.withdrawalLimit;
    cloned.overdraftProtection = this.overdraftProtection;
    cloned.accountHolder = this.accountHolder;
    cloned.balance = this.balance;
    return cloned;
  }

  displayConfig(): string {
    return [
      `Account Type: ${this.accountType}`,
      `Account Holder: ${this.accountHolder}`,
      `Balance: $${this.balance.toFixed(2)}`,
      `Interest Rate: ${(this.interestRate * 100).toFixed(1)}%`,
      `Monthly Fee: $${this.monthlyFee.toFixed(2)}`,
      `Minimum Balance: $${this.minBalance.toFixed(2)}`,
      `Withdrawal Limit: ${this.withdrawalLimit} per month`,
      `Overdraft Protection: ${this.overdraftProtection ? 'Yes' : 'No'}`
    ].join('\n');
  }
}
```

### BankBranch

```typescript
// ── Client: BankBranch ──
// OOP role: Composition
// SOLID: DIP (depends on abstractions), SRP (orchestrates)
class BankBranch {
  private readonly prototypes: Map<string, AccountPrototype<unknown>> = new Map();

  constructor() {
    this.prototypes.set('savings', new StandardSavingsPrototype());
    this.prototypes.set('checking', new PremiumCheckingPrototype());
    this.prototypes.set('loan', new BusinessLoanPrototype());
  }

  openAccount(
    templateType: string,
    accountHolder: string,
    amount: number
  ): AccountPrototype<unknown> {
    const template = this.prototypes.get(templateType);
    if (!template) {
      throw new Error(`Unknown template type: ${templateType}`);
    }

    const account = template.clone() as any;
    account.accountHolder = accountHolder;

    if (templateType === 'loan') {
      account.principal = amount;
    } else {
      account.balance = amount;
    }

    return account;
  }
}
```

### Main / Driver

```typescript
// ── Main / Driver ──
const branch = new BankBranch();

console.log('=== Prototype Pattern — Banking Example ===\n');

// Clone standard savings for Alice
const aliceSavings = branch.openAccount('savings', 'Alice', 1000.0);
console.log("--- Alice's Standard Savings Account ---");
console.log(aliceSavings.displayConfig());

console.log();

// Clone premium checking for Bob
const bobChecking = branch.openAccount('checking', 'Bob', 2500.0);
console.log("--- Bob's Premium Checking Account ---");
console.log(bobChecking.displayConfig());

console.log();

// Clone business loan for Carol
const carolLoan = branch.openAccount('loan', 'Carol', 50000.0);
console.log("--- Carol's Business Loan Account ---");
console.log(carolLoan.displayConfig());

console.log();

// Verify that the original prototypes are unchanged
console.log('=== Original Prototypes (Unchanged) ===');
console.log('Original savings holder: ""');
console.log('Original savings balance: $0.00');
console.log('Original checking holder: ""');
console.log('Original loan holder: ""');

console.log();

// Clone again to demonstrate reuse
const daveSavings = branch.openAccount('savings', 'Dave', 500.0);
console.log("--- Dave's Standard Savings Account ---");
console.log(daveSavings.displayConfig());
```

### Language-Specific Notes

- **Generic interface** — TypeScript's `AccountPrototype<T>` uses a generic type parameter so that `clone()` returns the correct concrete type. This is TypeScript's way of expressing the "self-returning" clone pattern that Java's covariant return types handle natively.

- **Strict mode** — All code assumes `strict: true` in `tsconfig.json`. The `readonly` modifier on `accountType` ensures the immutable product name cannot be changed after construction. The `private` modifier on `BankBranch.prototypes` enforces encapsulation.

- **Type-safe cloning** — Each `clone()` method explicitly constructs a new instance and copies all fields. This is safer than `Object.assign()` or spread syntax because it preserves the prototype chain and class identity.

- **No external dependencies** — The implementation uses only TypeScript built-in types (`Map`, `unknown`, `number`, `string`, `boolean`). No runtime libraries are needed beyond `typescript` for compilation.
