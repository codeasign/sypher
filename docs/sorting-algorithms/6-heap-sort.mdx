## Summary

Heap Sort imagines the list as a pyramid shape, called a heap, where every "parent" number is bigger than its two "children" numbers below it. Because of that rule, the single biggest number always ends up at the very top of the pyramid. Heap Sort keeps pulling the top (biggest) number off, putting it at the end of the sorted list, and then patching the pyramid back into shape — over and over until nothing is left. It's dependably medium-fast no matter what order the numbers started in.

## Pseudocode

```text
1. Arrange the numbers into a pyramid shape where every parent is bigger than its two children — this is called building the heap.
2. Take the number at the very top of the pyramid (it's always the biggest one left) and swap it with the last number still in the pyramid.
3. Set that last spot aside — it now holds a correctly placed, sorted number, and shrink the pyramid by one.
4. Fix the pyramid: let the number that just moved to the top sink down by swapping with its biggest child, repeating until every parent is bigger than its children again.
5. Repeat steps 2-4 until the pyramid has no numbers left in it — the list is now fully sorted from smallest to biggest.
```

## How It Works

- Look at the pyramid in the diagram: `9` sits on top because every parent number is bigger than the numbers below it. That's the whole point of a heap — the biggest number always floats to the top.
- Heap Sort takes that top number (`9`), moves it to the end of the list where sorted numbers live, and then re-shapes what's left of the pyramid so a new biggest number rises to the top.
- It repeats that "grab the top, fix the pyramid" step again and again. Each time, one more number gets locked into its correct sorted position, working backward from the end of the list.
- Heap Sort is "in-place," meaning it rearranges numbers within the same list you gave it — it doesn't need to build a second list to work.
- Heap Sort is not "stable." If two equal numbers are in the list, this method doesn't guarantee they'll end up in the same order they started in.
- How fast is it? Heap Sort takes roughly the same amount of time no matter how the list starts out — sorted, reversed, or shuffled, it's always about the same effort to build the pyramid and keep fixing it (this is `O(n log n)` time in every case, with best and worst case barely differing).
