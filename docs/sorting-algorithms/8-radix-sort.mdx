## Summary

Radix Sort sorts numbers by looking at one digit at a time instead of comparing whole numbers to each other. It starts with the ones digit and sorts the whole list by just that digit, then moves to the tens digit and sorts again, then the hundreds digit, and so on, until it's worked through every digit of the biggest number. Each of those digit-by-digit sorting passes uses the counting trick from Counting Sort. It stays fast and predictable, but the more digits the numbers have, the more passes it needs to make.

## Pseudocode

```text
1. If the list is empty, stop — there's nothing to sort.
2. Find the biggest number in the list, so you know how many digits you'll need to work through.
3. Look at the ones digit of every number, and sort the whole list based on just that digit (using the counting trick from Counting Sort).
4. Look at the tens digit of every number, and sort the whole list based on just that digit.
5. Keep moving one digit to the left each time — hundreds, thousands, and so on — sorting the whole list by that digit each pass.
6. Stop once you've processed the last digit of the biggest number — the list is now fully sorted.
```

## How It Works

- Radix Sort never compares two full numbers side by side — instead, it counts how many numbers have each digit value, one digit position at a time, the same way Counting Sort does.
- In the diagram, pass 1 sorts the numbers by their ones digit: `170` and `90` both end in `0`, so they come first; `45` and `75` both end in `5`, so they come after.
- Pass 2 then sorts by the tens digit, and pass 3 sorts by the hundreds digit. By the time every digit has had its turn, the whole list ends up fully sorted.
- Radix Sort is not "in-place" — each digit pass needs some extra space to build the newly-sorted order before moving to the next digit.
- Radix Sort is "stable," which matters a lot here: each pass must keep equal-digit numbers in the same relative order they already had, so the ordering built up by earlier (smaller-digit) passes doesn't get scrambled by later ones.
- How fast is it? Radix Sort takes roughly the same amount of time no matter how the list starts out, because it always makes one pass per digit regardless of order (this is `O(d × n)` time, where `d` is the number of digits in the biggest number). The more digits the numbers have, the more passes it needs — so Radix Sort works best when the numbers don't have too many digits.
