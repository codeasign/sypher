## Summary

Insertion Sort builds a sorted list one number at a time, the same way you'd sort playing cards in your hand. It picks up the next number and slides it left past any bigger numbers already in the sorted part, until it lands in the right spot. It's fast when the list is already close to sorted, but it slows down the more out of order the list is.

## Pseudocode

```text
1. Treat the first number as a sorted list of one.
2. Pick up the next number in line — call it the "key".
3. Compare the key to the number just before it in the sorted part.
4. If that number is bigger than the key, slide it one spot to the right.
5. Keep sliding numbers right, moving one spot left each time, as long as they're bigger than the key.
6. Once you hit a number that's smaller than the key (or you reach the start of the list), drop the key into that empty spot.
7. Repeat steps 2-6 for every remaining number in the list.
```

## How It Works

- Look at the diagram: the sorted part so far is `[1, 5, 8]`, and the next number to insert is `3`.
- Compare `3` to `8` — `8` is bigger, so slide `8` one spot to the right to make room.
- Compare `3` to `5` — `5` is bigger too, so slide it right as well.
- Compare `3` to `1` — `1` is smaller than `3`, so stop sliding and drop `3` into the empty spot right after `1`. The sorted part is now `[1, 3, 5, 8]`, one number longer than before.
- It doesn't need extra space to work — it rearranges the same list in place instead of building a new one.
- If two numbers are equal, Insertion Sort stops sliding as soon as it meets one, so it never moves an equal number out of its original order.
- If the list is already sorted, each new number only needs one quick check before it stays put — that's fast (this is called O(n) time). If the list is in reverse order, every new number has to slide all the way past everything already placed, which is much slower (this is called O(n²) time).
