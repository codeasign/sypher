## Summary

Bubble Sort works by comparing two numbers that are sitting right next to each other and swapping them if they're in the wrong order. It walks down the whole list doing this over and over, and each pass pushes the biggest number left in the unsorted part all the way to the end — like a bubble floating up to the top. It's one of the easiest sorting methods to picture, but it's slow on big lists because it may need a lot of passes and comparisons to finish.

## Pseudocode

```text
1. Look at the first two numbers in the list.
2. If the left one is bigger than the right one, swap them.
3. Move one step to the right and repeat step 2 with the next pair.
4. Keep going until you reach the end of the list — that's one full pass.
5. If you made at least one swap during that pass, do another pass from the start.
6. If you made zero swaps during a pass, the list is already sorted — stop.
```

## How It Works

- Start with `[5, 3, 8, 1]`. Look at the first pair, `5` and `3`. Since `5` is bigger than `3`, swap them: the list becomes `[3, 5, 8, 1]`.
- Move to the next pair, `5` and `8`. `5` is smaller, so leave them alone.
- Move to the last pair, `8` and `1`. `8` is bigger, so swap them: the list becomes `[3, 5, 1, 8]`.
- That's one full pass — notice the biggest number, `8`, has now bubbled all the way to the end. The next pass repeats the same neighbor-by-neighbor comparisons on what's left, so the second-biggest number ends up in place, and so on.
- It doesn't need extra space to work — it rearranges the same list in place instead of building a new one.
- If two numbers are equal, Bubble Sort never swaps them past each other, so it leaves them in the order they started in.
- If the list is already sorted, one pass finds no swaps and it stops right away — that's fast. If the list is in the worst possible order (reverse sorted), it has to compare and swap almost everything against everything else (this is called O(n²) time).
