Python's duck typing and abstract base classes make the Open/Closed Principle natural to implement. With the `abc` module for formal interfaces and `Protocol` for structural subtyping, Python offers multiple strategies for defining the abstraction boundary.

## The Domain: Notification Sender System

An application needs to send notifications through multiple channels: email, SMS, and push notifications. Each channel has a completely different delivery mechanism. New channels are added as the platform expands.

### The Violation

The following example shows a `send_notification` function that branches on a channel string. Every new notification channel requires editing this function.

```python
# VIOLATION: send_notification uses if-else on channel type.
# Adding a new channel (e.g., Slack, Telegram) means editing this
# function and retesting all existing channels.

from dataclasses import dataclass
from typing import Any


@dataclass
class Notification:
    recipient: str
    subject: str
    body: str
    priority: str = "normal"  # low, normal, high, urgent


# VIOLATION: Monolithic send function. Every new channel type adds a new
# elif branch. A mistake in any branch risks breaking all notifications.
def send_notification(notification: Notification, channel: str) -> dict[str, Any]:
    # VIOLATION: branching on channel string -- each new channel means adding
    # an elif and retesting every existing channel.
    if channel == "email":
        # 15 lines of email sending logic
        print(f"[EMAIL] Connecting to SMTP server...")
        print(f"[EMAIL] From: noreply@example.com")
        print(f"[EMAIL] To: {notification.recipient}")
        print(f"[EMAIL] Subject: {notification.subject}")
        print(f"[EMAIL] Body: {notification.body[:50]}...")
        print(f"[EMAIL] Priority: {notification.priority}")
        if notification.priority == "urgent":
            print(f"[EMAIL] Marking as high importance")
        print(f"[EMAIL] Email sent successfully")
        return {"status": "sent", "channel": "email", "id": "msg_abc123"}

    elif channel == "sms":
        # 10 lines of SMS sending logic
        print(f"[SMS] Connecting to SMS gateway...")
        print(f"[SMS] To: {notification.recipient}")
        # SMS has no subject, concatenate with body
        message = f"{notification.subject}: {notification.body}"
        sms_body = message[:160]  # Truncate to SMS length
        print(f"[SMS] Message: {sms_body}")
        print(f"[SMS] SMS sent successfully")
        return {"status": "sent", "channel": "sms", "id": "sms_def456"}

    elif channel == "push":
        # 12 lines of push notification logic
        print(f"[PUSH] Connecting to push notification service...")
        print(f"[PUSH] Device token: {notification.recipient}")
        print(f"[PUSH] Title: {notification.subject}")
        print(f"[PUSH] Body: {notification.body[:80]}...")
        print(f"[PUSH] Priority: {notification.priority}")
        print(f"[PUSH] Push notification sent")
        return {"status": "sent", "channel": "push", "id": "push_ghi789"}

    else:
        raise ValueError(f"Unknown notification channel: {channel}")


# Testing
note = Notification(
    recipient="user@example.com",
    subject="Welcome!",
    body="Thank you for joining our platform. We are excited to have you aboard."
)

print("=== Email ===")
result_email = send_notification(note, "email")
print(f"Result: {result_email}\n")

print("=== SMS ===")
result_sms = send_notification(note, "sms")
print(f"Result: {result_sms}\n")

print("=== Push ===")
result_push = send_notification(note, "push")
print(f"Result: {result_push}\n")
```

### The Correct Implementation: Abstract Base Class

The solution uses an abstract base class (ABC) to define the notification sender contract. Each channel implements the ABC. A registry maps channel identifiers to sender instances.

```python
# CORRECT: Abstract base class defines the sender contract. Each channel
# is a separate implementation. The notification service is closed.

from abc import ABC, abstractmethod
from dataclasses import dataclass, field
from typing import Any, Protocol


@dataclass
class Notification:
    recipient: str
    subject: str
    body: str
    priority: str = "normal"


@dataclass
class SendResult:
    status: str
    channel: str
    message_id: str
    details: dict[str, Any] = field(default_factory=dict)


# --- Abstract base class ---
class NotificationSender(ABC):
    """Interface contract for all notification senders."""

    @property
    @abstractmethod
    def channel_name(self) -> str:
        """Unique identifier for this channel."""
        ...

    @abstractmethod
    def send(self, notification: Notification) -> SendResult:
        """Send the notification and return the result."""
        ...

    def validate(self, notification: Notification) -> None:
        """Optional pre-flight validation. Subclasses can override."""
        if not notification.recipient:
            raise ValueError("Recipient is required")


# --- Concrete implementations ---

class EmailSender(NotificationSender):
    @property
    def channel_name(self) -> str:
        return "email"

    def send(self, notification: Notification) -> SendResult:
        print(f"[EMAIL] Connecting to SMTP server...")
        print(f"[EMAIL] From: noreply@example.com")
        print(f"[EMAIL] To: {notification.recipient}")
        print(f"[EMAIL] Subject: {notification.subject}")
        print(f"[EMAIL] Body: {notification.body[:50]}...")
        print(f"[EMAIL] Priority: {notification.priority}")
        if notification.priority == "urgent":
            print(f"[EMAIL] Marking as high importance")
        print(f"[EMAIL] Email sent successfully")

        return SendResult(
            status="sent",
            channel="email",
            message_id="msg_abc123",
            details={"smtp_server": "smtp.example.com"}
        )


class SmsSender(NotificationSender):
    @property
    def channel_name(self) -> str:
        return "sms"

    def send(self, notification: Notification) -> SendResult:
        print(f"[SMS] Connecting to SMS gateway...")
        print(f"[SMS] To: {notification.recipient}")
        message = f"{notification.subject}: {notification.body}"
        sms_body = message[:160]
        print(f"[SMS] Message: {sms_body}")
        print(f"[SMS] SMS sent successfully")

        return SendResult(
            status="sent",
            channel="sms",
            message_id="sms_def456",
            details={"segments": 1}
        )


class PushNotificationSender(NotificationSender):
    @property
    def channel_name(self) -> str:
        return "push"

    def send(self, notification: Notification) -> SendResult:
        print(f"[PUSH] Connecting to push notification service...")
        print(f"[PUSH] Device token: {notification.recipient}")
        print(f"[PUSH] Title: {notification.subject}")
        print(f"[PUSH] Body: {notification.body[:80]}...")
        print(f"[PUSH] Priority: {notification.priority}")
        print(f"[PUSH] Push notification sent")

        return SendResult(
            status="sent",
            channel="push",
            message_id="push_ghi789",
            details={"provider": "firebase"}
        )


# --- Registry ---
class NotificationRegistry:
    """Maps channel names to sender instances."""

    def __init__(self):
        self._senders: dict[str, NotificationSender] = {}

    def register(self, sender: NotificationSender) -> None:
        self._senders[sender.channel_name] = sender

    def get(self, channel: str) -> NotificationSender:
        sender = self._senders.get(channel)
        if sender is None:
            raise ValueError(f"Unknown channel: {channel}")
        return sender

    @property
    def available_channels(self) -> list[str]:
        return list(self._senders.keys())


# --- Core service -- closed for modification ---
class NotificationService:
    """Sends notifications through registered channels. Closed for
    modification -- new channels arrive as new sender implementations."""

    def __init__(self, registry: NotificationRegistry):
        self._registry = registry

    def send(self, notification: Notification, channel: str) -> SendResult:
        sender = self._registry.get(channel)
        sender.validate(notification)
        return sender.send(notification)

    def send_to_all(self, notification: Notification) -> dict[str, SendResult]:
        """Send notification through every registered channel."""
        results: dict[str, SendResult] = {}
        for channel in self._registry.available_channels:
            results[channel] = self.send(notification, channel)
        return results

    def broadcast(
        self,
        notification: Notification,
        channels: list[str]
    ) -> dict[str, SendResult]:
        """Send notification to a subset of channels."""
        results: dict[str, SendResult] = {}
        for channel in channels:
            results[channel] = self.send(notification, channel)
        return results


# --- Duck-typing with Protocol (alternative to ABC) ---
class SenderProtocol(Protocol):
    """Structural subtyping: any object with send() is a sender."""

    def send(self, notification: Notification) -> SendResult: ...


def send_with_protocol(sender: SenderProtocol, notification: Notification) -> SendResult:
    """This function works with any object that has a matching send() method,
    no inheritance required."""
    return sender.send(notification)


# --- Setup ---
registry = NotificationRegistry()
registry.register(EmailSender())
registry.register(SmsSender())
registry.register(PushNotificationSender())

service = NotificationService(registry)

# --- Usage ---
note = Notification(
    recipient="user@example.com",
    subject="Welcome!",
    body="Thank you for joining our platform. We are excited to have you aboard."
)

print("=== Single channel ===")
result = service.send(note, "email")
print(f"Result: {result}\n")

print("=== All channels ===")
results = service.send_to_all(note)
for channel, res in results.items():
    print(f"  {channel}: {res.message_id} ({res.status})")

# --- Adding a new channel without modifying existing code ---
#
# New requirement: Slack integration. We create a new sender class
# and register it. No existing files change.

class SlackSender(NotificationSender):
    @property
    def channel_name(self) -> str:
        return "slack"

    def send(self, notification: Notification) -> SendResult:
        print(f"[SLACK] Posting to Slack channel #{notification.recipient}")
        print(f"[SLACK] Message: *{notification.subject}*")
        print(f"[SLACK] {notification.body[:100]}")
        print(f"[SLACK] Slack message posted")

        return SendResult(
            status="sent",
            channel="slack",
            message_id="slack_jkl012",
            details={"workspace": "acme-corp"}
        )


registry.register(SlackSender())
print(f"\n=== Slack (new, no edits to existing code) ===")
slack_result = service.send(note, "slack")
print(f"Result: {slack_result}")

# Using Protocol duck-typing
print(f"\n=== Protocol-based dispatch ===")
send_with_protocol(SlackSender(), note)
```

## Key Takeaways

- **Abstract Base Classes** (`abc.ABC` with `@abstractmethod`) provide a formal interface contract. Subclasses must implement all abstract methods or Python raises `TypeError` on instantiation.
- **`@property` with `@abstractmethod`** enforces that each sender provides a unique `channel_name`. The registry uses this to dispatch.
- **Optional `validate` method** in the base class demonstrates that ABCs can include default behavior that subclasses override.
- **`Protocol` for structural subtyping** (PEP 544) allows duck-typed dispatch without inheritance. Any object with a matching `send()` method satisfies the protocol.
- **The registry** is a simple `dict[str, NotificationSender]`. In larger applications this could be a plugin loader that scans a package for sender classes.
- **Adding a sender** is purely additive: subclass `NotificationSender`, implement the abstract methods, and register an instance. `NotificationService` never changes.
- **`SendResult` dataclass** provides a consistent return type across all senders. The `details` dict allows channel-specific metadata without breaking the interface.
