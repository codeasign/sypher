TypeScript's static type system provides strong contracts for implementing the Open/Closed Principle. Interfaces define the abstraction boundary, union types support exhaustive pattern matching in the registry, and the `never` type ensures completeness.

## The Domain: Report Exporter System

A business intelligence platform exports reports in multiple formats. New export formats are requested frequently by clients. The core export pipeline must remain stable.

### The Violation

The following code violates OCP by branching on a format string inside a single `exportReport` function.

```typescript
// VIOLATION: exportReport uses if-else branching on format type.
// Every new format requires editing this function, risking regressions
// across all existing formats.

type ReportFormat = "pdf" | "csv" | "json" | "html";

interface Report {
  title: string;
  headers: string[];
  rows: Record<string, string | number>[];
  generatedAt: Date;
}

// VIOLATION: Monolithic export function. Adding "xml" or "xlsx" means
// editing this function and retesting every existing branch.
function exportReport(report: Report, format: ReportFormat): string {
  // VIOLATION: branching on enum-like union -- each new format adds a branch
  if (format === "pdf") {
    // 20 lines of PDF generation
    let output = `%PDF-1.4\n1 0 obj\n<< /Type /Catalog >>\nendobj\n`;
    output += `Title: ${report.title}\n`;
    for (const row of report.rows) {
      output += Object.values(row).join(", ") + "\n";
    }
    return output;
  } else if (format === "csv") {
    // 10 lines of CSV generation
    const headerLine = report.headers.join(",");
    const rows = report.rows.map((r) =>
      report.headers.map((h) => String(r[h] ?? "")).join(",")
    );
    return [headerLine, ...rows].join("\n");
  } else if (format === "json") {
    // 8 lines of JSON generation
    return JSON.stringify(report, null, 2);
  } else if (format === "html") {
    // 15 lines of HTML generation
    let html = `<!DOCTYPE html><html><head><title>${report.title}</title></head><body>`;
    html += `<h1>${report.title}</h1><table><thead><tr>`;
    for (const header of report.headers) {
      html += `<th>${header}</th>`;
    }
    html += `</tr></thead><tbody>`;
    for (const row of report.rows) {
      html += `<tr>${report.headers.map((h) => `<td>${row[h] ?? ""}</td>`).join("")}</tr>`;
    }
    html += `</tbody></table></body></html>`;
    return html;
  }

  throw new Error(`Unsupported format: ${format}`);
}

// Usage
const salesReport: Report = {
  title: "Q4 Sales",
  headers: ["Product", "Units", "Revenue"],
  rows: [
    { Product: "Widget", Units: 120, Revenue: 2400 },
    { Product: "Gadget", Units: 85, Revenue: 4250 },
  ],
  generatedAt: new Date(),
};

console.log("=== PDF ===");
console.log(exportReport(salesReport, "pdf"));
console.log("\n=== CSV ===");
console.log(exportReport(salesReport, "csv"));
console.log("\n=== JSON ===");
console.log(exportReport(salesReport, "json"));
```

### The Correct Implementation: Strategy with Interface

The fix defines a `ReportExporter` interface. Each format implements the interface. A registry maps format identifiers to exporters. Adding a new format means creating a new class -- no changes to existing code.

```typescript
// CORRECT: ReportExporter interface defines the contract. Each format is
// a separate implementation. The export pipeline is closed for modification.

// --- Interface contract ---
interface ReportExporter {
  readonly format: string;             // Unique format identifier
  export(report: Report): string;      // Produce the formatted output
  mimeType: string;                    // Content-Type for downloads
}

// --- Concrete implementations ---

class PdfExporter implements ReportExporter {
  readonly format = "pdf";
  readonly mimeType = "application/pdf";

  export(report: Report): string {
    let output = `%PDF-1.4\n1 0 obj\n<< /Type /Catalog >>\nendobj\n`;
    output += `Title: ${report.title}\nDate: ${report.generatedAt.toISOString()}\n`;
    for (const row of report.rows) {
      output += Object.values(row).join(", ") + "\n";
    }
    return output;
  }
}

class CsvExporter implements ReportExporter {
  readonly format = "csv";
  readonly mimeType = "text/csv";

  export(report: Report): string {
    const headerLine = report.headers.join(",");
    const rows = report.rows.map((r) =>
      report.headers.map((h) => String(r[h] ?? "")).join(",")
    );
    return [headerLine, ...rows].join("\n");
  }
}

class JsonExporter implements ReportExporter {
  readonly format = "json";
  readonly mimeType = "application/json";

  export(report: Report): string {
    return JSON.stringify(report, null, 2);
  }
}

class HtmlExporter implements ReportExporter {
  readonly format = "html";
  readonly mimeType = "text/html";

  export(report: Report): string {
    let html = `<!DOCTYPE html><html><head><title>${report.title}</title></head><body>`;
    html += `<h1>${report.title}</h1><p>Generated: ${report.generatedAt.toISOString()}</p>`;
    html += `<table><thead><tr>`;
    for (const header of report.headers) {
      html += `<th>${header}</th>`;
    }
    html += `</tr></thead><tbody>`;
    for (const row of report.rows) {
      html += `<tr>${report.headers.map((h) => `<td>${row[h] ?? ""}</td>`).join("")}</tr>`;
    }
    html += `</tbody></table></body></html>`;
    return html;
  }
}

// --- Registry with type-safe lookup ---
class ExporterRegistry {
  private exporters = new Map<string, ReportExporter>();

  register(exporter: ReportExporter): void {
    this.exporters.set(exporter.format, exporter);
  }

  get(format: string): ReportExporter {
    const exporter = this.exporters.get(format);
    if (!exporter) {
      throw new Error(`No exporter registered for format: ${format}`);
    }
    return exporter;
  }

  /** Returns all supported format identifiers */
  getFormats(): string[] {
    return Array.from(this.exporters.keys());
  }
}

// --- Generic constraint: the exporter type can be constrained ---
function exportWithExporter<T extends ReportExporter>(
  report: Report,
  exporter: T
): string {
  return exporter.export(report);
}

// --- Core pipeline -- closed for modification ---
class ReportPipeline {
  constructor(private registry: ExporterRegistry) {}

  export(report: Report, format: string): string {
    const exporter = this.registry.get(format);
    return exporter.export(report);
  }

  listSupportedFormats(): string[] {
    return this.registry.getFormats();
  }

  /**
   * Export to all registered formats, returning a map of format -> content.
   * This method never changes when new formats are added.
   */
  exportToAll(report: Report): Map<string, string> {
    const results = new Map<string, string>();
    for (const format of this.registry.getFormats()) {
      results.set(format, this.export(report, format));
    }
    return results;
  }
}

// --- Setup ---
const registry = new ExporterRegistry();
registry.register(new PdfExporter());
registry.register(new CsvExporter());
registry.register(new JsonExporter());
registry.register(new HtmlExporter());

const pipeline = new ReportPipeline(registry);

// --- Usage ---
const report: Report = {
  title: "Q4 Sales",
  headers: ["Product", "Units", "Revenue"],
  rows: [
    { Product: "Widget", Units: 120, Revenue: 2400 },
    { Product: "Gadget", Units: 85, Revenue: 4250 },
  ],
  generatedAt: new Date("2026-07-05"),
};

console.log("=== CSV Export ===");
console.log(pipeline.export(report, "csv"));

console.log("\n=== All Formats ===");
const allExports = pipeline.exportToAll(report);
for (const [format, content] of allExports) {
  console.log(`\n--- ${format.toUpperCase()} (${content.length} chars) ---`);
  console.log(content.substring(0, 120) + "...");
}

// --- Adding XML exporter without modifying existing code ---
//
// New requirement: XML export for legacy system integration.
// We create a new class and register it. No existing files change.

class XmlExporter implements ReportExporter {
  readonly format = "xml";
  readonly mimeType = "application/xml";

  export(report: Report): string {
    let xml = `<?xml version="1.0" encoding="UTF-8"?>\n`;
    xml += `<report title="${this.escapeXml(report.title)}">\n`;
    xml += `  <generated-at>${report.generatedAt.toISOString()}</generated-at>\n`;
    xml += `  <rows>\n`;
    for (const row of report.rows) {
      xml += `    <row>\n`;
      for (const header of report.headers) {
        const value = row[header] ?? "";
        xml += `      <${header}>${this.escapeXml(String(value))}</${header}>\n`;
      }
      xml += `    </row>\n`;
    }
    xml += `  </rows>\n`;
    xml += `</report>`;
    return xml;
  }

  private escapeXml(str: string): string {
    return str
      .replace(/&/g, "&amp;")
      .replace(/</g, "&lt;")
      .replace(/>/g, "&gt;")
      .replace(/"/g, "&quot;")
      .replace(/'/g, "&apos;");
  }
}

registry.register(new XmlExporter());
console.log("\n=== XML Export (new, no edits to existing code) ===");
console.log(pipeline.export(report, "xml"));

// --- Using the never type for exhaustiveness checking ---
// If you prefer discriminated union dispatch, you can combine it with
// the never type to ensure all variants are handled at compile time:

type ExportFormat = "pdf" | "csv" | "json" | "html" | "xml";

function assertNever(value: never): never {
  throw new Error(`Unhandled variant: ${value}`);
}

// This function uses the registry at runtime but the type system
// to enforce exhaustive format support:
function exportWithExhaustiveCheck(report: Report, format: ExportFormat): string {
  switch (format) {
    case "pdf":  return new PdfExporter().export(report);
    case "csv":  return new CsvExporter().export(report);
    case "json": return new JsonExporter().export(report);
    case "html": return new HtmlExporter().export(report);
    case "xml":  return new XmlExporter().export(report);
    default:     return assertNever(format); // compile error if a variant is missing
  }
}
```

## Key Takeaways

- **Interface contracts** define the shape every exporter must satisfy, providing compile-time safety that the pipeline's expectations are met.
- **Generic constraints** (`T extends ReportExporter`) let you write type-safe utility functions like `exportWithExporter`.
- **Union types** for the registry keys (`string`) keep the registry flexible. The type system does not enforce exhaustiveness on the registry -- only runtime registration matters. If you want compile-time exhaustiveness, use a discriminated union switch with `never`.
- **The `never` type** in the `assertNever` helper ensures that if you add a new format to the `ExportFormat` union but forget to handle it in the switch, TypeScript produces a compile error.
- **Adding an exporter** is purely additive: write a new class that implements `ReportExporter`, instantiate it, and register it. The `ReportPipeline` and all existing exporters remain untouched.
- **`mimeType` on the interface** demonstrates that interfaces can carry metadata beyond the core method, enabling richer behavior (download headers, content negotiation) without coupling to concrete types.
