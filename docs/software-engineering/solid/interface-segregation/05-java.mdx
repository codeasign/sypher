## Implementation

A Java 17+ implementation of ISP using a document management system. The violation shows a `DocumentHandler` interface with `open()`, `save()`, `print()`, and `preview()` — `ReadOnlyDocument` must implement `save()` even though it cannot write.

```java
// ISP — Java 17+ implementation
// Domain: Document management

// ============================================================
// VIOLATION: Fat DocumentHandler interface forces every document
// to implement save() and print() even if read-only
// ============================================================
//
// interface DocumentHandler {
//     String open();
//     String save();
//     String print();
//     String preview();
// }
//
// class ReadOnlyDocument implements DocumentHandler {
//     public String open() { return "Opening read-only document"; }
//     public String save() { throw new UnsupportedOperationException("Read-only"); }
//     public String print() { return "Printing document"; }
//     public String preview() { return "Previewing document"; }
// }

// ============================================================
// CORRECT IMPLEMENTATION — Segregated interfaces
// ============================================================

import java.util.List;

interface Openable {
    String open();
}

interface Saveable {
    String save();
}

interface Printable {
    String print();
}

interface Previewable {
    String preview();
}

// Only implements what it needs
record ReadOnlyDocument(String name, String content) implements Openable, Printable, Previewable {
    @Override
    public String open() {
        return "Opened read-only: " + name;
    }

    @Override
    public String print() {
        return "Printing: " + name + " (" + content.length() + " chars)";
    }

    @Override
    public String preview() {
        return "Preview: " + content.substring(0, Math.min(50, content.length())) + "...";
    }
}

// Implements all capabilities
class EditableDocument implements Openable, Saveable, Printable, Previewable {
    private final String name;
    private String content;

    public EditableDocument(String name, String content) {
        this.name = name;
        this.content = content;
    }

    @Override
    public String open() {
        return "Opened editable: " + name;
    }

    @Override
    public String save() {
        return "Saved: " + name + " (" + content.length() + " chars)";
    }

    public void edit(String newContent) {
        this.content = newContent;
    }

    @Override
    public String print() {
        return "Printing: " + name + " (" + content.length() + " chars)";
    }

    @Override
    public String preview() {
        return "Preview: " + content.substring(0, Math.min(50, content.length())) + "...";
    }
}

// --- Main driver ---
public class InterfaceSegregationDemo {
    public static void main(String[] args) {
        var readOnly = new ReadOnlyDocument("report.pdf", "Quarterly financial results for Q3 2026");
        var editable = new EditableDocument("notes.txt", "Meeting notes: discussed project timeline");

        System.out.println("=== Read-Only Document ===");
        System.out.println(readOnly.open());
        System.out.println(readOnly.preview());
        System.out.println(readOnly.print());
        // readOnly.save();  // Not available — correct!

        System.out.println("\n=== Editable Document ===");
        System.out.println(editable.open());
        editable.edit("Updated: project timeline confirmed");
        System.out.println(editable.save());
        System.out.println(editable.print());
    }
}
```

Expected output:

```bash

Preview: Quarterly financial results for Q3 2026...
Printing: report.pdf (47 chars)

=== Editable Document ===

Saved: notes.txt (40 chars)
Printing: notes.txt (40 chars)
```

## Key Language-Specific Notes

- **Small interfaces with one method** — Each interface (`Openable`, `Saveable`, `Printable`, `Previewable`) has exactly one abstract method, following ISP strictly.
- **`record` classes** (Java 16+) — `ReadOnlyDocument` is a `record` because it is immutable and its interface set (no `Saveable`) reflects that immutability at the type level.
- **Interface implementation as capability declaration** — Implementing `Saveable` is a declaration that the document can be written. The absence of `Saveable` on `ReadOnlyDocument` is a compile-time guarantee.
- **No `UnsupportedOperationException`** — The correct version has zero throwing stubs. Every implemented method is a real implementation.
