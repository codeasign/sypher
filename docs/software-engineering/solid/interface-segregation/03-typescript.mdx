## Implementation

A strict TypeScript implementation of ISP using a media processing system. The violation shows a `MediaPlayer` interface with `playAudio()`, `playVideo()`, and `showSubtitles()` — `AudioPlayer` must implement `playVideo()` even though it cannot play video.

```typescript
// ISP — TypeScript (strict mode) implementation
// Domain: Media processing

// ============================================================
// VIOLATION: Fat MediaPlayer interface forces AudioPlayer to
// implement video-related methods it cannot use
// ============================================================
//
// interface MediaPlayer {
//   playAudio(): string;
//   playVideo(): string;
//   showSubtitles(language: string): string;
// }
//
// class AudioPlayer implements MediaPlayer {
//   playAudio(): string { return "Playing audio track"; }
//   playVideo(): string { throw new Error("Audio-only device"); }
//   showSubtitles(_language: string): string { throw new Error("No display"); }
// }

// ============================================================
// CORRECT IMPLEMENTATION — Segregated interfaces
// ============================================================

interface AudioPlayable {
  playAudio(): string;
}

interface VideoPlayable {
  playVideo(): string;
}

interface Subtitlable {
  showSubtitles(language: string): string;
}

class AudioPlayer implements AudioPlayable {
  playAudio(): string {
    return "Playing audio track: stereo 320kbps";
  }
}

class VideoPlayer implements VideoPlayable, AudioPlayable, Subtitlable {
  playVideo(): string {
    return "Playing video: 1080p H.264";
  }

  playAudio(): string {
    return "Playing audio track: surround 5.1";
  }

  showSubtitles(language: string): string {
    return `Showing subtitles in ${language}`;
  }
}

class StreamingService {
  playAudioOnly(player: AudioPlayable): string {
    return player.playAudio();
  }

  playFullMedia(player: VideoPlayable & AudioPlayable & Subtitlable): string[] {
    return [
      player.playVideo(),
      player.playAudio(),
      player.showSubtitles("English")
    ];
  }
}

// --- Main driver ---
function main(): void {
  const audioPlayer = new AudioPlayer();
  const videoPlayer = new VideoPlayer();
  const service = new StreamingService();

  console.log("=== Audio Only ===");
  console.log(service.playAudioOnly(audioPlayer));
  console.log(service.playAudioOnly(videoPlayer));

  console.log("\n=== Full Media ===");
  service.playFullMedia(videoPlayer).forEach(msg => console.log(msg));

  console.log("\n=== AudioPlayer cannot play video ===");
  // audioPlayer.playVideo();  // TypeScript error: Property 'playVideo' does not exist
  console.log("TypeScript prevents this at compile time!");
}

main();
```

Showing subtitles in English

TypeScript prevents this at compile time!

## Key Language-Specific Notes

- **Small, focused interfaces** — `AudioPlayable`, `VideoPlayable`, and `Subtitlable` each have exactly one method. This is the canonical ISP pattern in TypeScript.
- **Intersection types** (`VideoPlayable & AudioPlayable & Subtitlable`) — TypeScript can compose segregated interfaces at the usage site without creating a third merged interface.
- **Compile-time safety** — Calling `playVideo()` on an `AudioPlayer` is a TypeScript error, not a runtime exception. ISP violations are caught before deployment.
- **`implements` multiple interfaces** — `VideoPlayer implements VideoPlayable, AudioPlayable, Subtitlable` shows how classes can compose exactly the capabilities they support.
- **Structural typing** (duck typing at the type level) — Any object with the right method shape satisfies an interface, even without explicit `implements`.
