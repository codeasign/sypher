## DIP in Rust

Rust implements the Dependency Inversion Principle through its trait system. Traits define the abstraction, and types implement them. Dependencies can be injected either via generics with trait bounds (static dispatch, zero-cost) or via trait objects with `dyn` (dynamic dispatch). This example uses both approaches.

This example builds a **data cache system**. A high-level `CacheService` stores and retrieves cached data. Without the DIP, it would be coupled to a specific caching backend (Redis). With the DIP, it depends on a `Cache` trait that any backend can implement.

## The Violation

In the violating version, `CacheService` directly creates a `RedisCache` instance and calls its methods. Switching from Redis to an in-memory cache or Memcached requires editing `CacheService`.

```rust
// VIOLATION: CacheService depends on concrete RedisCache.
// Changing the cache backend requires modifying CacheService.
// Testing requires a running Redis server.

use std::collections::HashMap;
use std::time::{Duration, Instant};

// ── Concrete Redis client simulation ──

struct RedisCache {
    connection_string: String,
    pool_size: u32,
}

impl RedisCache {
    fn new(connection_string: &str, pool_size: u32) -> Self {
        println!(
            "[RedisCache] Connecting to {} with pool size {}",
            connection_string, pool_size
        );
        RedisCache {
            connection_string: connection_string.to_string(),
            pool_size,
        }
    }

    fn get(&self, key: &str) -> Option<String> {
        println!("[RedisCache] GET {}", key);
        // In production, query Redis:
        // redis::cmd("GET").arg(key).query(&mut conn).ok()
        None
    }

    fn set(&self, key: &str, value: &str, ttl_seconds: u64) {
        println!(
            "[RedisCache] SET {} = {} (TTL: {}s)",
            key, value, ttl_seconds
        );
        // In production, write to Redis:
        // redis::cmd("SET")
        //     .arg(&[key, value])
        //     .arg(&["EX", &ttl_seconds.to_string()])
        //     .query(&mut conn)
        //     .unwrap();
    }

    fn delete(&self, key: &str) {
        println!("[RedisCache] DEL {}", key);
        // In production: redis::cmd("DEL").arg(key).query(&mut conn).ok();
    }
}

// ── Data types ──

#[derive(Debug, Clone)]
struct UserProfile {
    user_id: String,
    name: String,
    email: String,
    preferences: HashMap<String, String>,
}

// ── High-level module ──

struct CacheService {
    cache: RedisCache,
    default_ttl: u64,
}

impl CacheService {
    fn new() -> Self {
        // VIOLATION: Direct instantiation of concrete RedisCache.
        CacheService {
            cache: RedisCache::new("redis://localhost:6379", 10),
            default_ttl: 3600,
        }
    }

    fn get_user_profile(&self, user_id: &str) -> Option<UserProfile> {
        let key = format!("user:profile:{}", user_id);
        let cached = self.cache.get(&key);
        // Deserialize and return (simplified)
        cached.map(|json_str| {
            println!("[CacheService] Deserializing profile for {}", user_id);
            UserProfile {
                user_id: user_id.to_string(),
                name: "placeholder".to_string(),
                email: "placeholder@example.com".to_string(),
                preferences: HashMap::new(),
            }
        })
    }

    fn set_user_profile(&self, profile: &UserProfile) {
        let key = format!("user:profile:{}", profile.user_id);
        let json_str = format!(
            r#"{{"user_id":"{}","name":"{}","email":"{}"}}"#,
            profile.user_id, profile.name, profile.email
        );
        self.cache.set(&key, &json_str, self.default_ttl);
    }

    fn invalidate_user(&self, user_id: &str) {
        let key = format!("user:profile:{}", user_id);
        self.cache.delete(&key);
    }
}

// Usage
fn main() {
    let service = CacheService::new();

    let profile = UserProfile {
        user_id: "u-100".to_string(),
        name: "Alice".to_string(),
        email: "alice@example.com".to_string(),
        preferences: HashMap::new(),
    };

    service.set_user_profile(&profile);

    match service.get_user_profile("u-100") {
        Some(p) => println!("Found profile for {}", p.name),
        None => println!("Profile not found (cache miss)"),
    }

    service.invalidate_user("u-100");

    // PROBLEM: Cannot switch to in-memory cache or Memcached.
    // PROBLEM: Unit tests require Redis to be running.
}
```

## The Correct Implementation

We define a `Cache` trait. `RedisCache` and `InMemoryCache` implement it. `CacheService` receives its dependency through the constructor, using generics with trait bounds for zero-cost abstraction.

```rust
// CORRECT: CacheService depends on Cache trait.
// Multiple cache backends can be swapped without changing CacheService.

use std::collections::HashMap;
use std::time::{Duration, Instant};

// ── Abstractions ──

/// Trait defining the cache interface.
/// High-level modules depend on this trait; low-level modules implement it.
trait Cache {
    fn get(&self, key: &str) -> Option<String>;
    fn set(&self, key: &str, value: &str, ttl_seconds: u64);
    fn delete(&self, key: &str);
    fn clear(&self);
}

// ── Data types ──

#[derive(Debug, Clone)]
struct UserProfile {
    user_id: String,
    name: String,
    email: String,
    preferences: HashMap<String, String>,
}

// ── Implementation 1: Redis (via generics) ──

struct RedisCache {
    connection_string: String,
    pool_size: u32,
}

impl RedisCache {
    fn new(connection_string: &str, pool_size: u32) -> Self {
        println!(
            "[Redis] Connected to {} (pool: {})",
            connection_string, pool_size
        );
        RedisCache {
            connection_string: connection_string.to_string(),
            pool_size,
        }
    }
}

impl Cache for RedisCache {
    fn get(&self, key: &str) -> Option<String> {
        println!("[Redis] GET {}", key);
        // In production, query Redis:
        // let mut conn = self.pool.get().ok()?;
        // redis::cmd("GET").arg(key).query(&mut conn).ok()
        None
    }

    fn set(&self, key: &str, value: &str, ttl_seconds: u64) {
        println!("[Redis] SET {} = {} (TTL: {}s)", key, value, ttl_seconds);
        // In production:
        // let mut conn = self.pool.get().unwrap();
        // redis::cmd("SET")
        //     .arg(&[key, value])
        //     .arg(&["EX", &ttl_seconds.to_string()])
        //     .query(&mut conn)
        //     .unwrap();
    }

    fn delete(&self, key: &str) {
        println!("[Redis] DEL {}", key);
    }

    fn clear(&self) {
        println!("[Redis] FLUSHDB");
    }
}

// ── Implementation 2: In-memory cache ──

struct InMemoryCache {
    data: std::sync::Mutex<HashMap<String, (String, Instant)>>,
}

impl InMemoryCache {
    fn new() -> Self {
        InMemoryCache {
            data: std::sync::Mutex::new(HashMap::new()),
        }
    }
}

impl Cache for InMemoryCache {
    fn get(&self, key: &str) -> Option<String> {
        let data = self.data.lock().unwrap();
        if let Some((value, expires_at)) = data.get(key) {
            if Instant::now() < *expires_at {
                println!("[InMemory] HIT {}", key);
                Some(value.clone())
            } else {
                println!("[InMemory] EXPIRED {}", key);
                None
            }
        } else {
            println!("[InMemory] MISS {}", key);
            None
        }
    }

    fn set(&self, key: &str, value: &str, ttl_seconds: u64) {
        let mut data = self.data.lock().unwrap();
        let expires_at = Instant::now() + Duration::from_secs(ttl_seconds);
        data.insert(key.to_string(), (value.to_string(), expires_at));
        println!(
            "[InMemory] SET {} = {} (TTL: {}s, expires at {:?})",
            key,
            value,
            ttl_seconds,
            expires_at
        );
    }

    fn delete(&self, key: &str) {
        let mut data = self.data.lock().unwrap();
        data.remove(key);
        println!("[InMemory] DEL {}", key);
    }

    fn clear(&self) {
        let mut data = self.data.lock().unwrap();
        data.clear();
        println!("[InMemory] CLEAR");
    }
}

// ── Implementation 3: No-op cache (for testing) ──

struct NoOpCache;

impl Cache for NoOpCache {
    fn get(&self, key: &str) -> Option<String> {
        println!("[NoOp] GET {} → always miss", key);
        None
    }

    fn set(&self, key: &str, value: &str, ttl_seconds: u64) {
        println!("[NoOp] SET {} = {} (discarded)", key, value);
    }

    fn delete(&self, key: &str) {
        println!("[NoOp] DEL {} (discarded)", key);
    }

    fn clear(&self) {
        println!("[NoOp] CLEAR (discarded)");
    }
}

// ── High-level module ──
// CORRECT: Depends only on the Cache trait via generics (zero-cost abstraction).

struct CacheService<T: Cache> {
    cache: T,
    default_ttl: u64,
}

impl<T: Cache> CacheService<T> {
    /// CORRECT: Dependency is injected via constructor.
    fn new(cache: T) -> Self {
        CacheService {
            cache,
            default_ttl: 3600,
        }
    }

    fn get_user_profile(&self, user_id: &str) -> Option<UserProfile> {
        let key = format!("user:profile:{}", user_id);
        let cached = self.cache.get(&key)?;
        println!("[CacheService] Deserializing profile for {}", user_id);

        // Simple JSON parsing simulation:
        // In a real app, use serde_json::from_str(&cached).ok()
        Some(UserProfile {
            user_id: user_id.to_string(),
            name: "Profile from cache".to_string(),
            email: "cached@example.com".to_string(),
            preferences: HashMap::new(),
        })
    }

    fn set_user_profile(&self, profile: &UserProfile) {
        let key = format!("user:profile:{}", profile.user_id);
        let json_str = serde_json::to_string(profile).unwrap_or_else(|_| {
            format!(
                r#"{{"user_id":"{}","name":"{}","email":"{}"}}"#,
                profile.user_id, profile.name, profile.email
            )
        });
        self.cache.set(&key, &json_str, self.default_ttl);
    }

    fn invalidate_user(&self, user_id: &str) {
        let key = format!("user:profile:{}", user_id);
        self.cache.delete(&key);
    }

    fn set_with_custom_ttl(&self, key: &str, value: &str, ttl_seconds: u64) {
        self.cache.set(key, value, ttl_seconds);
    }

    fn clear_all(&self) {
        self.cache.clear();
    }
}

// ── Composition root ──

fn main() {
    println!("=== Scenario 1: Redis backend ===");
    let redis_cache = RedisCache::new("redis://localhost:6379", 10);
    let redis_service: CacheService<RedisCache> = CacheService::new(redis_cache);
    run_queries(&redis_service);
    println!();

    println!("=== Scenario 2: In-memory backend ===");
    let mem_cache = InMemoryCache::new();
    let mem_service: CacheService<InMemoryCache> = CacheService::new(mem_cache);
    run_queries(&mem_service);
    println!();

    println!("=== Scenario 3: No-op cache (test mode) ===");
    let noop_service: CacheService<NoOpCache> = CacheService::new(NoOpCache);
    run_queries(&noop_service);
}

fn run_queries<T: Cache>(service: &CacheService<T>) {
    let profile = UserProfile {
        user_id: "u-100".to_string(),
        name: "Alice Johnson".to_string(),
        email: "alice@example.com".to_string(),
        preferences: {
            let mut p = HashMap::new();
            p.insert("theme".to_string(), "dark".to_string());
            p.insert("language".to_string(), "en".to_string());
            p
        },
    };

    // Set and retrieve
    println!("Setting user profile...");
    service.set_user_profile(&profile);

    println!("Getting user profile...");
    match service.get_user_profile("u-100") {
        Some(p) => println!("  → Found: {} ({})", p.name, p.email),
        None => println!("  → Cache miss"),
    }

    // Custom TTL
    println!("Setting session key with short TTL...");
    service.set_with_custom_ttl("session:abc123", "user-100", 600);

    // Invalidate
    println!("Invalidating user profile...");
    service.invalidate_user("u-100");

    println!("Getting after invalidation...");
    match service.get_user_profile("u-100") {
        Some(p) => println!("  → Found: {}", p.name),
        None => println!("  → Cache miss (expected)"),
    }
}

/// Alias for trait-object based approach (alternative to generics)
/// Useful when you need runtime polymorphism (e.g., a Vec of different caches)
type BoxedCache = Box<dyn Cache>;

struct DynamicallyCachedService {
    cache: BoxedCache,
}

impl DynamicallyCachedService {
    fn new(cache: BoxedCache) -> Self {
        DynamicallyCachedService { cache }
    }

    fn get(&self, key: &str) -> Option<String> {
        self.cache.get(key)
    }

    fn set(&self, key: &str, value: &str, ttl: u64) {
        self.cache.set(key, value, ttl);
    }
}
```

## Key Points

- The `Cache` trait defines the abstraction. Any type implementing it can be used with `CacheService`.
- `CacheService<T: Cache>` uses generics with a trait bound. This gives **zero-cost abstraction** — the compiler monomorphizes the code for each concrete type, and there is no runtime overhead from virtual dispatch.
- The `BoxedCache` type alias shows the alternative approach using trait objects (`Box<dyn Cache>`), which provides runtime polymorphism at the cost of a vtable lookup.
- `RedisCache` connects to an external service; `InMemoryCache` stores data in-process; `NoOpCache` discards everything. All three implement the same trait and are interchangeable.
- The composition root (`main`) is the only place that decides which cache backend to use. `CacheService` never knows which implementation it received.
- Adding a new backend (e.g., `MemcachedCache`, `S3Cache`) requires creating a new type that implements `Cache` and wiring it in at the composition root. No changes to `CacheService`.

## Testing with DIP

```rust
#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_cache_service_with_noop_cache() {
        let service: CacheService<NoOpCache> = CacheService::new(NoOpCache);

        // Setting and getting should not panic
        service.set_user_profile(&UserProfile {
            user_id: "test-1".to_string(),
            name: "Test".to_string(),
            email: "test@test.com".to_string(),
            preferences: HashMap::new(),
        });

        assert!(service.get_user_profile("test-1").is_none());
    }

    #[test]
    fn test_cache_service_with_in_memory_cache() {
        let cache = InMemoryCache::new();
        let service: CacheService<InMemoryCache> = CacheService::new(cache);

        service.set_with_custom_ttl("key1", "value1", 60);
        assert_eq!(service.get_user_profile("nonexistent"), None);
    }

    #[test]
    fn test_in_memory_ttl_expiration() {
        let cache = InMemoryCache::new();
        cache.set("expires-fast", "value", 0);
        std::thread::sleep(Duration::from_millis(10));
        assert!(cache.get("expires-fast").is_none());
    }
}
```
