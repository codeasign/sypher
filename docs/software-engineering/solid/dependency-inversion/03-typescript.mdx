## DIP in TypeScript

TypeScript's interface system and strict type checking make the Dependency Inversion Principle natural to express. Interfaces define the contract, classes implement them, and the high-level module depends only on the interface.

This example builds a **data export system** that transforms records into various formats and uploads them to different storage backends. Without the DIP, the exporter would be coupled to specific formatters and storage engines.

## The Violation

In the violating version, `DataExporter` directly creates a `CsvFormatter` and a `FileSystemStorage` instance. Switching to JSON or S3 requires changing `DataExporter`.

```typescript
// VIOLATION: DataExporter depends on concrete CsvFormatter and FileSystemStorage.
// Changing the format or storage requires editing DataExporter.
// Testing requires actual file system access.

interface Record {
  id: string;
  name: string;
  email: string;
  createdAt: Date;
}

class CsvFormatter {
  format(records: Record[]): string {
    const header = 'id,name,email,createdAt';
    const rows = records.map(
      (r) => `${r.id},${r.name},${r.email},${r.createdAt.toISOString()}`
    );
    return [header, ...rows].join('\n');
  }
}

class FileSystemStorage {
  private basePath: string;

  constructor(basePath: string) {
    this.basePath = basePath;
  }

  save(filename: string, content: string): void {
    const fs = require('fs');
    const path = require('path');
    const fullPath = path.join(this.basePath, filename);
    fs.writeFileSync(fullPath, content, 'utf-8');
    console.log(`[FileSystem] Saved to ${fullPath}`);
  }
}

class DataExporter {
  private formatter: CsvFormatter;
  private storage: FileSystemStorage;

  constructor() {
    // VIOLATION: DataExporter creates its own concrete dependencies.
    this.formatter = new CsvFormatter();
    this.storage = new FileSystemStorage('./exports');
  }

  exportRecords(records: Record[], filename: string): void {
    const formatted = this.formatter.format(records);
    this.storage.save(filename, formatted);
    console.log(`Exported ${records.length} records to ${filename}`);
  }
}

// Usage
const exporter = new DataExporter();
const records: Record[] = [
  { id: '1', name: 'Alice', email: 'alice@example.com', createdAt: new Date('2025-01-15') },
  { id: '2', name: 'Bob', email: 'bob@example.com', createdAt: new Date('2025-02-20') },
];
exporter.exportRecords(records, 'users-export.csv');

/* Expected output:
   [FileSystem] Saved to ./exports/users-export.csv
   Exported 2 records to users-export.csv
*/
```

## The Correct Implementation

We extract `Formatter` and `Storage` interfaces. Multiple implementations of each can be created. `DataExporter` receives its dependencies through the constructor and works with any implementation that satisfies the interface.

```typescript
// CORRECT: DataExporter depends on Formatter and Storage interfaces.
// Implementations are swappable without changing the exporter.

// ── Abstractions ──

interface Record {
  id: string;
  name: string;
  email: string;
  createdAt: Date;
}

interface Formatter {
  format(records: Record[]): string;
}

interface Storage {
  save(filename: string, content: string): void;
}

// ── Formatter implementations ──

class CsvFormatter implements Formatter {
  format(records: Record[]): string {
    const header = 'id,name,email,createdAt';
    const rows = records.map(
      (r) => `${r.id},${r.name},${r.email},${r.createdAt.toISOString()}`
    );
    return [header, ...rows].join('\n');
  }
}

class JsonFormatter implements Formatter {
  format(records: Record[]): string {
    return JSON.stringify(records, null, 2);
  }
}

class XmlFormatter implements Formatter {
  format(records: Record[]): string {
    const recordsXml = records
      .map(
        (r) => `  <record>
    <id>${r.id}</id>
    <name>${r.name}</name>
    <email>${r.email}</email>
    <createdAt>${r.createdAt.toISOString()}</createdAt>
  </record>`
      )
      .join('\n');

    return `<?xml version="1.0" encoding="UTF-8"?>\n<records>\n${recordsXml}\n</records>`;
  }
}

// ── Storage implementations ──

class LocalStorage implements Storage {
  private basePath: string;

  constructor(basePath: string) {
    this.basePath = basePath;
  }

  save(filename: string, content: string): void {
    const fs = require('fs');
    const path = require('path');
    const fullPath = path.join(this.basePath, filename);
    fs.writeFileSync(fullPath, content, 'utf-8');
    console.log(`[LocalStorage] Saved to ${fullPath}`);
  }
}

class S3Storage implements Storage {
  private bucketName: string;
  private region: string;

  constructor(bucketName: string, region: string) {
    this.bucketName = bucketName;
    this.region = region;
  }

  save(filename: string, content: string): void {
    // In a real app, you would use @aws-sdk/client-s3.
    console.log(`[S3] Uploaded ${filename} to s3://${this.bucketName}/${filename}`);
    console.log(`[S3] Region: ${this.region}, Size: ${content.length} bytes`);
  }
}

class MemoryStorage implements Storage {
  private store: Map<string, string> = new Map();

  save(filename: string, content: string): void {
    this.store.set(filename, content);
    console.log(`[MemoryStorage] Stored ${filename} (${content.length} bytes)`);
  }

  get(filename: string): string | undefined {
    return this.store.get(filename);
  }

  list(): string[] {
    return Array.from(this.store.keys());
  }
}

// ── High-level module ──
// CORRECT: Depends only on Formatter and Storage interfaces.

class DataExporter {
  private formatter: Formatter;
  private storage: Storage;

  constructor(formatter: Formatter, storage: Storage) {
    this.formatter = formatter;
    this.storage = storage;
  }

  exportRecords(records: Record[], filename: string): void {
    if (records.length === 0) {
      console.log('No records to export.');
      return;
    }

    const formatted = this.formatter.format(records);
    this.storage.save(filename, formatted);
    console.log(`Exported ${records.length} records to ${filename}`);
  }

  exportSummary(records: Record[], formatType: string): string {
    const formatter = this.getFormatter(formatType);
    return formatter.format(records);
  }

  private getFormatter(formatType: string): Formatter {
    switch (formatType) {
      case 'csv': return new CsvFormatter();
      case 'json': return new JsonFormatter();
      case 'xml': return new XmlFormatter();
      default: throw new Error(`Unknown format: ${formatType}`);
    }
  }
}

// ── Composition root ──

function main(): void {
  const records: Record[] = [
    { id: '1', name: 'Alice Johnson', email: 'alice@example.com', createdAt: new Date('2025-01-15') },
    { id: '2', name: 'Bob Smith', email: 'bob@example.com', createdAt: new Date('2025-02-20') },
    { id: '3', name: 'Carol Davis', email: 'carol@example.com', createdAt: new Date('2025-03-10') },
  ];

  // Scenario 1: Export CSV to local file system
  console.log('=== Scenario 1: CSV to LocalStorage ===');
  const csvExporter = new DataExporter(new CsvFormatter(), new LocalStorage('./exports'));
  csvExporter.exportRecords(records, 'users.csv');

  // Scenario 2: Export JSON to S3
  console.log('\n=== Scenario 2: JSON to S3 ===');
  const jsonExporter = new DataExporter(new JsonFormatter(), new S3Storage('my-bucket', 'us-east-1'));
  jsonExporter.exportRecords(records, 'users.json');

  // Scenario 3: Export XML to in-memory storage (testing scenario)
  console.log('\n=== Scenario 3: XML to MemoryStorage ===');
  const memoryStorage = new MemoryStorage();
  const xmlExporter = new DataExporter(new XmlFormatter(), memoryStorage);
  xmlExporter.exportRecords(records, 'users.xml');
  console.log(`Stored files: ${memoryStorage.list().join(', ')}`);
}

main();

/* Expected output:
   === Scenario 1: CSV to LocalStorage ===
   [LocalStorage] Saved to ./exports/users.csv
   Exported 3 records to users.csv

   === Scenario 2: JSON to S3 ===
   [S3] Uploaded users.json to s3://my-bucket/users.json
   [S3] Region: us-east-1, Size: 249 bytes
   Exported 3 records to users.json

   === Scenario 3: XML to MemoryStorage ===
   [XML formatted output...]
   [MemoryStorage] Stored users.xml (922 bytes)
   Exported 3 records to users.xml
   Stored files: users.xml
*/
```

## Key Points

- `Formatter` and `Storage` are pure TypeScript interfaces — they define contracts with zero implementation.
- `DataExporter` never imports or instantiates any concrete implementation class. It works through abstractions.
- Adding a new format (e.g., `YamlFormatter`) or a new storage backend (e.g., `GcsStorage` for Google Cloud Storage) requires no changes to `DataExporter`.
- `MemoryStorage` is useful for unit tests — no disk I/O needed.
- The composition root (the `main()` function) is the only place that decides which implementations to use. This separation of wiring from business logic is a core benefit of DIP.

## Testing with DIP

```typescript
import { describe, it, expect } from 'vitest';

class SpyStorage implements Storage {
  public savedFiles: Array<{ filename: string; content: string }> = [];

  save(filename: string, content: string): void {
    this.savedFiles.push({ filename, content });
  }
}

describe('DataExporter', () => {
  it('exports records using the injected formatter and storage', () => {
    const spy = new SpyStorage();
    const exporter = new DataExporter(new JsonFormatter(), spy);

    const records: Record[] = [
      { id: '1', name: 'Test', email: 'test@test.com', createdAt: new Date() },
    ];

    exporter.exportRecords(records, 'test.json');

    expect(spy.savedFiles).toHaveLength(1);
    expect(spy.savedFiles[0].filename).toBe('test.json');
    expect(spy.savedFiles[0].content).toContain('"name": "Test"');
  });

  it('handles empty record lists gracefully', () => {
    const spy = new SpyStorage();
    const exporter = new DataExporter(new CsvFormatter(), spy);

    exporter.exportRecords([], 'empty.csv');

    expect(spy.savedFiles).toHaveLength(0);
  });
});
```
