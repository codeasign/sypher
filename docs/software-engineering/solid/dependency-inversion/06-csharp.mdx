## DIP in C#

C# with .NET 8 provides first-class support for the Dependency Inversion Principle through interfaces, the built-in dependency injection container (`Microsoft.Extensions.DependencyInjection`), and constructor injection patterns. ASP.NET Core applications naturally follow DIP through their controller and service architecture.

This example builds a **logging system**. A high-level `AppLogger` needs to write log messages to various destinations. Without the DIP, it would be coupled to a specific logging implementation. With the DIP, it depends on an abstraction that any logging target can implement.

## The Violation

In the violating version, `AppLogger` directly writes to a file using `StreamWriter`. Switching from file logging to console logging, database logging, or cloud logging requires editing `AppLogger`. Testing produces real files on disk.

```csharp
// VIOLATION: AppLogger depends on concrete file I/O.
// Changing the logging target requires modifying AppLogger.
// Testing creates real files on disk.

namespace LoggingSystem.Violation;

public class AppLogger
{
    private readonly string _logFilePath;

    public AppLogger(string logFilePath)
    {
        _logFilePath = logFilePath;

        // VIOLATION: Creating concrete file infrastructure inside the class.
        var directory = Path.GetDirectoryName(logFilePath);
        if (!string.IsNullOrEmpty(directory) && !Directory.Exists(directory))
        {
            Directory.CreateDirectory(directory);
        }
    }

    public void LogInformation(string message)
    {
        WriteLog("INFO", message);
    }

    public void LogWarning(string message)
    {
        WriteLog("WARN", message);
    }

    public void LogError(string message, Exception? exception = null)
    {
        var fullMessage = exception != null
            ? $"{message}\nException: {exception.GetType().Name}: {exception.Message}"
            : message;
        WriteLog("ERROR", fullMessage);
    }

    private void WriteLog(string level, string message)
    {
        // VIOLATION: Direct file writing — couples AppLogger to the file system.
        using var writer = new StreamWriter(_logFilePath, append: true);
        var timestamp = DateTime.UtcNow.ToString("yyyy-MM-dd HH:mm:ss.fff");
        writer.WriteLine($"[{timestamp}] [{level}] {message}");
    }
}

// Usage
class Program
{
    static void Main()
    {
        var logger = new AppLogger("app.log");
        logger.LogInformation("Application started");
        logger.LogWarning("Memory usage is high: 85%");
        logger.LogError("Failed to connect to database", new InvalidOperationException("Timeout"));

        // PROBLEM: Can't switch to console, database, or cloud logging.
        // PROBLEM: Unit tests produce real files on disk.
    }
}

/* Expected output:
   (Writes to app.log on disk)
*/
```

## The Correct Implementation

We define an `ILogger` interface. `FileLogger`, `ConsoleLogger`, and `DatabaseLogger` implement it. `AppLogger` receives its dependency through the constructor and works with any implementation that satisfies the interface.

```csharp
// CORRECT: AppLogger depends on ILogger abstraction.
// Multiple logging targets can be swapped without changing the logging logic.

namespace LoggingSystem.Correct;

// ── Abstractions ──

/// <summary>
/// Abstraction for log output destinations.
/// Low-level modules implement this interface.
/// High-level modules depend on it.
/// </summary>
public interface ILogger
{
    void Log(string level, string message);
}

// ── Implementation 1: File logging ──

public class FileLogger : ILogger
{
    private readonly string _logFilePath;

    public FileLogger(string logFilePath)
    {
        _logFilePath = logFilePath;
        var directory = Path.GetDirectoryName(logFilePath);
        if (!string.IsNullOrEmpty(directory) && !Directory.Exists(directory))
        {
            Directory.CreateDirectory(directory);
        }
    }

    public void Log(string level, string message)
    {
        using var writer = new StreamWriter(_logFilePath, append: true);
        var timestamp = DateTime.UtcNow.ToString("yyyy-MM-dd HH:mm:ss.fff");
        writer.WriteLine($"[{timestamp}] [{level}] {message}");
    }
}

// ── Implementation 2: Console logging ──

public class ConsoleLogger : ILogger
{
    private readonly bool _useColors;

    public ConsoleLogger(bool useColors = true)
    {
        _useColors = useColors;
    }

    public void Log(string level, string message)
    {
        var timestamp = DateTime.UtcNow.ToString("yyyy-MM-dd HH:mm:ss.fff");
        var formatted = $"[{timestamp}] [{level}] {message}";

        if (_useColors)
        {
            var originalColor = Console.ForegroundColor;
            Console.ForegroundColor = level switch
            {
                "ERROR" => ConsoleColor.Red,
                "WARN" => ConsoleColor.Yellow,
                "INFO" => ConsoleColor.Green,
                _ => ConsoleColor.White
            };
            Console.WriteLine(formatted);
            Console.ForegroundColor = originalColor;
        }
        else
        {
            Console.WriteLine(formatted);
        }
    }
}

// ── Implementation 3: Database logging ──

public class DatabaseLogger : ILogger
{
    private readonly string _connectionString;

    public DatabaseLogger(string connectionString)
    {
        _connectionString = connectionString;
    }

    public void Log(string level, string message)
    {
        // In production, write to a log table:
        // INSERT INTO Logs (Timestamp, Level, Message) VALUES (@ts, @level, @message)
        Console.WriteLine($"[DbLogger] Inserting log entry: Level={level}, Length={message.Length}");
    }
}

// ── Implementation 4: Composite logger (writes to multiple targets) ──

public class CompositeLogger : ILogger
{
    private readonly ILogger[] _loggers;

    public CompositeLogger(params ILogger[] loggers)
    {
        _loggers = loggers;
    }

    public void Log(string level, string message)
    {
        foreach (var logger in _loggers)
        {
            logger.Log(level, message);
        }
    }
}

// ── Implementation 5: In-memory logger (for testing) ──

public class InMemoryLogger : ILogger
{
    private readonly List<(string Level, string Message, DateTime Timestamp)> _entries = new();

    public IReadOnlyList<(string Level, string Message, DateTime Timestamp)> Entries =>
        _entries.AsReadOnly();

    public void Log(string level, string message)
    {
        _entries.Add((level, message, DateTime.UtcNow));
    }

    public void Clear() => _entries.Clear();
}

// ── High-level module ──
// CORRECT: Depends only on ILogger abstraction.

public class AppLogger
{
    private readonly ILogger _logger;

    public AppLogger(ILogger logger)
    {
        // CORRECT: Dependency is injected, not created.
        _logger = logger;
    }

    public void LogInformation(string message)
    {
        _logger.Log("INFO", message);
    }

    public void LogWarning(string message)
    {
        _logger.Log("WARN", message);
    }

    public void LogError(string message, Exception? exception = null)
    {
        var fullMessage = exception != null
            ? $"{message}\nException: {exception.GetType().Name}: {exception.Message}"
            : message;
        _logger.Log("ERROR", fullMessage);
    }

    /// <summary>
    /// Logs a structured message with key-value pairs.
    /// </summary>
    public void LogInformation(string template, params object[] args)
    {
        var message = string.Format(template, args);
        _logger.Log("INFO", message);
    }

    /// <summary>
    /// Times an action and logs the duration.
    /// </summary>
    public T LogTimed<T>(string operationName, Func<T> action)
    {
        var sw = System.Diagnostics.Stopwatch.StartNew();
        try
        {
            var result = action();
            sw.Stop();
            LogInformation($"{operationName} completed in {sw.ElapsedMilliseconds}ms");
            return result;
        }
        catch (Exception ex)
        {
            sw.Stop();
            LogError($"{operationName} failed after {sw.ElapsedMilliseconds}ms", ex);
            throw;
        }
    }

    public async Task<T> LogTimedAsync<T>(string operationName, Func<Task<T>> asyncAction)
    {
        var sw = System.Diagnostics.Stopwatch.StartNew();
        try
        {
            var result = await asyncAction();
            sw.Stop();
            LogInformation($"{operationName} completed in {sw.ElapsedMilliseconds}ms");
            return result;
        }
        catch (Exception ex)
        {
            sw.Stop();
            LogError($"{operationName} failed after {sw.ElapsedMilliseconds}ms", ex);
            throw;
        }
    }
}

// ── Composition root ──

class Program
{
    static void Main()
    {
        Console.WriteLine("=== Scenario 1: Console logging ===");
        var consoleLogger = new AppLogger(new ConsoleLogger());
        consoleLogger.LogInformation("Application started");
        consoleLogger.LogWarning("Memory usage is high: 85%");
        consoleLogger.LogError("Failed to connect to database", new InvalidOperationException("Timeout"));
        Console.WriteLine();

        Console.WriteLine("=== Scenario 2: Composite logging (Console + InMemory) ===");
        var inMemory = new InMemoryLogger();
        var composite = new AppLogger(new CompositeLogger(
            new ConsoleLogger(useColors: false),
            inMemory
        ));
        composite.LogInformation("User logged in: alice@example.com");
        composite.LogWarning("Rate limit approaching: 95/100 requests");
        Console.WriteLine($"InMemory has {inMemory.Entries.Count} entries:");
        foreach (var (level, msg, ts) in inMemory.Entries)
        {
            Console.WriteLine($"  [{level}] {msg}");
        }
        Console.WriteLine();

        Console.WriteLine("=== Scenario 3: Timed operation logging ===");
        var timedLogger = new AppLogger(new ConsoleLogger(useColors: false));
        var result = timedLogger.LogTimed("HeavyCalculation", () =>
        {
            Thread.Sleep(100);
            return 42;
        });
        Console.WriteLine($"Result: {result}");
        Console.WriteLine();

        Console.WriteLine("=== Scenario 4: Database logging ===");
        var dbLogger = new AppLogger(new DatabaseLogger("Server=localhost;Database=Logs;"));
        dbLogger.LogInformation("Processing batch job #1024");
    }
}
```

## Key Points

- `ILogger` is a single-method interface. Any class implementing `Log(string level, string message)` can be injected into `AppLogger`.
- `AppLogger` never imports `FileLogger`, `ConsoleLogger`, or `DatabaseLogger`. It works purely through the `ILogger` abstraction.
- The `CompositeLogger` demonstrates how the DIP enables composition patterns — multiple loggers can be combined without changing `AppLogger`.
- `InMemoryLogger` makes unit testing trivial. Tests can inspect logged entries without touching the file system or console.
- ASP.NET Core applications would register these with the built-in DI container in `Program.cs`:
  ```csharp
  builder.Services.AddSingleton<ILogger>(sp => new FileLogger("app.log"));
  builder.Services.AddScoped<AppLogger>();
  ```
- The `LogTimed` and `LogTimedAsync` methods show how the high-level module can add cross-cutting behavior (timing, structured logging) on top of the abstraction without coupling to any specific output.

## Testing with DIP

```csharp
using Xunit;

public class AppLoggerTests
{
    [Fact]
    public void LogInformation_StoresMessage()
    {
        var inMemory = new InMemoryLogger();
        var logger = new AppLogger(inMemory);

        logger.LogInformation("Test message");

        Assert.Single(inMemory.Entries);
        Assert.Equal("INFO", inMemory.Entries[0].Level);
        Assert.Equal("Test message", inMemory.Entries[0].Message);
    }

    [Fact]
    public void LogTimed_ReturnsResultAndLogsDuration()
    {
        var inMemory = new InMemoryLogger();
        var logger = new AppLogger(inMemory);

        var result = logger.LogTimed("TestOp", () => 99);

        Assert.Equal(99, result);
        Assert.Contains(inMemory.Entries, e =>
            e.Level == "INFO" && e.Message.Contains("TestOp completed in"));
    }
}
```
