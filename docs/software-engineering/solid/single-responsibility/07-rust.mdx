## Implementation

A Rust 2021 edition implementation of SRP using a configuration management system. The violation shows a `ConfigManager` struct handling file reading, parsing, validation, and environment variable resolution. The correct version splits each concern.

```rust
// SRP — Rust 2021 edition implementation
// Domain: Configuration management

use std::collections::HashMap;
use std::path::Path;

// ============================================================
// VIOLATION: ConfigManager handles file reading, YAML-like
// parsing, validation, and env var resolution — 4 concerns
// ============================================================
//
// use std::fs;
//
// struct ConfigManager {
//     raw: String,
//     parsed: HashMap<String, String>,
// }
//
// impl ConfigManager {
//     fn new(path: &str) -> Result<Self, String> {
//         let raw = fs::read_to_string(path).map_err(|e| e.to_string())?;
//         let mut parsed = HashMap::new();
//         for line in raw.lines() {
//             if let Some((k, v)) = line.split_once('=') {
//                 let val = v.trim();
//                 let resolved = if val.starts_with("${") && val.ends_with('}') {
//                     let env_key = &val[2..val.len()-1];
//                     std::env::var(env_key).unwrap_or_default()
//                 } else {
//                     val.to_string()
//                 };
//                 parsed.insert(k.trim().to_string(), resolved);
//             }
//         }
//         Self { raw, parsed }
//     }
// }

// ============================================================
// CORRECT IMPLEMENTATION — One trait per responsibility
// ============================================================

/// Responsibility: Read raw configuration text from a source
trait ConfigReader {
    fn read(&self, path: &Path) -> Result<String, String>;
}

struct FileConfigReader;

impl ConfigReader for FileConfigReader {
    fn read(&self, path: &Path) -> Result<String, String> {
        std::fs::read_to_string(path).map_err(|e| format!("Failed to read config: {e}"))
    }
}

/// Responsibility: Parse raw text into key-value pairs
trait ConfigParser {
    fn parse(&self, raw: &str) -> Result<HashMap<String, String>, String>;
}

struct KeyValueConfigParser;

impl ConfigParser for KeyValueConfigParser {
    fn parse(&self, raw: &str) -> Result<HashMap<String, String>, String> {
        let mut map = HashMap::new();
        for (line_num, line) in raw.lines().enumerate() {
            let line = line.trim();
            if line.is_empty() || line.starts_with('#') {
                continue;
            }
            let (key, value) = line.split_once('=')
                .ok_or_else(|| format!("Line {}: missing '=' separator", line_num + 1))?;
            map.insert(key.trim().to_string(), value.trim().to_string());
        }
        Ok(map)
    }
}

/// Responsibility: Validate configuration values
trait ConfigValidator {
    fn validate(&self, config: &HashMap<String, String>) -> Result<(), Vec<String>>;
}

struct AppConfigValidator;

impl ConfigValidator for AppConfigValidator {
    fn validate(&self, config: &HashMap<String, String>) -> Result<(), Vec<String>> {
        let mut errors = Vec::new();

        if !config.contains_key("db_host") {
            errors.push("Missing required key: db_host".to_string());
        }
        if !config.contains_key("db_port") {
            errors.push("Missing required key: db_port".to_string());
        }
        if let Some(port) = config.get("db_port") {
            if port.parse::<u16>().is_err() {
                errors.push(format!("db_port must be a valid port number, got: {port}"));
            }
        }
        if !config.contains_key("app_name") {
            errors.push("Missing required key: app_name".to_string());
        }

        if errors.is_empty() { Ok(()) } else { Err(errors) }
    }
}

/// Responsibility: Resolve environment variable references
trait ConfigResolver {
    fn resolve(&self, config: &HashMap<String, String>) -> HashMap<String, String>;
}

struct EnvConfigResolver;

impl ConfigResolver for EnvConfigResolver {
    fn resolve(&self, config: &HashMap<String, String>) -> HashMap<String, String> {
        config.iter().map(|(key, value)| {
            let resolved = if let Some(env_var) = value.strip_prefix("${")
                .and_then(|v| v.strip_suffix('}'))
            {
                std::env::var(env_var).unwrap_or_else(|_| {
                    eprintln!("Warning: env var {env_var} not set, using empty string");
                    String::new()
                })
            } else {
                value.clone()
            };
            (key.clone(), resolved)
        }).collect()
    }
}

/// Orchestrator — composes SRP components
struct ConfigLoader {
    reader: Box<dyn ConfigReader>,
    parser: Box<dyn ConfigParser>,
    validator: Box<dyn ConfigValidator>,
    resolver: Box<dyn ConfigResolver>,
}

impl ConfigLoader {
    fn new(
        reader: Box<dyn ConfigReader>,
        parser: Box<dyn ConfigParser>,
        validator: Box<dyn ConfigValidator>,
        resolver: Box<dyn ConfigResolver>,
    ) -> Self {
        Self { reader, parser, validator, resolver }
    }

    fn load(&self, path: &Path) -> Result<HashMap<String, String>, String> {
        let raw = self.reader.read(path)?;
        let parsed = self.parser.parse(&raw)?;
        self.validator.validate(&parsed)
            .map_err(|errors| format!("Validation failed:\n  - {}", errors.join("\n  - ")))?;
        Ok(self.resolver.resolve(&parsed))
    }
}

// --- Main driver ---
fn main() -> Result<(), String> {
    // Set up environment variable for testing
    std::env::set_var("DATABASE_URL", "localhost:5432");

    // Create a temporary config file
    let config_content = "\
app_name = MyApp
db_host = localhost
db_port = 5432
db_url = ${DATABASE_URL}
log_level = debug
";
    let config_path = Path::new("/tmp/app_config.txt");
    std::fs::write(config_path, config_content)
        .map_err(|e| format!("Failed to write test config: {e}"))?;

    // Compose SRP components
    let loader = ConfigLoader::new(
        Box::new(FileConfigReader),
        Box::new(KeyValueConfigParser),
        Box::new(AppConfigValidator),
        Box::new(EnvConfigResolver),
    );

    let config = loader.load(config_path)?;

    println!("=== Loaded Configuration ===");
    let mut keys: Vec<&String> = config.keys().collect();
    keys.sort();
    for key in keys {
        println!("{key} = {}", config[key]);
    }

    // Cleanup
    std::fs::remove_file(config_path).ok();

    Ok(())
}
```

## Key Language-Specific Notes

- **Traits per concern** — Each responsibility is modeled as a separate trait (`ConfigReader`, `ConfigParser`, `ConfigValidator`, `ConfigResolver`), which is the Rust idiomatic equivalent of Java/C# interfaces for SRP.
- **`Box<dyn Trait>`** — Dynamic dispatch via trait objects allows the orchestrator to accept any implementation of each concern at runtime, enabling easy swapping (e.g., `FileConfigReader` vs `HttpConfigReader`).
- **`Result<T, String>` for error handling** — Each SRP component returns its own error type, keeping failure modes local to the concern. A real system would use proper error types; `String` here keeps the example self-contained.
- **No external crates** — The implementation uses only `std::{collections, fs, path, env}`, demonstrating that SRP is a design principle, not a framework feature.
- **`split_once`** (stabilized in Rust 1.52) — A clean standard-library method for key-value parsing that avoids pulling in regex or nom dependencies.
