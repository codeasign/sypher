## Implementation

Below is a complete Java implementation of the Template Method pattern using a stock market trading domain. The abstract `TradingStrategy` class defines the `executeTrade()` template method as a `final` method — preventing subclasses from modifying the algorithm skeleton. Concrete strategies `MovingAverageCrossover` and `MeanReversion` implement the primitive operations and optionally override hooks. The driver exercises both strategies with realistic AAPL, GOOGL, and MSFT price data using Java records for immutable data structures.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌─────────────────────────────────────────────────┐
//   │ TradingStrategy (abstract)                      │← OOP: Abstraction
//   ├─────────────────────────────────────────────────┤   ISP: minimal abstract
//   │ + final executeTrade(ticker, prices)             │    method set
//   │ # abstract analyzeMarket(prices): Decision       │
//   │ # abstract calcPositionSize(confidence, max): int│
//   │ # preValidate(signal, conf): boolean              │← hook, default true
//   │ # applyRiskControls(ticker, shares): boolean      │← hook, default true
//   │ # placeOrder(ticker, shares, signal): Order       │← concrete
//   │ # final logTrade(...)                             │← concrete + final
//   └────────────────────────┬─────────────────────────┘
//                            ▲ extends
//   ┌────────────────────────┴─────────────────────────┐
//   │                                                   │
//   ┌──────────────────────────┐  ┌──────────────────────────┐
//   │ MovingAverageCrossover   │  │ MeanReversion            │
//   ├──────────────────────────┤  ├──────────────────────────┤
//   │ SHORT_WINDOW = 3         │  │ LOOKBACK = 10            │← OOP: Polymorphism
//   │ LONG_WINDOW = 8          │  │ THRESHOLD = 0.02         │   LSP: substitutable
//   │ analyzeMarket()          │  │ analyzeMarket()          │   OCP: extend with
//   │ calcPositionSize()       │  │ calcPositionSize()       │   new subclasses
//   └──────────────────────────┘  └──────────────────────────┘
```

#### OOP Concepts Applied

1. **Abstraction** — `TradingStrategy` is declared `abstract` with two `abstract` methods that subclasses must implement. Clients depend only on this abstract type.
2. **Encapsulation** — Each strategy class hides its configuration (window sizes, thresholds) behind `private static final` constants and exposes only the required method implementations.
3. **Inheritance** — `MovingAverageCrossover` and `MeanReversion` extend `TradingStrategy` and inherit `executeTrade()`, `placeOrder()`, and `logTrade()`.
4. **Polymorphism** — The driver declares `List<TradingStrategy>` and iterates polymorphically, calling `executeTrade()` on each reference.

#### SOLID Principles Applied

5. **SRP** — The template method owns the algorithm structure; each concrete strategy owns its analysis logic. No class mixes concerns.
6. **OCP** — New strategies extend `TradingStrategy` without modifying existing code. The template method is `final`, preventing accidental structural changes.
7. **LSP** — All concrete strategies are substitutable for `TradingStrategy`. The driver never checks `instanceof` — it relies on polymorphic dispatch.
8. **ISP** — The abstract class requires exactly two methods. Hooks have default implementations. No class implements methods it does not use.
9. **DIP** — The driver depends on the abstract `TradingStrategy` type. High-level trade execution is decoupled from low-level strategy logic.

### TradingStrategy (AbstractClass)

```java
// ── AbstractClass: TradingStrategy ──
// OOP role: Abstraction + ISP
// SOLID: ISP, DIP
abstract class TradingStrategy {
    protected static final double MIN_CONFIDENCE = 0.15;
    protected static final int MAX_POSITION = 1000;

    // ── Template Method ──
    // 'final' prevents subclasses from overriding the algorithm skeleton.
    public final void executeTrade(String ticker, List<Double> priceData) {
        Decision decision = analyzeMarket(priceData);

        // Hook 1: Pre-validate the signal
        if (!preValidate(decision.signal(), decision.confidence())) {
            System.out.printf("[%s] Trade rejected by pre-validation " +
                "(confidence: %.4f)%n", ticker, decision.confidence());
            return;
        }

        int shares = calculatePositionSize(decision.confidence(), MAX_POSITION);
        if (shares <= 0) {
            System.out.printf("[%s] No trade: position size is zero%n", ticker);
            return;
        }

        // Hook 2: Apply risk controls
        if (!applyRiskControls(ticker, shares)) {
            System.out.printf("[%s] Trade rejected by risk controls " +
                "(%d shares)%n", ticker, shares);
            return;
        }

        Order order = placeOrder(ticker, shares, decision.signal());
        logTrade(ticker, shares, decision.signal(), decision.confidence(), order);
    }

    // ── Primitive Operations ──
    protected abstract Decision analyzeMarket(List<Double> priceData);
    protected abstract int calculatePositionSize(double confidence, int maxPosition);

    // ── Hooks (optional overrides with defaults) ──
    protected boolean preValidate(int signal, double confidence) {
        return confidence >= MIN_CONFIDENCE;
    }

    protected boolean applyRiskControls(String ticker, int shares) {
        return true;
    }

    // ── Concrete Methods ──
    protected Order placeOrder(String ticker, int shares, int signal) {
        return Order.create(ticker, shares, signal);
    }

    protected void logTrade(String ticker, int shares, int signal,
                            double confidence, Order order) {
        String ts = LocalDateTime.now()
            .format(DateTimeFormatter.ofPattern("MM/dd/yyyy, hh:mm:ss a"));
        String direction = signal > 0 ? "BUY" : signal < 0 ? "SELL" : "HOLD";
        System.out.printf("[%s] %s: %s %d shares " +
            "(confidence: %.4f) — Order#%s [%s]%n",
            ts, ticker, direction, shares, confidence,
            order.orderId(), order.status());
    }
}
```

### MovingAverageCrossover (ConcreteClass)

```java
// ── ConcreteClass: MovingAverageCrossover ──
// OOP role: Inheritance/Polymorphism
// SOLID: LSP, OCP, SRP
class MovingAverageCrossover extends TradingStrategy {
    private static final int SHORT_WINDOW = 3;
    private static final int LONG_WINDOW = 8;

    private double movingAverage(List<Double> data, int window) {
        double sum = 0.0;
        for (int i = data.size() - window; i < data.size(); i++) {
            sum += data.get(i);
        }
        return sum / window;
    }

    @Override
    protected Decision analyzeMarket(List<Double> priceData) {
        if (priceData.size() < LONG_WINDOW) {
            return new Decision(0, 0.0);
        }

        double shortMA = movingAverage(priceData, SHORT_WINDOW);
        double longMA = movingAverage(priceData, LONG_WINDOW);

        // Compute previous period moving averages for crossover detection
        List<Double> prevData = priceData.subList(0, priceData.size() - 1);
        double prevShort = movingAverage(prevData, SHORT_WINDOW);
        double prevLong = movingAverage(prevData, LONG_WINDOW);

        if (shortMA > longMA && prevShort <= prevLong) {
            double confidence = Math.min((shortMA - longMA) / longMA, 1.0);
            return new Decision(1, confidence);
        } else if (shortMA < longMA && prevShort >= prevLong) {
            double confidence = Math.min((longMA - shortMA) / longMA, 1.0);
            return new Decision(-1, confidence);
        }

        return new Decision(0, 0.0);
    }

    @Override
    protected int calculatePositionSize(double confidence, int maxPosition) {
        return (int) Math.floor(maxPosition * Math.min(confidence * 1.2, 1.0));
    }
}
```

### MeanReversion (ConcreteClass)

```java
// ── ConcreteClass: MeanReversion ──
// OOP role: Inheritance/Polymorphism
// SOLID: LSP, OCP, SRP
class MeanReversion extends TradingStrategy {
    private static final int LOOKBACK = 10;
    private static final double DEVIATION_THRESHOLD = 0.02;
    private static final int MAX_TRADE_SIZE = 500;
    private static final double MIN_CONFIDENCE_STRICT = 0.25;

    @Override
    protected Decision analyzeMarket(List<Double> priceData) {
        if (priceData.size() < LOOKBACK + 1) {
            return new Decision(0, 0.0);
        }

        double currentPrice = priceData.get(priceData.size() - 1);
        double sum = 0.0;
        for (int i = priceData.size() - LOOKBACK; i < priceData.size(); i++) {
            sum += priceData.get(i);
        }
        double meanPrice = sum / LOOKBACK;
        double deviation = (meanPrice - currentPrice) / meanPrice;

        if (deviation > DEVIATION_THRESHOLD) {
            return new Decision(1, Math.min(deviation, 0.5));
        } else if (deviation < -DEVIATION_THRESHOLD) {
            return new Decision(-1, Math.min(Math.abs(deviation), 0.5));
        }

        return new Decision(0, 0.0);
    }

    @Override
    protected int calculatePositionSize(double confidence, int maxPosition) {
        return (int) Math.floor(maxPosition * confidence * 0.6);
    }

    // Override hook: tighter risk controls
    @Override
    protected boolean applyRiskControls(String ticker, int shares) {
        if (shares > MAX_TRADE_SIZE) {
            return false;
        }
        return super.applyRiskControls(ticker, shares);
    }

    // Override hook: stricter confidence threshold
    @Override
    protected boolean preValidate(int signal, double confidence) {
        return confidence >= MIN_CONFIDENCE_STRICT;
    }
}
```

public class TemplateMethodDemo {

        Map<String, List<Double>> marketData = new LinkedHashMap<>();

            178.50, 179.20, 180.10, 179.80, 181.30,
            182.00, 181.50, 183.20, 184.10, 183.80,
            185.00, 184.50
        ));
        marketData.put("GOOGL", Arrays.asList(
            141.20, 140.80, 140.10, 139.50, 140.00,
            139.20, 138.80, 138.10, 137.50, 138.00,
            137.20, 136.80
        ));
        marketData.put("MSFT", Arrays.asList(
            378.00, 379.50, 380.20, 378.80, 381.00,
            382.30, 383.00, 382.10, 384.50, 385.00,
            383.80, 386.20
        ));

        List<TradingStrategy> strategies = Arrays.asList(
            new MovingAverageCrossover(),
            new MeanReversion()
        );

        for (Map.Entry<String, List<Double>> entry : marketData.entrySet()) {
            String ticker = entry.getKey();
            System.out.println("\n========== " + ticker + " ==========");

            for (TradingStrategy strategy : strategies) {
                String name = strategy.getClass().getSimpleName();
                System.out.println("\n--- " + name + " ---");
                strategy.executeTrade(ticker, entry.getValue());
            }
        }

        System.out.println("\n==========================================");
        System.out.println("Portfolio Summary:");
        System.out.println("All trades executed via the Template Method pattern.");
        System.out.println("Java's 'final' keyword prevented structural changes to the algorithm.");
    }
}

### Language-Specific Notes

- **`final` keyword on template method** — Java's `final` keyword on `executeTrade()` is the strongest enforcement mechanism among all six languages: subclasses physically cannot override the template method. This is the idiomatic Java way to implement the "don't call us, we'll call you" guarantee.
- **Java 17+ records** — `Decision` and `Order` are Java records, providing immutable data carriers with automatically generated `equals()`, `hashCode()`, `toString()`, and accessor methods. The `Order.create()` static factory encapsulates the construction logic.
- **`protected` access control** — Primitive operations and hooks are `protected`, meaning they are visible to subclasses but not to external clients. The template method is `public` because it is the external API. Concrete helper methods (`placeOrder`, `logTrade`) are `protected` so subclasses can call them if needed.
- **`LinkedHashMap` for insertion order** — The market data map uses `LinkedHashMap` to preserve the insertion order of tickers, ensuring AAPL, GOOGL, MSFT appear in sequence in the output rather than alphabetically as `HashMap` would produce.
