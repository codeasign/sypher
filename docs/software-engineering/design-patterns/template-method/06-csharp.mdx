## Implementation

Below is a complete C# implementation of the Template Method pattern using a stock market trading domain. The abstract `TradingStrategy` class defines the `ExecuteTrade()` template method. Concrete strategies `MovingAverageCrossover` and `MeanReversion` implement the primitive operations and optionally override hooks. The driver exercises both strategies with realistic AAPL, GOOGL, and MSFT price data using C# records for immutable data structures and `readonly` structs for lightweight trading decisions.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌─────────────────────────────────────────────────┐
//   │ TradingStrategy (abstract class)                │ ← OOP: Abstraction
//   ├─────────────────────────────────────────────────┤    ISP: focused abstract
//   │ + sealed ExecuteTrade(ticker, prices)            │    method set
//   │ # abstract AnalyzeMarket(prices): Decision       │
//   │ # abstract CalcPositionSize(conf, max): int      │
//   │ # virtual PreValidate(signal, conf): bool         │ ← hook, default true
//   │ # virtual ApplyRiskControls(ticker, sh): bool     │ ← hook, default true
//   │ # PlaceOrder(ticker, shares, signal): Order       │ ← concrete
//   │ # LogTrade(...)                                   │ ← concrete
//   └────────────────────────┬─────────────────────────┘
//                            ▲ extends
//   ┌────────────────────────┴─────────────────────────┐
//   │                                                   │
//   ┌──────────────────────────┐  ┌──────────────────────────┐
//   │ MovingAverageCrossover   │  │ MeanReversion            │
//   ├──────────────────────────┤  ├──────────────────────────┤
//   │ SHORT_WINDOW = 3         │  │ LOOKBACK = 10            │ ← OOP: Polymorphism
//   │ LONG_WINDOW = 8          │  │ THRESHOLD = 0.02         │    LSP: substitutable
//   │ AnalyzeMarket()          │  │ AnalyzeMarket()          │    OCP: extend with
//   │ CalcPositionSize()       │  │ CalcPositionSize()       │    new subclasses
//   └──────────────────────────┘  └──────────────────────────┘
```

#### OOP Concepts Applied

1. **Abstraction** — `TradingStrategy` is declared `abstract` with two `abstract` methods. The `sealed` keyword on `ExecuteTrade()` prevents any further modification of the algorithm structure.
2. **Encapsulation** — Each strategy hides its internal constants and computation logic. Fields are `private const` or `private static readonly`.
3. **Inheritance** — Both concrete strategies inherit from `TradingStrategy` and reuse `PlaceOrder()` and `LogTrade()`.
4. **Polymorphism** — The driver uses a `List<TradingStrategy>` and iterates polymorphically through the strategies.

#### SOLID Principles Applied

5. **SRP** — The template method owns the algorithm structure; concrete strategies own their analysis. No class does more than one thing.
6. **OCP** — New strategies extend `TradingStrategy` without modifying existing code. The `sealed` template method guarantees the skeleton is invariant.
7. **LSP** — All concrete strategies substitute `TradingStrategy` without behavioral issues. The driver never checks concrete types.
8. **ISP** — The abstract contract requires only two method implementations. Hooks are declared `virtual` and provide default behavior.
9. **DIP** — The driver depends on the abstract `TradingStrategy` type. Low-level analysis is decoupled from the high-level workflow.

### TradingStrategy (AbstractClass)

```csharp
// ── AbstractClass: TradingStrategy ──
// OOP role: Abstraction + ISP
// SOLID: ISP, DIP
public abstract class TradingStrategy
{
    protected const double MinConfidence = 0.15;
    protected const int MaxPosition = 1000;

    // ── Template Method ──
    // 'sealed' prevents subclasses from overriding the algorithm skeleton.
    public sealed void ExecuteTrade(string ticker, List<double> priceData)
    {
        Decision decision = AnalyzeMarket(priceData);

        // Hook 1: Pre-validate the signal
        if (!PreValidate(decision.Signal, decision.Confidence))
        {
            Console.WriteLine(
                $"[{ticker}] Trade rejected by pre-validation " +
                $"(confidence: {decision.Confidence:F4})"
            );
            return;
        }

        int shares = CalcPositionSize(decision.Confidence, MaxPosition);
        if (shares <= 0)
        {
            Console.WriteLine($"[{ticker}] No trade: position size is zero");
            return;
        }

        // Hook 2: Apply risk controls
        if (!ApplyRiskControls(ticker, shares))
        {
            Console.WriteLine(
                $"[{ticker}] Trade rejected by risk controls ({shares} shares)"
            );
            return;
        }

        Order order = PlaceOrder(ticker, shares, decision.Signal);
        LogTrade(ticker, shares, decision.Signal, decision.Confidence, order);
    }

    // ── Primitive Operations ──
    protected abstract Decision AnalyzeMarket(List<double> priceData);
    protected abstract int CalcPositionSize(double confidence, int maxPosition);

    // ── Hooks (optional overrides with defaults) ──
    protected virtual bool PreValidate(int signal, double confidence)
    {
        return confidence >= MinConfidence;
    }

    protected virtual bool ApplyRiskControls(string ticker, int shares)
    {
        return true;
    }

    // ── Concrete Methods ──
    protected Order PlaceOrder(string ticker, int shares, int signal)
    {
        return Order.Create(ticker, shares, signal);
    }

    protected void LogTrade(string ticker, int shares, int signal,
                            double confidence, Order order)
    {
        string ts = DateTime.Now.ToString("MM/dd/yyyy, hh:mm:ss tt");
        string direction = signal > 0 ? "BUY" : signal < 0 ? "SELL" : "HOLD";
        Console.WriteLine(
            $"[{ts}] {ticker}: {direction} {shares} shares " +
            $"(confidence: {confidence:F4}) — Order#{order.OrderId} [{order.Status}]"
        );
    }
}
```

### MovingAverageCrossover (ConcreteClass)

```csharp
// ── ConcreteClass: MovingAverageCrossover ──
// OOP role: Inheritance/Polymorphism
// SOLID: LSP, OCP, SRP
public class MovingAverageCrossover : TradingStrategy
{
    private const int ShortWindow = 3;
    private const int LongWindow = 8;

    private static double MovingAverage(List<double> data, int window)
    {
        return data.TakeLast(window).Average();
    }

    protected override Decision AnalyzeMarket(List<double> priceData)
    {
        if (priceData.Count < LongWindow)
        {
            return new Decision(0, 0.0);
        }

        double shortMA = MovingAverage(priceData, ShortWindow);
        double longMA = MovingAverage(priceData, LongWindow);

        // Detect crossover by comparing with previous period
        var prevData = priceData.Take(priceData.Count - 1).ToList();
        double prevShort = MovingAverage(prevData, ShortWindow);
        double prevLong = MovingAverage(prevData, LongWindow);

        if (shortMA > longMA && prevShort <= prevLong)
        {
            double confidence = Math.Min((shortMA - longMA) / longMA, 1.0);
            return new Decision(1, confidence);
        }
        else if (shortMA < longMA && prevShort >= prevLong)
        {
            double confidence = Math.Min((longMA - shortMA) / longMA, 1.0);
            return new Decision(-1, confidence);
        }

        return new Decision(0, 0.0);
    }

    protected override int CalcPositionSize(double confidence, int maxPosition)
    {
        return (int)Math.Floor(maxPosition * Math.Min(confidence * 1.2, 1.0));
    }
}
```

### MeanReversion (ConcreteClass)

```csharp
// ── ConcreteClass: MeanReversion ──
// OOP role: Inheritance/Polymorphism
// SOLID: LSP, OCP, SRP
public class MeanReversion : TradingStrategy
{
    private const int Lookback = 10;
    private const double DeviationThreshold = 0.02;
    private const int MaxTradeSize = 500;
    private const double MinConfidenceStrict = 0.25;

    protected override Decision AnalyzeMarket(List<double> priceData)
    {
        if (priceData.Count < Lookback + 1)
        {
            return new Decision(0, 0.0);
        }

        double currentPrice = priceData[^1];
        double meanPrice = priceData.TakeLast(Lookback).Average();
        double deviation = (meanPrice - currentPrice) / meanPrice;

        if (deviation > DeviationThreshold)
        {
            return new Decision(1, Math.Min(deviation, 0.5));
        }
        else if (deviation < -DeviationThreshold)
        {
            return new Decision(-1, Math.Min(Math.Abs(deviation), 0.5));
        }

        return new Decision(0, 0.0);
    }

    protected override int CalcPositionSize(double confidence, int maxPosition)
    {
        return (int)Math.Floor(maxPosition * confidence * 0.6);
    }

    // Override hook: tighter risk controls
    protected override bool ApplyRiskControls(string ticker, int shares)
    {
        if (shares > MaxTradeSize)
        {
            return false;
        }
        return base.ApplyRiskControls(ticker, shares);
    }

    // Override hook: stricter confidence threshold
    protected override bool PreValidate(int signal, double confidence)
    {
        return confidence >= MinConfidenceStrict;
    }
}
```

### Language-Specific Notes

- **`sealed` keyword on template method** — C# provides the `sealed` keyword to prevent derived classes from overriding the template method. This is the direct equivalent of Java's `final` and enforces the pattern's invariant algorithm structure at compile time.
- **`readonly record struct` for lightweight decisions** — The `Decision` type uses `readonly record struct`, which is a value type (allocated on the stack) rather than a reference type. This minimizes allocation pressure when decisions are created frequently during backtesting or live trading. The `Order` type uses a reference `record` because orders carry more data and are passed around, not copied.
- **`virtual` for hooks, `abstract` for required methods** — C# distinguishes hooks (`virtual` with a default body) from primitive operations (`abstract` with no body). This design communicates intent clearly to developers extending the class: `abstract` methods must be implemented; `virtual` methods may be overridden if needed.
- **Collection expressions and `TakeLast()`** — The code uses C# 12 collection expressions (`new() { ... }`) and LINQ's `TakeLast()` and `Average()` for concise data manipulation. The `priceData[^1]` syntax (index from end operator) provides clean access to the most recent price.
