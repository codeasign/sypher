## Implementation

Below is a complete JavaScript ES2022+ implementation of the Flyweight pattern using a car configurator domain. The `CarSpec` interface defines the flyweight contract, and three concrete flyweights (`EngineSpec`, `WheelSpec`, `PaintSpec`) carry the intrinsic state shared across many car configurations. The `CarSpecFactory` caches and returns flyweight objects by key, and each `CarConfiguration` stores its own extrinsic state (VIN, owner, license plate, production date) alongside references to shared flyweights.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌──────────────────────────────────────┐
//   │   <<interface>>                      │
//   │   CarSpec                             │  ← OOP: Abstraction
//   ├──────────────────────────────────────┤     ISP: minimal contract
//   │ + display(): string                  │
//   └──────────────────┬───────────────────┘
//                      ▲ implements
//   ┌──────────────────┼───────────────────┐
//   │                  │                   │
//   ┌────────────────┐ ┌────────────────┐ ┌────────────────┐
//   │   EngineSpec   │ │   WheelSpec    │ │   PaintSpec    │
//   ├────────────────┤ ├────────────────┤ ├────────────────┤
//   │ - model: string│ │ - dia: number  │ │ - code: string │
//   │ - hp: number   │ │ - width: number│ │ - name: string │

//   │ - cyl: number  │ │ - brand: string│ │ - hex: string  │

//   └────────────────┘ └────────────────┘ └────────────────┘
//
//   ┌────────────────────────────────────────────────────────┐
//   │   CarSpecFactory (FlyweightFactory)                    │  ← SRP: manages
//   ├────────────────────────────────────────────────────────┤     flyweight
//   │ - engines: Map<string, EngineSpec>                     │     cache
//   │ - wheels:  Map<string, WheelSpec>                      │
//   │ - paints:  Map<string, PaintSpec>                      │
//   │ + getEngine(k, hp, disp, cyl, fuel): EngineSpec        │
//   │ + getWheel(k, dia, width, mat, brand): WheelSpec       │
//   │ + getPaint(k, name, finish, hex): PaintSpec            │
//   │ + getStats(): string                                   │
//   └────────────────────────────────────────────────────────┘
//
//   ┌────────────────────────────────────────────────────────┐
//   │   CarConfiguration (Client)                            │  ← OOP: Encaps
//   ├────────────────────────────────────────────────────────┤     SRP: per-car
//   │ - vin: string, - owner: string, - plate: string       │     extrinsic
//   │ - prodDate: string                                     │     state + refs
//   │ - engine: EngineSpec (shared)                          │     to flyweights
//   │ - wheels: WheelSpec (shared)                           │
//   │ - paint: PaintSpec (shared)                            │
//   │ + display(): string                                    │
//   └────────────────────────────────────────────────────────┘
```

#### OOP Concepts Applied

1. **Abstraction** — `CarSpec` defines the `display()` contract. All client code depends on this abstraction, not on concrete flyweight classes.

2. **Encapsulation** — Each flyweight hides its intrinsic state behind the `display()` method. The factory hides its cache implementation.

3. **Polymorphism** — `CarConfiguration.display()` calls `display()` on any `CarSpec` reference. The correct formatting logic runs per type.

#### SOLID Principles Applied

4. **SRP** — Each flyweight class has one responsibility: represent a specific type of car spec. The factory has one responsibility: manage the cache. `CarConfiguration` has one responsibility: represent a single car configuration.

5. **OCP** — Adding a new flyweight type (e.g., `InteriorSpec`) requires creating a new class that implements `CarSpec` and adding a new cache to the factory. No existing code needs modification.

6. **LSP** — Every concrete flyweight can substitute for `CarSpec`. The `CarConfiguration` treats all flyweights uniformly through the `display()` interface.

7. **ISP** — The `CarSpec` interface is minimal: only `display()`. No flyweight is forced to implement methods it does not need.

8. **DIP** — `CarConfiguration` depends on the abstract `CarSpec` interface, not on concrete flyweight classes. The configuration stores references typed as `CarSpec`.

### CarSpec (Flyweight Interface)

```javascript
// Flyweight Pattern — JavaScript — Car Configurator Example
// ==========================================================
// Participants: CarSpec (Flyweight), EngineSpec/WheelSpec/PaintSpec
//   (Concrete Flyweights), CarSpecFactory (FlyweightFactory),
//   CarConfiguration (Client)
// OOP: abstraction, encapsulation, polymorphism
// SOLID: SRP, OCP, LSP, ISP, DIP
// ==========================================================

// ── Flyweight interface ──
// OOP role: Abstraction + ISP
// SOLID: ISP, DIP
class CarSpec {
  display() {
    return '';
  }
}

```

### EngineSpec

```javascript
// ── Concrete Flyweight: EngineSpec ──
// Intrinsic state: model, horsepower, displacement, cylinders, fuel type
// OOP role: Inheritance/Polymorphism
// SOLID: LSP, SRP
class EngineSpec extends CarSpec {
  constructor(model, horsepower, displacement, cylinders, fuelType) {
    super();
    this.model = model;
    this.horsepower = horsepower;
    this.displacement = displacement;
    this.cylinders = cylinders;
    this.fuelType = fuelType;
    Object.freeze(this);
  }

  display() {
    return `${this.model} | ${this.horsepower} hp | ${this.displacement}L | ${this.cylinders} cyl | ${this.fuelType}`;
  }
}

```

### CarSpecFactory

```javascript
// ── Flyweight Factory ──
// OOP role: Composition
// SOLID: SRP (manages flyweight cache), DIP (returns CarSpec abstractions)
class CarSpecFactory {
  constructor() {
    this.engines = new Map();
    this.wheels = new Map();
    this.paints = new Map();
  }

  getEngine(model, horsepower, displacement, cylinders, fuelType) {
    const key = model;
    if (!this.engines.has(key)) {
      this.engines.set(
        key,
        new EngineSpec(model, horsepower, displacement, cylinders, fuelType)
      );
    }
    return this.engines.get(key);
  }

  getWheel(key, diameter, width, material, brand) {
    if (!this.wheels.has(key)) {
      this.wheels.set(key, new WheelSpec(diameter, width, material, brand));
    }
    return this.wheels.get(key);
  }

  getPaint(code, name, finish, hexColor) {
    if (!this.paints.has(code)) {
      this.paints.set(code, new PaintSpec(code, name, finish, hexColor));
    }
    return this.paints.get(code);
  }

  getStats() {
    return [
      `Flyweight Cache Stats:`,
      `  Engines: ${this.engines.size} unique`,
      `  Wheels:  ${this.wheels.size} unique`,
      `  Paints:  ${this.paints.size} unique`
    ].join('\n');
  }
}

```

### CarConfiguration

```javascript
// ── Client: CarConfiguration ──
// Extrinsic state: VIN, owner, license plate, production date
// References to shared flyweights: engine, wheels, paint
// OOP role: Encapsulation, Composition
// SOLID: SRP (represents one car), DIP (depends on CarSpec abstractions)
class CarConfiguration {
  constructor(vin, owner, licensePlate, productionDate, engine, wheels, paint) {
    this.vin = vin;
    this.owner = owner;
    this.licensePlate = licensePlate;
    this.productionDate = productionDate;
    this.engine = engine;
    this.wheels = wheels;
    this.paint = paint;
  }

  display() {
    const lines = [
      `--- Car Configuration ---`,
      `VIN:             ${this.vin}`,
      `Owner:           ${this.owner}`,
      `License Plate:   ${this.licensePlate}`,
      `Production Date: ${this.productionDate}`,
      `Engine:          ${this.engine.display()}`,
      `Wheels:          ${this.wheels.display()}`,
      `Paint:           ${this.paint.display()}`,
      `---`
    ];
    return lines.join('\n');
  }
}

```

### Language-Specific Notes

- **`Object.freeze()` for immutability** — The flyweight constructor calls `Object.freeze(this)` after setting all properties. This prevents any accidental mutation of shared intrinsic state. In JavaScript, frozen objects cannot have properties added, removed, or changed. Any attempt to modify a frozen flyweight silently fails in non-strict mode or throws in strict mode.

- **`Map` for factory cache** — The factory uses `Map` objects instead of plain `{}` objects for the cache. `Map` provides better performance for frequent get/set operations, accepts any value type as a key, and has a predictable iteration order. The `Map.prototype.has()` method serves as the cache check before creating a new flyweight.

- **Reference equality for sharing verification** — The driver uses `===` to verify that two `CarConfiguration` objects reference the same flyweight instance. This is only possible because the factory returns the same object reference for the same key. In a non-flyweight implementation, two cars with the same engine would have different `EngineSpec` objects and `===` would return `false`.

- **ES2022+ class syntax** — The implementation uses ES2022 class syntax with `extends` for inheritance. The `CarSpec` base class provides a default `display()` method that concrete flyweights override. The `constructor` pattern with `super()` ensures proper prototype chain setup.
