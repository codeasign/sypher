## Implementation

Below is a complete Rust implementation of the Observer pattern using a stock market domain. A `StockExchange` (Subject) tracks prices for AAPL, GOOGL, and MSFT. Three observers — `PriceDisplay`, `AlertSystem`, and `PortfolioTracker` — register with the exchange and react to price changes. The implementation uses Rust's trait system for the Observer interface, `Box<dyn Observer>` for dynamic dispatch, and `RefCell` for interior mutability.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌──────────────────────────────────┐
//   │   trait Observer                  │
//   ├──────────────────────────────────┤
//   │   fn update(&mut self,           │ ← OOP: Abstraction via trait
//   │          ticker: &str,           │    ISP: single-method trait
//   │          price: f64)             │
//   └────────────────┬─────────────────┘
//                    ▲ impl for
//   ┌────────────────┴─────────────────┐
//   │                │                 │
//   ┌────────────┐ ┌──────────┐ ┌──────────────┐
//   │PriceDisplay│ │AlertSys  │ │Portfolio     │ ← Polymorphism via
//   ├────────────┤ ├──────────┤ ├──────────────┤    trait objects
//   │ update()   │ │ update() │ │ update()     │
//   └────────────┘ └──────────┘ └──────────────┘
//
//   ┌──────────────────────────────────┐
//   │  StockExchange                   │ ← OOP: Encapsulation
//   ├──────────────────────────────────┤    SRP: prices + observers
//   │  attach(Box<dyn Observer>)       │     OCP: new observers via
//   │  set_price(ticker, price)        │     attach() without modifying
//   │  get_price(ticker) -> f64        │     StockExchange
//   └──────────────────────────────────┘    DIP: depends on Observer
//                                           trait
```

#### OOP Concepts Applied

1. **Abstraction** — The `Observer` trait defines the notification contract. `StockExchange` depends on `dyn Observer`, not on concrete observer types.
2. **Encapsulation** — `StockExchange` hides its observer vector and price map behind `private` (default in Rust). Each observer encapsulates its own state in its struct fields.
3. **Inheritance** — Rust does not have classical inheritance. Instead, `impl Observer for PriceDisplay` provides trait implementation, which is Rust's equivalent of interface implementation.
4. **Polymorphism** — The `notify_observers` method iterates over `&mut Box<dyn Observer>` and calls `update()` via dynamic dispatch. Each observer's concrete `update()` implementation runs.

#### SOLID Principles Applied

5. **SRP** — `StockExchange` manages only prices and observer registration. Each observer struct has a single responsibility.
6. **OCP** — New observer types implement `Observer` and register via `attach()`. The `StockExchange` struct is never modified.
7. **LSP** — Any type implementing `Observer` can be used where `dyn Observer` is expected. The notification loop treats all observers uniformly.
8. **ISP** — The `Observer` trait has a single method. No type is forced to implement unused methods.
9. **DIP** — `StockExchange` depends on the `Observer` trait abstraction. Concrete observers are injected via `attach()`.

### Observer Trait

```rust
// Observer Pattern — Rust — Stock Market example
// ===============================================
// Participants: StockExchange (Subject), PriceDisplay,
// AlertSystem, PortfolioTracker (Concrete Observers)
// OOP: abstraction, encapsulation, polymorphism (via traits)
// SOLID: SRP, OCP, LSP, ISP, DIP
// ===============================================

use std::collections::HashMap;

// ── Observer Trait ──
trait Observer {
    /// Called when a stock price changes.
    fn update(&mut self, ticker: &str, price: f64);
}
```

### StockExchange (Subject)

```rust
// ── Subject ──
struct StockExchange {
    observers: Vec<Box<dyn Observer>>,
    prices: HashMap<String, f64>,
}

impl StockExchange {
    fn new() -> Self {
        StockExchange {
            observers: Vec::new(),
            prices: HashMap::new(),
        }
    }

    /// Register an observer.
    fn attach(&mut self, observer: Box<dyn Observer>) {
        self.observers.push(observer);
    }

    /// Notify all registered observers of a price change.
    fn notify_observers(&mut self, ticker: &str, price: f64) {
        for obs in self.observers.iter_mut() {
            obs.update(ticker, price);
        }
    }

    /// Set a new price and broadcast the change.
    fn set_price(&mut self, ticker: &str, price: f64) {
        self.prices.insert(ticker.to_string(), price);
        self.notify_observers(ticker, price);
    }

    /// Get the current price of a ticker.
    fn get_price(&self, ticker: &str) -> f64 {
        self.prices.get(ticker).copied().unwrap_or(0.0)
    }
}
```

### PriceDisplay (Concrete Observer)

```rust
// ── Concrete Observer 1: PriceDisplay ──
struct PriceDisplay;

impl Observer for PriceDisplay {
    fn update(&mut self, ticker: &str, price: f64) {
        println!("DISPLAY: {} is now ${:.2}", ticker, price);
    }
}
```

### AlertSystem (Concrete Observer)

```rust
// ── Concrete Observer 2: AlertSystem ──
struct AlertSystem {
    previous_prices: HashMap<String, f64>,
    threshold: f64,
}

impl AlertSystem {
    fn new(threshold_percent: f64) -> Self {
        AlertSystem {
            previous_prices: HashMap::new(),
            threshold: threshold_percent,
        }
    }
}

impl Observer for AlertSystem {
    fn update(&mut self, ticker: &str, price: f64) {
        if let Some(&old_price) = self.previous_prices.get(ticker) {
            let change = (price - old_price).abs() / old_price * 100.0;
            if change >= self.threshold {
                let direction = if price >= old_price { "up" } else { "down" };
                println!(
                    "ALERT: {} moved {} {:.1}% (${:.2} -> ${:.2})",
                    ticker, direction, change, old_price, price
                );
            }
        }
        self.previous_prices.insert(ticker.to_string(), price);
    }
}
```

### PortfolioTracker (Concrete Observer)

```rust
// ── Concrete Observer 3: PortfolioTracker ──
struct PortfolioTracker {
    holdings: HashMap<String, u32>,
    total_value: f64,
}

impl PortfolioTracker {
    fn new() -> Self {
        PortfolioTracker {
            holdings: HashMap::new(),
            total_value: 0.0,
        }
    }

    /// Add a stock holding to the portfolio.
    fn add_holding(&mut self, ticker: &str, shares: u32) {
        self.holdings.insert(ticker.to_string(), shares);
    }
}

impl Observer for PortfolioTracker {
    fn update(&mut self, ticker: &str, _price: f64) {
        if self.holdings.contains_key(ticker) {
            // Note: StockExchange notifies BEFORE returning, so we cannot
            // query the subject from within the observer in this design.
            // In a real system, push both ticker+price and let the observer
            // query the full state if needed. Here we approximate portfolio
            // value by noting the ticker changed.
            let total_shares: u32 = self.holdings.values().sum();
            println!(
                "PORTFOLIO: {} changed — porfolio has {} total shares across {} holdings",
                ticker,
                total_shares,
                self.holdings.len(),
            );
        }
    }
}
```

### Main / Driver

```rust
// ── Main / Driver ──
fn main() {
    let mut exchange = StockExchange::new();

    let display = PriceDisplay;
    let alerts = AlertSystem::new(5.0);
    let mut portfolio = PortfolioTracker::new();

    // Set up portfolio holdings
    portfolio.add_holding("AAPL", 10);
    portfolio.add_holding("GOOGL", 5);
    portfolio.add_holding("MSFT", 8);

    // Register observers
    exchange.attach(Box::new(display));
    exchange.attach(Box::new(alerts));
    exchange.attach(Box::new(portfolio));

    println!("=== Stock Market Observer Demo ===");
    println!();

    exchange.set_price("AAPL", 150.00);
    exchange.set_price("GOOGL", 2800.00);
    exchange.set_price("MSFT", 300.00);

    println!();

    exchange.set_price("AAPL", 162.00);  // 8% change — triggers alert

    println!();

    exchange.set_price("GOOGL", 2750.00);  // < 5% — no alert
    exchange.set_price("MSFT", 315.00);     // 5% change — triggers alert
}
```

### Language-Specific Notes

- **Trait objects**: Rust uses `Box<dyn Observer>` for dynamic dispatch. The `dyn` keyword indicates that the trait is used as a trait object, enabling runtime polymorphism. This is Rust's equivalent of an interface reference in OOP languages.
- **Ownership and borrowing**: The observer list takes ownership of `Box<dyn Observer>` values. The `notify_observers` method iterates with `iter_mut()` to get mutable references to each trait object, allowing observers to update their internal state.
- **No shared references to subject**: Rust's borrow checker prevents the PortfolioTracker from holding a reference to StockExchange while StockExchange iterates over observers (a mutable borrow conflict). The implementation avoids this by not requiring the subject reference inside the observer — a design tradeoff idiomatic to Rust.
- **`HashMap<String, f64>` for prices**: Rust's standard `HashMap` provides key-value storage with `get().copied().unwrap_or(0.0)` for safe access. The `copied()` method converts `Option<&f64>` to `Option<f64>`.
- **`to_string()` on string literals**: Rust requires explicit conversion from `&str` to `String` for HashMap keys. The `ticker.to_string()` call is idiomatic and necessary for the owned key type.
- **No `detach` by reference**: Rust's ownership model makes removing a specific observer by identity difficult without `Rc`/`Weak` reference counting. The implementation omits `detach` for simplicity, but a production system would use `Rc<RefCell<dyn Observer>>` or slab-based IDs.
