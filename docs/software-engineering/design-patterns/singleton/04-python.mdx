## Implementation

Below is a complete Python implementation of the Singleton pattern using a banking domain. The `BankConfiguration` class overrides `__new__` to control instance creation, ensuring only one configuration object exists. Thread safety is provided by a `threading.Lock`. Three client classes — `LoanDepartment`, `TransactionService`, and `ComplianceSystem` — access the same `BankConfiguration` instance, guaranteeing consistent interest rates, fee structures, and regulatory settings across the entire banking application.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌──────────────────────────────────────┐
//   │   BankConfiguration (Singleton)      │  ← OOP: Encapsulation
//   ├──────────────────────────────────────┤     SRP: holds config data
//   │   - _instance: BankConfiguration     │
//   │   - _lock: threading.Lock            │
//   │   - _base_interest_rate: float       │
//   │   - _wire_transfer_fee: float        │
//   │   - ... (other config properties)    │
//   ├──────────────────────────────────────┤
//   │   + __new__(cls)                     │  ← Controls instance creation
//   │   + get_base_interest_rate(): float  │
//   │   + get_wire_transfer_fee(): float   │
//   │   + get_kyc_api_endpoint(): str      │
//   └──────────────────────────────────────┘
```

#### OOP Concepts Applied

1. **Encapsulation** — The `BankConfiguration` class hides all configuration data behind private attributes (prefix `_`) and exposes only public getter methods. Clients cannot accidentally modify configuration values.

2. **Abstraction** — Clients interact with the configuration through meaningful method names (`get_base_interest_rate()`, `get_wire_transfer_fee()`), without needing to know how the configuration is loaded or stored.

3. **Polymorphism** — Not directly applicable to the Singleton pattern, but the three client classes all use the same `BankConfiguration` interface polymorphically.

#### SOLID Principles Applied

4. **SRP** — `BankConfiguration` is responsible only for holding configuration data and controlling its single instance. Each client class has a distinct responsibility: loan processing, transaction handling, or compliance verification.

5. **OCP** — The system is open for extension. New configuration properties can be added to `BankConfiguration` without modifying any client code. New client classes can access the singleton without changes to the singleton.

6. **LSP** — While there is no inheritance hierarchy, the typed getter methods provide consistent behavior that any client can rely on.

7. **ISP** — Each getter method returns a single value. Clients that need only interest rates do not depend on methods related to KYC endpoints or SWIFT codes.

8. **DIP** — Clients depend on the `BankConfiguration` class through its public getter interface, not on the details of how the single instance is managed.

### BankConfiguration (Singleton)

```python
# Singleton Pattern — Python — Banking example
# ==================================================
# Participants: BankConfiguration (Singleton),
# LoanDepartment, TransactionService, ComplianceSystem (Clients)
# OOP: encapsulation, abstraction
# SOLID: SRP, OCP, ISP, DIP
# ==================================================

import threading
from typing import Optional


# ── Singleton ──
class BankConfiguration:
    """Singleton that holds global banking configuration.

    Overrides __new__ to ensure only one instance exists.
    Uses a threading.Lock for thread-safe lazy initialization.
    """

    _instance: Optional["BankConfiguration"] = None
    _lock: threading.Lock = threading.Lock()
    _initialized: bool = False

    def __new__(cls) -> "BankConfiguration":
        if cls._instance is None:
            with cls._lock:
                # Double-checked locking pattern
                if cls._instance is None:
                    instance = super().__new__(cls)
                    cls._instance = instance
        return cls._instance

    def __init__(self) -> None:
        # Prevent re-initialization on subsequent calls to __new__
        if self._initialized:
            return
        self._initialized = True

        # Load all configuration values
        self._base_interest_rate: float = 4.5
        self._savings_interest_rate: float = 3.75
        self._wire_transfer_fee: float = 25.00
        self._max_transaction_limit: float = 50000.00
        self._min_balance_requirement: float = 500.00
        self._kyc_api_endpoint: str = "https://api.globaltrustbank.com/kyc/v2"
        self._bank_name: str = "Global Trust Bank"
        self._swift_code: str = "GTBKUS44"
        self._reserve_requirement: float = 0.10

    # Public getter methods
    def get_base_interest_rate(self) -> float:
        return self._base_interest_rate

    def get_savings_interest_rate(self) -> float:
        return self._savings_interest_rate

    def get_wire_transfer_fee(self) -> float:
        return self._wire_transfer_fee

    def get_max_transaction_limit(self) -> float:
        return self._max_transaction_limit

    def get_min_balance_requirement(self) -> float:
        return self._min_balance_requirement

    def get_kyc_api_endpoint(self) -> str:
        return self._kyc_api_endpoint

    def get_bank_name(self) -> str:
        return self._bank_name

    def get_swift_code(self) -> str:
        return self._swift_code

    def get_reserve_requirement(self) -> float:
        return self._reserve_requirement
```

### LoanDepartment (Client 1)

```python
# ── Client 1: LoanDepartment ──
class LoanDepartment:
    """Handles loan processing using the singleton configuration."""

    def process_loan_application(self, amount: float, credit_score: int) -> None:
        config = BankConfiguration()
        rate = config.get_base_interest_rate()
        if credit_score > 700:
            rate = rate - 1.5
        annual_interest = amount * (rate / 100)
        print(f"LoanDepartment: Processing ${amount:,.2f} loan application")
        print(f"  Interest rate: {rate}%")
        print(f"  Annual interest: ${annual_interest:,.2f}")
        print(f"  Bank: {config.get_bank_name()}")
        print(f"  SWIFT: {config.get_swift_code()}")
```

### TransactionService (Client 2)

```python
# ── Client 2: TransactionService ──
class TransactionService:
    """Handles wire transfers and rate queries using the singleton configuration."""

    def process_wire_transfer(self, amount: float) -> None:
        config = BankConfiguration()
        if amount > config.get_max_transaction_limit():
            print(f"TransactionService: REJECTED — ${amount:,.2f} exceeds "
                  f"limit of ${config.get_max_transaction_limit():,.2f}")
            return
        fee = config.get_wire_transfer_fee()
        total = amount + fee
        print(f"TransactionService: Processing wire transfer of ${amount:,.2f}")
        print(f"  Wire fee: ${fee:,.2f}")
        print(f"  Total debit: ${total:,.2f}")
        print(f"  Bank: {config.get_bank_name()}")

    def check_savings_rate(self) -> None:
        config = BankConfiguration()
        rate = config.get_savings_interest_rate()
        print(f"TransactionService: Current savings interest rate is {rate}%")
```

### ComplianceSystem (Client 3)

### Main / Driver

```python
# ── Main / Driver ──
def main() -> None:
    print("=== Banking System Initialization ===\n")

    loan_dept = LoanDepartment()
    transaction_service = TransactionService()
    compliance = ComplianceSystem()

    loan_dept.process_loan_application(250000.00, 720)
    print()
    transaction_service.process_wire_transfer(15000.00)
    print()
    transaction_service.check_savings_rate()
    print()
    compliance.verify_customer("CUST-4201")
    print()

    # Singleton identity verification
    print("=== Singleton Identity Verification ===")
    ref1 = BankConfiguration()
    ref2 = BankConfiguration()
    ref3 = BankConfiguration()
    print(f"ref1 is ref2: {ref1 is ref2}")
    print(f"ref2 is ref3: {ref2 is ref3}")
    all_same = (ref1 is ref2) and (ref2 is ref3)
    print(f"All references point to the same instance: {all_same}")

    # Thread-safety demonstration
    print("\n=== Thread Safety Test ===")
    instances = []

    def create_ref() -> None:
        instances.append(BankConfiguration())

    threads = [threading.Thread(target=create_ref) for _ in range(10)]
    for t in threads:
        t.start()
    for t in threads:
        t.join()

    all_match = all(i is instances[0] for i in instances)
    print(f"10 threads all got the same instance: {all_match}")


if __name__ == "__main__":
    main()
```

### Language-Specific Notes

- **`__new__` override** — Python controls instance creation through `__new__`, not the constructor. Overriding `__new__` is the standard way to implement the Singleton pattern. The `__init__` method is guarded by the `_initialized` flag to prevent re-initialization on subsequent calls (since `__init__` is called every time `BankConfiguration()` is invoked, even if `__new__` returns the existing instance).

- **Thread safety with Lock** — Python's `threading.Lock` provides thread-safe lazy initialization. The double-checked locking pattern (check `_instance is None` outside the lock, then again inside) minimizes lock contention once the instance is initialized.

- **`is` identity comparison** — Python uses `is` (not `==`) to check whether two references point to the same object. The driver demonstrates that all calls to `BankConfiguration()` return the same object.

- **Module-level singleton** — In Python, modules themselves are singletons. An alternative approach is to create the instance at module level and import it: `config = BankConfiguration()`. This is simpler and thread-safe at module import time, but gives less control over initialization timing.

- **Metaclass alternative** — For production code, a `SingletonMeta` metaclass can centralize the singleton logic. Defining `class SingletonMeta(type)` with a custom `__call__` method allows any class using `metaclass=SingletonMeta` to become a singleton without duplicating the uniqueness logic.
