## Implementation

Below is a complete Python 3.11+ implementation of the Prototype pattern using a banking domain. The `AccountPrototype` abstract base class defines the `clone()` contract. Three concrete prototypes (`StandardSavingsPrototype`, `PremiumCheckingPrototype`, `BusinessLoanPrototype`) carry pre-configured default values. A `BankBranch` client maintains a prototype registry and clones templates on demand for customer onboarding.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌──────────────────────────────────────────┐
//   │   <<abstract class>>                     │
//   │   AccountPrototype (ABC)                 │  ← OOP: Abstraction
//   ├──────────────────────────────────────────┤     ISP: focused contract
//   │ + clone(): AccountPrototype              │     (clone + config)
//   │ + display_config(): str                  │
//   └────────────────────┬─────────────────────┘
//                        ▲ inherits
//   ┌────────────────────┼─────────────────────┐
//   │                    │                     │
//   ┌──────────────────┐ ┌──────────────────┐ ┌──────────────────────┐
//   │ StandardSavings  │ │ PremiumChecking  │ │ BusinessLoan        │
//   │ Prototype        │ │ Prototype        │ │ Prototype           │
//   ├──────────────────┤ ├──────────────────┤ ├──────────────────────┤
//   │ - rate: 0.035    │ │ - rate: 0.001   │ │ - apr: 0.0675       │
//   │ - fee: 0.0       │ │ - fee: 25.0     │ │ - fee: 500.0        │
//   │ - minBal: 100.0  │ │ - minBal: 1500.0│ │ - maxPrin: 50000.0  │
//   │ + clone()        │ │ + clone()       │ │ + clone()           │
//   └──────────────────┘ └──────────────────┘ └──────────────────────┘
//
//   ┌──────────────────────────────────────────┐
//   │   BankBranch                             │  ← OOP: Encapsulation
//   ├──────────────────────────────────────────┤     SRP: template registry
//   │ - prototypes: dict                       │     DIP: uses abstract type
//   │ + open_account(type, holder, amount)     │
//   └──────────────────────────────────────────┘
```

#### OOP Concepts Applied

1. **Abstraction** — `AccountPrototype` is an abstract base class (ABC) defining the `clone()` and `display_config()` interface. All client code depends on this abstraction.

2. **Encapsulation** — Each concrete prototype hides its internal configuration fields (interest rate, fees, limits) as instance attributes. `display_config()` provides controlled access to the state.

3. **Inheritance** — `StandardSavingsPrototype`, `PremiumCheckingPrototype`, and `BusinessLoanPrototype` all inherit from `AccountPrototype` and share the same interface.

4. **Polymorphism** — `BankBranch.open_account()` calls `clone()` on any `AccountPrototype` reference. The correct cloning logic executes per type without explicit type checks.

#### SOLID Principles Applied

5. **SRP** — Each prototype class has exactly one responsibility: represent a specific financial product template. `BankBranch` manages the registry and cloning orchestration.

6. **OCP** — Adding a new `StudentLoanPrototype` requires creating a new class that inherits from `AccountPrototype` and registering it in `BankBranch`. No existing prototype classes are modified.

7. **LSP** — Every concrete prototype is substitutable for `AccountPrototype`. The `BankBranch` treats all prototypes uniformly through the `clone()` interface.

8. **ISP** — The `AccountPrototype` interface is minimal with only two methods. No class is forced to implement methods it does not need.

9. **DIP** — `BankBranch` depends on the abstract `AccountPrototype` type, not on concrete prototype classes. The registry stores prototypes by their abstract type.

### Prototype (Abstract Base Class)

```python
# Prototype Pattern — Python — Banking example
# ==============================================
# Participants: AccountPrototype (Prototype),
#   StandardSavingsPrototype, PremiumCheckingPrototype,
#   BusinessLoanPrototype (Concrete Prototypes),
#   BankBranch (Client)
# OOP: abstraction, encapsulation, inheritance, polymorphism
# SOLID: SRP, OCP, LSP, ISP, DIP
# ==============================================

from abc import ABC, abstractmethod


# ── Prototype (Abstract Base Class) ──
# OOP role: Abstraction + ISP
# SOLID: ISP, DIP
class AccountPrototype(ABC):
    """Abstract prototype — all account templates implement this."""

    @abstractmethod
    def clone(self) -> 'AccountPrototype':
        """Create a copy of this prototype."""
        ...

    @abstractmethod
    def display_config(self) -> str:
        """Return a formatted string of all configuration fields."""
        ...
```

### StandardSavingsPrototype

```python
# ── Concrete Prototype: Standard Savings ──
# OOP role: Inheritance/Polymorphism
# SOLID: LSP, OCP, SRP
class StandardSavingsPrototype(AccountPrototype):
    """Pre-configured Standard Savings account template."""

    def __init__(self) -> None:
        self.account_type: str = 'Standard Savings'
        self.interest_rate: float = 0.035
        self.monthly_fee: float = 0.0
        self.min_balance: float = 100.0
        self.withdrawal_limit: int = 6
        self.overdraft_protection: bool = False
        self.account_holder: str = ''
        self.balance: float = 0.0

    def clone(self) -> 'StandardSavingsPrototype':
        cloned = StandardSavingsPrototype()
        cloned.interest_rate = self.interest_rate
        cloned.monthly_fee = self.monthly_fee
        cloned.min_balance = self.min_balance
        cloned.withdrawal_limit = self.withdrawal_limit
        cloned.overdraft_protection = self.overdraft_protection
        cloned.account_holder = self.account_holder
        cloned.balance = self.balance
        return cloned

    def display_config(self) -> str:
        return (
            f'Account Type: {self.account_type}\n'
            f'Account Holder: {self.account_holder}\n'
            f'Balance: ${self.balance:.2f}\n'
            f'Interest Rate: {self.interest_rate * 100:.1f}%\n'
            f'Monthly Fee: ${self.monthly_fee:.2f}\n'
            f'Minimum Balance: ${self.min_balance:.2f}\n'
            f'Withdrawal Limit: {self.withdrawal_limit} per month\n'
            f'Overdraft Protection: {"Yes" if self.overdraft_protection else "No"}'
        )
```

### BankBranch

```python
# ── Client: BankBranch ──
# OOP role: Composition
# SOLID: DIP (depends on abstractions), SRP (orchestrates)
class BankBranch:
    """Maintains a registry of account prototypes and clones them on demand."""

    def __init__(self) -> None:
        self.prototypes: dict[str, AccountPrototype] = {
            'savings': StandardSavingsPrototype(),
            'checking': PremiumCheckingPrototype(),
            'loan': BusinessLoanPrototype(),
        }

    def open_account(
        self, template_type: str, account_holder: str, amount: float
    ) -> AccountPrototype:
        if template_type not in self.prototypes:
            raise ValueError(f'Unknown template type: {template_type}')

        template = self.prototypes[template_type]
        account = template.clone()
        account.account_holder = account_holder

        if template_type == 'loan':
            account.principal = amount
        else:
            account.balance = amount

        return account
```

### Main / Driver

```python
# ── Main / Driver ──
def main() -> None:
    branch = BankBranch()

    print('=== Prototype Pattern — Banking Example ===')
    print()

    # Clone standard savings for Alice
    alice_savings = branch.open_account('savings', 'Alice', 1000.0)
    print("--- Alice's Standard Savings Account ---")
    print(alice_savings.display_config())

    print()

    # Clone premium checking for Bob
    bob_checking = branch.open_account('checking', 'Bob', 2500.0)
    print("--- Bob's Premium Checking Account ---")
    print(bob_checking.display_config())

    print()

    # Clone business loan for Carol
    carol_loan = branch.open_account('loan', 'Carol', 50000.0)
    print("--- Carol's Business Loan Account ---")
    print(carol_loan.display_config())

    print()

    # Verify that the original prototypes are unchanged
    print('=== Original Prototypes (Unchanged) ===')
    original_savings = branch.prototypes['savings']
    print(f'Original savings holder: "{original_savings.account_holder}"')
    print(f'Original savings balance: ${original_savings.balance:.2f}')
    original_checking = branch.prototypes['checking']
    print(f'Original checking holder: "{original_checking.account_holder}"')
    original_loan = branch.prototypes['loan']
    print(f'Original loan holder: "{original_loan.account_holder}"')

    print()

    # Clone again to demonstrate reuse
    dave_savings = branch.open_account('savings', 'Dave', 500.0)
    print("--- Dave's Standard Savings Account ---")
    print(dave_savings.display_config())


if __name__ == '__main__':
    main()
```

### Language-Specific Notes

- **ABCs for abstraction** — Python uses `abc.ABC` and `@abstractmethod` to define the abstract prototype interface. This enforces that all concrete prototypes implement `clone()` and `display_config()` at instantiation time.

- **Manual cloning** — Python's `copy.deepcopy()` could clone any object, but this implementation uses explicit field-by-field cloning for clarity. Each `clone()` method constructs a new instance and copies all fields, making the cloning logic explicit and avoiding issues with `__init__` side effects.

- **Type hints** — All methods and attributes use Python 3.11+ type hints. The `-> 'AccountPrototype'` forward reference in the ABC's `clone()` method handles the case where the class is still being defined.

- **No `@dataclass`** — Although `@dataclass` would reduce boilerplate, this implementation uses regular classes with explicit `__init__` to make the field initialization and cloning logic transparent. The cloning process is the focus of the pattern.
