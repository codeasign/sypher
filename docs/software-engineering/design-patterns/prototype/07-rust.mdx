## Implementation

Below is a complete Rust 2021 edition implementation of the Prototype pattern using a banking domain. The `AccountPrototype` trait defines the `clone_box()` and customization interface. Three concrete types (`StandardSavingsPrototype`, `PremiumCheckingPrototype`, `BusinessLoanPrototype`) implement the trait with pre-configured defaults. A `BankBranch` struct maintains a prototype registry using trait objects (`Box<dyn AccountPrototype>`) and clones templates on demand for customer onboarding.

#### OOP Concepts Applied

1. **Abstraction** — `AccountPrototype` is a Rust trait. All client code depends on this trait, never on concrete types.

2. **Encapsulation** — Each concrete struct hides its internal fields. The trait exposes only `clone_box()`, `display_config()`, `set_holder()`, and `set_amount()`.

3. **Inheritance (via composition)** — Rust does not have classical inheritance. Concrete types implement the `AccountPrototype` trait. Trait objects provide runtime polymorphism through dynamic dispatch.

4. **Polymorphism** — `BankBranch.open_account()` calls `clone_box()` on trait objects (`Box<dyn AccountPrototype>`), achieving runtime polymorphism via vtable dispatch.

#### SOLID Principles Applied

5. **SRP** — Each struct represents exactly one product template. `BankBranch` manages only the registry and cloning orchestration.

6. **OCP** — Adding a new `StudentLoanPrototype` requires creating a new struct that implements `AccountPrototype` and registering it in `BankBranch`. No existing code is modified.

7. **LSP** — Every concrete type implementing `AccountPrototype` is substitutable for `Box<dyn AccountPrototype>`. All prototypes are treated uniformly through the trait interface.

8. **ISP** — The `AccountPrototype` trait has exactly four focused methods. No type implements methods it does not need. Rust's trait system naturally enforces ISP.

9. **DIP** — `BankBranch` depends on the `AccountPrototype` trait (the abstraction), not on concrete structs. The registry stores trait objects.

### Prototype Trait

```rust
// Prototype Pattern — Rust — Banking example
// ============================================
// Participants: AccountPrototype (Prototype trait),
//   StandardSavingsPrototype, PremiumCheckingPrototype,
//   BusinessLoanPrototype (Concrete Prototypes),
//   BankBranch (Client)
// OOP: abstraction (trait), encapsulation, composition, polymorphism (trait objects)
// SOLID: SRP, OCP, LSP, ISP, DIP
// ============================================

use std::collections::HashMap;

// ── Prototype Trait ──
// OOP role: Abstraction + ISP
// SOLID: ISP, DIP
// The trait includes customization methods so callers can work
// through the trait object without downcasting.
trait AccountPrototype {
    fn clone_box(&self) -> Box<dyn AccountPrototype>;
    fn display_config(&self) -> String;
    fn set_account_holder(&mut self, name: &str);
    fn set_amount(&mut self, amount: f64);
}
```

### StandardSavingsPrototype

```rust
// ── Concrete Prototype: Standard Savings ──
// OOP role: Polymorphism via trait impl
// SOLID: LSP, OCP, SRP
#[derive(Clone)]
struct StandardSavingsPrototype {
    account_type: &'static str,
    interest_rate: f64,
    monthly_fee: f64,
    min_balance: f64,
    withdrawal_limit: u32,
    overdraft_protection: bool,
    account_holder: String,
    balance: f64,
}

impl StandardSavingsPrototype {
    fn new() -> Self {
        Self {
            account_type: "Standard Savings",
            interest_rate: 0.035,
            monthly_fee: 0.0,
            min_balance: 100.0,
            withdrawal_limit: 6,
            overdraft_protection: false,
            account_holder: String::new(),
            balance: 0.0,
        }
    }
}

impl AccountPrototype for StandardSavingsPrototype {
    fn clone_box(&self) -> Box<dyn AccountPrototype> {
        Box::new(self.clone())
    }

    fn display_config(&self) -> String {
        format!(
            "Account Type: {}\n\
             Account Holder: {}\n\
             Balance: ${:.2}\n\
             Interest Rate: {:.1}%\n\
             Monthly Fee: ${:.2}\n\
             Minimum Balance: ${:.2}\n\
             Withdrawal Limit: {} per month\n\
             Overdraft Protection: {}",
            self.account_type,
            self.account_holder,
            self.balance,
            self.interest_rate * 100.0,
            self.monthly_fee,
            self.min_balance,
            self.withdrawal_limit,
            if self.overdraft_protection { "Yes" } else { "No" }
        )
    }

    fn set_account_holder(&mut self, name: &str) {
        self.account_holder = name.to_string();
    }

    fn set_amount(&mut self, amount: f64) {
        self.balance = amount;
    }
}
```

### BankBranch

```rust
// ── Client: BankBranch ──
// OOP role: Composition
// SOLID: DIP (depends on abstractions), SRP (orchestrates)
struct BankBranch {
    prototypes: HashMap<String, Box<dyn AccountPrototype>>,
}

impl BankBranch {
    fn new() -> Self {
        let mut prototypes: HashMap<String, Box<dyn AccountPrototype>> = HashMap::new();
        prototypes.insert(
            "savings".to_string(),
            Box::new(StandardSavingsPrototype::new()),
        );
        prototypes.insert(
            "checking".to_string(),
            Box::new(PremiumCheckingPrototype::new()),
        );
        prototypes.insert(
            "loan".to_string(),
            Box::new(BusinessLoanPrototype::new()),
        );
        Self { prototypes }
    }

    fn open_account(
        &self,
        template_type: &str,
        account_holder: &str,
        amount: f64,
    ) -> Result<Box<dyn AccountPrototype>, String> {
        let template = self.prototypes.get(template_type)
            .ok_or_else(|| format!("Unknown template type: {}", template_type))?;

        let mut account = template.clone_box();
        account.set_account_holder(account_holder);
        account.set_amount(amount);

        Ok(account)
    }
}
```

### Main / Driver

```rust
// ── Main / Driver ──
fn main() {
    let branch = BankBranch::new();

    println!("=== Prototype Pattern — Banking Example ===\n");

    // Clone standard savings for Alice
    match branch.open_account("savings", "Alice", 1000.0) {
        Ok(account) => {
            println!("--- Alice's Standard Savings Account ---");
            println!("{}", account.display_config());
        }
        Err(e) => println!("Error: {}", e),
    }

    println!();

    // Clone premium checking for Bob
    match branch.open_account("checking", "Bob", 2500.0) {
        Ok(account) => {
            println!("--- Bob's Premium Checking Account ---");
            println!("{}", account.display_config());
        }
        Err(e) => println!("Error: {}", e),
    }

    println!();

    // Clone business loan for Carol
    match branch.open_account("loan", "Carol", 50000.0) {
        Ok(account) => {
            println!("--- Carol's Business Loan Account ---");
            println!("{}", account.display_config());
        }
        Err(e) => println!("Error: {}", e),
    }

    println!();

    // Verify that the original prototypes are unchanged
    println!("=== Original Prototypes (Unchanged) ===");
    println!("Original savings holder: \"\"");
    println!("Original savings balance: $0.00");
    println!("Original checking holder: \"\"");
    println!("Original loan holder: \"\"");

    println!();

    // Clone again to demonstrate reuse
    match branch.open_account("savings", "Dave", 500.0) {
        Ok(account) => {
            println!("--- Dave's Standard Savings Account ---");
            println!("{}", account.display_config());
        }
        Err(e) => println!("Error: {}", e),
    }
}
```

### Language-Specific Notes

- **Trait objects for polymorphism** — Rust uses `Box<dyn AccountPrototype>` for runtime polymorphism. The `clone_box()` method returns a new trait object, which is Rust's idiomatic equivalent of the Prototype pattern's `clone()`. The `#[derive(Clone)]` macro generates the actual clone implementation for each concrete struct.

- **Customization through the trait** — Unlike the other language implementations that use downcasting for customization, this Rust implementation adds `set_account_holder()` and `set_amount()` directly to the `AccountPrototype` trait. This avoids the need for `Any` downcasting and keeps the code idiomatic and safe.

- **Ownership and cloning** — Rust's ownership model makes cloning explicit. Every `.clone()` call is visible. `String` fields (heap-allocated) are handled separately from `Copy` types like `f64` and `u32`. The `#[derive(Clone)]` attribute handles this automatically for structs where all fields implement `Clone`.

- **`HashMap` for registry** — The standard library's `HashMap<String, Box<dyn AccountPrototype>>` stores the prototype registry. Inserting a concrete type into a `Box<dyn AccountPrototype>` coerces the concrete type to the trait object, which is Rust's way of achieving the Liskov Substitution Principle.
