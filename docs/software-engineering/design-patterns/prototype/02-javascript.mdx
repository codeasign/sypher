## Implementation

Below is a complete JavaScript ES2022+ implementation of the Prototype pattern using a banking domain. The `AccountPrototype` interface defines the `clone()` contract, and three concrete prototypes (`StandardSavingsPrototype`, `PremiumCheckingPrototype`, `BusinessLoanPrototype`) carry pre-configured defaults. A `BankBranch` registry stores the templates and clones them on demand, after which the clone is customized with customer-specific fields.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌──────────────────────────────────────────┐
//   │   <<prototype>>                          │
//   │   AccountPrototype                       │  ← OOP: Abstraction
//   ├──────────────────────────────────────────┤     ISP: focused contract
//   │ + clone(): AccountPrototype              │     (clone + display)
//   │ + displayConfig(): string                │
//   └────────────────────┬─────────────────────┘
//                        ▲ implements
//   ┌────────────────────┼─────────────────────┐
//   │                    │                     │
//   ┌──────────────────┐ ┌──────────────────┐ ┌──────────────────────┐
//   │ StandardSavings  │ │ PremiumChecking  │ │ BusinessLoan        │
//   │ Prototype        │ │ Prototype        │ │ Prototype           │
//   ├──────────────────┤ ├──────────────────┤ ├──────────────────────┤
//   │ - rate: 0.035    │ │ - rate: 0.001   │ │ - apr: 0.0675       │
//   │ - fee: 0         │ │ - fee: 25       │ │ - fee: 500          │
//   │ - minBal: 100    │ │ - minBal: 1500  │ │ - maxPrin: 50000    │
//   │ + clone()        │ │ + clone()       │ │ + clone()           │
//   └──────────────────┘ └──────────────────┘ └──────────────────────┘
//
//   ┌──────────────────────────────────────────┐
//   │   BankBranch (Client)                    │  ← OOP: Encapsulation
//   ├──────────────────────────────────────────┤     SRP: manages registry
//   │ - prototypes: Map<string, Proto>         │     DIP: depends on
//   │ + openAccount(type, holder, amount)      │     abstraction
//   └──────────────────────────────────────────┘
```

#### OOP Concepts Applied

1. **Abstraction** — `AccountPrototype` defines the `clone()` and `displayConfig()` contract. All client code depends on this abstraction, not on concrete prototype classes.

2. **Encapsulation** — Each prototype hides its internal configuration fields (interest rates, fees, limits) behind the `clone()` and `displayConfig()` methods.

3. **Inheritance** — Concrete prototypes inherit from `AccountPrototype` via `Object.create()` or by setting the prototype chain. In this implementation, each concrete prototype object is created with a shared base prototype.

4. **Polymorphism** — `BankBranch.openAccount()` calls `clone()` on any `AccountPrototype` reference. The correct cloning logic runs per type without type checks.

#### SOLID Principles Applied

5. **SRP** — Each prototype class has one responsibility: represent a specific product template. `BankBranch` has one responsibility: manage the registry and coordinate cloning.

6. **OCP** — Adding a new product template (e.g., `StudentLoanPrototype`) requires creating a new prototype object that implements the `AccountPrototype` interface. No existing code needs modification.

7. **LSP** — Every concrete prototype can substitute for `AccountPrototype`. The `BankBranch` treats all prototypes uniformly through the `clone()` interface.

8. **ISP** — The `AccountPrototype` interface is minimal: only `clone()` and `displayConfig()`. No prototype is forced to implement methods it does not need.

9. **DIP** — `BankBranch` depends on the abstract `AccountPrototype` interface, not on concrete prototype classes. The registry stores prototypes by their abstract type.

### Prototype Interface

```javascript
// Prototype Pattern — JavaScript — Banking example
// ================================================
// Participants: AccountPrototype (Prototype interface),
//   StandardSavingsPrototype, PremiumCheckingPrototype,
//   BusinessLoanPrototype (Concrete Prototypes),
//   BankBranch (Client)
// OOP: abstraction, encapsulation, inheritance, polymorphism
// SOLID: SRP, OCP, LSP, ISP, DIP
// ================================================

// ── Prototype (abstract base) ──
// OOP role: Abstraction + ISP
// SOLID: ISP, DIP
const AccountPrototype = {
  clone() {
    return Object.create(Object.getPrototypeOf(this));
  },
  displayConfig() {
    return '';
  }
};
```

### StandardSavingsPrototype

```javascript
// ── Concrete Prototype: Standard Savings ──
// OOP role: Inheritance/Polymorphism
// SOLID: LSP, OCP, SRP
function createStandardSavingsPrototype() {
  const proto = Object.create(AccountPrototype);
  proto.accountType = 'Standard Savings';
  proto.interestRate = 0.035;
  proto.monthlyFee = 0.0;
  proto.minBalance = 100.0;
  proto.withdrawalLimit = 6;
  proto.overdraftProtection = false;
  proto.accountHolder = '';
  proto.balance = 0.0;

  proto.clone = function () {
    const cloned = Object.create(Object.getPrototypeOf(this));
    return Object.assign(cloned, this);
  };

  proto.displayConfig = function () {
    return [
      `Account Type: ${this.accountType}`,
      `Account Holder: ${this.accountHolder}`,
      `Balance: $${this.balance.toFixed(2)}`,
      `Interest Rate: ${(this.interestRate * 100).toFixed(1)}%`,
      `Monthly Fee: $${this.monthlyFee.toFixed(2)}`,
      `Minimum Balance: $${this.minBalance.toFixed(2)}`,
      `Withdrawal Limit: ${this.withdrawalLimit} per month`,
      `Overdraft Protection: ${this.overdraftProtection ? 'Yes' : 'No'}`
    ].join('\n');
  };

  return proto;
}
```

### BankBranch

```javascript
// ── Client: BankBranch ──
// OOP role: Composition
// SOLID: DIP (depends on abstractions), SRP (orchestrates)
class BankBranch {
  constructor() {
    this.prototypes = new Map();
    this.prototypes.set('savings', createStandardSavingsPrototype());
    this.prototypes.set('checking', createPremiumCheckingPrototype());
    this.prototypes.set('loan', createBusinessLoanPrototype());
  }

  openAccount(templateType, accountHolder, amount) {
    if (!this.prototypes.has(templateType)) {
      throw new Error(`Unknown template type: ${templateType}`);
    }

    const template = this.prototypes.get(templateType);
    const account = template.clone();

    account.accountHolder = accountHolder;

    if (templateType === 'loan') {
      account.principal = amount;
    } else {
      account.balance = amount;
    }

    return account;
  }
}
```

### Main / Driver

```javascript
// ── Main / Driver ──
const branch = new BankBranch();

console.log('=== Prototype Pattern — Banking Example ===\n');

// Clone standard savings for Alice
const aliceSavings = branch.openAccount('savings', 'Alice', 1000.0);
console.log('--- Alice\'s Standard Savings Account ---');
console.log(aliceSavings.displayConfig());

console.log();

// Clone premium checking for Bob
const bobChecking = branch.openAccount('checking', 'Bob', 2500.0);
console.log('--- Bob\'s Premium Checking Account ---');
console.log(bobChecking.displayConfig());

console.log();

// Clone business loan for Carol
const carolLoan = branch.openAccount('loan', 'Carol', 50000.0);
console.log("--- Carol's Business Loan Account ---");
console.log(carolLoan.displayConfig());

console.log();

// Verify that the original prototypes are unchanged
console.log('=== Original Prototypes (Unchanged) ===');
const originalSavings = branch.prototypes.get('savings');
console.log(`Original savings holder: "${originalSavings.accountHolder}"`);
console.log(`Original savings balance: $${originalSavings.balance.toFixed(2)}`);
const originalChecking = branch.prototypes.get('checking');
console.log(`Original checking holder: "${originalChecking.accountHolder}"`);
const originalLoan = branch.prototypes.get('loan');
console.log(`Original loan holder: "${originalLoan.accountHolder}"`);

console.log();

// Clone again to demonstrate reuse
const daveSavings = branch.openAccount('savings', 'Dave', 500.0);
console.log("--- Dave's Standard Savings Account ---");
console.log(daveSavings.displayConfig());
```

### Language-Specific Notes

- **OOP via prototypes** — JavaScript uses prototypal inheritance, not classical classes. The `AccountPrototype` is a plain object used as a prototype. `Object.create()` sets up the prototype chain, and `Object.assign()` copies own properties for shallow cloning.
- **ES2022+ features** — The code uses `Map` for the prototype registry (cleaner than plain objects), template literals for string formatting, and `Object.create()` / `Object.assign()` for cloning. No class syntax is used in the prototypes themselves to stay true to the prototypal nature of the pattern.
- **Cloning approach** — This implementation uses shallow copy via `Object.assign()`. For the Banking domain, all fields are primitives (strings, numbers, booleans), so shallow copy is sufficient. Deep cloning with `structuredClone()` would be needed for nested objects.
- **No `class` for prototypes** — While JavaScript classes work, using factory functions and `Object.create()` better illustrates the Prototype pattern's intent since JavaScript's own prototypal inheritance is itself an instance of the pattern.
