## Implementation

Below is a complete Java 17+ implementation of the Interpreter pattern using a Stock Market domain. The pattern defines a composable expression grammar for stock filtering. The `Expression` interface declares the `interpret()` method. Terminal expressions (`PriceGreaterThan`, `VolumeGreaterThan`, `TickerEquals`, `SectorEquals`) implement atomic comparisons against a `StockData` record. Nonterminal expressions (`AndExpr`, `OrExpr`, `NotExpr`) compose expressions logically. The `StockScreener` evaluates an expression tree against a portfolio, collecting matching stocks.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌──────────────────────────────────────┐
//   │   <<interface>>                      │
//   │   Expression (AbstractExpression)    │ ← OOP: Abstraction
//   ├──────────────────────────────────────┤    ISP: single method
//   │   + interpret(StockData): boolean    │
//   └────────────────┬─────────────────────┘
//                    ▲ implements
//   ┌────────────────┴─────────────────────┐
//   │                │                     │
//   ┌────────────────┐  ┌──────────────────┐
//   │ TerminalExprs  │  │ NonterminalExprs │ ← OOP: Polymorphism
//   │ (PriceCheck,   │  │ (AndExpr,        │    LSP: all are
//   │  VolumeCheck,  │  │  OrExpr,         │    substitutable for
//   │  TickerMatch,  │  │  NotExpr)        │    Expression
//   │  SectorMatch)  │  │                  │
//   └────────────────┘  └──────────────────┘
//
//   ┌──────────────────────────────────────┐
//   │ StockScreener (Client)               │ ← SRP: only screens
//   │ + screen(List, Expression): List     │    DIP: depends on
//   └──────────────────────────────────────┘    Expression interface
```

#### OOP Concepts Applied

1. **Abstraction** — The `Expression` interface is the contract. Clients depend on `Expression`, never on concrete classes.
2. **Encapsulation** — Each expression class stores its private fields and exposes only `interpret()`.
3. **Inheritance** — Classes implement the `Expression` interface, providing concrete `interpret()` bodies.
4. **Polymorphism** — `screen()` calls `interpret()` on any `Expression` reference. The JVM dispatches to the correct implementation.

#### SOLID Principles Applied

5. **SRP** — Each expression class evaluates exactly one grammar rule. `StockScreener` only iterates and collects results.
6. **OCP** — New expression types implement `Expression` without modifying existing expressions or the screener.
7. **LSP** — Every concrete expression is substitutable for `Expression`. No `instanceof` checks needed.
8. **ISP** — The `Expression` interface is a single method — minimal and focused.
9. **DIP** — `StockScreener` depends on `Expression`, not on concrete implementations.

### StockData (Context)

```java
// ── Context ──
record StockData(String ticker, double price, int volume, String sector) {}
```

### Expression (AbstractExpression)

```java
// ── Abstract Expression ──
interface Expression {
    boolean interpret(StockData stock);
}
```

### PriceGreaterThan (TerminalExpression)

```java
class PriceGreaterThan implements Expression {
    private final double threshold;

    public PriceGreaterThan(double threshold) {
        this.threshold = threshold;
    }

    @Override
    public boolean interpret(StockData stock) {
        return stock.price() > threshold;
    }
}
```

### VolumeGreaterThan (TerminalExpression)

```java
class VolumeGreaterThan implements Expression {
    private final int threshold;

    public VolumeGreaterThan(int threshold) {
        this.threshold = threshold;
    }

    @Override
    public boolean interpret(StockData stock) {
        return stock.volume() > threshold;
    }
}
```

### TickerEquals (TerminalExpression)

```java
class TickerEquals implements Expression {
    private final String ticker;

    public TickerEquals(String ticker) {
        this.ticker = ticker.toUpperCase();
    }

    @Override
    public boolean interpret(StockData stock) {
        return stock.ticker().equals(this.ticker);
    }
}
```

### SectorEquals (TerminalExpression)

```java
class SectorEquals implements Expression {
    private final String sector;

    public SectorEquals(String sector) {
        this.sector = sector;
    }

    @Override
    public boolean interpret(StockData stock) {
        return stock.sector().equals(this.sector);
    }
}
```

### AndExpr (NonterminalExpression)

```java
class AndExpr implements Expression {
    private final Expression left;
    private final Expression right;

    public AndExpr(Expression left, Expression right) {
        this.left = left;
        this.right = right;
    }

    @Override
    public boolean interpret(StockData stock) {
        return left.interpret(stock) && right.interpret(stock);
    }
}
```

### OrExpr (NonterminalExpression)

```java
class OrExpr implements Expression {
    private final Expression left;
    private final Expression right;

    public OrExpr(Expression left, Expression right) {
        this.left = left;
        this.right = right;
    }

    @Override
    public boolean interpret(StockData stock) {
        return left.interpret(stock) || right.interpret(stock);
    }
}
```

### NotExpr (NonterminalExpression)

```java
class NotExpr implements Expression {
    private final Expression expr;

    public NotExpr(Expression expr) {
        this.expr = expr;
    }

    @Override
    public boolean interpret(StockData stock) {
        return !expr.interpret(stock);
    }
}
```

### StockScreener (Client)

```java
// ── Client ──
class StockScreener {
    public List<StockData> screen(List<StockData> stocks, Expression expression) {
        List<StockData> results = new ArrayList<>();
        for (StockData stock : stocks) {
            if (expression.interpret(stock)) {
                results.add(stock);
            }
        }
        return results;
    }
}
```

### Main / Driver

```java
// ── Main / Driver ──
public class InterpreterDemo {
    public static void main(String[] args) {
        List<StockData> stocks = List.of(
            new StockData("AAPL", 175.50, 85000000, "Technology"),
            new StockData("GOOGL", 142.30, 32000000, "Technology"),
            new StockData("MSFT", 378.90, 25000000, "Technology"),
            new StockData("JPM", 198.40, 12000000, "Financial"),
            new StockData("BAC", 39.20, 45000000, "Financial"),
            new StockData("AMZN", 178.25, 61000000, "Technology"),
            new StockData("TSLA", 245.60, 98000000, "Automotive"),
            new StockData("WMT", 72.80, 18000000, "Retail"),
            new StockData("V", 275.30, 15000000, "Financial"),
            new StockData("NVDA", 820.10, 52000000, "Technology")
        );

        StockScreener screener = new StockScreener();

        System.out.println("=== Stock Screener — Interpreter Pattern ===");
        System.out.println();

        // Query 1: price > 150 AND volume > 30000000
        Expression query1 = new AndExpr(
            new PriceGreaterThan(150.0),
            new VolumeGreaterThan(30000000)
        );
        List<StockData> result1 = screener.screen(stocks, query1);
        System.out.println("Query 1: price > 150 AND volume > 30000000");
        for (StockData s : result1) {
            System.out.printf("  %s — $%.2f — %.1fM vol%n",
                s.ticker(), s.price(), s.volume() / 1_000_000.0);
        }
        System.out.println("  Matched: " + result1.size() + " stocks");
        System.out.println();

        // Query 2: sector = "Financial" OR ticker = "MSFT"
        Expression query2 = new OrExpr(
            new SectorEquals("Financial"),
            new TickerEquals("MSFT")
        );
        List<StockData> result2 = screener.screen(stocks, query2);
        System.out.println("Query 2: sector = \"Financial\" OR ticker = \"MSFT\"");
        for (StockData s : result2) {
            System.out.printf("  %s — $%.2f — %s%n",
                s.ticker(), s.price(), s.sector());
        }
        System.out.println("  Matched: " + result2.size() + " stocks");
        System.out.println();

        // Query 3: NOT (price > 100) — stocks with price <= 100
        Expression query3 = new NotExpr(new PriceGreaterThan(100.0));
        List<StockData> result3 = screener.screen(stocks, query3);
        System.out.println("Query 3: NOT (price > 100) — stocks with price <= 100");
        for (StockData s : result3) {
            System.out.printf("  %s — $%.2f%n", s.ticker(), s.price());
        }
        System.out.println("  Matched: " + result3.size() + " stocks");
        System.out.println();

        // Query 4: (ticker = "AAPL" OR ticker = "GOOGL") AND volume > 40000000
        Expression query4 = new AndExpr(
            new OrExpr(
                new TickerEquals("AAPL"),
                new TickerEquals("GOOGL")
            ),
            new VolumeGreaterThan(40000000)
        );
        List<StockData> result4 = screener.screen(stocks, query4);
        System.out.println("Query 4: (ticker = \"AAPL\" OR ticker = \"GOOGL\") AND volume > 40000000");
        for (StockData s : result4) {
            System.out.printf("  %s — $%.2f — %.1fM vol%n",
                s.ticker(), s.price(), s.volume() / 1_000_000.0);
        }
        System.out.println("  Matched: " + result4.size() + " stocks");
        System.out.println();

        // Query 5: sector = "Technology" OR sector = "Financial"
        Expression query5 = new OrExpr(
            new SectorEquals("Technology"),
            new SectorEquals("Financial")
        );
        List<StockData> result5 = screener.screen(stocks, query5);
        System.out.println("Query 5: sector = \"Technology\" OR sector = \"Financial\"");
        for (StockData s : result5) {
            System.out.printf("  %s — $%.2f — %s%n",
                s.ticker(), s.price(), s.sector());
        }
        System.out.println("  Matched: " + result5.size() + " stocks");
    }
}
```

### Language-Specific Notes

- **OOP concepts** — Java uses `interface` for the abstraction participant (100% abstract contract). The `record` keyword (Java 16+) provides concise immutable data carriers with auto-generated `equals()`, `hashCode()`, `toString()`, and accessor methods.
- **SOLID in practice** — Java's nominal type system enforces ISP at compile time: a class `implements Expression` must provide `interpret()` or the compiler rejects it. The `final` keyword on fields in expression classes ensures immutability and thread safety.
- **Idiomatic patterns** — `List.of()` for immutable collections (Java 9+). `System.out.printf()` with format specifiers for precise output. `@Override` annotations for clarity. `private final` fields with constructor injection for dependency setup.
- **Single-file demo** — The `public class InterpreterDemo` with `main()` serves as the driver. All supporting classes are package-private (no `public` modifier), keeping the demo self-contained.
