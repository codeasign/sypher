## Implementation

Below is a complete JavaScript ES2022+ implementation of the Interpreter pattern using a Stock Market domain. The pattern defines a grammar for stock-filtering expressions and evaluates them against stock data. `Expression` is the abstract interface, concrete terminal expressions (`PriceGreaterThan`, `VolumeGreaterThan`, `TickerEquals`, `SectorEquals`) handle atomic comparisons, and nonterminal expressions (`AndExpr`, `OrExpr`, `NotExpr`) compose them into complex queries. The `StockScreener` client iterates over a portfolio and collects matching stocks.

#### OOP Concepts Applied

1. **Abstraction** вҖ” `Expression` defines the `interpret()` contract. All clients depend on `Expression`, never on concrete expression types.
2. **Encapsulation** вҖ” Each expression class hides its internal data (thresholds, ticker strings, child expressions) behind the `interpret()` method.
3. **Inheritance** вҖ” Terminal and nonterminal expression classes implement the `Expression` interface, sharing the common contract.
4. **Polymorphism** вҖ” The `StockScreener.screen()` method calls `interpret()` on any `Expression` without knowing the concrete type. The correct evaluation runs per type.

#### SOLID Principles Applied

5. **SRP** вҖ” Each expression class has exactly one job: evaluate one specific grammar rule. `StockScreener` only iterates and collects.
6. **OCP** вҖ” Adding a new expression type (e.g., `PriceLessThan`) requires adding a new class that implements `Expression`. No existing code is modified.
7. **LSP** вҖ” Every concrete expression can substitute `Expression` without breaking the client. The screener treats all expressions uniformly.
8. **ISP** вҖ” The `Expression` interface is a single method вҖ” `interpret()`. No class is forced to implement methods it does not need.
9. **DIP** вҖ” `StockScreener` depends on the `Expression` abstraction, not on concrete expression classes.

### StockData (Context)

```javascript
// в”Җв”Җ Context в”Җв”Җ
class StockData {
  constructor(ticker, price, volume, sector) {
    this.ticker = ticker;
    this.price = price;
    this.volume = volume;
    this.sector = sector;
  }
}
```

### Expression (AbstractExpression)

```javascript
// в”Җв”Җ Abstract Expression в”Җв”Җ
class Expression {
  interpret(stock) {
    throw new Error('Subclasses must implement interpret()');
  }
}
```

### PriceGreaterThan (TerminalExpression)

```javascript
class PriceGreaterThan extends Expression {
  constructor(threshold) {
    super();
    this.threshold = threshold;
  }

  interpret(stock) {
    return stock.price > this.threshold;
  }
}
```

### VolumeGreaterThan (TerminalExpression)

```javascript
class VolumeGreaterThan extends Expression {
  constructor(threshold) {
    super();
    this.threshold = threshold;
  }

  interpret(stock) {
    return stock.volume > this.threshold;
  }
}
```

### TickerEquals (TerminalExpression)

```javascript
class TickerEquals extends Expression {
  constructor(ticker) {
    super();
    this.ticker = ticker.toUpperCase();
  }

  interpret(stock) {
    return stock.ticker === this.ticker;
  }
}
```

### SectorEquals (TerminalExpression)

```javascript
class SectorEquals extends Expression {
  constructor(sector) {
    super();
    this.sector = sector;
  }

  interpret(stock) {
    return stock.sector === this.sector;
  }
}
```

### AndExpr (NonterminalExpression)

```javascript
class AndExpr extends Expression {
  constructor(left, right) {
    super();
    this.left = left;
    this.right = right;
  }

  interpret(stock) {
    return this.left.interpret(stock) && this.right.interpret(stock);
  }
}
```

### OrExpr (NonterminalExpression)

```javascript
class OrExpr extends Expression {
  constructor(left, right) {
    super();
    this.left = left;
    this.right = right;
  }

  interpret(stock) {
    return this.left.interpret(stock) || this.right.interpret(stock);
  }
}
```

### NotExpr (NonterminalExpression)

```javascript
class NotExpr extends Expression {
  constructor(expr) {
    super();
    this.expr = expr;
  }

  interpret(stock) {
    return !this.expr.interpret(stock);
  }
}
```

### StockScreener (Client)

```javascript
// в”Җв”Җ Client в”Җв”Җ
class StockScreener {
  screen(stocks, expression) {
    const results = [];
    for (const stock of stocks) {
      if (expression.interpret(stock)) {
        results.push(stock);
      }
    }
    return results;
  }
}
```

console.log(`  Matched: ${result4.length} stocks`);
console.log('');

// Query 5: sector = "Technology" OR sector = "Financial"

  new SectorEquals('Technology'),
  new SectorEquals('Financial')
);
const result5 = screener.screen(stocks, query5);
console.log('Query 5: sector = "Technology" OR sector = "Financial"');
result5.forEach(s => console.log(`  ${s.ticker} вҖ” $${s.price} вҖ” ${s.sector}`));
console.log(`  Matched: ${result5.length} stocks`);

### Language-Specific Notes

- **OOP concepts** вҖ” JavaScript uses prototype-based inheritance, but ES2022+ `class` syntax provides a familiar class-based interface. The `extends Expression` establishes the contract, and `super()` in the constructor initializes the base class.
- **SOLID in practice** вҖ” JavaScript's dynamic typing makes DIP and LSP natural: any object with an `interpret()` method can substitute for `Expression`. The `StockScreener` never checks types, relying on duck typing.
- **Idiomatic patterns** вҖ” JavaScript's `Array.prototype.forEach()` and `for...of` loops are used for iteration. Template literals (`${}`) provide clean string formatting for output. The `toUpperCase()` call in `TickerEquals` normalizes input for case-insensitive comparison.
- **No `instanceof` checks** вҖ” The screener's `screen()` method calls `interpret()` polymorphically without any type checking, demonstrating the purest form of the pattern.
