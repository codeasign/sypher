The Builder pattern in JavaScript relies on duck typing and method convention rather than formal interfaces. In this banking-domain example, three concrete builders (PersonalLoanBuilder, MortgageBuilder, AutoLoanBuilder) each implement the same set of builder methods to construct a LoanApplication step by step. A LoanDirector orchestrates the construction, hiding the build sequence from the caller while keeping each builder interchangeable through JavaScript's prototypal inheritance and class syntax.

## How the Implementation Works

The JavaScript Builder pattern relies on duck typing: any object that exposes the expected builder methods can serve as a builder. The LoanDirector accepts any builder following this convention, calls the same sequence of construction steps, and each concrete builder produces a LoanApplication with domain-specific fields. MortgageBuilder extends the protocol with `setPropertyInfo()`, while AutoLoanBuilder adds `setVehicleInfo()` and `setDealership()`. JavaScript ES2022+ private fields (the `#` prefix) protect internal state from accidental access, and the fluent `return this` pattern enables method chaining where desired.

### OOP Concepts Applied

- **Abstraction** — The builder protocol hides construction details. Callers interact only with `setPersonalInfo()`, `setLoanTerms()`, and `getResult()`, never touching internal fields directly.
- **Encapsulation** — Each builder uses ES2022+ `#` private fields to shield internal state. LoanApplication stores its own data and provides a `describe()` method for formatted output.
- **Polymorphism** — LoanDirector treats any object with the builder methods identically. PersonalLoanBuilder, MortgageBuilder, and AutoLoanBuilder each respond to the same message protocol with different internal behavior.
- **Composition** — LoanDirector holds a reference to a builder object rather than inheriting from one. The product (LoanApplication) aggregates all the fields set across builder calls.

### SOLID Principles Applied

- **Single Responsibility Principle** — Each concrete builder is responsible for constructing one type of loan product. LoanApplication owns only its data and its presentation. LoanDirector orchestrates the build sequence without knowing any product details.
- **Open/Closed Principle** — New loan types (e.g. StudentLoanBuilder, BusinessLoanBuilder) can be added without modifying existing builders or the director. The system is open for extension but closed for modification.
- **Liskov Substitution Principle** — Any concrete builder can replace another in the director as long as it exposes the same method names. MortgageBuilder and AutoLoanBuilder extend the base protocol without breaking the director's expectations.
- **Interface Segregation Principle** — The builder protocol is minimal: only the methods the director calls. Loan-type-specific extensions (setPropertyInfo, setVehicleInfo) are added only on the builders that need them.
- **Dependency Inversion Principle** — LoanDirector depends on the builder convention (an abstraction), not on concrete builder classes. Concrete builders depend on the same convention rather than on the director.

### LoanApplication (Product)
```javascript
// ── Participant 2: LoanApplication (Product) ──
// OOP: Encapsulation — data and presentation live together
// SOLID: SRP — owns its structure and display only
class LoanApplication {
  constructor() {
    this.applicant = '';
    this.loanType = '';
    this.amount = 0;
    this.termMonths = 0;
    this.interestRate = 0;
    this.documents = [];
    this.extraFields = {};
  }

  describe() {
    return `Applicant: ${this.applicant} | ` +
      `Loan: $${this.amount.toFixed(2)} over ${this.termMonths}mo ` +
      `@ ${this.interestRate}% APR`;
  }
}
```

### PersonalLoanBuilder (ConcreteBuilder)
```javascript
// ── Participant 3: PersonalLoanBuilder (ConcreteBuilder) ──
// OOP: Encapsulation — private fields via # prefix; Polymorphism — same protocol
// SOLID: SRP — builds only personal loans; OCP — extendable without modification
class PersonalLoanBuilder {
  #personalInfo = { name: '', loanType: '' };
  #amount = 0;
  #termMonths = 0;
  #interestRate = 0;
  #documents = [];

  setPersonalInfo(name, loanType) {
    this.#personalInfo = { name, loanType };
    return this;
  }

  setLoanTerms(amount, termMonths) {
    this.#amount = amount;
    this.#termMonths = termMonths;
    return this;
  }

  setInterestRate(rate) {
    this.#interestRate = rate;
    return this;
  }

  addDocument(doc) {
    this.#documents.push(doc);
    return this;
  }

  getResult() {
    const app = new LoanApplication();
    app.applicant = this.#personalInfo.name;
    app.loanType = this.#personalInfo.loanType;
    app.amount = this.#amount;
    app.termMonths = this.#termMonths;
    app.interestRate = this.#interestRate;
    app.documents = [...this.#documents];
    return app;
  }
}
```

### MortgageBuilder (ConcreteBuilder)
```javascript
// ── Participant 4: MortgageBuilder (ConcreteBuilder) ──
// OOP: Encapsulation — private fields; Polymorphism — same protocol
// SOLID: SRP — builds only mortgage loans; LSP — substitutable for builder
class MortgageBuilder {

  #amount = 0;
  #termMonths = 0;
  #interestRate = 0;
  #propertyAddress = '';
  #propertyAppraisal = 0;
  #downPayment = 0;

    this.#amount = amount;
    this.#termMonths = termMonths;
    return this;
  }

  setInterestRate(rate) {
    this.#interestRate = rate;
    return this;
  }

  setPropertyInfo(address, appraisalValue) {
    this.#propertyAddress = address;
    this.#propertyAppraisal = appraisalValue;
    return this;
  }

  setDownPayment(down) {
    this.#downPayment = down;
    return this;
  }

  addDocument(doc) {
    this.#documents.push(doc);
    return this;
  }

  getResult() {
    const app = new LoanApplication();
    app.applicant = this.#personalInfo.name;
    app.loanType = this.#personalInfo.loanType;
    app.amount = this.#amount;
    app.termMonths = this.#termMonths;
    app.interestRate = this.#interestRate;
    app.documents = [...this.#documents];

      propertyAddress: this.#propertyAddress,
      propertyAppraisal: this.#propertyAppraisal,

  }
}
```

### LoanDirector (Director)
```javascript
// ── Participant 6: LoanDirector (Director) ──
// OOP: Abstraction — depends on builder protocol, not concrete types
// SOLID: DIP — depends on abstraction; SRP — orchestrates construction only
class LoanDirector {
  constructPersonalLoan(builder) {
    return builder
      .setPersonalInfo('Alice Johnson', 'Personal Loan')
      .setLoanTerms(15000, 60)
      .setInterestRate(11.99)
      .addDocument('ID')
      .addDocument('Income Verification')
      .getResult();
  }

  constructMortgage(builder) {
    return builder
      .setPersonalInfo('Bob Martinez', 'Mortgage')
      .setLoanTerms(350000, 360)
      .setInterestRate(6.25)
      .setPropertyInfo('123 Main St, Springfield', 450000)
      .setDownPayment(70000)
      .addDocument('ID')
      .addDocument('Tax Returns')
      .addDocument('Appraisal Report')
      .getResult();
  }

  constructAutoLoan(builder) {
    return builder
      .setPersonalInfo('Carol Chen', 'Auto Loan')
      .setLoanTerms(32000, 60)
      .setInterestRate(7.49)
      .setVehicleInfo(
        '2024 Honda Accord',
        '1HGCV1F34PA012345',
        35000
      )
      .setDealership('City Motors')
      .addDocument('ID')
      .addDocument('Vehicle Registration')
      .getResult();
  }
}
```

### Main / Driver
```javascript
// ── Driver ──
function main() {
  const director = new LoanDirector();

  // Build a Personal Loan for Alice
  const personalBuilder = new PersonalLoanBuilder();
  const personalLoan = director.constructPersonalLoan(personalBuilder);
  console.log('=== Personal Loan Application ===');
  console.log(personalLoan.describe());
  console.log(`Documents: ${personalLoan.documents.join(', ')}`);
  console.log();

  // Build a Mortgage for Bob
  const mortgageBuilder = new MortgageBuilder();
  const mortgage = director.constructMortgage(mortgageBuilder);
  console.log('=== Mortgage Application ===');
  console.log(mortgage.describe());
  console.log(
    `Property: ${mortgage.extraFields.propertyAddress} ` +
    `(Appraised: $${mortgage.extraFields.propertyAppraisal.toFixed(2)})`
  );
  console.log(
    `Down Payment: $${mortgage.extraFields.downPayment.toFixed(2)} | ` +
    `LTV: ${mortgage.extraFields.ltv.toFixed(2)}%`
  );
  console.log(`Documents: ${mortgage.documents.join(', ')}`);
  console.log();

  // Build an Auto Loan for Carol
  const autoBuilder = new AutoLoanBuilder();
  const autoLoan = director.constructAutoLoan(autoBuilder);
  console.log('=== Auto Loan Application ===');
  console.log(autoLoan.describe());
  console.log(
    `Vehicle: ${autoLoan.extraFields.vehicleDescription} ` +
    `(VIN: ${autoLoan.extraFields.vehicleVin}, ` +
    `Value: $${autoLoan.extraFields.vehicleValue.toFixed(2)})`
  );
  console.log(`Dealership: ${autoLoan.extraFields.dealership}`);
  console.log(`Documents: ${autoLoan.documents.join(', ')}`);
}

main();
```

### Language-Specific Notes

- JavaScript has no formal `interface` keyword; the Builder pattern is enforced through convention and duck typing — any object with the expected method names is a valid builder.
- ES2022+ private fields use the `#` prefix (`#amount`, `#personalInfo`) to provide true hard-private encapsulation, unlike the weak privacy of underscore-prefixed properties.
- Class syntax in JavaScript is primarily syntactic sugar over the prototype chain, but it provides a clean, familiar structure for defining builders compared to raw constructor functions.
- Fluent API chaining works because every setter method returns `this` — this allows the director to chain calls in a single expression without intermediate variables.
- There is no compile-time type checking; the director relies on runtime method dispatch. If a builder object lacks a method, the error surfaces at runtime rather than during a build step.
- The `getResult()` method decouples the build process from the final product: the builder accumulates state incrementally and materializes the LoanApplication only when construction is complete.
