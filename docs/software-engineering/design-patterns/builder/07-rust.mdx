The Builder pattern separates the construction of a complex `LoanApplication` object from its representation, allowing the same construction process to create different loans. In Rust, the pattern leverages traits for the builder interface, concrete structs for each builder implementation, and generics with trait bounds for the director. This approach aligns with Rust's philosophy of zero-cost abstractions and type safety while maintaining the flexibility to assemble loan applications with varying structures (personal loans, mortgages, and auto loans) through a uniform step-by-step process.

## How the Implementation Works

### Class Diagram (Pseudocode)

The diagram above shows the Builder pattern structure in Rust. The `LoanApplicationBuilder` trait defines the steps common to all loan types: setting personal information, loan terms, interest rate, and adding documents. Each concrete builder (`PersonalLoanBuilder`, `MortgageBuilder`, `AutoLoanBuilder`) implements the trait and adds its own domain-specific fields and methods. The `LoanDirector` is generic over the builder type using a trait bound and orchestrates the construction sequence for each loan category.

### OOP Concepts Applied

- **Encapsulation**: Each builder struct owns its fields privately and exposes only the builder methods. The product structs (`PersonalLoan`, `MortgageLoan`, `AutoLoan`) keep their data immutable once constructed.
- **Polymorphism**: The `LoanApplicationBuilder` trait enables polymorphic behavior: the director calls the same trait methods on different concrete builder types, and each implementation produces the appropriate loan variant.
- **Composition**: Builders compose domain-specific data structures (`Appraisal`, `VehicleInfo`) alongside shared fields, favoring struct composition over inheritance hierarchies.
- **Single-responsibility through generics**: The director is generic over `B: LoanApplicationBuilder` and only orchestrates the construction steps, while each builder handles its own data aggregation and product creation.

### SOLID Principles Applied

- **Single Responsibility Principle**: Each builder handles exactly one loan product family. The director only manages the sequence of construction steps. The product structs only represent the final loan data.
- **Open/Closed Principle**: New loan types can be added by implementing the `LoanApplicationBuilder` trait without modifying existing builders or the director.
- **Liskov Substitution Principle**: Any struct implementing `LoanApplicationBuilder` can be passed where the trait bound `T: LoanApplicationBuilder` is expected, as the trait's interface guarantees consistent behavior.
- **Interface Segregation Principle**: The builder trait defines only the methods common to all loan constructions. Mortgage-specific and auto-loan-specific methods are added on the concrete builders directly, keeping the trait focused.
- **Dependency Inversion Principle**: The director depends on the `LoanApplicationBuilder` trait abstraction, not on concrete builder types. Builders depend on the same abstraction through the trait contract.

```rust
/// Personal loan application product.
#[derive(Debug, Clone)]
pub struct PersonalLoan {
    pub first_name: String,
    pub last_name: String,
    pub amount: f64,
    pub term_months: u32,
    pub annual_rate: f64,
    pub documents: Vec<String>,
}

impl fmt::Display for PersonalLoan {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        writeln!(f, "=== Personal Loan Application ===")?;
        writeln!(
            f,
            "Applicant: {} {} | Loan: ${:.2} over {}mo @ {:.2}% APR",
            self.first_name, self.last_name, self.amount, self.term_months, self.annual_rate
        )?;
        write!(f, "Documents: {}", self.documents.join(", "))
    }
}
```

### Products — MortgageLoan
```rust
/// Mortgage loan application product.
#[derive(Debug, Clone)]
pub struct MortgageLoan {
    pub first_name: String,
    pub last_name: String,
    pub amount: f64,
    pub term_months: u32,
    pub annual_rate: f64,
    pub property_address: String,
    pub property_city: String,
    pub appraised_value: f64,
    pub down_payment: f64,
    pub documents: Vec<String>,
}

impl MortgageLoan {
    /// Calculate the loan-to-value ratio as a percentage.
    fn ltv_ratio(&self) -> f64 {
        if self.appraised_value <= 0.0 {
            0.0
        } else {
            (self.amount / self.appraised_value) * 100.0
        }
    }
}

impl fmt::Display for MortgageLoan {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        writeln!(f, "=== Mortgage Application ===")?;
        writeln!(
            f,
            "Applicant: {} {} | Loan: ${:.2} over {}mo @ {:.2}% APR",
            self.first_name, self.last_name, self.amount, self.term_months, self.annual_rate
        )?;
        writeln!(
            f,
            "Property: {}, {} (Appraised: ${:.2})",
            self.property_address, self.property_city, self.appraised_value
        )?;
        writeln!(
            f,
            "Down Payment: ${:.2} | LTV: {:.2}%",
            self.down_payment,
            self.ltv_ratio()
        )?;
        write!(f, "Documents: {}", self.documents.join(", "))
    }
}
```

### LoanApplication (Unifying Enum)

```rust
/// Represents any loan application built through the builder process.
#[derive(Debug, Clone)]
pub enum LoanApplication {
    Personal(PersonalLoan),
    Mortgage(MortgageLoan),
    Auto(AutoLoan),

impl fmt::Display for LoanApplication {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            LoanApplication::Personal(loan) => write!(f, "{}", loan),
            LoanApplication::Mortgage(loan) => write!(f, "{}", loan),
            LoanApplication::Auto(loan) => write!(f, "{}", loan),
        }
    }
}
```

### Builder Trait
```rust
// ── Builder trait ───────────────────────────────────────────────────────────

/// Common builder interface for all loan application types.
pub trait LoanApplicationBuilder {
    /// Set the applicant's name.
    fn set_personal_info(&mut self, first: &str, last: &str);

    /// Set the loan principal amount and term.
    fn set_loan_terms(&mut self, amount: f64, term_months: u32);

    /// Set the annual interest rate (as a percentage, e.g. 6.25).
    fn set_interest_rate(&mut self, rate: f64);

    /// Add a required document name to the application.
    fn add_document(&mut self, document: &str);

    /// Consume the builder and produce the finished loan application.
    fn get_result(self: Box<Self>) -> LoanApplication;
}
```

### PersonalLoanBuilder (ConcreteBuilder)
```rust
// ── PersonalLoanBuilder ─────────────────────────────────────────────────────

/// Concrete builder for unsecured personal loans.
pub struct PersonalLoanBuilder {
    first_name: String,
    last_name: String,
    amount: f64,
    term_months: u32,
    annual_rate: f64,
    documents: Vec<String>,
}

impl PersonalLoanBuilder {
    pub fn new() -> Self {
        Self {
            first_name: String::new(),
            last_name: String::new(),
            amount: 0.0,
            term_months: 0,
            annual_rate: 0.0,
            documents: Vec::new(),
        }
    }
}

impl LoanApplicationBuilder for PersonalLoanBuilder {
    fn set_personal_info(&mut self, first: &str, last: &str) {
        self.first_name = first.to_string();
        self.last_name = last.to_string();
    }

    fn set_loan_terms(&mut self, amount: f64, term_months: u32) {
        self.amount = amount;
        self.term_months = term_months;
    }

    fn set_interest_rate(&mut self, rate: f64) {
        self.annual_rate = rate;
    }

    fn add_document(&mut self, document: &str) {
        self.documents.push(document.to_string());
    }

    fn get_result(self: Box<Self>) -> LoanApplication {
        let loan = PersonalLoan {
            first_name: self.first_name.clone(),
            last_name: self.last_name.clone(),
            amount: self.amount,
            term_months: self.term_months,
            annual_rate: self.annual_rate,
            documents: self.documents.clone(),
        };
        LoanApplication::Personal(loan)
    }
}
```

```rust
/// Domain struct for property appraisal data.
#[derive(Debug, Clone)]
pub struct Appraisal {
    pub address: String,
    pub city: String,
    pub appraised_value: f64,
}

/// Concrete builder for mortgage loan applications.
pub struct MortgageBuilder {
    first_name: String,
    last_name: String,
    amount: f64,
    term_months: u32,
    annual_rate: f64,
    documents: Vec<String>,
    appraisal: Option<Appraisal>,
    down_payment: f64,
}

impl MortgageBuilder {
    pub fn new() -> Self {
        Self {
            first_name: String::new(),
            last_name: String::new(),
            amount: 0.0,
            term_months: 0,
            annual_rate: 0.0,
            documents: Vec::new(),
            appraisal: None,
            down_payment: 0.0,
        }
    }

    /// Set the property address, city, and appraised value.
    pub fn set_property_info(&mut self, address: &str, city: &str, appraised_value: f64) {
        self.appraisal = Some(Appraisal {
            address: address.to_string(),
            city: city.to_string(),
            appraised_value,
        });
    }

    /// Set the down payment amount.
    pub fn set_down_payment(&mut self, down_payment: f64) {
        self.down_payment = down_payment;
    }
}

impl LoanApplicationBuilder for MortgageBuilder {
    fn set_personal_info(&mut self, first: &str, last: &str) {
        self.first_name = first.to_string();
        self.last_name = last.to_string();
    }

    fn set_loan_terms(&mut self, amount: f64, term_months: u32) {
        self.amount = amount;
        self.term_months = term_months;
    }

    fn set_interest_rate(&mut self, rate: f64) {
        self.annual_rate = rate;
    }

    fn add_document(&mut self, document: &str) {
        self.documents.push(document.to_string());
    }

    fn get_result(self: Box<Self>) -> LoanApplication {
        let appraisal = self
            .appraisal
            .expect("MortgageBuilder: property info must be set before building");
        let loan = MortgageLoan {
            first_name: self.first_name.clone(),
            last_name: self.last_name.clone(),
            amount: self.amount,
            term_months: self.term_months,
            annual_rate: self.annual_rate,
            property_address: appraisal.address,
            property_city: appraisal.city,
            appraised_value: appraisal.appraised_value,
            down_payment: self.down_payment,
            documents: self.documents.clone(),
        };
        LoanApplication::Mortgage(loan)
    }
}
```

### LoanDirector (Director)
```rust
// ── Director ────────────────────────────────────────────────────────────────

/// Orchestrates the construction of loan applications using any builder that
/// implements `LoanApplicationBuilder`.
pub struct LoanDirector;

impl LoanDirector {
    /// Build a standard personal loan application.
    pub fn construct_personal_loan(builder: &mut Box<dyn LoanApplicationBuilder>) {
        builder.set_personal_info("Alice", "Johnson");
        builder.set_loan_terms(15_000.0, 60);
        builder.set_interest_rate(11.99);
        builder.add_document("ID");
        builder.add_document("Income Verification");
    }

    /// Build a full mortgage application including property and down payment details.
    pub fn construct_mortgage(builder: &mut MortgageBuilder) {
        builder.set_personal_info("Bob", "Martinez");
        builder.set_loan_terms(350_000.0, 360);
        builder.set_interest_rate(6.25);
        builder.set_property_info("123 Main St", "Springfield", 450_000.0);
        builder.set_down_payment(70_000.0);
        builder.add_document("ID");
        builder.add_document("Tax Returns");
        builder.add_document("Appraisal Report");
    }

    /// Build an auto loan application including vehicle information and dealership.
    pub fn construct_auto_loan(builder: &mut AutoLoanBuilder) {
        builder.set_personal_info("Carol", "Chen");
        builder.set_loan_terms(32_000.0, 60);
        builder.set_interest_rate(7.49);
        builder.set_vehicle_info(
            2024,
            "Honda",
            "Accord",
            "1HGCV1F34PA012345",
            35_000.0,
        );
        builder.set_dealership("City Motors");
        builder.add_document("ID");
        builder.add_document("Vehicle Registration");
    }
}
```

```rust
fn main() {
```

### Language-Specific Notes

- **Traits as interfaces**: Rust uses `trait` definitions to express the builder contract, which is comparable to interfaces in other languages but allows default method implementations and associated types for additional flexibility.
- **Struct composition over inheritance**: Each concrete builder owns its fields directly and composes domain types (`Appraisal`, `VehicleInfo`) rather than inheriting from a base builder class, reflecting Rust's composition-oriented design philosophy.
- **Ownership and move semantics**: The `get_result` method consumes the builder via `self: Box<Self>`, transferring ownership of the built data into the product. This prevents accidental reuse of a builder after extraction and enforces a single-use lifecycle at the type level.
- **Generics with trait bounds for the director**: The `LoanDirector` uses either `Box<dyn LoanApplicationBuilder>` for dynamic dispatch (personal loan) or concrete generic parameters (mortgage and auto loan via direct struct methods), showing both approaches available in Rust 2021.
- **No null — Option\<T\> for optional fields**: Fields like `appraisal` and `vehicle` are wrapped in `Option<T>`, and the builder panics with `expect()` if `get_result` is called before setting them. This eliminates null-pointer bugs by making the absence of a value explicit in the type system.
- **Formatting controls**: The `Display` trait implementations use `format!()` with `{:.2}` for currency precision, `{:>7.2}` alignment patterns, and `join()` for document lists, giving full control over output formatting without runtime reflection.
