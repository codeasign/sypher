## Overview

The TypeScript implementation of the Memento pattern brings strict type safety,
encapsulation enforcement through private class fields (`#`), and a clean
separation between the opaque memento interface and the concrete snapshot class.
In this stock market example, a `Portfolio` originator manages a collection of
stock holdings and a cash balance. Before each trade, the caller requests a
snapshot via `save()`, which returns a `Snapshot` interface with no visible
properties — the caretaker (`PortfolioHistory`) can store and retrieve these
opaque snapshots but cannot inspect their contents. Only the Portfolio itself
casts a `Snapshot` back to the concrete `PortfolioSnapshot` to restore state.

## How the Implementation Works

The class diagram below maps each participant of the Memento pattern to its
TypeScript representation. Note how `Snapshot` is a narrow interface —
completely opaque to external code — while `PortfolioSnapshot` implements it
and carries the actual holdings map and cash balance. Portfolio accesses the
concrete class via an internal cast, preserving encapsulation.

**OOP and SOLID breakdown:**

- **Single Responsibility** — `Portfolio` handles domain logic (buy/sell/tracking)
  and snapshot creation/restoration. `PortfolioHistory` manages the snapshot
  stack. `PortfolioSnapshot` is a passive data carrier. Each class has one
  reason to change.
- **Open/Closed** — New state fields can be added to `PortfolioSnapshot` without
  modifying `PortfolioHistory` because the caretaker only sees the opaque
  `Snapshot` interface.
- **Liskov Substitution** — `PortfolioSnapshot` implements `Snapshot` and can be
  passed wherever `Snapshot` is expected. The caretaker never needs to know the
  concrete type.
- **Interface Segregation** — The `Snapshot` interface is deliberately empty (no
  methods or properties). External code gets the narrowest possible contract.
  Only `Portfolio` accesses the full `PortfolioSnapshot` class.
- **Dependency Inversion** — `PortfolioHistory` depends on the `Snapshot`
  abstraction, not on `PortfolioSnapshot` directly. The originator creates the
  concrete memento, but the caretaker stores it by its interface.

## TypeScript Implementation

### Holding (Value Object)

```typescript
// Memento Pattern — TypeScript (strict mode)
// Stock Market Portfolio with Undo Support
// Participants: Portfolio (Originator), PortfolioSnapshot / Snapshot (Memento),
//               PortfolioHistory (Caretaker), Holding (value object)
//
// Run with: ts-node --strict 03-typescript.mdx  (or copy to .ts file)

interface Holding {
  readonly shares: number;
  readonly totalCost: number;
}

function createHolding(shares: number, totalCost: number): Holding {
  return { shares, totalCost };
}

function avgCostPerShare(h: Holding): number {
  return h.totalCost / h.shares;
}
```

### Snapshot (Memento Interface)

```typescript
// Opaque memento interface — external code (PortfolioHistory, main) only sees this.
// It exposes no properties — the caretaker cannot inspect internals.
interface Snapshot {
  // Opaque — no public members
}
```

### PortfolioSnapshot (Concrete Memento)

```typescript
// Only Portfolio accesses this class directly, typically via a cast.
class PortfolioSnapshot implements Snapshot {
  public readonly holdings: ReadonlyMap<string, Holding>;
  public readonly cashBalance: number;

  constructor(holdings: Map<string, Holding>, cashBalance: number) {
    // Deep copy to prevent mutation after snapshot creation
    const copied = new Map<string, Holding>();
    for (const [symbol, h] of holdings) {
      copied.set(symbol, { shares: h.shares, totalCost: h.totalCost });
    }
    this.holdings = copied;
    this.cashBalance = cashBalance;
  }
}
```

### Portfolio (Originator)

```typescript
class Portfolio {
  #holdings: Map<string, Holding>;
  #cashBalance: number;

  constructor(initialCash: number) {
    this.#holdings = new Map<string, Holding>();
    this.#cashBalance = initialCash;
  }

  /** Deducts cash and adds or updates a holding by average-cost method. */
  buyTicker(symbol: string, shares: number, price: number): void {
    const cost = shares * price;
    if (cost > this.#cashBalance) {
      throw new Error(
        `Insufficient funds: need $${cost.toFixed(2)}, ` +
        `have $${this.#cashBalance.toFixed(2)}`
      );
    }
    this.#cashBalance -= cost;
    const existing = this.#holdings.get(symbol);
    if (existing !== undefined) {
      this.#holdings.set(symbol, {
        shares: existing.shares + shares,
        totalCost: existing.totalCost + cost,
      });
    } else {
      this.#holdings.set(symbol, { shares, totalCost: cost });
    }
  }

  /**
   * Sells shares from an existing position using average-cost basis.
   * Adds proceeds to cash. Removes the holding if all shares are sold.
   * Returns the proceeds amount.
   */
  sellTicker(symbol: string, shares: number): number {
    const existing = this.#holdings.get(symbol);
    if (existing === undefined) {
      throw new Error(`No holdings found for symbol "${symbol}"`);
    }
    if (shares > existing.shares) {
      throw new Error(
        `Insufficient shares of ${symbol}: ` +
        `want to sell ${shares}, have ${existing.shares}`
      );
    }
    const avgPrice = avgCostPerShare(existing);
    const proceeds = shares * avgPrice;
    this.#cashBalance += proceeds;
    const remaining = existing.shares - shares;
    if (remaining === 0) {
      this.#holdings.delete(symbol);
    } else {
      const remainingCost =
        existing.totalCost * (remaining / existing.shares);
      this.#holdings.set(symbol, {
        shares: remaining,
        totalCost: remainingCost,
      });
    }
    return proceeds;
  }

  /** Captures current state and returns an opaque Snapshot. */
  save(): Snapshot {
    return new PortfolioSnapshot(this.#holdings, this.#cashBalance);
  }

  /** Restores internal state from a previously saved Snapshot. */
  restore(snapshot: Snapshot): void {
    const memento = snapshot as PortfolioSnapshot;
    this.#holdings = new Map<string, Holding>();
    for (const [symbol, h] of memento.holdings) {
      this.#holdings.set(symbol, { shares: h.shares, totalCost: h.totalCost });
    }
    this.#cashBalance = memento.cashBalance;
  }

  /** Returns a human-readable summary of the portfolio. */
  toString(): string {
    const lines: string[] = [];
    for (const [symbol, h] of this.#holdings) {
      const avg = avgCostPerShare(h);
      lines.push(
        `  ${symbol}: ${h.shares} shares @ $${avg.toFixed(2)} avg = ` +
        `$${h.totalCost.toFixed(2)}`
      );
    }
    if (lines.length === 0) {
      lines.push('  (empty)');
    }
    lines.push(`  Cash: $${this.#cashBalance.toFixed(2)}`);
    return lines.join('\n');
  }
}
```

### PortfolioHistory (Caretaker)

```typescript
class PortfolioHistory {
  #snapshots: Snapshot[] = [];

  /** Pushes a snapshot onto the undo stack. */
  push(snapshot: Snapshot): void {
    this.#snapshots.push(snapshot);
  }

  /** Pops the most recent snapshot from the undo stack. */
  pop(): Snapshot {
    const snapshot = this.#snapshots.pop();
    if (snapshot === undefined) {
      throw new Error('No snapshots to undo');
    }
    return snapshot;
  }

  /** Returns true if there are snapshots available for undo. */
  canUndo(): boolean {
    return this.#snapshots.length > 0;
  }
}
```

### Main / Driver

```typescript
function main(): void {
  const portfolio = new Portfolio(300_000);
  const history = new PortfolioHistory();

  console.log('=== Stock Portfolio with Undo (Memento Pattern) ===');
  console.log('\nInitial state:');
  console.log(portfolio.toString());

  // --- Trade 1: Buy 100 shares of AAPL at $150 ---
  console.log('\n--- Buy 100 AAPL @ $150.00 ---');
  history.push(portfolio.save());
  console.log('Snapshot saved (can undo: ' + history.canUndo() + ')');
  portfolio.buyTicker('AAPL', 100, 150.00);
  console.log(portfolio.toString());

  // --- Trade 2: Buy 50 shares of GOOGL at $2,800 ---
  console.log('\n--- Buy 50 GOOGL @ $2800.00 ---');
  history.push(portfolio.save());
  console.log('Snapshot saved (can undo: ' + history.canUndo() + ')');
  portfolio.buyTicker('GOOGL', 50, 2800.00);
  console.log(portfolio.toString());

  // --- Trade 3: Buy 200 shares of MSFT at $330 ---
  console.log('\n--- Buy 200 MSFT @ $330.00 ---');
  history.push(portfolio.save());
  console.log('Snapshot saved (can undo: ' + history.canUndo() + ')');
  portfolio.buyTicker('MSFT', 200, 330.00);
  console.log(portfolio.toString());

  // --- Trade 4: Sell 30 shares of AAPL ---
  console.log('\n--- Sell 30 AAPL ---');
  history.push(portfolio.save());
  console.log('Snapshot saved (can undo: ' + history.canUndo() + ')');
  const proceeds = portfolio.sellTicker('AAPL', 30);
  console.log('Proceeds from sale: $' + proceeds.toFixed(2));
  console.log(portfolio.toString());

  // --- Undo the sell ---
  console.log('\n--- Undo last operation (sell) ---');
  const snap1 = history.pop();
  portfolio.restore(snap1);
  console.log('Restored to previous snapshot');
  console.log(portfolio.toString());

  // --- Undo the MSFT purchase ---
  console.log('\n--- Undo MSFT purchase ---');
  const snap2 = history.pop();
  portfolio.restore(snap2);
  console.log('Restored to previous snapshot');
  console.log(portfolio.toString());

  console.log('\n=== End ===');
}

main();
```

## Language-Specific Notes

- **Private fields (`#`)** — TypeScript's `#` private fields provide true
  runtime encapsulation (ES2022+), unlike `private` which is only a compile-time
  annotation. The Portfolio's holdings map and cash balance are fully hidden
  from external access, reinforcing the Memento pattern's encapsulation goal.

- **Narrow `Snapshot` interface** — By defining `Snapshot` as an empty
  interface and having `PortfolioSnapshot` implement it, the caretaker receives
  an opaque token it can store and return without ever inspecting internals.
  Only the Portfolio class performs the `as PortfolioSnapshot` cast, keeping the
  knowledge of the internal structure in one place.

- **`ReadonlyMap` for snapshot immutability** — The `PortfolioSnapshot` exposes
  its holdings as `ReadonlyMap<string, Holding>` rather than `Map`, preventing
  accidental mutations after the snapshot is created. The constructor performs a
  deep copy so that subsequent changes to the original Portfolio do not
  retroactively modify saved snapshots.

- **Strict mode enforcement** — TypeScript strict mode (`strict: true` in
  `tsconfig.json`) catches null-checks for `Map.get()`, ensures `pop()` handles
  the `undefined` case, and prevents implicit `any` types throughout the code.
  All method return types and parameter types are explicitly annotated.

- **Structural typing and brand safety** — TypeScript's structural type system
  means two shapes with the same properties are interchangeable. The opaque
  `Snapshot` interface with no members relies on convention rather than the
  compiler to enforce opacity — a deliberate design choice that mirrors
  languages where encapsulation is enforced by access modifiers rather than the
  type system.

- **Utility functions vs. class methods** — `createHolding` and `avgCostPerShare`
  are standalone functions rather than methods on `Holding` because `Holding` is
  an interface (not a class). This keeps the value object lightweight and avoids
  forcing a class-based design where a plain object suffices.
