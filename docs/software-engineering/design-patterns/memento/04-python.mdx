## Implementation

Below is a complete Python 3.11+ implementation of the Memento pattern using a stock market portfolio domain. The `Portfolio` (originator) manages stock holdings and cash balance, `PortfolioSnapshot` (memento) captures the portfolio's internal state as an opaque object, and `PortfolioHistory` (caretaker) maintains a stack of snapshots to support undo operations.

#### OOP Concepts Applied

1. **Encapsulation** — Portfolio hides its `_holdings` dict and `_cash_balance` behind private attributes. The `__str__` method provides controlled read access. PortfolioSnapshot hides its state behind attribute access methods that only Portfolio should call.

2. **Abstraction** — The save/restore interface on Portfolio defines a clean contract for state capture and restoration, decoupled from the business logic of buy/sell operations.

3. **Information Hiding** — PortfolioHistory stores snapshots as opaque `PortfolioSnapshot` objects. It never inspects or modifies their contents. The memento's internal fields are only meaningful to the Portfolio that created them.

#### SOLID Principles Applied

4. **SRP** — Portfolio owns business logic (buy/sell). PortfolioSnapshot owns state serialization. PortfolioHistory owns undo history management. Each class has a single, well-defined responsibility.

5. **OCP** — Adding new state fields to Portfolio (e.g., dividend tracking, margin balance) requires updating only Portfolio and PortfolioSnapshot. PortfolioHistory and the client code do not change.

6. **LSP** — PortfolioSnapshot is not subclassed, but the pattern ensures that any snapshot returned by `save()` can be consumed by `restore()` without type checks.

7. **ISP** — Each interface is minimal: save/restore on Portfolio, push/pop/can_undo on PortfolioHistory, get_holdings/get_cash_balance on PortfolioSnapshot.

8. **DIP** — Portfolio depends on the concrete PortfolioSnapshot type it creates, which is acceptable because the originator owns the memento definition. The client depends on the Portfolio and PortfolioHistory abstractions, not on snapshot internals.

### Holding (Value Object)

```python
from __future__ import annotations

import copy
from dataclasses import dataclass
from typing import Dict, List


@dataclass
class Holding:
    """A single stock holding: shares owned and total cost basis."""

    shares: int
    total_cost: float

    @property
    def avg_cost_per_share(self) -> float:
        """Average cost per share based on total cost basis."""
        if self.shares == 0:
            return 0.0
        return self.total_cost / self.shares
```

### PortfolioSnapshot (Memento)

```python
class PortfolioSnapshot:
    """Memento — opaque state container for Portfolio.

    Only Portfolio reads or writes the internal attributes.
    PortfolioHistory (the caretaker) holds references to these
    objects but never inspects their contents.
    """

    def __init__(self, holdings: Dict[str, Holding], cash_balance: float) -> None:
        self._holdings = holdings
        self._cash_balance = cash_balance

    def get_holdings(self) -> Dict[str, Holding]:
        """Return the holdings dict (called only by Portfolio)."""
        return self._holdings

    def get_cash_balance(self) -> float:
        """Return the cash balance (called only by Portfolio)."""
        return self._cash_balance
```

### Portfolio (Originator)

```python
class Portfolio:
    """Originator — stock portfolio with buy/sell and undo support."""

    def __init__(self, initial_cash: float = 0.0) -> None:
        self._holdings: Dict[str, Holding] = {}
        self._cash_balance: float = initial_cash

    def buy_ticker(self, symbol: str, shares: int, price: float) -> None:
        """Buy shares of a ticker at the given price per share."""
        cost = shares * price
        if cost > self._cash_balance:
            raise ValueError(
                f'Insufficient funds: need ${cost:,.2f}, '
                f'have ${self._cash_balance:,.2f}'
            )
        self._cash_balance -= cost
        if symbol in self._holdings:
            existing = self._holdings[symbol]
            total_shares = existing.shares + shares
            total_cost = existing.total_cost + cost
            self._holdings[symbol] = Holding(total_shares, total_cost)
        else:
            self._holdings[symbol] = Holding(shares, cost)

    def sell_ticker(self, symbol: str, shares: int) -> None:
        """Sell shares of a ticker using average cost basis."""
        if symbol not in self._holdings:
            raise ValueError(f'No position found for {symbol}')
        existing = self._holdings[symbol]
        if shares > existing.shares:
            raise ValueError(
                f'Cannot sell {shares} shares of {symbol}; '
                f'only {existing.shares} held'
            )
        avg_price = existing.avg_cost_per_share
        proceeds = shares * avg_price
        self._cash_balance += proceeds
        remaining = existing.shares - shares
        if remaining == 0:
            del self._holdings[symbol]
        else:
            remaining_cost = existing.total_cost * (remaining / existing.shares)
            self._holdings[symbol] = Holding(remaining, round(remaining_cost, 2))

    def save(self) -> PortfolioSnapshot:
        """Create a memento capturing the current state."""
        return PortfolioSnapshot(
            holdings=copy.deepcopy(self._holdings),
            cash_balance=self._cash_balance,
        )

    def restore(self, snapshot: PortfolioSnapshot) -> None:
        """Restore internal state from a memento."""
        self._holdings = copy.deepcopy(snapshot.get_holdings())
        self._cash_balance = snapshot.get_cash_balance()

    @property
    def cash_balance(self) -> float:
        """Current cash balance (read-only view)."""
        return self._cash_balance

    @property
    def holdings(self) -> Dict[str, Holding]:
        """Current holdings (read-only copy)."""
        return dict(self._holdings)

    def __str__(self) -> str:
        """Human-readable portfolio summary."""
        lines = [f'Cash: ${self._cash_balance:,.2f}']
        if not self._holdings:
            lines.append('Holdings: (empty)')
        else:
            lines.append('Holdings:')
            for symbol in sorted(self._holdings.keys()):
                h = self._holdings[symbol]
                lines.append(
                    f'  {symbol}: {h.shares} shares @ '
                    f'avg ${h.avg_cost_per_share:,.2f}'
                )
        return '\n'.join(lines)
```

### PortfolioHistory (Caretaker)

```python
class PortfolioHistory:
    """Caretaker — manages a stack of PortfolioSnapshot objects."""

    def __init__(self) -> None:
        self._snapshots: List[PortfolioSnapshot] = []

    def push(self, snapshot: PortfolioSnapshot) -> None:
        """Push a snapshot onto the undo stack."""
        self._snapshots.append(snapshot)

    def pop(self) -> PortfolioSnapshot:
        """Pop the most recent snapshot from the undo stack."""
        if not self._snapshots:
            raise IndexError('No snapshots to undo')
        return self._snapshots.pop()

    def can_undo(self) -> bool:
        """Return True if at least one snapshot is available."""
        return len(self._snapshots) > 0
```

### Main / Driver

```python
def main() -> None:
    """Demonstrate Memento pattern with realistic stock trades."""
    portfolio = Portfolio(initial_cash=1_000_000.0)
    history = PortfolioHistory()

    print('=== Memento Pattern — Stock Portfolio ===')
    print()
    print('Initial state:')
    print(portfolio)
    print()

    # Trade 1: Buy 100 shares of AAPL at $150 per share
    print('>>> Trade 1: Buy 100 AAPL @ $150')
    print('    Pushing snapshot...')
    history.push(portfolio.save())
    portfolio.buy_ticker('AAPL', 100, 150.0)
    print(portfolio)
    print()

    # Trade 2: Buy 50 shares of GOOGL at $2,800 per share
    print('>>> Trade 2: Buy 50 GOOGL @ $2,800')
    print('    Pushing snapshot...')
    history.push(portfolio.save())
    portfolio.buy_ticker('GOOGL', 50, 2800.0)
    print(portfolio)
    print()

    # Trade 3: Buy 200 shares of MSFT at $330 per share
    print('>>> Trade 3: Buy 200 MSFT @ $330')
    print('    Pushing snapshot...')
    history.push(portfolio.save())
    portfolio.buy_ticker('MSFT', 200, 330.0)
    print(portfolio)
    print()

    # Undo the MSFT trade
    print('>>> Undo: Remove MSFT position')
    snapshot = history.pop()
    portfolio.restore(snapshot)
    print(portfolio)
    print()

    # Undo the GOOGL trade
    print('>>> Undo: Remove GOOGL position')
    snapshot = history.pop()
    portfolio.restore(snapshot)
    print(portfolio)
    print()

    # Verify can_undo still works (one snapshot remains: initial state)
    print(f'>>> Can undo more trades? {history.can_undo()}')


if __name__ == '__main__':
    main()
```

### Language-Specific Notes

- **Underscore naming convention** — Python uses a single leading underscore (`_holdings`, `_cash_balance`, `_snapshots`) to mark attributes as internal. This is a convention, not a language-enforced restriction, but it signals to other developers that these attributes are private and should not be accessed directly.

- **`@dataclass` for value objects** — The `Holding` class is a `@dataclass` with auto-generated `__init__`, `__repr__`, and `__eq__` methods. This reduces boilerplate while keeping the data structure transparent. The `avg_cost_per_share` property adds computed behavior without breaking the data-class simplicity.

- **`copy.deepcopy` for state isolation** — The `save()` and `restore()` methods use `copy.deepcopy` to ensure that the snapshot holds an independent copy of the holdings dict and its `Holding` objects. This prevents accidental mutation of the snapshot's state if the portfolio is modified after saving.

- **List as a stack** — `PortfolioHistory` uses a plain `List` with `append` (push) and `pop` (pop). Python's list provides O(1) amortized push and pop from the end, making it a natural choice for the undo stack without requiring a separate `Stack` class.

- **Type hints throughout** — All method signatures and attributes use Python 3.11+ type hints (`Dict[str, Holding]`, `List[PortfolioSnapshot]`, `-> None`). The `from __future__ import annotations` import enables forward references and deferred evaluation of type annotations.

- **No abstract base class needed** — Unlike some patterns, the Memento pattern does not require an ABC. The memento is a concrete class that the originator owns. Python's duck typing and name-mangling conventions (`__` prefix for name mangling) could provide stronger encapsulation, but a single underscore is sufficient for idiomatic Python.
