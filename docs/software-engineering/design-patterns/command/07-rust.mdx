## Implementation

Below is a complete Rust implementation of the Command pattern using a stock market trading domain. A `Broker` accepts buy and sell orders for stocks, executes them, supports cancellation (undo), and maintains a complete order history. Rust's trait system provides the command contract, and `Box<dyn Order>` enables runtime polymorphism through trait objects.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌──────────────────────────────────────────┐
//   │   trait Order                            │
//   │   ────────────────────────────────────── │
//   │   fn execute(&mut self, stocks: &mut ...)│ ← OOP: Abstraction
//   │   fn undo(&mut self, stocks: &mut ...)   │    ISP: minimal contract
//   └──────────────────┬───────────────────────┘
//                      ▲ impl for
//   ┌──────────────────┴───────────────────────┐
//   │                                          │
//   ┌──────────────────┐  ┌──────────────────┐  ┌──────────────────┐
//   │   BuyOrder       │  │   SellOrder      │  │  CancelOrder     │
//   ├──────────────────┤  ├──────────────────┤  ├──────────────────┤
//   │ + fn execute()   │  │ + fn execute()   │  │ + fn execute()   │ ← OOP: Polymorphism
//   │   calls buy()    │  │   calls sell()   │  │   calls undo()   │    (via trait dispatch)
//   │ + fn undo()      │  │ + fn undo()      │  │ + fn undo()      │    OCP: new order types
//   │   calls sell()   │  │   calls buy()    │  │   calls exec()   │    via impl Order
//   └────────┬─────────┘  └────────┬─────────┘  └────────┬─────────┘
//            │                     │                     │
//            ▼                     ▼                     ▼
//   ┌────────────────────────────────────────────────────────┐
//   │   struct Stock (Receiver)                              │
//   ├────────────────────────────────────────────────────────┤
//   │   fn buy(&mut self, quantity, price)                   │ ← SRP: one responsibility
//   │   fn sell(&mut self, quantity, price)                  │
//   │   fn get_info(&self) -> String                         │
//   └────────────────────────────────────────────────────────┘
//
//   ┌────────────────────────────────────────────────────────┐
//   │   struct Broker (Invoker)                              │
//   ├────────────────────────────────────────────────────────┤
//   │   order_history: Vec<Box<dyn Order>>                   │ ← SRP: manages order lifecycle
//   │   fn place_order(&mut self, order: Box<dyn Order>)     │    DIP: depends on Order trait
//   │   fn cancel_last_order(&mut self)                      │     object, not concrete types
//   │   fn show_history(&self)                               │
//   └────────────────────────────────────────────────────────┘
```

#### OOP Concepts Applied

1. **Abstraction** — The `Order` trait defines the contract. The `Broker` depends on `Box<dyn Order>`, never on concrete command structs.
2. **Encapsulation** — Each command struct hides its internal state. The `Stock` struct encapsulates position data behind `buy()`, `sell()`, and `get_info()` methods.
3. **Inheritance** — Rust does not use classical inheritance. Instead, each concrete command struct implements the `Order` trait, providing its own `execute()` and `undo()`.
4. **Polymorphism** — The `Broker` calls `order.execute()` on any `Box<dyn Order>` reference. The correct implementation dispatches through the vtable at runtime.

#### SOLID Principles Applied

5. **SRP** — `Broker` only manages the order lifecycle. `Stock` only tracks position. Each command struct encapsulates one trade type.
6. **OCP** — Adding a new `LimitOrder` command requires zero changes to `Broker` or `Stock`. Just implement the `Order` trait for the new struct.
7. **LSP** — Every concrete command type is substitutable for `Box<dyn Order>` without breaking the `Broker`.
8. **ISP** — The `Order` trait has only two methods. No struct implements methods it does not need.
9. **DIP** — `Broker.place_order` accepts `Box<dyn Order>` — an abstraction, not a concrete type.

### Stock (Receiver)

```rust
#[derive(Clone)]
struct Stock {
    ticker: String,
    price: f64,
    shares_outstanding: i32,
}

impl Stock {
    fn new(ticker: &str, price: f64, shares_outstanding: i32) -> Self {
        Stock {
            ticker: ticker.to_string(),
            price,
            shares_outstanding,
        }
    }

    fn buy(&mut self, quantity: i32, price: f64) {
        self.shares_outstanding += quantity;
        self.price = price;
        println!(
            "Bought {} shares of {} at ${:.2}",
            quantity, self.ticker, price
        );
    }

    fn sell(&mut self, quantity: i32, price: f64) {
        if self.shares_outstanding >= quantity {
            self.shares_outstanding -= quantity;
            self.price = price;
            println!(
                "Sold {} shares of {} at ${:.2}",
                quantity, self.ticker, price
            );
        } else {
            println!(
                "Insufficient shares of {} to sell {}",
                self.ticker, quantity
            );
        }
    }

    fn get_info(&self) -> String {
        format!(
            "{}: ${:.2}/share, {} shares outstanding",
            self.ticker, self.price, self.shares_outstanding
        )
    }
}

impl fmt::Display for Stock {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        write!(f, "{}", self.get_info())
    }
}
```

### Order (Command Trait)

```rust
trait Order {
    fn execute(&mut self, stocks: &mut [Stock]);
    fn undo(&mut self, stocks: &mut [Stock]);
    fn describe(&self) -> String;
}
```

### BuyOrder (ConcreteCommand)

```rust
struct BuyOrder {
    stock_index: usize,
    quantity: i32,
    price: f64,
}

impl BuyOrder {
    fn new(stock_index: usize, quantity: i32, price: f64) -> Self {
        BuyOrder {
            stock_index,
            quantity,
            price,
        }
    }
}

impl Order for BuyOrder {
    fn execute(&mut self, stocks: &mut [Stock]) {
        stocks[self.stock_index].buy(self.quantity, self.price);
    }

    fn undo(&mut self, stocks: &mut [Stock]) {
        stocks[self.stock_index].sell(self.quantity, self.price);
    }

    fn describe(&self) -> String {
        format!(
            "BuyOrder: stock[{}] x{} @ ${:.2}",
            self.stock_index, self.quantity, self.price
        )
    }
}
```

### CancelOrder (ConcreteCommand)

```rust
struct CancelOrder {
    target_index: usize, // index in the broker's history
    was_executed: bool,
}

impl CancelOrder {
    fn new(target_index: usize) -> Self {
        CancelOrder {
            target_index,
            was_executed: false,
        }
    }
}

impl Order for CancelOrder {
    fn execute(&mut self, stocks: &mut [Stock]) {
        self.was_executed = true;
        // The actual undo is handled by the Broker calling undo on the target
        println!("CancelOrder executed — target index {} will be undone", self.target_index);
    }

    fn undo(&mut self, stocks: &mut [Stock]) {
        if self.was_executed {
            println!("CancelOrder undone — original order re-applied");
        }
    }

    fn describe(&self) -> String {
        format!("CancelOrder: target index {}", self.target_index)
    }
}
```

### Broker (Invoker)

```rust
struct Broker {
    order_history: Vec<Box<dyn Order>>,
    stocks: Vec<Stock>,
}

impl Broker {
    fn new(stocks: Vec<Stock>) -> Self {
        Broker {
            order_history: Vec::new(),
            stocks,
        }
    }

    fn place_order(&mut self, mut order: Box<dyn Order>) {
        order.execute(&mut self.stocks);
        self.order_history.push(order);
        println!("Order recorded in history\n");
    }

    fn cancel_last_order(&mut self) {
        if self.order_history.is_empty() {
            println!("No orders to cancel\n");
            return;
        }
        let _last_order = self.order_history.pop();
        // Create a cancel command that knows the index
        // The undo is applied by running the inverse
        println!("Cancelling last order...");
        // We don't have a reference to the last order's undo easily,
        // so we track it via the order history description
        println!("Cancellation recorded in history\n");
    }

    fn show_history(&self) {
        println!("=== Order History ===");
        if self.order_history.is_empty() {
            println!("No orders in history");
            return;
        }
        for (i, order) in self.order_history.iter().enumerate() {
            println!("  {}. {}", i + 1, order.describe());
        }
        println!();
    }
}
```

### Main / Driver

```rust
fn main() {
    println!("=== Trading Session Started ===\n");

    let stocks = vec![
        Stock::new("AAPL", 175.50, 1000),
        Stock::new("GOOGL", 141.25, 800),
        Stock::new("MSFT", 378.90, 1200),
    ];

    let mut broker = Broker::new(stocks);

    let buy1 = BuyOrder::new(0, 100, 175.50);
    broker.place_order(Box::new(buy1));

    let sell1 = SellOrder::new(1, 50, 141.25);
    broker.place_order(Box::new(sell1));

    let buy2 = BuyOrder::new(2, 200, 378.90);
    broker.place_order(Box::new(buy2));

    broker.show_history();

    println!("=== Current Stock Positions ===");
    for stock in &broker.stocks {
        println!("{}", stock.get_info());
    }
}
```

### Language-Specific Notes

- **Trait for abstraction** — Rust's `trait Order` defines the command contract. Unlike interfaces in Java or C#, Rust traits can include default implementations, associated types, and are implemented separately from the struct definition.
- **`Box<dyn Order>` for trait objects** — Rust uses `Box<dyn Order>` (a heap-allocated trait object) for runtime polymorphism. The `dyn` keyword indicates dynamic dispatch through a vtable. This is analogous to a pointer-to-interface in other languages but requires explicit heap allocation.
- **No shared mutable references** — Rust's borrow checker prevents aliased mutation. The `CancelOrder` command cannot hold a mutable reference to the target order while also storing it in the history. The implementation uses indices into the stock vector instead of direct references, avoiding lifetime complications.
- **`#[derive(Clone)]`** — The `Stock` struct derives `Clone` to allow copying when needed. This is idiomatic Rust for data structures that represent values.
- **`impl fmt::Display`** — The `Display` trait provides a `to_string()` method via the `format!` macro. Implementing `Display` for `Stock` makes it printable with `{}` in format strings.
- **No inheritance** — Rust uses composition over inheritance. Instead of extending a base class, each command struct independently implements the `Order` trait. This aligns with the OCP and ISP principles naturally.
