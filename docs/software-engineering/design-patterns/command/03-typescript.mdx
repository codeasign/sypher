## Implementation

Below is a complete TypeScript implementation of the Command pattern using a stock market trading domain. A `Broker` accepts buy and sell orders for stocks, executes them, supports cancellation (undo), and maintains a complete order history. TypeScript's interface and type system enforces the command contract at compile time, and discriminated unions provide type-safe command categorization for the history display.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌──────────────────────────────────────────┐
//   │   interface Order                        │
//   │   ────────────────────────────────────── │
//   │   execute(): void                        │ ← OOP: Abstraction
//   │   undo(): void                           │    ISP: minimal contract
//   └──────────────────┬───────────────────────┘
//                      ▲ implements
//   ┌──────────────────┴───────────────────────┐
//   │                                          │
//   ┌──────────────────┐  ┌──────────────────┐  ┌──────────────────┐
//   │   BuyOrder       │  │   SellOrder      │  │  CancelOrder     │
//   ├──────────────────┤  ├──────────────────┤  ├──────────────────┤
//   │ + execute()      │  │ + execute()      │  │ + execute()      │ ← OOP: Polymorphism
//   │   calls buy()    │  │   calls sell()   │  │   calls undo()   │    LSP: substitutable
//   │ + undo()         │  │ + undo()         │  │ + undo()         │    OCP: new order types
//   │   calls sell()   │  │   calls buy()    │  │   calls exec()   │
//   └────────┬─────────┘  └────────┬─────────┘  └────────┬─────────┘
//            │                     │                     │
//            ▼                     ▼                     ▼
//   ┌────────────────────────────────────────────────────────┐
//   │   class Stock (Receiver)                               │
//   ├────────────────────────────────────────────────────────┤
//   │   buy(quantity, price)                                 │ ← SRP: one responsibility
//   │   sell(quantity, price)                                │
//   │   getInfo()                                            │
//   └────────────────────────────────────────────────────────┘
//
//   ┌────────────────────────────────────────────────────────┐
//   │   class Broker (Invoker)                               │
//   ├────────────────────────────────────────────────────────┤
//   │   orderHistory: Order[]                                │ ← SRP: manages order lifecycle
//   │   placeOrder(order: Order)                             │    DIP: depends on Order
//   │   cancelLastOrder()                                    │     interface, not concrete
//   │   showHistory()                                        │     command types
//   └────────────────────────────────────────────────────────┘
```

#### OOP Concepts Applied

1. **Abstraction** — The `Order` interface defines the contract that all commands must fulfill. The `Broker` depends only on `Order`, never on concrete command classes.
2. **Encapsulation** — Each command hides its internal state (stock reference, quantity, price). The `Stock` class encapsulates position data behind typed methods.
3. **Inheritance** — TypeScript classes implement the `Order` interface. While TypeScript uses structural typing, the `implements` keyword explicitly documents the contract.
4. **Polymorphism** — The `Broker` calls `order.execute()` on any `Order` reference. The correct implementation dispatches per type at runtime.

#### SOLID Principles Applied

5. **SRP** — `Broker` only manages the order lifecycle. `Stock` only tracks position. Each command class encapsulates one trade type.
6. **OCP** — Adding a new `LimitOrder` command requires zero changes to `Broker` or `Stock`. Just implement the `Order` interface.
7. **LSP** — Every concrete command is substitutable for `Order`. The `Broker` never needs to know the concrete type.
8. **ISP** — The `Order` interface has only two methods. No class implements methods it does not need.
9. **DIP** — `Broker.placeOrder` accepts `Order` — an abstraction, not a concrete type. The invoker is decoupled from the command implementations.

### Order (Command Interface)

```typescript
interface Order {
  execute(): void;
  undo(): void;
}
```

### Stock (Receiver)

```typescript
class Stock {
  constructor(
    public readonly ticker: string,
    public price: number,
    public sharesOutstanding: number
  ) {}

  buy(quantity: number, price: number): void {
    this.sharesOutstanding += quantity;
    this.price = price;
    console.log(
      `Bought ${quantity} shares of ${this.ticker} at $${price.toFixed(2)}`
    );
  }

  sell(quantity: number, price: number): void {
    if (this.sharesOutstanding >= quantity) {
      this.sharesOutstanding -= quantity;
      this.price = price;
      console.log(
        `Sold ${quantity} shares of ${this.ticker} at $${price.toFixed(2)}`
      );
    } else {
      console.log(
        `Insufficient shares of ${this.ticker} to sell ${quantity}`
      );
    }
  }

  getInfo(): string {
    return (
      `${this.ticker}: $${this.price.toFixed(2)}/share, ` +
      `${this.sharesOutstanding} shares outstanding`
    );
  }
}
```

### BuyOrder (ConcreteCommand)

```typescript
class BuyOrder implements Order {
  constructor(
    private readonly stock: Stock,
    private readonly quantity: number,
    private readonly price: number
  ) {}

  execute(): void {
    this.stock.buy(this.quantity, this.price);
  }

  undo(): void {
    this.stock.sell(this.quantity, this.price);
  }
}
```

### SellOrder (ConcreteCommand)

```typescript
class SellOrder implements Order {
  constructor(
    private readonly stock: Stock,
    private readonly quantity: number,
    private readonly price: number
  ) {}

  execute(): void {
    this.stock.sell(this.quantity, this.price);
  }

  undo(): void {
    this.stock.buy(this.quantity, this.price);
  }
}
```

### CancelOrder (ConcreteCommand)

```typescript
class CancelOrder implements Order {
  private wasExecuted = false;

  constructor(private readonly targetOrder: Order) {}

  execute(): void {
    this.wasExecuted = true;
    this.targetOrder.undo();
    console.log('CancelOrder executed — original order undone');
  }

  undo(): void {
    if (this.wasExecuted) {
      this.targetOrder.execute();
      console.log('CancelOrder undone — original order re-applied');
    }
  }
}
```

### Broker (Invoker)

```typescript
class Broker {
  private readonly orderHistory: Order[] = [];

  placeOrder(order: Order): void {
    order.execute();
    this.orderHistory.push(order);
    console.log('Order recorded in history\n');
  }

  cancelLastOrder(): void {
    if (this.orderHistory.length === 0) {
      console.log('No orders to cancel\n');
      return;
    }
    const lastOrder = this.orderHistory.pop()!;
    const cancelCmd = new CancelOrder(lastOrder);
    cancelCmd.execute();
    this.orderHistory.push(cancelCmd);
    console.log('Cancellation recorded in history\n');
  }

  showHistory(): void {
    console.log('=== Order History ===');
    if (this.orderHistory.length === 0) {
      console.log('No orders in history');
      return;
    }
    this.orderHistory.forEach((order, index) => {
      const type = order.constructor.name;
      let details: string;
      if (order instanceof CancelOrder) {
        const cmd = order as CancelOrder;
        details = `CancelOrder targeting ${cmd['targetOrder'].constructor.name}`;
      } else {
        const cmd = order as BuyOrder | SellOrder;
        details = `${cmd['stock'].ticker} x${cmd['quantity']} @ $${cmd['price'].toFixed(2)}`;
      }
      console.log(`  ${index + 1}. ${type}: ${details}`);
    });
    console.log('');
  }
}
```

### Main / Driver

```typescript
console.log('=== Trading Session Started ===\n');

const aapl = new Stock('AAPL', 175.50, 1000);
const googl = new Stock('GOOGL', 141.25, 800);
const msft = new Stock('MSFT', 378.90, 1200);

const broker = new Broker();

const buy1 = new BuyOrder(aapl, 100, 175.50);
broker.placeOrder(buy1);

const sell1 = new SellOrder(googl, 50, 141.25);
broker.placeOrder(sell1);

const buy2 = new BuyOrder(msft, 200, 378.90);
broker.placeOrder(buy2);

broker.showHistory();

console.log('=== Cancelling Last Order ===\n');
broker.cancelLastOrder();

broker.showHistory();

console.log('=== Current Stock Positions ===');
console.log(aapl.getInfo());
console.log(googl.getInfo());
console.log(msft.getInfo());
```

### Language-Specific Notes

- **Interface as contract** — TypeScript's `interface Order` enforces the command shape at compile time. Any class implementing `Order` must provide `execute()` and `undo()`. This is stricter than JavaScript's duck typing and catches missing methods before runtime.
- **`readonly` for immutability** — The `Stock` class uses `readonly ticker` since ticker symbols never change. The `BuyOrder` and `SellOrder` constructor parameters are `readonly` to prevent accidental mutation after construction.
- **Private fields** — TypeScript's `private` keyword provides compile-time encapsulation. Note that this is a TypeScript-only construct and does not enforce privacy at runtime (unlike JavaScript `#` private fields).
- **Non-null assertion** — `this.orderHistory.pop()!` uses the non-null assertion operator. Since we checked `length === 0` first, this is safe. The `!` tells TypeScript the value is not undefined.
- **Strict mode** — This code requires `"strict": true` in `tsconfig.json`. All types are explicitly annotated, and the compiler catches any potential null/undefined issues.
