Java provides the most explicit expression of the Adapter pattern through formal interfaces and compile-time contract enforcement. In the car domain, a Java adapter wraps a TeslaSupercharger behind the standard CCSCharger interface, enabling a fleet management system to charge mixed-brand EVs through a single API. Java's strong typing ensures the adapter implements every method of the target interface, and checked exceptions force the adapter to handle all error conditions explicitly.

## How the Implementation Works

| Aspect | Implementation |
|---|---|
| **Target** | `CCSCharger` interface with `String chargeDC(int powerKw)` |
| **Adaptee** | `TeslaSupercharger` class with `String sendCharge(int voltage, int amperage)` |
| **Adapter** | `TeslaToCCSAdapter implements CCSCharger`, holds a `TeslaSupercharger` field |
| **Client** | `ChargingStation` accepts `CCSCharger` — works with any implementation |
| **Compile-Time** | Java verifies the adapter implements all interface methods at compile time |
| **Encapsulation** | Private field with constructor injection; no setter for the adaptee |

### Target Interface

```java
import java.text.SimpleDateFormat;
import java.util.Date;

// ── Target Interface ──
/**
 * The CCS (Combined Charging System) standard interface.
 * Any charger that implements this interface can communicate
 * with a CCS-compatible charging station.
 */
interface CCSCharger {
    /**
     * Deliver DC power at the requested kilowatt level.
     * @param powerKw Charging power in kilowatts (1 to 1000).
     * @return A human-readable charging confirmation string.
     * @throws IllegalArgumentException if powerKw is out of valid range.
     */
    String chargeDC(int powerKw);
}
```

### Adaptee

```java
// ── Adaptee ──
/**
 * Tesla's proprietary Supercharger hardware.
 * Uses a non-standard interface that expects voltage and amperage
 * separately, rather than the CCS convention of specifying power in kilowatts.
 */
class TeslaSupercharger {
    /**
     * Initiate a charge using Tesla's native protocol.
     * @param voltage Charging voltage in volts (typically 400 or 800).
     * @param amperage Charging current in amperes.
     * @return A confirmation from the Tesla charging hardware.
     */
    public String sendCharge(int voltage, int amperage) {
        int powerKw = (voltage * amperage) / 1000;
        return "Tesla Supercharger delivering " + powerKw
            + " kW at " + voltage + "V / " + amperage + "A via Tesla connector";
    }
}
```

### Adapter

```java
// ── Adapter ──
/**
 * Adapts a TeslaSupercharger to the CCSCharger interface.
 * Tesla vehicles can now be charged at any CCS-compatible station.
 *
 * Translation logic:
 * - Power < 350 kW uses 400V nominal (standard CCS).
 * - Power >= 350 kW uses 800V nominal (ultra-fast CCS).
 * - Amperage is derived from I = P / V, rounded up for safety.
 */
class TeslaToCCSAdapter implements CCSCharger {
    private final TeslaSupercharger teslaCharger;

    /**
     * @param teslaCharger The Tesla charging hardware to wrap.
     */
    public TeslaToCCSAdapter(TeslaSupercharger teslaCharger) {
        this.teslaCharger = teslaCharger;
    }

    @Override
    public String chargeDC(int powerKw) {
        if (powerKw <= 0) {
            throw new IllegalArgumentException(
                "Charging power must be a positive value in kilowatts."
            );
        }
        if (powerKw > 1000) {
            throw new IllegalArgumentException(
                "Requested power " + powerKw + " kW exceeds the maximum of 1000 kW."
            );
        }

        // Select voltage tier based on power requirement.
        int voltage = (powerKw > 350) ? 800 : 400;
        // Calculate required amperage: I = P / V (rounded up).
        int amperage = (int) Math.ceil((powerKw * 1000.0) / voltage);

        System.out.println(
            "  [Adapter] CCS " + powerKw + " kW -> "
            + "Tesla " + voltage + "V / " + amperage + "A"
        );

        return teslaCharger.sendCharge(voltage, amperage);
    }
}
```

### Client

```java
// ── Client ──
/**
 * A public CCS charging station.
 * Accepts any CCSCharger implementation. The station never knows
 * whether it is talking to native CCS hardware or an adapter wrapping a Tesla.
 */
class ChargingStation {
    private final String name;
    private final String location;

    /**
     * @param name The station operator name.
     * @param location The station address or location description.
     */
    public ChargingStation(String name, String location) {
        this.name = name;
        this.location = location;
    }

    /**
     * Service an electric vehicle at this station.
     *
     * @param charger Any object that implements the CCSCharger interface.
     * @param carModel The vehicle make and model for the log.
     * @param requiredKw The power the vehicle can accept in kilowatts.
     * @return A summary string of the charging session.
     */
    public String serviceVehicle(CCSCharger charger, String carModel, int requiredKw) {
        String timestamp = new SimpleDateFormat("yyyy-MM-dd'T'HH:mm:ss.SSS'Z'")
            .format(new Date());
        System.out.println("[" + timestamp + "] " + name + " at " + location);
        System.out.println("  Vehicle: " + carModel);
        System.out.println("  Request: " + requiredKw + " kW");

        try {
            String result = charger.chargeDC(requiredKw);
            System.out.println("  Result:  " + result);
            System.out.println("  Status:  Charging session completed successfully.");
            return result;
        } catch (IllegalArgumentException e) {
            System.out.println("  ERROR:   " + e.getMessage());
            return "Charge failed for " + carModel + ": " + e.getMessage();
        }
    }
}
```

### Main / Driver

```java
// ── Driver ──
/**
 * Demonstrates the Adapter pattern with a fleet of Tesla vehicles
 * charging at a CCS-compatible station through the TeslaToCCSAdapter.
 */
public class AdapterDemo {
    public static void main(String[] args) {
        System.out.println("=".repeat(60));
        System.out.println("ADAPTER PATTERN — Java Tesla to CCS Demo");
        System.out.println("=".repeat(60));
        System.out.println();

        // Create the adaptee (Tesla's proprietary charging hardware).
        TeslaSupercharger teslaHardware = new TeslaSupercharger();

        // Wrap it in the adapter so it speaks the CCS language.
        CCSCharger adapter = new TeslaToCCSAdapter(teslaHardware);

        // Create a CCS charging station that only knows the CCS interface.
        ChargingStation station = new ChargingStation(
            "Electrify America", "Walmart Parking Lot, Newark, DE"
        );

        // Define a fleet of Tesla vehicles needing a charge.
        String[][] vehicles = {
            {"Tesla Model 3 Standard Range", "170"},
            {"Tesla Model 3 Long Range", "250"},
            {"Tesla Model Y Performance", "250"},
            {"Tesla Model S Plaid", "200"},
            {"Tesla Cybertruck", "350"},
            {"Tesla Semi", "750"},
        };

        // Service each vehicle through the adapter.
        // The station never knows it is talking to Tesla hardware.
        String[] sessionLogs = new String[vehicles.length];
        for (int i = 0; i < vehicles.length; i++) {
            String model = vehicles[i][0];
            int powerKw = Integer.parseInt(vehicles[i][1]);
            System.out.println("--- " + model + " (" + powerKw + " kW) ---");
            sessionLogs[i] = station.serviceVehicle(adapter, model, powerKw);
            System.out.println();
        }

        System.out.println("=".repeat(60));
        System.out.println("SESSION SUMMARY");
        System.out.println("=".repeat(60));
        int successCount = 0;
        for (String log : sessionLogs) {
            if (!log.startsWith("Charge failed")) {
                successCount++;
            }
        }
        System.out.println("Vehicles serviced: " + vehicles.length);
        System.out.println("Successful charges: " + successCount);
        System.out.println("All Tesla vehicles charged through the TeslaToCCSAdapter.");
    }
}
```

## Language-Specific Notes

- Java's `interface` keyword provides the strictest contract enforcement of all languages shown here. If `TeslaToCCSAdapter` omits `chargeDC`, the code will not compile. This is the strongest form of adapter safety, catching missing translation methods before any code runs.

- The `@Override` annotation on `chargeDC` is not strictly required, but it causes a compile error if the method signature does not match the interface. This prevents silent mismatches when the interface changes (e.g., `chargeDC` renamed or its parameter type changed from `int` to `long`).

- Constructor injection (`TeslaToCCSAdapter(TeslaSupercharger teslaCharger)`) is the standard Java pattern for setting the adaptee reference. No setter is provided, making the adapter immutable after construction. This mirrors a physical adapter dongle that is wired permanently to one kind of hardware.

- Java's `Math.ceil` takes a `double` and returns a `double`, requiring an explicit cast to `int`. The `1000.0` literal (instead of `1000`) triggers floating-point division, which is necessary because integer division would truncate before the ceiling operation. This is a common gotcha in Java numeric code.

- The `SimpleDateFormat` provides ISO-like timestamps for the charging log. Java 8's `java.time` package (`Instant.now()` or `ZonedDateTime`) is preferred in modern Java, but `SimpleDateFormat` keeps the example compatible with Java 7 and earlier environments.

- Java's checked vs. unchecked exception design is visible here: `IllegalArgumentException` is an unchecked exception (extends `RuntimeException`). The adapter chooses unchecked exceptions because a power value error is a programming mistake, not a recoverable condition. A checked exception would be appropriate for network failures or hardware faults in a production charging system.
