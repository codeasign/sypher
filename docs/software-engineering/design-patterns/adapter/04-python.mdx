Python implements the Adapter pattern through abstract base classes (ABC) and duck typing. Python's dynamic nature means the adapter can be written with or without explicit interface enforcement. In the car domain, a Python adapter can wrap a Tesla charging API behind a standard CCS interface, enabling fleet management software to charge any EV through a uniform protocol regardless of the underlying hardware.

## How the Implementation Works

| Aspect | Implementation |
|---|---|
| **Target** | `CCSCharger` ABC with `@abstractmethod charge_dc(power_kw: int) -> str` |
| **Adaptee** | `TeslaSupercharger` with `send_charge(voltage: int, amperage: int) -> str` |
| **Adapter** | `TeslaToCCSAdapter(CCSCharger)` wraps a `TeslaSupercharger` instance, translates CCS kw to voltage/amperage |
| **Client** | `ChargingStation` accepts any `CCSCharger` subclass — duck typed |
| **Composition** | Adapter holds the Adaptee via `self._tesla_charger` |
| **SOLID** | Open for extension (new adapters for other brands), closed for modification (station unchanged) |

### Target Interface

```python
import math
from abc import ABC, abstractmethod
from datetime import datetime
from typing import List, Tuple


# ── Target Interface ──
class CCSCharger(ABC):
    """
    Abstract base class defining the CCS (Combined Charging System) interface.
    Any charger that speaks the CCS protocol must implement charge_dc().
    """

    @abstractmethod
    def charge_dc(self, power_kw: int) -> str:
        """
        Deliver DC power at the requested kilowatt level.

        Args:
            power_kw: Charging power in kilowatts (must be positive, max 1000).

        Returns:
            A human-readable charging confirmation string.

        Raises:
            ValueError: If power_kw is out of valid range.
        """
        raise NotImplementedError("Subclasses must implement charge_dc()")
```

### Adaptee

```python
# ── Adaptee ──
class TeslaSupercharger:
    """
    Tesla's proprietary Supercharger hardware.
    Uses a non-standard interface that expects voltage and amperage separately,
    rather than the CCS convention of specifying power in kilowatts.
    """

    def send_charge(self, voltage: int, amperage: int) -> str:
        """
        Initiate a charge using Tesla's native protocol.

        Args:
            voltage: Charging voltage in volts (typically 400 or 800).
            amperage: Charging current in amperes.

        Returns:
            A confirmation from the Tesla charging hardware.
        """
        power_kw = (voltage * amperage) // 1000
        return (
            f"Tesla Supercharger delivering {power_kw} kW at "
            f"{voltage}V / {amperage}A via Tesla connector"
        )
```

### Adapter

```python
# ── Adapter ──
class TeslaToCCSAdapter(CCSCharger):
    """
    Adapts a TeslaSupercharger to the CCSCharger interface.

    Tesla vehicles can now be charged at any CCS-compatible charging station.
    The adapter translates the CCS "power in kilowatts" convention into
    the Tesla "voltage and amperage" convention.

    Translation logic:
    - Power < 350 kW uses 400V nominal (standard CCS).
    - Power >= 350 kW uses 800V nominal (ultra-fast CCS).
    - Amperage is derived from I = P / V, rounded up for safety.
    """

    def __init__(self, tesla_charger: TeslaSupercharger) -> None:
        """
        Args:
            tesla_charger: The Tesla charging hardware to wrap.
        """
        self._tesla_charger = tesla_charger

    def charge_dc(self, power_kw: int) -> str:
        """
        Convert a CCS-standard charging request to Tesla's proprietary protocol.

        Args:
            power_kw: Kilowatts requested via the CCS standard.

        Returns:
            The result from the Tesla charging hardware.

        Raises:
            ValueError: If power_kw is zero, negative, or exceeds safe limits.
        """
        if power_kw <= 0:
            raise ValueError(
                "Charging power must be a positive value in kilowatts."
            )
        if power_kw > 1000:
            raise ValueError(
                f"Requested power {power_kw} kW exceeds the maximum of 1000 kW."
            )

        # Select voltage tier based on power requirement.
        voltage = 800 if power_kw > 350 else 400
        # Calculate required amperage: I = P / V (rounded up).
        amperage = math.ceil((power_kw * 1000) / voltage)

        print(
            f"  [Adapter] CCS {power_kw} kW -> "
            f"Tesla {voltage}V / {amperage}A"
        )

        return self._tesla_charger.send_charge(voltage, amperage)
```

### Client

```python
# ── Client ──
class ChargingStation:
    """
    A public CCS charging station.
    Accepts any CCSCharger implementation — it never knows whether it is
    talking to native CCS hardware or an adapter wrapping a Tesla.
    """

    def __init__(self, name: str, location: str) -> None:
        """
        Args:
            name: The station operator name.
            location: The station address or location description.
        """
        self.name = name
        self.location = location

    def service_vehicle(
        self, charger: CCSCharger, car_model: str, required_kw: int
    ) -> str:
        """
        Service an electric vehicle at this station.

        Args:
            charger: Any object that implements the CCSCharger interface.
            car_model: The vehicle make and model for the log.
            required_kw: The power the vehicle can accept in kilowatts.

        Returns:
            A summary string of the charging session.
        """
        timestamp = datetime.now().isoformat()
        print(f"[{timestamp}] {self.name} at {self.location}")
        print(f"  Vehicle: {car_model}")
        print(f"  Request: {required_kw} kW")

        try:
            result = charger.charge_dc(required_kw)
            print(f"  Result:  {result}")
            print("  Status:  Charging session completed successfully.")
            return result
        except (ValueError, NotImplementedError) as exc:
            print(f"  ERROR:   {exc}")
            return f"Charge failed for {car_model}: {exc}"
```

### Main / Driver

```python
# ── Driver ──
def main() -> None:
    """Demonstrate the Adapter pattern with a fleet of Tesla vehicles."""
    print("=" * 60)
    print("ADAPTER PATTERN — Python Tesla to CCS Demo")
    print("=" * 60)
    print()

    # Create the adaptee (Tesla's proprietary charging hardware).
    tesla_hardware = TeslaSupercharger()

    # Wrap it in the adapter so it speaks the CCS language.
    adapter: CCSCharger = TeslaToCCSAdapter(tesla_hardware)

    # Create a CCS charging station that only knows the CCS interface.
    station = ChargingStation(
        "EVgo Fast Charging", "Target Parking Lot, San Jose, CA"
    )

    # Define a fleet of Tesla vehicles needing a charge.
    vehicles: List[Tuple[str, int]] = [
        ("Tesla Model 3 Standard Range", 170),
        ("Tesla Model 3 Long Range", 250),
        ("Tesla Model Y Performance", 250),
        ("Tesla Model S Plaid", 200),
        ("Tesla Cybertruck", 350),
        ("Tesla Semi", 750),
    ]

    # Service each vehicle through the adapter.
    # The station never knows it is talking to Tesla hardware.
    session_logs: List[str] = []
    for model, power in vehicles:
        print(f"--- {model} ({power} kW) ---")
        log = station.service_vehicle(adapter, model, power)
        session_logs.append(log)
        print()

    print("=" * 60)
    print("SESSION SUMMARY")
    print("=" * 60)
    success_count = sum(
        1 for log in session_logs if not log.startswith("Charge failed")
    )
    print(f"Vehicles serviced: {len(vehicles)}")
    print(f"Successful charges: {success_count}")
    print("All Tesla vehicles charged through the TeslaToCCSAdapter.")


if __name__ == "__main__":
    main()
```

## Language-Specific Notes

- Python's `ABC` and `@abstractmethod` from the `abc` module provide optional interface enforcement. Unlike TypeScript where the check is at compile time, Python raises `TypeError` at instantiation time if a subclass does not implement all abstract methods. This gives runtime safety without requiring a type checker.

- The convention of naming the wrapped adaptee with a leading underscore (`self._tesla_charger`) signals private access by convention. Python does not enforce true privacy, but the underscore naming convention is universally understood by Python developers.

- Duck typing is a natural fit for the Adapter pattern in Python. The `ChargingStation.service_vehicle` method accepts any object with a `charge_dc` method — it does not strictly require a `CCSCharger` subclass. This is both a strength (flexibility) and a weakness (no enforcement). Using ABCs strikes a balance.

- `math.ceil` is imported and used for amperage calculation. This rounding-up behavior is important in charging contexts: rounding down could under-deliver the promised power, while rounding up guarantees the vehicle receives at least the requested kilowatts.

- Type hints (`typing.List`, `typing.Tuple`, return type annotations) are included for documentation and for tools like mypy and Pyright. Python ignores them at runtime, but they help IDEs and static analyzers catch type mismatches.

- Python's `datetime.now().isoformat()` provides a human-readable timestamp for each charging session. The `datetime` module is standard library, requiring no external dependencies for this demo.
