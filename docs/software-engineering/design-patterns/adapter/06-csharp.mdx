C# implements the Adapter pattern through interfaces with first-class language support. C#'s interface syntax is similar to Java but with additional features like properties, default implementations, and nullable reference types. In the car domain, a C# adapter integrates Tesla's proprietary charging protocol into a .NET fleet management system, allowing operators to manage mixed-brand EV fleets through a single CCSCharger interface.

## How the Implementation Works

| Aspect | Implementation |
|---|---|
| **Target** | `ICCSCharger` interface with `string ChargeDC(int powerKw)` |
| **Adaptee** | `TeslaSupercharger` class with `string SendCharge(int voltage, int amperage)` |
| **Adapter** | `TeslaToCCSAdapter : ICCSCharger` wraps `TeslaSupercharger` via private readonly field |
| **Client** | `ChargingStation` accepts `ICCSCharger` — works with any implementation |
| **Encapsulation** | Private readonly field initialized via constructor |
| **Naming** | .NET convention prefixes interfaces with `I` (e.g., `ICCSCharger`) |

### Target Interface

```csharp
using System;
using System.Collections.Generic;

// ── Target Interface ──
/// <summary>
/// The CCS (Combined Charging System) standard interface.
/// Any charger that implements this interface can communicate
/// with a CCS-compatible charging station.
/// </summary>
public interface ICCSCharger
{
    /// <summary>
    /// Deliver DC power at the requested kilowatt level.
    /// </summary>
    /// <param name="powerKw">Charging power in kilowatts (1 to 1000).</param>
    /// <returns>A human-readable charging confirmation string.</returns>
    /// <exception cref="ArgumentException">Thrown if powerKw is out of valid range.</exception>
    string ChargeDC(int powerKw);
}
```

### Adaptee

```csharp
// ── Adaptee ──
/// <summary>
/// Tesla's proprietary Supercharger hardware.
/// Uses a non-standard interface that expects voltage and amperage
/// separately, rather than the CCS convention of specifying power in kilowatts.
/// </summary>
public class TeslaSupercharger
{
    /// <summary>
    /// Initiate a charge using Tesla's native protocol.
    /// </summary>
    /// <param name="voltage">Charging voltage in volts (typically 400 or 800).</param>
    /// <param name="amperage">Charging current in amperes.</param>
    /// <returns>A confirmation from the Tesla charging hardware.</returns>
    public string SendCharge(int voltage, int amperage)
    {
        int powerKw = (voltage * amperage) / 1000;
        return $"Tesla Supercharger delivering {powerKw} kW at {voltage}V / {amperage}A via Tesla connector";
    }
}
```

### Adapter

```csharp
// ── Adapter ──
/// <summary>
/// Adapts a TeslaSupercharger to the ICCSCharger interface.
/// Tesla vehicles can now be charged at any CCS-compatible station.
///
/// Translation logic:
/// - Power less than 350 kW uses 400V nominal (standard CCS).
/// - Power 350 kW or greater uses 800V nominal (ultra-fast CCS).
/// - Amperage is derived from I = P / V, rounded up for safety.
/// </summary>
public class TeslaToCCSAdapter : ICCSCharger
{
    private readonly TeslaSupercharger _teslaCharger;

    /// <summary>
    /// Initializes a new instance of the TeslaToCCSAdapter.
    /// </summary>
    /// <param name="teslaCharger">The Tesla charging hardware to wrap.</param>
    public TeslaToCCSAdapter(TeslaSupercharger teslaCharger)
    {
        _teslaCharger = teslaCharger ?? throw new ArgumentNullException(nameof(teslaCharger));
    }

    /// <summary>
    /// Convert a CCS-standard charging request to Tesla's proprietary protocol.
    /// </summary>
    /// <param name="powerKw">Kilowatts requested via the CCS standard.</param>
    /// <returns>The result from the Tesla charging hardware.</returns>
    /// <exception cref="ArgumentException">Thrown if powerKw is out of valid range.</exception>
    public string ChargeDC(int powerKw)
    {
        if (powerKw <= 0)
        {
            throw new ArgumentException(
                "Charging power must be a positive value in kilowatts.", nameof(powerKw));
        }
        if (powerKw > 1000)
        {
            throw new ArgumentException(
                $"Requested power {powerKw} kW exceeds the maximum of 1000 kW.", nameof(powerKw));
        }

        // Select voltage tier based on power requirement.
        int voltage = powerKw > 350 ? 800 : 400;
        // Calculate required amperage: I = P / V (rounded up).
        int amperage = (int)Math.Ceiling((powerKw * 1000.0) / voltage);

        Console.WriteLine(
            $"  [Adapter] CCS {powerKw} kW -> Tesla {voltage}V / {amperage}A");

        return _teslaCharger.SendCharge(voltage, amperage);
    }
}
```

### Client

```csharp
// ── Client ──
/// <summary>
/// A public CCS charging station.
/// Accepts any ICCSCharger implementation. The station never knows
/// whether it is talking to native CCS hardware or an adapter wrapping a Tesla.
/// </summary>
public class ChargingStation
{
    private readonly string _name;
    private readonly string _location;

    /// <summary>
    /// Initializes a new instance of the ChargingStation.
    /// </summary>
    /// <param name="name">The station operator name.</param>
    /// <param name="location">The station address or location description.</param>
    public ChargingStation(string name, string location)
    {
        _name = name;
        _location = location;
    }

    /// <summary>
    /// Service an electric vehicle at this station.
    /// </summary>
    /// <param name="charger">Any object that implements the ICCSCharger interface.</param>
    /// <param name="carModel">The vehicle make and model for the log.</param>
    /// <param name="requiredKw">The power the vehicle can accept in kilowatts.</param>
    /// <returns>A summary string of the charging session.</returns>
    public string ServiceVehicle(ICCSCharger charger, string carModel, int requiredKw)
    {
        string timestamp = DateTime.UtcNow.ToString("o");
        Console.WriteLine($"[{timestamp}] {_name} at {_location}");
        Console.WriteLine($"  Vehicle: {carModel}");
        Console.WriteLine($"  Request: {requiredKw} kW");

        try
        {
            string result = charger.ChargeDC(requiredKw);
            Console.WriteLine($"  Result:  {result}");
            Console.WriteLine("  Status:  Charging session completed successfully.");
            return result;
        }
        catch (ArgumentException ex)
        {
            Console.WriteLine($"  ERROR:   {ex.Message}");
            return $"Charge failed for {carModel}: {ex.Message}";
        }
    }
}
```

### Main / Driver

```csharp
// ── Driver ──
/// <summary>
/// Demonstrates the Adapter pattern with a fleet of Tesla vehicles
/// charging at a CCS-compatible station through the TeslaToCCSAdapter.
/// </summary>
public class AdapterDemo
{
    public static void Main()
    {
        Console.WriteLine(new string('=', 60));
        Console.WriteLine("ADAPTER PATTERN — C# Tesla to CCS Demo");
        Console.WriteLine(new string('=', 60));
        Console.WriteLine();

        // Create the adaptee (Tesla's proprietary charging hardware).
        var teslaHardware = new TeslaSupercharger();

        // Wrap it in the adapter so it speaks the CCS language.
        ICCSCharger adapter = new TeslaToCCSAdapter(teslaHardware);

        // Create a CCS charging station that only knows the CCS interface.
        var station = new ChargingStation(
            "ChargePoint", "Buc-ee's, Daytona Beach, FL");

        // Define a fleet of Tesla vehicles needing a charge.
        var vehicles = new List<(string Model, int PowerKw)>
        {
            ("Tesla Model 3 Standard Range", 170),
            ("Tesla Model 3 Long Range", 250),
            ("Tesla Model Y Performance", 250),
            ("Tesla Model S Plaid", 200),
            ("Tesla Cybertruck", 350),
            ("Tesla Semi", 750),
        };

        // Service each vehicle through the adapter.
        // The station never knows it is talking to Tesla hardware.
        var sessionLogs = new List<string>();
        foreach (var (model, powerKw) in vehicles)
        {
            Console.WriteLine($"--- {model} ({powerKw} kW) ---");
            string log = station.ServiceVehicle(adapter, model, powerKw);
            sessionLogs.Add(log);
            Console.WriteLine();
        }

        Console.WriteLine(new string('=', 60));
        Console.WriteLine("SESSION SUMMARY");
        Console.WriteLine(new string('=', 60));
        int successCount = sessionLogs.FindAll(
            log => !log.StartsWith("Charge failed")).Count;
        Console.WriteLine($"Vehicles serviced: {vehicles.Count}");
        Console.WriteLine($"Successful charges: {successCount}");
        Console.WriteLine("All Tesla vehicles charged through the TeslaToCCSAdapter.");
    }
}
```

## Language-Specific Notes

- C# uses `ICCSCharger` (the `I` prefix convention) for the interface name. This is a .NET naming convention that visually distinguishes interfaces from classes. The compiler enforces the interface contract identically regardless of the naming convention.

- The `readonly` keyword on the private field (`private readonly TeslaSupercharger _teslaCharger`) ensures the field can only be assigned during construction. This immutability prevents the adapter from swapping its wrapped adaptee at runtime, matching the one-to-one wiring of a physical adapter.

- The null check in the constructor (`teslaCharger ?? throw new ArgumentNullException(...)`) uses C#'s null-coalescing operator. This is a defensive pattern that prevents the adapter from being constructed with a null adaptee reference, which would cause a `NullReferenceException` at the first charge call.

- C#'s `Math.Ceiling` returns a `double`, requiring an explicit cast to `int`. The `1000.0` (double literal) ensures floating-point division. If `1000` (int literal) were used, integer division would truncate before the ceiling operation, producing incorrect amperage values.

- String interpolation with `$"..."` makes the logging code more readable than concatenation. C# string interpolation is a compile-time feature that converts the expression into a `string.Format` call, with no runtime overhead compared to concatenation.

- Tuples with named elements (`List<(string Model, int PowerKw)>`) provide lightweight data grouping without defining a separate class. The deconstruction syntax `foreach (var (model, powerKw) in vehicles)` extracts the tuple elements inline, keeping the driver code concise and readable.
