TypeScript adds compile-time interface enforcement to the Adapter pattern, making the contract between Client and Target explicit. In the car domain, TypeScript's type system guarantees that a TeslaToCCSAdapter fully implements the CCSCharger interface before any code runs. This catches missing methods and type mismatches at compile time — critical for a charging system where a mistranslated parameter could cause electrical faults.

## How the Implementation Works

| Aspect | Implementation |
|---|---|
| **Target** | `CCSCharger` interface with `chargeDC(powerKw: number): string` |
| **Adaptee** | `TeslaSupercharger` class with `sendCharge(voltage: number, amperage: number): string` |
| **Adapter** | `TeslaToCCSAdapter` class `implements CCSCharger`, wraps `TeslaSupercharger` via private field |
| **Client** | `ChargingStation` class typed to accept `CCSCharger` — any implementation works |
| **Type Safety** | TypeScript enforces all interface methods at compile time, preventing partial adapter implementations |
| **Readability** | Explicit type annotations serve as living documentation for the translation logic |

### Target Interface

```typescript
// ── Target Interface ──
/**
 * The CCS (Combined Charging System) standard interface.
 * Any charger that speaks the CCS protocol implements this interface,
 * allowing vehicles to charge at any CCS-compatible station.
 */
interface CCSCharger {
  /**
   * Deliver DC power at the requested kilowatt level.
   * @param powerKw - The charging power in kilowatts (1 to 1000).
   * @returns A human-readable charging confirmation string.
   */
  chargeDC(powerKw: number): string;
}
```

### Adaptee

```typescript
// ── Adaptee ──
/**
 * Tesla's proprietary Supercharger hardware.
 * Uses a non-standard interface that expects voltage and amperage separately,
 * rather than the CCS convention of specifying power in kilowatts.
 */
class TeslaSupercharger {
  /**
   * Initiate a charge using Tesla's native protocol.
   * @param voltage - Charging voltage in volts (typically 400 or 800).
   * @param amperage - Charging current in amperes.
   * @returns A confirmation from the Tesla charging hardware.
   */
  sendCharge(voltage: number, amperage: number): string {
    const powerKw: number = (voltage * amperage) / 1000;
    return (
      `Tesla Supercharger delivering ${powerKw} kW at ` +
      `${voltage}V / ${amperage}A via Tesla connector`
    );
  }
}
```

### Adapter

```typescript
// ── Adapter ──
/**
 * Adapts a TeslaSupercharger to the CCSCharger interface.
 *
 * Tesla vehicles can now be charged at any CCS-compatible charging station.
 * The adapter translates the CCS "power in kilowatts" convention into
 * the Tesla "voltage and amperage" convention.
 */
class TeslaToCCSAdapter implements CCSCharger {
  private readonly teslaCharger: TeslaSupercharger;

  /**
   * @param teslaCharger - The Tesla charging hardware to wrap.
   */
  constructor(teslaCharger: TeslaSupercharger) {
    this.teslaCharger = teslaCharger;
  }

  /**
   * Convert a CCS-standard charging request to Tesla's proprietary protocol.
   *
   * Translation logic:
   * - Power under 350 kW uses 400V nominal (standard CCS).
   * - Power 350 kW or above uses 800V nominal (ultra-fast CCS).
   * - Amperage is derived from Power = Voltage * Current.
   *
   * @param powerKw - Kilowatts requested via the CCS standard.
   * @returns The result from the Tesla charging hardware.
   * @throws Error if powerKw is zero, negative, or exceeds safe limits.
   */
  chargeDC(powerKw: number): string {
    if (powerKw <= 0) {
      throw new Error("Charging power must be a positive value in kilowatts.");
    }
    if (powerKw > 1000) {
      throw new Error(
        `Requested power ${powerKw} kW exceeds the maximum of 1000 kW.`
      );
    }

    // Select voltage tier based on power requirement.
    const voltage: number = powerKw > 350 ? 800 : 400;
    // Calculate required amperage: I = P / V (rounded up for safety).
    const amperage: number = Math.ceil((powerKw * 1000) / voltage);

    console.log(
      `  [Adapter] CCS ${powerKw} kW -> Tesla ${voltage}V / ${amperage}A`
    );

    return this.teslaCharger.sendCharge(voltage, amperage);
  }
}
```

### Client

```typescript
// ── Client ──
/**
 * A public CCS charging station.
 * Accepts any CCSCharger implementation — it never knows whether it is
 * talking to native CCS hardware or an adapter wrapping a Tesla.
 */
class ChargingStation {
  private readonly name: string;
  private readonly location: string;

  /**
   * @param name - The station operator name.
   * @param location - The station address or location description.
   */
  constructor(name: string, location: string) {
    this.name = name;
    this.location = location;
  }

  /**
   * Service an electric vehicle at this station.
   *
   * @param charger - Any object that implements the CCSCharger interface.
   * @param carModel - The vehicle make and model for the log.
   * @param requiredKw - The power the vehicle can accept in kilowatts.
   * @returns A summary string of the charging session.
   */
  serviceVehicle(
    charger: CCSCharger,
    carModel: string,
    requiredKw: number
  ): string {
    const timestamp: string = new Date().toISOString();
    console.log(`[${timestamp}] ${this.name} at ${this.location}`);
    console.log(`  Vehicle: ${carModel}`);
    console.log(`  Request: ${requiredKw} kW`);

    try {
      const result: string = charger.chargeDC(requiredKw);
      console.log(`  Result:  ${result}`);
      console.log(`  Status:  Charging session completed successfully.`);
      return result;
    } catch (error: unknown) {
      const message: string =
        error instanceof Error ? error.message : String(error);
      console.error(`  ERROR:   ${message}`);
      return `Charge failed for ${carModel}: ${message}`;
    }
  }
}
```

### Main / Driver

```typescript
// ── Driver ──
function main(): void {
  console.log("=".repeat(60));
  console.log("ADAPTER PATTERN — TypeScript Tesla to CCS Demo");
  console.log("=".repeat(60));
  console.log("");

  const teslaHardware: TeslaSupercharger = new TeslaSupercharger();

  const adapter: CCSCharger = new TeslaToCCSAdapter(teslaHardware);

  const station: ChargingStation = new ChargingStation(
    "ChargePoint",
    "I-95 Rest Area, Mile 87, Delaware"
  );

  interface VehicleSpec {
    model: string;
    powerKw: number;
  }

  const vehicles: VehicleSpec[] = [
    { model: "Tesla Model 3 Standard Range", powerKw: 170 },
    { model: "Tesla Model 3 Long Range", powerKw: 250 },
    { model: "Tesla Model Y Performance", powerKw: 250 },
    { model: "Tesla Model S Plaid", powerKw: 200 },
    { model: "Tesla Cybertruck", powerKw: 350 },
    { model: "Tesla Semi", powerKw: 750 },
  ];

  const sessionLogs: string[] = [];
  for (const vehicle of vehicles) {
    console.log(`--- ${vehicle.model} (${vehicle.powerKw} kW) ---`);
    const log: string = station.serviceVehicle(
      adapter,
      vehicle.model,
      vehicle.powerKw
    );
    sessionLogs.push(log);
    console.log("");
  }

  console.log("=".repeat(60));
  console.log("SESSION SUMMARY");
  console.log("=".repeat(60));
  console.log(`Station: ${station["name"]} at ${station["location"]}`);
  console.log(`Vehicles serviced: ${vehicles.length}`);
  const successCount: number = sessionLogs.filter(
    (l) => !l.startsWith("Charge failed")
  ).length;
  console.log(`Successful charges: ${successCount}`);
  console.log(`All ${successCount} Tesla vehicles charged through the adapter.`);
}

main();
```

## Language-Specific Notes

- TypeScript's `interface` keyword provides compile-time contract enforcement. If `TeslaToCCSAdapter` omits `chargeDC` or declares it with the wrong signature, the TypeScript compiler produces an error before the code ever runs. This catches adapter implementation bugs at development time rather than at the charging station.

- The `readonly` modifier on private fields (`private readonly teslaCharger`) prevents accidental reassignment of the adaptee reference after construction. This is a defensive coding practice that mirrors the one-directional wiring of a physical adapter dongle.

- Union types and type narrowing (`error instanceof Error`) in the catch block ensure type-safe error handling without relying on the `any` type. The adapter pattern benefits from this because translation errors from different adaptees may produce different error types, all of which must be safely handled.

- The `VehicleSpec` inline interface in the driver demonstrates TypeScript's structural type system: the object literal is type-checked against the interface without explicit class instantiation, keeping the demo code concise.

- TypeScript's access modifiers (`private`) enforce encapsulation at compile time. The `teslaCharger` field on the adapter cannot be accessed from the client or any other class, ensuring the adaptee is only interacted with through the adapter's translation logic.

- Optional chaining and nullish coalescing are not needed in this implementation because the adapter and adaptee are non-nullable by construction. TypeScript's strict null checks would flag any undefined or null assignment at compile time, preventing a family of runtime errors common in JavaScript adapter implementations.
