The C# 12 / .NET 8 implementation of the State pattern models a stock market order lifecycle. Each order transitions through well-defined states — New, Submitted, PartiallyFilled, Filled, Cancelled, and Rejected — with the `Order` context delegating state-specific behavior to the current `IOrderState` implementation. Primary constructors reduce boilerplate, and collection expressions provide concise list initialization for the allowed operations.

## How the Implementation Works

### Class Diagram (Pseudocode)

```text
interface IOrderState {
    + HandleSubmit(Order)
    + HandleFill(Order, int, double)
    + HandleCancel(Order)
    + HandleReject(Order, string)
    + GetStatusLabel(): string
    + GetAllowedOperations(): List<string>
}

class NewState            : IOrderState { /* Submit -> SubmittedState, Cancel -> CancelledState */ }
class SubmittedState      : IOrderState { /* Fill -> FilledState or PartiallyFilledState, Cancel -> CancelledState, Reject -> RejectedState */ }
class PartiallyFilledState : IOrderState { /* Fill -> FilledState, Cancel -> CancelledState */ }
class FilledState         : IOrderState { /* terminal -- all operations fail */ }
class CancelledState      : IOrderState { /* terminal -- all operations fail */ }
class RejectedState       : IOrderState { /* terminal -- all operations fail */ }

class Order {                      // Context
    - symbol: string
    - quantity: int
    - filledQuantity: int
    - price: double
    - state: IOrderState
    + Submit(): void               // delegates to state.HandleSubmit(this)
    + Fill(qty, price): void       // delegates to state.HandleFill(this, qty, price)
    + Cancel(): void               // delegates to state.HandleCancel(this)
    + Reject(reason): void         // delegates to state.HandleReject(this, reason)
    + GetStatus(): string          // delegates to state.GetStatusLabel()
    + SetState(IOrderState): void  // transitions to new state
    + AddFill(qty, price): void    // accumulates filled quantity
    + IsFullyFilled(): bool        // checks if filledQuantity >= quantity
}
```

### OOP Concepts Applied

1. **Abstraction** — `IOrderState` is the interface that defines the contract for all lifecycle states. The `Order` never references concrete state classes directly — it calls methods on the `IOrderState` abstraction.

2. **Encapsulation** — Each state class encapsulates its own transition rules, validation logic, and error messages. The `Order` encapsulates its mutable state reference (`_state`). No external code can change the state directly — only through `SetState()`.

3. **Inheritance** — All six concrete state classes implement the `IOrderState` interface, providing their own version of each state method. The C# compiler enforces that all interface members are implemented.

4. **Polymorphism** — The `Order` calls `_state.HandleSubmit(this)` and the CLR dispatches to the correct implementation depending on which concrete state is currently active — without any `if`/`switch` checks on status strings.

### SOLID Principles Applied

5. **SRP** — `Order` manages only order data and state delegation. Each state class handles only the behavior for one specific lifecycle phase. No class mixes responsibilities.

6. **OCP** — Adding a new state (e.g., `PendingReviewState`) requires creating one new class implementing `IOrderState`. No existing state classes need modification unless they transition to the new state.

7. **LSP** — Every concrete state is a valid substitute for `IOrderState`. The `Order` calls the same interface methods regardless of which state is active, and every state provides correct behavior for its phase.

8. **ISP** — The `IOrderState` interface defines exactly the methods a state needs: four lifecycle handlers, a label getter, and an operations query. No class implements irrelevant methods.

9. **DIP** — `Order` depends on the `IOrderState` abstraction, not on concrete state classes. States are injected via `SetState()` and are fully interchangeable without changing the `Order` class.

## Implementation

```csharp
using System;
using System.Collections.Generic;

interface IOrderState
{
    void HandleSubmit(Order order);
    void HandleFill(Order order, int quantity, double price);
    void HandleCancel(Order order);
    void HandleReject(Order order, string reason);
    string GetStatusLabel();
    List<string> GetAllowedOperations();
}
```

### NewState

```csharp
sealed class NewState : IOrderState
{
    public void HandleSubmit(Order order)
    {
        Console.WriteLine($"{order.Symbol}: New -> Submitted");
        order.SetState(new SubmittedState());
    }

    public void HandleFill(Order order, int quantity, double price)
    {
        Console.WriteLine("ERROR: Cannot fill — order must be submitted first");
    }

    public void HandleCancel(Order order)
    {
        Console.WriteLine($"{order.Symbol}: New -> Cancelled");
        order.SetState(new CancelledState());
    }

    public void HandleReject(Order order, string reason)
    {
        Console.WriteLine("ERROR: Cannot reject — order is not yet submitted");
    }

    public string GetStatusLabel() => "New";
    public List<string> GetAllowedOperations() => ["submit", "cancel"];
}
```

### SubmittedState

```csharp
sealed class SubmittedState : IOrderState
{
    public void HandleSubmit(Order order)
    {
        Console.WriteLine("ERROR: Cannot submit — order is already submitted");
    }

    public void HandleFill(Order order, int quantity, double price)
    {
        order.AddFill(quantity, price);
        if (order.IsFullyFilled())
        {
            Console.WriteLine($"{order.Symbol}: Filled {order.FilledQuantity} shares at ${price:F2}");
            order.SetState(new FilledState());
        }
        else
        {
            Console.WriteLine($"{order.Symbol}: Partially filled {order.FilledQuantity}/{order.Quantity} shares at ${price:F2}");
            order.SetState(new PartiallyFilledState());
        }
    }

    public void HandleCancel(Order order)
    {
        Console.WriteLine($"{order.Symbol}: Submitted -> Cancelled");
        order.SetState(new CancelledState());
    }

    public void HandleReject(Order order, string reason)
    {
        Console.WriteLine($"{order.Symbol}: Rejected — {reason}");
        order.SetState(new RejectedState());
    }

    public string GetStatusLabel() => "Submitted";
    public List<string> GetAllowedOperations() => ["fill", "cancel", "reject"];
}
```

### PartiallyFilledState

```csharp
sealed class PartiallyFilledState : IOrderState
{
    public void HandleSubmit(Order order)
    {
        Console.WriteLine("ERROR: Cannot submit — order is partially filled");
    }

    public void HandleFill(Order order, int quantity, double price)
    {
        order.AddFill(quantity, price);
        if (order.IsFullyFilled())
        {
            Console.WriteLine($"{order.Symbol}: Filled {order.FilledQuantity} shares at ${price:F2}");
            order.SetState(new FilledState());
        }
        else
        {
            Console.WriteLine($"{order.Symbol}: Partially filled {order.FilledQuantity}/{order.Quantity} shares at ${price:F2}");
        }
    }

    public void HandleCancel(Order order)
    {
        int remaining = order.Quantity - order.FilledQuantity;
        Console.WriteLine($"{order.Symbol}: Cancelled {remaining} remaining shares");
        order.SetState(new CancelledState());
    }

    public void HandleReject(Order order, string reason)
    {
        Console.WriteLine("ERROR: Cannot reject — order is partially filled");
    }

    public string GetStatusLabel() => "Partially Filled";
    public List<string> GetAllowedOperations() => ["fill", "cancel"];
}
```

### Order (Context)

```csharp
sealed class Order(string symbol, int quantity, double price)
{
    private int _filledQuantity;
    private IOrderState _state = new NewState();

    public string Symbol => symbol;
    public int Quantity => quantity;
    public int FilledQuantity => _filledQuantity;
    public double Price => price;

    public void Submit() => _state.HandleSubmit(this);
    public void Fill(int qty, double p) => _state.HandleFill(this, qty, p);
    public void Cancel() => _state.HandleCancel(this);
    public void Reject(string reason) => _state.HandleReject(this, reason);
    public string GetStatus() => _state.GetStatusLabel();
    public List<string> GetAllowedOperations() => _state.GetAllowedOperations();
    public void SetState(IOrderState newState) => _state = newState;
    public void AddFill(int qty, double p) => _filledQuantity += qty;
    public bool IsFullyFilled() => _filledQuantity >= quantity;
}
```

### Main / Driver

```csharp
class Program
{
    static void Main()
    {
        Console.WriteLine("--- Order 1: AAPL 100 @ $175.50 ---");
        Order order1 = new("AAPL", 100, 175.50);
        order1.Submit();
        order1.Fill(100, 175.50);
        order1.Cancel();

        Console.WriteLine("\n--- Order 2: GOOGL 50 @ $140.25 ---");
        Order order2 = new("GOOGL", 50, 140.25);
        order2.Submit();
        order2.Fill(20, 140.25);
        order2.Cancel();

        Console.WriteLine("\n--- Order 3: MSFT 75 @ $378.80 ---");
        Order order3 = new("MSFT", 75, 378.80);
        order3.Submit();
        order3.Reject("Insufficient funds");

        Console.WriteLine("\n=== Final States ===");
        Console.WriteLine($"AAPL: {order1.GetStatus()}");
        Console.WriteLine($"GOOGL: {order2.GetStatus()}");
        Console.WriteLine($"MSFT: {order3.GetStatus()}");
    }
}
```

Expected output:

### Language-Specific Notes

- **Primary constructors** — C# 12 primary constructors (`class Order(string symbol, int quantity, double price)`) capture constructor parameters as private fields automatically. The `Symbol => symbol` expression-bodied properties expose them publicly without manual field declarations, reducing boilerplate significantly.

- **Collection expressions** — C# 12 collection expressions (`["submit", "cancel"]` for non-empty lists, `[]` for empty lists) provide concise, immutable list initialization. Terminal states return `[]` for `GetAllowedOperations()`, while active states return their valid operations using the inline syntax.

- **`sealed` classes** — Each concrete state is marked `sealed`, preventing accidental inheritance that could break the state machine. This is the C# equivalent of Java's `final` keyword and ensures the hierarchy remains controlled.

- **Expression-bodied members** — C# 6+ expression-bodied members (`GetStatusLabel() => "New"`, `HandleSubmit(order) => ...`) provide ultra-concise syntax for simple methods. The terminal state classes (`FilledState`, `CancelledState`, `RejectedState`) use single-line expression-bodied members for all four lifecycle handlers.

- **`double` for financial prices** — While `decimal` is preferred for financial calculations in production C# code, this example uses `double` to stay consistent with the shared domain model. The `F2` format specifier in interpolated strings (`${price:F2}`) ensures two decimal places in output.

- **`new()` with target-typed instantiation** — The driver uses `new("AAPL", 100, 175.50)` without repeating the type name. C# 9+ target-typed `new` expressions infer the type from the variable declaration, reducing redundancy while maintaining compile-time type safety.
