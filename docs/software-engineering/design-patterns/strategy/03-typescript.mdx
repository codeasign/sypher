## Implementation

Below is a complete TypeScript implementation of the Strategy pattern using the Stock Market domain. The `TradingBot` (Context) delegates trade signal generation to a pluggable `TradingStrategy` interface. Three concrete strategies implement different trading algorithms: `MovingAverageCrossover`, `MomentumStrategy`, and `MeanReversionStrategy`. The implementation uses TypeScript strict mode with interfaces, enums, generics, and access modifiers to enforce the pattern contracts at compile time. A fourth strategy (`RsiStrategy`) demonstrates OCP compliance by adding a new algorithm without modifying existing code.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌────────────────────────────────────────┐
//   │ TradingBot (Context)                   │ ← OOP: Encapsulation
//   ├────────────────────────────────────────┤    SRP: orchestrates strategy
//   │ - strategy: TradingStrategy            │     DIP: depends on interface
//   │ + setStrategy(strategy): void          │
//   │ + runAnalysis(ticker, prices): Signal  │
//   └────────────────┬───────────────────────┘
//                    │ delegates analyze()
//                    ▼
//   ┌────────────────────────────────────────┐
//   │ <<interface>> TradingStrategy          │ ← OOP: Abstraction
//   ├────────────────────────────────────────┤    ISP: single-method contract
//   │ + analyze(prices): Signal              │
//   └────────────────┬───────────────────────┘
//                    ▲ implements
//   ┌────────────────┴───────────────────────┐
//   │                                        │
//   ┌────────────────┐ ┌────────────────┐   ┌──────────────────┐
//   │MovingAvg       │ │Momentum        │   │MeanReversion     │
//   │Crossover       │ │Strategy        │   │Strategy          │
//   ├────────────────┤ ├────────────────┤   ├──────────────────┤
//   │+ analyze()     │ │+ analyze()     │   │+ analyze()       │
//   └────────────────┘ └────────────────┘   └──────────────────┘
//       OCP: new = new class, no existing code changed
```

#### OOP Concepts Applied

1. **Abstraction** — `TradingStrategy` is a TypeScript interface defining the `analyze()` contract. All client code depends on this interface.
2. **Encapsulation** — Each strategy hides its parameters via `private readonly` fields. `TradingBot` encapsulates the strategy reference and logging logic.
3. **Polymorphism** — `runAnalysis()` calls `this.strategy.analyze(prices)`. The correct implementation runs based on the actual concrete type.

#### SOLID Principles Applied

4. **SRP** — Each class has one responsibility: `TradingBot` orchestrates, each strategy implements one algorithm.
5. **OCP** — New strategies (like `RsiStrategy`) are added as new classes implementing `TradingStrategy`. No existing code changes.
6. **LSP** — Every concrete strategy is substitutable for `TradingStrategy`. The bot works with any implementation.
7. **ISP** — The interface has a single method. Implementations are not forced to depend on methods they do not use.
8. **DIP** — `TradingBot` depends on the `TradingStrategy` interface. Concrete strategies are injected from outside.

### MovingAverageCrossover (Concrete Strategy)

```typescript
// ── Concrete Strategy 1: Moving Average Crossover ──
// OOP role: Polymorphism — implements the strategy contract
// SOLID: SRP (only crossover), LSP (substitutable), OCP (extensible)
class MovingAverageCrossover implements TradingStrategy {
  private readonly shortWindow: number;
  private readonly longWindow: number;

  constructor(shortWindow: number = 20, longWindow: number = 50) {
    if (shortWindow >= longWindow) {
      throw new Error("shortWindow must be less than longWindow");
    }
    this.shortWindow = shortWindow;
    this.longWindow = longWindow;
  }

  analyze(prices: readonly number[]): Signal {
    if (prices.length < this.longWindow) {
      return Signal.HOLD;
    }

    const shortMA = this.computeAverage(prices, this.shortWindow);
    const longMA = this.computeAverage(prices, this.longWindow);

    if (shortMA > longMA) {
      return Signal.BUY;
    } else if (shortMA < longMA) {
      return Signal.SELL;
    }
    return Signal.HOLD;
  }

  private computeAverage(prices: readonly number[], window: number): number {
    const slice = prices.slice(-window);
    const sum = slice.reduce((acc, price) => acc + price, 0);
    return sum / slice.length;
  }
}

```

### MomentumStrategy (Concrete Strategy)

```typescript
// ── Concrete Strategy 2: Momentum Strategy ──
// OOP role: Polymorphism — implements the strategy contract
// SOLID: SRP (only momentum), LSP (substitutable), OCP (extensible)
class MomentumStrategy implements TradingStrategy {
  private readonly lookbackPeriod: number;
  private readonly threshold: number;

  constructor(lookbackPeriod: number = 10, threshold: number = 0.05) {
    this.lookbackPeriod = lookbackPeriod;
    this.threshold = threshold;
  }

  analyze(prices: readonly number[]): Signal {
    if (prices.length < this.lookbackPeriod + 1) {
      return Signal.HOLD;
    }

    const currentPrice = prices[prices.length - 1];
    const pastPrice = prices[prices.length - 1 - this.lookbackPeriod];
    const momentum = (currentPrice - pastPrice) / pastPrice;

    if (momentum > this.threshold) {
      return Signal.BUY;
    } else if (momentum < -this.threshold) {
      return Signal.SELL;
    }
    return Signal.HOLD;
  }
}

```

### MeanReversionStrategy (Concrete Strategy)

```typescript
// ── Concrete Strategy 3: Mean Reversion Strategy ──
// OOP role: Polymorphism — implements the strategy contract
// SOLID: SRP (only reversion), LSP (substitutable), OCP (extensible)
class MeanReversionStrategy implements TradingStrategy {
  private readonly window: number;
  private readonly deviationThreshold: number;

  constructor(window: number = 20, deviationThreshold: number = 0.03) {
    this.window = window;
    this.deviationThreshold = deviationThreshold;
  }

  analyze(prices: readonly number[]): Signal {
    if (prices.length < this.window) {
      return Signal.HOLD;
    }

    const slice = prices.slice(-this.window);
    const mean = slice.reduce((acc, p) => acc + p, 0) / slice.length;
    const current = prices[prices.length - 1];
    const deviation = (current - mean) / mean;

    if (deviation < -this.deviationThreshold) {
      return Signal.BUY;
    } else if (deviation > this.deviationThreshold) {
      return Signal.SELL;
    }
    return Signal.HOLD;
  }
}

```

### RsiStrategy (Concrete Strategy)

```typescript
// ── Concrete Strategy 4: RSI Strategy (added without modifying existing code) ──
// OOP role: Demonstrates OCP — new strategy, no existing code changed
// SOLID: OCP (extension without modification)
class RsiStrategy implements TradingStrategy {
  private readonly period: number;
  private readonly overbought: number;
  private readonly oversold: number;

  constructor(period: number = 14, overbought: number = 70, oversold: number = 30) {
    this.period = period;
    this.overbought = overbought;
    this.oversold = oversold;
  }

  analyze(prices: readonly number[]): Signal {
    if (prices.length < this.period + 1) {
      return Signal.HOLD;
    }

    const gains: number[] = [];
    const losses: number[] = [];

    const startIdx = prices.length - this.period;
    for (let i = startIdx; i < prices.length; i++) {
      const change = prices[i] - prices[i - 1];
      if (change >= 0) {
        gains.push(change);
        losses.push(0);
      } else {
        gains.push(0);
        losses.push(Math.abs(change));
      }
    }

    const avgGain = gains.reduce((a, b) => a + b, 0) / this.period;
    const avgLoss = losses.reduce((a, b) => a + b, 0) / this.period;

    if (avgLoss === 0) {
      return Signal.BUY;
    }

    const rs = avgGain / avgLoss;
    const rsi = 100 - 100 / (1 + rs);

    if (rsi >= this.overbought) {
      return Signal.SELL;
    } else if (rsi <= this.oversold) {
      return Signal.BUY;
    }
    return Signal.HOLD;
  }
}

```

### TradingBot (Context)

```typescript
// ── Context: TradingBot ──
// OOP role: Encapsulation — holds and delegates to strategy
// SOLID: SRP (only orchestrates), DIP (depends on TradingStrategy interface)
class TradingBot {
  private strategy: TradingStrategy | null = null;

  setStrategy(strategy: TradingStrategy): void {
    this.strategy = strategy;
  }

  getStrategyName(): string {
    if (this.strategy === null) {
      return "None";
    }
    return this.strategy.constructor.name;
  }

  runAnalysis(ticker: string, prices: readonly number[]): Signal {
    if (this.strategy === null) {
      throw new Error("No strategy set. Call setStrategy() first.");
    }
    const signal = this.strategy.analyze(prices);
    this.logSignal(ticker, signal);
    return signal;
  }

  private logSignal(ticker: string, signal: Signal): void {
    const now = new Date();
    const timestamp = now.toISOString().slice(0, 19).replace("T", " ");
    console.log(`[${timestamp}] ${ticker} -> ${signal} (strategy: ${this.getStrategyName()})`);
  }
}

```

### Language-Specific Notes

- **Interfaces with `readonly` arrays** — The `analyze()` method accepts `readonly number[]` to signal that strategies must not mutate the input data. This is a TypeScript-specific compile-time guarantee.
- **`enum` for type-safe constants** — TypeScript's native `enum Signal { BUY, SELL, HOLD }` provides compile-time checking. Invalid signal values are caught by the compiler rather than at runtime.
- **`private readonly` fields** — TypeScript access modifiers enforce encapsulation at compile time (along with ES2022 `#` private fields if using `target: ES2022`). The `readonly` keyword prevents reassignment after construction.
- **`implements` keyword** — Each concrete strategy explicitly declares `implements TradingStrategy`. The compiler verifies that the class satisfies the interface contract, catching missing methods at compile time.
- **Satisfies the Liskov Substitution Principle** — Because TypeScript uses structural typing, any object with a compatible `analyze` signature can be used as a `TradingStrategy`, even without explicit `implements`.
