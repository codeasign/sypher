## Implementation

Below is a complete Python implementation of the Strategy pattern using the Stock Market domain. The `TradingBot` (Context) delegates trade signal generation to a pluggable `TradingStrategy` abstract base class. Three concrete strategies implement different trading algorithms: `MovingAverageCrossover`, `MomentumStrategy`, and `MeanReversionStrategy`. The implementation uses Python 3.11+ features including ABCs, dataclasses, and type hints. A fourth strategy (`RsiStrategy`) demonstrates OCP compliance by adding a new algorithm without modifying existing code.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌────────────────────────────────────────────┐
//   │ TradingBot (Context)                       │ ← OOP: Encapsulation
//   ├────────────────────────────────────────────┤    SRP: orchestrates strategy
//   │ - _strategy: TradingStrategy               │     DIP: depends on ABC
//   │ + set_strategy(strategy): None             │
//   │ + run_analysis(ticker, prices) -> Signal   │
//   └────────────────┬───────────────────────────┘
//                    │ delegate analyze()
//                    ▼
//   ┌────────────────────────────────────────────┐
//   │ <<ABC>> TradingStrategy                    │ ← OOP: Abstraction
//   ├────────────────────────────────────────────┤    ISP: single abstract method
//   │ + analyze(prices) -> Signal  [@abstract]   │
//   └────────────────┬───────────────────────────┘
//                    ▲ inherits
//   ┌────────────────┴───────────────────────────┐
//   │                                            │
//   ┌────────────────┐ ┌────────────────┐       ┌──────────────────┐
//   │MovingAvg       │ │Momentum        │       │MeanReversion     │
//   │Crossover       │ │Strategy        │       │Strategy          │
//   ├────────────────┤ ├────────────────┤       ├──────────────────┤
//   │+ analyze()     │ │+ analyze()     │       │+ analyze()       │
//   └────────────────┘ └────────────────┘       └──────────────────┘
//       OCP: new = dataclass subclass, no existing code changed
```

#### OOP Concepts Applied

1. **Abstraction** — `TradingStrategy` is an ABC (Abstract Base Class) with `@abstractmethod`. All strategies adhere to this contract.
2. **Encapsulation** — Each strategy hides its parameters behind its `analyze()` method. `TradingBot` hides the strategy reference and logging logic.
3. **Polymorphism** — `run_analysis()` calls `self._strategy.analyze(prices)`. The correct implementation runs based on the concrete type.

#### SOLID Principles Applied

4. **SRP** — Each class has one responsibility. `TradingBot` orchestrates. Each strategy implements exactly one algorithm.
5. **OCP** — Adding `RsiStrategy` or any new strategy requires only a new dataclass subclass. No existing code is modified.
6. **LSP** — Any concrete strategy can substitute `TradingStrategy`. The bot calls `analyze()` on any strategy and gets a valid `Signal`.
7. **ISP** — The ABC has a single abstract method. No class implements methods it does not need.
8. **DIP** — `TradingBot` depends on the abstract `TradingStrategy`. Concrete strategies are injected.

        """Analyze price data and return a trading signal."""

### MovingAverageCrossover (Concrete Strategy)

```python
# ── Concrete Strategy 1: Moving Average Crossover ──
# OOP role: Polymorphism — implements the algorithm
# SOLID: SRP (only crossover), LSP (substitutable), OCP (extensible)
@dataclass
class MovingAverageCrossover(TradingStrategy):
    short_window: int = 20
    long_window: int = 50

    def __post_init__(self) -> None:
        if self.short_window >= self.long_window:
            raise ValueError("short_window must be less than long_window")

    def analyze(self, prices: list[float]) -> Signal:
        if len(prices) < self.long_window:
            return Signal.HOLD

        short_ma = self._compute_average(prices, self.short_window)
        long_ma = self._compute_average(prices, self.long_window)

        if short_ma > long_ma:
            return Signal.BUY
        elif short_ma < long_ma:
            return Signal.SELL
        return Signal.HOLD

    @staticmethod
    def _compute_average(prices: list[float], window: int) -> float:
        return sum(prices[-window:]) / window


```

### MomentumStrategy (Concrete Strategy)

```python
# ── Concrete Strategy 2: Momentum Strategy ──
# OOP role: Polymorphism — implements the algorithm
# SOLID: SRP (only momentum), LSP (substitutable), OCP (extensible)
@dataclass
class MomentumStrategy(TradingStrategy):
    lookback_period: int = 10
    threshold: float = 0.05

    def analyze(self, prices: list[float]) -> Signal:
        if len(prices) < self.lookback_period + 1:
            return Signal.HOLD

        current_price = prices[-1]
        past_price = prices[-(self.lookback_period + 1)]
        momentum = (current_price - past_price) / past_price

        if momentum > self.threshold:
            return Signal.BUY
        elif momentum < -self.threshold:
            return Signal.SELL
        return Signal.HOLD


```

### MeanReversionStrategy (Concrete Strategy)

```python
# ── Concrete Strategy 3: Mean Reversion Strategy ──
# OOP role: Polymorphism — implements the algorithm
# SOLID: SRP (only reversion), LSP (substitutable), OCP (extensible)
@dataclass
class MeanReversionStrategy(TradingStrategy):
    window: int = 20
    deviation_threshold: float = 0.03

    def analyze(self, prices: list[float]) -> Signal:
        if len(prices) < self.window:
            return Signal.HOLD

        recent = prices[-self.window:]
        mean = sum(recent) / self.window
        current = prices[-1]
        deviation = (current - mean) / mean

        if deviation < -self.deviation_threshold:
            return Signal.BUY
        elif deviation > self.deviation_threshold:
            return Signal.SELL
        return Signal.HOLD


```

### RsiStrategy (Concrete Strategy)

```python
# ── Concrete Strategy 4: RSI Strategy (added without modifying existing code) ──
# OOP role: OCP extension — new strategy, no existing code changed
# SOLID: OCP (open for extension, closed for modification)
@dataclass
class RsiStrategy(TradingStrategy):
    period: int = 14
    overbought: int = 70
    oversold: int = 30

    def analyze(self, prices: list[float]) -> Signal:
        if len(prices) < self.period + 1:
            return Signal.HOLD

        gains: list[float] = []
        losses: list[float] = []

        start_idx = len(prices) - self.period
        for i in range(start_idx, len(prices)):
            change = prices[i] - prices[i - 1]
            if change >= 0:
                gains.append(change)
                losses.append(0.0)
            else:
                gains.append(0.0)
                losses.append(abs(change))

        avg_gain = sum(gains) / self.period
        avg_loss = sum(losses) / self.period

        if avg_loss == 0:
            return Signal.BUY

        rs = avg_gain / avg_loss
        rsi = 100.0 - (100.0 / (1.0 + rs))

        if rsi >= self.overbought:
            return Signal.SELL
        elif rsi <= self.oversold:
            return Signal.BUY
        return Signal.HOLD


```

### TradingBot (Context)

```python
# ── Context: TradingBot ──
# OOP role: Encapsulation — holds strategy reference and delegates
# SOLID: SRP (only orchestrates), DIP (depends on abstraction)
class TradingBot:
    def __init__(self) -> None:
        self._strategy: TradingStrategy | None = None

    def set_strategy(self, strategy: TradingStrategy) -> None:
        self._strategy = strategy

    @property
    def strategy_name(self) -> str:
        if self._strategy is None:
            return "None"
        return type(self._strategy).__name__

    def run_analysis(self, ticker: str, prices: list[float]) -> Signal:
        if self._strategy is None:
            raise RuntimeError("No strategy set. Call set_strategy() first.")
        signal = self._strategy.analyze(prices)
        self._log_signal(ticker, signal)
        return signal

    def _log_signal(self, ticker: str, signal: Signal) -> None:
        timestamp = datetime.now().strftime("%Y-%m-%d %H:%M:%S")
        print(f"[{timestamp}] {ticker} -> {signal} (strategy: {self.strategy_name})")


```

### Language-Specific Notes

- **ABCs and abstract methods** — Python's `abc.ABC` and `@abstractmethod` provide the interface contract. Unlike Java or TypeScript, Python cannot enforce the contract at compile time — it raises `TypeError` at instantiation if a concrete class fails to implement all abstract methods.
- **`@dataclass` for concise strategy classes** — The `@dataclass` decorator auto-generates `__init__`, `__repr__`, and `__eq__` methods from class annotations. This keeps strategy definitions concise while maintaining full type hints and default values.
- **`Enum` for type-safe signals** — Python's `Enum` provides a type-safe way to represent the fixed set of signals (`BUY`, `SELL`, `HOLD`). The `auto()` function assigns automatic integer values.
- **`Final` type qualifier** — `Final` (from `typing`) indicates that price data lists should not be reassigned. Combined with `list[float]` type hints, this provides documentation-level immutability intent (Python itself does not enforce `Final` at runtime).
- **Duck typing and structural subtyping** — Python supports both nominal subtyping (explicit `extends TradingStrategy`) and structural subtyping via `Protocol`. The ABC approach is used here for explicit intent, but a `Protocol` would also work and is more idiomatic in modern Python for simple interfaces.
