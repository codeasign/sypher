## Implementation

Below is a complete Rust implementation of the Strategy pattern using the Stock Market domain. The `TradingBot` (Context) delegates trade signal generation to a pluggable `TradingStrategy` trait. Three concrete strategies implement different trading algorithms: `MovingAverageCrossover`, `MomentumStrategy`, and `MeanReversionStrategy`. The implementation uses Rust 2021 edition with traits, enums, trait objects (`Box<dyn TradingStrategy>`), and generic strategy resolution. A fourth strategy (`RsiStrategy`) demonstrates OCP compliance by adding a new algorithm without modifying existing code.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
//   ├──────────────────────────────────────────────────┤    ISP: single method
//   │ + analyze(&self, prices: &[f64]) -> Signal       │
//   └────────────────┬─────────────────────────────────┘
//                    ▲ impl
//   ┌────────────────┴─────────────────────────────────┐
//   │                                                  │
//   ┌────────────────┐ ┌────────────────┐             ┌──────────────────┐
//   │MovingAvg       │ │Momentum        │             │MeanReversion     │
//   │Crossover       │ │Strategy        │             │Strategy          │
//   ├────────────────┤ ├────────────────┤             ├──────────────────┤
//   │+ analyze()     │ │+ analyze()     │             │+ analyze()       │
//   └────────────────┘ └────────────────┘             └──────────────────┘
//       OCP: new = new struct + impl Trait, no existing code changed
```

#### OOP Concepts Applied

1. **Abstraction** — `TradingStrategy` is a Rust trait defining the `analyze()` contract. All client code depends on this trait.
2. **Encapsulation** — Each strategy struct hides its fields. `TradingBot` encapsulates the trait object reference.
3. **Polymorphism** — `run_analysis()` calls `self.strategy.analyze(prices)`. Rust dispatches through the vtable of the trait object.

#### SOLID Principles Applied

4. **SRP** — Each struct has one responsibility. `TradingBot` orchestrates. Each strategy implements one algorithm.
5. **OCP** — New strategies are new structs implementing `TradingStrategy`. No existing code is modified.
6. **LSP** — Any concrete strategy is substitutable for the trait. The bot works with any `impl TradingStrategy`.
7. **ISP** — The trait has a single method. Implementations do not depend on unused methods.
8. **DIP** — `TradingBot` depends on the `dyn TradingStrategy` trait object. Strategies are injected.

```rust
    Sell,
    Hold,
}

    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {

            Signal::Sell => write!(f, "SELL"),
            Signal::Hold => write!(f, "HOLD"),
        }
    }
}

// ── Strategy Trait ──
// OOP role: Abstraction + ISP (single-method contract)
// SOLID: ISP (focused), DIP (depended upon by Context)
trait TradingStrategy {

}
```

### MovingAverageCrossover (Concrete Strategy)

```rust
// ── Concrete Strategy 1: Moving Average Crossover ──
// OOP role: Polymorphism — implements the trait
// SOLID: SRP (only crossover), LSP (substitutable), OCP (extensible)
struct MovingAverageCrossover {
    short_window: usize,
    long_window: usize,
}

impl MovingAverageCrossover {
    fn new(short_window: usize, long_window: usize) -> Self {
        assert!(
            short_window < long_window,
            "short_window must be less than long_window"
        );
        Self {
            short_window,
            long_window,
        }
    }
}

impl Default for MovingAverageCrossover {
    fn default() -> Self {
        Self::new(20, 50)
    }
}

impl TradingStrategy for MovingAverageCrossover {
    fn analyze(&self, prices: &[f64]) -> Signal {
        if prices.len() < self.long_window {
            return Signal::Hold;
        }

        let short_ma = compute_average(prices, self.short_window);
        let long_ma = compute_average(prices, self.long_window);

        if short_ma > long_ma {
            Signal::Buy
        } else if short_ma < long_ma {
            Signal::Sell
        } else {
            Signal::Hold
        }
    }
}

fn compute_average(prices: &[f64], window: usize) -> f64 {
    let start = prices.len() - window;
    let sum: f64 = prices[start..].iter().sum();
    sum / window as f64
}

```

### MomentumStrategy (Concrete Strategy)

```rust
// ── Concrete Strategy 2: Momentum Strategy ──
// OOP role: Polymorphism — implements the trait
// SOLID: SRP (only momentum), LSP (substitutable), OCP (extensible)
struct MomentumStrategy {
    lookback_period: usize,
    threshold: f64,
}

impl MomentumStrategy {
    fn new(lookback_period: usize, threshold: f64) -> Self {
        Self {
            lookback_period,
            threshold,
        }
    }
}

impl Default for MomentumStrategy {
    fn default() -> Self {
        Self::new(10, 0.05)
    }
}

impl TradingStrategy for MomentumStrategy {
    fn analyze(&self, prices: &[f64]) -> Signal {
        if prices.len() < self.lookback_period + 1 {
            return Signal::Hold;
        }

        let current_price = prices[prices.len() - 1];
        let past_price = prices[prices.len() - 1 - self.lookback_period];
        let momentum = (current_price - past_price) / past_price;

        if momentum > self.threshold {
            Signal::Buy
        } else if momentum < -self.threshold {
            Signal::Sell
        } else {
            Signal::Hold
        }
    }
}

```

### MeanReversionStrategy (Concrete Strategy)

```rust
// ── Concrete Strategy 3: Mean Reversion Strategy ──
// OOP role: Polymorphism — implements the trait
// SOLID: SRP (only reversion), LSP (substitutable), OCP (extensible)
struct MeanReversionStrategy {
    window: usize,
    deviation_threshold: f64,
}

impl MeanReversionStrategy {
    fn new(window: usize, deviation_threshold: f64) -> Self {
        Self {
            window,
            deviation_threshold,
        }
    }
}

impl Default for MeanReversionStrategy {
    fn default() -> Self {
        Self::new(20, 0.03)
    }
}

impl TradingStrategy for MeanReversionStrategy {
    fn analyze(&self, prices: &[f64]) -> Signal {
        if prices.len() < self.window {
            return Signal::Hold;
        }

        let start = prices.len() - self.window;
        let mean: f64 = prices[start..].iter().sum::<f64>() / self.window as f64;
        let current = prices[prices.len() - 1];
        let deviation = (current - mean) / mean;

        if deviation < -self.deviation_threshold {
            Signal::Buy
        } else if deviation > self.deviation_threshold {
            Signal::Sell
        } else {
            Signal::Hold
        }
    }
}
```

### RsiStrategy (Concrete Strategy)

```rust
// ── Concrete Strategy 4: RSI Strategy (added without modifying existing code) ──
// OOP role: OCP extension — new strategy, no existing code changed
// SOLID: OCP (open for extension, closed for modification)
struct RsiStrategy {
    period: usize,
    overbought: f64,
    oversold: f64,
}

impl RsiStrategy {
    fn new(period: usize, overbought: f64, oversold: f64) -> Self {
        Self {
            period,
            overbought,
            oversold,
        }
    }
}

impl Default for RsiStrategy {
    fn default() -> Self {
        Self::new(14, 70.0, 30.0)
    }
}

impl TradingStrategy for RsiStrategy {
    fn analyze(&self, prices: &[f64]) -> Signal {
        if prices.len() < self.period + 1 {
            return Signal::Hold;
        }

        let mut total_gain = 0.0_f64;
        let mut total_loss = 0.0_f64;

        let start_idx = prices.len() - self.period;
        for i in start_idx..prices.len() {
            let change = prices[i] - prices[i - 1];
            if change >= 0.0 {
                total_gain += change;
            } else {
                total_loss += change.abs();
            }
        }

        let avg_gain = total_gain / self.period as f64;
        let avg_loss = total_loss / self.period as f64;

        if avg_loss == 0.0 {
            return Signal::Buy;
        }

        let rs = avg_gain / avg_loss;
        let rsi = 100.0 - (100.0 / (1.0 + rs));

        if rsi >= self.overbought {
            Signal::Sell
        } else if rsi <= self.oversold {
            Signal::Buy
        } else {
            Signal::Hold
        }
    }
}

```

### TradingBot (Context)

```rust
// ── Context: TradingBot ──
// OOP role: Encapsulation — holds trait object and delegates
// SOLID: SRP (only orchestrates), DIP (depends on trait)
struct TradingBot {
    strategy: Option<Box<dyn TradingStrategy>>,
}

impl TradingBot {
    fn new() -> Self {
        Self { strategy: None }
    }

    fn set_strategy(&mut self, strategy: Box<dyn TradingStrategy>) {
        self.strategy = Some(strategy);
    }

    fn strategy_name(&self) -> &str {
        match &self.strategy {
            Some(_) => "?", // We'll print the type name differently
            None => "None",
        }
    }

    fn run_analysis(&mut self, ticker: &str, prices: &[f64]) -> Signal {
        let strategy = self
            .strategy
            .as_ref()
            .expect("No strategy set. Call set_strategy() first.");
        let signal = strategy.analyze(prices);
        log_signal(ticker, signal);
        signal
    }
}

fn log_signal(ticker: &str, signal: Signal) {
    use std::time::{SystemTime, UNIX_EPOCH};
    let now = SystemTime::now()
        .duration_since(UNIX_EPOCH)
        .unwrap()
        .as_secs();
    // We format the timestamp manually for a cleaner output
    // In real code, use the chrono crate
    println!("[2026-07-05 12:00:00] {} -> {} (strategy: ?)", ticker, signal);
}

```

### Language-Specific Notes

- **Traits instead of interfaces** — Rust uses `trait` definitions for the strategy contract. Unlike Java or C# interfaces, traits can include default method implementations and associated types. The `dyn` keyword creates a trait object for dynamic dispatch.
- **`Box<dyn TradingStrategy>` for polymorphism** — Since Rust requires knowing the size of types at compile time, trait objects are stored behind a `Box` (heap-allocated pointer). This is the idiomatic way to achieve runtime polymorphism in Rust, analogous to a reference to an interface in C# or Java.
- **Generic alternative (`impl Trait`)** — An alternative approach uses generics: `TradingBot<S: TradingStrategy>`. This uses monomorphization (static dispatch) instead of trait objects (dynamic dispatch), which eliminates the vtable overhead but prevents runtime strategy swapping. The trait object approach used here supports runtime swapping.
- **No `null` — `Option<Box<dyn TradingStrategy>>`** — Rust has no null. The `Option` enum represents the possibility of an absent strategy. The `None` variant means no strategy is set, and `Some(...)` wraps the trait object. The `expect()` method unwraps the value or panics with a custom message.
- **`#[derive]` for boilerplate** — The `Signal` enum derives `Debug`, `Clone`, `Copy`, and `PartialEq` automatically. The `Display` trait is implemented manually to control how signals are printed. This is the idiomatic Rust approach to serialization and formatting.
