The Bridge pattern in Python uses abstract base classes (ABCs) from the `abc` module to define the Engine interface and the Car abstraction. The `Engine` ABC declares abstract methods `start()`, `sound()`, and `fuel_type()`. The `Car` ABC holds a reference to an `Engine` instance and declares `drive()` as abstract. Three concrete engine classes and three concrete car classes implement the respective interfaces. Python's duck typing means that any object with the required methods can serve as an engine — the ABCs serve primarily as documentation and enforcement tools through `@abstractmethod`. The `HybridEngine` composes two engines, showcasing how the implementation side of the bridge can itself use composition.

## How the Implementation Works

Python's `abc.ABC` and `@abstractmethod` decorator enforce the Engine and Car contracts at instantiation time — you cannot create an instance of a class that has unimplemented abstract methods. The `Car.__init__` accepts an `engine` parameter and stores it as a protected `_engine` attribute. Each refined car class overrides `drive()` and calls `self._engine.start()` and `self._engine.sound()`. Python's dynamic dispatch ensures the correct engine method is called regardless of the concrete engine type. The `HybridEngine` stores two engine references and conditionally delegates based on battery level, demonstrating that the Bridge pattern supports flexible implementation strategies.

### OOP Concepts Applied

- **Abstraction** — `Engine` ABC defines the engine contract. `Car` ABC defines the car contract. Neither knows the other's concrete details.
- **Encapsulation** — Each engine class hides its internal parameters (cylinders, battery capacity, motor power) as private or protected attributes.
- **Polymorphism** — `Car.drive()` calls `self._engine.start()` polymorphically. The same method name dispatches to different implementations based on the engine type.
- **Composition** — `Car` has an `Engine` reference. `HybridEngine` composes two `Engine` references. The bridge is built on composition, not inheritance.

### SOLID Principles Applied

- **Single Responsibility Principle** — GasEngine handles combustion, Sedan handles sedan driving, HybridEngine coordinates two sub-engines. Each class has one reason to change.
- **Open/Closed Principle** — New engine types and new car types can be added via new classes. No existing class needs modification.
- **Liskov Substitution Principle** — Any `Engine` subclass works in any `Car` subclass. `Car.__init__` accepts any `Engine` instance.
- **Interface Segregation Principle** — The `Engine` interface has three focused methods. No engine is forced to implement unused operations.
- **Dependency Inversion Principle** — `Car` depends on the `Engine` ABC (abstraction), not on `GasEngine` or `ElectricEngine` (concretions).

```python
# Bridge Pattern — Python — Cars Domain
# =========================================
# Participants: Car (Abstraction), Sedan/SUV/Truck (RefinedAbstractions),
# Engine (Implementor), GasEngine/ElectricEngine/HybridEngine
# (ConcreteImplementors)
# OOP: abstraction, encapsulation, polymorphism, composition
# SOLID: SRP, OCP, LSP, ISP, DIP
# =========================================

from abc import ABC, abstractmethod
```

### Participant 1: Engine (Implementor Interface)

```python
# OOP: Abstraction — defines the contract all engines must satisfy
# SOLID: ISP — minimal three-method interface; DIP — Car depends on this
class Engine(ABC):
    @abstractmethod
    def start(self) -> None:
        pass

    @abstractmethod
    def sound(self) -> str:
        pass

    @abstractmethod
    def fuel_type(self) -> str:
        pass
```

### Participant 2: GasEngine (ConcreteImplementor)

```python
# OOP: Encapsulation — internal parameters are name-mangled private
# SOLID: SRP — implements only gas combustion logic
class GasEngine(Engine):
    def __init__(self, cylinders: int, displacement: float) -> None:
        self.__cylinders = cylinders
        self.__displacement = displacement

    def start(self) -> None:
        print("Gas engine fires: *vroom vroom*")

    def sound(self) -> str:
        return "VROOOOM" + "." * self.__cylinders

    def fuel_type(self) -> str:
        return f"Gasoline ({self.__displacement}L {self.__cylinders}-cyl)"
```

### Participant 3: ElectricEngine (ConcreteImplementor)

```python
# OOP: Encapsulation — battery and motor specs are name-mangled private
# SOLID: SRP — implements only electric motor logic
class ElectricEngine(Engine):
    def __init__(self, battery_capacity: float, motor_power: int) -> None:
        self.__battery_capacity = battery_capacity
        self.__motor_power = motor_power

    def start(self) -> None:
        print("Electric motor engages: *whirrr*")

    def sound(self) -> str:
        return "WHIIIIRRRR"

    def fuel_type(self) -> str:
        return f"Electric ({self.__battery_capacity}kWh, {self.__motor_power}kW)"
```

### Participant 4: HybridEngine (ConcreteImplementor)

```python
# OOP: Composition — wraps both a gas and electric engine
# SOLID: SRP — coordinates delegation without duplicating engine logic
class HybridEngine(Engine):
    def __init__(self, gas_engine: Engine, electric_engine: Engine,
                 battery_level: int) -> None:
        self.__gas_engine = gas_engine
        self.__electric_engine = electric_engine
        self.__battery_level = battery_level

    def start(self) -> None:
        self.__electric_engine.start()
        if self.__battery_level < 20:
            print("Battery low — gas engine kicks in.")
            self.__gas_engine.start()
        else:
            print("Running on electric power — gas engine idle.")

    def sound(self) -> str:
        if self.__battery_level >= 20:
            return self.__electric_engine.sound()
        return (self.__gas_engine.sound()
                + " then " + self.__electric_engine.sound())

    def fuel_type(self) -> str:
        return f"Hybrid (Gas + Electric, battery at {self.__battery_level}%)"
```

### Participant 5: Car (Abstraction)

```python
# OOP: Abstraction — defines the car interface and holds Engine reference
# SOLID: DIP — depends on Engine ABC, not on concrete implementations
class Car(ABC):
    def __init__(self, engine: Engine) -> None:
        self._engine = engine

    @abstractmethod
    def drive(self) -> str:
        pass

    def get_specs(self) -> str:
        return f"Car with {self._engine.fuel_type()}"
```

### Participant 6: Sedan (RefinedAbstraction)

```python
# OOP: Polymorphism — overrides drive() with sedan-specific behavior
# SOLID: SRP — handles only sedan driving mechanics
class Sedan(Car):
    def __init__(self, engine: Engine, trunk_capacity: float) -> None:
        super().__init__(engine)
        self.__trunk_capacity = trunk_capacity

    def drive(self) -> str:
        self._engine.start()
        return f"Sedan cruising smoothly on the highway. {self._engine.sound()}"

    def open_trunk(self) -> str:
        return f"Trunk opens ({self.__trunk_capacity} cu ft)."

    def get_specs(self) -> str:
        return (f"Sedan with {self._engine.fuel_type()}"
                f" | Trunk: {self.__trunk_capacity} cu ft")
```

### Participant 7: SUV (RefinedAbstraction)

```python
# OOP: Polymorphism — overrides drive() with SUV-specific behavior
# SOLID: SRP — handles only SUV driving mechanics
class SUV(Car):
    def __init__(self, engine: Engine, ground_clearance: float,
                 awd: bool) -> None:
        super().__init__(engine)
        self.__ground_clearance = ground_clearance
        self.__awd = awd

    def drive(self) -> str:
        self._engine.start()
        return f"SUV tackling off-road terrain. {self._engine.sound()}"

    def engage_4wd(self) -> str:
        if self.__awd:
            return "All-wheel drive engaged. Traction optimized."
        return "This SUV does not have AWD."

    def get_specs(self) -> str:
        drivetrain = "AWD" if self.__awd else "2WD"
        return (f"SUV with {self._engine.fuel_type()}"
                f" | Ground: {self.__ground_clearance}\" {drivetrain}")
```

### Participant 8: Truck (RefinedAbstraction)

```python
# OOP: Polymorphism — overrides drive() with truck-specific behavior
# SOLID: SRP — handles only truck driving mechanics
class Truck(Car):
    def __init__(self, engine: Engine, payload_capacity: int,
                 towing_capacity: int) -> None:
        super().__init__(engine)
        self.__payload_capacity = payload_capacity
        self.__towing_capacity = towing_capacity

    def drive(self) -> str:
        self._engine.start()
        return f"Truck hauling heavy load. {self._engine.sound()}"

    def get_specs(self) -> str:
        return (f"Truck with {self._engine.fuel_type()}"
                f" | Payload: {self.__payload_capacity}lbs"
                f" | Towing: {self.__towing_capacity}lbs")
```

### Main / Driver

```python
def main() -> None:
    print("=== Bridge Pattern — Cars Domain (Python) ===")
    print()

    # Configuration 1: V6 Gasoline Sedan
    gas_engine = GasEngine(6, 3.5)
    sedan = Sedan(gas_engine, 15.5)
    print(sedan.get_specs())
    print(sedan.drive())
    print(sedan.open_trunk())
    print()

    # Configuration 2: Electric SUV with AWD
    electric_engine = ElectricEngine(75.0, 250)
    suv = SUV(electric_engine, 8.7, True)
    print(suv.get_specs())
    print(suv.drive())
    print(suv.engage_4wd())
    print()

    # Configuration 3: Hybrid Truck for heavy towing
    hybrid_engine = HybridEngine(
        GasEngine(4, 2.0),
        ElectricEngine(12.0, 80),
        35
    )
    truck = Truck(hybrid_engine, 2000, 7500)
    print(truck.get_specs())
    print(truck.drive())
    print()

    # Configuration 4: Electric Sedan — same abstraction, different engine
    electric_sedan = Sedan(ElectricEngine(60.0, 200), 14.0)
    print("Electric Sedan:")
    print(electric_sedan.get_specs())
    print(electric_sedan.drive())
    print()

    # Configuration 5: Gas SUV — same abstraction, different engine
    gas_suv = SUV(GasEngine(8, 5.3), 10.2, True)
    print("Gas SUV:")
    print(gas_suv.get_specs())
    print(gas_suv.drive())
    print(gas_suv.engage_4wd())


if __name__ == "__main__":
    main()
```

### Language-Specific Notes

- Python's `abc.ABC` and `@abstractmethod` enforce the Engine and Car contracts at instantiation time. Attempting to create a `Car()` directly or an `Engine()` directly raises `TypeError: Can't instantiate abstract class`.
- Python uses name mangling for double-underscore attributes (`self.__cylinders` becomes `_GasEngine__cylinders`). This provides stronger encapsulation than a single underscore convention, though it is not true private access.
- The `HybridEngine` constructor type-hints both sub-engines as `Engine` (the ABC type), enabling polymorphism at the implementation level. Any object implementing `Engine` — even future ones — can be composed.
- The driver demonstrates the Bridge pattern's key property: the Sedan abstraction works with GasEngine, ElectricEngine, and HybridEngine without any changes to the Sedan class. The two hierarchies truly vary independently.
- Python's duck typing means a plain object with `start()`, `sound()`, and `fuel_type()` methods can serve as an engine without inheriting from `Engine` ABC. The ABCs document the expected protocol and catch errors at class creation time.
- The `if __name__ == "__main__"` guard ensures the driver code only runs when the file is executed directly, not when imported as a module.
