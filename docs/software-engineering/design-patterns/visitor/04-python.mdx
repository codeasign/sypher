## Implementation

Below is a complete Python 3.11+ implementation of the Visitor pattern using a stock market portfolio domain. The `Portfolio` holds a heterogeneous list of financial instruments (`Stock` and `Bond` objects), and four visitor implementations perform distinct analyses: computing total market value, calculating annual dividend income, assessing risk levels, and generating a formatted portfolio report. Python's ABCs (Abstract Base Classes) provide the interface contracts, and the double-dispatch mechanism ensures the correct visit method runs for each instrument type.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌──────────────────────────────────────────┐
//   │   class Visitor(ABC)                     │  <- OOP: Abstraction
//   ├──────────────────────────────────────────┤     ISP: segregated per type
//   │   @abstractmethod                         │
//   │   visit_stock(self, stock: Stock)         │
//   │   @abstractmethod                         │
//   │   visit_bond(self, bond: Bond)            │
//   └────────────────┬─────────────────────────┘
//                    ▲ extends
//   ┌────────────────┴─────────────────────────┐
//   │                                          │
//   ┌────────────────────┐   ┌──────────────────────────┐
//   │ ValueVisitor       │   │ DividendVisitor          │  <- OOP: Polymorphism
//   ├────────────────────┤   ├──────────────────────────┤     SRP: one concern
//   │ total_value: float │   │ total_dividend: float    │     OCP: new = new class
//   │ visit_stock()      │   │ visit_stock()            │
//   │ visit_bond()       │   │ visit_bond()             │
//   └────────────────────┘   └──────────────────────────┘
//
//   ┌──────────────────────────────────────────┐
//   │   class Element(ABC)                     │  <- OOP: Abstraction
//   ├──────────────────────────────────────────┤     ISP: single accept method
//   │   @abstractmethod                         │
//   │   accept(self, visitor: Visitor)          │
//   └────────────────┬─────────────────────────┘
//                    ▲ extends
//   ┌────────────────┴─────────────────────────┐
//   │                                          │
//   ┌────────────────────┐   ┌──────────────────────────┐
//   │     Stock          │   │      Bond                │  <- OOP: Polymorphism
//   ├────────────────────┤   ├──────────────────────────┤     LSP: both
//   │ accept(v) {        │   │ accept(v) {              │     substitutable
//   │   v.visit_stock(this)│   │   v.visit_bond(this)   │     for Element
//   │ }                  │   │ }                        │
//   └────────────────────┘   └──────────────────────────┘
```

#### OOP Concepts Applied

1. **Abstraction** -- `Visitor` and `Element` are abstract base classes (ABCs) defining the contracts. Client code depends on these abstractions, not on concrete types.
2. **Encapsulation** -- Each class hides its internal state. Visitors use private attributes (e.g., `self._total_value`) that are not accessible from outside.
3. **Inheritance** -- Concrete visitors extend `Visitor`, and concrete elements extend `Element`. Each provides its own `accept` method that dispatches to the correct visitor method.
4. **Polymorphism** -- `Portfolio.accept_all()` calls `accept()` on each element without knowing whether it is a `Stock` or `Bond`. Through double dispatch, the correct `visit_stock` or `visit_bond` method runs.

#### SOLID Principles Applied

5. **SRP** -- Each visitor class has a single responsibility: value calculation, dividend computation, risk assessment, or report generation. The `Portfolio` only manages the element collection.
6. **OCP** -- Adding a new analysis requires only a new class extending `Visitor`. No existing code needs modification.
7. **LSP** -- Any `Visitor` subclass can be passed to `Portfolio.accept_all()` without breaking the iteration. `Stock` and `Bond` are fully substitutable for `Element`.
8. **ISP** -- The `Visitor` ABC has exactly two abstract methods -- one per visited element type. No implementor is forced to provide unused methods.
9. **DIP** -- `Portfolio` depends on the `Element` abstraction. Visitors depend on the `Visitor` ABC contract. High-level analysis code depends on abstractions, not concretions.

```python
from abc import ABC, abstractmethod
from dataclasses import dataclass
```

### Element (ABC)

```python
# OOP role: Abstraction
# SOLID: ISP (minimal contract), DIP (depended upon abstraction)
class Element(ABC):
    @abstractmethod
    def accept(self, visitor: "Visitor") -> None:
        """Accept a visitor and dispatch to the correct visit method."""
        pass
```

### Visitor (ABC)

```python
# OOP role: Abstraction
# SOLID: ISP (one method per element type), OCP (extensible)
class Visitor(ABC):
    @abstractmethod
    def visit_stock(self, stock: "Stock") -> None:
        """Visit a Stock element."""
        pass

    @abstractmethod
    def visit_bond(self, bond: "Bond") -> None:
        """Visit a Bond element."""
        pass
```

### Stock (Concrete Element)

```python
# OOP role: Polymorphism (extends Element)
# SOLID: SRP (represents only stock data), LSP (substitutable for Element)
@dataclass
class Stock(Element):
    ticker: str
    company_name: str
    shares: int
    price_per_share: float
    dividend_yield: float

    def accept(self, visitor: Visitor) -> None:
        visitor.visit_stock(self)
```

### Bond (Concrete Element)

```python
# OOP role: Polymorphism (extends Element)
# SOLID: SRP (represents only bond data), LSP (substitutable for Element)
@dataclass
class Bond(Element):
    name: str
    face_value: float
    coupon_rate: float
    years_to_maturity: int

    def accept(self, visitor: Visitor) -> None:
        visitor.visit_bond(self)
```

### ValueVisitor (Concrete Visitor)

```python
# OOP role: Polymorphism (extends Visitor)
# SOLID: SRP (only calculates total value), OCP (new analysis = new class)
class ValueVisitor(Visitor):
    def __init__(self) -> None:
        self._total_value: float = 0.0

    def visit_stock(self, stock: Stock) -> None:
        self._total_value += stock.shares * stock.price_per_share

    def visit_bond(self, bond: Bond) -> None:
        self._total_value += bond.face_value

    def get_total_value(self) -> float:
        return self._total_value
```

### DividendVisitor (Concrete Visitor)

```python
# OOP role: Polymorphism (extends Visitor)
# SOLID: SRP (only calculates annual income)
class DividendVisitor(Visitor):
    def __init__(self) -> None:
        self._total_dividend: float = 0.0

    def visit_stock(self, stock: Stock) -> None:
        value = stock.shares * stock.price_per_share
        self._total_dividend += value * stock.dividend_yield

    def visit_bond(self, bond: Bond) -> None:
        self._total_dividend += bond.face_value * bond.coupon_rate

    def get_total_dividend(self) -> float:
        return self._total_dividend
```

### RiskVisitor (Concrete Visitor)

```python
# OOP role: Polymorphism (extends Visitor)
# SOLID: SRP (only assesses risk)
class RiskVisitor(Visitor):
    def __init__(self) -> None:
        self._results: list[str] = []

    def visit_stock(self, stock: Stock) -> None:
        if stock.dividend_yield > 0.04:
            self._results.append(
                f"{stock.ticker}: LOW RISK -- established dividend payer"
            )
        elif stock.dividend_yield > 0.01:
            self._results.append(
                f"{stock.ticker}: MEDIUM RISK -- moderate dividend"
            )
        else:
            self._results.append(
                f"{stock.ticker}: HIGH RISK -- low or no dividend"
            )

    def visit_bond(self, bond: Bond) -> None:
        if bond.years_to_maturity > 20:
            self._results.append(
                f"{bond.name}: MEDIUM RISK -- long maturity"
            )
        elif bond.years_to_maturity > 10:
            self._results.append(
                f"{bond.name}: LOW RISK -- medium maturity"
            )
        else:
            self._results.append(
                f"{bond.name}: LOW RISK -- short maturity"
            )

    def get_results(self) -> list[str]:
        return list(self._results)
```

### ReportVisitor (Concrete Visitor)

```python
# OOP role: Polymorphism (extends Visitor)
# SOLID: SRP (only generates formatted report)
class ReportVisitor(Visitor):
    def __init__(self) -> None:
        self._lines: list[str] = []

    def visit_stock(self, stock: Stock) -> None:
        value = stock.shares * stock.price_per_share
        income = value * stock.dividend_yield
        line = (
            f"| {stock.ticker:<6} | {stock.company_name:<20} "
            f"| {stock.shares:>5}  "
            f"| ${stock.price_per_share:>8.2f} "
            f"| ${value:>8.2f} "
            f"| ${income:>8.2f} |"
        )
        self._lines.append(line)

    def visit_bond(self, bond: Bond) -> None:
        income = bond.face_value * bond.coupon_rate
        line = (
            f"| {bond.name:<6} | {'Bond':<20} "
            f"| {bond.years_to_maturity:>2}yr  "
            f"| ${bond.face_value:>8.2f} "
            f"| ${bond.face_value:>8.2f} "
            f"| ${income:>8.2f} |"
        )
        self._lines.append(line)

    def get_report(self) -> str:
        header = "\nPortfolio Report:"
        columns = (
            "| Ticker  | Name                 | Qty/Mat "
            "| Price/Val  | Mkt Value  | Income    |"
        )
        separator = (
            "|---------+----------------------+---------"
            "+------------+------------+-----------|"
        )
        return "\n".join([header, columns, separator, *self._lines])
```

### Portfolio (ObjectStructure)

```python
# OOP role: Composition (aggregates elements)
# SOLID: SRP (manages collection only), DIP (depends on Element abstraction)
class Portfolio:
    def __init__(self) -> None:
        self._instruments: list[Element] = []

    def add(self, instrument: Element) -> None:
        self._instruments.append(instrument)

    def accept_all(self, visitor: Visitor) -> None:
        for instrument in self._instruments:
            instrument.accept(visitor)
```

### Main / Driver

```python
if __name__ == "__main__":
    portfolio = Portfolio()

    # Add stocks with realistic data
    portfolio.add(Stock("AAPL", "Apple Inc.", 100, 175.50, 0.005))
    portfolio.add(Stock("GOOGL", "Alphabet Inc.", 50, 140.25, 0.004))
    portfolio.add(Stock("MSFT", "Microsoft Corp.", 80, 380.75, 0.007))
    portfolio.add(Stock("JPM", "JPMorgan Chase", 60, 155.30, 0.048))

    # Add bonds
    portfolio.add(Bond("US10Y", 10000.0, 0.043, 10))
    portfolio.add(Bond("US30Y", 5000.0, 0.048, 30))

    # 1. ValueVisitor -- calculate total portfolio value
    value_visitor = ValueVisitor()
    portfolio.accept_all(value_visitor)
    print(f"Total Portfolio Value: ${value_visitor.get_total_value():.2f}")

    # 2. DividendVisitor -- calculate annual income
    div_visitor = DividendVisitor()
    portfolio.accept_all(div_visitor)
    print(f"Annual Dividend/Coupon Income: ${div_visitor.get_total_dividend():.2f}")

    # 3. RiskVisitor -- assess risk for each holding
    risk_visitor = RiskVisitor()
    portfolio.accept_all(risk_visitor)
    print("\nRisk Assessment:")
    for result in risk_visitor.get_results():
        print(f"  {result}")

    # 4. ReportVisitor -- generate formatted portfolio report
    report_visitor = ReportVisitor()
    portfolio.accept_all(report_visitor)
    print(report_visitor.get_report())
```

### Language-Specific Notes

- **ABCs for interface contracts** -- Python's `abc.ABC` and `@abstractmethod` decorator provide the equivalent of interfaces. Any class extending `Visitor` or `Element` must implement all abstract methods or it cannot be instantiated. This prevents missing-method bugs at instantiation time.
- **`@dataclass` for concise elements** -- The `@dataclass` decorator auto-generates `__init__`, `__repr__`, `__eq__`, and `__hash__` methods. This keeps the `Stock` and `Bond` classes concise while maintaining full functionality. The `frozen=True` option (not used here for simplicity) would make them immutable.
- **Type hints for self-documentation** -- Python 3.11+ type hints (`list[str]`, `float`, `"Visitor"` forward reference) make the code self-documenting and enable IDE autocompletion. The `"Visitor"` forward reference in `Element.accept()` is needed because `Visitor` is defined after `Element`.
- **Private attribute convention** -- Python uses the `_name` convention (single underscore) for private attributes. While not enforced by the runtime, it signals to other developers that `_total_value`, `_results`, etc. are internal implementation details.
- **`list()` copy for defensive returns** -- `RiskVisitor.get_results()` returns `list(self._results)` instead of the internal list reference, preventing callers from modifying the visitor's accumulated state.
- **`f-string` formatting** -- Python's f-strings provide concise string formatting with alignment specifiers (`:<6`, `:>8.2f`). The `ReportVisitor` uses these to create the aligned table output without external libraries like `tabulate` or `prettytable`.
