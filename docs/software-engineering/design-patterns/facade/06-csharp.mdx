## Implementation

Below is a complete C# implementation of the Facade pattern using a cars domain. The `CarFacade` class provides a unified interface for starting, driving, and stopping a modern car. Six subsystem classes — `Engine`, `Transmission`, `CoolingSystem`, `ElectricalSystem`, `FuelSystem`, and `BrakingSystem` — each encapsulate a specific vehicle domain. The facade composes all six and orchestrates the correct startup sequence, with a battery voltage check as a precondition before the starter relay engages.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌───────────────────────────────────────────┐
//   │   CarFacade (Facade)                      │  ← OOP: Composition, Encapsulation
//   ├───────────────────────────────────────────┤
//   │   - Engine engine                         │
//   │   - Transmission transmission             │
//   │   - CoolingSystem cooling                 │
//   │   - ElectricalSystem electrical           │
//   │   - FuelSystem fuelSystem                 │
//   │   - BrakingSystem brakes                  │
//   ├───────────────────────────────────────────┤
//   │   + CarFacade()                           │
//   │   + void StartEngine()                    │
//   │   + void Drive(int speed)                 │
//   │   + void StopEngine()                     │
//   └───────────────────────────────────────────┘
```

#### OOP Concepts Applied

1. **Encapsulation** — C# uses `private readonly` fields to enforce strict encapsulation. Subsystem state is accessible only through controlled methods, never directly.

2. **Composition** — `CarFacade` composes six subsystem objects. The `readonly` keyword on the fields ensures the references are assigned only once in the constructor.

3. **Abstraction** — The driver calls `StartEngine()`, `Drive()`, and `StopEngine()`. The complex sequence of battery checks, fuel priming, and spark timing is hidden behind these three method names.

#### SOLID Principles Applied

4. **SRP** — `Engine` handles cranking and RPM. `FuelSystem` manages fuel pressure. `CarFacade` orchestrates the sequence but does not implement subsystem behavior.

5. **OCP** — New subsystems (e.g., `Turbocharger`, `HybridBattery`) can be added without changing existing subsystem classes. The facade is extended by adding a new field and wiring it into the orchestration methods.

6. **ISP** — Each subsystem class exposes a minimal interface. `BrakingSystem` has only `Release()`, `Apply()`, and `IsEngaged()`. No subsystem depends on methods it does not use.

### Code

#### Subsystem: Engine

```csharp
class Engine
{
    private int rpm = 0;

    public void Crank()
    {
        this.rpm = 250;
        Console.WriteLine("Engine: cranking at 250 RPM");
    }

    public void InjectFuel()
    {
        Console.WriteLine("Engine: injecting fuel into cylinders");
    }

    public void Spark()
    {
        Console.WriteLine("Engine: firing spark plugs");
    }

    public int GetIdleRPM()
    {
        return 850;
    }

    public int GetRPM()
    {
        return this.rpm;
    }

    public void Rev(int amount)
    {
        this.rpm += amount;
        Console.WriteLine($"Engine: revving to {this.rpm} RPM");
    }

    public void Stop()
    {
        this.rpm = 0;
        Console.WriteLine("Engine: stopped");
    }
}
```

#### Facade

```csharp
class CarFacade
{
    private readonly Engine engine;
    private readonly Transmission transmission;
    private readonly CoolingSystem cooling;
    private readonly ElectricalSystem electrical;
    private readonly FuelSystem fuelSystem;
    private readonly BrakingSystem brakes;

    public CarFacade()
    {
        this.engine = new Engine();
        this.transmission = new Transmission();
        this.cooling = new CoolingSystem();
        this.electrical = new ElectricalSystem();
        this.fuelSystem = new FuelSystem();
        this.brakes = new BrakingSystem();
    }

    public void StartEngine()
    {
        Console.WriteLine("--- Starting Engine ---");
        double voltage = this.electrical.GetBatteryVoltage();
        if (voltage < 12.0)
        {
            Console.WriteLine("ERROR: Battery voltage too low.");
            return;
        }
        this.electrical.PowerECU();
        this.electrical.PowerAccessories();
        this.electrical.CloseStarterRelay();
        this.cooling.RunWaterPump();
        this.fuelSystem.Prime(55);
        this.engine.Crank();
        this.engine.InjectFuel();
        this.engine.Spark();
        Console.WriteLine($"Engine started. Idle RPM: {this.engine.GetIdleRPM()}");
        this.cooling.RunRadiatorFan();
        this.brakes.Release();
    }

    public void Drive(int speed)
    {
        Console.WriteLine($"--- Driving at {speed} RPM ---");
        this.transmission.ShiftTo("Drive");
        this.engine.Rev(speed);
        double targetTemp = 90.0;
        this.cooling.SetThermostat(targetTemp);
        Console.WriteLine($"Driving: cruising at {speed} RPM");
    }

    public void StopEngine()
    {
        Console.WriteLine("--- Stopping Engine ---");
        this.transmission.ShiftTo("Park");
        this.brakes.Apply();
        this.engine.Stop();
        this.fuelSystem.Shutoff();
        Console.WriteLine("Engine stopped. All subsystems off.");
    }
}
```

#### Main / Driver

```csharp
class FacadeDemo
{
    static void Main(string[] args)
    {
        CarFacade car = new CarFacade();
        car.StartEngine();
        Console.WriteLine();
        car.Drive(2500);
        Console.WriteLine();
        car.StopEngine();
    }
}
```

### Language-Specific Notes

1. **`readonly` fields** — The facade uses `private readonly` for all subsystem references. The `readonly` keyword guarantees that the reference is assigned only during construction, preventing accidental reassignment elsewhere in the class.

2. **String interpolation** — C# uses `$"..."` string interpolation (`Console.WriteLine($"Engine: revving to {this.rpm} RPM")`) instead of concatenation or format strings. This is the idiomatic modern C# approach.

3. **PascalCase naming** — C# conventions use PascalCase for all method names (`StartEngine`, `GetBatteryVoltage`) and camelCase for local variables and fields. This differs from Java's camelCase methods and Python's snake_case.

4. **`Console.WriteLine` empty call** — An empty `Console.WriteLine()` produces a blank line for output separation. This is the standard C# idiom for a newline separator.

5. **Top-level statements** — C# 9 and later support top-level statements, eliminating the need for the `class FacadeDemo` wrapper. The code uses the traditional explicit class for clarity and compatibility with older C# versions.

6. **`using System`** — The single `using System;` directive provides access to the `Console` class. In a larger project, subsystems would likely be in separate namespaces with their own `using` directives.
