## Implementation

Below is a complete Java implementation of the Facade pattern using a cars domain. The `CarFacade` class provides a unified interface for starting, driving, and stopping a modern car. Six subsystem classes — `Engine`, `Transmission`, `CoolingSystem`, `ElectricalSystem`, `FuelSystem`, and `BrakingSystem` — each encapsulate a specific vehicle domain. The facade composes all six and orchestrates the correct startup sequence, ensuring that preconditions (battery voltage, fuel pressure) are met before the engine cranks.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌───────────────────────────────────────────┐
//   │   CarFacade (Facade)                      │  ← OOP: Composition, Encapsulation
//   ├───────────────────────────────────────────┤
//   │   - engine: Engine                        │
//   │   - transmission: Transmission            │
//   │   - cooling: CoolingSystem                │
//   │   - electrical: ElectricalSystem          │
//   │   - fuelSystem: FuelSystem                │
//   │   - brakes: BrakingSystem                 │
//   ├───────────────────────────────────────────┤
//   │   + CarFacade()                           │
//   │   + void startEngine()                    │
//   │   + void drive(int speed)                 │
//   │   + void stopEngine()                     │
//   └───────────────────────────────────────────┘
```

#### OOP Concepts Applied

1. **Encapsulation** — Java's `private` access modifier enforces strict encapsulation. Subsystem state (`rpm`, `gear`, `voltage`) is inaccessible from outside the owning class. The facade exposes only what the client needs.

2. **Composition** — `CarFacade` holds references to six subsystem objects. The constructor creates them eagerly, ensuring the facade is fully initialized when the client receives it.

3. **Abstraction** — The client calls `startEngine()`, `drive()`, and `stopEngine()`. The complex sequence of battery checks, fuel priming, and spark timing is abstracted behind these three method names.

#### SOLID Principles Applied

4. **SRP** — `Engine` handles cranking and RPM. `FuelSystem` manages fuel pressure. `CarFacade` orchestrates the sequence but does not implement subsystem behavior.

5. **OCP** — New subsystems (e.g., `Turbocharger`, `AdBlueSystem`) can be added without changing existing subsystem classes. The facade is extended by adding a new field and integrating it into the start/drive/stop methods.

6. **ISP** — Each subsystem class exposes a minimal, cohesive interface. `BrakingSystem` has only `release()`, `apply()`, and `isEngaged()`. Clients are not forced to depend on methods they do not use.

### Code

#### Subsystem: Engine

```java
class Engine {
    private int rpm = 0;

    public void crank() {
        this.rpm = 250;
        System.out.println("Engine: cranking at 250 RPM");
    }

    public void injectFuel() {
        System.out.println("Engine: injecting fuel into cylinders");
    }

    public void spark() {
        System.out.println("Engine: firing spark plugs");
    }

    public int getIdleRPM() {
        return 850;
    }

    public int getRPM() {
        return this.rpm;
    }

    public void rev(int amount) {
        this.rpm += amount;
        System.out.println("Engine: revving to " + this.rpm + " RPM");
    }

    public void stop() {
        this.rpm = 0;
        System.out.println("Engine: stopped");
    }
}
```

#### Facade

```java
class CarFacade {
    private Engine engine;
    private Transmission transmission;
    private CoolingSystem cooling;
    private ElectricalSystem electrical;
    private FuelSystem fuelSystem;
    private BrakingSystem brakes;

    public CarFacade() {
        this.engine = new Engine();
        this.transmission = new Transmission();
        this.cooling = new CoolingSystem();
        this.electrical = new ElectricalSystem();
        this.fuelSystem = new FuelSystem();
        this.brakes = new BrakingSystem();
    }

    public void startEngine() {
        System.out.println("--- Starting Engine ---");
        double voltage = this.electrical.getBatteryVoltage();
        if (voltage < 12.0) {
            System.out.println("ERROR: Battery voltage too low.");
            return;
        }
        this.electrical.powerECU();
        this.electrical.powerAccessories();
        this.electrical.closeStarterRelay();
        this.cooling.runWaterPump();
        this.fuelSystem.prime(55);
        this.engine.crank();
        this.engine.injectFuel();
        this.engine.spark();
        System.out.println("Engine started. Idle RPM: " + this.engine.getIdleRPM());
        this.cooling.runRadiatorFan();
        this.brakes.release();
    }

    public void drive(int speed) {
        System.out.println("--- Driving at " + speed + " RPM ---");
        this.transmission.shiftTo("Drive");
        this.engine.rev(speed);
        double targetTemp = 90.0;
        this.cooling.setThermostat(targetTemp);
        System.out.println("Driving: cruising at " + speed + " RPM");
    }

    public void stopEngine() {
        System.out.println("--- Stopping Engine ---");
        this.transmission.shiftTo("Park");
        this.brakes.apply();
        this.engine.stop();
        this.fuelSystem.shutoff();
        System.out.println("Engine stopped. All subsystems off.");
    }
}
```

#### Main / Driver

```java
public class FacadeDemo {
    public static void main(String[] args) {
        CarFacade car = new CarFacade();
        car.startEngine();
        System.out.println();
        car.drive(2500);
        System.out.println();
        car.stopEngine();
    }
}
```

### Language-Specific Notes

1. **Access modifiers** — Java's `private` modifier enforces strict encapsulation on subsystem fields. The facade class is package-private (no `public` modifier) to limit its visibility. Only `FacadeDemo` is public, serving as the entry point.

2. **Eager constructor initialization** — The facade creates all six subsystem instances in its constructor. For production code, consider dependency injection (constructor parameters or a DI framework) to decouple instantiation from orchestration.

3. **Primitive types** — The code uses `int` and `double` primitives rather than wrapper types for performance. In a real automotive system, sensor values would likely come from CAN bus data structured as domain objects.

4. **System.out.println** — The implementation uses stdout for demonstration. In a production system, replace with a logging framework (SLF4J, Log4j) that routes to a diagnostic bus or telemetry system.

5. **Single main class** — Java requires a public class matching the filename. The file should be named `FacadeDemo.java` and compiled with `javac FacadeDemo.java`, then run with `java FacadeDemo`.

6. **No inheritance hierarchy** — The subsystem classes do not extend a common interface. In a larger system, define interfaces (`IEngine`, `ITransmission`) to support multiple implementations (e.g., `V8Engine`, `ElectricMotor`) and apply the Dependency Inversion Principle.
