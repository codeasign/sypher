## Implementation

Below is a complete JavaScript implementation of the Facade pattern using a cars domain. The `CarFacade` class provides a unified interface for starting, driving, and stopping a modern car. Behind the facade, six subsystem classes — `Engine`, `Transmission`, `CoolingSystem`, `ElectricalSystem`, `FuelSystem`, and `BrakingSystem` — handle the real work. The driver code calls only three high-level methods without ever touching the subsystems directly.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌───────────────────────────────────────────┐
//   │   CarFacade (Facade)                      │  ← OOP: Encapsulation, Composition
//   ├───────────────────────────────────────────┤     SRP: orchestrates subsystems
//   │   - engine: Engine                        │
//   │   - transmission: Transmission            │
//   │   - cooling: CoolingSystem                │
//   │   - electrical: ElectricalSystem          │
//   │   - fuelSystem: FuelSystem                │
//   │   - brakes: BrakingSystem                 │
//   ├───────────────────────────────────────────┤
//   │   + startEngine()                         │
//   │   + drive(speed)                          │
//   │   + stopEngine()                          │
//   └───────────────────────────────────────────┘
```

#### OOP Concepts Applied

1. **Encapsulation** — Each subsystem class hides its internal state (RPM, coolant temperature, fuel pressure) behind getter methods. The facade hides the orchestration logic from the client.

2. **Composition** — `CarFacade` composes six subsystem objects in its constructor. This is a "has-a" relationship: a car facade has an engine, a transmission, a cooling system, etc.

3. **Abstraction** — The client interacts with three methods (`startEngine`, `drive`, `stopEngine`). The complex sequence of battery checks, fuel priming, and spark timing is abstracted away.

#### SOLID Principles Applied

4. **SRP** — Each subsystem class has a single responsibility. `Engine` manages cranking and injection. `FuelSystem` manages pressure. `CarFacade` orchestrates but does not implement subsystem behavior.

5. **OCP** — New subsystems (e.g., `Turbocharger`, `HybridBattery`) can be added without modifying existing client code. The facade is extended by adding internal wiring, not by changing its public API.

6. **DIP** — The facade depends on concrete subsystem classes. In a more advanced design, subsystems could implement interfaces, and the facade would depend on abstractions. The pattern's core intent (simplifying access) is achieved either way.

### Code

#### Subsystem: Engine

```javascript
class Engine {
    constructor() {
        this._rpm = 0;
    }

    crank() {
        this._rpm = 250;
        console.log("Engine: cranking at 250 RPM");
    }

    injectFuel() {
        console.log("Engine: injecting fuel into cylinders");
    }

    spark() {
        console.log("Engine: firing spark plugs");
    }

    getIdleRPM() {
        return 850;
    }

    getRPM() {
        return this._rpm;
    }

    rev(amount) {
        this._rpm += amount;
        console.log(`Engine: revving to ${this._rpm} RPM`);
    }

    stop() {
        this._rpm = 0;
        console.log("Engine: stopped");
    }
}
```

```javascript
class BrakingSystem {

    }

        this._engaged = true;
        console.log("Brakes: parking brake applied");

        return this._engaged;

```

#### Facade

```javascript
class CarFacade {
    constructor() {
        this.engine = new Engine();
        this.transmission = new Transmission();
        this.cooling = new CoolingSystem();
        this.electrical = new ElectricalSystem();
        this.fuelSystem = new FuelSystem();
        this.brakes = new BrakingSystem();
    }

    startEngine() {
        console.log("--- Starting Engine ---");
        const voltage = this.electrical.getBatteryVoltage();
        if (voltage < 12.0) {
            console.log("ERROR: Battery voltage too low.");
            return;
        }
        this.electrical.powerECU();
        this.electrical.powerAccessories();
        this.electrical.closeStarterRelay();
        this.cooling.runWaterPump();
        this.fuelSystem.prime(55);
        this.engine.crank();
        this.engine.injectFuel();
        this.engine.spark();
        console.log(`Engine started. Idle RPM: ${this.engine.getIdleRPM()}`);
        this.cooling.runRadiatorFan();
        this.brakes.release();
    }

    drive(speed) {
        console.log(`--- Driving at ${speed} RPM ---`);
        this.transmission.shiftTo("Drive");
        this.engine.rev(speed);
        const targetTemp = 90.0;
        this.cooling.setThermostat(targetTemp);
        console.log(`Driving: cruising at ${speed} RPM`);
    }

    stopEngine() {
        console.log("--- Stopping Engine ---");
        this.transmission.shiftTo("Park");
        this.brakes.apply();
        this.engine.stop();
        this.fuelSystem.shutoff();
        console.log("Engine stopped. All subsystems off.");
    }
}
```

#### Main / Driver

```javascript
function main() {
    const car = new CarFacade();
    car.startEngine();
    console.log("");
    car.drive(2500);
    console.log("");
    car.stopEngine();
}

main();
```

### Language-Specific Notes

1. **Dynamic dispatch** — JavaScript's prototypal inheritance means the facade works with any object that has the expected methods. Subsystem classes can be swapped at runtime as long as they match the duck-typed interface.

2. **Constructor composition** — The facade's constructor eagerly creates all six subsystem instances. In a resource-constrained environment, consider lazy initialization within each getter method.

3. **No private fields** — In this ES6 class version, the `_` prefix convention marks private fields. In modern JavaScript (ES2022+), use `#` private fields for true encapsulation: `#engine = new Engine()`.

4. **Console output** — The implementation uses `console.log` for simplicity. In a production automotive system, replace with a structured logger that writes to a diagnostic bus or telemetry stream.

5. **Error handling** — The `startEngine` method includes a battery voltage check. A production facade would add try/catch blocks around each subsystem call and implement fail-safe defaults for every failure mode.

6. **Testing advantage** — Each subsystem class can be individually unit-tested. The facade can be tested by mocking the six subsystems and verifying that method calls happen in the correct order with the correct arguments.
