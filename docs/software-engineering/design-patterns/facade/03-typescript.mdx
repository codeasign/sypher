## Implementation

Below is a complete TypeScript implementation of the Facade pattern using a cars domain. TypeScript's static type system enforces the subsystem interfaces at compile time, ensuring that the `CarFacade` only composes components that satisfy the expected contracts. The facade provides three high-level methods — `startEngine`, `drive`, and `stopEngine` — while internally coordinating an `Engine`, `Transmission`, `CoolingSystem`, `ElectricalSystem`, `FuelSystem`, and `BrakingSystem`.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌───────────────────────────────────────────┐
//   │   <<interface>> IEngine                   │  ← DIP: Facade depends on abstractions
//   ├───────────────────────────────────────────┤
//   │   + crank(): void                         │
//   │   + injectFuel(): void                    │
//   │   + spark(): void                         │
//   │   + getIdleRPM(): number                  │
//   │   + getRPM(): number                      │
//   │   + rev(amount: number): void             │
//   │   + stop(): void                          │
//   └───────────────────────────────────────────┘
//
//   ┌───────────────────────────────────────────┐
//   │   CarFacade (Facade)                      │  ← OOP: Composition over inheritance
//   ├───────────────────────────────────────────┤
//   │   - engine: IEngine                       │
//   │   - transmission: ITransmission           │
//   │   - ... (5 more interfaces)               │
//   ├───────────────────────────────────────────┤
//   │   + startEngine(): void                   │
//   │   + drive(speed: number): void            │
//   │   + stopEngine(): void                    │
//   └───────────────────────────────────────────┘
```

#### OOP Concepts Applied

1. **Abstraction** — Six interfaces define pure contracts without implementation details. `CarFacade` depends only on these interfaces, not on concrete subsystem classes.

2. **Composition** — `CarFacade` is composed of six interface-typed members. The concrete implementations are injected via the constructor, supporting dependency injection.

3. **Polymorphism** — Any class that implements `IEngine` (e.g., `V8Engine`, `ElectricMotor`, `HybridEngine`) can be substituted without changing the facade.

#### SOLID Principles Applied

4. **DIP (Dependency Inversion Principle)** — The facade depends on interfaces (`IEngine`, `ITransmission`, etc.), not on concrete classes. Concrete subsystem implement the interfaces. This is the hallmark of the DIP.

5. **SRP** — Each subsystem interface defines a focused contract. `IFuelSystem` only deals with fuel pressure, `IBrakingSystem` only with brake state.

6. **OCP** — New engine types require only a new class that implements `IEngine`. The facade remains unchanged because it depends on the interface, not the concrete engine class.

7. **ISP** — Each interface is small and focused. `IEngine` has seven methods that are all relevant to engine operation. A client that only needs engine RPM does not depend on braking methods.

### Code

#### Subsystem Interfaces

```typescript
interface IEngine {
    crank(): void;
    injectFuel(): void;
    spark(): void;
    getIdleRPM(): number;
    getRPM(): number;
    rev(amount: number): void;
    stop(): void;
}

interface ITransmission {
    shiftTo(gearName: string): void;
    getCurrentGear(): string;
}

interface ICoolingSystem {
    runWaterPump(): void;
    runRadiatorFan(): void;
    getCoolantTemperature(): number;
    setThermostat(temp: number): void;
}

interface IElectricalSystem {
    getBatteryVoltage(): number;
    powerAccessories(): void;
    powerECU(): void;
    closeStarterRelay(): void;
}

interface IFuelSystem {
    prime(pressure: number): void;
    getRailPressure(): number;
    shutoff(): void;
}

interface IBrakingSystem {
    release(): void;
    apply(): void;
    isEngaged(): boolean;
}
```

#### Concrete Subsystem: Engine

```typescript
class Engine implements IEngine {
    private rpm: number = 0;

    public crank(): void {
        this.rpm = 250;
        console.log("Engine: cranking at 250 RPM");
    }

    public injectFuel(): void {
        console.log("Engine: injecting fuel into cylinders");
    }

    public spark(): void {
        console.log("Engine: firing spark plugs");
    }

    public getIdleRPM(): number {
        return 850;
    }

    public getRPM(): number {
        return this.rpm;
    }

    public rev(amount: number): void {
        this.rpm += amount;
        console.log(`Engine: revving to ${this.rpm} RPM`);
    }

    public stop(): void {
        this.rpm = 0;
        console.log("Engine: stopped");
    }
}
```

#### Facade

```typescript
class CarFacade {
    constructor(
        private engine: IEngine,
        private transmission: ITransmission,
        private cooling: ICoolingSystem,
        private electrical: IElectricalSystem,
        private fuelSystem: IFuelSystem,
        private brakes: IBrakingSystem
    ) {}

    public startEngine(): void {
        console.log("--- Starting Engine ---");
        const voltage = this.electrical.getBatteryVoltage();
        if (voltage < 12.0) {
            console.log("ERROR: Battery voltage too low.");
            return;
        }
        this.electrical.powerECU();
        this.electrical.powerAccessories();
        this.electrical.closeStarterRelay();
        this.cooling.runWaterPump();
        this.fuelSystem.prime(55);
        this.engine.crank();
        this.engine.injectFuel();
        this.engine.spark();
        console.log(`Engine started. Idle RPM: ${this.engine.getIdleRPM()}`);
        this.cooling.runRadiatorFan();
        this.brakes.release();
    }

    public drive(speed: number): void {
        console.log(`--- Driving at ${speed} RPM ---`);
        this.transmission.shiftTo("Drive");
        this.engine.rev(speed);
        const targetTemp = 90.0;
        this.cooling.setThermostat(targetTemp);
        console.log(`Driving: cruising at ${speed} RPM`);
    }

    public stopEngine(): void {
        console.log("--- Stopping Engine ---");
        this.transmission.shiftTo("Park");
        this.brakes.apply();
        this.engine.stop();
        this.fuelSystem.shutoff();
        console.log("Engine stopped. All subsystems off.");
    }
}
```

#### Main / Driver

```typescript
function main(): void {
    const car = new CarFacade(
        new Engine(),
        new Transmission(),
        new CoolingSystem(),
        new ElectricalSystem(),
        new FuelSystem(),
        new BrakingSystem()
    );
    car.startEngine();
    console.log("");
    car.drive(2500);
    console.log("");
    car.stopEngine();
}

main();
```

### Language-Specific Notes

1. **Dependency injection via constructor** — Unlike the JavaScript version that creates subsystems internally, the TypeScript facade receives them via constructor parameters. This enables unit testing with mock subsystems and supports IoC containers.

2. **Interface contracts** — Each subsystem interface defines an explicit contract. A `HybridEngine` class implementing `IEngine` can be swapped in without changing the facade, demonstrating the Open/Closed Principle.

3. **Readonly and private** — TypeScript's `private` keyword (or `#` in modern JS) enforces encapsulation at compile time. Interface members are `public` by default and only expose the minimum surface area.

4. **Union types for safety** — The `Transmission.shiftTo` method accepts a `string`. In a production system, use a union type like `type Gear = "Park" | "Drive" | "Reverse" | "Neutral"` to prevent invalid gear arguments at compile time.

5. **No runtime overhead** — TypeScript interfaces are compile-time only. The runtime behavior is identical to the JavaScript version, with zero overhead for type checking.

6. **Narrowing with type guards** — The battery voltage check in `startEngine` benefits from TypeScript's type narrowing. The `const voltage: number` ensures the comparison is type-safe.
