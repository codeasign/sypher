The Factory Method pattern defines an interface for creating an object but lets subclasses alter the type of objects that will be created. In this Banking implementation, `AccountFactory` declares the factory method `create_account()`, and concrete subclasses of the factory produce specific account types — `SavingsAccount`, `CheckingAccount`, and `LoanAccount`. Each product class implements the same `Account` interface, so callers depend only on the abstraction, not on any concrete account class.

## How the Implementation Works

### Class Diagram (Pseudocode)

```python
class Account(ABC):
    @abstractmethod
    def deposit(self, amount: float) -> None: ...
    @abstractmethod
    def withdraw(self, amount: float) -> None: ...

class SavingsAccount(Account):
    def withdraw(self, amount) -> None:
        if self.balance - amount >= self.minimum_balance:
            self.balance -= amount
        else:
            raise ValueError("Insufficient funds")

class CheckingAccount(Account):
    def withdraw(self, amount) -> None:
        if self.balance - amount >= -self.overdraft_limit:
            self.balance -= amount
        else:
            raise ValueError("Overdraft limit exceeded")

class LoanAccount(Account):
    def deposit(self, amount) -> None:    self.balance -= amount  # repayment
    def withdraw(self, amount) -> None:   self.balance += amount  # disbursement

class AccountFactory(ABC):
    @abstractmethod
    def create_account(self, owner, type, init) -> Account: ...

class ConcreteAccountFactory(AccountFactory):
    def create_account(self, owner, type, init) -> Account:
        if type == "savings":  return SavingsAccount(owner, init)
        elif type == "checking":  return CheckingAccount(owner, init)
        elif type == "loan":  return LoanAccount(owner, init)
        else:  raise ValueError(f"Unknown type: {type}")
```

### OOP Concepts Applied

| Concept | How It Is Applied |
|---|---|
| **Abstraction** | `Account` defines an abstract interface; callers never touch concrete account internals. `AccountFactory` declares the creation contract without exposing instantiation logic. |
| **Encapsulation** | Each account hides its balance and fee rules behind property methods. The factory encapsulates the `if/elif` branching that decides which concrete class to instantiate. |
| **Inheritance** | `SavingsAccount`, `CheckingAccount`, and `LoanAccount` inherit from the `Account` ABC. `ConcreteAccountFactory` inherits from `AccountFactory` and provides the method body. |
| **Polymorphism** | Client code calls `account.deposit()` without knowing the concrete type. The correct `withdraw()` logic runs based on the actual runtime type via method dispatch. |

### SOLID Principles Applied

| Principle | How It Is Applied |
|---|---|
| **SRP** | `AccountFactory` has one job: create accounts. Each account class has one job: manage a specific banking product. |
| **OCP** | Adding a new account type (e.g., `MoneyMarketAccount`) requires zero changes to existing products or client code. Existing code stays closed for modification. |
| **LSP** | Every concrete account satisfies the `Account` contract. A `LoanAccount` can be used anywhere an `Account` is expected. |
| **ISP** | `Account` exposes only the methods every account needs. No class is forced to implement a method it does not use. |
| **DIP** | Client code depends on `Account` and `AccountFactory` abstractions, not on concrete classes. High-level modules remain decoupled from low-level details. |

## Implementation

```python
"""
Factory Method Pattern — Banking Domain (Python 3.11+)

Participants:
  - Product (Account):             ABC defining the account interface
  - ConcreteProduct:               SavingsAccount, CheckingAccount, LoanAccount
  - Creator (AccountFactory):      ABC declaring the factory method
  - ConcreteCreator:               ConcreteAccountFactory, PremiumAccountFactory

OOP:  Abstraction, Encapsulation, Inheritance, Polymorphism
SOLID: SRP, OCP, LSP, ISP, DIP
"""

from abc import ABC, abstractmethod
from dataclasses import dataclass
```

### Account (Product / ABC)
```python
class Account(ABC):
    @abstractmethod
    def deposit(self, amount: float) -> None: ...
    @abstractmethod
    def withdraw(self, amount: float) -> None: ...
    @abstractmethod
    def get_balance(self) -> float: ...
    @abstractmethod
    def get_account_type(self) -> str: ...
    @abstractmethod
    def get_owner(self) -> str: ...
    @abstractmethod
    def apply_monthly_fee(self) -> None: ...
```

### SavingsAccount (Concrete Product)
```python
@dataclass
class SavingsAccount(Account):
    owner: str
    _balance: float = 0.0
    interest_rate: float = 0.015
    minimum_balance: float = 25.0
    monthly_fee: float = 5.0

    def deposit(self, amount: float) -> None:
        if amount <= 0:
            raise ValueError("Deposit must be positive.")
        self._balance += amount

    def withdraw(self, amount: float) -> None:
        if amount <= 0:
            raise ValueError("Withdrawal must be positive.")
        if self._balance - amount < self.minimum_balance:
            raise ValueError(
                f"Cannot withdraw ${amount:.2f}. Minimum ${self.minimum_balance:.2f} "
                f"required. Balance: ${self._balance:.2f}."
            )
        self._balance -= amount

    def get_balance(self) -> float:
        return self._balance

    def get_account_type(self) -> str:
        return "Savings Account"

    def get_owner(self) -> str:
        return self.owner

    def apply_monthly_fee(self) -> None:
        self._balance = max(0.0, self._balance - self.monthly_fee)

    def apply_interest(self) -> None:
        self._balance += self._balance * (self.interest_rate / 12.0)
```

### LoanAccount (Concrete Product)
```python
@dataclass
class LoanAccount(Account):
    owner: str
    _balance: float = 0.0
    interest_rate: float = 0.0699

    def deposit(self, amount: float) -> None:
        """Make a repayment reducing the outstanding principal."""
        if amount <= 0:
            raise ValueError("Repayment must be positive.")
        if amount >= self._balance:
            refund = amount - self._balance
            print(f"  Loan paid off! Refund: ${refund:.2f}")
            self._balance = 0.0
        else:
            self._balance -= amount

    def withdraw(self, amount: float) -> None:
        """Disburse additional funds, increasing principal."""
        if amount <= 0:
            raise ValueError("Disbursement must be positive.")
        self._balance += amount + (amount * self.interest_rate / 12.0)

    def get_balance(self) -> float:
        return self._balance

    def get_account_type(self) -> str:
        return "Loan Account"

    def get_owner(self) -> str:
        return self.owner

    def apply_monthly_fee(self) -> None:
        self._balance += self._balance * (self.interest_rate / 12.0)
```

### PremiumSavingsAccount (Concrete Product / OCP Extension)
```python
@dataclass
class PremiumSavingsAccount(SavingsAccount):
    interest_rate: float = 0.035
    minimum_balance: float = 1000.0
    monthly_fee: float = 0.0
    cashback_rate: float = 0.005

    def deposit(self, amount: float) -> None:
        super().deposit(amount)
        self._balance += amount * self.cashback_rate

    def get_account_type(self) -> str:
        return "Premium Savings Account"
```

### AccountFactory (Creator / ABC)
```python
class AccountFactory(ABC):
    @abstractmethod
    def create_account(
        self, owner: str, account_type: str, initial_deposit: float = 0.0
    ) -> Account: ...

    def open_account(
        self, owner: str, account_type: str, initial_deposit: float = 0.0
    ) -> Account:
        account = self.create_account(owner, account_type, initial_deposit)
        if initial_deposit > 0:
            account.deposit(initial_deposit)
        print(f"  Created {account.get_account_type()} for {owner} "
              f"(deposit: ${initial_deposit:.2f})")
        return account
```

### ConcreteAccountFactory (Concrete Creator)
```python
class ConcreteAccountFactory(AccountFactory):
    def create_account(
        self, owner: str, account_type: str, initial_deposit: float = 0.0
    ) -> Account:
        t = account_type.lower().strip()
        if t == "savings":
            return SavingsAccount(owner=owner, _balance=0.0)
        elif t == "checking":
            return CheckingAccount(owner=owner, _balance=0.0)
        elif t == "loan":
            return LoanAccount(owner=owner, _balance=initial_deposit)
        elif t == "premium":
            return PremiumSavingsAccount(owner=owner, _balance=0.0)
        else:
            raise ValueError(f"Unknown type: '{t}'")
```

### Main / Driver
```python
def main() -> None:
    factory: AccountFactory = ConcreteAccountFactory()

    print("=" * 60)
    print("  STANDARD FACTORY")
    print("=" * 60)
    alice: Account = factory.open_account("Alice", "savings", 2500.00)
    bob: Account = factory.open_account("Bob", "checking", 1000.00)
    carol: Account = factory.open_account("Carol", "loan", 50000.00)
    dave: Account = factory.open_account("Dave", "premium", 10000.00)

    print("  --- Transactions ---")
    alice.deposit(500.00)
    alice.withdraw(200.00)
    print(f"  Alice (Savings):  ${alice.get_balance():.2f}")
    bob.withdraw(1200.00)
    print(f"  Bob (Checking):   ${bob.get_balance():.2f} (overdraft)")
    bob.deposit(1500.00)
    print(f"  Bob (Checking):   ${bob.get_balance():.2f} (after deposit)")
    carol.deposit(1000.00)
    print(f"  Carol (Loan):     ${carol.get_balance():.2f} (repayment)")
    dave.deposit(5000.00)
    print(f"  Dave (Premium):   ${dave.get_balance():.2f} (cashback+)")

    print("  --- Monthly Fees ---")
    alice.apply_monthly_fee(); bob.apply_monthly_fee()
    carol.apply_monthly_fee(); dave.apply_monthly_fee()
    print(f"  Alice: ${alice.get_balance():.2f} | Bob: ${bob.get_balance():.2f}")
    print(f"  Carol: ${carol.get_balance():.2f} | Dave: ${dave.get_balance():.2f}")
    alice.apply_interest()
    print(f"  Alice after interest: ${alice.get_balance():.2f}")

    print()
    print("=" * 60)
    print("  PREMIUM FACTORY")
    print("=" * 60)
    premium: AccountFactory = PremiumAccountFactory()
    alice_p: Account = premium.open_account("Alice", "savings", 50000.00)
    bob_p: Account = premium.open_account("Bob", "checking", 25000.00)
    carol_p: Account = premium.open_account("Carol", "loan", 100000.00)
    alice_p.deposit(10000.00)
    bob_p.withdraw(26000.00)
    carol_p.apply_monthly_fee()
    print(f"  Alice (PremSavings):  ${alice_p.get_balance():.2f}")
    print(f"  Bob   (PremCheck):    ${bob_p.get_balance():.2f}")
    print(f"  Carol (PremLoan):     ${carol_p.get_balance():.2f}")

    print()
    print("=" * 60)
    print("  POLYMORPHISM")
    print("=" * 60)
    accounts: list[Account] = [
        factory.create_account("Alice", "savings", 5000.00),
        factory.create_account("Bob", "checking", 3000.00),
        factory.create_account("Carol", "loan", 25000.00),
        factory.create_account("Dave", "premium", 15000.00),
    ]
    for a in accounts:
        before = a.get_balance()
        a.deposit(1000.00)
        print(f"  {a.get_owner()}'s {a.get_account_type()}: "
              f"${before:.2f} -> ${a.get_balance():.2f}")


if __name__ == "__main__":
    main()
```

## Language-Specific Notes

- **ABCs enforce the contract.** Python's `abc.ABC` and `@abstractmethod` guarantee every concrete product implements the full `Account` interface. Forgetting `deposit()` on a subclass raises `TypeError` at instantiation time — a compile-time-like safety net.
- **Dataclasses reduce boilerplate.** `@dataclass` eliminates writing `__init__`, `__repr__`, and `__eq__` for every account class. Default values on fields (like `_balance: float = 0.0`) set initial state without a constructor body.
- **Union type syntax (`str | None`).** Python 3.10+ `str | None` replaces the older `Optional[str]` from `typing`. No `from typing import Optional` needed anywhere in the code.
- **Polymorphism is natural in Python.** Because Python is dynamically typed, any object with `deposit()`, `withdraw()`, and `get_balance()` methods satisfies the `Account` protocol — even without inheriting from the ABC. The ABC documents intent, but duck typing works as a fallback.
- **DIP through ABCs.** `AccountFactory` and `Account` are both ABCs; client code accepts `AccountFactory` and `Account` parameters, never concrete types. Swapping from `ConcreteAccountFactory` to `PremiumAccountFactory` changes only the factory instance passed in.
- **OCP in practice.** Adding a `MoneyMarketAccount` requires: (1) write the class inheriting from `Account`, (2) add one `elif` branch in the factory. No existing class is touched, and no client code changes. The premium factory demonstrates this by reusing `CheckingAccount` with different arguments.
