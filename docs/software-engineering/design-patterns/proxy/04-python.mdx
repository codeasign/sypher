This page implements the Proxy design pattern in Python 3.12+ using a Cars domain. The Proxy pattern provides a surrogate object that controls access to another object. Here, a `Car` abstract base class is implemented by `SportsCar` (the real subject), while `LazyCarModelProxy` defers expensive 3D model loading and `ValetCarProxy` restricts speed and access for valet parking scenarios.

## How the Implementation Works

### Class Diagram (Pseudocode)

The following table maps the participants to their OOP/SOLID roles:

```
  ┌──────────────────────────────────────────────────────────────────────────┐
  │ Participant            ──  OOP Role         ──  SOLID Principle         │
  ├──────────────────────────────────────────────────────────────────────────┤
  │ Car (ABC)              ──  Abstraction      ──  DIP (depend on          │
  │                        ──  (contract)       ──  abstractions)           │
  ├──────────────────────────────────────────────────────────────────────────┤
  │ SportsCar              ──  Inheritance      ──  LSP (substitutable      │
  │ (real subject)         ──  Polymorphism     ──  for Car)                │
  │                        ──                  ──  OCP (open for           │
  │                        ──                  ──  extension)               │
  ├──────────────────────────────────────────────────────────────────────────┤
  │ ValetCarProxy          ──  Encapsulation    ──  SRP (single reason      │
  │ LazyCarModelProxy      ──  (access logic    ──  to change: access       │
  │ (proxies)              ──   hidden)         ──  control / lazy load)    │
  ├──────────────────────────────────────────────────────────────────────────┤
  │ Driver / client        ──  Composition      ──  ISP (depends only       │
  │ (caller)               ──  (uses Car API)   ──  on Car abstraction)     │
  └──────────────────────────────────────────────────────────────────────────┘
```

### OOP Concepts Applied

- **Abstraction** — The `Car` ABC defines the contract using `@abstractmethod` decorators. Python enforces that concrete subclasses implement all abstract methods at instantiation time.
- **Encapsulation** — Internal state such as `_full_3d_model`, `_trunk_contents`, and `_glovebox_contents` is stored with a single underscore prefix, indicating protected access. The proxies encapsulate their access control rules and lazy initialization logic.
- **Inheritance** — `SportsCar`, `LazyCarModelProxy`, and `ValetCarProxy` extend `Car`, inheriting common fields like `_vin`, `_make`, `_model`, and `_year`.
- **Polymorphism** — The driver calls methods on `Car` references without knowing the concrete type. Each subclass provides its own implementation of `drive()`, `display_details()`, and `access_trunk()`.

### SOLID Principles Applied

- **Single Responsibility Principle** — `SportsCar` handles real car behavior. `ValetCarProxy` handles access control. `LazyCarModelProxy` handles lazy initialization.
- **Open/Closed Principle** — Adding a new proxy type requires creating a new subclass of `Car`. Existing classes remain unchanged.
- **Liskov Substitution Principle** — Every proxy can replace `Car` in client code without breaking correctness.
- **Interface Segregation Principle** — The `Car` ABC exposes only the methods every car representation needs.
- **Dependency Inversion Principle** — High-level driver code depends on the `Car` abstraction, not on concrete classes.

## Implementation

The complete Python program below demonstrates the Proxy pattern in a Cars context. It is runnable with Python 3.12+ and uses ABCs, type hints, and strict validation.

### Utility

```python
from abc import ABC, abstractmethod
from typing import Optional
import random
import string


def generate_vin() -> str:
    """Generate a random 17-character vehicle identification number."""
    chars = string.ascii_uppercase + string.digits
    return ''.join(random.choice(chars) for _ in range(17))
```

### Subject: Car (Abstract Base Class)

```python
class Car(ABC):
    """Subject interface for all car types in the Proxy pattern."""

    def __init__(self, vin: str, make: str, model: str, year: int) -> None:
        self._vin = vin
        self._make = make
        self._model = model
        self._year = year

    @property
    def vin(self) -> str:
        return self._vin

    @property
    def make(self) -> str:
        return self._make

    @property
    def model(self) -> str:
        return self._model

    @property
    def year(self) -> int:
        return self._year

    def get_short_description(self) -> str:
        """Return a brief identifier for this car."""
        return f"{self._year} {self._make} {self._model} ({self._vin})"

    @abstractmethod
    def display_details(self) -> str:
        """Show full car specifications including 3D model data."""
        ...

    @abstractmethod
    def drive(self, speed: float) -> str:
        """Drive the car at the given speed."""
        ...

    @abstractmethod
    def get_max_speed(self) -> float:
        """Return the maximum speed of the car."""
        ...

    @abstractmethod
    def access_trunk(self) -> str:
        """Open and access the trunk compartment."""
        ...
```

### RealSubject: SportsCar

```python
class SportsCar(Car):
    """RealSubject: The real sports car with expensive 3D model and full power."""

    def __init__(self, vin: str, make: str, model: str, year: int) -> None:
        super().__init__(vin, make, model, year)
        self._horsepower: int = 750
        self._engine_type: str = "5.2L V10 Twin-Turbo"
        self._acceleration_0_60: float = 2.8
        self._max_speed: float = 200.0
        self._full_3d_model: Optional[dict] = None
        self._trunk_contents: list[str] = [
            "First aid kit", "Tool set", "Owners manual"
        ]
        self._glovebox_contents: list[str] = [
            "Registration", "Insurance card", "Service record"
        ]
        self._is_engine_running: bool = False
        self._current_speed: float = 0.0
        self._load_3d_model()

    def _load_3d_model(self) -> None:
        """Simulate expensive 3D model loading (200 MB of data)."""
        print(
            f"  [SportsCar] Loading 3D model for {self._vin}... "
            f"200 MB of polygon data, textures, and materials loaded."
        )
        self._full_3d_model = {
            "polygons": 450_000,
            "textures": ["carbon_fiber", "leather_black", "aluminum_brushed"],
            "materials": ["forged_carbon", "alcantara", "titanium"],
        }

    def display_details(self) -> str:
        """Show full car specifications including 3D model data."""
        model_data = self._full_3d_model
        trunk = ", ".join(self._trunk_contents)
        glovebox = ", ".join(self._glovebox_contents)
        return (
            f"--- {self.get_short_description()} ---\n"
            f"  Engine: {self._engine_type}\n"
            f"  Horsepower: {self._horsepower} hp\n"
            f"  0-60 mph: {self._acceleration_0_60}s\n"
            f"  Top Speed: {self._max_speed} mph\n"
            f"  3D Model: {model_data['polygons']:,} polygons, "
            f"{len(model_data['textures'])} textures\n"
            f"  Trunk: {trunk}\n"
            f"  Glovebox: {glovebox}"
        )

    def drive(self, speed: float) -> str:
        """Drive the car at the requested speed (full capabilities)."""
        actual_speed = min(speed, self._max_speed)
        self._is_engine_running = True
        self._current_speed = actual_speed
        engine_note = "ROAR! V10 screams as it climbs through the rev range."
        return (
            f"  [SportsCar] {engine_note}\n"
            f"  Speed: {actual_speed:.1f} mph | Gear: 7th | RPM: 8200"
        )

    def get_max_speed(self) -> float:
        return self._max_speed

    def access_trunk(self) -> str:
        """Open the trunk and reveal contents."""
        contents = ", ".join(self._trunk_contents)
        return f"  [SportsCar] Trunk opened. Contents: {contents}"
```

### Virtual Proxy: LazyCarModelProxy

```python
class LazyCarModelProxy(Car):
    """Virtual Proxy: Defers creation of the expensive SportsCar until needed."""

    def __init__(self, vin: str, make: str, model: str, year: int) -> None:
        super().__init__(vin, make, model, year)
        self._real_car: Optional[SportsCar] = None

    def _get_real_car(self) -> SportsCar:
        """Initialize the real car on first access."""
        if self._real_car is None:
            print(
                f"  [LazyCarModelProxy] Initializing real SportsCar for "
                f"{self.get_short_description()}..."
            )
            self._real_car = SportsCar(self._vin, self._make, self._model, self._year)
        else:
            print(f"  [LazyCarModelProxy] Reusing existing SportsCar instance.")
        return self._real_car

    def display_details(self) -> str:
        return self._get_real_car().display_details()

    def drive(self, speed: float) -> str:
        return self._get_real_car().drive(speed)

    def get_max_speed(self) -> float:
        return self._get_real_car().get_max_speed()

    def access_trunk(self) -> str:
        return self._get_real_car().access_trunk()
```

### Protection Proxy: ValetCarProxy

```python
class ValetCarProxy(Car):
    """Protection Proxy: Restricts what a valet can do with the car."""

    def __init__(self, real_car: Car, max_valet_speed: float = 30.0) -> None:
        super().__init__(
            real_car.vin, real_car.make, real_car.model, real_car.year
        )
        self._real_car: Car = real_car
        self._max_valet_speed: float = max_valet_speed
        self._trunk_denied_message: str = (
            "  [ValetCarProxy] ACCESS DENIED: Valet key does not "
            "permit trunk access."
        )

    def display_details(self) -> str:
        """Show limited information card for valet."""
        return (
            f"--- Valet Card ---\n"
            f"  VIN: {self._vin}\n"
            f"  Max Permitted Speed: {self._max_valet_speed} mph\n"
            f"  Trunk Access: DENIED\n"
            f"  Glovebox Access: DENIED"
        )

    def drive(self, speed: float) -> str:
        """Enforce the valet speed limit."""
        limited_speed = min(speed, self._max_valet_speed)
        if limited_speed < speed:
            print(
                f"  [ValetCarProxy] Speed limited to {self._max_valet_speed} mph."
            )
        return self._real_car.drive(limited_speed)

    def get_max_speed(self) -> float:
        return self._max_valet_speed

    def access_trunk(self) -> str:
        return self._trunk_denied_message
```

### Main / Driver

```python
def main() -> None:
    print("=" * 72)
    print("  PROXY PATTERN — Cars Domain (Python 3.12+)")
    print("=" * 72)
    print()

    # ── Scenario 1: Virtual Proxy (Lazy Loading) ────────────────────────────
    print("--- SCENARIO 1: Virtual Proxy (Lazy Loading) ---")
    lazy_car: Car = LazyCarModelProxy(generate_vin(), "Sypher", "GT", 2026)
    print("  Showroom: Created LazyCarModelProxy. No 3D model loaded yet.")
    print()

    print("  Customer browses thumbnail:")
    print(f"  {lazy_car.get_short_description()}")
    print()

    print('  Customer clicks "View Full Details":')
    details: str = lazy_car.display_details()
    print(details)
    print()

    # ── Scenario 2: Protection Proxy (Valet Key) ─────────────────────────────
    print("--- SCENARIO 2: Protection Proxy (Valet Key) ---")
    real_car: Car = SportsCar(generate_vin(), "Sypher", "GT-S", 2026)
    valet_key: Car = ValetCarProxy(real_car)
    print()

    print("  Owner hands the valet key to the parking attendant.")
    print()

    print("  Valet: Displays info card:")
    print(valet_key.display_details())
    print()

    print("  Valet: Attempts to drive at 80 mph:")
    print(valet_key.drive(80.0))
    print()

    print("  Valet: Attempts to access the trunk:")
    print(valet_key.access_trunk())
    print()

    # ── Scenario 3: Owner Returns — Full Access ─────────────────────────────
    print("--- SCENARIO 3: Owner Returns --- Full Access ---")
    print("  Owner uses the real car key directly:")
    print(real_car.drive(120.0))
    print(real_car.access_trunk())
    print()

    # ── Scenario 4: Polymorphic Iteration ────────────────────────────────────
    print("--- SCENARIO 4: Polymorphic Car Display ---")
    cars: list[Car] = [lazy_car, valet_key]
    for car in cars:
        print(f"  [{car.vin}] Max Speed: {car.get_max_speed()} mph")
    print()

    # ── Scenario 5: Demonstrating Lazy Loading Benefit ──────────────────────
    print("--- SCENARIO 5: Lazy Loading Benefit ---")
    showroom_cars: list[LazyCarModelProxy] = [
        LazyCarModelProxy(generate_vin(), "Sypher", "GT", 2026),
        LazyCarModelProxy(generate_vin(), "Sypher", "GT-S", 2026),
        LazyCarModelProxy(generate_vin(), "Falcon", "X1", 2026),
    ]
    print("  All 3 proxies created. Memory usage: minimal.")
    print("  Showing thumbnails...")
    for car in showroom_cars:
        print(f"    {car.get_short_description()}")
    print()

    print("  Customer examines one car in detail:")
    print(showroom_cars[0].display_details())
    print()

    print("=" * 72)
    print("  Proxy pattern demonstration complete.")
    print("=" * 72)


if __name__ == "__main__":
    main()
```

## Language-Specific Notes

- **`@abstractmethod` for interface enforcement** — Python's `abc.ABC` and `@abstractmethod` decorator enforce at instantiation time that every concrete subclass implements all abstract methods. Attempting to instantiate a `Car` subclass that does not override `display_details()` raises a `TypeError` at runtime. This provides the interface contract enforcement that Python lacks at the language level.

- **Properties for controlled attribute access** — The `vin`, `make`, `model`, and `year` attributes are exposed through `@property` getters, making them read-only from outside the class. This follows Python's EAFP (Easier to Ask for Forgiveness than Permission) philosophy and is more idiomatic than explicit `getVin()` methods used in Java-like languages.

- **Underscore-prefixed protected attributes** — Internal state such as `_full_3d_model`, `_trunk_contents`, and `_is_engine_running` uses a single underscore prefix to signal that they are protected implementation details. Python does not enforce access modifiers at compile time, so the underscore is a convention that indicates "do not access from outside the class."

- **Type hints with `Optional` for nullable references** — `LazyCarModelProxy._real_car` is typed as `Optional[SportsCar]`, explicitly representing the uninitialized state. The `_get_real_car()` method checks `if self._real_car is None` and creates the real car lazily. Type checkers like `mypy` and `pyright` can verify that the null check is performed before accessing the real car.

- **F-string formatting for readable output** — Python's f-strings with format specifiers (`:.1f` for one decimal place, `:,` for thousand separators) make the output formatting concise and readable. The `model_data['polygons']:,` expression formats 450000 as "450,000" automatically.

- **`__name__ == "__main__"` guard** — The driver code is protected by the standard `if __name__ == "__main__":` guard, ensuring the demonstration only runs when the file is executed directly and not when imported as a module. This is the idiomatic Python pattern for separating executable code from library code.
