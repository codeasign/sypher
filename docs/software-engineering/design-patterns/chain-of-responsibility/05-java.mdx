## Implementation

Below is a complete Java implementation of the Chain of Responsibility pattern using a stock market domain. A `TradeOrder` flows through an approval chain consisting of `JuniorTrader`, `SeniorTrader`, `PortfolioManager`, and `RiskDirector`. Each handler decides whether to approve the order based on its total value, or passes it to the next handler in the chain. The implementation uses Java 17+ features including `sealed` interfaces, `record` classes, and `switch` expressions, and demonstrates OOP concepts (abstraction, encapsulation, inheritance, polymorphism) along with SOLID principles (SRP, OCP, LSP, ISP, DIP).

### How the Implementation Works

#### Class Diagram (Pseudocode)

```

//

//   ├──────────────────────────────────────────┤
//   │  + setNext(OrderApprover): OrderApprover │ ← OOP: Abstraction via interface
//   │  + approveOrder(TradeOrder): String      │    ISP: focused interface with
```

#### OOP Concepts Applied

1. **Abstraction** — `OrderApprover` is an interface that defines the handler contract (`setNext`, `approveOrder`, `canApprove`). The chain traversal logic is implemented once in `AbstractApprover` and reused by all concrete handlers.
2. **Encapsulation** — Each handler hides its approval limit and next-handler reference behind private fields. `TradeOrder` as a record encapsulates its data with auto-generated accessors.
3. **Inheritance** — `AbstractApprover` provides a base implementation of `setNext` and `approveOrder`. All four concrete handlers extend it and only override `canApprove`. `RiskDirector` overrides `canApprove` to always return `true`.
4. **Polymorphism** — `approveOrder` calls `canApprove()` polymorphically on each handler. The correct limit check runs based on the handler's runtime type, without `instanceof` checks.

#### SOLID Principles Applied

5. **SRP** — Each handler has one reason to change: its approval threshold. `TradeOrder` is solely responsible for representing order data. The chain traversal logic lives in `AbstractApprover`, not in any concrete handler.
6. **OCP** — Adding a new handler (e.g., `VicePresident` with a $50,000 limit) requires creating a new class extending `AbstractApprover` and inserting it into the chain. No existing handler code is modified.
7. **LSP** — Any concrete handler can replace any other in the chain. `RiskDirector` (which always approves) can substitute for any handler without breaking the chain behavior.
8. **ISP** — The `OrderApprover` interface is minimal: `setNext`, `approveOrder`, and `canApprove`. No handler is forced to implement methods it does not need.
9. **DIP** — Handlers depend on the abstract `OrderApprover` interface when calling `setNext` and when delegating to the next handler. The chain is built by injecting concrete handlers into the abstract interface.

### TradeOrder (Request Record)
```java
import java.text.NumberFormat;
import java.util.Locale;

record TradeOrder(
    String ticker,
    int quantity,
    double pricePerShare,
    String orderType  // "buy" or "sell"
) {
    /**
     * Computes the total monetary value of the order.
     */
    public double totalValue() {
        return quantity * pricePerShare;
    }
}
```

### OrderApprover (Handler Interface)
```java
interface OrderApprover {
    /**
     * Sets the next handler in the chain. Returns the next handler for chaining.
     */
    OrderApprover setNext(OrderApprover approver);

    /**
     * Attempts to approve the order or passes it to the next handler.
     */
    String approveOrder(TradeOrder order);

    /**
     * Determines whether this handler can approve the given order.
     */
    boolean canApprove(TradeOrder order);
}
```

### AbstractApprover (Abstract Base Handler)
```java
abstract class AbstractApprover implements OrderApprover {
    protected final String name;
    protected final double approvalLimit;
    private OrderApprover nextApprover;

    public AbstractApprover(String name, double approvalLimit) {
        this.name = name;
        this.approvalLimit = approvalLimit;
        this.nextApprover = null;
    }

    @Override
    public OrderApprover setNext(OrderApprover approver) {
        this.nextApprover = approver;
        return approver;
    }

    @Override
    public String approveOrder(TradeOrder order) {
        if (canApprove(order)) {
            String message = String.format(
                "%s %s approved: %s %d %s at $%.2f (total: $%,.2f)",
                this.getClass().getSimpleName(),
                name,
                order.orderType().toUpperCase(),
                order.quantity(),
                order.ticker(),
                order.pricePerShare(),
                order.totalValue()
            );
            System.out.println(message);
            return message;
        }

        if (nextApprover != null) {
            System.out.printf("%s %s: Order $%,.2f exceeds my limit of $%,.2f. Escalating...%n",
                this.getClass().getSimpleName(),
                name,
                order.totalValue(),
                approvalLimit);
            return nextApprover.approveOrder(order);
        }

        String message = String.format(
            "REJECTED: No approver can handle order for %s ($%,.2f)",
            order.ticker(),
            order.totalValue()
        );
        System.out.println(message);
        return message;
    }
}
```

### JuniorTrader (Concrete Handler)
```java
class JuniorTrader extends AbstractApprover {
    public JuniorTrader(String name) {
        super(name, 1_000.00);
    }

    @Override
    public boolean canApprove(TradeOrder order) {
        return order.totalValue() <= approvalLimit;
    }
}
```

### Main / Driver
```java
public class Main {
    public static void main(String[] args) {
        // Build the chain: JuniorTrader -> SeniorTrader -> PortfolioManager -> RiskDirector
        JuniorTrader junior = new JuniorTrader("Alice");
        SeniorTrader senior = new SeniorTrader("Bob");
        PortfolioManager portfolioMgr = new PortfolioManager("Carol");
        RiskDirector riskDirector = new RiskDirector("Dave");

        junior.setNext(senior)
              .setNext(portfolioMgr)
              .setNext(riskDirector);

        // Define test orders
        TradeOrder[] orders = {
            new TradeOrder("AAPL", 10, 150.00, "buy"),
            new TradeOrder("GOOGL", 50, 175.00, "buy"),
            new TradeOrder("MSFT", 1000, 200.00, "sell"),
        };

        System.out.println("=".repeat(72));
        System.out.println("TRADE ORDER APPROVAL SYSTEM");
        System.out.println("=".repeat(72));
        System.out.println();

        for (TradeOrder order : orders) {
            System.out.printf("Processing: %s %d %s @ $%.2f (total: $%,.2f)%n",
                order.orderType().toUpperCase(),
                order.quantity(),
                order.ticker(),
                order.pricePerShare(),
                order.totalValue());
            System.out.println("-".repeat(72));
            junior.approveOrder(order);
            System.out.println();
        }
    }
}
```

### Language-Specific Notes

- **Records**: `TradeOrder` is a Java 17+ `record` — the compiler auto-generates the constructor, `equals()`, `hashCode()`, `toString()`, and accessor methods (`ticker()`, `quantity()`, etc.), eliminating boilerplate.
- **`interface` vs `abstract class`**: The pattern uses a clean `OrderApprover` interface for the contract and an `AbstractApprover` abstract class for shared implementation. This separation follows the ISP — concrete handlers only implement the interface if they need a different base.
- **Fluent chaining**: `setNext()` returns the next handler, enabling method chaining: `junior.setNext(senior).setNext(portfolioMgr).setNext(riskDirector)`.
- **`Double.MAX_VALUE`**: `RiskDirector` uses `Double.MAX_VALUE` as its limit, but overrides `canApprove()` to return `true` unconditionally — making the final handler explicit and self-documenting.
- **`String.format` and `printf`**: Java's `String.format()` and `System.out.printf()` with format specifiers (`%s`, `%d`, `%f`, `%,.2f`) handle formatted output including locale-aware number formatting with commas.
- **Single-file demo**: The entire implementation is in one `public class Main` file with inner classes, so it can be compiled and run with `javac Main.java && java Main` without a package structure or build tool.
