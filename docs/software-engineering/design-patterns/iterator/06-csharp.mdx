The Iterator pattern is widely used in the Stock Market domain to traverse collections of price data, trade records, and portfolio holdings without exposing the underlying data structure. In C#, the pattern is deeply integrated into the language through the `IEnumerable<T>` and `IEnumerator<T>` interfaces, which power `foreach` loops and LINQ queries. A stock market application might need to iterate over a price history chronologically, filter trades by volume thresholds, or walk through holdings within a date range -- the Iterator pattern decouples the traversal logic from the collection, making it easy to swap, compose, or extend iteration behavior.

## How the Implementation Works

### Class Diagram (Pseudocode)

```
interface IStockIterator {
    + HasNext(): bool
    + Next(): StockPrice
}

class PriceHistoryIterator : IStockIterator {
    - _prices: List<StockPrice>
    - _position: int
    + HasNext(): bool
    + Next(): StockPrice
    + Reset(): void
}

interface IStockPriceCollection : IEnumerable<StockPrice> {
    + CreateIterator(): IStockIterator
    + AddPrice(StockPrice): void
    + PriceCount: int
}

class PriceHistory : IStockPriceCollection {
    - _prices: List<StockPrice>
    + Symbol: string
    + CreateIterator(): IStockIterator
    + GetEnumerator(): IEnumerator<StockPrice>
    + AddPrice(StockPrice): void
}

class HighVolumeFilter : IStockIterator {
    - _inner: IStockIterator
    - _minVolume: long
    - _buffer: StockPrice?
    + HasNext(): bool
    + Next(): StockPrice
}

class DateRangeFilter : IStockIterator {
    - _inner: IStockIterator
    - _startDate: string
    - _endDate: string
    - _buffer: StockPrice?
    + HasNext(): bool
    + Next(): StockPrice
}
```

### OOP Concepts Applied

- **Encapsulation** -- The `PriceHistoryIterator` hides its internal position counter and list reference, exposing only `HasNext()` and `Next()`. The `PriceHistory` aggregate encapsulates the underlying `List<StockPrice>`.
- **Abstraction** -- The `IStockIterator` interface defines a clean contract for traversal without dictating how elements are stored or retrieved.
- **Inheritance** -- Although C# uses interface implementation rather than class inheritance for iterators, `PriceHistory` inherits from `object` (implicitly) and implements `IStockPriceCollection` and `IEnumerable<StockPrice>`.
- **Polymorphism** -- `HighVolumeFilter` and `DateRangeFilter` both implement `IStockIterator` and can be used interchangeably wherever an iterator is expected. The `foreach` loop works polymorphically on any `IEnumerable<StockPrice>`.

### SOLID Principles Applied

- **SRP (Single Responsibility)** -- Each class has one reason to change: `PriceHistory` stores and provides access to price data, `PriceHistoryIterator` handles traversal state, and `HighVolumeFilter` handles volume filtering.
- **OCP (Open/Closed)** -- New filter iterators (e.g., `PriceRangeFilter`, `MovingAverageFilter`) can be added by implementing `IStockIterator` without modifying existing code. The system is open for extension but closed for modification.
- **LSP (Liskov Substitution)** -- Any `IStockIterator` implementation can replace another without breaking the client. All filter iterators follow the same contract and can be chained or substituted freely.
- **ISP (Interface Segregation)** -- The `IStockIterator` interface is minimal with only two methods. Clients are not forced to depend on methods they do not use. `IStockPriceCollection` is similarly focused on collection operations.
- **DIP (Dependency Inversion)** -- High-level code (the `Program` class) depends on the `IStockIterator` abstraction, not on concrete iterator classes. Filter iterators depend on the same abstraction via constructor injection.

    string Symbol,

### PriceHistoryIterator (Concrete Iterator)

```csharp
// ── Concrete Iterator ──
public class PriceHistoryIterator : IStockIterator
{
    private readonly List<StockPrice> _prices;
    private int _position;

    public PriceHistoryIterator(List<StockPrice> prices)
    {
        _prices = prices;
        _position = 0;
    }

    public bool HasNext() => _position < _prices.Count;

    public StockPrice Next()
    {
        if (!HasNext())
            throw new InvalidOperationException("No more stock prices");
        return _prices[_position++];
    }

    public void Reset() => _position = 0;
}
```

### PriceHistory (Concrete Aggregate)

```csharp
// ── Concrete Aggregate ──
public class PriceHistory : IStockPriceCollection
{
    private readonly List<StockPrice> _prices = new();
    public string Symbol { get; }

    public PriceHistory(string symbol)
    {
        Symbol = symbol;
    }

    public void AddPrice(StockPrice price) => _prices.Add(price);

    public int PriceCount => _prices.Count;

    public IStockIterator CreateIterator() => new PriceHistoryIterator(_prices);

    // IEnumerable<StockPrice> implementation
    public IEnumerator<StockPrice> GetEnumerator() => _prices.GetEnumerator();

    IEnumerator IEnumerable.GetEnumerator() => GetEnumerator();
}
```

### HighVolumeFilter (Decorator: HighVolumeFilter (OCP))

```csharp
// ── Decorator: HighVolumeFilter (OCP) ──
public class HighVolumeFilter : IStockIterator
{
    private readonly IStockIterator _inner;
    private readonly long _minVolume;
    private StockPrice? _buffer;

    public HighVolumeFilter(IStockIterator inner, long minVolume)
    {
        _inner = inner;
        _minVolume = minVolume;
    }

    public bool HasNext()
    {
        while (_buffer == null && _inner.HasNext())
        {
            var candidate = _inner.Next();
            if (candidate.Volume >= _minVolume)
                _buffer = candidate;
        }
        return _buffer != null;
    }

    public StockPrice Next()
    {
        if (!HasNext())
            throw new InvalidOperationException("No more matching prices");
        var result = _buffer!;
        _buffer = null;
        return result;
    }
}
```

### DateRangeFilter (Decorator: DateRangeFilter (OCP))

```csharp
// ── Decorator: DateRangeFilter (OCP) ──
public class DateRangeFilter : IStockIterator
{
    private readonly IStockIterator _inner;
    private readonly string _startDate;
    private readonly string _endDate;
    private StockPrice? _buffer;

    public DateRangeFilter(IStockIterator inner, string startDate, string endDate)
    {
        _inner = inner;
        _startDate = startDate;
        _endDate = endDate;
    }

    public bool HasNext()
    {
        while (_buffer == null && _inner.HasNext())
        {
            var candidate = _inner.Next();
            if (string.Compare(candidate.Date, _startDate) >= 0 &&
                string.Compare(candidate.Date, _endDate) <= 0)
            {
                _buffer = candidate;
            }
        }
        return _buffer != null;
    }

    public StockPrice Next()
    {
        if (!HasNext())
            throw new InvalidOperationException("No more prices in date range");
        var result = _buffer!;
        _buffer = null;
        return result;
    }
}
```

### Language-Specific Notes

- **`IEnumerable<T>` integration** -- C# bakes the Iterator pattern directly into the language. Any collection implementing `IEnumerable<T>` can be used with `foreach`, LINQ, and collection initializers. The `PriceHistory` class implements `IEnumerable<StockPrice>` by delegating to the underlying `List<T>.GetEnumerator()`, which means it works with LINQ methods like `.ToList()` and `.Where()`.
- **Yield return / iterator methods** -- C# provides compiler-generated iterators via `yield return`. A method returning `IEnumerable<T>` or `IEnumerator<T>` can use `yield return` to produce elements lazily without writing a dedicated iterator class. This is a more concise alternative to the manual `PriceHistoryIterator` implementation shown above.
- **Null safety with nullable reference types** -- The filter iterators use `StockPrice? _buffer` (nullable annotation) to track buffered elements. The `_buffer!` null-forgiving operator is used in `Next()` because `HasNext()` guarantees a non-null value. This pattern keeps the code safe without runtime null checks in the hot path.
- **LINQ as a declarative iterator** -- Language Integrated Query (LINQ) provides a declarative way to filter, transform, and aggregate sequences. Under the hood, LINQ operators like `Where()`, `Select()`, and `OrderBy()` create iterator chains that execute lazily -- the same Decorator pattern shown with `HighVolumeFilter` and `DateRangeFilter`, but built into the framework.
- **Struct enumerators for performance** -- In performance-sensitive stock market applications (e.g., real-time tick processing), implementing `IEnumerator<T>` as a `struct` instead of a `class` avoids heap allocation during iteration. `List<T>` itself uses a struct enumerator internally, which is why `foreach` on a `List<T>` does not allocate.
- **Disposable iterators** -- `IEnumerator<T>` inherits from `IDisposable`. The `foreach` loop calls `Dispose()` automatically when iteration completes, which is important for iterators that hold resources (e.g., database cursors, file streams). Always implement `IDisposable` on custom iterators that manage unmanaged resources.
