The Iterator pattern is a fundamental behavioral design pattern in Java, deeply integrated into the language through the `java.util.Iterator` and `Iterable` interfaces. In the stock market domain, analysts and algorithmic trading systems need to traverse time-series price data, filter by criteria such as volume spikes or date windows, and apply custom traversal logic. Java's strong type system and interface-based design make the Iterator pattern especially clean and maintainable, with compile-time safety guarantees that ensure correct traversal behavior.

### OOP Concepts Applied

- **Encapsulation**: `PriceHistory` hides its internal `List<StockPrice>` behind add/query methods. `PriceHistoryIterator` encapsulates the position counter and traversal logic.
- **Abstraction**: The `StockIterator<T>` and `StockPriceCollection` interfaces define clean contracts that decouple clients from implementation details.
- **Polymorphism**: `PriceHistoryIterator`, `HighVolumeFilter`, and `DateRangeFilter` all implement `StockIterator<StockPrice>`, allowing them to be substituted interchangeably.
- **Inheritance**: The filter iterators implement the `StockIterator` interface, inheriting the contract and enabling decorator-style composition.

### SOLID Principles Applied

- **SRP (Single Responsibility)**: Each class has one job -- `PriceHistory` stores data, `PriceHistoryIterator` traverses, `HighVolumeFilter` filters by volume, `DateRangeFilter` filters by date range.
- **OCP (Open/Closed)**: The system is open for extension via new filter iterators (e.g., `PriceRangeFilter`, `SectorFilter`) without modifying existing classes.
- **LSP (Liskov Substitution)**: Any `StockIterator<StockPrice>` implementation can replace another transparently. The decorator filters wrap and delegate without breaking client expectations.
- **ISP (Interface Segregation)**: `StockIterator<T>` has only two methods (`hasNext`, `next`), keeping the interface minimal and focused.
- **DIP (Dependency Inversion)**: Client code depends on the abstract `StockIterator<T>` and `StockPriceCollection` interfaces, never on concrete implementations.

### StockPrice (Data record)

```java
// Iterator Pattern -- Java -- Stock Market example
// ==================================================
// Participants: StockIterator<T> (interface), PriceHistory (Aggregate),
//   PriceHistoryIterator (ConcreteIterator), StockPrice (record)
// OOP: abstraction, encapsulation, polymorphism
// SOLID: SRP, OCP, LSP, ISP, DIP
// ==================================================

import java.util.*;

// ── Data record ──
record StockPrice(String symbol, String date,
                  double openPrice, double high, double low,
                  double close, long volume) {
    @Override
    public String toString() {
        return String.format("%s | %-5s O:$%.2f H:$%.2f L:$%.2f C:$%.2f Vol:%,d",
                date, symbol, openPrice, high, low, close, volume);
    }
}
```

### StockIterator (Iterator Interface)

```java
// ── Iterator Interface ──
interface StockIterator<T> {

```

### PriceHistoryIterator (Concrete Iterator)

```java
// ── Concrete Iterator ──
class PriceHistoryIterator implements StockIterator<StockPrice> {
    private final List<StockPrice> prices;
    private int position;

    public PriceHistoryIterator(List<StockPrice> prices) {
        this.prices = prices;
        this.position = 0;
    }

    @Override
    public boolean hasNext() {
        return position < prices.size();
    }

    @Override
    public StockPrice next() {
        if (!hasNext()) {
            throw new NoSuchElementException("No more stock prices");
        }
        return prices.get(position++);
    }

    public void reset() {
        position = 0;
    }
}
```

### PriceHistory (Concrete Aggregate)

```java
// ── Concrete Aggregate ──
class PriceHistory implements StockPriceCollection, Iterable<StockPrice> {
    private final String symbol;
    private final List<StockPrice> prices = new ArrayList<>();

    public PriceHistory(String symbol) {
        this.symbol = symbol;
    }

    public void addPrice(StockPrice price) {
        prices.add(price);
    }

    public int priceCount() {
        return prices.size();
    }

    public String getSymbol() {
        return symbol;
    }

    @Override
    public StockIterator<StockPrice> createIterator() {
        return new PriceHistoryIterator(prices);
    }

    // Java Iterable interface: returns a java.util.Iterator
    @Override
    public Iterator<StockPrice> iterator() {
        return prices.iterator();
    }
}
```

### HighVolumeFilter (Decorator: High Volume Filter (OCP))

```java
// ── Decorator: High Volume Filter (OCP) ──
class HighVolumeFilter implements StockIterator<StockPrice> {
    private final StockIterator<StockPrice> iterator;
    private final long minVolume;
    private StockPrice buffer;

    public HighVolumeFilter(StockIterator<StockPrice> iterator, long minVolume) {
        this.iterator = iterator;
        this.minVolume = minVolume;
        this.buffer = null;
    }

    @Override
    public boolean hasNext() {
        while (buffer == null && iterator.hasNext()) {
            StockPrice candidate = iterator.next();
            if (candidate.volume() >= minVolume) {
                buffer = candidate;
            }
        }
        return buffer != null;
    }

    @Override
    public StockPrice next() {
        if (!hasNext()) {
            throw new NoSuchElementException("No more matching prices");
        }
        StockPrice result = buffer;
        buffer = null;
        return result;
    }
}
```

### DateRangeFilter (Decorator: Date Range Filter (OCP))

```java
// ── Decorator: Date Range Filter (OCP) ──
class DateRangeFilter implements StockIterator<StockPrice> {
    private final StockIterator<StockPrice> iterator;
    private final String startDate;
    private final String endDate;
    private StockPrice buffer;

    public DateRangeFilter(StockIterator<StockPrice> iterator,
                           String startDate, String endDate) {
        this.iterator = iterator;
        this.startDate = startDate;
        this.endDate = endDate;
        this.buffer = null;
    }

    @Override
    public boolean hasNext() {
        while (buffer == null && iterator.hasNext()) {
            StockPrice candidate = iterator.next();
            if (candidate.date().compareTo(startDate) >= 0
                    && candidate.date().compareTo(endDate) <= 0) {
                buffer = candidate;
            }
        }
        return buffer != null;
    }

    @Override
    public StockPrice next() {
        if (!hasNext()) {
            throw new NoSuchElementException("No more prices in date range");
        }
        StockPrice result = buffer;
        buffer = null;
        return result;
    }
}
```

### Language-Specific Notes

- Java's `Iterable<T>` interface and enhanced `for-each` loop provide native iteration support, making any class that implements `Iterable` directly usable in loop constructs.
- The `Iterator<T>` interface from `java.util` is the standard library equivalent. The custom `StockIterator<T>` interface in this example mirrors it to demonstrate the pattern explicitly.
- Java records (`record StockPrice(...)`) provide a concise way to define immutable data carriers with auto-generated `equals()`, `hashCode()`, `toString()`, and accessor methods.
- Type safety is enforced at compile time through generics (`StockIterator<T>`), eliminating the need for casting when iterating over typed collections.
- The decorator-style iterator pattern composes well with Java's `Stream` API -- filter iterators can be adapted to `Stream` pipelines using `Spliterator` or by collecting into collections first.
- Java's `NoSuchElementException` (from `java.util`) is the standard exception for signaling the end of an iteration, providing consistency with the JDK's own iterator implementations.
