## Implementation

TypeScript brings static typing to the Iterator pattern, providing compile-time safety for the Stock Market domain. Typed interfaces (`StockIterator<T>`, `Iterable<T>`) define the contract that concrete iterators and aggregates must fulfill. The `PriceHistory<T>` aggregate stores `StockPrice` objects for AAPL, GOOGL, and MSFT, while `PriceHistoryIterator<T>` traverses the collection. Generic filter iterators (`HighVolumeFilter<T>`, `DateRangeFilter<T>`) compose with the base iterator to add filtering logic without modifying the aggregate. TypeScript's generics ensure that all iterators work with the correct element type at compile time.

### How the Implementation Works

#### Class Diagram (Pseudocode)

```
// ── Participant Diagram ──
//
//   ┌──────────────────────────────────┐
//   │  interface StockIterator<T>      │ ← OOP: Abstraction via interface
//   ├──────────────────────────────────┤    ISP: minimal interface
//   │ + hasNext(): boolean             │     (hasNext + next)
//   │ + next(): T                      │
//   │ + reset(): void                  │
//   └──────────┬───────────────────────┘
//              ▲
//   ┌──────────┴───────────────────────┐
//   │  interface Iterable<T>           │ ← DIP: depends on abstract
//   ├──────────────────────────────────┤     StockIterator, not
//   │ + createIterator():              │     concrete classes
//   │   StockIterator<T>               │
//   └──────────┬───────────────────────┘
//              │
//              ▼
//   ┌──────────────────────────────────┐
//   │  PriceHistory<T> implements      │ ← OOP: Encapsulation
//   │    Iterable<T>                   │    SRP: stores prices only
//   ├──────────────────────────────────┤
//   │ + createIterator():              │
//   │   StockIterator<T>               │
//   │ + [Symbol.iterator]():           │
//   │   Iterator<T>                    │
//   └──────────┬───────────────────────┘
//              │
//              ▼
//   ┌──────────────────────────────────┐
//   │  PriceHistoryIterator<T>         │ ← SRP: iteration only
//   │    implements StockIterator<T>   │    OCP: new iterators via
//   └──────────┬───────────────────────┘     composition
//              │
//   ┌──────────┴──────────┐
//   │                     │
//   ┌────────────────┐ ┌──────────────┐
//   │ HighVolume     │ │ DateRange    │ ← OCP: extend behavior
//   │ Filter<T>      │ │ Filter<T>    │    without modifying
//   └────────────────┘ └──────────────┘     PriceHistory
```

#### OOP Concepts Applied

1. **Abstraction** — The `StockIterator<T>` and `Iterable<T>` interfaces define abstract contracts. Consumers depend on these interfaces, not on concrete implementations. The internal traversal logic is hidden behind `hasNext()` and `next()`.
2. **Encapsulation** — `PriceHistory<T>` hides its internal `prices` array behind `addPrice()` and `createIterator()`. The iterator encapsulates its own `position` state. Filter iterators encapsulate their buffering logic.
3. **Inheritance** — Not used directly (composition is preferred for iterators), but the `Symbol.iterator` protocol inherits the language-level iteration contract. Filter iterators implement the same `StockIterator<T>` interface as the base iterator.
4. **Polymorphism** — `HighVolumeFilter<T>` and `DateRangeFilter<T>` both implement `StockIterator<T>`. They can be used wherever a `StockIterator<T>` is expected, enabling polymorphic iteration behavior without type checks.

#### SOLID Principles Applied

5. **SRP** — `PriceHistory<T>` stores prices only. `PriceHistoryIterator<T>` handles iteration only. Each filter has a single filtering responsibility (volume threshold or date range).
6. **OCP** — New filtering iterators (e.g., `PriceRangeFilter<T>`, `MovingAverageIterator<T>`) can be added by creating new classes that implement `StockIterator<T>` and wrap an existing iterator. The `PriceHistory<T>` class and base iterator are never modified.
7. **LSP** — Any iterator implementing `StockIterator<T>` can substitute for any other. Filtered iterators behave identically to the base iterator from the consumer's perspective (same interface, same contract).
8. **ISP** — The `StockIterator<T>` interface is minimal: three methods (`hasNext`, `next`, `reset`). No client is forced to depend on methods it does not use.
9. **DIP** — Consumers depend on the abstract `StockIterator<T>` interface, not on concrete iterator classes. Iterators are injected via constructor parameters (dependency injection).

### StockPrice (Data class)

```typescript
// ============================================================
// Iterator Pattern -- TypeScript -- Stock Market example
// ============================================================
// Interfaces: StockIterator<T>, Iterable<T>
// Participants: PriceHistory<T> (Aggregate),
//   PriceHistoryIterator<T> (ConcreteIterator), StockPrice (data)
// OOP: abstraction, encapsulation, inheritance, polymorphism
// SOLID: SRP, OCP, LSP, ISP, DIP
// ============================================================

// ── Data class ──
class StockPrice {
  constructor(
    public readonly symbol: string,
    public readonly date: string,
    public readonly open: number,
    public readonly high: number,
    public readonly low: number,
    public readonly close: number,
    public readonly volume: number
  ) {}

  toString(): string {
    return `${this.date} | ${this.symbol.padEnd(5)} O:$${this.open.toFixed(2)} H:$${this.high.toFixed(2)} L:$${this.low.toFixed(2)} C:$${this.close.toFixed(2)} Vol:${this.volume.toLocaleString()}`;
  }
}
```

### PriceHistoryIterator (Concrete Iterator)

```typescript
// ── Concrete Iterator ──
class PriceHistoryIterator<T> implements StockIterator<T> {
  private position: number = 0;

  constructor(private readonly prices: T[]) {}

  hasNext(): boolean {
    return this.position < this.prices.length;
  }

  next(): T {
    if (!this.hasNext()) {
      throw new Error('No more elements');
    }
    const price = this.prices[this.position];
    this.position++;
    return price;
  }

  reset(): void {
    this.position = 0;
  }
}
```

### PriceHistory (Concrete Aggregate)

```typescript
// ── Concrete Aggregate ──
class PriceHistory<T extends StockPrice> implements Iterable<T> {
  private prices: T[] = [];

  constructor(public readonly symbol: string) {}

  addPrice(price: T): void {
    this.prices.push(price);
  }

  getPriceCount(): number {
    return this.prices.length;
  }

  createIterator(): StockIterator<T> {
    return new PriceHistoryIterator<T>(this.prices);
  }

  // Make the collection iterable with Symbol.iterator (native JS protocol)
  [Symbol.iterator](): Iterator<T> {
    let position = 0;
    const prices = this.prices;
    return {
      next(): IteratorResult<T> {
        if (position < prices.length) {
          return { value: prices[position++], done: false };
        } else {
          return { value: undefined as unknown as T, done: true };
        }
      }
    };
  }
}
```

### HighVolumeFilter (Extended Iterator: HighVolumeFilter)

```typescript
// ── Extended Iterator: HighVolumeFilter ──
// OCP: extends behavior without modifying PriceHistory or PriceHistoryIterator
// SRP: single responsibility -- filtering by volume threshold
class HighVolumeFilter<T extends StockPrice> implements StockIterator<T> {
  private buffer: T | null = null;

  constructor(
    private readonly iterator: StockIterator<T>,
    private readonly minVolume: number
  ) {}

  hasNext(): boolean {
    while (this.buffer === null && this.iterator.hasNext()) {
      const candidate = this.iterator.next();
      if (candidate.volume >= this.minVolume) {
        this.buffer = candidate;
      }
    }
    return this.buffer !== null;
  }

  next(): T {
    if (!this.hasNext()) {
      throw new Error('No more elements matching filter');
    }
    const result = this.buffer as T;
    this.buffer = null;
    return result;
  }

  reset(): void {
    this.iterator.reset();
    this.buffer = null;
  }
}
```

### DateRangeFilter (Extended Iterator: DateRangeFilter)

```typescript
// ── Extended Iterator: DateRangeFilter ──
// OCP: filter by date range without changing PriceHistory
class DateRangeFilter<T extends StockPrice> implements StockIterator<T> {
  private buffer: T | null = null;
  private readonly start: Date;
  private readonly end: Date;

  constructor(
    private readonly iterator: StockIterator<T>,
    startDate: string,
    endDate: string
  ) {
    this.start = new Date(startDate);
    this.end = new Date(endDate);
  }

  hasNext(): boolean {
    while (this.buffer === null && this.iterator.hasNext()) {
      const candidate = this.iterator.next();
      const candDate = new Date(candidate.date);
      if (candDate >= this.start && candDate <= this.end) {
        this.buffer = candidate;
      }
    }
    return this.buffer !== null;
  }

  next(): T {
    if (!this.hasNext()) {
      throw new Error('No more elements in date range');
    }
    const result = this.buffer as T;
    this.buffer = null;
    return result;
  }

  reset(): void {
    this.iterator.reset();
    this.buffer = null;
  }
}
```

### printPrices (Utility: print all prices from an iterator)

```typescript
// ── Utility: print all prices from an iterator ──
function printPrices<T extends StockPrice>(iterator: StockIterator<T>): void {
  while (iterator.hasNext()) {
    console.log(iterator.next().toString());
  }
}
```

### collectPrices (Utility: collect all prices from an iterator into an array)

```typescript
// ── Utility: collect all prices from an iterator into an array ──
function collectPrices<T extends StockPrice>(iterator: StockIterator<T>): T[] {
  const result: T[] = [];
  while (iterator.hasNext()) {
    result.push(iterator.next());
  }
  return result;
}
```

### Language-Specific Notes

- **Generic interfaces (`StockIterator<T>`, `Iterable<T>`)**: TypeScript's generics provide compile-time type safety for the Iterator pattern. The `PriceHistory<T>` aggregate is constrained to `T extends StockPrice`, ensuring that only valid price objects can be added to the collection. Filter iterators are also generic, preserving the element type through the composition chain.
- **`readonly` modifier on constructor parameters**: The `StockPrice` class uses TypeScript's `readonly` modifier in the constructor to create immutable data objects. This prevents accidental mutation of price data after creation and is more concise than manually declaring fields and assigning them.
- **Type-safe `IteratorResult<T>`**: The `[Symbol.iterator]()` method returns `Iterator<T>` with proper typing for `IteratorResult<T>`. The `done: true` case requires a cast because TypeScript's type system does not distinguish between `{value: T, done: false}` and `{value: undefined, done: true}` without discriminated unions.
- **Utility functions with generics**: The `printPrices<T>()` and `collectPrices<T>()` functions are generic, accepting any `StockIterator<T>` implementation. This demonstrates how TypeScript enables reusable, type-safe helper functions that work with the full Iterator hierarchy.
- **Interface segregation via two separate interfaces**: The pattern uses two interfaces: `StockIterator<T>` (three methods: `hasNext`, `next`, `reset`) and `Iterable<T>` (single method: `createIterator`). This separation follows ISP — a class only implements what it needs. The `PriceHistory<T>` class implements `Iterable<T>`, while filter iterators implement only `StockIterator<T>`.
- **To compile and run**: `npx tsc iterator.ts --target ES2020 --strict && node iterator.js` — or use `ts-node` for direct execution. The `--strict` flag enables all strict type-checking options, catching potential null/undefined issues at compile time.
