## Implementation

Below is a complete JavaScript implementation of the Iterator pattern using a stock market domain. A `PriceHistory` (Aggregate) stores `StockPrice` objects for AAPL, GOOGL, and MSFT. The `PriceHistoryIterator` (ConcreteIterator) provides sequential access to the price history without exposing the underlying array. Extended iterators (`HighVolumeFilter`, `DateRangeIterator`) compose with the base iterator to add filtering logic without modifying the aggregate. The implementation uses ES2022+ class syntax and the `Symbol.iterator` protocol for native `for...of` and spread operator support.

#### OOP Concepts Applied

1. **Abstraction** — The Iterator role is defined as a protocol (duck type): any object with `hasNext()` and `next()` methods can serve as an iterator. The consumer (`PriceHistory`) depends only on this protocol, not on concrete iteration logic.
2. **Encapsulation** — `PriceHistory` hides its internal `prices` array behind `addPrice()` and `createIterator()`. The iterator encapsulates its own `position` state, preventing external interference.
3. **Inheritance** — Not used explicitly (JavaScript favors composition for iterators), but the `Symbol.iterator` protocol inherits the language-level iteration contract.
4. **Polymorphism** — `HighVolumeFilter` and `DateRangeIterator` both expose the same `hasNext()`/`next()` interface as the base iterator. They can be used wherever a base iterator is expected without type checks.

#### SOLID Principles Applied

5. **SRP** — `PriceHistory` stores prices only. `PriceHistoryIterator` handles iteration only. Each filter has a single filtering responsibility (volume threshold or date range).
6. **OCP** — New filtering iterators (e.g., `PriceRangeFilter`, `MovingAverageIterator`) can be added by creating new classes that wrap an existing iterator. The `PriceHistory` class and base iterator are never modified.
7. **LSP** — Any iterator exposing `hasNext()` and `next()` can substitute for any other. Filtered iterators behave identically to the base iterator from the consumer's perspective.
8. **ISP** — The Iterator interface is minimal: two methods (`hasNext`, `next`). No client is forced to depend on methods it does not use.
9. **DIP** — Consumers depend on the abstract Iterator protocol (`hasNext`/`next`), not on concrete iterator classes. Iterators are injected via constructor parameters.

### StockPrice (Data class)

```javascript
// Iterator Pattern -- JavaScript -- Stock Market example
// ======================================================
// Participants: StockIterator (Iterable protocol), PriceHistory (Aggregate),
//   PriceHistoryIterator (ConcreteIterator), StockPrice (data)
// OOP: abstraction, encapsulation, polymorphism
// SOLID: SRP, OCP, LSP, ISP, DIP
// ======================================================

// ── Data class ──
class StockPrice {
  constructor(symbol, date, open, high, low, close, volume) {
    this.symbol = symbol;
    this.date = date;
    this.open = open;
    this.high = high;
    this.low = low;
    this.close = close;
    this.volume = volume;
  }

  toString() {
    return `${this.date} | ${this.symbol.padEnd(5)} O:$${this.open.toFixed(2)} H:$${this.high.toFixed(2)} L:$${this.low.toFixed(2)} C:$${this.close.toFixed(2)} Vol:${this.volume.toLocaleString()}`;
  }
}
```

### PriceHistoryIterator (Concrete Iterator)

```javascript
// ── Concrete Iterator ──
class PriceHistoryIterator {
  constructor(prices) {
    this.prices = prices;
    this.position = 0;
  }

  hasNext() {
    return this.position < this.prices.length;
  }

  next() {
    if (!this.hasNext()) {
      throw new Error('No more elements');
    }
    const price = this.prices[this.position];
    this.position++;
    return price;
  }

  reset() {
    this.position = 0;
  }
}
```

### PriceHistory (Concrete Aggregate)

```javascript
// ── Concrete Aggregate ──
class PriceHistory {
  constructor(symbol, prices = []) {
    this.symbol = symbol;
    this.prices = prices;
  }

  addPrice(price) {
    this.prices.push(price);
  }

  getPriceCount() {
    return this.prices.length;
  }

  createIterator() {
    return new PriceHistoryIterator(this.prices);
  }

  // Make the collection iterable with Symbol.iterator
  [Symbol.iterator]() {
    let position = 0;
    const prices = this.prices;
    return {
      next() {
        if (position < prices.length) {
          return { value: prices[position++], done: false };
        } else {
          return { done: true };
        }
      }
    };
  }
}
```

### HighVolumeFilter (Extended Iterator: FilteredIterator)

```javascript
// ── Extended Iterator: FilteredIterator ──
// OCP: extends behavior without modifying PriceHistory
// SRP: single responsibility -- filtering
class HighVolumeFilter {
  constructor(iterator, minVolume) {
    this.iterator = iterator;
    this.minVolume = minVolume;
    this.buffer = null;
  }

  hasNext() {
    while (this.buffer === null && this.iterator.hasNext()) {
      const candidate = this.iterator.next();
      if (candidate.volume >= this.minVolume) {
        this.buffer = candidate;
      }
    }
    return this.buffer !== null;
  }

  next() {
    if (!this.hasNext()) {
      throw new Error('No more elements matching filter');
    }
    const result = this.buffer;
    this.buffer = null;
    return result;
  }
}
```

### DateRangeIterator (Extended Iterator: RangeIterator)

```javascript
// ── Extended Iterator: RangeIterator ──
// OCP: filter by date range without changing PriceHistory
class DateRangeIterator {
  constructor(iterator, startDate, endDate) {
    this.iterator = iterator;
    this.startDate = new Date(startDate);
    this.endDate = new Date(endDate);
    this.buffer = null;
  }

  hasNext() {
    while (this.buffer === null && this.iterator.hasNext()) {
      const candidate = this.iterator.next();
      const candDate = new Date(candidate.date);
      if (candDate >= this.startDate && candDate <= this.endDate) {
        this.buffer = candidate;
      }
    }
    return this.buffer !== null;
  }

  next() {
    if (!this.hasNext()) {
      throw new Error('No more elements in date range');
    }
    const result = this.buffer;
    this.buffer = null;
    return result;
  }
}
```

### Language-Specific Notes

- **`Symbol.iterator` protocol**: JavaScript's built-in iteration protocol (`Symbol.iterator`) allows any object to be iterable with `for...of`, the spread operator (`[...obj]`), and destructuring. The `PriceHistory` class implements both a custom iterator (via `createIterator()`) and the native protocol (via `[Symbol.iterator]()`), demonstrating both approaches.
- **Duck-typed Iterator interface**: JavaScript has no formal interface system. The iterator protocol is defined by convention: any object with `next()` returning `{value, done}` qualifies. Filtered iterators (`HighVolumeFilter`, `DateRangeIterator`) follow the same convention as `PriceHistoryIterator` through structural typing.
- **Composition over inheritance for filtering**: Rather than subclassing `PriceHistoryIterator`, the filtered iterators wrap an existing iterator instance (Decorator-like composition). This is idiomatic JavaScript and follows the OCP without deep inheritance hierarchies.
- **Lazy evaluation**: The filtered iterators use a buffering strategy that advances the underlying iterator only when the consumer calls `hasNext()`/`next()`. No additional array storage is created — each element is inspected on demand, which is memory-efficient for large datasets.
- **No external dependencies**: The implementation uses only ES2022+ built-in features — no libraries, no frameworks, no Node.js-specific APIs. The `toLocaleString()` method on `volume` is a standard `Number` method for locale-aware formatting.
- **`for...of` vs manual iteration**: The code demonstrates both approaches. Manual iteration (via `while (iterator.hasNext())`) gives the consumer control over traversal flow, while `for...of` provides cleaner syntax for the common case of visiting every element. The custom iterator pattern is useful when you need multiple simultaneous traversals over the same collection — each iterator maintains its own position.
