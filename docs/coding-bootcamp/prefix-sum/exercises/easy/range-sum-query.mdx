# Range Sum Query

**Difficulty:** Easy

Given an integer array `nums`, implement a class that supports two operations:

1. **Constructor:** Initialize the object with the integer array `nums`.
2. **`sumRange(left, right)`:** Return the sum of the elements of `nums` between indices `left` and `right` (inclusive).

## Example 1

**Input:**
```
nums = [1, 2, 3, 4, 5]
sumRange(0, 2) → 6
sumRange(2, 4) → 12
sumRange(0, 4) → 15
```

**Output:** `6`, `12`, `15`

**Explanation:** 1 + 2 + 3 = 6, 3 + 4 + 5 = 12, 1 + 2 + 3 + 4 + 5 = 15.

## Input Specification

- First line: integer `n` (length of `nums`)
- Second line: `n` space-separated integers `nums[i]`
- Third line: integer `q` (number of queries)
- Next `q` lines: each contains two integers `left` and `right`

## Output Specification

Print `q` lines, each containing the sum of `nums[left..right]` for the corresponding query.

## Constraints

- `1 <= n <= 10^5`
- `-10^4 <= nums[i] <= 10^4`
- `1 <= q <= 10^5`
- `0 <= left <= right < n`

## Hints

<details>
<summary>Hint 1</summary>

Think about how to compute `sumRange` in O(1) time. What if you precompute cumulative sums?

</details>

<details>
<summary>Hint 2</summary>

Precompute a prefix sum array where `prefix[i]` is the sum of `nums[0..i]`. Then `sumRange(l, r) = prefix[r] - prefix[l-1]`.

</details>

<details>
<summary>Hint 3</summary>

Handle the edge case where `l == 0` — then the sum is just `prefix[r]`.

</details>

</ProblemEditor>
