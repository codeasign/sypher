# Subarray Sum Equals K

**Difficulty:** Medium

Given an array of integers `nums` and an integer `k`, return the **total number of subarrays** whose sum equals `k`.

A subarray is a contiguous non-empty sequence of elements within an array.

## Example 1

**Input:** `nums = [1, 1, 1]`, `k = 2`

**Output:** `2`

**Explanation:** The subarrays `[1, 1]` (indices 0-1) and `[1, 1]` (indices 1-2) both sum to 2.

## Example 2

**Input:** `nums = [1, 2, 3]`, `k = 3`

**Output:** `2`

**Explanation:** The subarrays `[1, 2]` (indices 0-1) and `[3]` (index 2) both sum to 3.

## Input Specification

- First line: integer `n` (length of `nums`)
- Second line: `n` space-separated integers `nums[i]`
- Third line: integer `k`

## Output Specification

Print a single integer — the count of subarrays whose sum equals `k`.

## Constraints

- `1 <= n <= 2 * 10^4`
- `-1000 <= nums[i] <= 1000`
- `-10^7 <= k <= 10^7`

## Hints

<details>
<summary>Hint 1</summary>

A brute-force approach checks every subarray in O(n^2) — too slow for large inputs. Can you use prefix sums?

</details>

<details>
<summary>Hint 2</summary>

If `prefix[j] - prefix[i] == k`, then the subarray `(i+1..j)` has sum `k`. Use a hash map to store prefix sum frequencies.

</details>

<details>
<summary>Hint 3</summary>

As you iterate, check if `current_prefix - k` exists in the hash map. Add its frequency to the count, then update the map with the current prefix sum.

</details>

</ProblemEditor>
