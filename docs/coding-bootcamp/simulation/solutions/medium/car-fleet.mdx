# Car Fleet — Solution

## Problem Overview

Given n cars with positions and speeds heading to a target, determine how many car fleets will arrive. Cars cannot pass; they form a fleet when a faster car catches up from behind.

## Brute Force Approach

Simulate the journey step by step in time increments. Advance all cars, check for collisions, merge into fleets. This is too slow for large constraints.

**Complexity**: O(target * n) time, O(n) space

## Optimal Approach

1. Pair each position with its speed and sort by position descending (closest to target first)
2. Compute each car's time to reach the target: `(target - position) / speed`
3. Iterate from closest to farthest: if a car's time is greater than the current max time, it forms a new fleet; otherwise it catches up to the fleet ahead

The key insight: a car behind the current fleet will catch it if its time-to-target is less than or equal to the fleet's time. We only count a new fleet when a car takes strictly longer.

## Deep Dive

Sorting by position ensures we process cars from the destination outward. By tracking the maximum time seen so far, we know which cars merge into existing fleets. A car with time  maxTime will catch the fleet ahead; a car with time  maxTime starts a new fleet.

def carFleet(target, position, speed):
    cars = sorted(zip(position, speed), reverse=True)
    fleets = 0
    max_time = 0
    for pos, spd in cars:
        time = (target - pos) / spd
        if time > max_time:
            fleets += 1
            max_time = time
    return fleets

class Solution {
    public int carFleet(int target, int[] position, int[] speed) {
        int n = position.length;
        int[][] cars = new int[n][2];
        for (int i = 0; i < n; i++) {
            cars[i][0] = position[i];
            cars[i][1] = speed[i];
        }
        Arrays.sort(cars, (a, b) -> b[0] - a[0]);
        int fleets = 0;
        double maxTime = 0;
        for (int[] car : cars) {
            double time = (double)(target - car[0]) / car[1];
            if (time  maxTime) {
                fleets++;
                maxTime = time;
            }
        }
        return fleets;
    }
}

        vector<pair<int,int>> cars;

fn car_fleet(target: i32, position: Vec<i32>, speed: Vec<i32>) -> i32 {

int carFleet(int target, int* position, int positionSize, int* speed, int speedSize) {
    int n = positionSize;
    int** cars = malloc(n * sizeof(int*));
    for (int i = 0; i < n; i++) {
        cars[i] = malloc(2 * sizeof(int));
        cars[i][0] = position[i];
        cars[i][1] = speed[i];
    }
    for (int i = 0; i < n; i++)
        for (int j = i + 1; j < n; j++)
            if (cars[j][0] > cars[i][0]) {
                int* tmp = cars[i]; cars[i] = cars[j]; cars[j] = tmp;
            }
    int fleets = 0;
    double maxTime = 0;
    for (int i = 0; i < n; i++) {
        double time = (double)(target - cars[i][0]) / cars[i][1];
        if (time  maxTime) { fleets++; maxTime = time; }
    }
    for (int i = 0; i < n; i++) free(cars[i]);
    free(cars);
    return fleets;
}

type Car struct { pos, spd int }

func carFleet(target int, position []int, speed []int) int {
    n := len(position)
    cars := make([]Car, n)
    for i := 0; i < n; i++ { cars[i] = Car{position[i], speed[i]} }
    sort.Slice(cars, func(i, j int) bool { return cars[i].pos > cars[j].pos })
    fleets := 0
    maxTime := 0.0
    for _, c := range cars {
        time := float64(target - c.pos) / float64(c.spd)
        if time  maxTime { fleets++; maxTime = time }
    }
    return fleets
}

## Key Takeaways

- Sorting by position and comparing times is a common simulation pattern
- The monotonic stack-like pattern (tracking max time) simplifies collision logic
- This problem tests the ability to model a physical process mathematically
