# Hash Maps / Hash Sets

A **hash map** (also called a hash table or dictionary) is a data structure that stores key-value pairs and offers near-instantaneous lookups, insertions, and deletions. A **hash set** is a close relative that stores only keys (no associated values) and is used when you simply need to track membership.

## How Hashing Works

At its core, a hash map uses a **hash function** to convert a key into an integer index within an internal array. The key's value is stored at that index. When two keys hash to the same index (a **collision**), the map resolves it using either **chaining** (linked list at each bucket) or **open addressing** (probing for the next free slot).

## Time Complexity

| Operation | Average Case | Worst Case |
|-----------|-------------|------------|
| get(key) | O(1) | O(n) |
| put(key, value) | O(1) | O(n) |
| delete(key) | O(1) | O(n) |
| containsKey(key) | O(1) | O(n) |

The worst case occurs when many keys collide into the same bucket, degrading the map into a linked list. A good hash function and a **load factor** threshold (typically 0.75) that triggers rehashing keep the average case O(1).

## Common Applications

- **Frequency counting** — Count occurrences of elements in a collection.
- **Deduplication** — Use a hash set to track seen elements and skip duplicates.
- **Fast lookup** — Store computed values in a cache (memoization) to avoid recomputation.
- **Two-sum / pair problems** — Check for complements in a single pass.
- **Grouping** — Partition items by a hashable key (e.g., group anagrams).
- **Index tracking** — Map a value to its position in an array.

## Worked Example: Two Sum

**Problem:** Given an array `nums = [2, 7, 11, 15]` and a target `target = 9`, return the indices of the two numbers that add up to the target.

**Approach (one-pass hash map):**

1. Create an empty hash map `seen = {}`.
2. Loop through the array with index `i` and value `num`.
3. Compute `complement = target - num`.
4. If `complement` is in `seen`, return `[seen[complement], i]`.
5. Otherwise, store `seen[num] = i`.

```
i=0, num=2, complement=7, 7 not in {} → store {2:0}
i=1, num=7, complement=2, 2 is in {2:0} → return [0, 1]
```

**Result:** `[0, 1]` because `nums[0] + nums[1] = 2 + 7 = 9`.

## Choosing Between HashMap and HashSet

| Use Case | Data Structure |
|----------|---------------|
| Count frequencies | `HashMap<T, Integer>` |
| Track seen items | `HashSet<T>` |
| Cache computed results | `HashMap<K, V>` |
| Remove duplicates | `HashSet<T>` |
| Build adjacency lists | `HashMap<K, List<V>>` |
