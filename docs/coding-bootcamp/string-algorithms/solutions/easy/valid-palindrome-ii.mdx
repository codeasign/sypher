# Valid Palindrome II — Solution

## Problem Overview

Given a string `s`, return `true` if it can become a palindrome by deleting **at most one** character.

## Brute Force Approach

**Idea:** Try deleting each character one at a time and check if the resulting string is a palindrome.

**Algorithm:**
1. For each index `i` from 0 to `n-1`:
   - Create a new string by removing `s[i]`
   - Check if the resulting string is a palindrome
2. Also check if the original string is already a palindrome
3. Return `true` if any check passes

**Complexity Analysis:**
- **Time:** O(n^2) — for each of n deletions we check palindrome in O(n)
- **Space:** O(n) — we create a new string for each deletion

**Why this could be better:** This is too slow for large inputs. We can solve this in O(n) time and O(1) space using two pointers.

## Optimal Approach: Two Pointers with Tolerance

**Idea:** Use two pointers starting at opposite ends. When a mismatch is found, try skipping **either** the left character or the right character, and check if the remaining substring is a palindrome.

**Algorithm:**
1. Initialize `left = 0` and `right = len(s) - 1`
2. While `left < right`:
   - If `s[left] == s[right]`, move both inward
   - If they differ:
     - Check if `s[left+1..right]` is a palindrome (skip left)
     - Check if `s[left..right-1]` is a palindrome (skip right)
     - Return `true` if either sub-palindrome check passes
3. Return `true` (the string was already a palindrome)

**Why This Works:** A palindrome must have matching outer characters. When we find a mismatch, at most one character can be deleted — so one of the two substrings (leaving out the left or right character) must be a palindrome.

**Complexity Analysis:**
- **Time:** O(n) — each character is visited at most twice (once in main loop, once in sub-palindrome check)
- **Space:** O(1) — only a few integer variables

class Solution {
    public boolean validPalindrome(String s) {
        int left = 0, right = s.length() - 1;  // Two pointers at ends
        while (left < right) {                  // Scan until pointers meet
            if (s.charAt(left) != s.charAt(right)) {
                // Try skipping left character OR skipping right character
                return isPalindrome(s, left + 1, right) ||
                       isPalindrome(s, left, right - 1);
            }
            left++;   // Characters match, move inward
            right--;
        }
        return true;  // Already a palindrome
    }

    // Helper: check if substring s[l..r] is a palindrome
    private boolean isPalindrome(String s, int l, int r) {
        while (l < r) {
            if (s.charAt(l) != s.charAt(r)) return false;
            l++;
            r--;
        }
        return true;
    }
}

fn valid_palindrome(s: &str) -> bool {
    let chars: Vec<char> = s.chars().collect();  // Collect for O(1) indexing
    let mut left = 0;
    let mut right = chars.len().saturating_sub(1);

    while left < right {
        if chars[left] != chars[right] {
            // Mismatch: try skipping left or right character
            return is_palindrome(&chars, left + 1, right) ||
                   is_palindrome(&chars, left, right - 1);
        }
        left += 1;
        right = right.saturating_sub(1);
    }
    true
}

fn is_palindrome(chars: &[char], mut l: usize, mut r: usize) -> bool {
    while l < r {
        if chars[l] != chars[r] {
            return false;
        }
        l += 1;
        r = r.saturating_sub(1);
    }
    true
}

## Key Takeaways

- **Two pointers with a tolerance counter** is the standard pattern for "at most one deletion" palindrome problems
- When a mismatch occurs, you **branch** into two possibilities — skip left or skip right
- The helper function isolates the palindrome-checking logic and avoids code duplication
- This runs in **O(n) time and O(1) space** — optimal for this problem
