# String Compression — Solution

## Problem Overview

Compress a character array in-place by replacing consecutive repeating characters with the character followed by its count. If the count is 1, omit it. Return the new length of the compressed array.

## Optimal Approach: Three Pointers

**Idea:** Use a read pointer to scan the array, a write pointer to track where to write compressed output, and an anchor pointer to mark the start of each group.

**Algorithm:**
1. Initialize `write = 0` (where to write next) and `anchor = 0` (start of current group)
2. For `read` from 0 to `n-1`:
   - If `read == n-1` or `chars[read] != chars[read+1]` (group ends):
     - Write `chars[anchor]` at position `write`, increment `write`
     - If group length > 1: write the count as individual digits
     - Set `anchor = read + 1` (start of next group)
3. Return `write` (new length)

**Complexity Analysis:**
- **Time:** O(n) — each character is visited exactly once
- **Space:** O(1) — modifications are done in-place, only integer variables

**Why this works:** The write pointer never outpaces the read pointer (the compressed string is always shorter than or equal in length to the original), so we safely write in-place without needing a buffer.

class Solution {
    public int compress(char[] chars) {
        int write = 0;   // Position to write next compressed character
        int anchor = 0;  // Start index of current group

        for (int read = 0; read < chars.length; read++) {
            // Group ends at last index or when next character differs
            if (read == chars.length - 1 || chars[read] != chars[read + 1]) {
                chars[write++] = chars[anchor];  // Write the character
                int count = read - anchor + 1;    // Size of this group
                if (count > 1) {
                    // Write each digit of the count
                    for (char digit : Integer.toString(count).toCharArray()) {
                        chars[write++] = digit;
                    }
                }
                anchor = read + 1;  // Move anchor to start of next group
            }
        }
        return write;  // New length of compressed array
    }
}

fn compress(chars: &mut Vec<char>) -> i32 {

    let n = chars.len();

                    chars[write] = digit;
                    write += 1;
                }
            }
            anchor = read + 1;  // Move to next group
        }
    }
    write as i32  // New compressed length
}

## Key Takeaways

- **Three pointers** (read, write, anchor) handle in-place compression efficiently
- The read pointer scans; the write pointer fills; the anchor marks group start
- Group length detection happens when the next character **differs** or at the **end** of the array
- Multi-digit counts are written as individual characters
- **O(n) time, O(1) space** — optimal for in-place compression
