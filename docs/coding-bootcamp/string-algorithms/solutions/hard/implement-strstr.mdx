# Implement strStr (KMP) — Solution

## Problem Overview

Find the first occurrence of `needle` in `haystack`. Return the index, or `-1` if not found. The optimal solution uses the **KMP (Knuth-Morris-Pratt)** algorithm — O(n+m) time.

## Approach 1: Naive (Sliding Window)

**Idea:** Slide the needle across the haystack one position at a time, comparing characters.

**Algorithm:**
1. For each position `i` from 0 to `n - m`:
   - Compare `haystack[i..i+m-1]` with `needle[0..m-1]`
   - If all characters match, return `i`
2. Return `-1`

**Complexity:**
- **Time:** O(n*m) — worst case when many partial matches
- **Space:** O(1)

## Approach 2: KMP (Optimal)

**Idea:** Preprocess the needle to build an **LPS (Longest Prefix Suffix)** array. When a mismatch occurs, use the LPS array to skip characters that have already been matched, avoiding redundant comparisons.

**Building the LPS Array:**
- `lps[i]` = length of the longest proper prefix of `needle[0..i]` that is also a suffix
- Example: `"abab"` → LPS = `[0, 0, 1, 2]`
  - `"a"`: no proper prefix = suffix → 0
  - `"ab"`: no match → 0
  - `"aba"`: prefix `"a"` = suffix `"a"` → 1
  - `"abab"`: prefix `"ab"` = suffix `"ab"` → 2

**Matching with LPS:**
1. Initialize `i = 0` (haystack index), `j = 0` (needle index)
2. While `i < n`:
   - If `haystack[i] == needle[j]`, increment both
   - If `j == m`, found a match at `i - j`. Set `j = lps[j-1]` to continue searching
   - If mismatch and `j > 0`, set `j = lps[j-1]` (don't move `i`)
   - If mismatch and `j == 0`, increment `i`
3. Return `-1`

**Complexity Analysis:**
- **Time:** O(n + m) — each character in haystack and needle is processed at most twice
- **Space:** O(m) — for the LPS array

class Solution {
    public int strStr(String haystack, String needle) {
        if (needle.isEmpty()) return 0;  // Empty needle is always found at start

        int n = haystack.length(), m = needle.length();
        if (m > n) return -1;  // Needle longer than haystack — impossible

        // Step 1: Build LPS (Longest Prefix Suffix) array
        int[] lps = new int[m];
        int len = 0;  // Length of previous longest prefix suffix
        int i = 1;
        while (i < m) {
            if (needle.charAt(i) == needle.charAt(len)) {
                len++;
                lps[i] = len;
                i++;
            } else {
                if (len != 0) {
                    len = lps[len - 1];  // Fall back to previous LPS
                } else {
                    lps[i] = 0;  // No match
                    i++;
                }
            }
        }

        // Step 2: Match using LPS
        i = 0;  // Index for haystack
        int j = 0;  // Index for needle
        while (i < n) {
            if (haystack.charAt(i) == needle.charAt(j)) {
                i++;
                j++;
                if (j == m) return i - j;  // Full match found
            } else if (j > 0) {
                j = lps[j - 1];  // Use LPS to skip ahead
            } else {
                i++;  // No match at j=0, advance haystack
            }
        }
        return -1;  // Not found
    }
}

## Key Takeaways

- **KMP** eliminates redundant comparisons by preprocessing the pattern into an LPS array
- The LPS array tells us the **longest proper prefix that is also a suffix** — this is the key insight
- On a mismatch, we shift the pattern so that the already-matched suffix aligns with the prefix, rather than restarting from scratch
- **O(n+m) time** is the theoretical optimum for single-pattern matching
- The naive approach is fine for small inputs, but KMP is essential for large strings (up to 10^5 characters)
