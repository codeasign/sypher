# Longest Repeating Character Replacement

**Difficulty:** Medium

Given a string `s` consisting of only uppercase English letters and an integer `k`, you may replace any character in the string with any other uppercase English letter at most `k` times total. Return the **length of the longest substring** that contains only one unique letter after performing at most `k` replacements.

## Example 1

**Input:** `s = "ABAB"`, `k = 2`
**Output:** `4`
**Explanation:** Replace both `B`s with `A` (or both `A`s with `B`). The entire string is one repeated character.

## Example 2

**Input:** `s = "AABABBA"`, `k = 1`
**Output:** `4`
**Explanation:** Replace one character to get `"AABBBBA"` or `"AAAABBA"` — the longest same-letter window is `4`.

## Example 3

**Input:** `s = "AAAB"`, `k = 0`
**Output:** `3`
**Explanation:** With no replacements allowed, the longest run of the same letter is `"AAA"`.

## Input Specification

- The first line contains the string `s` (uppercase English letters only).
- The second line contains the integer `k`.

## Output Specification

Print a single integer — the length of the longest substring of one repeated character achievable with at most `k` replacements.

## Constraints

- `1 <= s.length <= 10^5`
- `0 <= k <= s.length`
- `s` consists of uppercase English letters only.

## Hints

<details>
<summary>Hint 1</summary>

Think about what makes a window valid. If the most frequent character in the window appears `maxFreq` times, how many replacements do you need to make the whole window uniform?

</details>

<details>
<summary>Hint 2</summary>

A window of length `len` is valid if `len - maxFreq <= k`. Use a sliding window: expand `right` freely; when the window becomes invalid, advance `left` by one.

</details>

<details>
<summary>Hint 3</summary>

You do not need to recompute `maxFreq` when shrinking — keeping the historical maximum is enough. The window length can only grow when a new character beats the old `maxFreq`, which is the only time the answer can increase.

</details>
