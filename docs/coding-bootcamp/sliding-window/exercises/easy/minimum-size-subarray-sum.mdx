# Minimum Size Subarray Sum

**Difficulty:** Easy

Given an array of positive integers `nums` and a positive integer `target`, return the **minimal length** of a contiguous subarray whose sum is greater than or equal to `target`. If no such subarray exists, return `0`.

## Example 1

**Input:** `target = 7, nums = [2, 3, 1, 2, 4, 3]`
**Output:** `2`
**Explanation:** The subarray `[4, 3]` has sum `7` and length `2`, which is the smallest possible.

## Example 2

**Input:** `target = 4, nums = [1, 4, 4]`
**Output:** `1`
**Explanation:** The subarray `[4]` has sum `4 >= 4` with length `1`.

## Example 3

**Input:** `target = 11, nums = [1, 1, 1, 1, 1]`
**Output:** `0`
**Explanation:** The total sum is `5`, which is less than `11`. No valid subarray exists.

## Input Specification

The first line contains a single integer `target`.
The second line contains space-separated positive integers representing `nums`.

## Output Specification

Print a single integer: the minimal length of a subarray with sum >= target, or `0` if no such subarray exists.

## Constraints

- `1 <= target <= 10^9`
- `1 <= nums.length <= 10^5`
- `1 <= nums[i] <= 10^4`

## Hints

<details>
<summary>Hint 1</summary>

A brute-force approach checks every subarray — O(n^2) time. Think about how to avoid recomputing sums from scratch each time. Can you maintain a running window sum and adjust it incrementally?

</details>

<details>
<summary>Hint 2</summary>

Use a sliding window with two pointers (`left` and `right`). Expand the window by advancing `right` and adding `nums[right]` to the current sum. When the sum is >= target, try to shrink the window from the left.

</details>

<details>
<summary>Hint 3</summary>

While the current window sum is >= target, record the window length as a candidate answer, then subtract `nums[left]` and advance `left`. Keep track of the minimum length found. This runs in O(n) because each element is added and removed at most once.

</details>

</ProblemEditor>
