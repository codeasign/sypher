# Minimum Window Substring — Solution

## Problem Overview

Given two strings `s` and `t`, find the minimum (contiguous) substring of `s` that contains all characters of `t` (including duplicates). If no such substring exists, return an empty string.

## Brute Force Approach

**Idea:** For every possible starting and ending position in `s`, check if the substring contains all characters of `t`. Track the minimum length among all valid substrings.

**Algorithm:**
1. Initialize `minLen = INF`, `result = ""`
2. For each `i` from 0 to n-1:
   - For each `j` from i to n-1:
     - Extract substring `s[i:j+1]`
     - Check if it contains all characters of `t` (using frequency counts)
     - If valid and `j - i + 1 < minLen`:
       - Update `minLen` and `result`
3. Return `result`

**Complexity Analysis:**
- **Time:** O(n² * m) — O(n²) substrings, each checked against `t` of length m
- **Space:** O(m) — frequency array for counting characters of `t`

**Why this is suboptimal:** The nested loops and repeated frequency checks make this approach infeasible for n up to 10⁵. The algorithm rechecks the same characters many times across overlapping substrings.

## Optimal Approach: Sliding Window with Two Frequency Counters

**Idea:** Use a sliding window with two pointers (left and right). Expand the right pointer to include more characters until the window satisfies all requirements of `t`. Then contract from the left to find the minimum valid window. Use frequency counters to track character counts and a `matched` counter to track how many characters from `t` have been satisfied.

**Algorithm:**
1. Create a frequency map `need` for characters in `t`
2. Initialize `have = 0` (number of unique characters in `t` whose frequency is satisfied), `needCount = len(need)` (unique characters in `t`)
3. Initialize `left = 0`, `minLen = INF`, `minStart = 0`
4. For `right` from 0 to n-1:
   - Add `s[right]` to the window frequency map `window`
   - If `s[right]` is in `need` and `window[s[right]] == need[s[right]]`:
     - Increment `have`
   - While `have == needCount` (window is valid):
     - Update `minLen` and `minStart` if current window is smaller
     - Remove `s[left]` from the window
     - If `s[left]` is in `need` and `window[s[left]] < need[s[left]]`:
       - Decrement `have`
     - Increment `left` (shrink window from the left)
5. If `minLen` is still INF, return `""`; otherwise return `s[minStart:minStart + minLen]`

**Why This Works:** The algorithm maintains the invariant that the window between `left` and `right` always contains the most recent valid window. When the window becomes valid (contains all characters of `t`), we try to shrink it from the left to find the minimum. The `have` counter tracks how many of the required characters are present in the right quantity, avoiding O(m) checks at each step.

**Complexity Analysis:**
- **Time:** O(n + m) — each character in `s` is visited at most twice (once by `right`, once by `left`), and building the `need` map takes O(m)
- **Space:** O(1) — the frequency maps have at most 52 entries (uppercase and lowercase letters), so O(1) space

    window[C] = 1, need[C] = 1, window[C] < need[C]? Yes (0 < 1)

### Edge Cases to Consider

- **t longer than s**: The `have` counter will never reach `needCount`, so `minLen` stays INF, and we return `""`.
- **Exact match**: `s = t` — the algorithm finds the full string as the only valid window.
- **Single character**: `t = "a"` — the algorithm finds the first occurrence of `'a'` in `s`.
- **Repeated characters in t**: `t = "aa"` — the window must contain at least two `'a'` characters. The `need` map has `{A:2}`.
- **Characters in t not in s**: The `have` counter never reaches `needCount`, so we return `""`.
- **Multiple valid windows**: The algorithm always finds the minimum by checking every valid window and shrinking aggressively.

### Why the Two-Counter Approach Is Efficient

The `have` counter tracks how many *unique* characters from `t` are currently satisfied in the window. This avoids scanning the entire frequency map at each step. When `have == needCount`, we know the window is valid. The counters are updated incrementally as characters enter and leave the window, making the algorithm O(n) overall.

class Solution {
    public String minWindow(String s, String t) {
        if (s.length() < t.length()) return "";  // Impossible to have a valid window

        int[] need = new int[128];  // Frequency of characters needed from t
        for (char c : t.toCharArray()) need[c]++;

        int[] window = new int[128];  // Frequency of characters in the current window
        int have = 0;          // Number of unique characters satisfied
        int needCount = 0;     // Number of unique characters in t
        for (int count : need) if (count > 0) needCount++;

        int left = 0;          // Left boundary of the window
        int minLen = Integer.MAX_VALUE;  // Length of the minimum window found
        int minStart = 0;      // Start index of the minimum window

        for (int right = 0; right < s.length(); right++) {
            char c = s.charAt(right);
            window[c]++;  // Add the character to the window

            // If this character is needed and we now have enough, increment have
            if (need[c] > 0 && window[c] == need[c]) {
                have++;
            }

            // Shrink the window from the left while it remains valid
            while (have == needCount) {
                // Update the minimum window if this one is smaller
                if (right - left + 1 < minLen) {
                    minLen = right - left + 1;
                    minStart = left;
                }
                // Remove the leftmost character from the window
                char leftChar = s.charAt(left);
                window[leftChar]--;
                if (need[leftChar] > 0 && window[leftChar] < need[leftChar]) {
                    have--;  // No longer satisfied for this character
                }
                left++;  // Move the left boundary inward
            }
        }

        return minLen == Integer.MAX_VALUE ? "" : s.substring(minStart, minStart + minLen);
    }
}

def min_window(s: str, t: str) -> str:
    if len(s) < len(t):
        return ""  # No valid window possible

    need = {}  # Frequency of required characters
    for c in t:
        need[c] = need.get(c, 0) + 1

    window = {}  # Frequency of characters in the current window
    have = 0          # Number of unique characters satisfied
    need_count = len(need)  # Number of unique characters in t

    left = 0          # Left boundary of the window
    min_len = float('inf')  # Length of the minimum window
    min_start = 0     # Start index of the minimum window

    for right, c in enumerate(s):
        # Expand the window to include the current character
        window[c] = window.get(c, 0) + 1

        # If this character satisfies a requirement, increment have
        if c in need and window[c] == need[c]:
            have += 1

        # While the window is valid, try to shrink it from the left
        while have == need_count:
            # Update the minimum window if this one is smaller
            if right - left + 1 < min_len:
                min_len = right - left + 1
                min_start = left

            # Remove the leftmost character from the window
            left_char = s[left]
            window[left_char] -= 1
            if left_char in need and window[left_char] < need[left_char]:
                have -= 1  # No longer satisfied for this character
            left += 1  # Shrink the window from the left

    return "" if min_len == float('inf') else s[min_start:min_start + min_len]

fn min_window(s: &str, t: &str) -> String {
    if s.len() < t.len() {
        return String::new();  // No valid window possible
    }

    let s_bytes = s.as_bytes();
    let mut need = [0i32; 128];  // Frequency of required characters
    for &b in t.as_bytes() {
        need[b as usize] += 1;
    }

    let mut window = [0i32; 128];  // Frequency in the current window
    let mut have = 0;              // Number of unique characters satisfied
    let mut need_count = 0;        // Number of unique characters in t
    for &count in need.iter() {
        if count > 0 { need_count += 1; }
    }

    let mut left = 0;          // Left boundary of the window
    let mut min_len = usize::MAX;  // Length of the minimum window
    let mut min_start = 0;     // Start index of the minimum window

    for right in 0..s.len() {
        let c = s_bytes[right] as usize;
        window[c] += 1;  // Expand the window

        // If this character satisfies a requirement, increment have
        if need[c] > 0 && window[c] == need[c] {
            have += 1;
        }

        // While the window is valid, try to shrink it from the left
        while have == need_count {
            // Update the minimum window if this one is smaller
            if right - left + 1 < min_len {
                min_len = right - left + 1;
                min_start = left;
            }
            // Remove the leftmost character from the window
            let left_char = s_bytes[left] as usize;
            window[left_char] -= 1;
            if need[left_char] > 0 && window[left_char] < need[left_char] {
                have -= 1;  // No longer satisfied for this character
            }
            left += 1;  // Shrink from the left
        }
    }

    if min_len == usize::MAX {
        String::new()
    } else {
        s[min_start..min_start + min_len].to_string()
    }
}

    int needCount = 0;      // Number of unique characters in t

        // If this character satisfies a requirement, increment have

            have++;
        }

            // Update the minimum window if this one is smaller
            if (right - left + 1 < minLen) {
                minLen = right - left + 1;
                minStart = left;
            }
            // Remove the leftmost character from the window
            int leftChar = (int)s[left];
            window[leftChar]--;
            if (need[leftChar] > 0 && window[leftChar] < need[leftChar]) {
                have--;  // No longer satisfied for this character
            }
            left++;  // Shrink the window from the left
        }

    // Build the result string
    char* result = (char*)malloc((minLen + 1) * sizeof(char));
    if (minLen == 100005) {
        result[0] = '\0';  // No valid window found
    } else {
        strncpy(result, s + minStart, minLen);
        result[minLen] = '\0';
    }
    return result;
}

## Key Takeaways

- **Sliding window with frequency counters** is the standard pattern for "minimum window substring" problems
- The **two-counter approach** (`have` vs `needCount`) avoids scanning the entire frequency map at each step, keeping the algorithm O(n)
- First **expand** the window to make it valid, then **contract** from the left to find the minimum
- The algorithm is **O(n + m) time and O(1) space** — optimal for this problem
- Handle the case where `t` has duplicate characters correctly by using frequency counts, not just a set
- Use a fixed-size array (128 for ASCII) instead of a hash map for better performance when the character set is known
