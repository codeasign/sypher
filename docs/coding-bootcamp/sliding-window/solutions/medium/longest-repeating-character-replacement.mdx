# Longest Repeating Character Replacement — Solution

## Problem Overview

Given a string `s` consisting of only uppercase English letters and an integer `k`, you may replace any character in the string with any other uppercase English letter at most `k` times total. Return the length of the longest substring that contains only one unique letter after performing at most `k` replacements.

## Brute Force Approach

**Idea:** Check every possible substring. For each substring, count character frequencies. If the substring length minus the maximum frequency is ≤ k, it can be made uniform with at most k replacements.

**Algorithm:**
1. Initialize `maxLen = 0`
2. For each starting index `i` from 0 to n-1:
   - Initialize a frequency array for 26 letters
   - For each ending index `j` from i to n-1:
     - Increment frequency of `s[j]`
     - Let `maxFreq` = the highest frequency in the current window
     - If `(j - i + 1) - maxFreq <= k`, update `maxLen`
     - Otherwise, break (further expansion only increases the gap)
3. Return `maxLen`

**Complexity Analysis:**
- **Time:** O(n²) — two nested loops with frequency tracking
- **Space:** O(1) — fixed frequency array of size 26

**Why this is suboptimal:** The O(n²) approach is too slow for n up to 10⁵. The inner loop recomputes frequencies from scratch or updates incrementally but still explores every starting position.

## Optimal Approach: Sliding Window with Frequency Map

**Idea:** Maintain a sliding window and a frequency count of characters within it. Track the maximum frequency of any single character. A window of length `len` is valid if `len - maxFreq <= k`. When the window becomes invalid, shrink it from the left by one position.

**Algorithm:**
1. Initialize `freq = {}` (frequency map), `left = 0`, `maxFreq = 0`
2. For `right` from 0 to n-1:
   - Increment `freq[s[right]]`
   - Update `maxFreq = max(maxFreq, freq[s[right]])`
   - If `(right - left + 1) - maxFreq > k`:
     - Decrement `freq[s[left]]`
     - Increment `left` (shrink window from the left)
3. Return `n - left` (length of the longest valid window)

**Why This Works:** The key insight is that `maxFreq` never decreases — it only increases when a character beats the previous maximum. This is valid because if we could achieve a certain window length with a lower maxFreq, we can always achieve it with a higher one. The window length only grows from the right; when invalid, we shrink from the left by one. This ensures we don't miss a longer valid window.

**Why `maxFreq` Never Needs to Decrease:** Even if the most frequent character changes after shrinking, using the historical maximum is safe — it means there exists a valid window of that length, and we only care about the maximum possible length.

**Complexity Analysis:**
- **Time:** O(n) — each character is visited at most twice
- **Space:** O(26) = O(1) — fixed frequency array for uppercase letters

class Solution {
    public int characterReplacement(String s, int k) {
        int[] freq = new int[26];  // Frequency of each uppercase letter in the window
        int left = 0;              // Left boundary of the sliding window
        int maxFreq = 0;           // Highest frequency of any character in the window

        for (int right = 0; right < s.length(); right++) {
            freq[s.charAt(right) - 'A']++;  // Add the current character to the window
            maxFreq = Math.max(maxFreq, freq[s.charAt(right) - 'A']);  // Update max frequency

            // If the window is invalid, shrink it from the left
            if (right - left + 1 - maxFreq > k) {
                freq[s.charAt(left) - 'A']--;  // Remove the leftmost character
                left++;                        // Move the left boundary inward
            }
        }

        // Return the length of the longest valid window
        return s.length() - left;
    }
}

    let s_bytes = s.as_bytes();
    let mut freq = [0usize; 26];  // Frequency of each uppercase letter

    let mut max_freq = 0;         // Highest frequency of any character in the window

        if freq[idx] > max_freq {
            max_freq = freq[idx];  // Update max frequency

        if (right - left + 1) - max_freq > k {
            let left_idx = (s_bytes[left] - b'A') as usize;
            freq[left_idx] -= 1;   // Remove the leftmost character
            left += 1;             // Move the left boundary inward
        }
    }

    // Return the length of the longest valid window
    s_bytes.len() - left
}

## Key Takeaways

- **Sliding window with frequency tracking** is the optimal pattern for substring replacement problems
- The key optimization is that **`maxFreq` never needs to decrease** — the historical maximum is sufficient to find the answer
- The approach is **O(n) time and O(1) space** — optimal for this problem
- The final answer is `n - left` because the window only grows from the right and shrinks minimally from the left
