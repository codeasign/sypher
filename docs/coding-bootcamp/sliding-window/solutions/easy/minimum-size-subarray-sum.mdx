# Minimum Size Subarray Sum — Solution

## Problem Overview

Given an array of positive integers `nums` and a positive integer `target`, return the minimal length of a contiguous subarray whose sum is greater than or equal to `target`. If no such subarray exists, return `0`.

## Brute Force Approach

**Idea:** Check every possible subarray, compute its sum, and track the minimum length that satisfies the sum condition.

**Algorithm:**
1. Initialize `minLen = INF`
2. For each starting index `i` from 0 to n-1:
   - Initialize `sum = 0`
   - For each ending index `j` from i to n-1:
     - Add `nums[j]` to `sum`
     - If `sum >= target`, update `minLen = min(minLen, j - i + 1)` and break
3. Return `minLen` if set, else `0`

**Complexity Analysis:**
- **Time:** O(n²) — two nested loops over the array
- **Space:** O(1) — only constant extra space

**Why this is suboptimal:** Every subarray sum is computed independently, leading to O(n²) time which is too slow for n = 10⁵.

## Optimal Approach: Sliding Window (Two Pointers)

**Idea:** Maintain a sliding window with two pointers. Expand the right pointer to grow the window, and when the window sum reaches or exceeds the target, shrink the window from the left while tracking the minimum length.

**Algorithm:**
1. Initialize `left = 0`, `total = 0`, `minLen = INF`
2. For `right` from 0 to n-1:
   - Add `nums[right]` to `total`
   - While `total >= target`:
     - Update `minLen = min(minLen, right - left + 1)`
     - Subtract `nums[left]` from `total`
     - Increment `left`
3. Return `minLen` if set, else `0`

**Why This Works:** Each element is added once (by `right`) and removed at most once (by `left`), making it O(n). The while loop shrinks the window as much as possible, ensuring we find the minimum length for each valid window.

**Complexity Analysis:**
- **Time:** O(n) — each element is visited at most twice
- **Space:** O(1) — only a few integer variables

class Solution {
    public int minSubArrayLen(int target, int[] nums) {
        int left = 0;          // Left boundary of the sliding window
        int total = 0;         // Sum of elements in the current window
        int minLen = Integer.MAX_VALUE;  // Minimum length found

        for (int right = 0; right < nums.length; right++) {
            total += nums[right];  // Expand the window to the right

            // Shrink the window from the left while the sum >= target
            while (total >= target) {
                minLen = Math.min(minLen, right - left + 1);
                total -= nums[left];  // Remove the leftmost element
                left++;               // Move the left boundary inward
            }
        }

        return minLen == Integer.MAX_VALUE ? 0 : minLen;
    }
}

fn min_sub_array_len(target: i32, nums: &Vec<i32>) -> i32 {
    let mut left = 0;             // Left boundary of the sliding window
    let mut total = 0i32;         // Sum of elements in the current window
    let mut min_len = i32::MAX;   // Minimum length found

    for right in 0..nums.len() {
        total += nums[right];  // Expand the window to the right

        // Shrink the window from the left while the sum >= target
        while total >= target {
            let window_len = (right - left + 1) as i32;
            if window_len < min_len {
                min_len = window_len;  // Update minimum length
            }
            total -= nums[left];  // Remove the leftmost element
            left += 1;            // Move the left boundary inward
        }
    }

    if min_len == i32::MAX { 0 } else { min_len }
}

## Key Takeaways

- **Sliding window (two pointers)** is the optimal pattern for subarray sum problems with positive numbers
- **Expanding and contracting** the window is the core mechanism: expand right until valid, then contract left to minimize
- The approach is **O(n) time and O(1) space** — optimal for this problem
- When the window sum is exactly `target`, the next left-pointer move will make the sum less than target, correctly driving the loop
