# Find the Town Judge — Solution

## Problem Overview

In a town of `n` people labeled `1` to `n`, a **town judge** is the person who:
1. Trusts nobody.
2. Is trusted by everybody else (all `n - 1` other people).

Given an array `trust` where `trust[i] = [a, b]` means person `a` trusts person `b`, return the label of the town judge if they exist, or `-1` otherwise.

**Example:** `n = 3, trust = [[1,3],[2,3]]` → judge is person `3`.

## Brute Force Approach

Count the in-degree (number of people who trust this person) and out-degree (number of people this person trusts) for each person using two separate arrays.

**Algorithm:**
1. Create `inDegree` and `outDegree` arrays of size `n + 1` (1-indexed).
2. For each `[a, b]` in `trust`:
   - Increment `outDegree[a]` (a trusts someone).
   - Increment `inDegree[b]` (b is trusted by someone).
3. The judge is the person `i` where `inDegree[i] == n - 1` and `outDegree[i] == 0`.

**Complexity:** O(n + len(trust)) time, O(n) space.

## Optimal Approach: Trust Balance

Instead of two separate arrays, we can maintain a single **trust balance** array: `balance[i] = inDegree[i] - outDegree[i]`. The judge will have `balance[i] = (n - 1) - 0 = n - 1`. This is not a classic Union-Find problem, but it demonstrates graph-theoretic reasoning that complements Union-Find techniques. Union-Find is more appropriate for *dynamic* trust scenarios where trust relationships evolve over time and we need to query the judge incrementally.

**Algorithm:**
1. Initialize `balance` array of size `n + 1` with zeros.
2. For each `[a, b]` in `trust`:
   - Decrement `balance[a]` (a trusts someone — out-degree increases).
   - Increment `balance[b]` (b is trusted — in-degree increases).
3. Scan for person `i` where `balance[i] == n - 1`. Return `i` if found, else `-1`.

**Complexity:** O(n + len(trust)) time, O(n) space.

## Deep Dive

Walk through the example `n = 4, trust = [[1,3],[1,4],[2,3],[3,4]]`:

```
Initial: balance = [0, 0, 0, 0, 0]  (index 0 unused)

Process trust edges:
  [1,3]: balance[1]-- → -1, balance[3]++ → 1
  [1,4]: balance[1]-- → -2, balance[4]++ → 1
  [2,3]: balance[2]-- → -1, balance[3]++ → 2
  [3,4]: balance[3]-- → 1,  balance[4]++ → 2

Final balance: [0, -2, -1, 1, 2]
  n - 1 = 3
  No one has balance = 3 → return -1

Indeed, nobody is trusted by everyone else.
```

Another example `n = 2, trust = [[1,2]]`:

```
balance = [0, 0, 0]
  [1,2]: balance[1]-- → -1, balance[2]++ → 1
  n - 1 = 1
  balance[2] = 1 → judge is person 2 ✅
```

class Solution:
    def findJudge(self, n: int, trust: list[list[int]]) -> int:
        # balance[i] = in-degree - out-degree for person i
        balance = [0] * (n + 1)

        for a, b in trust:
            balance[a] -= 1  # a trusts someone (out-degree)
            balance[b] += 1  # b is trusted (in-degree)

        # The judge has balance = n - 1
        for i in range(1, n + 1):
            if balance[i] == n - 1:
                return i
        return -1

impl Solution {

        let n = n as usize;
        let mut balance = vec![0i32; n + 1];

        for t in &trust {

                return i as i32;
            }
        }

    }

## Key Takeaways

- The trust balance (`in-degree - out-degree`) elegantly captures both conditions for the judge in a single pass.
- This problem is a graph-theory application: the judge is a sink node with `n - 1` incoming edges and zero outgoing edges.
- While not a classic Union-Find problem, the degree-counting technique is an essential graph tool. For dynamic trust scenarios where edges are added over time, a Union-Find with additional metadata (tracking in-degree counts per root) could be used instead.
- Always watch for edge cases: `n = 1` (no trust edges, person 1 is the judge), empty trust array, and no candidate meeting both conditions.
