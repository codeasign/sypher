# Number of Provinces — Solution

## Problem Overview

There are `n` cities, numbered from `0` to `n-1`. Some cities are directly connected by roads, while others are not. A **province** is a group of directly or indirectly connected cities, with no other cities outside the group connected to them. Given an `n x n` adjacency matrix `isConnected` where `isConnected[i][j] = 1` if city `i` is directly connected to city `j` (and `isConnected[i][i] = 1` by convention), return the total number of provinces.

**Example:** `isConnected = [[1,1,0],[1,1,0],[0,0,1]]` has 2 provinces: `{0, 1}` and `{2}`.

## Brute Force Approach: DFS / BFS

The problem is equivalent to counting connected components in an undirected graph. We can run DFS or BFS from each unvisited node and mark all reachable nodes as visited.

**Algorithm:**
1. Maintain a `visited` boolean array of size `n`, initially all `false`.
2. For each city `i` from `0` to `n-1`:
   - If not visited, start a DFS/BFS from `i` and increment the province count.
   - In the traversal, mark every reachable city as visited.
3. Return the count.

**Complexity:** O(n^2) time (we must examine the adjacency matrix edges), O(n) space for the visited array and recursion stack.

**Why Union-Find is better:** Union-Find handles the same connected-component counting with the same time complexity but offers a cleaner, iterative approach that scales well to dynamic additions (new edges added over time) without re-running the entire traversal.

## Optimal Approach: Union-Find

Union-Find (Disjoint Set Union) is purpose-built for this problem. We union all pairs of directly connected cities, then count the number of distinct roots remaining.

**Algorithm:**
1. Initialize a Union-Find structure with `n` elements (each city is its own parent).
2. Iterate over every pair `(i, j)` where `j > i` (upper triangle of the matrix, since the graph is undirected and symmetric).
3. If `isConnected[i][j] == 1`, call `union(i, j)`.
4. After processing all pairs, count how many cities `i` satisfy `find(i) == i` — these are the distinct roots, one per province.

**Complexity:** O(n^2 * alpha(n)) time, where alpha(n) is the inverse Ackermann function (essentially constant for all practical inputs). O(n) space for the `parent` and `rank` arrays.

## Deep Dive

Walk through the example matrix `[[1,1,0],[1,1,0],[0,0,1]]`:

```
Initial state:
  parent = [0, 1, 2]   rank = [0, 0, 0]

Step 1: Check pair (0, 1) — isConnected[0][1] = 1
  find(0) = 0, find(1) = 1  → different roots, union
  rank[0] == rank[1], so attach 1 under 0, increment rank[0]
  parent = [0, 0, 2]   rank = [1, 0, 0]

Step 2: Check pair (0, 2) — isConnected[0][2] = 0 → skip
Step 3: Check pair (1, 2) — isConnected[1][2] = 0 → skip

Count distinct roots:
  find(0) = 0    → root
  find(1) = 0    → compressed
  find(2) = 2    → root
  Total: 2 provinces ✅
```

class Solution:
    def findCircleNum(self, isConnected: list[list[int]]) -> int:
        n = len(isConnected)
        parent = list(range(n))
        rank = [0] * n

        def find(x: int) -> int:
            # Path compression: make every node point directly to the root
            if parent[x] != x:
                parent[x] = find(parent[x])
            return parent[x]

        def union(x: int, y: int) -> None:
            px, py = find(x), find(y)
            if px == py:
                return
            # Union by rank: attach the shorter tree under the taller one
            if rank[px] < rank[py]:
                parent[px] = py
            elif rank[px] > rank[py]:
                parent[py] = px
            else:
                parent[py] = px
                rank[px] += 1

        for i in range(n):
            for j in range(i + 1, n):
                if isConnected[i][j] == 1:
                    union(i, j)

        # Count distinct roots
        return sum(1 for i in range(n) if find(i) == i)

class Solution {
    public int findCircleNum(int[][] isConnected) {
        int n = isConnected.length;
        int[] parent = new int[n];
        int[] rank = new int[n];
        for (int i = 0; i < n; i++) parent[i] = i;

        for (int i = 0; i < n; i++) {
            for (int j = i + 1; j < n; j++) {
                if (isConnected[i][j] == 1) {
                    union(parent, rank, i, j);
                }
            }
        }

        // Count distinct roots
        int count = 0;
        for (int i = 0; i < n; i++) {
            if (find(parent, i) == i) count++;
        }
        return count;
    }

    private int find(int[] parent, int x) {
        if (parent[x] != x) parent[x] = find(parent, parent[x]);
        return parent[x];
    }

        else if (rank[px] > rank[py]) parent[py] = px;
        else { parent[py] = px; rank[px]++; }
    }
}

    pub fn find_circle_num(is_connected: Vec<Vec<i32>>) -> i32 {

## Key Takeaways

- Union-Find naturally identifies connected components — count the number of distinct roots after processing all edges.
- Path compression and union by rank together keep each operation nearly O(1) amortized.
- This problem is a textbook application of Union-Find for connected-component counting in an undirected graph.
- The same approach works for any graph where you need to find the number of disjoint groups.
