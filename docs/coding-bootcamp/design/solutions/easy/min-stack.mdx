# Min Stack — Solution

## Problem Overview

Design a stack that supports push, pop, top, and retrieving the minimum element in O(1) time. The challenge is that a standard stack only gives O(1) access to the top element, not the minimum. We need to augment the stack with auxiliary data to track the minimum as elements are pushed and popped.

## Approach 1: Two Stacks (Auxiliary Min Stack)

**Idea:** Maintain a separate stack that tracks the current minimum. When pushing a value, compare it with the current minimum and push the smaller of the two onto the min stack. When popping, pop from both stacks.

**Algorithm:**
1. Create a main stack for all elements and a min stack for tracking minima.
2. `push(val)`: Push val onto the main stack. If the min stack is empty or val `<=` top of min stack, push val onto the min stack. Otherwise, push the current minimum onto the min stack again.
3. `pop()`: Pop from both the main stack and the min stack.
4. `top()`: Return the top of the main stack.
5. `getMin()`: Return the top of the min stack.

**Complexity Analysis:**
- **Time:** O(1) for all operations
- **Space:** O(n) — the min stack holds up to n elements

## Approach 2: Single Stack with Pairs

**Idea:** Store each element as a pair (value, currentMin) in a single stack. This eliminates the need for a separate stack.

**Algorithm:**
1. The stack stores pairs of (value, minimum at this point).
2. `push(val)`: If the stack is empty, push (val, val). Otherwise, push (val, min(val, top.min)).
3. `pop()`: Pop the top pair.
4. `top()`: Return the value from the top pair.
5. `getMin()`: Return the minimum from the top pair.

**Complexity Analysis:**
- **Time:** O(1) for all operations
- **Space:** O(n) — each element stores a pair

## Approach 3: Space-Optimized (Single Variable + Encoding)

**Idea:** Track the minimum with a single variable. When pushing a value smaller than the current minimum, encode the difference to recover the old minimum on pop.

**Algorithm:**
1. Maintain a single stack for values and a variable `min` for the current minimum.
2. `push(val)`: If val < min, push (2 * val - min) and set min = val. Otherwise, push val.
3. `pop()`: If the top is less than min, restore the old minimum: min = 2 * min - top.
4. `top()`: If the top is less than min, return min. Otherwise, return top.
5. `getMin()`: Return min.

**Complexity Analysis:**
- **Time:** O(1) for all operations
- **Space:** O(n) — single stack, no auxiliary structure

This approach relies on integer overflow behavior and can be tricky with large negative values, so Approach 1 or 2 is preferred in interviews.

import java.util.Stack;

class MinStack {
    // Main stack to store all elements
    private Stack<Integer> stack;
    // Auxiliary stack to track minimum at each state
    private Stack<Integer> minStack;

    public MinStack() {
        stack = new Stack<>();
        minStack = new Stack<>();
    }

    public void push(int val) {
        stack.push(val);
        // Push the new minimum onto the min stack
        // If the min stack is empty, val becomes the minimum
        // Otherwise, push the smaller of val and the current minimum
        if (minStack.isEmpty() || val <= minStack.peek()) {
            minStack.push(val);
        } else {
            minStack.push(minStack.peek());
        }
    }

    public void pop() {
        stack.pop();
        minStack.pop();  // Always pop from both stacks in sync
    }

    public int top() {
        return stack.peek();
    }

    public int getMin() {
        return minStack.peek();
    }
}

#include <stdbool.h>
#include <stdlib.h>
#include <limits.h>

#define MAX_SIZE 100005

typedef struct {
    int stack[MAX_SIZE];     // Main stack
    int minStack[MAX_SIZE];  // Auxiliary min-tracking stack
    int top;                 // Current top index (-1 = empty)
} MinStack;

MinStack* minStackCreate() {
    MinStack* obj = (MinStack*)malloc(sizeof(MinStack));
    obj->top = -1;
    return obj;
}

void minStackPush(MinStack* obj, int val) {
    obj->stack[++(obj->top)] = val;
    // Track the new minimum
    if (obj->top == 0) {
        obj->minStack[obj->top] = val;
    } else {
        int prevMin = obj->minStack[obj->top - 1];
        obj->minStack[obj->top] = (val < prevMin) ? val : prevMin;
    }
}

void minStackPop(MinStack* obj) {
    if (obj->top `>=` 0) {
        (obj->top)--;  // Decrement top to remove element
    }
}

int minStackTop(MinStack* obj) {
    return obj->stack[obj->top];
}

int minStackGetMin(MinStack* obj) {
    return obj->minStack[obj->top];
}

void minStackFree(MinStack* obj) {
    free(obj);
}

type MinStack struct {

    this.stack = append(this.stack, val)
    // Track the new minimum
    if len(this.minStack) == 0 || val <= this.minStack[len(this.minStack)-1] {
        this.minStack = append(this.minStack, val)
    } else {
        this.minStack = append(this.minStack, this.minStack[len(this.minStack)-1])

}

func (this *MinStack) Pop() {
    this.stack = this.stack[:len(this.stack)-1]
    this.minStack = this.minStack[:len(this.minStack)-1]  // Keep synchronized
}

func (this *MinStack) Top() int {
    return this.stack[len(this.stack)-1]
}

func (this *MinStack) GetMin() int {
    return this.minStack[len(this.minStack)-1]

## Key Takeaways

- **Two-stack approach** is the simplest and most intuitive solution for Min Stack
- The min stack always stays **synchronized** with the main stack — every push/pop affects both
- The space-optimized approach (single variable + encoding) is clever but **brittle** with integer overflow
- O(1) time for all operations with O(n) extra space is the expected trade-off
- This pattern of using **auxiliary data structures** to track statistics is common in design problems
