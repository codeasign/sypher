# Design Browser History

**Difficulty:** Medium

You have a **browser history** where you start on a `homepage`. You can visit other URLs, go back steps in history, and go forward steps in history.

Implement the `BrowserHistory` class:

- `BrowserHistory(string homepage)` initializes the object with the `homepage` of the browser.
- `void visit(string url)` visits `url` from the current page. Clears up all the forward history.
- `string back(int steps)` move `steps` back in history. If you can only return `x` steps in the history and `steps > x`, you will return only `x` steps. Return the current page after moving back.
- `string forward(int steps)` move `steps` forward in history. If you can only forward `x` steps in the history and `steps > x`, you will forward only `x` steps. Return the current page after moving forward.

## Example 1

**Input:**
```
["BrowserHistory","visit","visit","visit","back","back","forward","visit","forward","back","back"]
[["leetcode.com"],["google.com"],["facebook.com"],["youtube.com"],[1],[1],[1],["linkedin.com"],[2],[2],[7]]
```

**Output:**
```
[null,null,null,null,"facebook.com","google.com","facebook.com",null,"linkedin.com","google.com","leetcode.com"]
```

**Explanation:**
```
BrowserHistory browserHistory = new BrowserHistory("leetcode.com");
browserHistory.visit("google.com");       // You are at "google.com"
browserHistory.visit("facebook.com");     // You are at "facebook.com"
browserHistory.visit("youtube.com");      // You are at "youtube.com"
browserHistory.back(1);                   // You are back at "facebook.com"
browserHistory.back(1);                   // You are back at "google.com"
browserHistory.forward(1);                // You are forward at "facebook.com"
browserHistory.visit("linkedin.com");     // You are at "linkedin.com" (forward history cleared)
browserHistory.forward(2);                // You are at "linkedin.com" (no forward history)
browserHistory.back(2);                   // You are at "google.com"
browserHistory.back(7);                   // You are at "leetcode.com"
```

## Input Specification

The first line contains `BrowserHistory <homepage>`. Subsequent lines contain commands:
- `visit <url>` — visit a URL
- `back <steps>` — go back steps, output the current URL
- `forward <steps>` — go forward steps, output the current URL

## Output Specification

For each `back` and `forward` command, output the resulting URL on its own line.

## Constraints

- `1 <= homepage.length <= 20`
- `1 <= url.length <= 20`
- `1 <= steps <= 100`
- At most `5000` calls will be made to `visit`, `back`, and `forward`.

## Hints

<details>
<summary>Hint 1</summary>

Use two stacks (or a doubly linked list) to represent the history. One stack for the back history and one for the forward history.
</details>

<details>
<summary>Hint 2</summary>

When you visit a new URL, clear the forward stack (since forward history is no longer valid). Push the current page to the back stack.
</details>

<details>
<summary>Hint 3</summary>

For back, pop from the back stack and push to the forward stack. For forward, pop from the forward stack and push to the back stack. The current page is always the top of the back stack.
</details>
