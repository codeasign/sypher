# Unique Paths — Solution

## Problem Overview

Count the number of unique paths from the top-left to the bottom-right of an `m x n` grid, moving only down or right. This is a **2D dynamic programming** problem with a simple recurrence.

## Brute Force Approach: Recursion (DFS)

**Idea:** Use recursion to explore all paths. At each cell, recursively explore the right and down moves, summing the total paths.

**Algorithm:**
1. If at the bottom-right cell (m-1, n-1), return 1 (found a path).
2. If out of bounds, return 0.
3. Return `countPaths(r+1, c) + countPaths(r, c+1)`.

**Complexity Analysis:**
- **Time:** O(2^(m+n)) — each call branches 2 ways, depth = m+n.
- **Space:** O(m+n) — recursion stack depth.

**Why this could be better:** The recursive approach explores every path independently, resulting in exponential time. DP caches overlapping subproblems, reducing to O(m*n).

## Optimal Approach: Dynamic Programming

**Idea:** Let `dp[i][j]` = number of ways to reach cell `(i, j)`. The robot can only arrive from above or from the left, so `dp[i][j] = dp[i-1][j] + dp[i][j-1]`. Base case: `dp[0][j] = 1` and `dp[i][0] = 1` (only one way along the top row and left column).

**Algorithm:**
1. Initialize a 1D DP array `dp` of size `n` with all 1s (first row: all cells have exactly 1 way to reach them).
2. For each row from 1 to m-1:
   - For each column from 1 to n-1:
     - `dp[j] = dp[j] + dp[j-1]` (dp[j] from the previous row + dp[j-1] from the current row)
3. Return `dp[n-1]`.

**Why This Works:** `dp[j]` before the update represents `dp[i-1][j]` (the cell above). `dp[j-1]` after the update represents `dp[i][j-1]` (the cell to the left). By adding them, we compute the current cell's path count in O(1) space.

**Complexity Analysis:**
- **Time:** O(m * n) — nested loops over the grid.
- **Space:** O(n) — single row of DP values.

## Deep Dive: How the DP Table Evolves

### Visual Walkthrough

For `m = 3, n = 3`:

```
Initial dp (row 0): [1, 1, 1]   ← first row: only 1 way to reach each cell

Row 1:
  dp[0] = 1                      ← first column: always 1
  dp[1] = dp[1] + dp[0] = 1+1 = 2
  dp[2] = dp[2] + dp[1] = 1+2 = 3
  After row 1: [1, 2, 3]

Row 2:
  dp[0] = 1
  dp[1] = dp[1] + dp[0] = 2+1 = 3
  dp[2] = dp[2] + dp[1] = 3+3 = 6
  After row 2: [1, 3, 6]   ← answer: dp[2] = 6

Full table:
  ┌───┬───┬───┐
  │ 1 │ 1 │ 1 │
  ├───┼───┼───┤
  │ 1 │ 2 │ 3 │
  ├───┼───┼───┤
  │ 1 │ 3 │ 6 │
  └───┴───┴───┘
```

### Edge Cases

- **1 x 1 grid**: Only one cell — return 1.
- **Single row (m=1)**: Only one path (all right moves) — return 1.
- **Single column (n=1)**: Only one path (all down moves) — return 1.
- **Large grids ()**: The number of paths grows large but fits in 32-bit for m,n  100.

### Combinatorial Formula

This problem is equivalent to choosing `m-1` down moves out of `(m-1)+(n-1)` total moves. The answer is C(m+n-2, m-1) = (m+n-2)! / ((m-1)! * (n-1)!). This formula can compute the result in O(min(m,n)) but may overflow for large values.

def unique_paths(m: int, n: int) -> int:
    # dp[j] = number of ways to reach column j in the current row
    # Initially, the first row has all 1s (only one way: keep moving right)
    dp = [1] * n

    # Process remaining rows
    for i in range(1, m):
        for j in range(1, n):
            # dp[j] (from previous row) + dp[j-1] (from current row)
            dp[j] += dp[j - 1]

    return dp[-1]  # bottom-right cell

class Solution {
    public int uniquePaths(int m, int n) {
        // dp[j] = number of ways to reach column j in the current row
        int[] dp = new int[n];
        // First row: only one way to reach each cell (keep moving right)
        for (int j = 0; j < n; j++) dp[j] = 1;

        // Process remaining rows
        for (int i = 1; i < m; i++) {
            for (int j = 1; j < n; j++) {
                // dp[j] = above cell, dp[j-1] = left cell
                dp[j] += dp[j - 1];
            }
        }

        return dp[n - 1];  // bottom-right cell
    }
}

## Key Takeaways

- **2D DP on a grid**: The recurrence `dp[i][j] = dp[i-1][j] + dp[i][j-1]` is the foundation for many grid-based DP problems (Minimum Path Sum, Unique Paths II).
- **Space optimization**: By using a 1D array and updating left-to-right, we reduce space from O(m*n) to O(n).
- **Combinatorial insight**: The problem can also be solved with combinatorics (binomial coefficient).
