# Bit Manipulation

Bit manipulation involves operating directly on the binary representation of integers. It is one of the most powerful and efficient techniques in competitive programming, allowing you to solve problems with O(1) operations that would otherwise require loops or extra memory.

## Bitwise Operators

Every integer is stored in memory as a sequence of bits (0s and 1s). Bitwise operators work on these bits directly.

### AND (`&`)

Returns 1 only if both bits are 1.

```
  1 0 1 1 0  (22)
& 1 1 0 1 0  (26)
-----------
  1 0 0 1 0  (18)
```

**Use:** Clearing specific bits, checking if a number is even (`n & 1 == 0`).

### OR (`|`)

Returns 1 if at least one bit is 1.

```
  1 0 1 1 0  (22)
| 1 1 0 1 0  (26)
-----------
  1 1 1 1 0  (30)
```

**Use:** Setting specific bits, combining flags.

### XOR (`^`)

Returns 1 if the bits are different.

```
  1 0 1 1 0  (22)
^ 1 1 0 1 0  (26)
-----------
  0 1 1 0 0  (12)
```

**Key properties:**
- `a ^ a = 0` (a number XOR itself is 0)
- `a ^ 0 = a` (a number XOR 0 is itself)
- `a ^ b ^ a = b` (XOR is associative and commutative)
- `a ^ b = b ^ a` (commutative)

### NOT (`~`)

Flips all bits. `~n = -(n + 1)` in two's complement.

```
~ 0 1 1 0 1  (13)
-----------
  1 0 0 1 0  (-14)
```

### Left Shift (`<<`)

Shifts bits left, filling with 0s on the right. Equivalent to multiplying by 2^k.

```
  0 0 0 1 1  (3) << 2
  ----------
  0 1 1 0 0  (12)
```

`n << k` = `n * 2^k`

### Right Shift (`>>`)

Shifts bits right. Equivalent to integer division by 2^k.

```
  0 1 1 0 0  (12) >> 2
  ----------
  0 0 0 1 1  (3)
```

`n >> k` = `n // 2^k`

## Common Tricks

### Checking if a bit is set

```python
# Check if the k-th bit (0-indexed from LSB) is set
is_set = (n >> k) & 1
```

### Setting a bit

```python
# Set the k-th bit to 1
n = n | (1 << k)
```

### Clearing a bit

```python
# Set the k-th bit to 0
n = n & ~(1 << k)
```

### Toggling a bit

```python
# Flip the k-th bit
n = n ^ (1 << k)
```

### Checking if a number is a power of two

```python
is_power_of_two = n > 0 and (n & (n - 1)) == 0
```

### Counting set bits (Brian Kernighan's algorithm)

```python
count = 0
while n:
    n &= n - 1  # clears the lowest set bit
    count += 1
```

### Isolating the lowest set bit

```python
lowest_set_bit = n & -n
```

## Complexity

| Operation | Complexity | Notes |
|-----------|-----------|-------|
| Single bitwise operation | O(1) | Constant time, hardware-level |
| Counting set bits | O(num bits set) | Kernighan's: O(k) where k = number of 1 bits |
| Iterating over all bits | O(num bits) | Typically 32 or 64 |
| Bit manipulation on array | O(n) | Per element |

## Common Applications

- **Flags and permissions:** Use individual bits to represent boolean flags in a single integer (e.g., Unix file permissions)
- **Optimization:** Replace expensive operations (division, modulo) with shifts and masks
- **Compression:** Pack multiple boolean values into a single integer
- **Cryptography:** XOR is fundamental to many encryption algorithms (one-time pad, AES)
- **Graphics:** Bit blitting and mask operations in pixel data
- **Networking:** Subnet masks, IP address calculations

## Worked Example: Single Non-Repeating Element

**Problem:** Given an array where every element appears twice except for one, find the element that appears once.

**Example:** `[4, 1, 2, 1, 2]` → Output: `4`

**Approach:** XOR all elements together. Since `a ^ a = 0`, pairs cancel out, leaving only the unique element.

```python
def single_number(nums: list[int]) -> int:
    result = 0
    for num in nums:
        result ^= num  # XOR all elements
    return result
```

**Trace:**
```
result = 0
result ^ 4 = 4  (0 ^ 100 = 100)
result ^ 1 = 5  (100 ^ 001 = 101)
result ^ 2 = 7  (101 ^ 010 = 111)
result ^ 1 = 6  (111 ^ 001 = 110)
result ^ 2 = 4  (110 ^ 010 = 100)  ← unique element!
```

**Complexity:** O(n) time, O(1) space — no extra memory needed.
