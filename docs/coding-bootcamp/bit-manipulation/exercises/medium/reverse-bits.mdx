# Reverse Bits

**Difficulty:** Medium

Reverse the bits of a given 32-bit unsigned integer.

**Note:**
- In some languages (like Java), there are no unsigned integer types. In such cases, the input will be given as a signed integer. Do not worry about sign — the internal binary representation is the same.
- Your function should handle all 32-bit values from 0 to 2³² − 1.

## Example 1

**Input:** `n = 43261596` (binary: `00000010100101000001111010011100`)  
**Output:** `964176192` (binary: `00111001011110000010100101000000`)

## Example 2

**Input:** `n = 4294967293` (binary: `11111111111111111111111111111101`)  
**Output:** `3221225471` (binary: `10111111111111111111111111111111`)

## Input Specification

A single integer `n` (0 ≤ n < 2³²).

## Output Specification

Print the unsigned integer obtained by reversing the 32 bits of the input.

## Constraints

- `0 <= n < 2³²`

## Hints

<details>
<summary>Hint 1</summary>

Iterate through all 32 bits. Extract the least significant bit of `n` and place it in the most significant position of the result.
</details>

<details>
<summary>Hint 2</summary>

Use a loop: for each bit position `i` from 0 to 31, check if bit `i` of `n` is set by `(n >> i) & 1`, and if so, set bit `(31 - i)` of the result.
</details>

<details>
<summary>Hint 3</summary>

Can you optimize by processing 4 bits at a time using a precomputed lookup table?
</details>
