# Summary Ranges

**Difficulty:** Easy

Given a **sorted unique** integer array `nums`, return the smallest sorted list of ranges that cover all the numbers in the array exactly. That is, each element of `nums` is covered by exactly one of the ranges, and there is no integer in any range that is not in `nums`.

Each range `[a, b]` should be output as:
- `"a->b"` if `a != b`
- `"a"` if `a == b`

Return the ranges as a comma-separated string.

## Example 1

**Input:** `nums = [0, 1, 2, 4, 5, 7]`
**Output:** `"0->2,4->5,7"`
**Explanation:** The ranges are `[0,2]`, `[4,5]`, and `[7,7]`.

## Example 2

**Input:** `nums = [0, 2, 3, 4, 6, 8, 9]`
**Output:** `"0,2->4,6,8->9"`
**Explanation:** The ranges are `[0,0]`, `[2,4]`, `[6,6]`, and `[8,9]`.

## Example 3

**Input:** `nums = []`
**Output:** `""`
**Explanation:** There are no numbers so there are no ranges.

## Input Specification

A single line of space-separated integers representing the sorted array `nums`. The line may be empty to represent an empty array.

## Output Specification

Print the comma-separated range strings on a single line. Print an empty line if `nums` is empty.

## Constraints

- `0 <= nums.length <= 20`
- `-2^31 <= nums[i] <= 2^31 - 1`
- All values in `nums` are **unique** and sorted in **ascending order**.

## Hints

<details>
<summary>Hint 1</summary>

Walk through the array and track the **start** of each consecutive range. A new range begins whenever `nums[i] != nums[i-1] + 1`.

</details>

<details>
<summary>Hint 2</summary>

When you detect a break (or reach the last element), emit the range. If `start == end` emit `"start"`, otherwise emit `"start->end"`.

</details>

<details>
<summary>Hint 3</summary>

Don't forget to emit the last open range after the loop ends — it won't be closed by a break inside the loop.

</details>

</ProblemEditor>
