# Remove Covered Intervals

**Difficulty:** Medium

Given a list of intervals, remove all intervals that are **covered** by another interval in the list.

An interval `[a, b]` is covered by interval `[c, d]` if `c <= a` and `b <= d`.

Return the number of remaining intervals after removing all covered intervals.

## Example 1

**Input:**
```
intervals = [[1,4],[3,6],[2,8]]
```
**Output:** `2`

**Explanation:** Interval `[3,6]` is covered by `[2,8]` (since `2 <= 3` and `6 <= 8`), so we remove it. The remaining intervals are `[1,4]` and `[2,8]`.

## Example 2

**Input:**
```
intervals = [[1,4],[2,3]]
```
**Output:** `1`

**Explanation:** Interval `[2,3]` is covered by `[1,4]` (since `1 <= 2` and `3 <= 4`), so we remove it.

## Example 3

**Input:**
```
intervals = [[1,10],[2,5],[6,9]]
```
**Output:** `1`

**Explanation:** Both `[2,5]` and `[6,9]` are covered by `[1,10]`. Only `[1,10]` remains.

## Input Specification

The first line contains a single integer `n` — the number of intervals.

The next `n` lines each contain two space-separated integers `a` and `b` representing the interval `[a, b]`.

## Output Specification

Print a single integer — the count of intervals remaining after removing all covered intervals.

## Constraints

- `1 <= intervals.length <= 1000`
- `0 <= intervals[i][0] < intervals[i][1] <= 10^5`

## Hints

<details>
<summary>Hint 1</summary>

Try sorting the intervals. If you sort by start time ascending (and break ties by end time descending), you can process them left to right and greedily decide which ones are covered.

</details>

<details>
<summary>Hint 2</summary>

After sorting, track the maximum end value seen so far. If the current interval's end is less than or equal to `maxEnd`, the current interval is covered — because there already exists an interval that starts no later and ends no earlier.

</details>

<details>
<summary>Hint 3</summary>

When two intervals share the same start, the one with the larger end covers the smaller-end one. Sorting by start ascending and end **descending** handles this case automatically: the first interval seen for each start group is always the widest, and subsequent ones will be detected as covered.

</details>

</ProblemEditor>
