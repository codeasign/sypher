# Meeting Rooms — Solution

## Problem Overview

Given an array of meeting time intervals `[start, end]`, determine if a person could attend all meetings. This is fundamentally an interval overlap detection problem: if any two intervals overlap, the person cannot attend both meetings.

## Brute Force Approach

**Idea:** Compare every pair of intervals to check for overlaps.

**Algorithm:**
1. For each interval `i`, compare it with every other interval `j`
2. Two intervals `[a, b]` and `[c, d]` overlap if `a < d` and `c < b` (they share any common time)
3. If any pair overlaps, return `false`
4. Return `true` if no overlaps are found

**Complexity Analysis:**
- **Time:** O(n²) — we compare each pair of n intervals
- **Space:** O(1) — no extra space used

**Why this could be better:** This approach works but is inefficient for large inputs. We can do better by sorting the intervals first.

## Optimal Approach: Sort and Sweep

**Idea:** Sort intervals by start time, then scan to check if any interval's end overlaps with the next interval's start.

**Algorithm:**
1. Sort the intervals by start time
2. Iterate from `i = 0` to `n - 2`:
   - If `intervals[i][1] > intervals[i+1][0]`, return `false` (overlap found)
3. Return `true` if no overlaps are found

**Why This Works:** After sorting by start time, the only possible overlaps are between adjacent intervals. If interval A ends after interval B starts, then A overlaps with B (and B starts after A started, since they are sorted). Non-adjacent intervals cannot overlap without adjacent ones also overlapping.

**Complexity Analysis:**
- **Time:** O(n log n) — dominated by sorting
- **Space:** O(1) or O(n) depending on the sorting algorithm used

## Deep Dive: How the Sort and Sweep Works

### Visual Walkthrough

For the input `intervals = [[0, 30], [5, 10], [15, 20]]`:

```
Step 1: Sort by start time
        [(0,30), (5,10), (15,20)]

Step 2: Check adjacent pairs
        Pair 1: (0,30) and (5,10)
        ┌───── 0 ───────────── 30 ────┐
          ┌── 5 ── 10 ─┐
        30 > 5 → OVERLAP → return false

Result: false (cannot attend all meetings)
```

For the input `intervals = [[0, 5], [5, 10], [10, 15]]`:

```
Step 1: Sort by start time (already sorted)
        [(0,5), (5,10), (10,15)]

Step 2: Check adjacent pairs
        Pair 1: (0,5) and (5,10) → 5 > 5? No (equal is not overlap)
        Pair 2: (5,10) and (10,15) → 10 > 10? No (equal is not overlap)

Result: true (can attend all meetings)
```

### Key Insight: Overlap Condition

Two sorted intervals `[a, b]` and `[c, d]` (with `a <= c`) overlap if and only if `b > c`. Notice that:
- If `b == c`, end of one meeting equals the start of the next — the person can attend both (back-to-back)
- If `b < c`, there is a gap — no overlap
- If `b > c`, the meetings overlap — cannot attend both

### Edge Cases

- **Empty input (n = 0):** No meetings to attend, return `true`
- **Single meeting:** Always possible, return `true`
- **Duplicate intervals:** Two meetings at the same time always overlap, return `false`
- **Negative start times:** The algorithm handles negative values naturally since sorting and comparison work on any integers
- **Large ranges:** The O(n log n) time complexity handles up to 10⁴ intervals efficiently

import java.util.Arrays;

class Solution {
    public boolean canAttendMeetings(int[][] intervals) {
        // Sort intervals by start time
        Arrays.sort(intervals, (a, b) -> Integer.compare(a[0], b[0]));

        // Check adjacent intervals for overlaps
        for (int i = 0; i < intervals.length - 1; i++) {
            // If current interval ends after next starts → overlap
            if (intervals[i][1] > intervals[i + 1][0]) {
                return false;
            }
        }
        return true;
    }
}

## Key Takeaways

- **Sort then sweep** is the standard pattern for interval overlap detection
- The overlap condition for sorted intervals is simple: `current_end > next_start`
- Back-to-back meetings (end == start of next) are not considered overlapping — the person can attend both
- Sorting first reduces the O(n²) brute force approach to O(n log n)
- This problem is a foundation for harder interval problems like Merge Intervals and Meeting Rooms II
