# Minimum Number of Arrows to Burst Balloons — Solution

## Problem Overview

Balloons are represented as intervals `[x_start, x_end]` on the x-axis. An arrow shot at position `x` bursts any balloon where `x_start <= x <= x_end`. Find the **minimum** number of arrows needed to burst all balloons.

## Brute Force Approach

**Idea:** Try all possible arrow placements and combinations to find the minimum set of points that intersects all intervals.

**Algorithm:**
1. Generate all possible combinations of intersection points
2. For each combination, check if every balloon is intersected by at least one point
3. Return the minimum number of points needed

**Complexity Analysis:**
- **Time:** O(2^n) — exponential, testing subsets of arrow positions
- **Space:** O(n) — recursion stack

**Why this could be better:** The brute force is computationally infeasible for up to 10^5 balloons. The greedy approach solves it in O(n log n) time.

## Optimal Approach: Greedy Interval Scheduling

**Idea:** Sort intervals by their **end** coordinate. Shoot the first arrow at the end of the first balloon. Then traverse the remaining balloons: if a balloon's start is beyond the last arrow's position, it hasn't been burst — shoot a new arrow at its end.

**Algorithm:**
1. Sort `points` by the end coordinate (`x_end`) in ascending order
2. Initialize `arrows = 1` (first arrow) and `arrowPos = points[0][1]`
3. For each remaining balloon `[start, end]`:
   - If `start > arrowPos`, the balloon is not burst by the last arrow:
     - Increment `arrows` and set `arrowPos = end`
4. Return `arrows`

**Why This Works:** This is the classic "minimum points to cover all intervals" problem — a variant of interval scheduling. By sorting by end coordinate and always shooting at the end of the earliest-ending balloon that isn't yet burst, we maximize the overlap with remaining balloons. Any optimal solution must place an arrow within the first balloon's range; placing it at the end is always at least as good as any other position because it also covers all balloons that start at or before that end point.

**Complexity Analysis:**
- **Time:** O(n log n) — dominated by sorting
- **Space:** O(1) — ignoring sorting overhead

```
Arrow 1 at x=2: bursts [1,2] and [2,3] (2 <= 2 ✓)
Arrow 2 at x=4: bursts [3,4] and [4,5] (4 <= 4 ✓)

Result: 2 arrows
```

### Edge Cases

- **Single balloon**: Always requires 1 arrow
- **All balloons overlapping**: 1 arrow at the end of the earliest-ending balloon bursts everything
- **No overlapping balloons**: Each balloon needs its own arrow — result equals the number of balloons
- **Touching balloons** (e.g., `[1,2]` and `[2,3]`): One arrow at x=2 bursts both since `x_start <= x <= x_end` means the shared boundary counts as overlap
- **Integer overflow**: The problem uses the full 32-bit integer range. In statically typed languages, comparing `start > arrowPos` can overflow when `start = -2^31` and `arrowPos = 2^31 - 1`. Use `long long` or cast to a wider type for safe comparison.
- **Large coordinates near INT_MIN/INT_MAX**: Sorting by end coordinate works correctly, but the comparison in the traversal must use a type that accommodates the full range.

### Greedy Choice Proof

Let the balloons be sorted by end coordinate. Let `b1 = [x1, y1]` be the balloon with the smallest end coordinate. Any optimal solution must place at least one arrow somewhere in `[x1, y1]` to burst `b1`. Among all positions in `[x1, y1]`, shooting at `y1` (the end) is optimal because every other balloon that overlaps with any part of `b1` also overlaps with `y1` (since their start  y1 if they overlap with b1's range). Therefore, greedily shooting at `y1` leaves the rest of the problem no harder.

```python
def find_min_arrow_shots(points: list[list[int]]) -> int:
    if not points:
        return 0

    # Sort balloons by their end coordinate (greedy choice)
    points.sort(key=lambda x: x[1])

    arrows = 1             # at least one arrow is needed
    arrow_pos = points[0][1]  # first arrow at the end of the first balloon

    for start, end in points[1:]:
        # If this balloon starts after the last arrow's position,
        # it is not burst — need a new arrow
        if start > arrow_pos:
            arrows += 1
            arrow_pos = end  # shoot new arrow at this balloon's end

    return arrows
```

```java
        Arrays.sort(points, (a, b) ->

            }
        }
        return arrows;
    }
```

## Key Takeaways

- **Sort by end**: this is the critical greedy choice — shooting at the smallest end coordinate maximizes overlap with remaining balloons
- **Integer overflow**: the problem uses the full 32-bit integer range. In C++, C, Rust, and C# the solution uses `long long` / `i64` / `long` to safely compare `start > arrowPos`
- **Greedy choice proof**: let the first balloon in sorted order be `b1` (ends earliest). Any optimal solution must shoot an arrow somewhere in `b1`'s range. Shooting at `b1.end` is always at least as good as any other point because it also bursts all balloons that start at or before `b1.end`
- **Edge cases**: single balloon (1 arrow), non-overlapping balloons (each needs its own arrow), all overlapping (1 arrow), integer bounds
