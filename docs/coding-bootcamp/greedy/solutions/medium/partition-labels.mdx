# Partition Labels — Solution

## Problem Overview

Given a string `s`, partition it into as many parts as possible so that **each letter appears in at most one part**. Return a list of integers representing the size of each part.

## Brute Force Approach

**Idea:** Try every possible partition point, recursively checking if each segment satisfies the constraint that no character appears in more than one segment.

**Algorithm:**
1. Generate all possible partition boundaries
2. For each candidate partition, verify that each segment's characters do not appear in any other segment
3. Keep the partition with the maximum number of segments

**Complexity Analysis:**
- **Time:** O(2^n) — exponential, as each position can be a cut point or not
- **Space:** O(n) — recursion stack

**Why this could be better:** The brute force approach explores an exponential number of partitions. The greedy approach solves it in O(n) time with a single traversal.

## Optimal Approach: Greedy with Last Occurrence

**Idea:** Record the last occurrence index of each character. Then traverse the string, extending the current segment's end whenever we encounter a character whose last occurrence is further ahead. When the current index reaches the segment's end, we finalize that segment.

**Algorithm:**
1. **First pass:** Build a map/array `last` where `last[c]` = last index where character `c` appears
2. **Second pass:** Traverse the string with `start` and `end` pointers:
   - Extend `end = max(end, last[current_char])`
   - If `i == end`, the current segment is complete:
     - Record segment length: `end - start + 1`
     - Set `start = i + 1` for the next segment
3. Return the list of segment lengths

**Why This Works:** The last occurrence of a character defines the minimum end of any segment containing that character. By extending the segment's end to the furthest last occurrence seen so far, we guarantee that all characters in the current segment appear only within it. And by finalizing the segment as soon as `i == end`, we maximize the number of partitions (any delay would merge segments).

**Complexity Analysis:**
- **Time:** O(n) — two passes through the string
- **Space:** O(1) — fixed-size array of 26 integers (since `s` consists of lowercase English letters)

### Edge Cases

- **Single character (`s = "a"`)**: `end = last['a'] = 0`, `i = 0 == end`, result is `[1]`
- **All unique characters (`s = "abc"`)**: Each character's last occurrence is its own index, so each `i == end` immediately, result is `[1, 1, 1]`
- **All same character (`s = "aaaa"`)**: All last occurrences point to index 3, `end` becomes 3 immediately, one partition with length 4 — `[4]`
- **Character repeats across the entire string**: The whole string becomes one partition — the algorithm correctly handles this

### Maximizing Partitions

The algorithm creates the **maximum** number of partitions because it cuts at the **earliest** possible point — as soon as the current character's last occurrence is within the current segment. Any later cut would merge two or more segments, reducing the count.

```python
def partition_labels(s: str) -> list[int]:
    # First pass: record the last occurrence index of each character
    last = {c: i for i, c in enumerate(s)}

    result = []    # list of segment lengths
    start = 0      # start index of current segment
    end = 0        # end index of current segment (extended as we go)

    # Second pass: traverse and partition greedily
    for i, c in enumerate(s):
        # Extend the end if this character's last occurrence is further
        end = max(end, last[c])

        # If we've reached the end of the current segment
        if i == end:
            result.append(end - start + 1)  # record segment length
            start = i + 1                   # start new segment

    return result
```

```java
class Solution {
    public List<Integer> partitionLabels(String s) {
        // First pass: record last occurrence of each character
        int[] last = new int[26];
        for (int i = 0; i < s.length(); i++) {
            last[s.charAt(i) - 'a'] = i;
        }

        List<Integer> result = new ArrayList<>();
        int start = 0;  // start of current segment
        int end = 0;    // end of current segment (expanding)

        // Second pass: greedy partition
        for (int i = 0; i < s.length(); i++) {
            // Extend end to the last occurrence of current character
            end = Math.max(end, last[s.charAt(i) - 'a']);

            // If current index reaches the segment end, cut here
            if (i == end) {
                result.add(end - start + 1);  // record segment length
                start = i + 1;                // start next segment
            }
        }
        return result;
    }
}
```

```rust
use std::collections::HashMap;

pub fn partition_labels(s: String) -> Vec<i32> {
    // First pass: record the last occurrence of each character
    let mut last = HashMap::new();
    for (i, c) in s.chars().enumerate() {
        last.insert(c, i);
    }

    let mut result = Vec::new();
    let mut start = 0;  // start of current segment
    let mut end = 0;    // expanding end boundary

    // Second pass: greedy partition
    for (i, c) in s.chars().enumerate() {
        // Extend end if this character's last occurrence is further
        end = end.max(*last.get(&c).unwrap());

        // Reached segment boundary — finalize this segment
        if i == end {
            result.push((end - start + 1) as i32);  // segment length
            start = i + 1;                           // begin next segment
        }
    }
    result
}
```

```c
int* partitionLabels(char* s, int* returnSize) {

        }
```

## Key Takeaways

- **Two-pass algorithm**: first pass builds the last-occurrence map; second pass greedily segments
- **Greedy choice**: extending the current segment's end whenever a character's last occurrence lies beyond it ensures each letter appears in at most one part
- **Maximizing partitions**: by finalizing a segment as soon as `i == end`, we create the maximum number of partitions (any smaller partition would violate the constraint)
- **Edge cases**: single character, all unique characters (each is its own partition), all same character (one partition)
- **Space efficiency**: only 26 integers needed for the last-occurrence map (limited to lowercase English letters)
