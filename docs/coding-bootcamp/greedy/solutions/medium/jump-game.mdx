# Jump Game — Solution

## Problem Overview

You are given an integer array `nums`. You start at index 0. Each element `nums[i]` represents your **maximum jump length** from that position. Return `true` if you can reach the **last index**, `false` otherwise.

## Brute Force Approach (Backtracking / DFS)

**Idea:** From each position, try every possible jump length (from 1 to `nums[i]`) recursively to see if any path reaches the last index.

**Algorithm:**
1. Start at index 0
2. From the current index, try all jump lengths from 1 to `nums[i]`
3. Recursively check if any jump reaches the last index
4. Return `true` if any path succeeds

**Complexity Analysis:**
- **Time:** O(2^n) — exponential, as each position can branch to many others
- **Space:** O(n) — recursion stack depth

**Optimized brute force (DP with memoization):**
- Cache results for each index to avoid recomputation
- **Time:** O(n^2) — for each position, check up to `nums[i]` reachable positions
- **Space:** O(n) — memoization array

**Why this could be better:** The DP approach works but is O(n^2). The greedy solution achieves O(n) by tracking only the furthest reachable index rather than all possible paths.

## Optimal Approach: Greedy (Furthest Reach)

**Idea:** Traverse the array left to right, tracking the **furthest index reachable** so far. If we ever reach a position beyond the furthest reachable index, we're stuck. If the furthest reachable index reaches or exceeds the last position, we succeed.

**Algorithm:**
1. Initialize `maxReach = 0` (furthest index reachable from any visited position)
2. For each index `i` from 0 to `n-1`:
   - If `i > maxReach`, return `false` (current position is unreachable)
   - Update `maxReach = max(maxReach, i + nums[i])`
   - If `maxReach >= n - 1`, return `true` (can reach the last index)
3. Return `true` (completed the loop, so last index is reachable)

**Why This Works:** The key insight is that we don't need to know the exact path — we only need to know whether the last index is reachable. If we can reach position `i`, we can reach any position from `i` to `i + nums[i]`. By maintaining the maximum reachable index across all visited positions, we capture the union of all possible jumps without simulating each one. The greedy choice — always extending `maxReach` as far as possible — works because if a position `j` is reachable, it will be reachable by the time we reach it in our left-to-right traversal.

**Complexity Analysis:**
- **Time:** O(n) — single pass through the array
- **Space:** O(1) — constant extra space

## Deep Dive: How the Greedy Strategy Works

### Visual Walkthrough

For `nums = [2, 3, 1, 1, 4]`:

```
Index:  0  1  2  3  4
nums:  [2, 3, 1, 1, 4]

i=0:  maxReach = max(0, 0+2) = 2
i=1:  maxReach = max(2, 1+3) = 4  → reaches last index → return true

Path: index 0 → index 1 (jump 1) → index 4 (jump 3)
```

For `nums = [3, 2, 1, 0, 4]`:

```
Index:  0  1  2  3  4
nums:  [3, 2, 1, 0, 4]

i=0:  maxReach = max(0, 0+3) = 3
i=1:  maxReach = max(3, 1+2) = 3
i=2:  maxReach = max(3, 2+1) = 3
i=3:  maxReach = max(3, 3+0) = 3
i=4:  i=4 > maxReach=3 → return false (index 4 is unreachable)

The 0 at index 3 prevents reaching index 4.
```

### Edge Cases

- **Single element array (`nums = [0]`)**: Loop runs once, `maxReach = max(0, 0+0) = 0 >= 0`, return `true` (already at last index)
- **All zeros except first**: Only the first index is reachable if `nums[0] >= n-1`; otherwise stuck at 0
- **Large jump at start**: `nums = [10, 0, 0, 0, 0, ...]` — single jump reaches the end in one go
- **No possible path despite large jumps**: The array `[0, 1]` — index 0 cannot advance since `nums[0] = 0`
- **Integer overflow**: `i + nums[i]` could overflow in some languages — use a safe comparison or a larger type

### Why Greedy Beats DP

DP would track reachability of each index individually: `dp[i] = OR(dp[i-k] for k in 1..nums[i-k])`. This requires O(n^2) time because each position checks all its predecessors. The greedy approach realizes that we only need the **frontier** — the furthest index we can collectively reach — which simplifies to a single variable updated in O(1) per position.

def can_jump(nums: list[int]) -> bool:
    max_reach = 0  # furthest index reachable from any visited position

    for i in range(len(nums)):

## Key Takeaways

- **Greedy invariant**: `maxReach` always represents the farthest index reachable from any position visited so far
- **Early exit**: return `true` as soon as `maxReach` reaches the last index — no need to continue
- **Stuck detection**: if `i > maxReach` at any point, that position is unreachable — return `false`
- **Alternative view**: this is equivalent to solving "can you reach the end" without DP's O(n^2) overhead, because the greedy choice (always extending `maxReach` as far as possible) is provably optimal
