# Greedy

A **greedy algorithm** builds a solution piece by piece, always choosing
the option that looks best **at the moment** — the locally optimal choice —
hoping it leads to a globally optimal solution.

⏱ 20 minutes

## What You Will Learn

- What the greedy choice property is and when it applies
- How greedy algorithms differ from dynamic programming
- Common problem patterns solvable with greedy strategies
- How to reason about correctness and spot counterexamples

## The Problem

Many optimization problems ask you to find the best solution among many
candidates. The naive approach enumerates all possibilities (exponential
time). Greedy algorithms reduce this to a single pass by making the
best-looking choice at each step and never reconsidering.

For example, scheduling the most meetings in a room: trying every subset
is exponential. Sorting by finish time and always picking the next
compatible meeting gives the optimal answer in O(n log n).

## The Concept

Make the choice that is optimal **right now**, then solve the remaining
subproblem. This works only when the problem has two properties:

1. **Greedy choice property** — a globally optimal solution can be
   assembled from locally optimal choices.
2. **Optimal substructure** — the optimal solution to the whole problem
   contains optimal solutions to its subproblems.

## Greedy vs. Dynamic Programming

Greedy and DP both solve optimization problems, but they differ in how
they explore the solution space.

| Property | Greedy | Dynamic Programming |
|----------|--------|-------------------|
| Decision | One-shot, local choice | Explores all subproblems |
| Optimality | Requires proof of greedy choice property | Guaranteed by exhaustive search |
| Efficiency | Usually O(n) or O(n log n) | Often O(n²) or O(n × W) |
| Memory | O(1) or O(n) | Often O(n) or O(n²) |
| When to use | Subproblems don't overlap, local choice is safe | Overlapping subproblems, optimal substructure |

**Trade-off**: Greedy is faster but only works when the problem has the
**greedy choice property** (a global optimum can be reached by making
locally optimal choices) and **optimal substructure** (an optimal solution
to the whole problem contains optimal solutions to subproblems).

## Complexity

| Metric | Complexity |
|--------|-----------|
| Time | O(n log n) — sorting dominates in most cases; O(n) otherwise |
| Space | O(1) or O(n) — depends on whether sorting is in-place |

## Worked Example: Interval Scheduling

**Problem:** Given `n` intervals `[start_i, end_i]`, select the maximum
number of non-overlapping intervals.

**Greedy strategy:** Always pick the interval with the earliest finish
time.

```
Intervals:  [1,3]  [2,5]  [3,6]  [5,7]  [6,9]

Sorted by end:  [1,3]  [2,5]  [3,6]  [5,7]  [6,9]

Pick [1,3]  ──> last_end = 3
[2,5] overlaps (2 < 3)?  ✗ skip
[3,6] starts at 3 >= 3?  ✓  pick ──> last_end = 6
[5,7] overlaps (5 < 6)?  ✗ skip
[6,9] starts at 6 >= 6?  ✓  pick ──> last_end = 9

Result: 3 intervals  ([1,3], [3,6], [6,9])
```

```python
def interval_scheduling(intervals):
    # Sort by finish time (greedy choice)
    intervals.sort(key=lambda x: x[1])

    count = 0
    last_end = -float('inf')

    for start, end in intervals:
        if start >= last_end:
            count += 1
            last_end = end

    return count
```

**Why it works:** By picking the interval that finishes earliest, we
leave the maximum remaining room for other intervals. This is provably
optimal — any optimal solution's first interval can be replaced with the
earliest-finishing one without reducing the count.

## When to Use Greedy

- The problem has **optimal substructure** — the optimal solution to the
  problem contains optimal solutions to its subproblems.
- The **greedy choice property** holds — a locally optimal choice leads
  to a globally optimal solution (requires proof).
- **No look-ahead needed** — the best immediate choice never backfires.
- The problem involves **scheduling, intervals, or resource allocation**
- You need **minimum spanning trees, Huffman coding, or canonical coin
  change**
