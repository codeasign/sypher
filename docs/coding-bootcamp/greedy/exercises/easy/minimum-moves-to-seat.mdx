# Minimum Moves to Seat Everyone

**Difficulty:** Easy

A classroom has `n` seats and `n` students. You are given two arrays `seats` and `students`, each of length `n`, where `seats[i]` is the position of the `i-th` seat and `students[j]` is the starting position of the `j-th` student.

In one move, a student can move one step left or right. Return the **minimum number of moves** needed to seat every student so that no two students share the same seat.

## Example 1

**Input:** `seats = [3, 1, 5]`, `students = [2, 7, 4]`  
**Output:** `4`  
**Explanation:** Sort both arrays: seats = [1, 3, 5], students = [2, 4, 7]. Assign seat 1 to student at 2 (1 move), seat 3 to student at 4 (1 move), seat 5 to student at 7 (2 moves). Total = 4.

## Example 2

**Input:** `seats = [4, 1, 5, 9]`, `students = [1, 3, 2, 6]`  
**Output:** `7`  
**Explanation:** Sort: seats = [1, 4, 5, 9], students = [1, 2, 3, 6]. Moves: |1-1| + |4-2| + |5-3| + |9-6| = 0 + 2 + 2 + 3 = 7.

## Example 3

**Input:** `seats = [1, 2, 3, 4, 5]`, `students = [5, 4, 3, 2, 1]`  
**Output:** `0`  
**Explanation:** After sorting both, every student is already at their assigned seat.

## Input Specification

The first line contains space-separated integers representing the `seats` array.  
The second line contains space-separated integers representing the `students` array.

## Output Specification

Print a single integer — the minimum total number of moves required.

## Constraints

- `1 <= n <= 100`
- `1 <= seats[i], students[j] <= 100`
- Both arrays have the same length `n`.

## Hints

<details>
<summary>Hint 1</summary>

Think about what happens if you try to assign students to seats in a greedy way. Should the closest student always take the closest seat?

</details>

<details>
<summary>Hint 2</summary>

Sorting both arrays independently is the key insight. Once sorted, the optimal assignment is always to pair the i-th smallest seat with the i-th smallest student.

</details>

<details>
<summary>Hint 3</summary>

After sorting both arrays, the answer is simply the sum of `abs(seats[i] - students[i])` for all indices `i`. This works because any crossing of assignments can only increase the total cost.

</details>

</ProblemEditor>
