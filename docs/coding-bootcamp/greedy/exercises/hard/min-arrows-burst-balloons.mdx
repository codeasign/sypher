# Minimum Number of Arrows to Burst Balloons

There are some spherical balloons taped onto a flat wall that represents the XY-plane. The balloons are represented as a 2D integer array `points` where `points[i] = [x_start, x_end]` denotes a balloon whose **horizontal diameter** stretches between `x_start` and `x_end`. You do not know the exact y-coordinates of the balloons.

Arrows can be shot up **vertically** from different points along the x-axis. A balloon with `x_start` and `x_end` is burst by an arrow shot at `x` if `x_start <= x <= x_end`. There is **no limit** to the number of arrows that can be shot. An arrow once shot keeps traveling up infinitely, bursting any balloons in its path.

Given the array `points`, return the **minimum** number of arrows that must be shot to burst all balloons.

## Examples

**Example 1:**
```
Input: points = [[10,16],[2,8],[1,6],[7,12]]
Output: 2
Explanation: The balloons can be burst by 2 arrows:
- Shoot an arrow at x = 6, bursting balloons [2,8] and [1,6].
- Shoot an arrow at x = 11, bursting balloons [10,16] and [7,12].
```

**Example 2:**
```
Input: points = [[1,2],[3,4],[5,6],[7,8]]
Output: 4
Explanation: Each balloon requires its own arrow since none overlap.
```

**Example 3:**
```
Input: points = [[1,2],[2,3],[3,4],[4,5]]
Output: 2
Explanation: Balloons [1,2] and [2,3] can be burst by one arrow at x = 2.
Balloons [3,4] and [4,5] can be burst by one arrow at x = 4.
```

## Constraints

- `1 <= points.length <= 10^5`
- `points[i].length == 2`
- `-2^31 <= x_start < x_end <= 2^31 - 1`

## Hints

<details>
<summary>Hint 1</summary>

Sort the balloons by their end coordinate. This is the classic "interval scheduling" greedy choice where the optimal strategy is to shoot where it has the most impact.
</details>

<details>
<summary>Hint 2</summary>

Shoot an arrow at the end of the first balloon. Then skip all balloons that overlap with that arrow position and repeat the process for the remaining balloons.
</details>

<details>
<summary>Hint 3</summary>

Two balloons [x1, y1] and [x2, y2] overlap if x2 `<=` y1 (when sorted by end coordinate). Use this property to group overlapping balloons efficiently in a single pass.
</details>
