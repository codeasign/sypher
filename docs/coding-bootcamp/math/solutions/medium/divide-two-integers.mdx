# Divide Two Integers — Solution

## Approach

We use **bit shifting** to perform division efficiently. Instead of subtracting the divisor repeatedly (which would be O(n)), we double the divisor using left shifts until it exceeds the dividend. This gives us the largest multiple we can subtract, accelerating the process.

We handle signs separately and work with negative numbers to avoid overflow when dealing with `-2³¹`.

**Time Complexity:** O(log n) — each step doubles the divisor
**Space Complexity:** O(1)

## Steps

1. Handle overflow: if dividend is `-2³¹` and divisor is `-1`, return `2³¹ - 1`
2. Determine the sign of the result
3. Convert both numbers to negative (to avoid overflow when negating `-2³¹`)
4. While `dividend <= divisor` (both negative):
   - Find the largest multiple: repeatedly double divisor until it would exceed dividend
   - Subtract the largest multiple, add the corresponding power of 2 to quotient
5. Apply the sign and return

## Visual

```
Divide 100 by 7 (both converted to positive for clarity):

Step 1: Find largest multiple
  7 << 1 = 14     (2x)
  14 << 1 = 28    (4x)
  28 << 1 = 56    (8x)
  56 << 1 = 112   (16x — too big! 112 > 100)
  Largest: 56 (8x), subtract: 100 - 56 = 44, quotient = 8

Step 2: Continue with remainder 44
  7 << 1 = 14     (2x)
  14 << 1 = 28    (4x)
  28 << 1 = 56    (8x — too big! 56 > 44)
  Largest: 28 (4x), subtract: 44 - 28 = 16, quotient = 8 + 4 = 12

Step 3: Continue with remainder 16
  7 << 1 = 14     (2x)
  14 << 1 = 28    (4x — too big! 28 > 16)
  Largest: 14 (2x), subtract: 16 - 14 = 2, quotient = 12 + 2 = 14

Step 4: Remainder 2 < 7, stop

Result: 14 (100 / 7 = 14.285... truncates to 14)
```

```python
def divide(dividend: int, divisor: int) -> int:
    # Overflow edge case
    if dividend == -2**31 and divisor == -1:
        return 2**31 - 1

    # Determine sign
    negative = (dividend < 0) != (divisor < 0)

    # Work with negatives to avoid overflow of -2^31
    dividend = -abs(dividend)
    divisor = -abs(divisor)

    quotient = 0
    while dividend <= divisor:  # both negative, so <=
        # Find the largest multiple
        current_div = divisor
        multiple = 1
        while current_div >= (dividend >> 1):  # prevent overflow
            current_div <<= 1
            multiple <<= 1
        # Subtract the largest multiple
        dividend -= current_div
        quotient += multiple

    return -quotient if negative else quotient
```

```rust
fn divide(dividend: i32, divisor: i32) -> i32 {
    // Overflow edge case
    if dividend == i32::MIN && divisor == -1 {
        return i32::MAX;
    }

    // Determine sign
    let negative = (dividend < 0) != (divisor < 0);

    // Work with i64 to avoid overflow
    let mut dividend = (dividend as i64).abs();
    let divisor = (divisor as i64).abs();

    let mut quotient = 0i64;
    while dividend >= divisor {
        // Find the largest multiple
        let mut current_div = divisor;
        let mut multiple = 1i64;
        while current_div <= (dividend >> 1) {
            current_div <<= 1;
            multiple <<= 1;
        }
        // Subtract the largest multiple
        dividend -= current_div;
        quotient += multiple;
    }

    let result = if negative { -quotient } else { quotient };
    result as i32
}
```

```go
func divide(dividend int, divisor int) int {
    // Overflow edge case
    if dividend == math.MinInt32 && divisor == -1 {
        return math.MaxInt32
    }

    // Determine sign
    negative := (dividend < 0) != (divisor < 0)

    // Work with int64 to avoid overflow
    dvd := int64(dividend)
    if dvd < 0 {
        dvd = -dvd
    }
    dvs := int64(divisor)
    if dvs < 0 {
        dvs = -dvs
    }

    quotient := int64(0)
    for dvd >= dvs {
        // Find the largest multiple
        cur := dvs
        mul := int64(1)
        for cur <= (dvd >> 1) {
            cur <<= 1
            mul <<= 1
        }
        // Subtract the largest multiple
        dvd -= cur
        quotient += mul
    }

    if negative {
        quotient = -quotient
    }
    return int(quotient)
}
```

## Key Takeaways

- Repeated subtraction works but is O(n) — too slow for large values
- Doubling the divisor with bit shifts achieves O(log n) time
- Working with negative numbers avoids overflow when handling `-2³¹`
- The overflow edge case `(-2³¹) / (-1)` must be explicitly checked
- The approach is essentially binary long division
