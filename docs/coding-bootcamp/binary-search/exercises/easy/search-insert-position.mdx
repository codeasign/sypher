# Search Insert Position

**Difficulty:** Easy

Given a sorted array of **distinct** integers `nums` and an integer `target`, return the index where `target` is found. If `target` is not in the array, return the index where it **would be inserted** in order.

You must write an algorithm with **O(log n)** runtime complexity.

## Example 1

**Input:** `nums = [1, 3, 5, 6]`, `target = 5`
**Output:** `2`
**Explanation:** `5` is found at index `2`.

## Example 2

**Input:** `nums = [1, 3, 5, 6]`, `target = 2`
**Output:** `1`
**Explanation:** `2` is not in the array. It would be inserted between `1` (index 0) and `3` (index 1), so the answer is `1`.

## Example 3

**Input:** `nums = [1, 3, 5, 6]`, `target = 7`
**Output:** `4`
**Explanation:** `7` is larger than all elements, so it would be inserted at the end, index `4`.

## Input Specification

The first line contains space-separated integers representing the sorted array `nums`.
The second line contains a single integer `target`.

## Output Specification

Print a single integer: the index at which `target` is found or would be inserted.

## Constraints

- `1 <= nums.length <= 10^4`
- `-10^4 <= nums[i] <= 10^4`
- `nums` contains **distinct** values sorted in **ascending** order.
- `-10^4 <= target <= 10^4`

## Hints

<details>
<summary>Hint 1</summary>

A linear scan works but is O(n). To achieve O(log n), think about binary search — the sorted property means you can eliminate half the array at each step.

</details>

<details>
<summary>Hint 2</summary>

Use `left = 0` and `right = len(nums)` (not `len(nums) - 1`). Setting `right` to `len(nums)` lets the pointer naturally land on the insertion index even when target is larger than every element.

</details>

<details>
<summary>Hint 3</summary>

At each step compute `mid = (left + right) // 2`. If `nums[mid] < target`, the answer is to the right: set `left = mid + 1`. Otherwise set `right = mid`. When `left == right` the loop ends and `left` is your answer — whether the target was found or not.

</details>
