# Binary Search

**Difficulty:** Easy

Given a **sorted** array of `n` integers and a `target` value, return the
**index** of the target if it is found. If the target is not present,
return `-1`.

The array is sorted in **ascending** order. You must write an algorithm
with **O(log n)** runtime complexity.

## Example 1

**Input:** `n = 5, target = 3, nums = [1, 2, 3, 4, 5]`  
**Output:** `2`  
**Explanation:** The target `3` is at index 2 (0-based indexing).

## Example 2

**Input:** `n = 5, target = 6, nums = [1, 2, 3, 4, 5]`  
**Output:** `-1`  
**Explanation:** The target `6` is not present in the array.

## Example 3

**Input:** `n = 1, target = 7, nums = [7]`  
**Output:** `0`  
**Explanation:** The single element matches the target at index 0.

## Input Specification

The first line contains two space-separated integers: `n` (the length of
the array) and `target` (the value to search for).

The second line contains `n` space-separated integers representing the
sorted array `nums`.

## Output Specification

Print a single integer: the 0-based index of `target` in the array, or
`-1` if the target is not found.

## Constraints

- `1 <= n <= 10^5`
- `-10^9 <= nums[i] <= 10^9`
- `-10^9 <= target <= 10^9`
- `nums` is sorted in ascending order (may contain duplicates).

## Hints

<details>
<summary>Hint 1</summary>

The array is sorted. Use this property to eliminate half of the
remaining elements with each comparison.
</details>

<details>
<summary>Hint 2</summary>

Maintain two pointers: `left` and `right`. Compute `mid` as the middle
index. If `nums[mid] == target`, return `mid`. If `nums[mid] < target`,
search the right half (`left = mid + 1`). Otherwise, search the left
half (`right = mid - 1`).
</details>

<details>
<summary>Hint 3</summary>

Be careful about integer overflow when computing `mid`. Use
`mid = left + (right - left) / 2` instead of `(left + right) / 2`.
</details>
