# Find Minimum in Rotated Sorted Array

**Difficulty:** Medium

Suppose an array of length `n` sorted in ascending order is **rotated** between 1 and n times. For example, the array `nums = [0, 1, 2, 4, 5, 6, 7]` might become `[4, 5, 6, 7, 0, 1, 2]`.

Given the rotated sorted array `nums`, return the **minimum element** of this array.

You must write an algorithm that runs in **O(log n)** time.

## Example 1

**Input:** `nums = [3, 4, 5, 1, 2]`  
**Output:** `1`  

## Example 2

**Input:** `nums = [4, 5, 6, 7, 0, 1, 2]`  
**Output:** `0`  

## Example 3

**Input:** `nums = [11, 13, 15, 17]`  
**Output:** `11`  

## Input Specification

- A single line of space-separated integers.
- `1 <= nums.length <= 5000`

## Output Specification

Print the minimum element.

## Constraints

- `1 <= nums.length <= 5000`
- `-5000 <= nums[i] <= 5000`
- All elements are unique.
- `nums` is sorted and rotated.

## Hints

<details>
<summary>Hint 1</summary>

Find the pivot point where the array rotates. The middle element helps decide which half contains the minimum.
</details>

<details>
<summary>Hint 2</summary>

If `nums[mid] > nums[right]`, the minimum is in the right half. Otherwise, it's in the left half.
</details>

<details>
<summary>Hint 3</summary>

Compare `nums[mid]` with `nums[right]` (not `nums[left]`) to avoid off-by-one errors.
</details>
