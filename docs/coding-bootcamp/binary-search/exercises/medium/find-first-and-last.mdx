# Find First and Last Position of Element in Sorted Array

**Difficulty:** Medium

Given an array of integers `nums` sorted in non-decreasing order, find the starting and ending position of a given `target` value.

If `target` is not found in the array, return `[-1, -1]`.

You must write an algorithm with **O(log n)** runtime complexity.

## Example 1

**Input:** `nums = [5, 7, 7, 8, 8, 10]`, `target = 8`  
**Output:** `[3, 4]`  

## Example 2

**Input:** `nums = [5, 7, 7, 8, 8, 10]`, `target = 6`  
**Output:** `[-1, -1]`  

## Example 3

**Input:** `nums = []`, `target = 0`  
**Output:** `[-1, -1]`  

## Input Specification

- First line: space-separated integers representing `nums` (may be empty).
- Second line: the target integer.
- `0 <= nums.length <= 10^5`

## Output Specification

Print the first and last index space-separated.

## Constraints

- `0 <= nums.length <= 10^5`
- `-10^9 <= nums[i] <= 10^9`
- `nums` is sorted in non-decreasing order.

## Hints

<details>
<summary>Hint 1</summary>

Write a helper function that finds the first position where `nums[i] >= target`. Use it with different comparisons to find both boundaries.
</details>

<details>
<summary>Hint 2</summary>

The first occurrence uses `nums[mid] >= target` to find the left bound. The last uses `nums[mid] > target`.
</details>

<details>
<summary>Hint 3</summary>

Implement `lower_bound` (first >= target) and `upper_bound` (first > target). The answer is `[lower_bound, upper_bound - 1]`.
</details>
