# Search Insert Position — Solution

## Problem Overview

Given a sorted array of **distinct** integers and a target value, return the index where the target is found, or the index where it would be inserted to maintain sorted order.

## Approach: Binary Search with Inclusive Upper Bound

**Idea:** Use binary search with `right = len(nums)` (not `len(nums) - 1`). This allows the algorithm to naturally return the insertion index when the target is larger than all elements.

**Algorithm:**
1. `left = 0, right = len(nums)`.
2. While `left < right`:
   - `mid = (left + right) // 2`.
   - If `nums[mid] < target`: search right half → `left = mid + 1`.
   - Else: search left half (including mid) → `right = mid`.
3. Return `left` (which equals `right`).

**Complexity:** O(log n) time, O(1) space.

## Key Takeaways

- **Using `right = len(nums)`** instead of `len(nums) - 1` elegantly handles the case where the target belongs at the end.
- **The while condition `left < right`** (not `left <= right`) ensures the loop terminates with `left == right`, which is the insertion point.
- **This algorithm works for both found and not-found cases** — if the target is present, `left` will point to its first occurrence; if not, it points to the insertion index.
