# Median of Two Sorted Arrays — Solution

## Problem Overview

Given two sorted arrays `nums1` and `nums2` of sizes `n` and `m`, return the median of the combined sorted array. The required runtime complexity is O(log(min(n, m))).

## Brute Force Approach: Merge and Find

**Idea:** Merge the two sorted arrays into a single sorted array, then find the median.

**Algorithm:**
1. Merge both arrays using a two-pointer technique into a combined array of size `n + m`
2. If the total size is odd, return the middle element
3. If the total size is even, return the average of the two middle elements

**Complexity Analysis:**
- **Time:** O(n + m) — we merge both arrays
- **Space:** O(n + m) — we create a new merged array

**Why this could be better:** This ignores the fact that we only need the median, not the full sorted array. We can do better by avoiding the merge.

## Better Approach: Two-Pointer Without Full Merge

**Idea:** Use two pointers to traverse both arrays without building the merged array, stopping when we reach the median position.

**Algorithm:**
1. Compute the target median position(s)
2. Walk through both arrays with two pointers, counting elements
3. Stop when we have passed the median position(s)

**Complexity Analysis:**
- **Time:** O(n + m) — we still traverse through half the elements
- **Space:** O(1) — no extra array

## Optimal Approach: Binary Search on the Smaller Array

**Idea:** The median is the element that divides the combined set into two halves of equal size. We can partition both arrays such that the left half contains exactly `(n + m + 1) / 2` elements. Binary search on the **smaller** array to find the correct partition point.

**Algorithm:**
1. Ensure `nums1` is the smaller array (swap if `n > m`)
2. Set `low = 0`, `high = n` (size of the smaller array)
3. While `low <= high`:
   - `partition1 = (low + high) / 2` — elements taken from nums1
   - `partition2 = (n + m + 1) / 2 - partition1` — elements taken from nums2
   - Compute boundary values:
     - `maxLeft1 = nums1[partition1 - 1]` (or -inf if partition1 == 0)
     - `minRight1 = nums1[partition1]` (or +inf if partition1 == n)
     - `maxLeft2 = nums2[partition2 - 1]` (or -inf if partition2 == 0)
     - `minRight2 = nums2[partition2]` (or +inf if partition2 == m)
   - If `maxLeft1 <= minRight2` and `maxLeft2 <= minRight1`:
     - Valid partition found
     - If total is odd: return `max(maxLeft1, maxLeft2)`
     - If total is even: return `(max(maxLeft1, maxLeft2) + min(minRight1, minRight2)) / 2.0`
   - Else if `maxLeft1 > minRight2`: move left: `high = partition1 - 1`
   - Else: move right: `low = partition1 + 1`

**Why This Works:** The median position in the combined array is fixed at `(n + m + 1) / 2` elements from the left. By choosing how many elements to take from the smaller array (`partition1`), we automatically determine how many come from the larger array (`partition2`). The binary search finds the correct partition by ensuring that all elements on the left are less than or equal to all elements on the right.

**Complexity Analysis:**
- **Time:** O(log(min(n, m))) — binary search on the smaller array
- **Space:** O(1) — only a few variables

## Deep Dive: The Partition Approach

### Visual Walkthrough

For the input `nums1 = [1, 3, 5, 7]`, `nums2 = [2, 4, 6, 8, 10, 12]`, let's trace:

```
n = 4, m = 6 (nums1 is smaller)
Total = 10 (even)
Left half size = (10 + 1) / 2 = 5

low = 0, high = 4

Step 1: partition1 = 2, partition2 = 5 - 2 = 3
nums1 left: [1, 3]  |  nums1 right: [5, 7]
nums2 left: [2, 4, 6]  |  nums2 right: [8, 10, 12]

maxLeft1 = 3, minRight1 = 5
maxLeft2 = 6, minRight2 = 8

maxLeft1 (3) <= minRight2 (8)? ✓
maxLeft2 (6) <= minRight1 (5)? ✗ (6 > 5)
→ Need to decrease partition2 (increase partition1)
→ Move right: low = partition1 + 1 = 3

Step 2: partition1 = 3, partition2 = 5 - 3 = 2
nums1 left: [1, 3, 5]  |  nums1 right: [7]
nums2 left: [2, 4]  |  nums2 right: [6, 8, 10, 12]

maxLeft1 = 5, minRight1 = 7
maxLeft2 = 4, minRight2 = 6

maxLeft1 (5) <= minRight2 (6)? ✓
maxLeft2 (4) <= minRight1 (7)? ✓
→ Valid partition found!
→ Even total: median = (max(5, 4) + min(7, 6)) / 2 = (5 + 6) / 2 = 5.5
```

### Edge Cases to Consider

- **One array is empty:** The partition algorithm handles this naturally. The empty array contributes zero elements, and its boundary values use -inf/+inf sentinels.
- **Both arrays empty:** Return 0.0 (or handle as undefined).
- **Single element total:** `(n + m + 1) / 2 = 1`, so the left half has exactly one element. The median is that element.
- **Negative numbers:** The algorithm works the same way, since the arrays are sorted and comparisons work on negatives.
- **All elements in one array:** The partition may take all elements from one array and none from the other. The sentinel values handle this.
- **Identical values across arrays:** The condition `maxLeft1 <= minRight2` and `maxLeft2 <= minRight1` correctly handles equality.

### Why Binary Search on the Smaller Array

We always binary search on the smaller array to minimize the search space. The partition in the larger array is derived as `partition2 = (n + m + 1) / 2 - partition1`. If `partition1` is valid, then `partition2` is automatically valid since it accounts for the remaining elements. The time complexity is O(log(min(n, m))).

class Solution {
    public double findMedian(int[] nums1, int[] nums2) {
        // Ensure nums1 is the smaller array for efficient binary search
        if (nums1.length > nums2.length) {
            return findMedian(nums2, nums1);
        }

        int n = nums1.length;          // Size of smaller array
        int m = nums2.length;          // Size of larger array
        int low = 0;                   // Start of binary search range on nums1
        int high = n;                  // End of binary search range on nums1
        int totalLeft = (n + m + 1) / 2;  // Number of elements in left half

        while (low <= high) {
            int partition1 = (low + high) / 2;      // Elements taken from nums1
            int partition2 = totalLeft - partition1; // Elements taken from nums2

            // Handle edge cases with sentinel values
            int maxLeft1 = (partition1 == 0) ? Integer.MIN_VALUE : nums1[partition1 - 1];
            int minRight1 = (partition1 == n) ? Integer.MAX_VALUE : nums1[partition1];
            int maxLeft2 = (partition2 == 0) ? Integer.MIN_VALUE : nums2[partition2 - 1];
            int minRight2 = (partition2 == m) ? Integer.MAX_VALUE : nums2[partition2];

            // Valid partition found
            if (maxLeft1 <= minRight2 && maxLeft2 <= minRight1) {
                // If total length is odd, median is the max element of the left half
                if ((n + m) % 2 == 1) {
                    return Math.max(maxLeft1, maxLeft2);
                }
                // If total length is even, median is average of the two middle elements
                return (Math.max(maxLeft1, maxLeft2) + Math.min(minRight1, minRight2)) / 2.0;
            }
            // Too many elements taken from nums1: move left
            else if (maxLeft1 > minRight2) {
                high = partition1 - 1;
            }
            // Too few elements taken from nums1: move right
            else {
                low = partition1 + 1;
            }
        }

        return 0.0;  // Should never reach here for valid input
    }
}

func findMedian(nums1 []int, nums2 []int) float64 {
    // Ensure nums1 is the smaller array for efficient binary search
    if len(nums1) > len(nums2) {
        return findMedian(nums2, nums1)
    }

    n := len(nums1)                        // Size of smaller array
    m := len(nums2)                        // Size of larger array
    low := 0                               // Start of binary search on nums1
    high := n                              // End of binary search on nums1
    totalLeft := (n + m + 1) / 2           // Elements needed in left half

    for low <= high {
        partition1 := (low + high) / 2     // Elements taken from nums1
        partition2 := totalLeft - partition1 // Elements taken from nums2

        // Handle edge cases with sentinel values
        var maxLeft1, minRight1, maxLeft2, minRight2 int
        if partition1 == 0 { maxLeft1 = -1 << 31 } else { maxLeft1 = nums1[partition1 - 1] }
        if partition1 == n { minRight1 = (1 << 31) - 1 } else { minRight1 = nums1[partition1] }
        if partition2 == 0 { maxLeft2 = -1 << 31 } else { maxLeft2 = nums2[partition2 - 1] }
        if partition2 == m { minRight2 = (1 << 31) - 1 } else { minRight2 = nums2[partition2] }

        // Valid partition: all left elements <= all right elements
        if maxLeft1 <= minRight2 && maxLeft2 <= minRight1 {
            // Odd total: median is the max of the left half
            if (n + m) % 2 == 1 {
                if maxLeft1 > maxLeft2 { return float64(maxLeft1) }
                return float64(maxLeft2)
            }
            // Even total: median is average of the two middle elements
            maxLeft := maxLeft1
            if maxLeft2 > maxLeft { maxLeft = maxLeft2 }
            minRight := minRight1
            if minRight2 < minRight { minRight = minRight2 }
            return float64(maxLeft + minRight) / 2.0
        }
        // Too many elements from nums1: move partition left
        if maxLeft1 > minRight2 {
            high = partition1 - 1
        } else {
            low = partition1 + 1
        }
    }

    return 0.0  // Should never reach here for valid input
}

## Key Takeaways

- **Binary search on the smaller array** is the key to O(log(min(n, m))) complexity.
- **The partition approach** views the median as a dividing line between left and right halves of the combined sorted array.
- **Sentinel values** (-inf and +inf) elegantly handle edge cases where a partition boundary is at the edge of an array.
- **The condition `maxLeft <= minRight`** across both arrays ensures a valid partition.
- **Handling odd vs even total length** is the final step once the valid partition is found.
