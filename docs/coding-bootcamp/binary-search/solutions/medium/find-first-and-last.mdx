# Find First and Last Position — Solution

## Problem Overview

Find the first and last occurrence of a target in a sorted array in O(log n). This is **binary search with boundary finding**.

## Approach: Two Binary Searches

**Idea:** Use `lower_bound` (first position >= target) and `upper_bound` (first position > target). The answer is `[lower, upper - 1]`.

**Algorithm:**
1. Define `lower_bound(nums, target)`:
   - Binary search for the first index where `nums[i] >= target`.
   - Returns n if no such index exists.
2. Define `upper_bound(nums, target)`:
   - Binary search for the first index where `nums[i] > target`.
   - Returns n if no such index exists.
3. Compute `left = lower_bound(nums, target)`.
4. If `left == n` or `nums[left] != target`, return `[-1, -1]`.
5. Compute `right = upper_bound(nums, target) - 1`.
6. Return `[left, right]`.

**Complexity Analysis:**
- **Time:** O(log n) — two binary searches.
- **Space:** O(1) — constant extra space.

def search_range(nums: list[int], target: int) -> list[int]:
    # Helper: first index where nums[i] >= target
    def lower_bound(nums, target):
        left, right = 0, len(nums)
        while left < right:
            mid = (left + right) // 2
            if nums[mid] < target:
                left = mid + 1  # target is in the right half
            else:
                right = mid     # nums[mid] >= target, search left
        return left

    # Helper: first index where nums[i] > target
    def upper_bound(nums, target):
        left, right = 0, len(nums)
        while left < right:
            mid = (left + right) // 2
            if nums[mid] <= target:
                left = mid + 1  # target is still >=, go right
            else:
                right = mid     # nums[mid] > target, search left
        return left

    left = lower_bound(nums, target)
    # If target not found, return [-1, -1]
    if left == len(nums) or nums[left] != target:
        return [-1, -1]

    right = upper_bound(nums, target) - 1
    return [left, right]

int* searchRange(int* nums, int numsSize, int target, int* returnSize) {
    *returnSize = 2;
    int* result = (int*)malloc(2 * sizeof(int));
    result[0] = -1; result[1] = -1;

    int left = 0, right = numsSize;
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] < target) left = mid + 1;
        else right = mid;
    }
    int lower = left;
    if (lower == numsSize || nums[lower] != target) {
        return result;
    }

    left = 0; right = numsSize;
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] <= target) left = mid + 1;
        else right = mid;
    }
    result[0] = lower;
    result[1] = left - 1;
    return result;
}

## Key Takeaways

- **Lower/upper bound pattern**: The standard library pattern for binary search range queries.
- **Two searches**: Two O(log n) searches are still O(log n).
- **Edge cases**: Empty array, target at ends, all equal values, target not found.
