# Search in Rotated Sorted Array — Solution

## Problem Overview

A sorted array was rotated at some unknown pivot. Given the rotated array and a target, return the index of the target or `-1` if not found. The array has **distinct** values.

## Brute Force Approach: Linear Scan

**Idea:** Scan every element from left to right until you find the target.

**Algorithm:**
1. Loop through each index `i` from `0` to `n-1`
2. If `nums[i] == target`, return `i`
3. Return `-1` if not found

**Complexity Analysis:**
- **Time:** O(n) — examines every element in the worst case
- **Space:** O(1) — no extra space

**Why this could be better:** This ignores the partially sorted structure. The rotation preserves the sorted order within two contiguous segments, which we can exploit.

## Better Approach: Find Pivot Then Binary Search

**Idea:** First find the pivot (the smallest element) using binary search, then perform binary search on the appropriate half.

**Algorithm:**
1. Find the pivot index using a modified binary search
2. Decide which half of the array the target belongs to
3. Perform standard binary search on that half

**Complexity Analysis:**
- **Time:** O(log n) — two binary searches, each O(log n)
- **Space:** O(1)

## Optimal Approach: One-Pass Modified Binary Search

**Idea:** Perform a single binary search pass. At each step, at least one half of the array is fully sorted. Determine which half is sorted, then check if the target lies in that half.

**Algorithm:**
1. Initialize `left = 0`, `right = n - 1`
2. While `left <= right`:
   - Compute `mid = left + (right - left) / 2`
   - If `nums[mid] == target`, return `mid`
   - If the left half is sorted (`nums[left] <= nums[mid]`):
     - If `nums[left] <= target < nums[mid]`, search left: `right = mid - 1`
     - Otherwise, search right: `left = mid + 1`
   - Else (the right half is sorted):
     - If `nums[mid] < target <= nums[right]`, search right: `left = mid + 1`
     - Otherwise, search left: `right = mid - 1`
3. Return `-1`

**Why This Works:** In a rotated sorted array, when you examine the middle element, at least one of the two halves (left or right) must be completely sorted. If the left half is sorted, we can check whether the target lies within its range using standard binary search logic. If not, the target must be in the other half. The same reasoning applies when the right half is sorted.

**Complexity Analysis:**
- **Time:** O(log n) — each iteration halves the search space
- **Space:** O(1) — only a few integer variables

## Deep Dive: How the Modified Binary Search Works

### Visual Walkthrough

For the input `n = 7, target = 2, nums = [6, 7, 1, 2, 3, 4, 5]`, let's trace the algorithm:

```
Initial: left = 0, right = 6
nums = [6, 7, 1, 2, 3, 4, 5]
         ↑                    ↑
       left                 right

Step 1: mid = 0 + (6 - 0) / 2 = 3
nums[mid] = 2
nums[left] = 6, nums[mid] = 2
Is left half sorted? nums[0] <= nums[3]? 6 <= 2? No → right half is sorted

Right half: [2, 3, 4, 5] (indices 3-6)
Is target in right half? nums[3] < target <= nums[6]? 2 < 2 <= 5? No (2 is not < 2)
So search left: right = mid - 1 = 2

nums = [6, 7, 1, 2, 3, 4, 5]
         ↑     ↑
       left  right

Step 2: mid = 0 + (2 - 0) / 2 = 1
nums[mid] = 7
nums[left] = 6, nums[mid] = 7
Is left half sorted? 6 <= 7? Yes → left half is sorted

Left half: [6, 7] (indices 0-1)
Is target in left half? nums[0] <= target < nums[1]? 6 <= 2 < 7? No
So search right: left = mid + 1 = 2

nums = [6, 7, 1, 2, 3, 4, 5]
               ↑
             left=right=2

Step 3: mid = 2
nums[mid] = 1
nums[mid] != target (1 != 2)

Is left half sorted? nums[2] <= nums[2]? Yes (single element)
Is target in left half? nums[2] <= target < nums[2]? 1 <= 2 < 1? No
So search right: left = 3

nums = [6, 7, 1, 2, 3, 4, 5]
                  ↑
               left=right=3

Step 4: mid = 3
nums[mid] = 2 == target → return 3
```

### Edge Cases to Consider

- **No rotation (k = 0):** The left half is always sorted, so the algorithm behaves exactly like standard binary search.
- **Rotation at the middle:** Both halves are sorted, each with at least one element.
- **Single element:** `left == right == mid == 0`. Check if `nums[0] == target`.
- **Target not present:** The loop exits when `left > right`, returning `-1`.
- **Minimum element at index 0 (full rotation):** The array is effectively sorted. The algorithm identifies the left half as sorted and proceeds normally.
- **Two elements rotated:** e.g., `[2, 1]`. `left = 0, right = 1, mid = 0`. Left half sorted? `nums[0] <= nums[0]` is trivially true. Check bounds.

### Why the One-Pass Approach is Elegant

Finding the pivot and then binary searching requires two passes. The one-pass approach determines the sorted half on the fly and uses a single pass. It handles all cases uniformly without needing to find the pivot explicitly.

class Solution {
    public int search(int[] nums, int target) {
        int left = 0;                              // Start of search range
        int right = nums.length - 1;               // End of search range

        while (left <= right) {                     // Continue while range is valid
            int mid = left + (right - left) / 2;   // Compute mid to avoid overflow

            if (nums[mid] == target) {             // Found the target at mid
                return mid;
            }

            // Check if the left half is sorted
            if (nums[left] <= nums[mid]) {          // Left half is fully sorted
                // If target is within the sorted left half, search left
                if (nums[left] <= target && target < nums[mid]) {
                    right = mid - 1;               // Narrow search to left half
                } else {
                    left = mid + 1;                // Target is in the right half
                }
            } else {                                // Right half is fully sorted
                // If target is within the sorted right half, search right
                if (nums[mid] < target && target <= nums[right]) {
                    left = mid + 1;                // Narrow search to right half
                } else {
                    right = mid - 1;               // Target is in the left half
                }
            }
        }

        return -1;                                 // Target not found
    }
}

## Key Takeaways

- **At least one half is always sorted** in a rotated sorted array. Use this property to decide which direction to search.
- **The one-pass approach** combines pivot detection and binary search into a single pass, achieving O(log n) with minimal code.
- **The condition `nums[left] <= nums[mid]`** determines whether the left half is sorted. If true, the left half is sorted; otherwise, the right half is sorted.
- **Boundary checks** (`nums[left] <= target < nums[mid]`) use the sorted property to determine if the target lies in that half.
- This technique generalizes to many rotation-based problems (finding minimum in rotated array, etc.).
