# Prefix Count

**Difficulty:** Easy

Given a list of words and a string `prefix`, return the **count** of words in the list that start with the given prefix.

A word is considered to start with `prefix` if the word's first `len(prefix)` characters exactly match `prefix`. A word that equals the prefix exactly also counts.

## Example 1

**Input:**
```
apple app application apt
app
```

**Output:**
```
3
```

**Explanation:** `apple`, `app`, and `application` all start with `"app"`. The word `apt` does not start with `"app"`.

## Example 2

**Input:**
```
test testing tested tester
test
```

**Output:**
```
4
```

**Explanation:** All four words start with `"test"`.

## Example 3

**Input:**
```
abc def ghi
xyz
```

**Output:**
```
0
```

**Explanation:** No word starts with `"xyz"`.

## Input Specification

The first line contains space-separated words.  
The second line contains the prefix string.

## Output Specification

Print a single integer — the number of words that start with the given prefix.

## Constraints

- `1 <= words.length <= 10^4`
- `1 <= words[i].length, prefix.length <= 100`
- All words and the prefix consist only of lowercase English letters

## Hints

<details>
<summary>Hint 1</summary>

You can solve this without a trie by using each language's built-in string `startsWith` or `startswith` method. However, the trie solution teaches you how prefix lookups work at a structural level.

</details>

<details>
<summary>Hint 2</summary>

To use a trie: insert every word into the trie, tracking a **count** at each node of how many inserted words passed through it during insertion. When querying the prefix, walk down the trie following the prefix characters. The count stored at the last node of the prefix is your answer.

</details>

<details>
<summary>Hint 3</summary>

If the prefix does not exist in the trie (i.e., you encounter a missing child while traversing), the count is 0 — no word starts with that prefix.

</details>
