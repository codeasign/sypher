# Word Search II

**Difficulty:** Medium

Given an `m x n` board of characters and a list of strings `words`, return all words on the board.

Each word must be constructed from letters of sequentially adjacent cells, where adjacent cells are horizontally or vertically neighboring. The same letter cell may not be used more than once in a word.

## Example 1

**Input:**  
```
board = [
  ["o","a","a","n"],
  ["e","t","a","e"],
  ["i","h","k","r"]
]
words = ["oath","pea","eat","rain"]
```
**Output:** `["eat","oath"]`

## Example 2

**Input:**
```
board = [
  ["a","b"],
  ["c","d"]
]
words = ["ab","cd"]
```
**Output:** `["ab","cd"]`

## Input Specification

First line: two integers `rows` and `cols`.  
Next `rows` lines: each contains `cols` characters separated by spaces.  
Last line: space-separated list of words to find.

## Output Specification

Print the found words, sorted alphabetically, separated by spaces. If no words are found, print an empty line.

## Constraints

- `1 <= m, n <= 12`
- `1 <= words.length <= 3 * 10⁴`
- `1 <= words[i].length <= 10`
- `board` and `words[i]` consist only of lowercase English letters
- All `words[i]` are unique

## Hints

<details>
<summary>Hint 1</summary>

A naive DFS for each word would be too slow. Use a Trie to store all words and prune the search early.
</details>

<details>
<summary>Hint 2</summary>

Build a Trie from the word list. As you traverse the board, check the current path against the Trie. If the current prefix doesn't exist in the Trie, backtrack immediately.
</details>

<details>
<summary>Hint 3</summary>

When you find a word, mark it as found so you don't add duplicates. You can remove the word from the Trie to avoid searching for it again.
</details>

<details>
<summary>Hint 4</summary>

Use a visited set (or modify the board in-place) to avoid reusing the same cell within a single DFS path.
</details>
