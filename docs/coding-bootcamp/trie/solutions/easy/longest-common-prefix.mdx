# Longest Common Prefix — Solution

## Problem Overview

Given an array of strings, find the longest common prefix shared by all strings. If no common prefix exists, return an empty string.

## Approach 1: Horizontal Scanning

**Idea:** Start with the first string as the initial prefix, then repeatedly trim it against each subsequent string.

**Algorithm:**
1. If the array is empty, return `""`.
2. Set `prefix = strs[0]`.
3. For each remaining string `s`, check if `s` starts with `prefix`. If not, remove the last character from `prefix` and check again.
4. Continue until `prefix` is empty or all strings have been processed.

**Complexity:**
- **Time:** O(N * L) where N is the number of strings and L is the length of the common prefix
- **Space:** O(1)

## Approach 2: Trie (Optimal for Prefix-Intensive Problems)

**Idea:** Insert all strings into a Trie. Then traverse from the root, following nodes that have exactly one child and are not an end-of-word, until reaching a branching point.

**Algorithm:**
1. Insert all strings into a Trie.
2. Start from the root. While the current node has exactly one child and is not an end-of-word, append the child's character to the result and move to the child.
3. Return the accumulated prefix.

**Complexity:**
- **Time:** O(N * L) to build the Trie, O(L) to find the prefix
- **Space:** O(N * L) for the Trie

**Why This Works:** The longest common prefix is the path from the root where every node has only one child (no branching occurs). When a node has more than one child, the strings diverge at that point.

## Deep Dive

### Edge Cases

- **Empty array:** Return `""` immediately.
- **Single string:** The entire string is the common prefix.
- **No common prefix:** Return `""` after finding the first mismatch at index 0.
- **All strings identical:** Return the full string.
- **Strings of varying lengths:** The common prefix cannot exceed the shortest string.

### Trie vs. Horizontal Scanning

The horizontal scanning approach is simpler and uses O(1) extra space, making it the better choice for most scenarios. The Trie approach shines when you need to do multiple prefix queries on the same set of strings, as the Trie can be reused.

def longest_common_prefix(strs: list[str]) -> str:
    if not strs:
        return ""

    # Start with the first string as the prefix
    prefix = strs[0]

    for s in strs[1:]:
        # Trim the prefix until s starts with it
        while not s.startswith(prefix):
            prefix = prefix[:-1]  # Remove last character
            if not prefix:
                return ""         # No common prefix

    return prefix

        for (int i = 1; i < strs.length; i++) {

                prefix = prefix.substring(0, prefix.length() - 1);  // Remove last char

## Key Takeaways

- **Horizontal scanning** is the simplest approach: start with the first string and trim it against each subsequent string
- The **Trie approach** is useful when you need to find common prefixes across multiple queries on the same data set
- The common prefix is bounded by the shortest string in the array
- The prefix can only shrink, never grow, as you process more strings
