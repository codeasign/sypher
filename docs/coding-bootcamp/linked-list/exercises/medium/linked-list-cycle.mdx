# Linked List Cycle

**Difficulty:** Medium

Given `head`, the head of a linked list, determine if the linked list has a cycle in it.

A cycle occurs when a node's `next` pointer points back to a previous node in the list, forming a loop. Return `true` if there is a cycle, otherwise `false`.

## Example 1

**Input:** `head = [3, 2, 0, -4]`, `pos = 1`  
**Output:** `true`  
**Explanation:** The tail node (-4) connects back to the node at index 1 (value 2), forming a cycle.

## Example 2

**Input:** `head = [1, 2]`, `pos = 0`  
**Output:** `true`  
**Explanation:** The tail node (2) connects back to the head (1), forming a cycle.

## Example 3

**Input:** `head = [1]`, `pos = -1`  
**Output:** `false`  
**Explanation:** There is no cycle — the single node points to null.

## Input Specification

The first line contains an integer `n` (the number of nodes). The second line contains `n` space-separated integers representing the node values. The third line contains an integer `pos` (-1 for no cycle, or a 0-based index indicating where the tail connects).

## Output Specification

Print `true` if the list has a cycle, otherwise `false`.

## Constraints

- `0 <= n <= 10^5`
- `-10^9 <= Node.val <= 10^9`
- `-1 <= pos < n`

## Hints

<details>
<summary>Hint 1</summary>

Think about what happens when two runners race on a circular track. The faster runner will eventually lap the slower one.
</details>

<details>
<summary>Hint 2</summary>

Use two pointers: one moving one step at a time (slow) and one moving two steps at a time (fast). If there is a cycle, they will meet.
</details>

<details>
<summary>Hint 3</summary>

If the fast pointer reaches `null` (or `None`), there is no cycle. This is known as Floyd's Tortoise and Hare algorithm.
</details>

</ProblemEditor>
