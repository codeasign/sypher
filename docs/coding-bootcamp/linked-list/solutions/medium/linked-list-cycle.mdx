# Linked List Cycle — Solution

## Problem Overview

Given the `head` of a linked list, determine whether the list contains a cycle. A cycle exists when a node's `next` pointer points back to an earlier node in the list, forming a loop. Return `true` if a cycle exists, `false` otherwise.

## Brute Force Approach

**Idea:** Use a hash set to record every node visited. If we encounter a node that is already in the set, there is a cycle. If we reach `null`, there is no cycle.

**Algorithm:**
1. Initialize an empty hash set `visited`
2. Start at `head` and traverse the list
3. For each node, check if it is already in `visited`:
   - If yes, return `true` (cycle detected)
   - If no, add it to `visited` and move to the next node
4. If we reach `null`, return `false`

**Complexity Analysis:**
- **Time:** O(n) — each node is visited at most once
- **Space:** O(n) — we store up to n node references in the hash set

**Why this could be better:** The hash set approach works and is simple to understand, but it uses O(n) extra space. The optimal approach uses only O(1) space.

## Optimal Approach: Floyd's Tortoise and Hare (Two Pointers)

**Idea:** Use two pointers that move at different speeds. The slow pointer (`tortoise`) moves one step at a time, and the fast pointer (`hare`) moves two steps at a time. If there is a cycle, the fast pointer will eventually lap the slow pointer and they will meet. If there is no cycle, the fast pointer will reach `null`.

**Algorithm:**
1. Initialize `slow = head` and `fast = head`
2. While `fast` and `fast.next` are not null:
   - Move `slow` one step: `slow = slow.next`
   - Move `fast` two steps: `fast = fast.next.next`
   - If `slow == fast`, return `true` (cycle detected)
3. Return `false` (fast reached null, no cycle)

**Why This Works:** Imagine two runners on a circular track. The faster runner will eventually lap the slower one. If the track is straight (no cycle), the faster runner reaches the end first. Mathematically, if there is a cycle of length `C`, the fast pointer moves 2 steps per iteration while the slow moves 1 — the gap closes by 1 each step, so they will meet within `C` steps after entering the cycle.

**Complexity Analysis:**
- **Time:** O(n) — each node is visited at most once by each pointer
- **Space:** O(1) — only two pointer variables

## Deep Dive: Floyd's Cycle Detection

### Visual Walkthrough

For the list `3 -> 2 -> 0 -> -4 -> (back to node at index 1, value 2)`:

```
Step 0: slow = 3, fast = 3 (both start at head)
Step 1: slow = 2, fast = 0 (slow moves 1, fast moves 2)
Step 2: slow = 0, fast = 2 (slow moves 1, fast moves 2, within the cycle)
Step 3: slow = -4, fast = 0 (slow moves 1, fast moves 2)
Step 4: slow = 2, fast = -4 (slow moves 1, fast moves 2)
Step 5: slow = 0, fast = 0 (MEET! Cycle detected)
```

### Why the Pointers Always Meet

In a cycle of length `C`, the distance between the slow and fast pointers decreases by 1 each iteration (fast gains 1 step on slow). Starting from the first node inside the cycle, the maximum distance is `C - 1`. Therefore, within at most `C - 1` additional steps, the pointers will coincide. This guarantee holds regardless of where the cycle starts.

### Edge Cases to Consider

- **Empty list (head = null):** The while condition `fast != null && fast.next != null` fails immediately, returning `false`. Correct.
- **Single node, no cycle:** `fast.next` is null, so the loop exits and returns `false`. Correct.
- **Single node, cycle to itself:** `fast.next == head` (the node points to itself). The loop runs once: `slow = head`, `fast = head.next` (which is `head`). They are equal, so `true` is returned. Correct.
- **Two nodes, cycle at head:** The tail connects back to the head. The pointers will meet within the cycle. Correct.
- **Large list without cycle:** The fast pointer reaches `null` and the function returns `false`. Correct.

### Detecting the Cycle Start

Floyd's algorithm can also find the node where the cycle begins. After the pointers meet, reset `slow` to `head` and move both pointers one step at a time. The point where they meet again is the start of the cycle. This is a common follow-up question.

class Solution {
    public boolean hasCycle(ListNode head) {
        if (head == null) return false;          // Empty list has no cycle

        ListNode slow = head;                    // Tortoise: moves 1 step at a time
        ListNode fast = head;                    // Hare: moves 2 steps at a time

        while (fast != null && fast.next != null) {  // Fast pointer needs to be able to move 2 steps
            slow = slow.next;                        // Tortoise advances by 1
            fast = fast.next.next;                   // Hare advances by 2
            if (slow == fast) {                      // If they meet, a cycle exists
                return true;
            }
        }
        return false;                                // Fast hit the end — no cycle
    }
}

fn has_cycle(head: Option<Box<ListNode>>) -> bool {
    let mut slow = &head;                     // Tortoise: borrow the head reference
    let mut fast = &head;                     // Hare: borrow the head reference

    while fast.is_some() && fast.as_ref().unwrap().next.is_some() {
        slow = &slow.as_ref().unwrap().next;              // Tortoise advances by 1
        fast = &fast.as_ref().unwrap().next.as_ref().unwrap().next;  // Hare advances by 2

        // Compare pointer addresses: if both point to the same node, cycle exists
        if std::ptr::eq(slow.as_ref().unwrap() as *const _,
                        fast.as_ref().unwrap() as *const _) {
            return true;
        }
    }
    false                                              // Fast reached null — no cycle
}

## Key Takeaways

- **Floyd's Tortoise and Hare** is the standard O(n) time, O(1) space cycle detection algorithm
- Two pointers moving at different speeds will always meet if a cycle exists
- The fast pointer must check both `fast != null` and `fast.next != null` before advancing
- This algorithm also extends to finding the start of the cycle (a common follow-up)
- The same pattern is used for detecting cycles in other data structures (arrays, graphs)
