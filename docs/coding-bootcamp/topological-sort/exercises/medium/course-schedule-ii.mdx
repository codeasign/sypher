# Course Schedule II

**Difficulty:** Medium

There are a total of `numCourses` courses you have to take, labeled from `0` to `numCourses - 1`. You are given `prerequisites` where `prerequisites[i] = [a, b]` indicates that you must take course `b` before course `a`.

Return the ordering of courses you should take to finish all courses. If it is impossible (the graph has a cycle), return an empty array.

## Example 1

**Input:** `numCourses = 2, prerequisites = [[1, 0]]`

**Output:** `[0, 1]`

**Explanation:** Take course 0 first, then course 1.

## Example 2

**Input:** `numCourses = 4, prerequisites = [[1, 0], [2, 0], [3, 1], [3, 2]]`

**Output:** `[0, 1, 2, 3]` or `[0, 2, 1, 3]`

**Explanation:** Both are valid orderings.

## Example 3

**Input:** `numCourses = 2, prerequisites = [[1, 0], [0, 1]]`

**Output:** `[]`

**Explanation:** A cycle exists, no valid ordering.

## Input Specification

The first line contains `numCourses`. The second line contains `p`, the number of prerequisites. The next `p` lines each contain two integers `a` and `b` for `[a, b]`. Output the order as space-separated integers on one line, or an empty line if impossible.

## Output Specification

Print the course order as space-separated integers on one line. If impossible, print an empty line.

## Constraints

- `1 <= numCourses <= 2000`
- `0 <= p <= 5000`
- All prerequisite pairs are unique

## Hints

<details>
<summary>Hint 1</summary>

This is the direct application of Kahn's algorithm. Build an adjacency list and an in-degree array.
</details>

<details>
<summary>Hint 2</summary>

Enqueue all courses with in-degree 0. Process them one by one, decrementing in-degrees of their dependents. If a dependent's in-degree becomes 0, enqueue it.
</details>

<details>
<summary>Hint 3</summary>

After processing, if the result array length is less than numCourses, a cycle exists — return an empty array.
</details>
