# Course Schedule

**Difficulty:** Easy

There are a total of `numCourses` courses you have to take, labeled from `0` to `numCourses - 1`. You are given an array `prerequisites` where `prerequisites[i] = [a, b]` indicates that you **must** take course `b` first before you can take course `a` (course `b` is a prerequisite of course `a`).

Determine if it is possible to finish all courses.

## Example 1

**Input:** `numCourses = 2, prerequisites = [[1, 0]]`

**Output:** `true`

**Explanation:** Take course 0 first, then course 1. All courses can be completed.

## Example 2

**Input:** `numCourses = 2, prerequisites = [[1, 0], [0, 1]]`

**Output:** `false`

**Explanation:** Course 1 requires course 0, and course 0 requires course 1. This is a cycle; no valid order exists.

## Input Specification

The first line contains an integer `numCourses`. The second line contains an integer `p`, the number of prerequisites. The next `p` lines each contain two integers `a` and `b` representing a prerequisite `[a, b]`.

## Output Specification

Print `1` if all courses can be finished, otherwise `0`.

## Constraints

- `1 <= numCourses <= 2000`
- `0 <= p <= 5000`
- `0 <= a, b < numCourses`
- All prerequisite pairs are unique

## Hints

<details>
<summary>Hint 1</summary>

This is a cycle detection problem in a directed graph. Represent each course as a vertex and each prerequisite as a directed edge from `b` (prerequisite) to `a` (dependent course).
</details>

<details>
<summary>Hint 2</summary>

Kahn's algorithm works well here. Compute in-degrees, enqueue vertices with in-degree 0, and process them. If the number of processed vertices equals `numCourses`, there is no cycle.
</details>

<details>
<summary>Hint 3</summary>

Alternatively, use DFS with three states: unvisited, visiting, and visited. If you encounter a "visiting" vertex during DFS, a cycle exists.
</details>
