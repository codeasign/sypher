# Find if Path Exists in Graph

**Difficulty:** Easy

There is a bi-directional graph with `n` vertices, labeled from `0` to `n - 1`. You are given `edges` where `edges[i] = [u, v]` indicates that there is an undirected edge between vertex `u` and vertex `v`.

Determine if there is a valid path from `source` to `destination`.

## Example 1

**Input:** `n = 3, edges = [[0, 1], [1, 2]], source = 0, destination = 2`

**Output:** `true`

**Explanation:** Path: 0 → 1 → 2

## Example 2

**Input:** `n = 6, edges = [[0, 1], [0, 2], [3, 5], [5, 4], [4, 3]], source = 0, destination = 5`

**Output:** `false`

**Explanation:** There is no path between vertex 0 and vertex 5 — the graph has two disconnected components.

## Input Specification

The first line contains an integer `n`. The second line contains an integer `m`, the number of edges. The next `m` lines each contain two integers `u` and `v` representing an edge. The last line contains two integers `source` and `destination`.

## Output Specification

Print `1` if a path exists, otherwise `0`.

## Constraints

- `1 <= n <= 2000`
- `0 <= m <= 5000`
- `0 <= u, v < n`
- `0 <= source, destination < n`

## Hints

<details>
<summary>Hint 1</summary>

Build an adjacency list from the edges. Each edge is undirected, so add both directions.
</details>

<details>
<summary>Hint 2</summary>

Use DFS (recursive or iterative) or BFS starting from the source. Keep a visited set to avoid revisiting nodes.
</details>

<details>
<summary>Hint 3</summary>

If DFS or BFS reaches the destination, return true. If all reachable nodes are explored without finding the destination, return false.
</details>
