# Remove Duplicate Letters — Solution

## Problem Overview

Given a string `s`, remove duplicate letters so that every letter appears exactly once. Return the result as the **smallest lexicographical subsequence** possible while preserving the original relative order.

## Brute Force Approach

**Idea:** Generate all possible subsequences that contain each unique character exactly once, then pick the lexicographically smallest one.

**Algorithm:**
1. Find all unique characters in `s`
2. Generate all subsequences of `s` that contain each unique character exactly once
3. Compare lexicographically and return the smallest

**Complexity Analysis:**
- **Time:** O(2^n) — exponential, completely impractical for n > 20
- **Space:** O(2^n) — storing all valid subsequences

**Why this is suboptimal:** Generating all subsequences is exponential. The monotonic stack greedy approach solves this in O(n) time.

## Optimal Approach: Monotonic Stack with Greedy Selection

**Idea:** Use a monotonic increasing stack to build the result character by character. At each step, if the current character is smaller than the top of the stack and the character on the stack can appear later in the string, pop it (since we can add it later in a better position). Track which characters are already in the result to avoid duplicates.

**Algorithm:**
1. Count the frequency of each character in `s`
2. Initialize an empty stack (or list) for the result and a boolean array `inResult` to track which characters are in the result
3. For each character `c` in `s`:
   - Decrement the frequency of `c`
   - If `c` is already in the result, skip it
   - While the stack is not empty and `c < stack.top()` and `frequency[stack.top()] > 0`:
     - Pop the top character from the stack and mark it as not in the result
   - Push `c` onto the stack and mark it as in the result
4. Return the stack contents as a string

**Why This Works:** The monotonic increasing stack ensures the result is as lexicographically small as possible. By only popping characters that can appear later (frequency > 0), we guarantee we can still include all unique characters.

**Complexity Analysis:**
- **Time:** O(n) — each character is pushed and popped at most once
- **Space:** O(1) — the stack and counter both use at most 26 slots (lowercase alphabet)

### Key Insight: Why Greedy Works

The greedy algorithm works because at each position we make the locally optimal choice: if a larger character on the stack can appear later, we postpone it and use the smaller character now. The frequency counts guarantee we won't lose any unique character.

### Edge Cases

- **Already sorted ascending**: All characters are already in order, no popping occurs. Result is the input itself with duplicates removed.
- **Already sorted descending**: Each character pops all previous characters (if they can appear later). Result is essentially the reverse order of first occurrences.
- **Single character**: Return the character itself.
- **All same character**: Return a single copy of that character.
- **Full alphabet in order**: All 26 letters in ascending or descending order test the two extremes of the algorithm.

import java.util.Stack;

class Solution {
    public String removeDuplicateLetters(String s) {
        // Step 1: Count frequency of each character
        int[] freq = new int[26];
        for (char c : s.toCharArray()) {
            freq[c - 'a']++;
        }

        // Step 2: Track which characters are already in the result
        boolean[] inResult = new boolean[26];
        Stack<Character> stack = new Stack<>();  // Monotonic increasing stack

        // Step 3: Process each character
        for (char c : s.toCharArray()) {
            freq[c - 'a']--;  // Decrement frequency

            // Skip if already in the result
            if (inResult[c - 'a']) continue;

            // Pop larger characters that can appear later
            while (!stack.isEmpty() && c < stack.peek() && freq[stack.peek() - 'a'] > 0) {
                inResult[stack.pop() - 'a'] = false;
            }

            // Push current character
            stack.push(c);
            inResult[c - 'a'] = true;
        }

        // Step 4: Build result string
        StringBuilder sb = new StringBuilder();
        for (char c : stack) sb.append(c);
        return sb.toString();
    }
}

            st.push(c);

        while (!st.empty()) {
            result = st.top() + result;
            st.pop();
        }
        return result;
    }

def remove_duplicate_letters(s: str) -> str:
    # Step 1: Count frequency of each character
    freq = [0] * 26
    for c in s:
        freq[ord(c) - ord('a')] += 1

    # Step 2: Track which characters are already in the result
    in_result = [False] * 26
    stack = []  # Monotonic increasing stack

    # Step 3: Process each character
    for c in s:
        idx = ord(c) - ord('a')
        freq[idx] -= 1  # Decrement frequency

        # Skip if already in the result
        if in_result[idx]:
            continue

        # Pop larger characters that can appear later
        while stack and c < stack[-1] and freq[ord(stack[-1]) - ord('a')] > 0:
            in_result[ord(stack.pop()) - ord('a')] = False

        # Push current character
        stack.append(c)
        in_result[idx] = True

    # Step 4: Build result string
    return ''.join(stack)

fn remove_duplicate_letters(s: &str) -> String {
    // Step 1: Count frequency of each character
    let mut freq = [0i32; 26];
    for b in s.bytes() {
        freq[(b - b'a') as usize] += 1;
    }

    // Step 2: Track which characters are already in the result
    let mut in_result = [false; 26];
    let mut stack: Vec<u8> = Vec::new();  // Monotonic increasing stack

    // Step 3: Process each character
    for b in s.bytes() {
        let idx = (b - b'a') as usize;
        freq[idx] -= 1;  // Decrement frequency

        // Skip if already in the result
        if in_result[idx] {
            continue;
        }

        // Pop larger characters that can appear later
        while let Some(&top) = stack.last() {
            if b >= top || freq[(top - b'a') as usize] == 0 {
                break;
            }
            in_result[(stack.pop().unwrap() - b'a') as usize] = false;
        }

        // Push current character
        stack.push(b);
        in_result[idx] = true;
    }

    // Step 4: Build result string
    String::from_utf8(stack).unwrap()
}

#include <stdlib.h>
#include <string.h>

char* removeDuplicateLetters(char* s) {
    int n = strlen(s);
    // Step 1: Count frequency of each character
    int freq[26] = {0};
    for (int i = 0; i < n; i++) freq[s[i] - 'a']++;

    // Step 2: Track which characters are already in the result
    int inResult[26] = {0};
    char* stack = (char*)malloc((n + 1) * sizeof(char));
    int top = -1;  // Stack pointer

    // Step 3: Process each character
    for (int i = 0; i < n; i++) {
        int idx = s[i] - 'a';
        freq[idx]--;  // Decrement frequency

        // Skip if already in the result
        if (inResult[idx]) continue;

        // Pop larger characters that can appear later
        while (top >= 0 && s[i] < stack[top] && freq[stack[top] - 'a'] > 0) {
            inResult[stack[top] - 'a'] = 0;
            top--;
        }

        // Push current character
        stack[++top] = s[i];
        inResult[idx] = 1;
    }

    // Step 4: Build result string
    stack[++top] = '\0';
    return stack;
}

using System.Collections.Generic;

class Solution {
    public string RemoveDuplicateLetters(string s) {
        // Step 1: Count frequency of each character
        int[] freq = new int[26];
        foreach (char c in s) freq[c - 'a']++;

        // Step 2: Track which characters are already in the result
        bool[] inResult = new bool[26];
        Stack<char> stack = new Stack<char>();  // Monotonic increasing stack

        // Step 3: Process each character
        foreach (char c in s) {
            int idx = c - 'a';
            freq[idx]--;  // Decrement frequency

            // Skip if already in the result
            if (inResult[idx]) continue;

            // Pop larger characters that can appear later
            while (stack.Count > 0 && c < stack.Peek() && freq[stack.Peek() - 'a'] > 0) {
                inResult[stack.Pop() - 'a'] = false;
            }

            // Push current character
            stack.Push(c);
            inResult[idx] = true;
        }

        // Step 4: Build result string
        char[] arr = new char[stack.Count];
        for (int i = arr.Length - 1; i >= 0; i--) arr[i] = stack.Pop();
        return new string(arr);
    }
}

## Key Takeaways

- **Monotonic increasing stack** combined with **greedy selection** yields the lexicographically smallest result
- Use **frequency counts** to decide whether a larger character can be safely removed (it must appear later)
- A **boolean array** prevents duplicate characters from being added to the result
- This pattern applies to problems where you need the smallest subsequence that satisfies certain constraints
- Time complexity is O(n) even though we have nested loops — each character is pushed and popped at most once
