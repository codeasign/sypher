# Daily Temperatures — Solution

## Problem Overview

Given an array of integers `temperatures` representing daily temperatures, return an array `answer` where `answer[i]` is the number of days you have to wait after the `i`th day to see a warmer temperature. If no warmer temperature exists in the future, `answer[i]` is 0.

## Brute Force Approach

**Idea:** For each day, scan forward through the remaining days to find the first warmer temperature.

**Algorithm:**
1. Initialize `n = temperatures.length` and `result = [0] * n`
2. For each index `i` from 0 to n-1:
   - For each index `j` from i+1 to n-1:
     - If `temperatures[j] > temperatures[i]`, set `result[i] = j - i` and break
3. Return `result`

**Complexity Analysis:**
- **Time:** O(n²) — for each of n elements, we scan up to n elements ahead
- **Space:** O(n) — the output array (excluding input)

**Why this is suboptimal:** For n = 10^5, O(n²) is too slow. The monotonic stack approach reduces this to O(n) by avoiding redundant comparisons.

## Optimal Approach: Monotonic Decreasing Stack

**Idea:** Process the array from left to right, maintaining a stack of indices whose warmer day has not yet been found. The stack is kept in decreasing order of temperature (monotonic decreasing). When we encounter a temperature warmer than the top of the stack, we know the warmer day for all indices on the stack that are colder.

**Algorithm:**
1. Initialize `result = [0] * n` and an empty stack
2. For each index `i` from 0 to n-1:
   - While the stack is not empty and `temperatures[i] > temperatures[stack.top()]`:
     - Pop the top index `prev` from the stack
     - Set `result[prev] = i - prev` (the number of days to wait)
   - Push the current index `i` onto the stack
3. Return `result` (indices left on the stack remain 0 — no warmer day found)

**Why This Works:** The stack maintains a list of unresolved indices (colder days waiting for a warmer day). Since the stack is decreasing, each new temperature only needs to resolve indices colder than itself. Once a warmer day is found for an index, it is popped and never visited again.

**Complexity Analysis:**
- **Time:** O(n) — each index is pushed and popped at most once
- **Space:** O(n) — the stack can hold up to n indices in the worst case

## Deep Dive: How the Monotonic Stack Works

### Visual Walkthrough

### Why Store Indices Instead of Temperatures?

Storing indices in the stack allows us to compute the day difference (`i - prev`) when a warmer day is found. If we stored temperatures, we would know a warmer day exists but couldn't determine how many days apart they are.

### Edge Cases to Consider

- **Single element**: The loop pushes index 0, no warmer day is found, result remains `[0]`.
- **Strictly decreasing temperatures**: Every element is pushed onto the stack; none are ever resolved. All results are 0.
- **Strictly increasing temperatures**: Each new element resolves the previous one. All results are 1 except the last, which is 0.
- **All equal temperatures**: No warmer day exists for any element. All results are 0.

### The Monotonic Stack Pattern

A monotonic stack is a stack that maintains elements in either increasing or decreasing order. It is useful for problems where you need to find the "next greater element" or "next smaller element" in an array. The key insight is:

- **Monotonic decreasing stack**: When you find a greater element, process all smaller elements on the stack
- **Monotonic increasing stack**: When you find a smaller element, process all larger elements on the stack

Daily Temperatures is a classic "next greater element" problem solved with a monotonic decreasing stack.

import java.util.Stack;

class Solution {
    public int[] dailyTemperatures(int[] temperatures) {
        int n = temperatures.length;
        int[] result = new int[n];       // Initialize result array with zeros
        Stack<Integer> stack = new Stack<>();  // Monotonic decreasing stack of indices

        // Process each day from left to right
        for (int i = 0; i < n; i++) {
            // While the current temperature is warmer than the top of the stack
            while (!stack.isEmpty() && temperatures[i] > temperatures[stack.peek()]) {
                int prev = stack.pop();          // Resolve this index
                result[prev] = i - prev;          // Days until warmer temperature
            }
            stack.push(i);  // Push current index onto stack (waiting for warmer day)
        }

        return result;  // Indices left on the stack remain 0
    }
}

fn daily_temperatures(temperatures: Vec<i32>) -> Vec<i32> {
    let n = temperatures.len();
    let mut result = vec![0; n];          // Initialize result array with zeros
    let mut stack: Vec<usize> = Vec::new(); // Monotonic decreasing stack of indices

    // Process each day from left to right
    for i in 0..n {
        // While the current temperature is warmer than the top of the stack
        while let Some(&prev) = stack.last() {
            if temperatures[i] <= temperatures[prev] {
                break;
            }
            stack.pop();                     // Resolve this index
            result[prev] = (i - prev) as i32; // Days until warmer temperature
        }
        stack.push(i);  // Push current index onto stack (waiting for warmer day)
    }

    result  // Indices left on the stack remain 0
}

#include <stdlib.h>

int* dailyTemperatures(int* temperatures, int n, int* returnSize) {
    *returnSize = n;
    int* result = (int*)calloc(n, sizeof(int));  // Initialize result array with zeros
    int* stack = (int*)malloc(n * sizeof(int));  // Monotonic decreasing stack of indices
    int top = -1;  // Stack pointer, -1 means empty

    // Process each day from left to right
    for (int i = 0; i < n; i++) {
        // While the current temperature is warmer than the top of the stack
        while (top >= 0 && temperatures[i] > temperatures[stack[top]]) {
            int prev = stack[top--];  // Resolve this index
            result[prev] = i - prev;  // Days until warmer temperature
        }
        stack[++top] = i;  // Push current index onto stack (waiting for warmer day)
    }

    free(stack);
    return result;  // Indices left on the stack remain 0
}

## Key Takeaways

- **Monotonic decreasing stack** is the standard pattern for "next greater element" problems
- **Store indices, not values** — you need the index difference to compute the day count
- Each element is **pushed and popped at most once**, giving O(n) time complexity
- The stack maintains a **colder-to-warmer order** — elements on the stack are waiting for their warmer day
- This pattern generalizes to many problems: next greater element, stock span, and sliding window maximum
