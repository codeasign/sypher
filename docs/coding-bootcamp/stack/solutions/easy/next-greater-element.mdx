# Next Greater Element — Solution

## Problem Overview

Given an array of integers `nums`, return an array `result` where `result[i]` is the next greater element for `nums[i]` — the first larger element to its right. If no greater element exists, use `-1`.

## Brute Force Approach

**Idea:** For each element, scan forward through the remaining elements to find the first larger value.

**Algorithm:**
1. Initialize `n = nums.length` and `result = [-1] * n`
2. For each index `i` from 0 to n-1:
   - For each index `j` from i+1 to n-1:
     - If `nums[j] > nums[i]`, set `result[i] = nums[j]` and break
3. Return `result`

**Complexity Analysis:**
- **Time:** O(n²) — for each of n elements, we scan up to n elements ahead
- **Space:** O(n) — the output array (excluding input)

**Why this is suboptimal:** For n = 10^5, O(n²) is too slow. The monotonic stack approach reduces this to O(n) by avoiding redundant comparisons.

## Optimal Approach: Monotonic Decreasing Stack

**Idea:** Process the array from left to right, maintaining a stack of indices whose next greater element has not yet been found. The stack is kept in decreasing order of values (monotonic decreasing). When we encounter a value larger than the top of the stack, we know the next greater element for all indices on the stack with smaller values.

**Algorithm:**
1. Initialize `result = [-1] * n` and an empty stack
2. For each index `i` from 0 to n-1:
   - While the stack is not empty and `nums[i] > nums[stack.top()]`:
     - Pop the top index `prev` from the stack
     - Set `result[prev] = nums[i]` (the current element is the next greater element)
   - Push the current index `i` onto the stack
3. Return `result` (indices left on the stack remain -1 — no greater element found)

**Why This Works:** The stack maintains a list of unresolved indices in decreasing order of their values. Each new element resolves all smaller elements on the stack. Since the stack is monotonically decreasing, we only need to check the top.

**Complexity Analysis:**
- **Time:** O(n) — each index is pushed and popped at most once
- **Space:** O(n) — the stack can hold up to n indices in the worst case

## Deep Dive

### Visual Walkthrough

For `nums = [4, 5, 2, 25]`, let's trace the algorithm:

```
Initial: stack = [], result = [-1, -1, -1, -1]

i=0, val=4: stack is empty, push 0
            stack = [0]

i=1, val=5: val > nums[stack.top()] = 4
            Pop 0, result[0] = 5
            stack = [], push 1
            stack = [1]

i=2, val=2: val <= nums[stack.top()] = 5, push 2
            stack = [1, 2]

i=3, val=25: val > nums[stack.top()] = 2
             Pop 2, result[2] = 25
             stack = [1]
             val > nums[stack.top()] = 5
             Pop 1, result[1] = 25
             stack = [], push 3
             stack = [3]

Final: result = [5, 25, 25, -1]
```

### Edge Cases

- **Single element**: No greater element, result stays `[-1]`.
- **Strictly decreasing array**: Every element is pushed; none are resolved. All results are `-1`.
- **Strictly increasing array**: Each new element resolves the previous one. All results are valid except the last.
- **All equal values**: No greater element, all results are `-1`.

### The Monotonic Stack Pattern

A monotonic stack is a stack that maintains elements in either increasing or decreasing order. This pattern is useful for "next greater element" and "next smaller element" problems:

- **Monotonic decreasing stack**: When you find a greater element, process all smaller elements on the stack
- **Monotonic increasing stack**: When you find a smaller element, process all larger elements on the stack

import java.util.Stack;

class Solution {
    public int[] nextGreaterElement(int[] nums) {
        int n = nums.length;
        int[] result = new int[n];       // Initialize result array
        for (int i = 0; i < n; i++) {
            result[i] = -1;              // Default to -1 (no greater element)
        }
        Stack<Integer> stack = new Stack<>();  // Monotonic decreasing stack of indices

        // Process each element from left to right
        for (int i = 0; i < n; i++) {
            // While the current element is greater than the top of the stack
            while (!stack.isEmpty() && nums[i] > nums[stack.peek()]) {
                int prev = stack.pop();          // Resolve this index
                result[prev] = nums[i];          // Current element is the next greater
            }
            stack.push(i);  // Push current index onto stack
        }

        return result;  // Indices left on the stack remain -1
    }
}

fn next_greater_element(nums: Vec<i32>) -> Vec<i32> {
    let n = nums.len();
    let mut result = vec![-1; n];          // Initialize result array with -1
    let mut stack: Vec<usize> = Vec::new(); // Monotonic decreasing stack of indices

    // Process each element from left to right
    for i in 0..n {
        // While the current element is greater than the top of the stack
        while let Some(&prev) = stack.last() {
            if nums[i] <= nums[prev] {
                break;
            }
            stack.pop();                     // Resolve this index
            result[prev] = nums[i];          // Current element is the next greater
        }
        stack.push(i);  // Push current index onto stack
    }

    result  // Indices left on the stack remain -1
}

#include <stdlib.h>

int* nextGreaterElement(int* nums, int n, int* returnSize) {
    *returnSize = n;
    int* result = (int*)malloc(n * sizeof(int));  // Initialize result array
    for (int i = 0; i < n; i++) result[i] = -1;  // Default to -1
    int* stack = (int*)malloc(n * sizeof(int));   // Monotonic decreasing stack of indices
    int top = -1;  // Stack pointer, -1 means empty

    // Process each element from left to right
    for (int i = 0; i < n; i++) {
        // While the current element is greater than the top of the stack
        while (top >= 0 && nums[i] > nums[stack[top]]) {
            int prev = stack[top--];  // Resolve this index
            result[prev] = nums[i];   // Current element is the next greater
        }
        stack[++top] = i;  // Push current index onto stack
    }

    free(stack);
    return result;  // Indices left on the stack remain -1
}

## Key Takeaways

- **Monotonic decreasing stack** is the standard pattern for "next greater element" problems
- **Store indices, not values** — you can always look up the value using the index
- Each element is **pushed and popped at most once**, giving O(n) time complexity
- The stack maintains a **decreasing order** — elements on the stack are waiting for their greater element
- This pattern generalizes to: daily temperatures, stock span, sliding window maximum, and more
