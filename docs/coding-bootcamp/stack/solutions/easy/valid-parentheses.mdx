# Valid Parentheses — Solution

## Problem Overview

Given a string `s` containing just the characters `'('`, `')'`, `'{'`, `'}'`, `'['` and `']'`, determine if the input string is valid. Open brackets must be closed by the same type of brackets in the correct order, and every close bracket must have a corresponding open bracket of the same type.

## Brute Force Approach

**Idea:** Repeatedly replace adjacent matching bracket pairs with empty strings until no more replacements can be made.

**Algorithm:**
1. While the string contains `"()"`, `"[]"`, or `"{}"`:
   - Replace all occurrences of `"()"` with `""`
   - Replace all occurrences of `"[]"` with `""`
   - Replace all occurrences of `"{}"` with `""`
2. Return `true` if the resulting string is empty, `false` otherwise

**Complexity Analysis:**
- **Time:** O(n²) — each pass scans the string, and in the worst case we make O(n) passes
- **Space:** O(n) — we create new strings during replacement

**Why this is suboptimal:** Repeated string replacement is slow and creates many intermediate strings. The stack-based approach solves this in a single pass with O(1) extra space (excluding the stack).

## Optimal Approach: Stack

**Idea:** Use a stack to track opening brackets. When you encounter a closing bracket, check if it matches the most recent opening bracket on the stack.

**Algorithm:**
1. Create a mapping of closing brackets to their corresponding opening brackets: `')' -> '('`, `']' -> '['`, `'}' -> '{'`
2. Initialize an empty stack
3. For each character `c` in `s`:
   - If `c` is a closing bracket:
     - If the stack is empty or the top of the stack does not match `c`'s corresponding opening bracket, return `false`
     - Pop the top of the stack (the matching opening bracket)
   - If `c` is an opening bracket:
     - Push it onto the stack
4. Return `true` if the stack is empty, `false` otherwise

**Why This Works:** The stack enforces the Last-In-First-Out (LIFO) property of nested brackets. The most recent opening bracket must be closed first, which matches exactly how a stack operates. The mapping ensures that each closing bracket is matched with the correct type of opening bracket.

**Complexity Analysis:**
- **Time:** O(n) — we traverse the string once, and each push/pop operation is O(1)
- **Space:** O(n) — in the worst case, the stack holds all opening brackets (e.g., `"((((..."`)

## Deep Dive: How the Stack Approach Works

### Visual Walkthrough

For the input `s = "({[]})"`, let's trace the algorithm:

```
Initial: stack = []
s = "({[]})"

Step 1: c = '(' — opening bracket, push onto stack
        stack = ['(']

Step 2: c = '{' — opening bracket, push onto stack
        stack = ['(', '{']

Step 3: c = '[' — opening bracket, push onto stack
        stack = ['(', '{', '[']

Step 4: c = ']' — closing bracket for '['
        top of stack is '[' -> matches! Pop it.
        stack = ['(', '{']

Step 5: c = '}' — closing bracket for '{'
        top of stack is '{' -> matches! Pop it.
        stack = ['(']

Step 6: c = ')' — closing bracket for '('
        top of stack is '(' -> matches! Pop it.
        stack = []

Final: stack is empty -> return true
```

### Edge Cases to Consider

- **Empty string**: The loop never executes, and the stack is empty, so the function returns `true`. An empty string is considered valid.
- **Single opening bracket**: Pushed onto the stack but never popped. The stack is non-empty at the end, so the function returns `false`.
- **Single closing bracket**: The stack is empty when we encounter it, so the function immediately returns `false`.
- **Mismatched types**: `([)]` — the closing `)` expects `(` but finds `[` on top of the stack, so the function returns `false`.
- **Nested brackets**: `({[]})` — each opening bracket is pushed in order, and the closing brackets are popped in reverse order, which is exactly what the stack handles naturally.

### Why a Stack?

Bracket validation is a classic LIFO (Last-In-First-Out) problem. The most recently opened bracket must be closed first. This maps perfectly to a stack:

- **Push** = open a new bracket (it becomes the most recent)
- **Pop** = close the most recent bracket (it must match)

No other data structure captures this property as naturally.

import java.util.Stack;

class Solution {
    public boolean isValid(String s) {
        // Step 1: Create a mapping of closing to opening brackets
        java.util.Map<Character, Character> map = new java.util.HashMap<>();
        map.put(')', '(');
        map.put(']', '[');
        map.put('}', '{');

        // Step 2: Initialize a stack to track opening brackets
        java.util.Stack<Character> stack = new java.util.Stack<>();

        // Step 3: Traverse each character in the string
        for (char c : s.toCharArray()) {
            if (map.containsKey(c)) {
                // c is a closing bracket
                // Check if stack is empty or the top doesn't match
                if (stack.isEmpty() || stack.pop() != map.get(c)) {
                    return false;
                }
            } else {
                // c is an opening bracket — push onto stack
                stack.push(c);
            }
        }

        // Step 4: Valid if all brackets were closed (stack is empty)
        return stack.isEmpty();
    }
}

fn is_valid(s: &str) -> bool {
    // Step 1: Create a mapping of closing to opening brackets
    // Step 2: Initialize a stack to track opening brackets
    let mut stack: Vec<char> = Vec::new();

    // Step 3: Traverse each character in the string
    for c in s.chars() {
        match c {
            // Opening brackets — push onto stack
            '(' | '{' | '[' => stack.push(c),
            // Closing brackets — check the top of the stack
            ')' => {
                if stack.pop() != Some('(') {
                    return false;
                }
            }
            '}' => {
                if stack.pop() != Some('{') {
                    return false;
                }
            }
            ']' => {
                if stack.pop() != Some('[') {
                    return false;
                }
            }
            _ => {} // Ignore other characters
        }
    }

    // Step 4: Valid if all brackets were closed (stack is empty)
    stack.is_empty()
}

#include <stdbool.h>
#include <string.h>

bool isValid(char* s) {
    int n = strlen(s);
    char stack[100005];
    int top = -1;  // Stack pointer, -1 means empty

    // Step 1: Create a mapping of closing to opening brackets
    // Step 2: Traverse each character in the string
    for (int i = 0; i < n; i++) {
        char c = s[i];
        if (c == '(' || c == '{' || c == '[') {
            // Opening bracket — push onto stack
            stack[++top] = c;
        } else {
            // Closing bracket
            // Check if stack is empty
            if (top == -1) return false;

            char expected;
            if (c == ')') expected = '(';
            else if (c == '}') expected = '{';
            else if (c == ']') expected = '[';
            else return false;

            // Check if top of stack matches
            if (stack[top] != expected) return false;
            top--;  // Pop the matching opening bracket
        }
    }

    // Step 3: Valid if all brackets were closed (stack is empty)
    return top == -1;
}

                // c is an opening bracket — push onto stack
                stack.Push(c);

    }
}

## Key Takeaways

- **Stack is the natural data structure** for bracket validation due to the LIFO property of nested brackets
- A **hash map** simplifies the mapping from closing brackets to opening brackets, making the code cleaner
- Always check for **stack emptiness** before popping — a closing bracket with an empty stack means the string is invalid
- The final check ensures no **unclosed opening brackets** remain
- O(n) time and O(n) space is optimal for this problem
