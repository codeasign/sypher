# N-Queens вҖ” Solution

## Problem Overview

Place `n` queens on an `n x n` chessboard so that no two queens threaten each other. Queens attack along rows, columns, and diagonals. We must return all distinct board configurations as arrays of column positions per row.

## Brute Force Approach

**Idea:** Generate all possible placements of n queens on nВІ squares (C(nВІ, n) possibilities) and check each configuration for validity.

**Algorithm:**
1. Generate all combinations of n squares from the nВІ available cells
2. For each combination, check if any two queens share a row, column, or diagonal
3. If valid, convert to the column-position representation and add to result

**Complexity Analysis:**
- **Time:** O(C(nВІ, n) * nВІ) вҖ” astronomically large. For n=8, C(64, 8) = 4.4 billion
- **Space:** O(nВІ) вҖ” storing the board

**Why this could be better:** This is impractical even for n=8. The backtracking approach prunes invalid placements early.

## Optimal Approach: Backtracking with Constraint Propagation

**Idea:** Place queens one row at a time. At each row, try every column. Use sets to track occupied columns and diagonals, making the safety check O(1). Backtrack when no safe column exists.

**Algorithm:**
1. Initialize `cols`, `diag1`, `diag2` as sets to track occupied columns and diagonals
2. Initialize `board` as an array of size n (column position per row)
3. Define `backtrack(row)`:
   - If `row == n`, add a copy of `board` to result
   - For `col` from 0 to n-1:
     - Compute `d1 = row - col` (main diagonal), `d2 = row + col` (anti-diagonal)
     - Skip if `col in cols` or `d1 in diag1` or `d2 in diag2`
     - Place queen: `board[row] = col`, add to sets
     - Recurse: `backtrack(row + 1)`
     - Remove queen from sets (backtrack)
4. Return result

**Why This Works:** By placing queens row by row, we guarantee no two queens share a row. The sets ensure no two queens share a column or either diagonal. This prunes the search space dramatically вҖ” from C(nВІ, n) to approximately n! possibilities (about 40,000 for n=8 instead of 4.4 billion).

**Complexity Analysis:**
- **Time:** O(n!) вҖ” worst case explores all permutations, but pruning reduces this significantly
- **Space:** O(n) вҖ” recursion depth of n, plus three sets of size n

### Diagonal Index Key Insight

The key insight that makes this algorithm efficient is indexing the diagonals:

- **Main diagonal** (top-left to bottom-right): All cells on the same main diagonal have the same `row - col` value
- **Anti diagonal** (top-right to bottom-left): All cells on the same anti-diagonal have the same `row + col` value

Two queens share a main diagonal if `row1 - col1 == row2 - col2`.
Two queens share an anti-diagonal if `row1 + col1 == row2 + col2`.

This allows O(1) conflict checking using boolean arrays or sets.

### Edge Cases

- **n=1:** One queen, one solution: `[0]`. The loop places it at col=0 and the base case triggers.
- **n=2 / n=3:** No solutions. The algorithm explores all possibilities and returns an empty list.
- **n=4:** Exactly 2 solutions.
- **Large n (n=9):** 9! = 362,880 permutations to explore with pruning вҖ” well within the time limit.

import java.util.*;

class Solution {
    public List<List<Integer>> solveNQueens(int n) {
        List<List<Integer>> result = new ArrayList<>();
        int[] board = new int[n];               // Column position of queen in each row
        boolean[] cols = new boolean[n];         // Track occupied columns
        boolean[] diag1 = new boolean[2 * n];    // Track occupied main diagonals (row - col + n)
        boolean[] diag2 = new boolean[2 * n];    // Track occupied anti-diagonals (row + col)
        backtrack(0, n, board, cols, diag1, diag2, result);
        return result;
    }

    private void backtrack(int row, int n, int[] board, boolean[] cols,
                           boolean[] diag1, boolean[] diag2, List<List<Integer>> result) {
        if (row == n) {
            // All queens placed вҖ” add solution
            List<Integer> solution = new ArrayList<>();
            for (int col : board) solution.add(col);
            result.add(solution);
            return;
        }
        for (int col = 0; col < n; col++) {
            int d1 = row - col + n;  // Offset by n to make index non-negative
            int d2 = row + col;
            if (cols[col] || diag1[d1] || diag2[d2]) continue;  // Unsafe
            // Place queen
            board[row] = col;
            cols[col] = diag1[d1] = diag2[d2] = true;
            backtrack(row + 1, n, board, cols, diag1, diag2, result);
            // Remove queen (backtrack)
            cols[col] = diag1[d1] = diag2[d2] = false;
        }
    }
}

fn solve_n_queens(n: i32) -> Vec<Vec<i32>> {
    let n = n as usize;
    let mut result: Vec<Vec<i32>> = Vec::new();
    let mut board: Vec<i32> = vec![0; n];
    let mut cols: Vec<bool> = vec![false; n];
    let mut diag1: Vec<bool> = vec![false; 2 * n];
    let mut diag2: Vec<bool> = vec![false; 2 * n];

    fn backtrack(row: usize, n: usize, board: &mut Vec<i32>, cols: &mut Vec<bool>,
                 diag1: &mut Vec<bool>, diag2: &mut Vec<bool>, result: &mut Vec<Vec<i32>>) {
        if row == n {
            result.push(board.clone());  // All queens placed
            return;
        }
        for col in 0..n {
            let d1 = row - col + n;   // Offset for non-negative index
            let d2 = row + col;
            if cols[col] || diag1[d1] || diag2[d2] { continue; }  // Unsafe
            // Place queen
            board[row] = col as i32;
            cols[col] = true;
            diag1[d1] = true;
            diag2[d2] = true;

            // Remove queen (backtrack)
            cols[col] = false;
            diag1[d1] = false;
            diag2[d2] = false;
        }
    }

    backtrack(0, n, &mut board, &mut cols, &mut diag1, &mut diag2, &mut result);
    result
}

}

int** solveNQueens(int n, int* returnSize, int** returnColumnSizes) {
    int capacity = 1000;  // Sufficient for n <= 9 (max 352 solutions for n=9)

    *returnColumnSizes = (int*)malloc(capacity * sizeof(int));
    *returnSize = 0;

    bool* diag1 = (bool*)calloc(2 * n, sizeof(bool));

    free(board); free(cols); free(diag1); free(diag2);
    return result;

class Solution {
    public IList<IList<int>> SolveNQueens(int n) {
        IList<IList<int>> result = new List<IList<int>>();

            if (cols[col] || diag1[d1] || diag2[d2]) continue;  // Unsafe
            // Place queen
            board[row] = col;
            cols[col] = diag1[d1] = diag2[d2] = true;
            Backtrack(row + 1, n, board, cols, diag1, diag2, result);
            // Remove queen (backtrack)
            cols[col] = diag1[d1] = diag2[d2] = false;
        }
    }
}

## Key Takeaways

- **N-Queens is a classic constraint satisfaction problem** solved elegantly with backtracking
- **Row-by-row placement** ensures queens never share a row, reducing the problem to finding valid column permutations
- **O(1) conflict checking** using boolean arrays for columns and diagonals is the key optimization
- The diagonal index trick (`row - col` for main diagonal, `row + col` for anti-diagonal) is the critical insight
- This is the same **choose-explore-unchoose** pattern as subsets and permutations, just with an additional validity check
- For n=8, backtracking explores ~40,000 states instead of the brute force 4.4 billion вҖ” a 100,000x improvement
