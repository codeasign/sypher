# Subsets — Solution

## Problem Overview

Given an array of distinct integers, return all possible subsets (the power set). For `n = 3`, there are `2³ = 8` subsets. We must return every subset exactly once, without duplicates.

## Brute Force Approach

**Idea:** Use bit manipulation. Each subset corresponds to a binary number from `0` to `2ⁿ - 1`, where bit `i` indicates whether element `i` is included.

**Algorithm:**
1. Iterate `mask` from `0` to `(1 << n) - 1`
2. For each `mask`, build the subset by including `nums[i]` when bit `i` is set
3. Add the built subset to the result

**Complexity Analysis:**
- **Time:** O(n * 2ⁿ) — we generate 2ⁿ subsets, and each takes O(n) to build
- **Space:** O(n * 2ⁿ) — storing all subsets (not counting output space)

**Why this could be better:** While the bit manipulation approach works, it doesn't scale well for recursive understanding. The backtracking approach is the standard template for generating combinatorial structures.

## Optimal Approach: Backtracking

**Idea:** Use recursive backtracking (also called the "choose-explore-unchoose" pattern). At each index, we decide whether to include the current element. We add the current state to the result at every recursive step (not just the leaves), so the empty set is captured immediately.

**Algorithm:**
1. Initialize an empty `result` list and an empty `current` list
2. Define a recursive function `backtrack(start)`:
   - Add a copy of `current` to `result`
   - Loop `i` from `start` to `n - 1`:
     - Include `nums[i]` by pushing it onto `current`
     - Recurse: `backtrack(i + 1)`
     - Exclude `nums[i]` by popping it from `current` (backtrack)
3. Call `backtrack(0)` and return `result`

**Why This Works:** By always adding the current subset before any recursion, we capture the empty set first. Then for each element from `start` onward, we try including it and recurse further. After returning, we remove it (backtrack) so the next iteration starts fresh. This systematically enumerates every combination.

**Complexity Analysis:**
- **Time:** O(n * 2ⁿ) — each of the 2ⁿ subsets takes O(n) to copy
- **Space:** O(n) — recursion depth of at most n, plus the current list (output space is not counted)

### Why We Add at Every Node

Unlike permutations where we only add at leaf nodes (when the permutation is complete), for subsets we add the current state at **every** node of the recursion tree. The root node represents the empty set, the first level represents all 1-element subsets, the second level all 2-element subsets, etc. This is what gives us the 2ⁿ subsets.

### Edge Cases

- **Empty array (n=0):** The only subset is the empty set. The function adds `[]` immediately and returns `[[]]`.
- **Single element (n=1):** Two subsets: `[]` and `[nums[0]]`.
- **Negative numbers:** They work exactly the same way — backtracking treats all elements uniformly.
- **Large values:** Elements can be up to 10⁹, but they are only compared for equality and never used as array indices, so no issues.

import java.util.*;

class Solution {
    public List<List<Integer>> subsets(int[] nums) {
        List<List<Integer>> result = new ArrayList<>();  // Stores all subsets
        List<Integer> current = new ArrayList<>();        // Current subset being built
        backtrack(nums, 0, current, result);
        return result;
    }

    private void backtrack(int[] nums, int start, List<Integer> current, List<List<Integer>> result) {
        result.add(new ArrayList<>(current));  // Add the current subset (copy)
        for (int i = start; i < nums.length; i++) {
            current.add(nums[i]);              // Include nums[i]
            backtrack(nums, i + 1, current, result);  // Recurse with next elements
            current.remove(current.size() - 1);        // Exclude nums[i] (backtrack)
        }
    }
}

fn subsets(nums: Vec<i32>) -> Vec<Vec<i32>> {
    let mut result: Vec<Vec<i32>> = Vec::new();  // Stores all subsets
    let mut current: Vec<i32> = Vec::new();       // Current subset being built

    fn backtrack(nums: &[i32], start: usize, current: &mut Vec<i32>, result: &mut Vec<Vec<i32>>) {
        result.push(current.clone());  // Add a copy of the current subset
        for i in start..nums.len() {
            current.push(nums[i]);              // Include nums[i]
            backtrack(nums, i + 1, current, result);  // Recurse with next elements
            current.pop();                      // Exclude nums[i] (backtrack)
        }
    }

    backtrack(&nums, 0, &mut current, &mut result);
    result
}

#include <stdlib.h>

int** subsets(int* nums, int numsSize, int* returnSize, int** returnColumnSizes) {
    int total = 1 << numsSize;                   // 2^n subsets
    int** result = (int**)malloc(total * sizeof(int*));
    *returnColumnSizes = (int*)malloc(total * sizeof(int));
    *returnSize = 0;

    // Use bit manipulation: each mask from 0 to 2^n-1 represents a subset
    for (int mask = 0; mask < total; mask++) {
        int* subset = (int*)malloc(numsSize * sizeof(int));
        int col = 0;
        for (int i = 0; i < numsSize; i++) {
            if (mask & (1 << i)) {
                subset[col++] = nums[i];
            }
        }
        // Reallocate to exact size
        subset = (int*)realloc(subset, col * sizeof(int));
        result[*returnSize] = subset;
        (*returnColumnSizes)[*returnSize] = col;
        (*returnSize)++;
    }

    return result;
}

## Key Takeaways

- **Backtracking** is the standard technique for generating all subsets (power set)
- The **choose-explore-unchoose** pattern is the backbone of combinatorial search
- For subsets, we **add at every node** of the recursion tree (unlike permutations which add only at leaves)
- The loop starts at `start` to avoid reusing elements and generating duplicates
- Time complexity O(n * 2ⁿ) is optimal since we must generate all 2ⁿ subsets
