# Backtracking

Backtracking systematically searches for solutions by building candidates incrementally and abandoning them when they can't lead to a valid solution.

⏱ 20 minutes

## What You Will Learn

- The recursive backtracking template
- Pruning the search space with constraints
- Common patterns: permutations, combinations, subsets

## The Concept

Build solutions incrementally. At each step, check if the partial solution is valid. If not, backtrack (undo the last choice and try the next option). If valid and complete, record it.

## Backtracking Template

```
def backtrack(path, remaining, result):
    if is_valid_solution(path):
        result.add(path.copy())
        return
    for choice in choices(remaining):
        make_choice(path, choice)
        backtrack(path, remaining - choice, result)
        undo_choice(path, choice)
```

## When to Use Backtracking

- Generate all combinations, permutations, subsets
- Constraint satisfaction problems (N-Queens, Sudoku)
- The problem asks for "all possible" solutions
- Search space is bounded (prune early)

## Complexity

- **Time**: O(k^n) worst-case, pruned by constraints
- **Space**: O(n) recursion depth
