# Generate Parentheses

**Difficulty:** Easy

Given `n` pairs of parentheses, write a function to generate **all combinations of well-formed parentheses** in the order they are generated by a standard backtracking approach (always try `(` before `)`).

## Example 1

**Input:** `n = 1`  
**Output:** `()`  
**Explanation:** There is only one valid combination with one pair.

## Example 2

**Input:** `n = 2`  
**Output:** `(()) ()()`  
**Explanation:** There are exactly two well-formed combinations with two pairs.

## Example 3

**Input:** `n = 3`  
**Output:** `((())) (()()) (())() ()(()) ()()()`  
**Explanation:** There are five well-formed combinations with three pairs.

## Input Specification

A single line containing the integer `n` — the number of pairs of parentheses.

## Output Specification

Print all valid parentheses combinations on a single line, separated by spaces, in the order produced by backtracking (try `(` before `)` at each step).

## Constraints

- `1 <= n <= 8`

## Hints

<details>
<summary>Hint 1</summary>

Think recursively. At each step you can either place an open parenthesis `(` or a close parenthesis `)`. What conditions must be true before you place each one?

</details>

<details>
<summary>Hint 2</summary>

You can place `(` as long as the number of open parentheses used so far is less than `n`. You can place `)` as long as the number of close parentheses used is less than the number of open parentheses placed.

</details>

<details>
<summary>Hint 3</summary>

Use a backtracking helper that tracks `open_count`, `close_count`, and the current string being built. When `open_count == n` and `close_count == n`, add the string to your result list.

</details>

</ProblemEditor>
