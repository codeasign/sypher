# Remove Nth Node from End of List

**Difficulty:** Medium

Given the `head` of a singly linked list, remove the **nth node from the end** of the list and return its head.

You must solve this in **one pass** using the fast and slow pointer technique.

## Example 1

**Input:** `list = [1, 2, 3, 4, 5]`, `n = 2`
**Output:** `[1, 2, 3, 5]`
**Explanation:** The 2nd node from the end is `4`. After removing it, the list becomes `1 -> 2 -> 3 -> 5`.

## Example 2

**Input:** `list = [1]`, `n = 1`
**Output:** `[]`
**Explanation:** The only node is removed, resulting in an empty list.

## Example 3

**Input:** `list = [1, 2]`, `n = 2`
**Output:** `[2]`
**Explanation:** The 2nd node from the end is `1` (the head). After removing it, the list becomes `2`.

## Input Specification

The first line contains space-separated integers representing the linked list values (may be empty for a single-node list that gets fully removed).
The second line contains a single integer `n`.

## Output Specification

Print the space-separated values of the resulting linked list. Print an empty line if the list becomes empty.

## Constraints

- The number of nodes in the list is in the range `[1, 30]`
- `0 <= Node.val <= 100`
- `1 <= n <= length of list`

## Hints

<details>
<summary>Hint 1</summary>

Use two pointers — `fast` and `slow` — both starting at a dummy node before the head. Advance `fast` by `n + 1` steps first so there is a gap of `n` between the two pointers.

</details>

<details>
<summary>Hint 2</summary>

After placing `fast` at position `n + 1`, advance both `fast` and `slow` together one step at a time. When `fast` reaches `null`, `slow` is positioned exactly at the node **before** the one you want to remove.

</details>

<details>
<summary>Hint 3</summary>

Set `slow.next = slow.next.next` to skip over the target node. Using a dummy node at the start handles the edge case where the head itself must be removed, without any special-case code.

</details>

</ProblemEditor>
