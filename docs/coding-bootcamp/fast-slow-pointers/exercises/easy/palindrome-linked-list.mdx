# Palindrome Linked List

**Difficulty:** Easy

Given the head of a singly linked list, return `true` if it is a palindrome, or `false` otherwise.

A linked list is a palindrome if the sequence of node values reads the same forwards and backwards.

## Example 1

**Input:** `1 -> 2 -> 2 -> 1`  
**Output:** `true`  
**Explanation:** The values read the same in both directions.

## Example 2

**Input:** `1 -> 2`  
**Output:** `false`  
**Explanation:** `[1, 2]` reversed is `[2, 1]`, which is different.

## Example 3

**Input:** `1 -> 2 -> 3 -> 2 -> 1`  
**Output:** `true`  
**Explanation:** The values form a symmetric palindrome.

## Input Specification

A single line of space-separated integers representing the values of the linked list nodes from head to tail.

## Output Specification

Print `true` if the linked list is a palindrome, otherwise print `false`.

## Constraints

- The number of nodes in the list is in the range `[1, 10^5]`
- `0 <= Node.val <= 9`

## Hints

<details>
<summary>Hint 1</summary>

Use the fast and slow pointer technique to find the middle of the linked list. The fast pointer moves two steps at a time while the slow pointer moves one step, so when fast reaches the end, slow is at the middle.

</details>

<details>
<summary>Hint 2</summary>

Once you find the middle, reverse the second half of the linked list in-place. This lets you compare the first half with the reversed second half without extra space.

</details>

<details>
<summary>Hint 3</summary>

Walk two pointers simultaneously — one from the head and one from the start of the reversed second half — comparing values at each step. If all values match, the list is a palindrome. Remember: for odd-length lists the middle node does not need to be compared.

</details>
