# Middle of the Linked List — Solution

## Problem Overview

Find the middle node of a singly linked list in one pass. For even-length lists, return the second middle node.

## Brute Force Approach

Count the number of nodes in the list, then iterate again to the middle. This requires two passes through the list.

**Complexity**: O(n) time, O(1) space (two passes)

## Optimal Approach

Use the fast and slow pointer technique:

1. Both pointers start at the head
2. Slow moves one step per iteration
3. Fast moves two steps per iteration
4. When fast reaches the end (or null), slow is at the middle

For even-length lists (e.g., 6 nodes), slow stops at the 4th node (the second middle), which matches the problem's requirement.

## Deep Dive

This is the simplest application of the fast-slow pattern. The math: when the list has length n, the slow pointer moves n/2 steps while the fast pointer moves n. For odd lengths, slow lands at the exact middle. For even lengths, slow lands at the (n/2 + 1)th node — the second middle.

class Solution {

    }
}

## Key Takeaways

- Fast-slow pointers find the middle in one pass with O(1) space
- For even length, the second middle is automatically selected
- This is the foundational pattern for the rest of this section
