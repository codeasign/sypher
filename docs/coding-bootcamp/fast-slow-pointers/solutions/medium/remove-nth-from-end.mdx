# Remove Nth Node from End of List — Solution

## Problem Overview

Given the `head` of a singly linked list, remove the `n`-th node from the end of the list and return the head of the modified list. The solution must use a **one-pass** algorithm.

## Brute Force Approach

**Idea:** Traverse the list to find its length, then traverse again to the node just before the target, and remove it.

**Algorithm:**
1. Traverse the list to count the total number of nodes
2. Calculate the index of the node to remove from the start: `target = length - n`
3. Traverse to the node at position `target - 1` (the node before the target)
4. Update the `next` pointer to skip the target node
5. If the target is the head, update the head instead

**Complexity Analysis:**
- **Time:** O(n) — two passes through the list
- **Space:** O(1) — no extra space

**Why this could be better:** The two-pass approach works correctly but the problem requires a **one-pass** solution.

## Optimal Approach: One-Pass with Fast and Slow Pointers

**Idea:** Use two pointers with a gap of `n` nodes between them. When the fast pointer reaches `null`, the slow pointer is at the node just before the one to remove.

**Algorithm:**
1. Create a dummy node that points to the head (handles edge case of removing the head)
2. Initialize both `slow` and `fast` pointers at the dummy node
3. Advance the `fast` pointer by `n + 1` steps so there is a gap of `n` between `slow` and `fast`
4. Move both `slow` and `fast` one step at a time until `fast` reaches `null`
5. Now `slow` is at the node **before** the target — remove by setting `slow.next = slow.next.next`
6. Return `dummy.next` (the new head, or `null` if the list became empty)

The dummy node eliminates the special case of removing the head node.

**Complexity Analysis:**
- **Time:** O(n) — one pass through the list
- **Space:** O(1) — only pointer variables

## Deep Dive

The gap of `n` between slow and fast is the key insight. When fast reaches `null`, slow is exactly `n` nodes behind — at the node before the target. This works because the dummy node ensures even removing the first node from the end (the head) is handled uniformly.

For `n = 1` (remove the last node), fast advances by 2 from dummy, then both pointers advance together until fast is `null`. Slow ends up at the second-to-last node.

def remove_nth_from_end(head, n):
    dummy = ListNode(0)
    dummy.next = head
    slow = fast = dummy

fn remove_nth_from_end(head: Option<Box<ListNode>>, n: i32) -> Option<Box<ListNode>> {
    let mut dummy = Some(Box::new(ListNode { val: 0, next: head }));
    let mut slow = &mut dummy;
    let mut fast = &slow.clone();

    // Advance fast by n + 1 steps
    for _ in 0..=n {
        fast = &fast.as_ref().unwrap().next;
    }

    // Move both pointers until fast reaches the end
    while fast.is_some() {

        fast = &fast.as_ref().unwrap().next;

    // Remove the target node
    let target = &mut slow.as_mut().unwrap().next;
    slow.as_mut().unwrap().next = target.as_mut().unwrap().next.take();

}

## Key Takeaways

- **Fast and slow pointers with a gap** solve the "nth from end" problem in one pass
- A **dummy node** eliminates special-case handling for removing the head
- The gap of `n` ensures slow is at the node just before the target when fast reaches `null`
- This is O(n) time and O(1) space — optimal for the one-pass constraint
- The same gap technique can be adapted to other problems (e.g., finding the nth from end without removal)
