# Happy Number — Solution

## Problem Overview

Write an algorithm to determine if a number `n` is happy. A happy number is defined by repeatedly replacing it with the sum of the squares of its digits until it equals 1 (happy) or loops endlessly in a cycle that does not include 1 (unhappy). Return `true` if `n` is a happy number, `false` otherwise.

## Brute Force Approach

**Idea:** Use a hash set to detect cycles in the sequence of squared digit sums.

**Algorithm:**
1. Create an empty hash set `seen`
2. While `n` is not 1 and `n` is not in `seen`:
   - Add `n` to `seen`
   - Replace `n` with the sum of squares of its digits
3. Return `true` if `n == 1`, `false` otherwise

**Complexity Analysis:**
- **Time:** O(log n) per iteration, with a bounded number of iterations before reaching 1 or a cycle
- **Space:** O(log n) — the hash set stores all numbers seen in the sequence

**Why this could be better:** The hash set approach is simple and correct, but uses extra space proportional to the length of the sequence. The optimal approach uses only O(1) space.

## Optimal Approach: Floyd's Cycle Detection

**Idea:** Apply Floyd's cycle detection (Tortoise and Hare) directly to the sequence of numbers. The slow pointer computes one step of the sequence (sum of squares once), and the fast pointer computes two steps (sum of squares twice). If they meet at 1, the number is happy. If they meet at any other value, a cycle is detected and the number is unhappy.

**Algorithm:**
1. Define a helper function `sumSquares(n)` that computes the sum of squares of digits
2. Initialize `slow = n` and `fast = n`
3. Loop:
   - Set `slow = sumSquares(slow)` (one step)
   - Set `fast = sumSquares(sumSquares(fast))` (two steps)
   - If `fast == 1`, return `true` (reached 1)
   - If `slow == fast`, return `false` (cycle detected, not 1)
4. This loop always terminates because the sequence either reaches 1 or enters a cycle

**Why This Works:** The sequence of squared digit sums is deterministic. If it reaches 1, it stays at 1 forever. If it doesn't, it must eventually repeat a number (forming a cycle) because the number of possible values is bounded. Floyd's algorithm detects this cycle using O(1) space.

**Complexity Analysis:**
- **Time:** O(log n) per iteration, with a bounded number of iterations
- **Space:** O(1) — only two integer variables

def isHappy(n: int) -> bool:
    def sum_squares(num):
        total = 0
        while num:
            digit = num % 10
            total += digit * digit
            num //= 10
        return total

    slow = n
    fast = n
    while True:
        slow = sum_squares(slow)               # One step for slow
        fast = sum_squares(sum_squares(fast))  # Two steps for fast
        if fast == 1:                          # Reached 1 — happy number
            return True
        if slow == fast:                       # Cycle detected — unhappy number
            return False

## Key Takeaways

- **Floyd's cycle detection** works on any sequence, not just linked lists
- The sequence of squared digit sums is guaranteed to either reach 1 or enter a cycle
- The slow pointer computes one step; the fast pointer computes two steps
- If they meet at a value other than 1, a cycle (and thus an unhappy number) is confirmed
- O(1) space is the key advantage over the hash set approach
