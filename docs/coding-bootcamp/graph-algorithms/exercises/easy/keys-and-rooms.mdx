# Keys and Rooms

**Difficulty:** Easy

There are `n` rooms labeled from `0` to `n - 1` and all the rooms are locked except for room `0`. Your goal is to visit all the rooms. However, you cannot enter a locked room without having its key.

When you visit a room, you may find a set of distinct keys in it. Each key has a number on it, denoting which room it unlocks, and you can take all of them with you to unlock the other rooms.

Given an array `rooms` where `rooms[i]` is the set of keys you can obtain if you visit room `i`, return `true` if you can visit all the rooms, or `false` otherwise.

## Example 1

**Input:** `rooms = [[1],[2],[3],[]]`  
**Output:** `true`  
**Explanation:**  
Start in room 0 → pick up key to room 1 → visit room 1 → pick up key to room 2 → visit room 2 → pick up key to room 3 → visit room 3 → all rooms visited.

## Example 2

**Input:** `rooms = [[1,3],[3,0,1],[2],[0]]`  
**Output:** `false`  
**Explanation:**  
You cannot enter room 2 because it has no key leading to it from any reachable room.

## Input Specification

The first line contains an integer `n` representing the number of rooms.  
Each of the next `n` lines contains space-separated integers representing the keys in each room (an empty line means no keys).

## Output Specification

Print `true` if you can visit all rooms, otherwise `false`.

## Constraints

- `n == rooms.length`
- `2 <= n <= 1000`
- `0 <= rooms[i][j] < n`
- All the values of `rooms[i]` are unique

## Hints

<details>
<summary>Hint 1</summary>

This is a classic graph traversal problem. The rooms are nodes and keys are directed edges.
</details>

<details>
<summary>Hint 2</summary>

Use DFS (recursive or iterative) or BFS starting from room 0 and track which rooms you've visited.
</details>

<details>
<summary>Hint 3</summary>

After traversal, check if the number of visited rooms equals the total number of rooms.
</details>
