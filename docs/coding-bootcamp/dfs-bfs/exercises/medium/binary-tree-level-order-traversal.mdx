# Binary Tree Level Order Traversal

**Difficulty:** Medium

Given the `root` of a binary tree, return the **level order traversal** of its nodes' values. (i.e., from left to right, level by level).

## Example 1

```
Tree:
    3
   / \
  9  20
    /  \
   15   7
```

**Input:**
```
7
3 9 20 -1 -1 15 7
```

**Output:**
```
3
9 20
15 7
```

**Explanation:** The tree has 3 levels: level 0 has `[3]`, level 1 has `[9, 20]`, level 2 has `[15, 7]`.

## Example 2

```
Tree:
    1
   / \
  2   3
 / \
4   5
```

**Input:**
```
5
1 2 3 4 5
```

**Output:**
```
1
2 3
4 5
```

## Example 3

```
Tree: (empty)
```

**Input:**
```
0
```

**Output:**
```
(empty line)
```

## Input Specification

- First line: an integer `n` — number of nodes in the tree (`0 <= n <= 2000`)
- Second line: `n` space-separated integers representing the level-order traversal of the tree (use `-1` for null nodes). Omitted if `n == 0`.

## Output Specification

Print each level of the tree on its own line, with the values separated by spaces. Print an empty line if the tree is empty.

## Constraints

- The number of nodes in the tree is in the range `[0, 2000]`
- `-1000 <= Node.val <= 1000`

## Hints

<details>
<summary>Hint 1</summary>

Use a queue to perform BFS. Process nodes level by level by tracking the number of nodes at each level.
</details>

<details>
<summary>Hint 2</summary>

At each level, process all nodes currently in the queue. For each node, add its value to the current level's list, then enqueue its children.
</details>

<details>
<summary>Hint 3</summary>

The outer loop runs while the queue is not empty. The inner loop runs exactly `queue.size()` times to process one complete level.
</details>
