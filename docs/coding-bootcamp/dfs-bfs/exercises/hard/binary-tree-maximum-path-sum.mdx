# Binary Tree Maximum Path Sum

**Difficulty:** Hard

A **path** in a binary tree is a sequence of nodes where each pair of adjacent nodes in the sequence has an edge connecting them. A node can only appear in the sequence **at most once**. The path does **not** need to pass through the root.

The **path sum** of a path is the sum of the node values in the path.

Given the `root` of a binary tree, return the **maximum path sum** of any **non-empty path**.

## Example 1

```
Tree:
      -10
      /  \
     9   20
        /  \
       15   7
```

**Input:**
```
7
-10 9 20 -1 -1 15 7
```

**Output:** `42`

**Explanation:** The optimal path is `15 -> 20 -> 7`, with a sum of `15 + 20 + 7 = 42`.

## Example 2

```
Tree:
    1
   / \
  2   3
```

**Input:**
```
3
1 2 3
```

**Output:** `6`

**Explanation:** The optimal path is `2 -> 1 -> 3`, with a sum of `2 + 1 + 3 = 6`.

## Example 3

```
Tree:
    -3
   /  \
 -1   -2
```

**Input:**
```
3
-3 -1 -2
```

**Output:** `-1`

**Explanation:** All nodes have negative values, so the optimal path is the single node `-1`.

## Input Specification

- First line: an integer `n` — number of nodes in the tree (`0 <= n <= 3 * 10^4`)
- Second line: `n` space-separated integers representing the level-order traversal of the tree (use `-1` for null nodes). Omitted if `n == 0`.

## Output Specification

Print a single integer — the maximum path sum.

## Constraints

- The number of nodes in the tree is in the range `[1, 3 * 10^4]`
- `-1000 <= Node.val <= 1000`

## Hints

<details>
<summary>Hint 1</summary>

Use a recursive DFS that computes the maximum contribution a subtree can provide to its parent. Each node can combine its value with contributions from its left and right children.
</details>

<details>
<summary>Hint 2</summary>

Maintain a global maximum path sum variable. At each node, consider three cases: the path goes through the node using both children, or the path goes through the node using only one child (to continue upward).
</details>

<details>
<summary>Hint 3</summary>

When returning a value to the parent, you can only take one branch (the larger of left or right contribution) since the path can't fork when going upward. But when evaluating the max at the current node, you consider both branches.
</details>
