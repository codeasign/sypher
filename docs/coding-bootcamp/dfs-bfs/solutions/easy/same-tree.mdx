# Same Tree — Solution

## Problem Overview

Given the roots of two binary trees `p` and `q`, determine whether they are the **same tree**. Two binary trees are considered the same if they are structurally identical and every corresponding node has the same value.

## Approach: Recursive DFS (Depth-First Search)

**Idea:** Traverse both trees simultaneously using recursion. At each step, compare the current node from both trees. If both are `null`, the subtrees match. If one is `null` and the other is not, they differ. If both exist but have different values, they differ. Otherwise, recursively check the left subtrees and the right subtrees.

**Algorithm:**
1. If both `p` and `q` are `null`, return `true` (both subtrees are empty)
2. If one is `null` and the other is not, return `false` (structural difference)
3. If `p.val != q.val`, return `false` (value mismatch)
4. Recursively check `isSameTree(p.left, q.left)` and `isSameTree(p.right, q.right)`
5. Return `true` only if both recursive calls return `true`

**Why This Works:** The recursive definition mirrors the definition of tree equality itself — two trees are the same if their roots match and their left and right subtrees are each the same. By recursing on both children, we explore the entire structure of both trees in parallel, and the base cases handle the boundary conditions (null nodes, value mismatches).

**Complexity Analysis:**
- **Time:** O(n) — each node is visited exactly once, where n is the total number of nodes in the smaller tree (the algorithm short-circuits on the first mismatch)
- **Space:** O(h) — the recursion stack depth equals the height of the tree h. In the worst case (skewed tree), h = n. In the best case (balanced tree), h = log n

## Deep Dive: How Recursive DFS Compares Trees

### Visual Walkthrough

For the input trees `p = [1, 2, 3]` and `q = [1, 2, 3]`, let's trace the recursion:

```
Tree p:        Tree q:
    1             1
   / \           / \
  2   3         2   3

Step 1: isSameTree(p = node(1), q = node(1))
        → Both not null, values match (1 == 1)
        → Recurse left:  isSameTree(p = node(2), q = node(2))
        → Recurse right: isSameTree(p = node(3), q = node(3))

Step 2a: isSameTree(p = node(2), q = node(2))
         → Both not null, values match (2 == 2)
         → Recurse left:  isSameTree(p = null, q = null) → true
         → Recurse right: isSameTree(p = null, q = null) → true
         → Return true

Step 2b: isSameTree(p = node(3), q = node(3))
         → Both not null, values match (3 == 3)
         → Recurse left:  isSameTree(p = null, q = null) → true
         → Recurse right: isSameTree(p = null, q = null) → true
         → Return true

Step 3: Both left and right returned true → Return true
```

Now for the input `p = [1, 2, -1]` and `q = [1, -1, 2]` (different structure):

```
Tree p:        Tree q:
    1             1
   /               \
  2                 2

Step 1: isSameTree(p = node(1), q = node(1))
        → Both not null, values match (1 == 1)
        → Recurse left:  isSameTree(p = node(2), q = null)
        → Recurse right: isSameTree(p = null, q = node(2))

Step 2a: isSameTree(p = node(2), q = null)
         → One is null, the other is not → Return false

Step 2b: isSameTree(p = null, q = node(2))
         → One is null, the other is not → Return false

Step 3: Left returned false → Short-circuit, Return false
```

### Edge Cases to Consider

- **Both trees are empty**: The function immediately returns `true` since both roots are `null`.
- **One tree empty, one not**: The null-check catches this structural difference at the root level.
- **Single node trees**: If values match, both recursive calls return `true` (null == null). If values differ, the value check catches it.
- **Deeply nested trees**: The recursion naturally handles any depth, bounded only by the call stack. The O(n) time ensures performance even for 100-node trees.
- **Negative values**: The comparison uses `==` (or `.val` comparison), which works correctly for all integers including negative values.
- **Trees with null children at different positions**: The structural comparison via recursion naturally catches cases where one tree has a left child and the other does not, regardless of the values.

### Why Recursion Is the Natural Fit

Tree comparison is a textbook case for recursion because trees are recursively defined data structures. Each node is itself the root of a subtree. The recursive solution mirrors the problem definition directly:

- **Base case**: Two null nodes are trivially the same
- **Recursive case**: Two non-null nodes are the same if their values match AND their left subtrees are the same AND their right subtrees are the same

An iterative approach (using explicit stacks) would work but would be more verbose without any performance benefit. The recursive approach is the clearest and most concise.

class Solution {
    public boolean isSameTree(TreeNode p, TreeNode q) {
        // Base case: both nodes are null — structurally identical at this point
        if (p == null && q == null) return true;
        // One is null, the other is not — structural mismatch
        if (p == null || q == null) return false;
        // Values differ — not the same tree
        if (p.val != q.val) return false;
        // Recursively check left and right subtrees; both must match
        return isSameTree(p.left, q.left) && isSameTree(p.right, q.right);
    }
}

## Key Takeaways

- **Recursive DFS** is the natural solution for tree comparison problems — trees are recursively defined, so recursive solutions mirror the problem structure
- **Three base cases**: both null (true), one null (false), values differ (false) — these cover all termination conditions
- The solution is **O(n) time and O(h) space** — optimal for this problem
- The same recursive pattern extends to other tree comparison problems (subtree check, symmetric tree, etc.)
- **Short-circuit evaluation** (`&&` operator) stops as soon as any mismatch is found, so the algorithm may not visit every node
