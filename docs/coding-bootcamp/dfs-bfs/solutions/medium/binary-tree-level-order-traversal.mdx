# Binary Tree Level Order Traversal — Solution

## Problem Overview

Given the `root` of a binary tree, return its **level order traversal** — the values of its nodes grouped by level, from left to right, top to bottom.

## Approach: BFS with Queue

**Idea:** Use a queue to perform breadth-first search. Process the tree level by level by tracking how many nodes are at each level using the queue size at the start of each level.

**Algorithm:**
1. If `root` is `null`, return an empty list
2. Initialize a queue with `root`
3. While the queue is not empty:
   - Get the current queue size — this is the number of nodes at the current level
   - Create a new list for the current level's values
   - For each node at this level (loop `size` times):
     - Dequeue the node
     - Add its value to the current level's list
     - Enqueue its left child (if not null)
     - Enqueue its right child (if not null)
   - Add the current level's list to the result
4. Return the result

**Why This Works:** BFS naturally visits nodes in level order. By capturing the queue size before processing each level, we can accurately group nodes by their depth. The queue always contains nodes from at most two consecutive levels.

**Complexity Analysis:**
- **Time:** O(n) — each node is entered and exited from the queue exactly once
- **Space:** O(n) — the queue holds at most the width of the tree, which is O(n) in the worst case (a full tree's bottom level has n/2 nodes)

## Alternative: Recursive DFS with Level Tracking

**Idea:** Perform a pre-order DFS traversal while tracking the current depth. Append each node's value to the list corresponding to its level.

**Algorithm:**
1. Initialize an empty result list
2. Call a recursive helper `dfs(node, level)`:
   - If `node` is `null`, return
   - If `level` equals the result list size, add a new empty list for this level
   - Append `node.val` to `result[level]`
   - Recursively call `dfs(node.left, level + 1)`
   - Recursively call `dfs(node.right, level + 1)`
3. Return the result

**Complexity Analysis:**
- **Time:** O(n) — each node is visited once
- **Space:** O(h) for the recursion stack plus O(n) for the result

## Deep Dive: BFS Level Traversal

### Visual Walkthrough

For the tree `[3, 9, 20, -1, -1, 15, 7]`:

```
    3
   / \
  9  20
    /  \
   15   7
```

Queue trace:

```
Initial: queue = [3], result = []

Level 0: size = 1
  Dequeue 3 → level = [3], enqueue 9, 20
  queue = [9, 20], result = [[3]]

Level 1: size = 2
  Dequeue 9 → level = [9], enqueue nothing (no children)
  Dequeue 20 → level = [9, 20], enqueue 15, 7
  queue = [15, 7], result = [[3], [9, 20]]

Level 2: size = 2
  Dequeue 15 → level = [15], enqueue nothing
  Dequeue 7 → level = [15, 7], enqueue nothing
  queue = [], result = [[3], [9, 20], [15, 7]]

Result: [[3], [9, 20], [15, 7]]
```

### Edge Cases to Consider

- **Empty tree**: Return an empty list immediately.
- **Single node**: One level with one value.
- **Unbalanced tree**: BFS processes level by level regardless of shape. Nodes with missing children simply don't have those children enqueued.
- **Deep tree with narrow width**: The queue never grows large — only 1-2 nodes at a time for a skewed tree.
- **Complete binary tree**: The queue reaches its maximum size at the bottom level (n/2 nodes).

### Why BFS Is the Natural Fit

Level-order traversal is the quintessential BFS application. The queue-based approach directly mirrors the problem's requirement: process nodes layer by layer. While DFS with level tracking also works, BFS is more intuitive because it processes nodes in the exact order requested, without needing to pre-allocate or build the result structure dynamically.

class Solution {
    public List<List<Integer>> levelOrder(TreeNode root) {
        // Result list to hold each level's values
        List<List<Integer>> result = new ArrayList<>();
        if (root == null) return result;

        // Queue for BFS traversal
        Queue<TreeNode> queue = new LinkedList<>();
        queue.offer(root);

        while (!queue.isEmpty()) {
            // Number of nodes at the current level
            int levelSize = queue.size();
            List<Integer> currentLevel = new ArrayList<>();

            // Process all nodes at this level
            for (int i = 0; i < levelSize; i++) {
                TreeNode node = queue.poll();
                currentLevel.add(node.val);

                // Enqueue children for the next level
                if (node.left != null) queue.offer(node.left);
                if (node.right != null) queue.offer(node.right);
            }

            result.add(currentLevel);
        }

        return result;
    }
}

use std::collections::VecDeque;

fn level_order(root: Option<Box<TreeNode>>) -> Vec<Vec<i32>> {
    let mut result = vec![];
    if root.is_none() {
        return result;
    }

    let mut queue = VecDeque::new();
    queue.push_back(root);

    while !queue.is_empty() {
        let level_size = queue.len();
        let mut current_level = vec![];

        for _ in 0..level_size {
            if let Some(Some(node)) = queue.pop_front() {
                current_level.push(node.val);
                queue.push_back(node.left);
                queue.push_back(node.right);
            }
        }

        if !current_level.is_empty() {
            result.push(current_level);
        }
    }

    result
}

int** levelOrder(struct TreeNode* root, int* returnSize, int** returnColumnSizes) {
    if (root == NULL) {
        *returnSize = 0;
        return NULL;
    }

    int** result = malloc(2000 * sizeof(int*));
    *returnColumnSizes = malloc(2000 * sizeof(int));
    *returnSize = 0;

    struct TreeNode** queue = malloc(2000 * sizeof(struct TreeNode*));
    int front = 0, rear = 0;
    queue[rear++] = root;

    while (front < rear) {
        int levelSize = rear - front;
        int* level = malloc(levelSize * sizeof(int));

        for (int i = 0; i < levelSize; i++) {
            struct TreeNode* node = queue[front++];
            level[i] = node->val;

            if (node->left) queue[rear++] = node->left;
            if (node->right) queue[rear++] = node->right;
        }

        (*returnColumnSizes)[*returnSize] = levelSize;
        result[*returnSize] = level;
        (*returnSize)++;
    }

    free(queue);
    return result;
}

## Key Takeaways

- **BFS with queue** is the canonical solution for level-order traversal
- The **level size capture pattern** (`int size = queue.size()`) is a critical technique for processing BFS level by level
- Both BFS and DFS approaches achieve **O(n) time** — the key difference is space usage
- This pattern extends to many variants: zigzag level order, level order bottom-up, right side view, average of levels, etc.
- Level-order traversal also naturally computes the **maximum width** of a binary tree
