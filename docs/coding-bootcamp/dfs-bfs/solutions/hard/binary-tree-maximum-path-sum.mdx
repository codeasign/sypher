# Binary Tree Maximum Path Sum — Solution

## Problem Overview

Given the `root` of a binary tree, find the **maximum path sum** of any **non-empty path**. A path is a sequence of connected nodes where each pair of adjacent nodes has an edge, and each node can appear at most once. The path does not need to pass through the root.

## Brute Force Approach

**Idea:** For every pair of nodes, find the path between them and compute the sum. This is impractical for all but the smallest trees.

**Complexity Analysis:**
- **Time:** O(n^2) or worse, depending on how paths are computed
- **Space:** O(h) for recursion

**Why this could be better:** The brute force is computationally infeasible for large trees. We need a smarter approach that computes the maximum path sum in a single traversal.

## Optimal Approach: Post-Order DFS with Global Maximum

**Idea:** Perform a post-order DFS where each node returns the **maximum contribution** it can provide to its parent along a single path. Simultaneously, track the global maximum path sum that can pass through each node, considering both left and right branches.

**Algorithm:**
1. Initialize a global variable `maxSum` to a very negative value (or the minimum integer)
2. Define a recursive helper `dfs(node)` that returns the **maximum gain** from this node to its parent:
   - If `node` is `null`, return `0` (no contribution from an empty subtree)
   - Recursively compute `leftGain = max(dfs(node.left), 0)` — only take positive contributions
   - Recursively compute `rightGain = max(dfs(node.right), 0)` — only take positive contributions
   - Compute `currentPathSum = node.val + leftGain + rightGain` — the sum if the path goes through this node using both branches
   - Update `maxSum = max(maxSum, currentPathSum)`
   - Return `node.val + max(leftGain, rightGain)` — the maximum gain this node can provide to its parent (only one branch)
3. Call `dfs(root)` and return `maxSum`

**Why This Works:** The key insight is that at each node, there are two scenarios:
1. **Path that goes through this node using both children**: This is a candidate for the maximum path sum. The path starts somewhere in the left subtree, goes up to this node, and down to somewhere in the right subtree.
2. **Path that goes through this node to its parent**: Only one branch can be used (the larger one), because the path cannot fork when going upward.

By taking `max(gain, 0)` for each child's contribution, we effectively ignore negative contributions — it's always better to not include a negative subtree than to include it.

**Complexity Analysis:**
- **Time:** O(n) — each node is visited exactly once
- **Space:** O(h) — the recursion stack depth equals the height of the tree

## Deep Dive: Post-Order DFS Walkthrough

### Visual Walkthrough

For the tree `[-10, 9, 20, -1, -1, 15, 7]`:

```
     -10
     /  \
    9   20
       /  \
      15   7
```

Tracing the DFS:

```
dfs(9): left=max(dfs(null),0)=0, right=max(dfs(null),0)=0
        pathSum = 9 + 0 + 0 = 9
        maxSum = max(-inf, 9) = 9
        return 9 + max(0, 0) = 9   [gain to parent]

dfs(15): left=0, right=0
         pathSum = 15 + 0 + 0 = 15
         maxSum = max(9, 15) = 15
         return 15 + max(0, 0) = 15

dfs(7): left=0, right=0
        pathSum = 7 + 0 + 0 = 7
        maxSum = max(15, 7) = 15
        return 7 + max(0, 0) = 7

dfs(20): left=max(dfs(15),0)=15, right=max(dfs(7),0)=7
         pathSum = 20 + 15 + 7 = 42
         maxSum = max(15, 42) = 42
         return 20 + max(15, 7) = 35   [gain to parent]

dfs(-10): left=max(dfs(9),0)=9, right=max(dfs(20),0)=35
          pathSum = -10 + 9 + 35 = 34
          maxSum = max(42, 34) = 42
          return -10 + max(9, 35) = 25   [gain to parent]

Result: maxSum = 42
```

The optimal path is `15 -> 20 -> 7`, with sum `15 + 20 + 7 = 42`.

### Edge Cases to Consider

- **Single node with negative value**: The node itself is the only path. Returns its value (e.g., `-5`).
- **All negative values**: The maximum path sum is the largest single node value (since adding any neighbor would make it smaller).
- **Root with negative value but positive children**: The optimal path may skip the root entirely and stay within a subtree.
- **Deep tree with alternating signs**: The algorithm correctly evaluates all possible paths via the post-order traversal.
- **Tree with zero values**: Zero contributions are treated as neutral — they don't hurt but don't help either.

### Why Negative Gains Are Clipped to Zero

The `max(gain, 0)` step is crucial. Consider a node whose left subtree has a maximum path sum of `-5` to the left child. Including this negative contribution would only reduce the sum at the current node. By clipping to `0`, we effectively say "it's better to not include this subtree at all."

This doesn't mean we lose the possibility of a negative node being in the optimal path — if the node itself is in the path, its value is still counted. We only clip the *contribution* from subtrees, which represents the optional choice of extending the path into a subtree.

### Why One Branch for Return

When returning a value to the parent, we can only take one branch (`node.val + max(leftGain, rightGain)`). This is because a path cannot fork: if we went both left and right from the current node, the path would have a "V" shape and could not continue upward to the parent. The path that continues upward must be a straight line through the current node and one of its children.

class Solution {
    // Variable to track the global maximum path sum
    private int maxSum = Integer.MIN_VALUE;

    public int maxPathSum(TreeNode root) {
        dfs(root);  // Compute gains and track max path sum
        return maxSum;
    }

    // Returns the maximum gain this node can contribute to its parent
    // Also updates maxSum with paths that pass through this node
    private int dfs(TreeNode node) {
        if (node == null) return 0;  // Empty subtree contributes nothing

        // Compute maximum gains from left and right subtrees
        // Use max(gain, 0) to ignore negative contributions
        int leftGain = Math.max(dfs(node.left), 0);
        int rightGain = Math.max(dfs(node.right), 0);

        // Path through this node using both branches
        int currentPathSum = node.val + leftGain + rightGain;

        // Update the global maximum
        maxSum = Math.max(maxSum, currentPathSum);

        // Return the maximum gain to parent (only one branch)
        return node.val + Math.max(leftGain, rightGain);
    }
}

#include <limits.h>

// Helper that returns max gain to parent, updates maxSum via pointer
int dfs(struct TreeNode* node, int* maxSum) {
    if (node == NULL) return 0;

    // Compute gains from children, ignoring negative contributions
    int leftGain = dfs(node->left, maxSum);
    int rightGain = dfs(node->right, maxSum);
    if (leftGain < 0) leftGain = 0;
    if (rightGain < 0) rightGain = 0;

    // Path through this node using both branches
    int currentPathSum = node->val + leftGain + rightGain;

    // Update the global maximum
    if (currentPathSum > *maxSum) *maxSum = currentPathSum;

    // Return maximum gain to parent (only one branch)
    int maxChild = leftGain > rightGain ? leftGain : rightGain;
    return node->val + maxChild;
}

int maxPathSum(struct TreeNode* root) {
    int maxSum = INT_MIN;  // Track the global maximum
    dfs(root, &maxSum);
    return maxSum;
}

## Key Takeaways

- **Post-order DFS** is the essential pattern — we need children's information before we can compute at the parent
- The **max(gain, 0) clipping** elegantly handles negative subtrees
- The **split between "path through both children" and "gain to parent"** is the core insight — a path can fork at a node (making it a candidate for the global max), but cannot fork when continuing upward
- **O(n) time and O(h) space** — optimal for this problem
- This is a classic problem that demonstrates the power of **recursive tree traversal with state tracking**
