# Kth Smallest Element in a BST — Solution

## Problem Overview

Given the root of a Binary Search Tree and an integer `k`, return the kth smallest value (1-indexed) in the BST. Since in-order traversal of a BST yields sorted order, the kth visited node during an in-order traversal is the kth smallest element.

## Approach 1: Iterative In-Order Traversal (Optimal)

**Idea:** Use an explicit stack to perform an in-order traversal iteratively. Keep a counter of visited nodes. Stop when the counter reaches k and return the current node's value.

**Algorithm:**
1. Initialize an empty stack and set `curr = root`
2. While the stack is not empty or `curr` is not null:
   - Go as far left as possible, pushing each node onto the stack
   - Pop a node from the stack (this is the next in-order node)
   - Increment the counter
   - If the counter equals k, return the node's value
   - Move to the right child
3. Return -1 (should not reach here if k is valid)

**Why This Works:** In-order traversal (left → root → right) visits nodes in ascending order for a BST. By stopping at the kth visited node, we get the kth smallest element without traversing the entire tree.

**Complexity Analysis:**
- **Time:** O(h + k) — we traverse k nodes, where h is the height of the tree (for the initial descent to the leftmost node)
- **Space:** O(h) — the stack holds at most h nodes

## Approach 2: Recursive In-Order Traversal

**Idea:** Perform a recursive in-order traversal while maintaining a counter. When the counter reaches k, record the answer.

**Complexity Analysis:**
- **Time:** O(n) in the worst case (if k = n)
- **Space:** O(h) for the recursion stack

class Solution {
    public int kthSmallest(TreeNode root, int k) {
        Stack<TreeNode> stack = new Stack<>();    // Stack for iterative traversal
        TreeNode curr = root;                     // Start at the root
        int count = 0;                            // Track visited node count

        while (curr != null || !stack.isEmpty()) {
            // Go as far left as possible, pushing nodes onto the stack
            while (curr != null) {
                stack.push(curr);
                curr = curr.left;
            }

            // Pop the next in-order node
            curr = stack.pop();
            count++;

            // If this is the kth node, return its value
            if (count == k) return curr.val;

            // Move to the right subtree
            curr = curr.right;
        }

        return -1;                                 // k is invalid (should not reach)
    }
}

            // Move to the right subtree
            curr = node.borrow().right.clone();
        }

        while (curr != NULL) {

            curr = curr->left;

        curr = stack[top--];

        if (count == k) {

            free(stack);
            return result;
        }

        // Move to the right subtree

    }

func kthSmallest(root *TreeNode, k int) int {
    stack := []*TreeNode{}                         // Stack for iterative traversal
    curr := root                                   // Start at the root
    count := 0                                     // Track visited node count

    for curr != nil || len(stack) > 0 {

        for curr != nil {

## Key Takeaways

- **In-order traversal of a BST visits nodes in ascending order** — this is the key insight
- **The iterative approach allows early termination** — we stop as soon as we find the kth element, without traversing the rest of the tree
- **The stack-based approach uses O(h) space**, where h is the tree height
- **For follow-up questions** (what if the BST is modified frequently?), consider augmenting each node with the size of its left subtree to achieve O(log n) per query
- Both approaches handle the edge case of an empty tree properly (return -1 or handle gracefully)
