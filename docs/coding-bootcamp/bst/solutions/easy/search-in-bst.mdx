# Search in a Binary Search Tree — Solution

## Problem Overview

Given the `root` of a Binary Search Tree and a `target` value, return `true` if the target exists in the tree, and `false` otherwise. The BST property (left < root < right) allows us to search efficiently by eliminating one subtree at each step.

## Approach 1: Recursive BST Search

**Idea:** Use the BST property to search recursively. At each node, compare the target with the node's value. If equal, return true. If the target is smaller, search the left subtree. If larger, search the right subtree. Base case: if the node is null, return false.

**Algorithm:**
1. If the root is null, return false (target not found)
2. If `root.val == target`, return true
3. If `target < root.val`, recursively search `root.left`
4. If `target > root.val`, recursively search `root.right`

**Complexity Analysis:**
- **Time:** O(h) where h is the height of the tree. O(log n) average for balanced BST, O(n) worst case for skewed tree
- **Space:** O(h) for the recursion stack

## Approach 2: Iterative BST Search (Optimal)

**Idea:** Instead of recursion, use a while loop to traverse the tree. This eliminates the recursion stack overhead.

**Algorithm:**
1. Initialize a pointer `curr` to the root
2. While `curr` is not null:
   - If `curr.val == target`, return true
   - If `target < curr.val`, set `curr = curr.left`
   - If `target > curr.val`, set `curr = curr.right`
3. Return false (target not found)

**Complexity Analysis:**
- **Time:** O(h) — same as recursive but without function call overhead
- **Space:** O(1) — only a single pointer variable

## Why Binary Search Tree Search Works

The BST property guarantees that for any node, all values in the left subtree are smaller and all values in the right subtree are larger. This means at each node we can definitively choose one direction, eliminating the other subtree entirely. This is the same principle as binary search on a sorted array, but applied to a tree structure.

class Solution {
    public boolean searchBST(TreeNode root, int target) {
        TreeNode curr = root;                    // Start at the root

        while (curr != null) {                    // Continue while there are nodes
            if (curr.val == target) {             // Found the target
                return true;
            } else if (target < curr.val) {       // Target is smaller → go left
                curr = curr.left;
            } else {                               // Target is larger → go right
                curr = curr.right;
            }
        }

        return false;                              // Target not found
    }
}

## Key Takeaways

- **BST search is O(log n) on average** because the BST property eliminates half the remaining tree at each step
- **Iterative search uses O(1) space**, which is better than the recursive version's O(h) stack space
- **The BST property is essential** — without it we would need to search the entire tree O(n)
- This is the foundation for more complex BST operations like insertion, deletion, and validation
