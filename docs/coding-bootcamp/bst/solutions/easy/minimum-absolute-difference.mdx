# Minimum Absolute Difference in BST — Solution

## Problem Overview

Given the root of a Binary Search Tree, find the minimum absolute difference between the values of any two different nodes. Since in-order traversal of a BST yields sorted values, the minimum difference must occur between adjacent values in the in-order sequence.

## Approach: In-Order Traversal with Tracking

**Idea:** Perform an in-order traversal of the BST. Because in-order traversal visits nodes in ascending order, the minimum absolute difference between any two nodes will be the minimum difference between consecutive visited nodes.

**Algorithm:**
1. Initialize `prev` to null (previous node visited) and `min_diff` to infinity
2. Perform an in-order traversal (left, visit, right)
3. When visiting each node:
   - If `prev` is not null, compute `diff = node.val - prev.val` (positive since in-order is sorted)
   - Update `min_diff = min(min_diff, diff)`
   - Set `prev = node`
4. Return `min_diff`

**Why This Works:** In a sorted sequence, the minimum difference between any two elements is always found between adjacent elements. In-order traversal of a BST produces the sorted sequence, so we only need to check differences between consecutive elements in the traversal order.

**Complexity Analysis:**
- **Time:** O(n) — each node is visited exactly once during the in-order traversal
- **Space:** O(h) — recursion stack depth equals the height of the tree (O(n) worst case for skewed tree)

## Edge Cases
- **Single node:** No pair exists. Return the maximum integer value (or handle gracefully)
- **Empty tree:** No nodes to compare. Return 0
- **Negative values:** The algorithm handles negatives correctly since we always compute `current - previous`, which is positive in sorted order
- **Skewed tree (linked list):** The algorithm still works in O(n) time

class Solution {
    private Integer prev = null;                 // Previous node value in in-order
    private int minDiff = Integer.MAX_VALUE;     // Track minimum difference

    public int getMinimumDifference(TreeNode root) {
        inorder(root);                           // Perform in-order traversal
        return minDiff;
    }

    private void inorder(TreeNode node) {
        if (node == null) return;                // Base case: empty subtree

        inorder(node.left);                      // Visit left subtree first

        if (prev != null) {                      // Compare with previous value
            minDiff = Math.min(minDiff, node.val - prev);
        }
        prev = node.val;                         // Update previous value

        inorder(node.right);                     // Visit right subtree
    }
}

use std::cell::RefCell;
use std::rc::Rc;

fn get_minimum_difference(root: Option<Rc<RefCell<TreeNode>>>) -> i32 {
    let mut prev: Option<i32> = None;            // Previous node value in in-order
    let mut min_diff = i32::MAX;                 // Track minimum difference

    fn inorder(
        node: Option<Rc<RefCell<TreeNode>>>,
        prev: &mut Option<i32>,
        min_diff: &mut i32
    ) {
        if let Some(n) = node {                  // If node exists
            inorder(n.borrow().left.clone(), prev, min_diff); // Left subtree

            if let Some(p) = *prev {             // Compare with previous value
                *min_diff = (*min_diff).min(n.borrow().val - p);
            }
            *prev = Some(n.borrow().val);        // Update previous value

            inorder(n.borrow().right.clone(), prev, min_diff); // Right subtree
        }
    }

    inorder(root, &mut prev, &mut min_diff);
    min_diff
}

func getMinimumDifference(root *TreeNode) int {
    var prev *int                                // Previous node value in in-order
    minDiff := int(^uint(0) >> 1)                // Max int (sentinel)

    var inorder func(node *TreeNode)
    inorder = func(node *TreeNode) {
        if node == nil {                         // Base case: empty subtree
            return
        }

        inorder(node.Left)                       // Visit left subtree first

        if prev != nil {                         // Compare with previous value
            diff := node.Val - *prev
            if diff < minDiff {
                minDiff = diff
            }
        }
        val := node.Val
        prev = &val                              // Update previous value

        inorder(node.Right)                      // Visit right subtree
    }

    inorder(root)
    return minDiff
}

## Key Takeaways

- **In-order traversal of a BST gives sorted order** — this is the key insight for many BST problems
- **The minimum difference between any two numbers in a sorted array is always between adjacent numbers** — so we only need to check consecutive in-order nodes
- **Tracking the previous node** during traversal avoids storing the entire sorted sequence
- This technique extends to other problems like finding the kth smallest element, validating BST, and finding mode in BST
