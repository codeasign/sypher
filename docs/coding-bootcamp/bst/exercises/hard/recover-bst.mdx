# Recover Binary Search Tree

**Difficulty:** Hard

You are given the `root` of a Binary Search Tree where **exactly two nodes** have been swapped by mistake. Recover the tree by changing the values of the two nodes back to their correct positions without changing the structure.

You must solve this using **O(1) extra space** (excluding the recursion stack).

## Example 1

**Input:**
```
5
1 3 -1 -1 2
```

The BST is:
```
  1
 /
3
 \
  2
```

**Output:** `3 1 -1 -1 2`
**Explanation:** Nodes with values 1 and 3 were swapped. After swapping them back, the root becomes 3 and its left child becomes 1:
```
  3
 /
1
 \
  2
```

## Example 2

**Input:**
```
3
2 -1 1
```

The BST is:
```
  2
   \
    1
```

**Output:** `1 -1 2`
**Explanation:** After recovery:
```
  1
   \
    2
```

## Example 3

**Input:**
```
7
8 1 10 3 6 -1 14
```

The BST is:
```
      8
     / \
    1   10
   / \    \
  3   6   14
```

**Output:** `8 3 10 1 6 -1 14`
**Explanation:** Nodes with values 1 and 3 were swapped. After swapping them back:
```
      8
     / \
    3   10
   / \    \
  1   6   14
```

## Input Specification

- First line: an integer `n` — number of nodes in the BST (`0 <= n <= 10^4`)
- Second line: `n` space-separated integers representing the level-order traversal of the BST (use `-1` for null nodes)
- The tree is a valid BST except for exactly two swapped nodes

## Output Specification

Print the level-order traversal of the recovered BST (space-separated integers, `-1` for null). If the tree is empty, print `0`.

## Constraints

- The number of nodes is in the range `[0, 10^4]`
- `-2^31 <= Node.val <= 2^31 - 1`
- Exactly two nodes have been swapped
- Solve in O(n) time with O(1) extra space (excluding recursion stack)

## Hints

<details>
<summary>Hint 1</summary>

An in-order traversal of a BST should produce a strictly increasing sequence. After swapping two nodes, there will be either one or two pairs where the sequence is out of order.
</details>

<details>
<summary>Hint 2</summary>

When you find a pair where `prev.val > curr.val` during in-order traversal, the first occurrence identifies the first swapped node (prev), and the last occurrence identifies the second swapped node (curr).
</details>

<details>
<summary>Hint 3</summary>

To achieve O(1) extra space, use Morris traversal (threaded binary tree traversal) instead of recursion or an explicit stack.
</details>
