# Implement Queue using Stacks

**Difficulty:** Easy

Implement a first-in-first-out (FIFO) queue using only two stacks. The implemented queue should support all the functions of a normal queue (`push`, `pop`, `peek`, and `empty`).

## Example

**Input:**
```
push 1
push 2
pop
peek
empty
```

**Output:**
```
1
1
false
```

**Explanation:**
- `push(1)` adds 1 to the queue
- `push(2)` adds 2 to the queue
- `pop()` removes 1 (FIFO order)
- `peek()` returns 1 (the front element)
- `empty()` returns false (queue still has 2)

## Input Specification

Each line contains a command with optional arguments:
- `push x` — adds x to the back of the queue
- `pop` — removes and returns the front element
- `peek` — returns the front element without removing it
- `empty` — returns whether the queue is empty

## Output Specification

For each `pop` and `peek` command, output the result as an integer. For each `empty` command, output `true` or `false`.

## Constraints

- `1 <= x <= 1000`
- At most 1000 operations total
- All `pop` and `peek` calls are valid (queue is never empty)

## Hints

<details>
<summary>Hint 1</summary>

Use two stacks: one for input (push operations) and one for output (pop/peek operations). When the output stack is empty, transfer all elements from the input stack to the output stack.
</details>

<details>
<summary>Hint 2</summary>

The transfer from input to output reverses the order, effectively converting LIFO to FIFO.
</details>

<details>
<summary>Hint 3</summary>

For `peek()`, you can reuse the same logic as `pop()` but without removing the element.
</details>

</ProblemEditor>
