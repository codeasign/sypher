# Sliding Window Maximum — Solution

## Problem Overview

Given an array `nums` and a window size `k`, find the maximum element in each sliding window as it moves from left to right.

## Approach 1: Brute Force

**Idea:** For each window position, scan all k elements to find the maximum.

**Algorithm:**
1. Iterate through each possible window start position (0 to n-k).
2. For each window, scan k elements to find the maximum.
3. Add the maximum to the result.

**Complexity Analysis:**
- **Time:** O(n * k) — for each of n windows, we scan k elements.
- **Space:** O(1) — excluding the output array.

**Why this is suboptimal:** For large n and k, this is O(n*k) which can be O(n^2).

## Approach 2: Monotonic Deque (Optimal)

**Idea:** Use a deque to store indices of elements in decreasing order of value. The front of the deque always holds the index of the maximum element in the current window.

**Algorithm:**
1. Initialize an empty deque and an empty result array.
2. For each element `nums[i]`:
   - Remove indices from the back of the deque while the corresponding values are less than `nums[i]` (they can never be the maximum for any future window).
   - Add the current index `i` to the back of the deque.
   - If the front index is out of the window (i - k), remove it from the front.
   - If the window has at least k elements (i >= k-1), add `nums[deque.front()]` to the result.
3. Return the result.

**Why This Works:** The deque maintains a monotonically decreasing sequence of values. When a new element arrives, all smaller elements at the back of the deque become irrelevant — they can never be the maximum for the current or any future window. The front always contains the maximum for the current window.

**Complexity Analysis:**
- **Time:** O(n) — each element is added and removed from the deque at most once.
- **Space:** O(k) — the deque stores at most k indices.

## Deep Dive

### Visual Walkthrough

For `nums = [1,3,-1,-3,5,3,6,7]`, `k = 3`:

```
i=0, n=1: deque=[0],          result=[]       (window not full)
i=1, n=3: deque=[1],          result=[]       (1 removed, 3>1)
i=2, n=-1: deque=[1,2],       result=[3]      (nums[1]=3 is max)
i=3, n=-3: deque=[1,2,3],     result=[3,3]    (front=1, still in window)
i=4, n=5: deque=[4],          result=[3,3,5]  (all smaller values removed)
i=5, n=3: deque=[4,5],        result=[3,3,5,5] (5 remains max)
i=6, n=6: deque=[6],          result=[3,3,5,5,6] (3,5 removed)
i=7, n=7: deque=[7],          result=[3,3,5,5,6,7]
```

### Edge Cases

- **k=1**: Each element is its own window's maximum.
- **k=n**: One window containing the entire array.
- **Decreasing array**: The deque holds all elements, and the front is always the first element until it slides out.
- **Increasing array**: The deque always holds only the newest element (since all previous elements are smaller).
- **All equal values**: The deque holds indices, and the front slides out when its index is out of range.

```java
import java.util.*;

class Solution {
    public int[] maxSlidingWindow(int[] nums, int k) {
        if (nums.length == 0) return new int[0];

        int n = nums.length;
        int[] result = new int[n - k + 1];  // output array
        Deque<Integer> dq = new ArrayDeque<>();  // stores indices
        int ri = 0;  // result index

        for (int i = 0; i < n; i++) {
            // Remove indices from the back whose values are smaller than nums[i]
            // These can never be the maximum for any future window
            while (!dq.isEmpty() && nums[dq.peekLast()] < nums[i]) {
                dq.pollLast();
            }

            // Add the current index to the back of the deque
            dq.offerLast(i);

            // Remove the front index if it's outside the current window
            if (dq.peekFirst() == i - k) {
                dq.pollFirst();
            }

            // The front of the deque is the maximum for the current window
            if (i >= k - 1) {
                result[ri++] = nums[dq.peekFirst()];
            }
        }
        return result;
    }
}
```

```rust
use std::collections::VecDeque;

fn max_sliding_window(nums: &[i32], k: i32) -> Vec<i32> {
    if nums.is_empty() {
        return vec![];
    }

    let k = k as usize;
    let mut dq: VecDeque<usize> = VecDeque::new();  // stores indices
    let mut result = Vec::new();

    for (i, &n) in nums.iter().enumerate() {
        // Remove indices from the back whose values are smaller than nums[i]
        while let Some(&back) = dq.back() {
            if nums[back] < n {
                dq.pop_back();
            } else {
                break;
            }
        }

        // Add the current index to the back of the deque
        dq.push_back(i);

        // Remove the front index if it's outside the current window
        if let Some(&front) = dq.front() {
            if front == i - k {
                dq.pop_front();
            }
        }

        // The front of the deque is the maximum for the current window
        if i >= k - 1 {
            result.push(nums[dq.front().unwrap()]);
        }
    }
    result
}
```

```c
#include <stdlib.h>

int* maxSlidingWindow(int* nums, int numsSize, int k, int* returnSize) {
    if (numsSize == 0 || k == 0) {
        *returnSize = 0;
        return NULL;
    }

    *returnSize = numsSize - k + 1;
    int* result = (int*)malloc(*returnSize * sizeof(int));
    int* dq = (int*)malloc(numsSize * sizeof(int));  // stores indices
    int front = 0, back = 0;  // deque pointers
    int ri = 0;  // result index

    for (int i = 0; i < numsSize; i++) {
        // Remove indices from the back whose values are smaller than nums[i]
        while (back > front && nums[dq[back - 1]] < nums[i]) {
            back--;
        }

        // Add the current index to the back of the deque
        dq[back++] = i;

        // Remove the front index if it's outside the current window
        if (dq[front] == i - k) {
            front++;
        }

        // The front of the deque is the maximum for the current window
        if (i >= k - 1) {
            result[ri++] = nums[dq[front]];
        }
    }

    free(dq);
    return result;
}
```

```csharp
using System.Collections.Generic;

public class Solution {
    public int[] MaxSlidingWindow(int[] nums, int k) {
        if (nums.Length == 0) return new int[0];

        int n = nums.Length;
        int[] result = new int[n - k + 1];
        LinkedList<int> dq = new LinkedList<int>();  // stores indices
        int ri = 0;  // result index

        for (int i = 0; i < n; i++) {
            // Remove indices from the back whose values are smaller than nums[i]
            while (dq.Count > 0 && nums[dq.Last.Value] < nums[i]) {
                dq.RemoveLast();
            }

            // Add the current index to the back of the deque
            dq.AddLast(i);

            // Remove the front index if it's outside the current window
            if (dq.First.Value == i - k) {
                dq.RemoveFirst();
            }

            // The front of the deque is the maximum for the current window
            if (i >= k - 1) {
                result[ri++] = nums[dq.First.Value];
            }
        }
        return result;
    }
}
```

```go
func maxSlidingWindow(nums []int, k int) []int {
    if len(nums) == 0 {
        return []int{}
    }

    n := len(nums)
    result := make([]int, n-k+1)
    dq := make([]int, 0)  // stores indices
    ri := 0  // result index

    for i, nv := range nums {
        // Remove indices from the back whose values are smaller than nums[i]
        for len(dq) > 0 && nums[dq[len(dq)-1]] < nv {
            dq = dq[:len(dq)-1]
        }

        // Add the current index to the back of the deque
        dq = append(dq, i)

        // Remove the front index if it's outside the current window
        if dq[0] == i-k {
            dq = dq[1:]
        }

        // The front of the deque is the maximum for the current window
        if i >= k-1 {
            result[ri] = nums[dq[0]]
            ri++
        }
    }
    return result
}
```

## Key Takeaways

- **Monotonic deque** maintains a decreasing sequence of values, providing O(1) access to the maximum.
- **Indices, not values**, are stored in the deque so we can check if elements are still in the window.
- **O(n)** time complexity — each element is pushed and popped at most once.
- Key invariant: the deque is always sorted in decreasing order of values.
