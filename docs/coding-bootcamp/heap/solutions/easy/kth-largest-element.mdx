# Kth Largest Element in a Stream — Solution

## Problem Overview

Design a class `KthLargest` that:
- **Constructor:** Receives an integer `k` and an array of initial numbers `nums`
- **Method `add(val)`:** Adds `val` to the stream and returns the **kth largest element** in the stream so far

The challenge is to return the kth largest element efficiently after each insertion, without sorting the entire stream each time.

## Brute Force Approach

**Idea:** Maintain a sorted list. Each time a new value is added, insert it in the correct position and return the element at index `len - k`.

**Algorithm:**
1. In the constructor, sort `nums` and store the sorted list
2. In `add(val)`, insert `val` into the sorted list using binary search to find the correct position
3. Return the element at index `len - k`

**Complexity Analysis:**
- **Time:** O(n) per insertion — binary search is O(log n) but inserting into a list is O(n) due to shifting elements
- **Space:** O(n) — storing all elements

**Why this could be better:** For large streams or many `add()` calls, O(n) per insertion is slow. The optimal approach reduces each insertion to O(log k).

## Optimal Approach: Min-Heap of Size k

**Idea:** Maintain a **min-heap** of exactly size `k`. The heap stores the k largest elements seen so far. Since it's a min-heap, the smallest element in this group (the top) is exactly the kth largest overall.

**Algorithm:**
1. In the constructor, add all elements from `nums` to the min-heap. If the heap grows beyond `k` elements, pop the smallest.
2. In `add(val)`, push `val` onto the heap. If the heap now has more than `k` elements, pop the smallest. Return the top of the heap.

**Why This Works:** The min-heap of size k always contains the k largest elements seen so far. The smallest element in that heap (its root) is therefore the kth largest overall. Larger elements remain in the heap, and smaller ones are evicted.

**Complexity Analysis:**
- **Time:** O(n log k + m log k) where n is the initial length and m is the number of `add()` calls — each heap operation is O(log k)
- **Space:** O(k) — the heap stores at most k elements

## Deep Dive: Tracing the Min-Heap

### Visual Walkthrough

Let's trace `k = 3, nums = [4, 5, 8, 2]` through the algorithm.

**Initialization:**

```
nums = [4, 5, 8, 2], k = 3

Step 1: Push 4 → heap = [4]           size=1 < 3, keep
Step 2: Push 5 → heap = [4, 5]        size=2 < 3, keep
Step 3: Push 8 → heap = [4, 5, 8]     size=3 == k, keep
Step 4: Push 2 → heap = [2, 5, 8]     size=4 > 3, pop → heap = [4, 5, 8]
         (2 < 4, so min is 2, popped)  (After pop: [4, 5, 8])

Result: top = 4  (kth largest)
```

**add(3):**

```
heap = [4, 5, 8]  (min-heap, top = 4)

Step 1: Push 3 → heap = [3, 5, 8, 4]
         Bubble up: 3 < 4, so 3 moves to top
Step 2: Size = 4 > 3, pop → remove 3 (min)
         Heap after pop: [4, 5, 8]

top = 4  ← kth largest
```

**add(5):**

```
heap = [4, 5, 8]

Step 1: Push 5 → heap = [4, 5, 8, 5]
        5 > 4, no bubble needed
Step 2: Size = 4 > 3, pop → remove 4
         Heap after pop: [5, 5, 8]

top = 5  ← kth largest
```

**add(10):**

```
heap = [5, 5, 8]

Step 1: Push 10 → heap = [5, 5, 8, 10]
        10 > 5, no bubble needed
Step 2: Size = 4 > 3, pop → remove 5
         Heap after pop: [5, 8, 10]

top = 5  ← kth largest
```

### Key Insight

The min-heap acts as a **filter**: it only keeps the k largest elements. Any element smaller than the current kth largest gets pushed and immediately popped. This is why the heap never grows beyond size k, keeping each operation at O(log k).

### Edge Cases

- **k = 1:** The heap always has exactly 1 element (the maximum). Each `add()` compares and keeps the larger value.
- **Empty initial stream:** The heap starts empty. The first `k` additions fill it up before any element can be evicted.
- **Duplicate values:** Duplicates are handled naturally — they compete for space in the heap like any other value.
- **Constant values:** If every element is the same, the answer never changes. The heap maintains identical values correctly.

```java
import java.util.PriorityQueue;

class KthLargest {
    private PriorityQueue<Integer> minHeap;
    private int k;

    public KthLargest(int k, int[] nums) {
        this.k = k;
        minHeap = new PriorityQueue<>(k);           // Min-heap of size k
        for (int num : nums) {
            minHeap.offer(num);                      // Add each initial number
            if (minHeap.size() > k) minHeap.poll();  // Keep only k largest
        }
    }

    public int add(int val) {
        minHeap.offer(val);                          // Add new value to heap
        if (minHeap.size() > k) minHeap.poll();      // Evict if over size k
        return minHeap.peek();                       // Top is kth largest
    }
}
```

```typescript
class KthLargest {
    private heap: number[];
    private k: number;

    constructor(k: number, nums: number[]) {
        this.k = k;
        this.heap = [];
        for (const num of nums) this.add(num);
    }

    add(val: number): number {
        this.heap.push(val);
        this.bubbleUp(this.heap.length - 1);
        if (this.heap.length > this.k) {
            this.swap(0, this.heap.length - 1);
            this.heap.pop();
            this.bubbleDown(0);
        }
        return this.heap[0];
    }

    private bubbleUp(idx: number): void {
        while (idx > 0) {
            const parent = Math.floor((idx - 1) / 2);
            if (this.heap[parent] <= this.heap[idx]) break;
            this.swap(parent, idx);
            idx = parent;
        }
    }

    private bubbleDown(idx: number): void {
        const n = this.heap.length;
        while (true) {
            let smallest = idx;
            const left = 2 * idx + 1, right = 2 * idx + 2;
            if (left < n && this.heap[left] < this.heap[smallest]) smallest = left;
            if (right < n && this.heap[right] < this.heap[smallest]) smallest = right;
            if (smallest === idx) break;
            this.swap(idx, smallest);
            idx = smallest;
        }
    }

    private swap(i: number, j: number): void {
        [this.heap[i], this.heap[j]] = [this.heap[j], this.heap[i]];
    }
}
```

```rust
use std::collections::BinaryHeap;
use std::cmp::Reverse;

struct KthLargest {
    heap: BinaryHeap<Reverse<i32>>,  // BinaryHeap is max-heap; Reverse makes it min-heap
    k: usize,
}

impl KthLargest {
    fn new(k: i32, nums: Vec<i32>) -> Self {
        let k = k as usize;
        let mut heap = BinaryHeap::with_capacity(k + 1);
        let mut obj = KthLargest { heap, k };
        for num in nums {
            obj.add(num);
        }
        obj
    }

    fn add(&mut self, val: i32) -> i32 {
        self.heap.push(Reverse(val));
        if self.heap.len() > self.k {
            self.heap.pop();
        }
        self.heap.peek().unwrap().0
    }
}
```

```c
#include <stdlib.h>

typedef struct {
    int* heap;    // Min-heap array
    int size;     // Current number of elements
    int k;        // Max capacity (k)

void swap(int* a, int* b) { int t = *a; *a = *b; *b = t; }

void bubbleUp(int* heap, int idx) {
    while (idx > 0) {
        int parent = (idx - 1) / 2;
        if (heap[parent] <= heap[idx]) break;
        swap(&heap[parent], &heap[idx]);
        idx = parent;
    }
}

void bubbleDown(int* heap, int n, int idx) {
    while (1) {
        int smallest = idx;
        int left = 2 * idx + 1, right = 2 * idx + 2;
        if (left < n && heap[left] < heap[smallest]) smallest = left;
        if (right < n && heap[right] < heap[smallest]) smallest = right;
        if (smallest == idx) break;
        swap(&heap[idx], &heap[smallest]);
        idx = smallest;
    }
}

KthLargest* kthLargestCreate(int k, int* nums, int numsSize) {
    KthLargest* obj = (KthLargest*)malloc(sizeof(KthLargest));
    obj->k = k;
    obj->size = 0;
    obj->heap = (int*)malloc((k + 1) * sizeof(int));
    for (int i = 0; i < numsSize; i++) {
        obj->heap[obj->size++] = nums[i];
        bubbleUp(obj->heap, obj->size - 1);
        if (obj->size > k) {
            obj->heap[0] = obj->heap[--obj->size];
            bubbleDown(obj->heap, obj->size, 0);
        }
    }
    return obj;
}

int kthLargestAdd(KthLargest* obj, int val) {
    obj->heap[obj->size++] = val;
    bubbleUp(obj->heap, obj->size - 1);
    if (obj->size > obj->k) {
        obj->heap[0] = obj->heap[--obj->size];
        bubbleDown(obj->heap, obj->size, 0);
    }
    return obj->heap[0];
}

void kthLargestFree(KthLargest* obj) {
    free(obj->heap);
    free(obj);
}
```

```csharp
using System;
using System.Collections.Generic;

class KthLargest {
    private List<int> heap;  // Min-heap (manual implementation)
    private int k;

    public KthLargest(int k, int[] nums) {
        this.k = k;
        this.heap = new List<int>();
        foreach (int num in nums) Add(num);
    }

    public int Add(int val) {
        heap.Add(val);
        BubbleUp(heap.Count - 1);
        if (heap.Count > k) {
            heap[0] = heap[heap.Count - 1];
            heap.RemoveAt(heap.Count - 1);
            BubbleDown(0);
        }
        return heap[0];
    }

    private void BubbleUp(int idx) {
        while (idx > 0) {
            int parent = (idx - 1) / 2;
            if (heap[parent] <= heap[idx]) break;
            Swap(parent, idx);
            idx = parent;
        }
    }

    private void BubbleDown(int idx) {
        int n = heap.Count;
        while (true) {
            int smallest = idx;
            int left = 2 * idx + 1, right = 2 * idx + 2;
            if (left < n && heap[left] < heap[smallest]) smallest = left;
            if (right < n && heap[right] < heap[smallest]) smallest = right;
            if (smallest == idx) break;
            Swap(idx, smallest);
            idx = smallest;
        }
    }

    private void Swap(int i, int j) {
        int t = heap[i]; heap[i] = heap[j]; heap[j] = t;
    }
}
```

## Key Takeaways

- **Min-heap of size k** is the optimal data structure for finding the kth largest element in a stream
- The heap automatically **discards elements smaller than the current kth largest**, maintaining only what matters
- Each `add()` operation runs in **O(log k)**, making it efficient even for large streams
- The same pattern applies to finding the kth *smallest* element — simply use a max-heap instead
