# Kth Largest Element in a Stream

**Difficulty:** Easy

Design a class to find the **kth largest element** in a stream of numbers. The class should support adding new numbers to the stream and returning the kth largest element after each addition.

## Example 1

**Input:**  
`k = 3, nums = [4, 5, 8, 2]`  
`add(3) → 4`  
`add(5) → 5`  
`add(10) → 5`  
`add(9) → 8`  
`add(4) → 8`

**Explanation:** After adding 3, the sorted stream is `[3, 4, 5, 8, 2]` → 3rd largest is 4. After adding 5, `[3, 4, 5, 5, 8, 2]` → 3rd largest is 5. After adding 10, `[2, 3, 4, 5, 5, 8, 10]` → 3rd largest is 5. After adding 9, `[2, 3, 4, 5, 5, 8, 9, 10]` → 3rd largest is 8.

## Example 2

**Input:**  
`k = 1, nums = []`  
`add(5) → 5`  
`add(3) → 3`

**Explanation:** With k = 1, we always return the maximum element in the stream.

## Input Specification

The first line contains two integers `k` and `n`. The second line contains `n` integers representing the initial stream. Each subsequent line contains a single integer representing a new element added to the stream.

## Output Specification

After each addition, print the kth largest element in the stream on a new line.

## Constraints

- `1 <= k <= 10^4`
- `0 <= n <= 10^4`
- `-10^4 <= nums[i] <= 10^4`
- `-10^4 <= val <= 10^4`
- At most `10^4` calls to `add()` will be made

## Hints

<details>
<summary>Hint 1</summary>

Think about what data structure efficiently keeps track of the k largest elements without sorting the entire stream each time.
</details>

<details>
<summary>Hint 2</summary>

A min-heap of size k is the perfect fit — it always gives you the kth largest element as the smallest element in the heap.
</details>

<details>
<summary>Hint 3</summary>

When the heap has fewer than k elements, push the new value. Once it reaches size k, compare the new value with the top of the heap before deciding whether to push and pop.
</details>
