# Two Sum — Solution

## Problem Overview

Given an array of integers `nums` and an integer `target`, return the indices of the two numbers that add up to `target`. You may assume exactly one solution exists, and you may not use the same element twice. Return the answer in any order.

## Brute Force Approach

**Idea:** Check every possible pair of numbers in the array to see if they sum to the target.

**Algorithm:**
1. Loop through each element `nums[i]` from index 0 to n-2
2. For each `nums[i]`, loop through every subsequent element `nums[j]` from i+1 to n-1
3. If `nums[i] + nums[j] == target`, return `[i, j]`
4. If no pair is found, return an empty list

**Complexity Analysis:**
- **Time:** O(n²) — two nested loops, each iterating up to n times
- **Space:** O(1) — only a constant amount of extra space

**Why this is suboptimal:** The nested loop approach is slow for large arrays. We can trade space for time by using a hash map to achieve a single pass.

## Optimal Approach: Hash Map (One Pass)

**Idea:** As we iterate through the array, store each number's value and index in a hash map. For each element, check if the complement (`target - nums[i]`) already exists in the map.

**Algorithm:**
1. Initialize an empty hash map to store `{value: index}` pairs
2. For each index `i` and value `nums[i]`:
   - Compute `complement = target - nums[i]`
   - If `complement` exists in the map, return `[map[complement], i]`
   - Otherwise, store `nums[i]` with its index `i` in the map
3. Since the problem guarantees a solution, the loop will always find it

**Why This Works:** The hash map remembers numbers we've already seen. For each new number, we check if the number needed to reach the target was seen earlier. This is equivalent to checking all pairs but avoids the nested loop by using O(1) lookups.

**Complexity Analysis:**
- **Time:** O(n) — each element is processed once with O(1) hash map operations
- **Space:** O(n) — the hash map stores at most n entries

  def two_sum(nums, target):
      seen = {}  # val -> index
      for i, val in enumerate(nums):
          complement = target - val
          if complement in seen:
              return [seen[complement], i]
          seen[val] = i
      return [-1, -1]

## Key Takeaways

- **Hash maps** provide O(1) lookup for complement search
- Trade O(n) space for O(n) time improvement over O(n²)
- One-pass approach avoids self-matches by checking the complement before storing the current element
