# Best Time to Buy and Sell Stock — Solution

## Problem Overview

You are given an array `prices` where `prices[i]` is the price of a given stock on day `i`. You want to maximize your profit by choosing a single day to buy one stock and choosing a different day in the future to sell that stock. Return the maximum profit you can achieve. If no profit can be made, return 0.

## Brute Force Approach

**Idea:** Check every possible buy-sell pair and track the maximum profit.

**Algorithm:**
1. Initialize `max_profit = 0`
2. For each day `i` from 0 to n-2 (buy day):
   - For each day `j` from i+1 to n-1 (sell day):
     - Compute `profit = prices[j] - prices[i]`
     - Update `max_profit = max(max_profit, profit)`
3. Return `max_profit`

**Complexity Analysis:**
- **Time:** O(n²) — two nested loops check every buy-sell pair
- **Space:** O(1) — only a constant amount of extra space

**Why this is suboptimal:** Checking every pair is unnecessary. The optimal approach tracks the minimum price seen so far in a single pass, computing potential profit at each step.

## Optimal Approach: One Pass

**Idea:** Iterate through the array once, keeping track of the minimum price seen so far. At each day, compute the profit if we sold at the current price (current price minus minimum price seen so far) and update the maximum profit.

**Algorithm:**
1. Initialize `min_price = infinity` and `max_profit = 0`
2. For each `price` in `prices`:
   - Update `min_price = min(min_price, price)` — track the lowest buying price seen so far
   - Compute `profit = price - min_price` — profit if we sell today
   - Update `max_profit = max(max_profit, profit)` — track the best profit
3. Return `max_profit`

**Why This Works:** The optimal profit is achieved by buying at the lowest price and selling at the highest price after that low point. By tracking the minimum price as we go, we implicitly consider every possible buy day. The single pass captures the best pair without nested loops.

**Complexity Analysis:**
- **Time:** O(n) — each price is visited exactly once
- **Space:** O(1) — only two integer variables

def max_profit(prices: list[int]) -> int:
    # Track the minimum price seen so far and the maximum profit possible
    min_price = float('inf')
    max_profit = 0

    for price in prices:
        # Update the minimum buying price encountered so far
        if price < min_price:
            min_price = price
        else:
            # Calculate profit if we sell at the current price
            profit = price - min_price
            # Update the maximum profit if this is better
            if profit > max_profit:
                max_profit = profit

    return max_profit

## Key Takeaways

- **Track the minimum** seen so far to avoid nested loops — this is a classic "kadane-lite" pattern
- **O(n) time, O(1) space** is optimal since you must examine every price at least once
- The same approach extends to variants like Best Time to Buy and Sell Stock II (multiple transactions)
- Always handle the edge case where prices are strictly decreasing (profit = 0)
