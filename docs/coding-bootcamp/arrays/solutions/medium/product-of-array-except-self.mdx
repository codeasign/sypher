# Product of Array Except Self — Solution

## Problem Overview

Given an integer array `nums`, return an array `answer` such that `answer[i]` is equal to the product of all elements of `nums` except `nums[i]`. You must solve it without using division and in O(n) time.

## Brute Force Approach

**Idea:** For each element, compute the product of all other elements by iterating through the array and skipping the current index.

**Algorithm:**
1. Initialize an array `result` of length n filled with 1
2. For each index `i` from 0 to n-1:
   - For each index `j` from 0 to n-1:
     - If `j != i`, multiply `result[i]` by `nums[j]`
3. Return `result`

**Complexity Analysis:**
- **Time:** O(n²) — for each of n elements, we loop through all n elements
- **Space:** O(1) — excluding the output array

**Why this is suboptimal:** The nested loop is unnecessary. The optimal approach precomputes prefix and suffix products, reducing the time to O(n) by recognizing that each element's result is the product of all elements to its left multiplied by all elements to its right.

## Optimal Approach: Prefix and Suffix Products

**Idea:** Compute the product of all elements to the left of each index and all elements to the right of each index, then multiply them together.

**Algorithm:**
1. Initialize `result` array of length n with all 1s
2. **Left pass:** Compute prefix products from left to right
   - `result[i]` holds the product of all elements to the left of `i`
   - `prefix = 1` initially; for each `i`, set `result[i] = prefix`, then multiply `prefix` by `nums[i]`
3. **Right pass:** Multiply by suffix products from right to left
   - `suffix = 1` initially; for each `i` from n-1 down to 0:
     - Multiply `result[i]` by `suffix` (the product of all elements to the right of `i`)
     - Multiply `suffix` by `nums[i]`
4. Return `result`

**Why This Works:** The product of all elements except `nums[i]` = (product of elements before `i`) × (product of elements after `i`). By computing these in two passes and storing the left products directly in the result array, we achieve O(n) time with O(1) extra space (excluding the output).

**Complexity Analysis:**
- **Time:** O(n) — two passes over the array
- **Space:** O(1) — excluding the output array; we reuse the result array for the left pass

def product_except_self(nums: list[int]) -> list[int]:
    n = len(nums)
    # Initialize result array with 1s
    result = [1] * n

    # Left pass: compute prefix products
    # result[i] will hold the product of all elements to the left of i
    prefix = 1
    for i in range(n):
        result[i] = prefix          # Store product of elements to the left
        prefix *= nums[i]           # Update prefix for the next position

    # Right pass: multiply by suffix products
    # result[i] is now multiplied by the product of all elements to the right of i
    suffix = 1
    for i in range(n - 1, -1, -1):
        result[i] *= suffix         # Multiply by product of elements to the right
        suffix *= nums[i]           # Update suffix for the next position

    return result

## Key Takeaways

- **Prefix and suffix products** is the standard pattern for problems that need "all elements except self" computations
- **Reuse the output array** for the left pass to achieve O(1) extra space
- The two-pass approach eliminates the need for division, which avoids issues with zero elements
- This pattern extends to other problems like range product queries and rain water trapping
