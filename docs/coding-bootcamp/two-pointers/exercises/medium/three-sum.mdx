# Three Sum

**Difficulty:** Medium

Given an integer array `nums`, return all the triplets
`[nums[i], nums[j], nums[k]]` such that `i != j`, `i != k`, and
`j != k`, and `nums[i] + nums[j] + nums[k] == 0`.

The solution set must not contain duplicate triplets.

## Example 1

**Input:** `nums = [-1,0,1,2,-1,-4]`  
**Output:** `[[-1,-1,2],[-1,0,1]]`  
**Explanation:** The two distinct triplets that sum to 0 are
`[-1,-1,2]` and `[-1,0,1]`. Note that the order of output and
the order of triplets does not matter.

## Example 2

**Input:** `nums = [0,1,1]`  
**Output:** `[]`  
**Explanation:** The only possible triplet does not sum to 0.

## Example 3

**Input:** `nums = [0,0,0]`  
**Output:** `[[0,0,0]]`  
**Explanation:** The only possible triplet sums to 0.

## Input Specification

The first line contains an integer `n` (the length of the array).
The second line contains `n` space-separated integers representing
the array `nums`.

## Output Specification

Print each unique triplet on a separate line as three space-separated
integers, sorted in ascending order within each triplet. If no
triplets sum to zero, print a blank line.

## Constraints

- `3 <= nums.length <= 3000`
- `-10ŌüĄ <= nums[i] <= 10ŌüĄ`

## Hints

<details>
<summary>Hint 1</summary>

Sort the array first. This lets you use two pointers and also skips
duplicates easily.
</details>

<details>
<summary>Hint 2</summary>

Fix one element with a for loop, then use two pointers on the remaining
elements to find pairs that sum to `-nums[i]`.
</details>

<details>
<summary>Hint 3</summary>

Skip duplicate values to avoid duplicate triplets. Skip at both the
outer loop and the inner while loop.
</details>
