# Remove Duplicates from Sorted Array II — Solution

## Problem Overview

Given a sorted array `nums`, remove some duplicates in-place such that each unique element appears **at most twice**. The relative order must be kept. Modify the array in-place with O(1) extra memory and return `k` (the new length).

## Brute Force Approach

**Idea:** Count frequencies, then rebuild the array.

**Algorithm:**
1. Count the frequency of each unique element using a hash map
2. Rebuild the array: for each unique element, place at most 2 copies
3. Return the new length

**Complexity Analysis:**
- **Time:** O(n) — one pass to count, one pass to rebuild
- **Space:** O(n) — hash map to store frequencies

**Why this is suboptimal:** The problem explicitly requires O(1) extra space. Also, the array is sorted, which means we don't need a hash map — duplicates are adjacent.

## Optimal Approach: Same-Direction Two Pointers (Slow/Fast)

**Idea:** Since the array is sorted and each element may appear at most twice, we use a slow pointer that marks where the next valid element goes, and a fast pointer that scans forward. The first two elements are always kept, so both pointers start at index 2.

**Algorithm:**
1. If `nums.length <= 2`, return `nums.length` (trivially valid)
2. Initialize `slow = 2`
3. For `fast = 2` to `n - 1`:
   - If `nums[fast] != nums[slow - 2]`:
     - Set `nums[slow] = nums[fast]` and increment `slow`
4. Return `slow` (the new length)

**Why This Works:** The condition `nums[fast] != nums[slow - 2]` is the key insight. The slow pointer tracks the position two steps behind where we're writing. If the current value differs from the value at `slow - 2`, it means we've seen at most two copies so far, so it's safe to write. Since the array is sorted, this check guarantees no value appears more than twice.

**Visual Explanation:**
```
Initial: [1, 1, 1, 2, 2, 3]
          0  1  2  3  4  5
              slow=2
                 fast=2

nums[2] == nums[0] (1 == 1) → skip, fast=3
nums[3] != nums[1] (2 != 1) → write nums[2]=2, slow=3, fast=4
nums[4] != nums[2] (2 != 2)? Actually nums[4]=2, nums[2]=2 → equal → skip
Actually let's trace more carefully:
slow=2, fast=2: nums[2]=1, nums[0]=1 → equal → skip, fast=3
slow=2, fast=3: nums[3]=2, nums[0]=1 → different → write nums[2]=2, slow=3, fast=4
slow=3, fast=4: nums[4]=2, nums[1]=1 → different → write nums[3]=2, slow=4, fast=5
slow=4, fast=5: nums[5]=3, nums[2]=2 → different → write nums[4]=3, slow=5
Result: [1, 1, 2, 2, 3, ...], k=5
```

**Complexity Analysis:**
- **Time:** O(n) — single pass through the array
- **Space:** O(1) — in-place modification

## Deep Dive: How the Slow/Fast Pointer Technique Works

### Visual Walkthrough

For `nums = [1, 1, 1, 2, 2, 3]`:

```
Initial: nums = [1, 1, 1, 2, 2, 3]
                0  1  2  3  4  5
                       slow=2
                       fast=2

Step 1: nums[2]=1 == nums[0]=1 → duplicate, skip
        fast=3

Step 2: nums[3]=2 != nums[0]=1 → write nums[2]=2
        slow=3, fast=4

Step 3: nums[4]=2 != nums[1]=1 → write nums[3]=2
        slow=4, fast=5

Step 4: nums[5]=3 != nums[2]=2 → write nums[4]=3
        slow=5, fast=6 → exit loop

Result: [1, 1, 2, 2, 3, ...], k=5
```

### The Key Insight: `nums[fast] != nums[slow - 2]`

The condition `nums[fast] != nums[slow - 2]` works because:
- `slow - 2` points to the element that was written two positions ago
- Since the array is sorted, if `nums[fast]` equals `nums[slow - 2]`, it means we've already seen at least two copies of this value
- If they differ, it's safe to write because we have at most one copy of this value in the result so far

### Edge Cases

- **Array length ≤ 2**: Trivially valid — all elements can stay
- **All elements the same**: Only the first two copies are kept
- **No duplicates**: Every element is written, slow pointer reaches n
- **All duplicates more than twice**: Each value is reduced to exactly 2 copies

class Solution {
    public int removeDuplicates(int[] nums) {
        // Edge case: array with 2 or fewer elements is trivially valid
        if (nums.length <= 2) return nums.length;

        // Slow pointer: position to write the next valid element
        int slow = 2;

        // Fast pointer: scans each element in the array
        for (int fast = 2; fast < nums.length; fast++) {
            // Compare current element with the element at slow - 2
            // If they differ, we have at most one copy of this value so far
            if (nums[fast] != nums[slow - 2]) {
                // Write the current element to the slow position
                nums[slow] = nums[fast];
                // Advance slow to the next write position
                slow++;
            }
            // If equal, skip — this value already has at least two copies
        }

        // Slow is the new length (number of valid elements)
        return slow;
    }
}

## Key Takeaways

- **Same-direction (slow/fast) pointers** are the tool for in-place array compaction
- The condition `nums[fast] != nums[slow - 2]` elegantly enforces the "at most twice" rule
- The first two elements are always kept — the slow pointer starts at 2
- This is **O(n) time, O(1) space** — the most efficient solution possible
