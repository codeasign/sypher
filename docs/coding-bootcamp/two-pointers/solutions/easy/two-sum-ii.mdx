# Two Sum II — Solution

## Problem Overview

Given a **1-indexed** sorted array of integers `numbers` and a `target` value, find two numbers that add up to the target. Return their 1-indexed positions as an array `[index1, index2]`. There is exactly one solution, and you may not use the same element twice. Your solution must use only constant extra space.

## Brute Force Approach

**Idea:** Check every pair of elements using nested loops.

**Algorithm:**
1. For each index `i` from 0 to n-1:
   - For each index `j` from i+1 to n-1:
     - If `numbers[i] + numbers[j] == target`, return `[i+1, j+1]`

**Complexity Analysis:**
- **Time:** O(n²) — we check every pair in the worst case
- **Space:** O(1) — no extra space used

**Why this is suboptimal:** The array is sorted, which means we can use the sorted property to eliminate the need for nested loops.

## Optimal Approach: Two Pointers

**Idea:** Since the array is sorted, we can use two pointers from opposite ends. If the sum is too small, we need a larger value (move left pointer right). If the sum is too large, we need a smaller value (move right pointer left).

**Algorithm:**
1. Initialize `left = 0` and `right = numbers.length - 1`
2. While `left < right`:
   - Compute `sum = numbers[left] + numbers[right]`
   - If `sum == target`: return `[left + 1, right + 1]` (1-indexed)
   - If `sum < target`: increment `left` (need a larger sum)
   - If `sum > target`: decrement `right` (need a smaller sum)
3. Return `[-1, -1]` (should never reach here since a solution is guaranteed)

**Why This Works:** The sorted array creates a monotonic property: moving left increases the sum, moving right decreases it. This is essentially a binary search-like process that narrows down the solution in a single pass.

**Complexity Analysis:**
- **Time:** O(n) — each element is visited at most once
- **Space:** O(1) — only two pointer variables

## Deep Dive: How the Two-Pointer Technique Works

### Visual Walkthrough

For the input `numbers = [2, 7, 11, 15]`, `target = 9`:

```
Initial: left = 0, right = 3
numbers = [2, 7, 11, 15]
           ↑            ↑
          left        right

Step 1: sum = 2 + 15 = 17 > 9 → too large, decrement right
numbers = [2, 7, 11, 15]
           ↑        ↑
          left    right

Step 2: sum = 2 + 11 = 13 > 9 → still too large, decrement right
numbers = [2, 7, 11, 15]
           ↑     ↑
          left  right

Step 3: sum = 2 + 7 = 9 == target → return [1, 2] (1-indexed)
```

### Why This Works

The sorted array creates a monotonic property:
- Moving `left` rightward **increases** the sum (since the array is sorted ascending)
- Moving `right` leftward **decreases** the sum

This is essentially a binary search-like narrowing process. At each step, we eliminate one element from consideration, guaranteeing O(n) time.

### Edge Cases

- **Minimum array size (n=2)**: The pointers start at the only two elements. Single iteration determines the result.
- **Negative numbers**: The same logic applies — negative numbers are just smaller values.
- **Duplicate values**: The problem guarantees exactly one solution, so duplicates don't cause ambiguity.
- **Target at extremes**: Very small or very large targets still work because the pointers adjust correctly.

class Solution {
    public int[] twoSum(int[] numbers, int target) {
        // Initialize two pointers: left at start, right at end
        int left = 0, right = numbers.length - 1;

        // Keep narrowing until pointers meet
        while (left < right) {
            // Compute the current sum
            int sum = numbers[left] + numbers[right];

            // Found the target pair — return 1-indexed positions
            if (sum == target)
                return new int[]{left + 1, right + 1};
            // Sum too small — need a larger value, move left forward
            else if (sum < target) left++;
            // Sum too large — need a smaller value, move right backward
            else right--;
        }

        // Guaranteed to find a solution, so this is unreachable
        return new int[]{-1, -1};
    }
}

## Key Takeaways

- **Sorted input** is the key signal that tells you to use two pointers
- The **1-indexed return** is just an adjustment: add 1 to each index
- The two-pointer approach is **O(n) time, O(1) space** — far better than the O(n²) brute force
- This pattern generalizes to Three Sum and other pair-finding problems
