# Flood Fill — Solution

## Problem Overview

Starting from a pixel at `(sr, sc)`, replace its color and all 4-directionally connected pixels of the same original color with the new `color`. This is a classic graph traversal problem on a grid.

## DFS (Recursive) Approach

**Idea:** Use depth-first search. If the current pixel matches the original color, recolor it and recursively visit all 4 neighbors.

**Algorithm:**
1. If `image[sr][sc] == color`, return early (no change needed).
2. Store the original color.
3. Define a recursive DFS function:
   - If out of bounds or not the original color, return.
   - Set the pixel to the new color.
   - Recursively call DFS on all 4 neighbors.
4. Start DFS from `(sr, sc)`.

**Complexity Analysis:**
- **Time:** O(m × n) — each pixel visited at most once
- **Space:** O(m × n) — worst-case recursion stack depth for a full grid

## BFS (Queue) Approach

**Idea:** Use a queue to process the grid level by level. Start from the seed pixel and expand outward.

**Algorithm:**
1. If `image[sr][sc] == color`, return early.
2. Store the original color.
3. Initialize a queue with `(sr, sc)`.
4. While queue is not empty:
   - Pop a pixel, recolor it.
   - Push all 4-directional neighbors that match the original color.

**Complexity Analysis:**
- **Time:** O(m × n)
- **Space:** O(m × n) — queue size in worst case

## Deep Dive: How Grid DFS Works

### Direction Array Pattern

```
directions = [(-1, 0), (1, 0), (0, -1), (0, 1)]
```

This compact array replaces four separate `if` statements for up/down/left/right.

### Visual Walkthrough

For `image = [[1,1,1],[1,1,0],[1,0,1]]`, `sr = 1`, `sc = 1`, `color = 2`:

```
Original:         Step 1 (center):   Step 2:           Step 3:           Final:
1 1 1             1 1 1              2 1 1             2 2 1             2 2 2
1 1 0      →      2 1 0       →      2 1 0      →      2 2 0      →      2 2 0
1 0 1             1 0 1              1 0 1             1 0 1             2 0 1
```

### Edge Cases to Consider

- **Same color**: If `color == originalColor`, no changes — return immediately.
- **Single pixel**: Just change it.
- **Full-grid fill**: The entire grid is one color, all pixels change.
- **Disconnected regions**: Only the connected component starting from `(sr, sc)` is affected.

class Solution {
    public int[][] floodFill(int[][] image, int sr, int sc, int color) {
        int original = image[sr][sc];
        if (original == color) return image;  // No change needed
        dfs(image, sr, sc, original, color);
        return image;
    }

    private void dfs(int[][] image, int r, int c, int original, int color) {
        // Check bounds and color match
        if (r < 0 || r >= image.length || c < 0 || c >= image[0].length || image[r][c] != original)
            return;
        image[r][c] = color;  // Recolor current pixel
        // Visit all 4 neighbors
        dfs(image, r - 1, c, original, color);
        dfs(image, r + 1, c, original, color);
        dfs(image, r, c - 1, original, color);
        dfs(image, r, c + 1, original, color);
    }
}

fn flood_fill(image: Vec<Vec<i32>>, sr: i32, sc: i32, color: i32) -> Vec<Vec<i32>> {
    let mut img = image;
    let original = img[sr as usize][sc as usize];
    if original == color { return img; }

    fn dfs(img: &mut Vec<Vec<i32>>, r: i32, c: i32, original: i32, color: i32) {
        let m = img.len() as i32;
        let n = img[0].len() as i32;
        if r < 0 || r >= m || c < 0 || c >= n || img[r as usize][c as usize] != original {
            return;
        }
        img[r as usize][c as usize] = color;
        dfs(img, r - 1, c, original, color);
        dfs(img, r + 1, c, original, color);
        dfs(img, r, c - 1, original, color);
        dfs(img, r, c + 1, original, color);
    }

    dfs(&mut img, sr, sc, original, color);
    img
}

## Key Takeaways

- **Grid DFS/BFS** is the standard pattern for flood fill and connected-component problems.
- **Direction arrays** make the code cleaner and more maintainable than 4 separate conditionals.
- **Early return** when `original == color` prevents infinite recursion.
- The same pattern applies to problems like Number of Islands, Max Area of Island, and more.
