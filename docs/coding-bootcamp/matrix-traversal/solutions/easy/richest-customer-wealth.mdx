# Richest Customer Wealth — Solution

## Problem Overview

Given an `m x n` integer grid `accounts` where `accounts[i][j]` is the amount of money the `i-th` customer has in the `j-th` bank, return the maximum total wealth (sum of all accounts) across all customers.

## Brute Force Approach

There is no simpler approach than iterating through every element — every customer's wealth must be computed, which inherently requires visiting all `m * n` elements.

**Idea:** Sum each row and track the maximum.

**Complexity Analysis:**
- **Time:** O(m × n) — we visit each cell exactly once
- **Space:** O(1) — only a few integer variables

## Optimal Approach: Row Sum with Max Tracking

**Idea:** Iterate through the matrix row by row. For each row, compute the sum and update the maximum if this row's sum is larger.

**Algorithm:**
1. Initialize `maxWealth = 0`
2. For each row in `accounts`:
   - Compute the sum of the row
   - Update `maxWealth = max(maxWealth, rowSum)`
3. Return `maxWealth`

**Why This Works:** The wealth of each customer is independent — each row represents one customer's accounts. The problem reduces to finding the row with the maximum sum.

**Complexity Analysis:**
- **Time:** O(m × n) — must visit every element once
- **Space:** O(1) — just two integer variables

## Deep Dive: 2D Array Traversal

### Visual Walkthrough

For the input `accounts = [[1, 5], [7, 3], [3, 5]]`:

```
Row 0: [1, 5] → sum = 6  → maxWealth = 6
Row 1: [7, 3] → sum = 10 → maxWealth = 10
Row 2: [3, 5] → sum = 8  → maxWealth = 10 (unchanged)

Result: 10
```

### Edge Cases to Consider

- **Single customer, single bank**: Just return the only value.
- **All zeros**: Wealth is 0.
- **Negative values**: The algorithm handles negative values correctly — `maxWealth` should be initialized to the smallest possible value (`Integer.MIN_VALUE`) or the first row's sum.
- **Large values**: The constraints are modest (max 50×50 with values up to 100), so 32-bit integers are sufficient.

class Solution {
    public int maximumWealth(int[][] accounts) {
        int maxWealth = 0;                         // Track the richest customer
        for (int[] customer : accounts) {          // Iterate through each customer row
            int sum = 0;
            for (int bank : customer) {            // Sum all accounts for this customer
                sum += bank;
            }
            maxWealth = Math.max(maxWealth, sum);  // Update richest if current is higher
        }
        return maxWealth;
    }
}

## Key Takeaways

- **Simple row-wise traversal** is the foundation of many matrix problems.
- The problem is a straightforward **2D array iteration** — the pattern is just nested loops.
- **O(m×n) time, O(1) space** is the optimal complexity for problems that must visit every cell.
