Anthropic's Model Context Protocol is rapidly becoming the standard way to connect AI assistants with enterprise data. Companies like Replit, Codeium, and Zed have adopted MCP to give their AI features access to files, databases, and APIs. The challenge: building a production MCP server that handles authentication, rate limiting, error recovery, and multiple tool definitions while remaining secure and observable.

An enterprise MCP server lets your internal AI tools read from your databases, query your APIs, and act on your systems — all through a single, standardized protocol.

## What You'll Build

You'll build **Enterprise MCP Server** with:

- **Multiple tool definitions** — database query tools, API proxy tools, file system tools, notification tools
- **Resource exposure** — expose company data as MCP resources with URI templates
- **Authentication** — API key validation with scoped permissions per tool
- **Rate limiting** — per-user, per-tool rate limiting to protect downstream services
- **Error handling** — structured error responses, retry logic, circuit breakers
- **Observability** — structured logging, request tracing, usage metrics
- **configuration** — YAML-based tool and resource definitions
- **Transport** — both stdio (for local Claude integration) and SSE (for remote clients)

## Technology Decisions

| Component | Choice | Why this, not X |
|---|---|---|
| MCP SDK | `mcp` (official Python SDK) | Standard protocol implementation. Building from scratch would need protocol compliance testing |
| Transport | stdio + SSE | stdio for local (zero config), SSE for remote clients. WebSocket adds complexity without benefit |
| Config | YAML | Human-readable, supports comments. JSON is machine-friendly but harder to hand-edit |
| Auth | Bearer tokens + scopes | Simple, stateless. OAuth2 integration added as a tool rather than built-in |

## Data Model

```python
from pydantic import BaseModel
from typing import Optional, Dict, Any
from datetime import datetime

class ToolDefinition(BaseModel):
    name: str
    description: str
    input_schema: Dict[str, Any]
    required_scopes: list[str]
    rate_limit: int  # requests per minute
    backend: str  # "postgres", "http", "file", "notify"

class ResourceDefinition(BaseModel):
    uri: str
    name: str
    mime_type: str = "application/json"
    description: Optional[str] = None

class AuthToken(BaseModel):
    token_hash: str
    user: str
    scopes: list[str]
    expires_at: Optional[datetime] = None

class AuditEntry(BaseModel):
    id: str
    user: str
    tool: str
    input: str
    success: bool
    duration_ms: int
    timestamp: datetime
```
