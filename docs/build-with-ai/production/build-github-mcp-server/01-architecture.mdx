GitHub hosts over 200 million repositories and processes millions of events per day. Developers using AI coding assistants frequently need to search code, review PRs, check CI status, and manage issues — all without leaving their editor. With the Model Context Protocol, you can expose the full GitHub API as tools and resources that any MCP-compatible client can use.

## What You'll Build

You'll build **GitHub MCP Server** with:

- **Repository tools** — search repos, get repo metadata, list files, read file contents
- **Issue tools** — create, list, search, and update issues and comments
- **PR tools** — create PRs, review changes, check status, merge
- **Code search** — search code across repositories using GitHub's code search API
- **Workflow tools** — list workflows, trigger runs, check status
- **User tools** — get user/org info, list repos for a user
- **Resource templates** — expose repos, issues, PRs as MCP resources

## Technology Decisions

| Component | Choice | Why this, not X |
|---|---|---|
| API Client | httpx | Async HTTP, supports both REST and GraphQL. requests is sync-only |
| Auth | Personal Access Token | Simple, scoped. GitHub App auth adds complexity for single-user use |
| Cache | SQLite + TTL | Zero config, local. Redis would be faster but adds operational overhead |
| Transport | stdio (default) | Designed for integration with Claude Code and Claude Desktop |

## API Surface

| Tool | Description | Key Params |
|---|---|---|
| `search_repositories` | Search GitHub repos | `query`, `limit` |
| `get_repository` | Get repo metadata | `owner`, `repo` |
| `list_files` | List files in a repo path | `owner`, `repo`, `path`, `ref` |
| `get_file_content` | Get file contents | `owner`, `repo`, `path`, `ref` |
| `search_code` | Search code across GitHub | `query`, `limit` |
| `list_issues` | List/search issues | `owner`, `repo`, `state`, `labels` |
| `create_issue` | Create a new issue | `owner`, `repo`, `title`, `body`, `labels` |
| `create_pull_request` | Create a PR | `owner`, `repo`, `title`, `head`, `base`, `body` |
| `get_pull_request` | Get PR details | `owner`, `repo`, `pull_number` |
| `list_workflow_runs` | List CI runs | `owner`, `repo`, `branch`, `status` |
