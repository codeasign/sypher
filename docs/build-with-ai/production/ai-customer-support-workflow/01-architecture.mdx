Support teams handle thousands of tickets daily. The challenge isn't just answering questions — it's knowing when to answer, when to ask for clarification, when to escalate, and when to follow up. Each of these decisions is an implicit branching point in a workflow. LangGraph makes these branches explicit, stateful, and testable.

## What You'll Build

You'll build **SupportGraph**, a LangGraph-based customer support workflow with:

- **Stateful conversation graph** — tracks intent, context, resolution status, and escalation path
- **Classification node** — determines ticket category, sentiment, and priority
- **Knowledge base search node** — retrieves relevant articles
- **Resolution generation node** — produces answers with cited sources
- **Human escalation node** — pauses for human input with full context
- **Follow-up scheduling node** — marks tickets for later follow-up
- **Cycle detection** — prevents infinite loops in the graph
- **Persistence** — saves graph state to SQLite for recovery

## Technology Decisions

| Component | Choice | Why this, not X |
|---|---|---|
| Framework | LangGraph | Stateful graphs with built-in cycle detection. Simple if/else chains don't scale to complex branching |
| LLM | Claude Haiku 3.5 | Cheap enough for per-node calls. Sonnet reserved for resolution node only |
| Persistence | SQLite (checkpoint) | Graph state serialized after each node. In-memory loses state on crash |
| KB Search | ChromaDB | Same as Helpdesk project. Reuse your existing KB index |

## Data Model

```python
from typing import TypedDict, List, Optional, Literal
from datetime import datetime

class SupportState(TypedDict):
    ticket_id: str
    subject: str
    description: str
    customer_email: str
    category: Optional[str]
    priority: Optional[Literal["low", "medium", "high", "critical"]]
    sentiment: Optional[str]
    assigned_team: Optional[str]
    confidence: float
    relevant_articles: List[dict]
    resolution: Optional[str]
    status: Literal["new", "classified", "resolved", "escalated", "follow_up"]
    messages: List[dict]
    error_count: int
```
