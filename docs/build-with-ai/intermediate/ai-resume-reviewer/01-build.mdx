You spent two hours tailoring your resume for a dream role. You hit submit, and then you wait. Did you emphasise the right skills? Is your experience gap going to be a dealbreaker? Did you miss a keyword the recruiter is screening for? The uncertainty is the worst part — because you have no way to see your resume the way a recruiter or an ATS (Applicant Tracking System) sees it.

Recruiters spend an average of seven seconds on an initial resume scan. They look for role-specific keywords, measurable impact, career progression signals, and red flags like employment gaps or unexplained job hops. An ATS scores resumes against a job description, and if yours doesn't match the keyword profile, it never reaches human eyes.

In this project you'll build a tool that gives you that second set of eyes: an AI-powered resume reviewer that analyses a resume against a job description, scores the match, highlights gaps, and suggests specific improvements. By the end, you'll have a tool that can review any resume against any job description in under 10 seconds.

## What You'll Build

A Python CLI tool that takes a resume and a job description and produces a detailed analysis. The system:

- Accepts resume text (pasted or loaded from a file)
- Accepts a job description (pasted or loaded from a file)
- Analyses the match across multiple dimensions: skills, experience, education, keywords
- Produces a match score (0–100%) with a detailed breakdown
- Highlights strengths, gaps, and specific improvements
- Suggests resume rewrites for weak sections
- Generates a tailored cover letter outline

## Concepts You Need

**Multi-dimensional scoring with structured output.** Instead of asking the LLM for a single score, you define separate evaluation dimensions (skills, experience, education, achievements, keyword density) and prompt the model to score each one independently. This produces a detailed breakdown that reveals exactly where the candidate is strong or weak relative to the role.

**Prompting for constructive feedback.** The difference between "your resume is weak" (useless) and "Your resume lists 3 of 5 required skills; add the missing Python and SQL sections with concrete examples" (actionable) is the prompt design. You structure the prompt to request specific, actionable feedback organised by priority.

**Text extraction and normalisation.** Resumes come in many formats — plain text, Markdown, PDF-extracted text. Job descriptions have their own structure (requirements, responsibilities, nice-to-haves). Normalising both inputs into a consistent format before analysis ensures the LLM evaluates the content, not the formatting.

## Build It

### Step 1: Set up the LLM client and input handling

Create `reviewer.py`:

```python
import os
import sys
import json
from pathlib import Path
from dotenv import load_dotenv

load_dotenv()

ANTHROPIC_API_KEY = os.environ.get("ANTHROPIC_API_KEY")
if not ANTHROPIC_API_KEY:
    print("Error: ANTHROPIC_API_KEY not set in .env")
    sys.exit(1)

import anthropic
client = anthropic.Anthropic(api_key=ANTHROPIC_API_KEY)

def load_text(source: str) -> str:
    """Load text from a file or return the input as-is."""
    path = Path(source)
    if path.exists():
        return path.read_text(encoding="utf-8")
    return source
```

**What happened:** The `load_text` function accepts either a file path or raw text. If the input matches an existing file path, it reads the file; otherwise it returns the text directly. This lets users paste text or reference a file interchangeably.

### Step 4: Add the resume analysis function

```python
def analyze_resume(resume_text: str, job_description: str) -> dict:
    """Analyse a resume against a job description. Returns structured analysis."""
    user_message = f"""RESUME:
{resume_text}

JOB DESCRIPTION:
{job_description}

Analyse this resume against the job description. Return the JSON analysis."""

    response = client.messages.create(
        model="claude-3-5-haiku-20241022",
        max_tokens=2000,
        system=SYSTEM_PROMPT,
        messages=[{"role": "user", "content": user_message}]
    )

    response_text = response.content[0].text
    input_tokens = response.usage.input_tokens
    output_tokens = response.usage.output_tokens

    print(f"  Input tokens: {input_tokens}, Output tokens: {output_tokens} "
          f"(~${input_tokens * 3e-6 + output_tokens * 15e-6:.4f})")

    # Parse JSON from response
    try:
        # Find JSON block in response
        if "```json" in response_text:
            json_str = response_text.split("```json")[1].split("```")[0].strip()
        elif "```" in response_text:
            json_str = response_text.split("```")[1].split("```")[0].strip()
        else:
            json_str = response_text.strip()
        return json.loads(json_str)
    except json.JSONDecodeError:
        print("Warning: Could not parse JSON, returning raw text")
        return {"raw_response": response_text}
```

### Step 5: Build the display formatter

Add a function that formats the analysis as a readable report:

```python
def format_report(analysis: dict) -> str:
    """Format the analysis as a readable Markdown report."""
    if "raw_response" in analysis:
        return analysis["raw_response"]

    lines = []
    lines.append("# Resume Analysis Report\n")
    lines.append(f"## Overall Score: {analysis.get('overall_score', 'N/A')}/100")
    lines.append(f"ATS Fit Score: {analysis.get('ats_fit_score', 'N/A')}/100\n")

    lines.append("## Dimension Scores\n")
    dims = analysis.get("dimensions", {})
    for dim_key, dim_data in dims.items():
        label = dim_key.replace("_", " ").title()
        score = dim_data.get("score", "N/A")
        bar = "█" * (score // 10) + "░" * (10 - score // 10) if isinstance(score, int) else ""
        lines.append(f"### {label}: {score}/100")
        if bar:
            lines.append(f"`{bar}`")
        lines.append(f"- **Evidence:** {dim_data.get('evidence', 'N/A')}")
        lines.append(f"- **Gaps:** {dim_data.get('gaps', 'N/A')}")
        lines.append(f"- **Suggestion:** {dim_data.get('suggestion', 'N/A')}\n")

    lines.append("## Strengths\n")
    for s in analysis.get("strengths", []):
        lines.append(f"- ✅ {s}")

    lines.append("\n## Critical Gaps\n")
    for g in analysis.get("critical_gaps", []):
        lines.append(f"- ❌ {g}")

    lines.append("\n## Quick Wins\n")
    for q in analysis.get("quick_wins", []):
        lines.append(f"- ⚡ {q}")

    lines.append("\n## Recommended Actions\n")
    for a in analysis.get("recommended_actions", []):
        lines.append(f"- {a}")

    if analysis.get("ats_notes"):
        lines.append(f"\n## ATS Notes\n{analysis['ats_notes']}")

    return "\n".join(lines)
```

**What happened:** The formatter converts the JSON analysis into a readable Markdown report with score bars, emoji indicators, and clearly labelled sections. This is the output the user sees.

### Step 6: Build the interactive CLI

```python
def main():
    print("=" * 50)
    print("  AI Resume Reviewer")
    print("=" * 50)

    # Get resume
    print("\n--- Resume ---")
    resume_source = input("Enter resume text or file path: ").strip()
    resume_text = load_text(resume_source)
    print(f"Resume loaded: {len(resume_text)} characters")

    # Get job description
    print("\n--- Job Description ---")
    jd_source = input("Enter job description text or file path: ").strip()
    jd_text = load_text(jd_source)
    print(f"Job description loaded: {len(jd_text)} characters")

    print("\n" + "=" * 50)
    print("Analysing...")
    print("=" * 50 + "\n")

    analysis = analyze_resume(resume_text, jd_text)

    report = format_report(analysis)
    print(report)

    # Save option
    save = input("\nSave report to file? (y/n): ").strip().lower()
    if save == "y":
        from datetime import datetime
        output_dir = Path("reports")
        output_dir.mkdir(exist_ok=True)
        timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
        output_path = output_dir / f"resume_review_{timestamp}.md"
        output_path.write_text(report, encoding="utf-8")
        print(f"Report saved to: {output_path}")

if __name__ == "__main__":
    main()
```

**What happened:** The CLI prompts for the resume and job description, runs the analysis, displays the formatted report, and offers to save it.

## Understand What Happened

**What each API call actually did.** The API call sent the resume text, job description, and the structured output system prompt to Claude Haiku. The model analysed the match across all five dimensions and returned a JSON object with scores, evidence, gaps, and suggestions. The structured output prompt ensured the response was parseable without complex string manipulation.

**Token usage and cost for a typical run.** A single analysis uses ~1,000–1,500 input tokens (resume + job description + prompt) and generates ~400–500 output tokens. At Claude Haiku pricing, this is ~$0.002–$0.003 per review. A batch of 50 resume reviews costs under $0.15.

**Performance characteristics.** The LLM API call takes 2–4 seconds. The JSON parsing and report formatting are instant. The system is entirely network-bound on the API call.

**3 common mistakes teams make with this pattern:**

- **Not separating evaluation dimensions.** A single "overall score" prompt produces vague feedback. Multi-dimensional prompting (skills, experience, keywords, etc.) produces specific, actionable insights that the candidate can actually use to improve.

- **Prompting for subjective praise.** A prompt that says "find the positive aspects" produces a fluff piece. A prompt that says "list specific gaps and how to fix them" produces real value. Always bias toward constructive, specific criticism.

- **Overlooking ATS keyword matching.** Human reviewers and ATS software evaluate resumes differently. An ATS scores keyword density and formatting, not narrative quality. Including an ATS-specific score and notes ensures the candidate addresses both audiences.
