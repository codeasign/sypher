## Deploy It

### Docker Compose

```yaml
version: "3.8"
services:
  observe:
    build: .
    ports:
      - "8004:8004"
    environment:
      - DB_PATH=/data/observe.db
      - ANTHROPIC_API_KEY=${ANTHROPIC_API_KEY}
      - ALERT_COST_THRESHOLD=5.0
      - ALERT_ERROR_THRESHOLD=10
      - ALERT_WEBHOOK_URL=${ALERT_WEBHOOK_URL:-}
    volumes:
      - ./data:/data
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8004/api/v1/metrics"]
      interval: 30s
      timeout: 10s
      retries: 3
```

## Monitoring and Observability (meta)

- **Disk usage**: SQLite grows with call volume. Set up log rotation or time-partitioned tables
- **Ingestion rate**: Single SQLite handles ~100 calls/second. Use batched inserts for higher volume
- **Cost tracking accuracy**: Token counting is approximate. Use provider billing APIs for exact numbers

### 5 Technical Decisions to Highlight

1. **Decorator-based instrumentation** — zero code changes for existing apps
2. **Statistical anomaly detection** — no ML training data needed
3. **Real-time cost tracking** — per-call cost calculation from rate cards
4. **PII detection via regex** — fast, no external API needed for common patterns
5. **Webhook alerts** — integrates with Slack, PagerDuty, or custom endpoints
