## Deploy It

### GitHub Actions Integration

```yaml
# .github/workflows/ai-tests.yml
name: AI Test Suite
on: [pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"
      - run: pip install -r requirements.txt
      - name: Run AI tests
        run: testai run tests/example_suite.yaml
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
      - name: Publish Test Results
        uses: dorny/test-reporter@v1
        if: always()
        with:
          name: AI Test Results
          path: results.xml
          reporter: java-junit
```

## Portfolio Presentation

### 3-Sentence Summary

"I built TestAI, an AI testing framework that validates LLM outputs using semantic assertions — similarity, safety, factual accuracy, and content checks — rather than brittle exact-match comparisons. It runs as YAML-defined test suites, outputs JUnit XML for CI integration, and costs under $0.10 per full suite run. The framework caught 3 regressions in my team's prompt deployments within the first week."

### 5 Technical Decisions to Highlight

1. **LLM-as-judge** — semantic assertions over exact matching
2. **YAML test format** — non-engineers can add test cases
3. **JUnit XML output** — compatible with every CI platform
4. **Configurable model per suite** — test with Haiku, validate with Sonnet
5. **Regression comparison** — catch prompt drift over time
