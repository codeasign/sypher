# Why Playwright — Build It

**Reading time:** 10 minutes · **Difficulty:** Beginner

## Step by Step

### 1. Install Node.js

Playwright requires Node.js 18 or later. If you do not have Node.js installed, download the LTS version from [nodejs.org](https://nodejs.org). Verify your installation by running:

```bash
node --version
npm --version
```

You should see version numbers for both commands.

### 2. Create a new project

Create a directory for your Playwright experiments and navigate into it:

```bash
mkdir my-first-playwright
cd my-first-playwright
```

### 3. Initialize Playwright

Run the official setup command:

```bash
npm init playwright@latest
```

The wizard presents several choices. Here is what you will see and how to answer:

```text
✔ Do you want to use TypeScript or JavaScript? · TypeScript
✔ Where to put your end-to-end tests? · tests
✔ Add a GitHub Actions workflow? · false
✔ Install Playwright browsers? · true
```

The wizard creates the following files and folders:

```
my-first-playwright/
├── tests/
│   └── example.spec.ts      # A sample test file
├── playwright.config.ts     # Configuration file
├── package.json
└── node_modules/
```

### 4. Install browsers

If you answered `true` to installing browsers, Playwright downloads Chromium, Firefox, and WebKit. This takes a minute on a typical connection.

If you skipped it, install browsers manually:

```bash
npx playwright install
```

### 5. Run the example tests

Playwright ships with three example tests. Run them to verify everything works:

```bash
npx playwright test
```

Expected output (yours will vary slightly):

```text
Running 3 tests using 3 workers
  3 passed (2.1s)
```

To see the tests run in a visible browser window, use the UI mode:

```bash
npx playwright test --ui
```

This opens the Playwright UI, a graphical interface where you can watch each test step, inspect the DOM, and debug failures interactively.

### 6. Open the HTML report

After running tests, Playwright generates an HTML report:

```bash
npx playwright show-report
```

This opens a detailed report showing which tests passed, failed, how long each took, and any screenshots or traces captured during failure.

## Build Something Real

Now write a test from scratch — one that navigates to a website and takes a screenshot. Create a new file `tests/first-test.spec.ts`:

```typescript
import { test } from '@playwright/test';

test('take a screenshot of the homepage', async ({ page }) => {
  await page.goto('https://example.com');
  await page.screenshot({ path: 'screenshots/homepage.png', fullPage: true });
});
```

Run it:

```bash
npx playwright test tests/first-test.spec.ts
```

Check the `screenshots/` directory for the saved image. You just automated a browser, navigated to a URL, and captured a full-page screenshot — in four lines of code.

### Make it more interesting

Let the test interact with the page. Replace the contents of `tests/first-test.spec.ts`:

```typescript
import { test, expect } from '@playwright/test';

test('interact with the page', async ({ page }) => {
  await page.goto('https://example.com');

  // Read the page heading
  const heading = page.locator('h1');
  await expect(heading).toHaveText('Example Domain');

  // Click the "More information" link
  await page.click('a');
  await expect(page).toHaveURL(/iana\.org/);

  // Take a screenshot after navigation
  await page.screenshot({ path: 'screenshots/after-click.png' });
});
```

Run it again:

```bash
npx playwright test tests/first-test.spec.ts
```

The test navigates to example.com, asserts the heading text, clicks a link, waits for the new page to load, and takes a screenshot. Notice there are no `sleep()` calls, no `waitForTimeout()` — Playwright's auto-waiting handles everything.
