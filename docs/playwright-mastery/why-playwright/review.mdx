# Why Playwright — Review

**Reading time:** 5 minutes · **Difficulty:** Beginner

## Key Takeaways

1. Playwright is a browser automation framework that lets you write end-to-end tests that run in Chromium, Firefox, and WebKit — all from a single API.
2. Playwright's auto-waiting mechanism eliminates the need for manual `sleep()` or `waitForTimeout()` calls by automatically retrying actions until elements are visible, enabled, and stable.
3. Browser contexts provide fully isolated sessions, enabling parallel test execution without shared state interference — a key architectural advantage over Selenium.
4. Playwright includes a built-in test runner, code generator, trace viewer, and UI mode, giving you a complete testing toolkit without third-party dependencies.
5. Choose Playwright for web browser automation, but use dedicated tools for API-only checks, native mobile testing, and performance benchmarking.

## Recap in 60 Seconds

1. Playwright is a modern, fast, and reliable browser automation framework from Microsoft.
2. It supports all three major browser engines (Chromium, Firefox, WebKit) with a single API.
3. Everything auto-waits, parallel execution is built-in, and debugging is comprehensive — these are the three features that make Playwright the industry standard.

### Scenario Questions

**Scenario 1:** Your team has a suite of 200 Playwright tests that currently take 45 minutes to run sequentially. How would you reduce the execution time to under 5 minutes?

**Scenario 2:** A test frequently fails because a loading spinner takes a variable amount of time to disappear. You notice a colleague added `await page.waitForTimeout(5000)` before every assertion. How would you fix this properly?

### Answers

**Multiple Choice:** 1-B, 2-B, 3-C, 4-B, 5-A

**Scenario 1:** Enable Playwright's built-in parallel execution. By default, Playwright runs tests in parallel using multiple worker processes. With 3-4 workers, a 45-minute suite runs in roughly 12-15 minutes. With more workers (depending on your CI machine's CPU cores), you can push it under 5 minutes.

**Scenario 2:** Remove the `waitForTimeout` entirely. Use Playwright's auto-waiting by waiting for the element you need after the spinner, not the spinner itself. For example, instead of waiting for the spinner to disappear, wait for the actual content element to be visible: `await expect(page.locator('.results-table')).toBeVisible()`. Playwright will automatically retry until the element appears.
