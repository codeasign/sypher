# Self-Healing Locators

Difficulty: Advanced

## What You Will Learn

- Understand what self-healing locators are and why they matter for test resilience
- Use Playwright's built-in smart selector engine ranking
- Implement fallback selectors with the `.or()` method
- Build and register custom selector engines for project-wide patterns

## Prerequisites

- Playwright Locators lesson completed
- TypeScript basics (for custom engine registration)
- Understanding of CSS selectors and accessibility roles

## Why This Matters

DOM changes are the #1 reason Playwright tests break in CI. A button that changes from `<button>Submit</button>` to `<button data-testid="order-now">Place Order</button>` kills a test that used `getByRole('button', { name: 'Submit' })`. Self-healing locators keep your tests running through minor UI updates by chaining fallback strategies — no manual test edits needed for every design tweak.

## Quick Reference

| Concept | API / Syntax | Description |
|---|---|---|
| Locator chaining | `locator.or(otherLocator)` | Returns first matching element from either locator |
| Selector combination | `'button >> text=Submit'` | Playwright descends the DOM with piped selectors |
| Selector fallback chain | `loc1.or(loc2).or(loc3)` | Tries loc1, then loc2, then loc3 |
| Custom engine | `selectors.register(name, engine)` | Register a selector engine with `query` and `queryAll` |
| Engine unregister | `selectors.setTestIdAttribute(attr)` | Override default `data-testid` attribute |
| Built-in ranking | role > testid > text > css > xpath | Playwright's internal preference order |

## The Big Picture

When a primary selector fails, a self-healing locator falls through a chain of alternatives. The first match is used and the test continues without interruption.

```text
<AsciiDiagram id="playwright-mastery/self-healing-locators/overview" alt="Self-healing locator fallback chain" caption="Self-healing locator flow with fallback selectors" content={`
  page.getByRole('button', { name: 'Submit' })
       │  (fails: element not found)
       ▼
  page.locator('[data-testid="submit-btn"]')
       │  (fails: data-testid changed)
       ▼
  page.getByText('Submit')
       │  (found!)
       ▼
  Test continues with matched element
`} />
```
