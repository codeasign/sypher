# Build It: Playwright CLI Agent

Follow these steps to generate Playwright tests using the CLI's codegen mode.

## Step 1: Run Codegen with a URL

Open a terminal and run:

```bash
npx playwright codegen example.com
```

This opens two windows side by side:
- A browser window navigated to `https://example.com`
- The Playwright Inspector panel

The Inspector shows a blank action list and a code pane that updates in real time.

## Step 2: Record Interactions

In the browser window, perform these actions:

1. Click the "More information" link
2. Press the browser back button
3. Right-click anywhere on the page and select "Inspect" (this is done in the browser, not in devtools)

Watch the Inspector panel record each action as Playwright code. You will see something like:

```javascript
await page.goto('https://example.com');
await page.getByRole('link', { name: 'More information' }).click();
```

The Inspector records the locator strategy it chose for each element.

## Step 3: Use Device Emulation

Run codegen with a specific device profile:

```bash
npx playwright codegen --device "iPhone 14" https://example.com
```

The browser opens in iPhone 14 viewport dimensions. Any test generated from this session includes the device emulation configuration automatically. The resulting code looks like:

```javascript
import { test, expect, devices } from '@playwright/test';

const iPhone14 = devices['iPhone 14'];

test.use({ ...iPhone14 });

test('example on iPhone 14', async ({ page }) => {
  await page.goto('https://example.com');
  // recorded actions here
});
```

## Step 4: Set a Custom Viewport

Run codegen with a custom viewport:

```bash
npx playwright codegen --viewport-size 1280,720 https://example.com
```

The browser window opens at 1280x720 pixels. The generated test includes the viewport setting:

```javascript
test.use({ viewport: { width: 1280, height: 720 } });
```

## Step 5: Specify Output Path

Run codegen and save directly to a file:

```bash
npx playwright codegen https://example.com -o tests/e2e/homepage.spec.ts
```

When you close the Inspector, the recorded test is saved to `tests/e2e/homepage.spec.ts`. If you omit the `-o` flag, you can copy the code from the Inspector manually.

## Step 6: Build Something Real

Now generate a complete login test using codegen.

First, create a simple login page to test against. Save this as `login.html`:

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Login</title>
  <style>
    body { font-family: system-ui; max-width: 400px; margin: 4rem auto; padding: 1rem; }
    label { display: block; margin-top: 1rem; }
    input { width: 100%; padding: 0.5rem; box-sizing: border-box; }
    button { margin-top: 1.5rem; padding: 0.5rem 2rem; }
    .error { color: red; margin-top: 0.5rem; }
    .hidden { display: none; }
  </style>
</head>
<body>
  <h1>Sign In</h1>
  <form id="login-form">
    <label for="email">Email</label>
    <input type="email" id="email" name="email" required />
    <label for="password">Password</label>
    <input type="password" id="password" name="password" required />
    <button type="submit" id="login-btn">Sign In</button>
    <div id="error-message" class="error hidden">Invalid credentials. Please try again.</div>
  </form>
  <div id="dashboard" class="hidden">
    <h2>Welcome back!</h2>
    <button id="logout-btn">Log Out</button>
  </div>
  <script>
    document.getElementById('login-form').addEventListener('submit', function (e) {
      e.preventDefault();
      const email = document.getElementById('email').value;
      const password = document.getElementById('password').value;
      if (email === 'user@example.com' && password === 'password123') {
        document.getElementById('login-form').classList.add('hidden');
        document.getElementById('dashboard').classList.remove('hidden');
      } else {
        document.getElementById('error-message').classList.remove('hidden');
      }
    });
    document.getElementById('logout-btn').addEventListener('click', function () {
      document.getElementById('login-form').classList.remove('hidden');
      document.getElementById('dashboard').classList.add('hidden');
      document.getElementById('email').value = '';
      document.getElementById('password').value = '';
      document.getElementById('error-message').classList.add('hidden');
    });
  </script>
</body>
</html>
```

Serve the login page:

```bash
npx serve .
```

Run codegen targeting the login page:

```bash
npx playwright codegen http://localhost:3000/login.html -o tests/e2e/generated-login.spec.ts
```

Perform the following flow in the browser:

1. Enter email `user@example.com` in the email field
2. Enter password `password123` in the password field
3. Click the "Sign In" button
4. Verify you see the dashboard heading
5. Click the "Log Out" button
6. Close the Inspector

Now open the generated file:

```bash
cat tests/e2e/generated-login.spec.ts
```

Examine the generated code. Pay attention to:

- How each locator was chosen (by role, by label, by CSS, by text)
- Whether assertions were added (codegen does not add assertions by default)
- Whether the test includes comments

Refactor the generated test. Add assertions, extract selectors, and give the test a descriptive name:

```typescript
import { test, expect } from '@playwright/test';

const CREDENTIALS = {
  email: 'user@example.com',
  password: 'password123',
};

test.describe('Login flow', () => {
  test('successful login and logout', async ({ page }) => {
    await page.goto('http://localhost:3000/login.html');

    await page.getByLabel('Email').fill(CREDENTIALS.email);
    await page.getByLabel('Password').fill(CREDENTIALS.password);
    await page.getByRole('button', { name: 'Sign In' }).click();

    await expect(page.getByRole('heading', { name: 'Welcome back!' })).toBeVisible();

    await page.getByRole('button', { name: 'Log Out' }).click();

    await expect(page.getByRole('heading', { name: 'Sign In' })).toBeVisible();
  });

  test('shows error for invalid credentials', async ({ page }) => {
    await page.goto('http://localhost:3000/login.html');

    await page.getByLabel('Email').fill('wrong@example.com');
    await page.getByLabel('Password').fill('wrongpass');
    await page.getByRole('button', { name: 'Sign In' }).click();

    await expect(page.getByText('Invalid credentials')).toBeVisible();
  });
});
```

Notice the changes you made: you added assertions, used `getByLabel` for inputs (more robust than CSS selectors), used `getByRole` for buttons (more semantic), added a descriptive test description, and covered an error state that codegen did not generate.
