# Playwright CLI Agent

Difficulty: Intermediate.

## What You Will Learn

- Use the Playwright CLI's `codegen` command to generate tests from real browser interactions
- Leverage the CLI agent mode for self-directed test writing
- Debug interactively with CLI tools like `show-report` and `show-trace`
- Integrate CLI commands into your daily development workflow

## Prerequisites

You need Playwright installed in your project. Basic terminal skills are required. You should already know how to write a simple Playwright test by hand.

## Why This Matters

The Playwright CLI is faster than writing locators by hand. Instead of inspecting the DOM, guessing selectors, and rerunning tests to see if they work, you interact with a real browser and the CLI generates the code for you. This lets you focus on test logic and edge cases instead of selector discovery.

## Quick Reference

| Command | Purpose |
|---|---|
| `npx playwright test` | Run your test suite |
| `npx playwright codegen` | Open browser and inspector for test generation |
| `npx playwright show-report` | Open the HTML test report |
| `npx playwright show-trace` | Open a trace viewer for a failed test |
| `npx playwright open` | Open a page in the Playwright browser inspector |

The CLI opens a browser window and an inspector panel side by side. Every click, type, and navigation you perform in the browser is recorded in the inspector as a Playwright action. When you are done, the inspector shows the complete generated test code ready to save.
