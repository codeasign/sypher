# Setup — Review

**Reading time:** 5 minutes · **Difficulty:** Beginner

## Key Takeaways

1. Always use `npm init playwright@latest` for first-time setup — it generates the correct `playwright.config.ts`, example tests, and installs browsers in one command.
2. The `playwright.config.ts` file controls everything: which browsers to use, how many parallel workers, retry logic, screenshot/trace/video capture, and the base URL for all tests.
3. A `baseURL` in the config lets you write relative paths (`await page.goto('/login')`) instead of absolute URLs, making tests portable across environments.
4. Configure `screenshot: 'only-on-failure'`, `trace: 'retain-on-failure'`, and `video: 'retain-on-failure'` in the `use` block — these artifacts are invaluable for debugging without cluttering every test run.
5. Set an explicit `workers` count in CI environments to match your runner's CPU, and always add `test-results/` and `playwright-report/` to `.gitignore`.

## Recap in 60 Seconds

1. Run `npm init playwright@latest` and say yes to browsers.
2. Configure `playwright.config.ts` with `baseURL`, screenshot/trace/video on failure, and all three browser projects.
3. Install the VS Code extension, add `.gitignore`, and verify with a test run.

## Quick Quiz

### Multiple Choice

**1. What command initializes a new Playwright project with the setup wizard?**

A. `npm install playwright`
B. `npm init playwright@latest`
C. `npx playwright start`
D. `playwright new`

**2. If `baseURL` is set to `https://example.com`, what URL does `await page.goto('/about')` navigate to?**

A. `https://example.com/about`
B. `https://example.com/about/`
C. `/about` relative to the current page
D. The test fails — you must use absolute URLs

**3. Which configuration option captures a trace only when a test fails on its first attempt and is being retried?**

A. `trace: 'retain-on-failure'`
B. `trace: 'on-first-retry'`
C. `trace: 'always'`
D. `trace: 'on-failure-retry'`

**4. What should you add to `.gitignore` to avoid committing test artifacts?**

A. `node_modules/`
B. `screenshots/`
C. `test-results/` and `playwright-report/`
D. `tests/`

**5. Which command runs tests against only the Firefox browser?**

A. `npx playwright test --browser=firefox`
B. `npx playwright test --project=firefox`
C. `npx playwright test --firefox`
D. `npx playwright test firefox`

### Scenario Questions

**Scenario 1:** You join a new team and run `npx playwright test` for the first time. The terminal outputs `browserType.launch: Executable doesn't exist at ...` for every browser. What went wrong and how do you fix it?

**Scenario 2:** Your colleague is frustrated that every test file starts with `await page.goto('http://localhost:3000/products')`. They have to change this URL in 20 test files when switching from local development to staging. What configuration change would solve this?

### Answers

**Multiple Choice:** 1-B, 2-A, 3-B, 4-C, 5-B

**Scenario 1:** The browsers were not installed during setup. The person running `npm init playwright@latest` likely answered "No" when asked about installing browsers. Fix it by running `npx playwright install`.

**Scenario 2:** Set `baseURL: 'http://localhost:3000'` in `playwright.config.ts` and change all tests to use relative paths: `await page.goto('/products')`. To switch environments, change only the `baseURL` value.
