# Setup

**Reading time:** 5 minutes · **Difficulty:** Beginner

## What You Will Learn

- How to install Playwright and its browsers using the official setup command
- How to configure Playwright for your project with `playwright.config.ts`
- How to install and use the VS Code Playwright extension
- How to understand every key option in the Playwright configuration file

## Prerequisites

- Node.js 18 or later installed on your machine
- A code editor (VS Code recommended for the extension)
- Basic familiarity with the terminal and `npm`

## Why This Matters

A correct setup prevents 90% of the early errors beginners encounter. Browser not found, tests timing out, wrong Node version, missing dependencies — each of these has a single root cause that proper setup avoids. Investing ten minutes in configuration saves hours of debugging later.

## Quick Reference

| Task | Command |
|---|---|
| Initialize Playwright | `npm init playwright@latest` |
| Install browsers | `npx playwright install` |
| Install with system deps | `npx playwright install --with-deps` |
| Install a single browser | `npx playwright install chromium` |
| Run all tests | `npx playwright test` |
| Run tests in headed mode | `npx playwright test --headed` |
| Open HTML report | `npx playwright show-report` |
| Open UI mode | `npx playwright test --ui` |
| Run code generator | `npx playwright codegen` |
| Install VS Code extension | Search "Playwright" in VS Code extensions panel |

### Key `playwright.config.ts` Options

| Option | Purpose | Default |
|---|---|---|
| `testDir` | Directory containing test files | `./tests` |
| `fullyParallel` | Run all tests in parallel | `true` |
| `retries` | Number of retries on failure | `0` (2 in CI) |
| `workers` | Number of parallel workers | `50%` of CPU cores |
| `reporter` | Test output format | `list` |
| `use.baseURL` | Base URL for all tests | `http://localhost:3000` |
| `use.screenshot` | Screenshot capture mode | `'off'` |
| `use.trace` | Trace capture mode | `'off'` |
| `use.video` | Video recording mode | `'off'` |
| `projects` | Browser-specific configurations | Chromium, Firefox, WebKit |
