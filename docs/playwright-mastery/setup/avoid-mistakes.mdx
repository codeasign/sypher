# Setup — Avoid Mistakes

**Reading time:** 6 minutes · **Difficulty:** Beginner

## Common Mistakes

### Mistake 1: Running an incorrect Node.js version

Playwright requires Node.js 18 or later. Running on Node 16 or earlier causes cryptic errors, especially around the native fetch API and ES module support.

```bash
# Check your Node version
node --version
# Must be v18.x, v20.x, v22.x, or later
```

If you have the wrong version, use [nvm-windows](https://github.com/coreybutler/nvm-windows) (Windows) or [nvm](https://github.com/nvm-sh/nvm) (macOS/Linux) to switch:

```bash
nvm install 20
nvm use 20
```

### Mistake 2: Forgetting to install browsers after setup

Running `npm init playwright@latest` does not install browsers unless you explicitly say yes to the prompt. If you skip it, every test fails with a message like:

```text
browserType.launch: Executable doesn't exist at ...
```

Fix it by installing browsers:

```bash
npx playwright install
```

### Mistake 3: Not setting a `baseURL` in the configuration

Without a `baseURL`, every test must use an absolute URL:

```typescript
// Without baseURL — you must type the full URL every time
await page.goto('https://myapp.com/login');
await page.goto('https://myapp.com/dashboard');
await page.goto('https://myapp.com/settings');
```

With `baseURL` set in the config, you write cleaner, more maintainable tests:

```typescript
// Inside playwright.config.ts
use: {
  baseURL: 'https://myapp.com',
}

// In your tests — relative paths resolve against baseURL
await page.goto('/login');
await page.goto('/dashboard');
await page.goto('/settings');
```

When you switch environments (local, staging, production), you change one line in the config instead of every test file.

### Mistake 4: Ignoring `.gitignore` for test artifacts

Test runs generate screenshots, videos, trace files, and HTML reports. If you do not add these directories to `.gitignore`, they bloat your repository and slow down git operations.

```bash
# Add to .gitignore
test-results/
playwright-report/
```

The `test-results/` directory contains artifacts from each test run. The `playwright-report/` directory contains the generated HTML report. Neither belongs in version control.

### Mistake 5: Not configuring the worker count for CI

The default worker setting (`50%` of CPU cores) works well on a developer machine with 8-16 cores. On a CI machine with 2 cores, 50% means 1 worker — tests run sequentially and take much longer than they should.

```typescript
// In your playwright.config.ts
workers: process.env.CI ? 4 : undefined,
```

Set a fixed worker count in CI (typically 2-4, matching your CI runner's CPU allocation). Test the worker count with a small number of tests and adjust based on execution time and flakiness.

## Troubleshooting

| Problem | Likely Cause | Solution |
|---|---|---|
| Browser download times out | Slow network or corporate proxy | Set `PLAYWRIGHT_DOWNLOAD_HOST` environment variable to a mirror. Or install browsers individually: `npx playwright install chromium` (smaller download) |
| Permissions error during browser install | Missing admin rights on Windows, or missing system deps on Linux | Windows: run terminal as Administrator. Linux: `npx playwright install --with-deps` |
| VS Code extension does not detect tests | Project opened in wrong folder, or no `playwright.config.ts` at root | Ensure VS Code workspace root contains `playwright.config.ts`. Reload the window (Ctrl+Shift+P > Developer: Reload Window) |
| TypeScript errors in test files | Missing `@playwright/test` types or incorrect tsconfig | Run `npm install -D @playwright/test`. Ensure `tsconfig.json` includes `"types": ["node"]` |
| `browserType.launch: Executable doesn't exist at ...` | Browsers not installed, or installed for the wrong Playwright version | Run `npx playwright install`. If the error persists, run `npx playwright install --force` to reinstall |

## Production Checklist

Before moving on, verify your setup against this checklist:

- [ ] Node.js version is 18 or later (`node --version`)
- [ ] Playwright browsers are installed (`npx playwright install`)
- [ ] `baseURL` is set in `playwright.config.ts`
- [ ] `.gitignore` includes `test-results/` and `playwright-report/`
- [ ] CI worker count is explicitly configured
- [ ] Reporters are configured (at least HTML for CI)
- [ ] `screenshot: 'only-on-failure'` is enabled
- [ ] `trace: 'retain-on-failure'` is enabled
