# Review: AI Test Generation

## Key Takeaways

1. AI can generate the majority of a test suite from natural language feature descriptions, but every generated test must be reviewed, run, and hardened before it is production-ready.
2. Prompt specificity directly determines output quality. Include locator strategy, test scenarios, edge cases, and environment details in every prompt.
3. AI-generated locators default to CSS selectors unless you explicitly instruct the AI to use `getByRole`, `getByLabel`, or `getByTestId`.
4. AI struggles with complex async state verification, security-critical flows, and domain-specific edge cases — these always require human review.
5. The ideal workflow is iterative: generate a test, review and run it, refine with follow-up prompts, and repeat until the suite meets your quality bar.

## AI Practice Prompt

Copy this prompt and use it with an AI coding tool:

```
I am building a Playwright test suite for a task management app with these features:
- Create a task (title, priority, assignee)
- Edit a task's priority
- Assign a task to a user
- Filter tasks by priority
- Delete a task

Generate the complete test file. For each feature include a happy path test and at least one error state or edge case test. Use getByRole and getByLabel locators. Use beforeEach to navigate to the app. The app runs at http://localhost:3000.
```

### Scenario Questions

**Scenario 1:** You prompt an AI: "Generate a Playwright test for the search feature." The AI returns a single test that navigates to the search page, types a query, and clicks the search button. It passes in CI. A week later, a bug is found where the search results do not update when the user clears the search field. The test did not catch it. What went wrong, and how would you fix it?

**Answer:** The prompt was too vague. It did not specify that the test should cover multiple scenarios: valid search, no results, clearing the search, empty query, search with special characters, and search result count verification. The generated test had no meaningful assertions — it only performed actions. To fix it, rewrite the prompt to include specific scenarios and assertions: "Generate tests for the search feature. Test 1: search for a valid term and verify results are displayed. Test 2: search for an empty string and verify no results. Test 3: type a query, clear the field, and verify the results reset. Test 4: search for a term with no matches and verify a 'no results' message. Use getByRole for all locators."

**Scenario 2:** Your team decides to use AI to generate the entire test suite for a new feature overnight. The next morning, you find 40 generated test files. When you run them, 25 pass, 10 have syntax errors, and 5 fail because they use Playwright APIs from two versions ago. What process should your team adopt for AI-generated tests?

**Answer:**
- Generate tests in smaller batches (5-10 at a time) so you can review each batch
- Run compiled tests immediately after generation, not the next day
- Include the Playwright version number in every prompt: "Use Playwright 1.50+ syntax"
- Add a manual review step to the workflow: run the test, read the assertions, check locator quality
- Do not generate all tests in one shot — iterate on one feature at a time
- Create a shared prompt template that includes locator strategy, version, environment, and required scenarios so all generated tests are consistent
