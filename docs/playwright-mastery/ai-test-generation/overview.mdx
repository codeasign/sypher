# AI Test Generation

Difficulty: Advanced.

## What You Will Learn

- Use AI to generate Playwright tests from natural language feature descriptions
- Prompt for specific test scenarios including happy paths, error states, and edge cases
- Combine AI-generated tests with manual review and refinement
- Use AI for test data generation and data-driven testing setup

## Prerequisites

You should have completed Playwright basics and be comfortable writing tests by hand. You need access to an AI tool such as Claude, ChatGPT, or GitHub Copilot. Familiarity with the Page Object Model pattern is helpful.

## Why This Matters

AI can generate 80% of a test suite from natural language descriptions. Instead of writing every test case manually, you describe the feature in plain English and the AI produces the Playwright code. This dramatically reduces the time from feature specification to tested code, letting you focus on reviewing and hardening the output instead of typing boilerplate.

## Quick Reference

### AI Prompt Patterns

| Pattern | Example |
|---|---|
| Feature description | "Generate a Playwright test for user login with valid and invalid credentials" |
| Edge case focus | "Add a test for expired session token that redirects to login" |
| Data generation | "Generate 5 test user objects with name, email, and role fields" |
| POM refactoring | "Refactor these tests to use the Page Object Model pattern" |

### Test Scenarios Coverage Matrix

| Scenario | Description |
|---|---|
| Happy path | The primary successful flow |
| Error states | Validation errors, API errors, 404 pages |
| Empty states | No data, empty lists, null values |
| Boundary conditions | Min/max values, pagination edges |
| Authentication | Logged in, logged out, expired session |

### Review Checklist for AI-Generated Tests

- Do the locators follow best practices (role, label, testid)?
- Are there meaningful assertions that validate outcomes?
- Are error states and edge cases covered?
- Does the test handle async loading and wait properly?
- Can the test run independently without relying on state from other tests?

The workflow starts with a natural language feature description. The AI generates tests covering multiple scenarios. You review the output, validate it by running the tests, and refine it through follow-up prompts. The result is a production-ready test suite built in a fraction of the time manual coding would require.
