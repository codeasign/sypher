# Avoid Common Mistakes: AI Test Generation

## When NOT to Use This

Do not trust AI-generated tests without running them. AI models can produce plausible-looking code that does not work, does not compile, or passes without actually testing the right behavior. Every AI-generated test must be executed and verified before it is committed.

Also avoid using AI for tests that require domain-specific security knowledge. An AI does not understand your application's unique authentication flows, authorization rules, or encryption requirements. Tests for security-critical paths should always be written and reviewed by a security-aware developer.

## The Pattern to Remember

AI generates the first draft. You review and harden the output. Prompt specificity determines output quality — the more detailed your feature description, the better the AI-generated tests will be.

## Common Mistakes

**1. Not specifying the locator strategy in prompts**

If you do not tell the AI which locator strategy to use, it defaults to CSS selectors. These are brittle and prone to break on markup changes. Always include instruction in your prompts like "Use getByRole and getByLabel locators, not CSS selectors."

**2. Accepting all AI output without running the tests**

AI can generate code with syntax errors, undefined variables, or API calls that do not exist. It may import modules you do not have installed. It may use deprecated Playwright methods. Always run the generated tests (or at least compile them) before reviewing the logic.

**3. Not describing error states and edge cases in prompts**

When you write a brief prompt like "Generate a test for login," the AI produces only the happy path. Error states, empty states, and edge cases are missing unless you explicitly ask for them. Include a full list of scenarios in your prompt: valid flow, invalid credentials, empty fields, network error, expired session, rate limiting.

**4. AI generates tests that pass but do not validate meaningful behavior**

A common trap is a test that navigates, clicks, and fills fields but only checks that the page did not throw an error. The test passes even though it does not actually verify the expected outcome. Review each AI-generated test to ensure the assertions check meaningful behavior, not just page load or element existence.

**5. Over-relying on AI for complex async state verification**

AI struggles with multi-step asynchronous flows where state changes over time, such as polling for a job status, waiting for a WebSocket message, or monitoring a progress bar. These scenarios require careful use of `waitForFunction`, `waitForResponse`, or custom polling logic that is difficult for AI to get right without seeing the actual application behavior.

## Troubleshooting

| Problem | Likely Cause | Solution |
|---|---|---|
| AI generates outdated Playwright API | The model's training data cutoff | Ask the AI to use the current Playwright version; check the official docs for API changes |
| Generated tests pass but do not test the right thing | Assertions are too weak or missing entirely | Review every `expect` call; are you validating the outcome of the action or just the action itself? |
| AI uses the wrong base URL or production URLs | You did not specify the test environment in the prompt | Include the environment and base URL in every prompt; use `test.use({ baseURL })` |
| Generated test data does not match the schema | The AI guessed the field names and types | Provide an example JSON object in the prompt that matches your schema exactly |
| AI imports packages that are not installed | The AI assumes a standard dependency set | Review imports in generated code; install any missing packages or remove unused imports |

## Production Checklist

- [ ] Every AI-generated test has been reviewed by a human
- [ ] Locators use best practices (getByRole, getByLabel, getByTestId)
- [ ] Assertions check meaningful behavior, not just element existence
- [ ] Error states, empty states, and edge cases are covered
- [ ] Tests run and pass in CI
- [ ] Edge cases that the AI missed have been manually added
- [ ] No AI-generated code uses deprecated Playwright APIs
