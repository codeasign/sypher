# Build It — Waiting in Practice

## Step 1: Auto-Wait Demonstration

Playwright automatically waits for an element to be actionable before performing any click, type, or other interaction. The following test navigates to a page where a button appears after a 2-second delay. Playwright handles this without any explicit wait call.

```typescript
import { test, expect } from '@playwright/test';

test('auto-wait handles delayed button', async ({ page }) => {
  // Navigate to a page that shows a button after 2 seconds
  await page.goto('https://practice.sypher.dev/delayed-element');

  // Playwright auto-waits for this button to become visible and stable
  // No waitForTimeout needed — Playwright polls until the button is actionable
  await page.getByRole('button', { name: 'Show Modal' }).click();

  // After clicking, the modal should be visible
  await expect(page.getByText('Welcome to the modal')).toBeVisible();
});
```

Notice there is no `page.waitForTimeout(2000)` — Playwright's auto-wait retries the click until the button passes all five actionability checks. On a fast machine the click may happen in 2.1 seconds; on a slow CI runner it may take 10 seconds. The test passes either way because Playwright adapts to the actual timing.

## Step 2: Wait for a Loading Spinner to Disappear

When a page shows a loading spinner while fetching data, you can wait for it to vanish before proceeding with assertions.

```typescript
import { test, expect } from '@playwright/test';

test('wait for loading spinner to disappear', async ({ page }) => {
  await page.goto('https://practice.sypher.dev/data-table');

  // Click a button that triggers a data reload with a loading spinner
  await page.getByRole('button', { name: 'Reload Data' }).click();

  // Wait for the loading spinner to disappear
  // Using { state: 'hidden' } waits until the element is either
  // not attached to DOM, or is attached but not visible
  await page.getByText('Loading...').waitFor({ state: 'hidden' });

  // Now assert the data table is populated
  const rows = page.locator('table tbody tr');
  await expect(rows).not.toHaveCount(0);
});
```

The `{ state: 'hidden' }` option is more robust than checking `toBeHidden()` because it can also handle the case where the element is completely removed from the DOM.

## Step 3: Wait for Navigation

After clicking a link or submitting a form that triggers a page navigation, use `waitForURL` to confirm the new page has loaded.

```typescript
import { test, expect } from '@playwright/test';

test('wait for navigation after form submit', async ({ page }) => {
  await page.goto('https://practice.sypher.dev/login');

  // Fill in credentials
  await page.getByLabel('Email').fill('user@example.com');
  await page.getByLabel('Password').fill('securePassword123');

  // Click the submit button — this triggers a navigation to /dashboard
  await page.getByRole('button', { name: 'Sign In' }).click();

  // Wait for the URL to match the expected pattern
  // The glob pattern ** matches any path segments
  await page.waitForURL('**/dashboard');

  // Assert the dashboard is loaded
  await expect(page.getByText('Welcome back')).toBeVisible();
});
```

The glob pattern `'**/dashboard'` matches any URL ending in `/dashboard`, regardless of query parameters or base URL. This is more flexible than an exact URL match.

## Step 4: Wait for an API Response

When you need to wait for a specific network request to complete before asserting, use `waitForResponse`. This is essential for API-driven UIs where the page renders before the data arrives.

```typescript
import { test, expect } from '@playwright/test';

test('wait for API response before asserting table data', async ({ page }) => {
  await page.goto('https://practice.sypher.dev/user-list');

  // Set up a wait for the API response BEFORE triggering the action
  // This prevents a race condition where the response arrives before we start listening
  const responsePromise = page.waitForResponse(
    resp => resp.url().includes('/api/users') && resp.status() === 200
  );

  // Click the sort button — this triggers an API call
  await page.getByRole('button', { name: 'Sort by Name' }).click();

  // Wait for the API response to arrive
  const response = await responsePromise;
  const users = await response.json();

  // Assert against the API response
  expect(users).toHaveLength(10);

  // Assert the UI updated correctly
  const firstUserName = page.locator('table tbody tr').first().locator('td').first();
  await expect(firstUserName).toHaveText(users[0].name);
});
```

The key pattern here is creating the `responsePromise` **before** triggering the action. If you trigger the action first and then call `waitForResponse`, the response may arrive before Playwright begins listening, causing the wait to time out.

## Step 5: Wait for a Custom JavaScript Condition

Use `waitForFunction` when you need to wait for a condition that cannot be expressed with Playwright's built-in locator or network waits.

```typescript
import { test, expect } from '@playwright/test';

test('wait for custom JS condition', async ({ page }) => {
  await page.goto('https://practice.sypher.dev/dashboard');

  // Wait until the page title is no longer the default "Loading..."
  await page.waitForFunction(
    () => document.title === 'Dashboard',
    null,
    { timeout: 15000 }
  );

  // The title has changed — the page is fully initialized
  await expect(page.getByText('Dashboard Metrics')).toBeVisible();
});
```

The `waitForFunction` method accepts a predicate that runs in the browser context. It polls the page every 100ms by default and resolves when the predicate returns a truthy value. The second argument (`null`) is for passing arguments to the predicate function; the third argument accepts options like `timeout` and `polling` interval.

## Step 6: Build Something Real

Combine everything you have learned into a single comprehensive test. This test interacts with a page that has lazy-loaded images, an API-driven table, a form with a multi-step redirect, and an animated modal — all without a single fixed-time sleep.

```typescript
import { test, expect } from '@playwright/test';

test('real-world page with zero fixed-time sleeps', async ({ page }) => {
  // Navigate to the application
  await page.goto('https://practice.sypher.dev/dashboard');

  // --- Lazy-loaded images ---
  // Wait for all images in the gallery to have a non-empty naturalWidth
  // This confirms the images have actually loaded, not just appeared in the DOM
  await page.waitForFunction(() => {
    const images = document.querySelectorAll('img[data-lazy]');
    return Array.from(images).every(
      (img: HTMLImageElement) => img.naturalWidth > 0
    );
  });

  const galleryImages = page.locator('img[data-lazy]');
  await expect(galleryImages).toHaveCount(8);

  // --- API-driven table ---
  // Set up response wait BEFORE triggering the sort
  const sortResponsePromise = page.waitForResponse(
    resp => resp.url().includes('/api/products') && resp.status() === 200
  );

  // Click the sort button
  await page.getByRole('button', { name: 'Sort by Price' }).click();

  // Wait for the response
  const sortResponse = await sortResponsePromise;
  const products = await sortResponse.json();
  expect(products.length).toBeGreaterThan(0);

  // Verify the UI updated to reflect the sorted data
  const productRows = page.locator('table#products tbody tr');
  await expect(productRows).toHaveCount(products.length);

  // --- Form submit with 2-step redirect ---
  // Click "Add Product" button which navigates to a form page
  await page.getByRole('button', { name: 'Add Product' }).click();
  await page.waitForURL('**/products/new');

  // Fill in the form
  await page.getByLabel('Product Name').fill('Test Widget');
  await page.getByLabel('Price').fill('19.99');
  await page.getByRole('button', { name: 'Save' }).click();

  // First redirect: /products/new → /products/confirm
  await page.waitForURL('**/products/confirm');
  await page.getByRole('button', { name: 'Confirm' }).click();

  // Second redirect: /products/confirm → /products
  await page.waitForURL('**/products');

  // Verify the new product appears in the table
  await expect(page.getByText('Test Widget')).toBeVisible();

  // --- Modal with animation ---
  // Click a row to open a details modal that slides in with CSS animation
  await page.getByText('Test Widget').click();

  // Playwright auto-waits for the modal to be stable (animation complete)
  // before clicking any element inside it
  const closeButton = page.getByRole('button', { name: 'Close' });
  await closeButton.click();

  // Wait for the modal to close (the animation plays in reverse)
  await closeButton.waitFor({ state: 'hidden' });

  console.log('Test passed — no fixed-time sleeps used anywhere!');
});
```

This test demonstrates the complete waiting toolkit:
- Auto-wait handles all user interactions (clicks, fills)
- `waitForFunction` checks lazy image loading
- `waitForResponse` coordinates with API calls
- `waitForURL` follows multi-step navigation
- `waitFor({ state: 'hidden' })` waits for animated elements to close

Every wait targets a real condition. There are zero `waitForTimeout` calls.
