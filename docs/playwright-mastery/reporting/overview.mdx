# Reporting

**Reading time:** 6 minutes · **Difficulty:** Intermediate

## What You Will Learn

- Configure Playwright's built-in reporters for different audiences — humans, CI systems, and dashboards
- Create custom reporters that generate team-specific output or send notifications
- Integrate with external reporting services by extending Playwright's reporter interface
- Combine multiple reporters in a single test run so you get HTML reports locally, JUnit XML in CI, and blob reports for shard merging

## Prerequisites

This lesson assumes you have a Playwright project with a working `playwright.config.ts` and some tests. You should be comfortable with TypeScript/JavaScript classes and configuration objects.

## Why This Matters

A test run that produces nothing but a terminal exit code is nearly useless. When a test fails, you need to know:

- Which test failed and in which browser?
- What was on the screen at the moment of failure?
- What network requests happened?
- Was the failure a flake or a real regression?

A well-configured reporting pipeline answers all these questions automatically. It generates an HTML report with traces and screenshots for local debugging, a JUnit XML file for CI annotations, and a blob report for shard merging — all from a single test run.

Without good reporting, your team spends hours reproducing failures manually, scrolling through raw logs, and asking "did this test fail in CI too or just on my machine?"

## Quick Reference

| Reporter | Output | Best for |
|---|---|---|
| `html` | `playwright-report/index.html` | Human-readable local debugging |
| `list` | Terminal output | Watching test progress in real time |
| `line` | Single line per test | Quick terminal feedback |
| `dot` | Compact dot matrix | Quick pass/fail overview |
| `json` | `test-results.json` | Custom dashboards and tooling |
| `junit` | `junit.xml` | CI integration (GitHub Actions, Jenkins) |
| `blob` | `blob-report/` directory | Shard merging across CI runners |

Config format:

```typescript
// Single reporter
reporter: 'html'

// Multiple reporters
reporter: [
  ['html', { open: 'never' }],
  ['junit', { outputFile: 'results.xml' }],
]

// Environment variable override
// PLAYWRIGHT_REPORTER=dot npx playwright test
```
