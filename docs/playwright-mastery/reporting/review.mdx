# Reporting — Review

**Reading time:** 5 minutes · **Difficulty:** Intermediate

## Key Takeaways

1. Playwright supports seven built-in reporters: `html`, `list`, `line`, `dot`, `json`, `junit`, and `blob` — each designed for a specific use case.
2. Combine multiple reporters in a single config so you get HTML for local debugging, JUnit for CI annotations, JSON for dashboards, and blob for shard merging — all from one test run.
3. The blob reporter is essential for sharded CI pipelines because it preserves trace and screenshot data from every shard and enables merging with `npx playwright merge-reports`.
4. Custom reporters implement the `Reporter` interface with hooks like `onBegin`, `onTestEnd`, and `onEnd`, giving you full control over output format and destinations.
5. Always set `open: 'never'` or conditionally based on `process.env.CI` for the HTML reporter, and always upload report artifacts in CI with `if: always()`.

## Recap in 60 Seconds

1. Configure `['html', { open: 'never' }]` for CI, `['junit', { outputFile: 'test-results/junit.xml' }]` for CI annotations, `['blob']` for sharding.
2. Upload `playwright-report/` as a CI artifact with `if: always()`.
3. For custom output, implement the `Reporter` interface.
4. Merge blob reports with `npx playwright merge-reports --reporter html ./all-blob-reports`.
5. Override reporters at runtime with `PLAYWRIGHT_REPORTER` env var.

### Scenario Questions

**Scenario 1:** Your CI pipeline runs Playwright tests across 8 shards. After each run, the team can only see the results from one shard's test run. The traces and screenshots from the other 7 shards are missing. Review the reporters and suggest a fix.

**Scenario 2:** A team member configured the JUnit reporter without an `outputFile` option. When they check the GitHub Actions annotations, no test results appear. What is the likely cause and how do you fix it?

### Answers

**Multiple Choice:** 1-C, 2-B, 3-C, 4-A, 5-B

**Scenario 1:** The pipeline is using only the HTML reporter without the blob reporter. Each shard generates its own `playwright-report/` directory, and when the artifact upload step runs, only the last shard's report survives. The fix is to add `['blob']` to the reporter list, upload each shard's blob report as a separate artifact, and add a merge job that runs `npx playwright merge-reports --reporter html ./all-blob-reports` to produce a single comprehensive report.

**Scenario 2:** Without `outputFile`, the JUnit reporter writes the XML to stdout instead of a file. GitHub Actions expects a file at the path specified in the `dorny/test-reporter` action configuration. The fix is to add `outputFile: 'test-results/junit.xml'` to the JUnit reporter configuration and update the GitHub Actions step to reference that path.
