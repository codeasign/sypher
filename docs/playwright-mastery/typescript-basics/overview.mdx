# TypeScript Basics for Playwright

Difficulty: Beginner

Reading time: 8 minutes

## What You Will Learn

- Write Playwright tests in TypeScript with type annotations
- Use async/await patterns correctly in test code
- Configure `tsconfig.json` for Playwright projects
- Catch locator typos and missing awaits at compile time

## Prerequisites

- Basic JavaScript knowledge (variables, functions, objects)
- Playwright installed in your project (`npm init playwright@latest`)
- Node.js 18 or later

## Why This Matters

TypeScript catches locator typos, wrong return types, and missing `await` at compile time instead of at 2am in CI. When you write `page.locator('button').click()` without `await`, TypeScript warns you that the return type is `Promise<void>`, not `void`. When you pass a string where a `Locator` is expected, the compiler flags it immediately. For a tool like Playwright that orchestrates dozens of async browser operations, this safety net is invaluable.

## Quick Reference

| Feature | TypeScript Syntax | Playwright Use Case |
|---|---|---|
| Type annotations | `let page: Page` | Declare Playwright types |
| async/await | `await page.goto(url)` | All Playwright actions |
| Interfaces | `interface User { name: string }` | Test data shapes |
| Generics | `locator<HTMLElement>` | Type-safe element selection |
| Union types | `type Status = 'pass' | 'fail'` | Test outcomes |
| Optional chaining | `user?.name` | Safer property access on nullable values |

## The Big Picture

TypeScript sits between your test code and the Playwright runtime. It analyzes your types before the code ever reaches a browser, catching whole categories of mistakes in milliseconds.
