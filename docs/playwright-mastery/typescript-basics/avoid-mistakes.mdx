# TypeScript Basics — Avoid Mistakes

TypeScript makes Playwright tests safer, but only if you use it correctly. This lesson covers the most common TypeScript mistakes in Playwright projects and how to fix them.

## The Pattern to Remember

Always annotate `Page` and `BrowserContext` parameters when you write helper functions outside the test callback. Use `const` for locators so you never accidentally reassign them.

```typescript
// Correct pattern — types explicit, locators are const
async function getHeaderText(page: import('@playwright/test').Page): Promise<string> {
  const heading = page.locator('h1');
  return heading.textContent() ?? '';
}
```

## Common Mistakes

### 1. Forgetting `await` Before Playwright Actions

Every Playwright action returns a Promise. Omitting `await` means the action starts but never completes before the next line runs.

```typescript
// Wrong — missing await
test('broken', async ({ page }) => {
  page.goto('/');          // Navigation starts but doesn't complete
  page.click('button');    // Runs before page loads
  const text = page.textContent('.result'); // Returns Promise, not string
});

// Correct — every async call is awaited
test('working', async ({ page }) => {
  await page.goto('/');
  await page.click('button');
  const text = await page.textContent('.result');
});
```

TypeScript warns you about some missing awaits, but not all. Use the `@typescript-eslint/no-floating-promises` ESLint rule to catch every case.

### 2. Using `any` Instead of Proper Types

When you use `any`, you opt out of TypeScript's safety. A locator typo or wrong return type goes undetected until runtime.

```typescript
// Wrong — any hides mistakes
async function getData(page: any): Promise<any> {
  return page.locater('.data').innerText(); // Typo: locater vs locator
}

// Correct — explicit types surface errors
async function getData(page: Page): Promise<string> {
  return page.locator('.data').innerText(); // TypeScript validates locator exists on Page
}
```

### 3. Not Configuring `tsconfig.json` Paths for Playwright

Playwright test files need their own TypeScript configuration. If you use the default `tsconfig.json` without extending Playwright's config, type resolution fails.

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "types": ["node", "@playwright/test"]
  },
  "include": ["tests/**/*.ts"]
}
```

Without `"moduleResolution": "bundler"`, TypeScript cannot resolve Playwright's export maps in newer versions.

### 4. Mixing Sync and Async Patterns

Calling an async function without `await` inside a synchronous context creates a floating promise that never resolves properly.

```typescript
// Wrong — calling async function from non-async context
test.describe('suite', () => {
  test.beforeEach(() => {
    // This won't work — beforeEach expects async callback
    loginAsAdmin(page);
  });
});

// Correct — use async callback
test.describe('suite', () => {
  test.beforeEach(async ({ page }) => {
    await loginAsAdmin(page);
  });
});
```

### 5. Ignoring TypeScript Strict Mode Errors

It is tempting to set `"strict": false` or add `// @ts-ignore` when you see red squiggly lines. This defeats the purpose of using TypeScript.

```typescript
// Wrong — silencing the compiler
// @ts-ignore
const element = page.locator('.maybe-exists');

// Correct — handle the nullable type properly
const element = page.locator('.maybe-exists');
if (await element.isVisible()) {
  await element.click();
}
```

## Troubleshooting

| Error | Cause | Fix |
|---|---|---|
| `Property 'click' does not exist on type 'Locator'` | You imported `Locator` from the wrong module or your Playwright version is outdated | Run `npm install @playwright/test@latest` and import from `@playwright/test` |
| `Cannot find module '@playwright/test'` | Playwright is not installed or `tsconfig.json` paths are misconfigured | Run `npm install -D @playwright/test`; ensure `"types"` includes `"@playwright/test"` in `tsconfig.json` |
| `Type 'string \| null' is not assignable to type 'string'` | A Playwright method returns a nullable type (e.g., `textContent()`) | Use the nullish coalescing operator: `textContent() ?? ''` |
| `'await' has no effect on this kind of expression` | You are awaiting a value that is not a Promise, often from a synchronous Playwright method like `isVisible()` | Remove `await` or check the Playwright API docs for the correct return type |

## Production Checklist

- [ ] `tsconfig.json` has `"strict": true` enabled
- [ ] `"moduleResolution": "bundler"` is set in `compilerOptions`
- [ ] Playwright types resolve correctly — run `npx tsc --noEmit` without errors
- [ ] ESLint `@typescript-eslint/no-floating-promises` rule is active
- [ ] All helper functions outside the test callback have explicit return type annotations
- [ ] Locators are assigned to `const` variables, never `let`
- [ ] Every Playwright action is prefixed with `await`
