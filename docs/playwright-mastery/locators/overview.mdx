# Locators Overview

Difficulty: Beginner

## What You Will Learn

- How to use CSS, XPath, and text-based locators to find elements on a page
- How to chain locators for more precise targeting
- How to filter locators by text and visibility
- How to use role locators based on ARIA semantics

## Prerequisites

Before starting this lesson, you should have a basic understanding of HTML and the DOM. You should also have Playwright installed and set up in your project. If you need help with setup, see the Playwright setup lesson.

## Why This Matters

Bad locators are the number one cause of flaky tests. When your tests fail intermittently because a CSS class changed or a button text was updated, you waste time debugging false positives. Playwright's locator API is designed from the ground up to be resilient, user-facing, and accessible. By learning the right locator strategy, you make your tests robust against UI changes and ensure they interact with elements the same way a real user would.

## Quick Reference

| Locator Method | Description | Best Use |
|---|---|---|
| `page.locator()` | Accepts any CSS or XPath selector | When you need an exact CSS path or XPath |
| `page.getByRole()` | Finds by ARIA role and accessible name | Preferred for buttons, links, headings, form controls |
| `page.getByText()` | Finds by visible text content | Menus, paragraphs, dynamic labels |
| `page.getByTestId()` | Finds by `data-testid` attribute | When no semantic role fits (e.g., custom components) |
| `page.getByPlaceholder()` | Finds input by placeholder text | Form inputs with placeholders |
| `page.getByLabel()` | Finds form control by associated label | Inputs, selects, textareas in forms |
| `page.getByTitle()` | Finds by `title` attribute | Tooltips, iframes, elements with title |

The hierarchy above is the Playwright locator strategy priority, from most preferred at the top to least preferred at the bottom. You should always start with `getByRole()` because it mimics how assistive technology sees the page and is highly resilient to DOM changes. Only fall back to CSS or XPath when no other strategy works.
