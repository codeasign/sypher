# Locators — Build It

In this lesson, you will write real Playwright locators against a live page. You will practice every locator strategy from the overview and chain them together for precise targeting.

## Setup

Create a test file called `locators.spec.ts` and a helper HTML file to test against. You can also test against a public site like `https://the-internet.herokuapp.com/login`.

```typescript
// locators.spec.ts
import { test, expect } from '@playwright/test';

test.describe('Locator strategies', () => {
  test.beforeEach(async ({ page }) => {
    await page.goto('https://the-internet.herokuapp.com/login');
  });
});
```

## Step 1: getByRole

The `getByRole` method is your most preferred locator. It finds elements by their ARIA role and accessible name.

```typescript
test('locate button by role', async ({ page }) => {
  // Finds the login button by its role and name
  const loginButton = page.getByRole('button', { name: /login/i });
  await expect(loginButton).toBeVisible();
});
```

You can use a string or a regular expression for the name. Regular expressions are useful when text might have surrounding whitespace or case differences.

```typescript
// Exact name match
page.getByRole('button', { name: 'Login' });

// Case-insensitive partial match
page.getByRole('button', { name: /login/i });

// Role and exact: true for precise matching
page.getByRole('button', { name: 'Login', exact: true });
```

## Step 2: getByTestId

When a component has no semantic role, add a `data-testid` attribute and use `getByTestId`.

Add this attribute to your HTML:

```html
<input type="text" name="username" id="username" data-testid="username-input" />
```

Then locate it in your test:

```typescript
test('locate input by test id', async ({ page }) => {
  await page.getByTestId('username-input').fill('tomsmith');
  await expect(page.getByTestId('username-input')).toHaveValue('tomsmith');
});
```

### Configuring the test ID attribute

If your project uses a different attribute name (like `data-test` instead of `data-testid`), configure it in `playwright.config.ts`:

```typescript
// playwright.config.ts
import { defineConfig } from '@playwright/test';

export default defineConfig({
  use: {
    testIdAttribute: 'data-test',
  },
});
```

## Step 3: getByText

Use `getByText` when you need to locate an element by its visible text content.

```typescript
test('locate element by text', async ({ page }) => {
  // Find a heading or paragraph by its text
  const heading = page.getByText('Login Page');
  await expect(heading).toBeVisible();

  // With exact match option
  const exactHeading = page.getByText('Login Page', { exact: true });
  await expect(exactHeading).toBeVisible();
});
```

## Step 4: getByLabel and getByPlaceholder

Form controls are best located using their associated `<label>` element or placeholder text.

```typescript
test('locate form fields by label and placeholder', async ({ page }) => {
  // By associated label
  await page.getByLabel('Username').fill('tomsmith');
  await page.getByLabel('Password').fill('SuperSecretPassword!');

  // By placeholder (if no label exists)
  // Assumes an input with placeholder="Enter your email"
  // await page.getByPlaceholder('Enter your email').fill('user@example.com');
});
```

## Step 5: Chain Locators

Chaining combines a parent locator with a more specific child locator. This is useful when the same role appears in multiple sections of the page.

```typescript
test('chain locators for precision', async ({ page }) => {
  // Find the form first, then find the button inside it
  const form = page.locator('#login');
  const submitButton = form.getByRole('button', { name: /login/i });
  await expect(submitButton).toBeVisible();

  // Alternative: chain in one line
  await expect(page.locator('#login').getByRole('button')).toBeVisible();
});
```

## Step 6: Filter

When multiple elements match a locator, use `.filter()` to narrow down by text or visibility.

```typescript
test('filter matching elements', async ({ page }) => {
  // Assume a list with items: "Active Item", "Inactive Item", "Active Task"
  // Find list items and filter by text
  const activeItem = page.getByRole('listitem').filter({ hasText: 'Active' });
  await expect(activeItem).toHaveCount(2);

  // Filter by child element
  const itemWithButton = page.getByRole('listitem').filter({
    has: page.getByRole('button'),
  });

  // Filter by visibility
  const visibleItems = page.getByRole('listitem').filter({ hasNotText: 'Hidden' });
});
```

## Step 7: Build Something Real

Now combine everything into a complete test spec that tests a form with five fields, fills each one, and submits. This test works against the-internet login page, but the pattern applies to any form.

```typescript
import { test, expect } from '@playwright/test';

test.describe('Complete form interaction with locators', () => {
  test.beforeEach(async ({ page }) => {
    await page.goto('https://the-internet.herokuapp.com/login');
  });

  test('fill and submit login form using multiple locator strategies', async ({ page }) => {
    // 1. Use getByLabel to find username and password fields
    // The-internet login page labels are "Username" and "Password"
    const usernameInput = page.getByLabel('Username');
    const passwordInput = page.getByLabel('Password');

    await expect(usernameInput).toBeVisible();
    await expect(passwordInput).toBeVisible();

    // 2. Fill both fields
    await usernameInput.fill('tomsmith');
    await passwordInput.fill('SuperSecretPassword!');

    // 3. Use getByRole to find the login button
    const loginButton = page.getByRole('button', { name: /login/i });
    await expect(loginButton).toBeVisible();
    await loginButton.click();

    // 4. Assert successful login using URL and flash message
    await expect(page).toHaveURL(/secure/);
    const successMessage = page.getByText('You logged into a secure area');
    await expect(successMessage).toBeVisible();
  });

  test('show error with invalid credentials', async ({ page }) => {
    // Fill with bad credentials
    await page.getByLabel('Username').fill('wronguser');
    await page.getByLabel('Password').fill('wrongpassword');
    await page.getByRole('button', { name: /login/i }).click();

    // Assert error message
    const errorMessage = page.getByText('Your username is invalid');
    await expect(errorMessage).toBeVisible();
  });

  test('chained locator targeting form elements', async ({ page }) => {
    // Chain: locate the form, then find children inside it
    const form = page.locator('#login');
    const button = form.getByRole('button');
    await expect(button).toBeVisible();
    await expect(button).toHaveText(/login/i);
  });

  test('locator filtering for lists', async ({ page }) => {
    // Navigate to a page with a list
    await page.goto('https://the-internet.herokuapp.com/checkboxes');

    // Find all checkbox inputs
    const checkboxes = page.getByRole('checkbox');
    await expect(checkboxes).toHaveCount(2);

    // Filter checked checkboxes
    const checkedBox = checkboxes.filter({ has: page.locator(':checked') });
    const uncheckedBox = checkboxes.filter({ hasNot: page.locator(':checked') });
  });

  test('negative test for missing element', async ({ page }) => {
    // Assert that a button that doesn't exist is not found
    const logoutButton = page.getByRole('button', { name: /logout/i });
    await expect(logoutButton).not.toBeVisible();
  });
});
```
