# Authentication — Overview

Difficulty: Advanced

## What You Will Learn

- Implement login flows using `storageState` to persist authentication
- Reuse authentication across tests without repeating login steps
- Handle multi-role testing with separate storage states for admin, user, and guest
- Manage session cookies, tokens, and expiry in long-running test suites

## Prerequisites

- Playwright API testing fundamentals (`page.request.post`, `browserContext.addCookies`)
- Playwright fixtures (`test.extend`, `test.use`)
- Project configuration in `playwright.config.ts`

## Why This Matters

Running login before every test adds 10-30 seconds per test. In a suite with 200 tests, that is 30-100 minutes of wasted time. Playwright's `storageState` lets you log in once, save the browser state (cookies, localStorage, session tokens), and reuse it across all tests. Shared auth state makes your suites fast and realistic — real users stay logged in across pages, and your tests should too.

## Quick Reference

| API / Config | Purpose | Example |
|---|---|---|
| `storageState` | Save and load browser state (cookies, localStorage) | `storageState: '.auth/user.json'` |
| `browserContext.addCookies()` | Inject cookies into a context at creation | `addCookies([{ name, value, url }])` |
| `browserContext.addInitScript()` | Run script before each page load (e.g., set auth tokens) | `addInitScript(() => { window.token = '...'; })` |
| `globalSetup` | Run once before all tests (e.g., login once) | `globalSetup: './global-setup'` |
| `project.dependencies` | Ensure a setup project runs before test projects | `dependencies: ['setup']` |
| `test.use({ storageState })` | Apply storageState to a test or describe block | `test.use({ storageState: '.auth/admin.json' })` |
| `page.request.post()` | Make API login request without rendering a page | `page.request.post('/login', { data })` |

The flow is: a setup project logs in once via API, saves the browser state to a `.auth/user.json` file, and then every test project declares a dependency on the setup project. Tests load the saved state and are immediately authenticated — no login step in any test.
