# Docker

**Reading time:** 6 minutes · **Difficulty:** Intermediate

## What You Will Learn

- Run Playwright tests inside a Docker container using the official Playwright image
- Build custom Docker images that bundle your test code, dependencies, and browsers
- Use Docker Compose to spin up your application and test service together for integrated testing
- Configure VS Code Dev Containers for a consistent local development environment

## Prerequisites

This lesson assumes you have a Playwright project with tests ready to run, Docker Desktop installed on your development machine, and a basic understanding of Docker concepts (images, containers, Dockerfiles). No prior Docker Compose experience is needed.

## Why This Matters

The most frustrating bug report in software engineering is "it works on my machine." Different operating systems, different versions of system libraries, different browser builds — every environment has subtle differences that can cause tests to pass locally and fail in CI, or vice versa.

Docker solves this by packaging your entire test environment into a container image. The image includes Node.js, Playwright, all three browsers, the exact system libraries each browser needs, and your test code. This image runs identically on your laptop, your CI runner, and your colleague's machine.

For Playwright specifically, Docker is especially valuable because browsers depend on dozens of system libraries (libnss3, libnspr4, libatk-bridge, libcups, and many more). Getting these right on a fresh CI runner is error-prone. The official Playwright Docker image comes with everything pre-installed.

## Quick Reference

| Concept | Details |
|---|---|
| Official image | `mcr.microsoft.com/playwright:v1.52.0-jammy` |
| Image tags | `v{version}-{distro}` e.g., `v1.52.0-jammy`, `v1.52.0-focal` |
| Supported distros | `jammy` (Ubuntu 22.04, recommended), `focal` (Ubuntu 20.04) |
| User in image | `pwuser` (non-root) — switch to root with `--user root` or `USER root` in Dockerfile |
| Default browser location | `/ms-playwright/` inside container |
| DevContainer spec | `.devcontainer/devcontainer.json` references the same image |
