# Build Trace Viewer Skills

## Step 1: Configure Tracing in playwright.config.ts

Set the trace mode globally or per-project in your configuration.

```typescript
import { defineConfig } from '@playwright/test';

export default defineConfig({
  use: {
    // Retain traces only for failed tests — best for CI
    trace: 'retain-on-failure',
  },
  projects: [
    {
      name: 'chromium',
      use: { trace: 'retain-on-failure' },
    },
    {
      name: 'debug',
      use: {
        // Full tracing with snapshots for deep debugging
        trace: {
          mode: 'on',
          snapshots: true,
          screenshots: true,
        },
      },
    },
  ],
});
```

## Step 2: Run a Test That Generates a Trace

Write a test that fails intentionally so you can inspect the trace.

```typescript
import { test, expect } from '@playwright/test';

test('failing form submission — generates trace', async ({ page }) => {
  await page.goto('https://example.com/contact');

  // Fill out form
  await page.getByLabel('Name').fill('Jane Doe');
  await page.getByLabel('Email').fill('jane@example.com');

  // The submit button has a dynamic ID — it will fail first try
  await page.locator('#submit-btn').click();

  // Wait for confirmation — but the button ID was wrong
  await expect(page.getByText('Thank you for your message')).toBeVisible();
});
```

Run the test:

```bash
npx playwright test --trace on failing-form.spec.ts
```

After the run, trace files appear in `test-results/`.

## Step 3: Open the Trace in Trace Viewer

Use the CLI or the VS Code extension to open trace files.

```bash
# Open the most recent trace file
npx playwright show-trace test-results/trace-*.zip
```

This opens the Trace Viewer in your browser. The UI has:

- **Left panel**: Action timeline — each Playwright action with duration and status
- **Center panel**: DOM snapshot — what the page looked like at each action
- **Right panel**: Network log, console output, and source reference

## Step 4: Navigate the Trace Viewer

Click through each action to understand the failure.

```typescript
import { test, expect } from '@playwright/test';

test('inspect network failure via trace', async ({ page }) => {
  // Intercept and fail the API call
  await page.route('**/api/submit', route => route.abort('connectionrefused'));

  await page.goto('https://example.com/contact');
  await page.getByLabel('Name').fill('Jane Doe');
  await page.getByLabel('Email').fill('jane@example.com');
  await page.getByRole('button', { name: 'Submit' }).click();

  // Test fails because the API call was aborted
  await expect(page.getByText('Thank you')).toBeVisible();
});
```

After running, open the trace and:

1. Click the last action (`click`) — see the "Network" tab shows the API request failed with status `(failed)`
2. Click the "Console" tab to see any error messages logged by the page
3. Toggle between "Before" and "After" DOM snapshots to see the page state change

## Step 5: Add Metadata to Traces

Use test annotations to track issues and link traces to bug reports.

```typescript
import { test, expect } from '@playwright/test';

test('order flow with metadata', async ({ page }) => {
  // Add metadata that appears in the trace
  test.info().annotations.push({
    type: 'issue',
    description: 'https://github.com/example/issues/4321',
  });
  test.info().annotations.push({
    type: 'browser',
    description: 'Chrome 120 — rendering bug investigation',
  });

  await page.goto('https://example.com/orders');
  await page.getByTestId('order-1').click();

  await expect(page.locator('.order-detail')).toBeVisible();

  // Capture a specific snapshot point
  await page.screenshot({ path: 'order-detail.png' });
});
```

## Step 6: Build Something Real

Write two tests: one that fails and generates a trace, then use the trace viewer to diagnose the root cause. Write a debugging report based on your analysis.

```typescript
import { test, expect } from '@playwright/test';

// --- Test 1: Failing test with trace ---
test('checkout fails due to disabled button', async ({ page }) => {
  await page.goto('https://example.com/checkout');

  // Fill cart
  await page.getByTestId('add-item').click();
  await page.getByTestId('add-item').click();

  // Proceed to checkout
  await page.getByRole('button', { name: 'Checkout' }).click();

  // Fill shipping info
  await page.getByLabel('Address').fill('123 Main St');
  await page.getByLabel('City').fill('Springfield');

  // The "Place Order" button is disabled because payment is not configured
  await page.getByRole('button', { name: 'Place Order' }).click();

  // This assertion fails
  await expect(page.getByText('Order Confirmed')).toBeVisible({ timeout: 3000 });
});

// --- Test 2: Debugging report (this is what you'd write after inspecting the trace) ---
/*
DEBUGGING REPORT — Trace Analysis for "checkout fails due to disabled button"

Root Cause:
The "Place Order" button was disabled because the payment form was not filled.
The trace's DOM snapshot at the "click" action shows:
  <button disabled data-testid="place-order">Place Order</button>

Evidence from Trace Viewer:
1. Action timeline: "click" on "Place Order" succeeded but no navigation followed
2. DOM snapshot after click: same page, button still visible, no loading state
3. Console: no errors (button just didn't trigger anything)
4. Network: no POST request to /api/orders — the disabled button prevented submission

Fix:
Add payment form steps before clicking "Place Order":
  await page.getByLabel('Card Number').fill('4242424242424242');
  await page.getByLabel('Expiry').fill('12/28');
  await page.getByLabel('CVC').fill('123');
Then click "Place Order".
*/
```

Run the test with tracing:

```bash
npx playwright test checkout-trace.spec.ts --trace on
npx playwright show-trace test-results/trace-checkout-*.zip
```
