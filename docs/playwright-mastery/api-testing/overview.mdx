# API Testing in Playwright

Difficulty: Intermediate.

## What You Will Learn

- Use `APIRequestContext` to send HTTP requests (GET, POST, PUT, DELETE) directly from your Playwright tests
- Combine API calls with browser-based UI testing for setup, teardown, and state verification
- Mock backend responses using `page.route()` to test error handling and edge cases
- Test API authentication flows and manage session tokens across requests

## Prerequisites

Before taking this lesson, you should have completed the following Playwright Mastery lessons:

- [Locators](../locators/overview) — selecting elements on the page
- [Actions](../actions/overview) — clicking, typing, and interacting
- [Assertions](../assertions/overview) — verifying expected outcomes
- [Waiting](../waiting/overview) — synchronizing with page state and network events

You should also be comfortable with basic HTTP concepts (status codes, JSON, headers).

## Why This Matters

API testing in Playwright eliminates the need for a separate HTTP client library or a second test framework. By using Playwright's built-in `APIRequestContext`, you can seed test data before a UI test runs, verify the backend state after user actions, and mock backend failures to test error handling -- all within the same test file. This unified approach reduces tooling complexity, speeds up test setup, and gives you deeper coverage of your application's behavior.

## Quick Reference

| Method | Purpose | Common Use Case |
|---|---|---|
| `request.get(url, options)` | Send a GET request | Fetch data or verify endpoint availability |
| `request.post(url, options)` | Send a POST request | Create a resource or authenticate |
| `request.put(url, options)` | Send a PUT request | Update an existing resource |
| `request.delete(url, options)` | Send a DELETE request | Remove a resource or clean up test data |
| `request.fetch(url, options)` | Send any HTTP method | Interact with APIs that use non-standard methods |
| `request.storageState()` | Extract cookies and localStorage | Share authentication state with the browser context |
