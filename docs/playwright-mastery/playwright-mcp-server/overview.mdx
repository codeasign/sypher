# Playwright MCP Server

Difficulty: Advanced.

## What You Will Learn

- Understand what MCP (Model Context Protocol) is and how it enables AI-assisted browser automation
- Set up the Playwright MCP server and connect it to an AI assistant
- Use natural language commands to drive browser interactions through an AI tool
- Create tests faster by describing what you want in plain English

## Prerequisites

You should already have Playwright installed in your project. You also need access to an AI coding tool that supports MCP, such as Claude Code. Basic familiarity with JSON-RPC and client-server architecture helps but is not required.

## Why This Matters

MCP lets AI assistants control a browser directly. Instead of writing Playwright code by hand line by line, you describe what you want to test in natural language and the AI drives Playwright for you. This transforms test creation from a coding task into a conversation, dramatically reducing the time from idea to running test.

## Quick Reference

### MCP Concepts

| Concept | Description |
|---|---|
| MCP Server | A process that exposes tools and resources over the Model Context Protocol |
| MCP Client | An AI assistant that connects to an MCP server |
| Tools | Actions the server exposes (navigate, click, fill, screenshot) |
| Resources | Data the server provides (page content, DOM snapshots) |

### Playwright MCP Server Tools

| Tool | Purpose |
|---|---|
| navigate(url) | Go to a URL in the browser |
| click(selector) | Click an element on the page |
| fill(selector, text) | Type text into an input field |
| snapshot() | Get a structural snapshot of the current page |
| screenshot() | Capture a visual screenshot of the current page |

### Installation Commands

```bash
# Run the Playwright MCP server directly
npx @playwright/mcp@latest

# Or install it globally
npm install -g @playwright/mcp
```

The MCP protocol acts as the middle layer. Your AI assistant sends natural-language requests. The MCP server translates them into Playwright commands and returns snapshots or screenshots back to the AI. You never write a locator or a `page.goto` call manually.
