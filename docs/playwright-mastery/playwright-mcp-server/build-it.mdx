# Build It: Playwright MCP Server

Follow these steps to run the Playwright MCP server and control a browser using natural language.

## Step 1: Install the Playwright MCP Server

You can run the server directly without installing it globally:

```bash
npx @playwright/mcp@latest
```

Or install it globally so it is always on your PATH:

```bash
npm install -g @playwright/mcp
```

## Step 2: Start the MCP Server

Open a terminal and run:

```bash
npx @playwright/mcp@latest
```

You should see output similar to this:

```
Playwright MCP server listening on port 3100
```

The server starts on port 3100 by default. It is now ready to accept connections from any MCP-compatible AI assistant.

## Step 3: Connect from Claude Code

In a separate terminal, configure Claude Code to use the MCP server. You can do this through the Claude Code configuration file (`claude.json` in your project root) or by passing flags on the command line.

Create a `claude.json` file in your project root:

```json
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@latest"]
    }
  }
}
```

If the MCP server is already running on port 3100, you can also connect using the `--mcp` flag:

```bash
claude --mcp http://localhost:3100
```

## Step 4: Run Your First Command

Once connected, give the AI a natural language instruction:

```
Navigate to https://example.com, take a screenshot, and tell me what you see.
```

The AI will call the MCP server's `navigate` tool, then the `screenshot` tool, and return a description of the page.

Try a few more commands:

```
Click on the "More information" link.
Take a snapshot of the current page structure.
Fill in the search box with "Playwright" and press Enter.
```

## Step 5: Build Something Real

Start the Playwright MCP server and connect your AI assistant. Then run the following scenario.

You will need a local web app to test against. If you do not have one, create a simple HTML file:

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Test App</title>
</head>
<body>
  <h1>Welcome</h1>
  <form id="signup-form">
    <label for="name">Name:</label>
    <input type="text" id="name" name="name" required />
    <label for="email">Email:</label>
    <input type="email" id="email" name="email" required />
    <label for="password">Password:</label>
    <input type="password" id="password" name="password" required />
    <button type="submit" id="submit-btn">Sign Up</button>
  </form>
  <div id="success-message" style="display:none;">
    <h2>Thank you for signing up!</h2>
  </div>
  <script>
    document.getElementById('signup-form').addEventListener('submit', function (e) {
      e.preventDefault();
      document.getElementById('signup-form').style.display = 'none';
      document.getElementById('success-message').style.display = 'block';
    });
  </script>
</body>
</html>
```

Serve it locally:

```bash
npx serve .
```

Now use these natural language prompts with your AI assistant connected to the MCP server:

**Prompt 1:** Navigate and verify page structure

```
Navigate to http://localhost:3000. Take a snapshot of the page and describe all the elements you see.
```

The AI uses the `snapshot` tool to understand the DOM structure. It returns a structured description of the page.

**Prompt 2:** Fill in the form

```
Fill in the "name" input with "Jane Doe", the "email" input with "jane@example.com", and the "password" input with "securepass123".
```

The AI uses the `fill` tool for each input. Watch the browser window update with each action.

**Prompt 3:** Submit the form and verify

```
Click the "Sign Up" button. Take a screenshot and tell me what changed on the page.
```

The AI clicks the submit button, then uses `screenshot` to capture the result. The success message should now be visible.

**Prompt 4:** Navigate elsewhere

```
Navigate to https://playwright.dev and take a screenshot.
```

The AI navigates to a new URL. The MCP server manages the same browser session.

Write down the natural language prompts you used and what each one did. Record any issues you encountered — selector mismatches, timing problems, or unexpected behavior. This log will help you understand the strengths and limitations of MCP-based browser automation.
