# Review: Playwright MCP Server

## Key Takeaways

1. The Model Context Protocol (MCP) enables AI assistants to control a browser by exposing Playwright tools through a JSON-RPC interface.
2. The Playwright MCP server runs as a standalone process and provides tools like navigate, click, fill, snapshot, and screenshot.
3. MCP is a development-time tool — use it for prototyping, debugging, and rapid test creation, not for production CI execution.
4. The MCP browser context is separate from your Playwright test context; they do not share session state.
5. Keep your MCP server version in sync with your Playwright version to avoid compatibility issues.

### Scenario Questions

**Scenario 1:** You are debugging a failing test that navigates to a login page, fills in credentials, and submits. The test passes locally but fails only when you run it through an AI assistant connected to MCP. What could be different between the two environments?

**Answer:** The MCP browser context is separate from your Playwright test context. If the test depends on localStorage state, cookies, or session data set up outside the MCP session (such as a previous test's login), that state is not available. Verify that the MCP session handles authentication from scratch. Also check that the MCP server is using the same browser configuration as your test (viewport size, locale, device emulation).

**Scenario 2:** Your team wants to use MCP to let non-developers create automated tests by describing scenarios in natural language. What workflow would you recommend?

**Answer:** Have the non-developer describe the scenario step by step in an AI chat connected to MCP. The AI executes each step in the browser and returns screenshots and snapshots for verification. Once the flow is correct, ask the AI to export the sequence of Playwright commands as a test file. A developer then reviews the generated test: converts locators to best practices (getByRole, getByTestId), adds assertions, factors in error handling, and runs the test in CI without the MCP server.
