# Actions — Review

## Key Takeaways

1. **Prefer `fill` over `type`.** The `fill` method is faster and more reliable because it clears the input and sets the value in one operation. Only use `type` when you need character-by-character behavior.
2. **Let Playwright handle actionability checks.** Do not add manual `waitForTimeout` or `waitForSelector` before actions. Playwright automatically waits for elements to be visible, enabled, stable, and not obscured.
3. **Use `{ force: true }` sparingly.** Force-clicking bypasses actionability checks. Reserve it for intentionally hidden elements like file inputs or custom-styled checkboxes.
4. **Chain actions with navigation carefully.** When a click triggers page navigation, Playwright waits for the navigation to complete automatically. Ensure you `await` the action so the wait can happen.
5. **Match the action to the intent.** Use `check`/`uncheck` for checkboxes, `selectOption` for dropdowns, `dragTo` for drag-and-drop, and `click` for buttons. The dedicated methods are more expressive and reliable than generic click-based workarounds.

## Recap

- Every Playwright action goes through three phases: actionability checks, DOM interaction, and load waiting.
- Use `fill` for text inputs, `selectOption` for dropdowns, `check`/`uncheck` for checkboxes, `dragTo` for drag-and-drop.
- Keyboard and mouse actions provide low-level control for advanced scenarios.
- Force option bypasses checks but should be used rarely.

## Quick Quiz

### Multiple Choice Questions

**1. Which method should you use to type text into an input field in most cases?**

A. `locator.type('value')`
B. `locator.fill('value')`
C. `locator.press('value')`
D. `page.keyboard.type('value')`

**Answer: B.** `fill` is the preferred method because it clears the field and sets the value in one fast operation.

---

**2. What does Playwright check during actionability checks before clicking?**

A. That the element is visible
B. That the element is enabled
C. That the element is stable (not animating)
D. All of the above

**Answer: D.** Playwright checks that the element is visible, enabled, stable (no ongoing animations), and not obscured by other elements before performing an action.

---

**3. What option allows you to click an element that is obscured by an overlay?**

A. `{ bypass: true }`
B. `{ force: true }`
C. `{ ignoreVisibility: true }`
D. `{ override: true }`

**Answer: B.** The `{ force: true }` option skips all actionability checks including visibility and obstruction checks.

---

**4. How do you select an option from a `<select>` dropdown by the visible label text?**

A. `dropdown.selectOption('Option 1')`
B. `dropdown.selectOption({ label: 'Option 1' })`
C. `dropdown.fill('Option 1')`
D. `dropdown.click({ option: 'Option 1' })`

**Answer: B.** `selectOption({ label: 'Option 1' })` selects the option whose visible text matches "Option 1". Passing a string directly selects by value attribute.

---

**5. After clicking a button that triggers navigation, what does Playwright do automatically?**

A. Nothing — you must call `page.waitForNavigation()` manually
B. It waits up to 30 seconds for the page to load
C. It waits for the next navigation to complete before resolving the action promise
D. It sets a 5-second timeout and then continues

**Answer: C.** Playwright's action methods wait for any resulting navigation to complete before resolving. This is why you should `await` the action.

---

### Scenario Questions

**Scenario 1:** You have a form with a "Submit" button that is covered by a cookie consent banner. The banner has a "Accept" button. How do you submit the form?

```typescript
// Answer — accept the cookie banner first, then submit
await page.getByRole('button', { name: /accept/i }).click();
await page.getByRole('button', { name: /submit/i }).click();
```

---

**Scenario 2:** A test requires you to type into an auto-suggest field. The suggestions should appear after each keystroke. Which method should you use and why?

```typescript
// Answer — use type() because it fires keyboard events per character
await page.getByLabel('Search').type('playwright', { delay: 50 });
// fill() would set the entire value at once without triggering per-character events
```
