# Actions — Avoid Mistakes

## When NOT to Use This

- **Do not use `click` when `dispatchEvent` would suffice.** If you only need to test internal state changes (like a hidden counter), dispatching a synthetic event is faster and more reliable than simulating a full user click.
- **Do not use `type` instead of `fill`.** The `type` method is significantly slower and introduces flakiness because it types one character at a time. Only use `type` when you need to test character-by-character behavior like auto-complete.
- **Do not add manual `page.waitForTimeout()` before actions.** Playwright automatically waits for actionability. Adding arbitrary delays makes tests slower without improving reliability.

## The Pattern to Remember

Let Playwright's actionability checks work for you. Never add manual waits before actions. If an action fails with "action: target is obscured" or "element is not visible", investigate why the element is not actionable instead of adding `{ force: true }` as a workaround.

## Common Mistakes

### Mistake 1: Using `.type()` instead of `.fill()`

```typescript
// BAD — slow, character by character, prone to flakiness
await page.getByLabel('Email').type('user@example.com');

// GOOD — fast, clears then fills in one operation
await page.getByLabel('Email').fill('user@example.com');
```

### Mistake 2: Clicking when element is obscured by an overlay

```typescript
// BAD — this will throw "action: target is obscured"
await page.getByRole('button', { name: /submit/i }).click();

// GOOD — close the overlay first, then click
await page.getByRole('button', { name: /close/i }).click();
await page.getByRole('button', { name: /submit/i }).click();

// LAST RESORT — force click skips actionability checks
await page.getByRole('button', { name: /submit/i }).click({ force: true });
```

### Mistake 3: Not using `{ force: true }` for intentionally hidden elements

```typescript
// BAD — will fail if the file input is hidden
await page.locator('input[type="file"]').setInputFiles('photo.jpg');

// GOOD — force is supported for setInputFiles and other file operations
// Actually, setInputFiles does not need force for hidden inputs — it works by default

// But for clicking a hidden element:
await page.getByTestId('hidden-submit').click({ force: true });
```

### Mistake 4: Clicking without awaiting navigation when it triggers one

```typescript
// BAD — clicking launches navigation, but you don't wait for it
page.getByRole('link', { name: /dashboard/i }).click();
// Next line assumes we are already on the dashboard page
await expect(page.getByText('Welcome')).toBeVisible();

// GOOD — click triggers navigation, Playwright auto-waits
await page.getByRole('link', { name: /dashboard/i }).click();
await expect(page).toHaveURL(/dashboard/);
await expect(page.getByText('Welcome')).toBeVisible();
```

### Mistake 5: Using keyboard shortcuts that do not work in all browsers

```typescript
// BAD — Ctrl+A is browser-specific and might not work in headless mode
await page.keyboard.press('Control+A');
await page.keyboard.press('Delete');

// GOOD — use Playwright's built-in clear or fill with empty string
await page.getByLabel('Search').clear();
// Or fill with empty string
await page.getByLabel('Search').fill('');

// GOOD — if you do need keyboard shortcuts, use Meta for macOS compatibility
await page.keyboard.press(process.platform === 'darwin' ? 'Meta+A' : 'Control+A');
```

## Troubleshooting

| Error | Cause | Solution |
|---|---|---|
| `action: target is obscured` | Another element (modal, overlay) is on top of the target | Close the blocking element or use `{ force: true }` as a last resort |
| Filling an input that is not visible | The input is hidden (display:none, visibility:hidden) | Use `{ force: true }` or find a visible input alternative |
| `selectOption` with dynamic options | Options are loaded after the page loads, and you selected before they appeared | Wait for the option to appear: `await expect(page.getByLabel('Country')).toContainText('US')` before selecting |
| Click does not trigger navigation | The click event listener calls `event.preventDefault()` or uses AJAX | Use `noWaitAfter: false` (default) and wait for a specific response or URL change |
