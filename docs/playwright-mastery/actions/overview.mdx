# Actions Overview

Difficulty: Beginner

## What You Will Learn

- How to perform clicks, double-clicks, and right-clicks with configurable options
- How to fill form inputs, select options, and toggle checkboxes
- How to execute drag-and-drop operations
- How to use keyboard and mouse actions for advanced interactions

## Prerequisites

You should have completed the Locators lesson and understand how to find elements with `getByRole`, `getByLabel`, `getByTestId`, and other locator methods.

## Why This Matters

Knowing the right action API is critical for stable tests. Playwright automatically performs actionability checks before every action: it ensures the element is visible, enabled, stable (not animating), and not obscured by another element. This eliminates the need for manual `waitFor` calls and prevents flaky interactions that plague older testing frameworks.

## Quick Reference

| Action | Method | Notes |
|---|---|---|
| Click | `locator.click(options?)` | Options: `force`, `noWaitAfter`, `position`, `modifiers`, `delay`, `button` |
| Double-click | `locator.dblclick(options?)` | Same options as click |
| Fill text | `locator.fill(value)` | Clears field first; preferred over `type` |
| Type text | `locator.type(value)` | Types character by character; use only for keyboard-event tests |
| Select option | `locator.selectOption(value)` | Accepts value, label, or index |
| Check | `locator.check(options?)` | Toggles checkbox to checked |
| Uncheck | `locator.uncheck(options?)` | Toggles checkbox to unchecked |
| Hover | `locator.hover(options?)` | Triggers hover events |
| Drag to | `locator.dragTo(target)` | Drags element to another element |
| Focus | `locator.focus()` | Moves keyboard focus |
| Press key | `locator.press(key)` | Simulates a single keypress |
| Scroll | `page.mouse.wheel(dx, dy)` | Scrolls the page |
| Upload file | `locator.setInputFiles(path)` | Sets file input value |

## The Big Picture

<AsciiDiagram id="playwright-mastery/actions/overview" alt="Playwright action pipeline with actionability checks" caption="How Playwright actions execute" content={`
  Action called
       │
       ▼
  ┌────────────────┐
  │ Actionability   │  Checks: visible, enabled,
  │ checks          │  stable, not obscured
  └────────────────┘
       │ (all pass)
       ▼
  ┌────────────────┐
  │ DOM interaction │  click, fill, select...
  └────────────────┘
       │
       ▼
  ┌────────────────┐
  │ Wait for load  │  auto-wait for navigation
  └────────────────┘
`} />

Every Playwright action follows this three-phase pipeline. First, Playwright checks actionability (up to the configured timeout). Then it dispatches the DOM interaction. Finally, it waits for any resulting navigation or page load to complete.
