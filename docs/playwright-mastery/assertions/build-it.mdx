# Assertions — Build It

In this lesson, you will write every major Playwright assertion type. You will learn the difference between auto-retrying and non-retrying assertions, and when to use soft assertions to let a test continue after a failure.

## Setup

Create a test file called `assertions.spec.ts`.

```typescript
// assertions.spec.ts
import { test, expect } from '@playwright/test';

test.describe('Playwright assertions', () => {
  test.beforeEach(async ({ page }) => {
    await page.goto('https://the-internet.herokuapp.com/login');
  });
});
```

## Step 1: toBeVisible — Auto-Retrying Assertion

The most fundamental assertion checks that an element is visible on the page.

```typescript
test('assert element visibility', async ({ page }) => {
  const loginButton = page.getByRole('button', { name: /login/i });

  // Auto-retrying: polls until visible or timeout
  await expect(loginButton).toBeVisible();

  // Negative assertion: element should NOT be visible
  const logoutButton = page.getByRole('button', { name: /logout/i });
  await expect(logoutButton).not.toBeVisible();
});
```

Playwright retries `toBeVisible` every 500ms. If the button appears after 2 seconds (e.g., due to animation), the assertion will pass at that point without you adding any manual waits.

## Step 2: toHaveText and toContainText

These assertions check the text content of an element.

```typescript
test('assert text content', async ({ page }) => {
  const heading = page.getByText('Login Page');

  // Exact match — fails if there is extra whitespace or different text
  await expect(heading).toHaveText('Login Page');

  // Regex match — more flexible
  await expect(heading).toHaveText(/login/i);

  // Contains — only checks that the text appears somewhere
  await expect(heading).toContainText('Login');

  // toHaveText with options: ignore case and trim whitespace
  await expect(heading).toHaveText('  login page  ', { ignoreCase: true });
  // Note: by default, toHaveText trims whitespace and normalizes line breaks
});
```

## Step 3: toHaveCount

Use `toHaveCount` when you need to verify the number of matching elements.

```typescript
test('assert element count', async ({ page }) => {
  await page.goto('https://the-internet.herokuapp.com/checkboxes');

  // Assert there are exactly 2 checkboxes
  const checkboxes = page.getByRole('checkbox');
  await expect(checkboxes).toHaveCount(2);
});
```

This assertion is auto-retrying, so it handles lazy-loaded lists correctly. If items are added dynamically, the assertion will wait until the expected count is reached.

## Step 4: toHaveAttribute

Check that an element has a specific attribute, optionally with a specific value.

```typescript
test('assert element attributes', async ({ page }) => {
  // Check that the password input has type="password"
  const passwordInput = page.getByLabel('Password');
  await expect(passwordInput).toHaveAttribute('type', 'password');

  // Check that the login form has the id attribute (any value)
  await expect(page.locator('#login')).toHaveAttribute('id');

  // Check that a link has a specific href
  const link = page.getByText('Elemental Selenium');
  await expect(link).toHaveAttribute('href', /gitlab/);
});
```

## Step 5: Page-Level Assertions

Some assertions operate on the page itself rather than on a specific element.

```typescript
test('assert page URL and title', async ({ page }) => {
  // Assert the current URL
  await expect(page).toHaveURL('https://the-internet.herokuapp.com/login');

  // Assert URL with regex (useful for dynamic path segments)
  await expect(page).toHaveURL(/\/login$/);

  // Assert the page title
  await expect(page).toHaveTitle('The Internet');
});
```

## Step 6: Soft Assertions

By default, when an assertion fails, the test stops immediately at that line. Soft assertions let the test continue after a failure and report all failures at the end.

```typescript
test('soft assertions allow test to continue', async ({ page }) => {
  await page.goto('https://the-internet.herokuapp.com/checkboxes');

  const checkbox1 = page.getByRole('checkbox').nth(0);
  const checkbox2 = page.getByRole('checkbox').nth(1);

  // Soft assertion — test continues even if this fails
  await expect.soft(checkbox1).toBeChecked();

  // Hard assertion — test stops here if this fails
  await expect(checkbox2).not.toBeChecked();

  // Check the soft assertion results at the end
  // All soft assertion failures are collected and reported
  await checkbox1.check();

  // You can also check the test info for soft assertion failures
  // test.info().errors contains any soft assertion failures
});
```

Soft assertions are useful when you want to validate multiple things about a page and report all failures in one run rather than failing fast after the first issue.

## Step 7: Custom Matchers with expect.extend

You can create custom matchers using `expect.extend()`. This is useful for domain-specific assertions.

```typescript
// In a setup file or at the top of your test file
expect.extend({
  async toBeInputOfType(received, expectedType) {
    const actualType = await received.getAttribute('type');
    const pass = actualType === expectedType;
    return {
      message: () =>
        `expected input to have type "${expectedType}" but got "${actualType}"`,
      pass,
    };
  },
});

// Now use it in your tests
test('custom matcher for input type', async ({ page }) => {
  const passwordInput = page.getByLabel('Password');
  await expect(passwordInput).toBeInputOfType('password');
});
```

**Note:** For the custom matcher to work across all test files, define it in a setup file or in `playwright.config.ts` using `globalSetup` or `projects`.

## Step 8: Build Something Real

Now combine everything into a complete end-to-end test that fills a form, submits it, and asserts all expected outcomes using a mix of hard and soft assertions.

```typescript
import { test, expect } from '@playwright/test';

test.describe('Complete assertion workflow', () => {
  test('form submission assertions', async ({ page }) => {
    // Navigate to login page
    await page.goto('https://the-internet.herokuapp.com/login');

    // Assert initial state
    await expect(page).toHaveURL(/login/);
    await expect(page.getByRole('heading')).toContainText('Login');

    // Assert form elements exist
    await expect(page.getByLabel('Username')).toBeVisible();
    await expect(page.getByLabel('Password')).toBeVisible();
    await expect(page.getByRole('button', { name: /login/i })).toBeVisible();

    // Fill the form
    await page.getByLabel('Username').fill('tomsmith');
    await page.getByLabel('Password').fill('SuperSecretPassword!');

    // Assert values were set correctly
    await expect(page.getByLabel('Username')).toHaveValue('tomsmith');
    await expect(page.getByLabel('Password')).toHaveValue('SuperSecretPassword!');
    await expect(page.getByLabel('Password')).toHaveAttribute('type', 'password');

    // Submit the form
    await page.getByRole('button', { name: /login/i }).click();

    // Assert post-submit state with hard assertions
    await expect(page).toHaveURL(/secure/);
    await expect(page.getByText('You logged into a secure area')).toBeVisible();

    // Assert elements that should now be present
    await expect(page.getByRole('link', { name: /logout/i })).toBeVisible();
    await expect(page.getByRole('button', { name: /login/i })).not.toBeVisible();

    // Soft assertions: check secondary details (test continues if any fail)
    await expect.soft(page.getByText('Secure Area')).toBeVisible();
    await expect.soft(page.locator('.flash')).toHaveClass(/success/);
    await expect.soft(page.getByText('Welcome')).not.toBeVisible();
  });

  test('assert checkbox states with count and visibility', async ({ page }) => {
    await page.goto('https://the-internet.herokuapp.com/checkboxes');

    // Assert initial count
    const checkboxes = page.getByRole('checkbox');
    await expect(checkboxes).toHaveCount(2);

    // Check both checkboxes
    await checkboxes.nth(0).check();
    await checkboxes.nth(1).check();

    // Assert both are checked
    await expect(checkboxes.nth(0)).toBeChecked();
    await expect(checkboxes.nth(1)).toBeChecked();

    // Assert count with toHaveCount after changes
    await expect(checkboxes).toHaveCount(2); // still 2

    // Assert attributes
    await expect(checkboxes.nth(0)).toHaveAttribute('type', 'checkbox');
  });

  test('assert button state: enabled and disabled', async ({ page }) => {
    await page.goto('https://the-internet.herokuapp.com/add_remove_elements/');

    // The "Add Element" button should be enabled
    const addButton = page.getByRole('button', { name: /add element/i });
    await expect(addButton).toBeEnabled();

    // Click to add an element
    await addButton.click();

    // The "Delete" button should now exist and be visible
    const deleteButton = page.getByRole('button', { name: /delete/i });
    await expect(deleteButton).toBeVisible();
    await expect(deleteButton).toBeEnabled();

    // Click delete and assert it is gone
    await deleteButton.click();
    await expect(deleteButton).not.toBeVisible();
  });

  test('URL and navigation assertions', async ({ page }) => {
    await page.goto('https://the-internet.herokuapp.com/');

    // Assert base URL
    await expect(page).toHaveURL(/herokuapp/);

    // Click a link and assert the new URL
    await page.getByRole('link', { name: /login/i }).click();
    await expect(page).toHaveURL(/\/login$/);
    await expect(page).toHaveTitle('The Internet');

    // Go back and assert URL again
    await page.goBack();
    await expect(page).toHaveURL(/herokuapp\.com\/$/);
  });
});
```
