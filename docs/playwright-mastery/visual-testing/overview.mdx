# Visual Testing

Difficulty: Advanced

## What You Will Learn

- Capture screenshot comparisons with `toHaveScreenshot()` and `expect(page).toHaveScreenshot()`
- Configure pixel-diff thresholds and mask dynamic content
- Handle cross-browser and cross-platform rendering differences
- Integrate visual regression tests into your CI pipeline

## Prerequisites

- Playwright assertions and test configuration
- CI/CD basics (GitHub Actions, Jenkins, or similar)
- Understanding of PNG snapshots and pixel comparison

## Why This Matters

Functional assertions like `await expect(page.locator('h1')).toHaveText('Welcome')` verify content but miss layout shifts, font changes, broken CSS, misplaced elements, and rendering bugs across browsers. A single CSS change can push a button off-screen or overlap two elements — only visual testing catches these regressions. Without it, you ship visual bugs that users see immediately.

## Quick Reference

| API / Option | Type | Description |
|---|---|---|
| `expect(page).toHaveScreenshot(name)` | assertion | Captures full page and compares to named baseline |
| `expect(locator).toHaveScreenshot()` | assertion | Captures a specific element |
| `{ fullPage: true }` | option | Screenshot the entire scrollable page |
| `{ clip: { x, y, width, height } }` | option | Capture a specific region |
| `{ mask: [locator1, locator2] }` | option | Hide elements with a colored overlay |
| `{ maxDiffPixels: number }` | option | Fail if more than N pixels differ |
| `{ maxDiffPixelRatio: number }` | option | Fail if pixel diff ratio exceeds value (0-1) |
| `{ threshold: number }` | option | Color difference threshold per channel (0-1) |
| `{ animations: 'disabled' \| 'allow' }` | option | Disable or allow CSS animations during capture |
| `{ style: string }` | option | Inject CSS override into the page before capture |

## The Big Picture

Visual testing compares a stored baseline screenshot against a fresh capture. If the pixel difference exceeds your threshold, the test fails and produces a diff image for review.
