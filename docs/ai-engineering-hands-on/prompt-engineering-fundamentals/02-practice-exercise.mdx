## The Build — Step by Step

### Step 1: The Baseline — A Naive Prompt

Create `prompt_baseline.py`:

```python
import os
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

completion = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "user", "content": "Explain machine learning."}
    ]
)

print("Response:")
print(completion.choices[0].message.content)
```

Run it:

```bash
python prompt_baseline.py
```

This is a naive prompt. No system instruction, no format guidance, no constraints. Run it three times and notice how the output varies in length, tone, and content.

### Step 2: Add a System Prompt

Create `prompt_system.py`:

```python
import os
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

responses = []

for persona in ["a university professor", "a 10-year-old", "a stand-up comedian"]:
    completion = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {"role": "developer", "content": f"You are {persona}. Answer every question in character."},
            {"role": "user", "content": "Explain what machine learning is."}
        ],
        temperature=0.7
    )
    response = completion.choices[0].message.content
    responses.append((persona, response))

for persona, response in responses:
    print(f"=== {persona} ===")
    print(response[:200])
    print()
```

Run it:

```bash
python prompt_system.py
```

Notice how the same question produces completely different responses based solely on the system prompt. The system prompt sets the model's "frame" — you can dramatically change output behavior without changing the user's question.

### Step 3: Output Format Control

Create `prompt_format.py`:

```python
import os
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

formats = [
    "Respond in exactly one sentence.",
    "Respond as a bullet-point list with 3 items.",
    "Respond as a JSON object with 'title' and 'description' fields.",
    "Respond as a single paragraph of exactly 50 words."
]

for fmt in formats:
    completion = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {"role": "user", "content": f"Explain what Python is. {fmt}"}
        ],
        temperature=0.0
    )
    print(f"Format: {fmt}")
    print(completion.choices[0].message.content)
    print(f"Word count: {len(completion.choices[0].message.content.split())}")
    print("-" * 60)
```

Run it:

```bash
python prompt_format.py
```

Output format control is one of the most powerful prompt engineering techniques. When you explicitly specify the format, you eliminate the model's default behavior of generating varying structures.

### Step 4: Few-Shot Prompting

Create `few_shot.py`:

```python
import os
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

# Zero-shot (no examples)
zero_shot = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "user", "content": "Classify this tweet sentiment: 'Just got promoted! So excited!'"}
    ],
    temperature=0.0
)

print("Zero-shot result:")
print(zero_shot.choices[0].message.content)
print()

# Few-shot (three examples)
few_shot = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "user", "content": """Classify each tweet as Positive, Negative, or Neutral. Output only one word.

Tweet: Had a great lunch today!
Sentiment: Positive

Tweet: Stuck in traffic for two hours.
Sentiment: Negative

Tweet: The store opens at 9am.
Sentiment: Neutral

Tweet: Just got promoted! So excited!
Sentiment:"""}
    ],
    temperature=0.0
)

print("Few-shot result:")
print(few_shot.choices[0].message.content)
```

Run it:

```bash
python few_shot.py
```

The zero-shot version might say "This is clearly positive" or "The sentiment is positive" — varying formats. The few-shot version outputs exactly "Positive" because the examples established that pattern. Few-shot is the simplest way to control output format without structured output.

### Step 5: Chain-of-Thought Prompting

Create `chain_of_thought.py`:

```python
import os
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

# Without chain-of-thought
direct = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "user", "content": "A store has a 25% discount on all items. If a shirt originally costs $40 and you also have a $10 coupon, how much do you pay? Assume the coupon is applied after the discount."}
    ],
    temperature=0.0
)

print("Direct answer:")
print(direct.choices[0].message.content)
print()

# With chain-of-thought
cot = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "user", "content": """A store has a 25% discount on all items. If a shirt originally costs $40 and you also have a $10 coupon, how much do you pay? Assume the coupon is applied after the discount.

Think step by step, then give the final answer."""}
    ],
    temperature=0.0
)

print("Chain-of-thought answer:")
print(cot.choices[0].message.content)
```

Run it:

```bash
python chain_of_thought.py
```

Compare the two responses. The direct answer may skip intermediate steps and produce the wrong result. The chain-of-thought version walks through each calculation: original price $40, 25% off = $10 discount, sale price = $30, minus $10 coupon = $20. If the answer is wrong, you can see exactly which step failed.

### Step 6: Temperature Comparison — Deterministic vs Creative

Create `temperature_compare.py`:

```python
import os
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

prompt = "Write a one-sentence tagline for a coffee shop."

print("=== Temperature 0.0 (deterministic) ===")
for i in range(3):
    completion = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": prompt}],
        temperature=0.0
    )
    print(f"  Run {i+1}: {completion.choices[0].message.content}")

print("\n=== Temperature 0.3 (slightly varied) ===")
for i in range(3):
    completion = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": prompt}],
        temperature=0.3
    )
    print(f"  Run {i+1}: {completion.choices[0].message.content[:80]}...")

print("\n=== Temperature 1.0 (creative) ===")
for i in range(3):
    completion = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": prompt}],
        temperature=1.0
    )
    print(f"  Run {i+1}: {completion.choices[0].message.content[:80]}...")
```

Run it:

```bash
python temperature_compare.py
```

At temperature 0.0, all three runs produce identical output. At 0.3, minor variations. At 1.0, the outputs diverge significantly. This is why you develop prompts at 0.0 and only raise temperature for creative applications.

## Common Errors at This Stage

### Error 1: System prompt is too vague

```
Prompt: "You are helpful."
Result: Generic, unhelpful responses.
```

**Cause:** "Helpful" is too broad — the model does not know what kind of help you want.
**Fix:** Be specific: "You are a Python expert who writes production-ready code with error handling and type hints."

### Error 2: Contradictory instructions

```
Prompt: "Be concise. Provide a detailed explanation."
```

**Cause:** The model cannot satisfy contradictory constraints.
**Fix:** Prioritize: "Be concise. Use at most 3 sentences."

### Error 3: No output format specification

```
Prompt: "Give me a recipe for pancakes."
Result: Sometimes paragraph, sometimes list, sometimes with timestamps, sometimes without.
```

**Cause:** You left the format unspecified.
**Fix:** "Give me a recipe for pancakes. Use this format: Ingredients (bullet list), Instructions (numbered steps)."

### Error 4: Putting user data in the system prompt

```python
# WRONG — user data in system prompt enables injection
messages = [
    {"role": "system", "content": f"You are an assistant. The user says: {user_input}"}
]

# RIGHT — instructions and data separated
messages = [
    {"role": "system", "content": "You are an assistant."},
    {"role": "user", "content": user_input}
]
```

### Error 5: Assuming the model remembers your instructions across turns

```python
# First turn — instruction given
# Second turn — model may "forget" the instruction
```

**Cause:** The model has no persistent memory — it only sees the current context window.
**Fix:** Repeat key instructions at the start of each turn, or keep the entire conversation in context.

### Error 6: Not testing at temperature 0.0 first

```
Developing a prompt at high temperature → inconsistent results → "it works sometimes"
```

**Cause:** High temperature masks whether your prompt is actually correct.
**Fix:** Develop at 0.0. Once the prompt is reliable, raise temperature for creative applications.
