## Tier 1 — Beginner

### Exercise 1: Ask for a Recipe

**Task:** Write a Python script that asks the model for a one-paragraph recipe for chocolate chip cookies. Use `max_tokens=200`. Print only the recipe text, not the full response object.

**Hint:** You only need to pass one user message and access `response.content[0].text`.

<details>
<summary>Solution</summary>

```python
import os
from dotenv import load_dotenv
from anthropic import Anthropic

load_dotenv()

client = Anthropic(
    api_key=os.environ["ANTHROPIC_API_KEY"]
)

response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=200,
    messages=[
        {"role": "user", "content": "Give me a one-paragraph recipe for chocolate chip cookies."}
    ]
)

print(response.content[0].text)
```

Expected output begins with something like:

```text
Here's a simple chocolate chip cookie recipe: Preheat your oven to 350°F (175°C). In a large bowl, cream together 1 cup of softened butter...
```
</details>

### Exercise 3: Ask the Same Question Twice

**Task:** Write a script that asks the model "What color is the sky?" with `temperature=0` the first time and `temperature=1.0` the second time. Compare the two responses.

**Hint:** Create two separate `client.messages.create()` calls with different `temperature` values.

<details>
<summary>Solution</summary>

```python
import os
from dotenv import load_dotenv
from anthropic import Anthropic

load_dotenv()

client = Anthropic(
    api_key=os.environ["ANTHROPIC_API_KEY"]
)

prompt = "What color is the sky? Answer in one short sentence."

response1 = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=50,
    temperature=0,
    messages=[{"role": "user", "content": prompt}]
)

response2 = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=50,
    temperature=1.0,
    messages=[{"role": "user", "content": prompt}]
)

print("Temperature 0:")
print(response1.content[0].text)
print()
print("Temperature 1.0:")
print(response2.content[0].text)
```

With `temperature=0`, the response will be the same nearly every time. With `temperature=1.0`, the wording may vary between runs.
</details>

### Exercise 4: Check Your Key Before Calling

**Task:** Write a script that checks whether the `ANTHROPIC_API_KEY` environment variable exists and looks valid (starts with `sk-ant-` and is at least 50 characters long) before making any API call. If the key is missing or invalid, print a helpful error message and exit.

**Hint:** Use `os.environ.get("ANTHROPIC_API_KEY")` — it returns `None` instead of raising an error if the key is missing.

<details>
<summary>Solution</summary>

```python
import os
import sys
from dotenv import load_dotenv

load_dotenv()

api_key = os.environ.get("ANTHROPIC_API_KEY")

if not api_key:
    print("ERROR: ANTHROPIC_API_KEY environment variable is not set.")
    print("Create a .env file with: ANTHROPIC_API_KEY=sk-ant-...")
    sys.exit(1)

if not api_key.startswith("sk-ant-"):
    print("ERROR: ANTHROPIC_API_KEY does not start with 'sk-ant-'.")
    print("Check that you copied the correct key from console.anthropic.com.")
    sys.exit(1)

if len(api_key) < 50:
    print("ERROR: ANTHROPIC_API_KEY seems too short.")
    print(f"Found {len(api_key)} characters, expected at least 50.")
    sys.exit(1)

print("API key looks valid. Ready to make API calls.")
```

This pattern protects you from cryptic errors by validating the key before any API request is made.
</details>

## Tier 2 — Intermediate

### Exercise 1: Multi-Turn Conversation

**Task:** Write a script that carries on a three-turn conversation with the model. You ask the first question, the model answers. Then you append both messages to the conversation, ask a follow-up, get a second answer. Then append again and ask a third question. Print the entire conversation each time.

**Hint:** Each `client.messages.create()` call must include the full message history — every role/content pair from every previous turn.

<details>
<summary>Solution</summary>

```python
import os
from dotenv import load_dotenv
from anthropic import Anthropic

load_dotenv()

client = Anthropic(
    api_key=os.environ["ANTHROPIC_API_KEY"]
)

messages = []

# Turn 1
messages.append({"role": "user", "content": "What is the capital of Japan?"})
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=100,
    messages=messages
)
messages.append({"role": "assistant", "content": response.content[0].text})
print(f"Q: {messages[0]['content']}")
print(f"A: {messages[1]['content']}")
print()

# Turn 2
messages.append({"role": "user", "content": "What is the population of that city?"})
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=100,
    messages=messages
)
messages.append({"role": "assistant", "content": response.content[0].text})
print(f"Q: {messages[2]['content']}")
print(f"A: {messages[3]['content']}")
print()

# Turn 3
messages.append({"role": "user", "content": "What language do they speak there?"})
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=100,
    messages=messages
)
messages.append({"role": "assistant", "content": response.content[0].text})
print(f"Q: {messages[4]['content']}")
print(f"A: {messages[5]['content']}")
```

Each turn includes all prior messages, so the model remembers the full context.
</details>

### Exercise 2: Save Responses to a File

**Task:** Write a script that asks the model to write a short poem about programming, then saves the poem to a file called `poem.txt` along with the token usage. The file should look like:

```text
--- Poem ---
[poem text here]

--- Token Usage ---
Input tokens: N
Output tokens: N
```

**Hint:** Use Python's built-in `open()` and `write()` functions. Build the file content as a single string before writing.

<details>
<summary>Solution</summary>

```python
import os
from dotenv import load_dotenv
from anthropic import Anthropic

load_dotenv()

client = Anthropic(
    api_key=os.environ["ANTHROPIC_API_KEY"]
)

response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=300,
    messages=[
        {"role": "user", "content": "Write a short poem about programming."}
    ]
)

poem = response.content[0].text
input_tokens = response.usage.input_tokens
output_tokens = response.usage.output_tokens

content = f"--- Poem ---\n{poem}\n\n--- Token Usage ---\nInput tokens: {input_tokens}\nOutput tokens: {output_tokens}\n"

with open("poem.txt", "w") as f:
    f.write(content)

print("Saved to poem.txt")
print(content)
```
</details>

### Exercise 3: Format Output as JSON

**Task:** Ask the model to generate a list of three fictional characters with name, age, and role. Use a system prompt that instructs the model to respond with valid JSON only.

**Hint:** The system prompt can say "Respond with valid JSON and nothing else." Then parse the response with `json.loads()`.

<details>
<summary>Solution</summary>

```python
import os
import json
from dotenv import load_dotenv
from anthropic import Anthropic

load_dotenv()

client = Anthropic(
    api_key=os.environ["ANTHROPIC_API_KEY"]
)

response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=300,
    system="Respond with valid JSON only. Do not include any explanation or markdown formatting.",
    messages=[
        {"role": "user", "content": "Create a JSON array of 3 fictional characters. Each character has: name, age, and role."}
    ]
)

characters = json.loads(response.content[0].text)
print(json.dumps(characters, indent=2))
```

Expected output:

```json
[
  {
    "name": "Elena Vasquez",
    "age": 34,
    "role": "Starship Engineer"
  },
  {
    "name": "Marcus Chen",

  {
    "name": "Aria Patel",

    "role": "Mission Commander"

```
</details>

### Exercise 4: Handle a Rate Limit with Retry

**Task:** Write a script that makes 10 rapid API calls in a loop. If a rate limit error (HTTP 429) occurs, wait 5 seconds and retry. If the error happens more than 3 times, stop and print a message.

**Hint:** Catch `anthropic.RateLimitError` and use `time.sleep(5)` before retrying. Keep a retry counter.

<details>
<summary>Solution</summary>

```python
import os
import time
from dotenv import load_dotenv
from anthropic import Anthropic, RateLimitError

load_dotenv()

client = Anthropic(
    api_key=os.environ["ANTHROPIC_API_KEY"]
)

max_retries = 3

for i in range(10):
    retries = 0
    while retries < max_retries:
        try:
            response = client.messages.create(
                model="claude-sonnet-4-20250514",
                max_tokens=20,
                messages=[
                    {"role": "user", "content": f"Say the number {i} only."}
                ]
            )
            print(f"Request {i}: {response.content[0].text}")
            break
        except RateLimitError:
            retries += 1
            if retries < max_retries:
                print(f"Request {i}: rate limited, retrying in 5 seconds...")
                time.sleep(5)
            else:
                print(f"Request {i}: failed after {max_retries} retries.")
```

This pattern prevents your script from crashing when you hit API rate limits.
</details>
