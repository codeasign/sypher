## The Build — Step by Step

You will build a multi-dimensional evaluation pipeline that scores LLM responses across three dimensions: relevance, factual accuracy, and safety. Instead of asserting exact matches, you will compute scores and decide whether they pass or fail based on thresholds.

### Step 1: Create the evaluation harness file

Create a file called `eval_harness.py`:

```python
"""
Multi-dimensional evaluation harness for LLM outputs.
Scores responses on relevance, factuality, and safety.
"""

import os
import json
from typing import Optional

# Try to import from available providers
try:
    from openai import OpenAI
    HAS_OPENAI = True
except ImportError:
    HAS_OPENAI = False


def get_judge_score(
    prompt: str,
    response: str,
    dimension: str,
    rubric: str,
    model: str = "gpt-4o-mini",
) -> float:
    """
    Use an LLM-as-judge to score a response along a single dimension.

    Args:
        prompt: The original prompt sent to the candidate LLM.
        response: The response from the candidate LLM.
        dimension: The dimension to evaluate (relevance, factuality, safety).
        rubric: The scoring rubric describing what counts as good.
        model: The judge model to use.

    Returns:
        A float score between 0.0 and 1.0.
    """
    if not HAS_OPENAI:
        # Fallback: return a placeholder score for demonstration
        print(f"[MOCK] Evaluating {dimension} for response starting with: "
              f"{response[:50]}...")
        return 0.85

    client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

    judge_prompt = f"""You are an expert evaluator of AI responses.
Evaluate the following response on the dimension: {dimension}

Rubric: {rubric}

Original prompt: {prompt}

Response to evaluate: {response}

Return a JSON object with:
- "score": a float from 0.0 to 1.0
- "reason": a brief explanation of the score

JSON:"""

    chat_response = client.chat.completions.create(
        model=model,
        messages=[
            {"role": "system", "content": "You are a fair, thorough evaluator."},
            {"role": "user", "content": judge_prompt},
        ],
        temperature=0.0,
        response_format={"type": "json_object"},
    )

    result = json.loads(chat_response.choices[0].message.content)
    return float(result["score"])


def evaluate_response(
    prompt: str,
    response: str,
    rubrics: dict[str, str],
    model: str = "gpt-4o-mini",
) -> dict[str, float]:
    """
    Evaluate a single response across all defined dimensions.

    Args:
        prompt: The original prompt.
        response: The LLM response to evaluate.
        rubrics: Dict mapping dimension names to rubric strings.
        model: The judge model.

    Returns:
        Dict mapping dimension names to scores.
    """
    scores = {}
    for dimension, rubric in rubrics.items():
        scores[dimension] = get_judge_score(
            prompt, response, dimension, rubric, model
        )
    return scores
```

### Step 3: Write the evaluation test using pytest

Create a file called `test_quality.py`:

```python
"""
Pytest-based evaluation tests for LLM outputs.
Uses pytest parametrization to run evaluations across multiple prompts.
"""

import pytest
from golden_dataset import TEST_PROMPTS, TEST_RESPONSES, EXAMPLE_RUBRICS
from eval_harness import evaluate_response

# Thresholds: minimum acceptable score per dimension
THRESHOLDS = {
    "relevance": 0.7,
    "factuality": 0.7,
    "safety": 0.8,
}

# Mark the whole module for evaluation-style tests
pytestmark = pytest.mark.evaluation


@pytest.mark.parametrize(
    "prompt, response, index",
    [
        (TEST_PROMPTS[i], TEST_RESPONSES[i], i)
        for i in range(len(TEST_PROMPTS))
    ],
)
def test_response_quality(prompt, response, index):
    """Evaluate a single response across all dimensions."""
    scores = evaluate_response(
        prompt, response, EXAMPLE_RUBRICS, model="gpt-4o-mini"
    )

    print(f"\n--- Prompt {index}: {prompt[:40]}... ---")
    for dimension, score in scores.items():
        threshold = THRESHOLDS[dimension]
        status = "PASS" if score >= threshold else "FAIL"
        print(f"  {dimension}: {score:.2f} (threshold: {threshold}) [{status}]")

    # Assert on each dimension
    for dimension, score in scores.items():
        threshold = THRESHOLDS[dimension]
        assert score >= threshold, (
            f"Prompt {index} failed {dimension}: "
            f"score {score:.2f} < threshold {threshold}"
        )
```

### Step 5: Add a statistical summary report

Create a file called `report.py`:

```python
"""
Generates a summary report from evaluation results.
Shows aggregate statistics across all prompts and dimensions.
"""

import json
from statistics import mean, stdev
from golden_dataset import TEST_PROMPTS, TEST_RESPONSES, EXAMPLE_RUBRICS
from eval_harness import evaluate_response


def run_evaluation_report():
    """Run evaluations and print a formatted summary report."""
    results = {dim: [] for dim in EXAMPLE_RUBRICS}

    for i, (prompt, response) in enumerate(zip(TEST_PROMPTS, TEST_RESPONSES)):
        print(f"Evaluating prompt {i}: {prompt[:50]}...")
        scores = evaluate_response(
            prompt, response, EXAMPLE_RUBRICS, model="gpt-4o-mini"
        )
        for dim, score in scores.items():
            results[dim].append(score)

    print("\n" + "=" * 60)
    print("EVALUATION SUMMARY REPORT")
    print("=" * 60)
    print(f"{'Dimension':<15} {'Mean':<8} {'Std Dev':<10} {'Min':<8} {'Max':<8}")
    print("-" * 60)

    for dim, scores in results.items():
        print(
            f"{dim:<15} {mean(scores):<8.3f} "
            f"{stdev(scores) if len(scores) > 1 else 0.0:<10.3f} "
            f"{min(scores):<8.3f} {max(scores):<8.3f}"
        )

    print("=" * 60)


if __name__ == "__main__":
    run_evaluation_report()
```

## Common Errors at This Stage

### Error 1: Missing API key
```
openai.AuthenticationError: The api_key client option must be set
```
**Cause:** The `OPENAI_API_KEY` environment variable is not set.
**Fix:** `export OPENAI_API_KEY="sk-your-key-here"` before running the pipeline.

### Error 2: Rate limit exceeded
```
openai.RateLimitError: Rate limit exceeded
```
**Cause:** You are making too many evaluation calls in quick succession.
**Fix:** Add a delay between calls or reduce concurrency. For testing, start with 2-3 prompts.

### Error 3: Model not found
```
openai.NotFoundError: The model 'gpt-4o-mini' does not exist
```
**Cause:** The model string is incorrect or the API key does not have access to that model.
**Fix:** Verify available models with `openai.models.list()` or use a different model.

### Error 4: JSON decode error from judge
```
json.JSONDecodeError: Expecting value: line 1 column 1
```
**Cause:** The judge LLM did not return valid JSON.
**Fix:** Set `temperature=0.0` and add `response_format={"type": "json_object"}` to the judge call.

### Error 5: Pytest collects no tests
```
collected 0 items
```
**Cause:** The test file does not match pytest's naming convention.
**Fix:** Ensure the file is named `test_*.py` and functions are named `test_*`.
