## Setup

Before you start building, make sure your environment is ready.

### Requirements

- Python 3.10 or higher
- An Anthropic API key with access to Claude Sonnet 4
- 15-30 minutes of focused work

### Create a Project Directory

```bash
mkdir agent-loop-lab
cd agent-loop-lab
```

### Install Dependencies

```bash
pip install anthropic python-dotenv
```

### Set Up Your API Key

Create a `.env` file in the project directory:

```bash
echo "ANTHROPIC_API_KEY=your-api-key-here" > .env
```

Then create a Python file for your agent loop:

```bash
touch agent.py
```

Your directory structure should look like this:

```
agent-loop-lab/
  .env
  agent.py
```

### Verify Your Setup

Run this quick check to make sure the SDK is installed and your API key works:

```python
import os
from dotenv import load_dotenv
from anthropic import Anthropic

load_dotenv()

client = Anthropic(api_key=os.getenv("ANTHROPIC_API_KEY"))

response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=256,
    messages=[{"role": "user", "content": "say hello"}],
)

print(response.content[0].text)
```

If you see a greeting, your setup is working. If you see an error, check that
your API key is correct and that the `anthropic` package is installed.

## The Build -- Step by Step

You are going to build an agent loop from scratch using only the Anthropic
SDK. No frameworks, no abstractions -- just the raw loop. This is the same
pattern used by production agent systems, distilled to its essential form.

Your agent will have two tools:
- `get_weather(location: str)` -- returns the current weather for a city
- `get_time(city: str)` -- returns the current time for a city

Both tools are simulated (they return canned responses), which means you can
focus on the loop logic without needing live API integrations.

### Step 1: Define Your Tools

Start by defining the tool schemas that tell the model what tools are
available. Each tool needs a name, description, and input schema.

Open `agent.py` and add:

```python
import os
from dotenv import load_dotenv
from anthropic import Anthropic

load_dotenv()
client = Anthropic(api_key=os.getenv("ANTHROPIC_API_KEY"))

tools = [
    {
        "name": "get_weather",
        "description": "Get the current weather for a city",
        "input_schema": {
            "type": "object",
            "properties": {
                "location": {
                    "type": "string",
                    "description": "City name, e.g. Tokyo, Japan"
                }
            },
            "required": ["location"]
        }
    },
    {
        "name": "get_time",
        "description": "Get the current time for a city",
        "input_schema": {
            "type": "object",
            "properties": {
                "city": {
                    "type": "string",
                    "description": "City name, e.g. London"
                }
            },
            "required": ["city"]
        }
    }
]
```

### Step 2: Implement Tool Functions

Next, write the functions that execute when the model calls a tool. For this
lab, these return simulated data so you can test without external APIs.

Add below your tools definition:

```python
def execute_tool(tool_name, tool_args):
    """Run the requested tool and return the result as a string."""
    if tool_name == "get_weather":
        location = tool_args["location"]
        # Simulated weather data
        weather_data = {
            "Tokyo, Japan": "22 degrees, partly cloudy",
            "London": "15 degrees, light rain",
            "San Francisco": "18 degrees, foggy",
            "Sydney": "26 degrees, sunny",
        }
        # Default response for unknown locations
        result = weather_data.get(location, "20 degrees, clear sky")
        return f"Weather in {location}: {result}"

    elif tool_name == "get_time":
        city = tool_args["city"]
        # Simulated time data
        time_data = {
            "Tokyo": "14:30 JST",
            "London": "06:30 GMT",
            "San Francisco": "23:30 PST",
            "Sydney": "16:30 AEDT",
        }
        result = time_data.get(city, "12:00 local time")
        return f"Current time in {city}: {result}"

    else:
        return f"Error: unknown tool {tool_name}"
```

### Step 3: Build the Loop Core

Now build the loop itself. This is the control structure that sends messages
to the API, checks the response, and decides what to do next.

Add this below your tools and functions:

```python
def run_agent_loop(user_request, max_iterations=10):
    """Run the agent loop until the task is complete or max iterations reached."""
    messages = [{"role": "user", "content": user_request}]
    print(f"User: {user_request}")
    print(f"Max iterations: {max_iterations}")
    print("---")

    for iteration in range(1, max_iterations + 1):
        print(f"Iteration {iteration}")

        response = client.messages.create(
            model="claude-sonnet-4-20250514",
            max_tokens=4096,
            messages=messages,
            tools=tools,
        )

        input_tokens = response.usage.input_tokens
        output_tokens = response.usage.output_tokens
        print(f"  Input tokens: {input_tokens} | Output tokens: {output_tokens}")
        print(f"  Stop reason: {response.stop_reason}")

        if response.stop_reason == "end_turn":
            print(f"  Final response: {response.content[0].text}")
            print("Task complete.")
            return response.content[0].text

        elif response.stop_reason == "tool_use":
            for block in response.content:
                if block.type == "tool_use":
                    tool_name = block.name
                    tool_args = block.input
                    tool_id = block.id
                    print(f"  Tool call: {tool_name}({tool_args})")

                    tool_result = execute_tool(tool_name, tool_args)
                    print(f"  Tool result: {tool_result}")

                    messages.append({
                        "role": "assistant",
                        "content": [block],
                    })
                    messages.append({
                        "role": "user",
                        "content": [{
                            "type": "tool_result",
                            "tool_use_id": tool_id,
                            "content": tool_result,
                        }],
                    })

        elif response.stop_reason == "max_tokens":
            print("  Warning: Response hit max_tokens limit.")
            print(f"  Partial response: {response.content[0].text}")
            messages.append({
                "role": "assistant",
                "content": response.content,
            })
            continue

    print(f"Reached max iterations ({max_iterations}). Returning last response.")
    if response.content:
        return response.content[0].text
    return None


if __name__ == "__main__":
    result = run_agent_loop("What is the weather in Tokyo?")
```

This is the complete agent loop. Let us examine each part:

- The loop runs up to `max_iterations` times.
- Each iteration calls the API with the current messages and tools.
- It prints the token usage and stop reason for visibility.
- If `end_turn`, it returns the response and exits.
- If `tool_use`, it extracts the tool call, executes the function, appends
  the assistant message and tool result to the messages array, and continues.
- If `max_tokens`, it appends the partial response and continues.

### Step 6: Add Token Usage Tracking

Token usage tracking helps you understand the cost of each loop iteration and
detect runaway consumption early. Let us extend the loop to report cumulative
token usage.

Replace the `run_agent_loop` function with this enhanced version:

```python
def run_agent_loop(user_request, max_iterations=10, max_total_tokens=50000):
    """Run the agent loop with token usage tracking."""
    messages = [{"role": "user", "content": user_request}]
    total_input = 0
    total_output = 0

    print(f"User: {user_request}")
    print(f"Max iterations: {max_iterations}")
    print("---")

    for iteration in range(1, max_iterations + 1):
        print(f"Iteration {iteration}")

        response = client.messages.create(
            model="claude-sonnet-4-20250514",
            max_tokens=4096,
            messages=messages,
            tools=tools,
        )

        input_tokens = response.usage.input_tokens
        output_tokens = response.usage.output_tokens
        total_input += input_tokens
        total_output += output_tokens
        total = total_input + total_output

        print(f"  This call: {input_tokens} in / {output_tokens} out")
        print(f"  Cumulative: {total_input} in / {total_output} out ({total} total)")
        print(f"  Stop reason: {response.stop_reason}")

        if total > max_total_tokens:
            print(f"  Token budget exceeded ({total} > {max_total_tokens}). Stopping.")
            break

        if response.stop_reason == "end_turn":
            print(f"  Final response: {response.content[0].text}")
            print(f"Task complete after {iteration} iterations.")
            print(f"Total tokens: {total}")
            return response.content[0].text

        elif response.stop_reason == "tool_use":
            for block in response.content:
                if block.type == "tool_use":
                    tool_name = block.name
                    tool_args = block.input
                    tool_id = block.id
                    print(f"  Tool call: {tool_name}({tool_args})")

                    tool_result = execute_tool(tool_name, tool_args)
                    print(f"  Tool result: {tool_result}")

                    messages.append({
                        "role": "assistant",
                        "content": [block],
                    })
                    messages.append({
                        "role": "user",
                        "content": [{
                            "type": "tool_result",
                            "tool_use_id": tool_id,
                            "content": tool_result,
                        }],
                    })

        elif response.stop_reason == "max_tokens":
            print("  Warning: Response hit max_tokens limit.")
            messages.append({
                "role": "assistant",
                "content": response.content,
            })
            continue

    print(f"Reached max iterations ({max_iterations}). Total tokens: {total_input + total_output}")
    if response.content:
        return response.content[0].text
    return None
```

The enhanced version tracks cumulative input and output tokens across all
iterations and stops the loop if the total exceeds the budget. This is the
same pattern used in production systems to prevent cost blow-ups.

## Common Errors

Building an agent loop exposes you to several pitfalls. Here are the most
common errors and how to fix them.

### Error 1: Infinite Loop

You forget to set a maximum iteration limit, and the agent keeps calling
tools forever. The API bills you for every call. You notice 45 minutes later
when the cost alert email arrives.

**Fix**: Always set `max_iterations` before the loop starts. Use a low number
(5-10) during development. Add a token budget check as a second safety net.

```python
MAX_ITERATIONS = 10
for iteration in range(MAX_ITERATIONS):
    # agent logic here
    pass
```

**How to detect during development**: Print the iteration count on every pass.
If you see iteration 20 and the loop is still running, your termination logic
is broken.

### Error 2: Context Overflow

The loop runs fine for 5 iterations, then on iteration 6 the API returns a
400 error: "This model's maximum context length is 200000 tokens." The
messages array grew past the context window.

**Fix**: Monitor the cumulative token count and implement a context trimming
strategy before you hit the limit. Even for short loops, track input tokens
so you know how close you are to the boundary.

```python
# Add a safety check after each API call
total_tokens = response.usage.input_tokens + response.usage.output_tokens
if total_tokens > 150000:  # Leave 50k headroom
    print("Context window nearing limit. Trimming history.")
    messages = trim_context(messages)
```

### Error 3: Forgetting to Append Tool Results

You extract the tool call, execute the function, get the result, and ...
forget to append it to the messages array. The next API call does not include
the tool result. The model calls the same tool again with the same arguments.
An infinite loop of identical tool calls, each costing API tokens.

**Fix**: Always append both the assistant's tool_use block and the
tool_result block to messages. They must be in the correct order: assistant
message first, then user message with tool_result content.

```python
# Correct ordering and structure
messages.append({
    "role": "assistant",
    "content": [tool_use_block],
})
messages.append({
    "role": "user",
    "content": [{
        "type": "tool_result",
        "tool_use_id": tool_use_block.id,
        "content": result_string,
    }],
})
```

### Error 4: Wrong stop_reason Handling

You write an `if/elif` chain that handles `end_turn` and `tool_use` but
forgets `max_tokens`. When the model hits the token limit, none of your
branches match. The response is silently discarded. The loop calls the API
again with the same messages, hitting `max_tokens` again. Another billable
infinite loop.

**Fix**: Explicitly handle all three stop reasons. Even if your handling of
`max_tokens` is just to log a warning and break, include the branch. Never
leave it unhandled.

```python
if response.stop_reason == "end_turn":
    # handle completion
    break
elif response.stop_reason == "tool_use":
    # handle tool call
    pass
elif response.stop_reason == "max_tokens":
    print(f"WARNING: Response truncated at max_tokens.")
    # Option 1: continue with partial response
    # Option 2: break and return partial
    break
else:
    print(f"Unknown stop_reason: {response.stop_reason}")
    break
```

### Error 5: Not Including Tool Args With Tool Result

You append the tool result to messages, but the next API call fails because
the model cannot see which arguments it used for the previous call. The
context is missing the assistant message containing the tool_use call.

**Fix**: The messages array must contain both the assistant's `tool_use`
block (with the original call details including name and input) and the
`tool_result` block. The API uses the `tool_use_id` in the tool_result to
match it to the original call, but the model needs the full assistant message
to understand what it did.

```python
# BAD: only tool_result, no assistant message
messages.append({
    "role": "user",
    "content": [{"type": "tool_result", "tool_use_id": block.id, "content": result}],
})

# GOOD: assistant message with tool_use, then user message with tool_result
messages.append({
    "role": "assistant",
    "content": [block],  # block is the full tool_use content block
})
messages.append({
    "role": "user",
    "content": [{"type": "tool_result", "tool_use_id": block.id, "content": result}],
})
```

## What You Just Built

You built a working agent loop from scratch. Here is what you accomplished:

1. **You implemented the Observe-Think-Act cycle**: Your loop sends the
   conversation history to the LLM (observe), the LLM decides on an action
   (think), and your code executes the tool and captures the result (act).

2. **You handled all stop reasons**: Your loop correctly branches on
   `end_turn`, `tool_use`, and `max_tokens`, and takes appropriate action
   for each one.

3. **You constructed the correct messages format**: You appended `tool_use`
   blocks as assistant messages and `tool_result` blocks as user messages,
   maintaining the conversation structure the API requires.

4. **You added safety mechanisms**: Maximum iteration count and cumulative
   token budget prevent runaway loops from burning through your API budget.

5. **You tracked token usage**: Every iteration reports input and output
   tokens, giving you visibility into the cost structure of each agent run.

6. **You built a production-ready pattern without a framework**: The code you
   wrote is the same pattern used by agent frameworks under the hood. You now
   understand what those frameworks abstract away, and you can debug or
   customize any agent system built on the loop pattern.

This pattern -- a while loop with a messages array, API calls, stop_reason
checks, and tool execution -- is the foundation of every agent system you
will encounter. Every framework, every platform, every production agent runs
some variation of this loop. You built it yourself, and now you understand
it.
