In this exercise, you will build a basic AI agent from scratch using the Anthropic API. Your agent will be able to answer questions and perform simple actions — checking the current time and reading file contents — by deciding when to call tools and when to respond directly.

---

## Setup

### Requirements

- Python 3.10 or later
- An Anthropic API key (get one at https://console.anthropic.com)
- Internet access for API calls

### Installation

Create a new directory for this project and set up a virtual environment:

```bash
mkdir my-first-agent
cd my-first-agent
python -m venv venv
source venv/bin/activate   # On Windows: venv\Scripts\activate
```

Install the dependencies:

```bash
pip install anthropic python-dotenv
```

### Environment Variable

Create a file named `.env` in your project directory:

```bash
echo "ANTHROPIC_API_KEY=your-api-key-here" > .env
```

Replace `your-api-key-here` with your actual Anthropic API key. Never commit this file to version control. Add `.env` to your `.gitignore`:

```bash
echo ".env" >> .gitignore
```

## The Build — Step by Step

You will build the agent incrementally. Each step adds one piece of functionality. Run the code at each step and verify the output before moving on.

---

### Step 1: Basic Client Setup

Create a file named `agent.py` with the minimal setup:

```python
import os
from anthropic import Anthropic

client = Anthropic()

response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    messages=[
        {"role": "user", "content": "What is the capital of Japan?"}
    ]
)

print(response.content[0].text)
```

Run it:

```bash
python agent.py
```

**Expected output:**
```
The capital of Japan is Tokyo.
```

**Explanation:** The `Anthropic()` client reads `ANTHROPIC_API_KEY` from the environment automatically. The `messages.create()` call sends a single user message and returns a response containing the LLM's answer. This is a simple completion — no agent behavior yet, but it verifies your setup.

**Checkpoint:** If you see the expected output, your API key and environment are working correctly.

### Step 2: Define a Tool

Now define your first tool — a function that returns the current time. Add these lines after the `client` initialization:

```python
import datetime

TOOLS = [
    {
        "name": "get_current_time",
        "description": "Get the current date and time",
        "input_schema": {
            "type": "object",
            "properties": {
                "timezone": {
                    "type": "string",
                    "description": "The timezone (e.g., UTC, America/New_York, Asia/Tokyo)"
                }
            },
            "required": ["timezone"]
        }
    }
]
```

This is a **tool definition**. It tells the LLM: there is a function called `get_current_time` that accepts a `timezone` parameter. The `description` field is critical — the LLM uses it to decide when to call this tool. The `input_schema` defines the parameters the tool expects, in JSON Schema format.

### Step 4: Execute the Tool and Return the Result

Now connect the tool call to actual code. Replace everything after the `TOOLS` definition with:

```python
def execute_tool(name, input_data):
    """Execute a tool by name with the given input."""
    if name == "get_current_time":
        tz = input_data.get("timezone", "UTC")
        now = datetime.datetime.now(datetime.timezone.utc)
        return f"Current time in {tz}: {now.strftime('%Y-%m-%d %H:%M:%S')} UTC"
    raise ValueError(f"Unknown tool: {name}")

def agent_loop(user_message):
    """Run one iteration of the agent loop."""
    messages = [{"role": "user", "content": user_message}]

    response = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=1024,
        tools=TOOLS,
        messages=messages
    )

    # Collect all content from the response
    response_content = []
    for block in response.content:
        response_content.append(block)

    # If the LLM wants to call a tool, execute it and return the result
    if response.stop_reason == "tool_use":
        tool_block = None
        for block in response.content:
            if block.type == "tool_use":
                tool_block = block
                break

        if tool_block:
            result = execute_tool(tool_block.name, tool_block.input)
            # Show what the tool returned
            return f"Tool called: {tool_block.name}\nInput: {tool_block.input}\nResult: {result}"

    return response.content[0].text

# Test it
result = agent_loop("What time is it in Tokyo right now?")
print(result)
```

Run it:

```bash
python agent.py
```

**Expected output (similar):**
```
Tool called: get_current_time
Input: {'timezone': 'Asia/Tokyo'}
Result: Current time in Asia/Tokyo: 2026-07-04 14:30:00 UTC
```

**Explanation:** The `agent_loop` function sends the user message with the tools definition, checks if the response contains a `tool_use` block, and if so, executes the corresponding function. The tool executes and returns a result string.

**Checkpoint:** Your agent can now perceive a user request, decide to call a tool, and execute it.

### Step 5: Complete the Loop — Send Tool Results Back to the LLM

The agent is not done yet. After executing the tool, the result must go back to the LLM so it can formulate a final answer. Replace the `agent_loop` function with this version:

```python
def agent_loop(user_message, max_iterations=5):
    """Run the full agent loop with tool result feedback."""
    messages = [{"role": "user", "content": user_message}]
    iteration = 0

    while iteration < max_iterations:
        iteration += 1
        print(f"\n--- Iteration {iteration} ---")

        response = client.messages.create(
            model="claude-sonnet-4-20250514",
            max_tokens=1024,
            tools=TOOLS,
            messages=messages
        )

        if response.stop_reason == "tool_use":
            # Find the tool_use block
            for block in response.content:
                if block.type == "tool_use":
                    # Execute the tool
                    print(f"Calling tool: {block.name}")
                    tool_result = execute_tool(block.name, block.input)
                    print(f"Tool result: {tool_result}")

                    # Append assistant response to conversation
                    messages.append({
                        "role": "assistant",
                        "content": response.content
                    })

                    # Append tool result as a user message
                    messages.append({
                        "role": "user",
                        "content": [
                            {
                                "type": "tool_result",
                                "tool_use_id": block.id,
                                "content": tool_result
                            }
                        ]
                    })
        elif response.stop_reason == "end_turn":
            print("Agent finished.")
            return response.content[0].text
        else:
            print(f"Stopped for reason: {response.stop_reason}")
            return response.content[0].text

    return "Max iterations reached without completion."

# Test it
result = agent_loop("What time is it in Tokyo right now?")
print(f"\nFinal answer:\n{result}")
```

Run it:

```bash
python agent.py
```

**Expected output (similar):**
```
--- Iteration 1 ---
Calling tool: get_current_time
Tool result: Current time in Asia/Tokyo: 2026-07-04 14:30:00 UTC
Agent finished.

Final answer:
The current time in Tokyo (Asia/Tokyo timezone) is July 4, 2026 at 14:30 UTC, which is 23:30 JST (Japan Standard Time).
```

**Explanation:** This is the complete agent loop. The agent calls the tool, gets the result, sends it back to the LLM, and the LLM produces a final answer in natural language.

**The key insight:** The LLM does not execute the tool. Your code does. The LLM only requests the tool call. This separation is what makes agents safe and controllable — you control what functions are available and you verify the input before executing them.

**Checkpoint:** Your agent can perform the full perceive → think → act → perceive cycle.

### Complete Agent Code

Here is the full, final version of `agent.py`:

```python
import os
import datetime
from anthropic import Anthropic

client = Anthropic()

TOOLS = [
    {
        "name": "get_current_time",
        "description": "Get the current date and time in a given timezone",
        "input_schema": {
            "type": "object",
            "properties": {
                "timezone": {
                    "type": "string",
                    "description": "The timezone (e.g., UTC, America/New_York, Asia/Tokyo)"
                }
            },
            "required": ["timezone"]
        }
    },
    {
        "name": "read_file",
        "description": "Read the contents of a text file",
        "input_schema": {
            "type": "object",
            "properties": {
                "filepath": {
                    "type": "string",
                    "description": "The path to the file to read"
                }
            },
            "required": ["filepath"]
        }
    }
]

def execute_tool(name, input_data):
    if name == "get_current_time":
        tz = input_data.get("timezone", "UTC")
        now = datetime.datetime.now(datetime.timezone.utc)
        return f"Current time in {tz}: {now.strftime('%Y-%m-%d %H:%M:%S')} UTC"
    elif name == "read_file":
        filepath = input_data.get("filepath", "")
        try:
            with open(filepath, "r") as f:
                return f.read()
        except FileNotFoundError:
            return f"Error: File not found at {filepath}"
        except Exception as e:
            return f"Error reading file: {str(e)}"
    raise ValueError(f"Unknown tool: {name}")

def agent_loop(user_message, max_iterations=10):
    messages = [{"role": "user", "content": user_message}]

    for iteration in range(1, max_iterations + 1):
        response = client.messages.create(
            model="claude-sonnet-4-20250514",
            max_tokens=1024,
            tools=TOOLS,
            messages=messages
        )

        if response.stop_reason == "tool_use":
            for block in response.content:
                if block.type == "tool_use":
                    tool_result = execute_tool(block.name, block.input)
                    messages.append({
                        "role": "assistant",
                        "content": response.content
                    })
                    messages.append({
                        "role": "user",
                        "content": [
                            {
                                "type": "tool_result",
                                "tool_use_id": block.id,
                                "content": tool_result
                            }
                        ]
                    })
        elif response.stop_reason == "end_turn":
            return response.content[0].text
        else:
            return response.content[0].text

    return "Max iterations reached without completion."

if __name__ == "__main__":
    result = agent_loop("What time is it in Tokyo?")
    print(result)
```

## Common Errors

### Error 1: Missing API Key

```
anthropic.APIConnectionError: Connection error.
```

This usually means `ANTHROPIC_API_KEY` is not set. The SDK silently falls back to an empty string and the API rejects the connection.

**Fix:** Verify the environment variable is set:

```bash
echo $ANTHROPIC_API_KEY
# Should print your key, not a blank line
```

Ensure your `.env` file is in the current directory and loaded if you use `python-dotenv`:

```python
from dotenv import load_dotenv
load_dotenv()
```

Or export it directly:

```bash
export ANTHROPIC_API_KEY="sk-ant-..."
```

### Error 2: Wrong Model String

```
anthropic.BadRequestError: {"type":"error","error":{"type":"invalid_request_error","message":"model: value is not a valid model"}}
```

**Fix:** Verify the model name against the current Anthropic documentation. As of this writing, the correct strings are:

- `claude-sonnet-4-20250514` — the recommended model for agent tasks
- `claude-3-5-sonnet-latest` — an alternative alias

Typos like `claude-sonnet-4` (missing the date) or `claude-4-sonnet` (wrong order) will fail.

### Error 3: Tool Definition Schema Errors

```
anthropic.BadRequestError: {"type":"error","error":{"type":"invalid_request_error","message":"tools.0.input_schema.properties.timezone.type: value is not a valid enum"}}
```

This happens when your JSON Schema is invalid. Common mistakes:

- Using `"type": "string"` instead of `"type": "string"` (wrong case — JSON Schema types are lowercase).
- Missing `"required"` array.
- Nesting properties incorrectly.

**Fix:** Validate your tool definitions against the JSON Schema specification. Each tool must have `name`, `description`, and `input_schema` fields. The `input_schema` must have `type: "object"`, a `properties` object, and a `required` array.

### Error 4: Max Retries / Rate Limit Exceeded

```
anthropic.RateLimitError: {"type":"error","error":{"type":"rate_limit_error","message":"This request would exceed your rate limit"}}
```

The agent loop can trigger rate limits quickly because each iteration is a separate API call.

**Fix:** Add retry logic with exponential backoff:

```python
import time

def api_call_with_retry(messages, tools, max_retries=3):
    for attempt in range(max_retries):
        try:
            return client.messages.create(
                model="claude-sonnet-4-20250514",
                max_tokens=1024,
                tools=tools,
                messages=messages
            )
        except anthropic.RateLimitError:
            if attempt < max_retries - 1:
                wait = 2 ** attempt
                print(f"Rate limited. Retrying in {wait}s...")
                time.sleep(wait)
            else:
                raise
```

### Error 5: Tool Result Wrong Format

If the `tool_result` content block is malformed, the API returns:

```
anthropic.BadRequestError: ...
```

**Fix:** Ensure each `tool_result` block has exactly these fields:

```python
{
    "type": "tool_result",
    "tool_use_id": block.id,  # Must match the tool_use id exactly
    "content": tool_result    # Must be a string
}
```

The `tool_use_id` must match the `id` from the `tool_use` block exactly. The `content` must be a string (for simple text results) or a list of content blocks (for structured results).

## What You Just Built

- **A functional agent loop**: Your code sends a user message to the LLM, checks if a tool was requested, executes the tool, feeds the result back, and loops until the LLM decides the task is complete.
- **Tool definition and execution**: You defined tools as JSON schemas and connected them to real Python functions. The LLM never executes code — it only requests actions, and your code runs them.
- **Conversation state management**: You built the message history by appending assistant responses and tool results, maintaining the context across iterations.
- **Safety controls**: You implemented a `max_iterations` limit to prevent runaway loops — a critical safety pattern for production agents.
- **Multi-tool routing**: Your agent can choose between multiple tools (`get_current_time` and `read_file`) based on the user's request, demonstrating the flexibility of the agent pattern.
- **Error handling awareness**: You learned the most common failure modes and how to handle them — missing API keys, invalid schemas, rate limits, and malformed tool results.
