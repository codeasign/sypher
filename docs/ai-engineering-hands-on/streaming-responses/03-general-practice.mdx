## Tier 1 — Beginner

### Exercise 1: First Stream

**Task:** Write a Python script that sends the message "Tell me a fun fact about space" to GPT-4o-mini with `stream=True` and prints each token as it arrives.

**Hint:** Use `stream=True` and iterate over the response. Print with `end=""` and `flush=True`.

### Solution

```python
import os
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

stream = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Tell me a fun fact about space."}],
    max_tokens=200,
    stream=True
)

for chunk in stream:
    if chunk.choices[0].delta.content is not None:
        print(chunk.choices[0].delta.content, end="", flush=True)
```

This is the simplest possible streaming call. Each chunk contains one or a few tokens that you display immediately.

### Exercise 2: Word-by-Word Counter

**Task:** Stream a response and count how many chunks the API sends before the stream completes. Print the total.

**Hint:** Increment a counter inside the loop. Print the count after the stream ends.

### Solution

```python
import os
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

stream = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Count to 20."}],
    max_tokens=200,
    stream=True
)

chunk_count = 0
for chunk in stream:
    if chunk.choices[0].delta.content is not None:
        chunk_count += 1
        print(chunk.choices[0].delta.content, end="", flush=True)

print(f"\n\nTotal chunks received: {chunk_count}")
```

A single word like "Count" may arrive as one chunk, or be split across multiple. The chunk count reflects how the API chooses to batch tokens — it varies by provider and model.

### Exercise 4: Detect the First Token

**Task:** Print "Receiving response..." before the first token arrives, then "Stream started!" when the first token appears, then stream the rest normally.

**Hint:** Use a boolean flag that flips when `delta.content` is not None for the first time.

### Solution


## Tier 2 — Intermediate

### Exercise 1: Streaming with Early Termination

**Task:** Stream a response about "a long topic" but stop consuming the stream if the response contains the word "Python" — you only need the first relevant sentence.

**Hint:** Accumulate tokens and check after each chunk. Break out of the loop when your condition is met.

### Solution

```python
import os
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

stream = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Explain the history of programming languages from FORTRAN to Rust."}],
    max_tokens=500,
    stream=True
)

collected = []
stop_word = "Python"

for chunk in stream:
    if chunk.choices[0].delta.content is not None:
        content = chunk.choices[0].delta.content
        print(content, end="", flush=True)
        collected.append(content)
        full_so_far = "".join(collected)
        if stop_word in full_so_far:
            print(f"\n\n[Stopped early — found '{stop_word}' in response]")
            break

print(f"\nAccumulated {len(''.join(collected))} chars before stopping.")
```

Early termination can save costs on long generations when you only need enough context to make a decision.

### Exercise 2: Streaming Progress Indicator

**Task:** Create a stream that shows a spinner animation while waiting for the first token, then switches to a typewriter effect once streaming begins.

**Hint:** Use `sys.stdout.write("\\rLoading.")` with `time.sleep(0.2)` in a loop, then break when the first token arrives.

### Solution

```python
import os
import sys
import time
import threading
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])
streaming_started = threading.Event()

def show_spinner():
    chars = "|/-\\"
    i = 0
    while not streaming_started.is_set():
        sys.stdout.write(f"\rWaiting for response {chars[i % 4]}")
        sys.stdout.flush()
        time.sleep(0.1)
        i += 1
    sys.stdout.write("\r" + " " * 30 + "\r")

spinner = threading.Thread(target=show_spinner, daemon=True)
spinner.start()

stream = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Write a short poem about Python."}],
    max_tokens=200,
    stream=True
)

for chunk in stream:
    if chunk.choices[0].delta.content is not None:
        if not streaming_started.is_set():
            streaming_started.set()
        print(chunk.choices[0].delta.content, end="", flush=True)

print()
```

This demonstrates the "Loading..." → "Typing..." pattern used in production chat interfaces.

### Exercise 3: Multi-Stream Merger

**Task:** Send the same question to GPT-4o-mini and GPT-4o simultaneously (two separate streams) and display whichever responds first as soon as its first token arrives.

**Hint:** Use `concurrent.futures.ThreadPoolExecutor` to start both streams, then process results.

### Solution

```python
import os
import concurrent.futures
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

def stream_response(model):
    collected = []
    for chunk in client.chat.completions.create(
        model=model,
        messages=[{"role": "user", "content": "Explain what a qubit is in 100 words."}],
        max_tokens=200,
        stream=True
    ):
        if chunk.choices[0].delta.content is not None:
            collected.append(chunk.choices[0].delta.content)
    return model, "".join(collected)

with concurrent.futures.ThreadPoolExecutor(max_workers=2) as executor:
    future_mini = executor.submit(stream_response, "gpt-4o-mini")
    future_4o = executor.submit(stream_response, "gpt-4o")

    for future in concurrent.futures.as_completed([future_mini, future_4o]):
        model, response = future.result()
        print(f"\n=== {model} responded ===")
        print(response[:200] + "...\n")
```

## Tier 3 — Advanced

### Exercise 1: Streaming Proxy Server

**Task:** Build a simple Flask/FastAPI endpoint that accepts a user message, streams it from the OpenAI API, and returns a streaming HTTP response to the caller. The proxy should accumulate the full response for logging.

**Hint:** Use `StreamingResponse` from FastAPI or `Response(stream_with_context(...))` from Flask. The endpoint reads from the OpenAI stream and yields each chunk.

### Solution

```python
import os
from fastapi import FastAPI
from fastapi.responses import StreamingResponse
from openai import OpenAI

app = FastAPI()
client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

@app.get("/chat/{message}")
async def chat(message: str):
    def generate():
        stream = client.chat.completions.create(
            model="gpt-4o-mini",
            messages=[{"role": "user", "content": message}],
            max_tokens=300,
            stream=True
        )
        for chunk in stream:
            if chunk.choices[0].delta.content is not None:
                yield chunk.choices[0].delta.content

    return StreamingResponse(generate(), media_type="text/plain")
```

Run with `uvicorn filename:app --reload`, then visit `http://localhost:8000/chat/hello` to see streaming tokens in your browser.

### Exercise 2: Adaptive Stream Rate Limiter

**Task:** Create a wrapper around the streaming API that limits tokens-per-second displayed to the user, smoothing out the bursty nature of streaming. Characters should appear at a steady 30 characters per second regardless of how fast the server generates them.

**Hint:** Use `time.sleep()` based on the length of each chunk divided by the target rate: `sleep_time = len(content) / target_cps`.

### Solution

```python
import os
import time
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

TARGET_CHARS_PER_SECOND = 30

stream = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Tell me a story about a robot learning to paint."}],
    max_tokens=500,
    stream=True
)

for chunk in stream:
    if chunk.choices[0].delta.content is not None:
        content = chunk.choices[0].delta.content
        print(content, end="", flush=True)
        # Throttle to target rate
        sleep_time = len(content) / TARGET_CHARS_PER_SECOND
        time.sleep(sleep_time)
```

This creates a smooth reading experience that does not overwhelm the user with sudden bursts of text.

### Exercise 3: Streaming Chat with Token Counter and Rate Monitor

**Task:** Extend the streaming chat loop to track tokens per second, estimated time remaining, and accumulated cost in real-time. Display a status line that updates after each chunk: `[3.2s | 45 tokens | 14 tok/s | $0.00002]`

**Hint:** Track elapsed time per chunk and compute `len(content) / chunk_time` for instantaneous rate. Accumulate costs using the model's per-token pricing.

### Solution

```python
import os
import time
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])

messages = [{"role": "user", "content": "Explain the concept of recursion in programming, with examples."}]

start = time.time()
total_tokens = 0
INPUT_COST = 0.15 / 1_000_000
OUTPUT_COST = 0.60 / 1_000_000

stream = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=messages,
    max_tokens=500,
    stream=True,
    stream_options={"include_usage": True}
)

collected = []
for chunk in stream:
    if chunk.choices[0].delta.content is not None:
        content = chunk.choices[0].delta.content
        elapsed = time.time() - start
        print(content, end="", flush=True)
        collected.append(content)

    if chunk.usage:
        total_tokens = chunk.usage.total_tokens
        cost = (chunk.usage.prompt_tokens * INPUT_COST +
                chunk.usage.completion_tokens * OUTPUT_COST)
        total_time = time.time() - start
        print(f"\n\n--- Status ---")
        print(f"Time: {total_time:.1f}s")
        print(f"Total tokens: {total_tokens}")
        print(f"Avg rate: {total_tokens / total_time:.0f} tok/s")
        print(f"Estimated cost: ${cost:.6f}")
```

This is the closest exercise to a production monitoring setup — tracking every metric that matters when you pay per token.
