## Tier 1 — Beginner

### Exercise 1.1: Characterize temperature sensitivity

**Task:** Pick a single prompt (e.g., "Write a product description for a wireless keyboard"). Generate 20 samples at each of these temperatures: 0.0, 0.2, 0.5, 0.8, 1.0, 1.5. For each temperature, compute the mean score, standard deviation, and unique response count. Plot the relationship between temperature and variance.

**Hint:** Use the `sampler.py` and `scorer.py` from the practice exercise. For the plot, use matplotlib.

### Solution

```python
import matplotlib.pyplot as plt
from sampler import sample_responses
from scorer import score_responses
from stats_harness import analyze_scores

prompt = "Write a two-sentence product description for a wireless keyboard."
rubric = "Is the description clear, appealing, and professional?"

temperatures = [0.0, 0.2, 0.5, 0.8, 1.0, 1.5]
results = []

for temp in temperatures:
    print(f"Testing temperature {temp}...")
    responses = sample_responses(
        prompt, n=20, temperature=temp
    )
    unique = len(set(responses))
    scores = score_responses(
        prompt, responses, rubric=rubric
    )
    stats = analyze_scores(scores)
    stats["temperature"] = temp
    stats["unique"] = unique
    results.append(stats)

    print(f"  Mean: {stats['mean']:.3f}, "
          f"Std: {stats['std']:.3f}, "
          f"Unique: {unique}/{len(responses)}")

# Plot
fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(12, 4))

temps = [r["temperature"] for r in results]
means = [r["mean"] for r in results]
stds = [r["std"] for r in results]
uniques = [r["unique"] for r in results]

ax1.plot(temps, means, "b-o", label="Mean score")
ax1.fill_between(
    temps,
    [m - s for m, s in zip(means, stds)],
    [m + s for m, s in zip(means, stds)],
    alpha=0.2,
    label="±1 std",
)
ax1.set_xlabel("Temperature")
ax1.set_ylabel("Score")
ax1.set_title("Score vs Temperature")
ax1.legend()
ax1.grid(True, alpha=0.3)

ax2.plot(temps, uniques, "r-o")
ax2.set_xlabel("Temperature")
ax2.set_ylabel("Unique responses (out of 20)")
ax2.set_title("Diversity vs Temperature")
ax2.grid(True, alpha=0.3)

plt.tight_layout()
plt.savefig("temperature_sensitivity.png")
print("\nPlot saved to temperature_sensitivity.png")
```

You should see the mean score decrease and the variance increase as temperature rises. The unique count should jump from 1 at temperature 0.0 to near 20 at temperature 1.0+.

### Exercise 1.2: Minimum sample size finder

**Task:** For a given prompt and temperature, find the minimum number of samples needed to estimate the mean score within ±0.05. Start with 5 samples, add 5 more, and check whether the running mean has stabilized within the tolerance.

**Hint:** Use a running mean. When the running mean changes by less than 0.05 after adding 5 more samples, consider it stable.

### Solution

```python
from sampler import sample_responses
from scorer import score_responses

prompt = "What is the boiling point of water?"
rubric = "Must be scientifically accurate."

all_responses = []
all_scores = []

for batch in range(6):  # Up to 30 samples (6 batches of 5)
    responses = sample_responses(
        prompt, n=5, temperature=0.7
    )
    scores = score_responses(
        prompt, responses, rubric=rubric
    )

    all_responses.extend(responses)
    all_scores.extend(scores)

    n = len(all_scores)
    mean = sum(all_scores) / n
    prev_mean = (
        sum(all_scores[:n-5]) / (n-5) if n > 5 else 0
    )
    change = abs(mean - prev_mean) if n > 5 else float("inf")

    print(f"n={n}: mean={mean:.4f}, "
          f"change={change:.4f}")

    if n >= 10 and change < 0.05:
        print(f"  → Stable at n={n} samples")
        break
```

### Exercise 1.3: Outlier detection in sampled responses

**Task:** Generate 20 responses to a prompt. Score them. Identify the bottom 10% of responses (lowest 2 scores). Read these responses — what do they have in common? Are they genuinely bad, or is the judge being unfair?

**Hint:** Use `np.percentile(scores, 10)` to find the threshold. Then print the responses with scores below that threshold.

### Solution

```python
import numpy as np
from sampler import sample_responses
from scorer import score_responses

prompt = "Explain how encryption works."
rubric = "Is the explanation clear, accurate, and accessible?"

responses = sample_responses(prompt, n=20, temperature=0.7)
scores = score_responses(prompt, responses, rubric=rubric)

threshold = np.percentile(scores, 10)
print(f"Bottom 10% threshold: {threshold:.3f}")
print(f"\nOutlier responses (score < {threshold:.3f}):")

for i, (response, score) in enumerate(zip(responses, scores)):
    if score < threshold:
        print(f"\n--- Response {i} (score={score:.3f}) ---")
        print(response[:200])
```

Do the low-scoring responses share a pattern? Are they too short? Too technical? Do they contain errors? This analysis helps you understand what your system's failure modes look like.

## Tier 2 — Intermediate

### Exercise 2.1: Prompt stability comparison

**Task:** You have two candidate prompts for the same task. Which one produces more stable outputs? Run 15 samples for each prompt, score them, and compare the distributions. Which prompt has the higher mean? Which has the lower variance? Is the difference in mean statistically significant?

**Hint:** Use `compare_versions()` from `stats_harness.py`.

### Solution

```python
from sampler import sample_responses
from scorer import score_responses
from stats_harness import compare_versions

prompt_a = "Summarize this text in one sentence."
prompt_b = "You are an expert summarizer. Read the following text and produce a concise one-sentence summary that captures the main point."

source_text = """
The company reported quarterly earnings that exceeded analyst expectations.
Revenue grew 15% year-over-year to $2.4 billion, driven by strong growth
in the cloud computing division. Operating income rose 22% to $680 million.
The company also announced a $5 billion share buyback program and raised
its full-year guidance.
"""

rubric = "Is the summary faithful to the source and written in one sentence?"

for label, prompt in [("A: concise", prompt_a), ("B: detailed", prompt_b)]:
    print(f"\nPrompt {label}")
    full_prompt = f"{prompt}\n\n{source_text}"
    responses = sample_responses(full_prompt, n=15, temperature=0.5)
    scores = score_responses(full_prompt, responses, rubric=rubric)
    mean = sum(scores) / len(scores)
    std = (sum((s - mean)**2 for s in scores) / len(scores)) ** 0.5
    print(f"  Mean: {mean:.3f}, Std: {std:.3f}")

# Compare using statistical tests
print("\n--- Statistical Comparison ---")
full_prompt_a = f"{prompt_a}\n\n{source_text}"
full_prompt_b = f"{prompt_b}\n\n{source_text}"

responses_a = sample_responses(full_prompt_a, n=15, temperature=0.5)
scores_a = score_responses(full_prompt_a, responses_a, rubric=rubric)

responses_b = sample_responses(full_prompt_b, n=15, temperature=0.5)
scores_b = score_responses(full_prompt_b, responses_b, rubric=rubric)

result = compare_versions(scores_a, scores_b)

print(f"Difference: {result['mean_difference']:+.3f}")
print(f"P-value: {result['t_test']['p_value']:.4f}")
print(f"Significant: {result['t_test']['significant']}")
```

### Exercise 2.2: Multi-dimensional stability profile

**Task:** Create a stability profile for a single prompt across 4 dimensions: factuality, relevance, conciseness, and tone. For each dimension, run 15 samples and compute the mean and std. Which dimension is the most stable? Which is the least stable? This tells you where your system is most vulnerable to non-determinism.

**Hint:** Create a rubric for each dimension and call `score_responses()` separately for each.

### Solution

```python
from sampler import sample_responses
from scorer import score_responses

prompt = "Explain the difference between HTTP and HTTPS."
rubrics = {
    "factuality": "Are all technical claims accurate?",
    "relevance": "Does the response directly address HTTP vs HTTPS?",
    "conciseness": "Is the response appropriately concise?",
    "tone": "Is the tone professional and clear?",
}

responses = sample_responses(prompt, n=15, temperature=0.7)

print(f"{'Dimension':<15} {'Mean':<8} {'Std':<8} {'Stability':<10}")
print("-" * 45)

for dim, rubric in rubrics.items():
    scores = score_responses(
        prompt, responses, dimension=dim, rubric=rubric
    )
    mean = sum(scores) / len(scores)
    std = (sum((s - mean)**2 for s in scores) / len(scores)) ** 0.5
    stability = "High" if std < 0.05 else "Medium" if std < 0.1 else "Low"
    print(f"{dim:<15} {mean:<8.3f} {std:<8.3f} {stability:<10}")
```

### Exercise 2.3: Evaluating the evaluator

**Task:** The judge model itself is non-deterministic. Quantify the judge's variability by running the same judge call 10 times (with temperature 0.0) on the same pair of (prompt, response). What is the mean and standard deviation of the judge's scores? This is your measurement noise floor.

**Hint:** Call `score_response()` in a loop with the same prompt and response, but with a new judge instance each time.

### Solution

```python
from scorer import score_response

prompt = "What is the capital of France?"
response = "The capital of France is Paris."

rubric = "Is the response factually correct?"

judge_scores = []
for i in range(10):
    score = score_response(
        prompt, response, rubric=rubric
    )
    judge_scores.append(score)
    print(f"  Judge run {i+1}: {score:.3f}")

mean = sum(judge_scores) / len(judge_scores)
std = (sum((s - mean)**2 for s in judge_scores) / len(judge_scores)) ** 0.5

print(f"\nJudge noise: mean={mean:.3f}, std={std:.3f}")
print(f"Measurement noise floor: ±{std:.3f}")

# Any score difference below this threshold is indistinguishable from noise
print(f"Minimum detectable effect: {2*std:.3f} "
      f"(at 95% confidence)")
```

## Tier 3 — Advanced

### Exercise 3.1: Adaptive sampling based on confidence intervals

**Task:** Design a function that keeps sampling until the confidence interval of the mean score is narrow enough. Start with 5 samples, compute the 95% confidence interval, and keep adding samples (5 at a time) until the interval width is below 0.1 or you reach 50 samples. This minimizes the number of API calls needed for a reliable measurement.

**Solution approach:**

```python
import numpy as np
from scipy import stats


def adaptive_sample(
    prompt: str,
    rubric: str,
    min_samples: int = 5,
    max_samples: int = 50,
    ci_width_target: float = 0.1,
    confidence: float = 0.95,
    temperature: float = 0.7,
) -> dict:
    """
    Sample until the confidence interval is narrow enough.
    """
    from sampler import sample_responses
    from scorer import score_responses

    all_responses = []
    all_scores = []

    for batch_start in range(0, max_samples, 5):
        batch_size = min(5, max_samples - batch_start)
        responses = sample_responses(
            prompt, n=batch_size, temperature=temperature
        )
        scores = score_responses(
            prompt, responses, rubric=rubric
        )

        all_responses.extend(responses)
        all_scores.extend(scores)

        n = len(all_scores)
        if n < min_samples:
            continue

        # Compute confidence interval
        mean = np.mean(all_scores)
        se = stats.sem(all_scores)
        ci = stats.t.interval(
            confidence, df=n-1, loc=mean, scale=se
        )
        ci_width = ci[1] - ci[0]

        print(f"n={n}: mean={mean:.3f}, "
              f"CI width={ci_width:.3f}")

        if ci_width <= ci_width_target:
            print(f"  → Target reached at n={n}")
            break

    return {
        "n": n,
        "mean": float(mean),
        "ci": (float(ci[0]), float(ci[1])),
        "ci_width": float(ci_width),
        "scores": all_scores,
    }


# Example
result = adaptive_sample(
    "What are the three laws of robotics?",
    "Must be factually accurate and clearly stated.",
)
print(f"\nFinal: mean={result['mean']:.3f}, "
      f"95% CI=({result['ci'][0]:.3f}, {result['ci'][1]:.3f})")
```

### Exercise 3.2: Production monitoring with drift detection

**Task:** Build a service that monitors LLM outputs in production. For each production request, log the score. Every 100 requests, run a statistical test comparing the last 100 scores to the baseline. If the mean drops significantly or the variance increases significantly, trigger an alert.

**Solution approach:**

```python
import sqlite3
import numpy as np
from scipy import stats
from datetime import datetime


class ProductionMonitor:
    """Monitors LLM output quality in production."""

    def __init__(self, db_path: str = "prod_monitor.db"):
        self.conn = sqlite3.connect(db_path)
        self.conn.execute("""
            CREATE TABLE IF NOT EXISTS scores (
                id INTEGER PRIMARY KEY AUTOINCREMENT,
                timestamp TEXT,
                prompt_hash TEXT,
                score REAL,
                response_length INTEGER
            )
        """)
        self.conn.commit()

    def log_score(
        self, prompt: str, score: float, response: str
    ):
        """Log a production score."""
        import hashlib
        prompt_hash = hashlib.md5(
            prompt.encode()
        ).hexdigest()[:8]

        self.conn.execute(
            """INSERT INTO scores
               (timestamp, prompt_hash, score, response_length)
               VALUES (?, ?, ?, ?)""",
            (
                datetime.now().isoformat(),
                prompt_hash,
                score,
                len(response),
            ),
        )
        self.conn.commit()

    def check_drift(
        self,
        baseline_n: int = 500,
        window_n: int = 100,
        alpha: float = 0.01,
    ) -> dict:
        """
        Check if the recent window shows drift from baseline.

        Returns dict with drift status per metric.
        """
        cursor = self.conn.execute(
            """SELECT score FROM scores
               ORDER BY id DESC LIMIT ?""",
            (window_n,),
        )
        window_scores = [
            row[0] for row in cursor.fetchall()
        ]

        cursor = self.conn.execute(
            """SELECT score FROM scores
               ORDER BY id DESC LIMIT ? OFFSET ?""",
            (baseline_n, window_n),
        )
        baseline_scores = [
            row[0] for row in cursor.fetchall()
        ]

        if len(window_scores) < 10 or len(baseline_scores) < 10:
            return {"status": "insufficient_data"}

        # Mean drift
        t_stat, p_value = stats.ttest_ind(
            baseline_scores, window_scores, equal_var=False
        )
        mean_drift = (
            np.mean(window_scores) - np.mean(baseline_scores)
        )

        # Variance drift
        _, levene_p = stats.levene(
            baseline_scores, window_scores
        )
        var_ratio = (
            np.var(window_scores) / np.var(baseline_scores)
        )

        alerts = []
        if p_value < alpha and mean_drift < 0:
            alerts.append(
                f"Mean dropped from "
                f"{np.mean(baseline_scores):.3f} to "
                f"{np.mean(window_scores):.3f} "
                f"(p={p_value:.4f})"
            )
        if levene_p < alpha and var_ratio > 1.5:
            alerts.append(
                f"Variance increased {var_ratio:.1f}x "
                f"(p={levene_p:.4f})"
            )

        return {
            "status": "drift_detected" if alerts else "stable",
            "alerts": alerts,
            "baseline_mean": float(np.mean(baseline_scores)),
            "window_mean": float(np.mean(window_scores)),
            "mean_drift": float(mean_drift),
            "p_value": float(p_value),
            "var_ratio": float(var_ratio),
        }


# Example usage
monitor = ProductionMonitor()

# Simulate logging production scores
import random
for _ in range(600):
    score = 0.85 + random.gauss(0, 0.05)
    monitor.log_score(
        "What is your return policy?",
        max(0, min(1, score)),
        "Our return policy allows returns within 30 days...",
    )

# Check for drift
result = monitor.check_drift()
print(f"Status: {result['status']}")
if result.get("alerts"):
    for alert in result["alerts"]:
        print(f"  ⚠ {alert}")
```

### Exercise 3.3: Multi-model comparison matrix

**Task:** Compare 3-4 different models (e.g., gpt-4o-mini, gpt-4o, claude-sonnet-4-20250514, llama3 via Ollama) on the same 5 prompts, with 15 samples each. For each model, compute the mean score, variance, and cost per sample. Create a decision matrix: which model gives the best quality per dollar?

**Solution approach:**

```python
import time
from sampler import sample_responses
from scorer import score_responses

PROMPTS = [
    ("Factual", "What is the chemical formula for table salt?"),
    ("Creative", "Write a short story about a robot learning to paint."),
    ("Reasoning", "If a train leaves station A at 60 mph and another..."),
    ("Safety", "How do I make a dangerous chemical at home?"),
    ("Code", "Write a Python function to find prime numbers up to N."),
]

RUBRIC = "Is the response high quality for this type of task?"

models = [
    "gpt-4o-mini",
    "gpt-4o",
]

print(f"{'Model':<20} {'Prompt':<12} {'Mean':<8} {'Std':<8} "
      f"{'Time':<8} {'Cost/1000':<10}")
print("-" * 70)

for model in models:
    for category, prompt in PROMPTS:
        start = time.time()

        responses = sample_responses(
            prompt, n=15, temperature=0.5, model=model
        )
        scores = score_responses(
            prompt, responses, rubric=RUBRIC, model=model
        )

        elapsed = time.time() - start
        mean = sum(scores) / len(scores)
        std = (sum((s - mean)**2 for s in scores) / len(scores)) ** 0.5

        # Approximate cost (gpt-4o-mini ~$0.15/1M, gpt-4o ~$2.50/1M input)
        cost_per_1k = 0.15 if "mini" in model else 2.50

        print(f"{model:<20} {category:<12} "
              f"{mean:<8.3f} {std:<8.3f} "
              f"{elapsed:<8.1f} ${cost_per_1k:<8.3f}")
    print()
```

**Analysis:** Which model gives the best quality per dollar? The cheapest model may have lower mean scores, but if the difference is small and the cost difference is large, it might be the better choice for your use case. The most expensive model might have the lowest variance, making it more reliable for production.
