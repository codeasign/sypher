You have read about API keys, clients, and requests. Now you will do it for real. You will sign up for an API account, create a key, write a Python script, and see the model respond. Every step is here — follow them in order and check your output at each stage.

## Setup

Before you start, make sure you have:

- **Python 3.12+** installed and working. Run `python3 --version` to verify. If this fails, go back to the Python installation section.
- **pip** working. Run `pip3 --version` to verify.
- A text editor or IDE (VS Code recommended).

Install the Anthropic Python SDK:

```bash
pip install anthropic
```

Expected output:

```
Successfully installed anthropic-0.x.x httpx-0.x.x ...
```

If you see an error about `pip` not being found, try `python3 -m pip install anthropic` instead. On Windows, the command may be `python -m pip install anthropic`.

Create a project folder for this exercise:

```bash
mkdir first-api-call
cd first-api-call
```

Create a `.gitignore` file to keep your API key safe. This tells git to never track the `.env` file:

```
.env
```

**Checkpoint:** You are inside the `first-api-call` directory with a `.gitignore` file that contains `.env`.

## The Build — Step by Step

### Step 1: Create an Anthropic Account and Get Your Key

Go to [console.anthropic.com](https://console.anthropic.com) in your browser. Sign up for an account if you do not have one. You will need to provide an email address and verify it. Some providers also require phone verification.

Once logged in, navigate to the API Keys section. This is usually in a sidebar or a top-level menu item. Click **Create Key**. Give it a name like "learning" and click **Create**.

The console shows you the key once, and only once. It looks like:

```
sk-ant-api03-xxxxxxxxxxxx
```

The exact string is longer and contains a mix of letters, digits, and dashes. The format is specific to Anthropic — other providers generate keys in different formats but the concept is the same.

**Copy this key immediately** and paste it somewhere safe temporarily. A text editor or a password manager is fine for now. If you close the dialog without copying, you cannot see the full key again — the provider does not store the raw key, only a cryptographic hash. You would need to revoke this key and create a new one.

**Checkpoint:** You have a key string that starts with `sk-ant-`. You have it available to paste into a file.

### Step 2: Store the Key in a .env File

In your `first-api-call` project folder, create a file called `.env` with this content:

```
ANTHROPIC_API_KEY=sk-ant-api03-your-actual-key-here
```

Replace `sk-ant-api03-your-actual-key-here` with the actual key you copied from the console. Do not add spaces, quotes, or extra characters around the value. The format is `KEY=VALUE` with no spaces.

Install the `python-dotenv` package so Python can read the `.env` file:

```bash
pip install python-dotenv
```

Expected output:

```
Successfully installed python-dotenv-1.x.x ...
```

**Checkpoint:** Your `.env` file exists and contains `ANTHROPIC_API_KEY=` followed by your real key. Your `.gitignore` lists `.env`. If you run `git status`, the `.env` file should not appear in the untracked files list (if the directory is a git repository). No key will be committed to git.

### Step 3: Write the Script — Import Libraries

Create a new file called `my_first_call.py` and open it in your editor. Write these first lines:

```python
import os
from dotenv import load_dotenv
import anthropic
```

The `os` module is part of Python's standard library. It gives you access to operating system features, including environment variables via `os.environ`. The `dotenv` library reads your `.env` file and makes its contents available as environment variables. The `anthropic` library is the SDK that talks to the Anthropic API — it handles HTTP requests, serialization, authentication, and response parsing for you.

Add the next line to load your `.env` file:

```python
load_dotenv()
```

This function looks for a file named `.env` in the current working directory, reads it, and adds each key-value pair to `os.environ`. After this call, any variable defined in `.env` is available to your code through `os.environ`.

**Checkpoint:** Your file has the three imports and the `load_dotenv()` call. Save the file.

### Step 4: Write the Script — Create the Client

Add these lines to read your key and create the client:

```python
api_key = os.environ["ANTHROPIC_API_KEY"]
client = anthropic.Anthropic(api_key=api_key)
```

If the `ANTHROPIC_API_KEY` environment variable is not set — because `.env` is missing, `load_dotenv()` failed, or the variable name is misspelled — Python raises a `KeyError`. The error message will name the missing variable, which is your clue to check the `.env` file.

The `anthropic.Anthropic` constructor creates a client object. This client stores your API key internally. Every method you call on this client will include your key in the request headers. You create the client once and reuse it for all your API calls in the same script.

**Checkpoint:** Your script now reads the key and creates a client. If you were to run the script at this point, no error would appear if the environment variable is set correctly.

### Step 5: Make Your First API Call

Add these lines to send a message to the model:

```python
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=100,
    messages=[
        {"role": "user", "content": "Say hello in one short sentence."}
    ]
)
```

This is the actual API call — the moment your code talks to the LLM. Let us break down each parameter:

- **`model`**: Identifies which LLM to use. `claude-sonnet-4-20250514` is the Claude Sonnet 4 model, a capable mid-range model that is good for general-purpose use. Using an incorrect model string — like a typo or a model name from a different provider — produces a `BadRequestError`.
- **`max_tokens`**: Sets the upper limit on how many tokens the model can generate. Setting this to 100 prevents the model from generating a very long response. If the model would naturally generate more than 100 tokens, it stops at 100 and returns whatever it produced.
- **`messages`**: A list of conversation turns. Each turn has a `role` (usually `"user"` for your prompts and `"assistant"` for the model's responses) and `content` (the text). For a first call, you send a single user message.

The script pauses at this line. It sends the request, waits for the server to generate a response, and stores the result in the `response` variable. For a short prompt, this takes roughly 1-3 seconds.

**Checkpoint:** Your script now contains the API call. If you run it at this point, nothing will print yet — the response is stored in memory but not displayed.

### Step 6: Print the Response

Add these final lines to see what the model said:

```python
print(response.content[0].text)
```

The `response.content` is a list of content blocks. Most responses contain a single text block, which you access with `[0]`. The `.text` attribute holds the actual generated text.

The full script now looks like this:

```python
import os
from dotenv import load_dotenv
import anthropic

load_dotenv()

api_key = os.environ["ANTHROPIC_API_KEY"]
client = anthropic.Anthropic(api_key=api_key)

response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=100,
    messages=[
        {"role": "user", "content": "Say hello in one short sentence."}
    ]
)

print(response.content[0].text)
```

### Step 7: Run It

Run the script from your terminal:

```bash
python3 my_first_call.py
```

Expected output (the exact text will vary):

```
Hello! I'm ready to help you with whatever you need today.
```

If you see this output, congratulations. You have successfully:
- Created an API account with a provider
- Generated an API key for authentication
- Stored the key safely in a `.env` file so it stays out of version control
- Written a Python script that reads the key from the environment
- Made an authenticated API call to a frontier LLM
- Printed the model's response to your terminal

Now add lines to verify the token usage — how many tokens this call consumed:

```python
print(f"Input tokens: {response.usage.input_tokens}")
print(f"Output tokens: {response.usage.output_tokens}")
```

Your updated script:

```python
import os

import anthropic

load_dotenv()

api_key = os.environ["ANTHROPIC_API_KEY"]
client = anthropic.Anthropic(api_key=api_key)

        {"role": "user", "content": "Say hello in one short sentence."}
    ]

print(response.content[0].text)
print(f"Input tokens: {response.usage.input_tokens}")
print(f"Output tokens: {response.usage.output_tokens}")
```

Run again:

```bash
python3 my_first_call.py
```

Expected output (approximate):

```
Hello! I'm ready to help you with whatever you need today.

```

These numbers tell you how many tokens your call consumed. The input tokens are your prompt ("Say hello in one short sentence."). The output tokens are the model's response. For this example, roughly 26 total tokens — the cost is negligible. Using the mid-2026 pricing for Claude Sonnet 4 ($3/million input, $15/million output), this call cost roughly 0.00022 cents.

## Common Errors at This Stage

### Error 1: `KeyError: 'ANTHROPIC_API_KEY'`

```
KeyError: 'ANTHROPIC_API_KEY'
```

**Cause:** The environment variable is not set. Either the `.env` file does not exist, it is in the wrong directory, or `load_dotenv()` could not find it. Another common cause: you used `export` or `$env:` in a terminal session but ran the script from a different terminal window.

**Fix:** Verify the `.env` file exists in the same directory as your script. Verify it contains the exact line `ANTHROPIC_API_KEY=sk-ant-...` with no spaces around the `=`. Run this diagnostic command to test whether the variable loads:

```bash
python3 -c "from dotenv import load_dotenv; load_dotenv(); import os; print(os.environ.get('ANTHROPIC_API_KEY', 'NOT SET'))"
```

If this prints `NOT SET`, the `.env` file is missing, in the wrong directory, or has incorrect formatting. If it prints your key (or the first few characters), the loading works and the issue is elsewhere.

### Error 2: `anthropic.AuthenticationError`

```
anthropic.AuthenticationError: Authentication error
```

**Cause:** The API key is invalid, expired, or does not match any account. This can happen if you copied the key incorrectly, the key was revoked in the console, or you accidentally included a space or newline.

**Fix:** Check that you copied the key correctly with no extra spaces or characters. Verify the key starts with the expected prefix (`sk-ant-` for Anthropic). Go to the Anthropic console, navigate to API Keys, and check whether your key is still listed as active. If the key was revoked or expired, generate a new one.

### Error 3: `anthropic.BadRequestError`

```
anthropic.BadRequestError: Bad request: model parameter is required
```

**Cause:** You omitted a required parameter, used an incorrect parameter name, or provided an invalid model string. Common causes include spelling `model` as `Model`, forgetting `max_tokens`, or using an outdated model name.

**Fix:** Verify that `model`, `max_tokens`, and `messages` are all present in the `client.messages.create()` call. Check that the model string exactly matches a valid model ID. As of mid-2026, `claude-sonnet-4-20250514` is the correct identifier for Claude Sonnet 4. If you are using a different model, check the provider's documentation for the exact string.

### Error 4: `anthropic.RateLimitError`

```
anthropic.RateLimitError: Rate limit exceeded
```

**Cause:** You made too many requests in a short time. Free and low-tier plans typically allow 5-50 requests per minute. If you run the script many times rapidly or have a loop that makes multiple calls, you may hit this limit.

**Fix:** Wait 60 seconds and try again. If you hit this frequently, check your plan's rate limits in the provider console. For learning purposes, add a `time.sleep(2)` between calls to stay well under the limit:

```python
import time
time.sleep(2)
```

### Error 5: `ModuleNotFoundError: No module named 'anthropic'`

```
ModuleNotFoundError: No module named 'anthropic'
```

**Cause:** The Anthropic SDK is not installed in the Python environment you are currently using. This often happens when you have multiple Python versions or virtual environments.

**Fix:** Run `pip install anthropic` in the same terminal you will run the script from. Verify the installation with `pip list | grep anthropic` (or `pip list | findstr anthropic` on Windows). If you use virtual environments, make sure you have activated the environment where you installed the SDK with `source venv/bin/activate` (macOS/Linux) or `venv\Scripts\activate` (Windows) before running your script.
