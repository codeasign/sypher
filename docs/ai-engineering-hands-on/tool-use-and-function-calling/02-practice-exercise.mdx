## Setup

Before you start this exercise, make sure you have:

- **Python 3.10 or higher** installed (`python --version`)
- **An Anthropic API key** with credits. Sign up at [console.anthropic.com](https://console.anthropic.com)
- **pip** available (`pip --version`)

### Install the Anthropic SDK

```bash
pip install anthropic python-dotenv
```

```bash
export ANTHROPIC_API_KEY="sk-ant-your-key-here"
```

Or create a `.env` file in your project directory:

```
ANTHROPIC_API_KEY=sk-ant-your-key-here
```

## The Build -- Step by Step

You will build a weather agent that starts with a mocked data source and then graduates to a real API call. Along the way, you will learn every piece of the tool use lifecycle: defining tools, handling tool requests, executing functions, and returning results.

### Step 1: Define Your First Tool

Create a new file called `weather_agent.py`. Start by importing the SDK and defining a tool for fetching weather data.

```python
import os
from anthropic import Anthropic

client = Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])

TOOLS = [
    {
        "name": "get_weather",
        "description": "Get the current weather for a city. Returns temperature, conditions, humidity, and wind speed.",
        "input_schema": {
            "type": "object",
            "properties": {
                "city": {
                    "type": "string",
                    "description": "The name of the city, e.g. 'Tokyo' or 'New York'"
                },
                "units": {
                    "type": "string",
                    "enum": ["celsius", "fahrenheit"],
                    "description": "Temperature unit preference. Defaults to celsius."
                }
            },
            "required": ["city"]
        }
    }
]
```

The tool definition tells the LLM what the tool does (`description`), what parameters it accepts (`input_schema`), and which parameters are required (`required`). The `enum` on `units` restricts the LLM to valid values only.

### Step 2: Implement the Mock Weather Function

Add a function that returns mock weather data. This lets you test the tool use flow without depending on an external API.

```python
import random

def get_weather(city: str, units: str = "celsius") -> str:
    """Return mock weather data for a city."""
    conditions = ["Sunny", "Cloudy", "Rainy", "Partly cloudy", "Clear", "Overcast"]
    temp_range = (15, 35) if "celsius" else (59, 95)
    temp = random.randint(*temp_range)

    data = {
        "city": city,
        "temperature": temp,
        "units": units,
        "conditions": random.choice(conditions),
        "humidity": random.randint(30, 90),
        "wind_speed": random.randint(5, 30),
    }

    unit_symbol = "F" if units == "fahrenheit" else "C"
    return (
        f"Weather in {data['city']}: "
        f"{data['temperature']}°{unit_symbol}, "
        f"{data['conditions']}, "
        f"Humidity: {data['humidity']}%, "
        f"Wind: {data['wind_speed']} km/h"
    )
```

The function returns a formatted string. Your tool functions can return strings, lists of content blocks, or JSON-serializable dicts. A string is the simplest and most common format.

### Step 3: Write the Tool Dispatch Loop

Now write the core loop that sends a message to the LLM, checks for tool use requests, dispatches them to the function, and returns results. This is the engine of every tool-based agent.

```python
def chat_with_tools(user_message: str, max_tool_calls: int = 5) -> str:
    """Send a message to the LLM with tool support."""
    messages = [{"role": "user", "content": user_message}]
    tool_call_count = 0

    while tool_call_count < max_tool_calls:
        response = client.messages.create(
            model="claude-sonnet-4-20250514",
            max_tokens=1024,
            tools=TOOLS,
            messages=messages
        )

        if response.stop_reason != "tool_use":
            # LLM produced a final text response
            return response.content[0].text

        # LLM wants to call one or more tools
        tool_call_count += 1
        tool_results = []

        for block in response.content:
            if block.type == "tool_use":
                tool_name = block.name
                tool_args = block.input
                print(f"  [{tool_call_count}] Calling tool: {tool_name}({tool_args})")

                if tool_name == "get_weather":
                    result = get_weather(**tool_args)
                    print(f"  Result: {result}")
                else:
                    result = f"Error: Unknown tool '{tool_name}'"

                tool_results.append({
                    "type": "tool_result",
                    "tool_use_id": block.id,
                    "content": result
                })

        # Append assistant response and tool results to conversation
        messages.append({"role": "assistant", "content": response.content})
        messages.append({"role": "user", "content": tool_results})

    return "I could not complete your request within the allowed number of steps. Please try a simpler query."
```

The loop works like this:

1. Send the messages array with tool definitions to the LLM
2. Check `response.stop_reason`. If it is not `"tool_use"`, the LLM finished -- return the text
3. If it is `"tool_use"`, iterate over each `tool_use` block in the response
4. For each block, extract `name` and `input`, call the matching function
5. Package each result as a `tool_result` content block with the matching `tool_use_id`
6. Append the assistant's response and the tool results to the messages array
7. Loop back to step 1
8. If `max_tool_calls` is exceeded, return a fallback message


Run it:

Expected output (your temperatures and conditions will vary):

```
  [1] Calling tool: get_weather({'city': 'Tokyo', 'units': 'celsius'})
  Result: Weather in Tokyo: 24°C, Partly cloudy, Humidity: 65%, Wind: 15 km/h
  [1] Calling tool: get_weather({'city': 'London', 'units': 'celsius'})
```

### Step 6: Switch to a Real Weather API

Replace the mock function with a real API call. This example uses the Open-Meteo API, which is free and requires no API key. Add this function and replace `get_weather` in the dispatch loop.

```python
import urllib.request
import json

def get_weather_real(city: str, units: str = "celsius") -> str:
    """Fetch real weather data from the Open-Meteo API."""
    # First, geocode the city name
    geo_url = f"https://geocoding-api.open-meteo.com/v1/search?name={urllib.parse.quote(city)}&count=1"
    try:
        with urllib.request.urlopen(geo_url) as resp:
            geo_data = json.loads(resp.read())
    except Exception as e:
        return f"Error: Could not find city '{city}'. {str(e)}"

    if not geo_data.get("results"):
        return f"Error: Could not find city '{city}'."

    lat = geo_data["results"][0]["latitude"]
    lon = geo_data["results"][0]["longitude"]
    city_name = geo_data["results"][0]["name"]

    # Fetch weather data
    temp_unit = "fahrenheit" if units == "fahrenheit" else "celsius"
    weather_url = (
        f"https://api.open-meteo.com/v1/forecast?"
        f"latitude={lat}&longitude={lon}"
        f"&current=temperature_2m,relative_humidity_2m,weather_code,wind_speed_10m"
        f"&temperature_unit={temp_unit}"
    )
    try:
        with urllib.request.urlopen(weather_url) as resp:
            weather_data = json.loads(resp.read())
    except Exception as e:
        return f"Error: Could not fetch weather. {str(e)}"

    current = weather_data["current"]
    temp = current["temperature_2m"]
    humidity = current["relative_humidity_2m"]
    wind = current["wind_speed_10m"]
    unit_symbol = "F" if units == "fahrenheit" else "C"

    # Map weather codes to conditions
    weather_codes = {
        0: "Clear", 1: "Mainly clear", 2: "Partly cloudy", 3: "Overcast",
        45: "Foggy", 48: "Depositing rime fog", 51: "Light drizzle",
        61: "Slight rain", 63: "Moderate rain", 65: "Heavy rain",
        71: "Slight snow", 73: "Moderate snow", 75: "Heavy snow",
        80: "Slight rain showers", 81: "Moderate rain showers", 82: "Violent rain showers",
        95: "Thunderstorm", 96: "Thunderstorm with slight hail", 99: "Thunderstorm with heavy hail"
    }
    condition = weather_codes.get(current["weather_code"], "Unknown")

    return (
        f"Weather in {city_name}: "
        f"{temp}°{unit_symbol}, "
        f"{condition}, "
        f"Humidity: {humidity}%, "
        f"Wind: {wind} km/h"
    )
```

Update the dispatch to use the real function:

```python
if tool_name == "get_weather":
    result = get_weather_real(**tool_args)
```

Now your agent fetches real data. The tool use flow is identical -- only the function implementation changed.

## Common Errors

### Error 1: Malformed Tool Schema

If your `input_schema` is missing the `type: "object"` wrapper, the API rejects the request.

```
anthropic.BadRequestError: ...
```

**Fix:** Always wrap properties in `"type": "object"`:

```python
# Correct
"input_schema": {
    "type": "object",
    "properties": {
        "city": {"type": "string"}
    },
    "required": ["city"]
}

# Wrong -- missing "type": "object"
"input_schema": {
    "properties": {
        "city": {"type": "string"}
    }
}
```

### Error 2: Tool Name Mismatch

The tool name in the LLM's response must match exactly one of your tool definitions. If you define `get_weather` but the LLM responds with `GetWeather`, it will not match.

```python
# When dispatching:
if block.name == "get_weather":  # LLM said "GetWeather" -- no match
    result = get_weather(**block.input)
else:
    result = f"Error: Unknown tool '{block.name}'"  # This runs
```

**Fix:** Use lowercase snake_case for tool names. The LLM is trained to use this convention and rarely deviates. If you still get mismatches, add a case-insensitive fallback or a name alias map.

### Error 3: Missing Required Parameters

The LLM may omit a parameter you marked as required if the schema description is unclear.

```python
# Tool definition requires "city"
"required": ["city"]

# LLM produces:
# block.input = {"units": "celsius"}
# No "city" key -- get_weather(**block.input) will crash
```

**Fix:** Validate arguments before passing them:

```python
if "city" not in block.input:
    result = "Error: 'city' parameter is required but was not provided."
else:
    result = get_weather(**block.input)
```

### Error 4: Tool Result Too Large

If your tool returns a massive string -- for example, dumping an entire database table -- the LLM may not be able to process it within the context window, or it may truncate the result.

```python
# Bad -- returns too much data
def get_all_users() -> str:
    users = db.query("SELECT * FROM users")  # 10,000 rows
    return json.dumps(users)  # This string could be millions of tokens
```

**Fix:** Keep tool results concise. Return summaries, aggregates, or the most relevant subset:

```python
def get_all_users_summary() -> str:
    count = db.query("SELECT COUNT(*) FROM users")[0][0]
    return f"Total users: {count}. Active users: {count - db.query('SELECT COUNT(*) FROM users WHERE status='inactive''')[0][0]}."
```
