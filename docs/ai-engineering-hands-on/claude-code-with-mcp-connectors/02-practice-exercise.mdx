In this practice exercise, you will build a custom MCP server that connects to a SQLite database, configure Claude Code to use it, and run queries through natural language. By the end, you will have a working MCP connector that lets you inspect and modify data in a local database without writing SQL.

## Setup

### Prerequisites

Before you begin, make sure you have the following installed:

- **Node.js** version 18 or later (the MCP SDK runs on Node.js)
- **Claude Code** installed and authenticated
- **npm** (comes with Node.js)
- A terminal with internet access to install packages

### Create the Project Directory

Open your terminal and create a new directory for your MCP server:

```bash
mkdir -p ~/mcp-sqlite-server
cd ~/mcp-sqlite-server
```

### Initialize the Project

```bash
npm init -y
```

### Install Dependencies

You need two packages: the MCP SDK for building the server, and better-sqlite3 for the SQLite database driver.

```bash
npm install @modelcontextprotocol/sdk better-sqlite3
npm install --save-dev @types/better-sqlite3 typescript
```

### Verify Installation

```bash
node -e "require('@modelcontextprotocol/sdk'); console.log('MCP SDK installed')"
node -e "require('better-sqlite3'); console.log('SQLite driver installed')"
```

Both commands should print success messages. If you see errors, check the troubleshooting section below.

### Set Up a Sample Database

Create a sample database file that you will use throughout the exercise. You can do this with a quick Node.js script:

```bash
node -e "
const Database = require('better-sqlite3');
const db = new Database('sample.db');
db.exec('CREATE TABLE IF NOT EXISTS users (id INTEGER PRIMARY KEY, name TEXT, email TEXT, role TEXT)');
db.exec('INSERT INTO users VALUES (1, \"Alice\", \"alice@example.com\", \"admin\")');
db.exec('INSERT INTO users VALUES (2, \"Bob\", \"bob@example.com\", \"editor\")');
db.exec('INSERT INTO users VALUES (3, \"Charlie\", \"charlie@example.com\", \"viewer\")');
db.exec('CREATE TABLE IF NOT EXISTS projects (id INTEGER PRIMARY KEY, name TEXT, owner_id INTEGER, status TEXT)');
db.exec('INSERT INTO projects VALUES (1, \"Docs Site\", 1, \"active\")');
db.exec('INSERT INTO projects VALUES (2, \"API Service\", 2, \"active\")');
db.exec('INSERT INTO projects VALUES (3, \"Legacy App\", 3, \"archived\")');
console.log('Database created with sample data');
"
```

## The Build — Step by Step

### Step 1: Create the MCP Server Entry Point

Create a file called `server.js` in your project directory. This file will be the main entry point for your MCP server.

```javascript
const { McpServer } = require('@modelcontextprotocol/sdk/server/mcp.js');
const { StdioServerTransport } = require('@modelcontextprotocol/sdk/server/stdio.js');
const Database = require('better-sqlite3');
const path = require('path');

const dbPath = process.env.DB_PATH || './sample.db';
const db = new Database(dbPath);

db.pragma('journal_mode = WAL');

const server = new McpServer({
  name: 'sqlite-connector',
  version: '1.0.0',
  description: 'MCP server for querying a SQLite database'
});
```

This code imports the MCP SDK classes, creates a SQLite database connection, and initializes the MCP server. The database path comes from an environment variable with a sensible default. The WAL (Write-Ahead Logging) pragma improves concurrent read performance.

### Step 2: Define the Query Tool

Add this code after the server initialization to define a tool that executes read-only SQL queries:

```javascript
server.tool(
  'query_database',
  'Execute a read-only SQL query against the SQLite database',
  {
    sql: {
      type: 'string',
      description: 'The SQL query to execute (SELECT only)'
    }
  },
  async (args) => {
    const sql = args.sql.trim().toUpperCase();
    if (!sql.startsWith('SELECT') && !sql.startsWith('PRAGMA')) {
      return {
        content: [{ type: 'text', text: 'Error: Only SELECT and PRAGMA queries are allowed' }],
        isError: true
      };
    }
    try {
      const rows = db.prepare(args.sql).all();
      return {
        content: [{ type: 'text', text: JSON.stringify(rows, null, 2) }]
      };
    } catch (err) {
      return {
        content: [{ type: 'text', text: `Query error: ${err.message}` }],
        isError: true
      };
    }
  }
);
```

The `server.tool` method registers a new tool. The first argument is the tool name, the second is a description for Claude Code, the third is a JSON Schema object defining the parameters, and the fourth is the async handler function. Notice the safety check: only SELECT and PRAGMA queries are allowed, preventing accidental data modification.

### Step 3: Define the Insert Tool

Add a second tool for inserting data. This tool requires your explicit approval before running:

```javascript
server.tool(
  'insert_row',
  'Insert a new row into a table',
  {
    table: {
      type: 'string',
      description: 'The table name to insert into'
    },
    data: {
      type: 'object',
      description: 'Key-value pairs of column names and values',
      additionalProperties: true
    }
  },
  async (args) => {
    const allowedTables = ['users', 'projects'];
    if (!allowedTables.includes(args.table)) {
      return {
        content: [{ type: 'text', text: `Error: Table "${args.table}" is not in the allowed list` }],
        isError: true
      };
    }
    try {
      const columns = Object.keys(args.data).join(', ');
      const placeholders = Object.keys(args.data).map(() => '?').join(', ');
      const values = Object.values(args.data);
      const stmt = db.prepare(`INSERT INTO ${args.table} (${columns}) VALUES (${placeholders})`);
      const result = stmt.run(...values);
      return {
        content: [{ type: 'text', text: `Inserted row with ID ${result.lastInsertRowid}` }]
      };
    } catch (err) {
      return {
        content: [{ type: 'text', text: `Insert error: ${err.message}` }],
        isError: true
      };
    }
  }
);
```

This tool is deliberately restricted to only the `users` and `projects` tables to prevent arbitrary table access. The parameterized query prevents SQL injection. Because this tool modifies data, Claude Code will prompt you for permission before executing it.

### Step 4: Define the Schema Discovery Tool

Add a tool that lets Claude Code discover the database schema:

```javascript
server.tool(
  'get_schema',
  'Get the schema of all tables in the database',
  {
    table: {
      type: 'string',
      description: 'Optional table name to get schema for a specific table'
    }
  },
  async (args) => {
    try {
      let tables;
      if (args.table) {
        tables = db.prepare("SELECT name FROM sqlite_master WHERE type='table' AND name = ?").all(args.table);
      } else {
        tables = db.prepare("SELECT name FROM sqlite_master WHERE type='table'").all();
      }
      const schema = {};
      for (const t of tables) {
        schema[t.name] = db.prepare(`PRAGMA table_info(${t.name})`).all();
      }
      return {
        content: [{ type: 'text', text: JSON.stringify(schema, null, 2) }]
      };
    } catch (err) {
      return {
        content: [{ type: 'text', text: `Schema error: ${err.message}` }],
        isError: true
      };
    }
  }
);
```

This tool queries SQLite's internal schema tables and returns the column names, types, and constraints for each table. Claude Code uses this information to write correct queries without guessing the schema.

### Step 5: Start the Server

Add the server startup code at the bottom of `server.js`:

```javascript
async function main() {
  const transport = new StdioServerTransport();
  await server.connect(transport);
  console.error('SQLite MCP server running on stdio');
}

main().catch((err) => {
  console.error('Server failed to start:', err);
  process.exit(1);
});
```

The `StdioServerTransport` communicates over stdin and stdout. Log messages go to stderr (via `console.error`) so they do not interfere with the JSON-RPC messages on stdout. This is a common pattern in MCP server development.

### Step 6: Configure Claude Code to Use Your Server

Create or edit your project's `.claude/settings.json` file. If you are in the project directory where you want to use the SQLite MCP server, run:

```bash
mkdir -p .claude
```

Then create `.claude/settings.json` with this content:

```json
{
  "mcpServers": {
    "sqlite-connector": {
      "command": "node",
      "args": ["/absolute/path/to/mcp-sqlite-server/server.js"],
      "env": {
        "DB_PATH": "/absolute/path/to/mcp-sqlite-server/sample.db"
      }
    }
  }
}
```

Replace `/absolute/path/to/mcp-sqlite-server/` with the actual absolute path to your project directory. You can get this path by running `pwd` in the project directory.

### Step 7: Test the Connection

Start Claude Code in the project directory where you created the `.claude/settings.json`:

```bash
claude
```

When Claude Code starts, you should see a message indicating that an MCP server connected. Type `/mcp` to see the list of connected servers:

```text
/mcp
```

Expected output:

```text
Connected MCP Servers:
  sqlite-connector (running)
    Tools:
      query_database - Execute a read-only SQL query
      insert_row - Insert a new row into a table
      get_schema - Get schema information
```

If you do not see your server listed, check the error section below.

### Step 8: Run Your First Natural Language Query

Now that the MCP server is connected, you can ask Claude Code to query the database using natural language. Try this prompt:

```text
How many users are in my database, and what roles do they have?
```

Claude Code will use the `query_database` tool to run the appropriate SQL. You will see a permission prompt the first time:

```text
Allow the tool 'sqlite-connector/query_database' with these arguments?
  sql: "SELECT role, COUNT(*) as count FROM users GROUP BY role"

[Allow] [Allow Always] [Deny]
```

Type `Allow` to approve. Claude Code will then show the result. Try a more complex query:

```text
Show me all projects owned by Alice, including the project status.
```

Claude Code will figure out that it needs to join the `users` and `projects` tables. This is the power of MCP connectors: you never need to write SQL directly.

### Step 9: Insert Data Through Natural Language

Try inserting a new user through natural language:

```text
Add a new user named Diana with email diana@example.com as a viewer.
```

Claude Code will use the `insert_row` tool. Because this is a write operation, you will be prompted for permission:

```text
Allow the tool 'sqlite-connector/insert_row' with these arguments?
  table: "users"
  data: { "name": "Diana", "email": "diana@example.com", "role": "viewer" }

[Allow] [Allow Always] [Deny]
```

Approve the tool call. Then verify the insertion:

```text
List all users in the database now.
```

You should see four users including Diana. You have just modified a database through natural language, with the MCP connector handling all the SQL translation.

## Common Errors at This Stage

### Error 1: Server Not Starting

```text
Error: Cannot find module '@modelcontextprotocol/sdk'
```

**Cause:** The MCP SDK dependency was not installed correctly or you are running from the wrong directory.

**Fix:** Make sure you are in the `~/mcp-sqlite-server` directory and run `npm install` again. Also verify that `node_modules/@modelcontextprotocol/sdk` exists. If you moved the project, reinstall dependencies in the new location.

### Error 2: Transport Connection Failure

```text
Error: Server disconnected unexpectedly
```

**Cause:** The MCP server process crashed during startup, often because of an unhandled exception or a missing dependency.

**Fix:** Test the server standalone before connecting it to Claude Code. Run `node server.js` directly in your terminal. If you see errors, fix them first. Common issues include incorrect database path, missing environment variables, or syntax errors in your server code.

### Error 3: Tool Call Timeout

```text
The MCP server did not respond in time. The tool call was cancelled.
```

**Cause:** The database query is taking too long, or the server is stuck. This can happen with very large datasets or complex queries.

**Fix:** Check that your SQL queries are not doing full table scans on large tables. For production databases, consider adding query timeouts or limits to your tool handler. For this exercise, keep the sample data small.

### Error 4: Permission Denied on Insert

```text
Error: Table "payments" is not in the allowed list
```

**Cause:** You tried to insert into a table that is not in the allowed list defined in the `insert_row` tool.

**Fix:** This is a safety feature, not an error in your code. If you need to insert into additional tables, update the `allowedTables` array in the `insert_row` handler to include them. In production, always audit which tables your MCP server allows write access to.

### Error 5: JSON-RPC Parse Error

```text
[JsonRpcError]: Parse error
```

**Cause:** The MCP server is outputting unexpected text to stdout instead of JSON-RPC messages. This often happens when you accidentally use `console.log` instead of `console.error` for debug output.

**Fix:** Search your `server.js` for any `console.log` statements and change them to `console.error`. Only JSON-RPC messages should go to stdout. The MCP SDK uses stdout for protocol messages, so any extra text breaks the message framing.
