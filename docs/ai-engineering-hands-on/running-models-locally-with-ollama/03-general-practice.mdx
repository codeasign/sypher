### Exercise 1: Multi-Model Comparator

Build a script that sends the same prompt to multiple models and compares their responses side by side.

**Requirements:**
- Accept a list of model names as a configuration
- Send the same prompt to each model
- Print each model response with a header showing the model name
- Measure and report the response time for each model
- Handle models that are not installed gracefully

**Starter code:**

```python
import time
from ollama import chat, ResponseError

models = ['gemma3:2b', 'phi3:mini']
prompt = "Explain the water cycle in three sentences."

for model in models:
    print(f"\n{'='*60}")
    print(f"Model: {model}")
    print(f"{'='*60}")
    try:
        start = time.perf_counter()
        response = chat(model=model, messages=[
            {'role': 'user', 'content': prompt}
        ])
        elapsed = time.perf_counter() - start
        print(response.message.content)
        print(f"\nTime: {elapsed:.2f}s")
    except ResponseError as e:
        print(f"Error: {e.error}")
```

### Exercise 2: Streaming Chat Application

Build a command-line chat application that streams responses token by token, showing each token as it arrives.

**Requirements:**
- Maintain conversation history across turns
- Stream the response token by token
- Accept user input in a loop
- Exit when the user types /quit
- Show a typing indicator while the model is generating

**Starter code:**

```python
from ollama import chat

messages = [
    {'role': 'system', 'content': 'You are a helpful assistant.'}
]

print("Chat started. Type /quit to exit.")
print()

while True:
    user_input = input('You: ')
    if user_input.lower() == '/quit':
        break

    messages.append({'role': 'user', 'content': user_input})

    print('AI: ', end='', flush=True)

    # Use streaming
    stream = chat(model='gemma3:2b', messages=messages, stream=True)

    full_response = ''
    for chunk in stream:
        token = chunk['message']['content']
        print(token, end='', flush=True)
        full_response += token

    print()
    messages.append({'role': 'assistant', 'content': full_response})
```

**Challenge:** Add color to differentiate user input from AI output. On terminals that support ANSI, print user messages in blue and AI messages in green.

### Exercise 1: RAG Pipeline with Local Embeddings

Build a minimal Retrieval-Augmented Generation pipeline that searches a local document collection and answers questions using only the retrieved context.

**Requirements:**
- Store 5-10 short documents (strings hardcoded or loaded from files)
- Generate embeddings for each document
- On a query, embed the query and find the top-2 most similar documents by cosine similarity
- Build a prompt that includes the retrieved documents as context
- Send the prompt to the LLM and return the answer
- No external vector database: use in-memory lists

**Starter code:**

```python
from ollama import chat, embed
import math

documents = [
    "Python is a high-level programming language known for readability.",
    "Docker containers package software with all dependencies.",
    "REST APIs use HTTP methods for client-server communication.",
    "SQL databases store data in tables with rows and columns.",
    "Git tracks changes in source code during development.",
    "Machine learning models learn patterns from training data.",
    "Web servers handle HTTP requests and return responses.",
    "PostgreSQL is an open-source relational database system.",
]

def cosine_similarity(a, b):
    dot = sum(x * y for x, y in zip(a, b))
    na = math.sqrt(sum(x*x for x in a))
    nb = math.sqrt(sum(y*y for y in b))
    return dot / (na * nb)

# Generate document embeddings
doc_embeddings = []
for doc in documents:
    response = embed(model='gemma3:2b', input=doc)
    doc_embeddings.append(response.embeddings[0])

def answer_question(question):
    # Embed the question
    q_embed = embed(model='gemma3:2b', input=question).embeddings[0]

    # Find top-2 most similar documents
    similarities = [(cosine_similarity(q_embed, de), i)
                     for i, de in enumerate(doc_embeddings)]
    similarities.sort(reverse=True)
    top_indices = [i for _, i in similarities[:2]]

    # Build context
    context = "\n".join([documents[i] for i in top_indices])

    # Generate answer with context
    prompt = f"""Answer the question using only the context below.

Context:
{context}

Question: {question}

Answer:"""

    response = chat(model='gemma3:2b', messages=[
        {'role': 'user', 'content': prompt}
    ])
    return response.message.content


# Test
questions = [
    "What is Git used for?",
    "How do REST APIs communicate?",
    "What is PostgreSQL?"
]

for q in questions:
    print(f"Q: {q}")
    print(f"A: {answer_question(q)}")
    print()
```

### Exercise 2: Tool-Using Agent Simulation

Build a simple agent that can use a calculator tool. The agent decides whether to compute something or answer directly based on the user question.

**Requirements:**
- Define a Python calculator tool with operations: add, subtract, multiply, divide
- The LLM generates a structured response indicating whether a tool call is needed
- If a tool call is made, execute the tool and feed the result back to the LLM
- Use string parsing to detect tool-calling patterns (no function-calling API needed)

**Starter code:**

```python
from ollama import chat
import re

def calculator(expression):
    """Safe calculator that handles basic arithmetic."""
    # Only allow digits, operators, parentheses, spaces, and decimal points
    safe_pattern = re.compile(r'^[\d+\-*/().\s]+$')
    if not safe_pattern.match(expression):
        return "Error: Invalid characters in expression"
    try:
        result = eval(expression, {'__builtins__': {}}, {})
        return str(result)
    except Exception as e:
        return f"Error: {str(e)}"

tools_description = """
You have access to a calculator tool.
To use the calculator, respond with exactly:
CALCULATE: <expression>

For any other question, respond normally.

Examples:
User: What is 15 * 23?
Assistant: CALCULATE: 15 * 23

User: What is the capital of France?
Assistant: The capital of France is Paris.
"""

def agent_loop(user_input, max_turns=5):
    messages = [
        {'role': 'system', 'content': tools_description},
        {'role': 'user', 'content': user_input}
    ]

    for turn in range(max_turns):
        response = chat(model='gemma3:2b', messages=messages)
        content = response.message.content

        # Check if the model wants to use the calculator
        calc_match = re.match(r'^CALCULATE:\s*(.+)$', content.strip(), re.IGNORECASE)
        if calc_match:
            expression = calc_match.group(1)
            result = calculator(expression)
            print(f"Tool: calculator({expression}) => {result}")
            messages.append({'role': 'assistant', 'content': content})
            messages.append({'role': 'user', 'content': f"The result is {result}"})
        else:
            print(f"Assistant: {content}")
            break


# Test
agent_loop("What is 1234 + 5678?")
print("---")
agent_loop("What is the square root of 144?")
```

### Exercise 3: Prompt Variation Tester

Build a script that tests how different system prompts and parameter settings affect the model output for the same input. This is useful for prompt engineering and finding the optimal configuration.

**Requirements:**
- Define at least 3 different system prompts for the same task
- Test each prompt with the same user input
- Vary temperature parameter (0.1, 0.7, 1.5) for each prompt
- Record all responses and parameters in a structured format
- Save results to a CSV file for analysis

**Starter code:**

```python
import csv
import time
from ollama import chat

system_prompts = {
    "terse": "Answer in one sentence. Be direct.",
    "detailed": "Provide a thorough, detailed explanation with examples.",
    "educational": "Explain as if teaching a beginner. Use simple language."
}

temperatures = [0.1, 0.7, 1.5]
prompt = "What is the difference between HTTP and HTTPS?"

results = []

for style_name, system_prompt in system_prompts.items():
    for temp in temperatures:
        messages = [
            {'role': 'system', 'content': system_prompt},
            {'role': 'user', 'content': prompt}
        ]

        start = time.perf_counter()
        response = chat(
            model='gemma3:2b',
            messages=messages,
            options={'temperature': temp}
        )
        elapsed = time.perf_counter() - start

        result = {
            'style': style_name,
            'temperature': temp,
            'response': response.message.content,
            'time_seconds': round(elapsed, 2),
            'tokens': len(response.message.content.split())
        }
        results.append(result)

        print(f"Style: {style_name}, Temp: {temp}")
        print(f"Response: {result['response'][:100]}...")
        print()

# Save to CSV
with open('prompt_test_results.csv', 'w', newline='') as f:
    writer = csv.DictWriter(f, fieldnames=results[0].keys())
    writer.writeheader()
    writer.writerows(results)

print("Results saved to prompt_test_results.csv")
```
