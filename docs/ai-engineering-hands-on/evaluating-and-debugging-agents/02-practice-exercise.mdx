In this exercise you will take a basic agent loop and add structured logging, an evaluation function, a test suite, and a before-after comparison when you change the system prompt. By the end, you will have a reusable evaluation harness for any agent you build.

## Setup

Before you start, make sure you have the following prerequisites.

### Prerequisites

- Python 3.10 or later installed
- An Anthropic API key with access to Claude models
- Basic familiarity with the agent loop pattern

```bash

```


Never commit this file. Add `.env` to a `.gitignore` file. On Windows PowerShell, set the environment variable directly for the current session instead of writing it to a file:

## The Build -- Step by Step

You will build from a minimal agent loop and incrementally add evaluation infrastructure.

### Step 1: Create the Base Agent Loop

Create a file called `agent.py` with a minimal agent loop that has access to a simple tool. This agent will answer questions about weather data using a mock tool.

```python
import json
import os
import time
import uuid
from datetime import datetime, timezone
from anthropic import Anthropic

client = Anthropic()
MODEL = "claude-sonnet-4-20250514"
MAX_LOOPS = 5


def get_weather(location: str, date: str = "today") -> dict:
    """Mock weather data tool. Returns simulated weather for a location."""
    data = {
        "San Francisco": {"today": {"temp": 18, "condition": "foggy"},
                          "tomorrow": {"temp": 20, "condition": "sunny"}},
        "New York": {"today": {"temp": 25, "condition": "humid"},
                     "tomorrow": {"temp": 22, "condition": "rainy"}},
        "London": {"today": {"temp": 12, "condition": "cloudy"},
                   "tomorrow": {"temp": 10, "condition": "drizzle"}},
    }
    city_data = data.get(location, {})
    day_data = city_data.get(date, {"temp": "unknown", "condition": "unknown"})
    return {"location": location, "date": date, **day_data}


TOOLS = [
    {
        "name": "get_weather",
        "description": "Get the weather for a location on a given date",
        "input_schema": {
            "type": "object",
            "properties": {
                "location": {"type": "string", "description": "City name"},
                "date": {"type": "string",
                         "description": "today or tomorrow",
                         "enum": ["today", "tomorrow"]},
            },
            "required": ["location"],
        },
    }
]

SYSTEM_PROMPT = "You are a helpful weather assistant. Answer questions about the weather using the get_weather tool. Always check the weather for the requested date."


def run_agent(user_input: str) -> list:
    """Run the agent loop and return a list of step logs."""
    messages = [{"role": "user", "content": user_input}]
    steps = []

    for loop_count in range(MAX_LOOPS):
        step_start = time.time()
        response = client.messages.create(
            model=MODEL,
            system=SYSTEM_PROMPT,
            max_tokens=1024,
            messages=messages,
            tools=TOOLS,
        )

        step_log = {
            "run_id": "",
            "timestamp": datetime.now(timezone.utc).isoformat(),
            "step_number": loop_count + 1,
            "event_type": "llm_response",
            "input_tokens": response.usage.input_tokens,
            "output_tokens": response.usage.output_tokens,
            "finish_reason": response.stop_reason,
            "duration_ms": int((time.time() - step_start) * 1000),
        }

        if response.stop_reason == "tool_use":
            tool_block = response.content[-1]
            step_log["event_type"] = "tool_call"
            step_log["tool_name"] = tool_block.name
            step_log["tool_input"] = tool_block.input

            if tool_block.name == "get_weather":
                result = get_weather(**tool_block.input)
            else:
                result = {"error": f"Unknown tool: {tool_block.name}"}

            step_log["tool_output"] = result
            messages.append({"role": "assistant", "content": response.content})
            messages.append({
                "role": "user",
                "content": [
                    {"type": "tool_result",
                     "tool_use_id": tool_block.id,
                     "content": json.dumps(result)}
                ],
            })
        else:
            step_log["final_output"] = response.content[0].text if response.content else ""
            steps.append(step_log)
            return steps

        steps.append(step_log)

    return steps


if __name__ == "__main__":
    steps = run_agent("What is the weather in London today?")
    for s in steps:
        print(json.dumps(s, indent=2))
```

Run it to confirm the agent works:

```bash
python agent.py
```

Expected output: A JSON log of the agent steps, starting with a tool call to `get_weather` and ending with the final assistant response. You should see input and output token counts, timing, and the tool result.

### Step 2: Add Structured Logging with Run IDs

The base agent loop logs to the console but does not save to a file. Add a logging wrapper that assigns a run ID and writes each step to a JSON Lines file.

Create `evaluator.py` and add the logging infrastructure:

```python
import json
import os
import uuid
from datetime import datetime, timezone


class AgentLogger:
    """Structured logger that writes agent run steps to a JSON Lines file."""

    def __init__(self, log_dir: str = "logs"):
        self.log_dir = log_dir
        os.makedirs(log_dir, exist_ok=True)

    def start_run(self) -> str:
        """Start a new run and return the run ID."""
        run_id = str(uuid.uuid4())[:8]
        self.run_id = run_id
        self.steps = []
        return run_id

    def log_step(self, step_data: dict):
        """Record a step and append it to the run log file."""
        step_data["run_id"] = self.run_id
        step_data["timestamp"] = datetime.now(timezone.utc).isoformat()
        self.steps.append(step_data)

        filename = os.path.join(self.log_dir, f"run_{self.run_id}.jsonl")
        with open(filename, "a") as f:
            f.write(json.dumps(step_data) + "\n")

    def get_trajectory(self) -> list:
        """Return the list of tool names called in order."""
        return [
            s.get("tool_name", "none")
            for s in self.steps
            if s.get("event_type") == "tool_call"
        ]

    def get_total_tokens(self) -> dict:
        """Sum input and output tokens across all steps."""
        total_input = sum(s.get("input_tokens", 0) for s in self.steps)
        total_output = sum(s.get("output_tokens", 0) for s in self.steps)
        return {"input_tokens": total_input, "output_tokens": total_output,
                "total": total_input + total_output}
```

Now modify `agent.py` to use the logger. Add this import and replace the `run_agent` function:


### Step 3: Build an Evaluation Function

Add an evaluation function to `evaluator.py` that scores whether the agent achieved its goal on a given test case.

```python
def evaluate_run(logger: AgentLogger, expected_tools: list,
                 expected_keywords: list) -> dict:
    """Score an agent run against expected trajectory and outcome."""
    trajectory = logger.get_trajectory()
    tokens = logger.get_total_tokens()

    # Trajectory score: how many of the expected tools were called in order
    trajectory_score = 0.0
    if expected_tools:
        matches = 0
        for i, expected in enumerate(expected_tools):
            if i < len(trajectory) and trajectory[i] == expected:
                matches += 1
        trajectory_score = matches / len(expected_tools)

    # Outcome score: does the final response contain expected keywords?
    final_step = logger.steps[-1] if logger.steps else {}
    final_output = final_step.get("final_output", "")
    outcome_score = 0.0
    if expected_keywords and final_output:
        keyword_hits = sum(
            1 for kw in expected_keywords if kw.lower() in final_output.lower()
        )
        outcome_score = keyword_hits / len(expected_keywords)

    total_score = (trajectory_score + outcome_score) / 2

    return {
        "trajectory": trajectory,
        "trajectory_score": round(trajectory_score, 2),
        "outcome_score": round(outcome_score, 2),
        "total_score": round(total_score, 2),
        "tokens": tokens,
    }
```

### Step 5: Run the Full Evaluation Suite

Add a script at the bottom of `evaluator.py` to load the test cases, run the agent on each, and score the results:

```python
import json
from agent import run_agent


def run_evaluation_suite(test_cases_path: str = "test_cases.json"):
    """Run the agent against every test case and return scores."""
    with open(test_cases_path) as f:
        test_cases = json.load(f)

    results = []
    for tc in test_cases:
        print(f"Running test case: {tc['id']} -- {tc['input'][:50]}...")
        steps = run_agent(tc["input"])
        score = evaluate_run(logger, tc["expected_tools"],
                             tc["expected_keywords"])
        score["test_id"] = tc["id"]
        score["input"] = tc["input"]
        results.append(score)
        print(f"  Score: {score['total_score']}  "
              f"Trajectory: {score['trajectory']}")
        print(f"  Tokens: {score['tokens']}")

    print("\n=== EVALUATION SUMMARY ===")
    for r in results:
        print(f"{r['test_id']}: total={r['total_score']} "
              f"traj={r['trajectory_score']} outcome={r['outcome_score']} "
              f"tokens={r['tokens']['total']}")

    return results
```

### Step 6: Change the Prompt and Compare Results

Now see how a prompt change affects evaluation scores. Create a second system prompt and run the same test suite.

Create a new file called `compare_prompts.py`:

```python
import json
from agent import run_agent, SYSTEM_PROMPT, logger
from evaluator import evaluate_run

# Define a different system prompt
ALT_SYSTEM_PROMPT = (
    "You are a weather assistant. For every question, first call get_weather "
    "to retrieve the data, then summarize the result in a friendly sentence. "
    "If the location is unknown, say 'I could not find data for that location'."
)

# Load test cases
with open("test_cases.json") as f:
    test_cases = json.load(f)

print("=== BASELINE (original prompt) ===")
baseline_scores = []
for tc in test_cases:
    steps = run_agent(tc["input"])
    score = evaluate_run(logger, tc["expected_tools"],
                         tc["expected_keywords"])
    score["test_id"] = tc["id"]
    baseline_scores.append(score)
    print(f"  {tc['id']}: {score['total_score']} -- "
          f"traj={score['trajectory']}")

print("\n=== ALTERNATE (new prompt) ===")
# Override SYSTEM_PROMPT in agent module
import agent
original_prompt = agent.SYSTEM_PROMPT
agent.SYSTEM_PROMPT = ALT_SYSTEM_PROMPT

alternate_scores = []
for tc in test_cases:
    # Reset logger for each run by re-importing
    steps = run_agent(tc["input"])
    score = evaluate_run(logger, tc["expected_tools"],
                         tc["expected_keywords"])
    score["test_id"] = tc["id"]
    alternate_scores.append(score)
    print(f"  {tc['id']}: {score['total_score']} -- "
          f"traj={score['trajectory']}")

# Restore original
agent.SYSTEM_PROMPT = original_prompt

print("\n=== COMPARISON ===")
print(f"{'Test':<10} {'Baseline':<12} {'Alternate':<12} {'Change':<10}")
print("-" * 45)
for b, a in zip(baseline_scores, alternate_scores):
    change = a['total_score'] - b['total_score']
    sign = "+" if change > 0 else ""
    print(f"{b['test_id']:<10} {b['total_score']:<12} "
          f"{a['total_score']:<12} {sign}{change:.2f}")
```

Run the comparison:

```bash
python compare_prompts.py
```

Expected output: A table comparing baseline and alternate scores per test case. The alternate prompt should improve the outcome on test 3 (the unknown location case) because it explicitly instructs the agent to handle unknown locations gracefully. If test 1 or test 2 regress, the new prompt has a side effect you need to fix.

## Common Errors

### Error 1: Log File Permissions

The `AgentLogger` creates files in the `logs/` directory. If this directory does not exist or is not writable, the agent will crash when trying to write the first log entry.

**Symptom**: `FileNotFoundError` or `PermissionError` when running the agent.

**Fix**: The logger's constructor calls `os.makedirs(log_dir, exist_ok=True)`, which creates the directory if it does not exist. If you change the `log_dir` parameter, make sure the parent directory is writable. Run the script from a directory where you have write permissions.

### Error 2: PII in Logs

The current logger records every tool input and output without redaction. If your tools process personal data (names, email addresses, locations), the log files will contain this data.

**Symptom**: Log files contain sensitive user information that should not be stored.

**Fix**: Add a redaction filter in the `log_step` method. Before writing the JSON line, scan for patterns that match PII (email addresses, phone numbers, SSNs) and replace them with placeholders. Never log raw tool inputs or outputs in production without redaction.

### Error 3: Evaluation Dataset Too Small

A test suite with three cases is insufficient to evaluate an agent reliably. The agent might pass all three tests but fail on the fourth variation.

**Symptom**: Prompt changes that pass the evaluation suite still cause failures in production.

**Fix**: Grow your test suite to cover edge cases: missing data, ambiguous inputs, multi-turn conversations, unexpected parameter combinations. Add at least one test case per tool the agent might call. For each tool, include a success case, a failure case, and an edge case.

### Error 4: Evaluation Metric Does Not Match Actual Quality

The evaluation function in this exercise uses keyword matching to score outcomes. This is a weak signal -- the agent could say the right words in the wrong context and still get a high score.

**Symptom**: Evaluation scores are high but the agent's actual responses are wrong.

**Fix**: Use stronger evaluation methods. For trajectory evaluation, check exact tool call order and parameter values. For outcome evaluation, use an LLM-as-judge approach where another model evaluates whether the response meets the criteria. Combine automated checks with manual review for high-stakes tasks.

## What You Just Built

You built a complete evaluation and debugging harness for AI agents. Here is what you can now do:

- **Structured logging**: Every agent step is recorded with run ID, timestamps, tool calls, token counts, and timing, all stored in reusable JSON Lines files.
- **Evaluation scoring**: The evaluation function scores both trajectory (did the agent follow the correct path) and outcome (did the agent achieve the goal), giving you a quantitative measure of agent quality.
- **Test suite execution**: A reusable test suite with three edge cases that you can run on demand to validate agent behavior before deployment.
- **Before-and-after comparison**: The comparison script lets you test prompt changes against the same test suite and quantify whether the change improved or regressed performance.
- **Log replay**: Because every step is logged with run IDs, you can replay any agent run by reading the logged steps and understanding exactly what happened at each decision point.
- **Extensible framework**: You can add more tools, more test cases, and more evaluation metrics without restructuring the code. The logger and evaluator are decoupled from the agent loop.
