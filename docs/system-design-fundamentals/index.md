# System Design Fundamentals

Welcome to the System Design Fundamentals course. This is a comprehensive, FAANG-level guide to system design — from the foundations to advanced interview preparation.

## Foundations

| Topic | What You'll Learn |
|-------|------------------|
| [What is System Design](what-is-system-design) | The discipline of architecting large-scale software systems |
| [Why System Design Matters](why-system-design-matters) | The business and technical impact of good design |
| [Functional vs Non-Functional Requirements](functional-vs-non-functional-requirements) | Defining what a system does and how well it does it |
| [Scalability](scalability) | Growing a system from 100 to 100 million users |
| [Availability](availability) | Keeping the lights on — uptime, SLAs, and redundancy |
| [Reliability](reliability) | Doing the right thing even when things go wrong |
| [Durability](durability) | Never losing data — from disk to replication |
| [Consistency](consistency) | Making sure everyone sees the same truth |
| [Latency vs Throughput](latency-vs-throughput) | Speed vs volume — the two axes of performance |
| [Capacity Estimation](capacity-estimation) | Back-of-envelope math every engineer needs |
| [System Design Process](system-design-process) | The step-by-step framework for interviews and real systems |

Each topic is structured as a guided learning path:

- **Mental Model** — Build intuition with stories and analogies
- **Deep Dive** — Understand internals, data structures, and algorithms
- **Architecture** — See how it fits in a production-scale system
- **Trade-offs** — Learn the hard choices and failure modes
- **Real World** — Study how companies like Netflix, Uber, and Google implement it
- **Interview** — Ace the system design interview with frameworks and phrase banks
- **Challenges** — Prove your knowledge with tiered exercises
