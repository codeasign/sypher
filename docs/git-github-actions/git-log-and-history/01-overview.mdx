Your teammate pushed a commit that broke the build three days ago. You have 47 commits on main since then. You need to find which one caused the problem, understand what changed in a specific file over the last week, and see the branching history to know who merged what and when. Without the right commands, you are left scrolling through a wall of commit hashes, opening each one manually in the GitHub web UI, and piecing together the timeline by hand.

**Git log commands let you query your project's commit history with surgical precision — filtering by date, author, file, keyword, or structural pattern — and visualize the full Directed Acyclic Graph (DAG) of branches and merges.**

Think of `git log` as the flight recorder for your project. Every commit is a black-box data point: who made it, when, what changed, and why. The flags on `git log` are your filters — like search parameters on a dashboard that let you zoom into exactly the time window, person, or file pattern you care about. `git show` is the playback button for a single record, and `git diff` compares any two points in the recording.

## Command and Syntax Reference

| Command | Flags / Syntax | What it Does | Example |
|---|---|---|---|
| `git log` | (no flags) | Lists all commits in reverse chronological order | `git log` |
| `git log --oneline` | `--oneline` | Shows each commit as one line: abbreviated hash + subject | `git log --oneline` |
| `git log --graph` | `--graph` | Draws a text-based graph of branch and merge history | `git log --graph --oneline` |
| `git log --all` | `--all` | Shows commits from all branches, not just the current one | `git log --all --oneline` |
| `git log --decorate` | `--decorate` | Labels each commit with branch/tag names it points to (enabled by default in recent Git) | `git log --decorate=full` |
| `git log --since` / `--until` | `--since="2026-06-01" --until="2026-06-30"` | Filters commits by date range | `git log --since="7 days ago"` |
| `git log --author` | `--author="alice"` | Filters commits by author (partial match) | `git log --author="maria"` |
| `git log --grep` | `--grep="fix"` | Filters commits whose message matches a pattern | `git log --grep="hotfix" -i` |
| `git log -p` | `-p` or `--patch` | Shows the full diff (patch) for each commit | `git log -p --oneline -3` |
| `git log --stat` | `--stat` | Shows file-change statistics per commit (files changed, insertions, deletions) | `git log --stat --oneline` |
| `git log -S` | `-S"functionName"` | Searches for commits that added or removed a specific string (pickaxe search) | `git log -S"validate_email" --oneline` |
| `git log -L` | `-L1,20:src/app.js` | Traces history of a specific line range in a file | `git log -L10,30:src/config.py` |
| `git show` | `<commit-hash>` | Shows a single commit: metadata, message, and full diff | `git show a1b2c3d` |
| `git diff` | `git diff <ref1>..<ref2>` | Compares two arbitrary commits, branches, or tags | `git diff main..feature-x` |
| `git diff --staged` | `--staged` | Shows changes staged for the next commit (vs. HEAD) | `git diff --staged` |

**Platform notes:** These commands work identically on Windows (Git Bash, PowerShell with Git), macOS, and Linux. The only difference is date parsing: `--since="7 days ago"` works on all three, but locale-specific date formats (e.g., `"06/30/2026"`) may behave differently — always use ISO 8601 format (`2026-06-30`) for portability. On Windows, PowerShell users may need to quote arguments with single quotes escaped or use double quotes: `git log --author="maria"`.

## How It Actually Works — Step by Step

### Step 1: Understanding the Commit DAG Internally

Git stores history as a directed acyclic graph. Each commit object contains:
- The hash of its parent commit (or multiple parents for a merge)
- The author name, email, and timestamp
- The committer name, email, and timestamp
- The commit message
- The hash of the tree object (snapshot of the entire repository at that point)

When you run `git log`, Git starts at the current HEAD, reads the commit object, prints it, then follows the parent pointer to the next commit, and repeats.

### Step 2: Linear History with git log (no flags)

A plain `git log` walks the ancestry chain from HEAD, one commit at a time, following the single parent pointer. It prints the full hash, author, date, and message for each commit. Internally, Git reads the commit object from `.git/objects/` using the hash as the filename, decompresses it with zlib, and formats the output.

```
$ git log --oneline
a1b2c3d fix: resolve login redirect bug
b2c3d4e feat: add user authentication module
c3d4e5f chore: initial project setup
```

When you specify `--oneline`, Git abbreviates the hash to 7 characters (configurable with `--abbrev=12`) and prints only the subject line of the message.

### Step 3: Branching and Merging with --graph --all

With branches and merges, the DAG has fork points (a commit with multiple children) and merge points (a commit with multiple parents). The `--graph` flag draws ASCII lines showing this structure. `--all` tells Git to traverse not just from HEAD but from every ref (branches, tags, remote references) — this reveals commits that are not on the current branch.

<AsciiDiagram id="git-log-and-history/dag-graph" title="How --graph --all visualizes branches and merges" content={`
  $ git log --graph --oneline --all

  |\
  | * d4e5f6g feat: add search functionality
  | * e4d5c6b feat: add search UI component
  * | c3d4e5f fix: resolve login redirect bug

  |/

`} alt="Diagram showing how git log --graph --oneline --all visualizes branches, merges, and the commit DAG with timeline" />

Internally, `--graph` works by maintaining a column map of active branches. At each commit, it checks how many branches are currently diverged, draws the appropriate connector symbols (`|`, `\`, `/`, `*`), and updates the column map when branches merge or end.

### Step 4: Filtering with --since, --author, and --grep

These flags do not change which commits Git traverses — they change which commits Git prints. Git walks the entire ancestry chain but applies filters before outputting each commit:

- `--since` and `--until` compare the commit's author timestamp against the provided date. Timestamps are stored as Unix epoch seconds, so Git converts your date string to epoch internally.
- `--author` performs a case-sensitive substring match against the author name and email fields. Use `-i` for case-insensitive matching combined with `--grep`.
- `--grep` searches the commit message body. When combined with `--all-match`, all `--grep` patterns must match.
- `-S` (pickaxe) is fundamentally different: instead of filtering commits, Git recomputes the diff for each commit and checks whether the number of occurrences of the given string changed. This is significantly slower but finds structural changes that `--grep` misses.

```
$ git log --oneline --since="2026-06-01" --until="2026-06-28" --author="maria"
a1b2c3d fix: resolve login redirect bug
b2c3d4e feat: add user authentication module
```

### Step 5: Inspecting Changes with -p and --stat

The `-p` flag tells Git to output the diff for each commit. Internally, Git reads the tree object of the current commit and its parent, then performs a tree diff algorithm that recursively compares every file (by its blob hash). Files with different hashes are output as patch hunks using the same algorithm as `git diff`.

`--stat` is lighter: Git compares tree objects but only counts files changed, insertions, and deletions — it does not generate the full patch text.

```
$ git log --oneline --stat -2
a1b2c3d fix: resolve login redirect bug
 src/auth/login.js | 5 +++--
 src/config.js     | 1 +
 2 files changed, 4 insertions(+), 2 deletions(-)

b2c3d4e feat: add user authentication module
 src/auth/login.js    | 45 +++++++++++++++++++++++++++++++++
 src/auth/session.js  | 30 ++++++++++++++++++++++++
 2 files changed, 75 insertions(+)
```

### Step 6: Comparing Points with git diff and Inspecting with git show

`git diff` does not traverse history — it reads two tree objects (or a tree and the working directory) and produces a patch. When you run `git diff main..feature-x`, Git resolves `main` to its commit hash, reads its tree, then resolves `feature-x` and reads its tree, then computes the difference.

`git show` is a combination of `git log -1` and `git diff`: it prints the metadata for a single commit and the patch diff against its parent. For merge commits, `git show` shows a combined diff by default (only lines that differ from all parents).

## When to Use This / When Not To

| Scenario | Use | Why |
|---|---|---|
| Find when a bug was introduced | `git log -S"buggyFunction" --oneline` or `git bisect` | Pickaxe finds the commit that changed the string; bisect narrows via binary search |
| See what a teammate worked on this week | `git log --author="alice" --since="1 week ago" --oneline` | Filters by person and time window |
| Understand project branching structure | `git log --graph --oneline --all --decorate` | Shows the full DAG with branch labels |
| Review all changes in a feature branch | `git diff main..feature-branch` | Single diff of entire branch vs main |
| Check what is staged for commit | `git diff --staged` | Shows only staged changes, not working tree |
| Who changed a specific file last | `git log -n 5 --oneline -- src/app.js` | Path filter limits history to that file |
| View a quick summary of recent commits | `git log --oneline -10` | Concise 10-commit overview |
| Debug a regression across many commits | `git log -p -- src/auth/` | Patch view limited to relevant directory |
| Search commit messages for a ticket number | `git log --grep="JIRA-1234" --oneline` | Keyword search across all messages |

| Don't Use | Instead |
|---|---|
| `git log` for a single commit's full details | `git show <hash>` — gives metadata and diff in one command |
| `git log -p` across 500+ commits | Narrow by path, author, or date first — the output volume is unmanageable |
| `git log --graph` on a repo with 50+ parallel branches | Use a GUI tool (GitKraken, Git Desktop, `gitk`) — the ASCII graph becomes a jumbled wall of characters |
| `git diff` when you want per-commit history | `git log -p` — `diff` shows the aggregate difference, not the step-by-step changes |
| `git log` commands that pipe into `grep` for complex filtering | Use `--grep`, `--author`, `-S` natively — they are faster and handle edge cases (binary files, merge commits) correctly |
