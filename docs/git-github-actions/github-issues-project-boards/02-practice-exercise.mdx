You will create issues, organize them with labels and milestones, set up a project board, link issues to PRs, and automate board movement. This lab requires a GitHub account and repository.

## Setup

```bash
# Create a repo (replace 'your-username' with your actual username)
gh repo create issues-lab --public --clone
cd issues-lab
git config user.name "Your Name"
git config user.email "your.email@example.com"
```

Create the starting code:

```bash
cat > app.py << 'EOF'
def add(a, b):
    return a + b


def subtract(a, b):
    return a - b
EOF
git add app.py
git commit -m "Initial calculator"
git push origin main
```

## The Walkthrough

### Step 1: Create labels

```bash
gh label create bug --color d73a4a --description "Something is not working"
gh label create enhancement --color a2eeef --description "New feature or request"
gh label create documentation --color 0075ca --description "Documentation changes"
gh label create good-first-issue --color 7057ff --description "Good for newcomers"
gh label create priority-high --color d93f0b --description "Needs immediate attention"
gh label create priority-low --color bfd4f2 --description "Low priority"
```

Expected output:

```
✓ Created label bug
✓ Created label enhancement
✓ Created label documentation
✓ Created label good-first-issue
✓ Created label priority-high
✓ Created label priority-low
```

### Step 2: Create a milestone

```bash
gh api repos/:owner/:repo/milestones \
  --method POST \
  --field title="v1.0" \
  --field description="Initial release" \
  --field due_on="2026-08-01T00:00:00Z"
```

Expected output — shows the milestone JSON with id, number, and title.

### Step 3: Create issues

Create a bug report:

```bash
gh issue create \
  --title "Divide function crashes on zero division" \
  --label bug \
  --label priority-high \
  --milestone "v1.0" \
  --body "## Steps to Reproduce
1. Call divide(5, 0)
2. Observe crash

## Expected
Graceful error handling or return None

## Actual
ZeroDivisionError: division by zero"
```

Expected output:

```
https://github.com/your-username/issues-lab/issues/1
```

Create a feature request:

```bash
gh issue create \
  --title "Add multiply function" \
  --label enhancement \
  --label good-first-issue \
  --milestone "v1.0" \
  --body "## Problem
Calculator needs a multiply function.

## Proposed Solution
Add multiply(a, b) that returns a * b

## Acceptance Criteria
- [ ] Function exists
- [ ] Returns correct product
- [ ] Handles large numbers"
```

Expected output:

```
https://github.com/your-username/issues-lab/issues/2
```

Create a documentation issue:

```bash
gh issue create \
  --title "Add usage examples to README" \
  --label documentation \
  --label priority-low \
  --body "Add code examples showing how to use each function."
```

Expected output:

```
https://github.com/your-username/issues-lab/issues/3
```

### Step 4: Create a project board

```bash
gh project create --owner "@me" --title "Sprint 1" --repos issues-lab
```

Expected output — shows the project ID and URL. Note the project ID (a number) for the next step.

### Step 5: Add issues to the project board

Get the project ID:

```bash
gh project list --owner "@me"
```

Note the project number (e.g., 1). Add issues to it:

```bash
# Add issue 1 to the project board
gh project item-add 1 --url "$(gh issue view 1 --json url --jq .url)"
gh project item-add 1 --url "$(gh issue view 2 --json url --jq .url)"
gh project item-add 1 --url "$(gh issue view 3 --json url --jq .url)"
```

### Step 6: Work on an issue — link to a PR

Create a branch for issue #1:

```bash
git switch -c fix-zero-division
```

Fix the divide function:

```bash
cat > app.py << 'EOF'
def add(a, b):
    return a + b


def subtract(a, b):
    return a - b


def divide(a, b):
    if b == 0:
        return None
    return a / b
EOF
git add app.py
git commit -m "Fix divide function to handle zero division"
git push origin fix-zero-division
```

Create a PR that references the issue:

```bash
gh pr create \
  --title "Fix divide function to handle zero division" \
  --body "Fixes #1

Adds a check for zero division. The function now returns None
instead of crashing when the divisor is 0." \
  --assignee @me
```

Expected output:

```
https://github.com/your-username/issues-lab/pull/4
```

The phrase `Fixes #1` creates a link. When this PR is merged, issue #1 auto-closes.

### Step 7: Merge the PR and verify the issue closes

```bash
gh pr merge 4 --squash --delete-branch
```

Verify the issue is closed:

```bash
gh issue view 1
```

Expected output — shows `state: CLOSED` and the closing commit reference.

### Step 8: Organize with issue templates

Create an issue template directory:

```bash
mkdir -p .github/ISSUE_TEMPLATE
```

Create a bug report template:

```bash
cat > .github/ISSUE_TEMPLATE/bug_report.md << 'EOF'
---
name: Bug report
about: Create a report to help us improve
title: "[BUG] "
labels: bug
assignees: ''
---

## Describe the Bug
A clear and concise description of what the bug is.

## To Reproduce
Steps to reproduce the behavior:
1. Go to '...'
2. Click on '...'
3. See error

## Expected Behavior
A clear description of what you expected to happen.

## Environment
- OS: [e.g. Windows 11, macOS 14, Ubuntu 22.04]
- Browser: [e.g. Chrome 120, Firefox 121]
- Version: [e.g. v1.0.0]

## Additional Context
Add any other context about the problem here.
EOF
```

Create a feature request template:

```bash
cat > .github/ISSUE_TEMPLATE/feature_request.md << 'EOF'
---
name: Feature request
about: Suggest an idea for this project
title: "[FEATURE] "
labels: enhancement
assignees: ''
---

## Problem Statement
A clear description of what problem this feature would solve.

## Proposed Solution
A clear description of what you want to happen.

## Alternatives Considered
Any alternative solutions or features you have considered.

## Acceptance Criteria
- [ ] Criterion 1
- [ ] Criterion 2
EOF
```

Commit and push:

```bash
git add .github/
git commit -m "Add issue templates"
git push origin main
```

Now when anyone creates a new issue in your repository, GitHub shows a template chooser with "Bug report" and "Feature request" options.

## Common Errors at This Stage

### Error 1: `gh project create: GraphQL: Could not resolve to a ProjectV2`

You tried to create a project board but the `--owner` flag is incorrect, or you do not have permission to create projects in the specified owner.

**Fix:** Use `@me` for your personal account: `gh project create --owner "@me" --title "Sprint 1"`. For organization projects, use the organization name: `gh project create --owner "my-org" --title "Sprint 1"`.

### Error 2: `gh label create: Name already exists`

You tried to create a label that already exists in the repository. Labels are unique per repository.

**Fix:** Use `gh label list` to see existing labels. Edit the existing label with `gh label edit <name>` or use a different name.

### Error 3: Issue does not auto-close when PR merges

You wrote `Fixes #1` in the PR description but the issue did not close. This happens when the PR is merged to a different branch than the one specified, or the keyword is not recognized.

**Fix:** The keyword must be in the PR body (not the title) and reference the issue by number. Use `Fixes`, `Closes`, `Resolves`, `Fix`, `Close`, or `Resolve`. If the PR is already merged, close the issue manually: `gh issue close 1 --comment "Fixed in PR #4"`.

### Error 4: Project board column does not automatically update

You added an issue to a project board, but moving it to "In Progress" when a PR is opened did not happen automatically.

**Fix:** Configure GitHub Actions automation for the project board. Create a workflow file that listens for `pull_request` events and moves the linked issue. See the "[Automating project boards](https://docs.github.com/en/issues/planning-and-tracking-with-projects/automating-your-project)" guide for built-in workflows.

## What You Just Learned

- `gh issue create` creates a trackable issue with labels, assignees, and milestones
- `gh label create` manages custom labels for categorizing issues
- `gh project create` creates a project board for tracking work visually
- Issues link to PRs via keywords like `Fixes #42` — auto-close when the PR merges
- Issue templates standardize bug reports and feature requests
- `gh project item-add` adds issues to project boards
- Milestones group issues by target release or sprint
