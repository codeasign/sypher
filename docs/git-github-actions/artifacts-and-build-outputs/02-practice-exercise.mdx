You will create a workflow that builds a project, uploads artifacts, downloads them in a downstream job, and caches dependencies for faster builds. By the end you will understand how to move files between jobs and optimize build times.

## Setup

```bash
# Create a new repository for this practice
gh repo create artifacts-practice --public --clone
```

```bash
# Create the workflows directory
mkdir -p .github/workflows
```

```bash
# Create a project with build output
mkdir -p src dist tests

cat > src/app.py << 'EOF'
def greet(name):
    return f"Hello, {name}!"

def add(a, b):
    return a + b

if __name__ == "__main__":
    print(greet("GitHub Actions"))
EOF

cat > tests/test_app.py << 'EOF'
from src.app import greet, add

def test_greet():
    assert greet("World") == "Hello, World!"

def test_add():
    assert add(2, 3) == 5
EOF

cat > requirements.txt << 'EOF'
pytest>=7.0
pytest-cov>=4.0
EOF

cat > Makefile << 'EOF'
.PHONY: build test

build:
	mkdir -p dist
	echo "#!/usr/bin/env python3" > dist/app
	cat src/app.py >> dist/app
	chmod +x dist/app
	@echo "Build complete: dist/app"

test:
	pytest tests/ -v --cov=src --cov-report=html --cov-report=xml
	@echo "Tests complete. Reports in htmlcov/ and coverage.xml"
EOF
```

## The Walkthrough

### Step 1: Build and upload artifacts

Create `.github/workflows/artifacts.yml`:

```yaml
name: Artifacts Demo
on: workflow_dispatch

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - name: Install dependencies
        run: pip install -r requirements.txt
      - name: Build the application
        run: make build
      - name: Upload build artifact
        uses: actions/upload-artifact@v4
        with:
          name: built-app
          path: dist/
          if-no-files-found: error
```

**Checkpoint:** Commit and push:

```bash
git add .
git commit -m "Add build and artifact upload"
git push origin main
```

Go to the Actions tab, trigger the workflow manually, and verify the artifact appears in the run summary.

### Step 2: Run tests and upload reports

Add a `test` job that depends on the `build` job:

```yaml
name: Artifacts Demo
on: workflow_dispatch

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - name: Install dependencies
        run: pip install -r requirements.txt
      - name: Build the application
        run: make build
      - name: Upload build artifact
        uses: actions/upload-artifact@v4
        with:
          name: built-app
          path: dist/
          if-no-files-found: error

  test:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - name: Install dependencies
        run: pip install -r requirements.txt
      - name: Run tests with coverage
        run: make test
      - name: Upload coverage report
        uses: actions/upload-artifact@v4
        with:
          name: coverage-report
          path: |
            htmlcov/
            coverage.xml
          retention-days: 30
```

**Checkpoint:** Push and run the workflow. Two artifacts should appear: `built-app` and `coverage-report`.

### Step 3: Download artifact in a deploy job

Add a `deploy` job that downloads the `built-app` artifact:

```yaml
name: Artifacts Demo
on: workflow_dispatch

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - name: Install dependencies
        run: pip install -r requirements.txt
      - name: Build the application
        run: make build
      - name: Upload build artifact
        uses: actions/upload-artifact@v4
        with:
          name: built-app
          path: dist/
          if-no-files-found: error

  test:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - name: Install dependencies
        run: pip install -r requirements.txt
      - name: Run tests with coverage
        run: make test
      - name: Upload coverage report
        uses: actions/upload-artifact@v4
        with:
          name: coverage-report
          path: |
            htmlcov/
            coverage.xml
          retention-days: 30

  deploy:
    needs: [build, test]
    runs-on: ubuntu-latest
    steps:
      - name: Download built app
        uses: actions/download-artifact@v4
        with:
          name: built-app
          path: ./downloaded-app
      - name: Deploy the application
        run: |
          echo "Deploying from downloaded artifact..."
          ls -la ./downloaded-app/
          ./downloaded-app/app
          echo "Deployment complete!"
```

### Step 4: Use cache for faster dependency installation

Add caching to speed up pip installs:

```yaml
name: Artifacts Demo
on: workflow_dispatch

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Cache pip dependencies
        uses: actions/cache@v4
        with:
          path: ~/.cache/pip
          key: pip-${{ hashFiles('requirements.txt') }}
          restore-keys: |
            pip-
      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - name: Install dependencies
        run: pip install -r requirements.txt
      - name: Build the application
        run: make build
      - name: Upload build artifact
        uses: actions/upload-artifact@v4
        with:
          name: built-app
          path: dist/
          if-no-files-found: error

  test:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Cache pip dependencies
        uses: actions/cache@v4
        with:
          path: ~/.cache/pip
          key: pip-${{ hashFiles('requirements.txt') }}
          restore-keys: |
            pip-
      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - name: Install dependencies
        run: pip install -r requirements.txt
      - name: Run tests with coverage
        run: make test
      - name: Upload coverage report
        uses: actions/upload-artifact@v4
        with:
          name: coverage-report
          path: |
            htmlcov/
            coverage.xml
          retention-days: 30

  deploy:
    needs: [build, test]
    runs-on: ubuntu-latest
    steps:
      - name: Download built app
        uses: actions/download-artifact@v4
        with:
          name: built-app
          path: ./downloaded-app
      - name: Deploy the application
        run: |
          echo "Deploying from downloaded artifact..."
          ls -la ./downloaded-app/
          ./downloaded-app/app
          echo "Deployment complete!"
```

**Checkpoint:** Run the workflow twice. The first run installs dependencies from scratch. The second run uses the cache — the install step should be significantly faster.

### Step 5: Download all artifacts at once

Replace the deploy job's download step to download all artifacts:

```yaml
  deploy:
    needs: [build, test]
    runs-on: ubuntu-latest
    steps:
      - name: Download all artifacts
        uses: actions/download-artifact@v4
        with:
          path: ./artifacts
      - name: List downloaded artifacts
        run: |
          echo "=== Downloaded artifacts ==="
          ls -la ./artifacts/
          echo ""
          echo "=== built-app ==="
          ls -la ./artifacts/built-app/
          echo ""
          echo "=== coverage-report ==="
          ls -la ./artifacts/coverage-report/
```

When you omit the `name` input, `download-artifact` downloads ALL artifacts from the current workflow run into separate subdirectories.

### Step 6: Use a matrix with unique artifact names

Add a matrix strategy that produces uniquely-named artifacts:

```yaml
name: Artifacts Demo
on: workflow_dispatch

jobs:
  build:
    runs-on: ${{ matrix.os }}
    strategy:
      matrix:
        os: [ubuntu-latest, macos-latest]
        python-version: ['3.10', '3.11']
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: ${{ matrix.python-version }}
      - name: Build
        run: |
          mkdir -p dist
          echo "Built on ${{ runner.os }} with Python ${{ matrix.python-version }}" > dist/version.txt
      - name: Upload unique artifact
        uses: actions/upload-artifact@v4
        with:
          name: build-${{ runner.os }}-${{ matrix.python-version }}
          path: dist/

  deploy:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - name: Download all builds
        uses: actions/download-artifact@v4
        with:
          path: ./all-builds
      - name: List all builds
        run: |
          echo "=== All builds ==="
          find ./all-builds -name "version.txt" -exec cat {} \;
```

Each matrix variant uploads a uniquely-named artifact. The deploy job downloads all of them.

## Common Errors at This Stage

### Error 1: "No files were found with the provided path"
**Message:** `No files were found with the provided path: dist/. No artifacts will be uploaded.`

**Why:** The `path` you specified does not exist on the runner. Either the build step failed, or the path is wrong.

**Fix:** Verify the directory exists by listing it before the upload step:
```yaml
- name: Verify build output
  run: ls -la dist/
- name: Upload
  uses: actions/upload-artifact@v4
  with:
    name: built-app
    path: dist/
```

### Error 2: "Artifact already exists"
**Message:** `An artifact with the name "built-app" already exists for this workflow run.`

**Why:** You uploaded two artifacts with the same name. When using a matrix or multiple jobs, each artifact name must be unique.

**Fix:** Include the matrix variables or job name in the artifact name:
```yaml
name: build-${{ matrix.os }}-${{ matrix.python-version }}
```

### Error 3: Download artifact fails with 404
**Message:** `Artifact "built-app" was not found for this workflow run.`

**Why:** The artifact was never uploaded because the upload job failed, or the artifact was deleted due to retention policy.

**Fix:** Check that the upload job completed successfully. Verify the artifact exists in the Actions tab. Increase retention if needed:
```yaml
- uses: actions/upload-artifact@v4
  with:
    name: built-app
    path: dist/
    retention-days: 90
```

### Error 4: "Cache not found" warning
**Message:** `Cache not found for input keys: pip-...`

**Why:** This is a warning, not an error. The cache key did not match any existing cache entry. This is expected on the first run.

**Fix:** No action needed — the cache will be created after the first successful run and will be available for subsequent runs. The `restore-keys` fallback with `pip-` prefix will match any earlier cache even if the hash changed.

## What You Just Learned

- `actions/upload-artifact@v4` compresses and uploads files from the runner
- `actions/download-artifact@v4` fetches and extracts artifacts in downstream jobs
- Artifacts persist for a configurable retention period (default 90 days)
- Artifacts are accessible from the Actions tab after the workflow completes
- `actions/cache@v4` speeds up dependency installation across runs
- Matrix jobs need unique artifact names to avoid conflicts
- You can download all artifacts at once by omitting the `name` input
