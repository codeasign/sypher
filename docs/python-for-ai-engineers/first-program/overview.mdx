# Your First Python Program

⏱ 8 minutes · Difficulty: Beginner

## What You Will Learn

By the end of this lesson you will be able to:

- Write a Python script that accepts user input and prints output
- Understand the structure of a Python source file
- Use comments, expressions, and statements correctly
- Run a script from the command line and spot basic syntax errors

## Prerequisites

You should already understand:

- [How to install Python and run a terminal](https://docs.python.org/3/using/cmdline.html) — covered in the previous lesson
- What a text editor is — any editor works; VS Code recommended

## Why This Matters

Your first program is more than a tradition. It introduces the fundamental shape of every Python file you will write — statements evaluated top to bottom, expressions producing values, and `print()` as your window into what the code is doing. AI engineering work is full of scripts: preprocess a dataset, call an API and log the response, evaluate model outputs. Every one of those scripts follows the same structure as a ten-line "hello world." Mastering the basics of script structure means you can read and modify any Python file you encounter.

In AI teams, you will constantly run and re-run scripts with different parameters. The ability to write a script that accepts command-line arguments or interactive input is something you use every single day, from tweaking a prompt template to adjusting the batch size for evaluation.

## The Problem

You have a text editor and a terminal open. What do you actually type? Without a mental model of how a Python file is structured — what goes where, how data flows from one line to the next — the blank file is intimidating. Many beginners copy-paste code they do not understand, then cannot debug it when it breaks. You need a clear, repeatable template for any Python script.

## The Concept

A Python script is a plain text file containing a sequence of **statements**. Python executes them one at a time from top to bottom. A statement can be:

- An **expression** evaluated for its value: `2 + 2`
- A **function call** that does something: `print("hello")`
- An **assignment** that stores a value in a variable: `name = "world"`
- A **control structure** that changes flow: `if`, `for`, `while`

Comments — lines starting with `#` — are ignored by Python. Use them to explain why, not what. A script starts running at line 1 and continues until it reaches the end of the file, an `exit()`, or an unhandled error.
