# Strings — Review

⏱ 3 minutes

## Key Takeaways

- Strings are immutable sequences — every method returns a new string; the original is unchanged.
- Index and slice with `s[start:stop:step]` — stop is exclusive, negative indices count from the end.
- f-strings (`f"value: {var}"`) are the preferred formatting method in modern Python.
- Use `.split()` (no args) for whitespace splitting; use `",".join(list)` to assemble strings from parts.
- Always `.strip()` user input to remove accidental leading or trailing whitespace.

## Recap in 60 Seconds

If you only remember three things:

1. f-strings are your default for building strings — `f"Model: {name} | Score: {score:.2f}"` is readable and fast.
2. Strings never change in place — slicing, methods, and concatenation all create new strings.
3. Build large strings with a list and `.join()`, not with `+=` in a loop.
