# Context Managers — Common Mistakes

⏱ 5 minutes

## When NOT to Use a Custom Context Manager

- **Simple pairs of calls:** If you just call `open()` and `close()`, the built-in `with open(...)` already handles it.
- **One-off blocks:** If you only need the pattern once, do not create a reusable context manager — a simple try/finally is clearer.
- **Non-resource patterns:** Context managers are for setup/teardown. If the "before" and "after" are not related to resource management, a function call pair may be simpler.

## The Pattern to Remember

- **Pattern:** Use `with` for guaranteed cleanup. Write custom managers with `@contextmanager` (generator style) or `__enter__/__exit__` (class style).
- **When to use:** Any resource that needs cleanup — files, locks, connections, temporary state changes, timing.
- **When NOT to use:** One-off pairs of calls, simple operations that do not need cleanup guarantees.
- **Mental model:** A context manager is a safety net — it catches every exit path (normal, exception, return, break) and runs cleanup.

## Common Mistakes

### Mistake 1: Not Yielding in a `@contextmanager` Function

```python
@contextmanager
def bad_timer():
    start = time.time()
    # Forgot yield — the block body never runs!
    print(f"Took {time.time() - start:.2f}s")

with bad_timer():
    print("This never prints")
```

The function must `yield` exactly once. Without the yield, the block body never executes.

### Mistake 2: Swallowing Exceptions Accidentally

```python
def __exit__(self, exc_type, exc_val, exc_tb):
    # Don't return True unless you mean to suppress!
    return True  # Suppresses all exceptions

with my_manager():
    raise RuntimeError("Something went wrong")
# Exception is silently swallowed — you never see it
```

**Fix:** Return `False` or `None` (implicitly) to propagate exceptions. Only return `True` when you intentionally want to suppress specific exceptions.

### Mistake 3: Not Handling Exceptions in `@contextmanager`

```python
@contextmanager
def fragile():
    print("Setup")
    yield
    print("Teardown")  # This line never runs if the block raises!

with fragile():
    raise ValueError("Oops")
# Prints "Setup" but NOT "Teardown"
```

**Fix:** Use try/finally:

```python
@contextmanager
def robust():
    print("Setup")
    try:
        yield
    finally:
        print("Teardown")  # Always runs
```

### Mistake 4: Reusing a Context Manager Object

A context manager instance should be used once. Reusing it after the block exits has undefined behaviour:

```python
cm = MyManager()
with cm:
    pass
with cm:  # May or may not work depending on implementation
    pass
```

### Mistake 5: Forgetting That `__exit__` Can Suppress Exceptions

If `__exit__` returns `True`, any exception in the block is suppressed. This is intentional for `contextlib.suppress`, but accidental if you return `True` from a cleanup method without meaning to.

```python
def __exit__(self, exc_type, exc_val, exc_tb):
    self.cleanup()
    # Implicit return None — exceptions propagate
```
