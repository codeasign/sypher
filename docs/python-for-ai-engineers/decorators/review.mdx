# Decorators — Review

⏱ 3 minutes

## Key Takeaways

- A decorator is a function that wraps another function to add behaviour — use the `@decorator` syntax to apply it.
- Always use `@functools.wraps(func)` on the wrapper to preserve the original function's `__name__` and `__doc__`.
- The wrapper must accept `*args, **kwargs` and `return func(*args, **kwargs)` to pass arguments and results through.
- Decorators with arguments need three levels of nesting: `decorator(args) -> inner(func) -> wrapper(*args, **kwargs)`.
- Decorators apply bottom-up when stacked — the closest to the function runs first.

## Recap in 60 Seconds

If you only remember three things:

1. Write every decorator with the boilerplate: `def decorator(func):` → `@functools.wraps(func)` → `def wrapper(*args, **kwargs):` → `return func(*args, **kwargs)` → `return wrapper`.
2. A decorator without arguments receives the function directly; a decorator with arguments needs an extra nesting level.
3. Use decorators for cross-cutting concerns (timing, logging, retry) — they keep your core functions clean and focused.
