# Setting Up Python for AI Engineering

⏱ 8 minutes · Difficulty: Beginner

## What You Will Learn

By the end of this lesson you will be able to:

- Install Python 3.12+ on your operating system
- Verify your installation and understand the Python version scheme
- Run the Python REPL interactively
- Write and execute your first Python script
- Choose an appropriate code editor for AI development

## Prerequisites

You should already understand:

- Basic computer literacy — how to open a terminal, install software, and navigate file systems
- [What a programming language is](https://docs.python.org/3/faq/general.html#what-is-python) — a general concept, no specific language needed

## Why This Matters

Every AI engineering workflow starts with a working Python installation. You will install libraries like `numpy`, `transformers`, `openai`, and `langchain`, run training scripts, launch Jupyter notebooks, and deploy inference servers. If your Python installation is wrong — wrong version, broken PATH, missing tools — nothing else works. Companies standardise on a Python version and distribution for their ML infrastructure, and knowing how to set this up cleanly is the first skill any AI team expects.

Debugging environment issues consumes a surprising amount of junior engineer time. Misaligned Python versions between development and production, accidental use of the system Python, and missing `pip` are among the most common onboarding problems at AI startups. By learning a repeatable, clean setup process now, you avoid hours of head-scratching later and establish a foundation you will use daily.

## The Problem

You write your first AI script, install a package with `pip install openai`, and run the script — only to see `ModuleNotFoundError: No module named 'openai'`. You check: you definitely installed it. You try again. Same error.

This is nearly always a Python environment mismatch: the `pip` you ran belongs to a different Python interpreter than the one executing your script. Without understanding how Python discovers interpreters and packages, you cannot diagnose or fix it. A clean, deliberate setup process prevents this class of problem entirely and is the single highest-leverage thing you can do as a beginner.

## The Concept

Python is an interpreted language. When you install Python, you get two critical executables: the interpreter (`python3` or `python`) that runs your code, and `pip` that installs packages from the Python Package Index (PyPI). Every Python installation has its own isolated site-packages directory where `pip` places libraries. If you install a package using one interpreter's `pip` but run your script with another interpreter, the package is invisible.

The solution is straightforward: install one canonical version of Python, ensure `pip` is paired with that interpreter, and use virtual environments to keep projects separate. This lesson walks you through that exact process.
