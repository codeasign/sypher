# Setting Up Python — Common Mistakes

⏱ 6 minutes

## When NOT to Use This Setup

The installation process above is for **local development**. When you deploy to production — Docker containers, cloud VMs, serverless functions — you will use a different approach: pre-built Docker images (e.g. `python:3.12-slim`) or conda environments for data science platforms. The local setup is for your development machine. For production, use containerised environments that are reproducible and isolated from the host OS.

## The Pattern to Remember

- **Pattern:** Install one Python version intentionally, keep `pip` paired with it, use virtual environments per project.
- **When to use:** Every Python project, always.
- **When NOT to use:** Production deployments (use Docker) or data science platforms that mandate conda.
- **Mental model:** Your Python interpreter is the engine; `pip` is the parts store; a virtual environment is a dedicated garage for each project's parts.

## Common Mistakes

### Mistake 1: Using the System Python

**Mistake:** Running `/usr/bin/python3` (macOS/Linux) — the version Apple or your Linux distro bundles for system tools.

**Why it happens:** `python3` resolves to the system Python because it is already on your PATH. You do not realise your manually installed Python exists but is not the default.

**Symptoms:** `pip install` succeeds but `import` fails. You cannot install certain packages because the system Python is older and lacks support.

**Fix:** Always check `python3 --version`. If it shows a version below 3.10 on a modern OS in 2026, you are using the system Python. Install a fresh Python and either adjust your PATH or use the full path (`python3.12`).

### Mistake 2: Forgetting to Add Python to PATH on Windows

**Mistake:** You installed Python but `python` is not recognised as a command.

**Why it happens:** The installer checkbox "Add Python to PATH" is unchecked by default. It is easy to miss when clicking through quickly.

**Symptoms:** `'python' is not recognized as an internal or external command` in PowerShell.

**Fix:** Re-run the installer and check the box. Or manually add `C:\Users\<you>\AppData\Local\Programs\Python\Python312\` and `C:\Users\<you>\AppData\Local\Programs\Python\Python312\Scripts\` to your system PATH via System Properties > Environment Variables.

### Mistake 3: Running pip Instead of pip3

**Mistake:** Running `pip install` when `pip` points to a Python 2 installation or a different Python version than your `python3`.

**Why it happens:** On some systems `pip` and `pip3` are different executables, especially if both Python 2 and 3 are installed.

**Symptoms:** A package installs without error but your Python 3 script cannot import it.

**Fix:** Prefer `python3 -m pip install <package>` — this always installs packages into the site-packages of the Python interpreter you are using. The `-m` flag runs `pip` as a module of that specific interpreter, guaranteeing correct pairing.

### Mistake 4: Installing Python from the Microsoft Store (Windows)

**Mistake:** Installing Python via the Microsoft Store.

**Why it happens:** Searching "Python" in Windows leads to the Store as the first result. It appears convenient.

**Symptoms:** File permission issues, scripts not launching from the terminal, difficulty installing native extension packages.

**Fix:** Uninstall the Store version and use the official installer from python.org. The official build has full filesystem access and proper PATH integration.

### Mistake 5: Not Restarting the Terminal After Installation

**Mistake:** Installing Python and immediately running `python3 --version` in the same terminal session.

**Why it happens:** You are eager to verify, but the terminal may have cached the old PATH from before installation.

**Symptoms:** You still see `command not found` even though the installer reported success.

**Fix:** Close and reopen your terminal completely. On Windows, PowerShell does not reload environment variables mid-session — you must start a new window. On macOS, `exec zsh -l` reloads the login shell without a full restart.
