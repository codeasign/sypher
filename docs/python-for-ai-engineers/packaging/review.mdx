# Packaging — Review

⏱ 3 minutes

## Key Takeaways

- A `pyproject.toml` file declares your package's name, version, dependencies, and entry points in a standard format.
- Use the `src/` layout to keep package code separate from tests and config files.
- Install locally with `pip install -e .` for editable development — no reinstalling on every change.
- Build distributions with `python -m build` and publish with `twine upload dist/*`.
- Automate version bumps with git tags and setuptools-scm. Never manually edit version numbers.

## Recap in 60 Seconds

If you only remember three things:

1. `pyproject.toml` is the single source of truth for your package — name, dependencies, entry points, build system.
2. Install in editable mode (`pip install -e .`) during development; build and publish with `build` + `twine` for releases.
3. Automate publishing in CI on tagged releases — never upload from your local machine.
