# Project Structure — Review

⏱ 3 minutes

## Key Takeaways

- Separate concerns into `src/`, `tests/`, `configs/`, `data/`, `notebooks/`, and `scripts/` directories.
- Make your project installable with `pyproject.toml` and `pip install -e .` so imports work from anywhere.
- Mirror `src/` structure inside `tests/` so finding the test for any module is trivial.
- Keep credentials out of config files — use environment variables or `.env` with strict `.gitignore`.
- Define your public API in `__init__.py` with `__all__` so consumers import from the package root.

## Recap in 60 Seconds

If you only remember three things:

1. An installable package (`pyproject.toml` + `pip install -e .`) eliminates path hacks and makes imports work everywhere.
2. One directory, one responsibility: source code in `src/`, tests in `tests/`, configuration in `configs/`, data in `data/`.
3. Never commit secrets — use environment variables and keep `.env` in `.gitignore`.
