## Why This Matters

Functions are the fundamental unit of organisation in Python. Every AI library you use is a collection of functions. Every script you write becomes maintainable when you break it into functions.

Good functions have a single responsibility, a clear name, and a predictable contract — you give them inputs, they return outputs. This makes your code testable, reusable, and understandable by others (and by your future self). In AI engineering, you will write functions to preprocess data, call APIs, parse responses, calculate metrics, and orchestrate pipelines.

## The Problem

You write a script to load data, clean it, run inference, and save results — all in one flat file with no functions. It works. Then you need to run the same inference on a different dataset. You copy-paste the inference code, customise it, and now you have two copies of the same logic. When the API changes, you need to find and update both copies. When you find a bug, you fix it in one place and forget the other.

Functions eliminate this by packaging reusable logic into named, callable units. One definition, one place to fix, one source of truth.

## The Concept

A function is a named block of code that takes inputs (**arguments**), performs work, and optionally produces a **return value**. Defining a function with `def` does not run it — it creates a callable object. Executing the function requires calling it with parentheses.

Variables defined inside a function are local — they do not affect variables outside the function with the same name. This **scoping** rule is what lets functions be self-contained and reusable without side effects.
