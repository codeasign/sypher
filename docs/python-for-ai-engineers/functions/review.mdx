# Functions — Review

⏱ 3 minutes

## Key Takeaways

- Functions encapsulate reusable logic — define with `def`, call with `()`, return with `return`.
- Use type hints (`param: type`, `-> ReturnType`) and docstrings to document function contracts — they are not enforced but help tooling and teammates.
- Never use mutable objects (lists, dicts) as default parameters — use `None` and create the mutable inside the function.
- Python uses **pass-by-reference** — modifying a mutable argument changes the original; copy before mutating to avoid side effects.
- Use `*args` for variable positional arguments, `**kwargs` for variable keyword arguments, and lambdas for short anonymous functions.

## Recap in 60 Seconds

If you only remember three things:

1. A function should do one thing, do it well, and have a name that says what that thing is — single responsibility.
2. Copy mutable arguments before modifying them inside a function, and never use `[]` or `{}` as default values.
3. Use type hints and docstrings — they are free documentation consumed by IDEs, linters, and teammates.
