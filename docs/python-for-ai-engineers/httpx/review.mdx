# HTTPX — Review

⏱ 3 minutes

## Key Takeaways

- Use `httpx.Client()` for synchronous requests (same API as `requests`) and `httpx.AsyncClient()` for async requests.
- Always use `async with httpx.AsyncClient() as client:` to ensure connections are closed.
- Use `asyncio.gather()` for concurrent requests and `asyncio.Semaphore` to limit concurrency.
- httpx supports HTTP/2 (`http2=True`), granular timeouts (`httpx.Timeout(connect=..., read=...)`), and streaming.
- The async client lets you run many API calls concurrently, completing a batch in roughly the time of the slowest single request.

## Recap in 60 Seconds

If you only remember three things:

1. Sync: `with httpx.Client() as c: c.get(url)`. Async: `async with httpx.AsyncClient() as c: await c.get(url)`.
2. For concurrent requests: create tasks, `await asyncio.gather(*tasks)`, limit with `Semaphore`.
3. Always set a `timeout`, always call `raise_for_status()`, always use `async with` for the async client.
