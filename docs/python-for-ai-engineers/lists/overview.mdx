## Why This Matters

Lists are the workhorse collection in Python. You will use them to hold batches of input data, collect model outputs, store configuration values, and manage sequences of tasks. When you run inference on 1000 samples, the results go in a list. When you need to process records one at a time and collect the results, you use a list. When you split a large text into chunks for an LLM, you store the chunks in a list.

AI work produces lists of things — predictions, embeddings, token sequences, file paths, evaluation scores. Understanding how to create, modify, and iterate over lists efficiently is fundamental to writing any substantial Python program.

## The Problem

You have 5000 customer reviews to classify. You call your model on each one and need to store the results: `["positive", "negative", "neutral", ...]`. Then you need to count how many are positive, filter out low-confidence predictions, and batch them for further processing. Without a collection type, you would need 5000 separate variables. Lists solve this by letting you store multiple values in a single ordered structure and operate on them with concise syntax.

## The Concept

A list is an **ordered, mutable sequence** of elements. Ordered means elements have a position (index 0, 1, 2, ...) and stay in the order you insert them. Mutable means you can add, remove, or change elements after creation — unlike strings. Lists can contain elements of any type, including other lists (nested lists).

Think of a list as a train. Each carriage holds one item. You can add a carriage to the end (append), insert a carriage in the middle, uncouple a carriage (remove), or look inside any carriage by its number. The train stays the same; what is inside the carriages can change.
