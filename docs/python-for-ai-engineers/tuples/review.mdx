# Tuples — Review

⏱ 3 minutes

## Key Takeaways

- Tuples are immutable sequences — once created, they cannot be changed. This makes them hashable and usable as dictionary keys.
- The comma creates a tuple, not the parentheses — a single-element tuple needs a trailing comma: `(42,)`.
- Unpacking (`a, b, c = tuple`) is the most common way to work with tuples — it is how Python handles multiple return values.
- Use tuples for fixed records (function return values, coordinates, version numbers) and lists for variable-length collections.
- Tuples have only two methods: `.count()` and `.index()`.

## Recap in 60 Seconds

If you only remember three things:

1. Tuples are immutable lists — use them when the data should not change. They can be dictionary keys; lists cannot.
2. Pack with `(a, b, c)` and unpack with `x, y, z = record` — this is how Python functions return multiple values.
3. Never forget the trailing comma for single-element tuples: `(42,)` is a tuple; `(42)` is an integer.
