# Dataclasses — Review

⏱ 3 minutes

## Key Takeaways

- `@dataclass` auto-generates `__init__`, `__repr__`, and `__eq__` from type-annotated fields.
- Use `field(default_factory=list)` for mutable defaults — never `= []`.
- Use `frozen=True` for immutable, hashable dataclasses that can be dict keys.
- Use `__post_init__` for validation and derived field computation.
- Convert to dict with `asdict()` for JSON serialisation.

## Recap in 60 Seconds

If you only remember three things:

1. `@dataclass` eliminates boilerplate — declare fields with type annotations and get `__init__`, `__repr__`, `__eq__` for free.
2. Always use `field(default_factory=list/dict)` for mutable defaults — bare `= []` creates a shared list across all instances.
3. Use `asdict()` to convert to a dict for serialisation, `__post_init__` for validation.
