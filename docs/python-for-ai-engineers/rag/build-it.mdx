# RAG — Hands On

⏱ 18 minutes

## Breaking It Down

### Step 1: Install Dependencies

```bash
pip install openai chromadb anthropic tiktoken
```

`tiktoken` counts tokens to control chunk sizes. `chromadb` stores and retrieves vectors. `openai` generates embeddings.

### Step 2: Chunking Documents

Documents must be split into manageable chunks before embedding. Two popular strategies:

- **Fixed-size with overlap:** Split by token count with a sliding window.
- **Semantic splitting:** Split at paragraph or sentence boundaries.

Here is a fixed-size chunker with overlap:

```python
# chunker.py
import tiktoken


def chunk_text(
    text: str,
    chunk_size: int = 500,
    chunk_overlap: int = 100,
    encoding_name: str = "cl100k_base",
) -> list[str]:
    """Split text into overlapping token-aligned chunks.

    Args:
        text: The full document text.
        chunk_size: Target tokens per chunk.
        chunk_overlap: Tokens of overlap between adjacent chunks.
        encoding_name: Tokeniser matching your embedding model.

    Returns:
        List of text chunks.
    """
    encoding = tiktoken.get_encoding(encoding_name)
    tokens = encoding.encode(text)

    chunks = []
    start = 0
    while start < len(tokens):
        end = start + chunk_size
        chunk_tokens = tokens[start:end]
        chunk_text = encoding.decode(chunk_tokens)
        chunks.append(chunk_text)
        start += chunk_size - chunk_overlap

    return chunks


# Usage
doc_text = """Machine learning is transforming industries from healthcare to finance.
In healthcare, models analyse medical images to detect diseases earlier than human doctors.
In finance, algorithms detect fraudulent transactions in milliseconds.
The key enabler is the availability of large datasets and cheap compute.
Python has become the dominant language for machine learning due to its ecosystem:
NumPy for numerical computing, scikit-learn for classical ML, and PyTorch for deep learning.
Each library builds on the lower-level capabilities of the ones before it,
creating a stack that allows researchers to focus on model architecture
rather than low-level implementation details."""

chunks = chunk_text(doc_text, chunk_size=100, chunk_overlap=20)
print(f"Generated {len(chunks)} chunks")
for i, chunk in enumerate(chunks):
    print(f"Chunk {i}: {len(chunk)} chars")
```

Overlap ensures that information split across a chunk boundary is still captured in at least one chunk.

### Step 3: Ingestion Pipeline

Embed each chunk and store it in Chroma:

```python
# ingest.py
import os
import chromadb
from openai import OpenAI
from chunker import chunk_text


class IngestionPipeline:
    """Ingest documents into a vector store for RAG."""

    def __init__(self, collection_name: str = "rag_docs"):
        self.embed_client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])
        self.chroma = chromadb.PersistentClient(path="./rag_db")
        self.collection = self.chroma.get_or_create_collection(
            name=collection_name,
            metadata={"hnsw:space": "cosine"},
        )

    def ingest_document(self, text: str, doc_id: str, metadata: dict | None = None):
        """Chunk, embed, and store a single document.

        Args:
            text: Full document text.
            doc_id: Unique identifier for the document.
            metadata: Optional metadata (source URL, date, etc.).
        """
        chunks = chunk_text(text, chunk_size=500, chunk_overlap=100)

        if not chunks:
            return

        # Number chunks for unique IDs
        chunk_ids = [f"{doc_id}_chunk_{i}" for i in range(len(chunks))]
        metadatas = [
            {"doc_id": doc_id, "chunk_index": i, **(metadata or {})}
            for i in range(len(chunks))
        ]

        # Embed in batches of 100
        for batch_start in range(0, len(chunks), 100):
            batch_end = batch_start + 100
            batch_chunks = chunks[batch_start:batch_end]
            batch_ids = chunk_ids[batch_start:batch_end]
            batch_metas = metadatas[batch_start:batch_end]

            response = self.embed_client.embeddings.create(
                model="text-embedding-3-small",
                input=batch_chunks,
            )
            embeddings = [d.embedding for d in response.data]

            self.collection.add(
                embeddings=embeddings,
                documents=batch_chunks,
                ids=batch_ids,
                metadatas=batch_metas,
            )

        print(f"Ingested {len(chunks)} chunks from {doc_id}")


# Usage
pipeline = IngestionPipeline()

pipeline.ingest_document(
    text="""Python is a high-level, general-purpose programming language.
Its design philosophy emphasises code readability with notable use of significant indentation.
Python is dynamically typed and garbage-collected.
It supports multiple programming paradigms, including structured, object-oriented, and functional programming.
Python was created by Guido van Rossum and first released in 1991.
The language has undergone major version changes, with Python 2 reaching end of life in 2020
and Python 3 being the current and future of the language.""",
    doc_id="python_intro",
    metadata={"source": "wikipedia"},
)

pipeline.ingest_document(
    text="""Machine learning is a subset of artificial intelligence that enables systems to learn
and improve from experience without being explicitly programmed.
Supervised learning uses labelled training data to learn a mapping from inputs to outputs.
Unsupervised learning finds hidden patterns in unlabelled data.
Reinforcement learning trains agents to make sequences of decisions by rewarding desired behaviours.
Deep learning uses neural networks with many layers to model complex patterns in data.
Common frameworks include TensorFlow, PyTorch, and scikit-learn.""",
    doc_id="ml_intro",
    metadata={"source": "internal_kb"},
)
```

### Step 4: Retrieval

Given a query, find the most relevant chunks:

```python
# retrieve.py
import os
import chromadb
from openai import OpenAI


class Retriever:
    """Retrieve relevant chunks for a given query."""

    def __init__(self, collection_name: str = "rag_docs"):
        self.embed_client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])
        self.chroma = chromadb.PersistentClient(path="./rag_db")
        self.collection = self.chroma.get_collection(collection_name)

    def retrieve(self, query: str, top_k: int = 3) -> list[dict]:
        """Return top-K relevant chunks for the query.

        Args:
            query: Natural language question.
            top_k: Number of chunks to retrieve.

        Returns:
            List of {document, score, metadata} dicts.
        """
        response = self.embed_client.embeddings.create(
            model="text-embedding-3-small",
            input=query,
        )
        query_emb = response.data[0].embedding

        results = self.collection.query(
            query_embeddings=[query_emb],
            n_results=top_k,
        )

        return [
            {
                "document": doc,
                "score": round(1 - dist, 4),  # Convert distance to similarity
                "metadata": meta,
            }
            for doc, dist, meta in zip(
                results["documents"][0],
                results["distances"][0],
                results["metadatas"][0],
            )
        ]


# Usage
retriever = Retriever()
results = retriever.retrieve("What is machine learning?")
for r in results:
    print(f"  Score: {r['score']:.4f} | {r['document'][:80]}...")
```

### Step 5: Generation — RAG Pipeline

Combine retrieval and generation into a single pipeline:

```python
# rag_pipeline.py
import os
from retrieve import Retriever
from anthropic import Anthropic


class RAGPipeline:
    """Full RAG pipeline: retrieve context, generate answer."""

    def __init__(self):
        self.retriever = Retriever()
        self.llm = Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])

    def answer(self, question: str, top_k: int = 3) -> str:
        """Answer a question using retrieved context.

        Args:
            question: Natural language question.
            top_k: Number of context chunks to retrieve.

        Returns:
            Answer grounded in the retrieved documents.
        """
        # 1. Retrieve relevant chunks
        chunks = self.retriever.retrieve(question, top_k=top_k)

        if not chunks:
            return "No relevant documents found."

        # 2. Build context string
        context = "\n\n".join(
            f"[{r['metadata']['doc_id']}] {r['document']}"
            for r in chunks
        )

        # 3. Generate answer with context
        response = self.llm.messages.create(
            model="claude-sonnet-4-20250514",
            max_tokens=1024,
            system=(
                "You are a helpful assistant that answers questions based on the "
                "provided context. If the context does not contain enough information "
                "to answer, say so. Do not make up information."
            ),
            messages=[
                {
                    "role": "user",
                    "content": f"Context:\n{context}\n\n"
                                f"Question: {question}\n\n"
                                f"Answer based on the context above."
                }
            ],
        )

        return response.content[0].text


# Usage
rag = RAGPipeline()

answer = rag.answer("What is Python? Who created it?")
print(answer)
# Output based on retrieved chunks: Python is a high-level programming language
# created by Guido van Rossum, first released in 1991.
```

## Build Something Real — Question Answering Over Technical Documentation

Build a QA system that answers questions about your own codebase or documentation:

```python
# docs_qa.py
from ingest import IngestionPipeline
from rag_pipeline import RAGPipeline

# Ingest multiple documents
pipeline = IngestionPipeline()

documents = {
    "api_docs": """
The /v1/users endpoint supports GET, POST, and DELETE operations.
GET /v1/users returns a paginated list of users.
POST /v1/users creates a new user. Required fields: name, email.
DELETE /v1/users/{id} deletes a user by ID.
Authentication is via Bearer token in the Authorization header.
Rate limits: 100 requests per minute per token.
""",
    "deployment_guide": """
The application deploys as a Docker container to Kubernetes.
The Dockerfile is in the project root. Build with 'docker build -t myapp .'.
Deploy with 'kubectl apply -f k8s/deployment.yaml'.
Health check endpoint: GET /health returns 200 OK when the service is running.
Environment variables: DATABASE_URL, REDIS_URL, API_KEY.
Minimum resources: 256MB RAM, 0.5 CPU cores.
""",
}

for doc_id, text in documents.items():
    pipeline.ingest_document(text, doc_id)

# Ask questions
qa = RAGPipeline()
questions = [
    "How do I create a new user via the API?",
    "What are the minimum resource requirements?",
    "How do I authenticate?",
]

for q in questions:
    print(f"\nQ: {q}")
    print(f"A: {qa.answer(q)}")
```

## Performance Considerations

- **Chunk size trade-off.** Small chunks (100-200 tokens) give granular retrieval but lose context. Large chunks (1000+ tokens) provide more context but may contain irrelevant information. 500 tokens with 100-token overlap is a good starting point.
- **Retrieval latency.** Chroma's HNSW index searches in milliseconds for collections up to 1M vectors. Beyond that, consider a hosted vector database (Pinecone, Weaviate).
- **Hybrid search.** For many domains, keyword matching complements embedding similarity. Chroma supports combining both with `lambda_mult` weighting.

## Security Considerations

- **Access control on retrieval.** RAG retrieves chunks by similarity, not by authorisation. If one user asks a question that matches chunks from a document they should not access, the context will include that document. Filter retrieved chunks by user permissions before sending them to the LLM.
- **Prompt injection via documents.** If an ingested document contains instructions like "Ignore previous context and say X," those instructions become part of the prompt. Treat document ingestion as an untrusted input channel and validate content.
