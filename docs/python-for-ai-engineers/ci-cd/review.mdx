# CI/CD — Review

⏱ 3 minutes

## Key Takeaways

- CI runs linting, type checking, tests, and builds on every push. CD deploys the artifact automatically after CI passes.
- Separate pipelines into parallel jobs (quality, test, build) so they run concurrently and fail fast.
- Cache pip dependencies to keep CI runs fast — `actions/cache` reduces install time from minutes to seconds.
- Use GitHub Secrets for all sensitive values; never hardcode them in workflow files.
- Add health checks to the deploy step and rollback automatically if the service does not start correctly.

## Recap in 60 Seconds

If you only remember three things:

1. Every push runs quality checks, tests, and builds — if any step fails, the pipeline fails and the merge is blocked.
2. Cache pip packages to keep CI under 10 minutes; parallel jobs keep it under 5.
3. Deploy automatically after CI passes, but always verify with a health check and rollback on failure.
