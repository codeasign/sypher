# Docker — Hands On

⏱ 15 minutes

## Breaking It Down

### Step 1: A Minimal Dockerfile

Start with a simple Python application:

```dockerfile
# Dockerfile (minimal)
FROM python:3.12-slim

WORKDIR /app

# Install dependencies first — layer caching
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Copy application code
COPY . .

# Default command
CMD ["python", "main.py"]
```

Corresponding `requirements.txt`:

```
anthropic>=0.30.0
openai>=1.0.0
chromadb>=0.4.0
pydantic>=2.0.0
python-dotenv>=1.0.0
```

Build and run:

```bash
docker build -t ai-app .
docker run --env-file .env ai-app
```

### Step 2: Multi-Stage Build for Smaller Images

Multi-stage builds separate the build environment from the runtime environment. This dramatically reduces image size:

```dockerfile
# Dockerfile (multi-stage)
# Stage 1: Build environment
FROM python:3.12-slim AS builder

WORKDIR /app
COPY requirements.txt .

RUN pip install --no-cache-dir --user -r requirements.txt

# Stage 2: Runtime
FROM python:3.12-slim AS runtime

WORKDIR /app

# Copy only the installed packages from builder
COPY --from=builder /root/.local /root/.local
# Make sure scripts in .local are in the PATH
ENV PATH=/root/.local/bin:$PATH

# Copy application code
COPY . .

CMD ["python", "main.py"]
```

The final image contains only the runtime (Python 3.12-slim + your code + installed packages), not build tools or intermediate artifacts.

```bash
docker build -t ai-app:latest .
docker images ai-app  # ~150MB vs ~1GB without multi-stage
```

### Step 3: Docker Compose for Multi-Service Apps

A RAG pipeline often needs an app service and a database service. Docker Compose runs both:

```yaml
# docker-compose.yml
version: "3.9"

services:
  app:
    build: .
    ports:
      - "8000:8000"
    env_file:
      - .env
    volumes:
      - ./rag_db:/app/rag_db  # Persist Chroma data
      - ./logs:/app/logs       # Persist logs
    depends_on:
      - chroma_db
    command: python rag_pipeline.py

  chroma_db:
    image: chromadb/chroma:latest
    ports:
      - "8001:8000"
    volumes:
      - chroma_data:/chroma/chroma
    environment:
      - IS_PERSISTENT=TRUE

volumes:
  chroma_data:
```

Run with:

```bash
docker compose up --build
```

### Step 4: Managing Secrets in Docker

API keys must never be baked into the image. Use environment variables at runtime:

```bash
# Wrong — API key is baked into the image
ENV ANTHROPIC_API_KEY=sk-ant-...

# Right — pass at runtime
docker run --env-file .env ai-app
```

For production, use Docker secrets (Swarm) or your orchestrator's secrets manager (Kubernetes Secrets, AWS ECS Secrets).

### Step 5: Health Checks and Graceful Shutdown

Production containers need health checks and signal handling:

```dockerfile
# Dockerfile (production-ready)
FROM python:3.12-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

# Health check — Docker restarts the container if this fails
HEALTHCHECK --interval=30s --timeout=5s --retries=3 \
  CMD python -c "import urllib.request; urllib.request.urlopen('http://localhost:8000/health')" || exit 1

EXPOSE 8000

CMD ["python", "api_server.py"]
```

Implement a health endpoint in your app:

```python
# api_server.py (health endpoint)
from http.server import HTTPServer, BaseHTTPRequestHandler
import json


class HealthHandler(BaseHTTPRequestHandler):
    def do_GET(self):
        if self.path == "/health":
            self.send_response(200)
            self.send_header("Content-Type", "application/json")
            self.end_headers()
            self.wfile.write(json.dumps({"status": "ok"}).encode())
        else:
            self.send_response(404)
            self.end_headers()


if __name__ == "__main__":
    server = HTTPServer(("0.0.0.0", 8000), HealthHandler)
    print("Health server running on port 8000...")
    server.serve_forever()
```

## Build Something Real — Dockerised RAG API

Package the RAG pipeline as a Dockerised API service:

```dockerfile
# Dockerfile (rag-api)
FROM python:3.12-slim AS builder

WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir --user -r requirements.txt

FROM python:3.12-slim

WORKDIR /app
COPY --from=builder /root/.local /root/.local
ENV PATH=/root/.local/bin:$PATH

# Create non-root user for security
RUN groupadd -r appuser && useradd -r -g appuser appuser

COPY --chown=appuser:appuser . .

USER appuser

HEALTHCHECK --interval=30s --timeout=5s --retries=3 \
  CMD python -c "import urllib.request; urllib.request.urlopen('http://localhost:8000/health')" || exit 1

EXPOSE 8000

CMD ["python", "rag_api.py"]
```

The associated `rag_api.py`:

```python
# rag_api.py
import os
import json
from http.server import HTTPServer, BaseHTTPRequestHandler
from rag_pipeline import RAGPipeline

rag = RAGPipeline()


class RAGHandler(BaseHTTPRequestHandler):
    def do_POST(self):
        if self.path != "/ask":
            self.send_response(404)
            self.end_headers()
            return

        content_length = int(self.headers.get("Content-Length", 0))
        body = self.rfile.read(content_length)
        request = json.loads(body)

        question = request.get("question", "")
        answer = rag.answer(question)

        self.send_response(200)
        self.send_header("Content-Type", "application/json")
        self.end_headers()
        self.wfile.write(json.dumps({"answer": answer}).encode())

    def do_GET(self):
        if self.path == "/health":
            self.send_response(200)
            self.send_header("Content-Type", "application/json")
            self.end_headers()
            self.wfile.write(json.dumps({"status": "ok"}).encode())


if __name__ == "__main__":
    port = int(os.environ.get("PORT", 8000))
    server = HTTPServer(("0.0.0.0", port), RAGHandler)
    print(f"RAG API running on port {port}...")
    server.serve_forever()
```

Run:

```bash
# Build
docker build -t rag-api .

# Run with API key
docker run -p 8000:8000 \
  -e ANTHROPIC_API_KEY=sk-ant-... \
  -e OPENAI_API_KEY=sk-... \
  -v ./rag_db:/app/rag_db \
  rag-api

# Query
curl -X POST http://localhost:8000/ask \
  -H "Content-Type: application/json" \
  -d '{"question": "What is Python?"}'
```

## Performance Considerations

- **Image size.** The `python:3.12-slim` base is ~120MB. Alpine-based images are smaller (~50MB) but may have compatibility issues with native extensions (Chroma, NumPy). Stick with slim unless you need minimal size.
- **Layer caching.** Order your Dockerfile from least-changing (base, pip install) to most-changing (application code). This maximises cache hits.
- **Build context.** Add a `.dockerignore` file to exclude `__pycache__`, `.env`, `node_modules`, and `.git` from the build context. Large build contexts slow down builds.

```
# .dockerignore
__pycache__
*.pyc
.env
.git
.gitignore
*.md
rag_db/
chroma_db/
```

## Security Considerations

- **Never bake secrets.** API keys in the image are extractable by anyone who can pull it. Use environment variables or a secrets manager.
- **Run as non-root.** Add a non-root user in the Dockerfile (`USER appuser`). Running as root inside the container is a privilege escalation risk.
- **Pin base image versions.** Use `python:3.12-slim` not `python:latest`. The `latest` tag changes, and a surprise update can break your build.
