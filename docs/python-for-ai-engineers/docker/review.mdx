# Docker — Review

⏱ 3 minutes

## Key Takeaways

- A Dockerfile packages your application with its entire runtime: OS, Python, packages, code — one image that runs everywhere.
- Use multi-stage builds: builder stage installs dependencies, runtime stage copies only what is needed. This halves or thirds your image size.
- Never bake secrets into images. Pass them as environment variables at runtime.
- Run as a non-root user inside the container for security.
- Add a `.dockerignore` file to exclude build artifacts, environment files, and version control directories.

## Recap in 60 Seconds

If you only remember three things:

1. A Docker image is a frozen, portable filesystem — if it runs in one environment, it runs in all of them.
2. Multi-stage builds keep images small; `.dockerignore` keeps builds fast and keeps secrets out.
3. Run as non-root, never bake in secrets, and pin base image versions to `3.12-slim` not `latest`.
