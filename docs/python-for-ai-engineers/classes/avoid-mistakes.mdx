# Classes — Common Mistakes

⏱ 6 minutes

## When NOT to Use a Class

- **Pure data containers:** Use a dataclass or a NamedTuple instead.
- **Single function with state:** If a class has one method plus `__init__`, consider a function with a closure.
- **Over-engineering:** Not everything needs a class. If a plain function can do the job, start there.

## The Pattern to Remember

- **Pattern:** `class Name:` → `__init__(self, ...)` → instance methods with `self` → use via `instance.method()`.
- **When to use:** Bundling data with behaviour, representing domain concepts (Model, Dataset, Pipeline).
- **When NOT to use:** Pure data (use dataclass), single-function wrappers (use plain function).
- **Mental model:** A class is a cookie cutter. The `__init__` method puts the dough in; methods bake and decorate.

## Common Mistakes

### Mistake 1: Forgetting `self` as the First Method Parameter

```python
class Model:
    def predict(input_text):  # Missing self!
        return f"Prediction: {input_text}"

m = Model()
m.predict("hello")  # TypeError: predict() takes 1 positional argument but 2 were given
```

Every instance method must have `self` as its first parameter. When you call `m.predict("hello")`, Python passes `m` as the first argument — so the method receives two arguments (`m` and `"hello"`) but only accepts one.

### Mistake 2: Modifying a Mutable Class Attribute Through an Instance

```python
class Model:
    registry = []  # Class attribute — shared

m1 = Model()
m2 = Model()
m1.registry.append("gpt-4")  # Modifies the class-level list!
print(m2.registry)  # ['gpt-4'] — shared!
```

**Fix:** If each instance needs its own copy, initialise in `__init__`:

```python
class Model:
    def __init__(self):
        self.registry = []  # Instance attribute — separate per instance
```

### Mistake 3: Confusing `@classmethod` and `@staticmethod`

- `@classmethod` receives `cls` — use it for alternative constructors or factory methods.
- `@staticmethod` receives nothing extra — use it for utility functions that belong in the class namespace.

### Mistake 4: Using Bare `__init__` Instead of a Property for Validation

```python
class Config:
    def __init__(self, lr):
        self.lr = lr  # No validation

cfg = Config(lr=-1.0)  # Invalid but allowed
```

**Fix:** Validate in `__init__` or use a property setter.

### Mistake 5: Returning a Value from `__init__`

```python
class Bad:
    def __init__(self):
        return 42  # TypeError: __init__ should return None
```

`__init__` must return `None`. If you want to control object creation, use `__new__` (rare) or a class method factory.
