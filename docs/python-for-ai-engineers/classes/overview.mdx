# Classes

⏱ 10 minutes · Difficulty: Intermediate

## What You Will Learn

By the end of this lesson you will be able to:

- Define a class with `class` and create instances with `ClassName()`
- Write an `__init__` constructor that sets up instance attributes
- Use `self` to refer to the current instance
- Define instance methods, class methods, and static methods
- Control attribute access with properties

## Prerequisites

You should already understand:

- [Functions](../functions/overview) — methods are functions attached to a class
- [Dataclasses](../dataclasses/overview) — dataclasses are a shortcut for data-oriented classes
- [Variables and scope](../variables/overview) — `self` distinguishes instance variables from local variables

## Why This Matters

Classes let you bundle data and behaviour together. In AI engineering, classes are how you represent concepts that have both state and actions: a `Model` class with attributes (name, version, parameters) and methods (`predict()`, `load()`, `evaluate()`); a `Dataset` class with attributes (`path`, `size`, `labels`) and methods (`shuffle()`, `batch()`, `preprocess()`).

While dataclasses handle pure data containers, classes with methods are the right tool whenever the data has associated behaviour. Frameworks like PyTorch (`nn.Module`), Hugging Face, and scikit-learn all use classes as their fundamental building block.

## The Problem

You have a model with configuration, weights, and the ability to make predictions. Without a class, you manage these as separate pieces: a dict for config, a file path for weights, and a function that takes both as arguments. The relationship between these pieces is implicit — nothing forces you to pass the right config with the right weights. A class bundles them together, making the relationship explicit and enforced by the language.

## The Concept

A **class** is a blueprint for creating objects. An **instance** is a concrete object created from that blueprint. The `__init__` method initialises each instance's attributes. The `self` parameter refers to the instance being created or operated on — it is how methods access the instance's own data.

Every method in a class automatically receives `self` as its first argument. You do not pass it explicitly — Python provides it when you call the method on an instance.
