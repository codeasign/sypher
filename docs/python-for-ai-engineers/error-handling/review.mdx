# Error Handling — Review

⏱ 3 minutes

## Key Takeaways

- Use `try`/`except` for operations that can fail — catch specific exceptions, never bare `except:`.
- Use `else` for code that should run only on success, and `finally` for cleanup that must always run.
- Create custom exception classes that inherit from `Exception` for domain-specific errors.
- Use `raise ... from e` to chain exceptions and preserve the original traceback.
- Never swallow exceptions silently — at minimum log them.

## Recap in 60 Seconds

If you only remember three things:

1. Be specific about what you catch — `except FileNotFoundError:` not `except:`.
2. Use `else` for success-only code and `finally` for cleanup that always runs.
3. Define custom exceptions that inherit from `Exception` for your application domain.
