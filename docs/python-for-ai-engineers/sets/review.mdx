# Sets — Review

⏱ 3 minutes

## Key Takeaways

- Sets store unique, hashable elements and provide O(1) membership testing.
- Create empty sets with `set()`, not `{}` (which creates an empty dict).
- Use set operations — `|` (union), `&` (intersection), `-` (difference), `^` (symmetric difference).
- Use `frozenset` when you need an immutable, hashable set — useful for set-of-set structures.
- Sets are unordered and cannot contain mutable elements like lists or dictionaries.

## Recap in 60 Seconds

If you only remember three things:

1. Use `set()` for fast membership checks — `x in my_set` is O(1), compared to O(n) for lists.
2. Deduplicate any collection by passing it to `set()`: `unique = set(duplicates)`.
3. Sets have no order, no duplicates, no indexing — if you need any of those, use a different collection.
