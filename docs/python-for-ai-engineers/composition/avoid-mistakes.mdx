# Composition — Common Mistakes

⏱ 5 minutes

## When NOT to Use Composition

- **Tightly coupled components:** If the composed objects are so interdependent that swapping them is impossible, composition adds complexity without benefit.
- **Performance-critical code:** Every delegation adds a method call overhead. In hot loops, inlining or direct access may be necessary.
- **Framework-required inheritance:** PyTorch requires inheriting from `nn.Module`; Django requires inheriting from `Model`. In these cases, framework constraints override the general preference for composition.

## The Pattern to Remember

- **Pattern:** Inject dependencies via `__init__`, delegate work to them, swap implementations via the same interface.
- **When to use:** Has-a relationships — a Service has a Backend, a Pipeline has Transformers, a Classifier has a Tokeniser.
- **When NOT to use:** Framework-required inheritance, tightly coupled components, performance-critical hot paths.
- **Mental model:** Composition is LEGO bricks — you build complex structures by connecting simple pieces. Each piece does one thing and can be swapped independently.

## Common Mistakes

### Mistake 1: Composition Instead of Simple Function Calls

```python
# Over-engineering — a class for a single function
class Greeter:
    def greet(self, name):
        return f"Hello, {name}"

class App:
    def __init__(self):
        self.greeter = Greeter()

    def run(self, name):
        return self.greeter.greet(name)

# Simpler
def greet(name):
    return f"Hello, {name}"
```

Not everything needs a class. Use composition when the components have state or multiple methods, not for single functions.

### Mistake 2: Creating Dependencies Inside the Constructor (Tight Coupling)

```python
class Service:
    def __init__(self):
        self.backend = OpenAIBackend()  # Tightly coupled — cannot swap
```

**Fix:** Inject dependencies:

```python
class Service:
    def __init__(self, backend: ModelBackend):
        self.backend = backend  # Can be any ModelBackend
```

### Mistake 3: Breaking Encapsulation

```python
# ❌ Accessing the internals of a composed object
pipeline = PreprocessingPipeline(...)
pipeline.transformers.append(SomeTransformer())  # Directly modifying internal list

# ✅ Use the provided interface
pipeline.add(SomeTransformer())
```

### Mistake 4: Forgetting That Composition Creates a Lifecycle Dependency

If the composed object holds resources, the container is responsible for cleaning them up:

```python
class Service:
    def __init__(self, backend):
        self.backend = backend

    def cleanup(self):
        self.backend.close()  # Container manages child's lifecycle
```
