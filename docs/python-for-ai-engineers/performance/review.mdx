# Performance — Review

⏱ 3 minutes

## Key Takeaways

- Measure before you optimise — `cProfile` tells you the real bottleneck every time.
- Vectorisation replaces Python loops with C-level operations (NumPy, `re.findall`).
- `@lru_cache` eliminates redundant computation but set `maxsize` to avoid memory leaks.
- Asyncio + httpx overlays I/O-bound work; use a semaphore to respect rate limits.
- Optimise hot paths, not cold ones — profile cumulative time, not perceived complexity.

## Recap in 60 Seconds

If you only remember three things:

1. Profile first with `python -m cProfile`, sort by cumulative time, and fix the top entry.
2. Vectorise inner loops with NumPy/re, cache expensive function calls, and use asyncio for concurrent I/O.
3. Never optimise before measuring — you will almost certainly fix the wrong thing.
