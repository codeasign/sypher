# Control Flow — Review

⏱ 3 minutes

## Key Takeaways

- Use `if`/`elif`/`else` for branching logic — order conditions from most specific to most general.
- Falsy values in Python are `False`, `None`, `0`, `0.0`, `""`, `[]`, `()`, `{}`, `set()` — use `if not x` to check for any of them.
- Use ternary expressions (`x if condition else y`) for simple two-outcome assignments, not for complex logic.
- Use `match`/`case` (Python 3.10+) for matching fixed values against multiple patterns.
- Flatten deep nesting with early returns — never let indentation exceed 3-4 levels.

## Recap in 60 Seconds

If you only remember three things:

1. Put the most specific condition first in an if/elif chain — broader conditions after or they will catch everything.
2. Use guard clauses (early returns) instead of deep nesting — your code will be flatter and easier to follow.
3. Always compare both sides with `==` or `in` — `status == 401 or 403` is a bug; write `status in (401, 403)`.
