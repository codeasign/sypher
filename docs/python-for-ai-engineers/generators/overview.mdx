# Generators

⏱ 10 minutes · Difficulty: Intermediate

## What You Will Learn

By the end of this lesson you will be able to:

- Write generator functions using `yield` instead of `return`
- Use generator expressions for lazy evaluation
- Understand how generators save memory with large datasets
- Chain generators to build data pipelines
- Use `next()` and `for` loops with generator objects

## Prerequisites

You should already understand:

- [Functions](../functions/overview) — generators are a special kind of function
- [Comprehensions](../comprehensions/overview) — generator expressions are the lazy version of list comprehensions
- [Iteration and loops](../loops/overview) — generators are iterable

## Why This Matters

When you process a dataset with 10 million rows, you cannot load it all into memory at once. A list comprehension would create a list of 10 million items, consuming gigabytes of RAM. A generator produces each value on demand, using a constant amount of memory regardless of how many items it yields.

In AI engineering, you will use generators for: streaming data from files, producing batches for training, generating prompt variations, iterating over API responses page by page, and building processing pipelines that chain multiple transformations without intermediate storage.

## The Problem

You need to process a log file with 50 million lines. Each line must be parsed, filtered, and transformed. A list comprehension like `[process(line) for line in file]` would read every line into memory before processing starts. Your program would crash with `MemoryError` long before reaching the end of the file.

You need a lazy approach: read one line, process it, yield the result, discard it, and move to the next line. This is exactly what generators provide.

## The Concept

A generator function looks like a regular function but uses `yield` instead of `return`. When called, it does not execute the body — it returns a generator object. Each time you call `next()` on that object, the function runs until the next `yield`, produces the yielded value, and pauses. Local variables are preserved between calls.

A generator expression is the lazy equivalent of a list comprehension — use parentheses instead of brackets: `(x * 2 for x in range(10))`.
