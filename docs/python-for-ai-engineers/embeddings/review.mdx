# Embeddings — Review

⏱ 3 minutes

## Key Takeaways

- Embeddings map text to fixed-size vectors (1536 dims for `text-embedding-3-small`) where similar meaning means nearby vectors.
- Use cosine similarity to measure distance between embeddings: values near 1 mean similar, near 0 mean unrelated.
- Always use the same embedding model for both documents and queries.
- Normalise vectors after embedding for reliable similarity computation.
- Chunk large documents into passages before embedding for granular retrieval.

## Recap in 60 Seconds

If you only remember three things:

1. Embeddings turn meaning into coordinates — similar texts cluster together in vector space.
2. Cosine similarity measures how close two vectors are; use the same model for everything.
3. Chunk documents into passages before embedding — whole-document embeddings lose granularity.
