# Comprehensions — Common Mistakes

⏱ 5 minutes

## When NOT to Use Comprehensions

- **Complex logic:** If the expression or condition is hard to fit in one line, use a regular loop. Readability trumps brevity.
- **Side effects:** Comprehensions create a new collection. If your goal is to `print()`, write to a file, or modify existing data, use a loop. A comprehension with side effects is confusing.
- **Very long expressions:** If a comprehension stretches beyond 80 characters, break it into multiple lines or use a loop.
- **Performance-critical inner loops:** For numerical data at scale, use vectorised numpy operations instead — they are orders of magnitude faster.

## The Pattern to Remember

- **Pattern:** `[transform(item) for item in iterable if condition]` — map optionally followed by filter.
- **When to use:** Simple transformations and filters on sequences, dicts, or sets.
- **When NOT to use:** Complex multi-step logic, side effects, very large numerical data.
- **Mental model:** A comprehension is a compact for-loop that builds a collection. Read it as "make a list of `transform(item)` for each `item` in `iterable` where `condition` is true."

## Common Mistakes

### Mistake 1: Using a Comprehension When You Just Want a Loop

```python
# Side effect — confusing
[print(x) for x in data]  # Creates a list of None values!

# Better
for x in data:
    print(x)
```

**Why it happens:** The comprehension syntax looks concise, and the side effect works (it prints). But it creates and discards a list of `None` values, wasting memory and confusing readers.

**Fix:** Use a loop for side effects; use comprehensions only when you want the resulting collection.

### Mistake 2: Putting the Condition in the Wrong Position

```python
# Ternary (modifies each element, keeps all)
[s if s > 0.5 else 0.0 for s in scores]

# Filter (removes elements entirely)
[s for s in scores if s > 0.5]
```

**Why it happens:** The `if` at the end filters. The `if/else` in the expression position applies a conditional transformation. Beginners confuse them and get either missing elements or unexpected `None` values.

**Fix:** `if condition` at the end → filter (remove items). `value if cond else other` in expression → transform (keep all items, but change some).

### Mistake 3: Overly Complex Nested Comprehensions

```python
# Hard to read
result = [z for row in matrix for col in row for z in col if z > 0]
# What is row, col, z? Which level does the condition apply to?
```

**Fix:** Break into multiple steps or use a nested loop:

```python
result = []
for row in matrix:
    for col in row:
        for z in col:
            if z > 0:
                result.append(z)
```

### Mistake 4: Forgetting That Comprehensions Create a New Collection

Comprehensions always allocate memory for the full result. For very large sequences, use a generator expression (covered in the next lesson) instead:

```python
# List comprehension — builds entire list in memory
big = [x * 2 for x in range(10_000_000)]  # ~80 MB

# Generator expression — produces values on demand
big = (x * 2 for x in range(10_000_000))   # ~0 MB
```

### Mistake 5: Using a List Comprehension When a Dict Comprehension Is Needed

```python
# Produces a list of tuples, not a dict
result = [(x, x ** 2) for x in range(5)]
# [(0, 0), (1, 1), (2, 4), (3, 9), (4, 16)]

# What you probably wanted:
result = {x: x ** 2 for x in range(5)}
# {0: 0, 1: 1, 2: 4, 3: 9, 4: 16}
```
