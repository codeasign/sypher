# Comprehensions — Hands On

⏱ 12 minutes

## Breaking It Down

### List Comprehensions

A list comprehension builds a new list by iterating over an iterable. The syntax mirrors how you would describe the operation in English: "make a list of `expr` for each `item` in `iterable` where `condition` is true."

```python
# list_comp.py
numbers = [1, 2, 3, 4, 5]

# Map: transform every element
doubled = [n * 2 for n in numbers]
print(doubled)  # [2, 4, 6, 8, 10]

# Filter: keep only elements matching a condition
evens = [n for n in numbers if n % 2 == 0]
print(evens)  # [2, 4]

# Map + filter: transform kept elements
high_doubled = [n * 2 for n in numbers if n > 3]
print(high_doubled)  # [8, 10]

# With a string method
texts = ["  hello  ", "WORLD", "  Python  "]
cleaned = [t.strip().lower() for t in texts]
print(cleaned)  # ["hello", "world", "python"]
```

The key insight: the `if condition` at the end is optional. Without it you transform every element; with it you filter first, then transform only the survivors.

### Dict Comprehensions

Dict comprehensions use curly braces with a colon separating key and value. The pattern is `{key_expr: value_expr for key, value in iterable if condition}`.

```python
# dict_comp.py

# Build a dict from paired lists
models = ["gpt-4", "claude-3", "llama-3"]
latencies = [1200, 1800, 900]

model_latency = {m: l for m, l in zip(models, latencies)}
print(model_latency)
# {"gpt-4": 1200, "claude-3": 1800, "llama-3": 900}

# Transform keys or values
squared = {x: x ** 2 for x in range(1, 6)}
print(squared)  # {1: 1, 2: 4, 3: 9, 4: 16, 5: 25}

# Filter a dict
scores = {"alice": 0.95, "bob": 0.45, "carol": 0.88, "dave": 0.32}
passing = {name: score for name, score in scores.items() if score > 0.5}
print(passing)  # {"alice": 0.95, "carol": 0.88}

# Reverse a dict (swap keys and values)
original = {"a": 1, "b": 2, "c": 3}
reversed_dict = {v: k for k, v in original.items()}
print(reversed_dict)  # {1: "a", 2: "b", 3: "c"}
```

Dict comprehensions are especially useful in AI pipelines for building lookup tables — mapping model IDs to endpoints, class indices to labels, or configuration keys to values.

### Set Comprehensions

Set comprehensions use curly braces with a single expression (no colon). They automatically remove duplicates and produce an unordered collection.

```python
# set_comp.py

# Unique squared values — duplicates collapse
numbers = [1, 2, 2, 3, 3, 3, 4]
unique_squares = {n ** 2 for n in numbers}
print(unique_squares)  # {16, 1, 4, 9} — order not guaranteed

# Collect unique first letters
words = ["apple", "banana", "avocado", "blueberry", "cherry"]
first_letters = {w[0] for w in words}
print(first_letters)  # {"a", "b", "c"}

# Filtered set comprehension
long_words = {w for w in words if len(w) > 6}
print(long_words)  # {"blueberry", "avocado"}
```

Use set comprehensions when you care about uniqueness but not order — deduplicating a list of tokens, collecting the set of unique labels from a dataset, or finding distinct categories.

### Nested Comprehensions

A nested comprehension has multiple `for` clauses, read left-to-right — the same order as nested `for` loops.

```python
# nested_comp.py

# Flatten a matrix (list of lists)
matrix = [
    [1, 2, 3],
    [4, 5, 6],
    [7, 8, 9],
]
flat = [item for row in matrix for item in row]
print(flat)  # [1, 2, 3, 4, 5, 6, 7, 8, 9]
```

Read it as: for each row in matrix, for each item in that row, collect item.

```python
# Equivalent manual nested loop — same order
flat = []
for row in matrix:
    for item in row:
        flat.append(item)
```

You can combine nested loops with filters:

```python
# Filter in nested comprehension — keeps only even numbers
even_flat = [item for row in matrix for item in row if item % 2 == 0]
print(even_flat)  # [2, 4, 6, 8]

# Transpose a matrix — inner comprehension builds each row
transposed = [[row[i] for row in matrix] for i in range(3)]
print(transposed)  # [[1, 4, 7], [2, 5, 8], [3, 6, 9]]
```

Keep nested comprehensions to two levels maximum. Beyond that, a regular loop is easier to read and debug.

### Conditional Expressions in Comprehensions

There are two distinct ways to use `if` in a comprehension, and they mean different things:

```python
# ternary_in_comp.py
scores = [0.92, 0.45, 0.88, 0.32, 0.95]

# Ternary in expression position — modifies every element, keeps all
processed = [s if s > 0.5 else 0.0 for s in scores]
print(processed)  # [0.92, 0.0, 0.88, 0.0, 0.95]

# Filter at the end — removes elements entirely
filtered = [s for s in scores if s > 0.5]
print(filtered)  # [0.92, 0.88, 0.95]
```

- `if condition` at the end → **filter** (remove items from result)
- `value_if_true if cond else value_if_false` in the expression → **ternary transform** (keep all items, change some)

Mixing them up is one of the most common comprehension bugs. The filter at the end changes the length of the result; the ternary keeps every element but may change its value.

## How It Works Internally

Python does not execute a comprehension like a regular loop. The compiler detects the comprehension syntax and emits specialized bytecode that uses dedicated opcodes — `LIST_APPEND`, `MAP_ADD`, `SET_ADD` — rather than the general `FOR_ITER` loop.

```python
import dis

def with_loop():
    result = []
    for n in range(1000):
        if n % 2 == 0:
            result.append(n * 2)
    return result

def with_comp():
    return [n * 2 for n in range(1000) if n % 2 == 0]

# Compare bytecode
print("=== Loop ===")
dis.dis(with_loop)
print("\n=== Comprehension ===")
dis.dis(with_comp)
```

The loop version calls `append` at the Python level (a method call with attribute lookup overhead), while the comprehension version uses `LIST_APPEND` at the C level — no attribute lookup, no Python function call for each append. This is the primary reason comprehensions are faster: the iteration machinery stays inside the interpreter's C loop rather than bouncing back to Python bytecode for every iteration.

## Build Something Real — Token Feature Extraction

You are processing user prompts or raw text inputs for an AI pipeline. You need to extract numerical features from each text — word count, average word length, vocabulary size — to feed into a model or logging system. Comprehensions make this concise and readable.

```python
# feature_extraction.py

def extract_features(texts):
    """Extract word-level features from a list of text strings."""
    return {
        "word_counts": [len(t.split()) for t in texts],
        "avg_word_lengths": [
            sum(len(w) for w in t.split()) / max(len(t.split()), 1)
            for t in texts
        ],
        "unique_words_per_text": [
            len({w.lower() for w in t.split()})
            for t in texts
        ],
    }

texts = [
    "Transformer models process sequences of tokens",
    "Comprehensions make data transformation concise",
    "Attention",
]

features = extract_features(texts)
print("Word counts:", features["word_counts"])
print("Avg word lengths:", features["avg_word_lengths"])
print("Unique words:", features["unique_words_per_text"])
```

Each feature is built with a list comprehension (or set comprehension for unique words). The `zip` of these feature lists can feed directly into a DataFrame or analysis function:

```python
# Build a matrix of feature vectors — one row per text
import json

for i, text in enumerate(texts):
    row = {k: v[i] for k, v in features.items()}
    print(f"Text {i + 1}: {json.dumps(row)}")
```

Running this gives you structured data ready for downstream processing. The entire feature extraction fits in under 20 lines, with no explicit loops or accumulator variables.

## Performance Considerations

**Comprehensions are faster than manual loops** — typically 1.5–3x faster for simple operations — because the append step runs at C level via `LIST_APPEND` rather than as a Python method call. The gap widens as the number of elements grows.

**Memory tradeoff:** A list comprehension builds the entire result in memory at once. For sequences that fit in RAM (millions of elements), this is fine. For sequences in the tens of millions or larger, consider whether you need all results simultaneously or can process them one at a time — that is what generator expressions (covered in the next lesson) are for.

**Dict and set comprehensions have the same performance profile** as list comprehensions — C-level iteration with specialized opcodes.

**Nested comprehensions** add overhead proportional to the product of the dimensions. A two-level nested comprehension over a 1000x1000 matrix iterates 1,000,000 times. The comprehension syntax is still faster than the equivalent nested loop, but the complexity lives in the problem, not the syntax choice.
