# Dictionaries — Review

⏱ 3 minutes

## Key Takeaways

- Dictionaries map unique keys to values — use `.get()` for safe access, `[]` for guaranteed keys.
- Iterate with `.items()` for key-value pairs, `.keys()` for keys, `.values()` for values.
- Use `collections.defaultdict` to handle missing keys automatically without boilerplate.
- Dictionary keys must be hashable — strings, numbers, and tuples work; lists and sets do not.
- From Python 3.7+, dicts preserve insertion order by language specification.

## Recap in 60 Seconds

If you only remember three things:

1. Access values with `.get(key, default)` to avoid `KeyError` crashes — use bracket access only when you are certain the key exists.
2. Iterate with `for key, value in dict.items():` — it is the most readable and Pythonic pattern.
3. Use `defaultdict(int)` for counting, `defaultdict(list)` for grouping — they eliminate boilerplate key-existence checks.
