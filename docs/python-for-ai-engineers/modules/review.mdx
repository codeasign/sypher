# Modules — Review

⏱ 3 minutes

## Key Takeaways

- Every `.py` file is a module; a directory of modules with `__init__.py` is a package.
- Always use `if __name__ == "__main__":` to guard test/CLI code so it does not run on import.
- Never name a module after a standard library module (`json.py`, `csv.py`, etc.) — it shadows the real one.
- Put all imports at the top of the file — organised by stdlib, third-party, and local.
- Use `from module import specific_name` or `import module` — avoid `from module import *`.

## Recap in 60 Seconds

If you only remember three things:

1. Use the `if __name__ == "__main__"` guard in every reusable module so test code does not run on import.
2. Structure projects into modules by concern — one file for data loading, one for processing, one for configuration.
3. Never name a file after a standard library module, never use circular imports, and always activate your virtual environment before running.
