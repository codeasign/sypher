# Specialist Agents — Review

*Reading time: 3 minutes*

## Key Takeaways

- A specialist agent scopes its system prompt, tool set, and model configuration to exactly one domain
- The principle of bounded competence means each agent is an expert in one area and ignorant of others — this is a feature, not a limitation
- An orchestrator or router directs tasks to the appropriate specialist based on domain classification
- Every tool belongs to exactly one specialist — overlapping definitions cause inconsistent behavior
- Use an LLM classifier for routing in production; keyword-based routers become brittle as vocabulary grows

## Recap in 60 Seconds

1. Specialist agents narrow their focus to one domain, carrying only the tools and knowledge needed for that domain — this improves accuracy and safety over generalist agents
2. An orchestrator or router classifies each incoming task and delegates to the matching specialist
3. Split a generalist into specialists only when you have evidence of failure — premature splitting adds complexity without benefits
