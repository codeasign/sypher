# Review: Observability, Logging, and Tracing

⏱️ 5 minutes

## Key Takeaways

1. Every agent event should carry a trace ID — this is the single most important observability practice, enabling you to reconstruct any session
2. Use log levels with different retention: DEBUG (24h), INFO (30d), WARN (90d) to balance detail with storage cost
3. Redact PII and secrets (emails, credit cards, API keys) before writing logs to avoid compliance and security issues
4. Always ship logs to a central aggregator — local files are lost when servers restart or crash
5. Every dashboard metric should have an alert threshold; observability without alerting is just pretty charts

## Recap in 60 Seconds

If you only remember three things:

1. **Trace everything** — every log entry needs a trace ID that links it to the session
2. **Redact before logging** — strip PII and API keys from log entries automatically
3. **Alert on metrics** — a dashboard without alerts is a museum, not a monitoring tool
