## Why This Matters

Agents make mistakes. They call the wrong tool, misinterpret results, miss important information, and produce incorrect output. Without self-correction, these errors propagate through the rest of the task. Reflection gives the agent a chance to catch and fix its own errors.

Production agents at companies like Anthropic (Claude Code) and GitHub (Copilot) use self-correction loops. When the agent produces code, it runs tests, receives error feedback, and fixes the issues. This cycle — produce, test, reflect, fix — dramatically improves output quality without requiring human intervention at every step.

## The Problem

Your SQL agent generates a query: `SELECT * FROM orders WHERE date > '2026-01-01' ORDER BY total DESC`. It runs the query and returns the results. The user points out that the query only returns 10 rows because of an implicit LIMIT.

The agent could have caught this itself. If it had reflected on the query before executing — checking for common issues like missing LIMIT awareness, incorrect date format, or missing JOIN clauses — it would have fixed the query before the user saw the wrong result.

Without reflection, every mistake reaches the user. With reflection, many mistakes are caught and corrected before anyone sees them.

## The Concept

Reflection is a structured review phase where the agent evaluates its own work. It can happen at two points:

**Online reflection** (before acting). The agent generates a response, then reflects on it before sending it to the user. If the reflection finds issues, the agent revises. The user only sees the revised version.

**Offline reflection** (after acting). The agent reviews what it did and records lessons learned. These lessons inform future behavior through episodic memory.

Self-correction is the loop that follows reflection:

1. Produce output
2. Reflect on the output (what is good, what is wrong, what is missing)
3. If issues found, revise the output
4. Repeat until the reflection passes or a max revision limit is reached
5. Present the final output
