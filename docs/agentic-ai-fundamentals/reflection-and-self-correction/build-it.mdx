# Reflection and Self-Correction — Build It

*Reading time: 14 minutes*

## Breaking It Down

### 1. The Reflection Step

Reflection asks the model to critique its own output before it reaches the user.

```python
import anthropic

client = anthropic.Anthropic()


def reflect_on_output(output: str, criteria: list[str]) -> str:
    """Critique an output against a set of quality criteria."""
    criteria_text = "\n".join(f"- {c}" for c in criteria)

    response = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=300,
        temperature=0,
        messages=[{
            "role": "user",
            "content": (
                f"Evaluate this output against these criteria:\n"
                f"{criteria_text}\n\n"
                f"Output:\n{output}\n\n"
                f"Identify any issues. If the output is good, say 'PASS: ' followed by 'OK'. "
                f"If there are issues, say 'FAIL: ' followed by what needs to be fixed."
            )
        }]
    )
    return response.content[0].text


# Test reflection on a flawed output
output = "Python was created in 1991 by Guido van Rossum. It is a compiled language."
reflection = reflect_on_output(output, [
    "All facts are accurate",
    "No technical inaccuracies",
    "Language is clear and precise",
])

print(reflection)
# Should flag: Python is NOT a compiled language — it is interpreted
```

### 2. Self-Correction Loop

The self-correction loop produces output, reflects, and revises until the output passes.

```python
def self_correcting_agent(task: str, max_revisions: int = 3) -> dict:
    """Agent that produces, reflects, and revises before delivering."""
    quality_criteria = [
        "All facts are accurate and verified",
        "No technical inaccuracies or misleading statements",
        "Response directly addresses the user's question",
        "Language is clear, precise, and well-structured",
    ]

    revision_count = 0
    current_output = None

    # Step 1: Initial production
    response = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=500,
        temperature=0,
        messages=[{"role": "user", "content": task}]
    )
    current_output = response.content[0].text
    print(f"Initial output: {current_output[:100]}...")

    # Steps 2-4: Reflect and revise
    while revision_count < max_revisions:
        reflection = reflect_on_output(current_output, quality_criteria)
        print(f"Reflection {revision_count + 1}: {reflection[:80]}...")

        if reflection.startswith("PASS"):
            return {
                "output": current_output,
                "revisions": revision_count,
                "reflection": reflection,
            }

        # Revision: fix the identified issues
        revision_prompt = (
            f"Original task: {task}\n\n"
            f"Your previous output:\n{current_output}\n\n"
            f"Feedback:\n{reflection}\n\n"
            f"Revise the output to address all feedback issues. "
            f"Keep what is correct, fix what is wrong."
        )

        response = client.messages.create(
            model="claude-3-5-sonnet-20241022",
            max_tokens=500,
            temperature=0,
            messages=[{"role": "user", "content": revision_prompt}]
        )
        current_output = response.content[0].text
        revision_count += 1

    return {
        "output": current_output,
        "revisions": revision_count,
        "reflection": "Max revisions reached",
    }


result = self_correcting_agent(
    "Explain the difference between Python lists and tuples. "
    "Give an example of when to use each."
)
print(f"\nFinal output (after {result['revisions']} revisions):\n{result['output']}")
```

### 3. Structured Reflection with Scoring

For production, use a structured scoring rubric rather than free-text critique.

```python
def score_output(output: str, rubric: dict) -> dict:
    """Score an output against a rubric. Returns scores and issues."""
    rubric_text = "\n".join(
        f"- {criterion} (max {max_score} points)"
        for criterion, max_score in rubric.items()
    )

    response = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=300,
        temperature=0,
        messages=[{
            "role": "user",
            "content": (
                f"Score this output against the rubric:\n\n"
                f"Rubric:\n{rubric_text}\n\n"
                f"Output:\n{output}\n\n"
                f"For each criterion, provide:\n"
                f"1. A score out of the max\n"
                f"2. A brief justification\n"
                f"3. If score < max, what needs to be fixed\n\n"
                f"Format as:\n"
                f"Criterion: score/max - justification - fix"
            )
        }]
    )
    return {
        "raw": response.content[0].text,
        "passed": all("max" not in line.upper() or line.split()[0].isdigit()
                      for line in response.content[0].text.split("\n")
                      if "criterion" in line.lower()),
    }


rubric = {
    "Factual accuracy": 5,
    "Completeness": 5,
    "Clarity": 3,
    "Actionable advice": 3,
}
output = "Python lists are mutable, tuples are immutable. Use lists for collections that change."
scores = score_output(output, rubric)
print(scores["raw"])
```

## How It Works Internally

Reflection works because the model can switch from "producer" mode to "critic" mode. When asked to evaluate, the model applies a different set of attention patterns — it looks for flaws rather than generating content. This is the same model, but the prompt changes the task from generation to evaluation.

The self-correction loop exploits this by alternating between modes. The model generates, then evaluates its own generation, then generates a fix. Each iteration uses the evaluator's output as context for the next generation.

## Build Something Real

Build a self-correcting SQL query agent.

```python
import anthropic
import sqlite3
import re

client = anthropic.Anthropic()


class SQLReflectionAgent:
    """An agent that generates, reflects on, and fixes SQL queries."""

    def __init__(self):
        # In-memory database for testing queries
        self.db = sqlite3.connect(":memory:")
        self.db.executescript("""
            CREATE TABLE employees (
                id INTEGER PRIMARY KEY,
                name TEXT,
                department TEXT,
                salary REAL
            );
            INSERT INTO employees VALUES (1, 'Alice', 'Engineering', 120000);
            INSERT INTO employees VALUES (2, 'Bob', 'Engineering', 95000);
            INSERT INTO employees VALUES (3, 'Charlie', 'Marketing', 85000);
            INSERT INTO employees VALUES (4, 'Diana', 'Marketing', 110000);
        """)

    def generate_query(self, question: str) -> str:
        """Generate a SQL query from a natural language question."""
        response = client.messages.create(
            model="claude-3-5-sonnet-20241022",
            max_tokens=200,
            temperature=0,
            messages=[{
                "role": "user",
                "content": (
                    f"Database has a table 'employees' with columns: "
                    f"id (INTEGER), name (TEXT), department (TEXT), salary (REAL).\n\n"
                    f"Question: {question}\n\n"
                    f"Generate a SQL query. Output only the SQL, no explanation."
                )
            }]
        )
        return response.content[0].text.strip()

    def reflect_on_query(self, query: str, question: str) -> str:
        """Reflect on a query for correctness and safety."""
        response = client.messages.create(
            model="claude-3-5-sonnet-20241022",
            max_tokens=300,
            temperature=0,
            messages=[{
                "role": "user",
                "content": (
                    f"Question: {question}\n"
                    f"Generated SQL: {query}\n\n"
                    f"Check for:\n"
                    f"1. Does the query answer the question?\n"
                    f"2. Are all column names correct? (id, name, department, salary)\n"
                    f"3. Is the query safe? (no DROP, DELETE, INSERT, UPDATE)\n"
                    f"4. SQL syntax errors\n\n"
                    f"If OK, respond with 'PASS'. If issues, respond with "
                    f"'FAIL: ' and what is wrong."
                )
            }]
        )
        return response.content[0].text

    def execute_safe(self, query: str) -> str:
        """Execute a query safely. Only allows SELECT."""
        if not query.strip().upper().startswith("SELECT"):
            return "Error: Only SELECT queries are allowed."

        try:
            cursor = self.db.execute(query)
            rows = cursor.fetchall()
            columns = [desc[0] for desc in cursor.description]
            return f"Columns: {columns}\nRows: {rows}"
        except Exception as e:
            return f"SQL Error: {e}"

    def answer(self, question: str) -> dict:
        """Generate, reflect, and fix a query before executing."""
        max_attempts = 3

        for attempt in range(max_attempts):
            query = self.generate_query(question)
            print(f"Attempt {attempt + 1}: {query}")

            reflection = self.reflect_on_query(query, question)
            print(f"Reflection: {reflection[:80]}...")

            if reflection.startswith("PASS"):
                result = self.execute_safe(query)
                return {"query": query, "result": result, "attempts": attempt + 1}
            else:
                # Feed reflection back for revision
                question = f"{question}\n\nPrevious attempt had issues: {reflection}"

        return {"query": "Failed after max attempts", "result": None, "attempts": max_attempts}


agent = SQLReflectionAgent()
result = agent.answer("What is the average salary in Engineering?")
print(f"\nQuery: {result['query']}")
print(f"Result: {result['result']}")
print(f"Attempts: {result['attempts']}")
```

The SQL agent reflects on its own queries before executing them. It catches column name errors, unsafe operations, and logic mistakes. This prevents incorrect or destructive queries from reaching the database.

## Performance Considerations

Each revision doubles the token cost of generation. Set max_revisions to 1-3 for most tasks. Measure the pass rate at each revision level: if 90% of outputs pass on the first try, you do not need multiple revisions.

## Security Considerations

Self-correction creates a feedback loop. Ensure the reflection prompt does not contain sensitive information that the model should not see. The reflection output (critique) becomes part of the next generation context, so it must not leak internal instructions.
