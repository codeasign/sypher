# Review: Trajectory Evaluation

⏱️ 5 minutes

## Key Takeaways

1. Trajectory evaluation measures how the agent reached its answer — tool call correctness, efficiency, and safety — not just whether the final answer is correct
2. Score trajectories on three axes: correctness (40%), efficiency (35%), and safety (25%), with separate reporting to avoid conflating quality dimensions
3. Detect anti-patterns automatically: loops (same tool+args 3+ times), redundant reads (duplicate queries), fruitless calls (empty results), and error cascades
4. Tool argument quality matters as much as tool name correctness — an agent calling the right tool with wrong parameters is only partially succeeding
5. Parallel tool calls are a valid optimization; trajectory evaluation must handle unordered parallel groups

## Recap in 60 Seconds

If you only remember three things:

1. **Track the path, not just the destination** — a good trajectory is short, correct, and safe
2. **Score three dimensions separately** — correctness, efficiency, and safety should be reported independently
3. **Auto-detect anti-patterns** — loops, redundant reads, and fruitless calls are systemic issues you can fix
