# Shared Memory and State — Review

*Reading time: 3 minutes*

## Key Takeaways

- Three shared state patterns exist: centralized store (strong consistency), event log (append-only, replayable), and per-agent snapshots (isolation with write-back)
- The centralized store is the simplest pattern but the lock becomes a bottleneck under high concurrency
- The event log eliminates write locks but state reconstruction is O(N) in the number of events — use periodic snapshots to bound replay time
- Per-agent snapshots provide the strongest isolation but agents may work with slightly stale data
- Always use compare-and-swap or versioned writes when multiple agents write to the same key

## Recap in 60 Seconds

1. Centralized stores are the simplest and safest for small multi-agent systems — use a thread-safe dictionary or lightweight database with locks
2. Event logs eliminate write contention by making all writes append-only, but require snapshot compaction to prevent unbounded growth
3. Per-agent snapshots provide full isolation within a task but create write conflicts at merge time — validate snapshot freshness before accepting merges
