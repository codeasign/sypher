# MCP with Claude Code — Review

*Reading time: 3 minutes*

## Key Takeaways

- Claude Code has a built-in MCP client — you configure servers in `mcp_servers` within settings.json, no custom client code needed
- The config format uses `command`, `args`, and optional `env` per server; Claude Code spawns each server as a child process over stdio transport
- Settings files are loaded from global (`~/.claude/settings.json`), project shared (`.claude/settings.json`), and project local (`.claude/settings.local.json`), in that order
- Validate your settings JSON with `python -c "import json; json.load(open('settings.json'))"` — invalid JSON causes silent failure
- Use unique tool names that include the domain to prevent conflicts when multiple servers define similarly named tools

## Recap in 60 Seconds

1. Add MCP servers to Claude Code via the `mcp_servers` key in settings.json — each server is defined by its `command` and `args`, and Claude Code handles the MCP protocol automatically
2. Test your server independently with the MCP Inspector before configuring it in Claude Code — protocol errors are silent in Claude Code but visible in the Inspector
3. Use `.claude/settings.local.json` (gitignored) for servers that need environment-specific secrets; use `.claude/settings.json` (version-controlled) for shared team configurations
