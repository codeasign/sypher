# Prompt Engineering Basics — Review

*Reading time: 3 minutes*

## Key Takeaways

- A well-structured prompt includes role, task, context, format, constraints, and optionally examples
- The role assigned to the model dramatically affects the style and depth of the output
- Format constraints are essential for programmatically usable output
- Check prompts for contradictory instructions and ambiguous language before blaming the model
- Shorter, precise prompts often outperform longer, convoluted ones

## Recap in 60 Seconds

1. If the output is wrong, the bug is almost always in your prompt — examine roles, tasks, format, and constraints before anything else
2. Quantify everything: use word limits, specific formats, and concrete examples instead of vague descriptors
3. Test one variable at a time — change only the role or only the format and compare results
