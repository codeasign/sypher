# Review: Filesystem MCP Server

⏱️ 5 minutes

## Key Takeaways

1. A filesystem MCP server exposes file operations as discrete, sandboxed tools rather than giving the agent raw shell access
2. Path validation must resolve symlinks and check the resolved prefix, not just the raw string, to prevent directory traversal
3. The canonical way to configure a filesystem server in Claude Code is through `mcpServers` in `settings.json`
4. Always set file size limits — the server reads entire files into memory over the transport
5. For production use, build a custom server with the MCP SDK to add audit logging, file type restrictions, and concurrency controls

## Recap in 60 Seconds

If you only remember three things:

1. **Sandbox every path** — resolve symlinks, check prefix, reject anything outside the allowed directory
2. **Use the narrowest scope** — give the agent access to only the directory it needs, not the whole project
3. **Set size limits** — large files exhaust context windows and memory
