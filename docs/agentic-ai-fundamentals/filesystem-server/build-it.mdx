# Build It: Configuring and Running a Filesystem MCP Server

⏱️ 20 minutes

## Breaking It Down

### 1. Installing the Standard Filesystem Server

The quickest way to get a filesystem MCP server running is to use the official `@modelcontextprotocol/server-filesystem` package. Install it via npm:

```bash
npm install -g @modelcontextprotocol/server-filesystem
```

Alternatively, if you prefer running it on demand without a global install:

```bash
npx -y @modelcontextprotocol/server-filesystem /workspace/allowed
```

The single required argument is the path to the directory the server is allowed to access — every file operation is constrained to this sandbox.

### 2. Configuring the Server in Claude Code

In Claude Code, you register the filesystem server in your project's `settings.json` under `mcpServers`. Create or edit `.claude/settings.json` in your project root:

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/workspace/my-project"
      ]
    }
  }
}
```

In a global or user-level settings file (`~/.claude/settings.json`), you can make the server available across all your projects:

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/you/projects"
      ]
    }
  }
}
```

Start Claude Code and the server connects automatically. You will see tools like `read_file`, `write_file`, `edit_file`, `search_files`, `list_directory`, `directory_tree`, `move_file`, and `get_file_info` become available to the agent.

### 3. Using the Server Tools

Once connected, your agent can call filesystem tools directly. Here is how each major tool works:

| Tool | Purpose | Example Input |
|---|---|---|
| `read_file` | Read a file's contents | `path: "/workspace/project/src/main.py"` |
| `write_file` | Create or overwrite a file | `path: "...", content: "print('hello')"` |
| `edit_file` | Make surgical edits by line range | `path, oldText, newText` |
| `search_files` | Find files matching a pattern | `pattern: "*.py", path: "/workspace"` |
| `list_directory` | List entries in a directory | `path: "/workspace"` |
| `directory_tree` | Recursive tree view of a directory | `path: "/workspace"` |
| `move_file` | Rename or move a file | `source, destination` |
| `get_file_info` | Get metadata about a file | `path: "..."` |

The agent decides when to call these tools based on your requests. For example, when you ask "find all Python files that import requests and show me their network calls," the agent uses `search_files` to find `*.py` files, then `read_file` to inspect imports in each match.

### 4. Building a Custom Filesystem Server with MCP SDK

For production applications where you need custom logic — audit logging, file type restrictions, size limits — build your own server using the MCP Python SDK:

```python
# filesystem_server.py
import os
from pathlib import Path
from mcp import Server
from mcp.types import TextContent

# Define the sandbox — nothing outside this path is accessible
SANDBOX = Path("/workspace/allowed").resolve()

def safe_path(requested: str) -> Path:
    """Resolve a path and ensure it stays within the sandbox."""
    target = (SANDBOX / requested).resolve()
    if not str(target).startswith(str(SANDBOX)):
        raise PermissionError(f"Access denied: {requested} is outside the sandbox")
    if not target.exists():
        raise FileNotFoundError(f"Path not found: {requested}")
    return target

server = Server("filesystem-server")

@server.tool("read_file")
async def read_file(path: str) -> TextContent:
    """Read the complete contents of a file."""
    safe = safe_path(path)
    if not safe.is_file():
        raise ValueError(f"{path} is not a file")
    if safe.stat().st_size > 10 * 1024 * 1024:  # 10 MB limit
        raise ValueError(f"File too large: {path} ({safe.stat().st_size} bytes)")
    content = safe.read_text(encoding="utf-8")
    return TextContent(type="text", text=content)

@server.tool("write_file")
async def write_file(path: str, content: str) -> TextContent:
    """Write content to a file, creating parent directories if needed."""
    safe = (SANDBOX / path).resolve()
    if not str(safe).startswith(str(SANDBOX)):
        raise PermissionError(f"Access denied: {path} is outside the sandbox")
    safe.parent.mkdir(parents=True, exist_ok=True)
    safe.write_text(content, encoding="utf-8")
    return TextContent(type="text", text=f"Written {len(content)} bytes to {path}")

@server.tool("search_files")
async def search_files(pattern: str, path: str = ".") -> TextContent:
    """Recursively search for files matching a glob pattern."""
    safe = (SANDBOX / path).resolve()
    if not str(safe).startswith(str(SANDBOX)):
        raise PermissionError(f"Access denied")
    matches = [str(p.relative_to(SANDBOX)) for p in safe.rglob(pattern) if p.is_file()]
    return TextContent(type="text", text="\n".join(matches) if matches else "No matches found")
```

Run your custom server:

```bash
python filesystem_server.py --transport stdio
```

### 5. Connecting via SSE Transport

For remote filesystem access — useful when your agent runs in the cloud but needs to read files on your local machine — switch to SSE transport:

```python
# filesystem_server_sse.py
from mcp.server import Server
from mcp.server.sse import SseServerTransport

server = Server("filesystem-server")
sse = SseServerTransport("http://localhost:8000")

# (tool definitions same as above)

if __name__ == "__main__":
    from mcp.server import run_sse
    run_sse(server, "localhost", 8000)
```

Your client then connects via SSE URL instead of stdio:

```json
{
  "mcpServers": {
    "filesystem": {
      "url": "http://localhost:8000/sse"
    }
  }
}
```

## Build Something Real

Build a documentation search agent that reads your project's Markdown files and answers questions from them. Start by configuring the filesystem server with access to your docs folder:

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/workspace/my-project/docs"
      ]
    }
  }
}
```

Then, in Claude Code, ask something like:

> Find every `.md` file in the docs folder, read the one about authentication, and summarize the setup steps.

The agent calls `search_files` with `pattern: "*.md"`, then `read_file` on each match, and synthesizes the answer. You get a file-aware assistant without ever exposing shell access to the model.

## Security Considerations

The sandbox path is the most critical part of a filesystem MCP server. If you set the sandbox to the root directory, the agent can read and write every file on your system. Always:

- Use the narrowest sandbox directory that covers the files your agent needs
- Never set the sandbox to `/`, `C:\`, or your home directory
- Validate symlinks — a symlink inside the sandbox pointing outside bypasses the path check
- Set file size limits, as shown in the custom server example above, to prevent the agent from reading multi-gigabyte files

The standard `server-filesystem` package does resolve symlinks and reject paths outside the sandbox, but when building a custom server you must implement this yourself.
