# System Prompts — Avoid Mistakes

*Reading time: 6 minutes*

## When NOT to Use This

System prompts are powerful, but they are not always the best tool:

- **Short, one-turn interactions**: If the conversation is always one user message and one response, the system prompt's persistence gives no benefit. Put the instructions in the user message.
- **Dynamic instructions that change per turn**: If every user message needs different instructions, putting them in the user message (or alternating system prompts) makes more sense than one static system prompt.
- **API calls using tool definitions**: When using the Anthropic tool use API, tool definitions serve a similar purpose to system prompts for constraining behavior. Overloading both can create conflicts.
- **Fine-tuned models**: If the model is already fine-tuned for your task, a simple system prompt may be all you need. Over-engineering it adds unnecessary tokens.

## The Pattern to Remember

**Pattern**: System prompt for invariants, user message for specifics.

When to use:
- Multi-turn conversations where behavior must stay consistent
- Applications with strict tone, format, or domain requirements
- Systems where multiple prompts share common rules

When NOT to use:
- Single-turn interactions
- Tasks where every request has unique, unrelated instructions

**Mental model**: The system prompt is the constitution of your application. It sets the rules that all subsequent interactions follow. Write it carefully — it applies to every conversation turn.

## Common Mistakes

### Mistake 1: Putting Task Instructions in the System Prompt Instead of the User Message

**Why it happens**: You know the task, so you put everything in the system prompt for convenience.

**Symptoms**: The system prompt is a long paragraph mixing rules, task description, and examples. The model follows the rules but sometimes misses the specific task.

**Fix**: Put behavioral rules in the system prompt. Put the current task in the user message. The system prompt says who the model is and how to behave; the user message says what to do.

### Mistake 2: Contradicting the System Prompt in a User Message

**Why it happens**: A later user message includes instructions that conflict with the system prompt, and you expect the system prompt to overrule.

**Symptoms**: The model sometimes follows the system prompt, sometimes follows the conflicting instruction in the user message. Behavior is inconsistent.

**Fix**: If you need the model to deviate from the system prompt for a specific turn, explicitly acknowledge the override in the user message: "I know your general rule is X, but for this specific request, please do Y."

### Mistake 3: Making System Prompts Too Long

**Why it happens**: You keep adding rules and edge cases until the system prompt covers every scenario.

**Symptoms**: At 2000+ tokens, the model starts ignoring later rules. Performance degrades because the system prompt consumes context that could go to useful conversation.

**Fix**: Keep system prompts under 500 tokens when possible. Move detailed instructions or examples to user messages or a retrieval-augmented context.

### Mistake 4: Not Updating System Prompts When the Application Changes

**Why it happens**: The system prompt is set once during development and never revisited.

**Symptoms**: The application's behavior drifts from expectations. Rules in the system prompt reference outdated policies or products.

**Fix**: Version your system prompts. When you update policies or add features, update the system prompt. Test the new version against your evaluation suite.

### Mistake 5: Assuming the System Prompt Prevents All Bad Behavior

**Why it happens**: You write "Never make up facts" and assume the model will comply perfectly.

**Symptoms**: The model still occasionally hallucinates. The system prompt reduces but does not eliminate unwanted behavior.

**Fix**: Combine system prompts with output validation. The system prompt biases the model toward good behavior; validation catches when it still goes wrong. Guard with verification, not just prompting.
