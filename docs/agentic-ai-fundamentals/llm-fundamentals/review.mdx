# LLM Fundamentals — Review

*Reading time: 3 minutes*

## Key Takeaways

- LLMs are next-token predictors trained at enormous scale — every capability emerges from this objective
- Training has three stages: pretraining (raw text), fine-tuning (instruction following), and alignment (safety and helpfulness)
- Temperature controls the randomness of token selection — use 0 for deterministic tasks, higher for creative ones
- `max_tokens` limits output length, not total context — always leave room for the response within the context window
- Always check `stop_reason` in production — `"max_tokens"` means the response was truncated

## Recap in 60 Seconds

1. An LLM generates text by predicting one token at a time — the parameters you set control how it makes those predictions
2. Temperature 0 is deterministic; higher temperatures add randomness — pick based on whether your task needs consistency or variety
3. Every model is different — parameters that work on one model must be re-tuned when you switch models
