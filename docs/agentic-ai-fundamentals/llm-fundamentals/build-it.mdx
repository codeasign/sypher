# LLM Fundamentals — Build It

*Reading time: 10 minutes*

## Breaking It Down

### 1. Temperature — The Creativity Dial

Temperature controls the randomness of token selection. At `temperature=0`, the model always picks the most probable next token. At higher values, it draws from a broader distribution, producing more varied and creative outputs.

```python
import anthropic

client = anthropic.Anthropic()

def ask_with_temperature(prompt: str, temp: float):
    """Send a prompt at a specific temperature and return the response."""
    response = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=100,
        temperature=temp,
        system="You are a creative product namer.",
        messages=[{"role": "user", "content": prompt}]
    )
    return response.content[0].text

prompt = "Suggest a name for an AI that helps developers debug code."

print("=== Temperature 0 (deterministic) ===")
for i in range(3):
    print(f"Run {i+1}: {ask_with_temperature(prompt, 0.0)}")

print("\n=== Temperature 0.9 (creative) ===")
for i in range(3):
    print(f"Run {i+1}: {ask_with_temperature(prompt, 0.9)}")
```

At temperature 0, you get the same or very similar output each time. At 0.9, the names vary widely in style and creativity. Production systems that need consistent output (classification, extraction) should use `temperature=0`. Creative tasks (content generation, brainstorming) benefit from values between 0.5 and 1.0.

### 2. Max Tokens — Length Control

The `max_tokens` parameter limits how many tokens the model can generate in its response. The model stops early if it reaches a natural stopping point, but it will hard-stop at your limit.

```python
import anthropic

client = anthropic.Anthropic()

for limit in [50, 200, 500]:
    response = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=limit,
        temperature=0,
        system="Explain how gravity works in detail.",
        messages=[{"role": "user", "content": "Explain gravity"}]
    )
    print(f"--- max_tokens={limit} ---")
    print(f"Output tokens: {response.usage.output_tokens}")
    print(f"Stop reason: {response.stop_reason}")
    print(f"Text preview: {response.content[0].text[:80]}...\n")
```

When `stop_reason` is `"max_tokens"`, the response was truncated — you did not get the full answer. When it is `"end_turn"`, the model finished naturally.

### 3. Top-P (Nucleus Sampling)

Top-p selects from the smallest set of tokens whose cumulative probability exceeds `p`. A value of 0.9 means the model considers only the most probable tokens that together account for 90% of the probability mass. Lower values make output more focused; higher values allow more variety.

```python
def ask_with_topp(prompt: str, top_p: float):
    """Compare responses at different top_p values."""
    response = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=100,
        temperature=0.7,
        top_p=top_p,
        messages=[{"role": "user", "content": prompt}]
    )
    return response.content[0].text

prompt = "Write one sentence about the future of AI."

print(f"top_p=0.1 (very focused): {ask_with_topp(prompt, 0.1)}")
print(f"top_p=0.5 (moderate):    {ask_with_topp(prompt, 0.5)}")
print(f"top_p=1.0 (max variety): {ask_with_topp(prompt, 1.0)}")
```

In practice, most engineers set `temperature` and leave top-p at its default. Adjust top-p only when you need fine-grained control over the diversity of the output distribution.

## How It Works Internally

Every forward pass through the model produces a vector of logits — raw scores — for every token in the vocabulary (typically 100,000 to 200,000 tokens). The softmax function converts these logits into a probability distribution that sums to 1.0.

Temperature scales the logits before softmax. Dividing logits by a higher temperature makes the distribution flatter (more uniform), so lower-probability tokens get picked more often. Mathematically:

```
P(token_i) = exp(logit_i / T) / sum_j exp(logit_j / T)
```

When `T=0`, the model uses argmax — it always picks the highest-probability token, making output deterministic. In practice, implementation uses a slightly different approach for numerical stability, but the intuition is the same.

The number of parameters determines how much information the model can store. A 70B-parameter model has vastly more capacity than a 7B model. With more parameters comes better reasoning, more world knowledge, and better instruction following — but also higher latency and cost per token.

## Build Something Real

Let's build a parameter explorer tool that helps you find the right inference settings for different tasks:

```python
import anthropic
from dataclasses import dataclass, asdict

client = anthropic.Anthropic()


@dataclass
class InferenceConfig:
    """Holds the inference parameters for a single experiment."""
    temperature: float = 0.0
    max_tokens: int = 200
    top_p: float = 1.0


def run_experiment(
    system_prompt: str,
    user_message: str,
    configs: list[InferenceConfig]
) -> list[dict]:
    """Run the same prompt with multiple configurations and log results."""
    results = []

    for cfg in configs:
        response = client.messages.create(
            model="claude-3-5-sonnet-20241022",
            **asdict(cfg),
            system=system_prompt,
            messages=[{"role": "user", "content": user_message}]
        )

        results.append({
            "config": asdict(cfg),
            "output": response.content[0].text,
            "tokens": response.usage.output_tokens,
            "stop_reason": response.stop_reason
        })

    return results


# Test different configurations for a classification task
configs = [
    InferenceConfig(temperature=0.0, max_tokens=30),
    InferenceConfig(temperature=0.5, max_tokens=30),
    InferenceConfig(temperature=1.0, max_tokens=30),
]

results = run_experiment(
    system_prompt="Classify the sentiment as POSITIVE, NEGATIVE, or NEUTRAL. Respond with only the label.",
    user_message="The product works well but the shipping took too long.",
    configs=configs
)

for r in results:
    print(f"Config: {r['config']}")
    print(f"Output: {r['output']}")
    print(f"Tokens: {r['tokens']} | Stop: {r['stop_reason']}\n")
```

This pattern — running the same prompt across multiple configurations — is how you empirically find the right settings for your use case rather than guessing.
