# Build It: Configuring a Web Search MCP Server

⏱️ 20 minutes

## Breaking It Down

### 1. Getting a Brave Search API Key

The Brave Search API is a popular choice for MCP web search because it is affordable, privacy-respecting, and includes both web and news results. Sign up at [Brave Search API dashboard](https://api.search.brave.com/app/dashboard) and create an API key.

The free tier includes 2000 queries per month. Each query returns up to 20 results. For production use, the paid tier costs approximately $5 per 50,000 queries.

### 2. Configuring the Brave Search MCP Server

Install the official Brave Search MCP server:

```bash
npm install -g @anthropic/server-brave-search
```

Configure it in your project's `.claude/settings.json`:

```json
{
  "mcpServers": {
    "web-search": {
      "command": "npx",
      "args": [
        "-y",
        "@anthropic/server-brave-search"
      ],
      "env": {
        "BRAVE_API_KEY": "your_brave_api_key_here"
      }
    }
  }
}
```

For security, store the API key in an environment variable:

```json
{
  "mcpServers": {
    "web-search": {
      "command": "npx",
      "args": ["-y", "@anthropic/server-brave-search"],
      "env": {
        "BRAVE_API_KEY": "${BRAVE_API_KEY}"
      }
    }
  }
}
```

Set the environment variable in your shell profile:

```bash
export BRAVE_API_KEY="BSA_xxxxxxxxxxxxx"
```

### 3. Available Tools and Parameters

Once connected, the Brave Search server exposes these tools:

| Tool | Parameters | Description |
|---|---|---|
| `brave_web_search` | `query` (required), `count` (1-20, default 5) | General web search |
| `brave_local_search` | `query` (required), `count` (1-20) | Local business and places search |
| `brave_news_search` | `query` (required), `freshness` (day/week/month) | News article search |
| `brave_image_search` | `query` (required), `count` (1-20) | Image results (returning metadata URLs) |

Each tool returns structured results with titles, URLs, snippets, and (where available) publication dates and source names.

### 4. Real-World Usage Scenarios

**Current events research:**

Ask the agent: "What did the Fed announce about interest rates this week?"

The agent calls `brave_news_search(query: "Federal Reserve interest rate announcement this week", freshness: "week")`. It gets back 10 news articles with titles like "Fed Holds Rates Steady, Signals Two Cuts Later This Year," reads the most authoritative source using the file reader or a follow-up, and synthesizes a summary.

**Coding with live documentation:**

Ask: "How do I use the new `axios` v2 API? Find the migration guide."

The agent calls `brave_web_search(query: "axios v2 migration guide", count: 5)`, finds the official axios GitHub migration guide, fetches the page content, and uses it to update your code.

**Competitive monitoring:**

Ask: "What new features did our competitors announce this month?"

The agent searches news with queries for each competitor name and freshness set to month, compiles a report from the results.

### 5. Building a Custom Web Search Server with the MCP SDK

If you want to use a different search provider (Tavily, SerpAPI, Bing) or add custom logic:

```python
# web_search_server.py
import os
from mcp import Server
from mcp.types import TextContent
import httpx

BRAVE_API_KEY = os.environ["BRAVE_API_KEY"]
SEARCH_URL = "https://api.search.brave.com/res/v1/web/search"

server = Server("web-search")

@server.tool("web_search")
async def web_search(query: str, count: int = 5) -> TextContent:
    """Search the web using the Brave Search API."""
    async with httpx.AsyncClient() as client:
        resp = await client.get(
            SEARCH_URL,
            params={"q": query, "count": min(count, 20)},
            headers={
                "Accept": "application/json",
                "Accept-Encoding": "gzip",
                "X-Subscription-Token": BRAVE_API_KEY,
            },
            timeout=10,
        )
    resp.raise_for_status()
    data = resp.json()

    results = []
    for item in data.get("web", {}).get("results", []):
        results.append({
            "title": item["title"],
            "url": item["url"],
            "snippet": item.get("description", ""),
        })

    output = "\n\n".join(
        f"{i+1}. {r['title']}\n   URL: {r['url']}\n   {r['snippet']}"
        for i, r in enumerate(results)
    )
    return TextContent(type="text", text=output or "No results found.")

@server.tool("fetch_page")
async def fetch_page(url: str) -> TextContent:
    """Fetch and return the text content of a web page."""
    async with httpx.AsyncClient() as client:
        resp = await client.get(url, timeout=15, follow_redirects=True)
    resp.raise_for_status()

    import re
    text = resp.text
    # Strip HTML tags
    text = re.sub(r"<[^>]+>", " ", text)
    text = re.sub(r"\s+", " ", text).strip()
    # Truncate to 8000 characters
    text = text[:8000]

    return TextContent(type="text", text=text)
```

Run it:

```bash
BRAVE_API_KEY=BSA_xxx python web_search_server.py --transport stdio
```

## Performance Considerations

Each search API call costs between 100ms and 800ms depending on the provider and network latency. The Brave Search API is typically faster than general web scraping (200-400ms median). The fetch_page tool is slower (1-3 seconds per page) because it must download and parse the full HTML.

Set the `count` parameter to the minimum number of results you need. Fetching 20 results when you only need 3 wastes API quota and adds latency.

## Security Considerations

The `fetch_page` tool fetches arbitrary URLs. This means the agent could be prompted to visit malicious sites. Mitigate this by:

- Limiting `fetch_page` to `https://` URLs only (reject `http://` and `file://`)
- Adding a domain allow list if you only need results from specific sources (e.g., `*.github.com`, `*.python.org`)
- Setting a timeout of 10-15 seconds to prevent slow pages from hanging the server
- Never passing cookies or authentication tokens in the fetch request
