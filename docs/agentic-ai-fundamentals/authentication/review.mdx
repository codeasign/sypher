# Review: Authentication

⏱️ 5 minutes

## Key Takeaways

1. Never store credentials in source code, configuration files under version control, or plaintext environment variables — use a dedicated secrets manager with encryption at rest
2. Implement automatic OAuth 2.0 token refresh in the authentication layer, not in agent logic, so the agent never thinks about token expiry
3. Redact credentials from all log output — API keys and tokens in logs are a security incident waiting to happen
4. Use least-privilege scopes for every credential; a read-only agent should never have delete permissions on any service
5. Protect concurrent token refreshes with a lock to avoid race conditions when parallel tool calls trigger simultaneous refresh attempts

## Recap in 60 Seconds

If you only remember three things:

1. **Treat credentials like passwords** — encrypt at rest, rotate regularly, and never log them
2. **Automate the token lifecycle** — issuance, caching, refresh, and revocation must all happen without human intervention
3. **Scope every credential minimally** — the blast radius of a leaked token is limited by the permissions it carries
