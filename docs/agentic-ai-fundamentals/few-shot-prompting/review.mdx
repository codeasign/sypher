# Few-Shot Prompting — Review

*Reading time: 3 minutes*

## Key Takeaways

- Few-shot prompting uses input-output examples to teach the model desired behavior through in-context learning
- 3-5 diverse examples typically provide the best balance of effectiveness and token cost
- Example ordering matters — the last example has the strongest influence on output (recency bias)
- Every example must be correct — a single bad example can corrupt the learned pattern
- Validate each example before use and test across the full range of expected inputs

## Recap in 60 Seconds

1. Show examples rather than just describing the task — concrete patterns beat abstract descriptions
2. Start with 3 diverse examples covering edge cases, add more only if needed
3. Order matters: put your most representative or boundary-defining example last
