# Anatomy of an Agent — Review

*Reading time: 3 minutes*

## Key Takeaways

- Every agent has five components: model, orchestrator, tools, memory, and safety layer
- The orchestrator manages the loop: build prompt, call model, execute tools, check if done
- Each component has a single responsibility — the model decides, the orchestrator manages flow, tools execute actions, memory stores context, safety prevents harm
- The safety layer must intercept every tool call before execution, not after
- Component separation makes agents testable, debuggable, and maintainable

## Recap in 60 Seconds

1. The orchestrator is the loop, the model is the decision-maker, tools are the hands, memory stores context, and safety prevents harm
2. Build each component independently and wire them through the orchestrator's step function
3. The safety layer is the only component that can reject tool calls — it runs before the tool implementation
