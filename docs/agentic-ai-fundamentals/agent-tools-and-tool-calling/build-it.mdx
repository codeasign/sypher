# Agent Tools and Tool Calling — Build It

*Reading time: 14 minutes*

## Breaking It Down

### 1. Defining a Tool with JSON Schema

Every tool starts with a definition that tells the model what the tool does, what parameters it expects, and what values are valid.

```python
search_tool = {
    "name": "search_products",
    "description": "Search the product catalog by keyword. Returns product names, prices, and availability.",
    "input_schema": {
        "type": "object",
        "properties": {
            "keyword": {
                "type": "string",
                "description": "Product keyword to search for (e.g., 'laptop', 'wireless mouse')"
            },
            "category": {
                "type": "string",
                "description": "Optional category filter: 'electronics', 'clothing', 'home'",
                "enum": ["electronics", "clothing", "home"]
            },
            "max_price": {
                "type": "number",
                "description": "Maximum price filter",
                "exclusiveMinimum": 0
            },
            "sort_by": {
                "type": "string",
                "description": "Sort order",
                "enum": ["price_asc", "price_desc", "name"],
                "default": "name"
            }
        },
        "required": ["keyword"]
    }
}
```

Key schema design choices:

- Use `enum` to constrain categorical parameters. This prevents the model from inventing filter values.
- Use `default` for optional parameters. The model sends the defaults explicitly only when needed.
- The `description` field is critical — it is the model's primary signal for choosing between similar tools. Be specific about what the tool does and when to use it.

### 2. Tool Dispatch System

The dispatch system routes tool call requests to the correct implementation.

```python
from typing import Any, Callable


class ToolRegistry:
    """Registry that maps tool names to definitions and handlers."""

    def __init__(self):
        self._tools: list[dict] = []
        self._handlers: dict[str, Callable] = {}

    def register(self, definition: dict, handler: Callable):
        """Register a tool by its API definition."""
        name = definition["name"]
        self._tools.append(definition)
        self._handlers[name] = handler

    def get_definitions(self) -> list[dict]:
        """Get all tool definitions for the API call."""
        return list(self._tools)

    def execute(self, tool_name: str, arguments: dict) -> str:
        """Execute a tool by name with the given arguments."""
        handler = self._handlers.get(tool_name)
        if not handler:
            return f"Error: tool '{tool_name}' not found"

        try:
            result = handler(**arguments)
            return str(result)
        except TypeError as e:
            return f"Error: invalid arguments for {tool_name}: {e}"
        except Exception as e:
            return f"Error executing {tool_name}: {e}"


registry = ToolRegistry()


# Define handler functions
def search_products_handler(keyword: str, category: str | None = None,
                            max_price: float | None = None,
                            sort_by: str = "name") -> str:
    """Actual implementation of the search tool."""
    catalog = [
        {"name": "Wireless Mouse", "price": 29.99, "category": "electronics"},
        {"name": "Mechanical Keyboard", "price": 89.99, "category": "electronics"},
        {"name": "USB-C Hub", "price": 34.99, "category": "electronics"},
        {"name": "Cotton T-Shirt", "price": 19.99, "category": "clothing"},
        {"name": "Desk Lamp", "price": 45.00, "category": "home"},
    ]

    results = [
        p for p in catalog
        if keyword.lower() in p["name"].lower()
    ]
    if category:
        results = [p for p in results if p["category"] == category]
    if max_price:
        results = [p for p in results if p["price"] <= max_price]

    if sort_by == "price_asc":
        results.sort(key=lambda p: p["price"])
    elif sort_by == "price_desc":
        results.sort(key=lambda p: p["price"], reverse=True)

    if not results:
        return "No products found matching your criteria."

    return "\n".join(
        f"- {p['name']}: ${p['price']:.2f} ({p['category']})"
        for p in results
    )


registry.register(search_tool, search_products_handler)

# Test the dispatch
print(registry.execute("search_products", {"keyword": "mouse"}))
print()
print(registry.execute("search_products", {"keyword": "keyboard", "max_price": 50}))
```

### 3. Handling Parallel Tool Calls

Modern LLM APIs can return multiple tool calls in a single response. Your dispatcher must handle this.

```python
def process_response(response_content: list, registry: ToolRegistry,
                     messages: list[dict]) -> bool:
    """
    Process a model response that may contain text and multiple tool calls.
    Returns True if the response has no tool calls (we are done).
    """
    has_tool_calls = False

    for block in response_content:
        if block.type == "text":
            # The model included text alongside tool calls
            messages.append({"role": "assistant", "content": block.text})

        elif block.type == "tool_use":
            has_tool_calls = True
            result = registry.execute(block.name, block.input)

            messages.append({"role": "assistant", "content": block})
            messages.append({
                "role": "user",
                "content": [
                    {"type": "tool_result", "tool_use_id": block.id, "content": result}
                ]
            })

    return not has_tool_calls


# Example usage
messages = [{"role": "user", "content": "Find me wireless mice and USB hubs under $40."}]

for turn in range(5):
    response = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=1000,
        temperature=0,
        system="You are a shopping assistant.",
        messages=messages,
        tools=registry.get_definitions(),
    )

    done = process_response(response.content, registry, messages)
    if done:
        print(response.content[0].text)
        break
```

The model can request both `search_products("wireless mouse")` and `search_products("USB hub")` in one response. Your dispatcher runs both and returns both results. The model then synthesizes them.

## How It Works Internally

When the API receives tools in the request, it modifies the model's sampling process. The model can generate a special `tool_use` content block with structured JSON in the `input` field. The model's training teaches it to match the tool's `description` and parameter `description` fields to the user's request.

The tool `name` is critical for disambiguation. If two tools have similar descriptions, the model may choose the wrong one. Prefix tool names with domain context: `inventory_search_products` instead of just `search_products`.

## Build Something Real

Build a file management agent with multiple tools and robust error handling.

```python
import anthropic
import json
import os
from datetime import datetime
from typing import Any

client = anthropic.Anthropic()


class FileTools:
    """File management tools for an agent."""

    def __init__(self, workspace: str = "./agent_workspace"):
        self.workspace = workspace
        os.makedirs(workspace, exist_ok=True)

    def list_files(self, pattern: str = "*") -> str:
        """List files in the workspace matching a pattern."""
        import glob
        files = glob.glob(os.path.join(self.workspace, pattern))
        if not files:
            return "No files found."
        entries = []
        for f in files:
            stat = os.stat(f)
            entries.append(
                f"{os.path.basename(f)}  {stat.st_size} bytes  "
                f"{datetime.fromtimestamp(stat.st_mtime).strftime('%Y-%m-%d %H:%M')}"
            )
        return "\n".join(entries)

    def read_file(self, filename: str) -> str:
        """Read the contents of a file."""
        path = os.path.join(self.workspace, filename)
        if not os.path.exists(path):
            return f"Error: file '{filename}' not found."
        if not path.startswith(os.path.abspath(self.workspace)):
            return "Error: path traversal detected."
        with open(path, "r") as f:
            return f.read()

    def write_file(self, filename: str, content: str) -> str:
        """Write content to a file (creates or overwrites)."""
        path = os.path.join(self.workspace, filename)
        if not path.startswith(os.path.abspath(self.workspace)):
            return "Error: path traversal detected."
        with open(path, "w") as f:
            f.write(content)
        return f"File '{filename}' written successfully ({len(content)} bytes)."

    def append_file(self, filename: str, content: str) -> str:
        """Append content to an existing file."""
        path = os.path.join(self.workspace, filename)
        if not path.startswith(os.path.abspath(self.workspace)):
            return "Error: path traversal detected."
        with open(path, "a") as f:
            f.write(content)
        return f"Appended {len(content)} bytes to '{filename}'."


file_tools = FileTools()

TOOLS = [
    {
        "name": "list_files",
        "description": "List files in the workspace. Use a pattern like '*.txt' to filter.",
        "input_schema": {
            "type": "object",
            "properties": {
                "pattern": {
                    "type": "string",
                    "description": "File pattern (e.g., '*.txt', 'report*')",
                    "default": "*"
                }
            },
            "required": []
        }
    },
    {
        "name": "read_file",
        "description": "Read the contents of a file from the workspace",
        "input_schema": {
            "type": "object",
            "properties": {
                "filename": {"type": "string", "description": "Name of the file to read"}
            },
            "required": ["filename"]
        }
    },
    {
        "name": "write_file",
        "description": "Write content to a file in the workspace. Creates or overwrites.",
        "input_schema": {
            "type": "object",
            "properties": {
                "filename": {"type": "string", "description": "Name of the file to write"},
                "content": {"type": "string", "description": "Content to write"}
            },
            "required": ["filename", "content"]
        }
    },
    {
        "name": "append_file",
        "description": "Append content to an existing file in the workspace",
        "input_schema": {
            "type": "object",
            "properties": {
                "filename": {"type": "string", "description": "Name of the file"},
                "content": {"type": "string", "description": "Content to append"}
            },
            "required": ["filename", "content"]
        }
    },
]


HANDLERS = {
    "list_files": lambda **kw: file_tools.list_files(kw.get("pattern", "*")),
    "read_file": lambda **kw: file_tools.read_file(kw["filename"]),
    "write_file": lambda **kw: file_tools.write_file(kw["filename"], kw["content"]),
    "append_file": lambda **kw: file_tools.append_file(kw["filename"], kw["content"]),
}


def run_file_agent(user_input: str) -> str:
    """Run the file management agent."""
    messages = [{"role": "user", "content": user_input}]

    for turn in range(10):
        response = client.messages.create(
            model="claude-3-5-sonnet-20241022",
            max_tokens=1000,
            temperature=0,
            system="You are a file management assistant. Help users manage files in the workspace.",
            messages=messages,
            tools=TOOLS,
        )

        for block in response.content:
            if block.type == "text":
                # If there is text alongside tool calls, just continue
                continue

            if block.type == "tool_use":
                handler = HANDLERS.get(block.name)
                if handler:
                    result = handler(**block.input)
                else:
                    result = f"Unknown tool: {block.name}"

                messages.append({"role": "assistant", "content": block})
                messages.append({
                    "role": "user",
                    "content": [
                        {"type": "tool_result", "tool_use_id": block.id, "content": result}
                    ]
                })

        # Check if the last block was text (no tool calls)
        if all(block.type != "tool_use" for block in response.content):
            # Extract just the response text
            text_parts = [b.text for b in response.content if b.type == "text"]
            return "\n".join(text_parts)

    return "File agent did not complete within turn limit."


# Test: create a report, then read it back
result = run_file_agent(
    "Create a file called 'report.txt' with the text "
    "'Q4 Revenue: $1.2M. Profit margin: 22%.' "
    "Then read it back and list all files."
)
print(result)
```

This agent demonstrates production-quality tool patterns: path traversal protection, clear error messages, and structured file operations. Each tool returns a result string that the model can understand and act on.

## Security Considerations

Every tool argument must be validated before execution. The path traversal check in `read_file` and `write_file` prevents the agent from accessing files outside the workspace. Apply similar validation to any tool that accesses the filesystem, network, or external APIs.

## Performance Considerations

When the model returns multiple parallel tool calls, execute them concurrently rather than sequentially. Use Python's `concurrent.futures.ThreadPoolExecutor` to run independent tool calls in parallel, reducing wall-clock time.
