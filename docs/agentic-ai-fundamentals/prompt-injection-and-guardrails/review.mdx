# Review: Prompt Injection and Guardrails

⏱️ 5 minutes

## Key Takeaways

1. Prompt injection exploits the fact that LLMs cannot inherently distinguish between instructions and data — all tokens share the same attention mechanism
2. Defense-in-depth combines system prompt structure, input sanitization, output validation, tool-level safety, and human oversight — no single layer is sufficient
3. Always sanitize tool return values and database contents, not just user input and web page text
4. Output validation (deterministic code checks on planned actions) is a stronger defense than system prompt directives
5. Test against multi-turn attacks and maintain risk-based human-in-the-loop oversight, avoiding confirmation fatigue

## Recap in 60 Seconds

If you only remember three things:

1. **No silver bullet** — system prompts alone are weak; combine all five defense layers
2. **Output validation catches what prompts miss** — deterministic code checks on tool calls are your strongest defense
3. **Sanitize all data paths** — web pages, files, database results, and tool outputs can all carry injections
