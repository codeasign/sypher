# Agent Memory — Review

*Reading time: 3 minutes*

## Key Takeaways

- Agent memory has three tiers: short-term (context window), long-term (external store), and episodic (learning from mistakes)
- Sliding window memory is simple but lossy — old information is dropped regardless of importance
- Summarizing memory compresses old turns into a shorter form, preserving the gist at reduced fidelity
- Retrieval-augmented memory stores everything externally and selectively injects relevant information back into context
- Design your memory strategy before building the orchestrator — the memory interface shapes the entire message flow

## Recap in 60 Seconds

1. Sliding window = always loses old data; summarization = compresses old data; retrieval = selectively restores old data
2. Store critical facts (names, IDs, preferences) in a key-value store that is never summarized or dropped
3. Memory strategy is an architectural decision, not an afterthought — it affects every part of the agent loop
