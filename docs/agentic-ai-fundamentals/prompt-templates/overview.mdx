# Prompt Templates

## Why This Matters

In production, you do not write prompts by hand for every request. Your application builds prompts dynamically from user input, database records, conversation history, and configuration. You need a system for constructing prompts that is correct, safe, maintainable, and testable.

Hardcoding prompt strings with f-string interpolation works for the first few prompts, but as your application grows, you end up with prompt logic scattered across your codebase. Templates centralize your prompts, making them versionable, reviewable, and independently testable. This is the difference between an ad hoc integration and a production prompt management system.

## The Problem

You have an email classification system. The prompt is built inline in your application code:

```python
response = client.messages.create(
    model="claude-3-5-sonnet-20241022",
    system=f"Classify this email into {categories}.",
    messages=[{"role": "user", "content": f"Subject: {subject}\nBody: {body}"}]
)
```

It works. Then you add a second prompt for summarization. Then a third for response generation. Each has its own format, its own parameter names, its own way of handling missing data. When you need to update the classification categories, you have to find the right file, edit the f-string, and hope you did not break anything.

The prompt logic is mixed with application logic, which makes both harder to maintain. You need a template system that separates prompt design from application code.

## The Concept

A prompt template is a parameterized string with variable placeholders that your application fills in at runtime. The template defines the static structure of the prompt; the variables inject dynamic content.

A good template system includes:

**Variable substitution**: Placeholders like `{{category_list}}` or `{user_input}` that are replaced with actual values at runtime.

**Default values**: Fallback content when a variable is not provided. Example: `"Classify into {{categories | default('general')}}"`.

**Conditional sections**: Content that is included only when a variable has a specific value or is present. Example: `"{% if urgent %} This is urgent. {% endif %}"`.

**Formatting helpers**: Functions to transform variables before insertion — truncate text, format lists, escape special characters.
