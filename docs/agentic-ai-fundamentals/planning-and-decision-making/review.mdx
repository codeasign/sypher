# Planning and Decision Making — Review

*Reading time: 3 minutes*

## Key Takeaways

- Flat planning generates a complete plan upfront; hierarchical planning decomposes into subtasks; dynamic replanning adapts when steps fail
- Plan only 2-3 steps ahead (receding horizon) rather than generating a full plan that becomes invalid
- Always validate plans before executing — reject plans with impossible or out-of-scope steps
- Set a maximum number of replan rounds to prevent infinite planning loops
- Keep the plan and the execution history in separate data structures to avoid confusion

## Recap in 60 Seconds

1. Let the model plan a few steps at a time rather than the entire task — the situation changes as the agent acts
2. Distinguish between critical failures (replan) and minor failures (handle inline) to avoid excessive replan costs
3. Keep the plan document separate from the execution history — the model should not confuse intentions with results
