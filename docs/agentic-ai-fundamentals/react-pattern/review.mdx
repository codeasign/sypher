# ReAct Pattern — Review

*Reading time: 3 minutes*

## Key Takeaways

- ReAct interleaves reasoning (Thought) and action (Tool call) in every turn
- The reasoning trace makes the agent's decisions auditable and debuggable
- Use structured tags or explicit "Thought:" prefixes to separate reasoning from action
- Extract and log the reasoning trace separately from the conversation history
- ReAct is ideal for tasks requiring 3-5 steps with mixed reasoning and tool calls

## Recap in 60 Seconds

1. Each turn produces "Thought: what I am thinking" followed by "Action: what I am doing" — this trace is your primary debugging tool
2. Keep the last 2-3 thoughts in context and discard older ones to save window space
3. Validate that actions match stated intentions — diverging thought and action is a warning sign
