## Why This Matters

An agent without safety guards is a liability. In production, agents encounter API errors, network timeouts, malformed tool results, and their own loop bugs. Without recovery logic, these errors cause failed tasks, wasted tokens, and in worst cases, unintended side effects — a stuck agent calling a "send_email" tool in a loop could email hundreds of people.

Production agent infrastructure at companies like Google and Microsoft includes extensive safety systems. Rate limiting prevents runaway costs. Circuit breakers halt the loop when errors exceed a threshold. Dead-man's switches detect when the agent has gone silent. These systems are not optional — they are the difference between a demo and a deployed service.

## The Problem

Your agent is running a batch process overnight. It processes orders by querying the database, updating statuses, and sending confirmation emails. At 2 AM, the database API starts returning intermittent timeouts. The agent retries each call three times, as configured. But each retry adds 30 seconds of timeout delay. By 3 AM, the agent is stuck in a retry loop on a single order. The other 500 orders are never processed.

The problem is not the timeout — it is the lack of a circuit breaker. After N consecutive failures, the agent should recognize the pattern and either pause, use a fallback, or escalate. Instead, it retried indefinitely.

## The Concept

Loop safety consists of three layers:

**Hard limits**. Fixed boundaries that the agent cannot exceed: maximum turns, maximum wall-clock time, maximum token spend, maximum consecutive errors. These are enforced by the orchestrator and halt the agent immediately.

**Recovery strategies**. What the agent does when something goes wrong. Recovery can include retry with backoff, fallback to a simpler tool, skip the step and continue, or escalate to a human.

**Circuit breakers**. Detect patterns of failures. If the error rate exceeds a threshold (e.g., 50% of the last 10 tool calls failed), the circuit breaker trips. Once tripped, the agent stops making new tool calls and enters a recovery mode.
