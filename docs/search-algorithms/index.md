# Search Algorithms

Estimated completion: 12-18 hours - Difficulty: Beginner friendly, no prior algorithms experience needed

## What This Is

A beginner-friendly course covering the search algorithms every engineer should know
cold — from a plain linear scan to `A*` pathfinding — written for total beginners
first. Each module is a single self-contained page written in plain English, with an
everyday analogy, an ASCII diagram walking through a real example step by step, and a
plain-language "How It Works" explanation of how fast the algorithm is. Every module
also has an interactive in-browser code editor supporting nine languages — Python, C,
C++, Java, C#, JavaScript, TypeScript, Rust, and Go — plus a full set of worked
solutions.

New to algorithms? Start with Module 0 — it covers Big-O notation and what "sorted"
means in plain language before the algorithm modules assume you know them.

## How to Use This Course

Module 0 is a beginner primer — no code, just the vocabulary (Big-O notation, what
"sorted" means) that every later module assumes. Skip it if you already know these
terms. Every module after that (1-18) follows the same five-part format: Summary,
Pseudocode, ASCII Diagram, How It Works, and Practice test cases — followed by an
interactive code editor (9 languages) and a Solutions section with a worked answer in
each. Work through modules 1-7 first — each builds on plain array-search fundamentals.
Modules 8-10 shift to graph/tree traversal, ending with weighted-graph traversal plus
heuristics. Modules 11-18 cover pointer-, grid-, and structure-based search patterns
(two pointers, matrices, rotated arrays, graphs, trees, and tries) common in
real-world code and technical interviews.
