## Summary

Exponential Search is a fast way to find something in a sorted list by first guessing a rough neighborhood, then zooming in. It grows a "reach" by doubling it — 1, 2, 4, 8, 16, and so on — until it jumps past where the target could be, then it runs a Binary Search inside just that small stretch to pin down the exact spot. Think of flipping through a big sorted phone book by jumping further and further ahead each time, then narrowing down between your last two jumps once you've overshot the name. It only works on sorted lists, but it's especially quick when the item you want is near the front, and it's also handy when you don't even know how long the list is.

## Pseudocode

```text
1. If the list is empty, report "not found."
2. Check the first item. If it matches the target, report position 0.
3. Set a "reach" counter to 1.
4. While the reach hasn't passed the end of the list, and the item at the reach is still smaller than the target, double the reach.
5. Now the target (if it's in the list) must be somewhere between half the current reach and the current reach (or the end of the list, whichever comes first).
6. Run a Binary Search inside just that narrow window: check the middle item, and keep narrowing to the left half or right half depending on whether the middle is too big or too small.
7. If that search finds a match, report its position. Otherwise, report "not found."
```

## How It Works

- In the diagram, we're hunting for `9` in a sorted row of boxes. We check the first box (`1`) — not a match — so we start doubling the reach.
- Reach 1: `array[1] = 2`, still less than `9`, so double.
- Reach 2: `array[2] = 3`, still less than `9`, so double again.
- Reach 4: `array[4] = 5`, still less than `9`, so double again.
- Reach 8: `array[8] = 9`. That's no longer less than `9`, so the doubling stops right there.
- Because the reach only just passed the target, we know the target has to live between the previous reach (`4`) and the current one (`8`). That narrow five-box window is exactly what gets handed to Binary Search.
- Binary Search checks the middle of that window (`7`), sees it's too small, moves to the right half, checks again, and lands on `8`. Box `8` holds the value `9` — match found.
- This only works if the list is already sorted from smallest to largest — the doubling trick relies on being able to tell "too big" from "too small" at a glance, the same way you can only jump ahead in a phone book because the names are alphabetical.
- If you're lucky and the target is near the very front of the list, this is almost instant — the doubling barely has to grow before it overshoots. If the target is far from the front (or missing entirely), the doubling has to grow a lot more before the narrow Binary Search step even begins. Either way, it's much faster than checking every item one by one on a huge list (this is called `O(log i)` time, where `i` is roughly how far into the list the target sits).
