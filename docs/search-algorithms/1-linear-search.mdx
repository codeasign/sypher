## Summary

Linear Search means checking items one at a time, in order, until you find the one you're looking for — or run out of items. Imagine looking for your friend's name on a class roster by reading down the list line by line: that's Linear Search. It works on any list, in any order, but it can be slow: the more items there are, the more you might have to check.

## Pseudocode

```text
1. Start at the first item in the list.
2. Look at the current item. Is it the one you want?
3. If yes, stop — you found it. Report where it is.
4. If no, move to the next item.
5. If you've checked every item and never found a match, report "not found."
```

## How It Works

- You're looking for the number `8` in the row of boxes above. You start at the first box (`5`) and ask: "is this an `8`?" No, so you move on.
- You check the second box (`3`). Still no. You move on again.
- You check the third box (`8`). Yes! You stop right there and report "found it, at position 2." You never even look at the boxes after it.
- That's the whole idea: check one item, then the next, then the next, until you find a match or you've checked everything.
- There's no trick or shortcut — Linear Search never guesses or skips ahead. That's exactly why it works on a list in *any* order, sorted or not.
- How fast is it? If you're lucky and the item you want is near the front, it's almost instant. If it's near the end — or not in the list at all — you end up checking every single item. (In Big-O terms: best case is `O(1)`, worst case is `O(n)` — the "n" just means "however many items are in the list.")
- It also barely uses any extra memory — just a single counter to remember which box you're looking at. (That's `O(1)` space.)
